## R01

威佐夫博弈（Wythoff's Game）有两堆石子 `(a, b)`（约定 `a <= b`）。每回合允许三类操作之一。

1. 仅从第一堆取走任意正数。
2. 仅从第二堆取走任意正数。
3. 同时从两堆各取走相同的任意正数。

无法行动者失败（普通规则）。威佐夫博弈算法的核心任务是判定一个局面是：

- `P-position`（必败态）：当前玩家在最优对抗下会输。
- `N-position`（必胜态）：当前玩家存在一步进入 `P-position` 的策略。

经典结论：所有 `P-position` 恰好是
`(⌊kφ⌋, ⌊kφ⌋ + k)`，`k >= 0`，其中 `φ = (1+sqrt(5))/2`。

## R02

- 命名来源: 荷兰数学家 Willem A. Wythoff（1907）提出该博弈并给出结构性研究。
- 理论补强: 后续与 Beatty 序列互补定理（1926, Samuel Beatty）建立紧密联系。
- 历史定位: 这是组合博弈论中“数论显式刻画必败态”的经典案例之一。

## R03

朴素解法要对博弈图做递归或动态规划标记，状态空间随堆大小二次增长，且不便解释“为什么这个点必败”。

威佐夫博弈算法的重要性在于：

- 不依赖全图搜索即可直接判定状态性质。
- 把博弈结论压缩为一个数论公式（黄金分割与取整）。
- 可直接构造策略：若是必胜态，可找到一步走向必败态的落子。

## R04

复杂度分两层讨论。

- 理论判定复杂度（把 `⌊kφ⌋` 视作常数时间）:
  - 判定 `(a,b)` 是否必败：`O(1)`。
  - 其中 `k = |b-a|`，只需一次取整比较。
- 本目录 MVP 的策略求解:
  - `legal_moves` 枚举合法后继约 `O(a+b)` 个。
  - `winning_move` 线性扫描这些后继找冷局面，整体 `O(a+b)`。

## R05

- 纯判定算法额外空间: `O(1)`。
- 本目录 MVP 实现:
  - `legal_moves` 返回完整后继列表，额外空间 `O(a+b)`。
  - 若改为“边生成边检查”，可把额外空间降到 `O(1)`。

## R06

示例局面 `(10, 16)`。

- 差值 `k = 16 - 10 = 6`。
- `⌊6φ⌋ = 9`，而当前较小堆是 `10`，所以 `(10,16)` 不是必败态（是必胜态）。
- 一步策略: 同时取走 `4`，得到 `(6,10)`。
- 验证 `(6,10)`:
  - `k = 4`，`⌊4φ⌋ = 6`，符合 `(⌊kφ⌋, ⌊kφ⌋+k)`，因此是必败态。

## R07

该算法的意义体现在三方面。

- 理论意义: 展示了组合博弈与数论（Beatty 序列、黄金分割）的深层耦合。
- 方法意义: 提供“从结构公式直接判定状态”的范式，避免全局状态搜索。
- 工程意义: 在双堆取石类规则中，可快速构造 AI 策略和自动判题逻辑。

## R08

核心数学依赖。

- 黄金分割常数: `φ = (1+sqrt(5))/2`。
- Beatty 互补序列性质:
  - `A_k = ⌊kφ⌋`
  - `B_k = ⌊kφ^2⌋ = A_k + k`
  - 两序列互补覆盖正整数。
- 威佐夫必败态定理:
  - `P = {(A_k, B_k) | k >= 0}`。
- 正常博弈论基本结论:
  - `P` 态的所有后继都不是 `P`；每个 `N` 态至少有一个后继在 `P`。

## R09

适用条件与边界。

- 输入必须是非负整数堆大小。
- 两堆是“无序”的，通常先规范化为 `(min(a,b), max(a,b))`。
- 规则必须与威佐夫博弈一致（单堆取或双堆等量取）。
- 若规则改变（如允许不等量双取、多堆版本、misere 规则），本结论不直接适用。

## R10

正确性要点（简述）。

1. 用 `k = b-a` 固定“对角差值”。
2. 定义 `A_k=⌊kφ⌋, B_k=A_k+k`，得到候选必败态对 `(A_k, B_k)`。
3. 可证明不同 `k` 的候选对互不冲突，且覆盖结构满足 Beatty 互补关系。
4. 从任一 `P` 态出发，任何合法一步都无法到达另一个 `P` 态。
5. 对任一非 `P` 态，总能构造一步使其落到某个 `P` 态。
6. 由博弈论归纳法可得：这些且仅这些态为必败态。

## R11

数值稳定性与误差传播。

- 数学层面: 结论是离散且精确的，不含概率误差。
- 实现层面风险:
  - 若用二进制浮点直接算 `floor(k*phi)`，在超大 `k` 处可能出现临界误判。
  - 临界误判会把 `P/N` 分类翻转，属于高影响错误。
- 本目录 demo 处理:
  - 使用 `decimal.Decimal` 提高精度来计算 `⌊kφ⌋`，降低浮点边界误差风险。

## R12

真实运行成本（与 R04 一致）。

- 对单局面“仅判定 P/N”，成本接近常数级，瓶颈是一次高精度乘法+取整。
- 对“找一步必胜着法”（本 demo）：
  - 先枚举后继状态数量约 `a+b`。
  - 对每个后继做一次 `is_cold_position` 判定。
  - 因此整体是线性级 `O(a+b)`，常数项由高精度取整与 Python 循环决定。
- 若采用闭式构造法，可把找策略进一步压到接近 `O(1)`，但实现更复杂。

## R14

鲁棒性与失效模式。

- 非法输入:
  - 负数或非整数输入应拒绝；否则语义不成立。
- 规则不匹配:
  - 若游戏规则与威佐夫定义不一致，公式失效。
- 数值实现问题:
  - 用低精度浮点可能误判临界 `k`。
- 性能问题:
  - 枚举后继的策略搜索在超大堆规模下变慢；应改为闭式策略构造。

## R18

源码级算法流程拆解（本条目未使用第三方黑箱库；核心逻辑全部在 `demo.py`）。

针对 `is_cold_position + winning_move`，实现可分 8 步。

1. 读取输入 `(a,b)`，`normalize` 规整为 `x<=y` 并校验非负。
2. 在 `is_cold_position` 中计算差值 `k = y-x`。
3. 调用 `floor_mul_phi(k)`，用 `Decimal` 计算 `A_k = ⌊kφ⌋`。
4. 比较 `x == A_k`，成立则 `(x,y)` 为 `P-position`。
5. `winning_move` 先调用 `is_cold_position`，若当前已是 `P` 态则返回 `None`。
6. 否则 `legal_moves` 枚举三类合法一步后继（单堆减、另一堆减、双堆等减）。
7. 依次检查每个后继是否满足 `is_cold_position`。
8. 找到第一个冷后继即返回；若未找到则返回 `None`（理论上热态必能找到）。

该流程与 R17 中函数边界一一对应，也与 R10 的正确性结构保持一致。
