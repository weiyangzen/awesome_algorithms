## R01

问题定义（Jump Game / LeetCode 55）：
- 给定非负整数数组 `nums`，`nums[i]` 表示从位置 `i` 最多可向前跳的步数；
- 初始位于下标 `0`；
- 目标是判断是否可以到达最后一个下标 `n-1`。

本条目 MVP 在输出可达性布尔值之外，还给出一条可达路径（若存在），便于检查算法过程可解释。

## R02

形式化描述：
- 输入：一维非负整数数组 `nums`。
- 约束：若当前在 `i`，可选择任意 `j`，满足 `i < j <= i + nums[i]`。
- 输出：
  - `reachable`：是否可到达 `n-1`；
  - `path_indices`：若可达，给出一条从 `0` 到 `n-1` 的合法路径；
  - `jumps`：路径中每一步跳跃长度。

约定：空数组 `[]` 视为不可达（返回 `False`）。

## R03

动态规划状态设计：
- 令 `reachable[i]` 表示“位置 `i` 是否可从起点到达”；
- 初始状态：`reachable[0] = True`（当 `n>0`）；
- 状态转移：若 `reachable[i] = True`，则区间 `[i+1, min(n-1, i+nums[i])]` 全可标记为可达。

这是典型可达性 DP：由已知可达点向前扩展可达边界。

## R04

路径重建状态：
- 额外维护 `parent[j]`，表示首次把 `j` 标记为可达时的前驱位置；
- 当 `reachable[n-1]` 为真时，从 `n-1` 逆向沿 `parent` 回溯到 `0`；
- 反转后得到正向路径。

该方案把“可达判断”和“解释路径”统一在一套状态中完成。

## R05

正确性直觉：
- `reachable` 的含义是“存在合法跳跃序列到达该点”；
- 初始化时只有起点可达；
- 每一步转移只会把当前可达点一步可跳到的位置标记为可达，因此不会引入非法状态；
- 任何合法路径都可分解为连续边 `i -> j`，这些边都会在扩展过程中被覆盖。

因此，`reachable[n-1]` 与“存在从 `0` 到 `n-1` 的合法跳跃序列”等价。

## R06

复杂度：
- DP+路径重建实现：
  - 时间复杂度最坏 `O(n^2)`（如每个位置都能覆盖后续大量位置）；
  - 空间复杂度 `O(n)`（`reachable` 与 `parent`）。
- 贪心实现：
  - 时间复杂度 `O(n)`；
  - 空间复杂度 `O(1)`。
- 暴力 DFS（仅用于小规模校验）：指数级分支，借助记忆化后通常明显缩减，但不用于大规模主计算。

## R07

`demo.py` 中的三条求解路径：
- `jump_game_dp_trace`：主实现，返回可达性与一条路径；
- `jump_game_greedy`：线性贪心，只返回可达性，用于对照；
- `jump_game_bruteforce`：小规模精确搜索，用于回归测试交叉验证。

脚本会自动断言三者在可达性上保持一致。

## R08

输入校验策略（`to_1d_nonnegative_int_array`）：
- 输入必须为一维序列；
- 必须是有限数值（拒绝 `nan/inf`）；
- 只能是非负整数值（浮点若非整数会报错）；
- 统一转为 `numpy.int64` 便于后续计算稳定。

将数据清洗逻辑集中在单函数，避免重复与漂移。

## R09

边界情形：
- `[]`：不可达；
- `[0]`：已在终点，可达；
- `[k, 0, ..., 0]`：取决于首步是否能覆盖终点；
- 存在“陷阱 0”（如 `[3,2,1,0,4]`）时应判定不可达；
- 多路径可达时，DP 会输出“首次记录前驱”得到的一条稳定路径。

## R10

验证机制：
- 固定案例：覆盖可达、不可达、空数组、单元素、含陷阱等场景；
- 随机回归：随机生成小数组，对比 `DP`、`贪心`、`暴力` 的可达性是否一致；
- 路径校验：若 `reachable=True`，检查路径起终点、单调性与每步跳长合法性。

## R18

`demo.py` 源码级算法流程（8 步）：
1. `main` 构造固定测试用例并逐个调用 `run_case`。  
2. `run_case` 首先调用 `jump_game_dp_trace`：用 `reachable` 数组从左到右扩展可达区间。  
3. `jump_game_dp_trace` 在标记新可达点时同步记录 `parent`，用于后续路径重建。  
4. 若终点可达，函数从 `n-1` 反向追溯 `parent` 直到 `0`，反转得到正向路径，并计算 `jumps`。  
5. `run_case` 再调用 `jump_game_greedy`，用“最远可达下标 `farthest`”在 `O(n)` 时间内复算可达性。  
6. 对小规模输入，`run_case` 额外调用 `jump_game_bruteforce`（记忆化 DFS）获得精确可达结论。  
7. `is_valid_path` 检查路径是否从 `0` 到 `n-1`、严格递增且每步不超过当前位置可跳上限。  
8. 固定案例全部通过后，`randomized_regression` 执行 300 轮随机交叉验证，确保三种实现长期一致。
