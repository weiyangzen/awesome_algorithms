# 连分数算法

## R01

连分数算法把一个实数表示为
`x = [a0; a1, a2, ...] = a0 + 1/(a1 + 1/(a2 + ...))`。

在数论里常见三类任务：

- 有理数展开：输入 `p/q`，输出有限系数序列 `[a0, ..., an]`
- 收敛分数构造：输入系数，输出一串有理逼近 `p_k/q_k`
- 二次无理数周期展开：输入非平方整数 `D`，输出 `sqrt(D) = [a0; overline(period)]`

## R02

历史脉络与欧几里得算法紧密相关：

- 古典时期：辗转相除法已经隐含“商序列”结构
- 17 世纪：连分数被系统用于有理逼近
- 18 世纪：欧拉与拉格朗日完善理论
- 拉格朗日定理：二次无理数的连分数展开当且仅当最终周期

因此它既是“算法工具”，也是“数论结构定理”的计算化表达。

## R03

提出动机主要有三点：

- 用离散步骤表示连续实数，便于计算机处理
- 在所有分母不超过给定上界的分数中，收敛分数通常给出最优或近最优逼近
- 服务 Pell 方程、模结构分析、丢番图逼近等问题

相比“盲目枚举分子分母”，连分数方法利用了欧几里得结构，搜索路径更短、信息密度更高。

## R04

时间复杂度（按任务分开）：

- 有理数 `p/q` 展开：
  - 迭代次数与欧几里得算法一致，为 `O(log min(|p|, |q|))`
- 从系数构造前 `k` 个收敛分数：
  - `O(k)`
- `sqrt(D)` 周期部分求解：
  - `O(L)`，`L` 为周期长度（每轮常数个整数运算）

若考虑大整数位复杂度，单步除法/乘法成本会随位宽增长，但总步数仍受上述界控制。

## R05

空间复杂度：

- 仅求有理数系数并流式输出：额外 `O(1)`
- 若保存全部系数：`O(k)`（`k` 为系数个数）
- 保存全部收敛分数：`O(k)`
- `sqrt(D)` 周期求解：`O(L)`（存储周期）

## R06

示例 1：`415/93`

1. `415 = 4*93 + 43`
2. `93 = 2*43 + 7`
3. `43 = 6*7 + 1`
4. `7 = 7*1 + 0`

所以 `415/93 = [4; 2, 6, 7]`。
对应收敛分数：`4/1, 9/2, 58/13, 415/93`。

示例 2：`sqrt(23)`

- `sqrt(23) = [4; overline(1, 3, 1, 8)]`
- 周期长度为 `4`

## R07

算法意义：

- 理论：连通欧几里得算法、二次无理数、丢番图逼近
- 工程：在低成本下提供高质量有理逼近
- 应用：Pell 方程、数值近似、符号计算中的分式重建

它是“离散整数计算”和“实数逼近”之间的经典桥梁。

## R08

核心前置知识：

- 整数除法与余数
- 欧几里得算法
- 分数与互素概念（`gcd(p, q)=1`）
- 基本递推关系（收敛分数的 `p_k, q_k` 递推）

本条目 MVP 不依赖外部数值库，仅使用 Python 标准库。

## R09

适用前提与边界：

- 有理数输入要求分母非零
- `sqrt(D)` 周期算法要求 `D > 0` 且 `D` 非完全平方
- 对非常大的整数，理论正确性不变，但运行时间受大整数算术成本影响
- 浮点数直接展开会受舍入影响，工程上更推荐“有理数/整数驱动”的精确流程

## R10

正确性要点：

1. 有理数展开正确性：
   - 每轮做 `x = a*y + r`，等价于 `x/y = a + r/y`
   - 下一轮在 `y/r` 上重复，直到余数为 `0`
   - 这正是有限连分数定义
2. 收敛分数递推正确性：
   - `p_k = a_k p_{k-1} + p_{k-2}`，`q_k = a_k q_{k-1} + q_{k-2}`
   - 与连分数截断值逐层代回完全一致
3. `sqrt(D)` 周期正确性：
   - 状态 `(m, d, a)` 按整数递推更新
   - 对固定非平方 `D`，可达状态有限，必进入循环
   - 对标准 `sqrt(D)` 递推，遇到 `a = 2*a0` 即完成一个周期

## R11

数值稳定性：

- 有理数展开与收敛分数递推在整数域完成，理论上无浮点误差传播
- 风险主要来自“把近似浮点当成精确值”
- 因此实现应优先使用整数与 `Fraction` 等精确表示

本目录 `demo.py` 仅在“误差展示”环节把结果转为 `float`，不影响主算法正确性。

## R12

真实成本（超越渐近阶）：

- 主成本是整数除法与乘法，而非循环本身
- 系数值偶尔会很大（取决于输入结构），会影响常数开销
- 若批量处理多组输入，可复用内存对象或按需流式输出，减少峰值内存
- Python 中 `Fraction` 可读性高但对象开销不小；性能敏感场景可改为手写 `(p, q)` 整数对

## R13

本算法具备明确逼近误差结论（非 `N/A`）：

- 对无理数 `x` 的第 `k` 个收敛分数 `p_k/q_k`，有经典界：
  `|x - p_k/q_k| < 1/(a_{k+1} q_k^2) <= 1/q_k^2`
- 同时满足“最佳逼近”性质之一：
  对某些范围内分母更小的分数，很难达到同等误差级别

这也是连分数在高质量有理逼近中长期占优的原因。

## R14

鲁棒性与失效模式：

- 失效模式 1：分母为 `0`（输入非法）
- 失效模式 2：把完全平方数送入周期算法（不满足问题定义）
- 失效模式 3：用浮点反推系数导致错误截断
- 失效模式 4：仅用近似值做等式判断（如 Pell 关系）

缓解手段：

- 强制输入校验
- 核心流程坚持整数运算
- 断言关键恒等式（例如重建分数、Pell 残差）

## R15

工程实现要点：

- 有理数展开直接复用辗转相除法循环，代码短且可验证
- 收敛分数用两步历史状态 `(p_{k-1}, p_{k-2})`、`(q_{k-1}, q_{k-2})` 递推
- `sqrt(D)` 周期部分使用标准状态更新：
  - `m' = d*a - m`
  - `d' = (D - m'^2)/d`
  - `a' = floor((a0 + m')/d')`
- 对演示结果加入 `assert`，保证“可运行 + 可自检”

## R16

前驱/后继关系与应用：

- 前驱：欧几里得算法、最大公约数计算
- 后继：
  - Pell 方程求解器
  - 丢番图逼近与 Farey/Stern-Brocot 相关结构
  - 连分数在密码分析与计算数论中的近似模块
- 典型应用：
  - 构造高质量分式近似
  - 判断与利用二次无理数周期结构
  - 作为更复杂数论算法的子程序

## R18

源码级流程拆解（本实现未调用第三方黑箱）：

1. **输入归一化**：检查分母非零、分母符号归一。
2. **欧几里得循环取商**：反复执行 `a = x//y` 与 `(x, y) <- (y, x-a*y)`，得到有限系数序列。
3. **收敛分数递推**：用两阶递推更新 `(p_k, q_k)`，逐项构造 `C_k = p_k/q_k`。
4. **精确重建校验**：把最后一个收敛分数与原始 `p/q` 比较，确保展开正确。
5. **二次无理数状态更新**：对 `sqrt(D)` 迭代 `(m, d, a)` 三元组。
6. **周期终止判定**：当 `a == 2*a0` 时结束，得到一个完整周期。
7. **周期截断逼近**：把周期项按需重复，生成前 `t` 项截断收敛分数。
8. **误差与恒等式验证**：输出 `|sqrt(2)-p/q|` 与 `|p^2-2q^2|`，并断言 `|误差| < 1/q^2`。

这 8 步对应 `demo.py` 中的实际函数调用路径，形成可直接运行、可审计的最小闭环实现。
