## R01

LL(1) 语法分析是一种自顶向下的预测分析方法：
- 第一个 `L` 表示从左到右读入输入串；
- 第二个 `L` 表示产生式按最左推导展开；
- `1` 表示每次决策只看 1 个向前看符号（lookahead）。

本条目实现一个最小可运行 MVP，完整覆盖：`FIRST/FOLLOW` 计算、分析表构造、冲突检测、以及栈驱动的预测分析过程追踪。

## R02

问题定义（本实现的输入输出）：
- 输入 1：一个上下文无关文法（CFG），用 Python 字典表示；
- 输入 2：一组已分词 token 序列（示例中直接给定，不做交互输入）；
- 输出：
  - 各非终结符的 `FIRST` 集与 `FOLLOW` 集；
  - LL(1) 预测分析表 `M[A, a]`；
  - 每个测试样例的逐步分析轨迹（stack / input / action）；
  - 接受/拒绝结论与统计信息。

## R03

`FIRST` 集规则（设 `X` 是文法符号，`α` 是符号串）：
1. 若 `X` 是终结符，则 `FIRST(X) = {X}`；
2. 若有产生式 `A -> ε`，则 `ε ∈ FIRST(A)`；
3. 若 `A -> X1 X2 ... Xk`，把 `FIRST(X1)` 去掉 `ε` 加入 `FIRST(A)`；
4. 若 `ε ∈ FIRST(X1)`，继续看 `X2`，依次传播；
5. 若 `X1..Xk` 均可推出 `ε`，则 `ε ∈ FIRST(A)`。

`demo.py` 通过不动点迭代实现以上规则，直到所有 `FIRST(A)` 不再变化。

## R04

`FOLLOW` 集规则（`A` 为非终结符）：
1. 开始符号 `S` 满足 `$ ∈ FOLLOW(S)`；
2. 若有 `B -> α A β`，则 `FIRST(β) - {ε} ⊆ FOLLOW(A)`；
3. 若有 `B -> α A` 或 `ε ∈ FIRST(β)`，则 `FOLLOW(B) ⊆ FOLLOW(A)`。

实现中同样使用迭代收敛。`compute_follow_sets` 每轮遍历全部产生式，直到集合稳定。

## R05

预测分析表构造规则：
- 对每个产生式 `A -> α`，对所有 `a ∈ FIRST(α) - {ε}`，填 `M[A, a] = A -> α`；
- 若 `ε ∈ FIRST(α)`，则对所有 `b ∈ FOLLOW(A)`，填 `M[A, b] = A -> α`。

若某个格子被不同产生式重复填充，说明存在冲突，文法不是 LL(1)。`build_ll1_table` 会记录冲突并在主流程中抛错。

## R06

栈式预测分析算法（输入末尾追加 `$`）：
1. 初始化栈为 `[$, S]`；
2. 设栈顶为 `X`，当前输入符号为 `a`；
3. 若 `X` 是终结符且 `X == a`，执行匹配并前进；
4. 若 `X` 是非终结符，查表 `M[X, a]`：
   - 有条目：按产生式右部逆序压栈（`ε` 不入栈）；
   - 无条目：报错拒绝；
5. 当栈顶与输入同时为 `$` 时接受。

## R07

正确性要点（MVP 层面）：
- 分析表仅由 `FIRST/FOLLOW` 规则生成，遵循标准 LL(1) 构造法；
- 解析过程中每一步都记录在 `ParseStep`，避免“黑盒接受”；
- 提供正例与反例断言：
  - 正例必须 `accepted=True`；
  - 反例必须 `accepted=False`；
- 若断言失败，脚本直接异常退出，避免静默错误。

## R08

复杂度分析（记）：
- 非终结符数 `|N|`，终结符数 `|T|`，产生式总长度 `P`，输入长度 `n`。

主要开销：
- `FIRST/FOLLOW` 迭代：教学规模下近似 `O(P * I)`，`I` 为收敛轮次；
- 构造分析表：`O(P + |N|*|T|)`（填表 + 输出）；
- 单次语法分析：`O(n)`（每步匹配或展开都会推进栈/输入）。

## R09

MVP 取舍：
- 使用手写 LL(1) 算法，不依赖 parser generator 黑盒；
- 不引入大型编译框架，最小技术栈为 `Python + numpy`；
- `numpy` 仅用于输出统计（表填充率、分析步数统计），核心算法均在源码中逐步实现；
- 不做词法分析器，本示例直接给 token 序列，聚焦语法分析核心。

## R10

- `build_expression_grammar`：构造示例表达式文法；

## R12

示例文法（经典算术表达式，已消除左递归，满足 LL(1)）：

- `E  -> T E'`
- `E' -> + T E' | ε`
- `T  -> F T'`
- `T' -> * F T' | ε`
- `F  -> ( E ) | id`

该文法体现了常见优先级与结合性：`*` 高于 `+`，括号可改变结合结构。

## R13

示例测试用例：
- `valid_1`: `id + id * id`（应接受）；
- `valid_2`: `( id + id ) * id`（应接受）；
- `invalid_1`: `id + * id`（应拒绝）。

输出重点包括：
- FIRST/FOLLOW 集；
- 预测分析表文本；
- 每个样例的 step-by-step 动作（expand/match/error/accept）；
- 最终 `All LL(1) checks passed.`。

## R14

常见实现误区与本实现的处理：
1. 误把 `ε` 当普通终结符压栈。
   - 处理：`production == (ε,)` 时不压栈。
2. `FOLLOW` 传播漏掉“右侧可空”场景。
   - 处理：`if not beta or ε in FIRST(beta)` 时并入 `FOLLOW(lhs)`。
3. 冲突检测缺失，错误地把非 LL(1) 文法当可用。
   - 处理：填表时若同格不同产生式即记录冲突并终止。
4. 解析失败信息不透明。
   - 处理：失败时返回期望 lookahead 集合与完整步骤轨迹。

## R15

与其他语法分析方法对比：
- 相比递归下降：LL(1) 把决策前置到表，行为更显式、便于调试；
- 相比 LR 系列：LL(1) 实现更简单，但可处理文法范围更小；
- 相比 PEG/回溯：LL(1) 无回溯，性能稳定，错误定位更直接。

工程上常把 LL(1) 用于教学语言、配置语言或 DSL 的快速原型。

## R18

`demo.py` 的源码级算法流程（9 步）：
1. `run_demo` 调用 `build_expression_grammar` 生成示例 CFG 文法。  
2. `compute_first_sets` 迭代计算每个非终结符的 FIRST 集直到收敛。  
3. `compute_follow_sets` 以 `$` 初始化开始符号 FOLLOW，继续迭代求不动点。  
4. `build_ll1_table` 按 `FIRST/FOLLOW` 规则填表，并在同格多产生式时记录冲突。  
5. 若冲突非空，主流程立即报错退出，防止错误文法继续进入分析阶段。  
6. 对每组 token，`parse_tokens` 初始化栈 `[$, S]` 并循环执行“匹配终结符 / 非终结符查表展开”。  
7. 每一步都写入 `ParseStep(step_id, stack, remaining_input, action)`，得到完整可追踪解析轨迹。  
8. 主流程对正反样例执行断言，确保接受/拒绝行为符合预期。  
9. 使用 `numpy` 计算分析表填充率与步数统计并打印总结，最后输出通过信息。

说明：本实现没有把任务交给第三方解析器库的一两行黑盒 API，而是把 FIRST、FOLLOW、填表与栈驱动分析拆成可读源码步骤。
