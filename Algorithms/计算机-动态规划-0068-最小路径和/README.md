# 最小路径和

- UID: `CS-0051`
- 学科: `计算机`
- 分类: `动态规划`
- 源序号: `68`
- 目标目录: `Algorithms/计算机-动态规划-0068-最小路径和`

## R01

问题定义（Minimum Path Sum）：
- 给定一个 `m x n` 网格 `grid`；
- 起点为左上角 `(0,0)`，终点为右下角 `(m-1,n-1)`；
- 每一步只能向右或向下移动一格；
- 路径代价为经过格子数值之和，目标是最小化总代价。

本条目不仅计算最小路径和，还重建一条对应最优路径（坐标与路径值）。

## R02

MVP 输入输出约定：
- 输入：`grid: Sequence[Sequence[float]]`。
- 约束：
  - `grid` 必须是二维且非空；
  - 行列长度均大于 0；
  - 全部元素必须是有限数（拒绝 `NaN/Inf`）。
- 输出：`GridMinPathResult`
  - `min_sum`：最小路径和；
  - `path_cells`：路径坐标列表；
  - `path_values`：路径经过值列表。

## R03

状态定义（自底向上 DP）：
- `dp[i][j]`：从起点 `(0,0)` 到格子 `(i,j)` 的最小路径和；
- `parent_r[i][j], parent_c[i][j]`：到达 `(i,j)` 时最优前驱坐标，用于路径重建。

初始条件：
- `dp[0][0] = grid[0][0]`；
- 第一行只能从左侧到达；
- 第一列只能从上方到达。

## R04

状态转移方程（`i>0, j>0`）：

`dp[i][j] = grid[i][j] + min(dp[i-1][j], dp[i][j-1])`

并同步记录前驱：
- 若 `dp[i-1][j] <= dp[i][j-1]`，前驱取上方 `(i-1, j)`；
- 否则前驱取左方 `(i, j-1)`。

终值为 `dp[m-1][n-1]`。

## R05

最优子结构说明：
- 任意到达 `(i,j)` 的最优路径，其最后一步只能来自：
  - 上方 `(i-1,j)`；
  - 左方 `(i,j-1)`。
- 若最后一步来自某前驱，但该前驱并非其自身最优路径，则可替换为更优前缀，得到更小总和，矛盾。

因此局部最优前驱可递推得到全局最优路径和。

## R06

路径重建方法：
- 从终点 `(m-1,n-1)` 反向追踪 `parent` 指针直到 `(0,0)`；
- 将反向坐标序列翻转得到正向路径；
- 同步提取 `path_values`。

重建后应满足：
- `path_cells[0] == (0,0)`；
- `path_cells[-1] == (m-1,n-1)`；
- 每步坐标差仅为 `(1,0)` 或 `(0,1)`。

## R07

复杂度分析：
- 主算法（自底向上 DP）：
  - 时间复杂度 `O(mn)`；
  - 空间复杂度 `O(mn)`（`dp + parent`）。
- 记忆化递归基线：
  - 时间复杂度 `O(mn)`；
  - 空间复杂度 `O(mn)`（缓存）+ 递归栈深度 `O(m+n)`。
- 暴力 DFS 基线：
  - 时间复杂度指数级，约 `O(C(m+n-2, m-1))`，仅用于小规模对拍。

## R08

`demo.py` 结构：
- `to_grid`：校验并标准化输入网格；
- `min_path_sum_bottom_up`：主算法（最小和 + 路径重建）；
- `min_path_sum_top_down`：记忆化递归基线；
- `min_path_sum_bruteforce`：暴力 DFS 基线；
- `is_valid_grid_path`：校验路径是否满足“右/下移动”规则；
- `run_case`：执行固定样例并断言；
- `randomized_cross_check`：随机回归测试；
- `main`：组织样例并执行全部验证。

## R09

核心接口语义：
- `min_path_sum_bottom_up(grid) -> GridMinPathResult`
  - 返回最小路径和及一条可解释路径。
- `min_path_sum_top_down(grid) -> float`
  - 独立基线，仅返回最小路径和。
- `min_path_sum_bruteforce(grid) -> float`
  - 小规模精确基线，避免主算法“自证正确”。

## R12

输出解读：

- `top-down`：记忆化递归结果；

三者数值应一致；若不一致程序会抛出 `AssertionError`。

## R13

随机对拍策略：
- 默认 `250` 轮；
- `rows ∈ [1, 6]`，`cols ∈ [1, 6]`；
- 元素值随机取自 `[-8, 15]`；
- 每轮断言：
  - 主算法 `min_sum` == 记忆化结果；
  - 主算法 `min_sum` == 暴力 DFS 结果；
  - 重建路径合法且路径值求和等于 `min_sum`。

## R14

边界与异常处理：
- 输入不是规则二维网格或为空：`ValueError`；
- 存在 `NaN/Inf`：`ValueError`；
- 重建得到路径与网格移动规则不一致：`AssertionError`；
- 若 `parent` 链断裂（理论上不应发生）：`RuntimeError`。

## R15

为什么该实现是“最小但诚实”的 MVP：
- 仅依赖 `numpy + 标准库`；
- 没有调用第三方黑盒“最短路径 API”；
- 主算法与两种独立基线（记忆化、暴力）交叉验证；
- 代码规模小，便于手工审计与扩展。

## R18

`demo.py` 源码级算法流程（8 步）：
1. `main` 定义四个固定样例，逐个调用 `run_case`，最后执行 `randomized_cross_check`。  
2. `run_case` 首先调用 `min_path_sum_bottom_up`，`to_grid` 会检查输入是有限数二维非空数组。  
3. 主算法初始化 `dp` 与 `parent`：设置 `dp[0][0]`，再填第一行和第一列（各自只有单一来源）。  
4. 对每个内部格子 `(i,j)`，比较 `dp[i-1][j]` 与 `dp[i][j-1]`，选较小前驱并计算 `dp[i][j] = grid[i][j] + best_prev`。  
5. `dp` 填完后，从终点按 `parent` 反向回溯到起点，翻转得到正向 `path_cells`，并提取 `path_values`。  
6. `run_case` 再调用 `min_path_sum_top_down`（`lru_cache`）独立计算最小路径和。  
7. `run_case` 同时调用 `min_path_sum_bruteforce` 作为小规模精确基线，并校验路径合法性与路径值求和一致性。  
8. `randomized_cross_check` 对 250 组随机网格重复上述一致性断言，确认实现稳定可靠。
