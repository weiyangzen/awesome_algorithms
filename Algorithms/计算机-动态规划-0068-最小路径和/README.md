# 最小路径和

- UID: `CS-0051`
- 学科: `计算机`
- 分类: `动态规划`
- 源序号: `68`
- 目标目录: `Algorithms/计算机-动态规划-0068-最小路径和`

## R01

问题定义（Minimum Path Sum）：
- 给定一个 `m x n` 的网格 `grid`；
- 从左上角 `(0,0)` 出发，到右下角 `(m-1,n-1)` 结束；
- 每一步只能向右或向下移动一格；
- 路径代价为经过格子数值之和；
- 目标是求最小路径和。

这是二维动态规划的入门经典题，核心在于“当前位置最优值只依赖上方和左方两个子问题”。

## R02

MVP 输入输出约定：
- 输入：`grid: Sequence[Sequence[float]]`；
- 约束：
  - 网格不能为空；
  - 每行长度必须一致（拒绝 ragged array）；
  - 所有元素必须是有限数（拒绝 `NaN/Inf`）。
- 输出（主函数）：
  - `min_sum`：最小路径和；
  - `path_indices`：一条最优路径坐标；
  - `path_values`：路径上的对应值。

## R03

状态定义：
- `dp[i][j]`：从起点 `(0,0)` 到单元格 `(i,j)` 的最小路径和。

边界初始化：
- `dp[0][0] = grid[0][0]`；
- 第一行只能从左侧到达：`dp[0][j] = dp[0][j-1] + grid[0][j]`；
- 第一列只能从上侧到达：`dp[i][0] = dp[i-1][0] + grid[i][0]`。

## R04

状态转移（`i>0` 且 `j>0`）：

`dp[i][j] = grid[i][j] + min(dp[i-1][j], dp[i][j-1])`

同时记录前驱方向（来自上方或左方），用于后续路径重建：
- 若 `dp[i-1][j] <= dp[i][j-1]`，前驱记为 `(i-1,j)`；
- 否则前驱记为 `(i,j-1)`。

## R05

最优子结构说明：
- 到达 `(i,j)` 的合法路径最后一步只可能来自 `(i-1,j)` 或 `(i,j-1)`；
- 若某条全局最优路径在其前驱位置没有采用对应子问题的最优解，则可替换成更优子路径使总和更小，产生矛盾。

因此局部最优转移可以递推出全局最优。

## R06

路径重建方法：
1. 在计算 `dp` 时维护 `parent[i][j]`；
2. 从终点 `(m-1,n-1)` 反向按 `parent` 回溯至 `(0,0)`；
3. 反转得到正向路径；
4. 按路径坐标提取 `path_values`，并可验证其和等于 `min_sum`。

该方法让结果可解释，不只是输出一个标量。

## R07

复杂度分析：
- 时间复杂度：`O(mn)`，每个格子只处理一次；
- 空间复杂度：
  - 当前实现（含路径重建）为 `O(mn)`（`dp + parent`）；
  - 若只求最小和可降为 `O(n)`（滚动数组）。

## R08

`demo.py` 的函数结构：
- `to_matrix`：输入校验与标准化；
- `min_path_sum_dp`：主算法（二维 DP + 路径重建）；
- `min_path_sum_top_down`：记忆化递归基线；
- `min_path_sum_bruteforce`：小规模暴力 DFS 基线；
- `is_valid_grid_path`：检查路径是否合法（仅向右/向下）；
- `run_case`：执行单例并断言一致性；
- `randomized_cross_check`：随机网格对拍；
- `main`：组织固定样例与随机验证。

## R09

核心接口语义：
- `min_path_sum_dp(grid) -> GridMinPathResult`
  - 返回最小和与一条可重建路径。
- `min_path_sum_top_down(grid) -> float`
  - 返回最小和，用于独立交叉验证。
- `min_path_sum_bruteforce(grid) -> float`
  - 指数复杂度，仅用于小网格真值校验。

## R10

固定样例覆盖：
1. 经典样例 `[[1,3,1],[1,5,1],[4,2,1]]`，最小和应为 `7`；
2. 含负数样例，验证算法不依赖“非负权”假设；
3. 单格网格 `[[42]]`，验证最小边界输入。

每个样例都会检查：
- 主算法与基线一致；
- 路径结构合法；
- 路径值求和等于 `min_sum`。

## R12

输出字段解读：
- `bottom-up`：主算法输出，包含最小和与路径；
- `top-down`：记忆化递归输出；
- `bruteforce`：小规模时的暴力输出；
- 三者最小和应一致，否则抛出 `AssertionError`。

## R13

随机对拍策略：
- 默认 `300` 轮；
- 网格尺寸随机采样在 `1..6`；
- 元素值采样在 `[-8, 15]`；
- 每轮验证：
  - 主算法最小和 == 记忆化递归最小和；
  - 主算法最小和 == 暴力 DFS 最小和；
  - 主算法重建路径合法且和正确。

## R14

异常与边界处理：
- 空网格：`ValueError`；
- 行长度不一致：`ValueError`；
- 非有限数：`ValueError`；
- 路径重建后不满足“右/下相邻”：断言失败。

这些检查帮助在数据不合规时尽早定位问题。

## R15

为什么该实现是“最小但诚实”的 MVP：
- 仅依赖 `numpy + 标准库`；
- 没有调用第三方黑盒路径优化器；
- 使用主算法 + 两类基线（记忆化/暴力）做结果互证；
- 代码结构紧凑，且包含必要的输入校验与路径可解释输出。

## R18

`demo.py` 源码级算法流程（8 步）：
1. `main` 构造固定样例并调用 `run_case`，随后执行 `randomized_cross_check`。  
2. `run_case` 首先调用 `min_path_sum_dp`，其中 `to_matrix` 会完成空输入、矩阵形状与有限数校验。  
3. `min_path_sum_dp` 初始化 `dp` 与 `parent`，先填第一行和第一列的前缀最优和及其前驱。  
4. 对内部格子 `(i,j)` 执行状态转移：比较 `dp[i-1,j]` 与 `dp[i,j-1]`，取较小者并记录对应前驱到 `parent[i][j]`。  
5. 填表结束后，从终点 `(m-1,n-1)` 沿 `parent` 反向回溯到 `(0,0)`，再反转得到正向路径坐标。  
6. 根据路径坐标提取 `path_values`，并返回 `GridMinPathResult(min_sum, path_indices, path_values)`。  
7. `run_case` 再调用 `min_path_sum_top_down`（`lru_cache` 记忆化）与 `min_path_sum_bruteforce`（无记忆化 DFS）计算独立基线，逐项断言一致。  
8. `randomized_cross_check` 在随机网格上重复上述一致性校验，确认主实现稳定可靠。
