# 分岔理论 (Bifurcation Theory)

- UID: `PHYS-0119`
- 学科: `物理`
- 分类: `非线性动力学`
- 源序号: `119`
- 目标目录: `Algorithms/物理-非线性动力学-0119-分岔理论_(Bifurcation_Theory)`

## R01

分岔理论研究的是：当控制参数 `mu` 连续变化时，动力系统定性结构（平衡点数量、稳定性、周期轨道）在临界值附近发生突变。它关注的不是“解数值上变大一点”，而是“系统相图拓扑发生了类别变化”。

## R02

在物理和工程里，分岔对应“机制切换点”：

- 激光阈值、化学反应自振、流体失稳；
- 控制系统从单稳态变多稳态；
- 离散映射从稳态到倍周期再到混沌。

因此分岔分析是把“参数调节”变成“相态切换诊断”的核心工具。

## R03

本条目 MVP 的目标是最小闭环地完成三件事：

- 对两个经典连续系统标准型（鞍结、超临界叉形）做参数扫描；
- 自动判定平衡点稳定性并估计分岔点 `mu*`；
- 用 logistic 映射检测首个倍周期分岔区间（`r≈3`）。

即：`定义模型 -> 扫描参数 -> 识别分岔 -> 用轨道仿真验证`。

## R04

本实现使用的核心模型与判据：

1. 鞍结标准型：`x' = mu - x^2`。
2. 超临界叉形标准型：`x' = mu*x - x^3`。
3. 连续系统平衡点：`f(x, mu)=0`。
4. 局部稳定性（1D）：看 `df/dx` 符号；`df/dx<0` 稳定，`df/dx>0` 不稳定。
5. logistic 映射：`x_{n+1} = r*x_n*(1-x_n)`。
6. 离散轨道“周期代理”：统计尾部轨道的近似唯一值个数，识别从 1 到 2 的第一次跳变。

## R05

设参数网格大小为 `M`，每个参数点平均平衡点数为 `K`，logistic 每个 `r` 迭代 `T` 步：

- 连续模型扫描：`O(M*K)`；
- 稳定性判定：`O(M*K)`；
- logistic 周期代理：`O(M*T)`；
- 总体复杂度约 `O(M*(K+T))`，空间复杂度 `O(M*K)`（保存表格结果）。

MVP 中 `K` 很小（1~3），主要成本在 logistic 迭代。

## R06

`demo.py` 包含三类演示与断言：

- **Demo A（鞍结）**：扫描 `mu`，检测平衡点数量从 0 到 2 的跳变，估计 `mu*=0`。
- **Demo B（叉形）**：扫描 `mu`，检测平衡点数量从 1 到 3 的跳变，并检查 `x=0` 稳定性翻转。
- **Demo C（logistic）**：扫描 `r in [2.8, 3.2]`，检测首个 period-1 -> period-2 的过渡位置。

脚本运行后打印估计分岔点、轨道末态和局部表格，同时用 `assert` 做自动校验。

## R07

优点：

- 选用标准型，分岔机制清晰、可解释性强；
- 不依赖黑盒动力系统框架，算法步骤都在源码中展开；
- 连续系统与离散映射都覆盖，适合作为非线性动力学入门基线。

局限：

- 只处理低维标准型，未覆盖高维中心流形约化；
- logistic 周期识别是“近似唯一值计数”，在混沌边界会受阈值影响；
- 未绘制分岔图，仅做数值与表格验证。

## R10

正确性直觉：

1. 鞍结中，`mu<0` 无实平衡点，`mu>0` 出现一稳一不稳，数量应突变。
2. 叉形中，`mu<0` 仅 `x=0` 且稳定；`mu>0` 出现对称双稳态，`x=0` 变不稳。
3. logistic 在 `r≈3` 附近会从单点吸引子过渡到 2 周期。
4. 轨道仿真末态应收敛到对应稳定平衡支。

## R11

数值稳定与精度策略：

- 参数网格包含 `0`，避免临界点错位；
- 稳定性判定设置 `tol`，避免浮点零附近抖动；
- Euler 步长取 `dt=0.01`，并用足够长步数观察收敛；
- logistic 使用“先丢弃暂态，再统计尾部”的流程降低初值污染。

## R12

关键参数：

- `mu_values`: 连续系统参数扫描网格；
- `dt, steps`: ODE 轨道验证的离散精度与时长；
- `r_values`: logistic 参数扫描区间与分辨率；
- `transient, keep, tol`: period 代理统计的稳健性参数。

调参建议：

- 若分岔点估计偏粗，先加密 `mu_values` / `r_values`；
- 若轨道未收敛，增大 `steps` 或减小 `dt`；
- 若 period 误判，适当增大 `transient` 并收紧 `tol`。

## R14

常见失效模式：

1. 参数网格太稀导致临界点跳过；
2. 对 marginal 点（导数近零）误判稳定性；
3. logistic 的阈值聚类过松或过紧，导致 period 代理失真；
4. 用过短轨道判断稳态，混入暂态造成错误结论。

## R18

`demo.py` 源码级算法流程（8 步）：

1. 用 `ContinuousModel` 封装每个连续标准型的 `f(x,mu)`、平衡点枚举函数和导数 `df/dx`。  
2. 在 `sweep_equilibria` 中遍历 `mu` 网格，逐点生成平衡点列表并计算局部斜率。  
3. 用 `stability_label` 按斜率符号把每个分支标记为 `stable/unstable/marginal`。  
4. 在 `detect_count_change_bifurcation` 中统计每个 `mu` 的平衡点数量，找最大跳变位置作为 `mu*` 估计。  
5. 用 `simulate_1d_trajectory`（显式 Euler）在代表参数下推进轨道，检验是否收敛到理论稳定平衡。  
6. 对 logistic 映射逐 `r` 迭代：先去暂态，再保留尾部样本。  
7. 在 `orbit_period_proxy` 里对尾部点排序并按容差计数“近似唯一值”，把该计数作为周期代理，再在 `detect_logistic_first_period_doubling` 中找首次 `1->2` 跳变。  
8. `main` 汇总打印分岔估计与快照表，并通过断言验证连续/离散两类分岔都满足预期区间。  

第三方库没有承担黑盒分岔判断：`numpy/pandas` 仅用于数组与表格，分岔检测、稳定性判据和周期代理逻辑都在源码中显式实现。
