## R01

点在多边形内判定（Point-in-Polygon, PIP）要回答的问题是：给定二维简单多边形 `P` 与查询点 `q`，`q` 属于以下哪一类：

- `inside`：严格在内部。
- `outside`：严格在外部。
- `on_boundary`：落在边或顶点上。

本题的 MVP 采用射线法作为主算法，并把边界点作为单独类别处理。

## R02

该问题是经典计算几何基础问题，长期用于图形学、GIS、CAD 与机器人路径规划。工程上最常见的两条路线是：

- 射线交点奇偶规则（Jordan 曲线思想）。
- 缠绕数（winding number）规则。

二者在“简单多边形 + 一致边界规则”下可得到一致分类结果，差异主要在实现细节和数值鲁棒处理方式。

## R03

本任务聚焦静态多边形、多点查询场景：

- 输入：一个二维多边形顶点序列 `[(x0,y0), ... , (xn-1,yn-1)]`，以及一个或多个查询点。
- 输出：每个点的分类标签（`inside / outside / on_boundary`）。

`demo.py` 中支持：
- 单点判定函数。
- 批量点判定函数。
- 确定性测试样例与随机回归样例。

## R04

射线法核心思想：

1. 从查询点 `q` 向 `+x` 方向发出半无限射线。
2. 遍历多边形每条边，统计边与射线是否相交。
3. 交点数为奇数则 `inside`，偶数则 `outside`。
4. 若 `q` 恰好在某条边上，直接返回 `on_boundary`。

实现关键在于“半开区间”处理顶点交会，避免同一个顶点被重复计数。

## R05

设多边形顶点数为 `n`，查询点数量为 `m`。

- 单点查询时间复杂度：`O(n)`。
- `m` 个点批量查询时间复杂度：`O(mn)`。
- 额外空间复杂度：`O(1)`（不含输入输出存储）。

对于频繁查询同一多边形的场景，可进一步引入空间索引或分解结构做加速，但不属于本 MVP 范围。

## R06

示例（`demo.py` 中包含）：

- 凹多边形：`[(0,0),(6,0),(6,4),(3.5,2.2),(1.5,4.8),(0,4)]`
- 测试点包含：
  - 严格内部点 `strictly_inside`
  - 明显外部点 `outside_right`
  - 边上点 `on_left_edge`
  - 顶点点 `on_vertex`
  - 凹陷区域外部点 `in_concavity_outside`

程序会打印每个点的分类，并用缠绕数结果逐项交叉验证。

## R07

优点：

## R10

正确性思路（射线法）：

1. 对简单闭曲线，任意从点发出的射线与边界相交次数的奇偶性决定内外。  
2. 点在外部时，可连续变形到无穷远且不跨边界，交点奇偶保持偶数。  
3. 点在内部时，通向无穷远必跨越边界奇数次。  
4. 边界点不参与奇偶规则，单独提前判定为 `on_boundary`。  
5. 通过半开区间处理顶点，保证每次真实穿越只计一次。  

因此，在简单多边形假设下算法分类是正确的。

## R11

鲁棒性处理（`demo.py` 已实现）：

- 统一 `eps = 1e-9` 处理“接近共线/接近重合”。
- `point_on_segment` 先判叉积接近 0，再判投影范围。
- 射线交点计数采用 `(ay > py) != (by > py)`，避免顶点重复统计。
- 输入归一化会去掉重复尾点和相邻重复点。
- 面积近零多边形视作退化并报错。

## R12

参数与工程权衡：

- 关键参数仅 `eps`。
- `eps` 太小：可能把边界点错判为内/外。
- `eps` 太大：可能把近边但不在边上的点误判为边界。

建议：
- 若坐标尺度约在 `1e0 ~ 1e3`，`1e-9 ~ 1e-8` 通常够用。
- 若坐标尺度更大，按数据量级相应放大 `eps`。

## R14

常见失败模式：

1. 多边形自交但仍按简单多边形处理。  
2. 忽略边界点单独判定，导致边上点被当作内/外。  
3. 顶点交会重复计数，造成奇偶翻转错误。  
4. 输入含大量重复点或退化边。  
5. `eps` 与数据尺度不匹配。

对策：
- 增加输入校验与预清洗。
- 固化边界语义（本题为 `on_boundary` 独立类别）。
- 用第二实现（缠绕数）做回归对照。

## R18

`demo.py` 源码级算法流程（8 步）：

1. `normalize_polygon` 清洗输入：去闭合重复尾点、去相邻重复点，并拒绝近零面积退化多边形。  
2. `point_on_segment` 用“叉积接近 0 + 投影区间”判断点是否在边上。  
3. `point_in_polygon_ray` 遍历每条边，先调用 `point_on_segment` 做边界短路判定。  
4. 对非边界情况，使用半开条件 `(ay > py) != (by > py)` 判断边是否穿过查询点水平线。  
5. 计算交点横坐标 `x_intersection`，若在点右侧则翻转奇偶标记 `inside`。  
6. 遍历结束后依据奇偶标记输出 `inside` 或 `outside`。  
7. `point_in_polygon_winding` 以缠绕数规则独立实现同任务，用于结果交叉校验。  
8. `run_named_case` 与 `run_random_regression` 对比两种实现的每个输出，`main` 汇总打印并在不一致时抛错终止。

整个 MVP 没有把第三方几何库作为黑盒；核心判定、鲁棒细节和验证流程都在源码中可逐步追踪。
