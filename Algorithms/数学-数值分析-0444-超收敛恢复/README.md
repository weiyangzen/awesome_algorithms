# 超收敛恢复

## R01

超收敛恢复（Superconvergent Recovery）是有限元后处理中的经典技术，目标是在已有有限元解 `u_h` 基础上恢复更高精度的导数或应力场。

直观上：
- 原始梯度 `grad(u_h)` 往往只有一阶精度且在单元间不连续；
- 某些“超收敛采样点”（如线性单元中点）上，梯度误差会更小；
- 把这些更准的局部信息做拟合/平均，可得到收敛阶更高、且更平滑的恢复场。

## R02

本条目采用最小可运行问题：一维 Poisson 方程

- 强形式：`-u''(x)=f(x), x in (0,1)`
- 边界：`u(0)=u(1)=0`
- 取 `f(x)=pi^2 sin(pi x)`，精确解 `u(x)=sin(pi x)`，精确梯度 `u'(x)=pi cos(pi x)`

输入：网格剖分数 `N`（单元数）。  
输出：
- 原始梯度 `L2` 误差；
- 恢复梯度 `L2` 误差；
- 收敛阶（网格加密一倍时的 rate）；
- 线性系统残差与条件数诊断。

## R03

离散模型使用一维线性有限元（P1）：

- 网格节点：`x_i=i/N`；
- 有限元解 `u_h` 在每个单元上线性；
- 原始梯度在每个单元上为常数：
  `g_e=(u_{i+1}-u_i)/h`。

恢复目标：从单元中点处的 `g_e` 重建节点梯度 `G_i`，再按分片线性插值得到恢复梯度场 `G_h(x)`。

## R04

本 MVP 选用“局部多项式最小二乘恢复”（PPR/SPR 思路）：

1. 对每个节点 `x_i`，取邻域 patch 的若干单元中点 `x_{m,j}`；
2. 用中点梯度样本 `g_j` 拟合局部多项式 `p_i(x)`（默认二次）；
3. 令恢复节点梯度 `G_i = p_i(x_i)`；
4. 在每个单元上用 `G_i, G_{i+1}` 线性插值得 `G_h(x)`。

该流程不依赖黑盒库，所有步骤都在 `demo.py` 明确展开。

## R05

复杂度（设单元数为 `N`，patch 半径为常数 `r`）：

- 有限元装配：`O(N)`；
- 线性系统求解（此处稠密 `numpy.linalg.solve`）：`O(N^3)`；
- 恢复步骤：每个节点做常规模型最小二乘，整体 `O(N)`；
- 误差积分：`O(N)`。

在演示规模 `N<=128` 下，运行开销很小。

## R06

微型示意（`N=4`，`h=0.25`）：

- 原始梯度由 4 个单元常数构成，存在明显分段跳跃；
- 在节点 `x_2=0.5` 附近，patch 可取 3~4 个中点样本；
- 二次拟合后在 `x_2` 处求值，得到 `G_2`；
- 对所有节点重复后可构成连续的恢复梯度曲线。

该过程把“局部离散常数斜率”转成“全局更平滑且更高精度的梯度近似”。

## R07

超收敛恢复的重要性：

- 工程后处理：应力/通量可视化比原始离散导数更可靠；
- 误差估计：Zienkiewicz-Zhu 类自适应常用 `||G_h-grad(u_h)||` 作为指示量；
- 网格自适应：恢复场可驱动局部加密策略；
- 兼容性强：对很多有限元离散都可叠加使用。

## R08

理论关键点（本例对应）：

1. 线性元下，某些点（如中点）梯度误差可呈现超收敛行为。  
2. 局部多项式拟合可“提纯”这些更准样本。  
3. 恢复后的节点梯度 `G_i` 通常达到高于原始 `grad(u_h)` 的收敛阶。  
4. 本例常见现象：原始梯度 `L2` 误差约 `O(h)`，恢复梯度约 `O(h^2)`。  
5. 该改进依赖网格质量、解的光滑性、patch 选择与拟合阶数。

## R09

适用条件与局限：

- 适用：解较光滑、网格较规则、导数后处理需求明确；
- 局限：奇异点、强不连续系数、畸形网格会削弱超收敛；
- 本实现仅覆盖 1D P1 均匀网格，不直接代表复杂 2D/3D 工程问题；
- 当前求解器是教学型稠密线代，不是大规模生产配置。

## R10

正确性验证框架：

- 离散方程层：检查 `||A u - b||_inf`；
- 收敛层：`N` 翻倍时原始与恢复误差都应下降；
- 提升层：每个 `N` 下恢复误差应明显小于原始误差；
- 阶数层：观测 raw rate 约 1、recovered rate 约 2；
- 数值层：条件数、误差、残差均需为有限值。

## R11

`demo.py` 的典型现象（默认参数）：

- `L2_raw_grad` 随网格加密约按 `O(h)` 收敛；
- `L2_recovered` 约按 `O(h^2)` 收敛；
- `raw/rec` 改善倍数随 `N` 增加持续扩大；
- 恢复场在节点处的最大误差快速下降。

这符合“恢复后更高阶”的超收敛预期。

## R12

性能与工程取舍：

- 如需扩展到工程规模，应替换为稀疏装配+稀疏求解，并做并行 patch 恢复。

## R14

鲁棒性与常见失效模式：

- patch 太小或多项式阶过高，会导致拟合欠定或过拟合；
- 极粗网格下边界 patch 样本不足，恢复质量会下降；
- 积分阶过低会污染误差统计；
- 若问题解不光滑，恢复阶数可能退化。

本实现通过“最小样本数约束 + 边界自动扩 patch”降低失败概率。

## R15

`demo.py` 结构映射：

- `assemble_poisson_system`：装配 FE 刚度矩阵与载荷向量；
- `solve_fem_solution`：求解离散系统并返回残差/条件数；
- `element_gradients`：构造原始分片常数梯度；
- `recover_nodal_gradients`：基于 patch 最小二乘恢复节点梯度；
- `l2_error_raw_gradient` / `l2_error_recovered_gradient`：计算误差；
- `validate_results`：执行收敛与准确性断言；
- `main`：批量网格实验并打印报告。

## R16

相关方法链路：

- ZZ 恢复（Zienkiewicz-Zhu）：最常见后验误差估计恢复框架；
- SPR（Superconvergent Patch Recovery）：基于 patch 拟合应力/梯度；
- PPR（Polynomial Preserving Recovery）：强调多项式保持性质；
- 与本例关系：本实现属于 PPR/SPR 风格的一维最小化版本。

## R18

源码级算法流程拆解（`demo.py`，8 步）：

1. `main()` 构造 `RecoveryConfig`，设定网格序列 `N=(8,16,32,64,128)`。  
2. `run_suite()` 逐个网格调用 `run_case()`，进入“求解 + 恢复 + 评估”闭环。  
3. `run_case()` 在 `solve_fem_solution()` 中先由 `assemble_poisson_system()` 组装 `A,b`，再解线性系统得到节点解 `u_h`。  
4. `element_gradients()` 计算原始梯度 `g_e=(u_{i+1}-u_i)/h`，这是分片常数场。  
5. `recover_nodal_gradients()` 对每个节点选 patch（`_select_patch_indices`），在中点样本上做局部最小二乘多项式拟合，并取常数项作为恢复节点梯度 `G_i`。  
6. `l2_error_raw_gradient()` 以高斯积分计算原始梯度对真梯度的 `L2` 误差；`l2_error_recovered_gradient()` 用节点恢复值线性重建 `G_h(x)` 后计算其 `L2` 误差。  
7. `print_report()` 输出每个网格的 `L2_raw_grad`、`L2_recovered`、改进倍数与观测收敛阶。  
8. `validate_results()` 断言恢复误差严格小于原始误差、收敛率达到阈值、残差与数值状态正常，最终打印 `All checks passed.`。

说明：实现未调用任何“超收敛恢复黑盒函数”，恢复流程（patch 选取、局部拟合、求值）均在源码中显式实现。
