## R01

Gauss-Seidel 迭代法用于求解线性方程组 `Ax=b`，属于经典定常迭代方法。

把矩阵分解为：

- `A = D + L + U`
- `D`：对角部分
- `L`：严格下三角部分
- `U`：严格上三角部分

其迭代公式为：

- 向量形式：`x^{k+1} = -(D+L)^{-1} U x^k + (D+L)^{-1} b`
- 分量形式：
  `x_i^{k+1} = ( b_i - Σ_{j<i} a_{ij} x_j^{k+1} - Σ_{j>i} a_{ij} x_j^k ) / a_{ii}`

核心特点是“同一轮内立刻使用新值”，通常比 Jacobi 迭代收敛更快。

## R02

历史脉络可概括为“高斯思想 + 塞德尔整理”：

- 19 世纪早期：Gauss 在处理最小二乘与线性方程求解时已使用逐次代入式迭代思想。
- 1874 年：Philipp Ludwig von Seidel 对该迭代流程做了系统化表述与传播。
- 20 世纪：该方法成为数值线性代数与偏微分方程离散求解中的基础模块。

所以命名常写为 Gauss-Seidel，强调两位学者对方法形成与传播的共同贡献。

## R03

提出背景与动机：

- 大规模线性系统中，直接法（如高斯消元）在时间和存储上可能代价过高。
- 工程问题常得到稀疏矩阵（例如有限差分/有限元离散后），适合迭代法。
- 目标不一定是机器精度精确解，而是“足够精度的近似解”。

Gauss-Seidel 的价值在于：

- 每步更新简单、实现门槛低。
- 可利用稀疏结构。
- 常作为 SOR、多重网格平滑器、预条件思想的基础原件。

## R04

时间复杂度（每轮迭代）：

- 稠密矩阵：`O(n^2)`
- 稀疏矩阵：`O(nnz)`，其中 `nnz` 为非零元个数

若总迭代轮数为 `k`，则总复杂度约为：

- 稠密：`O(k n^2)`
- 稀疏：`O(k nnz)`

其中 `k` 强依赖于矩阵谱性质、初值与停止阈值。

## R05

空间复杂度：

- 仅迭代求解时，除输入 `A,b` 外，额外状态主要是当前解向量与少量标量。
- 额外空间约 `O(n)`。
- 若记录完整收敛历史（如每轮残差），需再加 `O(k)`。

## R06

手算示例（两元一次）：

线性系统：

- `4x1 + x2 = 9`
- `2x1 + 3x2 = 13`

迭代格式：

- `x1^{k+1} = (9 - x2^k)/4`
- `x2^{k+1} = (13 - 2x1^{k+1})/3`

取初值 `x^0=(0,0)`：

- 第 1 轮：
  - `x1^1 = 9/4 = 2.25`
  - `x2^1 = (13 - 2*2.25)/3 = 17/6 ≈ 2.8333`
- 第 2 轮：
  - `x1^2 = (9 - 2.8333)/4 ≈ 1.5417`
  - `x2^2 = (13 - 2*1.5417)/3 ≈ 3.3056`

可见序列正向精确解 `(1.4, 3.4)` 逼近。

## R07

算法意义：

- 数学意义：把线性系统求解转化为不动点迭代问题。
- 工程意义：在大规模稀疏系统中，以低内存实现可控精度求解。
- 方法论意义：体现“局部更新立即反馈”对收敛速度的改进。

它是“从直接法到迭代法”的关键入门算法之一。

## R08

前置知识：

- 线性方程组与矩阵分解（`D/L/U`）。
- 向量范数与残差 `r=b-Ax`。
- 迭代收敛概念（谱半径、收敛判据）。
- 基本浮点数值计算常识。

## R09

适用前提与边界：

- 必须满足 `a_{ii} != 0`，否则分量更新不可定义。
- 常见充分收敛条件之一：`A` 严格对角占优。
- 对对称正定（SPD）系统，Gauss-Seidel 也具有良好收敛性质。
- 若矩阵条件不佳或谱半径过大，可能收敛很慢甚至发散。

因此在工程中通常会结合：

- 变量重排
- 预处理
- 改进法（如 SOR、CG、GMRES）

## R10

正确性依据（摘要）：

令迭代矩阵 `T = -(D+L)^{-1}U`，常向量 `c=(D+L)^{-1}b`，则

- `x^{k+1}=Tx^k+c`
- 精确解 `x*` 满足 `x*=Tx*+c`

误差 `e^k=x^k-x*` 满足：

- `e^{k+1}=Te^k`
- `e^k=T^k e^0`

若 `ρ(T)<1`（谱半径小于 1），则 `T^k -> 0`，故 `e^k -> 0`，迭代收敛到唯一不动点 `x*`。

## R11

数值稳定性与误差传播：

- 每步只做加减乘除，局部数值操作稳定。
- 误差传播主要受 `T` 的谱性质控制，不是“单步舍入”主导。
- 条件数较大时，残差小不必然等于前向误差小，应结合业务精度判断。
- 对非常病态系统，建议使用更强鲁棒算法或预条件策略。

## R12

真实成本分析：

- 单轮成本低，但迭代轮数可能大，整体成本未必低于直接法。
- Python 逐元素循环存在解释器开销；超大规模场景应考虑稀疏格式和底层加速。
- 工程中常用“较宽松阈值 + 外层校验”平衡速度与精度。

本条 MVP 重点是算法透明性与可验证性，而不是极限性能。

## R13

近似性与收敛保证：

Gauss-Seidel 本质是近似迭代法，输出通常为近似解 `x^k`。

- 不是概率算法，无随机成功率概念。
- 在 `ρ(T)<1` 时，理论上保证收敛。
- 误差上界可写为 `||e^k|| <= ||T||^k ||e^0||`（在相容范数下）。

因此它的“保证”是条件性数学保证，不是无条件保证。

## R14

鲁棒性与失效模式：

- 失效模式 1：`a_{ii}=0` 或接近 0，更新公式不稳定或不可用。
- 失效模式 2：矩阵不满足收敛条件，残差震荡或发散。
- 失效模式 3：仅看迭代步长，不看残差，可能误判“已收敛”。
- 失效模式 4：把稠密 `O(n^2)` 实现直接用于超大规模问题，效率不可接受。

规避策略：

- 输入校验 + 残差与步长双指标停止。
- 监控最大迭代上限并给出未收敛告警。
- 结合矩阵性质选择更合适方法。

## R15

工程实现要点：

- 先做维度一致性检查：`A` 必须是 `n x n` 且 `b` 长度为 `n`。
- 先检查对角元素非零。
- 每轮更新时：
  - 前半段 `j<i` 使用本轮新值 `x[:i]`
  - 后半段 `j>i` 使用上一轮旧值 `x_old[i+1:]`
- 终止判据使用双阈值：
  - `||b-Ax||_inf < tol`
  - `||x^{k+1}-x^k||_inf < tol`
- 提供收敛历史，便于调参与排障。

## R18

源码级算法流程拆解（对应 `demo.py`，非黑箱）：

1. 输入规约：把 `A,b,x0` 转成 `numpy` 浮点数组并检查维度、对角非零。  
2. 初始化：设置初始向量 `x`，并准备历史记录容器。  
3. 外层迭代：从第 1 轮到 `max_iter` 循环。  
4. 旧值快照：复制 `x_old = x.copy()`，用于访问“上一轮”分量。  
5. 前向扫更新：对每个 `i` 按公式计算 `sigma_left` 与 `sigma_right`，更新 `x[i]`。  
6. 收敛监控：计算残差范数 `||b-Ax||_inf` 与步长范数 `||x-x_old||_inf` 并记录。  
7. 终止判定：若残差和步长都小于阈值，立即返回 `converged=True`。  
8. 失败返回：达到 `max_iter` 仍未满足阈值，则返回最后近似解和 `converged=False`，由上层决定如何处理。
