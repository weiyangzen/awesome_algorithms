## R01

几何规划（Geometric Programming, GP）是一类在 `x_i > 0` 域上的优化问题，标准形式是：

- 目标：最小化一个正多项式（posynomial）`f0(x)`；
- 约束：`fi(x) <= 1`（`fi` 为 posynomial）；
- 可选等式：`hj(x) = 1`（`hj` 为 monomial）。

其中：

- monomial 形如 `c * Π x_i^{a_i}`（`c > 0`，指数可为实数）；
- posynomial 是若干 monomial 的和。

GP 的关键价值在于：通过对数变量替换 `y = log x`，可转成凸优化问题。

## R02

几何规划常见于工程设计和资源分配，例如：

- 电路尺寸与功耗-延迟折中；
- 无线通信中的功率控制；
- 生产/库存中的乘性结构成本模型；
- 一些机器学习超参数的乘性近似模型。

它在“原问题看起来非凸、但结构上是 posynomial/monomial”时尤其有用。核心思想并非“暴力数值搜索”，而是利用代数结构把问题转成凸问题再求解。

## R03

本目录 MVP 解决的问题是一个 2 变量 GP：

- 最小化：`x^(-1) * y^(-1/2)`
- 约束：
  - `(x + y) / 10 <= 1`
  - `2 / x <= 1`
  - `3 / y <= 1`
  - `x > 0, y > 0`

该例有解析最优解 `x*=20/3, y*=10/3`，因此可作为可验证基准。

## R04

实现采用“建模层 + 变换层 + 求解层”三段：

1. 建模层：
   - `Monomial` 表达单项；
   - `Posynomial` 表达多项和；
   - 提供原变量值、对数值、梯度计算。
2. 变换层：
   - 令 `y = log x`；
   - 目标变为 `log f0(exp(y))`（与原目标同最优点）；
   - 约束 `fi(x)<=1` 变为 `log fi(exp(y))<=0`。
3. 求解层：
   - 使用 `scipy.optimize.minimize(method="SLSQP")`；
   - 显式传入目标梯度和约束雅可比；
   - 解出 `y*` 后再映射 `x*=exp(y*)`。

## R05

设变量维度为 `n`，每个 posynomial 含 `k` 个 monomial。

- 单次 `log_value` 计算：`O(k*n)`（主要是若干 `a^T y`）；
- 单次 `grad_log_value`：`O(k*n)`；
- 若总约束数为 `m`，一次 SLSQP 迭代的大头是目标/约束及梯度评估，约 `O((m+1) * k * n)` 量级（忽略求解内部 QP 的常数项）。

MVP 面向“可运行与可解释”，不是大规模稀疏 GP 的高性能实现。

## R06

`demo.py` 内置的是一个可验证小例子：

- 变量：`x, y`；
- 目标：`x^(-1) * y^(-1/2)`；
- 约束：线性和倒数下界混合（都可写成 posynomial<=1）；
- 初始点：`x0=[5,5]`（可行）。

程序输出：

- SLSQP 状态；
- 数值解 `x*`；
- 目标函数值；
- 与解析解的 2-范数误差；
- 每条约束残差 `fi(x*) - 1`。

## R07

优点：

- 完整体现 GP 的核心理论（log 变换 + log-sum-exp 凸化）；
- 没有把“建模逻辑”藏在黑盒库里，monomial/posynomial 都是显式代码；
- 同时给出解析解对照，便于回归测试。

局限：

- 只覆盖标准 GP（`fi<=1` 与 `hj=1`）；
- 求解器依赖 SLSQP，规模大时并非最优选择；
- 未实现稀疏结构利用、自动尺度归一化、warm start 策略。

代码环境：

## R09

适用场景：

- 问题可自然写成 monomial/posynomial；
- 变量要求严格正值；
- 需要一个“透明、可审计”的小型 GP 原型。

不适用场景：

- 大规模工业级 GP（需更专业内点法/锥优化器）；
- 含离散变量的混合整数优化；
- 非 GP 结构且无法通过等价变换进入标准形式的问题。

## R10

正确性直觉：

1. `log` 是单调递增函数，因此最小化 `f0(x)` 与最小化 `log f0(x)` 在最优点上等价；
2. 在 `x=exp(y)` 下，monomial 变成仿射函数 `log c + a^T y`；
3. posynomial 的对数是 `log-sum-exp`，为凸函数；
4. 因而目标凸、约束凸（`<=0`）且可选等式仿射，问题是标准凸规划；
5. 数值求解得到的 KKT 点即全局最优解（在可行且常规正则条件下）。

## R11

数值稳定策略：

- 使用 `stable_logsumexp`（先减最大值）避免指数上溢；
- 全流程在 log 域计算，规避原始域极小/极大幂导致的下溢与上溢；
- 为 SLSQP 提供解析梯度，减少有限差分噪声；
- 结果端做约束残差检查，防止“成功退出但轻微不可行”。

若数据尺度跨越特别大，建议先对系数和变量做归一化。

## R12

关键参数：

- `maxiter`：SLSQP 最大迭代次数；
- `ftol`：收敛精度；
- `x0`：初始点（必须严格正）。

调参建议：

- 若 `success=False`，优先增大 `maxiter`；
- 若出现轻微约束违背，适当收紧 `ftol`；
- 初始点尽量选在可行域内部，可明显提升收敛稳定性。

## R14

常见失败模式：

1. 输入存在非正系数（违反 GP 定义）；
2. 初始点含非正值，`log(x0)` 无定义；
3. 模型实际上不是 GP（例如出现减法）；
4. 规模过大或病态，SLSQP 收敛慢或停在数值边界。

诊断方式：

- 看异常信息（系数/维度检查）；
- 看 `result.message`；
- 看约束残差与解析解误差；
- 必要时打印每轮目标与残差（可扩展 callback）。

## R16

相关算法/模型：

- 线性规划（LP）：目标与约束线性；
- 凸二次规划（QP）：二次目标 + 线性约束；
- 锥规划（SOCP/SDP）：更广泛的凸问题框架；
- 符号规划（Signomial Programming）：允许正负项，通常非凸；
- 序列凸近似（SCA）：用于处理非凸乘性结构的近似方法。

GP 是“结构化非线性优化”里一类非常实用且可凸化的问题族。

## R18

`demo.py` 的源码级算法流程（8 步）：

1. `main()` 中把目标和约束写成 `Monomial` / `Posynomial` 对象，显式构造 GP。  
2. `GeometricProgram.solve()` 接收正初值 `x0`，先做 `y0 = log(x0)` 变量替换。  
3. `objective_fun(y)` 调用 `Posynomial.log_value(y)`，其内部把每个 monomial 变成 `log c + a^T y`。  
4. `stable_logsumexp` 用“减去最大值再指数求和”的方式稳定计算 `log(sum(exp(.)))`，避免上溢。  
5. `objective_jac(y)` 调用 `grad_log_value(y)`：先算 softmax 权重，再对指数向量加权求和得到梯度。  
6. 不等式 `fi(x)<=1` 被编码为 `-log fi(exp(y)) >= 0`，等式 `hj(x)=1` 编码为 `log hj(exp(y)) = 0`，并都提供解析雅可比。  
7. `scipy.optimize.minimize(..., method="SLSQP")` 在每轮迭代中使用这些函数值和梯度，解局部二次子问题并更新 `y`，直到满足收敛条件。  
8. 把 `y*` 映射回 `x*=exp(y*)`，然后在原始域计算目标、约束残差，并与解析解做误差校验。

说明：本实现没有把第三方库当“纯黑盒”。GP 的核心变换、`log-sum-exp`、梯度和约束编译都在源码中明确展开，SLSQP 仅承担数值迭代求解角色。
