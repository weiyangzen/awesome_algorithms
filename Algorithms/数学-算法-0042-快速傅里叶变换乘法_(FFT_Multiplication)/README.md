# 快速傅里叶变换乘法 (FFT Multiplication)

- UID: `MATH-0042`
- 学科: `数学`
- 分类: `算法`
- 源序号: `42`
- 目标目录: `Algorithms/数学-算法-0042-快速傅里叶变换乘法_(FFT_Multiplication)`

## R01

快速傅里叶变换乘法把“大整数乘法”转化为“多项式卷积”：

- 把整数按某个进位基底 `B` 拆成系数序列（低位在前），得到两个多项式 `A(x), B(x)`。
- 整数乘积对应系数卷积：`C(x)=A(x)B(x)`。
- 通过 FFT 在频域计算卷积，再逆变换回时域并处理进位，恢复最终整数。

输入通常是两个整数 `a, b`；输出是精确整数积 `a*b`。

## R02

历史脉络可概括为三步：

- 1965 年 Cooley-Tukey 重新普及 FFT，使离散傅里叶变换可在 `O(n log n)` 完成。
- 1971 年 Schönhage-Strassen 展示了“FFT/变换域 + 整数乘法”的大整数加速路线。
- 之后多精度库在不同规模区间组合 schoolbook / Karatsuba / Toom-Cook / FFT(NTT) 等策略。

因此“FFT 乘法”本质是把信号处理中的卷积加速迁移到整数算术。

## R03

提出背景是传统竖式乘法在大规模输入上过慢：

- 朴素乘法需要 `O(n^2)` 次系数乘加。
- 当位数达到几千、几万甚至更大时，二次复杂度成为瓶颈。
- 卷积可由频域点乘实现，结合 FFT 后降到 `O(n log n)`，在大规模显著更优。

它解决的是“超大整数乘法的可扩展性”问题。

## R04

设两个数拆分后长度分别为 `n, m`，令 `L = n + m - 1`，并取 `N` 为不小于 `L` 的 2 的幂。

- FFT 路径时间复杂度：`O(N log N)`。
- 频域点乘：`O(N)`。
- 进位处理：`O(L)`。
- 总体：`O(N log N)`，常写为 `O((n+m) log(n+m))`。

对比：朴素卷积是 `O(nm)`，在规模增大时增长更快。

## R05

空间复杂度（按本目录 MVP 实现）：

- 两个输入系数数组 + 频域数组 + 逆变换结果数组，主量级均为 `O(N)`。
- 进位后的结果列表也是 `O(L)`。
- 故额外空间整体为 `O(N)`。

实际峰值受 FFT 实现中间缓冲影响，但渐近不变。

## R06

示例：`12345 * 6789`

1. 取基底 `B=10`（便于说明），拆分：
   - `12345 -> [5,4,3,2,1]`
   - `6789  -> [9,8,7,6]`
2. 计算卷积系数（未进位）得到长度 `5+4-1=8` 的序列。
3. 将卷积结果按基底 `10` 做逐位进位。
4. 逆序拼接即得 `83810205`。

FFT 版本与手算逻辑等价，只是第 2 步由频域点乘加速。

## R07

算法意义：

- 理论上把整数乘法从二次复杂度推进到准线性级别。
- 工程上是大整数库、高精度计算、密码学和计算代数系统的关键基础。
- 方法论上体现了“问题重表示（整数 -> 多项式）+ 变换域求解”的通用范式。

## R08

直接依赖：

- 多项式与卷积的等价关系。
- 离散傅里叶变换与逆变换。
- Cooley-Tukey 类 FFT 分治思想（或库中等价实现）。
- 进位归并规则（把卷积系数还原为基底表示的整数）。

R08 关注“前置数学与算法构件”，R16 关注“后续应用链条”。

## R09

适用前提与边界：

- 适用于整数乘法（含正负号）；浮点误差需通过四舍五入与校验控制。
- 需要选择合适基底 `B`：过大可能放大频域数值误差，过小会增加序列长度。
- 当输入很小（几十位以内）时，FFT 常数开销可能不如朴素法。
- 长度为 0、全 0、负数等边界需显式处理。

## R10

正确性要点：

- 设 `a = Σ a_i B^i`, `b = Σ b_j B^j`。
- 则 `a*b = Σ_k (Σ_{i+j=k} a_i b_j) B^k`，括号内即卷积系数 `c_k`。
- FFT/逆 FFT 精确对应线性卷积（在零填充长度足够时不发生循环混叠）。
- 得到 `c_k` 后做标准进位，得到唯一基底 `B` 表示，因此恢复整数积。

本质上，FFT 只改变“求卷积的计算路径”，不改变代数结果。

## R11

数值稳定性（浮点 FFT 路径）主要风险：

- 频域复数运算引入舍入误差。
- 误差在逆变换后体现在系数偏离整数（如 `x.0000001` 或 `x.9999998`）。

常见缓解：

- 使用 `np.rint` 将逆变换结果四舍五入到最近整数。
- 选较保守基底（本 MVP 用 `10^4`）避免中间量过大。
- 对关键结果做校验（本 MVP 用 Python 内建大整数乘法做对拍）。

## R12

真实工程成本（超越渐近）：

- FFT 常数项大，包含复数数组分配、缓存访存和变换计划开销。
- 小规模输入下，`O(n^2)` 的朴素法可能更快。
- 规模越大，FFT 优势越明显；阈值与实现、硬件、基底选择相关。
- Python 层循环（如进位）也有开销，因此实践常用 C/C++ 后端库。

## R13

本算法在数学上是确定性精确算法，但实现层有“数值近似中间态”：

- 理论模型（精确复数域）下无误差。
- 浮点实现里，逆变换系数有微小误差，通过四舍五入回到整数域。
- 只要误差幅度小于 `0.5` 且进位逻辑正确，最终结果可精确恢复。

因此可把它视作“近似中间计算 + 精确离散恢复”的确定性流程。

## R14

鲁棒性与失效模式：

- 基底过大或输入过长可能导致浮点误差累积，出现错位进位。
- 未做足够零填充会把线性卷积误算成循环卷积。
- 忽略符号或前导零清理会导致格式错误。
- 负数/零处理不当会造成边界 bug。

防护策略：参数约束、四舍五入、随机对拍、断言检查与回退方案。

## R15

工程实现建议：

- 将“表示转换、FFT 卷积、进位恢复、符号处理”拆成独立函数，便于测试。
- 用可复现随机种子做批量自检，覆盖正负数、0、大位数。
- 提供朴素乘法基线用于回归验证。
- 明确阈值策略：小规模走朴素法，大规模走 FFT（本 MVP 以演示为主，统一走 FFT 并做对拍）。

## R16

前驱/后继关系：

- 前驱基础：DFT、FFT、卷积定理、位权表示。
- 后继方向：
  - NTT（模域 FFT）避免浮点误差；
  - 更高阶大整数算法（Schönhage-Strassen、Fürer、Harvey–van der Hoeven 路线）；
  - 多精度库中的混合乘法策略调度。

应用包括密码学大数运算、计算机代数、科学计算高精度内核。

## R17

Python MVP 设计（见本目录 `demo.py`）：

- 工具栈：`numpy` + Python 标准库（`random`、`time`、`typing`）。
- 核心函数：
  - `int_to_digits` / `digits_to_int`：整数与基底数组互转；
  - `fft_convolution`：基于 `numpy.fft` 的线性卷积；
  - `multiply_integers_fft`：完整乘法流程；
  - `schoolbook_multiply_digits`：朴素基线。
- 运行方式：`python3 demo.py`，无需交互输入。
- 输出内容：样例乘法、随机正确性校验、FFT 与朴素法耗时对比。

## R18

源码级流程拆解（对应 `demo.py`，非黑箱叙述）：

1. `int_to_digits` 把输入整数按 `BASE=10_000` 拆成低位在前的系数数组，并分离符号。
2. `fft_convolution` 根据 `len(a)+len(b)-1` 计算线性卷积需要的长度，再扩展到最近 2 的幂 `n_fft`。
3. 调用 `np.fft.fft` 分别对两个系数数组做前向变换，得到频域采样值。
4. 在频域做逐点相乘 `fa * fb`，利用卷积定理把时域卷积转为点乘。
5. 调用 `np.fft.ifft` 回到时域，并对实部用 `np.rint` 四舍五入到整数卷积系数。
6. `carry_normalize` 逐位执行 `total = coeff + carry` 的进位归并，确保每位落在 `[0, BASE)`。
7. `digits_to_int` 把归并后的位数组还原成整数，并根据原符号得到最终结果。
8. `run_self_checks` 用随机样本将结果与 Python 内建 `a*b` 对拍，验证流程正确性。

以上 8 步中，第三方库只承担 FFT 数值核；表示转换、长度策略、进位恢复与正确性验证都在源码中显式实现。
