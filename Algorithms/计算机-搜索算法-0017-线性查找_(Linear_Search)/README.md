# 线性查找 (Linear Search)

## R01

线性查找用于在一个可迭代序列中，从左到右逐个比较元素，找到目标值第一次出现的位置。
若目标不存在，则返回“未找到”标记（本实现中为 `-1`）。

## R02

输入：
- `data`：可按顺序访问的序列（如 `list`、`tuple`、`numpy.ndarray`）
- `target`：待查找目标值
- `key`（可选）：映射函数，用于从复杂元素中提取可比较键值

输出：
- `SearchResult(index, comparisons)`
- `index` 为首次命中位置，未命中则 `-1`
- `comparisons` 为实际比较次数

## R03

核心思想：
- 顺序扫描，不依赖排序条件
- 每走一步做一次“当前元素是否等于目标”的判断
- 一旦命中立即返回，避免无意义后续扫描

## R04

伪代码：

```text
for i from 0 to n-1:
    if project(data[i]) == target:
        return i
return -1
```

其中 `project` 是 `key` 映射函数（若未提供则为恒等映射）。

## R05

正确性要点（循环不变式）：
- 在第 `i` 轮比较开始前，区间 `[0, i)` 已被完整检查且都不等于目标。
- 若在位置 `i` 命中，则它必是“第一个命中位置”。
- 若循环结束仍未命中，则整个序列都不等于目标，返回 `-1` 正确。

## R06

复杂度：
- 最好时间复杂度：`O(1)`（首元素命中）
- 最坏时间复杂度：`O(n)`（末尾命中或未命中）
- 平均时间复杂度：`O(n)`
- 空间复杂度：`O(1)`（原地顺序扫描）

## R07

前置条件与约束：
- 元素与目标在 `key` 映射后应可比较（支持 `==`）
- 序列可进行顺序迭代
- 本算法不修改输入数据

## R08

与二分查找对比：
- 线性查找：不要求有序，代价 `O(n)`
- 二分查找：要求有序，代价 `O(log n)`
- 当数据规模小、数据无序、或一次性临时查找时，线性查找更直接且实现成本最低

## R12

边界情况：
- 空序列：比较次数为 `0`，返回 `index=-1`
- 重复元素：返回首次出现位置
- 所有元素都不匹配：扫描完整序列后返回未命中

## R13

测试与验证策略：
- 使用 `assert` 做最小自检，覆盖命中与未命中分支
- 验证比较次数与扫描路径一致
- 验证 `key` 模式在复杂结构上可用

## R14

常见错误：
- 把“返回值”误写成布尔值，丢失命中位置信息
- 忘记“首次命中即返回”，导致返回最后一次命中位置
- 在复杂对象上直接比较而未提供 `key`

## R18

`demo.py` 中线性查找的源码级流程（6 步）：

1. 进入 `linear_search(data, target, key)`，根据 `key` 是否为空决定投影函数：为空用 `_identity`，否则用调用方提供函数。
2. 初始化 `comparisons = 0`，用于记录实际比较次数。
3. 使用 `enumerate(data)` 从左到右遍历，每轮拿到 `(idx, item)`。
4. 每访问一个元素先执行 `comparisons += 1`，保证统计与真实比较一致。
5. 计算 `projector(item)` 与 `target` 做相等比较；若相等，立即返回 `SearchResult(index=idx, comparisons=comparisons)`。
6. 若循环结束仍无命中，返回 `SearchResult(index=-1, comparisons=comparisons)`，表示已完整扫描且未找到目标。
