## R01

Inception 网络（典型代表是 GoogLeNet / Inception v1）是一类多分支卷积架构。它在同一层里并行使用 `1x1`、`3x3`、`5x5` 卷积和池化分支，再在通道维拼接输出，以较低参数量获得多尺度特征表达能力。

核心思想是：在“同一个空间分辨率”上同时看不同感受野，让网络自己学习该更依赖局部细节还是更大范围上下文。

## R02

Inception 的提出背景是：网络越做越深、越宽时，计算与参数成本急剧上升。传统“单路径堆叠卷积”在提升表达能力时，常遇到显存和算力瓶颈。

GoogLeNet 的关键工程改进包括：
- 用 `1x1` 卷积做通道降维，显著降低 `3x3/5x5` 的计算量；
- 模块化设计（Inception block）便于堆叠；
- 用全局平均池化替代大规模全连接层，进一步减参。

## R03

本算法要解决的问题：
1. 如何在不暴涨计算量的前提下引入多尺度特征提取；
2. 如何在 CNN 中同时建模细粒度纹理与更大感受野语义；
3. 如何构造易堆叠、可训练、参数效率较高的视觉骨干网络。

Inception 的解法是“并行分支 + 降维后大核卷积 + 拼接融合”。

## R04

设输入特征图为 `X in R^(B x C x H x W)`，一个 Inception 模块可表示为：

- 分支1：`Y1 = Conv1x1(X)`
- 分支2：`Y2 = Conv3x3(Conv1x1(X))`
- 分支3：`Y3 = Conv5x5(Conv1x1(X))`
- 分支4：`Y4 = Conv1x1(MaxPool3x3(X))`

最终输出：`Y = Concat(Y1, Y2, Y3, Y4)`（按通道拼接）。

其中 `Conv1x1` 的作用既是引入非线性，也是在进入昂贵卷积前做通道压缩。

## R05

复杂度分析（单层主项，忽略常数）：

- 普通卷积：`O(H * W * Cin * Cout * K^2)`
- Inception 中带降维的大核分支：
  - `1x1` 降维：`O(H * W * Cin * Cred)`
  - 后续 `KxK`：`O(H * W * Cred * Cout * K^2)`

当 `Cred << Cin` 时，整体计算显著下降。代价是模块结构更复杂、分支调参更细。

## R06

形状示例（本目录 MVP 的一个模块）：

输入 `X: [B, 16, 8, 8]`
- `1x1` 分支输出 `[B, 8, 8, 8]`
- `1x1->3x3` 分支输出 `[B, 12, 8, 8]`
- `1x1->5x5` 分支输出 `[B, 4, 8, 8]`
- `pool->1x1` 分支输出 `[B, 8, 8, 8]`

拼接后 `Y: [B, 32, 8, 8]`。

## R07

优点：
- 多尺度并行特征提取，表达力强；
- 参数效率通常优于“直接宽卷积堆叠”；
- 模块化良好，易扩展到更深网络。

局限：
- 分支结构使实现与调参复杂度上升；
- 对硬件和内核融合不友好时，实际吞吐未必最优；
- 后续架构（ResNet、EfficientNet、ConvNeXt、ViT）在不同场景可能更优。

## R08

实现和理解 Inception 需要的基础：
- CNN 基础：卷积、池化、激活函数、通道拼接；
- 张量形状管理：`NCHW` 维度变换与对齐；
- 训练基础：交叉熵、反向传播、优化器；
- PyTorch 模块化编程与 DataLoader 使用。

## R09

适用场景：
- 图像分类、特征提取等二维网格数据任务；
- 需要兼顾不同尺度信息（局部纹理与全局结构）的任务；
- 在参数预算有限时仍希望提高表示能力。

不太适用：
- 极端资源受限且更依赖算子融合的部署环境；
- 任务更适合 Transformer 或轻量移动架构时。

## R10

正确性验证要点（本目录 MVP）：
1. 各分支输出 `H/W` 必须一致，才能 `torch.cat`；
2. `1x1` 降维通道数配置正确，避免计算图维度错误；
3. 训练过程中 `loss` 有下降趋势；
4. 测试集准确率显著高于随机猜测（10 类约 10%）；
5. 模型参数与损失值保持有限（非 NaN/Inf）。

## R11

数值稳定性与训练策略：
- 输入归一化到 `[0,1]`，并支持可选高斯平滑（默认关闭）；
- 使用 `ReLU` 与 `Adam` 获得稳定收敛；
- 采用保守学习率与轻微权重衰减抑制过拟合；
- 固定随机种子，保证实验可复现；
- 使用 mini-batch 训练，降低梯度方差。

## R12

关键超参数（MVP 默认值）：
- `batch_size=128`
- `epochs=24`
- `lr=1e-3`
- `weight_decay=1e-4`
- `gaussian_sigma=0.0`（默认不平滑）

结构超参数：
- stem 通道数、每个 Inception 分支通道数、模块堆叠层数。

调参经验：
- 准确率不足可先增大 epochs；
- 若过拟合，增大权重衰减或减少通道数；
- 若收敛慢，微调学习率而非盲目加深网络。

## R13

理论性质（工程视角）：
- 深度网络训练是非凸优化，不保证全局最优；
- Inception 的收益主要来自结构归纳偏置（多尺度 + 降维）；
- 在有限样本下，该结构常能以更少参数达到可接受泛化性能。

本条目是监督分类任务，不涉及近似比保证，评价以经验指标（loss、accuracy）为准。

## R14

常见失效模式与排查：
- 维度不匹配：检查各分支 padding/stride；
- loss 不下降：检查学习率、归一化、标签类型（`long`）；
- 测试精度低：可能欠拟合（模型太小或训练不足）或数据预处理不当；
- 训练震荡：尝试更小学习率或增加权重衰减。

## R15

本目录实现策略：
- 数据：`sklearn.datasets.load_digits`（8x8 灰度手写数字）；
- 预处理：归一化 + 可选 `scipy.ndimage.gaussian_filter`；
- 模型：手写 `InceptionBlock`，不调用 torchvision 的黑盒 Inception；
- 训练与评估：PyTorch + `pandas` 记录 epoch 日志；
- 验收：断言损失下降、准确率达阈值、参数有限。

## R16

相关算法对比：
- LeNet/VGG：单路径卷积堆叠，结构直观但多尺度融合较弱；
- Inception：并行多尺度分支，参数效率较高；
- ResNet：残差连接改善深层训练；
- MobileNet：深度可分离卷积强调移动端效率；
- ViT：以注意力替代卷积归纳偏置，数据规模需求通常更高。

## R18

`demo.py` 的源码级算法流程（8 步）：
1. `set_seed` 固定 `numpy/torch` 随机种子，确保实验复现。  
2. `load_preprocessed_digits` 加载 digits 数据，执行归一化、可选高斯平滑和分层切分。  
3. `make_dataloader` 把 `numpy` 数组转成 `[N,1,8,8]` 张量并构造批迭代器。  
4. `InceptionBlock.forward` 分别计算 `1x1`、`1x1->3x3`、`1x1->5x5`、`pool->1x1` 四个分支并做通道拼接。  
5. `TinyInceptionNet.forward` 执行 `stem -> inception1 -> downsample -> inception2 -> global avg pool -> linear` 得到 logits。  
6. `train_one_epoch` 在每个 batch 上执行 `zero_grad -> forward -> cross_entropy -> backward -> step`。  
7. `evaluate` 在训练集/测试集计算平均损失、准确率并返回预测结果用于诊断。  
8. `main` 组织完整训练流程，记录 `pandas` 日志，输出混淆矩阵与分类报告，并通过断言完成验收。

说明：核心 Inception 算法流已在源码中显式展开，不是“调用第三方模型一行完成”的黑盒实现。
