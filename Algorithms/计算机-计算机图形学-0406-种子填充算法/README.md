## R01

种子填充算法（Seed Fill / Flood Fill）是计算机图形学中的经典区域填充方法：
从一个种子点出发，只在“与种子点初始颜色相同且连通”的像素集合内扩展，并把这些像素统一改写为目标填充值。

它的核心语义是“按连通分量改色”，而不是“按边界色阻挡”改色。

## R02

本条目要解决的 MVP 问题：

给定二维整数栅格 `canvas`、种子点 `seed=(row,col)`、填充值 `fill_value`、连通方式 `connectivity∈{4,8}`，实现一个可运行脚本：
- 自动识别 `seed` 位置的原始颜色 `source_value`；
- 将与 seed 连通且值等于 `source_value` 的像素改为 `fill_value`；
- 输出填充前后画布、填充统计和若干一致性检查。

## R03

选择该算法作为 MVP 的理由：
- 是图形学、图像处理和地图编辑中的基础操作；
- 算法结构清晰（图遍历 + 条件写入），适合教学与验证；
- 使用 `numpy + Python` 即可完整落地，不依赖 GUI 或交互输入。

## R04

离散模型定义：

设输入画布为 `A`，种子点为 `s`，种子初始值为 `v0=A[s]`，目标填充值为 `vf`。
定义可达集合 `R` 为：
- 包含 `s`；
- 任取 `p,q∈R`，两者可通过 `4` 邻域或 `8` 邻域路径连接；
- 路径上所有像素在原始画布中的取值均为 `v0`。

输出画布 `B` 满足：
- 若 `x∈R`，则 `B[x]=vf`；
- 若 `x∉R`，则 `B[x]=A[x]`。

## R05

MVP 高层流程（迭代版，非递归）：

1. 校验输入（二维数组、seed 合法、连通参数合法）。
2. 读取 `source_value = canvas[seed]`。
3. 若 `source_value == fill_value`，直接返回（0 像素改写）。
4. 使用显式栈从 `seed` 开始遍历。
5. 每次弹出像素，若其当前值不是 `source_value` 则跳过。
6. 否则写入 `fill_value`，并把邻域像素压栈。
7. 栈清空后结束，输出结果画布与统计信息。

## R06

正确性要点：

- 安全性（不会误改）：只有值等于 `source_value` 的像素才会被写为 `fill_value`。
- 完备性（不会漏改）：从 seed 出发可达且值为 `source_value` 的像素，最终都会被遍历并写入。
- 终止性：每个像素最多从“源值”变为“填充值”一次，状态单调，有限步终止。

## R07

复杂度分析：

设画布大小为 `H*W`，可填充连通域大小为 `K`。
- 时间复杂度：`O(K)` 到 `O(HW)`（最坏填满全图）。
- 空间复杂度：`O(K)`（显式栈在最坏情况下存储整个连通域）。

常数因子由邻域大小决定：`4` 邻域每步扩展 4 个方向，`8` 邻域为 8 个方向。

## R08

边界与异常处理策略：
- `canvas` 必须是二维 `numpy.ndarray`；
- `seed` 必须落在有效索引内；
- `connectivity` 仅允许 `4` 或 `8`；
- 若 `source_value == fill_value`，返回稳定结果（不做无效遍历）；
- 算法不要求预先给出边界值，停止条件仅由“值是否等于 source_value”决定。

## R09

MVP 取舍说明：
- 仅实现核心 seed-fill，不引入 OpenCV/PIL 的黑盒填充 API；
- 不做扫描线优化、并行化或 GPU 版本；
- 优先保证“可读、可运行、可验证”，并保持依赖最小化。

## R10

`demo.py` 主要函数职责：
- `FillStats`：记录 seed、连通方式、源值、填充值、填充数量、弹栈步数；
- `neighbor_offsets`：返回 `4/8` 邻域偏移；
- `seed_fill`：核心种子填充实现（显式栈 DFS）；
- `build_component_canvas`：构造“同色但不连通”的演示画布；
- `build_diagonal_canvas`：构造“仅对角连通”的演示画布；
- `render_canvas`：终端友好的 ASCII 渲染；
- `run_component_case` / `run_diagonal_cases`：执行并检查示例；
- `main`：无交互自动运行全部演示。

## R11

运行方式（无交互输入）：

脚本会自动打印两个实验组：

## R12

输出字段说明：
- `source_value`：seed 的原始像素值（填充判定基准）；
- `fill_value`：写入值；
- `filled_pixels`：实际改写像素数；
- `popped_steps`：从栈弹出处理的次数；
- `only_source_changed`：是否仅改写了源值像素；
- `disconnected_component_preserved`：不连通同色区域是否保持不变。

## R13

脚本内置最小测试集：

1. 连通分量实验：
- 画布中有两块同色区域（值 `3`）彼此不连通；
- 以第一块区域为 seed 填充为 `2`；
- 预期：仅第一块区域被改写，第二块保持 `3`。

2. 对角连通实验：
- 构造一条仅对角相接的同色链（值 `4`）；
- 以链首作为 seed：
  - `4` 邻域预期只填充 `1` 个像素；
  - `8` 邻域预期填充整条链（`5` 个像素）。

## R14

关键参数与使用建议：
- `seed`：应放在目标区域内部；
- `fill_value`：建议与目标区域当前值不同，避免无效调用；
- `connectivity`：
  - 选择 `4` 可避免“对角接触即连通”；
  - 选择 `8` 可把对角接触视作同一区域。

工程上要先统一连通性定义，再讨论填充结果，否则容易出现“看起来错误、其实是定义不同”的歧义。

## R15

与相关算法对比：
- 对比边界填充（Boundary Fill）：
  - 种子填充依赖“源颜色一致”；
  - 边界填充依赖“边界值阻挡”。
- 对比扫描线填充（Scanline Fill）：
  - 扫描线在多边形栅格化中更高效；
  - 种子填充实现更直接，适合局部区域交互改色。
- 对比递归实现：
  - 递归代码短但受调用栈深度限制；
  - 显式栈更稳健，适合生产代码。

## R16

典型应用场景：
- 绘图工具“油漆桶”功能；
- 栅格地图编辑中的区域改色；
- 图像分割前的快速连通域操作；
- 教学演示图遍历（DFS/BFS）与连通定义差异。

## R17

可扩展方向：
- 加入颜色容差（`abs(pixel-source)<=tol`）支持近似填充；
- 使用队列实现 BFS 并统计波前层级；
- 引入扫描线 seed fill 以减少重复邻域压栈；
- 扩展到 3D 体素填充（6/18/26 连通）；
- 对超大图像加入分块与并行策略。

## R18

`demo.py` 源码级算法流程（8 步）：

1. `main` 先调用 `run_component_case`，构造包含两块不连通同色区域的测试画布。  
2. `run_component_case` 调用 `seed_fill`，传入 `seed=(2,2)`、`fill_value=2`、`connectivity=4`。  
3. `seed_fill` 做输入校验并读取 `source_value=canvas[seed]`；若源值已等于填充值则直接返回。  
4. 初始化结果画布副本与显式栈 `stack=[seed]`，进入循环弹栈。  
5. 每次弹出 `(r,c)` 后先做越界检查；再检查当前值是否仍等于 `source_value`，否则跳过。  
6. 若满足条件则把该像素写为 `fill_value`，并把所有邻域坐标（4 或 8 连通）压入栈。  
7. 栈清空后返回填充后画布和 `FillStats`，`run_component_case` 用布尔检查验证“仅源值被改写且不连通同色块保留”。  
8. `main` 再调用 `run_diagonal_cases`，在对角链画布上分别运行 4 邻域和 8 邻域，输出 `1` 与 `5` 的对比结果以验证连通定义差异。
