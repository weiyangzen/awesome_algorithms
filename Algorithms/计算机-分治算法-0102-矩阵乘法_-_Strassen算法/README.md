# 矩阵乘法 - Strassen算法

- UID: `CS-0082`
- 学科: `计算机`
- 分类: `分治算法`
- 源序号: `102`
- 目标目录: `Algorithms/计算机-分治算法-0102-矩阵乘法_-_Strassen算法`

## R01

Strassen 算法用于计算矩阵乘法 `C = A x B`，核心目标是在大规模方阵乘法场景下，通过减少乘法次数来降低渐进时间复杂度。经典三重循环需要 `O(n^3)` 次标量乘法，而 Strassen 通过分治把 8 次子块乘法降到 7 次。

## R02

核心思想是把两个 `n x n` 矩阵切成 4 个 `n/2 x n/2` 子块，使用线性组合构造 7 个中间乘积 `M1..M7`，再组合回 `C11/C12/C21/C22`。通过“多做加减、少做乘法”换取更低的理论复杂度。

## R03

递推关系为：`T(n) = 7T(n/2) + O(n^2)`，由主定理可得 `T(n) = O(n^log2(7)) ≈ O(n^2.8074)`。其中 `O(n^2)` 来自每层子块加减和结果拼接。

## R04

输入为二维实数矩阵 `A(m x k)` 与 `B(k x n)`。为了应用 Strassen，MVP 会自动将其补零到统一的 `p x p`（`p` 为不小于 `max(m,k,n)` 的 2 的幂），运算结束后裁剪回 `m x n`。

## R05

数据结构采用 `numpy.ndarray`：
- 输入矩阵统一转换为 `float64`。
- 递归过程中通过切片拿到 4 个象限子块。
- 用临时矩阵存储 `M1..M7` 与 `C11..C22`。
- 顶层返回裁剪后的目标形状结果。

## R06

MVP 执行流程：
1. 校验输入维度与可乘条件（`A.shape[1] == B.shape[0]`）。
2. 计算补零目标边长（2 的幂）。
3. 将原矩阵拷贝到补零后的方阵左上角。
4. 进入递归 Strassen；当子问题规模不大于阈值 `leaf_size` 时退化为 `a @ b`。
5. 递归结束后将补零结果裁剪到原目标尺寸。

## R07

正确性基于两点：
- 分块矩阵乘法恒等式：`Cij` 可由子块乘法和加法表示。
- Strassen 的 7 个中间量与 `C11/C12/C21/C22` 的组合公式与标准分块乘法代数等价。

因此在精确算术下与普通矩阵乘法完全等价；浮点实现中只会引入常规舍入误差。

## R08

复杂度：
- 时间复杂度：`O(n^log2(7))`（大规模时优于 `O(n^3)`）。
- 空间复杂度：递归与临时矩阵带来额外空间，工程实现通常高于朴素算法。
- 小规模矩阵上，递归与内存分配开销可能使其慢于直接 `numpy.matmul`。

## R09

数值稳定性方面，Strassen 会比经典算法引入更多加减操作，可能放大浮点舍入误差。MVP 采用 `float64`，并在 demo 中通过 `np.allclose(..., atol=1e-9)` 校验结果与 NumPy 基线一致。

## R10

工程实现细节：
- `leaf_size` 用于控制递归停止阈值，平衡“算法收益”和“函数调用/内存开销”。
- 非方阵与非 2 的幂尺寸统一走自动 padding，避免限制输入形状。
- 代码保持单文件、少依赖，便于教学与验证。

## R11

边界与异常处理：
- 非二维输入直接抛错。
- 维度不匹配（不可乘）抛错。
- `leaf_size <= 0` 抛错。
- `1x1` 或很小矩阵会直接走基线乘法分支。

## R12

`demo.py` 内容包括：
- Strassen 递归实现 `_strassen_recursive`。
- 非 2 的幂尺寸补零 `_pad_operands`。
- 对外接口 `strassen_multiply`。
- 基准与正确性校验 `_benchmark_once` + `main`。

## R15

可扩展方向：
- 针对不同硬件自动调优 `leaf_size`。
- 引入并行化（多进程或 GPU）提升大矩阵性能。
- 针对稀疏矩阵或块稀疏场景做混合策略。
- 结合 Winograd 变体减少加法次数。

## R16

常见错误：
- 误把 Strassen 公式中的加减号写错，导致 `C11..C22` 结果偏差很大。
- 忘记在补零后裁剪输出尺寸，得到错误形状。
- 在很小矩阵上强行递归到底，性能劣化明显。

## R17

建议测试集：
- 非方阵：`(3x5) * (5x4)`。
- 非 2 的幂：`31x31`。
- 2 的幂方阵：`64x64`。
- 异常输入：不可乘形状、非二维数组、非法阈值。

MVP 已内置前 3 类正向测试与数值一致性检查。

## R18

源码级算法流（对应 `demo.py`）可拆为 8 步：
1. `strassen_multiply` 接收 `A/B`，完成二维与形状合法性校验。  
2. 调用 `_pad_operands`，把 `A/B` 扩展到统一 `2^k x 2^k` 方阵。  
3. 进入 `_strassen_recursive`；若当前块边长 `<= leaf_size`，直接执行 `a @ b` 并返回。  
4. 否则把 `a`、`b` 分割为四个象限：`a11/a12/a21/a22` 与 `b11/b12/b21/b22`。  
5. 依据 Strassen 公式递归计算七个中间积 `m1..m7`（每个都是子问题）。  
6. 用 `m1..m7` 线性组合得到 `c11/c12/c21/c22` 四个结果象限。  
7. 将四个象限拼装为当前层完整结果矩阵 `c`，逐层返回到顶层。  
8. 顶层把补零结果裁剪回原目标尺寸 `m x n`，得到最终乘积。
