# 正规方程求解

- UID: `MATH-0075`
- 学科: `数学`
- 分类: `线性代数`
- 源序号: `75`
- 目标目录: `Algorithms/数学-线性代数-0075-正规方程求解`

## R01

正规方程用于求解线性最小二乘问题。给定 \(A\in\mathbb{R}^{m\times n}\)、\(b\in\mathbb{R}^m\)，目标是：
\[
\min_x \|Ax-b\|_2^2
\]
其一阶最优条件可写为：
\[
A^TAx=A^Tb
\]
这就是“正规方程（Normal Equation）”。

## R02

核心思想：把“残差平方和最小”转成“线性方程求解”。
- 残差向量：\(r=Ax-b\)
- 目标函数：\(f(x)=\frac12\|r\|_2^2\)
- 令梯度为 0：\(\nabla f(x)=A^T(Ax-b)=0\)
- 得到正规方程：\(A^TAx=A^Tb\)

当 \(A\) 满列秩时，\(A^TA\) 对称正定，解唯一。

## R03

输入：
- 设计矩阵 \(A\)（通常 \(m\ge n\)）。
- 观测向量 \(b\)。
- 可选正则参数 \(\lambda\ge 0\)（岭正则）。

输出：
- 参数向量 \(x\)（最小二乘解或正则化解）。
- 可选中间量：\(G=A^TA\)、\(c=A^Tb\)、残差范数等。

## R04

适用前提与边界：
- 基础正规方程假设数据可近似线性建模。
- 若 \(A\) 满列秩，直接解 \(A^TAx=A^Tb\) 数学上成立且唯一。
- 若列相关严重或秩亏，\(A^TA\) 可能奇异/病态，此时应：
  - 使用 \(\lambda I\) 做岭正则，或
  - 使用 QR/SVD 路径（如 `lstsq`）作为更稳健方案。

## R05

推导（最小二乘到正规方程）：
\[
f(x)=\frac12(Ax-b)^T(Ax-b)
\]
展开并求导：
\[
\nabla f(x)=A^T(Ax-b)
\]
令 \(\nabla f(x)=0\)：
\[
A^TAx=A^Tb
\]

若加入岭正则 \( \frac{\lambda}{2}\|x\|_2^2 \)，则得到：
\[
(A^TA+\lambda I)x=A^Tb
\]
当 \(\lambda>0\) 时，系数矩阵至少正定（数值上通常更可解）。

## R06

伪代码：

```text
输入 A (m×n), b (m), 正则参数 λ>=0
检查维度: A.ndim=2, b.ndim=1, A.shape[0]==b.shape[0]
G ← A^T A
c ← A^T b
if λ > 0:
    G ← G + λ I
try:
    x ← solve(G, c)          # 优先线性方程求解
except SingularMatrixError:
    x ← pinv(G) @ c          # 兜底（最小范数/广义逆）
返回 x, G, c
```

## R07

正确性要点：
- 目标函数 \(f(x)=\frac12\|Ax-b\|_2^2\) 是凸二次函数。
- 一阶最优条件 \(A^T(Ax-b)=0\) 与正规方程等价。
- 若 \(A\) 满列秩，则 Hessian \(A^TA\) 正定，驻点即全局唯一最优解。
- 若不满秩，正规方程对应最小二乘解集合；加 \(\lambda>0\) 后可得到唯一正则化解。

## R08

复杂度（\(A\in\mathbb{R}^{m\times n}\)）：
- 构造 \(A^TA\)：\(O(mn^2)\)
- 构造 \(A^Tb\)：\(O(mn)\)
- 解 \(n\times n\) 线性系统：\(O(n^3)\)
- 总体：\(O(mn^2+n^3)\)
- 额外存储：\(O(n^2)\)（不计输入 \(A\) 本身）

当 \(m\gg n\) 时，正规方程通常计算量较低；但稳定性不如 QR/SVD。

## R09

数值稳定性关键点：
- 条件数放大：\(\kappa(A^TA)\approx\kappa(A)^2\)，误差可能显著放大。
- 共线性会让 \(A^TA\) 接近奇异，直接求解容易不稳定。
- 常见缓解手段：
  - 特征标准化（避免尺度失衡）
  - 岭正则 \((A^TA+\lambda I)\)
  - 高精度/稳健分解（QR 或 SVD）

## R10

与常见最小二乘解法对比：
- 正规方程：实现简单、速度快（尤其 \(m\gg n\)），但稳定性较弱。
- QR 分解：稳定性更好，代价中等。
- SVD：最稳健，可处理秩亏并给最小范数解，但代价最高。
- 迭代法（GD/CG/LSQR）：适合超大规模稀疏场景，不必显式形成 \(A^TA\)。

## R11

常见工程坑：
- 忽略截距项：需要给 \(A\) 显式添加全 1 列（若模型含偏置）。
- 直接在原始量纲上求解：易被尺度差异放大数值误差。
- 对秩亏矩阵强行 `solve`：会触发奇异矩阵错误或得到不可信结果。
- 把训练误差很小误判为数值稳定：需同时看条件数与泛化表现。

## R15

典型应用：
- 线性回归参数估计（小到中等规模特征维度）。
- 传感器标定与曲线拟合。
- 最小二乘型参数反演问题。
- 作为更复杂优化算法中的子问题（局部二次近似）。

## R16

建议最小测试集：

- 列重复/线性相关矩阵（秩亏鲁棒性）。
- 高条件数矩阵（数值稳定性观察）。
- 不同 \(\lambda\)（如 0、1e-6、1e-2）对残差和参数范数影响。

## R17

可扩展方向：
- 加权最小二乘：\((A^TWA)x=A^TWb\)。
- 在线/增量式更新 \(A^TA\) 与 \(A^Tb\)（流式数据）。

## R18

第三方实现的源码级流程可拆成 7 步（以 `NumPy` 路径为例）：
1. Python 层执行 `G = A.T @ A`、`c = A.T @ b`，分别触发矩阵乘和矩阵向量乘。
2. `@` 运算在 NumPy C 层调度到 BLAS（如 `dgemm`/`dgemv`）完成高效线性代数计算。
3. 调用 `np.linalg.solve(G, c)` 后，先做维度检查、dtype 统一和内存布局整理。
4. `numpy.linalg` 将请求转发到 `_umath_linalg` 的 solve 入口。
5. 该入口调用 LAPACK `xGESV`（双精度常见为 `dgesv`）：先 LU 分解，再前后代回求解。
6. 若 `xGESV` 返回奇异信息（`info>0`），NumPy 抛 `LinAlgError: Singular matrix`。
7. 本 MVP 的兜底路径使用 `pinv(G) @ c`（SVD 思路）或加 \(\lambda I\) 后再 `solve`，保证秩亏时仍可得到可用解。

因此，库函数并非黑箱：核心仍是“构造正规方程 -> 选择分解/求解内核 -> 返回最小二乘相关解”。
