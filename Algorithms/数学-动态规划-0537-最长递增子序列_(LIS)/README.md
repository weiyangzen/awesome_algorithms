# 最长递增子序列 (LIS)

- UID: `MATH-0537`
- 学科: `数学`
- 分类: `动态规划`
- 源序号: `537`
- 目标目录: `Algorithms/数学-动态规划-0537-最长递增子序列_(LIS)`

## R01

本条目实现 `Longest Increasing Subsequence (LIS)` 的最小可运行版本（MVP），目标是：
- 给定数列，求出**严格递增**子序列的最大长度；
- 返回一条可复现的 LIS（含下标与数值）；
- 使用固定样例直接运行并输出结果，不依赖交互输入；
- 用 `O(n^2)` 版本对 `O(n log n)` 版本做一致性校验，保证可解释性与正确性。

## R02

问题定义（MVP 范围）：
- 输入：一维数值序列 `a[0..n-1]`（可为空，可含重复值与负数）。
- 约束：采用**严格递增**定义，即若选择下标 `i1 < i2 < ... < ik`，则要求  
  `a[i1] < a[i2] < ... < a[ik]`。
- 输出：
  - `length`：LIS 长度；
  - `indices`：一条 LIS 的下标序列；
  - `subsequence`：对应的数值序列。

说明：LIS 通常不唯一，MVP 返回其中一条合法解即可。

## R03

动态规划与状态表达：

1. 经典 `O(n^2)` 形式：  
   `dp[i] = 以 i 结尾的 LIS 长度`，转移为  
   `dp[i] = 1 + max(dp[j])`（其中 `j < i` 且 `a[j] < a[i]`）。
2. 高效 `O(n log n)` 形式（耐心排序思想）：  
   `tails[len-1]` 记录“长度为 `len` 的递增子序列的最小可能尾值”。
3. 若处理到元素 `x`，在 `tails` 中二分找到第一个 `>= x` 的位置 `pos`：
   - 若 `pos` 在尾部，扩展长度；
   - 否则用更小尾值 `x` 覆盖该位置，提升后续延展能力。
4. 严格递增通过 `bisect_left` 实现（相等值不会扩展长度）。
5. 结合 `prev` 与 `tails_indices` 可在结束后回溯出一条具体 LIS。

## R04

算法流程（MVP）：
1. 把输入序列转为 `numpy` 一维数组并校验有限性。
2. 若序列为空，直接返回长度 `0` 与空序列。
3. 初始化：
   - `tails_values`：各长度最优尾值；
   - `tails_indices`：对应尾值在原数组中的下标；
   - `prev[i]`：下标 `i` 在所选 LIS 中的前驱位置。
4. 逐个扫描元素 `a[i]`，用二分找到 `pos = lower_bound(tails_values, a[i])`。
5. 更新 `tails_values[pos]` 与 `tails_indices[pos]`（追加或替换）。
6. 若 `pos > 0`，设置 `prev[i] = tails_indices[pos-1]`，建立回溯链。
7. 扫描结束后，从 `tails_indices[-1]` 沿 `prev` 逆向回溯得到下标序列，再反转。
8. 根据下标提取具体数值，输出 `length / indices / subsequence`。
9. 可选：用 `O(n^2)` 版本再算一遍长度并断言一致。

## R05

核心数据结构：
- `numpy.ndarray`：保存输入序列并执行有限性检查；
- `list[float] tails_values`：最优尾值数组（单调不降）；
- `list[int] tails_indices`：每个长度层对应的“尾元素下标”；
- `list[int] prev`：前驱链用于路径重建；
- `LISResult`（`dataclass`）：
  - `length: int`
  - `indices: list[int]`
  - `subsequence: list[float]`

## R06

正确性要点（简述）：
- 不变量：处理完前 `i` 个元素后，`tails_values[L-1]` 是“长度为 `L` 的递增子序列最小可能尾值”。
- 二分替换不会降低已知可达长度，只会让同长度尾值更优（更小），因此对后续扩展更有利。
- 当某元素追加在 `tails_values` 末尾时，说明发现了更长递增子序列，LIS 长度增加 1。
- 结束时 `len(tails_values)` 即全局 LIS 长度。
- 通过 `prev` 链连接“当前元素”和“上一长度层尾元素”，可还原一条合法 LIS。

## R07

复杂度：
- 主算法（耐心排序 + 二分）：
  - 时间复杂度：`O(n log n)`
  - 空间复杂度：`O(n)`
- 校验算法（经典 DP）：
  - 时间复杂度：`O(n^2)`
  - 空间复杂度：`O(n)`

## R08

边界与异常处理：
- 空序列：返回长度 `0`、空下标、空子序列；
- 单元素序列：LIS 长度为 `1`；
- 全降序：LIS 长度为 `1`；
- 含重复值：因“严格递增”约束，相同值不能连续进入 LIS；
- 非一维输入或含 `nan/inf`：抛出 `ValueError`。

## R09

MVP 取舍说明：
- 仅使用 `numpy + Python 标准库`，避免重型框架依赖；
- 不调用第三方“黑盒 LIS”接口，核心逻辑自行实现并可逐行追踪；
- 保留 `O(n^2)` 对照实现，仅用于验证与教学透明；
- 不实现以下增强项：在线更新、分布式处理、并行化、所有 LIS 枚举。

## R10

`demo.py` 职责划分：
- `to_1d_finite_array`：输入校验与一维数组转换；
- `lis_patience_strict`：`O(n log n)` 主算法，返回长度和一条具体 LIS；
- `lis_dp_quadratic`：`O(n^2)` 基线算法（校验长度与参考路径）；
- `is_strictly_increasing`：检查输出序列合法性；
- `run_case`：执行单个样例并打印结果；
- `main`：组织固定样例并完成一致性断言。

## R12

输出内容解读：
- `Input`：原始序列；
- `Fast O(n log n)`：高效算法给出的长度、下标、子序列；
- `DP O(n^2)`：基线算法给出的长度、下标、子序列；
- `Checks`：
  - `length_equal=True` 表示两算法长度一致；
  - `fast_is_strict=True` 表示快速算法输出确为严格递增序列。

若断言失败，脚本会抛出异常，方便验证流程捕捉问题。

## R14

可调参数：
- 递增定义：当前实现为“严格递增”；若改为“非严格递增”，二分应从 `bisect_left` 改为 `bisect_right`；
- 数据类型：默认转成 `float`；如需整数输出可在展示层格式化；
- 样例集合：在 `main()` 的 `cases` 字典中增删即可；
- 校验策略：可关闭/保留 `O(n^2)` 对照以平衡速度与自检强度。

## R15

方法对比：
- `O(n^2)` DP：
  - 优点：概念直观，易证明与教学；
  - 缺点：大规模数据性能较差。
- `O(n log n)` 耐心排序：
  - 优点：性能更好，工业实践常用；
  - 缺点：若不额外维护前驱信息，默认只能得到长度。
- 线段树 / 树状数组（坐标压缩）：
  - 适合值域操作更复杂场景；
  - 实现复杂度更高，不是本 MVP 重点。

## R18

源码级算法流（对应 `demo.py`，9 步）：
1. `main` 构造固定测试集并逐个调用 `run_case`。  
2. `run_case` 先调用 `lis_patience_strict` 执行 `O(n log n)` 主算法。  
3. `lis_patience_strict` 先经 `to_1d_finite_array` 把输入转成一维有限数组。  
4. 对每个元素 `a[i]`，在 `tails_values` 上做 `bisect_left` 找到替换/追加位置 `pos`。  
5. 同步更新 `tails_values[pos]` 与 `tails_indices[pos]`，维护“每种长度的最优尾值与尾下标”。  
6. 若 `pos > 0`，设置 `prev[i] = tails_indices[pos-1]`，把当前元素链接到上一层最优尾部。  
7. 扫描结束后，从 `tails_indices[-1]` 沿 `prev` 回溯并反转，得到一条完整 LIS。  
8. `run_case` 再调用 `lis_dp_quadratic` 得到基线结果，与快速算法做长度一致性检查。  
9. `run_case` 最后调用 `is_strictly_increasing` 验证递增合法性并打印两种算法结果与检查状态。
