# 雷诺数 (Reynolds Number)

- UID: `PHYS-0085`
- 学科: `物理`
- 分类: `流体力学`
- 源序号: `85`
- 目标目录: `Algorithms/物理-流体力学-0085-雷诺数_(Reynolds_Number)`

## R01

雷诺数（Reynolds Number, `Re`）是流体力学中最常用的无量纲参数之一，用于衡量“惯性力”相对“黏性力”的强弱：

`Re = (惯性效应) / (黏性效应)`

直观上：
- `Re` 小，黏性主导，流动更平滑、更稳定；
- `Re` 大，惯性主导，流动更容易出现扰动放大和湍流。

## R02

历史与工程定位：
- Osborne Reynolds 在 19 世纪通过染色实验系统研究了管内层流与湍流转捩；
- 雷诺数成为相似理论中的核心量，支撑风洞、水洞、管路设计、换热器、空气动力学等工程缩比分析；
- 在 CFD 和实验流体力学中，`Re` 常作为首要工况标签。

## R03

本条目 MVP 的目标不是求解 Navier-Stokes 方程，而是完成一个可运行、可审计的 `Re` 计算与判别流程：

1. 输入一组典型流动工况（管流 + 平板边界层）；
2. 计算每个工况的雷诺数；
3. 依据场景阈值进行流态分类；
4. 自动进行一致性与物理合理性校验并输出结果表。

## R04

使用的定义与单位：

1. 以动力黏度表示：
`Re = rho * V * L / mu`

2. 以运动黏度表示：
`Re = V * L / nu`，且 `nu = mu / rho`

符号说明：
- `rho`：密度，单位 `kg/m^3`
- `V`：特征速度，单位 `m/s`
- `L`：特征长度，单位 `m`（例如管径、特征弦长、发展长度）
- `mu`：动力黏度，单位 `Pa*s`
- `nu`：运动黏度，单位 `m^2/s`

## R05

物理意义（为什么它有效）：
- 惯性项尺度约为 `rho * V^2 / L`；
- 黏性项尺度约为 `mu * V / L^2`；
- 二者比值约化后得到 `rho * V * L / mu`。

因此雷诺数本质上是“流动自我维持/放大扰动的能力”与“黏性耗散扰动能力”的竞争指标。

## R06

本 MVP 中采用的场景化判别规则：

1. 管内流（`scenario=pipe`）：
- `Re < 2300`：`laminar`
- `2300 <= Re <= 4000`：`transitional`
- `Re > 4000`：`turbulent`

2. 平板边界层（`scenario=flat_plate`）：
- `Re_x < 5e5`：`laminar_boundary_layer`
- `Re_x >= 5e5`：`turbulent_boundary_layer`

说明：这些阈值是工程常用经验范围，具体转捩还受扰动水平、入口条件、表面粗糙度等影响。

## R07

`demo.py` 的数据构造策略：
- 使用确定性的 7 个案例覆盖低 `Re`、过渡区和高 `Re`；
- 包含甘油微通道、水管不同流速、空气管道、平板短程与长程；
- 每个案例自带 `expected_regime`，用于程序内自动断言。

这样保证脚本运行后不仅“给数值”，还会验证分类逻辑与量纲一致性是否正确。

## R08

复杂度分析（设样本数为 `n`）：
- 雷诺数计算：每条样本 `O(1)`，总计 `O(n)`；
- 分类判定：每条样本 `O(1)`，总计 `O(n)`；
- 汇总统计（groupby）：`O(n)` 级；
- 总体时间复杂度：`O(n)`；
- 空间复杂度：`O(n)`（存储记录表）。

## R09

数值稳定性与工程细节：
- 先做输入合法性检查：`rho, V, L, mu` 必须为正；
- 同时计算 `Re(mu)` 与 `Re(nu)`，用 `np.isclose` 校验公式一致性；
- 对结果执行有限性和正值检查，防止异常输入导致无意义输出；
- 用同流体同几何不同速度样本做单调性断言（速度升高 `Re` 应升高）。

## R10

技术栈：
- Python 3
- `numpy`：数值计算与断言
- `pandas`：结果表和分组汇总
- `dataclasses`：结构化案例与结果记录

说明：核心算法逻辑（公式计算、阈值判别、校验）均由源码显式实现，不依赖第三方“黑盒雷诺数接口”。

## R12

输出字段说明：
- `case`：案例名称
- `scenario`：场景类型（`pipe` / `flat_plate`）
- `rho_kg_m3`：密度
- `mu_Pa_s`：动力黏度
- `nu_m2_s`：运动黏度（由 `mu/rho` 计算）
- `V_m_s`：特征速度
- `L_m`：特征长度
- `Re`：雷诺数
- `regime`：判定流态

随后会输出 `regime` 分组统计（数量、平均 `Re`、最小值、最大值）。

## R13

正确性校验清单（`demo.py` 已实现）：
1. 公式等价校验：`rho*V*L/mu` 与 `V*L/nu` 必须一致；
2. 结果有效性校验：所有 `Re` 必须有限且大于 0；
3. 标签校验：计算得到的 `regime` 必须与预设 `expected_regime` 一致；
4. 单调性校验：水管案例中速度越高，`Re` 严格上升。

以上校验失败会直接抛异常，防止“静默错误”。

## R14

当前 MVP 的局限：
- 只做经典经验阈值分类，不直接求转捩位置或湍流统计量；
- 未显式纳入粗糙度、入口湍强、几何突变等高阶影响；
- 平板转捩阈值采用单一经验值 `5e5`，未做环境依赖修正；
- 案例规模小，定位在教学演示与快速核查，不是工业级工况库。

## R15

可扩展方向：
- 增加更多几何场景（环隙流、外绕圆柱、喷流等）；
- 引入温度依赖黏度模型（`mu(T)`）实现热工况耦合；
- 增加不确定性传播（输入测量误差到 `Re` 置信区间）；
- 对接实验/CFD 数据文件，实现批量自动评估与报告。

## R16

边界与异常处理建议：
- 当 `mu` 极小导致 `Re` 极大时，需结合实际可压缩性/马赫数条件共同判断；
- 当 `L` 选取不当（例如不代表主导尺度）时，`Re` 解释会失真；
- 若场景未知，本实现会走回退规则（含 `creeping/laminar/transitional/turbulent`），应在工程中优先提供明确场景标签。

## R17

最小测试建议：
1. 单元测试：已知输入直接验证 `Re` 数值；
2. 阈值测试：在 `2300`、`4000`、`5e5` 附近构造边界样本；
3. 鲁棒性测试：非法输入（负密度、零黏度等）应触发异常；
4. 回归测试：固定案例运行结果在代码变更后保持一致。

## R18

`demo.py` 源码级流程（8 步）：

1. `build_demo_cases()` 构建 7 个确定性 `FlowCase`，覆盖管流和平板边界层两类工况。  
2. `main()` 对每个案例调用 `evaluate_case()`，先由 `validate_case()` 检查 `rho/V/L/mu` 的正值约束。  
3. `evaluate_case()` 先算 `nu = mu / rho`，再分别调用 `reynolds_number_from_mu()` 与 `reynolds_number_from_nu()`。  
4. 使用 `np.isclose` 对两种公式进行一致性校验，确保不是把公式当黑箱直接套用。  
5. `classify_reynolds()` 按 `scenario` 分支应用阈值，输出 `laminar / transitional / turbulent` 或边界层标签。  
6. 每条结果写入 `ReynoldsRecord`，`records_to_dataframe()` 汇总成结构化表格。  
7. `run_checks()` 执行全局断言：数量一致、`Re` 有限且正、预期标签一致、水管样本单调性成立。  
8. `main()` 打印案例表与 `groupby` 统计摘要，全部校验通过后输出 `All checks passed.`。  

说明：第三方库仅承担数组/表格基础操作；雷诺数计算、分类逻辑与校验流程均在源码中逐步展开。
