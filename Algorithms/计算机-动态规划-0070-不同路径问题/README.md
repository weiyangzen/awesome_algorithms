## R01

不同路径问题（Unique Paths）是动态规划入门经典题：
- 在 `m x n` 网格中，从左上角 `(0,0)` 走到右下角 `(m-1,n-1)`；
- 每一步只能向右或向下；
- 求一共有多少条不同路径。

本目录提供最小可运行 MVP，强调“可解释 + 可校验”。

## R02

MVP 输入输出定义：
- 输入：
  - `m`：网格行数，正整数；
  - `n`：网格列数，正整数。
- 输出：
  - `paths`：路径总数（非负整数）。

约束：
- `m >= 1` 且 `n >= 1`；
- `demo.py` 无交互输入，直接运行固定样例和随机回归。

## R03

二维 DP 状态定义：
- `dp[i][j]` 表示到达格子 `(i,j)` 的路径数。

初值：
- 第一行任意格子只能一直向右到达，所以 `dp[0][j] = 1`；
- 第一列任意格子只能一直向下到达，所以 `dp[i][0] = 1`。

## R04

状态转移方程：

`dp[i][j] = dp[i-1][j] + dp[i][j-1]`（`i>0, j>0`）

含义：
- 到 `(i,j)` 的最后一步要么来自上方 `(i-1,j)`，要么来自左方 `(i,j-1)`；
- 两类路径互斥且完备，所以直接相加。

## R05

最优子结构 / 子问题拆解性质：
- 虽然本题不是“最优化”而是“计数”，但同样具备可分解结构；
- 大问题 `f(i,j)` 由两个更小子问题 `f(i-1,j)` 与 `f(i,j-1)` 组成；
- 子问题结果可复用，避免指数级重复计算。

因此可以用 DP 从小到大填表。

## R06

一维滚动数组优化：
- 观察到 `dp[i][j]` 只依赖“当前行左侧”和“上一行同列”；
- 可把二维表压缩为一维 `row[j]`：
  - 初始化全 1（第一行）；
  - 逐行更新：`row[j] = row[j] + row[j-1]`。

含义对应：
- 更新前 `row[j]` 是“上方值”；
- `row[j-1]` 是“左方值”；
- 更新后 `row[j]` 即当前位置路径数。

## R07

复杂度分析：
- 二维 DP：
  - 时间复杂度 `O(mn)`；
  - 空间复杂度 `O(mn)`。
- 一维 DP（滚动优化）：
  - 时间复杂度 `O(mn)`；
  - 空间复杂度 `O(n)`。
- 组合数公式法：
  - 时间复杂度近似 `O(min(m,n))`（按实现而定）；
  - 空间复杂度 `O(1)`。

## R08

`demo.py` 结构：
- `validate_grid_shape`：校验 `m,n` 为正整数；
- `unique_paths_dp_2d`：二维 DP 基线实现；
- `unique_paths_dp_1d`：一维滚动 DP 主实现；
- `unique_paths_combinatorial`：组合数公式实现（数学对照）；
- `run_case`：执行样例并断言三种方法一致；
- `randomized_cross_check`：随机回归测试；
- `main`：组织固定样例并执行全流程。

## R09

核心接口说明：
- `unique_paths_dp_1d(m, n) -> int`
  - 推荐主接口，复杂度均衡、实现简洁。
- `unique_paths_dp_2d(m, n) -> int`
  - 教学友好的完整状态表版本。
- `unique_paths_combinatorial(m, n) -> int`
  - 公式法：
  `C(m+n-2, m-1)` 或 `C(m+n-2, n-1)`。

## R12

输出解读：
- 每个 case 会打印：
  - 输入维度 `m,n`；
  - `dp_2d` 结果；
  - `dp_1d` 结果；
  - `comb` 结果；
  - `expected`（若设置）。
- 若任意两种方法不一致，程序会抛出 `AssertionError`。

## R13

随机回归策略：
- 默认 `250` 轮；
- 随机采样 `m,n ∈ [1, 20]`；
- 每轮断言：
  - `dp_2d == dp_1d == comb`。

意义：
- 固定样例覆盖典型边界；
- 随机样例提升对实现细节错误（初始化、循环边界等）的发现概率。

## R14

边界与异常处理：
- `m <= 0` 或 `n <= 0`：抛 `ValueError`；
- `m,n` 非整数（如 `3.5`、字符串）：抛 `ValueError`；
- 超大尺寸时，结果会非常大：
  - Python `int` 可自动扩展，不会像定长整数那样溢出；
  - 但计算时间会随 `mn` 增长。

## R18

`demo.py` 源码级算法流程（8 步）：

1. `main` 定义固定样例，并逐个调用 `run_case`。  
2. `run_case` 先调用 `unique_paths_dp_2d`：创建 `m x n` 的 DP 表并初始化第一行、第一列为 `1`。  
3. `unique_paths_dp_2d` 双层循环填表，对每个内部格子执行 `dp[i][j] = dp[i-1][j] + dp[i][j-1]`，最终返回 `dp[m-1][n-1]`。  
4. `run_case` 再调用 `unique_paths_dp_1d`：创建长度 `n` 的一维数组并初始化为 `1`。  
5. `unique_paths_dp_1d` 逐行更新，内部执行 `row[j] = row[j] + row[j-1]`，结束后返回 `row[n-1]`。  
6. `run_case` 同时调用 `unique_paths_combinatorial`，用 `comb(m+n-2, m-1)` 计算数学真值。  
7. `run_case` 断言三种结果相同，并在给定 `expected` 时再做一次期望值断言。  
8. 所有固定样例通过后，`main` 调用 `randomized_cross_check` 进行多轮随机维度对拍，确保实现稳定。
