## R01

Tarjan 离线 LCA（Lowest Common Ancestor）用于一次性回答一批树上最近公共祖先查询。
它把“树 DFS 遍历”和“并查集连通归并”结合起来，在离线场景下可以用接近线性的代价得到所有查询结果。

## R02

问题定义（本任务 MVP 版本）：

- 输入：
1. 节点数 `n`（编号 `0..n-1`）
2. 无向树/森林边集 `edges=[(u,v), ...]`
3. 查询列表 `queries=[(u,v), ...]`
- 输出：
1. `answers[i]` 为第 `i` 个查询的 LCA 节点编号
2. 若 `u` 与 `v` 不在同一连通分量，返回 `None`

## R03

Tarjan 离线 LCA 的核心思想：

- DFS 到节点 `u` 时，先把 `u` 视为当前并查集代表的祖先
- 每处理完一个子树 `v`，执行 `union(u, v)`，再把合并后集合祖先设回 `u`
- 当节点 `u` 标记为“已访问完成”后，若查询 `(u, x)` 的另一端 `x` 也已完成访问，则答案为 `ancestor[find(x)]`

这个结论成立的原因是：在 DFS 回溯语义下，`find(x)` 对应的是“已经封闭处理的子树代表”，其记录的祖先正是当前可见的最低公共祖先。

## R04

MVP 的关键数据结构：

- 邻接表 `adj`：存树/森林
- 查询桶 `query_bucket[u]`：挂在每个节点上的查询 `(other, query_id)`
- 并查集 `parent/rank`：维护已回溯子树的集合合并
- `ancestor[rep]`：并查集代表节点 `rep` 对应的“当前祖先”
- `visited[u]`：节点是否已完成 DFS（后序时刻）
- `component[u]`：连通分量编号，用于跨分量查询返回 `None`

## R05

为什么该算法适合“离线批量查询”：

- 每条查询只在其两个端点被扫描时触发常数次判断
- 不需要为每个查询单独向上跳父节点
- 不需要像倍增 LCA 那样预处理 `O(n log n)` 的祖先表

因此当查询集合提前已知、且只需要一次性求解时，Tarjan 离线通常是很实用的方案。

## R06

算法流程概览：

1. 构建邻接表与查询桶。
2. 初始化并查集、`ancestor`、`visited`、`component`、`answers`。
3. 对每个未访问节点启动 DFS（支持森林）。
4. 进入 `dfs(u,p)`：先记录 `component[u]`，并令 `ancestor[find(u)] = u`。
5. 遍历子节点 `v`（`v != p`）：递归 `dfs(v,u)` 后执行 `union(u,v)`，再置 `ancestor[find(u)] = u`。
6. 子树处理结束后标记 `visited[u] = True`。
7. 扫描 `u` 的查询桶：若另一端 `x` 已访问且同分量，答案是 `ancestor[find(x)]`，否则为 `None`。
8. 所有 DFS 完成后返回答案数组。

## R07

关键不变量：

- 若 `visited[u] == True`，说明 `u` 的整棵 DFS 子树已经处理完成。
- 并查集中的每个集合都对应“一个已回溯封闭的子树块”。
- `ancestor[find(z)]` 始终是该集合在当前 DFS 路径下可代表的最高已知祖先节点。
- 查询仅在“另一端已访问完成”时求值，避免读到未稳定状态。

## R08

正确性直觉：

- LCA 的本质是“两条根路径的最后公共节点”。
- DFS 从叶向上回溯时，子树合并顺序恰好对应路径封闭顺序。
- 当 `(u, x)` 两端都已访问完成时，`x` 所在并查集代表的 `ancestor` 指向当前已封闭结构中的最近公共祖先。

因此每个查询在第一次满足“另一端已访问”时即可得到正确且最终的答案。

## R09

时间复杂度：`O((n + q) * α(n))`。

- 构图与挂查询：`O(n + q)`（树边数按 `n-1` 计）
- DFS 扫边：`O(n)`
- 并查集 `find/union` 摊还：`α(n)`
- 每条查询被处理常数次：`O(q * α(n))`

其中 `α(n)` 为反 Ackermann 函数，增长极慢，可视为近常数。

## R10

空间复杂度：`O(n + q)`。

- 邻接表与查询桶：`O(n + q)`
- 并查集和辅助数组：`O(n)`
- 结果数组：`O(q)`

## R11

边界与异常处理：

- `n=0`：返回空答案。
- 查询端点越界：抛出 `ValueError`。
- 孤立点查询 `(u,u)`：LCA 为 `u`（同分量条件下）。
- 跨连通分量查询：返回 `None`。
- 输入若不是树而含回边，本实现用 `seen` 防止重复 DFS，但语义上仍应视作“面向树/森林”。

## R12

实现中的高频错误：

- `union(u,v)` 后忘记刷新 `ancestor[find(u)] = u`。
- 在“发现节点”而不是“完成节点”时标记 `visited`，导致提前回答错误。
- 查询处理时误用 `ancestor[u]` 而非 `ancestor[find(x)]`。
- 未做连通分量判断，导致跨分量误给出伪 LCA。

## R13

与其他 LCA 方法对比：

- 倍增（Binary Lifting）：
1. 预处理 `O(n log n)`，单次查询 `O(log n)`
2. 适合在线多次查询
- Euler Tour + RMQ：
1. 预处理可做到 `O(n log n)`（或更复杂优化）
2. 单次查询可达 `O(1)`/`O(log n)`
- Tarjan 离线：
1. 需要提前拿到全部查询
2. 总复杂度近线性，代码短且适合批处理

## R17

工程化扩展建议：

- 大规模深树可改成显式栈 DFS，规避递归深度限制。
- 若要在线查询，可改用倍增 LCA 或树链剖分。
- 若输入来自数据库/文件流，可将查询桶构建与结果落盘封装成批处理管线。
- 在竞赛场景中，可用紧凑数组替代对象封装降低常数开销。

## R18

`demo.py` 中 Tarjan 离线 LCA 的源码级流程（9 步）：

1. `tarjan_offline_lca` 先校验 `n` 与边/查询端点，构建邻接表 `adj`。
2. 把每个查询 `(u,v)` 以 `(other, query_id)` 形式双向挂入 `query_bucket[u]` 与 `query_bucket[v]`。
3. 初始化并查集 `DSU`、`ancestor`、`visited`、`seen`、`component`、`answers`。
4. 外层遍历每个节点，对未访问节点启动新的 DFS 连通分量，并分配 `comp_id`。
5. 在 `dfs(u,p,comp_id)` 入口写入 `component[u]=comp_id`，并令 `ancestor[find(u)] = u`。
6. 对每个子节点 `v` 递归处理；回溯时执行 `union(u,v)`，随后把合并集合的祖先重设为 `u`。
7. 子树全部处理后设置 `visited[u]=True`，表示 `u` 进入可回答查询的稳定状态。
8. 扫描 `u` 的查询桶：若另一端 `x` 已访问且 `component[x]==component[u]`，答案写为 `ancestor[find(x)]`；若不同分量，写 `None`。
9. 全部 DFS 结束后返回 `answers`；`main` 中再用朴素法复核并打印对照结果。
