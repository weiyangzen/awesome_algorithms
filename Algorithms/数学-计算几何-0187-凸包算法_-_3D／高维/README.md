## R01

凸包（Convex Hull）问题：给定点集 `P ⊂ R^d`，求最小凸集 `conv(P)` 包含全部点。  
在 3D/高维中，凸包可用两类等价表示：

- V-representation：由顶点集合生成；
- H-representation：由半空间不等式 `A x + b <= 0` 描述。

本条目 MVP 目标：

- 在 3D 与 6D 上构建凸包；
- 输出顶点数、面数、半空间违反量等可验证指标；
- 保持实现可读、可审计、可直接运行。

## R02

历史与算法脉络（简要）：

- 2D 经典凸包算法包括 Graham Scan、Jarvis March；
- 3D/高维常见方法包括 Incremental、Divide-and-Conquer、Beneath-Beyond、Quickhull；
- 本条目为了“最小可运行 + 无外部几何依赖”，采用小规模枚举支撑超平面的实现路线。

## R03

3D/高维凸包的实际价值：

- 3D 几何：点云外包、碰撞边界、场景包围体；
- 高维数据：线性可分边界、异常值几何外壳、可行域外层近似；
- 优化建模：在顶点表示和不等式表示之间建立桥梁。

与 2D 相比，难点主要来自组合复杂度上升与退化情况增多。

## R04

本 MVP 的复杂度（枚举方案）：

- 对 `n` 个点、`d` 维空间，需枚举 `C(n, d)` 个候选超平面；
- 每个候选需做一次 SVD（求法向）与一次全点同侧检查，故总体约为  
  `O(C(n, d) * (d^3 + n*d))`；
- 这是“教学/验证型”复杂度，不适合大规模场景。

因此 demo 选取了小规模点集，以保证可运行性。

## R05

几何与代数基础：

- 若一个超平面满足所有点都在同侧（含边界），它是支撑超平面候选；
- 若点集对该超平面取等号的点数不少于 `d`，可形成一个面；
- 所有有效面的交就是凸包的 H-representation；
- 凸包顶点可由所有面的“活跃点索引并集”得到。

## R06

本条目问题设定：

- 输入：脚本内部生成的有限点集 `P`；
- 输出：
  - 面集合（以“共面点索引集”表示）；
  - 半空间方程矩阵 `equations`；
  - 顶点索引集合；
  - 校验指标（`max_violation`、`random_inside_rate`）。

高维演示固定为 `d=6`，覆盖“3D 以上”的基本流程。

## R07

`demo.py` 的输入输出约定（无交互）：

- 输入数据：
  - 3D：6 个八面体极点 + 内部随机点；
  - 6D：12 个高维交叉多胞体极点 `±e_i` + 内部随机点。
- 输出报告：
  - `dimension`、`n_points`、`n_vertices`、`n_facets`；
  - `max_violation`（所有点对所有面不等式的最大超界值）；
  - `random_inside_rate`（包围盒内随机采样的“落入凸包比例”）；
  - `combinations_tested/kept`（组合枚举统计）。

## R08

数据构造动机：

- 注入极点可保证外壳形状稳定、减少退化；
- 保留内部随机点可模拟真实噪声背景；
- 使用固定随机种子确保结果可复现。

该设计能在小规模点数下稳定展示 3D/高维凸包过程。

## R09

正确性检查策略：

- 半空间一致性：验证所有输入点满足 `A x + b <= tol`；
- 结构合法性：要求 `n_vertices >= d + 1` 且 `n_facets >= d + 1`；
- 报告关键指标：`max_violation` 越接近 0 越好；
- 使用断言在运行时自动验收。

这是一种工程可执行的后验检查方案。

## R12

输出字段说明：

- `dimension`：空间维度；
- `n_points`：输入点总数；
- `n_vertices`：识别出的凸包顶点数；
- `n_facets`：识别出的凸包面数；
- `max_violation`：最大半空间违反量（应接近 0 或为微小负数）；
- `random_inside_rate`：包围盒随机点中落在凸包内的比例（用于诊断，不是精确体积）；
- `combinations_tested`：枚举过的候选组合总数；
- `combinations_kept`：最终保留的有效面数量。

## R13

边界与鲁棒性：

- 若 `n_points <= d`，无法形成非退化 `d` 维凸包；
- 若存在大量共面/共线，候选面去重会更敏感；
- 浮点误差会带来接近零的正负波动，需要容忍阈值；
- 高维下 `C(n,d)` 增长快，计算会迅速变慢。

本实现通过 rank 阈值、同侧阈值、共面阈值来做稳定化处理。

## R15

工程实现建议：

- 先做去重与尺度规范化，减少数值病态；
- 限制点数或先降维，再做高维凸包；
- 记录 `max_violation` 和组合数量，便于性能/稳定性回归；
- 若需要工业级性能，可切换到成熟库并保留同样的后验校验。

## R16

前驱与后继关系：

- 前驱：线性代数（SVD、法向量）、凸分析（半空间交）；
- 后继：
  - 可行域边界分析；
  - 碰撞包围体构造；
  - 作为近邻搜索/异常检测的几何先验。

凸包常作为更复杂几何管线的基础模块。

## R17

方案对比：

- 本条目（枚举支撑超平面）：
  - 优点：源码透明、可审计、依赖少；
  - 缺点：组合爆炸明显，仅适合小规模。
- Quickhull/Incremental（成熟实现）：
  - 优点：性能更好，适用规模更大；
  - 缺点：实现复杂，调试和维护难度更高。
- 近似凸包方法：
  - 优点：可处理超大规模；
  - 缺点：边界精度受采样策略影响。

MVP 阶段优先选择可解释与可运行性。

## R18

`demo.py` 的源码级算法流程（8 步）：

1. `make_3d_points()` / `make_high_dim_points()` 构造“极点 + 内点”数据，保证外壳稳定且可复现。  
2. 在 `compute_convex_hull_enumeration()` 中枚举所有 `d` 点组合（共 `C(n,d)` 个候选超平面）。  
3. 对每个候选组合，用 SVD 求超平面法向量；若仿射秩不足 `d-1`，直接丢弃。  
4. 计算全部点到候选面的符号距离；仅保留“所有点同侧（含边界）”的支撑超平面。  
5. 把距离接近 0 的点收集为该面的活跃点集；活跃点数 `< d` 的候选无效。  
6. 以“活跃点索引元组”做去重，得到唯一面集合 `facets` 和对应 `equations`。  
7. 将全部面上的点索引并集作为凸包顶点 `vertices`，并统计候选/保留组合数量。  
8. `run_case()` 调用 `max_halfspace_violation()` 与随机采样检查，做断言并输出最终报告。  

这套流程完全在源码中显式实现，没有把“凸包构建”委托给外部黑箱函数。
