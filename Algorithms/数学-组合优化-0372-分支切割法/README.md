## R01

分支切割法（Branch-and-Cut）是求解整数规划的一类经典框架：
- 用分支定界（Branch-and-Bound）维护搜索树与上下界；
- 在节点 LP 松弛上动态加入割平面（Cuts）收紧可行域；
- 通过“割平面增强 + 分支剪枝”协同逼近整数最优解。

它不是单一公式，而是一套“节点求松弛 -> 分离割 -> 必要时分支”的迭代机制。

## R02

本目录 MVP 选择 `0-1` 背包问题作为载体来演示分支切割主流程：

- 决策变量：`x_i ∈ {0,1}`，表示是否选物品 `i`；
- 目标：最大化 `Σ p_i x_i`；
- 约束：`Σ w_i x_i <= C`。

背包模型小而完整，适合清晰展示：
- LP 松弛产生分数解；
- cover cut 能显式切掉部分分数解；
- 仍未整数时继续分支。

## R03

数学模型（最大化）如下：

- `max Σ_i p_i x_i`
- `s.t. Σ_i w_i x_i <= C`
- `x_i ∈ {0,1}`

节点 LP 松弛把整数条件放宽为 `0 <= x_i <= 1`，用于提供上界。分支时固定某些变量为 `0/1`，形成子问题。

## R04

为什么要“分支 + 切割”结合：

- 只做分支定界：树可能很深，节点数较多；
- 只做割平面：有时仍无法直接得到整数解；
- 二者结合：割平面先压缩分数可行域，分支树更小，剪枝更早。

MVP 里采用的是全局 cover cuts（对所有节点都有效），这在背包场景下实现简单且便于教学验证。

## R05

`demo.py` 的节点 LP 求解策略：

- 不依赖外部 LP 求解器黑盒；
- 在代码中实现了一个最小可用的原始单纯形（`simplex_maximize`）；
- 每个节点把“已固定变量”代回约束，得到剩余自由变量的 LP；
- 目标值作为该节点上界，用于剪枝判定。

这样可以完整追踪“LP 松弛求解”在源码中的每一步。

## R06

割平面采用背包常见的 cover inequality：

- 若集合 `C` 满足 `Σ_{i∈C} w_i > Capa`，称为一个 cover；
- 则对 `0-1` 背包有有效不等式 `Σ_{i∈C} x_i <= |C|-1`。

分离过程（启发式）：
- 按 `x` 值、`x*w`、`w` 三种顺序构造候选 cover；
- 做一次“最小化 cover”收缩；
- 若当前 LP 解违反该不等式，则加入全局 cuts 集。

## R07

分支与剪枝规则：

- 深度优先 DFS 遍历节点；
- 若节点 LP 不可行，直接剪枝；
- 若节点上界 `<=` 当前最优整数值，按界剪枝；
- 若 LP 解已整数，更新 incumbent；
- 否则选一个最接近 `0.5` 的分数变量做二分支（`x_j=0/1`）。

## R08

正确性直觉（MVP 层面）：

1. 节点 LP 松弛给出该子树可达到目标的上界；
2. cover cut 是对原整数可行域有效的不等式，不会切掉真实整数最优解；
3. 分支把可行域划分成互斥子域并最终覆盖所有整数赋值；
4. 剪枝仅在“不可行或不可能更优”条件下触发；
5. 因而最终保留的最优整数解是全局最优。

`demo.py` 还用小规模暴力枚举做 oracle 交叉验证，进一步保障实现正确性。

## R09

复杂度（本 MVP）主要受三部分影响：

- 搜索树规模：最坏可到 `O(2^n)`；
- 每节点 LP：使用小型单纯形，复杂度与约束/变量规模相关；
- 分离割次数：受 `max_cuts` 和 `max_cut_rounds_per_node` 限制。

由于示例规模小（`n=8`），运行时间在普通 CPU 上很快，适合演示与回归测试。

## R10

数值与鲁棒性处理：

- 统一容差 `EPS = 1e-9`；
- 单纯形 pivot 时跳过接近 0 的主元；
- 整数判定使用 `int_tol`，避免浮点抖动；
- 对 `rhs < -EPS` 的约束及时判 infeasible。

MVP 目标是“可运行且可读”，不是工业级数值鲁棒 LP 内核。

## R11

`demo.py` 内置示例：

- `profits = [20, 18, 14, 12, 10, 8, 7, 6]`
- `weights = [11, 10, 8, 7, 6, 5, 4, 3]`
- `capacity = 23`

该实例会触发“加割 + 分支”，最终最优值为 `41`（脚本会自动与暴力枚举结果对比并断言一致）。

## R12

关键参数：

- `max_nodes`：最大搜索节点数；
- `max_cuts`：全局最多添加多少条割平面；
- `max_cut_rounds_per_node`：每节点最多做几轮分离；
- `int_tol`：整数判定阈值。

调参建议：
- 若希望更快：减小 `max_cuts`/`max_nodes`；
- 若希望更强：适度提高 `max_cuts`，但节点 LP 规模会变大。

输出解读：

- `best_solution`：对应 `0/1` 选择向量；

## R18

`demo.py` 源码级流程（8 步）：

1. `main` 构造固定背包实例，调用 `branch_and_cut_knapsack`。  
2. `branch_and_cut_knapsack` 从根节点开始 DFS，节点状态是“部分变量固定表”。  
3. 每个节点调用 `solve_node_lp_relaxation`：将固定变量代回模型，形成自由变量 LP。  
4. LP 由 `simplex_maximize` 求解（单纯形 tableau），返回节点上界与分数解。  
5. `find_violated_cover_cut` 在当前分数解上做 cover 分离；若发现违反不等式，则把 cut 加入全局 cuts 并重解当前节点 LP。  
6. 若 LP 解已整数，更新 incumbent；若上界不优于 incumbent，则剪枝。  
7. 若仍分数，选最接近 `0.5` 的变量二分支（`x_j=0` 与 `x_j=1`），压栈继续搜索。  
8. 搜索结束后回到 `main`，再用 `brute_force_knapsack` 做 oracle 对照，执行断言并打印 `PASS`。

该流程完整展开了“LP 松弛上界 + 割平面收紧 + 分支定界”的分支切割闭环，且关键步骤都能在源码中逐行追踪。
