# 打家劫舍问题

- UID: `CS-0055`
- 学科: `计算机`
- 分类: `动态规划`
- 源序号: `72`
- 目标目录: `Algorithms/计算机-动态规划-0072-打家劫舍问题`

## R01

本题是经典线性动态规划：
- 一条街上有若干房屋，`nums[i]` 表示第 `i` 间房可偷金额；
- 不能偷相邻两间房；
- 目标是在不触发警报的约束下，最大化总金额。

MVP 不仅输出“最大金额”，还输出一组可执行的偷窃下标方案，便于验证结果可解释。

## R02

形式化定义：
- 输入：长度为 `n` 的非负整数数组 `nums`。
- 约束：若选择房屋下标集合 `S`，则对任意相邻下标 `i` 和 `i+1`，不能同时属于 `S`。
- 输出：
  - `best_total`：最大可偷金额；
  - `indices`：一个满足约束的最优下标序列（递增且不相邻）；
  - `values`：`indices` 对应的金额。

## R03

状态设计（前缀 DP）：
- 定义 `dp[i]`：只考虑前 `i` 间房（下标 `0..i-1`）时的最大金额。
- 边界：`dp[0] = 0`。
- 转移：
  - 不偷第 `i-1` 间：收益 `dp[i-1]`；
  - 偷第 `i-1` 间：收益 `dp[i-2] + nums[i-1]`（`i>=2` 时）。
- 递推式：
  `dp[i] = max(dp[i-1], dp[i-2] + nums[i-1])`。

## R04

正确性直觉：
- 任意最优解对最后一间房只有两种选择：偷或不偷；
- 若偷最后一间，则前一间必须不偷，问题规模缩成前 `i-2` 间；
- 若不偷最后一间，问题规模缩成前 `i-1` 间；
- 两种互斥决策覆盖全部可行解，因此取二者最大值即可。

## R05

复杂度：
- 主 DP（带路径重建）：
  - 时间复杂度 `O(n)`；
  - 空间复杂度 `O(n)`（`dp` 表 + 重建过程）。
- 值优化版 DP：
  - 时间复杂度 `O(n)`；
  - 空间复杂度 `O(1)`（只保留 `dp[i-1]` 与 `dp[i-2]`）。
- 小规模暴力校验：指数复杂度，仅用于测试正确性，不用于生产计算。

## R06

`demo.py` 的实现包含三条计算路径：
- `house_robber_dp_trace`：主实现，返回金额 + 方案路径；
- `house_robber_dp_optimized_value`：空间优化版本，只返回最优金额；
- `house_robber_bruteforce`：小规模精确枚举，用于回归测试交叉验证。

三者结果在脚本中会自动断言一致。

## R07

数据与输入校验策略：
- 输入必须是一维序列；
- 必须是有限数值（拒绝 `nan` / `inf`）；
- 金额要求为非负整数（浮点若不是整数值会报错）；
- 空数组合法，输出最优金额 `0`。

本校验由 `to_1d_nonnegative_int_array` 统一完成，避免重复逻辑。

## R08

路径重建策略：
- 主循环先构建 `dp`；
- 再从后向前回溯：
  - 若 `dp[i-1] >= dp[i-2] + nums[i-1]`，说明可选择“跳过当前房”；
  - 否则选择“偷当前房”，并向前跳两步；
- 回溯得到的下标是逆序，最后反转得到递增序列。

平局时优先跳过（`>=`）用于保证输出稳定、可复现。

## R09

边界条件覆盖：
- `[]` -> `0`；
- `[x]` -> `x`；
- 全 `0` 数组 -> `0`；
- 存在多个最优方案（如 `[2,2,2,2]`）时输出稳定方案；
- 一般混合样例（大小交错）可验证递推有效性。

## R10

验证策略（脚本自动执行）：
- 固定样例：展示可读输出，便于人工核对；
- 随机回归：
  - 随机生成小规模数组；
  - 对比主 DP、O(1) DP、暴力枚举结果是否一致；
  - 同时检查返回方案是否严格递增且无相邻下标。

## R12

`demo.py` 输出字段说明：
- `dp_trace`：主 DP 的总金额与选中房屋；
- `dp_optimized`：O(1) 空间版金额；
- `bruteforce`：小规模精确最优解；
- `checks`：
  - `valid_plan`：下标合法且不相邻；
  - `traced_sum_match`：方案金额求和是否等于输出总额；

## R13

MVP 取舍：
- 优先实现“线性街道（House Robber I）”这一核心版本；
- 不引入复杂工程框架，仅依赖 `numpy + 标准库`；
- 不调用第三方黑盒求解器，所有核心递推逻辑均在源码中可追踪；
- 暂不覆盖环形街道、树形街道等变体（可在后续扩展）。

## R14

与常见变体关系：
- House Robber II（环形）：可拆成两次线性 DP（排除首或排除尾）；
- House Robber III（树形）：转为树 DP（每个节点考虑“偷/不偷”）；
- 本条目实现的一维线性版本是这些扩展题的基础模块。

## R15

可扩展方向：
- 增加“返回所有最优方案”或“字典序最小方案”的策略控制；
- 引入性能基准（`n` 从 `1e3` 到 `1e7`）评估线性实现常数；
- 扩展到环形与树形版本并复用输入校验和测试框架；
- 增加属性测试（property-based testing）提升边界覆盖。

## R16

工程注意点：
- 不要把“最大金额计算”与“路径重建”混在同一层状态里，先算值再回溯更稳；
- 平局策略必须固定，否则不同运行可能输出不同方案；
- 暴力枚举仅用于小规模验证，避免误用在大输入；
- 输入校验应集中在单函数，避免多处逻辑漂移。

## R18

`demo.py` 源码级算法流程（8 步）：
1. `main` 构造固定测试用例，逐个调用 `run_case`。  
2. `run_case` 首先调用 `house_robber_dp_trace`，以 `dp[i]=max(dp[i-1], dp[i-2]+nums[i-1])` 计算最优金额。  
3. `house_robber_dp_trace` 完成 `dp` 表后，从尾部回溯决定每个位置“偷/不偷”，生成一组合法下标方案。  
4. `run_case` 再调用 `house_robber_dp_optimized_value`，使用两个滚动变量复算最优金额。  
5. 对小规模输入，`run_case` 额外调用 `house_robber_bruteforce`（记忆化 DFS）得到精确最优值作对照。  
6. `is_valid_plan` 校验方案下标有序、范围合法且不存在相邻房屋。  
7. `run_case` 断言三种求值结果一致、方案金额求和与总金额一致，并打印可读结果。  
8. 所有固定样例通过后，`randomized_regression` 再做 200 轮随机小规模交叉校验，确认实现稳定正确。
