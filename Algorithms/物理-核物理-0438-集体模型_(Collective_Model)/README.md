# 集体模型 (Collective Model)

## R01

本条目实现核结构“集体模型”的最小可运行 MVP，覆盖 Bohr-Mottelson 框架中最常用的两个极限：
- 转动极限（deformed rotor）：`E(J)=E0 + A J(J+1)`。
- 振动极限（spherical vibrator）：`E(n)=E0 + ħω n`。

MVP 目标不是做高精度核数据库拟合，而是给出“可执行、可解释、可校验”的算法流程：输入一组低激发能级，比较两类集体模型拟合效果并输出判别诊断。

## R02

集体模型用于描述“核子单粒子运动之外”的整体自由度（表面形变、转动、振动）。在偶偶核低能区，经常使用经验指标
`R4/2 = E(4+)/E(2+)` 区分结构类型：
- 近振动核：`R4/2 ~ 2.0`
- 近转动核：`R4/2 ~ 3.33`

因此本实现将“能级拟合 + R4/2 指标”结合成一个最小判别流水线。

## R03

MVP 主方程：

1. 刚性转子（rigid rotor）
\[
E(J)=E_0 + A\,x,\quad x=J(J+1)
\]

2. 含离心伸缩项（可选）
\[
E(J)=E_0 + A\,x - B\,x^2
\]

3. 振动模型（谐振近似）
\[
E(n)=E_0 + \hbar\omega\, n,\quad n=J/2\;(\text{even-}J\ \text{yrast 近似})
\]

4. 集体性指标
\[
R_{4/2}=E(4^+)/E(2^+)
\]

## R04

`demo.py` 使用内置样本（无外部文件、无交互）：
- `168Er-like (deformed rotor)`：近转动能带样本。
- `112Cd-like (quadrupole vibrator)`：近振动样本。

脚本输出：
- 每个核素的逐能级观测值与三种模型预测值表格；
- 诊断信息（`R4/2`、按阈值分类、各模型 RMSE）；
- 汇总表（模型参数、`R^2`、相对转动惯量指标等）。

## R05

变量与单位说明：
- `J`：自旋量子数（偶偶核能带常取 `0,2,4,...`）
- `E_obs_keV`：观测（或示例）激发能，单位 `keV`
- `A, B`：转动参数，单位分别约为 `keV` 与 `keV`
- `hw_kev`：振动量子能量 `ħω`，单位 `keV`
- `E0`：带头能量偏置，单位 `keV`
- `R4/2`：无量纲结构判据

## R06

转动模型拟合算法（`fit_rotor_sklearn`）：
1. 构造特征 `x=J(J+1)`。
2. 用 `LinearRegression` 拟合 `E = E0 + A*x`。
3. 输出 `E0, A`，并计算 `RMSE` 与 `R^2`。

该路径对应可解析线性模型，适合做基准估计。

## R07

振动模型拟合算法（`fit_vibrator_sklearn`）：
1. 采用偶 `J` yrast 近似 `n=J/2`。
2. 拟合 `E = E0 + hw*n`。
3. 输出 `E0, ħω` 与误差指标。

该步骤给出集体模型另一极限，用于与转动拟合做并列比较。

## R08

为避免单一路径依赖，本 MVP 增加两条独立数值路线：
- `fit_rotor_torch`：用 PyTorch + Adam 最小化 MSE，`A=softplus(raw_a)` 保证 `A>0`；
- `fit_rotor_stretching_scipy`：用 `scipy.optimize.curve_fit` 拟合 `E0 + A*x - B*x^2`，用于评估离心伸缩校正。

因此输出可交叉验证“线性最小二乘、梯度优化、非线性拟合”三类实现。

## R09

复杂度（单核素 `N` 个能级，PyTorch 迭代 `K`）：
- 线性回归（转动/振动）：`O(N)`
- SciPy 三参数拟合：通常 `O(N * I)`，`I` 为迭代评估次数
- PyTorch 拟合：`O(KN)`
- 汇总与表格输出：`O(N)`

当前 `N<=5`、`K=2000`，运行时间远低于 1 秒量级（常见开发机）。

## R10

数值稳定性处理：
- `curve_fit` 给定参数边界，抑制不合理负刚度或爆炸参数。
- PyTorch 路径对 `A` 用 `softplus` 约束正值，避免物理量符号错误。
- `R^2` 计算对 `SST=0` 情况提供保护分支。
- `R4/2` 在缺失 `2+` 或 `4+` 时返回 `NaN`，避免误判。

## R11

正确性检查策略：
- 近转动样本应表现为：转动模型 RMSE 明显小于振动模型、`R4/2` 接近 3.3。
- 近振动样本应表现为：振动模型 RMSE 更小、`R4/2` 接近 2。
- `rotor_lr` 与 `rotor_torch` 的 `A` 结果应相互接近，说明不同优化路径一致。

这三条检查形成“物理判据 + 拟合误差 + 多算法一致性”的闭环。

## R18

`demo.py` 源码级算法流程（8 步）：

1. `build_samples()` 构造两组偶偶核低能级样本（转动型、振动型）。
2. 对每组样本计算 `x=J(J+1)` 与 `R4/2`，给出基于阈值的先验分类标签。
3. `fit_rotor_sklearn()` 在线性空间拟合 `E=E0+A*x`，得到刚性转子基线参数。
4. `fit_vibrator_sklearn()` 拟合 `E=E0+ħω*(J/2)`，得到振动极限参数。
5. `fit_rotor_torch()` 用 Adam 迭代最小化 MSE，并通过 `softplus` 约束 `A>0`。
6. `fit_rotor_stretching_scipy()` 用 `curve_fit` 在有界条件下拟合 `E0+A*x-B*x^2`。
7. 对四条模型路径统一计算 RMSE，并按最小 RMSE 选择当前样本最优模型。
8. 将逐能级预测表和全局汇总表打印输出，形成“判据 + 拟合 + 选择”的完整闭环。
