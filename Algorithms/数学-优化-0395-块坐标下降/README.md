# 块坐标下降

## R01

块坐标下降（Block Coordinate Descent, BCD）是坐标下降的推广：
每次不是只更新单个变量，而是更新一个“变量块”（block），其余变量固定。

本条目选取最小可复现且有闭式块更新的场景：
- `L2` 正则线性回归（Ridge）
- 将特征维度按固定块大小划分
- 每次对一个块求解小型线性系统作为精确子问题解

## R02

本 MVP 求解问题：

`min_w  f(w) = (1/(2n)) * ||y - Xw||_2^2 + (lambda/2) * ||w||_2^2`

其中：
- `X in R^(n*p)`，`y in R^n`
- `lambda > 0`
- `w` 按块划分为 `w_(B1), w_(B2), ..., w_(BK)`

输出：
- 估计参数 `w`
- 每轮 epoch 的目标函数值、最大块更新量
- 与闭式 Ridge 解的差异（用于正确性对照）
- 收敛状态

## R03

选择该问题的原因：
- BCD 在该目标上有清晰的块子问题闭式解，不需要黑盒优化器；
- 相比单坐标下降，能直观看到“块更新”与小型线性代数求解的关系；
- 可与全局闭式 Ridge 解直接对比，便于做数值校验。

## R04

设残差 `r = y - Xw`，当前更新块为 `B`，对应特征子矩阵 `X_B`，参数子向量 `w_B`。
固定其余块后，子问题为：

`min_u (1/(2n)) * ||r + X_B w_B - X_B u||_2^2 + (lambda/2) * ||u||_2^2`

令：
- `G_B = (X_B^T X_B)/n + lambda * I`
- `b_B = (X_B^T (r + X_B w_B))/n`

则块最优更新：

`w_B_new = G_B^{-1} b_B`

并做残差增量维护：

`r <- r - X_B (w_B_new - w_B)`

## R05

算法流程（循环块坐标下降）：

1. 检查输入维度、有限值与超参数合法性。
2. 根据 `block_size` 划分特征块。
3. 初始化 `w=0`，`r=y`。
4. 预计算各块的 Gram 矩阵 `G_B`。
5. 每个 epoch 依次遍历所有块：
   - 计算 `b_B`
   - 解线性系统得到 `w_B_new`
   - 增量更新 `w` 与 `r`
6. 记录本轮 `objective`、`max_block_delta`。
7. 若 `max_block_delta < tol` 则判定收敛。
8. 到 `max_epochs` 仍未满足阈值则返回未收敛状态。

## R06

正确性要点：
- 固定其它块时，块子问题是严格凸二次问题（`lambda > 0` 时 Hessian 正定）；
- 每次块更新都取该子问题精确最优解，因此目标函数应非增（数值误差下允许极小扰动）；
- 整体问题为强凸，循环块更新会收敛到唯一全局最优解；
- `demo.py` 中提供目标函数近似单调性检查和闭式解误差检查。

## R07

复杂度分析（设样本数 `n`、特征数 `p`、块数 `K`、平均块宽 `b≈p/K`、迭代轮次 `T`）：
- 每个块更新主要成本：
  - 计算 `b_B`：`O(n*b)`
  - 解线性系统：`O(b^3)`
  - 残差增量：`O(n*b)`
- 单轮 epoch 约为：`O(n*p + K*b^3)`
- 总体：`O(T*(n*p + K*b^3))`
- 空间：
  - `w`: `O(p)`
  - `r`: `O(n)`
  - 块 Gram 矩阵总和约 `O(K*b^2)`

## R08

边界与异常处理：
- `X` 必须为二维，`y` 必须为一维，且样本数一致；
- 输入必须全为有限值（禁止 `nan/inf`）；
- `lambda <= 0`、`block_size <= 0`、`tol <= 0`、`max_epochs <= 0` 直接报错；
- 若某块矩阵病态，使用 `np.linalg.solve` 失败会抛异常并附带块索引提示。

## R09

MVP 取舍：
- 仅使用 `numpy` 与标准库，避免引入重依赖；
- 采用循环（cyclic）块顺序，不实现随机块采样和并行块更新；
- 不使用现成黑盒优化器，核心迭代显式实现；
- 以“可读、可审计、可运行”为首要目标，而非工业级性能优化。

## R10

`demo.py` 函数职责：
- `validate_inputs`：检查输入与超参数合法性。
- `build_blocks`：按 `block_size` 生成特征块切分。
- `ridge_objective`：计算当前目标函数。
- `closed_form_ridge`：计算闭式 Ridge 解用于对照。
- `block_coordinate_descent_ridge`：核心 BCD 训练循环。
- `make_synthetic_regression`：生成可复现实验数据。
- `is_almost_monotone`：检查目标函数是否近似单调下降。
- `run_case`：运行单个超参数组合并打印结果。
- `main`：执行两组块大小实验并汇总。

## R12

输出字段说明：
- `epoch`：迭代轮次。
- `objective`：当前 Ridge 目标值。
- `max_block_delta`：本轮所有块中最大的参数更新范数。
- `converged`：是否在 `max_epochs` 内达到阈值。
- `final_objective`：最终目标值。
- `l2_to_closed_form`：与闭式 Ridge 解的 L2 距离。
- `objective_monotone`：目标值是否近似单调下降。

## R13

最小测试覆盖（由 `main` 自动执行）：

- 两组块大小：
  - `block_size=1`（退化为单坐标下降）
  - `block_size=6`（标准块更新）

  - 收敛轮次与最终目标

  - 目标函数单调性检查

## R14

关键参数建议：
- `lambda_`：正则强度，越大解越平滑，条件数更稳定。
- `block_size`：块宽度，较小块更细粒度，较大块每步更“激进”。
- `tol`：收敛阈值，越小要求越严格。
- `max_epochs`：最大外层迭代轮数。

经验设置：
- 先用中等 `lambda_`（如 `0.1~1.0`）保证稳定；
- 再根据精度需求调小 `tol`。

## R15

与相关方法对比：
- 对比梯度下降：
  - 梯度下降用全向量步进；
  - BCD 每次精确求解子问题，常见于可分块结构问题。
- 对比单坐标下降：
  - 单坐标每步更便宜；
  - 块更新能利用块内耦合，常减少迭代轮次。
- 对比直接闭式解：
  - 闭式解需一次求解 `p x p` 系统；
  - BCD 避免一次性大求解，更适合分块和可扩展实现。

## R16

典型应用场景：
- 大规模线性模型的分块优化；
- 具有天然分组结构的参数学习（按特征组/模块更新）；
- 作为更复杂交替最优化算法（如矩阵分解）的基础内核。

## R17

可扩展方向：
- 随机块选择或重要性采样；
- 异步/并行块更新（需处理冲突与收敛性）；
- 加入块预条件与自适应块划分；
- 扩展到非光滑正则（如 Group Lasso 的块近端更新）；
- 引入验证集与早停策略。

## R18

`demo.py` 源码级算法流程（8 步）：

1. `main` 固定随机种子构造合成回归数据，并定义两组 `block_size` 实验。
2. 每组实验由 `run_case` 调用 `block_coordinate_descent_ridge` 执行训练。
3. 训练开始时 `validate_inputs` 校验维度、数值有效性和超参数范围。
4. `build_blocks` 根据 `block_size` 切分特征索引，并为每个块预计算 `G_B=(X_B^T X_B)/n + lambda I`。
5. 初始化 `w=0`、`residual=y` 后，按块顺序循环：先算 `b_B = X_B^T(residual + X_B w_B)/n`。
6. 用 `np.linalg.solve(G_B, b_B)` 得到块精确解 `w_B_new`，计算 `delta = w_B_new - w_B_old`。
7. 通过 `residual -= X_B @ delta` 做增量更新，并记录每轮 `objective` 与 `max_block_delta`；若低于 `tol` 则收敛返回。
8. `run_case` 再调用 `closed_form_ridge` 做对照，输出 `l2_to_closed_form` 与目标函数单调性检查结果。
