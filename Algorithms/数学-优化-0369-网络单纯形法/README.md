## R01

网络单纯形法（Network Simplex）是最小费用流（minimum-cost flow）的经典算法。
它把线性规划单纯形法的“基、入基、出基、枢轴”思想搬到网络结构上：
- 基对应一棵生成树（tree basis）；
- 非基弧通过“约化费用（reduced cost）”决定是否入基；
- 入基后在唯一基环上做增广，保持可行性并下降目标值。

本目录给出一个可运行且可审计的 MVP：
- 采用“运输问题（transportation problem）”这一最小费用流特例；
- 使用运输单纯形（transportation simplex），它是网络单纯形在二分网络上的特化形式；
- 核心枢轴逻辑完全手写，不依赖黑盒求解器。

## R02

本实现求解的模型为平衡运输问题：

- 决策变量：`x_ij >= 0`，表示从供给点 `i` 运到需求点 `j` 的流量。
- 目标函数：
  - `min Σ_i Σ_j c_ij * x_ij`
- 约束：
  - `Σ_j x_ij = supply_i`（每个供给点出流等于供给）
  - `Σ_i x_ij = demand_j`（每个需求点入流等于需求）
  - `Σ_i supply_i = Σ_j demand_j`（平衡条件）

输入：`cost` 矩阵、`supply`、`demand`。  
输出：最优流矩阵 `x*`、最终基、迭代历史。

## R03

数学要点（网络单纯形视角）：

1. 在运输问题里，基变量集合大小为 `m+n-1`，可对应二分图上的一棵树基。  
2. 对基单元满足对偶关系：`c_ij = u_i + v_j`。  
3. 对非基单元定义约化费用：`r_ij = c_ij - u_i - v_j`。  
4. 最优性条件：若所有非基单元 `r_ij >= 0`，当前可行解最优。  
5. 若存在 `r_ij < 0`，该单元可入基；入基后会与树基形成唯一环。  
6. 在环上按 `+ - + -` 交替增减流量，步长取所有 `-` 边流量最小值 `theta`，保证非负可行。

## R04

算法流程（本实现）如下：

1. 校验输入合法性（维度、非负、有限性、供需平衡）。
2. 用西北角法构造初始可行流 `x`。
3. 若基单元不足 `m+n-1`，补零基单元并保证“无环”（树基）。
4. 由基方程 `c_ij = u_i + v_j` 计算势 `u,v`。
5. 扫描非基单元的约化费用，选最负者入基（最陡下降）。
6. 若最小约化费用已非负，则终止并返回最优解。
7. 将入基单元加入基后，在当前树基中找唯一环。
8. 沿环执行枢轴更新并确定出基单元，回到第 4 步迭代。

## R05

核心数据结构：

- `x: np.ndarray[m, n]`：当前运输流矩阵。  
- `basis: set[(i, j)]`：当前基单元集合，规模保持 `m+n-1`。  
- `u, v: np.ndarray`：对偶势。  
- `HistoryItem = (iter, objective, min_reduced_cost, entering, leaving, theta)`：每轮日志。  
- `DisjointSet`：并查集，用于补零基时避免形成环。

## R06

正确性关键点：

- 可行性保持：
  - 初始解满足行列和约束；
  - 环上 `+/-` 更新不改变各行各列总和；
  - `theta` 取 `-` 侧最小流量，保证更新后仍 `x>=0`。
- 基合法性保持：
  - 基规模固定 `m+n-1`；
  - 每轮“入基一个、出基一个”，保持树基结构。
- 下降性：
  - 仅在存在负约化费用时枢轴，目标值不增，通常下降。
- 最优性判据：
  - 全部非基约化费用非负时满足运输问题的最优条件。

## R07

复杂度（`m` 个供给点，`n` 个需求点）粗略估计：

- 单轮计算势 `u,v`：`O(m+n)`（在树基上 BFS/传播）。
- 单轮扫描约化费用：`O(mn)`。
- 单轮找环并更新：`O(m+n)`。
- 单轮主导成本：`O(mn)`。
- 总体：`O(T * mn)`，`T` 为枢轴轮数。
- 空间：
  - 流矩阵 `O(mn)`；
  - 基和辅助结构 `O(m+n)`；
  - 迭代日志 `O(T)`。

## R08

边界与异常处理：

- `cost` 不是二维矩阵、或与 `supply/demand` 维度不一致：抛 `ValueError`。
- `supply/demand` 有负值、或总供给不等于总需求：抛 `ValueError`。
- 输入含 `nan/inf`：抛 `ValueError`。
- 初始基补全失败、基图不连通导致势不可定：抛 `RuntimeError`。
- 迭代超过 `max_iter` 仍未满足最优性：抛 `RuntimeError`。

## R09

MVP 取舍说明：

- 为保证实现小而清晰，仅做“平衡运输问题”版本，不覆盖一般上下界最小费用流的全部细节（如显式容量上界、下界平移、人工根弧等）。
- 核心枢轴过程手写，不把求解过程交给外部黑盒。
- `scipy.optimize.linprog` 仅作可选“结果校验”，不参与核心迭代。

## R10

`demo.py` 主要函数职责：

- `validate_inputs`：输入合法性检查。  
- `build_initial_solution_northwest`：西北角法构造初始可行解。  
- `augment_basis_with_zero_cells`：补零基单元到 `m+n-1` 且避免成环。  
- `compute_potentials`：由基方程解出对偶势 `u,v`。  
- `select_entering_cell`：按最负约化费用选入基单元。  
- `find_cycle_cells`：在树基中找到入基后唯一环。  
- `pivot_on_cycle`：执行 `+/-` 更新、确定出基单元。  
- `transportation_network_simplex`：主循环。  
- `check_feasibility`：验证行列和与非负性。  
- `solve_with_linprog`：可选 LP 交叉验证。

## R14

关键参数：

- 若希望更强可重复性，保持固定输入实例或固定随机种子生成实例。

## R15

与其他最小费用流方法对比：

- 对比逐次最短路（SSP）：
  - SSP 更偏“路径增广”；
  - 网络单纯形更偏“基变换”，在很多稠密运输场景表现稳定。
- 对比环消去（cycle-canceling）：
  - 网络单纯形通常更高效，且目标下降路径更结构化。
- 对比通用 LP 黑盒：
  - 黑盒实现快但过程不透明；
  - 本实现能逐轮审计入基/出基/环更新细节。

## R18

`demo.py` 源码级算法流程（9 步）：

1. `main` 定义固定运输实例（`cost/supply/demand`），调用 `transportation_network_simplex`。  
2. `validate_inputs` 校验维度、有限性、非负性和供需平衡。  
3. `build_initial_solution_northwest` 用西北角法生成初始可行流矩阵 `x`。  
4. `augment_basis_with_zero_cells` 用并查集补零基单元，保证基规模 `m+n-1` 且无环。  
5. 迭代中 `compute_potentials` 在树基上解出 `u,v`，使基单元满足 `c_ij = u_i + v_j`。  
6. `select_entering_cell` 扫描非基单元约化费用 `r_ij`，选最负者入基；若最小 `r_ij >= -tol` 则判最优并停止。  
7. `find_cycle_cells` 在当前树基中找“行节点到列节点”的唯一路径，与入基单元构成唯一环。  
8. `pivot_on_cycle` 按 `+/-` 交替更新环上流量，`theta` 取负号位最小流量，确定并删除出基单元。  
9. 循环执行直到最优；最后 `check_feasibility` 验证可行性，并可选用 `solve_with_linprog` 做目标值交叉校验。
