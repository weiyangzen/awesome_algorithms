# 紫外-可见光谱 (UV-Vis Spectroscopy)

- UID: `PHYS-0464`
- 学科: `物理`
- 分类: `光谱学`
- 源序号: `487`
- 目标目录: `Algorithms/物理-光谱学-0487-紫外-可见光谱_(UV-Vis_Spectroscopy)`

## R01

紫外-可见光谱（UV-Vis）通过测量样品在 `200-800 nm` 波段的吸光度 `A(λ)` 来识别组分并估计浓度。核心定律是 Beer-Lambert 定律：

- `A(λ) = ε(λ) * l * c`

其中 `ε(λ)` 为摩尔吸光系数，`l` 为光程，`c` 为浓度。实际任务中常需从含噪、含基线漂移、含干扰物的整段光谱反推出目标分析物浓度。

## R02

UV-Vis 是分析化学最经典的定量方法之一，早期以单波长标定为主。随着计算能力提升，现代实践逐步演进为：

- 从“单峰读数”扩展到“多波长联合建模”；
- 从手工基线扣除扩展到数字滤波与自动基线拟合；
- 从线性标定扩展到机器学习回归（Ridge、PLS、神经网络等）。

因此它既保留了物理可解释性，也可在复杂样品条件下借助算法提升稳健性。

## R03

本问题要解决的是：给定一批 UV-Vis 光谱，估计目标组分浓度 `c_target`。难点在于：

1. 基线漂移（仪器零点、浊度散射）会抬高/拉低全谱；
2. 干扰组分与目标组分峰位重叠；
3. 噪声导致单点读数不稳定。

对应思路是“物理先验 + 统计学习”结合：
- 先做平滑与基线校正；
- 用 Beer-Lambert 做可解释基线模型；
- 再用多波长回归（Ridge / MLP）提升抗干扰能力。

## R04

本目录 MVP 使用的数学模型：

- 混合光谱：`A(λ) = c_a*ε_a(λ) + c_b*ε_b(λ) + b(λ) + n(λ)`
- `c_a` 是目标分析物浓度，`c_b` 是干扰物浓度；
- `b(λ)` 是低阶多项式基线 + 瑞利散射项近似；
- `n(λ)` 是高斯噪声。

算法分三层：
1. 单波长估计：`c_hat = A_corr(λ*) / (ε_a(λ*)*l)`；
2. 多波长线性回归：`c_hat = w^T x + b`（Ridge）；
3. 多波长非线性回归：MLP 拟合 `x -> c_hat`。

## R05

记样本数 `N`、波长点数 `M`、MLP 训练轮数 `E`、隐藏维 `H`：

- Savitzky-Golay 平滑 + 基线拟合：约 `O(N*M)`；
- Beer-Lambert 单波长预测：`O(N)`；
- Ridge 训练（闭式/数值解）常见量级约 `O(N*M^2 + M^3)`；
- MLP 训练：约 `O(E*N*M*H)`。

在本 MVP 中 `M=581`，数据规模小，可在 CPU 上快速完成。

## R06

单波长定量直观例子：

- 选择分析波长 `λ*=280 nm`；
- 已知 `ε_a(280)=1.20`（示例单位归一化）；
- 测得基线校正后吸光度 `A_corr(280)=0.48`，`l=1 cm`。

则浓度估计：

- `c_hat = 0.48 / (1.20*1) = 0.40`

这展示了 UV-Vis 的物理可解释优势：每个量都有明确实验含义。

## R07

优点：

- 物理模型清晰，便于解释和审计；
- 仪器普及、成本低、实验速度快；
- 可与多变量算法结合提升复杂样品鲁棒性。

局限：

- 峰重叠严重时单波长法误差明显；
- 基线漂移、散射和噪声会引入系统偏差；
- 纯数据驱动模型在跨仪器/跨批次时可能退化。

## R08

前置知识：

- Beer-Lambert 定律与吸收光谱基础；
- 线性代数与回归（最小二乘、Ridge）；
- 基本信号处理（平滑、峰检测、基线校正）；
- Python 科学计算栈（NumPy/SciPy/pandas/scikit-learn/PyTorch）。

## R09

适用场景：

- 定量分析（溶液浓度、反应进程监测）；
- 工艺在线检测（配方一致性、杂质趋势）；
- 教学与算法验证（将物理模型和 ML 融合）。

不适用或需谨慎：

- 样品强浑浊/强荧光导致吸收假设破坏；
- 目标组分吸收太弱，信噪比不足；
- 缺少代表性标定数据却直接上复杂模型。

## R10

实现正确性的检查点：

1. 预处理后光谱基线应接近 0（至少边缘区不应系统偏移）；
2. `find_peaks` 选择的 `λ*` 应落在目标组分主吸收峰附近；
3. Beer-Lambert 估计结果应非负且量级合理；
4. Ridge/MLP 在 ID 测试集上应显著优于随机预测；
5. shifted 数据集性能下降应可解释，而非数值发散导致。

## R11

数值稳定性处理：

- Savitzky-Golay 使用奇数窗口并限制多项式阶次，避免过拟合噪声；
- 基线拟合只使用低吸收分位点，降低峰区对基线估计的污染；
- 除法 `A/(ε*l)` 对分母加下界 `1e-12`，防止零除；
- MLP 输入做标准化，避免梯度尺度失衡。

## R12

调参经验（对应本 MVP）：

- `window_length`（平滑窗口）过小会保留噪声，过大会抹平窄峰；
- Ridge 的 `alpha` 决定偏差-方差折中；
- MLP 轮数与学习率共同决定收敛速度；
- 评估时应同时看 `MAE`、`RMSE`、`R2`，不只看单一指标。

## R13

理论性质：

- Beer-Lambert 单波长估计在“纯组分 + 线性吸收 + 无漂移”条件下是一致且可解释的；
- Ridge 目标是凸优化，有唯一全局最优解（`alpha > 0`）；
- MLP 为非凸优化，不保证全局最优，但可逼近复杂非线性映射；
- 本任务不是近似算法问题，不涉及近似比或随机化成功概率界。

## R14

常见失效模式与防护：

- 失效：峰位漂移导致固定 `λ*` 失真。  
  防护：使用峰搜索 + 多波长模型并行。
- 失效：强基线漂移让单点法系统偏高。  
  防护：先做基线拟合扣除。
- 失效：干扰物与目标峰重叠，单波长不可分。  
  防护：改用 Ridge/MLP 全谱建模。
- 失效：训练集覆盖不足导致 shifted 集退化。  
  防护：扩展标定范围并做分布外验证。

## R15

工程落地建议：

- 固定光程、比色皿与预处理流程，先保证实验一致性；
- 用 Beer-Lambert 作为“可解释基线模型”，再叠加 ML 提升性能；
- 保存全流程参数（平滑窗口、`alpha`、模型种子）以便复现；
- 建立 ID + OOD（shifted）双测试集作为上线前门槛。

## R16

相关方法对比：

- 单波长标定：解释性最好，但抗干扰最弱；
- 多元线性回归 / Ridge：在重叠峰场景更稳健；
- PLS：化学计量学常用，适合高共线特征；
- 神经网络：可处理更复杂非线性，但解释性和数据需求更高。

本 MVP 选择“Beer-Lambert + Ridge + MLP”三者并列，是为了展示从物理可解释到数据驱动的连续谱。

## R18

`demo.py` 的源码级算法流程（8 步）：

1. `extinction_profiles` 构造目标物 `ε_a(λ)` 与干扰物 `ε_b(λ)` 的吸收峰形。  
2. `simulate_uvvis_dataset` 按 `A=c_aε_a+c_bε_b+b+n` 生成训练/测试光谱（含噪声、基线、散射与分布偏移）。  
3. `preprocess_spectra` 对每条光谱执行 Savitzky-Golay 平滑，并用低分位点二次多项式拟合基线后扣除。  
4. `select_analytical_wavelength` 用 `scipy.signal.find_peaks` 在 `ε_a` 上找候选峰，再按 `ε_a/(ε_b+eps)` 选出干扰最小的分析波长 `λ*`。  
5. `beer_lambert_estimate` 在 `λ*` 处执行 `c=A_corr/(ε*l)`，得到可解释的单波长浓度估计。  
6. `train_ridge_model` 使用 `StandardScaler + Ridge` 在全谱特征上拟合线性多波长回归器。  
7. `train_torch_mlp` 构建两层 MLP（PyTorch），用 Adam 最小化 MSE，学习非线性光谱到浓度映射。  
8. `main` 在 ID 与 shifted 数据集上计算 `MAE/RMSE/R2`，用 `pandas` 汇总打印并给出样例预测对比。  

该实现没有把“UV-Vis 定量”留给黑盒库一键完成，关键步骤（数据生成、预处理、选峰、物理估计、回归训练与评估）均可直接在源码中追踪。
