## R01

潮汐理论研究的是天体引力在延展天体（如地球海洋）上造成的**差分引力效应**。在最小平衡潮模型里，核心是二阶引潮势：

`U_b(t) = (G * M_b / R_b^3) * Re^2 * P2(cos(psi_b(t)))`

其中 `b` 表示月球或太阳，`P2(x)=0.5*(3x^2-1)`，`psi_b` 是观测点与天体方向夹角。将总引潮势转成平衡潮位近似：

`eta(t) ~= (1 + k2) * U_total(t) / g`

`k2` 是 Love 数，反映地球弹性响应对潮位的放大。

## R02

在天体力学中，潮汐理论连接了轨道运动（天体位置变化）与地表周期性响应（海潮/固体潮），是以下问题的基础：

- 半日潮与全日潮频率来源解释；
- 月日潮相对强弱（如 `M2` 通常强于 `S2`）解释；
- 由观测潮位反演参数（如有效 `k2`）的反问题建模。

## R03

本条目 MVP 聚焦一个“小而完整”的闭环：

- 用简化天体几何（纬度、赤纬、时角）构造月球+太阳引潮势；
- 通过平衡潮公式生成潮位，并加入噪声形成“观测”；
- 用最小二乘反演比例系数并换算 `k2`；
- 对拟合潮位做 `M2/S2` 谐波分解，验证半日潮结构。

目标是把潮汐理论从公式落到可运行、可验证、可扩展的代码基线。

## R04

实现采用的物理与数值方程：

1. 夹角余弦：
`cos(psi) = sin(phi)sin(delta) + cos(phi)cos(delta)cos(H)`
2. 二阶勒让德多项式：`P2(x)=0.5*(3x^2-1)`。
3. 单天体引潮势：`U_b=(G*M_b/R_b^3)*Re^2*P2(cos(psi_b))`。
4. 总引潮势：`U_total = U_moon + U_sun`。
5. 平衡潮位：`eta=(1+k2)*U_total/g`。
6. 一参数反演：`eta_obs ~= alpha * U_total`，并令 `k2 = alpha*g - 1`。
7. 谐波展开：`y ~= c0 + a1*sin(w1 t)+b1*cos(w1 t)+a2*sin(w2 t)+b2*cos(w2 t)`。

## R05

设时间采样点数为 `n`：

- 引潮势计算：`O(n)`；
- `curve_fit` 一参数拟合：每次残差评估 `O(n)`，总计近似 `O(n)`；
- `M2/S2` 线性回归：设计矩阵固定 5 列，复杂度 `O(n)`。

总体时间复杂度 `O(n)`，空间复杂度 `O(n)`。

## R06

`demo.py` 的输出包含三类信息：

- 反演结果：`true_k2` 与 `estimated_k2`；
- 拟合误差：`RMSE/MAE`（米）；
- 谐波结果：`M2` 与 `S2` 的振幅和相位。

脚本内置断言：

- `RMSE < 0.05 m`；
- `|k2_est - k2_true| < 0.06`；
- 月球引潮势波动标准差显著大于太阳（`std(U_moon)/std(U_sun) > 1.5`）；
- `M2` 与 `S2` 振幅均为非平凡正值。

这使其可直接作为非交互回归样例执行。

## R07

## R10

正确性直觉：

1. 月日天体在天球上的相对运动决定主周期，故潮位应呈半日主导；
2. 月球 `M/R^3` 贡献较太阳更强，因此月引潮势波动量级通常更大；
3. 若把 `eta` 视作 `U_total` 的线性映射，反演出的比例系数应回到设定 `k2` 附近；
4. 加入小噪声后，拟合误差应可控但非零。

## R11

数值稳定与精度策略：

- 统一使用秒级均匀时间网格，避免离散不一致；
- 拟合只用 1 个参数（`alpha`），降低病态风险；
- 谐波分解采用固定频率线性回归，避免频率-相位-振幅同时非线性耦合；
- 随机噪声使用固定随机种子，保证可复现。

## R12

关键参数及建议：

- `latitude_deg`：站点纬度，改变 `cos(psi)` 结构；
- `sim_hours` 与 `dt_s`：决定频谱分辨率与估计方差；
- `k2_true`：合成数据真值；
- 噪声标准差（脚本中 `0.025 m`）：过大将显著拉低反演稳定性；
- `omega_m2`、`omega_s2`：谐波频率设定。

调参经验：

- 若 `k2` 估计波动大，先增加总时长（如 5-7 天）再减小 `dt`；
- 若 `M2/S2` 分离不明显，优先延长观测窗口而不是盲目加特征项。

## R14

常见失效模式：

1. 把本模型用于真实港口潮位而不引入海盆动力学，导致系统偏差；
2. 时间窗口太短，`M2/S2` 拟合互相污染；
3. 噪声过大或存在强非潮信号（风暴潮）时仍做单参数反演；
4. 频率单位混淆（小时与秒）造成相位/振幅全局错误。

## R18

`demo.py` 源码级算法流程（8 步）：

1. `main` 构造时间网格、站点纬度、月球和太阳参数（质量、距离、赤纬周期、时角周期）。  
2. `tidal_potential_body` 对每个天体调用 `body_declination_rad` 和 `body_hour_angle_rad`，根据球面角关系算 `cos(psi)`。  
3. `legendre_p2` 计算 `P2(cos(psi))`，并乘 `G*M/R^3*Re^2` 得到该天体引潮势。  
4. 在 `main` 中把月球与太阳势叠加成 `U_total`，再经 `equilibrium_tide_height_m` 映射为平衡潮位。  
5. 通过固定随机种子加入高斯噪声，得到合成观测 `eta_obs`。  
6. `estimate_linear_scale` 用 `scipy.curve_fit` 解一参数最小二乘 `eta_obs = alpha * U_total`，再换算 `k2_est = alpha*g - 1`。  
7. `harmonic_decompose` 组装 `1/sin/cos` 设计矩阵，并用 `LinearRegression(fit_intercept=False)` 求 `M2/S2` 的 `sin`、`cos` 系数，再转换为振幅相位。  
8. `main` 输出误差与谐波表，并执行四条断言形成自动验收。  

第三方库没有被当作黑盒：

- `scipy` 仅做一参数最小二乘求解，目标函数与参数映射在源码中显式给出；
- `scikit-learn` 仅做固定设计矩阵线性回归，谐波振幅/相位转换公式在代码中手动实现。
