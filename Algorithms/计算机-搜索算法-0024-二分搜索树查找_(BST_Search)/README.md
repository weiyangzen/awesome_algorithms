# 二分搜索树查找 (BST Search)

- UID: `CS-0024`
- 学科: `计算机`
- 分类: `搜索算法`
- 源序号: `24`
- 目标目录: `Algorithms/计算机-搜索算法-0024-二分搜索树查找_(BST_Search)`

## R01

二分搜索树查找（BST Search）用于在二分搜索树（Binary Search Tree）中定位目标键值。  
它利用 BST 的有序性质，在每个节点只走左子树或右子树之一，因此平均情况下查找效率较高。

## R02

问题形式化：
- 输入：一棵满足 BST 性质的树根节点 `root`；目标键 `target`
- 输出：若存在键值等于 `target` 的节点，返回该节点（或其键值）；否则返回“未找到”（如 `None`）

## R03

BST 性质与前提：
- 对任意节点 `x`：
  - 左子树所有键 `< x.key`
  - 右子树所有键 `> x.key`（本条目采用“不存重复键”策略）
- 树由合法插入流程构建，保证上述性质成立
- 键类型支持全序比较（`<`, `>`, `==`）

## R04

核心思想：
- 从根节点开始，与目标键比较：
  - 相等：命中，结束
  - 目标更小：只可能在左子树，转到左孩子
  - 目标更大：只可能在右子树，转到右孩子
- 若走到空指针（`None`），表示不存在该键

## R05

简化伪代码：

```text
cur = root
while cur is not None:
    if target == cur.key:
        return cur
    if target < cur.key:
        cur = cur.left
    else:
        cur = cur.right
return None
```

## R06

正确性要点（循环不变式）：
- 任意时刻 `cur` 都是“仍可能包含 target 的子树根”
- 每一步都根据 BST 性质丢弃一个不可能包含目标的分支
- 一旦 `cur is None`，候选集合为空，故目标不存在

## R07

复杂度：
- 时间复杂度：
  - 平均情况：`O(log n)`（树近似平衡）
  - 最坏情况：`O(n)`（退化为链表）
- 空间复杂度：
  - 迭代查找：`O(1)`
  - 若用递归查找，额外调用栈为 `O(h)`（`h` 为树高）

## R08

边界与特殊情况：
- 空树：直接未命中
- 只有根节点的树
- 查找最小键、最大键
- 目标值小于最小键或大于最大键
- 插入重复键时策略要明确（本条目采用“忽略重复，不新增节点”）

## R09

示例（按顺序插入 `8,3,10,1,6,14,4,7,13`）：
- 查找 `6`：路径 `8 -> 3 -> 6`，命中
- 查找 `13`：路径 `8 -> 10 -> 14 -> 13`，命中
- 查找 `2`：路径 `8 -> 3 -> 1 -> None`，未命中

## R12

典型应用：
- 动态集合中的键存在性查询（插入与查询混合）
- 符号表 / 词典结构基础实现（教学与原型）
- 区间结构（如平衡树、顺序统计树）设计的基础概念
- 作为 AVL / 红黑树等平衡搜索树的入门模型

## R13

常见错误：
- 违反 BST 性质（插入逻辑错误）导致查找失真
- 比较方向写反（`<` 与 `>` 分支互换）
- 忘记推进当前指针，造成死循环
- 重复键策略不一致，导致行为与预期不符

## R14

常见变体：
- 递归版 BST 查找
- 返回父节点、层深或访问路径
- 支持重复键（节点计数或链式挂载）
- 在平衡 BST（AVL/红黑树）中执行相同查找逻辑以保证 `O(log n)` 最坏界

## R15

与有序数组二分查找对比：
- 都利用“有序性”逐步缩小搜索空间
- 有序数组二分：
  - 查找稳定 `O(log n)`，但动态插入代价高（通常 `O(n)`）
- BST：
  - 插入与查找都可在平均 `O(log n)` 完成
  - 但若不平衡，查找可退化到 `O(n)`

## R16

工程实践建议：
- 生产场景优先使用平衡 BST（或现成有序映射容器），避免退化
- 明确并文档化重复键语义（拒绝、覆盖、计数）
- 对外 API 统一返回约定（返回节点 / 布尔值 / 可选值）
- 单元测试覆盖空树、单节点、深路径、重复键

## R17

最小测试清单：
- 空树查找（应未命中）
- 根节点命中
- 左侧深路径命中、右侧深路径命中
- 未命中且落到左空指针、右空指针
- 重复插入是否符合约定（本条目应返回“未插入”）
- 中序遍历是否严格升序（验证 BST 结构正确）

## R18

源码级算法流程拆解（以本目录 `demo.py` 的 `BST.search` 为准）：
1. 接收目标键 `target`，把游标 `current` 初始化为根节点 `self.root`。
2. 进入 `while current is not None` 循环，只要当前子树非空就继续判定。
3. 读取当前节点键 `key = current.key`，与 `target` 进行比较。
4. 若 `target == key`，立即返回当前节点，查找成功。
5. 若 `target < key`，将 `current` 更新为 `current.left`，剪去右子树。
6. 否则（`target > key`），将 `current` 更新为 `current.right`，剪去左子树。
7. 重复步骤 2-6，沿一条根到叶路径逐层向下。
8. 当 `current` 变为 `None` 时循环结束，返回 `None`，表示目标不存在于该 BST。
