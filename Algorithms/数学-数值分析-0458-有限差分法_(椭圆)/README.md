## R01

有限差分法（Finite Difference Method, FDM）是求解椭圆型偏微分方程的经典离散方法。它通过把连续导数替换成网格上的差分商，把 PDE 转换为稀疏线性方程组。本条目聚焦二维椭圆问题 `-Delta u = f` 的五点差分离散与求解。

## R02

本目录 MVP 求解如下边值问题（单位方形域）：

- 方程：`-Delta u = f`，`(x,y) in (0,1)^2`
- 边界：`u = g`，`(x,y) in ∂Omega`（Dirichlet）
- 选取解析真解：`u*(x,y)=exp(x+y)`
- 对应右端：`f(x,y)=-2exp(x+y)`，边界函数 `g=u*|∂Omega`

由于有解析解，可直接评估离散误差与收敛阶。

## R03

离散采用均匀网格步长 `h=1/(n+1)`，内部未知点为 `n x n`。对内部点 `(i,j)` 使用五点模板：

`(4u(i,j)-u(i-1,j)-u(i+1,j)-u(i,j-1)-u(i,j+1))/h^2 = f(i,j)`

该模板对应二阶精度（截断误差 `O(h^2)`），是二维 Poisson/椭圆问题最常见的基础离散格式。

## R04

离散后可抽象为线性系统 `A u = b`，其中：

- `A` 来自五点模板，是对称正定（SPD）离散算子
- 每个未知点仅与上下左右四邻耦合，天然稀疏
- 本实现采用“矩阵无关（matrix-free）”方式：不显式存储 `A`，只实现 `v -> A v` 的算子应用

这样可减少依赖并保持数值流程透明。

## R05

对 Dirichlet 边界的处理方式是“移项到右端”：

- 当内部点邻接边界点时，把已知边界值按 `+g/h^2` 加到 `b`
- 不把边界点当作未知量参与求解
- 求解完成后再把内部解与边界值拼接回完整网格解

这样既保持了未知量规模最小化，也让边界条件施加过程显式可审计。

## R06

`demo.py` 的主流程：

1. 构造网格与解析解对应的 `f, g`
2. 组装包含边界贡献的右端向量 `b`
3. 实现五点离散算子的矩阵无关应用 `A v`
4. 用共轭梯度法（CG）迭代求解 `A u = b`
5. 重建完整二维数值解 `u_num`
6. 与解析解比较，输出 `L2/Linf` 误差
7. 在多个网格尺度上估计经验收敛阶并记录 CG 收敛信息

## R07

正确性依据：

- 差分离散与原方程一致（Consistency）
- 离散矩阵为 SPD，离散问题解存在唯一
- 五点格式在规则网格上满足二阶逼近
- 当 `h` 变小时，数值解应向真解收敛，误差近似按 `O(h^2)` 下降

## R08

复杂度（令内部未知量总数 `N=n^2`）：

- 右端组装：`O(N)`
- 单次算子应用 `v -> A v`：`O(N)`
- 共轭梯度 `k` 次迭代总成本：`O(kN)`

MVP 以“透明可验证”优先，不追求极限性能。

## R09

数值性质与工程注意点：

- 条件数随网格加密增大，CG 迭代数通常也会增加
- 边界项若漏加或符号错写，会显著破坏精度
- 扁平化顺序（`(i,j)` 到一维索引）必须与算子应用一致
- 若需要更快收敛，可加入预条件器或多重网格

- `n = [8, 16, 32, 64]`（每个方向内部点数）
- 输出字段：`h`、未知量规模、`cg_iters`、相对残差、`l2_error`、`linf_error`、经验阶 `order_l2/order_linf`

## R11

MVP 设计取舍：

- 依赖最小：核心仅 `numpy`（`pandas` 仅用于更友好的可选输出）
- 不引入大型 PDE 框架，避免算法流程被封装隐藏
- 显式写出边界贡献、算子应用与 CG 迭代，便于检查每一步数学含义
- 牺牲部分性能优化，换取实现可读性与可审查性

## R12

## R14

期望现象：

- 当 `n` 翻倍（`h` 约减半）时，`l2_error` 与 `linf_error` 明显下降
- `order_l2`、`order_linf` 通常接近 2（受边界与离散误差影响会有小幅波动）
- 说明五点差分在该光滑椭圆问题上达到了预期二阶收敛

## R15

常见错误清单：

- 把 `-Delta u = f` 写成 `Delta u = f` 导致矩阵符号整体反向
- 边界贡献应加到右端而非减去（取决于离散方程写法）
- `reshape` 与索引展开顺序不一致，造成解场“扭曲”
- 在 `apply_operator` 中把 `h^2` 缩放放错位置，导致量纲不匹配
- 忘记把边界值回填到完整解中，后续误差评估失真

## R16

可扩展方向：

- 非均匀网格与变系数椭圆方程（如 `-div(a grad u)=f`）
- Robin/Neumann 边界条件离散
- 为 CG 增加预条件器（IC/Jacobi）或改用 GMRES
- 使用多重网格作为椭圆方程加速器
- 扩展到三维 `3D Poisson` 问题

## R17

与常见替代方案对比：

- 对比有限元法：FDM 在规则矩形域实现更直接；FEM 在复杂几何和非均匀介质上更灵活
- 对比谱方法：谱方法高精度但对几何/边界规则性要求高；FDM 更通用易落地
- 对比纯迭代法（Jacobi/GS）：FDM 是离散框架，迭代法是线性系统求解器，二者可组合
- 对比黑盒 PDE 库：MVP 更透明，适合教学、验证与算法基线构建

## R18

`demo.py` 的源码级算法流（非黑盒）如下：

1. `main` 设定网格序列 `n=[8,16,32,64]`，逐个调用 `solve_once`。
2. `solve_once` 计算步长 `h`，`build_rhs` 在内部点计算 `f`，并把四条边界的已知值按 `+g/h^2` 加入 `b`。
3. `apply_operator` 把一维向量 reshape 成 `n x n` 网格，按五点模板计算 `A v` 后再 flatten 回向量，实现矩阵无关离散算子。
4. `conjugate_gradient` 用 `x0=0` 初始化，计算初始残差 `r0=b-Ax0`，并令初始搜索方向 `p0=r0`。
5. 每次迭代调用 `apply_operator(p_k)` 得到 `A p_k`，据此计算步长 `alpha_k=(r_k^T r_k)/(p_k^T A p_k)`。
6. 更新解与残差：`x_{k+1}=x_k+alpha_k p_k`，`r_{k+1}=r_k-alpha_k A p_k`，并用残差范数判断收敛。
7. 若未收敛，计算 `beta_k=(r_{k+1}^T r_{k+1})/(r_k^T r_k)` 并更新方向 `p_{k+1}=r_{k+1}+beta_k p_k`，继续迭代。
8. 收敛后把内部解回填到完整网格，计算 `L2/Linf` 误差与经验阶，输出收敛表。

这 8 步完整展示了从椭圆方程离散到迭代求解与误差验证的数据流。
