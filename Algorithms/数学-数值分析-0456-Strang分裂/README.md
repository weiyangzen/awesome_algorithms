## R01

Strang 分裂（Strang Splitting）用于求解可拆分为两部分的演化方程：

`u_t = (A + B)u`

其中 `A` 与 `B` 分别代表两种不同机制（如扩散、反应、对流、势场等）。

核心更新格式是对称组合：

`u^{n+1} = exp((dt/2)A) exp(dt B) exp((dt/2)A) u^n`

或交换 `A/B` 顺序。该对称结构使方法在时间上达到二阶精度，是算子分裂中最常用的基线方法之一。

## R02

本条目 MVP 求解一维反应-扩散方程：

`u_t = D * u_xx + r * u * (1-u), x in [0,L)`（周期边界）

并做如下拆分：
- `A(u) = r*u*(1-u)`（反应子问题，点上可解析）
- `B(u) = D*u_xx`（扩散子问题，用 FFT 在频域精确推进）

输入是固定配置参数（网格数、`D`、`r`、`dt`、终止时间）；
输出是不同 `dt` 下终态解、相对细步长参考解的 L2 误差，以及与一阶 Lie 分裂的对照误差。

## R03

本实现采用的 Strang 时间推进：

1. 半步反应：`u* = Phi_A(dt/2, u^n)`
2. 整步扩散：`u** = Phi_B(dt, u*)`
3. 半步反应：`u^{n+1} = Phi_A(dt/2, u**)`

其中：
- Logistic 反应精确流：
  `Phi_A(tau,u) = u*exp(r*tau) / (1 + u*(exp(r*tau)-1))`
- 扩散频域流：
  若 `u_hat_k` 为傅里叶系数，则
  `u_hat_k(t+tau) = exp(-D*k^2*tau) * u_hat_k(t)`。

该组合避免了把整个 PDE 丢给黑箱求解器，子问题与分裂流程都可直接审计。

## R04

时间复杂度（设网格点数 `N`、总步数 `M`）：
- 每步主要成本是一次 FFT + 一次 IFFT，复杂度 `O(N log N)`；
- 反应半步是逐点代数更新，复杂度 `O(N)`；
- 总复杂度 `O(M * N log N)`。

`demo.py` 里还会额外跑若干 `dt` 与参考解，用于误差评估，因此总成本是常数倍放大。

## R05

空间复杂度：
- 状态向量 `u` 是 `O(N)`；
- 频域临时数组与波数数组也是 `O(N)`；
- 误差统计表只占常数级。

总体空间复杂度 `O(N)`。

## R06

单步结构微例（只展示分裂逻辑）：

取某点初值 `u=0.30`，参数 `r=2, dt=0.1`。

- 第一步半反应（`tau=0.05`）：
  `u1 = Phi_A(0.05,0.30) ≈ 0.3445`
- 第二步整扩散：
  在真实程序中此步由 FFT 对整个空间场统一处理，
  某点新值依赖全局谱系数，不能只靠局部点值闭式给出。
- 第三步半反应：
  对扩散后值再做 `Phi_A(0.05, ·)`。

这说明 Strang 不是“在原方程上一步显式 Euler”，而是“子流映射的对称复合”。

## R07

Strang 分裂的重要性：
- 把复杂多物理问题拆成更易求解的子问题；
- 在保持实现简洁的同时获得二阶时间精度；
- 对反应-扩散、薛定谔方程、Vlasov 型方程等都非常常见；
- 工程中常作为高阶方法或自适应策略的基础模块。

## R08

理论基础（简要）：
- 若精确演化算子为 `exp(dt(A+B))`，Strang 使用对称复合
  `S(dt)=exp(dtA/2)exp(dtB)exp(dtA/2)`；
- 基于 Baker-Campbell-Hausdorff（BCH）展开，局部截断误差是 `O(dt^3)`；
- 因此全局时间误差为 `O(dt^2)`；
- 当 `A` 与 `B` 可交换时，分裂误差进一步减弱；不可交换时该二阶结论仍成立。

## R09

适用条件与局限：
- 适用于 `A`、`B` 子问题可高效稳定推进的场景；
- 若某子问题本身近似很粗，整体精度会被该子步限制；
- 高刚性问题可能仍需更小 `dt` 或隐式/IMEX 变体；
- 边界条件处理若不与子算子一致，会引入额外误差。

## R10

正确性框架（工程可验证）：
1. 子流实现正确：`Phi_A` 是 Logistic 方程解析解，`Phi_B` 是扩散方程频域精确更新。  
2. 组合顺序正确：严格采用 `A(dt/2)->B(dt)->A(dt/2)`。  
3. 时间步拼接正确：循环 `M = T/dt` 次得到终态。  
4. 结果可验证：用更小 `dt` 的 Strang 结果作参考，检查误差随 `dt` 下降。  
5. 对照基准：同 `dt` 下与一阶 Lie 分裂比较，Strang 误差应更小。

## R11

误差与稳定性：
- 分裂时间误差：二阶 `O(dt^2)`；
- 空间离散：扩散子步采用谱方法，对光滑周期解精度高；
- 反应子步为解析流，无子步离散误差；
- 稳定性主要受模型参数与时间步影响，`demo.py` 通过有限值检查与误差单调性检查做守护。

## R12

性能视角：
- 核心开销集中在 FFT；
- 与“整方程隐式求解”相比，本 MVP 代码短、可解释性高；
- 与纯显式差分相比，扩散子步可用更稳健的频域精确推进；
- 对中等规模教学/验证任务，NumPy 即可满足。

## R13

本目录可验证保证：
- `demo.py` 无交互输入，固定参数可重复；
- 输出每个 `dt` 的
  - `strang_L2_error`
  - `lie_L2_error`
  - `min(u), max(u)`；
- 内置断言包括：
  - Strang 误差随 `dt` 细化严格下降；
  - 相邻误差比大于阈值（体现二阶趋势）；
  - 同 `dt` 下 Strang 误差小于 Lie 误差；
  - 解保持有限，且数值范围不过度越界。

## R14

鲁棒性与常见失效模式：
- `dt` 不能整除 `final_time` 时立即报错；
- 非法参数（负扩散系数、非有限值、网格过小）会在配置阶段报错；
- 若步长过大导致数值异常，会触发 `NaN/Inf` 运行时检查；
- 若更换为非周期问题，需同步改写扩散子步与边界处理策略。

## R15

实现设计（`demo.py`）：
- `SimulationConfig`：参数集中管理与校验；
- `reaction_flow_logistic`：反应子流解析更新；
- `diffusion_flow_fft`：扩散子流频域更新；
- `strang_step`、`lie_step`：二阶与一阶分裂单步；
- `simulate`：主循环推进；
- `build_report`：生成误差对照表；
- `run_checks`：执行自动正确性断言。

实现中没有调用 SciPy 的黑箱 PDE 求解器，关键算法均在源码中显式展开。

## R16

相关算法链路：
- 一阶基线：Lie-Trotter 分裂；
- 高阶扩展：Yoshida/Composition 型高阶对称分裂；
- 其他时间推进：IMEX、指数积分、Crank-Nicolson；
- 应用场景：反应-扩散、量子动力学、等离子体与输运方程。

## R18

源码级算法流程拆解（对应 `demo.py`，8 步）：

1. `main` 创建 `SimulationConfig`，在 `__post_init__` 中校验所有参数与时间步可整除性。  
2. `build_report` 先调用 `simulate(..., reference_dt, scheme="strang")` 生成细步长参考终态 `u_ref`。  
3. `simulate` 内构造周期网格与波数 `k`，并计算 `k^2` 供扩散子步使用。  
4. `initial_condition` 生成 `[0,1]` 范围内的平滑初值场。  
5. 每个时间步执行 `strang_step`：先 `reaction_flow_logistic(dt/2)`，再 `diffusion_flow_fft(dt)`，再 `reaction_flow_logistic(dt/2)`。  
6. `diffusion_flow_fft` 里显式执行 `FFT -> 乘 exp(-D*k^2*dt) -> IFFT`，完成扩散精确流。  
7. 对每个候选 `dt`，同时计算 Strang 与 Lie 终态，基于 `l2_error` 得到误差并写入表格。  
8. `run_checks` 验证误差下降、误差比阈值、Strang 优于 Lie 及解范围稳定，最后由 `print_report` 输出结果。
