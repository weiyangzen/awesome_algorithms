# 洛伦兹吸引子 (Lorenz Attractor)

- UID: `PHYS-0118`
- 学科: `物理`
- 分类: `混沌理论`
- 源序号: `118`
- 目标目录: `Algorithms/物理-混沌理论-0118-洛伦兹吸引子_(Lorenz_Attractor)`

## R01

洛伦兹吸引子是三维常微分方程系统在特定参数下出现的经典混沌吸引子，标准形式为：

`dx/dt = sigma * (y - x)`  
`dy/dt = x * (rho - z) - y`  
`dz/dt = x * y - beta * z`

在常用参数 `sigma=10, rho=28, beta=8/3` 下，系统呈现对初值高度敏感、轨道长期有界但不可周期预测的动力学行为。本条目实现一个可审计的最小 MVP：显式 RK4 数值积分 + Lyapunov 指数估计 + 参考解交叉验证。

## R02

历史定位（简要）：

- 1963 年 Edward Lorenz 在简化大气对流模型时提出该系统；
- 它是“确定性系统也可产生不可长期预测行为”的代表案例；
- “蝴蝶效应”在科学传播中常以洛伦兹系统作为可视化原型；
- 对现代混沌理论、非线性动力系统、可预报性研究有基础意义。

## R03

MVP 输入输出约定（无交互输入）：

- 输入（脚本内部固定）：
1. 参数 `sigma=10, rho=28, beta=8/3`；
2. 初值 `x0=[1, 1, 1]`；
3. 时间步长 `dt=0.01`，总步数 `10000`，前 `3000` 步作为热身段；
4. Lyapunov 估计扰动量 `d0=1e-8`。
- 输出：
1. 轨道基本统计（均值、标准差、坐标包围盒、最终状态）；
2. 最大 Lyapunov 指数估计值；
3. 与 `scipy.solve_ivp` 的短时参考误差；
4. 自动校验通过标记 `All checks passed.`。

## R04

典型应用：

- 非线性动力系统课程中的混沌入门示例；
- 数值积分器稳定性对比（Euler/RK4/自适应方法）；
- 对初值敏感性实验（误差增长与可预报时间估计）；
- 吸引子重构、相空间分析、时间序列非线性特征研究。

## R05

数学要点：

- 系统是自治连续时间动力系统 `x'(t)=f(x(t))`；
- 在经典参数组下存在奇异平衡点，但典型轨道不会收敛到单一平衡点，而是在奇异吸引子上演化；
- 对两条近邻轨道，距离近似满足 `d(t) ~ d(0) * exp(lambda * t)`；
- 若最大 Lyapunov 指数 `lambda_max > 0`，可视为混沌行为的重要证据之一；
- 数值上常用 Benettin 重标定法估计 `lambda_max`。

## R06

本实现核心流程：

1. 定义洛伦兹系统右端函数 `f(x)`；
2. 用四阶 Runge-Kutta（RK4）推进单步；
3. 循环积分得到完整轨道 `trajectory`；
4. 丢弃热身段后统计吸引子区域的均值/方差/包围盒；
5. 用微扰双轨道 + 每步重标定估计最大 Lyapunov 指数；
6. 用 `solve_ivp` 在短时间窗内做参考解交叉验证；
7. 通过断言确保结果有限、有界、可复现且呈现正 Lyapunov。

## R07

工程细节（MVP 关键判据）：

- 输入校验：参数有限、`dt > 0`、`steps > 0`、初值为三维向量；
- 采用固定步长 RK4，保证实现短小且流程透明；
- 混沌系统长期会轨道分离，因此与 `solve_ivp` 只做短时比较，避免“正确实现被长期指数分离误判”；
- Lyapunov 估计每步重标定到 `d0`，避免浮点下溢/上溢；
- 加入确定性复跑检查（同配置两次积分结果一致）。

## R08

时间复杂度（设积分步数为 `N`）：

- 每个 RK4 步需要常数次向量运算，复杂度 `O(1)`；
- 主轨道积分 `O(N)`；
- Lyapunov 双轨道估计同量级 `O(N)`；
- 总体时间复杂度 `O(N)`，常数项来自 RK4 的 4 次斜率评估。

## R09

空间复杂度：

- 若保存整段轨道，轨道数组大小约为 `(N+1) x 3`，空间 `O(N)`；
- 若只做在线统计可降到 `O(1)`，但本 MVP 为可审计性保留全轨道；
- 额外临时变量（状态、斜率、误差指标）均为常数空间。

## R12

输出字段说明：

- `params`：洛伦兹系统参数 `(sigma, rho, beta)`；
- `dt / steps / burn_in`：积分步长、总步数与热身段长度；
- `final_state`：积分结束时状态向量；
- `mean_state_after_burnin`：热身后轨道均值；
- `std_state_after_burnin`：热身后轨道标准差；
- `bbox_min / bbox_max`：热身后三维包围盒最小/最大坐标；
- `wing_switches`：`x` 坐标符号切换次数（左右翼切换粗指标）；
- `largest_lyapunov`：最大 Lyapunov 指数估计；
- `reference_short_horizon_rmse`：与 `solve_ivp` 短时对照误差。

## R13

鲁棒性与边界处理：

- 对非有限参数、非法步长、错误维度输入立刻抛异常；
- 每步积分后检查有限性，发现数值爆炸立即失败；
- Lyapunov 估计时限制最小距离阈值，避免除零；
- 对轨道幅值设置上界检查，防止无意参数导致发散误判；
- 输出前执行多项断言，不满足就中止而非静默输出错误结果。

## R14

当前局限：

- 固定步长 RK4 不是自适应误差控制方法；
- 这里只估计最大 Lyapunov 指数，未计算完整 Lyapunov 谱；
- 未做分岔扫描（如对 `rho` 的参数连续扫描）；
- 未实现可视化绘图（例如三维相图），聚焦于最小可运行与数值校验。

## R16

最小测试建议：

- 正确性：与 `solve_ivp` 的短时轨道均方根误差应足够小；
- 有界性：热身后轨道坐标绝对值应落在合理范围（经典参数通常远小于 100）；
- 混沌性信号：最大 Lyapunov 指数应显著大于 0；
- 可复现性：同参数同初值重复运行结果一致；
- 输入鲁棒性：非法维度、`dt<=0`、`steps<=0` 应触发异常。

## R17

方案对比：

- 显式 Euler：实现最简单，但精度和稳定性不足；
- RK4（本实现）：在“实现复杂度/精度/可审计性”上平衡较好；
- `solve_ivp` 自适应方法：工程稳健但内部步骤更黑箱，不利于教学拆解；
- 辛积分器：更适于哈密顿系统，洛伦兹系统是耗散系统，收益不如 RK4 明确。

## R18

`demo.py` 源码级算法流程（8 步）：

1. `LorenzParams` 定义 `(sigma, rho, beta)`，`lorenz_rhs()` 返回三维导数向量。  
2. `rk4_step()` 对当前状态执行 `k1/k2/k3/k4` 四次斜率评估并合成下一步状态。  
3. `integrate_lorenz()` 分配轨道数组，循环 `steps` 次调用 `rk4_step()`，逐步生成 `trajectory[t]`。  
4. `summarize_trajectory()` 在去除热身段后计算均值、标准差、包围盒与翼间切换次数。  
5. `estimate_largest_lyapunov()` 同步推进“基准轨道 + 微扰轨道”，每步记录 `log(dist/d0)` 后重标定距离，最后按总时间归一化得到 `lambda_max`。  
6. `reference_error_against_scipy()` 用同初值在短时间窗调用 `solve_ivp`，与 RK4 轨道做 RMSE 对照。  
7. `run_checks()` 统一执行有限性、有界性、Lyapunov 正值、参考误差、确定性复跑等断言。  
8. `main()` 组织参数与调用顺序，打印关键指标并在所有断言通过后输出 `All checks passed.`。  

说明：第三方库并未替代主算法；RK4 推进、Lyapunov 估计、统计与校验路径均在源码中展开为可追踪步骤。
