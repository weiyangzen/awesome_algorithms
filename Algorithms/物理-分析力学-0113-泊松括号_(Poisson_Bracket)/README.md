# 泊松括号 (Poisson Bracket)

## R01

泊松括号是哈密顿力学中的核心双线性算子，用来度量两个相空间函数（观测量）之间的“正则耦合关系”。

对 `n` 自由度系统，设相空间坐标为 `(q_1,...,q_n,p_1,...,p_n)`，定义：

`{f,g} = sum_i (∂f/∂q_i * ∂g/∂p_i - ∂f/∂p_i * ∂g/∂q_i)`。

## R02

在分析力学里，泊松括号统一连接了三件事：

- 结构层：正则坐标与辛结构；
- 动力学层：哈密顿方程 `dot(A)={A,H}+∂A/∂t`；
- 守恒层：若 `{A,H}=0` 且 `A` 无显式时依，则 `A` 守恒。

因此它不仅是一个公式，更是“观测量演化规则”。

## R03

本条目 MVP 目标是把泊松括号从定义落到可执行验证：

- 用数值差分实现通用 `poisson_bracket(f,g,z)`；
- 自动检验基本正则关系 `{q_i,q_j}=0`、`{p_i,p_j}=0`、`{q_i,p_j}=delta_ij`；
- 自动检验 Jacobi 恒等式残差；
- 用 `{q,H}`、`{p,H}` 驱动一维简谐振子并对比解析解。

## R04

`demo.py` 使用的核心关系：

1. 泊松括号定义：`{f,g}=sum_i(df/dq_i*dg/dp_i-df/dp_i*dg/dq_i)`。
2. 哈密顿方程：`dot(q_i)={q_i,H}`，`dot(p_i)={p_i,H}`。
3. 简谐振子哈密顿量：`H(q,p)=p^2/(2m)+k*q^2/2`。
4. 解析轨道：
   `q(t)=q0*cos(omega t)+p0/(m*omega)*sin(omega t)`，
   `p(t)=p0*cos(omega t)-m*omega*q0*sin(omega t)`，
   `omega=sqrt(k/m)`。
5. Jacobi 恒等式：`{f,{g,h}}+{g,{h,f}}+{h,{f,g}}=0`。

## R05

设自由度为 `n`，每次函数评估成本为 `C_f`，轨道步数为 `T`。

- 单次泊松括号：每个 `i` 需要 4 个偏导，每个偏导用中心差分 2 次评估，总体约 `8n` 次评估，复杂度 `O(n*C_f)`；
- 正则关系验证：固定维度小矩阵，近似 `O(n^2)` 次括号调用；
- Jacobi 验证：每个采样点常数次嵌套括号，复杂度 `O(S*n*C_f)`（`S` 为采样点数）；
- RK4 演化：每步 4 次向量场评估，整体 `O(T*n*C_f)`。

本 MVP 维度很小，运行耗时在秒级。

## R06

脚本输出三组报告：

- 正则基本括号最大误差；
- Jacobi 残差的最大值与均值，并给出样本行；
- 简谐振子轨道误差、能量相对漂移、哈密顿方程右端一致性 RMSE。

最后统一打印阈值检查并给出 `Validation: PASS/FAIL`。

## R10

正确性直觉：

1. 对线性坐标函数 `q_i,p_i`，差分偏导应恢复常数导数，因此基本括号应精确接近理论值；
2. Jacobi 恒等式是泊松结构一致性的核心约束，残差应接近 0；
3. 若 `dot(z)={z,H}` 实现正确，简谐振子数值轨道应贴近解析解；
4. 对自治哈密顿系统，`H` 应近似守恒，能量漂移应保持较小。

## R11

数值稳定策略：

- 偏导使用中心差分而非前向差分，降低截断误差；
- `fd_eps` 设为中间量级（过小会放大舍入误差，过大会放大截断误差）；
- 时间积分选 RK4 并用小步长 `dt`，平衡精度和成本；
- 用多指标联合判定（轨道误差、能量漂移、方程残差），避免单指标“假通过”。

## R14

常见失效模式：

1. `fd_eps` 过小导致数值消减误差，Jacobi 残差放大；
2. `fd_eps` 过大导致偏导近似粗糙，基本括号偏离；
3. `dt` 过大导致轨道相位误差与能量漂移上升；
4. 相空间向量维度与 `2*n_dof` 不一致会导致括号计算逻辑错误。

## R18

`demo.py` 的源码级算法流程（8 步）：

1. `central_partial_derivative` 用中心差分计算任意观测量对相空间坐标的偏导。  
2. `poisson_bracket` 按定义式累加各自由度项，得到 `{f,g}` 数值值。  
3. `q_observable`/`p_observable` 构造坐标观测量，`canonical_bracket_report` 逐项验证 `{q_i,q_j}`、`{p_i,p_j}`、`{q_i,p_j}`。  
4. `bracket_observable` 把括号结果再次封装成观测量，`jacobi_identity_report` 计算 `J={f,{g,h}}+{g,{h,f}}+{h,{f,g}}` 残差。  
5. `hamiltonian_1d_harmonic` 定义 `H=p^2/(2m)+kq^2/2`，`hamiltonian_vector_field` 通过 `{q,H}`、`{p,H}` 生成动力学右端。  
6. `rk4_step` 对 `dot(z)=X_H(z)` 做四阶 Runge-Kutta 推进，构建数值轨道。  
7. `harmonic_oscillator_report` 将数值轨道与解析解对比，统计 `q/p` 最大误差、能量漂移与方程残差 RMSE。  
8. `main` 汇总三类报告，执行阈值检查并输出 `Validation: PASS/FAIL`。  

第三方库没有被当作“结论黑盒”：`numpy/pandas` 仅承担数值数组与表格展示，泊松括号、Jacobi 结构检验、哈密顿向量场与积分验证逻辑均在源码中显式实现。
