## R01

问题定义：实现一个可运行、可审计的单量子比特最小 MVP。

本条目关注如下核心能力：
- 用明确的数学表示描述单比特纯态 `|ψ>` 与密度矩阵 `ρ`；
- 用 Born 规则计算在 `X/Y/Z` 基下的测量概率；
- 用有限采样（shots）模拟测量统计波动；
- 施加简单幺正门后，比较演化前后的 Bloch 向量与测量分布。

## R02

理论背景：量子比特是二维复希尔伯特空间中的量子态，最常见写法为

`|ψ> = α|0> + β|1>,  α,β∈C, |α|^2 + |β|^2 = 1`

对纯态可转为密度矩阵：

`ρ = |ψ><ψ|`

单比特还可等价表示为 Bloch 球形式：

`ρ = (I + r·σ)/2,  r=(rx,ry,rz), |r|<=1`

其中纯态满足 `|r|=1`，混合态满足 `|r|<1`。

## R03

本 MVP 的具体计算任务：
1. 由角参数 `(θ, φ)` 生成单比特纯态；
2. 将纯态转换为密度矩阵并检查物理合法性（Hermitian、迹为 1、半正定）；
3. 在 `X/Y/Z` 轴上计算理论测量概率 `P(+1), P(-1)`；
4. 对每个测量轴进行 Monte Carlo 采样并估计经验期望值；
5. 对状态施加门序列 `H -> Rz` 后重复上面流程并比较差异。

## R04

建模假设（MVP 级别）：
- 仅处理单比特系统，希尔伯特空间维度固定为 2；
- 只演示理想投影测量，不引入器件误差、暗计数、漂移等实验噪声；
- 统计不确定性仅由有限样本数 `shots` 导致；
- 只展示最小门集（`H` 与 `Rz`）来体现可控演化。

这些假设用于教学与算法演示，不等同于完整实验平台建模。

## R05

关键公式：

1. 角参数态构造：
`|ψ(θ,φ)> = cos(θ/2)|0> + exp(iφ) sin(θ/2)|1>`

2. 密度矩阵：
`ρ = |ψ><ψ|`

3. Bloch 分量：
`rx = Tr(ρσx), ry = Tr(ρσy), rz = Tr(ρσz)`

4. Born 规则（测量轴 `n`）：
- 投影算符 `Π_± = (I ± n·σ)/2`
- 概率 `P(±1) = Tr(ρ Π_±)`
- 期望 `E = P(+1)-P(-1) = n·r`

5. 幺正演化：
`ρ' = UρU†`

## R06

`demo.py` 的执行流程：
1. 定义 Pauli 矩阵、测量轴、默认参数与门矩阵；
2. 由 `(θ, φ)` 构造初态 `|ψ>` 并转成 `ρ`；
3. 检查 `ρ` 的合法性并提取 Bloch 向量；
4. 在 `X/Y/Z` 轴计算理论概率和理论期望；
5. 在每个轴做有限抽样，得到采样期望与计数；
6. 对 `ρ` 施加 `H` 再施加 `Rz(θg)` 获得新态 `ρ'`；
7. 对新态重复诊断与测量；
8. 输出 DataFrame 表格并执行断言校验。

## R07

复杂度分析（记测量轴数量为 `A`，每轴抽样次数为 `K`）：
- 单次理论概率计算是 `2x2` 矩阵运算，视作 `O(1)`；
- 抽样阶段每轴 `K` 次，复杂度 `O(K)`；
- 总复杂度约为 `O(AK)`，在本脚本 `A=3` 时基本由抽样主导；
- 空间复杂度为 `O(K)`（主要是保存抽样结果向量）。

默认参数 `K=5000` 时本地开销很小。

## R08

数值稳定性策略：
- 所有核心矩阵使用 `complex128`；
- 对概率做 `clip + normalize`，避免浮点误差导致负概率或和不为 1；
- 通过 `ATOL=1e-10` 判断 Hermitian 偏差、迹偏差与等式误差；
- 固定随机种子 `seed=2026` 以保证采样结果可复现。

## R09

适用场景：
- 量子信息课程中的单比特表示与测量演示；
- 量子算法原型开发前的线性代数 sanity check；
- 检验“Bloch 向量 <-> Born 概率”计算链路是否一致。

不适用场景：
- 多比特纠缠系统或量子线路大规模仿真；
- 噪声通道拟合与实验级误差建模；
- 量子硬件脉冲级控制优化。

## R10

脚本内置正确性检查：
1. `is_valid_density` 验证密度矩阵三条件：Hermitian、迹为 1、最小本征值非负（容差内）；
2. 检查 `E_theory` 与 `E_from_bloch` 的最大差值 `< 1e-10`；
3. 检查每个测量轴的 `P(+1)+P(-1)=1`；
4. 检查采样期望与理论期望偏差上界（默认 `< 0.08`）。

全部断言通过时输出 `All checks passed.`。

## R13

正确性边界说明：
- 本实现只覆盖单比特；不能直接外推到纠缠态与多比特电路；
- 仅示范纯态主流程，混合态仅通过通用密度矩阵接口“可表示”，未做完整噪声模型；
- 采样误差阈值依赖 `shots`，若样本过少断言可能不稳定；
- 相位全局因子不可观测，本脚本通过密度矩阵计算自然规避该歧义。

## R14

常见失败模式与修复：
- 失败：状态范数为 0 或异常输入导致 `normalize_state` 报错。
  - 修复：保证输入振幅至少一个非零并做参数校验。
- 失败：概率和偏离 1。
  - 修复：先 `clip` 再归一化，避免浮点噪声扩散。
- 失败：采样偏差超过阈值。
  - 修复：增大 `shots_per_axis` 或调整误差上限。
- 失败：密度矩阵最小本征值出现微负值。
  - 修复：确认幺正演化链路和数值精度，必要时使用更宽容差。

## R18

`demo.py` 源码级算法流拆解（8 步，非黑盒）：
1. 定义 `I, σx, σy, σz` 与 `X/Y/Z` 测量轴，建立统一线性代数基础。
2. 通过 `ket_from_angles(θ,φ)` 生成纯态振幅，并在 `density_from_ket` 中显式构造 `ρ=|ψ><ψ|`。
3. `is_valid_density` 逐项检查 Hermitian 偏差、迹偏差和最小本征值，确保状态物理合法。
4. `bloch_vector_from_density` 用 `Tr(ρσi)` 提取 Bloch 三分量，实现表示转换。
5. `projector` 与 `probabilities_for_axis` 按 Born 规则计算 `P(+1),P(-1)`，不依赖任何量子 SDK 黑盒。
6. `sampled_expectation` 用 `rng.choice` 对 ±1 测量结果做有限次采样，得到经验期望与计数统计。
7. `apply_unitary` 用 `ρ' = UρU†` 执行门演化，本例中 `U = Rz * H`，得到演化后新态。
8. `main` 汇总诊断表和测量表，执行一致性断言（Born 与 Bloch 一致、概率归一、采样误差上界）并输出最终结果。
