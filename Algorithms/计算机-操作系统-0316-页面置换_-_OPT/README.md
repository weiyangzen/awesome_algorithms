## R01

OPT（Optimal Page Replacement，最优页面置换，也称 Belady 最优）用于在缺页且页框已满时选择被淘汰页面。其决策原则是：淘汰“下一次被访问时间最晚”的驻留页；如果某页未来不再访问，则优先淘汰该页。该策略在同一访问序列与同一页框容量下，理论上给出最少缺页次数。

## R02

本问题属于操作系统虚拟内存管理中的经典基准策略：
- OPT 不是现实系统可直接实现的在线算法，因为它依赖未来访问信息；
- 但它是评估 FIFO/LRU/Clock 等在线策略性能上界的重要参照；
- 教学与实验中常用 OPT 来衡量某策略“距离最优有多远”。

## R03

输入与输出定义：

- 输入：
  - 页面访问序列 `reference = [p0, p1, ...]`（非负整数）；
  - 页框数量 `frame_count > 0`。
- 输出：
  - `hits`（命中次数）、`faults`（缺页次数）、命中率与缺页率；
  - 每一步决策轨迹（命中/缺页、被替换页、被替换页下一次访问位置、当前页框状态）。

`demo.py` 固定输入直接运行，无需交互。

## R04

本实现采用“未来位置队列 + 线性评估受害页”的透明实现：
- 先预处理每个页面在访问序列中的出现下标队列；
- 扫描到第 `i` 步时，先弹出当前页的下标 `i`，保证队列头部代表真实未来；
- 缺页且满框时，对每个驻留页读取“下一次出现下标”；若不存在则记为 `inf`；
- 选择 `next_use` 最大者淘汰（同值时按较小槽位下标稳定打破平局）。

## R05

MVP 主流程：

1. 校验输入并初始化页框状态。
2. 预构建 `future_positions`（页面 -> 未来出现位置队列）。
3. 逐步处理访问请求，先更新当前页的未来队列。
4. 命中时仅计数。
5. 缺页且有空框时直接装入。
6. 缺页且无空框时遍历驻留页，计算各自下一次访问位置。
7. 选取“下一次访问最晚”的页面替换并记录事件。
8. 汇总统计并打印对照结果。

## R06

固定样例（`demo.py`）：
- `reference = [7, 0, 1, 2, 0, 3, 0, 4, 2, 3, 0, 3, 2]`
- `frame_count = 3`

结果：
- OPT：`faults = 7`, `hits = 6`
- LRU：`faults = 9`, `hits = 4`
- FIFO：`faults = 10`, `hits = 3`

该结果体现了 OPT 作为理论最优基线的作用。

## R07

复杂度分析（访问长度 `n`，页框数 `m`）：
- 预处理未来位置：`O(n)`；
- 每次缺页时扫描页框选择受害页：`O(m)`；
- 总体时间复杂度：最坏 `O(nm)`；
- 空间复杂度：
  - 未来位置索引 `O(n)`；
  - 页框状态 `O(m)`；
  - 若保留全量事件日志则额外 `O(n)`。

## R08

边界与异常处理：
- `frame_count <= 0`：抛出 `ValueError`；
- 访问序列出现负页号：抛出 `ValueError`；
- 空访问序列：命中/缺页均为 0；
- `frame_count >= 不同页面数`：初始装填后基本全命中；
- `frame_count = 1`：除连续重复外通常频繁缺页。

## R09

OPT 与常见策略对比：
- 对比 FIFO：OPT 会利用未来信息，通常显著更少缺页；
- 对比 LRU：LRU 依赖历史最近性，OPT 依赖未来最远性；
- 对比 Clock：Clock 是低开销近似 LRU，通常进一步偏离 OPT；
- 工程意义：OPT 不可在线实现，但可作为“理论下界”用于评估策略质量。

## R12

关键输出字段说明：
- `action`：`hit` / `fault_fill` / `fault_replace`；
- `replaced`：被替换页（命中或空框填充时为空）；
- `replaced_next_use`：被替换页下一次访问下标，若为 `inf` 表示“不再访问”；

- `fault_ratio`：`faults / len(reference)`。

这些字段可直接用于人工核对 OPT 决策是否合理。

## R14

常见实现错误与防护：
- 错误：未弹出当前访问下标，导致“未来位置”被污染。  
  防护：每步先 `popleft()` 当前页位置。

- 错误：把“下一次访问最早”当成受害页，实际与 OPT 相反。  
  防护：受害页选择使用 `max(next_use)`。

- 错误：未来不再访问未特殊处理，导致决策偏差。  
  防护：未再访问统一视为 `inf`。

- 错误：平局策略不确定，复现实验困难。  
  防护：同 `next_use` 下按较小槽位下标固定裁决。

## R18

`demo.py` 中 OPT 的源码级流程（8 步）：

1. `main()` 构造固定访问序列与页框数，并调用 `opt_page_replacement`。  
2. `opt_page_replacement` 先通过 `_build_future_positions` 建立“页面 -> 出现位置队列”。  
3. 每处理一步访问，先对当前页执行 `popleft()`，把“当前时刻”从未来队列中移除。  
4. 若页已在 `frames` 中，记一次 `hit`，不做替换。  
5. 若缺页且仍有空框，执行 `fault_fill` 直接装入。  
6. 若缺页且无空框，遍历每个驻留页读取 `next_use`（无未来访问则记为 `inf`）。  
7. 通过 `max(next_use)` 选出受害页并替换（平局按槽位下标稳定处理），记录 `replaced` 与 `replaced_next_use`。  
8. 全序列完成后汇总 `hits/faults/fault_ratio`，并在 `main()` 中与 FIFO/LRU 对照及断言校验。

第三方库边界：
- `numpy` 只用于生成随机访问序列；
- `pandas` 只用于表格化展示；
- OPT 决策主干（未来索引、受害页选择、替换）完全由源码实现，不是黑盒调用。
