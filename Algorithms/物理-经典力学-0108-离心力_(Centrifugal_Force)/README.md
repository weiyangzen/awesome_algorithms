# 离心力 (Centrifugal Force)

## R01

离心力是**旋转参考系中的惯性力（伪力）**。在角速度为 `Ω` 的非惯性系里，质点位置为 `r` 时，离心加速度写作：

`a_cf = - Ω × (Ω × r)`

当 `r` 与转轴垂直时，其大小化为：

`|a_cf| = ω^2 r`

对应离心力大小：`F_cf = m ω^2 r`，方向沿径向向外（远离转轴）。

## R02

经典力学里，离心力并非“真实相互作用力”，而是为了在旋转系中继续使用牛顿第二定律而引入的补偿项。它常与向心力一起出现：

- 惯性系看：物体做圆周运动需要向心力（指向圆心）；
- 旋转系看：引入离心力后，很多稳态问题可写成静力平衡。

因此它在旋转机械、离心分离、转台实验和车辆转弯分析中非常实用。

## R03

本条目要解决的 MVP 问题：

- 给定 `m`、`ω`、`r`，计算离心加速度/离心力（向量与标量两种视角）；
- 在“质量块 + 径向弹簧 + 匀速旋转”系统中，求并验证平衡半径；
- 使用数值仿真（非交互）验证解析解，形成可运行最小闭环。

## R04

核心方程：

1. 旋转系动力学（匀速转动、忽略欧拉力）
   `m a_rot = F_real + F_cf + F_cor`
2. 离心力
   `F_cf = m[-Ω × (Ω × r)]`
3. 本 MVP 的一维径向弹簧模型（旋转系，含线性阻尼 `c`）
   `m r¨ = -k(r - L0) + mω^2 r - c r˙`
4. 稳态（`r˙ = r¨ = 0`）解析半径
   `r_eq = k L0 / (k - mω^2)`，要求 `k > mω^2`。

## R05

设角速度采样数量为 `n`，每个角速度仿真步数为 `T`：

- 向量离心力批量计算：`O(n)`；
- 旋转弹簧时间推进：`O(nT)`；
- 空间复杂度：若只保留当前状态则 `O(1)`，本实现保留时序用于检查为 `O(T)`。

该复杂度对教学与小规模工程计算足够。

## R09

适用场景：

## R10

正确性直觉：

1. `-Ω×(Ω×r)` 天然给出“垂直于转轴方向的径向外推”项；
2. 对纯圆周问题，离心力大小与 `mω^2r` 一致；
3. 在旋转弹簧系统里，稳态解由 `dU_eff/dr = 0` 得到，与力平衡一致；
4. 数值仿真加入阻尼后应收敛到该稳态，作为实现正确性的动态证据。

## R11

数值稳定与精度策略：

- 统一使用 `float64`；
- Euler 步长 `dt` 取较小值，避免振荡发散；
- 阻尼项 `-c r˙` 用于抑制长时振荡并加速收敛；
- 用误差阈值断言（而非追求机器零误差）评估解析-数值一致性。

## R12

关键参数：

- `m`：质量；
- `k`：弹簧刚度；
- `L0`：自然长度；
- `ω`：角速度；
- `dt`、`steps`：离散积分步长与步数；
- `damping`：阻尼系数。

调参建议：

- 若误差偏大，优先减小 `dt` 或增大 `steps`；
- 若收敛慢，可适度增大 `damping`；
- 必须满足 `k > mω^2`，否则线性模型平衡半径失去有界解。

## R14

常见失效模式：

1. `k <= mω^2` 导致分母非正，解析平衡半径无界/失效；
2. `dt` 过大导致 Euler 发散或大幅振荡；
3. 阻尼过小导致收敛过慢，给人“公式不对”的错觉；
4. 误把离心力当作惯性系真实外力，导致受力图混淆。

## R18

`demo.py` 的源码级算法流程（8 步）：

1. `centrifugal_acceleration` 计算 `-Ω×(Ω×r)`，返回旋转系离心加速度向量。  
2. `centrifugal_force` 将加速度乘以质量，得到离心力向量。  
3. `centrifugal_force_magnitude` 给出标量公式 `mω^2r`，作为向量实现的对照基准。  
4. `spring_equilibrium_radius` 根据 `kL0/(k-mω^2)` 给出旋转弹簧系统解析平衡半径，并检查可行域 `k>mω^2`。  
5. `simulate_radial_dynamics` 用显式 Euler 推进 `m r¨ = -k(r-L0)+mω^2r-c r˙`，生成径向时序。  
6. `run_vector_consistency_demo` 对多组位置点计算离心力，逐点检查方向（向外）与大小（匹配 `mω^2r`）。  
7. `run_rotating_spring_demo` 扫描多组角速度，输出解析半径与仿真末态半径，并检查误差阈值。  
8. `main` 汇总打印并执行断言（误差与单调性），保证脚本可在非交互模式下一次性验证通过。

整个流程未把第三方库当黑盒：核心动力学方程、离散更新、解析解和校验逻辑都在源码中逐步展开。
