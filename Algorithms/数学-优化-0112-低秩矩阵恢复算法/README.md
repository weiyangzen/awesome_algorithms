## R01

低秩矩阵恢复（Low-Rank Matrix Recovery）关注：在观测不完整或含噪的情况下，重建一个潜在低秩矩阵 `M`。

典型建模（矩阵补全场景）为：
- 已知观测集合 `Omega` 上的元素 `P_Omega(M)`；
- 求解 `X` 使其在观测位置拟合数据，同时保持低秩。

常见优化形式：
- 原始（非凸）目标：`min rank(X), s.t. P_Omega(X)=P_Omega(M)`；
- 凸松弛：`min ||X||_* , s.t. P_Omega(X)=P_Omega(M)`；
- 含噪版本：`min 0.5*||P_Omega(X-M)||_F^2 + lambda*||X||_*`。

其中 `||X||_*` 是核范数（奇异值之和），是秩函数的凸替代。

## R02

该方向起源于压缩感知与凸优化交叉研究。核心里程碑是核范数最小化用于矩阵补全与鲁棒主成分分解（RPCA）等问题，推动了推荐系统、成像重建和系统辨识等应用。

工程上常见算法族包括：
- SVT（Singular Value Thresholding）；
- 近端梯度 / 加速近端梯度（ISTA / FISTA）；
- 交替最小化（因子分解视角，非凸）；
- ADMM 类方法。

本条目采用最小可运行且可解释的 SVT/近端梯度实现。

## R03

要解决的核心矛盾：
1. 仅有部分条目可见，直接插值通常不稳定。
2. 直接最小化 `rank(X)` 是组合优化，求解困难。
3. 观测可能带噪，需要拟合与模型复杂度平衡。

核范数正则提供了可计算的凸目标：
- 数据项保证观测一致性；
- 核范数项抑制高秩噪声成分；
- 通过 `lambda` 控制“拟合-低秩”折中。

## R04

本 MVP 使用的迭代流程（近端梯度 + 奇异值软阈值）：
1. 初始化 `X0`（通常全零）。
2. 计算观测位置梯度：`Gk = P_Omega(Xk - M_obs)`。
3. 梯度步：`Yk = Xk - eta*Gk`。
4. 近端步（核范数）：
   - 对 `Yk` 做 SVD：`Yk = U*diag(s)*V^T`；
   - 软阈值 `s' = max(s - eta*lambda, 0)`；
   - 重构 `X_{k+1} = U*diag(s')*V^T`。
5. 若相对变化量足够小则停止，否则继续。

这就是典型 SVT 机制：通过对奇异值做 shrinkage 达到“自动降秩”。

## R05

设矩阵大小为 `m x n`，`k=min(m,n)`。

- 单次迭代主成本来自 SVD：约 `O(m*n*k)`（致密全 SVD）。
- 额外成本：梯度计算 `O(m*n)`。
- 空间复杂度：`O(m*n)`（存储矩阵与中间量）。

因此该方法更适合中等规模问题；大规模场景常用截断 SVD 或随机 SVD 降成本。

## R06

一个小例子：
- 真实矩阵由 `U(60x4) @ V(50x4)^T` 生成，理论秩约 4；
- 仅观测 35% 条目；
- 用上述迭代恢复。

预期现象：
- 观测误差（observed RMSE）快速下降；
- 全矩阵误差（full RMSE）显著优于“未观测置零”基线；
- 恢复矩阵的有效秩接近真实秩（不一定完全相同）。

## R07

优点：
- 建模清晰，目标函数凸（核范数版）。
- 迭代规则简单，易于实现与调试。
- 对噪声有一定鲁棒性，可直接加入正则。

局限：
- SVD 成本高，规模上去后变慢。
- 对采样模式敏感（极不均匀采样可能失败）。
- 核范数偏置会压缩大奇异值，可能导致幅值偏差。

## R08

前置知识建议：
- 线性代数：SVD、秩、Frobenius 范数；
- 凸优化：正则化、近端算子；
- 数值优化：步长、收敛判据、迭代稳定性；
- Python/Numpy 基础矩阵运算。

## R09

适用场景：
- 推荐系统评分补全；
- 传感器缺失数据修复；
- 图像/视频低秩背景建模（简化版）。

不适用或需谨慎：
- 真实矩阵本身并不低秩；
- 观测比例过低且不满足基本覆盖（某些行/列几乎全缺失）；
- 需要超大规模实时求解但无法使用近似 SVD。

## R10

实现正确性的关键检查：
1. 梯度必须只在观测集合 `Omega` 上生效。
2. 近端步骤必须是“奇异值软阈值”而非硬截断。
3. 阈值应为 `eta*lambda`。
4. 停止条件建议基于相对变化量 `||X_{k+1}-X_k||/max(||X_k||,eps)`。
5. 评估时区分：观测误差与全矩阵误差。

`demo.py` 中这些点均显式实现并打印指标。

## R11

数值稳定性注意点：
- 分母保护：相对变化量分母加入 `eps` 防止除零。
- 观测计数保护：RMSE 分母至少为 1。
- 数据尺度：构造数据时按 `1/sqrt(rank)` 缩放，避免奇异值过大。
- 步长：本问题梯度 Lipschitz 常数为 1（投影算子），取 `eta<=1` 通常稳定；MVP 采用 `eta=1.0`。

## R12

参数经验：
- `lambda` 越大，恢复秩越低但偏差可能增大。
- `observe_ratio` 越高，恢复通常越稳定。
- `max_iter` 决定收敛充分程度；`tol` 控制停止精度。

实务调参顺序建议：
1. 先固定 `eta=1.0`；
2. 扫描 `lambda`（如 `0.1 ~ 1.0`）；
3. 再按误差曲线决定迭代上限。

## R13

- 近似比保证：N/A（不是离散近似算法语境）。
- 随机化成功概率保证：在满足非严格条件时有经典恢复理论（与 incoherence、采样量相关），但本 MVP 不给出形式化概率界证明。

常见理论结论（高层次）：
- 当真实矩阵满足一定非相干性，且采样量达到 `O(mu * r * (m+n) * polylog(max(m,n)))` 级别时，核范数法可精确或近似恢复。

## R14

常见失败模式与处理：
- 失败：采样过低导致不可辨识。
  - 处理：提高观测率，确保每行每列都有观测。
- 失败：`lambda` 过大，结果过度平滑。
  - 处理：减小 `lambda`。
- 失败：迭代早停。
  - 处理：增大 `max_iter` 或减小 `tol`。
- 失败：SVD 计算慢。
  - 处理：用截断/随机 SVD（大规模时）。

## R15

工程实践建议：
- 固定随机种子，保证可复现调参。
- 同时监控 `obs_rmse`、`full_rmse`、估计秩。
- 保留“零填充基线”，避免误把噪声拟合当作改进。
- 将数据预处理（中心化/标准化）与恢复过程解耦。

## R16

相关方法脉络：
- 前驱思想：PCA/SVD、Tikhonov 正则、压缩感知。
- 同类方法：
  - 核范数最小化（凸）；
  - 因子分解 `X=UV^T`（非凸但高效）；
  - RPCA（低秩 + 稀疏分解）。
- 扩展方向：
  - 加速近端（FISTA）；
  - 带约束/加权采样；
  - 在线或增量式低秩恢复。

## R18

`demo.py` 的源码级流程拆解（8 步）：
1. `make_low_rank_matrix` 生成低秩真值矩阵 `M_true = U @ V^T`，并按秩做尺度归一化。
2. `make_observation_mask` 采样观测掩码 `Omega`，并强制每行每列至少一个观测避免退化。
3. 构造观测矩阵 `M_obs = Omega * M_true`，未观测位置置零。
4. `recover_low_rank_matrix_svt` 进入主循环，先计算梯度 `Gk = Omega*(Xk-M_obs)`。
5. 做梯度步得到 `Yk = Xk - eta*Gk`，再调用 `singular_value_thresholding`。
6. 在 `singular_value_thresholding` 内执行 SVD，并对奇异值做软阈值 `max(s-eta*lambda,0)`，得到 `X_{k+1}` 与估计秩。
7. 每轮记录目标值、观测 RMSE、相对变化量、估计秩；满足 `tol` 则提前停止。
8. `main` 计算 full/observed/unobserved RMSE，与零填充基线对比并断言改进，最后打印结论。

该实现未依赖第三方黑盒矩阵补全接口，关键优化步骤都可在源码中逐行追踪。
