# 阻抗匹配 (Impedance Matching)

- UID: `PHYS-0181`
- 学科: `物理`
- 分类: `电磁学`
- 源序号: `182`
- 目标目录: `Algorithms/物理-电磁学-0182-阻抗匹配_(Impedance_Matching)`

## R01

本条目实现一个可运行的阻抗匹配 MVP：使用四分之一波长变换器（Quarter-Wave Transformer）把 `ZL` 匹配到系统阻抗 `Z0`，并对比匹配前后反射系数与回波损耗。

## R02

问题定义：
给定传输系统参考阻抗 `Z0`、负载阻抗 `ZL` 和设计频率 `f0`，设计一段长度为 `λ/4` 的传输线使得在 `f0` 附近反射最小。

核心目标：
- 在 `f0` 处使输入阻抗尽量接近 `Z0`
- 量化扫频范围内的匹配带宽特性

## R03

物理背景：
- 负载不等于系统阻抗时，会在接口产生反射波
- 反射系数定义为 `Γ = (Zin - Z0) / (Zin + Z0)`
- `|Γ|` 越小，反射越弱，功率传输效率越高

四分之一波长变换器是一种经典窄带匹配结构，常用于射频/微波传输线匹配。

## R04

本 MVP 使用的公式：

1. 变换器特性阻抗（负载和系统均取正实数时）
`Zt = sqrt(Z0 * ZL)`

2. 传输线输入阻抗
`Zin = Zt * (ZL + j Zt tan(βl)) / (Zt + j ZL tan(βl))`

3. 传播常数
`β = 2πf / vp`

4. 回波损耗
`RL(dB) = -20 log10(|Γ|)`

5. 失配损耗
`ML(dB) = -10 log10(1 - |Γ|^2)`

## R05

适用条件与边界：
- MVP 假设 `Z0` 和 `ZL` 为正实数（纯电阻）
- 模型为无损传输线近似
- 四分之一波长匹配属于窄带匹配，偏离 `f0` 后性能会下降

## R06

输入与输出：

输入（代码中 `MatchCase`）：
- `z0_system`: 系统阻抗（ohm）
- `z_load`: 负载电阻（ohm）
- `f0_hz`: 设计中心频率（Hz）
- `phase_velocity`: 相速度（m/s）

输出：
- 匹配网络设计值 `Zt`
- 扫频表格（`freq_MHz`、`|Gamma|`、`RL_dB`、`ML_dB`、`Zin`）
- `f0` 附近关键指标对比

## R08

代码结构（`demo.py`）：
- `MatchCase`: 参数数据类
- `quarter_wave_impedance`: 计算匹配线特性阻抗
- `input_impedance_quarter_wave`: 扫频计算输入阻抗
- `reflection_coefficient`: 计算反射系数
- `return_loss_db` / `mismatch_loss_db`: 指标换算
- `build_report`: 生成 `pandas.DataFrame` 报表
- `main`: 组装流程并打印结果

## R09

复杂度分析：

设扫频点数为 `N`：
- 时间复杂度：`O(N)`
- 空间复杂度：`O(N)`

主要计算为向量化复数运算（`numpy`），常数因子较小。

## R10

数值稳定性处理：
- `RL` 与 `ML` 计算中使用 `np.maximum(..., 1e-15)` 防止 `log10(0)`
- 输入参数检查：`z0_system`、`z_load` 必须为正
- 所有阻抗/反射量按复数路径计算，避免隐式类型问题

## R11

默认示例参数：
- `Z0 = 50 Ω`
- `ZL = 200 Ω`
- `f0 = 1.0 GHz`
- `vp = 2e8 m/s`

据此得到：
- `Zt = 100 Ω`
- 线长 `l = vp/(4f0) = 5 cm`

## R12

正确性自检点：
- 不匹配直连时：`|Γ| = |(200-50)/(200+50)| = 0.6`
- 在 `f0` 处匹配后理论上 `Zin -> 50 Ω`，因此 `|Γ| -> 0`
- 扫频远离 `f0` 时，`|Γ|` 重新上升，体现窄带行为

## R13

与其他匹配方式对比：
- L 型网络：器件少、实现简单，但参数依赖元件 Q 值
- 单支路短截线：常用于微波传输线实现
- 四分之一波长变换器：结构直观、易算，但中心频率依赖更强

本条目选择四分之一波长方案，是因为它能最小化 MVP 的实现复杂度并保留电磁学本质。

## R14

参数敏感性：
- `f0` 越高，`λ/4` 物理长度越短
- 负载偏离设计值时，最佳 `Zt` 也会偏移
- `phase_velocity`（介质决定）会直接改变所需线长

## R15

常见失败模式：
- 误把复数负载当纯电阻套用 `Zt = sqrt(Z0*ZL)`
- 使用错误相速度导致长度偏差
- 把完美匹配频点误解为“全频匹配”

## R16

可扩展方向：
- 扩展到复数负载匹配（先消虚部再阻抗变换）
- 引入有损传输线模型（复传播常数）
- 增加多节变换器以提升带宽
- 增加自动参数搜索（例如最小化带内平均 `|Γ|`）

## R17

工程落地建议：
- 射频 PCB 设计中先用该模型做初始值估算
- 再结合电磁仿真器（HFSS/CST/ADS 等）做版图与寄生修正
- 最后通过 VNA（矢网）实测 `S11` 对比并回调结构尺寸

## R18

`demo.py` 的源码级算法流程（8 步）：

1. 在 `main()` 创建 `MatchCase`，确定 `Z0`、`ZL`、`f0`、`vp`。  
2. 调用 `quarter_wave_impedance()` 计算设计阻抗 `Zt = sqrt(Z0*ZL)`。  
3. 在 `build_report()` 生成扫频网格 `frequencies_hz`。  
4. 调用 `input_impedance_quarter_wave()`：按每个频点计算 `β`、`tan(βl)`，再代入传输线输入阻抗公式得到 `Zin(f)`。  
5. 调用 `reflection_coefficient()` 计算匹配前常量反射 `Γ_before` 和匹配后频率相关反射 `Γ_after(f)`。  
6. 调用 `return_loss_db()` 与 `mismatch_loss_db()` 把 `Γ` 转为工程常用 dB 指标。  
7. 使用 `pandas.DataFrame` 组织输出列（频率、反射、回波损耗、失配损耗、输入阻抗实虚部）。  
8. 在 `main()` 中打印完整扫频表，并提取 `f0` 附近一行作为关键匹配结果摘要。
