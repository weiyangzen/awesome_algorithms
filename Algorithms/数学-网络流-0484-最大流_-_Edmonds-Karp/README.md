## R01

Edmonds-Karp 是 Ford-Fulkerson 方法的一个经典实现：每轮都在残量网络里用 BFS 找到“边数最少”的增广路，然后沿该路径增广。

本目录 MVP 的目标是：
- 手写 Edmonds-Karp 主算法，不调用图算法黑盒；
- 输出可审计的运行信息（BFS 次数、增广次数）；
- 通过经典样例 + 随机对拍验证正确性。

## R02

最大流问题定义（单源单汇）：

- 输入：有向图 `G=(V,E)`、容量 `c(u,v) >= 0`、源点 `s`、汇点 `t`。
- 约束：
  - 容量约束：`0 <= f(u,v) <= c(u,v)`；
  - 反对称：`f(u,v) = -f(v,u)`；
  - 流守恒：对任意 `u != s,t`，`sum_x f(x,u) = sum_x f(u,x)`。
- 目标：最大化总流量 `|f| = sum_v f(s,v)`。

Edmonds-Karp 在残量网络 `G_f` 上迭代改进 `f`，直到不存在 `s -> t` 增广路。

## R03

Edmonds-Karp 的关键约束是“增广路选择策略”而不是“增广动作”本身：

1. 用 BFS 在残量网络中找一条从 `s` 到 `t` 的最短路径（按边数计）。
2. 计算该路径上的瓶颈容量 `delta`。
3. 沿路径减少前向残量、增加反向残量。
4. 累计 `max_flow += delta`。

与任意路径的 Ford-Fulkerson 相比，BFS 最短路规则让复杂度具有确定上界。

## R04

复杂度结论（Edmonds-Karp 标准结论）：

- 单次 BFS：`O(E)`。
- 每次增广会使某些边变饱和，且最短路距离单调不减。
- 总增广次数上界 `O(VE)`。
- 总时间复杂度：`O(VE^2)`。

本实现采用容量矩阵表示残量图；在稠密矩阵扫描实现下，常数项更偏向教学可读性而非大规模性能。

## R05

空间复杂度：

- 容量矩阵 `n x n`：`O(V^2)`；
- 残量矩阵 `n x n`：`O(V^2)`；
- BFS 辅助结构（`parent`、队列）：`O(V)`。

合计为 `O(V^2)` 级别。对于教学与中小规模验证图，矩阵写法直观且便于调试。

## R06

微型手算示例：

- `s -> a (3)`
- `s -> b (2)`
- `a -> b (1)`
- `a -> t (2)`
- `b -> t (3)`

增广过程：
- 第 1 轮可走 `s-a-t`，增广 2。
- 第 2 轮可走 `s-b-t`，增广 2。
- 第 3 轮可走 `s-a-b-t`，增广 1。

最终最大流为 `5`。

## R07

为什么 Edmonds-Karp 仍然重要：

- 它是网络流入门最常用基线，证明结构清晰。
- 相比 Dinic / Push-Relabel，代码路径更短，便于审计。
- 经常作为“参考真值算法”对拍更快算法（例如本目录用 Dinic 参考实现做随机对拍）。

## R08

核心理论点：

1. 残量网络中的可达路径等价于“仍可继续增流”的通道。
2. 反向边记录撤销空间，保证已推流量可被后续路径重分配。
3. BFS 选择最短增广路，保证每次增广后距离标签具有单调性。
4. 某条边被反复成为最短路关键边的次数受限，从而导出 `O(VE^2)` 上界。
5. 当 BFS 找不到汇点时，残量网络不存在增广路，流量达到最优。

## R09

适用条件与限制：

- 适用于单源单汇、非负容量网络。
- 支持重边与反平行边（本实现会把平行边容量合并到矩阵单元）。
- 若图规模很大且稀疏，邻接表 Dinic 通常更高效。
- 本实现针对整数容量，使用 `numpy.int64` 存储残量。

## R10

正确性框架（简版）：

1. 每次增广量 `delta` 不超过路径上最小残量，容量约束保持成立。
2. 同步维护反向残量，保证流可撤销与守恒结构可维持。
3. 重复增广直到不存在 `s->t` 路径。
4. 依据增广路定理：不存在增广路当且仅当当前流为最大流。
5. 结合最大流最小割定理：最终流值等于某个割容量。

## R11

数值与工程注意点：

- 使用 `numpy.int64`，避免 Python 循环中频繁对象分配；
- 容量过大时要关注 `int64` 上限（本 demo 的测试数据规模远低于上限）；
- 随机测试固定种子 `20260407`，保证结果可复现；
- 通过对拍与定理校验（`max_flow == min_cut`）双重保障结果可信。

## R16

相关算法对比：

- Ford-Fulkerson（任意增广路）：实现简单，但复杂度不稳定。
- Edmonds-Karp：可读性高、复杂度有保证，适合基线与教学。
- Dinic：分层图 + 阻塞流，通常更快。
- Push-Relabel：在部分图族上性能优秀，但实现更复杂。

Edmonds-Karp 的定位是“稳健、透明、易验证”的标准参考实现。

## R18

源码级算法流程拆解（`edmonds_karp_max_flow` 主路径，8 步）：

1. 调用 `_validate_input` 做规模、端点、容量合法性检查。
2. 用 `build_capacity_matrix` 构建容量矩阵，并复制为残量矩阵 `residual`。
3. 初始化 `max_flow=0`，进入主循环；每轮先清空 `parent` 并从 `source` 开始 BFS。
4. BFS 在 `residual` 上仅沿 `cap>0` 的边扩展，记录每个节点前驱，直到找到 `sink` 或队列为空。
5. 若 `sink` 不可达，主循环结束并返回当前 `max_flow`（无增广路）。
6. 若可达，则沿 `parent` 从 `sink` 回溯到 `source`，取路径最小残量作为瓶颈 `aug`。
7. 再次沿路径回溯，执行残量更新：`residual[u,v]-=aug`，`residual[v,u]+=aug`。
8. 执行 `max_flow += aug` 并进入下一轮 BFS；最终返回 `EKResult(max_flow, residual, bfs_runs, augmentations)`。

本实现的 Edmonds-Karp 主过程完全手写展开，没有调用第三方最大流黑盒接口。
