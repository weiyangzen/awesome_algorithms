## R01

树状数组（Binary Indexed Tree, BIT / Fenwick Tree）是一种维护“前缀可加型信息”的轻量级数据结构，典型用途是：

- 单点更新（`a[i] += delta`）；
- 前缀查询（`sum(1..i)`）；
- 区间求和（`sum(l..r)=prefix(r)-prefix(l-1)`）。

核心优势是把这三类操作都压到 `O(log n)`，并且实现比线段树更短、更易审计。

## R02

Fenwick Tree 由 Peter M. Fenwick 在 1994 年提出，最初用于压缩累计频率表。它在算法竞赛与工程系统中长期流行，原因有三点：

- 常数小：数组 + 位运算即可完成；
- 语义稳定：非常适合“可逆前缀聚合”（和、计数等）；
- 易扩展：可做顺序统计（按前缀和反查下标）、离散化计数、逆序对等。

## R03

本条目关注的基础问题是动态前缀和：

- 输入：长度为 `n` 的可变数组 `a`；
- 操作：多次执行 `update(i, delta)` 与 `query(l, r)`；
- 目标：在大量交替更新/查询下，维持较低总复杂度。

若每次查询都线性扫描，复杂度会退化到 `O(n)`。BIT 通过“分块前缀覆盖”把查询和更新都降为 `O(log n)`。

## R04

BIT 的结构定义：

- 使用 1-based 数组 `tree[1..n]`；
- 每个节点 `tree[i]` 维护一段长度为 `lowbit(i)` 的区间和；
- `lowbit(i) = i & -i`（二进制最低位的 `1` 所对应的值）。

两个核心跳转：

- 更新时向上跳：`i += lowbit(i)`，把增量传播到所有受影响桶；
- 查询时向下跳：`i -= lowbit(i)`，逐段累加覆盖前缀。

本目录 `demo.py` 直接给出源码级实现，不依赖黑盒数据结构库。

## R05

复杂度（数组长度 `n`，操作次数 `m`）：

- 单点更新：`O(log n)`；
- 前缀和：`O(log n)`；
- 区间和：`O(log n)`（两次前缀和）；
- 空间：`O(n)`。

在 `m` 次操作下总复杂度为 `O(m log n)`，明显优于朴素的 `O(mn)` 扫描方案。

## R06

一个小例子（1-based）：

- 初始 `a=[2,1,3,4,5]`；
- `prefix(3)=2+1+3=6`；
- 执行 `add(2,+4)` 后数组变为 `[2,5,3,4,5]`；
- 此时 `range_sum(2,4)=5+3+4=12`。

BIT 不直接重算整段，而是仅更新与索引 2 相关的少量桶节点。

## R07

优点：

- 代码量小，维护成本低；
- 常数开销小，实战性能好；
- 对“可逆前缀聚合”问题非常高效。

局限：

- 不擅长区间最值（max/min）这类不可逆聚合；
- 对复杂区间更新通常需技巧（差分 BIT 或双 BIT）；
- 必须严格处理索引边界（1-based vs 0-based）。

## R08

前置知识：

- 二进制与位运算（尤其 `x & -x`）；
- 前缀和概念；
- 基本复杂度分析（`log n` 层级跳转）。

运行环境（本 MVP）：

- Python 3.10+；
- `numpy`（仅用于生成对照数据与自动校验）。

## R09

适用场景：

- 高频单点更新 + 区间求和；
- 频次统计、前缀计数、逆序对；
- 需要按累计频率反查位置（order statistics）。

不适用场景：

- 主要操作是区间最值查询；
- 需要复杂懒标记区间赋值；
- 结构需要支持任意结合但不可逆操作。

## R10

实现正确性的关键检查点：

1. `lowbit(i)` 必须写成 `i & -i`；
2. 更新循环必须是 `while i <= n: i += lowbit(i)`；
3. 前缀查询必须是 `while i > 0: i -= lowbit(i)`；
4. 区间和必须使用前缀差分，不应重复线性扫描；
5. 更新后与朴素数组结果要逐次对齐。

`demo.py` 内置随机操作回放断言，逐操作核对 BIT 与基准数组一致性。

## R11

数值与工程稳健性策略：

- 对索引做边界检查，防止越界写入；
- 支持负增量，允许撤销类更新；
- 前缀查询对 `i <= 0` 返回 0，对 `i > n` 自动截断到 `n`；
- 使用 `int64` 存储，避免中等规模累计溢出。

## R12

调参与实现选择（本 MVP）：

- 构建方式：
  - 简洁方案 `O(n log n)`：逐点 `add`；
  - 快速方案 `O(n)`：通过 `tree[parent] += tree[i]` 线性构建。

- 本实现采用 `O(n)` 构建，并保留单点更新接口，兼顾初始化效率和在线更新能力。

此外，`find_by_prefix` 使用二进制提升，时间复杂度 `O(log n)`。

## R13

理论性质：

- 正确性来自二进制区间覆盖：`tree[i]` 保存一个尾对齐区间；
- 任何前缀都可拆为不重叠若干 `lowbit` 区间，故查询可累加得到精确和；
- 更新沿父链传播，确保所有覆盖目标位置的桶都被同步。

这是一种精确数据结构，不涉及近似比或随机成功概率。

## R14

常见失效模式与对策：

- 0-based / 1-based 混用导致错位。  
  对策：对外接口使用 1-based，内部保持一致。
- 查询循环写成 `i += lowbit(i)`。  
  对策：查询必须向下跳（减法）。
- `find_by_prefix` 在目标超总和时未处理。  
  对策：明确返回 `n+1` 作为“未命中”。
- 构建时忘记复制输入，导致副作用。  
  对策：`np.asarray(...).copy()`。

## R15

工程落地建议：

- 若业务只需加法统计，优先 BIT，避免过早上复杂线段树；
- 把索引规范写进接口文档（强制 1-based 或统一转换）；
- 对关键路径保留随机回归测试（更新/查询混合序列）；
- 频次检索场景可直接复用 `find_by_prefix` 做 rank/select。

## R16

相关结构对比：

- 前缀和数组：查询 `O(1)`，但更新 `O(n)`；
- 线段树：功能更强（最值、懒标记），代码与常数更大；
- 平衡树 / Order Statistic Tree：适合动态有序集合，但实现复杂。

BIT 的定位是“在前缀可加问题上，用最小工程成本拿到 `O(log n)` 动态能力”。

## R18

`demo.py` 的源码级算法流程（7 步）：

1. `FenwickTree.__init__` 接收 1-based 数据，分配 `tree` 数组并调用 `_build_linear`。  
2. `_build_linear` 先把原数组值写入 `tree[i]`，再把每个节点贡献加到其父节点 `i + lowbit(i)`，完成 `O(n)` 构建。  
3. `add(index, delta)` 从叶位置开始沿父链上跳，把增量写入所有覆盖该点的桶。  
4. `prefix_sum(index)` 从 `index` 向 0 方向跳，累加经过桶值，得到 `sum(1..index)`。  
5. `range_sum(left, right)` 用两次前缀和差分得到区间和，复用同一查询逻辑。  
6. `find_by_prefix(target)` 用二进制提升测试“若走到 next_idx，前缀是否仍小于 target”，最终得到最小满足 `prefix >= target` 的位置。  
7. `main` 构造随机样例，执行基准对照、随机操作回放与顺序统计验证，形成完整可复现实验闭环。
