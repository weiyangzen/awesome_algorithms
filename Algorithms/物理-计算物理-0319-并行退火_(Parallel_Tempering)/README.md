# 并行退火 (Parallel Tempering)

## R01

并行退火（Parallel Tempering，也称 Replica Exchange Monte Carlo）是一种用于多峰能量地形采样的 Monte Carlo 技术。核心思想是同时运行多条不同温度的链：

- 高温链容易跨越能垒；
- 低温链能精细采样低能区域；
- 周期性尝试交换相邻温度链的状态，让“高温探索能力”传递给低温链。

## R02

在统计物理和计算物理中，普通 Metropolis 在低温下容易陷入局部极小值。并行退火通过温度维度上的“扩散”缓解这一问题，常用于：

- 自旋玻璃、Ising/Potts 模型；
- 粗糙自由能面上的分子模拟；
- 贝叶斯推断中的困难后验采样。

## R03

本条目 MVP 解决的问题是：

- 构建一个可运行的并行退火实现，而不是调用黑盒库；
- 用二维 Ising 模型作为测试系统；
- 与“仅在最低温度跑单链 Metropolis”做对照，展示低温观测量去相关改进。

## R04

本实现采用以下物理/算法关系：

1. Ising 能量（周期边界）：`E = -J * sum_{<ij>} s_i s_j`。  
2. 每个温度槽 `beta_k = 1/T_k` 下执行 Metropolis 单点翻转。  
3. 邻温复制体交换接受率：  
   `A = min(1, exp((beta_i - beta_j) * (E_i - E_j)))`。  
4. 交换按奇偶交替邻对进行（odd-even scheme），提升温度轴混合效率。  
5. 在 burn-in 后记录冷端链（最低温度槽）的能量与 `|m|` 序列。

## R05

设：

- `R` 为复制体数（温度数），
- `N = L^2` 为格点数，
- `S` 为 sweeps 数。

则每个 sweep 的主要成本：

- 局部更新：`O(R * N)`；
- 邻温交换：`O(R)`。

总体时间复杂度 `O(S * R * N)`，空间复杂度约 `O(R * N)`（保存所有复制体状态）。

## R06

`demo.py` 内置固定配置：

- 格子大小：`10x10`；
- 温度梯度：`(1.5, 1.8, 2.1, 2.4, 2.7, 3.0)`；
- sweeps：`1200`，burn-in：`400`；
- 固定随机种子，保证输出可复现。

输出包含：

- 各相邻温度对交换接受率；
- 各温度槽局部 Metropolis 接受率；
- 初始冷端复制体访问过多少温度槽；
- 冷端 `|m|` 的 lag-1 自相关（PT vs baseline）。

运行环境：

## R09

适用场景：

- 单温链明显卡住、能垒高、混合慢；
- 需要低成本验证 PT 思路或教学演示；
- 中小规模格点/简化模型的快速实验。

不适用场景：

- 需要大规模并行/高性能优化的生产模拟；
- 对统计误差控制有严格发表级要求但只做短链运行；
- 没有时间调温度梯度时，交换率可能极差。

## R10

正确性直觉：

1. 固定温度下，局部 Metropolis 满足对应 Boltzmann 分布的不变性；
2. 交换步骤使用 `exp((beta_i-beta_j)(E_i-E_j))`，保证扩展状态空间上的 detailed balance；
3. 局部更新 + 交换更新交替执行，整体仍保持目标联合分布；
4. 由于复制体可在温度轴迁移，低温槽更容易获得“先在高温解困、再回低温细化”的样本。

## R11

数值与稳定性策略：

- 所有接受判定在对数域实现（`log(u) < min(0, log_accept_ratio)`），避免指数上溢；
- 每 100 sweeps 重新精确计算能量，校验增量更新无漂移；
- 固定随机种子，确保脚本验证可复现；
- 使用 burn-in 截断初态偏置。

## R12

关键参数与调参建议：

- `temperatures`：决定交换率。相邻温度太疏会导致交换几乎失败；
- `lattice_size`：系统越大，能量分布越窄，常需更密温度梯度；
- `sweeps` / `burn_in`：过短会导致统计不稳定；
- `coupling`：控制能量尺度；改变它应联动温度范围。

实务上可先把相邻交换率调到约 `0.2 ~ 0.6` 的量级，再增加采样长度。

## R13

- 近似比保证：N/A（非组合优化近似算法）。
- 随机成功率保证：N/A（采样算法，关注统计收敛而非一次性成功率）。

本实现可验证的“工程保证”是：

- 所有邻温对至少发生一次成功交换（在当前固定配置下）；
- 冷端初始复制体能访问多个温度槽；
- 冷端 `|m|` 序列较 baseline 显示更低 lag-1 自相关。

## R14

常见失效模式：

1. 温度梯度设计不当，交换接受率接近 0；
2. sweeps 太少，统计量高方差，结论不稳定；
3. 格子太大但复制体数不够，温度扩散变慢；
4. 忽略能量增量更新校验，可能出现隐蔽实现错误。

诊断建议：

- 先看 `swap_rates` 是否有极低项；
- 再看 `id0_visited_temps` 是否显著小于温度总数；
- 最后看 PT 与 baseline 自相关对比是否反常。

## R15

工程化扩展建议：

- 自适应温度梯度（目标交换率区间）；
- 并行化复制体更新（多进程或多线程/GPU）；
- 增加 binning / blocking 分析与有效样本数估计；
- 将 Ising 替换为连续势函数系统（如 Lennard-Jones 或分子势）。

## R16

相关算法与对比：

- 单温 Metropolis：实现简单但低温易卡住；
- Simulated Annealing：单链降温优化，不是平衡采样；
- Hamiltonian Monte Carlo：连续变量高维采样强，但与离散自旋模型不直接同构；
- Wang-Landau / Multicanonical：直接重权采样能量空间，目标不同。

## R18

`demo.py` 源码级算法流程（8 步）：

1. `PTConfig` 定义实验参数（格子大小、温度梯度、采样步数、随机种子）。  
2. `total_energy` 计算周期边界 Ising 总能量，作为状态评价函数。  
3. `metropolis_sweep` 在每个温度槽执行 `L*L` 次单点提案，按 Metropolis 准则接受/拒绝。  
4. `run_parallel_tempering` 初始化所有复制体，并维护 `replica_ids` 跟踪“哪条逻辑链”位于哪个温度槽。  
5. 每个 sweep 先做全部局部更新，再按奇偶交替邻对尝试交换，接受率依据 `exp((beta_i-beta_j)(E_i-E_j))`。  
6. 主循环中定期重算能量做一致性校验，防止增量更新实现漂移。  
7. burn-in 后收集冷端观测量（能量/自旋绝对磁化）和复制体温度访问统计。  
8. `main` 额外运行 `run_baseline_single_chain`，计算 lag-1 自相关并进行断言，形成非交互可验证 MVP。  

说明：本实现没有把第三方库当黑盒，`numpy` 仅用于数组与随机数，PT 的局部更新、交换判定、诊断与验证逻辑都在源码中显式实现。
