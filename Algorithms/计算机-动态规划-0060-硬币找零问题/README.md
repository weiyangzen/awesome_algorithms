## R01

“硬币找零问题（Coin Change）”是动态规划经典问题。  
给定若干硬币面额（每种数量无限），目标金额为 `amount`：
- 版本 A：最少需要多少枚硬币（不可达则返回无解）；
- 版本 B：共有多少种凑法（按组合计数，不区分顺序）。

本条目在一个 MVP 中同时实现这两个版本，便于对照理解状态设计差异。

## R02

问题定义：
- 输入：
  - `coins`：正整数数组，表示硬币面额，例如 `[1, 2, 5]`；
  - `amount`：非负整数目标金额。
- 输出：
  - `min_coins`：最少硬币数；若不可达则为 `None`；
  - `combination`：一组达到 `min_coins` 的具体硬币组合；
  - `ways`：凑出 `amount` 的组合总数（顺序不敏感）。

示例：`coins=[1,2,5], amount=11`  
- 最少硬币数 `min_coins=3`（如 `[5,5,1]`）；  
- 组合总数 `ways=11`。

## R03

核心 DP 建模分两条线：

1) 最少硬币数（一维最短路式 DP）  
- 状态：`dp[a]` 表示凑出金额 `a` 的最少硬币数。  
- 初始化：`dp[0]=0`，其余为无穷大。  
- 转移：对每个 `coin` 和 `a>=coin`，执行  
  `dp[a] = min(dp[a], dp[a-coin] + 1)`。  
- 为了重建方案，记录 `pick[a]` 表示让 `dp[a]` 变优时选择的硬币。

2) 组合计数（完全背包计数）  
- 状态：`ways[a]` 表示凑出金额 `a` 的组合数。  
- 初始化：`ways[0]=1`（“什么都不选”是一种方式）。  
- 转移：对每个 `coin` 和 `a>=coin`，执行  
  `ways[a] += ways[a-coin]`。  
- 先遍历硬币、再遍历金额，天然避免排列重复计数。

## R04

正确性直觉：
- `dp` 的每次转移都代表“最后再放一枚 `coin`”，因此 `dp[a-coin]+1` 覆盖了所有可行构造；取最小即可得到全局最优。
- `ways` 的转移遵循“按硬币种类逐层扩展”：
  - 处理到第 `k` 种硬币时，`ways[a]` 只由前 `k` 种硬币构成；
  - 不会把同一组硬币在不同顺序下重复统计。
- 两个状态都只依赖更小金额，满足动态规划无后效性。

## R05

复杂度（`n=len(coins)`，目标金额为 `A`）：
- 时间复杂度：`O(n * A)`（最少硬币与组合计数各一次）。
- 空间复杂度：`O(A)`（`dp`、`pick`、`ways` 均为一维数组）。

这是硬币找零问题最常用且工程可落地的复杂度级别。

## R06

`demo.py` 的实现模块：
- `normalize_coins`：校验并标准化面额（去重、升序）。
- `validate_amount`：校验金额合法性。
- `min_coins_dp`：最少硬币数 DP + `pick` 回溯重建。
- `count_ways_dp`：组合计数 DP。
- `solve_coin_change`：整合两类输出。
- `brute_force_min_and_ways`：小规模精确解（用于回归校验）。
- `run_case`：单例执行、打印和断言。
- `randomized_regression`：随机回归测试。

## R07

最小工具栈：
- `numpy`：用于构建一维数值 DP 数组（`dp` / `pick` / `ways`）。
- Python 标准库：`dataclasses`、`functools`、`random`。

没有调用外部“黑盒求解器”，核心转移与回溯全部手写可追踪。

## R08

输入与边界处理：
- `coins` 必须是非空正整数数组，否则抛异常。
- `amount` 必须是非负整数。
- `amount=0` 时：
  - `min_coins=0`；
  - `combination=[]`；
  - `ways=1`。
- 若目标金额不可达（如 `coins=[2], amount=3`）：
  - `min_coins=None`；
  - `combination=[]`；
  - `ways=0`。

## R09

最优方案重建策略：
- `min_coins_dp` 在每次改善 `dp[a]` 时记录 `pick[a]=coin`。
- 结束后从 `amount` 反向回溯：
  - 取 `coin = pick[cur]`；
  - 将该硬币加入答案；
  - `cur -= coin`，直到 `cur=0`。
- 若 `dp[amount]` 仍是无穷大，说明不可达，直接返回无解。

## R10

为什么加入暴力校验：
- DP 代码常见错误在边界、循环顺序和回溯一致性。
- `brute_force_min_and_ways` 对小规模输入给出精确真值：
  - 同时验证最少硬币数；
  - 同时验证组合总数。
- `run_case` 和 `randomized_regression` 都做 DP vs 暴力对照，显著提升可信度。

## R12

输出字段说明（控制台）：
- `dp_result -> min_coins, combination, ways`：MVP 主算法结果。
- `bruteforce_check -> min_coins, ways`：精确校验结果。
- `checks`：
  - `sum_match`：重建组合金额是否等于 `amount`；
  - `len_match`：组合长度是否等于 `min_coins`；
  - `coin_domain_ok`：组合中的硬币是否都来自输入面额；
  - `min_match / ways_match`：是否与暴力真值一致。

## R13

建议测试样例：
- 经典可达：`[1,2,5], 11`。
- 明确不可达：`[2], 3`。
- 多最优候选：`[1,3,4], 6`。
- 输入乱序：`[2,5,10,1], 27`（验证标准化逻辑）。
- 零金额边界：`[7,10], 0`。
- 随机回归：小面额集合 + 小金额范围，和暴力解持续对照。

## R14

工程注意点：
- 组合计数若把“金额循环放外层”，会把排列当组合，导致重复计数。
- 最少硬币回溯必须保留 `pick`，仅有 `dp` 值无法恢复具体解。
- 输入中重复面额会导致不必要重复计算，需先去重。
- 不可达状态要统一表示（本实现用 `None` 和空组合）。

## R15

与其他方法对比：
- 相比纯暴力枚举：从指数级降到 `O(nA)`。
- 相比 BFS 求最少硬币：DP 更容易同时扩展到“组合计数”并复用状态。
- 相比直接调用第三方黑盒函数：本实现完整保留状态定义、递推和回溯细节，更适合作为算法学习与审计基线。

## R16

应用场景：
- 自动售货、收银系统中的最少找零策略。
- 支付拆分、预算离散化中的组合数量评估。
- 背包类问题教学：完全背包的“优化目标”与“计数目标”对比示例。
- 面试与竞赛中的经典 DP 基础题。

## R17

可扩展方向：
- 增加“硬币数量有限”的约束，转为有界背包变体。
- 输出所有最优解而非单一解（需额外去重与路径管理）。
- 增加大规模基准测试，观察 `amount` 增大时性能曲线。
- 扩展为多目标优化（例如最少硬币优先、其次最大面额最少等）。

## R18

`demo.py` 源码级流程（8 步）：
1. `main` 定义固定测试集，逐个调用 `run_case`，最后执行随机回归。  
2. `run_case` 先调用 `solve_coin_change`，后者依次执行 `min_coins_dp` 与 `count_ways_dp`。  
3. `min_coins_dp` 创建一维数组 `dp` 与 `pick`，初始化 `dp[0]=0`、其余为无穷大。  
4. `min_coins_dp` 按“硬币外层、金额内层”更新 `dp[a] = min(dp[a], dp[a-coin]+1)`，并在变优时写入 `pick[a]`。  
5. `min_coins_dp` 若 `dp[amount]` 可达，则基于 `pick` 反向回溯，恢复一组最优硬币组合。  
6. `count_ways_dp` 用独立的一维 `ways` 执行 `ways[a] += ways[a-coin]`，得到组合总数。  
7. `run_case` 调用 `brute_force_min_and_ways` 产出小规模精确真值，并检查金额、长度、面额合法性及两类结果一致性。  
8. 固定样例通过后，`randomized_regression` 进行 200 轮随机对照，最后打印 `All coin-change checks passed.`。
