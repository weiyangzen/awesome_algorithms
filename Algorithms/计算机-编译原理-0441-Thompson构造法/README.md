## R01

Thompson构造法是把正则表达式机械地转换为带 `ε` 边的 NFA（非确定有限自动机）的方法。它的价值在于：
- 构造规则局部且统一，便于实现和证明。
- 生成的自动机规模线性于正则长度，适合作为词法分析器前端。
- 可继续做子集构造（NFA -> DFA）与最小化，形成完整正则匹配流水线。

## R02

核心思想是“表达式语法树的后序构造 + 片段拼接”。
- 每个子表达式都对应一个 NFA 片段（fragment），只暴露 `start` 和 `accept`。
- 三个运算（并 `|`、连接、闭包 `*`）都能用固定模板把片段拼成更大片段。
- 最终栈顶片段就是整条正则的 NFA。

## R03

设正则字母表为 `Σ`，NFA 定义为五元组 `M=(Q, Σ, δ, q0, F)`，其中 `δ: Q × (Σ ∪ {ε}) -> P(Q)`。
Thompson模板：
- 字符 `a`：`s --a--> t`
- 连接 `R1R2`：`R1.accept --ε--> R2.start`
- 并 `R1|R2`：新建 `s,t`，`s --ε--> R1.start/R2.start`，两支末端都 `--ε--> t`
- 闭包 `R*`：新建 `s,t`，`s --ε--> R.start/t`，`R.accept --ε--> R.start/t`

## R04

本目录 MVP 的输入输出约定：
- 输入：正则字符串（支持单字符字面量、`|`、`*`、括号），在 `demo.py` 中写死测试样例。
- 输出：
  - 带显式连接符的中间表达式
  - 后缀表达式
  - NFA 状态数与迁移列表
  - 多个测试串的匹配布尔值

## R05

实现使用的数据结构：
- `Fragment(start, accept)`：栈内的“可拼接片段”。
- `transitions: dict[int, dict[str|None, set[int]]]`：邻接表；`None` 表示 `ε` 边。
- 栈 `list[Fragment]`：按后缀表达式驱动构造。
- 集合闭包：`epsilon_closure(states)` 进行 NFA 运行时扩展。

## R06

算法流程：
1. 对中缀正则插入显式连接符 `.`。
2. 用 Shunting-yard 把中缀转后缀。
3. 扫描后缀：字面量建基础片段，运算符按模板弹栈拼接。
4. 得到完整 NFA 的 `start/accept`。
5. 运行匹配时，维护当前状态集合：
   - 初始为 `ε-closure({start})`
   - 读入字符后 `move` 再做 `ε-closure`
6. 输入结束后，若 `accept` 在当前集合内则匹配成功。

## R07

正确性要点（归纳证明思路）：
- 基础情形：单字符模板严格接受且仅接受该字符。
- 归纳假设：子表达式片段语义正确。
- 归纳步：
  - 连接模板只允许先走左子表达式再走右子表达式。
  - 并模板只引入两条候选分支，语义为集合并。
  - 闭包模板允许 0 次（直接到新终点）或多次（回边）重复。
因此组合后片段语义与原正则等价。

## R08

复杂度（设正则长度为 `m`，输入串长度为 `n`，NFA 状态数为 `|Q|`）：
- 构造：`O(m)` 时间，`O(m)` 空间。
- 运行（朴素集合模拟）：每字符最坏触达 `O(|Q| + |E|)`，总计约 `O(n * |Q|)` 量级。
- Thompson 生成状态数通常与 token 数同阶，工程上可接受。

## R10

示例（逻辑）：
- 正则 `a(b|c)*d` 应接受：`ad`、`abd`、`acbd`、`abcbcd`
- 同时拒绝：`ab`、`aecd`
- 正则 `(ab|c)*` 应接受空串、`ab`、`c`、`abcab`

## R11

边界与限制：
- 支持单字符字面量，不支持转义序列与字符类（如 `[a-z]`）。
- 若传入非法括号或运算符序列，会抛出 `ValueError`。
- 空正则在实现中按“仅匹配空串”的 `ε` 自动机处理。

## R12

常见实现错误：
- 忘记插入显式连接符，导致后缀表达式错误。
- `*` 作为一元后缀运算符处理不当。
- `ε-closure` 只扩展一层而不是传递闭包。
- 并/闭包模板中漏连某条 `ε` 边，出现误拒绝。

## R13

可扩展方向：
- 增加 `+`、`?`、字符类、转义等更完整 regex 语法。
- 在 NFA 基础上做子集构造得到 DFA，再做最小化。
- 将 token 从“单字符”升级为“词法单元”，用于更真实词法器。

## R14

与编译原理流水线关系：
- Thompson构造解决“正则语义 -> 可执行自动机”第一步。
- 子集构造把 NFA 的非确定性消除，得到确定状态转移。
- 最小化降低状态规模，最终用于高效词法扫描。

## R15

工程落地建议：
- 先固定语法子集（本项目即单字符 + `|` `*` `()` + 连接）。
- 用可打印的边列表做白盒调试，比只看匹配结果更快定位问题。
- 将“构造”和“运行”分层，后续替换 DFA 运行器时无需改构造器。

## R18

`demo.py` 的源码级算法分解（非黑箱）：
1. `insert_concat_ops`：扫描中缀表达式，在“原子后接原子起始”处插入 `.`。
2. `to_postfix`：使用运算符栈按优先级把中缀转后缀。
3. `NFA.from_regex`：调度前两步并进入片段构造。
4. `NFA._build_from_postfix` 读 token：
   - 字面量 -> `_literal_fragment`
   - `.` -> `_concat`
   - `|` -> `_union`
   - `*` -> `_star`
5. 每个模板只做“弹栈、建新状态、补 `ε` 边、压栈”四件事。
6. 构造结束后得到唯一片段的 `start/accept`，并写入 NFA 对象。
7. `matches` 运行输入串：`current = epsilon_closure({start})`，逐字符执行 `move + epsilon_closure`。
8. 输入结束检查 `accept in current`，输出匹配真假。

这 8 步覆盖了 Thompson 构造与 NFA 解释执行的全部主干逻辑。
