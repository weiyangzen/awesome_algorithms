## R01

石子合并问题（线性版本）定义如下：
- 给定一行石子堆，重量数组为 `w[0..n-1]`；
- 每次只能合并相邻两段（可理解为先把区间拆成左右两部分，再把两部分合成一堆）；
- 一次合并代价等于“被合并区间的总重量”；
- 目标是让全部石子最终合成一堆时，总合并代价最小。

这是经典区间动态规划问题，MVP 聚焦最小总代价及一种最优分割方案重建。

## R02

MVP 输入输出约定：
- 输入：
  - `weights`：一维非负数值序列，表示每堆石子的重量；
- 输出：
  - `min_cost`：最小合并总代价；
  - `merge_plan`：按后序顺序给出的一组合并操作（由最优分割表重建）；
  - `n`：石子堆数量。

约束约定：
- `weights` 必须是一维有限数值；
- 允许 `n=0` 或 `n=1`，此时代价为 `0`。

## R03

状态定义（区间 DP）：
- `dp[i][j]`：把闭区间 `[i, j]` 合并成一堆的最小代价；
- `split[i][j]`：达到 `dp[i][j]` 时最优切分点 `k`，表示先合并 `[i,k]` 与 `[k+1,j]`。

边界：
- `dp[i][i] = 0`，单堆无需合并。

## R04

状态转移方程：

设区间和 `sum(i,j) = w[i] + ... + w[j]`，则

`dp[i][j] = min_{i <= k < j} (dp[i][k] + dp[k+1][j] + sum(i,j))`

解释：
- 先把左右子区间各自最优合并；
- 最后做一次“左右两堆”的合并，代价是整个 `[i,j]` 的总重量。

## R05

最优子结构成立原因：
- 若 `[i,j]` 的最优解最后一步把区间分成 `[i,k]` 与 `[k+1,j]`；
- 则在该最优解中，左右子区间必须分别采用它们各自的最优策略；
- 否则用更优子策略替换后会得到更小总代价，与原最优性矛盾。

因此可用“区间长度递增”方式进行 DP 填表。

## R06

区间和优化：
- 直接在转移中计算 `sum(i,j)` 会导致额外开销；
- 使用前缀和 `prefix`，其中 `prefix[t] = sum(w[0..t-1])`；
- 可在 `O(1)` 时间得到：
  - `sum(i,j) = prefix[j+1] - prefix[i]`。

这样总时间复杂度保持在经典 `O(n^3)`，不会退化。

## R07

复杂度分析：
- 主算法（自底向上区间 DP）：
  - 时间复杂度：`O(n^3)`（区间长度、起点、切分点三层循环）；
  - 空间复杂度：`O(n^2)`（`dp` 与 `split`）。
- 记忆化递归基线：
  - 时间复杂度同阶 `O(n^3)`；
  - 空间复杂度 `O(n^2)`（缓存状态）。

## R09

核心接口：

  - 返回 `n / min_cost / merge_plan`。

  - 返回同问题的最小代价（仅用于交叉验证）。
- `brute_force_min_cost_adjacent(weights) -> float`
  - 仅用于小规模样本正确性检验。

## R13

- 每轮随机生成长度 `n ∈ [0, 7]` 的重量数组，取值 `0..20`；
- 对每轮数据断言：
  - 区间 DP 结果 == 记忆化递归结果；
  - 且（`n <= 7`）区间 DP 结果 == 暴力枚举结果。

说明：`n<=7` 时暴力复杂度仍可接受，用于增强正确性信心。

## R18

`demo.py` 源码级算法流程（8 步）：

1. `main` 组织固定样例并调用 `run_case`，最后执行随机对拍。  
2. `run_case` 先调用 `stone_merge_min_cost`：初始化 `dp`、`split` 以及前缀和。  
3. 主算法按区间长度 `2..n` 递增，遍历每个区间 `[i,j]`。  
4. 对每个 `[i,j]` 枚举切分点 `k`，计算 `dp[i][k] + dp[k+1][j] + sum(i,j)`，更新最小值与 `split[i][j]`。  
5. 填表后通过 `split` 递归重建最优合并顺序，生成 `merge_plan`（后序操作列表）。  
6. `run_case` 再调用 `stone_merge_min_cost_top_down`（`lru_cache`）得到独立基线结果。  
7. 对小规模样例，`run_case` 再调用 `brute_force_min_cost_adjacent` 做穷举最优校验。  
8. 若主算法、递归基线、暴力结果（适用时）不一致，立即断言失败；全部通过后打印结果与随机对拍通过信息。
