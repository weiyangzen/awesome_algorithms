## R01

`SHA-3` 是 NIST 标准化的安全哈希家族（FIPS 202），底层基于 `Keccak` 海绵结构而非传统 Merkle-Damgard。它接收任意长度输入，输出固定长度摘要（224/256/384/512 bit），核心优势是结构与 SHA-2 路线不同，提供了密码学多样性。

## R02

本条目目标是提供一个“可运行、可审阅、非黑盒”的最小 MVP：
- 在 `demo.py` 中手写 `Keccak-f[1600]` 置换与海绵吸收/挤出流程；
- 支持 `SHA3-224/256/384/512` 四种变体；
- 通过标准向量与 `hashlib` 做交叉验证；
- 保证 `uv run python demo.py` 无交互直接运行。

## R03

问题定义（本条目）：
- 输入：任意字节串 `data: bytes`；
- 输出：固定长度摘要 `digest`（按所选 SHA-3 变体决定长度）。

实现接口：
- `sha3_digest(data, digest_bits)` 返回摘要字节与统计信息；
- `sha3_hexdigest(data, digest_bits)` 返回十六进制字符串；
- `main()` 自动自检并输出演示结果。

## R04

SHA-3 关键参数（本实现）：
- 内部状态大小：`1600 bit`（25 个 64-bit lane）；
- 容量与速率关系：`capacity + rate = 1600`；
- 变体参数：
  - `SHA3-224: rate=1152, capacity=448`
  - `SHA3-256: rate=1088, capacity=512`
  - `SHA3-384: rate=832, capacity=768`
  - `SHA3-512: rate=576, capacity=1024`
- SHA-3 域分离后缀：`0x06`。

## R05

状态表示与字节序：
- 状态为 `state[25]`，按 `index = x + 5*y` 存储 5x5 lane 网格；
- 每个 lane 64-bit，使用 little-endian 字节装载；
- 吸收时把输入块按字节 XOR 到前 `rate_bytes` 区域；
- 挤出时按同样 little-endian 顺序读取输出字节。

## R06

`Keccak-f[1600]` 每轮由 5 个步骤组成（共 24 轮）：
1. `theta`：列奇偶校正，做全局线性扩散；
2. `rho`：对每个 lane 进行不同偏移的循环位移；
3. `pi`：重排 lane 位置；
4. `chi`：逐行非线性混合；
5. `iota`：向 `state[0]` 注入轮常量。

`demo.py` 中显式实现了这些步骤与轮常量表，无第三方哈希黑箱。

## R07

填充与吸收流程：
- 先追加 SHA-3 域分离字节 `0x06`；
- 再做 `pad10*1`：补 `0x00` 到块边界前一字节，末尾追加 `0x80`；
- 将填充后消息按 `rate_bytes` 分块；
- 每块先 XOR 入状态，再执行一次 `Keccak-f[1600]`。

这部分逻辑由 `_pad10star1_sha3` 与 `sha3_digest` 中 absorb 循环完成。

## R08

挤出（squeeze）流程：
- 从状态前 `rate_bytes` 依次读字节作为输出；
- 若输出长度不足，继续执行置换后再读下一批；
- 对 SHA-3 标准输出长度（224~512 bit）通常首轮就足够，但实现保持通用循环。

## R09

复杂度分析：
- 设消息长度为 `n` 字节，速率为 `r` 字节；
- 吸收块数约为 `ceil((n + padding)/r)`；
- 每块一次固定 24 轮置换，因此总体时间复杂度对输入长度是线性增长（`O(n)`）；
- 额外状态空间固定为 25 个 64-bit lane 与少量临时数组，空间复杂度 `O(1)`（不计输入输出）。

## R10

正确性与一致性保障：
- 使用 SHA3-256 常见标准向量（空串与 `abc`）做硬校验；
- 对四个 SHA-3 变体在多组确定性样本上与 `hashlib` 比对；
- 若任何样本不一致，脚本会抛异常并非零退出，避免静默错误。

## R11

`demo.py` 模块结构：
- `_rotl64`：64-bit 循环左移；
- `_keccak_f1600`：24 轮置换核心；
- `_pad10star1_sha3`：SHA-3 域分离 + 填充；
- `sha3_digest / sha3_hexdigest`：海绵吸收与摘要导出；
- `_self_test_known_vectors`：标准向量断言；
- `_self_test_against_hashlib`：跨变体验证；
- `_build_digest_report`：用 `pandas` 生成结果表；
- `main`：运行全部测试并打印报告。

## R12

第三方库使用策略（最小且透明）：
- `numpy`：生成可复现随机字节样本（测试输入）；
- `pandas`：格式化输出对照表；
- `hashlib`：仅用于交叉验证，不作为主算法实现。

也就是说，真正的 SHA-3 计算路径由源码中的 Keccak 步骤完成。

## R13

运行输出包含：
- 自检是否通过（标准向量 + hashlib 全量比对）；
- 各变体摘要表（`variant`、摘要位数、块数、置换次数、是否匹配）；
- 每个变体对应的完整十六进制摘要。

这让验证人员可以同时看到“正确性”与“执行轨迹摘要”。

## R14

边界条件处理：
- 空消息 `b""`：明确测试并通过；
- 长度非块对齐消息：由填充逻辑自动处理；
- 二进制消息（含 `0x00` 与高位字节）可直接处理；
- 输出位数若不在 224/256/384/512 内，主动抛出 `ValueError`。

## R15

安全与工程注意事项：
- SHA-3 适合完整性摘要、签名前哈希等场景；
- 哈希函数本身不等于加密，不提供机密性；
- 密码存储应优先使用专门口令哈希（如 Argon2/scrypt/bcrypt），而非直接 SHA-3。

## R16

MVP 取舍说明：

- 统计信息只保留关键指标（块数、置换次数），不展开逐轮全状态日志，控制复杂度。

## R18

`demo.py` 源码级算法流程拆解（8 步，非黑箱）：
1. `main` 先执行 `_self_test_known_vectors` 与 `_self_test_against_hashlib`，确保实现与标准结果一致。
2. `sha3_digest` 按输出位数选择 `(rate, capacity)`，初始化 25 个 64-bit lane 状态为全零。
3. `_pad10star1_sha3` 对输入消息追加域分离后缀 `0x06`，再按 `pad10*1` 规则补齐到 `rate` 的整数块。
4. 吸收阶段逐块处理：将块字节按 little-endian XOR 到状态前 `rate_bytes`，随后调用 `_keccak_f1600`。
5. `_keccak_f1600` 每轮执行 `theta -> rho -> pi -> chi -> iota`，共 24 轮，完成扩散与非线性混合。
6. 吸收完成后进入挤出阶段：从状态前 `rate_bytes` 依序读取字节拼接输出；若输出不足则继续置换再读取。
7. 得到目标长度摘要后，`sha3_hexdigest` 将字节摘要转为十六进制字符串，并携带统计信息返回。
8. `_build_digest_report` 汇总各 SHA-3 变体结果，用 `pandas` 表格展示并与 `hashlib` 对照，若不匹配则直接报错。
