## R01

线段树（Segment Tree）是一种维护区间信息的二叉分治数据结构，适合处理“区间查询 + 区间/单点更新”的混合场景。

本条目实现一个最小可运行 MVP：
- 维护整数数组的区间和（`range sum`）；
- 支持区间加法更新（`range add`，带懒标记）；
- 支持单点赋值更新（`point set`，复用区间加法实现）；
- 内置确定性样例与随机回归测试，自动对照暴力结果。

## R02

问题定义（本目录实现）：
- 输入：
  - 初始数组 `a[0..n-1]`（`n >= 1`）；
  - 操作序列：
    - `sum(l, r)`：查询闭区间 `[l, r]` 元素和；
    - `add(l, r, delta)`：对闭区间 `[l, r]` 每个元素加上 `delta`；
    - `set(i, x)`：将 `a[i]` 赋值为 `x`。
- 输出：
  - 每次查询的区间和结果；
  - 所有断言校验通过时输出成功信息。

`demo.py` 使用固定随机种子生成数据，不需要任何交互输入。

## R03

数学与结构基础：

1. 区间可分性：
   `sum([L,R]) = sum([L,M]) + sum([M+1,R])`，这是线段树可递归维护的核心。  
2. 懒标记思想：
   对整段 `[L,R]` 加 `delta` 时，不必立刻下沉到叶子，可先记录 `lazy += delta`。  
3. 节点值更新：
   节点覆盖长度为 `len = R-L+1`，若整段加 `delta`，节点和增加 `delta * len`。  
4. 正确性直觉：
   查询路径上保证先下推必要懒标记，再汇总子区间，得到的值与真实数组一致。

## R04

算法流程（高层）：
1. 用数组 `tree`、`lazy` 构建线段树，`tree[node]` 表示该节点区间和。  
2. `range_add(l,r,delta)`：
   - 若当前节点区间被完全覆盖，直接更新 `tree[node]` 与 `lazy[node]`；
   - 否则先 `push`，再递归左右子树，最后 `pull` 回父节点。  
3. `range_sum(l,r)`：
   - 若当前节点区间被完全覆盖，直接返回 `tree[node]`；
   - 否则先 `push`，递归子树并求和。  
4. `point_set(i,x)`：先查当前值 `cur = sum(i,i)`，再做 `add(i,i,x-cur)`。

## R05

核心数据结构：
- `tree: list[int]`：长度约 `4*n`，存每个树节点维护的区间和；
- `lazy: list[int]`：长度约 `4*n`，存区间加法的延迟标记；
- `n`：原数组长度；
- 递归参数 `(node, left, right)`：表示节点编号与其覆盖的闭区间。

## R06

正确性要点：
- `build` 正确：叶子等于原数组值，内部节点由左右子和构成。  
- `apply` 正确：整段加法按区间长度修正节点和，并累积懒标记。  
- `push` 正确：把父节点未下传增量精确分发给左右子节点后清零父标记。  
- `pull` 正确：父节点和始终等于两子节点和。  
- `query/update` 正确：基于“覆盖/相交”分支递归，覆盖即返回或更新，相交则拆分并合并。

## R07

复杂度分析（`n` 为数组长度）：
- 建树：`O(n)`；
- 单次区间查询：`O(log n)`；
- 单次区间加法更新：`O(log n)`；
- 单次单点赋值：`O(log n)`（本实现通过一次点查询 + 一次点加法）；
- 额外空间：`O(n)`（`tree` 与 `lazy`）。

## R08

边界与异常处理：
- 空数组：构造时报 `ValueError`。  
- 非法区间（`l>r` 或越界）：报 `IndexError`。  
- 非法单点索引：报 `IndexError`。  
- 随机回归中所有查询均和暴力法比较，若不一致会触发 `assert`，立即暴露错误。

## R09

MVP 取舍说明：
- 只实现一个聚合函数（区间和），不扩展到 `min/max/gcd`。  
- 只实现一种懒更新（区间加法），不叠加区间赋值等复杂标记组合。  
- 不依赖第三方数据结构库，核心逻辑全部手写，便于审计每一步。  
- 使用 `numpy` 仅用于生成可复现测试数据，不把算法本身交给黑盒。

## R10

`demo.py` 主要函数职责：
- `SegmentTree.__post_init__`：初始化并触发建树。  
- `_build`：递归建树。  
- `_apply`：对节点施加“整段加 delta”的即时效果。  
- `_push`：懒标记下传。  
- `_pull`：由子节点回收父节点值。  
- `_range_add / range_add`：区间加法更新。  
- `_range_sum / range_sum`：区间求和查询。  
- `point_set`：单点赋值。  
- `run_deterministic_case`：固定操作序列演示。  
- `run_randomized_regression`：随机操作回归校验。  
- `main`：执行全部测试并打印结果。

## R15

方法对比：
- 对比暴力数组：
  - 更新或查询区间都需 `O(n)`；
  - 线段树将单次操作降到 `O(log n)`。  
- 对比树状数组（Fenwick）：
  - 树状数组常用于前缀和，扩展复杂区间操作时技巧性更强；
  - 线段树更通用，便于支持更多聚合与更新类型。  
- 对比稀疏表：
  - 稀疏表适合静态查询；
  - 线段树适合动态更新。

## R18

`demo.py` 源码级算法流程（8 步）：
1. `main` 依次调用 `run_deterministic_case` 与 `run_randomized_regression`。  
2. 每个测试先构造原始数组，再初始化 `SegmentTree`；`__post_init__` 分配 `tree/lazy` 并触发 `_build`。  
3. `_build` 递归划分区间，叶子写入原值，内部节点通过 `_pull` 合并为区间和。  
4. 执行 `range_add` 时，若节点被完全覆盖则 `_apply` 直接更新节点和并累加懒标记。  
5. 若仅部分覆盖，则先 `_push` 把父节点懒标记下传，再递归左右子区间，最后 `_pull` 回收父值。  
6. 执行 `range_sum` 时，完全覆盖直接返回节点和；部分覆盖同样先 `_push` 再递归求和。  
7. 执行 `point_set(i,x)` 时，先查询 `sum(i,i)` 得当前值，再转成 `add(i,i,x-cur)`。  
8. 所有查询结果都与暴力数组即时对比（`assert`），随机回归结束后再做全区间总和一致性校验。
