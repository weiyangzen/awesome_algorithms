# 单词拆分

- UID: `CS-0057`
- 学科: `计算机`
- 分类: `动态规划`
- 源序号: `74`
- 目标目录: `Algorithms/计算机-动态规划-0074-单词拆分`

## R01

问题定义（Word Break / LeetCode 139）：
- 给定字符串 `s` 与词典 `wordDict`（可重复使用词典中的单词）；
- 判断是否能把 `s` 拆分为若干个词典单词的拼接；
- 要求拆分后不留空隙、不重叠、顺序与原字符串一致。

本条目在返回布尔值之外，还输出一条可行拆分方案（若存在），便于解释 DP 状态转移过程。

## R02

形式化描述：
- 输入：
  - 字符串 `s`，长度记为 `n`；
  - 词典集合 `D`（元素为非空字符串）。
- 输出：
  - `can_break`：`s` 是否可由 `D` 中单词拼接得到；
  - `segments`：若可拆分，给出一条分词结果，如 `["leet", "code"]`。

约定：
- 空字符串 `""` 视为可拆分（可由“0 个单词”组成）；
- 词典中的空字符串不允许进入计算（实现中会拒绝）。

## R03

动态规划状态设计：
- 定义 `dp[i]`：前缀 `s[:i]` 是否可被词典拆分（`0 <= i <= n`）；
- 初始状态：`dp[0] = True`（空前缀可拆分）；
- 目标状态：`dp[n]`；
- 转移条件：
  - 若存在 `j < i`，满足 `dp[j] = True` 且 `s[j:i] in D`，则 `dp[i] = True`。

这是典型“前缀可达性”DP：从已可达前缀扩展到更长前缀。

## R04

路径重建状态：
- `parent[i]`：使 `dp[i]=True` 的前驱下标 `j`；
- `picked_word[i]`：该次转移使用的词 `s[j:i]`；
- 当 `dp[n]=True` 时，从 `i=n` 逆推 `parent` 直到 `0`，反转得到正向分词序列。

这样可以在 `O(n)` 额外回溯代价下给出可解释结果，而不仅是布尔判定。

## R05

正确性直觉：
- `dp[i]` 的语义是“前缀 `s[:i]` 存在合法拆分”；
- `dp[0]=True` 对应空拆分基例；
- 转移只在 `dp[j]=True` 且 `s[j:i]` 属于词典时触发，因此每一步都合法；
- 任意一条合法拆分都可映射成一串严格递增下标链 `0 = i0 < i1 < ... < ik = n`，并在 DP 中逐步激活。

故 `dp[n]` 与“存在合法单词拆分”充要等价。

## R06

复杂度分析：
- 设 `n = len(s)`，`L =` 词长种类数；
- 实现中按候选词长枚举转移，时间复杂度约为 `O(n * L * slice_cost)`；
- 在 Python 中切片 `s[j:i]` 需拷贝，严格估计最坏可达 `O(n^3)`，但在常见数据上通常明显低于朴素三重枚举；
- 空间复杂度 `O(n + |D|)`（`dp`、`parent`、`picked_word` 与词典集合）。

对照算法：
- BFS 可达性：通常 `O(n^2)` 级；
- 记忆化 DFS：最坏仍可较高，但小规模校验很实用。

## R07

`demo.py` 的三条求解路径：
- `word_break_dp_trace`：主实现，返回可拆分布尔值与一条分词方案；
- `word_break_bfs`：基于队列的可达性搜索，输出布尔值；
- `word_break_dfs_memo`：记忆化 DFS（小规模精确对照），同时可返回一条方案。

脚本会自动断言三者在可拆分性上保持一致。

## R08

输入校验策略（`normalize_inputs`）：
- `s` 必须为字符串；
- `word_dict` 必须为字符串序列；
- 自动去重，保留非空词；
- 若去重后为空词典且 `s` 非空，直接不可拆分；
- 预计算并排序词长列表，减少无效转移枚举。

该步骤统一了错误处理与数据标准化，避免主算法重复防御逻辑。

## R09

边界情形：
- `s=""`：可拆分，方案为空列表；
- `s` 非空但 `word_dict=[]`：不可拆分；
- 词典包含重复词：不影响结果（集合去重）；
- 存在前缀陷阱（如 `s="catsandog"`）应正确返回 `False`；
- 多方案并存时，DP 输出“按固定词长顺序首次命中”的稳定方案。

## R10

验证机制：
- 固定案例：覆盖可拆分、不可拆分、空串、重叠词、重复词等；
- 随机回归：随机生成小字符串与随机词典，比较 `DP/BFS/DFS` 三种实现；
- 方案校验：
  - 连接后应还原原串；
  - 每个分词都在词典中；
  - 空串时方案必须为空列表。

## R18

`demo.py` 源码级算法流程（8 步）：
1. `main` 构造固定测试集，逐个调用 `run_case`，最后执行随机回归。  
2. `run_case` 先调用 `word_break_dp_trace`：初始化 `dp[0]=True`，其余为 `False`。  
3. 对每个位置 `i`，`word_break_dp_trace` 枚举候选词长 `wlen`，令 `j=i-wlen`；若 `dp[j]` 为真且 `s[j:i]` 在词典内，则置 `dp[i]=True`。  
4. 命中转移时同步记录 `parent[i]=j` 与 `picked_word[i]=s[j:i]`，用于后续解释路径。  
5. 若 `dp[n]` 为真，从 `n` 逆向沿 `parent` 回溯至 `0`，反转得到 `segments` 与 `cut_positions`。  
6. `run_case` 再调用 `word_break_bfs`，用队列在下标图上做可达性搜索，得到独立布尔结论。  
7. 小规模输入时，`run_case` 还调用 `word_break_dfs_memo`（记忆化递归）做精确交叉验证。  
8. `is_valid_segmentation` 检查分词合法性，并在 `randomized_regression` 中执行多轮随机一致性断言。
