# Coppersmith方法

## R01

Coppersmith 方法用于求解模方程 `f(x) ≡ 0 (mod N)` 的“小整数根”问题。典型目标是：在已知 `|x| < X` 的前提下，从大模数 `N` 和多项式 `f` 中恢复未知根 `x`。

## R02

典型应用场景：
- RSA 相关攻击中的低位泄露、短填充、部分已知明文。
- 结构化同余方程（尤其是单变量、首一多项式）的小根恢复。
- 需要把“模约束 + 小范围”联合利用的问题。

## R03

核心思想：
- 构造一组与 `f(x)`、`N` 相关的多项式，使它们在真实小根处都“模 `N^m` 消失”。
- 将这些多项式的系数嵌入整数格，做 LLL 降基，得到一个“短向量对应的新多项式”。
- 该新多项式在整数域里更容易把小根显式暴露出来，再做候选验证。

## R04

本目录 `demo.py` 的输入输出约定：
- 输入：脚本内部构造固定 demo 实例（无需交互输入）。
- 输出：
  - 打印 `N`、`f(x)`、参数 `m,t,X`。
  - 打印格维度与恢复出的候选小根。
  - 对候选根打印 `f(root) mod N` 验证结果。

## R05

MVP 参数（Howgrave-Graham 风格单变量版本）：
- `d = deg(f)`，要求 `f` 为首一多项式。
- `m`：幂次层数（控制格大小和约束强度）。
- `t`：附加位移多项式数量。
- `X`：小根上界（搜索区间 `(-X, X)`）。
- 格维度 `n = d*m + t`。

## R06

伪代码（简化版）：

```text
输入: monic f(x), N, X, m, t
构造多项式族:
  g_{i,j}(x) = x^j * N^(m-i) * f(x)^i, 0<=i<m, 0<=j<d
  h_j(x)    = x^j * f(x)^m,             0<=j<t
把每个多项式做变量缩放 x -> xX，按系数向量组成整数格基 B
B' = LLL(B)
对 B' 的向量还原成多项式 q(x)
在 (-X, X) 扫描整数 cand:
  若 q(cand)=0 且 f(cand) mod N = 0，则 cand 为小根
```

## R07

正确性直觉：
- 构造的基多项式在真实根处都带有 `N` 的高次因子，因此其整数线性组合也保留该结构。
- LLL 倾向产出系数较小的组合；在 Howgrave-Graham 条件下，这个组合不仅“模意义小”，还会提升到“整数意义为零”。
- 于是小根会成为新多项式的整数根，之后可被有限区间枚举或因式分解检出。

## R08

复杂度（MVP 量级）：
- 设格维度 `n = d*m + t`。
- LLL 大致主导成本，常见估计可写为多项式时间，实践中约随 `n^4` 到 `n^5` 增长（取决于实现细节）。
- 根扫描成本 `O(X * r)`，`r` 为参与检测的还原多项式个数。

## R09

数据结构设计：
- 多项式使用“低次到高次”的整数系数列表表示。
- 格基使用 `List[List[int]]` 的整数矩阵。
- Gram-Schmidt 与 `mu` 系数使用 `Fraction`，避免浮点误差破坏 LLL 判定。

## R10

边界与异常处理：
- 非首一多项式会直接报错（本 MVP 仅实现 monic 场景）。
- `m<=0`、`t<=0` 或 `X<=1` 抛出 `ValueError`。
- 若降基后未恢复到目标根，脚本会抛出 `RuntimeError`，提示调参或更换实例。

## R11

本目录 MVP 的工程取舍：
- 不依赖 Sage/fpylll 等专用库，避免黑箱。
- 直接在源码中实现：

  - LLL 降基
  - Coppersmith 格构造与缩放还原
- 目标是“可读、可跑、可追踪”，而非极限性能。

## R14

常见实现错误：
- 忘记在格构造时做 `x -> xX` 缩放，导致短向量不对应正确小根结构。
- LLL 的 size-reduction/交换顺序写错，造成降基失败或死循环。
- 从 `Q(xX)` 还原 `Q(x)` 时未按 `X^k` 逐项除回。
- 只检查 `q(c)=0` 不检查 `f(c) mod N = 0`，导致假阳性。

## R15

最小测试清单：
- 功能测试：固定 demo 是否能恢复 `x=123`。
- 模验证：对每个恢复根检查 `f(root) % N == 0`。
- 参数鲁棒：在可行范围内调整 `X,m,t`，观察是否仍可恢复。
- 异常测试：给非首一 `f`、非法 `m/t/X` 应触发预期异常。

- 该实现是教学版，不适合大规模真实攻击任务。
- LLL 采用 `Fraction`，维度升高后会明显变慢。
- 当前示例是单变量、低维、人工可控参数；真实场景需严谨参数推导。

## R18

源码级算法流程（本实现，无黑箱第三方）
1. 在 `build_demo_instance()` 中固定构造 `N`、首一二次多项式 `f(x)`、真实小根 `x0` 和上界 `X`。  
2. `build_coppersmith_lattice()` 生成两类基多项式：`x^j*N^(m-i)*f(x)^i` 与 `x^j*f(x)^m`，并做 `x -> xX` 缩放。  
3. 将缩放后的系数按同维度补零，组装为整数格基矩阵 `B`。  
4. `lll_reduction()` 调用 `gram_schmidt()` 计算 `mu` 与正交基，循环执行 size-reduction 与 Lovasz 条件交换，得到约化基 `B'`。  
5. 对 `B'` 每个向量，`vector_to_unscaled_poly()` 按列次幂除以 `X^k`，把 `Q(xX)` 还原为 `Q(x)`。  
6. 在 `(-X, X)` 内扫描候选 `cand`，先检查 `Q(cand)==0`，再检查 `f(cand) mod N == 0`。  
7. 汇总全部候选根并去重，输出 `Recovered roots`。  
8. `main()` 断言植入根在结果中并打印逐根模验证值，作为 MVP 端到端正确性确认。
