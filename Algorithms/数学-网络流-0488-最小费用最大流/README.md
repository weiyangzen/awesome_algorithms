## R01

最小费用最大流（Min-Cost Max-Flow, MCMF）解决的问题是：
- 在有向容量网络中，先把从源点 `s` 到汇点 `t` 的流量做到最大；
- 在所有“最大流”方案里，再选总费用最小的那一个。

它是最大流与最短路的组合问题，广泛用于物流、排产、任务分配、通信路由等场景。

本目录提供一个可运行、可审计的 Python MVP：
- 核心算法手写实现，不依赖图算法黑盒；
- 使用“逐次最短增广路（SSP）+ 势能重标（Johnson potential）”；
- 支持负费用边（无负环前提）；
- 提供 `scipy.optimize.linprog` 的交叉验证，但不用于主求解路径。

## R02

问题定义（本实现）：

- 输入：
  - 顶点数 `n`；
  - 有向边 `[(u, v, capacity, cost), ...]`；
  - 源点 `source`、汇点 `sink`。
- 约束：
  - 边流量 `0 <= f_e <= capacity_e`；
  - 中间点满足流守恒；
  - 源汇净流满足 `|f|`（流值）。
- 目标（字典序）：
  1. 最大化 `|f|`；
  2. 在 `|f|` 最大时最小化总费用 `sum(cost_e * f_e)`。

输出：
- `max_flow`：最大流值；
- `min_cost`：该最大流下的最小费用；
- 每条原始边的最终流量明细。

## R03

数学要点：

1. 残量网络中，前向边表示可继续增流，反向边表示可撤销流量。  
2. 每轮在残量网络找一条从 `s` 到 `t` 的最短费用路径，并沿路径增广。  
3. 若直接在残量图跑 Dijkstra，会被负权边破坏；因此引入势能 `pi(v)`。  
4. 定义约化费用：`c'(u,v) = c(u,v) + pi(u) - pi(v)`。  
5. 若 `pi` 合法（即对可用残量边满足 `c' >= 0`），则可用 Dijkstra 在 `c'` 上求最短路。  
6. 每次最短路后更新势能 `pi(v) += dist'(v)`，保持后续约化费用非负。  
7. 当残量图中已不存在 `s->t` 路径时，流量已达最大；因为每次都选最短增广路，得到该最大流下的最小费用。

## R04

本实现采用的算法流程：

1. 校验输入合法性（维度、端点、容量、有限性）。  
2. 构建残量图：每条原边配一条反向边（费用取负）。  
3. 用 Bellman-Ford 计算初始势能，处理可能存在的负费用边。  
4. 循环执行：
   - 在“约化费用”图上用 Dijkstra 求 `s->t` 最短路；
   - 若不可达则停止（达到最大流）；
   - 回溯路径求可增流量 `delta`；
   - 更新残量容量、累计流量与总费用；
   - 势能重标 `pi += dist`。
5. 返回 `max_flow`、`min_cost` 和每条原边流量。

## R05

核心数据结构：

- `ResidualEdge(to, rev, cap, cost)`：
  - `to`：终点；
  - `rev`：反向边在 `graph[to]` 的下标；
  - `cap`：当前残量容量；
  - `cost`：该残量边单位费用。
- `graph: List[List[ResidualEdge]]`：残量邻接表。
- `potential: List[float]`：节点势能数组。
- `prev_v / prev_e`：最短路前驱节点和边索引，用于回溯增广路径。
- `original_refs`：记录每条原始边对应的前向残量边位置，便于回读最终流量。
- `MinCostMaxFlowResult`：结构化输出容器。

## R06

正确性关键点（对应代码）：

- 可行性保持：
  - 每次增广都不超过路径最小残量，容量约束始终成立；
  - 通过同步更新反向边，实现流量可撤销与守恒维护。
- 最大流性：
  - 当残量网络不存在 `s->t` 路径时，依据最大流理论，已无法继续增广。
- 最小费用性（在给定流量下）：
  - 每次选择最短费用增广路；
  - 势能重标保证 Dijkstra 等价于在原费用上做最短路；
  - 逐次增广到最大流后得到“最大流中的最小费用解”。
- 负环处理：
  - 若源点可达区域存在负费用环，费用无下界，本实现显式报错停止。

## R07

复杂度（`V` 点、`E` 边）：

- 初始 Bellman-Ford：`O(VE)`。  
- 每轮最短路（Dijkstra + 堆）：`O(E log V)`。  
- 设总增广轮数为 `K`，则主循环复杂度约 `O(K * E log V)`。  
- 总体：`O(VE + K * E log V)`。  
- 空间复杂度：`O(V + E)`（不含日志输出）。

说明：`K` 受网络结构与容量分布影响，在整数容量场景可视为增广次数。

## R08

边界与异常处理：

- `n <= 1`、`source == sink`、端点越界：抛 `ValueError`。  
- 容量为负或边属性非有限数：抛 `ValueError`。  
- 若最短路回溯失败（理论上不应出现）：抛 `RuntimeError`。  
- 若检测到源点可达负费用环：抛 `RuntimeError`。  
- 数值容差通过 `EPS = 1e-12` 控制，减少浮点误差影响。

## R09

MVP 取舍说明：

- 优先“短小+透明”实现：
  - 不引入 NetworkX/OR-Tools 等图优化黑盒作为主解器；
  - 明确展示残量、势能、最短路、增广四个核心动作。
- 额外使用 `scipy.optimize.linprog` 仅做结果校验：
  - 固定在“已知目标流量”的线性规划上比较最优费用；
  - 不参与主循环决策。
- 不覆盖的高级特性：
  - 费用缩放（cost scaling）；
  - 容量缩放；
  - 多源多汇（可通过超级源汇扩展）。

## R15

与相关算法对比：

- 对比最大流（如 Dinic / Push-Relabel）：
  - 最大流只关心“流量最大”，不关心费用；
  - MCMF在最大流基础上进一步优化成本。
- 对比“环消去法（Cycle Canceling）”：
  - 环消去实现直观但常较慢；
  - SSP + potential 在工程上通常更高效。
- 对比线性规划黑盒：
  - 黑盒调用短，但过程不透明；
  - 本实现能追踪每轮最短路与增广细节，适合教学和排障。

## R18

`demo.py` 源码级算法流程（9 步）：

1. `main` 定义两组固定网络样例并调用 `run_case`。  
2. `run_case` 调用 `min_cost_max_flow` 求解，并执行 `_flow_conservation_check`。  
3. `min_cost_max_flow` 先经 `_validate_input` 校验输入，再通过 `_add_edge` 构建残量图与反向边。  
4. `_bellman_ford_initial_potential` 计算初始势能：在可达残量边上做松弛，并检测负费用环。  
5. 进入主循环：在约化费用 `c'(u,v)=c(u,v)+pi(u)-pi(v)` 上用 Dijkstra 求 `s->t` 最短路，记录 `prev_v/prev_e`。  
6. 若汇点不可达，说明无法再增广，算法终止（已达最大流）。  
7. 若可达，则按前驱回溯路径，取最小残量作为本轮 `delta`，并更新路径上前向/反向残量容量。  
8. 同步累计 `total_flow += delta`、`total_cost += delta * edge_cost`，并执行势能重标 `pi(v) += dist(v)`。  
9. 循环结束后回读每条原边流量，返回结果；`run_case` 再用 `_linprog_cost_check` 做最小费用交叉验证并断言通过。
