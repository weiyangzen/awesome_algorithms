# 弗兰克-赫兹实验 (Franck-Hertz Experiment)

- UID: `PHYS-0250`
- 学科: `物理`
- 分类: `原子物理`
- 源序号: `253`
- 目标目录: `Algorithms/物理-原子物理-0253-弗兰克-赫兹实验_(Franck-Hertz_Experiment)`

## R01

弗兰克-赫兹实验展示了电子与原子碰撞的**能量量子化**：当加速电压 `V_a` 逐步升高时，集电极电流 `I_c` 不是单调上升，而会出现周期性峰谷。其核心机制是：

- 电子动能达到原子第一激发能 `E_exc`（汞原子约 `4.9 eV`）后，发生非弹性碰撞并损失近似固定能量；
- 若电子在到达栅极前刚经历非弹性碰撞，剩余动能可能不足以克服反向电压 `V_r`，导致集电流下降。

因此相邻峰/谷间距可用于估计 `E_exc/e`。

## R02

该实验是“离散能级”最经典的直接证据之一，历史上支持了玻尔原子模型。工程和教学价值包括：

- 以宏观可测量电流曲线验证微观量子跃迁；
- 连接原子物理、碰撞过程、气体放电与电子器件建模；
- 为更复杂的等离子体/气体电子输运模型提供最小可解释基线。

## R03

本条目使用简化 1D 蒙特卡洛模型重建 `I_c - V_a` 曲线。主要方程：

1. 均匀电场近似下，电子沿管长 `L` 每段 `dx` 获得能量 `dE = V_a * dx / L`（单位 eV）。
2. 两次碰撞间自由程 `s ~ Exp(lambda)`，`lambda` 为平均自由程。
3. 若碰撞时 `E >= E_exc`，以概率 `p_inelastic` 发生非弹性碰撞并执行 `E <- E - E_exc`。
4. 电子到达栅极后，若 `E >= V_r` 则被集电，记为一次成功。
5. 归一化集电流估计为
   `I_c/I_0 = (收集到的电子数) / (总发射电子数)`。

## R04

模型假设（MVP 级）：

- 一维漂移，忽略角散射与空间电荷；
- 电场沿加速区均匀；
- 激发能取单一阈值 `E_exc`，不区分多条激发谱线；
- 非弹性碰撞后仅扣除一份激发能，弹性碰撞不改电子动能；
- 用 `V_r` 作为集电判据的能量阈值。

这些假设牺牲细节精度，但保留了 Franck-Hertz 峰谷结构的主导机制。

## R05

复杂度分析：

- 设扫描电压点数为 `M`，每点模拟电子数为 `N`；
- 设每个电子平均碰撞次数为 `C ~ L/lambda`；
- 时间复杂度约 `O(M * N * C)`；
- 空间复杂度约 `O(M)`（保存电压-电流曲线）。

在 `demo.py` 的默认参数下，`M=161`、`N=5000`，可在普通 CPU 上快速完成。

## R06

算法流程（高层）：

1. 对每个 `V_a` 进行一次独立蒙特卡洛估计；
2. 每个电子重复“采样自由程 -> 累积加速能量 -> 可能发生非弹性碰撞”直到到达栅极；
3. 统计是否越过 `V_r`，得到当前电压点的 `I_c/I_0`；
4. 扫描完成后做平滑与局部峰值检测；
5. 计算峰间距均值，验证其与 `E_exc` 的一致性。

## R07

正确性直觉：

- 当 `V_a < E_exc`，电子几乎不会发生激发碰撞，`I_c` 主要随 `V_a` 上升；
- 当 `V_a` 允许“恰好一次激发碰撞且发生在后段”，部分电子在末端失能，`I_c` 会出现下降；
- 随 `V_a` 提高，电子可经历 2 次、3 次激发碰撞，形成近似周期峰谷；
- 因每次主要损失固定能量 `E_exc`，峰间距应接近 `E_exc/e`（单位 V）。

## R08

`demo.py` 的 MVP 目标：

- 非交互、一次运行即可输出关键数值；
- 仅依赖 `numpy`，不调用黑盒物理仿真器；
- 输出采样点电流、检测到的峰电压、峰间距均值；
- 带内置校验：至少检测到 3 个峰，且平均峰间距接近激发能。

## R09

`demo.py` 主要函数接口：

- `simulate_single_electron(...) -> bool`
- `collector_current_ratio(...) -> float`
- `moving_average(x, window=5) -> np.ndarray`
- `local_maxima_indices(y) -> np.ndarray`
- `main() -> None`

输入均为显式参数，便于后续做参数扫描、批处理或单元测试。

## R10

验证策略：

1. **形状验证**：`I_c(V_a)` 需出现多个局部峰；
2. **物理量验证**：峰间距均值应靠近 `E_exc`（默认容忍误差 `<=1.4 V`）；
3. **稳定性验证**：固定随机种子后输出可复现；
4. **数值可读性**：打印若干采样点，便于人工检查曲线趋势。

## R11

边界与异常处理：

- `V_a <= 0` 时电子直接判定不可收集；
- `moving_average` 要求窗口是正奇数，否则抛 `ValueError`；
- 若峰数量不足，抛 `RuntimeError` 提示需调参；
- 若峰间距与 `E_exc` 偏差过大，抛 `RuntimeError` 表示本轮标定失败。

## R12

默认参数（`demo.py`）及含义：

- `E_exc = 4.9 eV`：汞第一激发能的典型量级；
- `V_r = 1.5 V`：集电区反向势垒；
- `L = 1.0`, `lambda = 0.18`：控制平均碰撞次数；
- `p_inelastic = 0.92`：高于阈值时发生非弹性碰撞的概率；
- `N = 5000`：每个电压点的统计样本量；
- `V_a in [0,40]`（161 点）：覆盖多个周期。

## R13

本算法不是最优化近似算法，因此不存在“近似比”概念。可给出的统计保证是：

- 估计量 `I_c/I_0` 为伯努利样本均值；
- 标准误差约为 `sqrt(p(1-p)/N)`；
- 在最不利 `p=0.5` 下，默认 `N=5000` 时标准误差约 `0.007`；
- 因而峰位与峰间距的估计会随 `N` 增大而稳定。

## R14

常见失效模式：

1. `N` 太小导致曲线噪声大，峰检测不稳定；
2. `lambda` 太大（几乎无碰撞）或太小（过度耗散）都会削弱振荡结构；
3. `V_r` 取值不合理会使几乎全收集或几乎全截止；
4. 过度平滑会抹平真实峰值，平滑不足会引入伪峰。

## R18

`demo.py` 源码级流程拆解（8 步）：

1. `main` 设定物理参数（`E_exc`, `V_r`, `lambda` 等）与数值参数（电压网格、样本数、随机种子）。
2. 对每个电压 `V_a` 调用 `collector_current_ratio`，执行一次独立蒙特卡洛估计。
3. `collector_current_ratio` 内部循环 `N` 个电子，逐个调用 `simulate_single_electron`。
4. `simulate_single_electron` 用 `rng.exponential(lambda)` 采样自由程，按 `dE = V_a * ds / L` 累积能量。
5. 当能量超过阈值时，以 `rng.random() < p_inelastic` 决策是否发生非弹性碰撞，并执行 `E <- E - E_exc`。
6. 电子到达栅极后与 `V_r` 比较：`E >= V_r` 记为收集成功，最终得到 `I_c/I_0`。
7. 全扫描后，`moving_average` 做轻量平滑，`local_maxima_indices` 用三点比较法检测局部峰。
8. 由峰电压差分得到平均峰间距，与 `E_exc` 对比并做断言，完成“模型 -> 曲线 -> 物理量”闭环验证。

这里没有把第三方库当黑盒：`numpy` 仅提供随机采样与数组运算，Franck-Hertz 的碰撞判定、能量更新、峰值检测与验证逻辑均在源码中显式实现。
