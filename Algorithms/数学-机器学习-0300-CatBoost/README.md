## R01

CatBoost（Categorical Boosting）是梯度提升树家族中专门面向表格数据（尤其含大量类别特征）的方法。
它的核心不是“再造一套 GBDT 名字”，而是解决传统 boosting 在类别特征上常见的两个痛点：

- 类别编码容易泄漏目标信息，导致离线分数虚高。
- 稀疏高基数类别在 one-hot 下维度爆炸、泛化不稳。

CatBoost 用“有序目标统计 + 对称树（oblivious tree）+ boosting”形成一套完整训练范式。

## R02

二分类情形可抽象为：给定

- 数值特征 `x_num in R^d`
- 类别特征 `x_cat in C^k`
- 标签 `y in {0,1}`

学习加法模型

`F_M(x) = F_0 + sum_{m=1..M} eta * h_m(x)`，`p(x)=sigmoid(F_M(x))`

其中每个 `h_m` 是深度受限的对称树，目标最小化对数损失（log-loss）。

## R03

CatBoost 的关键思想：

- 用有序（ordered）方式为类别构造目标统计特征，降低 target leakage。
- 使用对称树：同一层所有节点共享同一个分裂条件，结构规则、推理快。
- 用梯度提升逐轮拟合残差方向（常配合二阶信息与正则）。

这三者组合后，既保留 boosting 对非线性和特征交互的表达力，又改善类别特征处理的稳定性。

## R04

输入：

- 训练集 `(X_num, X_cat, y)`
- 超参数：`n_estimators`、`depth`、`learning_rate`、`l2_leaf_reg`

输出：

- 初始偏置 `F0`
- 有序类别编码器（训练态统计）
- 一组按顺序训练的对称树 `{h_1,...,h_M}`
- 预测概率 `p(x)` 与类别预测 `y_hat`

## R05

本条目采用二分类 log-loss 作为主目标：

`L = -mean(y*log(p) + (1-y)*log(1-p))`

原因：

- 与概率输出天然一致。
- 既能看分类对错，也能衡量概率校准质量。
- 与 CatBoost 默认的分类训练目标一致性较高。

## R06

训练流程（MVP 版）可概括为：

1. 先对类别列做 ordered target encoding（训练时按随机排列的历史前缀统计）。
2. 用正类比例初始化 `F0`，得到初始 raw score。
3. 每轮计算 `p=sigmoid(F)`，并形成梯度 `g=y-p` 与 Hessian `h=p(1-p)`。
4. 训练一棵深度固定的对称树，按叶子聚合梯度/二阶量求叶值。
5. 执行 `F <- F + eta * h_m(x)`，记录训练损失。
6. 迭代到 `n_estimators` 轮后输出模型。

## R07

复杂度（粗略）：

- 设样本数 `n`，特征数 `d`，候选分裂数 `S`，树深 `D`，轮数 `M`。
- 每轮对称树搜索近似 `O(D * S * n)`。
- 总训练量约 `O(M * D * S * n)`。
- 推理时每棵树仅需 `D` 次比较，单样本约 `O(M * D)`。

对称树在推理吞吐和实现简洁性上通常优于同深度的任意结构树。

## R08

优点：

- 类别特征友好，不依赖大规模 one-hot。
- 通过 ordered 统计缓解目标泄漏，离线评估更可信。
- 对称树结构规则，工程上易优化。

局限：

- 参数较多（深度、学习率、迭代轮数、正则等）需要调优。
- 如果类别编码策略或数据切分不当，仍可能过拟合。
- 在样本极少且类别极稀疏时，统计编码方差会较大。

## R09

适用场景：

- 表格建模任务，且包含中高基数类别特征。
- 需要稳定离线验证和较好线上推理效率。
- 希望在“树模型效果”和“类别处理能力”之间取得平衡。

不太适用：

- 纯图像/纯语音等非表格主任务。
- 仅有超高维稀疏文本且需要线性可解释基线时。

## R11

与相近方法对比（简化）：

- GBDT（基础版）：通常把类别当数值或手工 one-hot，类别处理不原生。
- XGBoost：工程能力强，但类别处理策略取决于配置与版本。
- LightGBM：训练速度快，类别支持较强，但具体行为依赖数据分布与参数。
- CatBoost：以 ordered 统计与对称树为核心设计，更强调类别特征鲁棒性。

## R12

工程注意事项：

- 类别编码必须仅用训练信息，禁止把测试集统计泄漏进训练流程。
- 训练/验证分离要先做，再在训练子集上构造统计编码。
- log-loss 计算要做概率裁剪，避免 `log(0)`。
- 若类别极不平衡，阈值与评估指标（F1/AUC）要联合观察。

结果解读建议：

- 再看 `f1`/`auc`，检查是否仅靠阈值碰巧提升。

## R16

常见问题与排查：

- 指标不升反降：降低学习率、增大轮数或增强正则（`l2_leaf_reg`）。
- 训练集明显好于测试集：减小树深、增大 `min_data_in_leaf`。
- 类别很多且样本少：增大先验权重，避免编码值方差过大。

## R17

可扩展方向：

- 增加多分类支持（softmax / one-vs-rest）。
- 加入 early stopping 与独立验证集监控。
- 实现特征组合 CTR（高阶类别交互统计）。
- 支持缺失值专门分裂策略与更完整的量化流程。

## R18

CatBoost 源码级训练流（抽象成 8 步，不当黑箱）：

1. 读取与解析训练配置（损失函数、树深、迭代轮数、正则项等），构建训练上下文。  
2. 对数值特征做分桶/边界量化（binarization），把连续值映射到离散桶以便高效分裂搜索。  
3. 对类别特征构造在线/有序 CTR：按排列顺序只用“当前样本之前”的标签统计来编码，抑制泄漏。  
4. 进入 boosting 主循环，基于当前预测计算一阶/二阶统计（梯度与 Hessian）。  
5. 用贪心策略逐层选择对称树分裂：每层一个全局分裂条件，作用于该层全部节点。  
6. 在最终叶子上聚合梯度统计并计算叶值（含 `l2_leaf_reg` 等正则），形成本轮树输出。  
7. 以学习率缩放后更新全体样本 raw score，并记录训练目标与评估指标。  
8. 迭代结束后导出模型：量化边界、CTR 规则、树结构与叶值，推理时按同流程复现特征变换与树累加。

本目录的 `demo.py` 对应地复现了其中最核心的三段：ordered CTR、对称树构建、boosting 更新闭环。
