# Manacher算法

- UID: `MATH-0529`
- 学科: `数学`
- 分类: `字符串算法`
- 源序号: `529`
- 目标目录: `Algorithms/数学-字符串算法-0529-Manacher算法`

## R01

Manacher 算法用于在线性时间内求解字符串的回文半径信息，最常见目标是：
1. 最长回文子串。
2. 全部中心点的回文半径。
3. 回文子串总数（可由半径数组导出）。

相较于中心扩展法的 `O(n^2)` 最坏复杂度，Manacher 的核心优势是把总复杂度稳定在 `O(n)`。

## R02

核心思想是把原串统一转换为“带分隔符”的新串，让奇偶长度回文都变成同一种扩展模式。
例如原串 `abba` 转成 `^#a#b#b#a#$`（首尾哨兵用于边界保护）。
在该新串上维护每个位置的扩展半径 `p[i]`，即可统一处理原串里的奇回文和偶回文。

## R03

本目录 MVP 使用的数据结构：
1. `t`：变换后的字符串（含哨兵与分隔符）。
2. `p`：与 `t` 同长度的整型数组，`p[i]` 表示以 `i` 为中心可扩展的最大半径。
3. `center`：当前已知“最右回文”的中心。
4. `right`：当前已知“最右回文”的右边界（含）。

这四者即可完成线性扫描，不需要复杂外部依赖。

## R04

不变量定义：
1. 在扫描到位置 `i` 前，区间 `[center - p[center], right]` 是已处理范围内最靠右的回文。
2. 若 `i < right`，则 `i` 落在该回文覆盖区内，可利用镜像点 `mirror = 2 * center - i` 的半径信息做“下界初始化”。
3. 初始化后仅需继续向外补充扩展，避免重复比较。

这保证多数字符比较被复用，而不是每个中心都从 0 开始暴力扩展。

## R05

镜像初始化规则：
1. 当 `i < right` 时，先令 `p[i] = min(right - i, p[mirror])`。
2. 当 `i >= right` 时，令 `p[i] = 0` 从头扩展。

直觉上，`mirror` 在已知大回文内部与 `i` 对称，已知半径可以直接迁移，但最多只能迁移到右边界 `right`，不能越界假设。

## R06

扩展与边界更新：
1. 在 `t[i + 1 + p[i]] == t[i - 1 - p[i]]` 时持续扩展 `p[i] += 1`。
2. 若扩展后 `i + p[i] > right`，则刷新 `center = i`、`right = i + p[i]`。

因为 `right` 只会单调右移，整轮比较次数被摊薄到线性级别。

## R07

最长回文子串还原方式：
1. 在 `p` 中找到最大半径 `max_len` 与对应中心 `max_center`。
2. 原串起点为 `start = (max_center - max_len) // 2`。
3. 最长回文即 `text[start : start + max_len]`。

该映射来自“每个原串字符在变换串中被 `#` 隔开”的索引关系。

## R08

复杂度：
1. 时间复杂度：`O(n)`，`n` 为原串长度。
2. 空间复杂度：`O(n)`，主要是 `t` 与 `p`。

与 DP 回文表的 `O(n^2)` 空间相比，Manacher 在超长字符串场景更实用。

## R09

与其他方法对比：
1. 中心扩展法：实现简单，但最坏 `O(n^2)`。
2. DP（区间回文）：便于回答任意区间是否回文，但时间/空间常为 `O(n^2)`。
3. Manacher：专注回文半径与最长回文问题，复杂度最稳。

如果需求是“最长回文子串/回文计数”，Manacher 通常优先。

## R10

边界情况：
1. 空串：最长回文应为 `""`，计数为 `0`。
2. 单字符串：最长回文为该字符，计数为 `1`。
3. 全相同字符（如 `aaaaa`）：回文数量高，适合检验半径与计数公式。
4. 无长回文结构（如 `abcd`）：最长长度应退化为 `1`。
5. Unicode 文本：Python `str` 可直接比较字符，本实现可直接处理。

## R11

`demo.py` 暴露的核心函数：
1. `manacher(text)`：返回完整结果对象（变换串、半径数组、最长回文、总回文数）。
2. `brute_force_longest_palindrome(text)`：暴力基线（用于对拍）。
3. `brute_force_palindrome_count(text)`：暴力统计回文子串总数。
4. `run_deterministic_cases()`：固定样例验证。
5. `run_randomized_regression()`：随机回归测试。

## R12

运行方式（无交互输入）：

```bash
uv run python demo.py
```

脚本会自动：
1. 打印若干样例的 Manacher 结果。
2. 与暴力法逐例对照。
3. 执行随机回归并在失败时抛出断言。

## R13

主要输出字段解释：
1. `longest`：最长回文子串。
2. `start` / `length`：最长回文在原串中的起点与长度。
3. `pal_count`：回文子串总数（包含重复位置）。
4. `radius_preview`：半径数组前若干项，便于快速人工检查。
5. `All Manacher checks passed.`：所有断言通过。

## R15

常见实现错误：
1. 变换串缺少左右哨兵，导致扩展越界。
2. `mirror` 公式写错（应为 `2 * center - i`）。
3. `right` 更新条件写成 `>=` 并伴随错误中心更新，导致状态抖动。
4. 原串下标映射错误（`(center - len) // 2` 写错）。
5. 回文计数公式误用，导致计数偏大或偏小。

## R16

本目录 MVP 的验证策略：
1. 固定样例覆盖：空串、偶回文、奇回文、重复字符、混合结构。
2. 暴力对拍：最长回文与回文总数均与暴力结果比较。
3. 随机回归：固定随机种子，批量生成短串进行自动断言。

这样可以在低成本下显著降低实现偏差风险。

## R18

`demo.py` 的源码级算法流程（8 步）：
1. `main()` 调用 `run_deterministic_cases` 与 `run_randomized_regression`，形成“固定样例 + 随机样例”的验收闭环。
2. `manacher(text)` 先调用 `transform_text` 把原串变为 `^#...#$` 结构，初始化 `p`、`center`、`right`。
3. 对每个位置 `i` 扫描时，若 `i < right`，用镜像位置 `mirror = 2 * center - i` 初始化 `p[i] = min(right - i, p[mirror])`。
4. 在初始化基础上执行 while 扩展：比较 `t[i + 1 + p[i]]` 与 `t[i - 1 - p[i]]`，相等则半径递增。
5. 若 `i + p[i]` 超过当前 `right`，更新最右回文的 `center` 和 `right`，保证后续位置可复用更多已知信息。
6. 扫描结束后遍历 `p` 选出最优中心，按 `start = (center - len) // 2` 映射回原串，得到最长回文子串。
7. 同时通过 `sum((radius + 1) // 2 for radius in p)` 计算回文子串总数（包含重复位置），并打包为 `ManacherResult`。
8. 验证阶段用 `brute_force_longest_palindrome` 与 `brute_force_palindrome_count` 做真值对照；任一差异直接触发断言，全部通过才输出成功信息。

说明：本 MVP 未调用第三方字符串黑盒 API，核心 Manacher 扩展逻辑在源码中完整显式实现。
