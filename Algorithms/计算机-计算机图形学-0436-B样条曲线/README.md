## R01

B样条曲线（B-spline Curve）是一类由控制点、次数和结点向量共同定义的分段多项式参数曲线。对 `p` 次 B 样条：

`C(u) = sum_{i=0..n} N_{i,p}(u) P_i`

其中 `P_i` 为控制点，`N_{i,p}(u)` 为 B 样条基函数，`u` 是参数。相比单段高次曲线，B 样条通过“局部基函数 + 分段结构”实现更稳定、更可控的曲线建模。

## R02

B样条在计算机图形学和几何建模中非常常见，典型场景包括：

- CAD/CAM 路径设计与工业曲线拟合；
- 3D 建模软件中的样条编辑；
- 字体、矢量图形、动画路径平滑；
- 机器人与数控路径规划中的平滑轨迹生成。

其核心优势是局部控制：调整一个控制点通常只影响曲线局部区间，不会像高次全局多项式那样整体剧烈变化。

## R04

核心数学关系如下：

1. 曲线表达式：`C(u) = sum_i N_{i,p}(u) P_i`。
2. 基函数零次定义：
   `N_{i,0}(u) = 1` 当 `U_i <= u < U_{i+1}`，否则为 `0`（最右端点按闭区间处理）。
3. Cox-de Boor 递推：
   `N_{i,p}(u) = ((u-U_i)/(U_{i+p}-U_i)) N_{i,p-1}(u) + ((U_{i+p+1}-u)/(U_{i+p+1}-U_{i+1})) N_{i+1,p-1}(u)`。
4. de Boor 点求值（几何递推）：在参数 `u` 对应区间中选取 `p+1` 个相邻控制点，按层线性插值得到 `C(u)`。
5. 开放均匀结点向量：首尾各重复 `p+1` 次，使曲线插值首末控制点。

## R05

设控制点数为 `n_ctrl`，次数为 `p`，采样点数为 `m`：

- de Boor 单点评估复杂度 `O(p^2)`，整条采样约 `O(m p^2)`；
- Cox-de Boor 基函数矩阵显式求值约 `O(m n_ctrl p)`（递归实现常数更大）；
- 空间复杂度主要为采样输出与基矩阵，约 `O(m * dim)` 或 `O(m * n_ctrl)`。

在图形学常见低次数（2~5）条件下，de Boor 方案非常实用。

## R10

正确性直觉：

1. de Boor 与基函数加权本质上是同一 B 样条定义的两种计算路径，应数值一致；
2. 开放均匀结点会让端点结点重数达到 `p+1`，因此曲线应插值首末控制点；
3. 基函数是局部支撑，任一点只由少量邻近控制点影响；
4. 基函数和为 1，保证曲线是控制点的仿射组合。

## R11

数值稳定性策略：

- 主求值采用 de Boor 递推，避免直接高次多项式展开；
- 通过第二实现（Cox-de Boor 基函数）做交叉校验；
- 对分母为 0 的重复结点情形进行显式保护；
- 断言阈值控制在 `1e-10 ~ 1e-12`，兼顾精度与浮点误差。

## R12

关键参数：

- `control_points`：控制点数组，形状 `(n_ctrl, dim)`；
- `degree`：样条次数 `p`；
- `knots`：结点向量，长度 `n_ctrl + p + 1`；
- `u_values`：采样参数数组（本实现用 `[0,1]`）。

调参建议：

- 提高 `degree` 可增大平滑性，但会扩大单点影响范围；
- 增大 `u_values` 采样密度可更准确检查几何性质；
- 通过改变结点分布（非均匀）可构造更灵活的局部参数化效果。

## R14

常见失效模式：

1. `n_ctrl <= degree` 导致结点向量与样条定义不合法；
2. 输入参数超出结点定义域，可能触发外推语义；
3. 重复结点导致分母为 0，若未防护会产生 NaN；
4. 递归基函数在大规模采样时性能较差。

## R18

`demo.py` 的源码级算法流程（8 步）：

1. `_as_control_points` 校验控制点输入形状，统一为浮点矩阵。  
2. `open_uniform_knot_vector` 根据 `n_ctrl` 和 `degree` 构造首尾重复 `p+1` 的结点向量。  
3. `find_knot_span` 用二分搜索定位参数 `u` 所在结点区间，为局部递推准备索引。  
4. `de_boor_point` 取 `p+1` 个局部控制点，按 de Boor 递推逐层线性插值得到 `C(u)`。  
5. `evaluate_de_boor` 对全部 `u_values` 调用第 4 步，得到整条曲线采样结果。  
6. `cox_de_boor_basis_matrix` 按 Cox-de Boor 递推显式生成基函数矩阵，再由 `evaluate_by_basis` 做 `basis @ control_points`。  
7. `main` 计算两条结果差异，并检查基函数和为 1、局部支撑、端点插值与长度下界。  
8. 若 `scipy.interpolate.BSpline` 可用，再做第三方对照验证；全部断言通过后输出 `All checks passed.`。  

其中第三方库仅用于“结果交叉验证”，核心 B 样条求值流程（区间定位、基函数递推、de Boor 递推）均在源码中显式展开，不是黑盒调用。
