# 最小生成树 - Prim

- UID: `MATH-0217`
- 学科: `数学`
- 分类: `图论`
- 源序号: `217`
- 目标目录: `Algorithms/数学-图论-0217-最小生成树_-_Prim`

## R01

问题定义：给定一个**连通无向带权图** `G=(V,E)`，在所有能连接全部顶点且无环的子图中，找到总权重最小的一棵树，即最小生成树（MST）。

Prim 算法从某个起点出发，逐步把一个新顶点接入当前生成树，每一步都选“跨越当前割的最小边”。

## R02

输入（MVP 约定）：
- `num_nodes`：顶点数，顶点编号为 `0..num_nodes-1`；
- `edges`：无向边列表，元素格式 `(u, v, w)`；
- `start`：Prim 起点。

输出：
- `mst_edges`：MST 中的 `n-1` 条边；
- `total_weight`：MST 总权重。

异常：若图不连通，抛 `ValueError`（全图 MST 不存在）。

## R03

Prim 的核心贪心思想：
1. 维护一个已选顶点集合 `S`（初始只含起点）；
2. 在所有一端在 `S`、另一端不在 `S` 的边中，选择权重最小者；
3. 将该边和新顶点加入生成树；
4. 重复直到覆盖所有顶点。

## R04

正确性依据是割性质（Cut Property）：
- 对任意割 `(S, V-S)`，跨越该割的最小权边一定属于某棵 MST；
- Prim 每一步都在当前割上取最小边，因此不会破坏最优性；
- 连续执行 `n-1` 次后得到的树即为某棵 MST。

## R05

本实现使用的数据结构：
- 邻接表：`List[List[Tuple[int, float]]]`，用于存图；
- 最小堆：`heapq`，维护候选横切边；
- `visited` 布尔数组，标记已进入树的顶点；
- `mst_edges` 列表，保存结果边。

额外提供 `UnionFind + kruskal_mst` 仅用于结果交叉校验。

## R06

算法步骤（堆优化 Prim）：
1. 构建无向邻接表；
2. 将 `(0.0, start, -1)` 放入最小堆；
3. 循环弹出最小候选边；
4. 若目标点已访问则跳过；
5. 否则把该点并入生成树，若不是起点则记录这条边；
6. 把该点发出的可扩展边压入堆；
7. 当访问完全部顶点时结束；
8. 若最终访问数小于 `n`，说明图不连通，抛异常。

## R07

复杂度分析：
- 建图：`O(E)`；
- 每条边最多入堆一次数量级（无向图视实现约 `2E`），堆操作 `log V`；
- 总时间复杂度：`O(E log V)`；
- 空间复杂度：`O(V + E)`。

## R08

适用前提与限制：
- 图必须是无向图；
- 允许负权边（MST 问题不受此限制）；
- 若图不连通，本实现不返回森林，而是显式报错；
- 允许存在多条平行边，算法会自然选择更优边。

## R09

与常见方法对比：
- Kruskal：全局按边排序，适合边集驱动实现；
- Prim：从“点扩张”视角出发，配合邻接表+堆在稀疏图上高效；
- Boruvka：并行友好，工程中常与其他算法混合。

本条目主角是 Prim，但在 `demo.py` 中用 Kruskal 对总权重做交叉验证，提升 MVP 可核验性。

## R10

工程实现要点：
- 明确无向边需双向入邻接表；
- 使用 `visited` 防止成环；
- 起点入堆用“虚拟父边”`parent=-1`，避免特判复杂化；
- 使用断言验证 `|mst_edges| = n-1` 与已知答案权重。

## R11

`demo.py` 模块划分：
- `build_adjacency`：构建无向图邻接表；
- `prim_mst`：Prim 主算法；
- `UnionFind` + `kruskal_mst`：参考实现，用于交叉校验；
- `normalize_edges`：规范化无向边打印顺序；
- `main`：运行样例、打印结果、执行断言与异常分支测试。

## R12

最小可运行 MVP 特征：
- 仅依赖 Python 标准库（`heapq`, `dataclasses`, `typing`）；
- 无交互输入，直接 `python3 demo.py` 可运行；
- 内置固定样例图，输出确定且可复现；
- 包含“正确样例 + 异常分支”两个维度的自动校验。

## R13

内置样例图（6 个点、10 条边）中，MST 的一个可行解总权重为 `11.0`。

`demo.py` 会输出 Prim 选中的边，并校验：
- Prim 总权重等于 `11.0`；
- Prim 与 Kruskal 计算出的 MST 总权重一致。

## R14

正确性直觉（实现层）：
- 不变量 1：`visited` 中顶点始终连通；
- 不变量 2：`mst_edges` 始终无环（每次只把新点接入）；
- 不变量 3：每次新增边都是当前割最小边；
- 终止时满足“连通 + 无环 + `n-1` 条边”，因此是生成树；
- 由割性质可知该生成树权重最小。

## R15

边界与异常处理：
- `num_nodes == 0`：返回空边集和 `0.0`；
- 起点越界：抛 `ValueError`；
- 边端点越界：抛 `ValueError`；
- 图不连通：抛 `ValueError`；
- 代码中显式覆盖了不连通分支的测试。

## R16

建议测试集合：

2. 存在等权边（验证并列候选时仍能得到最优权重）；
3. 含负权边（验证算法适用性）；

5. `n=0`、`n=1` 退化输入。

## R18

源码级算法流程（`demo.py`，8 步）：
1. `main` 定义固定无向带权图 `edges`，并设置 `num_nodes=6`。  
2. 调用 `prim_mst`，其内部先用 `build_adjacency` 把每条无向边双向写入邻接表。  
3. `prim_mst` 初始化 `heap=[(0.0, start, -1)]`、`visited`、`mst_edges`、`total_weight`。  
4. 循环弹出堆顶最小候选边；若目标顶点已访问则跳过，否则将该顶点并入生成树，并记录真实父边。  
5. 将新并入顶点的所有外连边压入最小堆，继续扩展当前割，直到所有顶点都被访问。  
6. 若循环结束后访问顶点数不足 `num_nodes`，`prim_mst` 抛出“图不连通”异常；否则返回 `mst_edges` 与 `total_weight`。  
7. `main` 再调用 `kruskal_mst`（基于 `UnionFind`）求参考 MST，总权重用于与 Prim 做一致性断言。  
8. `main` 打印结果，执行 `11.0` 目标权重断言、Prim/Kruskal 一致性断言，以及不连通图应抛异常的分支断言，形成完整可验证闭环。
