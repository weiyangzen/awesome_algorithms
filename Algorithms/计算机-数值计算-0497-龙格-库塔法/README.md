## R01

龙格-库塔法（Runge-Kutta Methods）是一类常微分方程（ODE）数值积分方法。  
本条目实现经典四阶显式 RK4，用于求解初值问题：

`y'(t) = f(t, y), y(t0) = y0`。

`demo.py` 提供最小可运行 MVP，包含：
- 手写 RK4 单步更新（非黑盒）；
- 标量 ODE 的收敛阶验证（理论四阶）；
- 向量 ODE（简谐振子）示例与误差/能量漂移检查。

## R02

问题定义（本目录实现）：
- 输入：
  - 右端函数 `f(t, y)`；
  - 起点 `t0`、终点 `t_end`；
  - 初值 `y0`（标量或一维向量）；
  - 固定步数 `n_steps`。
- 输出：
  - 时间网格 `t`；
  - 数值解轨迹 `y`，形状 `(n_steps + 1, dim)`。

约束：
- 本 MVP 仅支持 `t_end > t0` 的正向积分；
- 使用固定步长，不做自适应步长控制；
- 全流程无交互输入，直接运行可复现输出。

## R03

数学基础（经典 RK4）：

记步长 `h`，在时刻 `t_n`、状态 `y_n` 下：
- `k1 = f(t_n, y_n)`
- `k2 = f(t_n + h/2, y_n + h*k1/2)`
- `k3 = f(t_n + h/2, y_n + h*k2/2)`
- `k4 = f(t_n + h, y_n + h*k3)`
- `y_(n+1) = y_n + (h/6) * (k1 + 2*k2 + 2*k3 + k4)`

性质：
- 局部截断误差 `O(h^5)`；
- 全局误差 `O(h^4)`（在足够光滑条件下）。

## R04

算法流程（高层）：
1. 校验输入：时间、步数、初值维度与有限性。  
2. 计算固定步长 `h = (t_end - t0) / n_steps`。  
3. 初始化 `t` 网格与 `y` 结果数组。  
4. 对每个时间步调用 `rk4_step` 计算下一状态。  
5. 将状态写入轨迹数组。  
6. 结束后返回 `IntegrationResult(t, y)`。  
7. 在示例中额外计算解析解误差与观测阶。  
8. 打印关键指标用于人工审计。

## R05

核心数据结构：
- `IntegrationResult`（`dataclass`）：
  - `t: np.ndarray`，长度 `n_steps + 1`；
  - `y: np.ndarray`，形状 `(n_steps + 1, dim)`。
- `State = np.ndarray`：
  - 状态向量统一为一维数组；
  - 标量初值会被提升为长度 1 向量。
- `step_list: list[int]`：
  - 用于标量案例的收敛阶评估（如 10/20/40/80）。

## R06

正确性要点：
- 更新式正确：`rk4_step` 严格对应 RK4 四斜率加权公式。  
- 维度安全：`_eval_rhs` 强制 RHS 输出形状与状态一致。  
- 数值安全：输入与 RHS 结果均检查 `nan/inf`。  
- 验证机制：
  - 标量方程对照解析解，输出终点误差；
  - 通过 `log2(e(h)/e(h/2))` 估计收敛阶，接近 4 则符合 RK4 理论。  
- 向量方程额外观察守恒量漂移（简谐振子能量）。

## R07

复杂度分析（维度 `d`，步数 `N=n_steps`）：
- 单步 RK4 需 4 次 RHS 评估；
- 若 RHS 单次开销记为 `C_f(d)`，则：
  - 时间复杂度 `O(4N*C_f(d)) = O(N*C_f(d))`；
  - 对常见稠密向量 RHS，可近似 `O(Nd)`。
- 空间复杂度：
  - 存全轨迹为 `O(Nd)`；
  - 若只保留当前步可降到 `O(d)`（本实现选择可审计的全轨迹）。

## R08

边界与异常处理：
- `n_steps <= 0`：抛 `ValueError`。  
- `t_end <= t0`：抛 `ValueError`。  
- 初值不是标量/一维数组：抛 `ValueError`。  
- 初值含非有限值：抛 `ValueError`。  
- RHS 返回形状不匹配或含非有限值：抛 `ValueError`。  
- 计算步长后非有限或非正：抛 `ValueError`。

## R09

MVP 取舍说明：
- 只实现经典显式 RK4，不扩展到 RK2/RKF45。  
- 不实现事件检测、刚性方程隐式法、误差自适应控制。  
- 重点放在：
  - 源码可读；
  - 可验证（解析解 + 观测阶）；
  - 可复用（标量/向量统一接口）。  
- 这是一个“诚实且可运行”的最小教学/验证版本。

## R10

`demo.py` 主要函数职责：
- `_ensure_state_vector`：统一初值形状并做有限性检查。  
- `_eval_rhs`：统一 RHS 输出并做形状/有限性校验。  
- `rk4_step`：执行一次 RK4 更新。  
- `rk4_integrate`：固定步长积分主循环。  
- `scalar_rhs` / `scalar_exact_solution`：标量测试方程与解析解。  
- `oscillator_rhs`：二维简谐振子 RHS。  
- `observed_orders`：由误差比估算观测收敛阶。  
- `run_scalar_convergence_demo`：标量收敛阶实验。  
- `run_vector_demo`：向量系统误差与能量漂移实验。  
- `main`：组织并打印全部实验结果。

## R12

输出字段说明：
- 标量实验表：
  - `steps`：步数 `N`；
  - `h`：步长；
  - `|error(T)|`：终点绝对误差。  
- `Observed order p`：
  - 用相邻步数误差比估计 `p`，应接近 `4`。  
- 向量实验：
  - `max state error`：整个时间段状态误差最大值；
  - `max energy drift`：能量守恒误差最大值；
  - `final state (num)`：终点状态与理论值对照。

## R13

建议最小测试集（本脚本已覆盖）：
- 标量线性非齐次方程：`y' = y - t^2 + 1`。  
  - 有解析解，可做绝对误差和阶数检查。  
- 二维简谐振子：`x' = v, v' = -x`。  
  - 可检查向量状态处理和守恒量漂移。  

建议补充异常测试：
- `n_steps = 0`（应报错）；
- `t_end <= t0`（应报错）；
- RHS 返回错误维度（应报错）。

## R14

可调参数与调参建议：
- `n_steps`：最关键参数，越大误差越小但耗时越高。  
- `t_end - t0`：区间越长，累计误差越明显。  
- `step_list`（标量实验）：
  - 用倍增步数验证阶数最直观。  

经验建议：
- 若观测阶偏离 4，先检查 RHS 是否足够光滑和误差是否已进入舍入误差区间；
- 若能量漂移过大，通常可通过增大 `n_steps` 改善。

## R15

方法对比（简述）：
- 对比欧拉法（Euler）：
  - 欧拉是一阶，全局误差 `O(h)`；
  - RK4 四阶，在同等精度下可用更大步长。  
- 对比中点法/Heun（二阶）：
  - RK4 每步计算更多（4 次 RHS），但高精度更有优势。  
- 对比自适应 `RK45`：
  - `RK45` 自动调步、工程实用性更强；
  - 本实现固定步长，更适合教学和算法透明审计。

## R18

`demo.py` 源码级算法流程（8 步）：
1. `main` 先设置打印选项，再依次调用 `run_scalar_convergence_demo` 与 `run_vector_demo`。  
2. 标量演示中构造固定步数列表（10/20/40/80），循环调用 `rk4_integrate`。  
3. `rk4_integrate` 先执行 `_ensure_state_vector`，把标量/向量初值统一成一维数组。  
4. `rk4_integrate` 计算固定步长 `h`，创建 `t_grid` 和 `y_grid`，写入初值。  
5. 每一步调用 `rk4_step`：内部按 `k1/k2/k3/k4` 四次 RHS 评估组合出 `y_(n+1)`。  
6. `_eval_rhs` 在每次 RHS 调用后验证输出维度和有限性，防止静默数值污染。  
7. 标量演示完成后，用解析解计算 `|error(T)|`，再由 `observed_orders` 计算观测阶 `p=log2(e_h/e_{h/2})`。  
8. 向量演示使用简谐振子轨迹，计算 `max state error` 与 `max energy drift`，最终打印结果完成可审计闭环。
