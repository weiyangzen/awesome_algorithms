# Reeb图计算

- UID: `MATH-0254`
- 学科: `数学`
- 分类: `计算拓扑`
- 源序号: `254`
- 目标目录: `Algorithms/数学-计算拓扑-0254-Reeb图计算`

## R01

本条目实现一个**离散 Reeb 图计算 MVP**：
- 输入是二维标量场（规则网格）；
- 输出是由关键事件点构成的拓扑骨架图；
- 事件点包括 `minimum / saddle / maximum`；
- 采用“下水平集扫描 + 并查集”显式实现，不调用 TDA 黑盒。

该 MVP 的目标是把 “Reeb 图” 的核心机制跑通并可检查，而不是追求完整工业级泛化。

## R02

问题定义（MVP 范围）：
- 输入：`f: V -> R`，其中 `V` 是二维网格顶点集合（4 邻接）。
- 输出：有向图 `G=(N,E)`，其中：
  - `N`：关键拓扑事件节点；
  - `E`：按函数值从低到高连接事件节点的边。

在本实现中，`G` 是**Reeb 图的离散近似（merge-tree 风格骨架）**，特别适用于单连通规则网格上的教学与原型验证。

## R03

数学背景（简化版）：
- 连续定义里，Reeb 图可视作把同一等值集连通分量“收缩”后的商空间图；
- 离散实现常借助过滤（filtration）思想：按函数值递增激活顶点，追踪下水平集连通分量变化；
- 当新连通分量诞生时对应 `minimum`，多个分量合并时对应 `saddle`，扫描结束时分量封顶对应 `maximum`。

这与 0 维持久同调中的并查集事件追踪是同一技术族。

## R04

算法主流程：
1. 校验标量场（二维、非空、有限值）。
2. 将网格展平，按 `(value, vertex_id)` 排序形成扫描序列（解决并列值）。
3. 逐点激活顶点并查看“已激活邻居所属连通分量集合”。
4. 若无已激活邻居：创建 `minimum` 节点。
5. 若仅 1 个连通分量：普通点，分量继续，不建事件节点。
6. 若有 >=2 个连通分量：创建 `saddle` 节点，并从各分量前沿节点连边到该鞍点。
7. 用并查集合并相关分量，并更新“该分量当前前沿事件节点”。
8. 扫描结束后，为每个最终分量创建 `maximum` 节点并闭合边。

## R05

核心数据结构：
- `UnionFind`：
  - `parent/rank` 维护连通分量；
  - `peak` 记录分量内当前最高值顶点。
- `component_frontier: Dict[root, node_id]`：每个活动分量最近一次关键事件节点。
- `ReebNode`：记录节点编号、类型、网格坐标、函数值。
- `edge_set`：存储事件图边（去重）。
- `ReebGraphResult`：统一封装节点、边及计数统计。

## R06

正确性直觉（MVP 层面）：
- 扫描顺序保证“拓扑事件只会从低值向高值发生”；
- 并查集保证下水平集连通分量追踪正确；
- `minimum` 对应分量出生，`saddle` 对应分量合并，`maximum` 对应分量终结；
- `component_frontier` 把“分量演化链”转写为图边，因此最终图反映关键拓扑变化。

## R07

复杂度分析：
- 设顶点数 `N=H*W`，网格边数 `M`，4 邻域下 `M=O(N)`。
- 排序复杂度：`O(N log N)`。
- 并查集操作近似：`O((N+M) * alpha(N))`。
- 总时间复杂度：`O(N log N)`（排序主导）。
- 空间复杂度：`O(N)`（并查集数组、活动标记、节点边结构）。

## R08

边界与异常处理：
- 非二维输入、空数组、含 `NaN/inf`：抛出 `ValueError`；
- 并列值（平台）通过 `(value, vertex_id)` 稳定排序打破并列，保证可复现；
- 输出边做单调性校验：若出现从高到低的边则触发断言。

## R09

MVP 取舍说明：
- 仅依赖 `numpy + 标准库`，方便在最小环境运行；
- 不引入 `gudhi / giotto-tda / networkx` 等现成封装，避免黑盒；
- 当前实现聚焦下水平集事件图（merge-tree 风格），未实现完整 contour tree 合并流程；
- 适合作为 Reeb 图计算入门骨架与后续扩展基础。

## R10

`demo.py` 模块职责：
- `validate_scalar_field`：输入校验；
- `grid_neighbors`：规则网格 4 邻接；
- `UnionFind`：连通分量维护与峰值追踪；
- `compute_reeb_graph`：核心事件扫描与图构建；
- `build_demo_field`：提供固定示例标量场；
- `validate_graph_monotonicity`：结果健全性检查；
- `print_result`：打印节点、边、统计；
- `main`：无交互入口。

## R12

输出解读：
- `Critical nodes`：每个关键点的 `id / kind / value / (row,col)`；
- `Graph edges`：关键事件图中的有向边 `source -> target`；
- `Summary`：网格尺寸、节点边数量、三类关键点计数；
- `All checks passed.`：表示基础一致性校验通过。

## R13

最小测试建议：

## R15

与相关方法对比：
- 对比完整 Reeb/Contour Tree 工程实现：
  - 本实现更小、更透明；
  - 但功能上更偏教学型骨架。
- 对比持久同调库直接调用：
  - 黑盒更快拿结果；
  - 本实现更容易逐行审计算法逻辑。
- 对比纯图最短路或聚类：
  - Reeb 图关注的是“等值集连通结构变化”，语义不同。

## R18

`demo.py` 源码级算法流（9 步）：
1. `main()` 构造固定 6x6 标量场并调用 `compute_reeb_graph`。
2. `compute_reeb_graph` 先用 `validate_scalar_field` 校验输入，再展平为一维值数组。
3. 按 `(value, vertex_id)` 对所有顶点排序，得到确定性的下水平集扫描顺序。
4. 每激活一个顶点，`grid_neighbors` 收集已激活邻居并通过 `UnionFind.find` 映射到分量根。
5. 若邻居分量数为 0：`NodeBuilder.add("minimum")` 产生极小值节点，并把该节点登记为分量前沿。
6. 若邻居分量数为 1：仅执行并查集合并，前沿节点沿用，不创建关键事件。
7. 若邻居分量数 >= 2：创建 `saddle`，把各分量前沿连边到该鞍点，再合并分量并更新前沿为该鞍点。
8. 扫描完成后，对每个最终分量读取 `UnionFind.peak` 顶点，创建 `maximum` 并从分量前沿连边闭合。
9. `validate_graph_monotonicity` 校验边方向单调，`print_result` 打印关键点、边和统计信息。
