# 矩阵快速幂优化DP

- UID: `MATH-0547`
- 学科: `数学`
- 分类: `动态规划`
- 源序号: `547`
- 目标目录: `Algorithms/数学-动态规划-0547-矩阵快速幂优化DP`

## R01

本条目聚焦的核心问题是：
- 对满足线性递推的动态规划（DP），如何把 `O(n)` 的逐步转移，优化为 `O(log n)`；
- 典型形式：
  \[
  f(n)=c_1f(n-1)+c_2f(n-2)+\cdots+c_kf(n-k)
  \]
- 当 `n` 很大（如 `10^12`、`10^18`）时，普通 DP 无法在可接受时间内完成。

## R02

优化思路是把“递推”改写成“矩阵乘法”：
- 定义状态向量 `S(n)=[f(n), f(n-1), ..., f(n-k+1)]^T`；
- 构造转移矩阵 `T` 使得 `S(n)=T·S(n-1)`；
- 则 `S(n)=T^(n-n0)·S(n0)`；
- 关键难点变为计算 `T^p`，可用二进制快速幂将复杂度从线性降为对数。

## R03

本目录 MVP 采用三阶递推示例（Tribonacci）：
\[
f(n)=f(n-1)+f(n-2)+f(n-3),\quad f(0)=0,f(1)=1,f(2)=1
\]

输入：
- `n`：目标下标（非负整数）；
- `mod`：模数（正整数，默认 `1_000_000_007`）。

输出：
- `f(n) mod mod`；
- 与朴素 DP 的一致性校验结果；
- 简单运行耗时对比。

## R04

状态设计：
\[
S(n)=\begin{bmatrix}f(n)\\f(n-1)\\f(n-2)\end{bmatrix}
\]

为了从 `S(n-1)` 得到 `S(n)`：
- 第一行实现 `f(n)=f(n-1)+f(n-2)+f(n-3)`；
- 第二、三行实现“状态下移”。

因此这是一个标准的“线性 DP -> 线性代数”建模。

## R05

对三阶递推，转移矩阵是：
\[
T=
\begin{bmatrix}
1 & 1 & 1 \\
1 & 0 & 0 \\
0 & 1 & 0
\end{bmatrix}
\]

并有：
\[
S(n)=T\,S(n-1),\quad
S(2)=\begin{bmatrix}1\\1\\0\end{bmatrix}
\]

所以：
\[
S(n)=T^{n-2}S(2)\quad(n\ge 2)
\]
最终答案是 `S(n)` 第一维。

## R06

伪代码（矩阵快速幂优化 DP）：

```text
function tribonacci_matrix_mod(n, mod):
    if n == 0: return 0
    if n == 1 or n == 2: return 1

    T <- [[1,1,1],[1,0,0],[0,1,0]]
    base <- [1,1,0]^T
    P <- fast_pow(T, n-2, mod)
    Sn <- P * base (mod mod)
    return Sn[0]

function fast_pow(M, e, mod):
    R <- I
    while e > 0:
        if e is odd: R <- R * M (mod mod)
        M <- M * M (mod mod)
        e <- e // 2
    return R
```

## R07

正确性要点：
1. 由状态定义和递推式可直接验证 `S(n)=T·S(n-1)`；
2. 递推展开得 `S(n)=T^(n-2)·S(2)`；
3. 二进制快速幂与普通幂等价，只是按二进制位分解指数：
   - 若 `e = Σ b_i 2^i`，则 `T^e = Π (T^(2^i))^{b_i}`；
4. 因此输出的第一维即 `f(n)`，算法正确。

## R08

复杂度对比（`k=3` 固定）：
- 朴素 DP：
  - 时间 `O(n)`；
  - 空间 `O(1)`（滚动变量）。
- 矩阵快速幂：
  - 时间 `O(log n)`（每次常数维矩阵乘法）；
  - 空间 `O(1)`（固定维矩阵）。

对超大 `n`（如 `10^18`）差异非常明显：线性算法不可用，而对数算法可行。

## R09

数值与工程注意点：
- 使用模运算可避免整数无限增长；
- `numpy.int64` 在本例 `3x3` 且每步都取模时足够稳定；
- 若改成更高维或更大系数，需要评估乘加溢出风险（必要时改 Python 大整数或分块乘法）。

## R10

边界条件：
- `n=0` 返回 `0`；
- `n=1,2` 返回 `1`；
- `n<0` 或 `mod<=0` 应报错；
- 非整数输入应报错（防止隐式截断造成错误结果）。

## R11

常见错误：
- 状态向量顺序写反，导致转移矩阵第一行系数错位；
- 快速幂漏写“奇数位乘 result”；
- 忘记初始化单位矩阵，导致所有结果偏差；
- 只比较少量小样本，未覆盖大指数与边界值。

## R18

本 MVP 的源码级算法流可拆成 6 步：
1. 在 `tribonacci_matrix_mod` 中处理边界输入，直接返回 `n<=2` 的基值。  
2. 构造固定 `3x3` 转移矩阵 `T` 与初始状态 `S(2)`。  
3. 调用 `mat_pow_mod(T, n-2, mod)`，按指数二进制位做快速幂：遇到奇位就累乘 `result`。  
4. 在每次矩阵乘法 `matmul_mod` 中执行 `A @ B`，随后立即 `% mod`，保持数值有界。  
5. 得到 `P=T^(n-2)` 后计算 `S(n)=P·S(2)`，取第一维作为 `f(n)`。  
6. 用 `verify_consistency` 把矩阵法与朴素 `dp_tribonacci_mod` 对拍，确保实现与递推定义一致。

这里没有把第三方库当黑箱：
- 只把 `numpy` 用作基础矩阵乘法算子；
- 快速幂与状态转移逻辑由 `demo.py` 显式实现并可逐行审计。
