## R01

Stirling 数通常指两族三角数表，分别编码“排列的循环结构”和“集合划分结构”。

- 第一类 Stirling 数:
  - 有符号记作 `s(n, k)`，满足
    `x^{\underline{n}} = \sum_{k=0}^{n} s(n, k) x^k`。
  - 无符号记作 `c(n, k) = |s(n, k)|`，表示 `n` 元排列恰有 `k` 个循环的个数。
- 第二类 Stirling 数:
  - 记作 `S(n, k)`，表示把 `n` 个有标号元素划分为 `k` 个非空无序子集的方案数。

本条目 MVP 计算 `s(n,k)`、`c(n,k)`、`S(n,k)`，并给出可验证的组合恒等式。

## R02

- 命名来源: James Stirling（18 世纪），后世将相关系数体系以其姓氏命名。
- 发展脉络:
  - 第一类与排列循环分解、下降阶乘展开紧密相关。
  - 第二类与集合划分、Bell 数和指数型生成函数相关。
- 现代位置: 在组合计数、概率论矩展开、算法分析、离散结构课程中是基础工具。

## R03

很多离散对象可以从两套“基”观察。

- 代数基变换:
  - 普通幂 `x^n` 与下降阶乘 `x^{\underline{n}}` 之间要互相展开。
  - 两类 Stirling 数就是这两种基之间的变换系数。
- 组合对象计数:
  - “排列按循环数分层”对应第一类。
  - “集合按分块数分层”对应第二类。

因此 Stirling 数既是计数对象，又是代数变换矩阵元素，具有双重解释价值。

## R04

对 `0..N` 的完整三角表做动态规划：

- 时间复杂度: `O(N^2)`
  - 需要填充约 `N(N+1)/2` 个有效单元。
  - 每个单元是 `O(1)` 的整数加减乘。
- 若只算单点 `(n,k)` 且用记忆化，最坏仍是 `O(nk)`。

## R05

- 完整表空间: `O(N^2)`（本目录 `demo.py` 采用此策略，便于后续校验与展示）。
- 滚动数组优化: 对单族单行递推可降为 `O(N)` 额外空间。
- 本题选择完整表的原因:
  - 要同时做多种恒等式验证。
  - 要打印多行结果并交叉检查第一类/第二类关系。

## R06

以 `n=5` 为例：

- 第二类 `S(5,k)`:
  - `S(5,1)=1, S(5,2)=15, S(5,3)=25, S(5,4)=10, S(5,5)=1`
- 第一类无符号 `c(5,k)`:
  - `c(5,1)=24, c(5,2)=50, c(5,3)=35, c(5,4)=10, c(5,5)=1`
- 第一类有符号 `s(5,k)`:
  - `s(5,1)=24, s(5,2)=-50, s(5,3)=35, s(5,4)=-10, s(5,5)=1`

递推示例（第二类）:
`S(5,2)=S(4,1)+2*S(4,2)=1+2*7=15`。

## R07

Stirling 数的重要性体现在三点。

- 结构分解能力:
  - 把“按元素数 n”问题分解到“按块数/循环数 k”维度。
- 代数连接能力:
  - 在幂基与下降阶乘基之间提供可计算桥梁。
- 工程可复用性:
  - Bell 数、Touchard 多项式、泊松矩、分区型 DP 都会直接使用。

## R08

最小理论依赖。

- 组合计数基本概念:
  - 排列、循环分解、集合划分。
- 下降阶乘定义:
  - `x^{\underline{n}} = x(x-1)...(x-n+1)`。
- 边界条件:
  - `s(0,0)=c(0,0)=S(0,0)=1`。
  - 对 `n>0`，`s(n,0)=c(n,0)=S(n,0)=0`。

## R09

适用条件与边界。

- 输入域: 非负整数 `n, k`，且通常 `0 <= k <= n`。
- 越界行为:
  - 当 `k>n` 时值为 `0`。
- 数值规模:
  - Stirling 数增长很快，小型整数类型会溢出。
  - Python `int` 任意精度，可正确承载但速度会变慢。

## R10

正确性可由递推构造归纳证明。

- 第一类无符号:
  - 递推 `c(n,k)=c(n-1,k-1)+(n-1)c(n-1,k)`。
  - 解释: 新元素 `n` 要么单独成长度 1 循环（贡献前项），要么插入已有排列的循环位置（贡献后项）。
- 第一类有符号:
  - 递推 `s(n,k)=s(n-1,k-1)-(n-1)s(n-1,k)`。
  - 符号由代数展开中奇偶置换效应决定。
- 第二类:
  - 递推 `S(n,k)=S(n-1,k-1)+kS(n-1,k)`。
  - 解释: 新元素要么单独成新块，要么加入 `k` 个已有块之一。
- 归纳基 `n=0` 与递推封闭性共同保证整张表正确。

  - 幂基/下降阶乘基变换。

## R18

源码级算法流（`demo.py`，未把第三方库当黑箱，核心逻辑完全自实现）。

1. 读取 `n_max`，创建 `(n_max+1) x (n_max+1)` 的零表。
2. 设置共同基值 `table[0][0] = 1`。
3. 对每个 `n=1..n_max`，只遍历 `k=1..n` 的三角区。
4. 按目标族别写入递推值:
   - 第一类无符号: `c[n][k]=c[n-1][k-1]+(n-1)c[n-1][k]`
   - 第一类有符号: `s[n][k]=s[n-1][k-1]-(n-1)s[n-1][k]`
   - 第二类: `S[n][k]=S[n-1][k-1]+kS[n-1][k]`
5. 构表后执行三组校验:
   - `c(n,k)=|s(n,k)|`
   - 第二类与容斥闭式一致
   - 基变换恒等式与 Bell 数行和正确
6. 打印有限结果窗口（而非整张大表）并结束，保证脚本可直接自动运行。
