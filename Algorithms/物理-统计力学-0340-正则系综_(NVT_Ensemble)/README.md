## R01

正则系综（NVT Ensemble）描述与热库接触、粒子数 `N` 和体积 `V` 固定、温度 `T` 固定的平衡系统。  
微观态 `x` 的概率密度满足玻尔兹曼权重：

`P(x) = exp(-beta * H(x)) / Z`

其中 `beta = 1/(k_B T)`，`H(x)` 是哈密顿量，`Z` 是配分函数。

## R02

本任务的最小目标是做一个可运行、可验证的 NVT 采样器，而不是完整分子动力学平台。  
`demo.py` 选择“1D 经典谐振子”的正则分布作为基准，用 Metropolis 采样验证：

- `q^2` 与 `p^2` 的均值是否符合解析解；
- 平均能量 `E` 是否满足均分定理（`<E>=T`，取 `k_B=1`）；
- 由能量涨落给出的 `C_V` 是否接近理论值 1。

## R03

模型定义（单粒子一维谐振子，采样空间为 `(q,p)`）：

- 哈密顿量：`H(q,p) = p^2/(2m) + (k q^2)/2`
- 正则分布：`P(q,p) ∝ exp(-beta H(q,p))`

`demo.py` 用多个并行 walker 组成“采样系综”，在每个温度点独立运行，得到统计量并与理论值对照。

## R04

该模型的解析结果（`k_B=1`）：

1. 配分函数：
   `Z = ∫dq∫dp exp(-beta H) = (2*pi/beta) * sqrt(m/k)`
2. 二阶矩：
   - `<q^2> = T / k`
   - `<p^2> = m * T`
3. 平均能量（两项二次自由度）：
   - `<E> = T`
4. 定容热容（涨落公式）：
   - `C_V = Var(E) / T^2 = 1`

这四个量直接用于代码中的正确性检查。

## R05

采样算法：Metropolis-Hastings（对称高斯提案）。

每次迭代对每个 walker 执行：
1. 提议：
   - `q' = q + Normal(0, sigma_q)`
   - `p' = p + Normal(0, sigma_p)`
2. 计算能量差 `dE = H(q',p') - H(q,p)`。
3. 接受概率：
   - `a = min(1, exp(-beta * dE))`
4. 以概率 `a` 接受新状态，否则保留旧状态。

对称提案 + 接受率定义保证细致平衡，因此平衡分布即目标正则分布。

## R06

统计流程分三段：

1. `burn_in`：丢弃预热样本，减弱初始态偏差；
2. `thin`：每隔若干步记录一次，降低自相关；
3. `sample_steps`：累计样本用于估计 `q^2`、`p^2`、`E`、`C_V`。

每个温度点都会输出：

- 接受率；
- 经验值与理论值；
- 各指标相对误差。

## R07

复杂度分析（设并行 walker 数为 `W`，总迭代数为 `S = burn_in + sample_steps * thin`）：

- 单温度时间复杂度：`O(W * S)`
- 单温度空间复杂度：`O(W * sample_steps)`（存能量样本用于涨落估计）
- 多温度点 `K`：时间复杂度线性扩展到 `O(K * W * S)`

在当前默认参数下，`uv run python demo.py` 可在本地快速完成。

## R10

正确性直觉：

1. 若采样确实收敛到 `exp(-beta H)`，则边缘分布应是高斯；
2. 高斯二阶矩与均分定理给出可检验解析值；
3. 能量涨落公式可额外验证采样是否捕捉到了正确波动尺度；
4. 多温度同时通过，比单温度更能排除“偶然拟合”。

## R11

数值稳定性策略：

- 使用固定随机种子，保证复现；
- 预热 + 抽稀，降低初态与自相关影响；
- 多 walker 并行提升有效样本量；
- 断言阈值设置为“宽松但有约束”（避免把随机波动误判为失败）。

## R12

程序输出包含：

- `temperature`：温度点；
- `acceptance_rate`：平均接受率（通常 `0.2~0.8` 都可工作）；
- `q2_emp`, `q2_theory`；
- `p2_emp`, `p2_theory`；
- `energy_emp`, `energy_theory`；
- `cv_emp`, `cv_theory`；
- `max_relative_error`：该温度下最差相对误差。

若断言通过，说明 NVT 采样与理论预测在允许误差内一致。

## R14

常见失效模式：

1. 预热不足导致偏置（`burn_in` 太短）；
2. 抽稀不足导致相关样本过多（`thin` 太小）；
3. 提案尺度过大或过小导致接受率极端；
4. 误将低样本噪声当作物理偏差；
5. 把此简化模型结论直接外推到强相互作用多体系统。

## R18

`demo.py` 源码级算法流程（8 步）：

1. `NVTConfig` 定义质量、弹簧常数、温度网格、采样步数与随机种子。  
2. `hamiltonian(q, p, m, k)` 显式计算 `p^2/(2m) + kq^2/2`，作为 Metropolis 的能量基准。  
3. `run_metropolis_nvt` 在给定温度下初始化并行 walkers，并计算温度自适应提案尺度 `sigma_q/sigma_p`。  
4. 每次迭代对全部 walkers 生成 `(q', p')` 提议，计算 `dE` 并按 `exp(-beta*dE)` 批量接受/拒绝。  
5. 预热后按 `thin` 抽样，记录每个 walker 的能量快照，并累计 `q^2`、`p^2` 的样本均值。  
6. 用样本均值和能量方差构造经验统计量：`q2_emp`、`p2_emp`、`energy_emp`、`cv_emp`。  
7. 在 `theoretical_observables` 中计算解析目标值（`q2_theory`、`p2_theory`、`energy_theory`、`cv_theory`）。  
8. `main` 汇总成 `pandas.DataFrame`，打印每温度误差，并执行断言保证所有温度点均在容差内。  

第三方库说明：`numpy` 仅用于随机数、数组和基础统计；`pandas` 仅用于展示结果。NVT 算法逻辑（能量函数、接受规则、统计估计）均由源码逐步展开，未依赖黑箱物理库。
