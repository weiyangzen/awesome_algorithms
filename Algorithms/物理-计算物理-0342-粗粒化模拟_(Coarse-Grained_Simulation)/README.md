# 粗粒化模拟 (Coarse-Grained Simulation)

- UID: `PHYS-0335`
- 学科: `物理`
- 分类: `计算物理`
- 源序号: `342`
- 目标目录: `Algorithms/物理-计算物理-0342-粗粒化模拟_(Coarse-Grained_Simulation)`

## R01

粗粒化模拟（Coarse-Grained Simulation, CG）通过“降维建模”把高分辨率原子自由度压缩为更少的有效自由度（beads），以更低计算代价保留主要统计行为。

本条目实现一个最小可运行 MVP：
- 先构造细粒度链状体系；
- 再按分组平均映射到粗粒化珠子；
- 然后在有效势下做随机动力学演化；
- 最后输出结构与动力学统计量并自动验证。

## R02

问题定义（本 MVP 范围）：
- 输入：脚本内配置参数（珠子数、每珠原子数、温度、扩散系数、时间步长、步数、势函数参数等）。
- 输出：时间序列表格（势能、回转半径、端到端距离、平均键长、质心 MSD）、验证指标与 `PASS/FAIL`。
- 目标：展示“映射 -> 有效相互作用 -> 粗粒化动力学 -> 可审计验证”的完整链路。

本实现聚焦教学与可验证最小流程，不追求工业级力场拟合。

## R03

动力学模型采用过阻尼 Langevin 形式：

`r(t+dt) = r(t) + (D/(k_B T)) * F(r) * dt + sqrt(2Ddt) * xi`

其中：
- `r` 为粗粒化坐标；
- `D` 为扩散系数；
- `F = -∇U` 为总有效力；
- `xi` 为标准高斯噪声。

代码中使用无量纲单位并取 `k_B = 1`，因此迁移率 `mobility = D/T`。

## R04

粗粒化映射采用“块平均映射”：把每 `m` 个细粒度原子映射为一个 bead。

若第 `I` 个 bead 对应原子集合 `G_I`，则：

`R_I = (1/|G_I|) * Σ_{i in G_I} r_i`

`demo.py` 中：
- `build_fine_grained_chain` 构造细粒度链；
- `map_atoms_to_beads` 通过 reshape + mean 实现映射。

这样可直观看到从细模型到粗模型的状态转换。

## R05

有效势能由两部分组成：

1. 键相互作用（相邻 bead）：
`U_bond = Σ 0.5 * k_bond * (|r_{i+1}-r_i| - r0)^2`

2. 排斥非键相互作用（WCA，截断 LJ）：
`U_wca = 4*eps*((sigma/r)^12 - (sigma/r)^6) + eps,  r < 2^(1/6)*sigma`
`U_wca = 0,  r >= 2^(1/6)*sigma`

总势能 `U = U_bond + U_wca`，总力为两部分力之和。

## R06

`demo.py` 输入输出约定：
- 输入：无命令行输入，参数由 `CGConfig` 固定给出（可在代码中修改）。
- 输出：
1. 关键参数回显；
2. 抽样轨迹统计表；
3. 验证指标（有限性、平均键长偏差、MSD 增长、位置尺度代理）；
4. `Validation: PASS/FAIL`。

若校验不通过，脚本以非零退出码结束，便于自动化检查。

## R07

算法主流程（高层）：
1. 初始化随机数种子与模型参数。
2. 生成细粒度链坐标。
3. 执行块平均映射得到粗粒化 bead 初态。
4. 在每个时间步计算总力和总势能。
5. 用过阻尼 Langevin 更新 bead 坐标。
6. 采样并记录结构/动力学观测量。
7. 汇总 DataFrame 并执行验证规则。
8. 打印报告并给出 `PASS/FAIL`。

## R08

设 bead 数为 `N`，步数为 `T`：
- 键力计算复杂度：`O(N)`；
- WCA 两两作用复杂度：`O(N^2)`；
- 总时间复杂度：`O(T * N^2)`；
- 空间复杂度：`O(N + N_sample)`（`N_sample` 为采样行数）。

本 MVP 取 `N=12`，即使用朴素 `O(N^2)` 也足够快、实现清晰。

## R09

数值稳定性策略：
- 小时间步 `dt=5e-4`，避免硬弹簧导致积分发散；
- 距离下限保护（`r < 1e-12` 时跳过除零路径）；
- 固定随机种子，保证可复现；
- 验证中检查 `nan/inf` 与尺度上限，及时捕获数值爆炸。

这些措施使脚本在最小实现下保持稳定可运行。

## R10

MVP 技术栈：
- Python 3
- `numpy`：向量与随机数计算
- `pandas`：结果表格组织与打印
- 标准库 `dataclasses`

算法核心（映射、势能、力、时间推进）均在源码中显式实现，没有调用粗粒化黑箱库。

## R12

输出字段说明：
- `step`: 仿真步号
- `time`: 物理时间（`step * dt`）
- `potential`: 总势能 `U_bond + U_wca`
- `rg`: 回转半径（链团尺寸）
- `ree`: 端到端距离
- `mean_bond`: 相邻 bead 平均键长
- `msd_com`: 质心相对初始时刻的均方位移

这些指标覆盖“结构 + 动力学”两类核心信息。

## R13

内置正确性验证规则：
1. 全表数值必须有限（无 `nan/inf`）。
2. `mean_bond` 全程平均值接近目标 `r0`（偏差不超过阈值）。
3. `msd_com` 总增长量为正且超过最小阈值（体现扩散趋势）。
4. `rg/ree` 的尺度代理不得异常爆炸（上限阈值约束）。

四项同时满足输出 `Validation: PASS`。

## R14

当前实现局限：
- 势函数参数为手工设定，未由底层全原子轨迹反演标定。
- 未考虑显式溶剂、长程静电、多体势等复杂效应。
- 未引入周期边界、邻居表、并行加速。
- 主要用于“方法演示 + 最小验证”，不直接对应具体材料体系定量预测。

## R15

可扩展方向：
- 用力匹配（force matching）或相对熵方法拟合有效势参数。
- 引入邻居表和 cell list，把大规模体系复杂度从 `O(N^2)` 降低。
- 增加温度扫描/参数扫描，比较结构因子、径向分布函数等统计量。
- 接入更高阶积分器（如 BAOAB 变体）并做误差-效率对比。

## R16

典型应用场景：
- 聚合物/软物质的大时间尺度构象动力学。
- 生物大分子的低分辨率结构波动研究。
- 全原子模拟前的快速参数探索与预筛选。
- 计算物理教学中“多尺度建模”流程演示。

## R17

方案对比：
- 相比全原子 MD：粗粒化显著降低自由度、可到达更长时空尺度，但细节分辨率下降。
- 相比仅给理论公式：本条目提供从映射到演化再到验证的可执行闭环。
- 相比黑箱仿真软件调用：本实现把关键步骤（力、势、积分、验证）完全展开，可审计和二次开发成本更低。

## R18

`demo.py` 源码级流程拆解（9 步）：
1. `CGConfig` 定义粗粒化模型、动力学参数、采样和验证阈值。
2. `build_fine_grained_chain` 生成细粒度链坐标（带小热扰动）。
3. `map_atoms_to_beads` 按 `atoms_per_bead` 分块平均得到 bead 坐标。
4. `bond_forces_and_energy` 计算相邻 bead 的谐振子键力与键能。
5. `wca_forces_and_energy` 计算非键 WCA 排斥力与能量。
6. `total_forces_and_energy` 汇总总力和总势能。
7. `run_simulation` 在时间循环中执行 Langevin 更新并记录 `potential/rg/ree/mean_bond/msd_com`。
8. `validate` 对采样表执行有限性、键长误差、MSD 增长、尺度上限四项检查。
9. `main` 打印样本表与指标，输出 `Validation: PASS/FAIL` 并在失败时返回非零状态。

上述每一步都在源码中明确可见，第三方库仅用于基础数值与表格，不隐藏算法主干。
