# 图着色 - 回溯

## R01

图着色（Graph Coloring）的经典版本是“顶点着色”问题：

- 给定无向图 `G = (V, E)`。
- 给每个顶点分配一个颜色编号。
- 对任意边 `(u, v) in E`，要求 `color[u] != color[v]`。

“回溯法”是最直观且可精确求解的方法：按顺序给顶点试色，若某步导致冲突则立即撤销并尝试下一个颜色。

## R02

问题形态通常有两类：

- 判定问题：给定 `m` 种颜色，图是否可被 `m`-着色？
- 优化问题：图的最小可用颜色数（色数）是多少？

本条目实现的是判定问题的核心求解器（`m`-着色回溯），并可通过对 `m=1..|V|` 的顺序尝试来近似得到色数。

## R03

回溯搜索树的含义：

- 第 `k` 层表示“第 `k` 个待着色顶点”做出颜色选择。
- 分支数最多为 `m`。
- 若某选择与已着色邻居冲突，则该分支被剪枝。

因此它是“构造 + 约束检查 + 失败回退”的典型 DFS 组合问题。

## R04

时间复杂度（最坏情形）：

- 搜索空间上界约为 `m^|V|`。
- 每次尝试颜色都要检查邻边冲突，单步约 `O(deg(v))`。
- 总体最坏复杂度可写为 `O(m^|V| * (|V| + |E|))` 量级。

实际运行通常显著优于最坏上界，因为大量非法分支会被提前剪枝。

## R05

空间复杂度：

- 颜色数组 `color[0..|V|-1]`：`O(|V|)`。
- 递归调用栈深度最多 `|V|`：`O(|V|)`。
- 图邻接表存储：`O(|V| + |E|)`（输入本身成本）。

除输入图之外，额外工作空间主要是线性级别。

## R06

一个小例子（`C5`，5 个顶点环图）：

- 用 2 色时，沿环交替着色到最后一个顶点会与起点冲突，因此失败。
- 用 3 色时，前 4 个顶点可交替两色，第 5 个顶点可用第 3 色，成功。

回溯过程会在 2 色时遍历完可行分支并确认“不可解”，在 3 色时找到第一组可行赋值后返回。

## R07

算法意义：

- 理论上，它提供了对 NP 完全问题的精确求解基线。
- 工程上，它是分支限界、约束规划（CSP）和 SAT 编码前的常用直解模板。
- 教学上，它是理解“可行性剪枝”最清晰的入门模型之一。

## R08

前置知识：

- 无向图与邻接表表示。
- 深度优先搜索（DFS）与递归栈。
- 回溯三件套：选择、约束检查、撤销。
- 基本复杂度估算。

## R09

适用边界：

- 顶点规模较小或中等，且需要精确解时适用。
- 大规模稠密图上纯回溯可能过慢，需要启发式顶点选择、位运算剪枝或转 SAT/ILP。
- 本实现针对无向简单图；若输入包含自环，则任何 `m` 下都不可着色。

## R10

正确性要点（归纳视角）：

- 不变式：递归深度 `k` 时，前 `k` 个已着色顶点始终满足“相邻异色”。
- 扩展：给新顶点尝试颜色时，仅在满足局部约束时继续递归，因此不变式保持。
- 完备性：每个顶点都枚举 `1..m`，所有合法赋值都会被遍历到。
- 终止：当 `k == |V|` 时得到一个完整合法着色，返回成功。

## R11

鲁棒性与输入约束：

- 需要保证顶点编号有效（`0..n-1`）。
- 自环会立即导致不可解（可预检查）。
- 重边不影响正确性，但会造成重复检查；可在构图阶段去重。
- 若图不连通，算法仍成立，因为约束是局部边约束，递归会自然覆盖所有连通分量。

## R12

性能优化方向（不改变答案正确性）：

- 顶点排序：优先着色高阶顶点（高约束先处理）。
- 颜色顺序：优先尝试更可能成功的颜色（如最少冲突值）。
- 前向检查：提前记录邻点可用色集合，进一步剪枝。
- 位掩码：把颜色可用性检查改为位运算，常数显著下降。

本 MVP 已采用“按度数降序”的简单启发式。

## R13

与贪心着色的关系：

- 贪心着色速度快，但结果依赖顶点顺序，可能非最优。
- 回溯着色可作为“真值判定器”，用于验证某个 `m` 是否足够。
- 实务中常见组合：先用贪心得到上界，再用回溯在更小 `m` 上尝试可行性。

## R14

常见失效模式与应对：

- 失效模式 1：图太大，搜索爆炸。
  - 应对：加入更强启发式、剪枝、并行分支或改用近似/松弛方法。
- 失效模式 2：输入图结构异常（自环、脏数据）。
  - 应对：运行前做图合法性校验。
- 失效模式 3：递归层数过深。
  - 应对：改写为显式栈或提高递归限制（谨慎）。

## R15

`demo.py` 的实现结构：

- `GraphColoringBacktracking`：核心求解器类。
- `_is_safe(v, c, colors)`：检查顶点 `v` 是否可用颜色 `c`。
- `_backtrack(pos, m, colors, stats)`：递归枚举颜色并统计搜索节点数与回溯次数。
- `color_with_m(m)`：返回是否可行 + 一组可行着色 + 统计信息。
- `main()`：构造多个演示图并打印求解结果。

## R16

前驱与后继知识点：

## R17

MVP 运行说明：

- 无需任何交互输入。
- 默认演示三个图：
  - `C5`（奇环）

## R18

源码级算法流程拆解（`demo.py`，8 步）：

1. **构图与预处理**：初始化 `GraphColoringBacktracking` 时把边表转为邻接表，检查顶点编号，并检测自环。
2. **顶点排序**：按顶点度数降序形成 `order`，让高约束顶点先着色以提升剪枝效率。
3. **启动求解**：`color_with_m(m)` 创建全 `-1` 的颜色数组，初始化统计器并进入 `_backtrack`。
4. **递归终止判断**：在 `_backtrack(pos, ...)` 中，若 `pos == n`，表示所有顶点已合法着色，立即返回成功。
5. **尝试颜色**：取当前顶点 `v = order[pos]`，依次尝试颜色 `0..m-1`。
6. **安全性检查**：`_is_safe` 遍历 `v` 的邻居，若任一邻居已是该颜色，则本颜色不可用并剪枝。
7. **深入与撤销**：可用则写入 `colors[v] = c` 继续递归；若后续失败则把 `colors[v]` 恢复为 `-1` 并记录一次回溯。
8. **结果返回**：若任一路径成功，逐层返回该着色；若所有颜色都失败，则该层返回失败并由上层继续搜索。
