## R01

本条目实现 `0/1 Knapsack`（01 背包）的最小可运行版本（MVP），目标是：
- 给定每个物品的重量与价值，以及背包容量；
- 在每件物品最多选择一次的约束下，求最大总价值；
- 返回一个可复现的最优选取方案（物品下标）；
- 使用固定样例直接运行，不依赖交互输入。

## R02

问题定义（MVP 范围）：
- 输入：
  - `weights[i]`：第 `i` 件物品重量（非负整数）；
  - `values[i]`：第 `i` 件物品价值（非负整数）；
  - `capacity`：背包容量（非负整数）。
- 约束：每件物品只能选 `0` 次或 `1` 次。
- 输出：
  - `best_value`：最大可达总价值；
  - `selected_indices`：一组最优物品下标；
  - `total_weight`：该组方案总重量（必须 `<= capacity`）。

说明：最优解可能不唯一，本 MVP 返回其中一组合法最优解。

## R03

动态规划建模：
- 令 `dp[i][c]` 表示“只考虑前 `i` 件物品（下标 `0..i-1`），容量为 `c` 时的最大价值”。
- 状态转移：
  - 不选第 `i-1` 件：`dp[i-1][c]`；
  - 选第 `i-1` 件（前提 `weights[i-1] <= c`）：`dp[i-1][c-weights[i-1]] + values[i-1]`；
  - 取二者最大：
    `dp[i][c] = max(dp[i-1][c], dp[i-1][c-w] + v)`。
- 初值：
  - `dp[0][c] = 0`（没有物品时价值为 0）；
  - `dp[i][0] = 0`（容量为 0 时价值为 0）。

## R04

算法流程（MVP）：
1. 校验输入并转成一维整数数组。
2. 构建 `dp` 二维表（`(n+1) x (capacity+1)`）与 `keep` 决策表。
3. 双层循环填表：外层遍历物品，内层遍历容量。
4. 若“选当前物品”更优，则写入 `dp` 并标记 `keep[i][c] = True`。
5. 填表完成后，从 `dp[n][capacity]` 开始逆向回溯物品选择。
6. 输出最大价值、选中下标和方案总重量。
7. 额外用一维 DP 计算最优值做一致性校验。
8. 对小规模样例再用暴力枚举做结果兜底校验。

## R05

核心数据结构：
- `numpy.ndarray`：保存 `weights` 与 `values`；
- `dp: np.ndarray[int64]`：二维动态规划值表；
- `keep: np.ndarray[bool]`：回溯路径的决策记录；
- `KnapsackResult`（`dataclass`）：
  - `best_value: int`
  - `selected_indices: list[int]`
  - `total_weight: int`

## R06

正确性要点（简述）：
- 完备性：对每个状态 `dp[i][c]`，转移同时覆盖“选/不选当前物品”两种合法决策。
- 最优子结构：若最优解选择了第 `i-1` 件，则剩余部分必须是 `dp[i-1][c-w]` 的最优解；若不选，则是 `dp[i-1][c]` 的最优解。
- 无后效性：`dp[i][c]` 只依赖上一层 `i-1` 的状态。
- 因此按物品层逐步推进可得到全局最优，`dp[n][capacity]` 即最优值。
- 回溯依据 `keep` 表得到的下标集合与 `dp` 最优值一致。

## R07

复杂度：
- 二维 DP（含回溯）：
  - 时间复杂度：`O(n * capacity)`
  - 空间复杂度：`O(n * capacity)`
- 一维 DP（值校验）：
  - 时间复杂度：`O(n * capacity)`
  - 空间复杂度：`O(capacity)`
- 暴力校验（仅小规模）：
  - 时间复杂度：`O(2^n)`

## R08

边界与异常处理：
- `capacity = 0`：最优值应为 0，方案可为空；
- 所有物品都超重：最优值为 0；
- `weights` 与 `values` 长度不一致：抛出 `ValueError`；
- 输入非一维数组：抛出 `ValueError`；
- 输入出现负重量/负价值或容量为负：抛出 `ValueError`。

## R09

MVP 取舍说明：
- 仅采用 `numpy + 标准库`，避免引入重型依赖；
- 不调用第三方黑盒背包求解器，核心 DP 逐行可追踪；
- 选择“二维 DP + 回溯”确保结果可解释；
- 不实现分支定界、位运算优化、GPU 并行等高级版本。

## R10

`demo.py` 职责划分：
- `to_nonnegative_int_1d`：输入数组合法性检查；
- `validate_knapsack_inputs`：统一校验 `weights/values/capacity`；
- `knapsack_01_dp_reconstruct`：二维 DP 求最优值并回溯方案；
- `knapsack_01_dp_value_1d`：一维 DP 求最优值（交叉校验）；
- `knapsack_bruteforce_small`：小规模暴力验证；
- `run_case`：执行单个样例、打印结果并断言；
- `main`：组织固定样例，保证脚本可直接运行。

## R12

输出解读：
- `Weights / Values / Capacity`：问题实例；
- `DP 2D result`：可回溯版本给出的最优值、选中下标、总重量；
- `DP 1D value`：空间优化版得到的最优值；
- `Checks`：
  - `weight_ok=True`：回溯方案总重量不超容量；
  - `value_ok=True`：回溯方案价值与 `best_value` 一致；
  - `crosscheck_1d=True`：2D 与 1D 的最优值一致；
  - `bruteforce_ok=True`：小规模样例与暴力解一致。

## R13

建议最小测试集：
- 经典中等规模样例（有多种可行组合）；
- `capacity = 0`；
- 所有物品重量都大于容量；
- 含零重量物品（验证 0/1 约束）；
- 小规模实例与暴力枚举比对；
- 稍大容量实例验证 `O(n*C)` 实现稳定性。

## R14

可调参数：
- `capacity`：容量上限，直接影响状态空间大小；
- 样例集合：在 `main()` 的 `cases` 中增删；
- 暴力阈值：`knapsack_bruteforce_small` 的 `max_n` 可调整；
- 决策 tie-break：当前“相等时倾向不选”，可改成其他策略。

## R15

方法对比：
- 二维 DP + 回溯：
  - 优点：易解释、能直接产出方案；
  - 缺点：空间开销较大。
- 一维 DP：
  - 优点：空间降为 `O(capacity)`；
  - 缺点：若不额外记录路径，通常只返回最优值。
- 暴力枚举：
  - 优点：实现直接、可作真值基线；
  - 缺点：`O(2^n)`，仅适合小规模验证。

## R16

典型应用场景：
- 预算约束下的项目组合选择；
- 资源有限时的任务优先级打包；
- 容量受限的装载/缓存策略；
- 作为更复杂组合优化问题（多维背包、分组背包）的基础模块。

可扩展方向：

## R18

源码级算法流（对应 `demo.py`，9 步）：
1. `main` 构造固定测试样例并逐个调用 `run_case`。  
2. `run_case` 调用 `knapsack_01_dp_reconstruct` 执行二维 DP 主算法。  
3. `validate_knapsack_inputs` 将 `weights/values` 转为一维非负整数数组，并校验容量。  
4. 主循环按物品 `i=1..n` 与容量 `c=0..capacity` 填充 `dp[i][c]`。  
5. 当 `dp[i-1][c-w] + v > dp[i-1][c]` 时，写入更优值并标记 `keep[i][c]=True`。  
6. 填表结束后，从 `(n, capacity)` 逆向读取 `keep`，恢复选中物品下标集合。  
7. 计算回溯方案的总重量与总价值，并与 `dp[n][capacity]` 做一致性断言。  
8. `run_case` 再调用 `knapsack_01_dp_value_1d`，用空间优化 DP 复算最优值并比对。  
9. 对小规模样例执行 `knapsack_bruteforce_small`，最后打印结果与全部检查项。
