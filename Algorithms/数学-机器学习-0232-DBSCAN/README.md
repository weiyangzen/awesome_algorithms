## R01

DBSCAN（Density-Based Spatial Clustering of Applications with Noise）是一种基于密度连通性的聚类算法。它不要求预先指定簇数量，能识别任意形状簇，并显式把离群点标记为噪声，适合包含非球形结构和异常点的数据。

## R02

问题建模：
- 输入：样本矩阵 `X in R^(n x d)`，距离阈值 `eps`，最小样本数 `min_samples`。
- 输出：长度为 `n` 的标签向量 `labels`，其中 `-1` 表示噪声，其余非负整数表示簇编号。
- 目标：基于“局部密度足够高 + 密度可达链路”将样本划入簇，并将低密度孤立样本剔除。

## R03

DBSCAN 的三类点定义：
- 核心点（core point）：`|N_eps(x_i)| >= min_samples`。
- 边界点（border point）：自身不是核心点，但落在某个核心点的 `eps` 邻域内。
- 噪声点（noise point）：既不是核心点，也不属于任何核心点扩展得到的簇。

其中 `N_eps(x_i) = {x_j | dist(x_i, x_j) <= eps}`。

## R04

算法直觉：
1. 先找“足够拥挤”的核心点。
2. 从某个未访问核心点出发，以“密度可达”关系做 BFS/队列扩展。
3. 扩展过程中吸收邻域中的边界点。
4. 无法被任何核心点吸收的样本最终记为噪声。

因此 DBSCAN 聚的是“连成片的高密度区域”，而非固定形状或固定中心。

## R05

关键数学关系：
- 直接密度可达：若 `x_j in N_eps(x_i)` 且 `x_i` 是核心点，则 `x_j` 从 `x_i` 直接密度可达。
- 密度可达：存在链 `x_1 -> x_2 -> ... -> x_k`，每一步都直接密度可达。
- 密度相连：存在某点 `x_o`，使 `x_i` 与 `x_j` 都从 `x_o` 密度可达。

簇可理解为一个极大密度相连集合。

## R06

复杂度分析（典型实现）：
- 若使用空间索引（KD-Tree/BallTree）进行半径查询，平均时间复杂度常见为 `O(n log n)` 到 `O(n^2)` 之间，依赖数据维度与分布。
- 若使用暴力距离矩阵，邻域查询为 `O(n^2)`。
- 内存开销主要来自邻域索引结果与标签数组；若显式保存全距离矩阵，则内存为 `O(n^2)`。

## R07

参数选择经验：
- `eps` 太小：大量样本被标记为噪声，簇被切碎。
- `eps` 太大：不同簇会被错误合并。
- `min_samples` 越大：算法更保守，对噪声更鲁棒，但小簇更易被吞没。
- 常见起点：`min_samples` 取 `d+1` 到 `2d`（`d` 为特征维度），再结合 k-distance 曲线选 `eps`。

## R08

距离度量与预处理：
- DBSCAN 对距离定义敏感，欧氏距离常见于连续数值特征。
- 特征量纲不一致会扭曲邻域结构，建议先标准化。
- 高维空间下距离集中效应明显，`eps` 解释性会下降；必要时先做降维或特征筛选。

## R09

与常见聚类方法对比：
- 相比 K-Means：DBSCAN 不要求簇是凸/球形，也不需要指定 `k`。
- 相比层次聚类：DBSCAN 更偏局部密度判定，且可自然识别噪声。
- 相比 OPTICS/HDBSCAN：DBSCAN 更简单，但在“不同簇密度差异较大”场景下不够稳健。

## R10

适用场景：
- 空间点云、轨迹热点、异常检测预处理。
- 不规则几何簇（弯月形、环形、链状）识别。

不理想场景：
- 各簇密度差异极大（单一 `eps` 很难兼顾）。
- 维度很高且样本稀疏，邻域定义失真。

## R15

常见失败模式：
- 参数不匹配：`eps` 与数据尺度不一致导致全噪声或全并簇。
- 边界链桥接：两个簇被一串点误连成一个簇。
- 高维失效：所有点互相“差不多远”，半径邻域失去区分力。

排查顺序建议：先看标准化，再扫 `eps`，最后调 `min_samples`。

## R18

源码级算法流程（对应本目录 `demo.py` 的 `dbscan_mvp`）：
1. 根据 `eps` 为每个样本预计算邻域索引 `neighbors[i]`（半径查询）。
2. 统计 `neighbor_count[i]` 并得到核心点布尔向量 `core_mask[i] = (count >= min_samples)`。
3. 初始化 `labels=-99`（未分配）、`visited=False`、`cluster_id=0`。
4. 顺序扫描样本：若点已访问则跳过；未访问则先标记访问。
5. 若当前点不是核心点，临时标记为噪声 `-1`，继续扫描下一个点。
6. 若是核心点，创建新簇并将其入队；循环弹出队首点 `j`：
   - 若 `j` 未访问则访问；
   - 若 `j` 为核心点，则把其邻域点追加进队列（密度扩展）。
7. 对队列中每个点，只要标签是未分配或噪声，就赋值为当前 `cluster_id`（边界点会在这里被吸收）。
8. 队列耗尽后该簇闭包完成，`cluster_id += 1`，继续外层扫描直到全部样本处理完。
