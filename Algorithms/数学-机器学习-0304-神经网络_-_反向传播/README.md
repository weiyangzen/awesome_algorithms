## R01

反向传播（Backpropagation）是训练神经网络的核心算法：
- 前向传播计算预测值与损失；
- 反向传播用链式法则把损失对输出的梯度逐层传回输入侧；
- 优化器使用这些梯度更新参数。

本条目聚焦“算法机制透明”的最小实现，而不是直接调用高层黑箱训练接口。

## R02

问题建模（本 MVP）：
- 输入：二维特征 `X in R^(n x 2)`，标签 `y in {0,1}^n`。
- 模型：单隐层 MLP（`2 -> h -> 1`，隐层 `tanh`，输出 `sigmoid`）。
- 目标：最小化二分类交叉熵（附加 L2 正则）。

记参数为：
- `W1 in R^(2 x h)`, `b1 in R^h`
- `W2 in R^(h x 1)`, `b2 in R`

## R03

前向传播公式：
- `Z1 = XW1 + b1`
- `A1 = tanh(Z1)`
- `Z2 = A1W2 + b2`
- `P = sigmoid(Z2)`

损失函数（平均 BCE + L2）：
- `L = -mean(y*log(P+eps) + (1-y)*log(1-P+eps)) + 0.5*lambda*(||W1||^2 + ||W2||^2)`

`eps` 用于避免 `log(0)`。

## R04

反向传播的核心是链式法则。对输出层：
- `dZ2 = (P - y) / n`
- `dW2 = A1^T dZ2 + lambda*W2`
- `db2 = sum(dZ2)`

这是二分类 `sigmoid + BCE` 的标准简化形式，避免显式写出 `sigmoid` 导数后再与 BCE 相乘。

## R05

隐层梯度：
- `dA1 = dZ2 W2^T`
- `dZ1 = dA1 * (1 - A1^2)`（因为 `d/dx tanh(x) = 1 - tanh(x)^2`）
- `dW1 = X^T dZ1 + lambda*W1`
- `db1 = sum(dZ1, axis=0)`

有了这些梯度，即可进行参数更新（本实现用 SGD）。

## R06

为什么反向传播高效：
- 不做逐参数独立求导，而是复用中间量（`A1`, `dZ2`, `dZ1`）；
- 每层只需矩阵乘法与按元素运算；
- 复杂度随参数量线性增长，而不是数值微分那样高代价。

这也是现代深度学习能训练大模型的基础。

## R07

本 MVP 的最小技术栈与职责分工：
- `numpy`：前向/反向传播和参数更新主实现。
- `scipy`：`scipy.special.expit` 做稳定 sigmoid，`scipy.optimize.check_grad` 做数值梯度校验。
- `pandas`：整理训练日志表格。
- `scikit-learn`：生成 `make_moons` 数据、划分训练测试集、标准化。
- `PyTorch`：仅用自动求导做梯度对照，不把训练主流程黑箱化。

## R08

复杂度（单轮、全批量）：
- 前向：`O(n*2*h + n*h)`，主项约 `O(nh)`。
- 反向：同量级 `O(nh)`。
- 参数更新：`O(2h + h + h + 1) = O(h)`。
- 总体：每 epoch 约 `O(nh)`；训练 `T` 轮约 `O(Tnh)`。

空间复杂度主要是激活与梯度缓存，约 `O(nh)`。

## R09

数值稳定性与工程处理：
- `sigmoid` 使用 `expit`，较手写 `1/(1+exp(-x))` 更稳健。
- BCE 中加入 `eps`，避免 `log(0)`。
- 初始化权重按 `1/sqrt(fan_in)` 缩放，减小激活饱和概率。
- 固定随机种子，保证可复现实验。

## R10

正确性验证不能只看 loss 下降。本目录给两层验证：
1. `scipy.optimize.check_grad`：用有限差分检验手写梯度与数值梯度一致性。
2. `PyTorch autograd` 对照：同一批数据、同一参数下，对比每个参数梯度。

若两者误差都很小，说明反向传播实现大概率正确。

## R11

和自动微分框架的关系：
- 自动微分（如 PyTorch）本质上也在做链式法则，只是自动构图与回传。
- 手写 backprop 的价值在于：
  - 理解梯度流向；
  - 能定位梯度爆炸/消失与维度错误；
  - 可在特殊算子上实现自定义梯度。

本 MVP 把“手写实现”和“自动微分校验”放在同一脚本里，便于学习比对。

## R12

`demo.py` 的程序结构：
- `make_dataset`：生成并标准化二分类数据。
- `init_params`：初始化 `W1/b1/W2/b2`。
- `forward_pass`：计算中间量与概率。
- `loss_and_grads`：返回损失、梯度、预测概率。
- `scipy_gradient_check`：有限差分校验。
- `torch_gradient_compare`：与 PyTorch 自动求导逐参数对比。
- `train`：执行迭代训练并记录日志。
- `main`：组织实验并打印结果。

## R14

结果解释建议：
- `check_grad` 误差通常应很小（如 `1e-6` 或更低量级，受步长与数据影响）。
- PyTorch 梯度最大差异应接近数值精度（如 `1e-9 ~ 1e-6`）。
- 若训练 loss 下降且测试准确率明显高于随机猜测，说明反向传播和更新流程有效。

## R15

常见失败模式：
- 梯度符号写反：loss 上升或震荡。
- 维度广播错误：程序能跑但梯度含义错。
- `tanh` 饱和：学习率过大或初始化不当导致梯度变小。
- 忘记除以批大小 `n`：步长等效变大，训练不稳定。

处理策略：先做梯度检查，再调学习率与隐层宽度。

## R16

可扩展方向：
- 换优化器：Momentum、Adam（更新规则更稳）。
- 换激活函数：ReLU/LeakyReLU 并调整初始化。
- 多层网络：把“每层前向缓存 + 每层反向链式”扩展为循环。
- 小批量训练：将全批量改为 mini-batch 提升效率与泛化。

## R18

源码级算法流程（对应本目录 `demo.py`，9 步）：
1. 用 `make_moons` 生成非线性二分类数据，随后划分训练/测试并标准化特征。  
2. `init_params` 初始化 `W1,b1,W2,b2`，确保参数尺度合理。  
3. `forward_pass` 计算 `Z1 -> A1 -> Z2 -> P`，缓存中间量供反向传播使用。  
4. `loss_and_grads` 先计算 BCE+L2 损失，再按链式法则求 `dW2,db2,dW1,db1`。  
5. `scipy_gradient_check` 将参数打平，调用 `check_grad` 与有限差分比对手写梯度。  
6. `torch_gradient_compare` 在同一参数与同一数据上运行 PyTorch 自动求导，并与手写梯度逐项比较。  
7. `train` 进入 epoch 循环：每轮执行“前向 -> 反向 -> SGD 更新”，并定期记录 loss/acc。  
8. 用 `pandas.DataFrame` 汇总训练日志，输出头尾与最佳测试精度轮次。  
9. 在 `main` 中给出最终训练/测试准确率并做基本断言，确认 MVP 可用。
