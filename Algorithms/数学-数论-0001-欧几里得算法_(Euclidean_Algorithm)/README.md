# 欧几里得算法 (Euclidean Algorithm)

- UID: `MATH-0001`
- 学科: `数学`
- 分类: `数论`
- 源序号: `1`
- 目标目录: `Algorithms/数学-数论-0001-欧几里得算法_(Euclidean_Algorithm)`

## R01

欧几里得算法用于计算两个整数 `a, b` 的最大公约数 `gcd(a, b)`。

- 核心思想: 利用不变量 `gcd(a, b) = gcd(b, a mod b)`，不断把“大问题”降成更小余数问题。
- 输入: 两个整数 `a, b`（可为负；通常不同时为 0）。
- 输出: `g = gcd(a, b)`，满足 `g` 整除 `a` 和 `b`，且是所有公约数中最大的非负值。
- 主要步骤:
  1. 归一化为 `a = |a|, b = |b|`。
  2. 当 `b != 0` 时，执行 `(a, b) <- (b, a mod b)`。
  3. 循环结束时返回 `a`。
- 关键机制: 每轮余数严格小于除数，序列单调下降，保证有限步终止。

## R02

- 提出时代: 古希腊，约公元前 300 年。
- 主要文献来源: 欧几里得《几何原本》(Elements) 第 VII 卷（整数理论部分）。
- 历史定位: 是现存文献中最早、最系统的高效整数算法之一。

## R03

在古代数学中，分数约分、比例比较、整除关系分析都依赖“公约数”计算。朴素做法是枚举所有约数，成本高且不可扩展。欧几里得算法把“找最大公约数”转化为“反复做带余除法”，将问题规模快速缩小，形成可证明正确、可手算、可程序化的通用方法。

## R04

时间复杂度可从两层理解。

- 单位成本模型（把除法视作 O(1)）:
  - 迭代次数为 `O(log min(|a|, |b|))`。
  - 最坏输入可由相邻 Fibonacci 数达到（迭代次数最多）。
- 位复杂度模型（大整数）:
  - 若输入位长为 `n`，总成本约为 `O(M(n) * log n)`，`M(n)` 是 `n` 位整数乘法复杂度（除法复杂度同阶讨论）。

## R05

- 迭代实现额外空间: `O(1)`（除输入与常数个临时变量外不额外分配）。
- 递归实现额外空间: `O(log min(|a|, |b|))`（调用栈深度）。
- 扩展欧几里得算法同样可做迭代版，保持 `O(1)` 额外工作内存。

## R06

示例: 计算 `gcd(252, 198)`。

1. `252 = 198 * 1 + 54`
2. `198 = 54 * 3 + 36`
3. `54 = 36 * 1 + 18`
4. `36 = 18 * 2 + 0`

最后一个非零余数是 `18`，所以 `gcd(252, 198) = 18`。

## R07

欧几里得算法的意义体现在三个层面。

- 理论层: 是整数环上“理想由最大公约数生成”的基础计算过程，连接整除理论与 Bézout 恒等式。
- 算法层: 是“循环不变量 + 单调下降 + 终止证明”的经典范式。
- 工程层: 广泛用于分数约分、模逆、同余方程、RSA 等密码协议实现。

## R08

算法路径上的直接依赖。

- 带余除法定理: `a = bq + r, 0 <= r < |b|`。
- 整除关系基本性质: 若 `d | a` 且 `d | b`，则 `d | (a - kb)`。
- 不变量等价: `gcd(a, b) = gcd(b, a mod b)`。
- （扩展版本）Bézout 恒等式: `ax + by = gcd(a, b)`。

## R09

适用前提与边界条件。

- 输入域: 整数（正、负、零均可）。
- 约定: `gcd(a, 0) = |a|`；`gcd(0, 0)` 在严格数学定义下未定，工程中常约定返回 `0`。
- 不适用场景:
  - 非整数对象（浮点数、符号表达式）若无离散化定义，不能直接套用。
  - 需要“约数分解”而非“公约数”时，欧几里得算法不能替代质因数分解算法。

## R10

正确性依据（简述）。

1. 不变量: 每一步把 `(a, b)` 替换为 `(b, a mod b)`，最大公约数不变。
2. 终止性: 余数序列 `b, r1, r2, ...` 为非负严格下降整数序列，必在有限步到 0。
3. 最终性: 当 `b = 0` 时，`gcd(a, 0) = a`（取非负），因此返回值为原问题的最大公约数。

## R11

数值稳定性与误差传播（整数算法视角）。

- 舍入误差: N/A（纯整数运算，不涉及浮点舍入传播）。
- 溢出风险:
  - 固定宽度整数语言（如 32/64 位）可能在中间计算时溢出，需要大整数库或边界保护。
  - Python `int` 为任意精度整数，无溢出但会带来更高常数开销。
- 结论: 数学结果精确，主要风险来自实现层的整数表示上限，而非数值稳定性。

## R12

真实计算成本（超越渐近记号）。

- 单步成本主项是整数除法/取模，不是加减法。
- 对机器字整数，分支预测和除法指令延迟决定常数项。
- 对大整数，底层多精度除法复杂度和内存分配占主导。
- 并行化: 单次 `gcd` 数据依赖强，不适合细粒度并行；批量 `gcd` 可在任务级并行。
- 向量化/GPU: 对单实例收益有限，批量场景才可能受益。

## R13

- 近似比/误差界/概率保证: N/A。
- 原因: 欧几里得算法是确定性精确算法，不是近似算法，也不依赖随机化。

## R14

鲁棒性与失效模式。

- 输入同时为 0: 数学上无唯一 gcd，工程实现必须显式给出约定。
- 负数输入: 若不做绝对值归一化，返回符号可能与预期不一致。
- 递归实现: 在极大输入下可能触发递归深度限制；迭代版更稳健。
- 跨语言移植: `%` 对负数的定义在不同语言可能不同，需要统一规范。

## R18

源码追踪与算法拆解（本条目未把第三方包作为黑箱；核心实现在 `demo.py`）。

以下是 `gcd_euclidean` 的源级流程（6 步）。

1. 读取输入整数 `a, b`，先取绝对值，消除符号差异。
2. 进入循环条件 `b != 0`。
3. 计算余数关系，通过并行赋值执行 `(a, b) = (b, a % b)`。
4. 利用不变量保持 `gcd(a, b)` 不变，同时让第二参数严格变小。
5. 重复步骤 2-4，直到 `b` 变为 `0`。
6. 返回 `a` 作为最终 gcd（若初始 `a=b=0`，按工程约定返回 `0`）。

扩展欧几里得 `extended_gcd` 在同样循环中额外维护系数对 `(s, t)`，使得每步都保持
`old_r = a*old_s + b*old_t`，终止时直接得到 Bézout 系数。
