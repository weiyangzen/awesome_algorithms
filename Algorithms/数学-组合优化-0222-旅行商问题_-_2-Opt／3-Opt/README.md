# 旅行商问题 - 2-Opt/3-Opt

- UID: `MATH-0222`
- 学科: `数学`
- 分类: `组合优化`
- 源序号: `222`
- 目标目录: `Algorithms/数学-组合优化-0222-旅行商问题_-_2-Opt／3-Opt`

## R01

旅行商问题（TSP）要求在给定城市集合上找到一条哈密顿回路，使总路程最短。该问题是经典 NP-hard 组合优化问题。
本条目聚焦局部搜索启发式 `2-Opt/3-Opt`：
- 不保证全局最优；
- 通过反复改进当前回路快速得到高质量可行解；
- 适合作为工程中的轻量近似求解器与大规模算法的后处理器。

## R02

输入与输出定义：
- 输入：`n` 个城市的二维坐标 `coords in R^(n×2)`；
- 输出：一个城市排列 `tour`（长度为 `n`，每个城市恰好一次），其闭环长度最小化（局部意义）。

`demo.py` 中默认：
- 固定随机种子生成 `24` 个城市；
- 用最近邻法生成初始解，再依次进行 2-Opt 和 3-Opt 改进；
- 打印各阶段路径长度、改进比例与耗时。

## R03

`2-Opt` 核心思想：
- 在回路中选两条不相邻边 `(a,b)` 与 `(c,d)`；
- 改连为 `(a,c)` 与 `(b,d)`；
- 对应地将中间路径片段反转；
- 若新回路更短则接受该交换。

这相当于去除路径交叉，通常能带来明显下降。

## R04

`3-Opt` 核心思想：
- 切断三条边，把回路分成 `A|B|C|D` 四段；
- 枚举若干合法重连模式（本实现显式枚举 7 类）；
- 选取最短候选并在有改进时替换当前回路。

相比 2-Opt，3-Opt 的局部邻域更大，能跳出部分 2-Opt 局部最优。

## R05

目标函数：
- 回路长度 `L(tour) = sum_i dist[tour[i], tour[i+1]]`，其中 `tour[n]=tour[0]`（闭环）。

局部搜索终止条件：
- 在一次完整扫描中找不到严格改进（`delta < -1e-12`）即停止；
- 或达到轮数上限（防止异常数据下过慢）。

## R06

`demo.py` 的整体流程：
1. 随机生成城市坐标并计算欧氏距离矩阵；
2. 最近邻法得到初始回路；
3. 执行 2-Opt 直到局部最优；
4. 在 2-Opt 结果上执行 3-Opt；
5. 校验回路合法性并打印结果。

## R07

复杂度（`n` 为城市数）：
- 单次 2-Opt 扫描：`O(n^2)`；
- 2-Opt 总体：`O(r2 * n^2)`，`r2` 为改进轮数；
- 单次 3-Opt 扫描：约 `O(n^3 * c)`，`c` 为每组三切点的重连候选数（本实现 `c=7`）；
- 3-Opt 总体：`O(r3 * n^3)`。

因此 3-Opt 质量更高但代价更大。

## R08

适用前提与场景：
- 距离为对称欧氏距离或一般度量距离时表现稳定；
- 需要“较好解 + 快速实现”的工程场景；
- 可作为遗传算法、模拟退火、蚁群等方法中的局部改进模块。

## R09

优势与局限：
- 优势：实现简单、可解释、无需大规模外部依赖、解质量通常明显优于原始构造解；
- 局限：仅给局部最优；对初始解敏感；3-Opt 在 `n` 很大时计算成本上升明显。

## R10

实现要点（对应代码）：
- `pairwise_distance_matrix` 使用 `numpy` 广播计算完整距离矩阵；
- `two_opt_pass` 用边替换增量公式快速评估交换是否改进；
- `_three_opt_candidates` 不调用黑盒库，显式构造 7 类重连候选；
- `three_opt_pass` 在三切点组合上做 best-improvement 选择。

## R11

正确性检查策略：
- 每阶段后调用 `_validate_tour`，确保回路是 `0..n-1` 的排列；
- 断言 `L_after_2opt <= L_init` 与 `L_after_3opt <= L_after_2opt`；
- 浮点比较使用微小容忍量 `1e-9`/`1e-12`，避免数值噪声误判。

## R12

MVP 技术栈与可复现性：
- 依赖：`Python 3 + numpy`；
- 不依赖交互输入，不依赖网络或外部数据；
- 固定随机种子（`seed=222`）保证可复现。

## R13

本实现不是精确算法，不提供全局最优保证。

但有以下“可验证保证”：
- 任一被接受的 move 都使目标函数严格下降；
- 局部搜索终止时，当前邻域内不存在更优候选；
- 故最终结果至少是对应邻域（2-Opt 或 3-Opt）下的局部最优。

## R14

边界与异常：
- 城市数过小（如 `<4`）时 3-Opt 邻域极小，改进可能为 0；
- 若输入回路缺失/重复城市，会在 `_validate_tour` 中抛出异常；
- 距离矩阵若含非法值（`NaN/Inf`）会破坏比较逻辑，实践中应先做数据清洗。

## R15

建议测试集：
1. 小规模手工点集（可肉眼判断路径交叉是否消除）；
2. 随机点集（验证平均改进幅度）；
3. 不同初始构造（最近邻起点变化）对结果的影响；
4. 增大 `n` 观察 2-Opt 与 3-Opt 的质量-耗时权衡。

## R16

调参建议：
- `n_cities`：先用 `20~60` 调试，保证 3-Opt 可在秒级运行；
- `max_rounds`（2-Opt/3-Opt）：默认足够，若问题更大可限制 3-Opt 轮数；
- 初始解策略：可替换为多起点最近邻，再选最优初解继续局部搜索。

## R18

`demo.py` 源码级算法流程（8 步）：
1. `build_cities` 用固定种子生成二维城市坐标，`pairwise_distance_matrix` 计算 `n×n` 欧氏距离矩阵。  
2. `nearest_neighbor_tour` 从起点 `0` 出发贪心选最近未访问城市，构造初始 Hamilton 回路。  
3. `two_opt_pass` 枚举可交换边对 `(a,b),(c,d)`，用增量 `dist[a,c]+dist[b,d]-dist[a,b]-dist[c,d]` 评估收益。  
4. `two_opt_local_search` 重复执行 2-Opt best-improvement，直到无改进，得到 2-Opt 局部最优。  
5. `three_opt_pass` 枚举三切点 `(i,j,k)`，调用 `_three_opt_candidates` 生成 7 类重连候选并计算完整回路长度。  
6. 在所有候选中选择最优且严格更短的回路，`three_opt_local_search` 迭代直至无改进或触发轮数上限。  
7. `main` 中对初始解、2-Opt 解、3-Opt 解分别计算长度，执行排列合法性检查与单调不劣化断言。  
8. 最后打印阶段长度、改进百分比、耗时与部分最终路径，形成可复现的最小闭环验证。  

本实现未调用任何 TSP 黑盒求解器，邻域构造、候选评估与迭代策略均可在源码中逐行追踪。
