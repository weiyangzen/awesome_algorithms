# 覆盖空间算法

- UID: `MATH-0251`
- 学科: `数学`
- 分类: `代数拓扑`
- 源序号: `251`
- 目标目录: `Algorithms/数学-代数拓扑-0251-覆盖空间算法`

## R01

问题定义（离散化版本）：
给定一个连通无向图 \(G\)（视为 1 维 CW 复形），构造其覆盖空间中的可计算对象。  
本 MVP 选择“截断万有覆盖”作为目标：从根点出发，用长度不超过 `max_depth` 的约化路径集合来近似万有覆盖 \(\widetilde{G}\)，并给出投影映射
\[
p:\widetilde{G}_{\le d} \to G
\]
其中覆盖节点对应约化路径，投影到该路径在基图中的终点。

## R02

数学背景：
- 覆盖空间：映射 \(p:E\to B\) 使得 \(B\) 中每点有邻域 \(U\)，满足 \(p^{-1}(U)\) 是若干个与 \(U\) 同胚的片。
- 对图（1 维复形）而言，局部条件等价于“每个节点的出射半边（star）在覆盖图和基图间一一对应”。
- 万有覆盖对应基本群作用下的展开空间；对连通图，基本群是自由群，秩为 `m - n + 1`（`m` 边数，`n` 顶点数）。
- 无限万有覆盖不可直接全部枚举，因此采用深度截断。

## R03

输入与输出（`demo.py`）：
- 输入（代码内置示例）：
  - 基图边列表 `[(u, v, label), ...]`。
  - 根顶点 `root_vertex`。
  - 截断深度 `max_depth`。
- 输出：
  - 截断覆盖图节点/边统计；
  - 每层节点数；
  - 前若干节点的约化词表示；
  - 局部覆盖性质检查结果（PASS/FAIL）。

## R04

核心思想：
- 把覆盖节点表示为“从根出发的约化弧序列（word）”。
- 约化规则：禁止相邻弧互为逆（避免立即抵消）。
- 从每个节点向所有合法出射弧扩展，形成一棵树（有限深度）。
- 覆盖映射 `p` 直接读取该 word 在基图中的终点顶点。

## R05

算法步骤（高层）：
1. 将每条无向边拆成两条定向弧，记录逆弧映射 `inverse`。
2. 根节点设为空词 `()`，终点为根顶点。
3. BFS 遍历覆盖节点；若深度达到上限则停止扩展。
4. 对当前节点的每条出射弧：
   - 若等于“上一条弧的逆”则跳过（保持约化）；
   - 否则追加到 word 形成新节点。
5. 添加覆盖边，并在两端记录对应的局部出射弧编号。
6. 结束后得到截断覆盖图与投影信息。

## R06

正确性要点（MVP 级）：
- 节点语义正确：每个节点唯一对应一条约化路径。
- 边语义正确：相邻节点只差一个合法弧扩展，正是“路径延长一步”。
- 覆盖局部性：对内部节点（`depth < max_depth`），其邻接弧集合应与基图对应顶点的出射弧集合完全一致。
- 代码中 `check_local_covering_property` 对上述局部性进行程序化验证。

## R07

复杂度分析：
- 设基图最大度数为 \(\Delta\)，截断深度为 \(d\)。
- 节点数近似：
  - 根层为 1；
  - 第一层约 \(\Delta\)；
  - 之后每层约乘 \((\Delta-1)\)（因禁止立即回退）。
- 总节点规模约为
\[
1 + \Delta\sum_{k=0}^{d-1}(\Delta-1)^k
\]
（当 \(\Delta>2\) 时指数增长）。
- 时间与空间复杂度均与生成节点/边总量线性同阶。

## R08

边界与鲁棒性：
- 支持自环边（`u == v`），每条自环仍拆成两条互逆弧。
- 支持多重边（不同 `label` 或不同 edge_id）。
- `max_depth = 0` 时仅返回根节点。
- 若图不连通，本实现默认用户选取了目标连通分量内根点（MVP 不自动分量分解）。

## R09

MVP 范围声明：
- 实现“图上的覆盖空间算法”，不是一般拓扑空间上的全覆盖构造器。
- 实现“万有覆盖截断近似”，不是无限对象的完整表示。
- 提供基础正确性检查（局部星状邻域），不做形式化证明器。

## R10

代码结构：
- `DirectedEdge`：定向弧数据结构。
- `UndirectedGraph`：无向图 + 定向弧展开 + 逆弧映射。
- `CoverNode`：覆盖节点（约化词、终点、父节点等）。
- `TruncatedUniversalCover`：覆盖图容器与统计属性。
- `build_truncated_universal_cover`：核心构造算法。
- `check_local_covering_property`：局部覆盖条件校验。
- `summarize_levels`：用 `numpy` 做分层统计。
- `main`：演示入口。

## R12

示例解释（默认基图）：
- 基图为“一点双圈”（两个自环），可看作 \(S^1 \vee S^1\) 的离散模型。
- 其基本群是秩 2 自由群，对应万有覆盖是 4-正则树（离散意义下）。
- 截断后各层节点数通常呈倍增趋势（除根层外接近按 3 倍扩展）。

## R13

- 若基图度数较大，需更保守深度（例如 `2~4`）。
- 如需更大规模实验，可在后续版本加入磁盘缓存或流式生成。

## R16

常见错误与规避：
- 错误 1：把“禁止回退”写成“禁止所有重复边”，会误删合法路径。
- 错误 2：只记录无向边，不记录定向弧，导致局部星状对应失真。
- 错误 3：未在子节点端写入逆弧，局部校验会假失败。
- 错误 4：深度过大导致节点爆炸，应先估计增长率再设置参数。

## R17

与相关算法对比：
- 与“最短路/BFS 树”：
  - BFS 树关注最短路径覆盖；
  - 本算法关注拓扑展开与路径同伦信息（通过约化词）。
- 与“并查集连通性”：
  - 并查集只判断连通；
  - 本算法显式构造覆盖空间近似结构。
- 与“直接调用图库黑盒生成器”：
  - 本实现显式编码了约化词扩展、逆弧约束和局部覆盖验证，流程可审计。

## R18

`demo.py` 源码级流程拆解（8 步）：
1. **图展开**：`UndirectedGraph.__init__` 把每条无向边拆成两条定向弧，建立 `outgoing` 和 `inverse`。
2. **初始化根**：`build_truncated_universal_cover` 创建空词根节点 `word=()`，并放入 BFS 队列。
3. **逐层遍历**：弹出当前覆盖节点，若 `depth >= max_depth` 就只保留不再扩展。
4. **约化过滤**：读取当前词尾弧的逆弧 `forbidden`，遍历基图出射弧时跳过该逆弧，避免“走一步立刻回退”。
5. **生成新节点**：将合法弧拼到词尾得到 `new_word`，创建子节点并记录其基图终点顶点。
6. **连接覆盖边**：在父子两端分别写入局部弧编号（子端写逆弧），形成可用于局部性校验的邻接结构。
7. **局部覆盖检查**：`check_local_covering_property` 对每个内部节点比较“覆盖邻接弧集合”与“基图出射弧集合”是否一致。
8. **结果汇总输出**：`main` 计算基本群秩 `m-n+1`、打印规模统计、用 `numpy` 汇总每层节点数并展示前若干约化词。
