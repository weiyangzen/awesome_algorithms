# Vigenere密码

## R01

Vigenere 密码（维吉尼亚密码）是一种经典的多表替换密码（polyalphabetic substitution）。
它通过“关键字驱动的字母位移”替代单一固定移位（如凯撒密码），能在一定程度上弱化简单频率分析。

## R02

核心思想：
- 把明文和密钥都映射为 `A-Z` 上的数字 `0-25`；
- 密钥按长度循环扩展，与明文逐字母对齐；
- 每个位置使用该位置的密钥位移量进行加法替换。

## R03

符号约定：
- 明文字母值 `P_i ∈ {0,...,25}`；
- 密钥字母值 `K_i ∈ {0,...,25}`；
- 密文字母值 `C_i ∈ {0,...,25}`；
- 字母表大小 `m = 26`。

## R04

加密公式：

`C_i = (P_i + K_i) mod 26`

其中 `K_i` 来自循环密钥。若原字符是大写，输出保持大写；原字符是小写，输出保持小写。

## R05

解密公式：

`P_i = (C_i - K_i) mod 26`

解密与加密共用同一密钥，且必须保证字母位置对齐规则一致（仅对字母推进密钥索引）。

## R06

本目录 MVP 的字符处理规则：
- 仅对英文字母 `A-Z/a-z` 做 Vigenere 变换；
- 非字母字符（空格、数字、标点）原样保留；
- 密钥中非字母字符会被忽略；
- 若密钥过滤后为空，直接抛 `ValueError`。

## R07

经典示例：
- 明文：`ATTACKATDAWN`
- 密钥：`LEMON`
- 密文：`LXFOPVEFRNHR`

该示例将作为 `demo.py` 的正确性断言之一。

## R08

复杂度分析：
- 设文本长度为 `n`，密钥长度为 `k`；
- 预处理密钥 `O(k)`；
- 加密或解密各做一次线性扫描，时间复杂度 `O(n)`；
- 输出缓冲 `O(n)`，额外状态为常数级。

## R09

安全性说明：
- Vigenere 比单表替换更强，但不具备现代密码学安全性；
- 当密钥较短且循环复用时，可被 Kasiski 检验、重合指数（IC）等方法推测密钥长度；
- 一旦密钥长度暴露，问题可分解为若干凯撒子问题并被频率分析破解。

## R10

MVP 范围与目标：
- 提供无交互、可直接运行的加密/解密最小实现；
- 覆盖：标准教材示例、通用文本往返验证、错误密钥负例；
- 提供一个简单统计指标（IC）用于展示经典密码分析入口。

## R11

`demo.py` 主要函数：
- `_key_to_shifts(key)`：清洗密钥并映射为位移序列；
- `_shift_char(ch, shift)`：对单字符做位移（保留大小写）；
- `vigenere_encrypt(plaintext, key)`：加密主流程；
- `vigenere_decrypt(ciphertext, key)`：解密主流程；
- `coincidence_index(text)`：计算重合指数（字母过滤后）。

## R13

预期输出与断言：
- 经典示例 `ATTACKATDAWN + LEMON -> LXFOPVEFRNHR` 通过；
- 通用消息 `decrypt(encrypt(msg,key), key) == msg`；
- 同一消息和同一密钥重复加密结果相同（确定性）；
- 使用错误密钥解密不会还原原文。

## R14

边界条件：
- 明文可为空串；
- 明文全是非字母字符时，加解密结果与输入一致；
- 密钥包含数字/符号时会自动忽略这些字符；
- 密钥若无任何字母，抛出异常阻止继续执行。

## R15

工程风险与限制：
- 不具备抗主动攻击能力（无认证）；
- 不具备语义安全，同明文同密钥得到同密文；
- 面对现代算力和统计分析，不能用于真实敏感数据保护。

## R17

测试建议：

- 性质测试：随机文本做 `decrypt(encrypt(x,k),k)` 恒等；
- 负例测试：错误密钥恢复失败且输出长度保持一致。

## R18

源级算法流程拆解（对应 `demo.py`，8 步）：
1. `main()` 先运行教材标准样例，调用 `vigenere_encrypt("ATTACKATDAWN", "LEMON")` 并断言结果为 `LXFOPVEFRNHR`。  
2. `_key_to_shifts` 过滤密钥中的非字母字符，把字母映射到 `0-25` 位移列表；若列表为空立即报错。  
3. `vigenere_encrypt` 线性扫描明文：仅当字符是字母时，取当前密钥位移并调用 `_shift_char` 执行 `(p+k) mod 26`。  
4. `_shift_char` 根据输入字符大小写选择基址（`A` 或 `a`），完成移位后恢复对应大小写字符。  
5. `vigenere_decrypt` 与加密同结构扫描，但把移位改为负移位，实现 `(c-k) mod 26`。  
6. `main()` 对业务示例文本执行加密后再解密，断言恢复值与原文完全一致。  
7. `main()` 再次对同一明文同一密钥加密，验证输出一致，展示该算法的确定性特征。  
8. 通过 `coincidence_index` 统计明文与密文的字母重合指数并打印，展示后续密码分析可利用的统计入口。  

以上流程完全由源码显式实现，不依赖第三方密码学黑盒。
