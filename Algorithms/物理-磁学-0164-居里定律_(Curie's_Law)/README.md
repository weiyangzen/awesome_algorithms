## R01

居里定律描述理想顺磁体在较高温区（远离磁有序转变温度）下的磁化率与温度关系：

`chi(T) = C / T`

其中：

- `chi`：体磁化率（或摩尔磁化率，取决于单位体系）；
- `T`：绝对温度（K）；
- `C`：居里常数，反映磁矩强度与粒子浓度。

该关系体现热涨落对磁矩取向的无序化作用：温度越高，平均磁响应越弱。

## R02

本条目的目标是给出一个可执行的最小 MVP，用于“从 `chi-T` 数据回推居里常数 `C`”：

- 生成带噪声的合成磁化率数据；
- 使用居里定律线性化关系估计初值；
- 用非线性最小二乘在原始模型空间精修参数；
- 输出误差指标与断言，确保脚本可自动验证。

## R03

MVP 的建模假设：

- 系统处于适用居里定律的顺磁区间；
- 温度 `T > 0` 且单位一致；
- 观测噪声近似为小幅乘性噪声；
- 忽略温度无关背景项（例如 `chi0`）；
- 参数识别聚焦单参数 `C`，不引入额外自由度。

## R04

核心推导：

1. 原式为 `chi = C / T`。
2. 取倒数可得线性关系：`1/chi = (1/C) * T`。
3. 设 `y = 1/chi`, `x = T`，则 `y = a*x`，其中 `a = 1/C`。
4. 线性回归得到斜率 `a` 后，反推 `C = 1/a`。

因此可以先在倒数空间做稳定初估，再回到原始 `chi(T)` 空间进行非线性精修。

## R05

`demo.py` 的数值流程与工具分工：

- `numpy`：生成温度网格、构造噪声、执行基础向量化计算；
- `pandas`：组织观测值、拟合值与输出表；
- `scikit-learn`：在 `1/chi-T` 线性化空间回归求初值；
- `PyTorch`：独立解同一线性最小二乘系统，用于交叉校验；
- `scipy.optimize.curve_fit`：在原始模型 `chi=C/T` 上做有界非线性最小二乘精修。

## R06

脚本运行后输出：

- 真值 `C_true`；
- 线性化估计 `C_linear` 及 `R2(1/chi~T)`；

## R07

优点：

- 只拟合一个核心参数，解释清晰；
- 线性化初值 + 非线性精修的流程贴近实际参数反演；
- 多求解器（`sklearn` 与 `torch`）一致性检查可降低实现错误风险。

局限：

- 未建模温度无关背景磁化率 `chi0`；
- 靠近磁相变区时，居里定律本身可能失效。

正确性直觉：

1. 居里定律在理想条件下是一参数模型，识别难度低；
2. 倒数线性化能快速定位合理参数区间；
3. 非线性最小二乘直接在 `chi` 空间最小化残差，避免仅在倒数空间拟合带来的偏差；
4. 独立线性求解器结果一致，说明代数实现正确；

## R14

常见失败模式：

1. 温度单位混用（K 与 °C）导致参数物理意义错误；
2. 噪声过大且样本过少，线性初值偏差显著；
3. 数据存在背景项但模型仍强行使用 `chi=C/T`，出现系统偏差；
4. 温度范围太窄，参数可辨识性下降；
5. 忽略异常值处理，回归被少量离群点主导。

## R18

`demo.py` 的源码级算法流程（8 步）：

1. `main` 创建 `CurieLawConfig`，固定真值 `C`、温度区间、噪声强度与随机种子。  
2. `generate_synthetic_data` 生成温度网格 `T`，按 `chi_clean = C/T` 计算真值并叠加乘性噪声得到 `chi_observed`。  
3. `fit_linearized_sklearn` 将观测映射到 `y=1/chi`，对 `y = aT` 做无截距最小二乘，得到 `a` 并反推 `C=1/a`。  
4. `fit_linearized_torch` 使用 `torch.linalg.lstsq` 解同一线性系统，得到独立斜率估计并与 `sklearn` 做一致性断言。  
5. `fit_nonlinear_scipy` 以线性估计作为初值，在 `C>0` 约束下调用 `curve_fit` 最小化目标：`sum_i (chi_obs_i - C/T_i)^2`。  
6. `curve_fit` 内部会在每轮迭代计算残差向量与雅可比（本模型对 `C` 的导数是 `1/T`），再解受约束的局部最小二乘子问题更新参数。  
7. `summarize_results` 计算 `MAE/RMSE`、相对误差，并打印头尾样本表，比较线性与非线性结果。  
8. 断言层检查 `R2`、参数误差、求解器一致性与误差上限，保证脚本在 CI/批处理中可自动验收。  

第三方库不是黑箱替代：物理模型表达、参数反推公式、边界设计、验收阈值与一致性校验都在源码显式实现，库仅承担线性代数与优化求解内核。
