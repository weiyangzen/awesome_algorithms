# 深度优先搜索 (DFS)

- UID: `MATH-0473`
- 学科: `数学`
- 分类: `图论`
- 源序号: `473`

## R01

深度优先搜索（Depth-First Search, DFS）是图遍历的基础算法之一。核心策略是：

- 从起点出发，优先沿着一条路径尽可能向深处探索；
- 当无法继续深入时回溯到最近的分叉点，再探索未访问分支。

DFS 既可用于无向图，也可用于有向图；既可用于单源可达性，也可作为更复杂算法（如拓扑排序、强连通分量、桥/割点检测）的底层骨架。

## R02

DFS 的历史可追溯到 20 世纪图论和组合搜索的发展过程。它和 BFS 一起构成了图遍历的两大基本范式：

- BFS 偏“层次扩展”；
- DFS 偏“路径深入+回溯”。

随着算法理论成熟，DFS 被系统化用于线性时间图算法：例如基于时间戳（discovery/finish time）的有向图分析、Tarjan 系列算法等。

## R03

DFS 的现实动机很直接：

- 你需要判断两个点是否连通；
- 你需要找出图的连通分量；
- 你需要检测是否存在环；
- 你需要在搜索空间中进行“试探-回退”式遍历。

相比“每次盲目全图扫描”，DFS 能在可达子图范围内高效完成任务，是图问题和回溯问题的通用起点。

## R04

设图有 `V` 个顶点、`E` 条边（邻接表存储），DFS 的时间复杂度为：

- 单次完整遍历：`O(V + E)`；
- 从单个起点出发的可达遍历：`O(V_reach + E_reach)`，最坏仍为 `O(V + E)`。

原因是每个顶点最多被首次访问一次，每条边最多被检查常数次（无向图约两次，有向图一次）。

## R05

空间复杂度主要来自：

- `visited` 标记：`O(V)`；
- 递归调用栈或显式栈：最坏 `O(V)`；
- 邻接表本身：`O(V + E)`（这是图输入存储成本）。

因此算法“额外空间”可记为 `O(V)`。

## R06

一个最小无向图示例：

- 顶点：`0,1,2,3,4`
- 边：`(0,1), (1,2), (1,3), (3,4)`

从 `0` 开始 DFS（按邻居升序）可能得到访问序：`[0, 1, 2, 3, 4]`。

如果再加入边 `(2,4)`，图中就出现环，DFS 的回边判定逻辑会检测到这一点。

## R07

DFS 的价值不仅在“能遍历”，更在“可组合”：

- 作为基础能力：可达性、连通分量、路径存在性；
- 作为结构分析工具：环检测、时间戳分析、边分类；
- 作为上层算法组件：拓扑排序、SCC、桥/割点、回溯搜索。

所以 DFS 往往是图论工程代码库里最先实现、复用率最高的模块之一。

## R08

本目录 MVP 采用两种等价实现思路：

1. 递归 DFS（代码简洁，逻辑直观）；
2. 迭代 DFS（显式栈，避免深递归风险）。

此外在同一个 demo 中复用 DFS 完成三类任务：

- 单源遍历顺序；
- 无向图连通分量拆分；
- 无向图环检测。

这体现了 DFS 的“一个核心框架，多种问题映射”的特点。

## R09

适用边界与前提：

- 适用：静态图或阶段性静态图的遍历分析；
- 适用：稀疏图与中等规模图（邻接表）；
- 不适用：超深递归环境下直接使用递归版（可能触发递归深度限制）；
- 不适用：需要最短路层次信息时（应优先 BFS 或最短路算法）。

本目录实现默认顶点编号为 `0..n-1`，并在建图时做范围校验。

## R10

正确性直觉：

- 通过 `visited` 保证每个顶点只会“首次进入”一次，不会无限循环；
- 对每个已访问顶点，DFS 会枚举其所有邻居并递归/入栈处理未访问邻居；
- 因此从起点可达的所有顶点最终都会被访问，且不会遗漏。

用于连通分量时，对每个“未访问顶点”启动一次 DFS，即可得到一个分量；重复直到全部顶点访问完成，分量划分完备且互不重叠。

## R11

稳定性与实现注意事项：

- 无向图环检测时要排除“指向父节点”的边，否则会误报；
- 如果需要可复现遍历顺序，应固定邻居顺序（本 demo 采用升序邻接）；
- 迭代版若要模拟递归前序，入栈邻居需逆序压栈；
- 递归版在深链图上可能触发 `RecursionError`，工程中优先准备迭代备选。

## R12

实际性能主要受以下因素影响：

- 图的存储方式（邻接表通常优于邻接矩阵）；
- 顶点度分布（高出度节点会拉高局部遍历成本）；
- 邻居排序策略（排序有稳定性收益，但会增加建图常数开销）。

在 Python 中，合理的数据结构选择（`list` + `bool visited` + 显式栈）对性能影响很大。

## R13

保证类型：

- 该算法是确定性算法，不依赖随机化概率保证；
- 在输入图合法、遍历逻辑正确实现时，`O(V+E)` 上界成立；
- 本目录 demo 通过“随机对拍（DFS vs BFS/并查集判定）”做工程级正确性补强。

## R14

常见失效模式：

- 忘记设置 `visited`，导致无限递归/死循环；
- 无向图环检测未排除父边，导致假阳性；
- 迭代 DFS 入栈顺序错误，导致遍历顺序和预期不一致；
- 顶点越界或脏数据未校验，导致运行时异常或隐式错误。

这些问题在小图上可能不明显，但在批量任务中会迅速放大。

## R16

方法谱系与关联：

- 同层基础：BFS（按层扩展）；
- DFS 下游：拓扑排序、Kosaraju/Tarjan SCC、桥与割点、欧拉路径构造中的子步骤；
- 搜索问题扩展：回溯、状态空间搜索、约束求解中的深搜框架。

掌握 DFS 后，很多图算法都能看成“DFS 主干 + 额外状态维护”。

## R17

本目录 `demo.py` 提供的 MVP 内容：

- 建图与输入校验：`build_graph`；
- 核心算法：`dfs_recursive`、`dfs_iterative`；
- 典型应用：`reachable_dfs`、`connected_components`、`has_cycle_undirected`；
- 验证模块：
  - 固定样例演示；
  - 随机图对拍（可达性对比 BFS、环检测对比并查集判定）。

运行方式：

```bash
python3 demo.py
```

脚本无交互输入，运行后会打印演示结果和随机测试通过信息。

## R18

源码级算法流程拆解（对应 `demo.py`，8 步）：

1. `build_graph` 把边集转换为邻接表，并对顶点范围、重边进行规范化处理。
2. `dfs_recursive` 从起点开始，执行“标记访问 -> 记录顺序 -> 递归未访问邻居”。
3. `dfs_iterative` 用显式栈复现深搜流程，通过逆序压栈保持与递归版一致的前序语义。
4. `reachable_dfs` 复用栈式 DFS，在搜索到目标顶点时提前返回，实现路径存在性判定。
5. `connected_components` 对每个未访问顶点启动一次 DFS，收集一个分量并写入结果列表。
6. `has_cycle_undirected` 在 DFS 状态中携带父节点，遇到“已访问且非父节点”的邻居即判环。
7. `run_fixed_demo` 在固定图上输出遍历序、分量和环检测结果，并断言关键性质。
8. `run_random_crosscheck` 批量生成随机图，使用 BFS 与并查集判定进行对拍，确保 DFS 实现的可达性与环检测正确。
