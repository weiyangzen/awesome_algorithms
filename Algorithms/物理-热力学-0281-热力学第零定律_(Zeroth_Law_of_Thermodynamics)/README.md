## R01

热力学第零定律的核心表述是：
若系统 `A` 与 `C` 处于热平衡，且系统 `B` 与 `C` 也处于热平衡，则 `A` 与 `B` 也处于热平衡。

它把“热平衡”关系提升为一个可传递的等价关系，从而允许我们为每个等价类赋予一个统一温度标记。第零定律本身不直接给出热量守恒或熵增方向，但它为“温度可定义、可比较、可测量”提供逻辑基础。

## R02

工程与建模意义：

- 它保证温度不是局部偶然量，而是系统状态可比较的全局标量；
- 温度计之所以可用，依赖的正是“与温度计平衡即同温”的传递结构；
- 在代码里可转化为图上的等价类构造问题：由成对实验关系推出整体分类。

本条目目标是把第零定律做成一个可运行 MVP：
通过成对接触实验推断热平衡关系，再用并查集构造热平衡类，并验证传递性。

## R03

`demo.py` 的输入输出（无交互）：

- 输入（脚本内固定参数）：
1. 一组系统（`A` 到 `H`）及其合成真温度；
2. 接触导热系数 `conductance`、测量噪声幅值 `noise_amplitude`；
3. 热平衡判据阈值 `flux_tol`（`|heat_flux| <= flux_tol` 视为平衡）。

- 输出：
1. 所有成对实验表（真温差、测得热流、预测平衡与真值标签）；
2. 推断得到的热平衡等价类表（成员、温度跨度、温标估计误差）；
3. 分类指标（precision/recall/accuracy）与传递性检查计数；
4. 断言通过后打印 `All checks passed.`。

## R04

MVP 使用的物理/算法模型：

1. 接触热流近似：`q_ij = k * (T_i - T_j) + eps_ij`；
2. 判定规则：`|q_ij| <= q_tol` 则预测 `i ~ j`（热平衡）；
3. 温差真值阈值：`|T_i - T_j| <= q_tol / k`；
4. 将 `~` 关系边输入并查集（Union-Find）得到等价类；
5. 在每个等价类上用温度计原始读数均值做类温度估计；
6. 用两点标定（273.15K 与 373.15K）将原始读数映射回 Kelvin。

说明：脚本没有调用黑盒热力学求解器，平衡关系、等价类推断和标定都在源码里逐步展开。

## R05

复杂度（`n` 个系统）：

- 需要处理所有成对实验，数量 `m = n(n-1)/2`；
- 构图与并查集合并总复杂度约为 `O(m * alpha(n))`（近似线性）；
- DataFrame 汇总也是 `O(m)`；
- 总体时间复杂度 `O(n^2)`，空间复杂度 `O(n^2)`（保存成对结果）。

对于教学型或小规模实验集，这个复杂度非常轻量。

## R06

`demo.py` 的最小闭环：

- 闭环 A（实验生成）：`run_all_pair_measurements` 生成所有成对接触数据；
- 闭环 B（关系推断）：`infer_equilibrium_classes` 用并查集构建热平衡类；
- 闭环 C（定律验证）：`verify_zeroth_transitivity` 检查传递性三元组；
- 闭环 D（温标赋值）：按类平均温度计读数并做两点标定得到数值温度。

## R10

正确性直觉：

1. 第零定律关注的是关系结构而非能量守恒；
2. 先通过成对热流阈值建立“是否平衡”的边；
3. 并查集把这些边闭包成等价类，本质上在实现自反/对称/传递结构；
4. 一旦 `A` 与 `C` 同类、`B` 与 `C` 同类，则 `A` 与 `B` 必同类；
5. 每个等价类可映射到一个温度值，形成可测量温标。

## R11

数值稳定性与鲁棒策略：

- 使用确定性噪声函数，避免随机波动导致复现实验困难；
- 阈值判定只做绝对值比较，避免符号混淆；
- 指标评估（precision/recall/accuracy）独立于类标号顺序；
- 断言门槛采用宽松但明确的容差（`>=0.99`、平均误差 `<0.08K`）。

## R12

关键参数及影响：

- `conductance`：把温差映射为热流的比例系数，越大越易区分不同温度；
- `flux_tol`：平衡判定阈值，越大则更容易把两体判为平衡；
- `noise_amplitude`：测量噪声强度，越大越可能导致错判；
- 样本温度分布：类内间距越小、类间间距越大，分类越稳定。

调参建议：

- 先固定 `conductance`，扫描 `flux_tol` 看误判曲线；
- 逐步增大 `noise_amplitude`，观察传递性与指标何时退化；
- 若真实数据偏噪，可先做重复测量取中位数再判定。

## R14

常见失效模式：

1. 阈值过宽导致不同温度簇被错误合并；
2. 阈值过窄导致同温簇被错误拆分；
3. 把热流方向（正负）误认为平衡判据，忽略了“接近 0”才关键；
4. 温度计标定点选错，导致类温度整体偏移；
5. 数据只覆盖稀疏边时，关系推断可能不稳定。

## R18

`demo.py` 源码级算法流程（9 步）：

1. `main` 固定实验参数（`conductance/flux_tol/noise_amplitude`）和系统温度样本。  
2. `run_all_pair_measurements` 枚举所有系统对，调用 `measure_pair` 生成热流观测。  
3. `measure_pair` 用 `q_ij = k*(T_i-T_j)+eps_ij` 计算热流，并按阈值输出平衡预测。  
4. `infer_equilibrium_classes` 把预测为平衡的边喂给 `UnionFind.union`，得到连通等价类。  
5. `verify_zeroth_transitivity` 在类映射上逐个检查三元组 `(A,B,C)` 的传递性条件。  
6. `build_pair_frame` 生成实验明细表，同时按 `|delta_T| <= flux_tol/k` 生成合成真值标签。  
7. `classification_metrics` 计算 `TP/FP/FN/TN` 与 `precision/recall/accuracy`。  
8. `build_class_frame` 对每个等价类聚合温度计读数，调用 `calibrate_kelvin_from_raw` 估计类温度。  
9. `main` 输出三张结果表并执行断言（高精度分类、传递性成立、类温估计误差受控）。

说明：第三方库只用于数值与表格展示（`numpy/pandas`）。第零定律相关的关系推断、等价类构造和标定流程均在源码中显式实现，不是黑盒函数调用。
