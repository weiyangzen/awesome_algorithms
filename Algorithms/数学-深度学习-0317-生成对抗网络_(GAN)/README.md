## R01

生成对抗网络（Generative Adversarial Network, GAN）是通过“博弈”学习数据分布的生成模型。它包含两个神经网络：

- 生成器 `G(z)`：把随机噪声 `z` 映射为伪样本 `x_fake`。
- 判别器 `D(x)`：输出样本来自真实分布的概率（或 logits）。

训练目标是让 `G` 生成越来越逼真的样本，同时让 `D` 学会识别真假，最终达到动态平衡。

## R02

经典 GAN 的极小极大目标可写为：

`min_G max_D V(D, G) = E_{x~p_data}[log D(x)] + E_{z~p_z}[log(1 - D(G(z)))]`

本目录 MVP 使用非饱和生成器损失（更稳定）：

- 判别器损失：`L_D = BCE(D(x_real), 1) + BCE(D(x_fake), 0)`
- 生成器损失：`L_G = BCE(D(G(z)), 1)`

其中 `BCE` 在代码中通过 `BCEWithLogitsLoss` 实现。

## R03

GAN 要解决的问题是：在没有显式似然函数的情况下，直接从随机噪声采样并生成贴近真实分布的新样本。

相比“显式密度建模”的方法，GAN 的核心特点是：

- 不要求写出 `p(x)` 的解析形式；
- 通过判别器作为“可学习损失函数”驱动生成器改进；
- 训练质量高度依赖对抗过程的稳定性。

## R04

本任务 MVP 的输入与输出：

输入：

- 真实样本：一维双峰高斯混合分布（`n_samples=2048`）
- 先验噪声：`z ~ N(0, I)`，维度 `latent_dim=4`
- 网络：两层 MLP 生成器与判别器

输出：

- 训练后的 `Generator` 参数
- 生成样本 `x_fake`
- 统计对比（均值、方差、分位数）与分布距离指标（Wasserstein、KS、直方图 L1）

## R05

本实现选择 Vanilla GAN（BCE 对抗目标），理由：

- 结构最小、实现直接，适合算法入门与源码追踪。
- 不依赖外部高层训练框架，能完整展示对抗更新细节。
- 在 1D toy 数据上可快速看到“从噪声到双峰分布”的学习过程。

附加稳定化措施：

- 对真实标签使用 `0.9`（one-sided label smoothing），减轻判别器过强导致的梯度问题。

## R06

训练流程（对应 `demo.py`）：

1. 固定随机种子，构造真实数据分布。
2. 初始化 `Generator` 与 `Discriminator`。
3. 对每个 mini-batch，先更新 `D`：
   - 用真实样本和 `G(z)` 生成的伪样本计算 `L_D`。
4. 再更新 `G`：
   - 重新采样 `z`，最小化 `L_G = BCE(D(G(z)), 1)`。
5. 重复多个 epoch，记录 `d_loss` 与 `g_loss`。
6. 训练后从噪声采样，生成新样本并评估分布贴合度。

## R07

复杂度粗略分析（设样本数 `N`、epoch 数 `E`、生成器/判别器单次前向成本分别为 `C_G/C_D`）：

- 训练复杂度约为：`O(E * N * (C_G + C_D))`（常数项含反向传播）
- 采样复杂度约为：`O(M * C_G)`（生成 `M` 个样本）

在本 1D MLP 场景下，复杂度和内存都很低，可在 CPU 快速运行。

## R08

GAN 的典型优缺点：

优点：

- 采样速度快（一次前向即可生成样本）。
- 可生成高保真样本（在复杂任务中潜力高）。
- 生成器学习的是隐式分布映射，表达能力强。

缺点：

- 训练不稳定，易出现震荡或模式崩塌（mode collapse）。
- 损失值与感知质量不总是单调对应。
- 超参数对结果敏感，需要较多调参经验。

## R09

适用场景：

- 图像、语音、时序等生成任务的原型探索。
- 需要快速采样生成数据的离线场景。
- 希望研究“对抗训练机制”而不仅是单模型拟合。

不太适用：

- 对训练稳定性要求极高但算力/调参预算有限的项目。
- 需要显式似然评估或严格概率解释的任务。

## R11

与相邻生成模型对比：

- 对比 VAE：GAN 往往样本更锐利，但训练稳定性通常更差。
- 对比扩散模型：GAN 采样更快；扩散模型通常训练更稳、覆盖模式更充分。
- 工程常见折中：用 GAN 追求快速高质量采样，用扩散追求稳定和覆盖。

## R12

工程实现注意点：

- 判别器过强会导致生成器梯度变弱，可通过标签平滑、学习率平衡缓解。
- 更新节奏需控制（本例每个 batch 更新一次 `D`、一次 `G`）。
- 训练时 `x_fake` 在判别器更新阶段要 `detach()`，避免误更新生成器。
- 指标不应只看损失，需结合分布距离和样本统计。

## R13

## R15

结果解读建议：

- `d_loss` 和 `g_loss` 不需要单调下降，关键是是否在合理区间内稳定振荡。
- 看 `Distribution Summary`：`mean/std/q10/q50/q90` 越接近真实分布越好。
- 看 `wasserstein_1` 与 `hist_l1`：越小表示分布更接近。
- 看 `positive_mode_fraction_gap`：越小表示对双峰左右模式的覆盖更均衡。

## R16

常见失败模式与排查：

- 模式崩塌（只生成单峰）：
  - 尝试减小判别器学习率、增加生成器容量或调整训练轮数。
- 损失剧烈震荡：
  - 检查学习率是否过大，或加入更稳的变体（WGAN-GP、谱归一化）。
- 判别器过快收敛：
  - 增强标签平滑、限制判别器更新强度。
- 指标很差：
  - 先确认数据管线和 `detach()` 是否正确，再调参。

## R17

- 从 1D 扩展到 2D 点云，再到图像（DCGAN/StyleGAN 风格卷积结构）。

## R18

本实现的源码级算法流（8 步）：

1. `make_toy_data` 生成真实分布样本 `x_real`（双峰高斯），作为对抗训练目标分布。
2. `Generator` 定义映射 `z -> x_fake`，`Discriminator` 定义映射 `x -> logit`。
3. `train_gan` 在每个 batch 先采样 `z`，生成 `x_fake.detach()`，计算 `d_loss_real + d_loss_fake` 更新判别器。
4. 同一 batch 再重新采样 `z`，计算 `g_loss = BCE(D(G(z)), 1)` 更新生成器（非饱和目标）。
5. 训练过程中持续记录 `epoch/d_loss/g_loss`，并定期打印，观察对抗博弈是否稳定。
6. `generate_samples` 用训练后的生成器从高斯噪声一次前向采样得到 `x_fake`。
7. `evaluate_distribution` 计算统计表与距离指标（Wasserstein、KS、`hist_l1`、双峰占比偏差）。
8. `run_sanity_checks` 做基础质量门禁（有限值、非塌缩方差、距离阈值），保证 MVP 输出可用于验证。

该流程完整展开了 GAN 的“对抗目标定义 -> 交替优化 -> 采样评估”闭环，没有将核心训练逻辑交给黑盒 API。
