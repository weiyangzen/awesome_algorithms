## R01

朗之万动力学（Langevin Dynamics）用随机微分方程描述粒子在热浴中的演化：

- 确定性部分来自势能对应的力（保守动力学）；
- 耗散部分来自摩擦（能量流向热浴）；
- 随机部分来自热噪声（热浴回馈涨落）。

本条目实现一维欠阻尼谐振子模型，并用 BAOAB 分裂积分器进行离散推进，验证其平衡统计是否符合正则系综理论。

## R02

该算法在计算物理中的核心价值是把“热涨落 + 阻尼 + 势场”统一到一个可模拟框架中：

- 分子动力学中常用作恒温器（thermostat）原型；
- 可用来构造与校验采样器，检验是否能重现目标平衡分布；
- 相比纯哈密顿动力学，它能直接模拟与环境耦合下的统计行为。

本 MVP 目标是给出可运行、可核验、非黑盒的最小实现。

## R03

4. 自动断言通过后打印 `All checks passed.`。

## R04

连续模型（1D 欠阻尼 Langevin）：

`dx = v dt`

`dv = -(k/m)x dt - gamma*v dt + sqrt(2*gamma*kBT/m) dW_t`

其中 `U(x)=0.5*k*x^2`，`F(x)=-k*x`。

理论平衡矩（正则系综）：

- `<x>=0, <v>=0`
- `<x^2>=kBT/k`
- `<v^2>=kBT/m`
- `<U>=0.5*k<x^2>=0.5*kBT`
- `<K>=0.5*m<v^2>=0.5*kBT`

离散推进采用 BAOAB（每步 `dt`）：

1. `v <- v + (dt/2m) F(x)`
2. `x <- x + (dt/2) v`
3. `v <- c1*v + c2*xi`，`c1=exp(-gamma*dt)`，`c2=sqrt((kBT/m)(1-c1^2))`
4. `x <- x + (dt/2) v`
5. `v <- v + (dt/2m) F(x)`

`xi ~ N(0,1)`，该步骤显式体现涨落-耗散配对，而非调用黑盒 SDE 求解器。

## R05

设 `N = n_steps`，`M = n_trajectories`：

- 时间复杂度：`O(N*M)`；
- 空间复杂度：`O(M)`（只保留当前 `x,v`，不存全轨迹）；
- 额外统计与快照开销：`O(N/sample_stride)`，相对主项可忽略。

## R06

MVP 的三条核心链路：

- 链路 A（动力学推进）：按 BAOAB 逐步更新 `x,v`；
- 链路 B（统计累积）：在 `burn_in` 后累计一阶和二阶矩；
- 链路 C（理论核验）：将经验矩与解析平衡矩比较并断言误差阈值。

## R07

优点：

- 物理模型、离散格式、验证指标一一对应，易审计；
- BAOAB 对该类系统数值稳定性较好，且实现简洁；
- 使用并行轨道平均，有限步数下统计误差较小。

局限：

- 未展示复杂势能面或约束系统下的高级积分细节。

## R10

正确性直觉：

1. 连续模型的平衡分布已知，且二阶矩有解析值；
2. BAOAB 将确定性和随机项分开处理，OU 热浴子步保持噪声-阻尼一致性；
3. 多轨道长时间平均应收敛到系综平均；
4. 若 `<x^2>, <v^2>, <U>, <K>` 同时匹配理论，说明实现链路基本正确。

## R11

数值稳定策略：

- 使用 BAOAB 而非最朴素显式欧拉，减小离散偏差；
- 使用 `burn_in_steps` 丢弃初始非平衡区间；
- 通过较大的 `n_trajectories` 降低统计噪声；
- 固定随机种子确保复现，便于回归测试；
- 对参数做正值检查，防止非物理输入导致发散。

## R12

关键参数及影响：

- `dt`：过大将放大离散误差，过小则计算成本增大；
- `gamma`：控制阻尼强度和热浴耦合快慢；
- `spring_k`：决定位置方差理论值 `kBT/k`；
- `n_steps` 与 `burn_in_steps`：影响平衡采样充分性；
- `n_trajectories`：影响统计误差。

调参建议：

1. 先固定物理参数，逐步减小 `dt` 观察矩统计收敛；
2. 再增大 `n_trajectories` 或 `n_steps-burn_in_steps` 压低方差；
3. 若断言偶发失败，优先增加样本量而非放宽阈值。

## R14

常见失效模式：

1. `dt` 过大导致离散误差显著，统计偏离理论；
2. `n_steps` 太少或 `burn_in` 不足，样本仍处于过渡态；
3. `n_trajectories` 太小，采样噪声大引发误判；
4. 参数非法（负质量、负温度、负阻尼）造成模型无意义；
5. 将噪声幅度写错（未满足涨落-耗散关系）会系统性偏差。

## R18

`demo.py` 源码级算法流程（8 步）：

1. `main` 构造默认 `LangevinConfig`，并进入 `run_baoab`。  
2. `validate` 检查质量、温度、阻尼、步长、样本规模等合法性，提前阻断无效参数。  
3. 初始化 `x,v` 与随机数发生器；预计算 `exp(-gamma*dt)` 和噪声系数 `sqrt((kBT/m)(1-c1^2))`。  
4. 在每个时间步执行 BAOAB 子步：半 kick、半 drift、OU 热浴、半 drift、半 kick。  
5. 过了 `burn_in_steps` 后累计 `sum_x,sum_v,sum_x2,sum_v2`，形成系综平均估计。  
6. 按 `sample_stride` 记录探针轨道和当前系综二阶矩，生成可读快照表。  
7. 仿真结束后计算经验矩、能量及误差表，并与 `theoretical_moments` 的解析值逐项对比。  
8. `main` 打印结果并执行断言（均值接近 0，二阶矩相对误差 < 5%），全部通过则报告成功。

第三方库边界：

- `numpy` 仅用于数组运算和高斯随机数；
- `pandas` 仅用于结果表格展示；
- Langevin 动力学的积分流程、统计累积与正确性校验全部在源码中显式实现，不依赖黑盒 SDE 求解器。
