# 强连通分量 (SCC) - Tarjan算法

- UID: `CS-0031`
- 学科: `计算机`
- 分类: `图算法`
- 源序号: `41`
- 目标目录: `Algorithms/计算机-图算法-0041-强连通分量_(SCC)_-_Tarjan算法`

## R01

强连通分量（Strongly Connected Component, SCC）定义在有向图中：

- 若一个顶点集合中任意两点 `u, v` 均互相可达（`u -> v` 且 `v -> u`），则该集合强连通；
- 极大强连通子图称为一个 SCC。

Tarjan 算法是求 SCC 的经典线性算法，特点是只需一次 DFS 即可完成分量提取。

## R02

本条目要解决的问题：

- 输入：有向图 `G=(V,E)`，顶点编号 `0..n-1`，边列表 `[(u,v), ...]`；
- 输出：
1. 所有 SCC 的顶点集合；
2. 每个顶点所属 SCC 编号；
3. DFS 发现序（`discovery_time`）与 low-link 值；
4. 缩点图（condensation DAG）边集合与拓扑序。

`demo.py` 使用固定样例，不依赖交互输入。

## R03

Tarjan 的核心理论：

1. DFS 访问每个顶点时赋予一个递增时间戳 `disc[u]`；
2. 维护 `low[u]`：从 `u` 出发，经 DFS 树边 + 至多一条回边，能够回到的最小 `disc`；
3. 用栈维护“当前 DFS 路径上尚未归属分量”的顶点集合；
4. 当且仅当 `low[u] == disc[u]` 时，`u` 是一个 SCC 根，从栈顶弹出直到 `u`，得到一个完整 SCC。

该方法把“分量边界识别”嵌入一次 DFS 回溯过程。

## R04

本 MVP 的算法流程：

1. 构建有向图邻接表；
2. 初始化 `disc`、`low`、`on_stack`、`stack`、`time`；
3. 对每个未访问顶点执行 DFS；
4. 在 DFS 中更新 `disc[u]`、`low[u]`，并将 `u` 入栈；
5. 遍历边 `u -> v`：
   - 若 `v` 未访问，递归后用 `low[v]` 回传更新 `low[u]`；
   - 若 `v` 仍在栈中，用 `disc[v]` 更新 `low[u]`；
6. 若 `low[u] == disc[u]`，连续出栈组成一个 SCC；
7. 对 SCC 做稳定排序，生成顶点到分量映射；
8. 扫描原边构建缩点边，并对缩点 DAG 做 Kahn 拓扑排序。

## R05

设 `V=|V|`，`E=|E|`：

- 时间复杂度：`O(V + E)`；
- 空间复杂度：`O(V + E)`（邻接表、DFS 状态数组、辅助栈）。

Tarjan 与 Kosaraju 同为线性复杂度，但 Tarjan 不需要显式构建反图。

## R06

`demo.py` 的核心样例（Case-1）：

- `0 -> 1 -> 2 -> 0` 构成 SCC-A；
- `3 -> 4 -> 5 -> 3` 构成 SCC-B；
- `6 <-> 7` 构成 SCC-C；
- 再加边 `2 -> 3`、`5 -> 6` 形成分量链。

期望输出：

- SCC：`{0,1,2}`、`{3,4,5}`、`{6,7}`；
- 缩点边：`A -> B`、`B -> C`。

## R07

优点：

- 一次 DFS 即可求 SCC；
- 无需反图，状态收敛在单次遍历中；
- low-link 值可直接用于定位回路结构。

局限：

- 递归实现在超深图上受 Python 递归深度限制；
- 对高频动态图（持续增删边）并非最优；
- 实现细节较 Kosaraju 更容易写错（尤其是栈内判定）。

## R08

前置知识与依赖：

- 有向图 DFS、回边概念；
- 栈与数组状态维护；
- Python 3.10+。

MVP 依赖策略：

- SCC、缩点、拓扑排序均手写实现，不依赖 `networkx` 等黑盒；
- 仅使用 `numpy` 组织固定测试边数据。

## R09

适用场景：

- 模块依赖环检测（编译/构建系统）；
- 工作流状态图回路分析；
- 图数据库中“互相可达团簇”提取。

不适用场景：

- 在线实时维护 SCC 的动态图系统；
- 目标是最短路、最小生成树、最大流等不同图任务。

## R10

正确性直觉（基于栈不变式）：

1. 栈中只保留“已发现但尚未归属任何 SCC”的顶点；
2. `low[u]` 始终表示 `u` 能回到的最早祖先发现时间；
3. 若 `u` 能通过后代回到更早祖先，则 `low[u] < disc[u]`，说明 `u` 不是 SCC 根；
4. 当 `low[u] == disc[u]` 时，`u` 及其栈顶连续可达段无法再回到更早祖先，恰形成一个极大强连通块；
5. 逐根弹栈得到的各 SCC 互不重叠且覆盖全部顶点。

## R11

实现细节与防坑：

- 只有当 `v` 仍在栈中时，边 `u -> v` 才参与 low-link 回边更新；
- 不可用“已访问”替代“在栈中”，否则会错误跨分量更新 `low`；
- 弹栈时必须持续到根 `u`，不能只弹一个顶点；
- 输入边端点需做范围校验，防止越界；
- 为了测试稳定性，分量内部和分量列表都做确定性排序。

## R12

工程成本评估：

- 代码规模小，百行级可维护；
- 调试抓手清晰：`disc`、`low`、`stack`、分量弹栈时机；
- 测试应覆盖：
1. 多 SCC 链式图；
2. 非连通图与孤立点；
3. DAG（每个点单独成 SCC）。

## R13

算法性质：

- 精确算法（非近似）；
- 线性复杂度，适合静态大图；
- 可解释性强（可追踪每个点的 `disc/low`）；
- 输出可直接用于缩点 DAG 的后续 DP/拓扑分析。

## R14

常见失效模式与防护：

- 失效：对任意已访问顶点都更新 `low`，导致跨分量串联。  
  防护：仅当 `on_stack[v]` 为真时才按回边更新。

- 失效：忘记在出栈时同步 `on_stack[w]=False`。  
  防护：弹栈和状态清理写在同一循环体。

- 失效：`low[u] == disc[u]` 时只弹一个点，分量被拆碎。  
  防护：`while True` 直到弹到 `u` 为止。

- 失效：输出顺序受邻接插入顺序影响，测试抖动。  
  防护：统一排序后再对比断言。

## R15

实践建议：

- 先核对 SCC 集合，再核对缩点边，定位更快；
- 调试时打印 `disc` 和 `low`，可直接看出根判定是否正确；
- 大规模深链图建议改显式栈 DFS，避免递归限制；
- 若接入业务系统，建议把顶点 ID 映射回业务对象名再输出。

## R16

相关算法与扩展：

- Kosaraju：两次 DFS + 反图；
- Gabow：双栈 SCC 算法；
- 无向图中的双连通分量与有向图 SCC 定义不同；
- SCC 缩点后可做 DAG 动态规划、关键路径与可达性压缩分析。

## R17

本目录 `demo.py` 的主要函数：

- `build_directed_graph`：构建邻接表并校验输入边；
- `tarjan_scc`：单次 DFS 求 SCC，同时输出 `disc/low`、缩点边；
- `topological_sort_dag`：对缩点图做 Kahn 拓扑排序；
- `assert_case`：校验 SCC 与缩点结果；
- `run_demo_cases`：运行 3 个固定样例并断言通过。

## R18

`demo.py` 的源码级算法流（9 步，非黑盒）：

1. `run_demo_cases` 用 `numpy` 构造固定边数组并转为 `list[tuple[int,int]]`。  
2. `tarjan_scc` 调用 `build_directed_graph` 创建邻接表与规范化边列表。  
3. 初始化 `disc`、`low`、`stack`、`on_stack` 与 DFS 时间戳。  
4. 对每个未访问顶点执行 `dfs(u)`：写入 `disc[u]=low[u]=time` 并入栈。  
5. 遍历每条 `u -> v`：未访问则递归并用 `low[v]` 回传；在栈中则用 `disc[v]` 更新 `low[u]`。  
6. 当 `low[u] == disc[u]` 时，从栈顶持续弹出到 `u`，形成一个原始 SCC。  
7. 对原始 SCC 做稳定排序并重映射分量编号，得到 `component_id_of_vertex`。  
8. 扫描原图边，提取跨分量边构成缩点图，再调用 `topological_sort_dag` 求拓扑序。  
9. `assert_case` 对 SCC 集合、缩点边和顶点覆盖率做断言，保证实现正确。

第三方库拆解说明：

- `numpy` 只用于样例数据组织；
- SCC 求解、low-link 更新、缩点构建、拓扑排序均为源码手写实现。
