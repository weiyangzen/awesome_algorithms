# 奥本海默-沃尔科夫极限 (Oppenheimer-Volkoff Limit)

- UID: `PHYS-0060`
- 学科: `物理`
- 分类: `天体物理`
- 源序号: `60`
- 目标目录: `Algorithms/物理-天体物理-0060-奥本海默-沃尔科夫极限_(Oppenheimer-Volkoff_Limit)`

## R01

奥本海默-沃尔科夫极限（OV limit）是指在广义相对论静力平衡下，中子星可稳定支撑的最大质量。  
当中心密度持续升高时，质量-中心密度曲线 `M(rho_c)` 先上升后下降，峰值点对应“稳定支”的终点；再往高密度方向通常进入径向不稳定区并趋向坍缩。

## R02

本条目目标是给出一个可运行、可审查的最小数值实现：

- 用 TOV 方程积分单个恒星结构；
- 扫描一组中心密度得到 `M-R` 序列；
- 用曲线峰值估计 OV 极限（`M_max`）；
- 通过第二积分器交叉计算，验证数值结果稳健。

## R03

MVP 建模假设：

- 球对称、静态、各向同性压力流体；
- 采用广义相对论 TOV 方程，不考虑自转、磁场、双星潮汐、温度梯度；
- 方程状态使用相对论多项式近似：`P = K rho^Gamma`，默认 `Gamma=2.0`；
- 以 cgs 单位积分：`rho[g/cm^3]`，`P[dyn/cm^2]`，`m[g]`，`r[cm]`。

该设定不追求真实核物质 EOS 细节，而是聚焦“OV 极限的算法闭环”。

## R04

核心方程：

1. 方程状态（EOS）

`P = K rho^Gamma`

`rho(P) = (P/K)^(1/Gamma)`

2. TOV 质量方程

`dm/dr = 4 pi r^2 rho`

3. TOV 压力梯度方程

`dP/dr = -G (rho + P/c^2) (m + 4 pi r^3 P/c^2) / (r^2 (1 - 2Gm/(rc^2)))`

4. 紧致度

`C = 2GM/(Rc^2)`

5. OV 极限估计

对离散扫描点，取 `M_max = max_i M(rho_c,i)`，对应 `rho_c*`、`R*` 即近似极限点。

## R05

算法流程（高层）：

1. 设定 EOS 参数、中心密度扫描区间、积分容差与表面阈值；
2. 对每个 `rho_c` 构造中心初值 `(m0, P_c)`；
3. 调用 `solve_ivp` 积分 TOV ODE，事件函数在 `P -> P_surface` 终止；
4. 提取该模型的 `(M, R, compactness)`；
5. 形成 `M-R-rho_c` 数据表；
6. 在主积分器结果中定位 `M_max`；
7. 用次积分器重复扫描并与主结果做 `MAE/R^2` 比较；
8. 输出表格与断言，给出是否通过。

## R06

复杂度分析（`N` 为中心密度扫描点数，`S` 为单次积分平均步数）：

- 单模型积分复杂度约 `O(S)`；
- 整体扫描复杂度约 `O(N*S)`；
- 主+次积分器双跑约 `O(2NS)`；
- 空间复杂度约 `O(N)`（存储每个模型的聚合结果）。

默认 `N=24`，可在秒级完成。

## R07

正确性直觉：

- TOV 方程给出“引力压缩 vs 压力支撑”的相对论平衡条件；
- 随 `rho_c` 增大，低密区通常增加总质量，高密区相对论效应增强并触发质量回落；
- `M(rho_c)` 峰值对应稳定性转折（常见判据 `dM/drho_c = 0`）；
- 因此通过密度扫描找峰值，是 OV 极限的可执行近似算法。

## R08

数值稳定策略：

- 从小但非零半径 `r0=1 cm` 起步，避免 `r=0` 奇点；
- 用事件函数在 `P_surface` 终止，避免在 `P -> 0` 附近出现步长塌缩；
- 当 `1 - 2Gm/(rc^2)` 过小（接近视界）时保护返回，避免分母灾难；
- 主积分器 `DOP853` + 次积分器 `RK45` 交叉对照，减少“单求解器偶然正确”。

## R09

`demo.py` 使用的最小工具栈：

- `numpy`：中心密度几何扫描、差分和向量统计；
- `scipy`：`solve_ivp` 实现 TOV 常微分方程积分与事件终止；
- `pandas`：构建并打印质量-半径结果表；
- `scikit-learn`：`mean_absolute_error`、`r2_score` 做跨积分器一致性评估；
- `PyTorch`：对质量序列做张量差分并统计正/负斜率段，验证转折点存在。

## R11

关键输出指标：

- `M_max (M_sun)`：OV 极限质量估计；
- `rho_c*`：达到 `M_max` 的中心密度；
- `R* (km)`：峰值质量对应半径；
- `compactness`：峰值点紧致度 `2GM/(Rc^2)`；
- `mass_mae_msun`, `mass_r2`：主次积分器质量曲线一致性；
- `radius_mae_km`, `radius_r2`：半径曲线一致性；
- `n_positive_deltas`, `n_negative_deltas`：质量序列差分符号统计。

## R12

默认参数（`TOVConfig`）：

- EOS：`Gamma=2.0`, `K=1.5e5`；
- 中心密度扫描：`rho_c in [3e14, 1e16] g/cm^3`，`n_models=24`；
- 表面阈值：`rho_surface=1e8 g/cm^3`（转为 `P_surface` 终止）；
- 积分区间：`r in [1 cm, 3e7 cm]`；
- 步长与容差：`max_step=2e4 cm`, `rtol=2e-6`, `atol=1e-6`；
- 求解器：主 `DOP853`，次 `RK45`。

在该参数下，MVP 通常得到约 `2.0 M_sun` 的峰值质量。

## R13

理论保证类型：

- 近似比保证：N/A（非组合优化问题）；
- 随机成功率保证：N/A（确定性数值积分流程）。

本实现提供的可执行保证：

- 峰值位置必须是内部点（非扫描边界）；
- 峰值质量/半径/紧致度必须落在物理合理区间；
- 质量曲线在峰值前后分别满足“近似上升/下降”断言；
- 主次积分器一致性指标必须达标，否则脚本失败。

## R14

常见失效模式：

1. EOS 参数不当，导致曲线始终单调，扫描不到峰值；
2. 扫描区间过窄，峰值落在边界点，误判 OV 极限；
3. 终止压力设得过高/过低，半径和质量产生系统偏差；
4. 忽略 `1 - 2Gm/(rc^2)` 分母保护，导致数值发散；
5. 把该简化 EOS 的结果当作真实核物质精确上限。

## R18

`demo.py` 源码级算法流程（8 步）：

1. `TOVConfig` 定义 EOS 参数、密度扫描区间、事件终止阈值、积分器与容差。  
2. `pressure_from_density` / `density_from_pressure` 实现 `P <-> rho` 转换，为 TOV RHS 提供闭式 EOS。  
3. `tov_rhs` 按 TOV 公式显式计算 `dm/dr` 与 `dP/dr`，并加入 `schwarzschild_factor` 防护。  
4. `make_surface_event` 构造 `P - P_surface = 0` 终止事件，避免在 `P -> 0` 附近数值病态。  
5. `solve_single_star` 对单个 `rho_c` 调 `solve_ivp`，提取终止点质量、半径和紧致度。  
6. `build_mass_radius_sequence` 在 `geomspace(rho_c)` 上批量求解，得到 `DataFrame` 质量-半径曲线。  
7. `estimate_ov_limit` 直接在离散曲线上取 `mass_msun` 峰值；`compare_solvers` 用 `MAE/R^2` 比较主次积分器结果；`analyze_turning_point_with_torch` 用差分符号统计验证“先升后降”。  
8. `main` 打印结果表，执行物理区间、单峰性与跨求解器一致性断言，最终给出“OV 极限估计通过/失败”。

第三方库未被当作黑盒“直接给出 OV 极限”：`scipy` 仅负责通用 ODE 积分；EOS、TOV RHS、峰值判定、稳定段判据和验收逻辑均在源码中逐步实现。
