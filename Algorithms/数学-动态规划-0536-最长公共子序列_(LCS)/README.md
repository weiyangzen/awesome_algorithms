# 最长公共子序列 (LCS)

- UID: `MATH-0536`
- 学科: `数学`
- 分类: `动态规划`
- 源序号: `536`
- 目标目录: `Algorithms/数学-动态规划-0536-最长公共子序列_(LCS)`

## R01

本条目实现 `Longest Common Subsequence (LCS)` 的最小可运行版本（MVP），目标是：
- 给定两个序列，计算它们的 LCS 长度；
- 回溯并返回一条具体的公共子序列；
- 同时返回该子序列在两个原序列中的下标位置；
- 用一个空间优化版本交叉校验长度，保证实现可验证且非黑箱。

## R02

问题定义（MVP 范围）：
- 输入：两个序列 `A` 与 `B`（可为字符串、列表或 1D `numpy.ndarray`）。
- 输出：
  - `length`：LCS 长度；
  - `subsequence`：一条公共子序列；
  - `indices_a`：`subsequence` 对应在 `A` 中的下标；
  - `indices_b`：`subsequence` 对应在 `B` 中的下标。

约束与说明：
- 子序列要求保持相对顺序，但不要求连续；
- LCS 可能不唯一，MVP 返回其中一条确定性解（由回溯 tie-break 决定）；
- 允许空序列输入。

## R03

动态规划建模：

设 `A` 长度为 `m`，`B` 长度为 `n`，定义
`dp[i][j] = A[:i]` 与 `B[:j]` 的 LCS 长度（`i=0..m, j=0..n`）。

状态转移：
- 若 `A[i-1] == B[j-1]`，则  
  `dp[i][j] = dp[i-1][j-1] + 1`
- 否则  
  `dp[i][j] = max(dp[i-1][j], dp[i][j-1])`

边界条件：`dp[0][*] = 0`, `dp[*][0] = 0`。  
答案为 `dp[m][n]`。

## R04

算法流程（MVP）：
1. 将输入统一转换为 token 列表，支持字符串、1D ndarray、一般序列。
2. 构造 `(m+1) x (n+1)` 的整型 DP 表并初始化为 0。
3. 双层循环填表，按 R03 的转移规则更新 `dp[i][j]`。
4. 从 `(m, n)` 开始回溯：
   - 若当前 token 相等，加入 LCS 并左上移动；
   - 否则比较 `dp[i-1][j]` 与 `dp[i][j-1]`，向较大者移动。
5. 反转回溯得到的 token 与下标，形成正向输出。
6. 再用双行滚动数组实现一次 `O(min(m,n))` 内存版本，只校验长度。
7. 检查重建结果是否确为两边的子序列并输出。

## R05

核心数据结构：
- `numpy.ndarray[int32] dp`：完整 DP 表，维度 `(m+1, n+1)`；
- `LCSResult(dataclass)`：
  - `length: int`
  - `indices_a: list[int]`
  - `indices_b: list[int]`
  - `subsequence: list[Any]`
- 两行 DP 校验器：
  - `prev`, `cur` 两个长度为 `min(m,n)+1` 的数组。

## R06

正确性要点：
- 最优子结构：若 `A[i-1] == B[j-1]`，最优解可拆为更短前缀最优解加 1；
- 无后效性：`dp[i][j]` 仅依赖更小下标状态，与达到状态的具体路径无关；
- 非匹配时取上/左最大值等价于“丢弃某一侧末尾 token 的最优选择”；
- 回溯沿着 DP 值保持不增且在匹配时取对角，保证得到合法公共子序列；
- 长度由完整表与双行版本一致性校验，降低实现错误风险。

## R07

复杂度分析：
- 完整 DP + 回溯：
  - 时间复杂度：`O(mn)`
  - 空间复杂度：`O(mn)`
- 双行长度校验版本：
  - 时间复杂度：`O(mn)`
  - 空间复杂度：`O(min(m,n))`

其中 `m=len(A)`，`n=len(B)`。

## R08

边界与异常处理：
- 任一序列为空：直接得到长度 `0`、空下标、空子序列；
- 两序列无公共元素：长度为 `0`；
- 输入为 `ndarray` 但维度不是 1D：抛出 `ValueError`；
- 输入既不是字符串也不可迭代：抛出 `ValueError`；
- 重复元素场景可正常处理，回溯返回其中一条合法 LCS。

## R09

MVP 取舍说明：
- 只依赖 `numpy + 标准库`，不引入重型框架；
- 不调用第三方 LCS 黑盒 API，核心状态转移和回溯均手写可读；
- 额外实现双行 DP 仅做长度交叉校验，提高可验证性；
- 暂不实现“所有 LCS 枚举”“编辑脚本输出”“超长序列并行化”等增强功能。

## R10

`demo.py` 模块职责：
- `to_token_list`：统一输入类型并做基础合法性检查；
- `lcs_dynamic_programming`：完整 DP 填表 + 回溯重建；
- `lcs_length_two_row`：空间优化版 LCS 长度；
- `is_subsequence`：验证输出序列合法性；
- `format_tokens`：将字符序列或一般序列转为可读文本；
- `run_case`：执行单个样例并打印结果、做断言；
- `main`：组织固定测试样例，无交互执行入口。

## R12

输出解读：

- `LCS length (two-row)`：空间优化版本长度；
- `LCS indices in A/B`：重建子序列在两边原序列中的位置；
- `LCS subsequence`：具体公共子序列；
- `Checks`：

  - `indices_well_formed=True`：回溯下标合法。

## R15

方法对比：
- 完整二维 DP：
  - 优点：可直接回溯具体子序列；
  - 缺点：内存开销 `O(mn)`。
- 双行滚动 DP：
  - 优点：空间降到 `O(min(m,n))`；
  - 缺点：默认只得到长度，重建路径较困难。
- 记忆化搜索（Top-down）：
  - 表达直观但在 Python 中递归开销更高；
  - 对大输入可能受递归深度限制。

## R18

源码级算法流（对应 `demo.py`，9 步）：
1. `main` 构造固定案例并逐一调用 `run_case`。  
2. `run_case` 先用 `to_token_list` 将两侧输入转成统一 token 列表。  
3. 调用 `lcs_dynamic_programming`，初始化 `(m+1) x (n+1)` 的 `dp` 全零表。  
4. 双层循环填表：相等走对角 `+1`，否则取上/左最大值。  
5. 填表完成后从右下角开始回溯：相等则收集 token 与双侧下标，不等则按 DP 值方向移动。  
6. 将回溯得到的逆序结果反转，得到正向 `subsequence / indices_a / indices_b`。  
7. 同一案例再调用 `lcs_length_two_row`，用滚动数组独立计算 LCS 长度。  
8. `run_case` 对比两种长度，并用 `is_subsequence` 验证重建序列确实属于两侧输入。  
9. 所有断言通过后打印结果，形成可复现、无交互的最小可验证 MVP。
