## R01

随机梯度下降（Stochastic Gradient Descent, SGD）是求解可微优化问题的经典一阶方法。

核心思想：
- 全量梯度下降每一步都使用全部样本计算梯度，计算成本高；
- SGD 每一步只使用一个样本或一个小批量（mini-batch）估计梯度；
- 用“低成本但有噪声”的梯度更新参数，从而在大规模数据上快速推进。

本条目 MVP 用 SGD 训练一个带 L2 正则的逻辑回归二分类器。

## R02

本条目问题定义（离散计算版）：

- 输入：
  - 训练集 `X_train in R^(n*d)`、标签 `y_train in {0,1}^n`；
  - 测试集 `X_test, y_test`；
  - 超参数：学习率 `lr0`、衰减 `lr_decay`、批大小 `batch_size`、轮数 `epochs`、正则系数 `lambda`。
- 输出：
  - 学到的参数 `w, b`；
  - 训练过程中若干日志点（`train_loss/test_loss/test_acc`）；
  - 终态检查结果（损失下降、精度达标、参数有限）。

## R03

MVP 的目标函数与更新公式：

- 逻辑回归概率：`p = sigmoid(Xw + b)`。
- 带 L2 正则的经验风险：

`L(w,b) = -mean(y*log(p) + (1-y)*log(1-p)) + (lambda/2)*||w||_2^2`

对 mini-batch `B` 的梯度估计：

`g_w = (X_B^T (p_B - y_B))/|B| + lambda*w`

`g_b = mean(p_B - y_B)`

SGD 更新：

`w <- w - eta_t * g_w`

`b <- b - eta_t * g_b`

`eta_t = lr0 / (1 + lr_decay * t)`。

## R04

时间复杂度（设样本数 `n`、特征维数 `d`、轮数 `E`）：

- 单个 mini-batch 梯度计算复杂度约 `O(batch_size * d)`；
- 每轮约处理 `n / batch_size` 个 batch，因此每轮是 `O(n*d)`；
- 总复杂度 `O(E*n*d)`。

相比全量梯度下降，SGD 在“每次参数更新”的成本上更低，且能更频繁地更新参数。

## R05

空间复杂度：

- 训练数据占 `O(n*d)`；
- 参数向量 `w` 与同维梯度占 `O(d)`；
- 单个 batch 视图/切片占 `O(batch_size*d)`；
- 日志仅 `O(E/report_every)`。

若数据流式读取，额外工作内存可近似视为 `O(d + batch_size*d)`。

## R06

一次 mini-batch 更新的微例：

假设当前某批次上已有：
- `g_w = [0.30, -0.10, 0.05]`
- `g_b = -0.20`
- `eta_t = 0.1`
- 当前参数 `w = [0.40, 0.20, -0.30], b = 0.05`

则更新后：
- `w_new = w - 0.1*g_w = [0.37, 0.21, -0.305]`
- `b_new = b - 0.1*g_b = 0.07`

这体现了 SGD 的本质：每个小批量都立即推动参数向降低该批次损失的方向移动。

## R07

SGD 重要性：

- 深度学习与大规模机器学习训练的基础优化器；
- 对海量样本，单步计算成本可控；
- 带噪声梯度有时能帮助跳出尖锐局部结构；
- 实现简单，便于与动量、Adam、学习率调度等机制组合。

## R08

理论基础（工程化摘要）：

1. 可微优化问题可用一阶泰勒近似得到局部下降方向。  
2. 若 `eta_t` 足够小，沿负梯度方向更新可使目标函数下降。  
3. 用样本/小批量梯度替代全量梯度时，得到无偏或近似无偏估计。  
4. 梯度噪声会带来波动，但在适当学习率策略下，期望上仍可收敛到稳定区域。  
5. L2 正则项通过惩罚大权重改善泛化并提升数值稳定性。

## R09

适用条件与局限：

- 适用于可微或次可微、可分批计算梯度的目标；
- 适用于样本规模大、全量梯度代价高的场景；
- 对学习率较敏感，过大易震荡/发散，过小收敛慢；
- SGD 轨迹通常非单调，单步损失可能上升；
- 在病态条件数问题上，原始 SGD 可能很慢，常需预处理或改进优化器。

## R10

正确性框架（本实现）：

1. 用 `logistic_loss_and_grads` 显式实现逻辑回归损失和梯度。  
2. 用 mini-batch 计算 `g_w, g_b`，与目标函数定义保持一致。  
3. 严格按 `theta <- theta - eta_t * grad` 更新。  
4. 每轮后在全训练集/测试集上重算指标，防止只看批次噪声。  
5. 运行结束后断言：
   - 训练损失下降；
   - 测试精度达到阈值；
   - 参数全为有限值。

## R11

误差与稳定性：

- mini-batch 梯度相对全量梯度存在方差，导致迭代曲线抖动；
- 学习率衰减 `eta_t = lr0/(1+decay*t)` 可缓和后期震荡；
- 特征标准化可显著改善条件数，减少更新方向畸变；
- L2 正则抑制权重过大，有助于稳定训练。

本 MVP 通过固定随机种子和保守超参数，使结果可复现且稳定。

## R12

性能视角：

- 主成本在 `X_B @ w` 和 `X_B.T @ err` 两次矩阵向量乘；
- NumPy 向量化减少 Python 循环开销；
- 与二阶法相比，SGD 单步便宜、内存需求小；
- 该实现优先保证可读性和算法透明，不引入复杂框架。

## R14

鲁棒性与常见失效模式：

- 学习率过大：可能出现数值爆炸或损失震荡不收敛；
- 批大小过小：梯度噪声过大，曲线抖动明显；
- 特征尺度差异大：下降方向被大尺度特征主导；
- 正则过强：欠拟合，精度上不去；
- 数据噪声过大：可分性下降，损失下界变高。

## R18

源码级算法流程拆解（`train_sgd` 主流程，8 步）：

1. 初始化：读取 `SGDConfig`，设置随机种子，令参数 `w=0, b=0`。  
2. 基线评估：在未训练参数上计算初始训练损失和测试精度。  
3. epoch 循环：每轮先对训练样本索引做随机打乱，避免固定顺序偏差。  
4. batch 循环：按 `batch_size` 取切片，形成 `X_B, y_B`。  
5. 梯度计算：调用 `logistic_loss_and_grads` 得到该批次 `g_w, g_b`。  
6. 学习率调度：按 `eta_t = lr0/(1+lr_decay*t)` 计算当前步长。  
7. 参数更新：执行 `w -= eta_t*g_w`, `b -= eta_t*g_b`，并做有限值检查。  
8. 轮末记录与验收：计算全量 `train_loss/test_loss/test_acc`，结束后断言“损失下降 + 精度达标”。

说明：这里没有调用 scikit-learn/PyTorch 的现成优化器；SGD 的核心计算链路（梯度估计、步长、更新）均可在源码中逐行追踪。
