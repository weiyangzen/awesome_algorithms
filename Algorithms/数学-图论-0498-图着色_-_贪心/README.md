# 图着色 - 贪心

- UID: `MATH-0498`
- 学科: `数学`
- 分类: `图论`

- 目标目录: `Algorithms/数学-图论-0498-图着色_-_贪心`

## R01

图着色（Graph Coloring）这里指**顶点着色**问题：

- 给定无向图 `G=(V,E)`；
- 为每个顶点分配一个颜色编号；
- 对每条边 `(u,v) in E`，要求 `color[u] != color[v]`。

本条目采用贪心策略：按给定顶点顺序逐个着色，每次给当前顶点分配“最小可用颜色”。

## R02

问题有两种常见形式：

- 判定：给定 `k` 种颜色，是否能合法着色？
- 优化：最少需要多少种颜色（色数，chromatic number）？

贪心法本质上是优化问题的启发式近似：能快速产出一个合法着色，但通常不保证最优色数。

## R03

本 MVP 的目标定位：

- 主算法：实现可复现的贪心顶点着色；
- 评估基线：提供小规模回溯精确色数，用于比较贪心与最优差距；
- 演示重点：同一张图在不同顶点顺序下，贪心用色数可显著不同。

## R04

贪心着色步骤（最小可用色规则）：

1. 选定一个顶点顺序 `order`；
2. 按顺序遍历顶点 `v`；
3. 收集已着色邻居使用的颜色集合 `used`；
4. 从 `0` 开始找不在 `used` 的最小颜色 `c`；
5. 令 `color[v]=c`；
6. 直到所有顶点完成。

该规则保证输出一定是合法着色，但颜色数量依赖 `order`。

## R05

复杂度分析（邻接矩阵实现）：

- 设顶点数为 `n`；
- 每个顶点着色时需扫描其邻接关系，最坏 `O(n)`；
- 共 `n` 个顶点，总时间 `O(n^2)`；
- 邻接矩阵空间 `O(n^2)`，颜色数组 `O(n)`。

## R06

关键不变量：

- 在处理到顺序位置 `i` 时，`order[:i]` 中顶点均已着色且满足边约束；
- 给下一个顶点分配颜色时，显式排除邻居已用色，不会破坏已有合法性；
- 因而算法结束时得到全局合法着色。

## R07

正确性与最优性的边界：

- 正确性：贪心一定返回合法着色（若输入是无向简单图）；
- 非最优性：贪心不保证最小颜色数；
- 顺序敏感：改变顶点顺序，结果可能从接近最优变成明显偏差。

## R08

演示样例选择 `Crown Graph`（皇冠图）：

- 该图是二分图，理论最优色数恒为 `2`；
- 但在某些交替顺序下，贪心会使用远多于 2 的颜色；
- 这能直观看到“贪心合法但不稳健”的特性。

本目录 `demo.py` 使用 `k=5` 的皇冠图（10 个顶点）做固定演示。

## R09

适用场景：

- 需要快速得到可行着色，而非严格最优；
- 图规模较大，精确算法开销不可接受；
- 可将结果用作后续改进算法（局部搜索、回溯剪枝）的初始上界。

不适用场景：

- 必须保证最小色数；
- 任务对最优性误差容忍度极低。

## R10

输入建模假设：

- 无向简单图（无自环、可容忍重复边但会自动去重）；
- 顶点编号范围 `0..n-1`；
- 邻接矩阵需方阵、对称、主对角线为 0。

`demo.py` 在构图和主流程中都做了相应合法性检查。

## R11

常见失效模式与应对：

- 顶点顺序不合法（遗漏/重复顶点）
  - 应对：运行前验证 `order` 是否为 `0..n-1` 的排列。
- 输入图不是无向简单图
  - 应对：构图函数与矩阵校验函数直接抛出异常。
- 误把贪心结果当最优
  - 应对：在小图上用精确回溯给出对照色数。

## R12

与精确算法的关系：

- 贪心提供快速上界（找到一个可行颜色数 `U`）；
- 精确回溯可在 `1..U` 之间判定最小可行 `k`；
- 实务上常见流程：`greedy upper bound + backtracking/SAT/ILP exact check`。

本 MVP 就实现了这条小型流程链。

## R13

MVP 使用的技术栈：

- `numpy`：邻接矩阵与度数计算；
- Python 标准库：`dataclasses`、`typing`。

没有调用图着色黑盒库，核心逻辑全部在 `demo.py` 源码中展开，便于教学与审计。

## R14

可复现性设计：

- 使用固定图实例（`k=5` 皇冠图）；
- 使用固定三种顺序（自然、交替、度数降序）；
- 并列时使用稳定规则（编号升序）；
- 关键结果通过断言验证，确保每次运行得到一致结论。

## R15

`demo.py` 模块划分：

- `build_undirected_adjacency`：构造无向邻接矩阵并做合法性检查；
- `greedy_coloring`：核心贪心着色；
- `is_proper_coloring`：合法性验证；
- `exact_chromatic_number_backtracking`：小规模精确色数基线；
- `build_crown_graph` / `interleaved_crown_order`：演示图与特定顺序；
- `main`：组织实验、打印报告与断言。

## R16

可扩展方向：

- 更强顶点顺序：DSATUR、largest-first + tie-break 策略；
- 后处理改进：Kempe chain / 局部重着色减少颜色数；
- 精确求解增强：分支限界、位运算剪枝、SAT/ILP 建模；
- 工程化：邻接表 + 位集降低常数开销。

## R18

`demo.py` 的源码级算法流程（8 步）：

1. `main` 调用 `build_crown_graph(5)` 构建固定皇冠图邻接矩阵，并生成三种顶点顺序。  
2. 对每个顺序调用 `greedy_coloring`：逐顶点读取邻居已用色，分配最小可用颜色。  
3. `greedy_coloring` 完成后由 `is_proper_coloring` 校验边约束，保证输出合法。  
4. 计算每个贪心结果的颜色数 `max(colors)+1`，形成“顺序 -> 用色数”映射。  
5. `exact_chromatic_number_backtracking` 先用“度数降序贪心”得到上界 `U`，再从 `k=1` 到 `U` 逐个做可着色判定。  
6. `can_color_with_k` 使用 DFS 回溯：对当前顶点试 `0..k-1` 色，若与已着色邻居冲突则剪枝。  
7. 第一次判定成功的 `k` 即精确色数；函数返回该 `k`、一组最优着色和搜索节点统计。  
8. `main` 汇总并打印报告，断言“交替顺序劣于最优、自然顺序等于最优”，最后输出 `All checks passed.`。
