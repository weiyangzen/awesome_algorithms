## R01

LOD 方法（Locally One-Dimensional Method，局部一维化方法）是一类经典算子分裂法。

核心思想：
- 把多维抛物型 PDE 的算子分解为若干一维方向算子；
- 每个时间步按方向依次做隐式子步（例如先 `x` 方向，再 `y` 方向）；
- 每个子步只需解一组一维三对角线性系统，显著降低实现与求解难度。

本条目使用二维热方程作为最小可运行示例。

## R02

本条目问题定义（离散计算版）：

- 方程：`u_t = kappa * (u_xx + u_yy)`，定义域 `(0,1)^2`。
- 边界：齐次 Dirichlet，边界上 `u=0`。
- 初值：`u(x,y,0)=sin(pi x)sin(pi y)`。
- 目标：
  - 用 LOD 分裂推进到 `t_end`；
  - 对比解析解 `u_exact(x,y,t)=exp(-2*pi^2*kappa*t)*sin(pi x)sin(pi y)`；
  - 输出相对 `L2` 误差和 `L2` 范数衰减曲线（采样点）。

## R03

MVP 采用的离散方案：

1. 空间离散：二维均匀网格内部点，二阶中心差分。  
2. 时间分裂（Lie 型 LOD）：

`(I - dt*kappa*Dxx) u* = u^n`  
`(I - dt*kappa*Dyy) u^(n+1) = u*`

其中 `Dxx`、`Dyy` 分别是一维二阶差分算子。

每个子步都会退化为大量独立三对角方程，示例代码使用 Thomas 算法显式求解。

## R04

时间复杂度（`n` 为每个方向内部网格数，`N_t` 为时间步数）：

- 单个子步：要解 `n` 个长度 `n` 的三对角系统，复杂度 `O(n^2)`；
- 每个时间步有两个子步（`x` 扫 + `y` 扫），仍为 `O(n^2)` 量级；
- 总复杂度 `O(N_t * n^2)`。

与直接解二维稠密系统相比，LOD 的线性代数成本更低且结构更规则。

## R05

空间复杂度：

- 主场变量 `u` 为 `n x n`，`O(n^2)`；
- 中间场 `u_star` 同阶，`O(n^2)`；
- 三对角系数和日志均为 `O(n)` 或 `O(N_t/report_every)`。

总空间复杂度为 `O(n^2)`。

## R06

单步结构微例（抽象演示）：

设 `n=4`，某时刻 `u^n` 是 `4x4` 内点矩阵。第 1 个子步（`x` 方向）对每一列求解

`A_x * u_star[:, j] = u^n[:, j]`，`j=0..3`

其中 `A_x` 为三对角矩阵，主对角 `1+2r`，副对角 `-r`，`r=kappa*dt/h^2`。

第 2 个子步（`y` 方向）对每一行求解

`A_y * u^(n+1)[i, :] = u_star[i, :]`，`i=0..3`

这样就把二维隐式更新拆成两组一维隐式更新，保持了“每步只解三对角”的结构优势。

## R07

LOD 的工程价值：

- 将多维耦合问题分解为方向可分问题，便于实现和优化；
- 对扩散型问题可获得良好稳定性（隐式子步）；
- 非常适合作为 ADI、Strang 分裂等更高阶分裂法的入门与基线；
- 在热传导、扩散、反应-扩散等问题中是常见基础组件。

## R08

理论基础（本条目对应）：

1. 原方程半离散后可写为 `u_t = (Lx + Ly)u`。  
2. 精确推进是 `exp(dt*(Lx+Ly))`。  
3. LOD（Lie 分裂）近似为 `exp(dt*Ly) exp(dt*Lx)`。  
4. 在离散实现中，`exp` 被相应隐式步近似，得到两次一维三对角求解。  
5. 该分裂是时间一阶精度，分裂误差主要由算子交换子项导致。

## R09

适用条件与局限：

- 适用：以方向二阶导为主的抛物型 PDE，规则网格场景尤其友好。
- 局限：
  - Lie 型 LOD 仅一阶时间精度；
  - 在强非线性、强耦合或复杂几何边界下需要改造；
  - 分裂顺序会引入方向偏置，误差比二阶 Strang 分裂更大。

## R10

正确性验证框架（本 MVP）：

1. 算法层：每个子步都严格对应 `(I - dt*kappa*Ddir)` 的离散方程。  
2. 线性代数层：三对角系统由源码中的 Thomas 流程求解，不依赖黑盒 PDE 库。  
3. 物理行为层：热方程 `L2` 范数应随时间衰减。  
4. 解析对照层：与已知解析解比较相对误差，检查最终误差阈值。

`demo.py` 同时做了以上 4 类检查。

## R11

误差与稳定性：

- 空间离散：中心差分二阶，`O(h^2)`；
- 时间离散：LOD Lie 分裂一阶，`O(dt)`；
- 对扩散方程，隐式方向子步使稳定性较好；
- 当 `dt` 偏大时，虽然不易爆炸，但分裂误差会明显增大。

本演示参数（`n=40, dt=1e-3, t_end=0.2`）下，最终相对误差约为百分之几量级，符合“一阶时间 + 二阶空间”的预期。

## R12

性能视角：

- 单次求解均为长度 `n` 的三对角线性系统，缓存友好；
- 代码中对固定系数矩阵先做一次“Thomas 系数预分解”，减少重复前处理；
- 相比每步构造并求解二维大系统，LOD 更轻量、易扩展；
- 若进一步追求速度，可并行化按列/按行的独立三对角求解。

## R14

鲁棒性与常见失效模式：

- 配置非法（如 `dt<=0`、`n<3`、`t_end/dt` 非整数）会在参数校验阶段报错；
- 若实现误写（例如行列方向混淆、三对角回代错误），误差会迅速变大并触发断言；
- 若切换到非齐次边界，需在 RHS 中显式加入边界贡献；
- 对更复杂 PDE（反应项、对流项），该 MVP 需要额外离散策略。

## R15

实现设计（`demo.py`）：

- `LODConfig`：参数定义与合法性校验。  
- `build_grid` / `exact_solution`：生成网格与解析解。  
- `factorize_tridiagonal`：对固定三对角矩阵做预分解。  
- `solve_tridiagonal_factored`：Thomas 前消元 + 回代。  
- `lod_step`：一次 `x` 扫 + 一次 `y` 扫。  
- `run_lod_simulation`：主循环推进并记录误差/范数。  
- `run_checks`：自动验收条件。

整个流程没有调用第三方“黑盒 PDE 求解器”。

## R16

相关算法链路：

- 邻近方法：ADI（交替方向隐式）方法。  
- 更高阶分裂：Strang splitting（时间二阶）。  
- 统一视角：算子分裂 `exp(dt(A+B))` 的近似族。  
- 实践延伸：可与 Crank-Nicolson、反应项 IMEX 方案组合。

## R18

源码级算法流程拆解（`run_lod_simulation` 主流程，8 步）：

1. 读取并校验 `LODConfig`，确保网格、时间参数合法。  
2. 调用 `build_grid` 构造内点网格，并用 `exact_solution(..., t=0)` 初始化 `u`。  
3. 根据 `r = kappa*dt/h^2` 组装三对角系数 `lower/diag/upper`。  
4. 在 `factorize_tridiagonal` 中预计算 Thomas 所需 `c_prime` 与 `inv_denom`。  
5. 每个时间步先执行 `x` 方向隐式扫：逐列调用 `solve_tridiagonal_factored` 得到 `u_star`。  
6. 再执行 `y` 方向隐式扫：逐行调用同一求解器得到 `u_next`。  
7. 在采样步计算解析解对照误差 `relative_l2_error` 与解的 `l2_norm`，并检查数值是否有限。  
8. 循环结束后返回 `SimulationResult`，由 `print_report` 输出报告，`run_checks` 完成自动验收。  

说明：这里的“分裂 + 三对角求解”全过程都在源码中显式展开，可逐行追踪，不是黑箱调用。
