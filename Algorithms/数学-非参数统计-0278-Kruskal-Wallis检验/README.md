## R01

Kruskal-Wallis 检验（常记为 K-W 检验）是单因素方差分析（one-way ANOVA）的非参数替代方法，用于比较 `k>=2` 个独立样本是否来自同一分布位置（常解释为总体中位数/位置参数是否一致）。

它把原始数值转换为“全局秩（rank）”，再比较各组秩和差异，因此对偏态分布、重尾分布和离群值通常比参数法更稳健。

## R02

问题定义：

- 输入：`k` 个彼此独立的样本组 `G1, G2, ..., Gk`，每组大小分别为 `n1, n2, ..., nk`。
- 原假设 `H0`：各组总体分布位置相同（同分布位置）。
- 备择假设 `H1`：至少有一组的位置不同。
- 输出：统计量 `H`、自由度 `df=k-1`、p 值、拒绝结论。

注意：K-W 检验只能告诉“是否存在差异”，不能直接指出“哪两组不同”，后者需要事后检验（如 Dunn 检验）。

## R03

核心思想：

1. 把所有组数据合并后做统一排序，得到每个观测的秩；
2. 计算每个组的秩和 `R_i`；
3. 若 `H0` 成立，各组秩和应接近其样本量比例；
4. 若某些组系统性偏大/偏小，其秩和会偏离，导致 `H` 变大。

因此 K-W 本质是“基于秩的组间差异度量”。

## R04

无并列值（tie）时的统计量：

- 总样本量 `N = n1 + n2 + ... + nk`；
- 第 `i` 组秩和为 `R_i`；
- 原始统计量

`H_raw = (12 / (N*(N+1))) * sum_i(R_i^2 / n_i) - 3*(N+1)`。

在样本量较大时，`H_raw` 近似服从 `chi^2(k-1)`。

## R05

并列值（ties）校正：

实际数据常出现相同取值（如 1~5 的问卷评分），需要 tie correction：

- 设所有并列块大小为 `t_j`；
- 校正因子

`C = 1 - sum_j(t_j^3 - t_j) / (N^3 - N)`；

- 校正后统计量

`H = H_raw / C`。

本条目 `demo.py` 会显式实现平均秩与 `C` 的计算，不把关键过程留在黑盒里。

## R06

适用前提与边界：

- 各组样本独立；
- 观测至少是有序尺度（ordinal）或连续数值；
- 各组分布形状应大致可比，否则“位置差异”解释会变弱；
- 每组至少 1 个样本（实践上建议 >=5）。

若样本是“配对/重复测量”，应考虑 Friedman 检验而非 K-W。

## R07

复杂度分析（`k` 组，总样本 `N`）：

- 合并排序以计算秩：`O(N log N)`；
- 扫描计算并列块与秩和：`O(N)`；
- 统计量与 p 值计算：`O(k)`。

总体时间复杂度 `O(N log N)`，额外空间复杂度 `O(N)`。

## R08

与常见方法的关系：

- 与 one-way ANOVA：
  - ANOVA 比较均值并依赖更强分布假设；
  - K-W 比较秩位置，分布假设更弱。
- 与 Mann-Whitney U：
  - Mann-Whitney 用于两组；
  - K-W 是多组推广。
- 与中位数检验：
  - K-W 通常更有统计功效。

## R09

手算级示例（3 组）：

- A: `[1, 3, 2]`
- B: `[4, 5, 6]`
- C: `[2, 2, 3]`

合并后赋秩（有并列用平均秩），可得每组秩和 `R_A, R_B, R_C`。代入 `H_raw` 与 tie 校正后得到 `H`。若 `H` 对应 p 值小于显著性水平（如 `alpha=0.05`），拒绝 `H0`，说明至少一组位置不同。

## R11

数值与实现注意点：

- 使用 `float64` 计算统计量；
- 排序采用稳定排序，确保并列块处理一致；
- `C` 极端接近 0 时应报错（数据退化，无可用秩信息）；
- 不接受空组、`NaN`、`Inf` 输入。

## R12

结果解释常见误区：

- “显著”不等于“差异很大”；
- “不显著”不等于“完全相同”；
- K-W 显著后需要事后多重比较来定位差异组；
- 仅报告 p 值不够，建议同时报告效应量（如 `epsilon^2`）。

## R13

本 MVP 输出字段：

- `H_statistic`：K-W 统计量（含 tie 校正）；
- `df`：自由度 `k-1`；
- `p_value`：卡方近似下的尾概率；
- `epsilon_squared`：效应量近似；
- `reject_h0`：在给定 `alpha` 下是否拒绝原假设。

还会输出每组 `n/mean/median` 方便快速诊断。

## R16

失败模式与防护：

- 失败：把重复测量数据当独立组。  
  防护：改用 Friedman 或混合效应模型。
- 失败：显著后直接下“某两组不同”结论。  
  防护：补做多重比较并控制错误率。
- 失败：大量并列值但未做校正。  
  防护：显式使用 tie correction。
- 失败：组大小极不平衡且样本很少。  
  防护：增加样本并结合效应量解释。

## R18

`demo.py` 源码级算法流程（8 步，非黑盒）：

1. `main` 构造两类可复现数据：
   - `shifted-locations`（各组位置有差异）；
   - `null-like`（各组分布接近）。
2. `kruskal_wallis_manual` 调用 `_validate_groups` 检查组数、空组、有限值约束，并汇总总样本量 `N`。  
3. 把所有组拼接为一维数组，调用 `_average_ranks_with_ties`：
   - 先稳定排序；
   - 对每个并列块分配平均秩；
   - 记录每个并列块大小 `t_j`。
4. 按各组样本量切分秩数组并求每组秩和 `R_i`。  
5. 依据公式计算 `H_raw = (12/(N(N+1))) * Σ(R_i^2/n_i) - 3(N+1)`。  
6. 用并列块大小计算校正因子 `C = 1 - Σ(t_j^3-t_j)/(N^3-N)`，得到 `H = H_raw / C`。  
7. 以 `df=k-1` 使用 `chi2.sf(H, df)` 计算 p 值，并计算效应量 `epsilon^2` 与拒绝结论。  
8. `compare_with_scipy` 调用 `scipy.stats.kruskal`，对 `H` 与 `p` 做误差断言；`main` 再汇总打印 DataFrame 供人工复核。

说明：SciPy 在本条目中只承担“对照验证”，核心秩统计与 tie 校正逻辑均已在源码中显式实现。
