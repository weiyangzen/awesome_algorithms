## R01

k均值聚类（K-Means）是最常用的原型聚类算法之一，目标是把 `n` 个样本划分为 `k` 个簇，使簇内样本尽量紧凑、簇间尽量分离。

标准目标函数（WCSS / Inertia）：

`min_{C_1...C_k, mu_1...mu_k} sum_{j=1..k} sum_{x in C_j} ||x - mu_j||_2^2`

其中：
- `C_j` 是第 `j` 个簇；
- `mu_j` 是第 `j` 个簇中心（均值向量）；
- 距离通常使用欧氏距离平方。

K-Means 的经典迭代由两步组成：
1. 分配步（Assignment）：把每个样本分到最近中心。
2. 更新步（Update）：把每个簇中心更新为该簇样本均值。

## R02

历史脉络（常见引用）：
- 1965 年：Forgy 提出“随机选点做中心”的初始化思路。
- 1967 年：MacQueen 发表在线更新版本的 k-means。
- 1982 年：Lloyd 算法（最常见批处理版本）公开发表并广泛传播。
- 2007 年：Arthur 与 Vassilvitskii 提出 k-means++ 初始化，显著改善随机初值不稳定问题。

工程实践里提到的 K-Means，默认通常是“Lloyd 迭代 + k-means++ 初始化 + 多次随机重启”。

## R03

K-Means 解决的问题是“无标签数据的结构发现”：
- 输入：样本矩阵 `X in R^(n x d)` 与簇数 `k`。
- 输出：簇标签 `labels`、簇中心 `centers`、目标函数值 `inertia`。

它适用于“簇近似球形、方差相近、用均值可代表簇”的场景。  
核心价值是把高维样本压缩成少量中心与离散簇标签，便于后续分析、压缩、可视化或作为下游特征。

## R04

Lloyd 版 K-Means 基本流程：
1. 选择 `k` 个初始中心（如 k-means++）。
2. 计算每个样本到各中心的距离，分配到最近中心。
3. 对每个簇求均值，更新中心。
4. 判断收敛：若中心位移很小或标签不再变化则停止。
5. 若未收敛，重复步骤 2-4。
6. 若配置了 `n_init > 1`，重复多次不同初始化，取 inertia 最小解。

性质：每轮迭代都会让目标函数不增（单调下降或持平），因此有限步内收敛到局部最优。

## R05

复杂度（Lloyd 批处理）：
- 单次迭代时间：`O(n * k * d)`（主耗时在距离计算）。
- 总时间：`O(T * n * k * d)`，`T` 是迭代次数。
- 多次重启：`O(n_init * T * n * k * d)`。
- 空间复杂度：`O(n * d + k * d + n)`，若显式保留全部距离矩阵则约 `O(n * k)`。

实际瓶颈通常是：
- 大样本数 `n`；
- 大簇数 `k`；
- 高维特征 `d`；
- 多次重启 `n_init`。

## R06

简化示例（二维、`k=2`）：
- 样本：`(1,1), (1.5,2), (3,4), (5,7), (3.5,5), (4.5,5), (3.5,4.5)`。
- 初始中心（示例）：`mu1=(1,1), mu2=(5,7)`。

第一轮分配后：
- 靠近 `mu1` 的点形成簇 1；
- 靠近 `mu2` 的点形成簇 2。

更新中心后：
- `mu1`、`mu2` 都会向各自簇样本的几何中心移动；
- 重复若干轮后，中心与簇标签稳定，inertia 收敛。

## R07

优势：
- 实现简单、速度快、可扩展到中大规模数据。
- 结果可解释（中心可视为“原型”）。
- 常作为无监督学习的强基线。

局限：
- 需要预先指定 `k`。
- 对初始化敏感，容易陷入局部最优。
- 假设簇近似球形；对非凸簇、密度差异大或强噪声数据表现不佳。
- 对特征尺度敏感，通常需要标准化。

## R08

前置知识与实现依赖：
- 向量化距离计算（欧氏距离平方）。
- 均值统计与广播操作。
- 迭代收敛判断（中心位移/标签稳定）。
- 基础数值工具：`numpy`。
- 评估与对照：`scikit-learn`（`KMeans`、`adjusted_rand_score`、`silhouette_score`）。
- 结果展示：`pandas`。
- 中心对齐比较：`scipy.optimize.linear_sum_assignment`。

## R09

适用场景：
- 数据天然呈现“团块状”结构。
- 需要快速聚类基线或中心原型。
- 作为下游模型前处理（如聚类特征、样本压缩）。

不适用或需谨慎：
- 非凸簇（如月牙形、同心环）。
- 簇大小与密度差异极大。
- 类别边界更接近密度连通而非均值最小二乘。
- 含大量离群点且未做鲁棒预处理。

## R10

正确性检查要点：
1. 分配步是否选择“最近中心”。
2. 更新步是否对每簇取“样本均值”。
3. 目标函数（inertia）是否非增。
4. 收敛条件是否明确（中心位移 `<= tol` 或标签不变）。
5. 空簇处理是否合理（重采样/最远点回填）。
6. 多次重启是否取最优 inertia 的结果。

## R11

数值稳定性与工程细节：
- 距离计算建议使用平方距离，避免不必要 `sqrt`。
- 标准化特征可避免某一维尺度主导聚类。
- 使用 `float64` 可减少累计误差。
- 距离矩阵可能出现极小负值（浮点误差），可 `clip` 到 0。
- 空簇是常见数值/结构问题，必须显式处理。

## R12

真实成本与调参经验：
- `k` 的影响最大，可结合轮廓系数、肘部法、业务约束选择。
- `n_init` 直接影响解质量与耗时，默认至少 10 次在多数场景更稳。
- `max_iter` 过小可能提前停止，过大收益递减。
- 初始化策略上，k-means++ 一般显著优于随机初始化。
- 大规模数据可转向 MiniBatchKMeans，以少量精度换训练速度。

## R13

理论性质（简述）：
- K-Means 全局最优求解是 NP-hard（一般情形）。
- Lloyd 迭代可保证有限步收敛到局部最优。
- k-means++ 初始化对期望目标值给出 `O(log k)` 近似保证（相对最优值）。
- 实务上需通过多次重启和指标评估降低局部最优风险。

## R14

常见失效模式与防护：
- 失效：初始化不佳导致差的局部最优。  
  防护：增大 `n_init`，使用 k-means++。
- 失效：簇并非球形，K-Means 强行切分。  
  防护：尝试 GMM、DBSCAN、谱聚类等替代方法。
- 失效：特征量纲差异大。  
  防护：标准化或白化。
- 失效：离群点拉偏均值中心。  
  防护：离群点检测、鲁棒缩放或改用 k-medoids。

## R15

工程实现建议：
- 固定随机种子，保证可复现。
- 输出至少包含：`inertia`、簇大小分布、外部指标（若有真值）。
- 加入质量守卫（断言），防止静默退化。
- 对自实现结果与成熟库（如 sklearn）做基线对照。
- 在生产侧记录参数版本：`k`、`n_init`、`tol`、标准化配置。

## R16

相关方法与演化路线：
- K-Means++：改进初始化。
- MiniBatchKMeans：小批量近似更新，适合大规模数据。
- K-Medoids：中心受限于样本点，对异常值更鲁棒。
- GMM（EM）：软分配、可建模椭圆簇。
- DBSCAN/HDBSCAN：基于密度，不需预设 `k`，可识别噪声与非凸簇。

## R18

源码级流程拆解（`demo.py` + `scikit-learn`，8 步）：
1. `generate_synthetic_blobs` 生成 3 个高斯簇数据并打乱顺序，得到 `X` 与真值标签 `y_true`。  
2. `standardize` 对特征做标准化，降低尺度差异对欧氏距离的影响。  
3. `KMeansScratch.fit` 在每次重启中先调用 `kmeans_plusplus_init` 选初始中心（按当前最短距离平方构造采样概率）。  
4. `fit` 的每轮迭代执行 `_assign`（E-step）得到最近中心标签，再对每簇取均值更新中心（M-step）；遇到空簇时用当前最远样本重置中心。  
5. 收敛后做最终分配，计算 inertia；多次重启后保留 inertia 最小解。  
6. `main` 同时训练 `sklearn.cluster.KMeans(..., algorithm="lloyd")` 做对照。  
7. 在本环境 `sklearn/cluster/_kmeans.py` 中，`KMeans.fit` 会做参数与数据检查，调用 `_init_centroids` 初始化，然后进入 `_kmeans_single_lloyd` 的“分配-更新”循环，最终通过 `_labels_inertia` 生成一致标签与 inertia。  
8. `main` 汇总两种实现的 `inertia/ARI/silhouette`、簇大小和中心匹配差距，并通过断言确保 MVP 结果达标。
