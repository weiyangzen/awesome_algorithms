## R01

SimRank 是一种图结构相似度算法，核心思想是：
- 两个节点相似，当且仅当“指向它们的节点”也相似；
- 相似性通过递归定义逐步传播；
- 常用于有向图中的“结构角色”比较，而不依赖节点文本特征。

它适用于引用网络、网页链接图、推荐图等场景，属于经典的结构相似度方法。

## R02

给定有向图 `G=(V,E)`，SimRank 要回答的问题是：
- 任意两个节点 `a,b` 的结构相似分数 `s(a,b)` 是多少；
- 能否得到一个全局相似度矩阵 `S` 用于检索 Top-K 相似节点对。

与最短路径、连通性不同，SimRank 关注的是“结构角色相似”，即两个节点不必直接相连，也可能高度相似。

## R03

经典递推定义（`I(x)` 为节点 `x` 的入邻居集合，`C` 为衰减系数）：

- `s(a,a)=1`
- 若 `a!=b` 且 `I(a), I(b)` 非空：
  - `s(a,b)=C/(|I(a)||I(b)|) * Σ_{i∈I(a)} Σ_{j∈I(b)} s(i,j)`
- 若 `a!=b` 且任一入邻居为空：
  - `s(a,b)=0`

该递推通常采用迭代近似求解。

## R04

矩阵化后可写为：
- 构造列随机矩阵 `P`，其中 `P[i,j]=1/|I(j)|`（当 `i->j`）否则为 `0`；
- 迭代更新：`S_{k+1}=C * P^T * S_k * P`；
- 每轮强制 `diag(S_{k+1})=1`。

`demo.py` 使用这个矩阵形式实现，便于并行与块分解。

## R05

复杂度（`n=|V|`）：
- 朴素矩阵迭代每轮约 `O(n^3)`；
- 共 `T` 轮时约 `O(T*n^3)`；
- 存储 `S` 需要 `O(n^2)`。

在稀疏图中，可结合稀疏矩阵与块分解降低常数开销，但最坏上界仍较高。

## R06

并行/分布式友好点：
- 单轮更新可写成矩阵乘法链，天然可并行化；
- `S_{k+1}` 可按行块拆分，多个 worker 独立计算不同块；
- 只需共享当前轮 `S_k` 与 `P`，然后汇总块结果。

本实现在 `simrank_step_blockwise` 中模拟了这种 row-block 执行路径，并支持线程池并发。

## R07

本条目 MVP 范围：
- 固定 8 节点有向图，确保可复现；
- 显式构建入链转移矩阵 `P`；
- 实现 SimRank 迭代 + 收敛统计（`delta_max`、`mse`）；
- 输出 Top 相似节点对与重点节点对分数。

目标是展示算法机制与工程可运行性，而不是追求大规模极致性能。

## R09

正确性关键约束：
- 对角线恒为 `1`；
- `S` 保持对称；
- 块分解版本与致密基准版本数值一致；
- PyTorch 路径与 NumPy 路径数值一致（容差内）。

`run_self_check` 与 `run_simrank` 中包含了这些断言。

## R10

收敛判据：
- 每轮计算 `delta_max = max(|S_{k+1}-S_k|)`；
- 若 `delta_max < tol` 则提前停止；
- 同时记录 `mse(S_k, S_{k+1})` 观察迭代稳定趋势。

该做法能在“固定迭代上限”和“自适应收敛”之间取得平衡。

## R11

边界与鲁棒性处理：
- 空图直接报错；
- 边中出现未知节点报错；
- `C` 必须在 `(0,1)` 内；
- `blocks` 必须为正；
- 零入度列在 `P` 中保持零列，不触发除零。

这些检查保证脚本在最小输入约束下稳定运行。

## R12

`demo.py` 结构映射：
- `build_demo_graph`：定义可复现测试图
- `build_inlink_transition_matrix`：构造列随机 `P`
- `simrank_step_dense`：矩阵公式基准实现
- `simrank_step_blockwise`：行块并行友好实现
- `simrank_step_torch`：PyTorch 交叉验证实现
- `run_simrank`：多轮迭代、收敛统计与一致性检查
- `top_similar_pairs`：输出 Top-K 节点对
- `run_self_check` / `main`：自检与演示入口

## R13

输出解读建议：
- `Iteration History` 中 `delta_max`、`mse` 应总体下降；
- `Top Similar Node Pairs` 展示全局最相似的非对角节点对；
- `Focus Pairs` 提供关键对照（如 `C-D`、`E-H`、`A-B`），便于快速人工判断。

由于图结构是人工构造，可看到共享入邻居越多的节点对往往得分更高。

## R14

常见实现错误：
1. 用出邻居而不是入邻居做递推；
2. 忘记每轮把对角线重置为 `1`；
3. 块分解后没有正确拼接行块；
4. 只看最终分数，不做跨实现一致性校验；
5. 使用黑盒 API 一步出结果，无法追踪算法步骤。

本条目通过显式公式实现与三路径一致性检查规避上述问题。

## R15

局限性：
- SimRank 对大图计算和存储成本较高；
- 收敛速度依赖图结构与 `C` 参数；
- 结果只反映结构相似，不直接编码语义信息；
- 当前 MVP 为单机演示，不含真实分布式通信。

## R16

可扩展方向：

- 与 MapReduce/Spark 矩阵分块结合做分布式求解；
- 对热点节点使用增量更新；
- 与内容特征融合，做结构+语义混合相似度。

## R18

`demo.py` 的源码级算法流程（8 步）：
1. `build_demo_graph` 构造固定有向图节点和边，保证实验可复现。  
2. `build_inlink_transition_matrix` 把边集转为稀疏邻接矩阵，并按列做 `L1` 归一化得到 `P`。  
3. `run_simrank` 以单位阵 `S_0=I` 初始化，并进入迭代循环。  
4. 每轮调用 `simrank_step_blockwise`：先计算 `right=S_k*P`，再按行块计算 `C*(P_block^T*right)` 并汇总成 `S_{k+1}`。  
5. 同轮并行计算 `simrank_step_dense` 与 `simrank_step_torch`，分别作为 NumPy 基准和 PyTorch 交叉验证。  
6. 对三条路径结果做 `allclose` 一致性断言，并强制 `diag(S_{k+1})=1`。  
7. 记录 `delta_max` 与 `mse` 作为收敛诊断，满足 `delta_max < tol` 时提前终止。  
8. `top_similar_pairs` 从最终 `S` 提取非对角 Top-K 相似节点对，`main` 打印结果并结束。  

这个流程把 SimRank 的递推定义完整落地成可执行代码，同时保留了并行/分布式分块计算的实现入口。
