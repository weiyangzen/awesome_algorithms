## R01

中国剩余定理（Chinese Remainder Theorem, CRT）用于求解同余方程组：

\[
\begin{cases}
x \equiv a_1 \pmod{m_1}\\
x \equiv a_2 \pmod{m_2}\\
\cdots\\
x \equiv a_k \pmod{m_k}
\end{cases}
\]

当模数两两互素时，解在模 \(M=\prod m_i\) 意义下唯一。CRT 本质是把多个小模空间的信息“拼接”为一个大模空间中的唯一代表元。

## R02

应用场景：

- 大整数运算与分解（把大模计算拆成多个小模并行计算后再重构）
- 密码学与同余协议（如 RSA 优化、同余检验）
- 数论构造题与竞赛题（多条件同余合并）
- 工程中的周期对齐问题（多个周期约束求公共相位）

## R03

形式化结论（两两互素版本）：

- 条件：\(\gcd(m_i,m_j)=1\), \(i\neq j\)
- 结论：同余组存在唯一解 \(x \pmod{M}\), \(M=\prod_{i=1}^k m_i\)

构造式：

\[
x \equiv \sum_{i=1}^k a_i M_i y_i \pmod{M}
\]

其中：

- \(M_i = M/m_i\)
- \(y_i \equiv M_i^{-1}\pmod{m_i}\)（即 \(M_i y_i \equiv 1 \pmod{m_i}\)）

## R04

直观理解：

- 每一项 \(a_i M_i y_i\) 在模 \(m_i\) 下等于 \(a_i\)
- 同时在任意 \(j\neq i\) 的模 \(m_j\) 下，由于 \(M_i\) 含有因子 \(m_j\)，该项等于 0
- 所有项叠加后，恰好在每个模通道保留对应余数，互不干扰

## R05

正确性要点（互素版本）：

1. 对固定 \(i\)，有 \(M_i y_i \equiv 1\pmod{m_i}\)，所以该项贡献 \(a_i\)。
2. 对 \(j\neq i\)，\(m_j\mid M_i\)，所以该项在模 \(m_j\) 下为 0。
3. 总和对每个 \(m_i\) 都满足目标余数，因此是方程组解。
4. 若 \(x,x'\) 都是解，则对每个 \(i\)，\(m_i\mid (x-x')\)。
5. 由两两互素可得 \(M=\prod m_i\mid (x-x')\)，即 \(x\equiv x' \pmod M\)，唯一性成立。

## R06

复杂度分析：

- 设有 \(k\) 个方程，模数量级约为 \(B\)（位数）
- 扩展欧几里得或逆元计算：每次约 \(O(\log m_i)\)
- 总体为 \(O(k\log B)\) 次基础大整数算术（忽略乘法位复杂度细节）
- 空间复杂度约 \(O(1)\) 额外变量（不含输入与大整数表示）

## R07

互素约束下的实现策略：

1. 预处理 \(a_i \leftarrow a_i \bmod m_i\)
2. 计算总模 \(M=\prod m_i\)
3. 对每个 i：
   - \(M_i=M/m_i\)
   - 求逆元 \(y_i=M_i^{-1}\pmod{m_i}\)
   - 累加 \(x = (x + a_iM_iy_i)\bmod M\)
4. 输出 \((x,M)\) 表示所有解 \(x+tM\)

## R08

非两两互素时的广义 CRT：

- 两个同余可合并当且仅当：
  \[
  a_1 \equiv a_2 \pmod{\gcd(m_1,m_2)}
  \]
- 若可合并，得到新同余：
  \[
  x \equiv a \pmod{\operatorname{lcm}(m_1,m_2)}
  \]
- 多个方程可用“逐对合并”迭代处理，最终要么得到唯一模 lcm 的解类，要么判定无解

## R09

逐对合并公式（广义 CRT 核心）：

已知
\[
x\equiv r \pmod m,\quad x\equiv a \pmod n
\]

设 \(g=\gcd(m,n)\)，若 \((a-r)\not\equiv 0\pmod g\) 则无解。否则：

\[
t \equiv \frac{a-r}{g}\cdot (m/g)^{-1}\pmod{n/g}
\]
\[
x = r + mt,\quad \text{新模}= \operatorname{lcm}(m,n)=m/g\cdot n
\]

用扩展欧几里得可直接得到 \((m/g)^{-1}\) 或等价系数。

## R11

关键边界条件：

- `modulus <= 0`：非法，必须抛错
- 余数可为负：统一归一化到 `[0, m-1]`
- 只有一个方程：答案即该方程归一化后结果
- 非互素冲突：必须显式报错（不能静默返回错误值）
- 空输入：无数学意义，通常视为非法输入

## R12

正确性验证建议：

- 样例 1（互素经典）：`x≡2(mod3), x≡3(mod5), x≡2(mod7)`，应得 `x≡23(mod105)`
- 样例 2（非互素可解）：`x≡1(mod4), x≡3(mod6)`，应得 `x≡9(mod12)`
- 样例 3（非互素无解）：`x≡1(mod2), x≡0(mod4)`，应判定无解
- 每次输出后逐式验证：`x % m_i == a_i % m_i`

## R13

与其他方法对比：

- 暴力枚举：实现简单，但复杂度随模乘积增长，几乎不可扩展
- CRT：利用代数结构直接构造，复杂度低且可扩展到大整数
- 搜索/回溯：在无结构约束时可用，但在同余问题上通常不如 CRT 直接

## R14

工程实现注意点：

- Python 大整数天然支持，适合直接实现
- 逆元建议通过扩展欧几里得，避免依赖版本特性
- 统一返回 `(x, mod)`，便于后续组合与验证
- 报错信息应包含冲突方程，方便排障

## R15

最小可运行示例（互素）：

```text
remainders = [2, 3, 2]
moduli     = [3, 5, 7]
=> x = 23, mod = 105
```

含义：所有整数解可写为 `x = 23 + 105t`（`t` 为任意整数）。

## R16

扩展方向：

- Garner 算法：避免显式构造超大乘积，适合某些多模重建场景
- 与快速幂结合：先在多小模下计算，再 CRT 合并
- 应用于多项式同余、NTT/FFT 混合算法中的结果重建

## R18

源码级算法流程拆解（对应 `demo.py`，共 7 步）：

1. **输入归一化**  
   对每个方程执行 `a_i %= m_i`，并检查 `m_i > 0`。

2. **扩展欧几里得求系数**  
   在 `extended_gcd(a, b)` 中递归返回 `(g, x, y)`，满足 `ax + by = g`。

3. **互素 CRT（快速路径）**  
   `crt_pairwise` 先算总模 `M`，再对每个方程计算 `Mi=M//mi` 与 `inv=Mi^{-1}(mod mi)`，累计 `x=(x+ai*Mi*inv)%M`。

4. **广义 CRT 初始化**  
   `crt_general` 从平凡同余 `x≡0(mod 1)` 开始，逐个方程并入。

5. **两式合并与可解判定**  
   合并 `x≡r(mod m)` 和 `x≡a(mod n)`：令 `g=gcd(m,n)`，若 `(a-r)%g!=0`，立即判无解。

6. **计算偏移并更新解类**  
   若可解，计算  
   `t = ((a-r)//g * p) % (n//g)`（`p` 来自 `extended_gcd(m,n)`），  
   更新 `x = (r + m*t) % lcm`，`lcm = m//g*n`。

7. **结果校验与展示**  
   `verify_solution` 逐式检查 `x % mi == ai % mi`，主程序打印互素可解、非互素可解与无解三类结果。
