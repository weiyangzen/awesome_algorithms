## R01

双向 RNN（Bidirectional RNN, BiRNN）是在时间维度上同时建模“从左到右”和“从右到左”上下文的序列网络。对序列中任意位置 `t`，它的表示不仅包含历史信息，也包含未来信息，因此特别适合需要双侧上下文的序列标注任务。

本条目提供一个可运行、可审计的最小 MVP：
- 手写单向 RNN 与双向 RNN（基于 `RNNCell` 循环，不走黑盒）；
- 在同一任务上做基线对比；
- 通过指标和断言验证“双向”确实带来收益。

## R02

经典单向 RNN 在位置 `t` 只能读取 `1..t` 的前缀信息。若标签依赖 `t+1` 或更右侧 token，单向结构会天然受限。双向 RNN 通过两条链路并行编码：
- 正向链负责历史上下文；
- 反向链负责未来上下文；
- 最终拼接表示后再分类。

因此它常用于命名实体识别、词性标注、语音帧标注等“每个位置都需要双侧语境”的任务。

## R03

BiRNN 的核心公式（以简单 RNN 单元表示）如下：

- 正向隐藏态：`h_f[t] = phi(W_xf x[t] + W_hf h_f[t-1] + b_f)`
- 反向隐藏态：`h_b[t] = phi(W_xb x[t] + W_hb h_b[t+1] + b_b)`
- 位置融合：`h[t] = concat(h_f[t], h_b[t])`
- 分类输出：`p[t] = softmax(W_o h[t] + b_o)`

其中 `phi` 在本实现中由 `RNNCell` 内部激活给出，输出层做二分类（标签 0/1）。

## R04

本目录把问题设置为“邻居匹配序列标注”：
- 输入：离散 token 序列；
- 标签：若当前位置 token 与左邻或右邻任一相等，则标签为 1，否则为 0。

这个任务有明确的“未来信息需求”：
- 判断 `x[t] == x[t+1]` 需要右侧信息；
- 单向 RNN 很难在位置 `t` 直接获得该信息；
- BiRNN 刚好能通过反向链补足。

## R05

MVP 的实现取舍：
- 不直接调用 `nn.RNN(..., bidirectional=True)` 作为黑盒；
- 显式使用两个 `nn.RNNCell` 写出前向和后向循环；
- 保留单向模型作为对照组，验证“结构改动”而非“训练偶然”。

这样既保证代码短小可运行，也能追溯算法数据流。

## R06

`demo.py` 的数据构造方式：
- 词表大小 `vocab_size=12`，实际 token 取值为 `[1, 11]`；
- 序列长度 `seq_len=18`；
- 训练集 `1800` 条，测试集 `500` 条；
- 标签规则：
  - 边界位 `t=0` 只比较 `x[0]` 与 `x[1]`；
  - 边界位 `t=L-1` 只比较 `x[L-1]` 与 `x[L-2]`；
  - 中间位判断 `x[t]` 是否等于 `x[t-1]` 或 `x[t+1]`。

## R07

关键张量与形状：
- 输入 token：`X -> [B, L]`（`int64`）；
- Embedding 输出：`[B, L, E]`；
- 单向隐藏序列：`[B, L, H]`；
- 双向拼接隐藏序列：`[B, L, 2H]`；
- 分类 logits：`[B, L, 2]`；
- 训练损失：将 `[B, L, 2]` 与 `[B, L]` reshape 后做 token 级交叉熵。

## R08

复杂度估计（忽略常数）：
- 单向 RNN：每步约 `O(B*(E*H + H^2))`，总计 `O(L*B*(E*H + H^2))`；
- 双向 RNN：两条链，约为单向两倍；
- 输出层：`O(B*L*H)` 或 `O(B*L*2H)`。

本任务参数较小（`L=18, H=20`），CPU 下可快速完成训练和对比。

## R09

数值与复现策略：
- 固定 `numpy` 和 `torch` 随机种子；
- 训练与测试数据使用不同 seed，避免数据泄漏；
- 指标采用 `accuracy/precision/recall/F1` 多视角判断；
- 用断言门禁限制最低性能与双向增益。

## R10

`demo.py` 函数分工：
- `build_neighbor_match_dataset`：生成样本与标签；
- `UniDirectionalRNNTagger`：单向基线模型；
- `BidirectionalRNNTagger`：双向模型（手写两条 RNNCell 链）；
- `train_model`：训练循环与 loss 记录；
- `evaluate_model`：计算分类指标；
- `summarize_samples`：构造可读样例表；
- `main`：组织训练、评估、断言与打印。

## R12

输出指标解释：
- `accuracy`：全部 token 的整体正确率；
- `precision`：预测为正时有多少是真的；
- `recall`：真实正类被找回的比例；
- `f1`：`precision` 与 `recall` 的调和均值；
- `pred_positive_rate`：模型输出正类的比例；
- `loss_start/loss_end`：用于确认训练确实收敛。

## R13

代码中的正确性门禁：
1. 单向和双向模型的训练 loss 都必须下降；
2. 双向模型 `accuracy >= 0.97`；
3. 双向模型 `F1 >= 0.97`；
4. 双向模型准确率相对单向至少提升 `0.05`；
5. 双向模型 F1 必须高于单向模型。

这些断言保证 demo 不是“只跑通不验证”。

## R14

常见失败模式与处理：
- 失败 1：双向收益不明显。  
  可能原因：任务设计不依赖未来信息。对策：改成明显依赖右侧上下文的标签规则。

- 失败 2：训练不收敛。  
  可能原因：学习率过高/过低。对策：优先调 `lr` 与 `epochs`。

- 失败 3：类别不平衡导致 F1 偏低。  
  可能原因：词表太大使“相邻相等”概率过低。对策：缩小词表或引入加权损失。

## R15

与相关序列模型对比：
- 相比单向 RNN：BiRNN 计算更重，但能利用未来上下文；
- 相比 BiLSTM/BiGRU：BiRNN 结构最简单，门控能力较弱但更易教学；
- 相比 Transformer：BiRNN 对短序列和小数据更轻量，但长程依赖能力通常不如注意力机制。

本条目聚焦“最基本双向递归思想”，因此选用最小 RNNCell 版本。

## R18

`demo.py` 的源码级算法流程（8 步）：
1. `build_neighbor_match_dataset` 生成离散 token 序列，并按“是否匹配左右邻居”构造逐位置二分类标签。  
2. `UniDirectionalRNNTagger.forward` 逐时间步更新 `h_t`，得到单向隐藏序列并通过线性层输出 logits。  
3. `BidirectionalRNNTagger.forward` 先跑正向链，再反向遍历跑后向链，在每个位置拼接 `h_f[t]` 与 `h_b[t]`。  
4. `train_model` 把 token 级 logits reshape 后用交叉熵训练，执行 Adam 反向传播更新参数。  
5. 在同一数据和训练轮次下分别训练单向模型与双向模型，形成公平对照。  
6. `evaluate_model` 展平全部位置，计算 accuracy、precision、recall、F1 与预测正类率。  
7. `main` 组装 `metric_summary`（`pandas.DataFrame`）并打印，直观看到双向模型在该任务上的增益。  
8. 断言阶段检查“loss 下降 + 双向性能达标 + 双向优于单向”，保证结果可验证而非仅可运行。
