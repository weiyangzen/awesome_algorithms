## R01

Whitted 光线追踪是经典递归光线追踪模型：
- 从相机向像素发射主光线；
- 命中物体后计算局部光照（环境光 + 漫反射 + 高光）；
- 再递归发射反射光线；
- 对每个光源发阴影光线做可见性判断。

本条目实现一个可运行 MVP，覆盖球体/平面求交、硬阴影、镜面反射和固定场景渲染。

## R02

问题定义（MVP 范围）：
- 输入（代码内固定参数）：
1. 相机参数：分辨率、FOV、相机原点；
2. 场景参数：球体、平面、材质；
3. 光照参数：两个点光源；
4. 追踪参数：最大递归深度。
- 输出：
1. 渲染结果 `render_whitted.ppm`；
2. 控制台统计（光线计数、求交计数）；
3. 若断言通过则输出 `All checks passed.`。

## R03

核心数学关系：
- 光线方程：`p(t) = o + t d`，其中 `t > 0`。
- 球求交：解二次方程 `||o + td - c||^2 = r^2`，取最小正根。
- 平面求交：`t = dot(p0 - o, n) / dot(d, n)`，且 `t > eps`。
- 漫反射：`I_diff = kd * max(dot(n, l), 0)`。
- 高光：`I_spec = ks * max(dot(v, r), 0)^shininess`。
- 反射向量：`r = d - 2 * dot(d, n) * n`。

## R04

`demo.py` 的执行流程：
1. 构造默认场景（2 个球 + 1 个地面平面 + 2 个点光源）；
2. 初始化 `WhittedTracer` 及统计计数器；
3. 对每个像素发射主光线并求最近交点；
4. 命中后执行 Phong 局部光照；
5. 对每个光源发阴影光线判断是否被遮挡；
6. 根据材质反射率递归追踪反射光线；
7. 合成颜色、裁剪到 `[0, 1]`；
8. 输出 PPM 图像并打印统计与摘要表。

## R05

关键数据结构：
- `Material`：颜色、环境/漫反射/高光系数、光泽度、反射率；
- `Sphere` / `Plane`：几何体与求交函数；
- `HitRecord`：最近命中记录（距离、点、法线、材质、对象名）；
- `PointLight`：点光源位置、颜色、强度；
- `WhittedTracer.stats`：`rays_cast / shadow_rays / reflection_rays / intersection_tests / surface_hits`。

## R06

正确性直觉：
- “最近正交点”保证可见表面选择正确；
- 阴影光线在到达光源前遇到物体即判定阴影，形成硬阴影；
- 反射光线递归让镜面材质获取间接可见物体颜色；
- 颜色合成使用局部项与反射项插值，保证结果稳定可控。

## R07

复杂度（记像素数 `P = W*H`，物体数 `O`，光源数 `L`，递归深度 `D`）：
- 时间复杂度：`O(P * D * O * (1 + L))`。
  解释：每次命中至少一次最近求交 `O(O)`，每个光源阴影测试 `O(O)`。
- 空间复杂度：`O(P)`（图像缓存）+ `O(D)`（递归栈）。

## R08

边界与鲁棒性处理：
- 使用 `EPSILON` 避免自相交（shadow acne / reflection acne）；
- 法线按入射方向矫正，避免平面背面法线导致高光异常；
- 递归深度上限 `max_depth` 防止无限反射；
- 最终颜色统一裁剪到 `[0, 1]`；
- 内置断言验证亮度范围与关键统计量非零。

## R09

MVP 设计取舍：
- 只做 Whitted 经典路径，不做折射与全局蒙特卡洛积分；
- 采用 `numpy` 做向量运算、`pandas` 做输出摘要，便于教学与验证；
- 图像写成二进制 PPM，避免额外图像库依赖；
- 几何体仅保留球与平面，保证实现短小、机制完整。

## R10

`demo.py` 函数职责：
- `normalize / reflect`：基础向量运算；
- `Sphere.intersect / Plane.intersect`：显式求交流程；
- `WhittedTracer.find_nearest_hit`：最近命中查询；
- `WhittedTracer.occluded`：阴影可见性判定；
- `WhittedTracer.shade`：局部光照 + 递归反射；
- `WhittedTracer.trace_ray`：单条光线递归追踪；
- `WhittedTracer.render`：像素扫描渲染；
- `save_ppm`：图像输出；
- `run_sanity_checks`：自动化有效性检查。

## R15

与相关方法差异：
- 光栅化：主要做可见面投影，不天然包含递归反射；
- 路径追踪：通过随机采样估计全局光照，噪声大但更真实；
- Whitted（本条目）：确定性递归，擅长镜面反射/硬阴影，速度快于全路径积分但全局照明有限。

## R16

适用场景：
- 计算机图形学教学中的“第一版光线追踪”；
- 验证求交、阴影和反射逻辑；
- 为后续折射、BVH、路径追踪打基础。

不适用场景：
- 追求电影级全局光照（间接漫反射、焦散、体积散射）；
- 超大规模场景实时渲染（需要加速结构和并行优化）。

## R18

`demo.py` 源码级算法流（9 步，非黑箱）：
1. `main` 固定相机与分辨率，调用 `build_default_scene` 生成材质、几何和灯光。  
2. 创建 `WhittedTracer`，初始化统计计数器并设置递归深度上限。  
3. `render` 对每个像素构造主光线方向（由 FOV 与像素坐标反投影得到）。  
4. `trace_ray` 调用 `find_nearest_hit`，遍历所有物体执行 `Sphere/Plane.intersect`，选最近正距离命中。  
5. 若未命中，返回渐变背景色；若命中，进入 `shade`。  
6. `shade` 先算环境光，再对每个光源计算 `ndotl` 漫反射与 `max(dot(v,r),0)^s` 高光。  
7. 每个光源都调用 `occluded` 发阴影光线；若在到光源前被阻挡，则跳过该光源贡献。  
8. 若材质反射率大于 0 且深度未超限，按反射公式生成新方向并递归 `trace_ray`，再按反射率混合颜色。  
9. 渲染完成后 `save_ppm` 写出图像，`run_sanity_checks` 做断言，最后打印统计与 `All checks passed.`。
