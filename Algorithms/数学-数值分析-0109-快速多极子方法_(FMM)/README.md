# 快速多极子方法 (FMM)

- UID: `MATH-0109`
- 学科: `数学`
- 分类: `数值分析`
- 源序号: `109`
- 目标目录: `Algorithms/数学-数值分析-0109-快速多极子方法_(FMM)`

## R01

本条目实现一个可运行的 FMM 最小版本（MVP），用于近似计算一维核函数
`K(x, y) = 1 / (x - y)` 的势场求和：

\[
\phi(x_i)=\sum_{j=1}^{N} \frac{q_j}{x_i-y_j}
\]

目标是展示 FMM 的关键结构（多极展开、层次树、远近场分离），并给出与直接法 `O(NM)` 的误差与耗时对比。

## R02

直接法需要对每个目标点遍历全部源点，复杂度为 `O(NM)`。当 `N, M` 变大时成本急剧上升。FMM 的核心思想是：

- 近场（几何上邻近）继续直接计算，保证精度。
- 远场（几何上分离）用“簇”的级数展开近似，不再逐点累加。
- 通过树结构复用中间结果，使整体复杂度下降到接近线性。

## R03

本实现采用 1D 幂级数矩（moments）表示多极展开。

- 叶子盒的多极矩：
  \[
  M_k=\sum_j q_j (y_j-c)^k
  \]
  其中 `c` 是该盒中心。
- 远场点 `x` 的近似：
  \[
  \sum_{k=0}^{p} \frac{M_k}{(x-c)^{k+1}}
  \]
- M2M 使用二项式平移矩。
- M2L 把源盒多极系数变换为目标盒局部系数。
- L2L 把父盒局部展开下推到子盒。

这些公式在 `demo.py` 中分别对应 `translate_m2m`、`translate_m2l`、`translate_l2l`。

## R04

数据结构是“均匀二叉树 + 盒子对象 `Cell`”：

- `level, index`: 盒子的层与同层索引。
- `center, half_width`: 几何范围。
- `source_ids, target_ids`: 叶子中源点/目标点索引。
- `multipole`: 多极系数向量 `M[0..p]`。
- `local`: 局部系数向量 `L[0..p]`。

树节点按 `(level, index)` 存在字典中，方便随机访问与层次遍历。

## R05

整体流程：

1. 建立 `[xmin, xmax]` 上的均匀二叉树。
2. 将源点与目标点分桶到叶子。
3. 上行阶段：`P2M` 计算叶多极，`M2M` 汇总到根。
4. 下行阶段：每层先 `L2L` 继承父局部，再执行同层 `M2L`。
5. 叶子评估：`L2P` 得到远场贡献。
6. 邻域叶子（`leaf-1, leaf, leaf+1`）直接求和补齐近场。
7. 与直接法比较误差和时间。

## R06

正确性直觉：

- 对于远离某一源簇的目标点，`1/(x-y)` 可在簇中心做截断展开。
- 截断误差随展开阶数 `p` 增大而下降。
- 对近邻盒不做展开，直接求和避免近场发散误差。
- 最终结果 = 远场近似 + 近场精确，保证“效率与精度”平衡。

## R07

复杂度（设 `N≈M`）：

- 直接法：`O(N^2)`。
- FMM MVP：
  - 建树/分桶约 `O(N)`。
  - 上下行在固定 `p` 下约 `O(#cells)`。
  - 近场直接和在叶点数受控时近似 `O(N)`。
- 综合为接近线性（常数受 `p`、层数、每叶点数影响）。

注：当前 Python MVP 重在结构清晰，不追求极致常数优化。

## R08

数值注意事项：

- 核 `1/(x-y)` 在 `x≈y` 处接近奇异，数据构造需避免精确重合。
- `p` 太小会增加远场截断误差；太大则带来更高常数和潜在舍入误差。
- `max_level` 过浅会导致近场负担大；过深会导致树开销上升。
- 本脚本采用确定性数据构造，避免交互输入导致不稳定复现实验。

## R09

默认参数：

- `n = 2000`
- `p = 6`
- `max_level = 6`

经验解释：

- `p=6` 在该数据分布上可把相对 `L2` 误差压到 `1e-5` 量级。
- 64 个叶子使每叶约几十个点，近场直接和成本可控。

## R13

结果解读建议：

- 先看 `Relative L2 error` 是否达到预期精度。
- 再看 `FMM time` 与 `Direct time`：
  - 若样本较小，Python 常数开销可能使加速比不明显。
  - 随规模增大，`O(N^2)` 与近线性方法差异会逐步体现。

## R14

常见失败模式与排查：

- 出现 `inf`/`nan`：检查是否存在 `x == y` 或极端接近。
- 误差过大：提高 `p` 或提高树层数（减少远场近似压力）。
- 速度不升反降：样本过小、参数不匹配或 Python 循环开销占主导。

## R18

`demo.py` 的源码级算法流（8 步）：

1. `make_dataset(n)` 生成确定性 `xs, q, xt`，确保脚本可复现且无需输入。
2. `build_uniform_tree(...)` 创建完整二叉树，并为每个 `Cell` 初始化 `multipole/local`。
3. `assign_points_to_leaves(...)` 把源点与目标点映射到叶子盒索引。
4. `upward_pass(...)`：
   - 叶子执行 `P2M`（按 `dx^k` 计算矩）；
   - 再自底向上执行 `M2M`（`translate_m2m`）汇总多极系数。
5. `downward_pass(...)` 逐层执行：
   - 先 `L2L`（`translate_l2l`）把父局部场传到子盒；
   - 再按 `interaction_list_1d` 做同层 `M2L`（`translate_m2l`）累加远场。
6. `evaluate_potential(...)` 在叶子上做 `L2P`，得到每个目标点远场值。
7. 同函数内对邻域叶子执行直接求和，补齐近场贡献并与远场相加。
8. `main()` 同时运行 `direct_sum(...)` 与 `fmm_1d(...)`，打印耗时、相对 `L2` 误差与最大绝对误差。
