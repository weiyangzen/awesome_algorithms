# 平面嵌入算法

- UID: `MATH-0502`
- 学科: `数学`
- 分类: `图论`
- 源序号: `502`
- 目标目录: `Algorithms/数学-图论-0502-平面嵌入算法`

## R01

平面嵌入算法的目标是：给定无向图 `G=(V,E)`，判断它是否可在平面（或球面）无边交叉地绘制，并在可行时给出一个嵌入表示。

本目录采用“组合嵌入（combinatorial embedding）”形式输出：
- 每个顶点周围邻接点的循环顺序（rotation system）
- 该循环顺序诱导出的面（faces）

## R02

MVP 问题定义：
- 输入：`n` 个顶点（编号 `0..n-1`）和无向边列表 `edges=[(u,v), ...]`
- 输出：
1. `is_planar`：是否平面可嵌入
2. `rotation`：若可嵌入，`rotation[v]` 为顶点 `v` 的邻居循环顺序
3. `faces`：由 rotation 推导出的面边界序列
4. `message` 与 `states_explored`：诊断信息与搜索状态计数

## R03

核心数学对象：
- **Dart（有向半边）**：每条无向边 `(u,v)` 对应两个 dart：`u->v` 与 `v->u`
- **Rotation system**：每个顶点 `v` 给出邻居的循环顺序
- **面游走变换**：`(u->v) -> (v->next_v(u))`，其中 `next_v(u)` 是 `u` 在 `v` 的循环序中的后继

对连通图，若某组 rotation 诱导的面数为 `f`，满足
`n - m + f = 2`
则对应球面 genus 0 嵌入，即平面嵌入成立。

## R04

正确性直觉：
- rotation system 完全决定了“每个顶点处边的局部绕序”
- 局部绕序一旦固定，面边界可由 dart 变换唯一追踪
- 若追踪到的面结构满足 Euler 球面特征 `2`，则该嵌入可视为无交叉平面嵌入

因此，本方法通过“枚举局部绕序 + 全局 Euler 验证”完成判定与构造。

## R05

`demo.py` 的关键数据结构：
- `adj: List[List[int]]`：去重后的无向邻接表
- `rotation: Dict[int, Tuple[int,...]]`：顶点的循环邻接顺序
- `successor[(v, incoming)] = outgoing`：面追踪所需的后继表
- `faces: List[List[int]]`：面边界顶点序列
- `EmbeddingResult`：封装 `is_planar/rotation/faces/states/message`

## R06

算法流程（组件级）：
1. 构建简单无向图（去重边，拒绝自环）
2. 分解连通分量
3. 对每个分量先做必要条件剪枝（Euler 上界、二分图上界）
4. 枚举每个顶点的循环顺序候选
5. 回溯搜索一组完整 rotation
6. 对完整 rotation 追踪全部面
7. 检查 `n-m+f==2`，成立则该分量平面可嵌入
8. 所有分量都成功时给出全图嵌入

## R07

状态空间说明：
- 顶点度为 `d` 时，循环顺序候选数约为 `(d-1)!`（仅对旋转去重，保留方向）
- 总搜索空间近似为各顶点候选数的乘积

该 MVP 是**精确搜索**，适合小中规模图的教学/验证，不追求大规模工业性能。

## R08

实现中的剪枝与稳健性：
- 必要条件 1：`m > 3n-6`（`n>=3`）直接非平面
- 必要条件 2：若分量是二分图且 `m > 2n-4`（`n>=3`）直接非平面
- `max_states` 限制回溯状态，避免组合爆炸导致不可控耗时
- 对孤立点分量直接返回平面结果

## R09

时间复杂度：
- 预处理（建图/连通分量/必要条件）：`O(n+m)`
- 主体搜索：最坏为指数级，
  近似 `O(Π_v ((deg(v)-1)!) * m)`

因此本 MVP 的理论最坏复杂度较高，但在小图上可精确工作。

## R10

空间复杂度：
- 图与辅助结构：`O(n+m)`
- 回溯状态：`O(n)`（当前路径）
- 面追踪中 dart 与访问集合：`O(m)`

总空间约 `O(n+m)`（不含搜索树隐式时间代价）。

## R11

边界情况处理：
- `n=0`：返回空图结果（平面）
- 孤立点：作为单独分量，视为平面且有外部面
- 重边：自动去重（按简单图处理）
- 自环：本 MVP 直接报错（避免把多重图/伪图处理混入最小实现）
- 非连通图：逐分量嵌入并汇总

## R12

常见实现错误：
- 把邻居“线性顺序”误当“循环顺序”，导致面追踪不闭合
- 忘记对循环顺序做“旋转去重”，状态空间被重复放大
- 错把局部镜像当重复删除，会丢失合法嵌入解
- face 追踪时使用了错误变换（应是 `(u->v)->(v->next_v(u))`）
- 只做必要条件剪枝就当作充分条件

## R13

与经典线性算法对比：
- Boyer-Myrvold、Hopcroft-Tarjan 可在 `O(n)` 时间做平面性测试并构造嵌入
- 本 MVP 采用“rotation 系统枚举 + Euler 验证”，复杂度更高但实现透明
- 教学价值：便于直接观察“局部绕序如何决定全局面结构”

## R14

`demo.py` 实现特性：
- 纯 Python 标准库，无交互输入
- 提供 `planar_embedding(...)` 主函数
- 输出可读的 rotation system 与 faces
- 内置 3 个示例：`Cube`（平面）、`K3,3`（非平面）、`K5`（非平面）

## R15

运行方式：

```bash
python3 Algorithms/数学-图论-0502-平面嵌入算法/demo.py
```

脚本会自动执行三个样例并打印结果。

样例预期：

- `K3,3`：应判定非平面（典型 Kuratowski 反例）
- `K5`：应判定非平面（另一 Kuratowski 反例）

## R17

可扩展方向：
- 将枚举法替换为 Boyer-Myrvold 线性算法以支持大图
- 在得到 rotation 后继续生成几何坐标（如 Tutte embedding）
- 支持多重边/自环并扩展到更一般嵌入模型
- 对搜索加入更多局部一致性剪枝（分离对、低点结构等）

## R18

`demo.py` 源码级算法流程（8 步）：

1. `_build_simple_undirected_graph` 对输入边做合法性检查、去重并建邻接表。
2. `_connected_components` 把图拆成连通分量，后续逐分量处理。
3. `_find_component_embedding` 先做 `m<=3n-6` 与二分图 `m<=2n-4` 的必要条件剪枝。
4. `_cyclic_orders` 为每个顶点生成“去旋转重复、保留方向”的循环邻接顺序候选。
5. `dfs` 回溯枚举各顶点候选顺序，形成完整 `rotation`。
6. `_trace_faces` 用 dart 变换 `(u->v)->(v->next_v(u))` 追踪所有面轨道。
7. 对连通分量检查 Euler 条件 `n-m+f==2`；成立则该 rotation 即一个平面嵌入。
8. `planar_embedding` 汇总所有分量结果，返回 `EmbeddingResult` 并在 `main` 中打印示例输出。
