## R01

Graham 扫描（Graham Scan）是二维凸包的经典算法：
- 给定平面点集，输出按逆时针顺序排列的凸包顶点；
- 通过“极角排序 + 栈维护左转”逐步剔除凹陷点；
- 对一般点集时间复杂度为 `O(n log n)`（主要来自排序）。

本目录提供一个可运行 MVP：`demo.py` 纯手写 Graham 扫描主流程，不调用现成黑盒凸包 API。

## R02

问题定义（本实现）：
- 输入：二维点集 `P = {(x_i, y_i)}`，形状 `n x 2`；允许重复点。
- 输出：`HullResult`，包含：
  - `unique_points`：去重后的点集；
  - `hull`：凸包顶点（逆时针，不重复首点）；
  - `removed_duplicates`：被移除重复点数。

特殊情形约定：
- `n = 0`：返回空凸包；
- `n = 1`：凸包是该点；
- 全部共线：返回两端点（若退化成同一点则返回 1 点）。

## R03

核心几何判定：
1. 叉积方向测试
   - `cross(o,a,b) = (a-o) x (b-o)`。
   - `cross > 0` 表示 `o->a->b` 为左转（逆时针）。
2. 枢轴点选择
   - 取 `y` 最小（并列取 `x` 最小）的点作为 `p0`。
3. 极角排序
   - 将其它点按相对 `p0` 的极角 `atan2` 升序排序；
   - 同角度时按到 `p0` 距离升序，便于在扫描时保留更远点。
4. 栈回退规则
   - 新点入栈前，若栈顶两点与新点不是左转（`cross <= eps`），持续出栈。

## R04

算法流程（高层）：
1. 校验输入维度与数值合法性。
2. 用 `eps` 量化哈希做近重复点去重。
3. 处理 `0/1` 点和“全部共线”退化分支。
4. 选枢轴点 `p0`，其余点做极角排序。
5. 初始化栈为 `[p0, first_sorted_point]`。
6. 依次扫描剩余点：若出现非左转则弹栈，直到恢复左转后再入栈。
7. 扫描结束，栈中点序即为逆时针凸包顶点序列。
8. 输出 `HullResult`，并在 `demo` 中做凸性与覆盖性校验。

## R05

数据结构设计：
- `np.ndarray (n,2)`：输入/去重点/凸包点坐标。
- `HullResult`：
  - `unique_points: np.ndarray`
  - `hull: np.ndarray`
  - `removed_duplicates: int`
- 扫描栈：`List[np.ndarray]`，维护当前候选凸包链。

该设计保持最小实现复杂度，便于源码审计。

## R06

正确性直觉：
- 极角排序保证点按围绕枢轴的“环绕顺序”被处理。
- 栈内始终维护一条不产生右转的边界链。
- 一旦出现 `cross <= 0`（右转或共线内点），当前栈顶不会是最终凸包顶点，可安全移除。
- 扫描完所有点后，剩余点正是最外层边界点，且按逆时针排序。

因此得到的多边形是凸的，且包含所有输入点。

## R07

复杂度分析（`n` 为去重后点数）：
- 去重：`O(n)`（哈希期望复杂度）。
- 枢轴选择：`O(n)`。
- 极角排序：`O(n log n)`。
- 扫描入栈/出栈：每点最多进出各一次，总计 `O(n)`。
- 总时间复杂度：`O(n log n)`。
- 空间复杂度：`O(n)`（排序结果与栈）。

## R08

数值稳定性与容差策略：
- `eps` 用于：
  - 去重量化阈值；
  - 左转判定中的 `cross <= eps`；
  - 共线检测阈值。
- 输入必须全部有限值（拒绝 `nan/inf`）。
- 全共线时不强行构造“面积>0”的多边形，而返回端点，避免不稳定结果。

这属于工程化 MVP 的稳健方案，而非精确算术几何核。

## R09

边界情况处理：
- 空输入：返回空凸包。
- 单点输入：返回该点。
- 两点输入：作为共线分支返回两端点。
- 重复点：自动去重并记录数量。
- 含大量共线点：扫描阶段会自动剔除中间共线点，仅保留边界端点。

## R10

`demo.py` 关键函数说明：
- `check_points`：输入校验。
- `deduplicate_points`：按容差去重。
- `cross` / `is_all_collinear`：几何谓词。
- `pivot_index` / `sort_by_polar`：枢轴与极角排序。
- `graham_scan`：主算法。
- `is_strictly_convex_ccw`：检查凸包严格凸且逆时针。
- `all_points_inside_or_on_hull`：检查所有点被凸包覆盖。
- `summarize_case` / `main`：组织样例与打印结果。

## R14

可调参数：
- `eps`（默认 `1e-12`）
  - 增大：更容易把近点判为重复/共线，结果更“粗”；
  - 减小：几何判定更敏感，但更易受浮点噪声影响。

实践建议：
- 坐标量级较大或噪声明显时，可先做坐标归一化再执行凸包。

## R15

与常见凸包方法对比：
- Graham 扫描：
  - 优点：实现直接、教学友好、`O(n log n)`；
  - 局限：依赖全局排序，不是在线算法。
- Jarvis March（礼物包装）：
  - 复杂度 `O(nh)`（`h` 为凸包点数），当 `h` 小时有优势。
- Andrew Monotone Chain：
  - 也为 `O(n log n)`，实现上常更短（按 x 排序构建上下壳）。

## R18

`demo.py` 源码级算法流程拆解（8 步）：
1. `main` 构造三组二维测试点（随机云、方形+内点+重复、共线点）。
2. `summarize_case` 调用 `graham_scan`，拿到 `HullResult`。
3. `graham_scan` 内先执行 `check_points` 与 `deduplicate_points`，清理非法/重复输入。
4. 对 `0/1` 点或 `is_all_collinear=True` 的输入直接走退化返回（端点或空壳）。
5. 对一般情况，`pivot_index` 选最低点 `p0`，`sort_by_polar` 对其余点做极角+距离排序。
6. 初始化扫描栈后逐点处理：当 `cross(stack[-2], stack[-1], p) <= eps` 时持续弹栈，直到恢复左转。
7. 新点入栈，循环结束后栈内容即为逆时针凸包顶点序列（不重复首点）。
8. `summarize_case` 再调用 `is_strictly_convex_ccw` 与 `all_points_inside_or_on_hull` 校验结果并打印摘要。
