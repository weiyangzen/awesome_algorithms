## R01

最小二乘法用于求解“方程组通常无精确解”的参数估计问题。

给定设计矩阵 `A in R^{m*n}`（通常 `m>=n`）和观测向量 `b in R^m`，目标是找到 `x in R^n`，使残差平方和最小：

`min_x ||Ax - b||_2^2`

它本质上是在欧氏空间里找“`b` 在 `A` 的列空间上的最佳近似投影”。

## R02

核心思想：把“过定约束下无法同时满足所有方程”转化为“使误差能量最小”。

- 若 `Ax=b` 无解，最小二乘仍可给出最优近似解。
- 几何上，最优残差 `r=b-Ax*` 与列空间 `Col(A)` 正交。
- 这个正交条件给出正规方程：`A^T A x = A^T b`。
- 在工程里常通过 QR / SVD 解最小二乘，数值稳定性通常优于直接解正规方程。

## R03

输入与输出：

- 输入:
  - `A`: 形状 `(m, n)` 的实数矩阵。
  - `b`: 形状 `(m,)` 的实数向量。
  - 可选 `ridge_lambda>=0`（用于 `A^T A + lambda I` 正则化）。
- 输出:
  - 近似最优解 `x`。
  - 残差范数 `||Ax-b||_2`。
  - 可选诊断量：`rank(A)`、`cond(A)`、`cond(A^T A)`、与参考解差异。

## R04

适用前提与边界：

- 数据关系可由线性模型近似描述。
- `m>=n` 时常见（回归）；`m<n` 时问题欠定，解不唯一，需要最小范数约束或正则化。
- 当 `A` 列相关严重、病态或秩亏时，直接正规方程不稳健。
- 实务中推荐优先 `lstsq`（底层 QR/SVD 路径）或显式正则化。

## R05

数学推导（从目标函数到最优条件）：

设

`f(x) = 1/2 ||Ax-b||_2^2 = 1/2 (Ax-b)^T (Ax-b)`

对 `x` 求梯度：

`grad f(x) = A^T(Ax-b)`

令梯度为零得到一阶最优条件：

`A^T A x = A^T b`

若 `A` 满列秩，`A^T A` 对称正定，解唯一；若秩亏，最小二乘解一般不唯一，需 SVD/伪逆给出最小范数解。

## R06

伪代码：

```text
输入 A, b, ridge_lambda
检查维度合法性

# 路径1: 正规方程（可选正则）
G <- A^T A
c <- A^T b
if ridge_lambda > 0:
    G <- G + ridge_lambda * I
尝试 x_ne <- solve(G, c)
若奇异则回退 x_ne <- pinv(G) @ c

# 路径2: 稳健参考
x_ref <- lstsq(A, b)

输出 x_ne, x_ref, 各自残差和差异指标
```

## R07

正确性要点：

1. `f(x)=1/2||Ax-b||^2` 是凸二次函数。
2. 任意全局最优点必须满足 `A^T(Ax-b)=0`（一阶条件）。
3. 该条件等价于残差向量对 `Col(A)` 正交。
4. 满列秩时 Hessian `A^T A` 正定，最优解唯一。
5. 秩亏时最优解构成仿射集合，`lstsq`/SVD 返回其中最小二范数解。

## R08

复杂度（`A` 为 `m*n`）：

- 构造 `A^T A`: `O(m n^2)`
- 构造 `A^T b`: `O(m n)`
- 解 `n*n` 线性系统: `O(n^3)`
- 正规方程总成本: `O(m n^2 + n^3)`
- 空间开销（额外）: `O(n^2)`（存 Gram 矩阵）

`lstsq` 的常数项更大，但稳定性通常更好。

## R09

数值稳定性：

- 直接正规方程会放大条件数：`cond(A^T A) ~= cond(A)^2`。
- 当特征共线或尺度差异大时，浮点误差会被明显放大。
- 常见缓解策略：
  - 特征标准化；
  - 岭正则（`lambda I`）；
  - 使用 QR/SVD 路径（例如 `np.linalg.lstsq`）。

## R10

常见解法对比：

- 正规方程:
  - 优点：实现最直接，`m>>n` 时较快。
  - 缺点：稳定性最弱，秩亏时易失败。
- QR 分解:
  - 优点：稳定性更好，适合作为通用方案。
  - 缺点：比正规方程常数更大。
- SVD / `lstsq`:
  - 优点：最稳健，可处理秩亏并返回最小范数解。
  - 缺点：计算代价通常最高。

## R11

工程常见坑：

- 忘记添加截距列（全 1 列），导致模型偏置缺失。
- 直接在强共线特征上解正规方程，得到不稳定参数。
- 仅看训练残差，不看条件数/参数范数，误判“模型可靠”。
- 数据类型不一致（整数/低精度）引入额外误差。

## R18

第三方库并非黑箱。以 `np.linalg.lstsq(A, b)` 的典型实现链路为例，可拆成 8 步：

1. Python 层接收 `A, b`，检查维度并做 dtype/内存布局标准化。
2. `numpy.linalg` 将调用转发到其 C 扩展线性代数入口（`_umath_linalg`）。
3. 根据矩阵形状与驱动配置，选择 LAPACK 最小二乘内核（常见 `xGELSD`，也可能是 `xGELS`/`xGELSS`）。
4. 若走 `xGELSD`，先对 `A` 做双对角化并通过分治 SVD 计算奇异值与右奇异向量相关结构。
5. 按奇异值阈值 `rcond` 判定有效秩，过滤近零奇异值，避免不稳定除法。
6. 在有效子空间中求解最小二乘意义下的最小范数解 `x`。
7. 返回 `x`、残差平方和、估计秩 `rank`、奇异值 `s` 到 Python 层。
8. 调用方（本 MVP）再计算 `||Ax-b||_2`、条件数和与其他解法差异，完成可解释验证。

对比之下，正规方程路径只是把同一问题改写为 `solve(A^T A, A^T b)`；稳定性差异来自“是否显式平方条件数”和“底层分解策略”不同。
