# 学习率调度

- UID: `MATH-0402`
- 学科: `数学`
- 分类: `深度学习`
- 源序号: `402`
- 目标目录: `Algorithms/数学-深度学习-0402-学习率调度`

## R01

学习率调度（Learning Rate Scheduling）是在训练过程中按预设规则动态调整学习率 `lr_t` 的方法。它不是独立优化器，而是“优化器外层控制器”：

- 优化器决定“用什么梯度更新公式”；
- 调度器决定“每一步用多大步长”。

常见目标是：前期用较大步长快速下降，后期减小步长稳定收敛并提升泛化。

## R02

发展脉络大致如下：

- 早期：固定学习率（constant lr）最简单，但经常出现“前期慢/后期抖”。
- 经典阶段：Step Decay、Exponential Decay 等分段或指数衰减被广泛使用。
- 深度学习普及后：Cosine Decay、Warmup+Decay、OneCycle、Cyclic LR 等成为主流。
- 大模型训练中：Warmup 几乎是标配，后接 cosine/linear 衰减，兼顾稳定性与最终性能。

调度并非“越复杂越好”，而是需要匹配模型规模、batch size、训练总步数和噪声水平。

## R03

学习率调度主要解决三类问题：

1. 固定学习率难兼顾训练早中后期需求。
2. 初始学习率过大可能导致梯度爆炸或 loss 发散。
3. 后期若学习率不降，模型可能在最优点附近震荡，难以细化收敛。

对应策略：

- Warmup：前几轮逐步升高学习率，降低早期不稳定风险。
- 衰减（Step/Cosine/Exponential）：中后期逐步减小步长，利于收敛与泛化。

## R04

典型调度公式如下（`t` 为当前 epoch/step）：

- Constant: `lr_t = lr0`
- Step Decay: `lr_t = lr0 * gamma^k`，`k` 为已跨过里程碑数量
- Exponential: `lr_t = lr0 * gamma^t`
- Cosine: `lr_t = lr_min + 0.5*(lr0-lr_min)*(1+cos(pi*progress))`
- Warmup + Cosine:
  - 若 `t < T_warmup`：`lr_t = lr0 * (t+1)/T_warmup`
  - 否则按 cosine 从 `lr0` 衰减到 `lr_min`

本目录 MVP 实现了 `Constant / Step / Warmup+Cosine` 三种，且每轮显式写回 optimizer 的 `param_groups`。

## R05

设参数量为 `P`、训练轮数为 `T`。

- 调度器本身每轮计算通常是 `O(1)`；
- 与优化更新 `O(P)` 相比，调度器额外开销可忽略；
- 总体训练开销仍主要由前向+反向决定。

因此学习率调度通常是“低成本高收益”的训练组件。

## R06

简化示例（`lr0=0.1`，总 10 轮）：

- Constant：每轮都 `0.1`。
- Step（里程碑 `[4, 7]`, `gamma=0.1`）：
  - epoch 0-3: `0.1`
  - epoch 4-6: `0.01`
  - epoch 7-9: `0.001`
- Warmup+Cosine（warmup=2, `lr_min=0.005`）：
  - epoch 0: `0.05`
  - epoch 1: `0.1`
  - 后续按余弦逐步降到接近 `0.005`

可见三者体现了“恒定、突降、平滑降”的不同节奏。

## R07

优点：

- 往往能显著改善收敛速度与最终精度。
- 减少手动反复试错固定学习率的成本。
- 与大多数优化器兼容（SGD、AdamW 等）。

局限：

- 需要额外超参数（warmup 比例、milestones、gamma 等）。
- 训练步数变化时，原调度配置可能失效。
- 不同任务最优曲线差异大，没有单一万能策略。

## R08

建议具备的前置知识：

- 梯度下降与随机优化基本原理。
- 训练/验证曲线解读（欠拟合、过拟合、震荡）。
- 常见优化器（SGD/AdamW）与 batch size 对有效步长的影响。
- 基础数值稳定手段（梯度裁剪、标准化、合理初始化）。

## R09

适用场景：

- 几乎所有深度学习训练流程。
- 大模型或大 batch 训练（通常需要 warmup）。
- 需要在固定训练预算内提高收敛质量。

谨慎场景：

- 训练轮数极少（调度收益有限）。
- 在线学习或步数不确定场景（固定里程碑不易定义）。
- 已经做强自适应控制且对调度不敏感的小模型任务。

## R10

实现正确性的关键检查点：

1. 学习率更新时机一致（按 epoch 或按 step，不能混用）。
2. 每次更新后确实写入 optimizer 参数组。
3. warmup 阶段应单调上升到目标峰值。
4. step 里程碑处应发生预期倍数下降。
5. 日志中记录的 lr 与实际训练使用值一致。

本目录 `demo.py` 对这些性质做了简单断言。

## R11

数值与训练稳定策略（本 MVP 使用）：

- 对输入特征做标准化，减少病态尺度。
- SGD + momentum 搭配梯度裁剪，降低异常梯度风险。
- Warmup 防止初期大步长破坏参数分布。
- 后期降学习率，缓解最优点附近来回震荡。
- 固定随机种子，便于复现实验曲线。

## R12

调参建议（经验起点）：

- `base_lr`：先做对数网格（如 `1e-3 ~ 1e-1`）。
- `warmup_epochs`：常取总轮数 `3% ~ 10%`。
- Step 调度：里程碑常放在 `50%/75%` 进度附近。
- `gamma`：常见 `0.1` 或 `0.2`。
- Cosine 调度：设置合理 `min_lr_ratio`（如 `0.02 ~ 0.1`）。

原则是先保证稳定收敛，再追求更激进的前期速度。

## R13

理论性质说明：

- 学习率调度本身不提供“近似比”这类组合优化保证。
- 在凸优化中，衰减步长通常有经典收敛分析；
- 在深度网络非凸场景中，主要依赖经验表现与泛化结果。

因此它是强工程实用工具，而非带统一最优性保证的算法模板。

## R14

常见失效模式与防护：

- 失效：前期 loss 爆炸。  
  防护：减小 `base_lr`，增加 warmup，启用梯度裁剪。
- 失效：后期 loss 停滞或抖动。  
  防护：增加衰减力度或延长训练轮数。
- 失效：调度太激进导致欠拟合。  
  防护：提高 `min_lr` 或减少衰减次数。
- 失效：日志 lr 与实际 lr 不一致。  
  防护：统一从 param group 读取并记录。

## R15

工程实践建议：

- 把“调度配置”作为实验配置的一等公民（与模型结构同等记录）。
- 对比实验需固定随机种子和初始化，避免误判调度收益。
- 监控 `lr/loss/accuracy` 三条曲线，定位问题更直接。
- 先在小数据快速验证调度逻辑，再扩大到正式训练规模。

## R16

与其他技术的关系：

- 与优化器：调度器控制全局步长，优化器控制梯度变换。
- 与正则化：两者都影响泛化，但作用机制不同。
- 与 batch size：大 batch 往往需要更长 warmup。
- 与早停（early stopping）：调度改善收敛轨迹，早停控制训练终点。

常见组合是 `AdamW + warmup + cosine` 或 `SGD+momentum + step`。

## R17

本目录 `demo.py` 的 MVP 内容：

- 使用 `PyTorch` 构建一个小型二层 MLP，训练 moons 二分类任务。
- 手写三种调度器：`ConstantLRScheduler`、`StepLRScheduler`、`WarmupCosineScheduler`。
- 每轮显式设置 optimizer 学习率并记录轨迹，不调用黑盒 scheduler API。
- 输出三种调度的初末 loss、测试精度与关键学习率点，并做基本断言。

运行方式：

```bash
cd Algorithms/数学-深度学习-0402-学习率调度
uv run python demo.py
```

## R18

`demo.py` 的源码级流程拆解（8 步）：

1. `make_moons_dataset` 生成二分类数据，随后标准化并切分训练/测试集。  
2. `TinyMLP` 定义两层全连接网络（ReLU 隐层 + 线性输出），用于二分类 logits。  
3. 三个调度类实现 `lr_at(epoch)`：常数、里程碑阶梯衰减、warmup 后余弦衰减。  
4. `train_with_scheduler` 每轮先调用 `scheduler.lr_at` 计算学习率，再写入 `optimizer.param_groups`。  
5. 在当前学习率下执行前向、`BCEWithLogitsLoss`、反向传播、梯度裁剪与 `optimizer.step`。  
6. 同步记录 `lr_history` 与 `loss_history`，得到可审计的“步长-损失”轨迹。  
7. `evaluate_accuracy` 在测试集计算精度，`summarize_result` 汇总每种调度的最终表现。  
8. `main` 并行运行三种调度配置，打印对比表并断言：loss 下降、warmup 先升后降、step 在里程碑处跳变。

该实现把学习率调度逻辑显式写成可追踪源码，而不是把它隐藏在框架黑盒里。
