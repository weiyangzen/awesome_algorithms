## R01

置信传播（Belief Propagation, BP）是图模型中的消息传递推断算法，用于计算变量的边缘后验或近似边缘后验。最常见形式是和-积（sum-product）BP，在因子图或成对马尔可夫随机场上迭代传递消息：

- 变量节点接收来自邻居因子的消息并相乘，得到“信念”（belief）。
- 因子节点把局部势函数与其他输入消息相乘后，对本变量以外的维度求和，再发回变量节点。

在树图上，BP 是精确推断；在含环图上，通常称为“Loopy BP”，是近似推断。

## R02

BP 源于概率图模型与编码理论的交叉发展：

- 在 1980s-1990s，Pearl 等工作把局部消息传递系统化为贝叶斯网络/马尔可夫网络推断工具。
- 在纠错码（如 LDPC、Turbo）中，BP 成为核心解码机制。
- 后续在计算机视觉、NLP、推荐系统、生物信息等领域广泛用于近似边缘推断。

今天 BP 仍是“结构化概率推断”的基础方法之一，尤其适合稀疏图与可分解势函数。

## R03

BP 主要解决的问题是：

1. 变量间存在依赖（图结构约束），不能把每个变量独立估计。
2. 直接枚举所有状态组合复杂度指数级（例如二元变量 `n` 个需 `2^n`）。
3. 需要边缘概率 `P(x_i | evidence)` 或局部联合概率 `P(x_i, x_j | evidence)`，而不只是单一最优解。

BP 通过“局部消息复用”把全局推断分解成邻接计算，显著降低树图上的计算复杂度。

## R04

和-积 BP 的核心递推（成对 MRF 视角）如下。设一元势 `phi_i(x_i)`、二元势 `psi_{ij}(x_i, x_j)`：

- 消息更新：
  `m_{i->j}(x_j) = sum_{x_i} phi_i(x_i) * psi_{ij}(x_i, x_j) * prod_{k in N(i) \ {j}} m_{k->i}(x_i)`
- 节点信念：
  `b_i(x_i) propto phi_i(x_i) * prod_{k in N(i)} m_{k->i}(x_i)`

在树上可用“两趟传播”完成：

1. 从叶到根（收集证据）做上行消息。
2. 从根到叶（分发上下文）做下行消息。

## R05

设变量数 `n`，每个变量离散状态数 `K`，边数 `|E|`。

- 树图 BP：每条有向边更新一次消息，单次消息约 `O(K^2)`，总复杂度约 `O(|E| * K^2)`。
- 节点边缘恢复：`O(n * deg * K)`，通常低于消息传递主成本。
- 空间复杂度：存储有向边消息约 `O(|E| * K)`。

对二元变量（`K=2`）而言，复杂度几乎线性于图规模。

## R06

直观例子：医疗诊断中的症状-疾病依赖网络。

- 变量可表示疾病状态或症状状态（有/无）。
- 一元势编码先验或观测证据。
- 二元势编码“疾病影响症状”的相关性。

BP 把某个症状节点的证据通过邻接关系逐步扩散，最终更新所有相关节点的后验信念，而不需要全局穷举所有疾病组合。

## R07

优势：

- 树图上精确、快速、可解释。
- 稀疏图上计算代价低，易并行化。
- 统一处理“有证据/无证据”的边缘推断问题。

局限：

- 含环图上不保证收敛，收敛也不保证全局精确。
- 高阶因子或大状态空间时，消息维度可能很大。
- 势函数建模不当时，推断结果会系统性偏差。

## R08

前置知识建议：

- 概率论：边缘化、条件概率、归一化常数。
- 图论：树、邻接关系、有向/无向边。
- 线性代数：向量化乘法、广播。
- 数值计算：归一化、防下溢。

## R09

适用场景：

- 树状或近树状依赖结构（层级诊断、链式时序局部模型）。
- 纠错码解码、稀疏图概率推断。
- 需要节点边缘后验而非单点预测的任务。

不太适用：

- 图非常稠密且环很强，Loopy BP 易震荡。
- 状态空间极大且势函数不可分解。
- 对全局最优与严格误差界有强保证需求时。

## R10

实现正确性检查要点：

1. 每条消息都应归一化后非负且和为 1。
2. 每个节点信念 `b_i` 归一化后和为 1。
3. 在树图上，BP 边缘应与穷举精确边缘一致（允许小浮点误差）。
4. 若计算边缘配分函数估计，应保证与直接穷举同号且合理。

本目录 demo 使用“树图 + 穷举真值”做主验证。

## R11

数值稳定策略：

- 每次消息更新后立即归一化，防止连乘导致上溢/下溢。
- 对归一化分母设置 `eps` 下界（如 `1e-15`），避免除零。
- 势函数保持正值（如由 `exp` 或加小常数构造），避免无效概率质量。

对超大图常见改造是改为 log-domain BP，但本 MVP 保持最小实现。

## R12

调参与实现经验：

- 图含环时可加 damping：`m_new = alpha * m_new + (1-alpha) * m_old`。
- 初始化可用均匀消息，必要时多随机初值对比收敛。
- Loopy BP 可设最大迭代步与残差阈值双停止条件。
- 势函数尺度差异过大时先做缩放，减少数值不稳定。

本 demo 使用树图，因此无需阻尼与多轮迭代，强调“精确两趟传播”。

## R13

理论性质（核心保证）：

- 在无环图（树/森林）上，和-积 BP 在有限次传递后给出精确边缘分布。
- 其正确性来自树分解与条件独立结构，等价于动态规划式消元。
- 在含环图上，BP 通常仅为近似算法，不具备普适收敛与误差界保证。

## R14

常见失效模式与防护：

- 失效：消息全零或接近零（势函数过尖锐）。
  - 防护：势函数加平滑、消息归一化加 `eps`。
- 失效：Loopy BP 振荡不收敛。
  - 防护：阻尼、残差监控、改用变分/采样方法。
- 失效：图结构或势函数方向写反，结果看似“合理但错”。
  - 防护：用小图穷举对照测试（本 demo 的主验收手段）。

## R15

工程实践建议：

- 把“图结构”“消息更新”“边缘恢复”“真值验证”解耦成独立函数。
- 固定随机种子，保证可复现实验与回归测试。
- 对每次改动保留小规模可穷举单测，避免悄然退化。
- 在生产系统中记录消息残差与迭代次数，便于诊断收敛问题。

## R16

与相关算法关系：

- 变量消元（Variable Elimination）：本质同类精确推断，BP 更偏局部并行消息视角。
- Junction Tree：把一般图转为团树后做消息传递，是 BP 的泛化精确框架。
- 均值场（Mean Field）：更便宜但通常更粗糙的变分近似。
- MCMC：采样近似后验，精度-速度权衡与 BP 不同。

## R17

本目录 `demo.py` 提供一个可运行 MVP：

- 构造一个二元变量的树形成对 MRF（随机一元势与二元势）。
- 使用两趟和-积 BP 计算每个节点边缘概率。
- 使用穷举 `2^n` 状态计算精确边缘。
- 比较两者最大绝对误差并断言误差足够小。

```bash
cd Algorithms/数学-统计-0326-置信传播

```

无需任何交互输入，成功时会输出 `All checks passed.`。

## R18

`demo.py` 源码级算法流程（8 步）：

1. `build_random_tree_mrf` 生成树结构、二元一元势 `phi` 与二元势 `psi`（全部正值并可复现）。
2. `build_parent_order` 以根节点做 BFS，得到父子关系与遍历顺序，为两趟消息传递做拓扑准备。
3. 上行阶段（叶到根）：按逆序对每个 `child -> parent` 调用 `compute_message`，聚合来自子树的信息。
4. 下行阶段（根到叶）：按正序对每个 `parent -> child` 调用 `compute_message`，把祖先上下文传向叶节点。
5. `belief_from_messages` 把入边消息与节点一元势相乘并归一化，得到每个节点边缘信念 `b_i`。
6. `enumerate_exact_marginals` 穷举全部状态组合，计算归一化后精确边缘，作为基准真值。
7. 在 `main` 中比较 BP 与真值的逐节点误差，输出最大绝对误差与每个节点后验。
8. 通过断言（误差阈值、消息/边缘归一化合法性）形成自动验收闭环。

该 MVP 没有调用图推断黑盒库，而是把 BP 的消息更新与双向传播路径完整展开为可审计源码。
