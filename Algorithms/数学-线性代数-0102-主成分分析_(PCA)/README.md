# 主成分分析 (PCA)

- UID: `MATH-0102`
- 学科: `数学`
- 分类: `线性代数`
- 源序号: `102`
- 目标目录: `Algorithms/数学-线性代数-0102-主成分分析_(PCA)`

## R01

PCA（Principal Component Analysis）用于把高维特征线性变换到一组彼此正交的新坐标轴（主成分），并按方差大小排序。

本条目 MVP 目标：
- 在可复现数据上实现“标准化 + 协方差矩阵 + 特征分解”的完整流程；
- 自动选择满足累计解释方差阈值的主成分个数；
- 给出降维表示与重构误差，并做自动正确性校验。

## R02

历史定位（简要）：
- PCA 最早由 Pearson（1901）和 Hotelling（1933）系统化，是经典线性降维方法；
- 在统计学、信号处理、机器学习中长期作为基线方法；
- 相比深度非线性降维，PCA 具有可解释、训练快、闭式解明确等优点；
- 在工业实践中常用于特征压缩、去噪、可视化前处理。

## R03

MVP 输入输出约定（无交互输入）：
- 输入（脚本内部固定生成）：
1. `build_demo_data()` 构造 `480 x 6` 的相关特征数据；
2. `variance_threshold=0.95`（自动选维）；
3. `standardize=True`（按列标准化）。
- 输出：
1. 选出的主成分个数 `selected_n_components`；
2. 前若干特征值与解释方差比；
3. 累计解释方差序列；
4. 重构 RMSE 与 `All checks passed.`。

## R04

典型应用场景：
- 特征降维：降低模型输入维度与训练成本；
- 去噪：保留主方差方向，抑制小方差噪声分量；
- 可视化：把高维样本映射到 2D/3D；
- 多重共线性处理：在回归前做正交化表征；
- 作为后续算法（聚类、分类、异常检测）的预处理步骤。

## R05

数学要点：
- 给定中心化数据矩阵 `X`（`n x d`），样本协方差矩阵为
  `C = X^T X / (n-1)`；
- 求解 `C v_i = lambda_i v_i`，其中 `lambda_i` 为第 `i` 个主方向方差；
- 按 `lambda` 降序排序，主成分方向由对应特征向量给出；
- 前 `k` 个方向组成投影矩阵 `W_k`，降维表示 `Z = X W_k`；
- 解释方差比：`ratio_i = lambda_i / sum_j lambda_j`。

## R06

本实现的主流程（算法层面）：
1. 输入校验（维度、有限值、`n_components` 合法性）；
2. 特征中心化与标准化（避免尺度差异主导 PCA）；
3. 显式构造协方差矩阵；
4. 对称特征分解 `eigh`；
5. 按特征值降序重排主成分；
6. 根据阈值自动选择最小 `k`；
7. 计算降维表示 `Z` 与重构 `X_hat`；
8. 进行正交性、谱一致性、重构质量等自动检查。

## R07

选维策略：
- 若显式给定 `n_components`，直接使用该值；
- 否则按累计解释方差阈值选维：
  `k = min { i | sum_{j<=i} ratio_j >= threshold }`；
- 本 demo 使用 `threshold=0.95`，保证保留至少 95% 方差信息；
- 这种策略比固定维数更稳健，适合不同数据集规模。

## R08

时间复杂度（设样本数 `n`，特征维 `d`）：
- 标准化：`O(nd)`；
- 协方差构造：`O(nd^2)`；
- 对称特征分解：`O(d^3)`；
- 投影与重构：`O(ndk)`。

总体可近似看为 `O(nd^2 + d^3)`，当 `d` 较大时主要由谱分解主导。

## R09

空间复杂度：
- 数据与中间矩阵：`O(nd)`；
- 协方差矩阵：`O(d^2)`；
- 特征向量矩阵：`O(d^2)`；
- 低维表示：`O(nk)`。

因此总空间约为 `O(nd + d^2)`。

## R12

输出字段说明：
- `data_shape`：输入样本矩阵形状；
- `selected_n_components`：按阈值选出的主成分数；
- `top_eigenvalues`：协方差谱中最大的若干特征值；
- `top_explained_variance_ratio`：对应解释方差比；
- `cumulative_explained_variance`：累计解释方差序列；
- `reconstruction_rmse`：从低维重构回原空间后的均方根误差；
- `transformed_shape`：降维后矩阵形状。

## R13

鲁棒性与边界处理：
- 输入必须是二维数值矩阵，且样本数至少为 2；
- 非有限值（NaN/Inf）直接报错；
- `n_components` 不得超过 `min(n-1, d)`；
- 标准差过小的特征自动以 `1.0` 替代，避免除零；
- 特征分解后将极小负特征值截断为 0，降低浮点误差影响。

## R14

当前局限：
- 仅支持线性降维，无法建模强非线性流形；
- 对异常值敏感，极端离群点会扭曲协方差结构；
- 解释方差最大不等于任务目标最优（如分类可分性）；
- 本 MVP 不包含增量式 PCA、稀疏 PCA 等扩展版本。

## R15

可扩展方向：
- 增加白化（whitening）选项；
- 增量 PCA：支持流式数据与大规模分块处理；
- 使用随机 SVD 加速超高维场景；
- 与下游任务联合：PCA + 线性模型/聚类统一评估；
- 增加稳健 PCA（如低秩+稀疏分解）以提升抗异常值能力。

## R16

最小测试建议：
- 正确性：与 SVD 推导的特征值对齐；
- 正交性：`components @ components.T` 接近单位阵；
- 单调性：特征值降序、解释方差和约等于 1；
- 有效性：重构误差优于“仅用均值重构”基线；
- 边界性：`n_components` 非法、含 NaN 输入时应抛异常。

## R17

方案对比：
- 直接调用 `sklearn.PCA`：开发最快，但不便展示核心线代流程；
- 基于 SVD 的 PCA：数值稳定，且在某些维度关系下更高效；
- 本实现（协方差 + `eigh`）：概念直观，便于教学与审计；
- 非线性方法（Kernel PCA/Autoencoder）：表达能力强，但复杂度与可解释性成本更高。

## R18

`demo.py` 的源码级流程（8 步）如下：
1. `build_demo_data()` 先生成 3 维潜变量，再经线性混合与噪声注入构造 `480x6` 的相关特征数据。  
2. `validate_inputs()` 检查输入是否二维、有限、样本数是否合法，以及 `n_components` 是否超过有效秩。  
3. `standardize_features()` 对每列做中心化与标准化，并对近零标准差做保护，得到 `X_std`。  
4. `pca_fit_transform()` 显式计算协方差 `C = X_std^T X_std/(n-1)`，并用 `np.linalg.eigh(C)` 求特征值/向量。  
5. 将特征值降序重排后计算 `explained_variance_ratio`，再通过 `choose_components_by_variance()` 找到满足阈值的最小 `k`。  
6. 用前 `k` 个主轴做线性投影 `Z = X_std W_k`，得到低维表示 `transformed_`。  
7. 将 `Z` 反投影并逆标准化，得到 `reconstructed_`，用于衡量信息损失。  
8. `run_checks()` 依次验证正交性、谱排序、方差比归一性、重构优于均值基线，并与 SVD 谱交叉校验后在 `main()` 输出报告。  

说明：`numpy.linalg.eigh` 仅负责“对称矩阵特征分解”这一步算子；数据预处理、排序选维、投影重构、质量校验都在源码中显式实现。
