## R01

卡方检验（Chi-square test）是针对“频数数据”的经典假设检验方法，核心思想是比较“观测频数 `O`”与“原假设下期望频数 `E`”之间的偏离程度：

- 若偏离较小，说明数据与原假设相容；
- 若偏离显著偏大，说明数据不支持原假设。

本条目覆盖两个最常见版本：

- 拟合优度检验（Goodness-of-Fit）：检验单个分类变量是否符合给定分布；
- 独立性检验（Independence）：检验列联表中两个分类变量是否独立。

## R02

方法定位（简要）：

- Pearson 在 1900 年提出卡方检验框架；
- 它是离散统计推断里最常用、最易解释的检验之一；
- 广泛用于问卷分析、A/B 结果分层、医学分组计数、质量抽检和异常分布检测。

与均值类检验（如 t 检验）不同，卡方检验直接作用于计数结构，而不是连续变量均值。

## R04

拟合优度卡方检验定义：

- 原假设：`H0` 下每个类别概率已给定（或可换算为期望频数）；
- 统计量：`X^2 = sum((O_i - E_i)^2 / E_i)`；
- 自由度：`df = k - 1 - ddof`（`k` 为类别数）；
- p 值：`p = P(ChiSquare(df) >= X^2)`。

当 `p < alpha` 时，拒绝 `H0`，说明观测分布与期望分布差异显著。

## R05

列联表独立性卡方检验定义：

- 输入 `r x c` 列联表 `O`；
- 在独立性原假设下，期望频数 `E_ij = (row_i_sum * col_j_sum) / n`；
- 统计量：`X^2 = sum((O_ij - E_ij)^2 / E_ij)`；
- 自由度：`df = (r - 1) * (c - 1)`。

该检验回答的是“变量是否独立”，不是“是否存在因果关系”。

## R06

复杂度：

- 拟合优度检验：`O(k)` 时间，`O(k)` 空间；
- 独立性检验：`O(r*c)` 时间，`O(r*c)` 空间。

MVP 主要是向量化聚合和分布尾概率计算，无迭代优化过程。

## R07

简化手算示例（拟合优度）：

假设六面骰子投掷 60 次，观测频数 `O=[8,10,12,10,9,11]`，公平骰期望 `E=[10,10,10,10,10,10]`。

1. 逐项计算 `(O-E)^2/E`：`0.4,0,0.4,0,0.1,0.1`。  
2. `X^2 = 1.0`。  
3. `df = 6-1=5`。  
4. 对应 p 值较大（大于 0.05），通常不拒绝“公平骰”假设。

这个例子说明：轻微波动不必然显著。

## R08

适用前提与经验条件：

- 观测应来自独立抽样或近似独立过程；
- 计数数据必须非负，且总样本量应足够；
- 经验上多数单元格 `E >= 5` 更适合卡方近似；
- 若样本过小或稀疏严重，优先考虑 Fisher 精确检验或合并类别。

## R09

标准检验流程：

1. 明确 `H0/H1` 与显著性水平 `alpha`；
2. 计算期望频数 `E`；
3. 计算 `X^2` 与 `df`；
4. 由卡方分布求 p 值；
5. 给出拒绝/不拒绝结论；
6. 同时报告效应量和残差，避免只看 p 值。

## R10

本 MVP 的正确性校验：

- 拟合优度结果与 `scipy.stats.chisquare` 做逐项对齐；
- 独立性检验结果与 `scipy.stats.chi2_contingency(correction=False)` 对齐；
- 对 `chi2`、`p_value`、`df` 和期望频数做数值断言。

这保证手写实现不是“看起来正确”，而是可执行可验证。

## R11

数值与实现稳健性：

- 统一使用 `float64`；
- 输入必须有限数值，拒绝 `NaN/Inf`；
- 期望频数要求严格大于 0，避免除零；
- 对拟合优度中期望和观测总和不一致的输入，代码会自动按总量缩放期望频数。

## R12

解释结果常见误区：

- “拒绝独立”不等于“证明因果”；
- p 值不是“原假设为真的概率”；
- 大样本下很小差异也可能显著，需结合效应量；
- “不显著”不等于“完全相同”，可能是统计功效不足。

## R13

与其他检验方法关系：

- Fisher 精确检验：适合 2x2 小样本稀疏计数；
- G-test（Likelihood-ratio）：与卡方检验近似同目标，但基于似然比；
- 二项检验/比例检验：适合二分类比例问题；
- 多类别情况下也可结合 Bootstrap 或贝叶斯后验做稳健性补充。

## R14

常见失败模式与防护：

- 失败：低期望频数单元格过多仍直接用卡方近似。  
  防护：合并类别或改用精确方法。
- 失败：把百分比当频数直接代入。  
  防护：先恢复样本计数或统一到可比总量。
- 失败：只报显著不报强度。  
  防护：同时输出 `phi/Cramer's V` 与残差。
- 失败：把相关当因果。  
  防护：在报告里明确“独立性检验仅反映关联结构”。

## R15

`demo.py` 的实现结构：

- `goodness_of_fit_chi_square_manual`：手写拟合优度检验；
- `independence_chi_square_manual`：手写列联表独立性检验；
- `_prepare_expected_counts`：从期望频数或期望概率构造 `E`；
- `compare_with_scipy`：与 SciPy 对照并断言一致；
- `main`：构造两组 GOF 场景和两组独立性场景，输出结果表。

## R16

工程实践建议：

- 在日志中保留原始频数表与预处理策略；
- 对大量维度批量检验时加多重检验校正（Bonferroni/FDR）；

## R18

`demo.py` 源码级算法流程（8 步，非黑盒）：

1. `main` 固定四组计数数据：两组用于拟合优度（近似公平与明显偏置骰子），两组用于独立性检验（弱关联与强关联列联表）。  
2. `goodness_of_fit_chi_square_manual` 调 `_as_1d_nonnegative` 做输入校验，再由 `_prepare_expected_counts` 把期望概率或期望频数转成与观测总量一致的 `E`。  
3. 在同一函数内按公式 `sum((O-E)^2/E)` 计算 `X^2`，用 `df=k-1-ddof`，再通过 `stats.chi2.sf` 得到 p 值，并给出 `phi=sqrt(X^2/n)`。  
4. `independence_chi_square_manual` 对列联表做二维校验，计算行和、列和与总数，并用外积 `row_sum @ col_sum / n` 得到期望频数矩阵。  
5. 该函数继续按矩阵版卡方公式计算 `X^2`，设 `df=(r-1)(c-1)`，再由 `stats.chi2.sf` 求 p 值，同时输出 `Cramer's V` 与标准化残差。  
6. `compare_with_scipy` 分别调用 `scipy.stats.chisquare` 与 `scipy.stats.chi2_contingency(correction=False)`，对手写结果的 `chi2`、`p`、`df` 和期望频数进行断言。  
7. 若断言通过，说明“核心统计量计算链路”与标准库实现一致，SciPy 在本脚本中仅承担交叉验证角色。  
8. `main` 将四个实验结果汇总为 `pandas.DataFrame`，统一输出 `chi2`、`df`、`p`、拒绝决策及效应量，形成可复核的最小可运行报告。
