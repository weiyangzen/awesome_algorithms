## R01

二次筛法（Quadratic Sieve, QS）是用于分解大整数（尤其是中等位数半素数）的经典数论算法。它通过构造同余平方
`a^2 ≡ b^2 (mod n)`，再利用 `gcd(a-b, n)` 提取非平凡因子。

本目录给出的 `demo.py` 是一个“可运行、可读、诚实”的 MVP：
- 不依赖黑盒第三方分解器。
- 完整覆盖 QS 的关键阶段：因子基、平滑关系、GF(2) 消元、依赖向量提因子。
- 可直接 `python3 demo.py` 运行，无需交互输入。

## R02

核心思想是把“直接分解 n”的难题，转化为“收集足够多的平滑关系并做线性代数”。

设
- `Q(x) = x^2 - n`
- 选定因子基 `FB = {p1, p2, ..., pm}`（通常是小素数，且 `n` 在模 `p` 上是二次剩余）

如果找到许多 `x_i` 使得 `Q(x_i)` 可以完全分解在 `FB` 上（即 B-smooth），则把指数向量按模 2 组成矩阵，找到一组线性相关行，得到
`Π Q(x_i) = y^2`（完全平方）。
又因为 `Π x_i^2 ≡ Π Q(x_i) (mod n)`，于是得到 `a^2 ≡ b^2 (mod n)`，再做 gcd 提因子。

## R03

关键数学点：
1. 同余平方差：若 `a^2 ≡ b^2 (mod n)`，则 `n | (a-b)(a+b)`。
2. 指数奇偶性：若素因子指数和全为偶数，则乘积是完全平方。
3. 二次剩余过滤：对于奇素数 `p`，只有当 `n` 是模 `p` 的二次剩余时，`x^2 ≡ n (mod p)` 才有解，`p` 才值得放入因子基。
4. 线性代数过 `GF(2)`：指数向量只关心奇偶，故在模 2 域上做消元。

## R04

`demo.py` 的输入输出约定：
- 输入：脚本内置一个示例合数 `n = 10007 * 10009`。
- 输出：打印

  - 校验 `factor1 * factor2 == n`

你也可以直接在 `main()` 中修改 `n` 做实验。

## R05

实现中的核心数据结构：
- `Relation`
  - `x`: 关系点
  - `qx`: `x^2 - n`
  - `exponents`: `qx` 在因子基上的指数向量
  - `parity`: `exponents mod 2`
- `QSResult`
  - 保存分解结果和统计信息，便于验证与调参。

矩阵表示采用 Python `List[List[int]]`（元素 0/1），保证可读性优先。

## R06

因子基构造（`build_factor_base`）：
- 枚举 `<= bound` 的素数。
- 特判 `p=2` 加入因子基。
- 对奇素数 `p`，用 `pow(n % p, (p-1)//2, p)` 计算勒让德符号对应值：
  - 值为 `1` 表示二次剩余，加入因子基。
  - 否则跳过。

这是 QS 中降低无效搜索空间的关键筛选。

## R07

平滑关系收集（`find_smooth_relations`）：
- 从 `x = ceil(sqrt(n))` 开始递增尝试。
- 计算 `Q(x)=x^2-n`。
- 用因子基反复试除，若最终余数是 `1`，说明 `Q(x)` 完全由因子基素数构成，即得到一条关系。
- 收集到 `len(FB)+8` 条关系后进入线性代数阶段。

该 MVP 用“试除”代替工程化大筛，代码更短、更透明。

## R08

线性代数阶段（`find_dependencies`）：
- 把每条关系的 `parity` 作为一行，得到 `R x M` 的 0/1 矩阵。
- 用高斯消元（`GF(2)`）做行变换。
- 同步维护 `combos` 位掩码，记录“当前行由哪些原始行 XOR 而来”。
- 消元后的零行对应线性依赖，即若干原始行 XOR 为零。

输出是多个依赖集合（行下标列表），供后续尝试提因子。

## R09

因子提取（`try_dependency`）：
- 对某个依赖集合 `S`：
  - `a = Π x_i mod n`
  - 累加指数向量得到 `sum_exp`
  - `b = Π p_j^(sum_exp_j/2) mod n`
- 计算 `gcd(a-b, n)` 与 `gcd(a+b, n)`。
- 若结果在 `(1, n)` 内，则得到非平凡因子。

并非所有依赖都成功，所以脚本会按依赖逐个尝试。

## R10

正确性直觉：
1. 每条关系都满足 `x_i^2 ≡ Q(x_i) (mod n)`。
2. 依赖保证 `Π Q(x_i)` 的每个素指数为偶数，因此是平方 `b^2`。
3. 于是 `a^2 ≡ b^2 (mod n)`，其中 `a = Π x_i`。
4. 若 `a ≠ ±b (mod n)`，则 `gcd(a-b, n)` 给出非平凡因子。

这与 QS 文献中的标准推导一致。

## R11

复杂度（定性）：
- 对整数分解，QS 的渐近复杂度优于试除、费马法等基础方法。
- 理论上常写成亚指数型 `L_n[1/2, 1]` 量级（实现细节会影响常数）。

本 MVP 复杂度热点：
- 平滑关系搜索：大量 `Q(x)` 试除。
- `GF(2)` 消元：矩阵尺寸约为“关系数 × 因子基大小”。

由于是教学 MVP，优先可读性而非极限性能。

## R12

边界与失败模式：
- `n` 偶数或完全平方会被快速特判。
- 若 `n` 含很小素因子，`small_trial_factor` 会提前返回。
- 当因子基过小或候选 `x` 不够时，关系不足，脚本会增大 `bound` 重试。
- 某些依赖可能退化为平凡 gcd（1 或 n），需继续尝试其它依赖。

若多轮仍失败，会抛出异常并提示增加参数。

## R13

MVP 取舍说明：
- 保留：QS 的完整数学链路（非黑盒）。
- 简化：
  - 不实现大规模 sieve array 与多项式 QS（MPQS/SIQS）。
  - 不做稀疏矩阵优化（如 block Lanczos）。
  - 不引入外部大整数库。

这样可以在较短代码中体现算法本质，适合入门与验证。

## R16

与常见分解法对比：
- 试除法：实现最简单，但对大数极慢。
- Pollard Rho：对中小规模常很实用，但不总是稳定。
- Quadratic Sieve：在中等规模整数分解中具有经典优势，是进入更高级筛法（如 NFS）前的重要台阶。

可扩展方向：

## R18

`demo.py` 源码级算法流程（8 步）：
1. `main()` 设定待分解数 `n`，调用 `quadratic_sieve(n)`。
2. `quadratic_sieve` 先做快速特判：偶数、完全平方、小素因子（`small_trial_factor`）。
3. 进入循环，用 `build_factor_base` 按当前 `bound` 生成因子基（包含 `p=2` 与二次剩余素数）。
4. 设定所需关系数 `needed = len(FB)+8`，调用 `find_smooth_relations` 搜索 `Q(x)=x^2-n` 的 B-smooth 值并保存指数向量。
5. 将关系奇偶向量输入 `find_dependencies`，在 `GF(2)` 上做高斯消元，同时用位掩码追踪原始行组合。
6. 对每个依赖集合，调用 `try_dependency` 计算
   - `a = Πx_i (mod n)`
   - `b = Πp_j^(sum_exp_j/2) (mod n)`。
7. 在 `try_dependency` 中尝试 `gcd(a-b, n)` 和 `gcd(a+b, n)`；命中非平凡因子即返回。
8. `main()` 打印因子与统计信息，完成可复现实验闭环。

本实现未调用第三方分解黑盒；每一步都在源码中可直接追踪。
