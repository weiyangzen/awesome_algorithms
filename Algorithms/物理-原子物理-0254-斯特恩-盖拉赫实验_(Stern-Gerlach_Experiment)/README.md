## R01

斯特恩-盖拉赫实验展示了原子磁矩在非均匀磁场中的空间分裂，并直接给出“角动量投影离散化”的实验图像。沿 `z` 方向磁场梯度下，原子受力近似为：

`F_z = mu_z * (dB_z/dz)`

若 `mu_z` 只能取离散值（如自旋 `1/2` 的两种投影），探测屏会出现离散斑点；若 `mu_z` 连续分布，则得到连续展宽分布。

## R02

该实验的核心价值：

- 为量子化投影（而非经典连续取向）提供了直观证据；
- 把“内部自由度（自旋/磁矩）”映射到“可见的空间轨迹偏转”；
- 是后续自旋测量、量子态制备与量子信息测量链路的基础原型；
- 在教学上可把哈密顿量层面的抽象概念落到可视化统计分布。

## R03

本目录 MVP 的输入输出约定（无交互）：

- 输入（脚本内固定参数）：
1. 银原子束参数（质量、平均前向速度、速度散度、初始束斑宽度）；
2. 磁铁几何参数（磁场梯度、磁铁长度、漂移段长度）；
3. 两种模型：
   - 量子模型：`mu_z = ±(g_e/2) * mu_B`；
   - 经典模型：`mu_z = (g_e/2) * mu_B * cos(theta), cos(theta)~U[-1,1]`。

- 输出：
1. 探测屏 `z` 方向分布统计表（`pandas`）；
2. `scipy` KDE 峰检结果（量子双峰 vs 经典单峰）；
3. `sklearn` 双高斯混合拟合得到的两峰中心、宽度、权重；
4. 可选 `torch` 反演 `|mu_z|` 估计值；
5. 自动断言通过后输出 `All checks passed.`。

## R04

MVP 使用的物理与数值模型：

1. 受力与加速度：
`a_z = F_z/m = mu_z * (dB_z/dz) / m`。

2. 飞行时间：
`t_m = L_m / v_x`（磁铁内），`t_d = L_d / v_x`（漂移段）。

3. 偏转位移：
`z = z0 + a_z * t_m * (0.5*t_m + t_d)`。

4. 量子模型磁矩投影：
`mu_z in {-(g_e/2)mu_B, +(g_e/2)mu_B}`。

5. 经典模型磁矩投影：
`mu_z = (g_e/2)mu_B*cos(theta)`，`cos(theta)` 连续分布。

## R05

设粒子数为 `N`、KDE 网格点数为 `G`（脚本中 `G=800`）：

- 束流采样与轨迹计算：`O(N)`；
- KDE 评估：约 `O(NG)`；
- 峰检测：`O(G)`；
- 两分量 GMM 拟合（EM）：约 `O(I*N*K)`，其中 `K=2`、`I` 为迭代次数；
- 可选 PyTorch 单参数回归：`O(T*N)`。

空间复杂度约 `O(N + G)`。

## R06

`demo.py` 的最小闭环：

- **闭环 A（动力学映射）**：由 `mu_z -> F_z -> a_z -> z` 计算探测屏位移；
- **闭环 B（量子/经典对照）**：同一实验几何下比较两种磁矩投影模型；
- **闭环 C（统计可验证）**：KDE 峰数、GMM 分离度、中心位置、权重与可选 `torch` 反演一起做自动断言。

## R07

优点：

- 把 Stern-Gerlach 核心机理（非均匀磁场 + 投影分布）直接编码；
- 同时给出“量子离散 vs 经典连续”的可比较输出，不是单一模型演示；
- 数学步骤透明，第三方库只做统计求解与容器。

局限：

- 采用简化一维偏转模型，未包含三维束线细节与碰撞效应；
- 未显式求解完整 Pauli/薛定谔演化，仅做实验统计层面的近似；
- 没有接入真实实验噪声链路（真空散射、磁场非理想边缘等）。

## R10

正确性直觉：

1. 同号磁矩应向同一方向偏转、异号向相反方向偏转；
2. 若只允许两种 `mu_z`，应形成双峰；若 `mu_z` 连续，应倾向单个连续峰；
3. 双峰中心应与理论位移量级一致；
4. 两峰权重在对称采样下应接近 `1:1`。

## R11

数值与工程稳定策略：

- 固定随机种子保证可复现；
- 速度采样设下界，避免极小 `v_x` 导致不现实超大飞行时间；
- KDE 峰检使用高度、间距、prominence 三重阈值降低伪峰；
- `torch` 路径对特征做尺度放大，改善梯度数值条件。

## R12

关键参数：

- `field_gradient_t_per_m`：直接控制分裂尺度；
- `magnet_length_m`, `drift_length_m`：分别控制加速时间与漂移放大；
- `velocity_mean_m_s`, `velocity_std_m_s`：影响中心偏移与峰宽；
- `initial_spot_sigma_m`：决定无磁场情况下的初始束斑展宽。

调参建议：

- 双峰不明显时，优先增大梯度或延长漂移段；
- 峰过于重叠时，减小初始束斑或速度散度；
- 若峰检误报，可增大 `find_peaks` 的 `prominence` 与 `distance`。

## R14

常见失效模式：

1. 把磁场梯度符号写反，导致峰位方向全部翻转；
2. 混用 `mm` 与 `m`，会导致中心位置量级错误；
3. 忘记速度分布裁剪，产生极端慢粒子导致异常长尾；
4. 直接用直方图峰检且无平滑，容易被采样噪声伪峰误导。

## R18

`demo.py` 源码级算法流程（8 步）：

1. `SGConfig` 固定实验参数（磁场梯度、几何长度、速度统计、银原子质量），并定义量子模型参考 `|mu_z|=(g_e/2)mu_B`。  
2. `sample_forward_velocity` 采样前向速度并裁剪下界；`ballistic_factor` 计算 `k(v)=t_m(0.5t_m+t_d)`，把动力学统一为 `z=z0+a_z*k`。  
3. `simulate_quantum_beam` 令 `spin_sign∈{-1,+1}`，构造离散 `mu_z`，生成量子位移样本表（含 `z_m/z_mm`、`k_s2`、`vx` 等字段）。  
4. `simulate_classical_beam` 令 `cos(theta)∈[-1,1]` 连续采样，构造经典 `mu_z`，在同一几何下生成对照分布。  
5. `analyze_peaks_kde` 先用 `gaussian_kde` 估计连续密度，再用 `find_peaks` 提取峰位置与峰数，实现“模型行为判别”的可审计统计步骤。  
6. `fit_quantum_gmm` 调用 `GaussianMixture(n_components=2)` 做 EM 拟合，并在源码里显式提取排序后的均值/方差/权重与分离度 `separation_sigma`。  
7. `estimate_mu_torch_optional` 把线性关系 `z = beta*(spin_sign*k)` 写成单参数回归，用 Adam 迭代得到 `beta`，再换算出 `|mu_z|` 估计。  
8. `main` 汇总表格与关键指标，并执行断言（量子双峰、经典单峰、中心对称与理论一致、权重接近 `0.5`、可选 torch 误差阈值），全部通过后打印 `All checks passed.`。  

说明：第三方库在本实现中承担“密度估计/峰检/EM/优化器”角色；核心物理映射 `mu_z -> a_z -> z`、阈值判定和单位换算都在源码中显式展开，不是黑盒一步调用。
