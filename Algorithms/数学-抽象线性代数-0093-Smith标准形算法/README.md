# Smith标准形算法

- UID: `MATH-0093`
- 学科: `数学`
- 分类: `抽象线性代数`
- 源序号: `93`
- 目标目录: `Algorithms/数学-抽象线性代数-0093-Smith标准形算法`

## R01

Smith 标准形（Smith Normal Form, SNF）用于把整数矩阵 `A \in Z^{m\times n}` 通过可逆整变换化为对角形：

`U A V = D`

其中：

- `U \in GL(m, Z)`、`V \in GL(n, Z)` 是整可逆（unimodular）矩阵，`det(U)=±1`、`det(V)=±1`；
- `D` 是对角矩阵（非对角元为 0）；
- 非零对角元满足整除链：`d1 | d2 | ... | dr`，且常取 `di > 0`。

本条目给出一个可运行、可验证、非黑盒的最小实现。

## R02

问题背景（抽象线性代数视角）：

- 整数环 `Z` 是 PID（主理想整环）；
- 因此任意有限生成 `Z`-模都可分解为自由部分与扭部分；
- Smith 标准形正是该结构定理在线性代数中的矩阵化表达。

它常用于：

- 有限阿贝尔群结构分解；
- 同调群/边界算子的结构化计算；
- 整数线性方程组与同余系统的可解性分析。

## R03

本目录 MVP 要解决的问题：

- 输入：任意二维整数矩阵（方阵或矩形矩阵）。
- 输出：`(D, U, V)`，满足 `U*A*V = D`。
- 目标性质：
  - `D` 非对角项全为 0；
  - `D` 非零对角元正且满足整除链；
  - `U` 与 `V` 的行列式绝对值为 1。

`demo.py` 额外提供自动断言，保证运行时直接验证这些性质。

## R04

时间复杂度（工程估计）：

- 设矩阵规模为 `m x n`，`s=min(m,n)`；
- 每个主元阶段会进行多次欧几里得式行/列消元；
- 单次行/列操作代价约 `O(m+n)` 到 `O(mn)`（视实现和缓存方式）；
- 总体可近似看作多轮 `O(mn)` 级更新叠加，实践中对小中规模矩阵足够可用。

严格位复杂度会受整数位数增长影响，不在本 MVP 范围内展开。

## R05

空间复杂度：

- 主矩阵 `A`：`O(mn)`；
- 变换矩阵 `U`、`V`：分别 `O(m^2)` 与 `O(n^2)`；
- 其余临时变量为常数级。

因此总空间为 `O(mn + m^2 + n^2)`。

## R06

微型示例：

给定

`A = [[4,6],[3,9]]`

算法输出

`D = [[1,0],[0,18]]`

满足：

- `1 | 18`；
- 存在整可逆 `U,V` 使 `UAV=D`；
- 与行列式关系一致：`|det(A)| = 18 = 1*18`。

## R07

算法意义：

- 把“整数矩阵”问题转为“标准不变量”问题；
- 对同一个整数线性映射，`D` 的非零对角元唯一（差符号约定）；
- 便于后续推导：秩、余核结构、商群结构都可直接读出。

## R08

核心数学要点：

1. 允许的初等变换（行交换、列交换、加整倍行/列、乘以 `-1`）对应左/右乘 unimodular 矩阵。  
2. 这些变换保持 `Z`-模同构类型不变。  
3. 欧几里得算法用于把主元行列上的元素逐步约化到 gcd。  
4. 对角化后再施加整除链约束，得到 Smith 规范形。  
5. 非零对角元即不变量因子（invariant factors）。

## R09

适用条件与局限：

适用：

- 系数环为整数环 `Z`；
- 输入是有限大小的二维整数矩阵；
- 关注结构分解而非浮点近似。

局限：

- 大整数与大规模矩阵时，运行时间和中间数值膨胀会明显增加；
- 本 MVP 追求清晰与可验证，未做高阶性能优化；
- 仅覆盖 `Z` 上 SNF，不直接扩展到一般 PID 的工程实现细节。

## R10

正确性要点（实现层）：

- 每一步仅使用 unimodular 行/列操作，因此始终保持 `U*A0*V = A_current`；
- 每轮主元 `k`：
  - 通过欧几里得消元清零主元列和主元行的尾部；
  - 若尾部子矩阵存在“不被主元整除”的元素，注入主元列并继续约化，直到满足整除条件；
- 主元阶段完成后，`k` 位置满足局部 Smith 条件；
- 所有阶段完成后得到全局对角形与整除链。

`demo.py` 用断言检查 `U*A*V=D`、对角性、整除链、`det(U/V)=±1`。

## R11

数值与稳定性说明：

- 本算法是“精确整数算法”，不涉及浮点误差累积；
- 主要风险是整数中间量增长（expression swell）；
- 实现使用 Python `int`（`numpy dtype=object`）避免 64 位溢出；
- 为避免主元符号干扰，阶段内会把主元规范化为非负值。

## R12

性能视角：

- MVP 以可读性优先，行/列更新采用显式循环；
- 小中规模矩阵（教学、验证）可直接使用；
- 若追求更高吞吐，可扩展方向：
  - 分块更新；
  - 更激进的主元选择策略；
  - 与 C/Fortran 后端或符号库联动。

## R13

本目录实现的可验证保证：

`demo.py` 固定 3 个测试用例：

- 满秩 `2x2`；
- 矩形 `2x3`；
- 秩亏 `3x3`。

每个用例都会验证：

- `U*A*V == D`（逐元素整数相等）；
- `D` 为对角形且非零对角满足整除链；
- `U`、`V` 的行列式绝对值为 1（Bareiss 精确行列式）。

## R14

常见失效模式：

- 输入不是二维矩阵；
- 存在非整数元素（如非整浮点）；
- 自定义修改时若引入非 unimodular 操作，会破坏 `U*A*V=D` 不变量；
- 极大规模输入可能出现性能瓶颈（非正确性问题）。

## R15

`demo.py` 结构设计：

- `SmithResult`：封装 `D/U/V`；
- `to_object_int_matrix`：输入规范化与整数校验；
- 行列基础操作：`swap/add/scale` 与对应 `op_*`（同步更新 `A` 与 `U/V`）；
- `reduce_pivot_block`：主元阶段核心（欧几里得消元 + 整除性修复）；
- `smith_normal_form_int`：阶段循环，生成最终 SNF；
- `bareiss_determinant`：精确行列式，用于 unimodular 断言；
- `run_case/main`：批量样例执行与打印。

## R16

相关算法链路：

- Hermite Normal Form（HNF）：仅行（或列）方向标准化；
- Smith Normal Form（SNF）：行列同时标准化，得到更强结构不变量；
- 扩展欧几里得算法：SNF 主元消元核心子程序思想来源；
- 在代数拓扑与整数规划中，SNF 常作为基础模块。

## R18

`demo.py` 的源码级算法流程（8 步）：

1. `main` 定义固定整数矩阵样例（满秩/矩形/秩亏），逐个调用 `run_case`。  
2. `run_case` 先把输入转为 `dtype=object` 的整数矩阵，随后调用 `smith_normal_form_int`。  
3. `smith_normal_form_int` 初始化 `U=I_m`、`V=I_n`，并按 `k=0..min(m,n)-1` 逐块处理右下子矩阵。  
4. 每一轮先在 `A[k:,k:]` 选择绝对值最小非零元，交换到 `(k,k)` 作为主元。  
5. `reduce_pivot_block` 对主元列执行欧几里得式行消元、对主元行执行欧几里得式列消元，把该行列尾部清零。  
6. 若尾部子矩阵存在元素不被主元整除，则把该元素所在列加到主元列，再回到第 5 步继续约化，直到满足整除性。  
7. 全部主元阶段结束后得到 `D`，并由构造过程保留 `U*A0*V=D`。  
8. `run_case` 验证四项：矩阵恒等式、对角形、整除链、`det(U/V)=±1`，全部通过后打印结果。

说明：本实现没有把第三方库当作 SNF 黑盒，核心变换流程在源码中逐步展开。
