# 平面图判定

- UID: `MATH-0501`
- 学科: `数学`
- 分类: `图论`
- 源序号: `501`
- 目标目录: `Algorithms/数学-图论-0501-平面图判定`

## R01

平面图判定问题：给定无向图 `G=(V,E)`，判断是否存在一种画法，使得所有边仅在端点相交、内部不发生交叉。

本目录 MVP 采用的判定口径是：
- 输入视为无向简单图（自环忽略，重边合并）；
- 输出 `True/False` 表示是否为平面图；
- 算法核心基于禁止 minor（`K5` 与 `K3,3`）。

## R02

经典理论背景：
- Kuratowski 定理：图非平面当且仅当包含 `K5` 或 `K3,3` 的细分（subdivision）。
- Wagner 定理：图非平面当且仅当包含 `K5` 或 `K3,3` 的 minor。

工程实现里，常见线性时间算法是 Hopcroft-Tarjan / Boyer-Myrvold 族。该 MVP 为了保持源码可读，使用“约化 + 禁止 minor 搜索”的小规模精确方案。

## R03

要解决的核心矛盾：
- 目标是布尔判定（planar / non-planar）；
- 但直接“找交叉”并不稳定，因为平面嵌入依赖顶点周边旋转顺序；
- 更稳妥的路径是转化为结构判定：是否存在 `K5` 或 `K3,3` minor。

因此本实现不是几何坐标法，而是纯组合图论法。

## R04

MVP 总流程：
1. 按连通分量分别处理（任一分量非平面则全图非平面）。
2. 先做保平面性的图约化：
   - 删除度 `<=1` 顶点；
   - 平滑度 `2` 顶点（把路径 `u-v-w` 收缩为 `u-w`）。
3. 做 Euler 必要条件快速剪枝：
   - 一般简单图：`m <= 3n-6`；
   - 二分图：`m <= 2n-4`。
4. 在约化图上精确搜索 `K5` minor。
5. 若未命中，再精确搜索 `K3,3` minor。
6. 若两者都不存在，则判定平面。

## R05

复杂度（设约化后分量顶点数为 `n_r`）：
- 约化与分量遍历：`O(n_r + m_r)`。
- Euler/二分图检查：`O(n_r + m_r)`。
- minor 精确搜索：最坏指数级（分支集划分回溯）。

因此本 MVP 是“小规模精确判定”，不是大规模线性时间工业实现。

## R06

示例：
- `K4`：平面图。
- `K5`：非平面（直接命中禁止 minor）。
- `K3,3`：非平面。
- `K3,3` 的边细分版本：约化后可回到 `K3,3`，仍非平面。

`demo.py` 内置以上案例并自动断言结果。

## R07

优点：
- 纯 Python、无第三方依赖。
- 判定逻辑可追踪，适合作为 Kuratowski/Wagner 教学实现。
- 对小图是精确判定，不是启发式。

局限：
- 受指数回溯影响，不适合大图。
- 为控制可运行性，代码设置了状态上限与规模边界。

## R08

前置知识：
- 图基础：连通分量、二分图、顶点度数。
- 平面图基本不等式（Euler 推导结果）。
- minor 与 branch set（分支集）定义。
- 回溯搜索与状态缓存（`lru_cache`）。

## R09

适用边界：
- 适用：教学、验证、单元测试、几十条边内的小图判定。
- 不适用：大规模网络的高性能判定任务。

实现中通过 `max_exact_vertices` 和 `max_states` 显式限制搜索规模；超限会抛错，而不是给出不可靠“猜测结果”。

## R10

正确性要点（在实现边界内）：
1. 删除度 `<=1` 顶点与平滑度 `2` 顶点保持平面性等价。
2. Euler 不等式违背可直接判非平面（必要条件剪枝）。
3. 若存在 `K5` 或 `K3,3` minor，根据 Wagner 定理可判非平面。
4. 回溯搜索枚举 branch set 划分，并逐一验证：
   - 每个 branch set 连通；
   - 目标图每条边在原图分支集间有跨边。

满足以上条件时结论与禁止 minor 定理一致。

## R11

稳健性与实现细节：
- 图表示采用整数 bitset，避免 Python 对象层级过深导致的高开销。
- 通过 `lru_cache` 缓存回溯状态，减少重复搜索。
- 对输入边做边界检查，忽略自环，合并平行边。
- 对大实例不静默降级，直接抛异常提示规模超界。

## R12

真实工程成本与调参：
- `max_exact_vertices`：限制约化后顶点规模，越大越慢。
- `max_states`：限制回溯状态数，防止长时间卡死。

经验上：
- 教学/示例可用 `max_exact_vertices=10~12`。
- 若追求工业性能，应换为 Boyer-Myrvold 等线性时间实现。

## R14

常见失效模式与防护：
- 失效：输入图约化后仍较大，回溯爆炸。
  - 防护：`max_exact_vertices` + `max_states` 双重上限。
- 失效：把 MVP 当成大规模生产判定器。
  - 防护：README 明确边界并给出替代算法建议。
- 失效：输入含非法顶点编号。
  - 防护：`from_edges` 中抛 `ValueError`。

## R17

本目录 `demo.py` 内容：
- `SimpleGraph`：bitset 图结构。
- `reduce_graph_for_planarity`：保平面性的约化。
- `has_target_minor`：对固定目标图 (`K5` / `K3,3`) 的分支集回溯搜索。
- `is_planar_mvp`：总控判定入口。
- `build_demo_graphs` + `main`：内置 6 个确定性案例并断言。

运行方式：

```bash
cd Algorithms/数学-图论-0501-平面图判定
python3 demo.py
```

脚本不需要交互输入。

## R18

`demo.py` 的源码级算法流程（8 步）：
1. `build_demo_graphs` 构造测试图，包括平面图与非平面图（含 `K3,3` 细分）。
2. `main` 对每个案例调用 `is_planar_mvp`，并记录耗时与断言期望标签。
3. `is_planar_mvp` 先按 `connected_components` 分解，每个连通分量独立判定。
4. 对每个分量执行 `reduce_graph_for_planarity`：反复剥离度 `<=1` 顶点、平滑度 `2` 顶点，得到更小等价实例。
5. 在约化图上先做 Euler 必要条件与二分图上界剪枝，快速排除明显非平面分量。
6. 若未剪枝，调用 `has_target_minor(..., K5_TARGET)`：回溯枚举 branch set 划分，并在叶子态检查连通性与跨分支边条件。
7. 若 `K5` 未命中，继续 `has_target_minor(..., K33_TARGET)`，同样执行精确 minor 搜索。
8. 任一目标 minor 命中则返回非平面；全部分量都未命中则返回平面，并在 `main` 打印结果。

该流程没有调用外部黑盒判定器，`K5/K3,3` 目标与判定逻辑都在本文件中显式实现。
