## R01

k近邻（k-Nearest Neighbors, KNN）是一种基于实例的监督学习方法。它在训练阶段几乎不做参数拟合，只保存训练样本；在预测阶段对查询点寻找最近的 `k` 个邻居，再通过“多数投票”（分类）或“均值/加权均值”（回归）输出结果。

对分类任务，可写成：
- 给定查询样本 `x`，找出训练集最近邻集合 `N_k(x)`；
- 预测标签 `y_hat = argmax_c sum_{(x_i,y_i) in N_k(x)} w_i * I(y_i = c)`；
- 常见权重是 `w_i = 1 / (dist(x, x_i) + eps)`，不加权时 `w_i = 1`。

## R02

KNN 的核心思想可追溯到 1950s-1960s 的最近邻研究：
- Fix 与 Hodges 早期提出了最近邻分类思想；
- Cover 与 Hart（1967）系统分析了 1-NN 的误差上界性质；
- 后续大量工作围绕一致性、距离度量、索引结构和近似搜索展开。

工程里常说的 KNN，通常指“基于某个距离度量 + k 邻居投票”的非参数方法。

## R03

KNN 解决的问题是：当我们认为“特征空间里相近的样本应有相近标签”时，如何在不显式学习复杂模型参数的前提下完成预测。

其关键假设与挑战：
1. 局部平滑假设成立（近邻标签相似）。
2. 合适的距离度量可表达“相似性”。
3. 高维下距离退化（维度灾难）会削弱近邻意义。
4. 预测成本较高（需要与很多训练样本比较距离）。

## R04

KNN 分类基本流程：
1. 准备训练集 `X_train, y_train`（训练阶段主要是“存储”）。
2. 选择超参数：`k`、距离度量（如欧氏距离）、是否距离加权。
3. 对每个查询点 `x`，计算其到所有训练样本的距离。
4. 选取距离最小的 `k` 个样本索引。
5. 统计这 `k` 个邻居的类别票数（或加权票数）。
6. 得到预测类别（并处理并列票的确定性规则）。
7. 在验证集评估准确率，并调节 `k`/度量方式。

## R05

设训练样本数 `n`、特征维度 `d`、查询样本数 `q`：
- 训练时间复杂度：`O(1)`（不计数据读取，仅存储）。
- 训练空间复杂度：`O(n*d)`（需保存训练数据）。
- 预测时间复杂度（暴力搜索）：
  - 距离计算 `O(q*n*d)`；
  - 取前 `k` 邻居常见为 `O(q*n)`（`argpartition`）或 `O(q*n log n)`（全排序）。
- 预测空间复杂度：
  - 若一次性构建距离矩阵，约 `O(q*n)`。

KNN 的典型成本特征是“训练便宜、预测昂贵”。

## R06

简化示例（二分类，二维）：
- 训练点：
  - 类别 0：`(1,1), (2,1), (1,2)`
  - 类别 1：`(4,4), (5,4), (4,5)`
- 查询点：`x=(2.5,2.0)`，取 `k=3`。

计算欧氏距离后，最近的 3 个邻居通常是：
- `(2,1)`（类 0）
- `(1,2)`（类 0）
- `(4,4)`（类 1）

多数票为类 0，因此预测 `x` 属于类 0。

## R07

优点：
- 思路直观、实现简单，适合作为基线模型。
- 无需显式训练复杂参数，易于增量加入样本。
- 可自然扩展到多分类与回归。

局限：
- 预测开销大，数据量大时延迟高。
- 对特征尺度敏感，未标准化时距离会失真。
- 对噪声点和无关特征敏感。
- 在高维空间中效果常明显下降。

## R08

实现 KNN 需要的前置知识：
- 向量距离计算（欧氏距离、闵可夫斯基距离等）。
- 排序/选择算法（`argpartition`、`argsort`）。
- 分类评价指标（准确率、混淆矩阵）。
- 数据预处理（标准化、异常值处理、特征筛选）。
- 随机种子控制与可复现实验。

## R09

适用场景：
- 中小规模数据集，预测实时性要求不极端。
- 类边界复杂但“局部邻域”可分的数据。
- 需要强可解释基线（可直接查看近邻样本）。

不适合或需谨慎：
- 超大规模训练集且在线预测低延迟要求高。
- 高维稀疏特征且未做降维/特征选择。
- 特征量纲差异巨大但未标准化。
- 类别严重不均衡且无重采样/权重策略。

## R10

实现正确性的关键检查点：
1. 距离计算是否正确，且与设定度量一致。
2. 邻居选择是否确实拿到最小距离的 `k` 个样本。
3. 投票逻辑是否正确，含加权与非加权两种情况。
4. 并列票时是否有确定且稳定的决策规则。
5. 训练样本数是否满足 `k <= n_train`。
6. 预测输出标签是否与原始标签空间一致（编码/解码正确）。

## R11

数值与工程稳定性要点：
- 加权投票里使用 `1 / (dist + eps)`，避免除零。
- 距离计算可先用平方距离做排序，减少 `sqrt` 开销。
- 使用向量化计算降低 Python 循环误差和性能损耗。
- 保证输入类型统一为 `float64`，减少类型混杂问题。
- 固定随机种子，保证数据划分与结果可复现。

## R12

真实成本与调参经验：
- `k` 是最关键超参数：
  - `k` 太小易过拟合、对噪声敏感；
  - `k` 太大易欠拟合、边界过平滑。
- 经验上先从奇数 `k`（如 3/5/7/11）网格尝试，降低平票概率。
- 当类别不均衡时，距离加权通常比纯多数票更稳。
- 可先做 PCA/特征选择再做 KNN，常能显著提速并提升泛化。

## R13

理论保证与边界：
- 经典结果表明 1-NN 的误差率在一定条件下可由 Bayes 误差上界约束。
- 对一般 k-NN，在样本量趋于无穷时，若 `k -> ∞` 且 `k/n -> 0`，可得到一致性结论（在常见正则条件下）。
- 这些保证依赖数据分布、度量空间性质和样本规模；有限样本下表现仍需以验证集结果为准。

## R14

常见失效模式与防护：
- 失效：特征尺度差异导致距离被某一维主导。
  - 防护：标准化/归一化。
- 失效：噪声点成为近邻，导致局部误判。
  - 防护：增大 `k`、距离加权、异常值清理。
- 失效：高维下近邻不再“近”。
  - 防护：降维、特征选择、改用更适合高维的模型。
- 失效：类别不均衡，多数类淹没少数类。
  - 防护：重采样、类别权重、阈值调整。

## R15

工程实践建议：
- 先实现一个可复现 MVP（固定数据、固定 seed、可打印指标）。
- 在小样本上手工核对若干查询点的近邻与投票结果。
- 保持预测流程可追踪：距离矩阵、邻居索引、票数统计。
- 在上线前做延迟压测；必要时引入 KD-Tree/Ball-Tree/ANN。
- 记录 `k`、度量、预处理版本和评估指标，便于回溯。

## R16

相关方法与演化：
- 半径邻域法（Radius Neighbors）：固定半径而非固定邻居数。
- 加权 KNN：距离越近权重越大。
- KD-Tree / Ball-Tree：加速近邻检索（低中维更有效）。
- 近似近邻（ANN，如 HNSW/FAISS）：在大规模下以少量精度换速度。
- 与核方法关系：局部加权平均可视作一种非参数平滑思想。

## R18

源码级算法流程拆解（对应 `demo.py`，8 步）：
1. `make_synthetic_dataset` 生成 3 个高斯簇并打乱顺序，得到 `x, y`。
2. `train_test_split` 依据固定随机种子划分训练集与测试集。
3. `KNNClassifier.fit` 保存训练特征，使用 `np.unique(..., return_inverse=True)` 做标签编码。
4. `predict` 调用 `_pairwise_sq_euclidean`，以向量化方式计算查询样本到全部训练样本的平方欧氏距离矩阵。
5. `_predict_encoded` 使用 `np.argpartition` 先取每行前 `k` 小，再用 `argsort` 在局部排序，得到稳定的近邻索引。
6. 对每个查询点统计邻居类别票数：
   - 非加权：`np.bincount(neighbor_labels)`；
   - 加权：`np.bincount(..., weights=1/(dist+eps))`。
7. 用 `np.argmax` 选出得票最高类别（并列时自然选最小类索引），再映射回原始标签空间。
8. `main` 调用 `evaluate` 计算准确率与混淆矩阵，并执行质量断言（准确率阈值、矩阵形状），确保 MVP 行为可验证。

该实现把 KNN 的距离计算、邻居选择、投票决策全部显式展开，可直接追踪每一步。
