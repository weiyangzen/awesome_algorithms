# 基本群计算

- UID: `MATH-0249`
- 学科: `数学`
- 分类: `代数拓扑`
- 源序号: `249`
- 目标目录: `Algorithms/数学-代数拓扑-0249-基本群计算`

## R01

问题定义（MVP 离散版本）：
给定一个**有限连通无向图** `G=(V,E)`（视作 1 维 CW 复形）和基点 `root`，计算其基本群
\[
\pi_1(G, root)
\]
的可计算表示。

本实现输出：
- 基本群自由秩 `rank = |E| - |V| + 1`；
- 一组自由生成元 `g1, g2, ...`；
- 每个生成元在原图中的“基点回路词”（由边及其逆组成）。

## R02

数学背景（图上的标准结论）：
- 连通图可看作 1 维 CW 复形；
- 选取任意生成树 `T` 后，收缩 `T` 不改变同伦类型中的环结构；
- 每条非树边对应一个独立环；
- 因此
\[
\pi_1(G, root) \cong F_r, \quad r = |E| - |V| + 1,
\]
其中 `F_r` 为秩 `r` 的自由群。

## R03

`demo.py` 的输入输出：
- 输入（代码内置，无交互）：
  - 边列表 `[(u, v, label), ...]`；
  - 基点 `root`。
- 输出：
  - `n, m` 与秩公式值 `m-n+1`；
  - 生成树得到的非树边数量；
  - 关联矩阵法得到的回路维数（数值校验）；
  - 自由群表示 `<g1, g2, ... | >`；
  - 每个生成元的具体回路词。

## R04

离散化建模与假设：
- 顶点用整数标识；
- 无向边存储一个参考方向 `u->v`，反向记作逆元；
- 支持自环与多重边；
- 当前 MVP 聚焦**连通图**，若不连通会抛出异常（避免“单基点基本群”语义混乱）。

## R05

算法步骤（高层）：
1. 从边列表构造图结构与邻接表；
2. 以 `root` 做 BFS，得到生成树 `T`（记录父节点与父边）；
3. 识别非树边集合 `E \ T`；
4. 对每条非树边 `e=(u,v)`：
   - 求树路径 `root -> u`；
   - 拼接 `e`；
   - 再拼接 `root -> v` 的逆路径；
   - 对得到的词做相邻逆元约化；
5. 将每条非树边映射为一个生成元 `gi`；
6. 输出自由群秩和生成元回路。

## R06

正确性要点（MVP 级）：
- 生成树覆盖所有顶点且无环，树边在同伦意义下可收缩；
- 每条非树边与树路径共同形成一个独立基点环；
- 非树边数量正好是图的第一 Betti 数 `m-n+1`；
- 因此生成元集合大小与理论秩一致，得到的群为秩 `m-n+1` 的自由群。

## R07

复杂度分析：
- 设 `n=|V|`, `m=|E|`。
- BFS 构造生成树：`O(n+m)`；
- 对每条非树边回溯树路径，最坏 `O(n)`，共 `m-n+1` 条；
- 总体最坏 `O((n+m) + (m-n+1)*n)`。

在中小图场景下该实现足够直观；若追求大规模，可改为 LCA/倍增优化路径拼接。

## R08

边界与鲁棒性：
- `root` 不在顶点集：抛出 `ValueError`；
- 图不连通：抛出 `ValueError`；
- 树图（`m=n-1`）：返回平凡群（无生成元）；
- 自环边：在关联矩阵中对应零列，正确增加回路维数；
- 多重边：通过 `edge_id` 区分，不会混淆。

## R09

MVP 范围声明：
- 本实现处理“图模型下的基本群计算”，不是一般 CW 复形/流形上的全泛化求解器；
- 不做群同构化简（如 Nielsen 变换最简基）；
- 不输出形式化证明，仅提供可审计计算流程与数值一致性检查。

## R10

代码结构：
- `Edge`：无向边数据结构；
- `GeneratorLoop`：生成元及其回路词；
- `UndirectedGraph`：图容器（邻接、词格式化）；
- `invert_word` / `reduce_word`：自由群词运算；
- `build_spanning_tree`：BFS 生成树；
- `path_from_root`：恢复树路径；
- `incidence_matrix`：构造顶点-边关联矩阵；
- `compute_fundamental_group`：主计算流程；
- `run_case` / `main`：示例运行与打印。

## R13

- `label` 建议语义化（如 `a,b,c`），输出更可读；

## R15

建议测试清单：
- 树图（应输出平凡群）；
- 单圈图（应输出 1 个生成元）；
- 一点多自环图（秩应等于自环数）；
- 含多重边图（生成元区分正确）；
- 错误输入：非法 `root`、不连通图。

## R16

常见错误与规避：
- 错误 1：把“无向边”当作单向边，导致路径逆元错误；
- 错误 2：未记录父边方向，拼接回路时词方向颠倒；
- 错误 3：把树边也当成生成元，导致秩虚高；
- 错误 4：忽略自环在关联矩阵中的零列效应，导致校验不一致。

## R17

与相关算法的区别：
- 与并查集：并查集只判连通，不给基本群生成元；
- 与最短路：最短路给路径代价，不给同伦环结构；
- 与黑盒图库 API：本实现显式暴露“生成树-非树边-回路词”链路，便于审计与教学。

## R18

`demo.py` 源码级算法流（8 步）：
1. `UndirectedGraph.__init__` 读取边列表，建立 `edges` 与邻接 `adj`，并为每条边保存参考方向。
2. `build_spanning_tree` 从 `root` 做 BFS，写入 `parent_vertex` 与 `parent_arc`，同时收集 `tree_edges`。
3. `compute_fundamental_group` 用 `edge_id not in tree_edges` 得到全部非树边。
4. 对每条非树边，分别调用 `path_from_root` 计算 `root->u` 与 `root->v` 的树路径。
5. 按 `root->u + e + inverse(root->v)` 拼接成候选基点回路词。
6. `reduce_word` 对词执行相邻逆元消去，得到约化词；并将其注册为生成元 `gi`。
7. `incidence_matrix` 构造 `B`，用 `numpy.linalg.matrix_rank` 计算 `m-rank(B)` 作为回路维数数值校验。
8. `run_case` 打印 `<g1,...|>` 与生成元词表（`pandas.DataFrame`），完成可读输出。
