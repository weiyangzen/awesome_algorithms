## R01

迭代加深搜索（Iterative Deepening Search, IDS）是把 DFS 的“低空间占用”与 BFS 的“按深度逐层保证最浅解”结合起来的搜索策略。它通过反复执行“深度受限 DFS（DLS）”，并逐步增大深度上限，最终在单位步长代价场景下找到最短步数解。

## R02

问题形式化（本目录 MVP 版本）：
- 输入：有向图 `graph`（邻接表）、起点 `start`、终点 `goal`、最大深度 `max_depth`
- 状态：图中的节点
- 动作：沿一条有向边 `u -> v` 扩展
- 目标：找到从 `start` 到 `goal` 的路径
- 输出：
  - 可达时返回路径、步数代价、命中时的深度上限
  - 不可达时返回 `found=False`

## R03

适用前提与约束：
- 图是有限分支的离散状态空间
- 目标深度未知或难以预估时，IDS 通常比单次深 DFS 更稳妥
- 单位边代价（每步成本相同）下，IDS 找到的首个解具备最短步数意义
- 本实现加入“路径内去环（on-path）”以防止 DFS 在环上无限递归

## R04

核心思想：
- 第 0 轮：只允许搜索深度 `0`
- 第 1 轮：允许深度 `1`
- ...
- 第 `d` 轮：允许深度 `d`
- 每一轮都执行一次 DLS；若某轮命中目标，立即停止并返回

直观上，前几层节点会被重复访问，但这部分开销通常被几何增长的最深层节点数掩盖，因此 IDS 在很多问题上依旧高效。

## R05

简化伪代码：

```text
for limit in 0..max_depth:
    result = depth_limited_dfs(start, goal, limit)
    if result.found:
        return result
    if result.cutoff == false:
        # 说明本轮未因深度上限被截断，继续加深已无意义
        break
return not_found
```

`depth_limited_dfs` 的关键行为：
- 到达 `goal` 立即成功返回
- `remaining_depth == 0` 且未命中时记为 cutoff
- 用 `on_path` 集合避免当前递归路径上的回边环

## R06

正确性要点：
- 完备性（有限分支 + 足够深度）：若存在深度 `d*` 的解，IDS 在第 `d*` 轮会覆盖所有深度不超过 `d*` 的合法路径，因此必能找到解
- 最浅解保证（单位步长）：IDS 按深度上限递增，首次找到的解深度最小，即步数最短
- 路径合法性：DLS 仅沿图中边扩展，并在回溯时维护父子路径关系，返回路径可直接验证

## R07

复杂度（令分支因子为 `b`，最浅目标深度为 `d`）：
- 时间复杂度：`O(b^d)`（更精确地是多轮前缀和，主导项仍为最深层）
- 空间复杂度：`O(d)`（递归栈 + 当前路径集合）

对比：
- BFS 空间通常是 `O(b^d)`
- IDS 用额外重复访问换取显著更低内存

## R08

边界与异常：
- `max_depth < 0`：应拒绝
- `start` 或 `goal` 不在图中：应拒绝
- 邻接表中出现未定义节点：应拒绝（避免隐性 KeyError）
- `start == goal`：应在深度 0 即命中
- 不可达：应返回 `found=False`，而不是抛出语义不清异常

## R09

示例：
- 可达图：`A -> ... -> J`，最短深度为 4，IDS 在 `limit=4` 轮找到路径
- 不可达图：`S` 所在连通部分不包含 `T`，IDS 在达到 `max_depth` 或提前无 cutoff 后返回失败
- 退化场景：`start=goal=E`，`limit=0` 轮直接成功

本目录 MVP 实现策略：

## R12

典型应用：
- 状态空间较深但内存受限的路径搜索
- 棋类/博弈中的逐步加深搜索框架（与启发式评估配合）
- 机器人离散动作规划中的浅层快速试探
- 需要“随时可中断并返回当前最浅解边界”的搜索场景

## R13

常见错误：
- 在 DLS 中使用全局 visited，导致同层不同路径被错误剪枝
- 不区分“未找到”与“被深度截断（cutoff）”，使 IDS 终止条件错误
- 忘记处理环，导致递归不收敛
- 将 IDS 误用到非单位代价最短路，并错误宣称最优性

## R14

常见变体：
- IDDFS（经典 IDS）：按深度上限加深的 DFS
- IDA*：把“深度上限”替换为 `f=g+h` 阈值（启发式版本）
- 双向迭代加深：从起点与终点两侧逐步扩张（实现更复杂）
- 带时间预算的迭代加深：随时返回当前最佳结果

## R15

与 BFS / DFS 对比：
- DFS：空间低，但可能先钻入很深分支，不能保证最浅解
- BFS：最浅解保证好，但内存占用大
- IDS：在单位代价场景下保留最浅解保证，同时空间接近 DFS

## R16

工程实践建议：
- 明确 cutoff 语义，并让上层 IDS 根据 cutoff 决定是否继续加深
- 保留每轮扩展计数，便于观察重复访问开销
- 固定邻接遍历顺序，保证测试可复现
- 用参考算法（如 BFS）做回归验证，防止“看起来能跑”但最优性失真

## R17

最小测试清单：
- 可达图：验证路径合法且代价等于 BFS 最短步数
- 不可达图：验证 `found=False`
- `start==goal`：验证零代价与零深度命中
- 非法图（邻接指向未定义节点）：验证输入校验
- 非法参数（负深度上限）：验证输入校验

## R18

源码级算法流程拆解（对应本目录 `demo.py`）：
1. `iterative_deepening_search` 入口先执行 `_validate_graph`，并检查 `max_depth >= 0`。
2. 从深度上限 `0` 开始循环，每一轮调用 `depth_limited_dfs(graph, start, goal, limit)`。
3. `depth_limited_dfs` 初始化 `path=[start]` 与 `on_path={start}`，用递归函数在当前上限内做 DFS。
4. 递归访问每个节点时先计入 `expanded_nodes`；若当前节点就是 `goal`，立即复制并返回完整路径。
5. 若 `remaining_depth == 0` 且尚未命中，则打上 `cutoff=True`，表示“这条分支因深度限制被截断”。
6. 对每个邻居，若已在 `on_path` 则跳过（防环）；否则入栈递归，失败后回溯恢复现场。
7. 单轮 DLS 返回后，IDS 记录该轮统计；若 `found=True` 则立刻返回最终结果；若 `found=False` 且 `cutoff=False` 则提前终止（继续加深已无新状态）。
8. `main()` 中对返回路径做合法性检查，并用 `_bfs_shortest_distance` 交叉验证步数最优性，最后打印固定格式摘要与通过结论。
