# 根据身高重建队列

- UID: `CS-0072`
- 学科: `计算机`
- 分类: `贪心算法`
- 源序号: `92`
- 目标目录: `Algorithms/计算机-贪心算法-0092-根据身高重建队列`

## R01

根据身高重建队列（Queue Reconstruction by Height）的问题定义是：

- 每个人用二元组 `(h, k)` 表示；
- `h` 是身高；
- `k` 是该人前面“身高大于等于 `h`”的人数。

给定一个被打乱顺序的人群数组，要求恢复出一个满足所有 `(h, k)` 约束的队列。

## R02

该题是贪心算法里非常典型的“排序 + 局部构造”范式，常用于训练：

- 如何把全局约束转化为可增量维护的局部不变式；
- 为什么“先处理高优先级对象”可以降低后续决策复杂度；
- 如何利用数据结构（如 Python 列表插入）实现构造式解法。

## R03

形式化描述：

- 输入：`people = [(h1,k1), (h2,k2), ..., (hn,kn)]`。
- 输出：队列 `queue`，是 `people` 的一个排列。
- 约束：对队列中每个位置 `i` 的人 `(h_i, k_i)`，满足
  `count(j < i and h_j >= h_i) == k_i`。

注意：输入不一定总是可行；工程实现应考虑不可行输入的检测与报错。

## R04

核心贪心策略：

1. 按 `h` 降序排序；若 `h` 相同，按 `k` 升序排序。
2. 依次把人插入到结果队列的下标 `k` 处。

直觉解释：

- 先放高个子时，后续插入的矮个子不会影响“前面有多少个 >= 当前身高”的计数；
- 对同身高人群，`k` 小的先放，能保证后放的人在其正确位置被插入。

## R05

复杂度分析：

- 排序：`O(n log n)`；
- 逐个插入列表：最坏 `O(n^2)`（每次 `insert` 可能线性搬移）；
- 总时间复杂度：`O(n^2)`；
- 额外空间复杂度：`O(n)`（结果队列与排序副本）。

## R06

经典示例：

输入：
`[(7,0),(4,4),(7,1),(5,0),(6,1),(5,2)]`

排序后（按 `-h, k`）：
`[(7,0),(7,1),(6,1),(5,0),(5,2),(4,4)]`

按 `k` 插入过程：

1. `[(7,0)]`
2. `[(7,0),(7,1)]`
3. `[(7,0),(6,1),(7,1)]`
4. `[(5,0),(7,0),(6,1),(7,1)]`
5. `[(5,0),(7,0),(5,2),(6,1),(7,1)]`
6. `[(5,0),(7,0),(5,2),(6,1),(4,4),(7,1)]`

即为重建结果。

## R09

适用条件与边界：

- 输入元素应为长度为 2 的整数对 `(h, k)`；
- 常规语义下 `h >= 0, k >= 0`；
- 当输入不可行时（不存在满足约束的队列），实现应返回错误而不是静默输出伪结果；
- `n=0` 时输出空队列。

## R10

正确性要点（简述）：

1. 排序后，当前处理的人要么更高，要么同高且 `k` 更小/相等。
2. 当把当前人插入到位置 `k` 时，当前队列中的人都满足“身高 >= 当前人”这一前提（因为更矮的人尚未加入）。
3. 因而插入后，该人前方恰有 `k` 个满足 `>= h` 的人。
4. 之后插入的更矮人不会增加“前方 >= h 的人数”，不会破坏该人的约束。
5. 归纳到全部插入完成，所有人约束均成立。

## R11

鲁棒性与输入校验：

- 检查每条记录长度是否为 2；
- 检查 `h/k` 是否为非负整数；
- 插入时若 `k > 当前队列长度`，可提前判定输入不可行；
- 构造结束后再做一次全局合法性验证，防止遗漏特殊不可行情形。

## R12

工程实现层面的成本：

- Python `list.insert` 的搬移成本在 `n` 大时明显；
- 若要提升性能，可用平衡树 / Fenwick 树维护“可插入空位”，将整体优化到 `O(n log n)`；
- 对于教学与中小规模输入，当前 `O(n^2)` 版本通常足够清晰实用。

## R14

常见错误：

1. 排序写成 `h` 升序，导致构造不满足不变式；
2. 同身高时按 `k` 降序，导致插入顺序错误；
3. 忽略不可行输入，得到“看似有输出但不满足约束”的结果；
4. 只比较结果长度，不做逐人约束验证。

## R18

`demo.py` 源码级算法流程（8 步，不依赖第三方黑箱求解）：

1. `main()` 依次执行固定样例、随机回归和性能快照，确保脚本可直接跑通。  
2. `_normalize_people()` 把输入统一成整数二元组，并完成基础格式校验。  
3. `reconstruct_queue_greedy()` 先按 `(-h, k)` 排序，建立贪心处理顺序。  
4. 对排序后的每个人 `(h, k)`，执行 `queue.insert(k, (h, k))`，逐步构造队列。  
5. 构造结束后调用 `is_queue_valid_for_people()`，逐个位置复核“前方 >= h 的人数是否等于 k”。  
6. `reconstruct_queue_bruteforce()` 在小规模下用回溯搜索合法队列，作为独立基线。  
7. 随机测试中，先生成一个真实队列并反推 `(h, k)` 约束，再打乱输入喂给贪心与回溯对拍。  
8. 最后输出通过统计与耗时摘要，确认实现既正确又可运行。  

`numpy` 在本 MVP 中不是求解器，只用于样例数据构造；核心贪心流程均可逐行追踪。
