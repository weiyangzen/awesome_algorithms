# 欧拉路径/回路

- UID: `MATH-0481`
- 学科: `数学`
- 分类: `图论`

## R01

欧拉路径（Euler trail）指在图中恰好经过每条边一次的路径；欧拉回路（Euler circuit）是起点和终点相同的欧拉路径。

- 无向图：
- 欧拉回路存在当且仅当所有非零度顶点连通，且每个顶点度数为偶数。
- 欧拉路径（非回路）存在当且仅当所有非零度顶点连通，且恰有 2 个奇度顶点。
- 有向图：
- 欧拉回路存在当且仅当每个顶点入度等于出度，且所有非零度顶点在同一弱连通分量内。
- 欧拉路径（非回路）存在当且仅当恰有一个顶点满足 `out-in=1`（起点），一个顶点满足 `in-out=1`（终点），其余顶点入度等于出度，且所有非零度顶点在同一弱连通分量内。

## R02

该问题最早来自 1736 年欧拉对“柯尼斯堡七桥问题”的研究，通常被视为图论起点之一。欧拉给出的关键思想是：

- 只看“能否一笔画完每条边一次”，本质由顶点度数约束决定。
- 连通性和奇偶性（或入/出度平衡）是存在性的核心条件。

## R03

要解决的核心是两个子问题：

- 判定：给定图是否存在欧拉路径/回路。
- 构造：若存在，输出一条具体路径（顶点序列或边序列）。

判定条件仅给“有无”，实际工程通常还要求可复现构造，因此本条目采用 Hierholzer 算法实现线性时间构造。

## R04

Hierholzer 算法流程（适用于无向/有向）如下：

1. 先做存在性判定（度数条件 + 连通性条件）。
2. 选择起点（奇度点或 `out-in=1` 点；若回路则任取有边顶点）。
3. 维护栈 `stack` 与结果 `path`。
4. 当栈顶顶点还有未用边时，取一条未用边走到下一顶点并压栈。
5. 当栈顶无可用边时，将该顶点弹栈并追加到 `path`。
6. 最终将 `path` 反转，得到欧拉路径/回路。

该流程本质是在“走到死胡同再回退拼接回路”，每条边恰好被消费一次。

## R05

设顶点数为 `V`，边数为 `E`。

- 时间复杂度：`O(V + E)`。
- 空间复杂度：`O(V + E)`（邻接表、访问标记、栈与输出路径）。

这是该问题的经典最优量级实现之一。

## R06

无向图例子：`edges = {(0,1),(1,2),(2,0),(0,3),(3,2)}`。

- 度数：`deg(0)=3, deg(1)=2, deg(2)=3, deg(3)=2`。
- 奇度顶点为 `0` 与 `2`，共 2 个，且非零度顶点连通。
- 因此存在欧拉路径（非回路），例如：`0 -> 1 -> 2 -> 0 -> 3 -> 2`。

该路径长度为 `E+1=6`，且每条边恰好一次。

## R07

优点：

- 判定条件简单，构造算法线性复杂度。
- 可处理重边与较大稀疏图（邻接表实现）。
- 结果可直接用于路线编排、边遍历任务。

局限：

- 只关注“边恰好一次”约束，不优化路径代价（非最短路问题）。
- 对动态图增删边后通常需重新判定与构造。
- 若图不满足度数/连通性条件，必须退化为“近似覆盖”类问题。

## R08

前置知识：

- 图的表示：邻接表、边编号。
- 图遍历：DFS/BFS（用于连通性检查）。
- 度数、入度、出度定义。
- 栈结构与回溯思想。

## R09

适用场景：

- 一笔画判定。
- 巡检/清扫类“每条边一次”的任务规划。
- DNA 组装中的某些图模型（如 De Bruijn 图路径构造思想）。

不适用或需改造：

- 目标是最短路径、最小代价巡回（应考虑最短路/中国邮差/TSP 等）。
- 图规模超大且在线更新频繁（需增量维护策略）。

## R10

正确性关键点：

1. 先验判定必须完整：度数条件与连通性条件缺一不可。
2. 每条边必须有唯一“已使用”状态，避免重复消费。
3. 构造结果应满足长度 `len(path)=E+1`。
4. 无向图中每条无向边应被计数恰好一次；有向图中每条有向边方向必须匹配。

当以上检查都通过时，输出路径即是有效欧拉路径/回路。

## R11

实现细节与稳定性：

- 使用“边 ID + used 数组”比直接删邻接边更稳健，尤其面对重边。
- 使用每个顶点的游标 `next_idx` 跳过已用边，避免反复线性扫描。
- 连通性只需检查非零度顶点，孤立点不影响欧拉性。
- 无边图可视为平凡欧拉回路（返回单点路径）。

## R14

常见失效模式与防护：

- 失效：仅检查度数，不检查连通性，导致“局部可走但全局不可覆盖”。
- 防护：对非零度顶点做一次 BFS/DFS 连通性检测。

- 失效：无向重边被错误合并，造成边数不守恒。
- 防护：每条边分配唯一 ID，不以 `(u,v)` 键去重。

- 失效：路径输出了但未覆盖全部边。
- 防护：构造后强制校验 `len(path)==E+1` 且边计数完全匹配。

## R15

实践建议：

- 把“判定”和“构造”拆成两个函数，便于单测覆盖。
- 对有向图优先打印 `in/out` 差分分布，快速定位失败原因。
- 先用小图（含重边、非连通、无解图）做回归，再上大图。
- 输出中保留类型标签（`none/path/circuit`），便于上层系统分支处理。

## R17

本目录 MVP（`demo.py`）内容：

- `find_euler_undirected`：无向图欧拉性判定与构造。
- `find_euler_directed`：有向图欧拉性判定与构造。
- `hierholzer_undirected` / `hierholzer_directed`：核心构造算法。
- `verify_*`：结果合法性检查（每条边恰一次）。
- `run_demo_cases`：自动跑 6 个用例（无向/有向各覆盖回路、路径、无解）。

运行方式：

```bash
cd Algorithms/数学-图论-0481-欧拉路径／回路
python3 demo.py
```

脚本无交互输入，执行后会打印每个测试图的类型与路径，并在失败时抛出异常。

## R18

源码级流程拆解（对应 `demo.py`，9 步）：

1. `run_demo_cases` 使用 `numpy` 构造无向/有向测试边集，并转为 Python 元组列表。 
2. `find_euler_undirected` 先统计度数并调用 `is_connected_nonzero_undirected` 检查非零度连通性。 
3. 同函数根据奇度顶点个数判定 `none/path/circuit`，并确定 Hierholzer 起点。 
4. `hierholzer_undirected` 用“邻接表 + 边 ID + used 标记 + 栈回溯”逐条消费边，生成顶点序列。 
5. `verify_undirected_path` 对输出路径做多重计数校验，确保每条无向边恰好一次。 
6. `find_euler_directed` 统计每个点的 `in/out`，检查弱连通与差分条件（`out-in` 模式）。 
7. `hierholzer_directed` 在有向邻接表上执行同样的栈式拼接，构造有向欧拉路径/回路。 
8. `verify_directed_path` 校验有向边方向与使用次数，避免方向错配。 
9. `main` 汇总输出并断言所有用例都满足预期类型，形成可直接运行的最小可用实现。 

该实现未调用第三方黑盒图算法库，判定与构造步骤全部可在源码中逐行追踪。
