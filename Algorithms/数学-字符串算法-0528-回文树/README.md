# 回文树

- UID: `MATH-0528`
- 学科: `数学`
- 分类: `字符串算法`
- 源序号: `528`
- 目标目录: `Algorithms/数学-字符串算法-0528-回文树`

## R01

回文树（Palindromic Tree, Eertree）用于在线维护一个字符串的全部不同回文子串。
核心产出通常有三类：
1. 不同回文子串个数。
2. 每个回文子串出现次数。
3. 最长回文子串（或按长度排序的回文信息）。

## R02

回文树的关键思想是“每个节点对应一个唯一回文串”，并用 `suffix_link` 把当前回文连接到“最长真回文后缀”。
当新字符 `ch` 追加到末尾时，只可能新增一个回文节点，因此总体节点数最多 `n + 2`（含两个根）。
这使得去重与计数可以在接近线性时间内完成。

## R03

本实现中的节点字段如下：
1. `length`: 该节点表示的回文长度。
2. `suffix_link`: 指向其最长真回文后缀节点。
3. `next`: 从当前回文外侧加同一字符后得到的新回文转移（`dict[char] -> node_id`）。
4. `occ`: 该回文在扫描过程中作为“当前后缀回文”出现的次数（后续回传得到总出现次数）。
5. `first_pos`: 该回文首次出现时的结尾下标，用于反推字符串内容。

## R04

初始化有两个特殊根节点：
1. 奇根：长度 `-1`，用于统一处理边界扩展。
2. 偶根：长度 `0`，表示空串回文。
`last` 初始指向偶根，表示当前最长回文后缀为空串。

## R05

单字符插入流程（`add_char`）：
1. 从 `last` 出发，沿 `suffix_link` 回退，找到可被新字符两侧扩展的最长后缀回文。
2. 若该扩展结果已存在于 `next[ch]`，复用节点并增加 `occ`。
3. 若不存在则新建节点，长度为 `old_len + 2`，并建立转移边。
4. 为新节点寻找 `suffix_link`：继续在候选后缀链上找可扩展节点，链接到对应转移。
5. 更新 `last` 指向新（或复用）节点。

## R06

`suffix_link` 的作用：
1. 保证插入时回退路径短，避免暴力比较所有子串。
2. 把长回文的统计信息传递给短回文（出现次数回传）。
3. 构成“按回文后缀关系组织”的 DAG 风格结构（实际是带根有向结构）。

## R07

正确性直觉：
1. 每次插入最多新增一个回文节点：因为新回文必须以新字符结尾，且最长可扩展后缀唯一。
2. 每个不同回文只会首次出现时创建一次节点：后续通过 `next` 命中复用。
3. `suffix_link` 总是指向最长真回文后缀，保证后续扩展搜索完备且不重不漏。
4. 长度降序做 `occ` 回传后，父后缀节点累计到所有子回文贡献，即得到真实出现次数。

## R08

复杂度（`n` 为字符串长度）：
1. 构建：均摊 `O(n)`（字符转移用哈希表近似 `O(1)`）。
2. 节点数：`O(n)`（最多 `n + 2`）。
3. 统计回传：`O(n log n)` 若显式排序；也可通过桶按长度降序做到 `O(n)`。
4. 本 MVP 为简洁实现，使用排序版本。

## R09

与 Manacher 对比：
1. Manacher 强项是“最长回文半径”类问题，线性求解半径数组。
2. 回文树强项是“不同回文去重 + 计数 + 在线更新”。
3. 若需求是查询某位置半径，优先 Manacher；若要枚举不同回文并统计频次，优先回文树。

## R10

边界情况：
1. 空串：无普通节点，`distinct_count = 0`。
2. 全相同字符（如 `aaaaa`）：节点少但频次高，验证计数回传是否正确。
3. 无重复回文结构（如随机串）：节点接近 `n`。
4. Unicode 字符：本实现使用 Python `str` 与 `dict`，可直接处理。

## R11

`demo.py` 暴露的核心接口：
1. `Eertree.build(text)`: 构建整棵回文树。
2. `Eertree.distinct_count()`: 不同回文数。
3. `Eertree.longest_palindrome()`: 最长回文串。
4. `Eertree.palindrome_occurrences()`: `{回文串: 出现次数}`。
5. `validate_with_bruteforce(text)`: 与暴力法对拍（小样本）确保正确性。

## R12

运行方式（无交互输入）：

```bash
uv run python demo.py
```

程序会对内置样例批量执行：
1. 回文树构建。
2. 与暴力法逐项比对。
3. 输出 distinct 数量、最长回文、按长度排序的前若干项。

## R13

输出字段说明：
1. `distinct palindromes`：不同回文子串个数。
2. `longest palindrome`：该样例中的最长回文子串。
3. `top palindromes by length`：按长度降序展示若干回文及其出现次数。
4. 末尾 `All samples passed...` 表示回文树结果与暴力结果一致。

## R14

工程化落地建议：
1. 大数据场景建议把 `dict` 转为定长数组（字符集可控时）提升常数。
2. 若需要高频在线查询，可保留插入过程中的实时统计而非离线回传。
3. 若输入极长，建议避免频繁切片，可仅保存区间索引。

## R15

常见错误：
1. 忘记设置长度 `1` 节点的 `suffix_link = 偶根`。
2. 回退条件下标写错（`pos - cur_len - 1` 是关键）。
3. 只统计创建节点次数，没做长度降序回传，导致频次偏小。
4. 把根节点也当普通回文输出，污染结果。

## R16

本目录 MVP 的验证策略：
1. 使用 4 组内置样例覆盖常见结构（重复、高度对称、一般情况、长回文嵌套）。
2. 采用暴力 `O(n^3)` 枚举回文子串做真值对照（仅小样本）。
3. 当 `tree_map != brute_map` 时直接抛出异常，保证失败可见。

## R17

可扩展方向：
1. 支持区间查询：结合前缀版本化或离线分块。
2. 支持流式数据：持续 `add_char`，按时间窗口维护统计。
3. 联动其他结构：与后缀自动机/后缀数组结合做更复杂文本分析。

## R18

`demo.py` 的源码级算法流（8 步）：
1. 初始化两根节点（长度 `-1` 与 `0`），`last` 指向偶根，文本为空。
2. 对输入串逐字符调用 `add_char(ch)`，把字符追加到 `self.text`。
3. 在 `_find_longest_suffix` 中，从 `last` 沿 `suffix_link` 回退，直到找到可被 `ch` 包裹扩展的后缀回文。
4. 若该扩展在 `next[ch]` 已存在，直接跳到旧节点并将其 `occ += 1`。
5. 若不存在，创建新节点（长度 `old_len + 2`、记录 `first_pos`、`occ = 1`），并挂到 `next[ch]`。
6. 为新节点建立 `suffix_link`：若新长度为 `1` 则指向偶根；否则继续沿候选后缀链回退，找到可扩展节点后连到其 `next[ch]`。
7. 全部字符处理后执行 `propagate_occurrences`：按长度降序把每个节点的 `occ` 累加给其 `suffix_link`，得到总出现次数。
8. 通过 `node_to_string`（`first_pos` + `length` 反推区间）恢复回文文本，导出 distinct 数、最长回文、频次表，并与暴力法对拍确认正确性。
