## R01

本条目给出 Muller 方法（Muller's Method）的最小可运行实现，用于求解一元非线性方程 `f(x)=0`。该方法每轮使用最近三个迭代点拟合一条二次曲线，并取该二次曲线的一个根作为下一步迭代点。

与牛顿法相比，Muller 方法不要求显式导数；与割线法相比，它利用二次插值，通常在局部具有更快收敛速度，并且天然支持复数迭代。

## R02

问题定义（有限精度版）：
- 输入：
  - 连续函数或可计算函数 `f`；
  - 三个初始点 `x0, x1, x2`（可为实数或复数）；
  - 停机参数 `tol`、`ftol`、`max_iter`。
- 输出：
  - 近似根 `x*`；
  - 对应函数值 `f(x*)`；
  - 迭代历史（每轮的 `x_k`、`f(x_k)`、步长等）；
  - 是否收敛与总迭代次数。

## R03

数学核心（按 `x0, x1, x2` 三点构造）：

设 `f_i = f(x_i)`，定义：
- `h1 = x1 - x0`
- `h2 = x2 - x1`
- `delta1 = (f1 - f0) / h1`
- `delta2 = (f2 - f1) / h2`
- `d = (delta2 - delta1) / (h2 + h1)`

则过三点的二次插值在 `x2` 处可写成：
`Q(t) = f2 + b t + d t^2`，其中 `t = x - x2`，`b = delta2 + h2*d`。

解 `Q(t)=0` 得
`t = -2f2 / (b ± sqrt(b^2 - 4f2d))`，
故下一点 `x3 = x2 + t`。

为了减少消去误差，分母采用“绝对值更大”的分支。

## R04

算法流程总览：
1. 读取三点初值并计算 `f(x0), f(x1), f(x2)`。
2. 用差商构建二次插值参数 `b, d`。
3. 计算判别量 `sqrt(b^2 - 4f2d)`。
4. 选择数值稳定的分母（模更大的分支）。
5. 计算增量 `step = -2f2 / denom`，得到 `x3`。
6. 记录迭代轨迹并检查收敛。
7. 若未收敛，窗口前移：`(x0, x1, x2) <- (x1, x2, x3)`。
8. 达到阈值或上限后返回结果。

## R05

数据结构设计（MVP）：
- `MullerResult`：
  - `root`：最终近似根；
  - `f_at_root`：函数值；
  - `iterations`：迭代次数；
  - `converged`：收敛标记；
  - `history`：逐轮记录。
- `history` 中每项为字典，包含：
  - `iter`、`x`、`fx`、`step_abs`、`fx_abs`。

这种结构便于后续做日志可视化或收敛统计。

## R06

正确性要点：
- 每轮都严格由三点函数值构造唯一二次插值（只要点不重合且差商可计算）；
- 下一点由插值二次式的根直接给出；
- 分母分支选择遵循稳定计算原则（避免 `b ± sqrt(...)` 的严重抵消）；
- 当 `|step|` 和/或 `|f(x)|` 进入阈值内时，近似根满足数值意义下收敛。

## R07

复杂度分析：
- 设迭代步数为 `T`。
- 时间复杂度：`O(T)`（每轮常数次复数四则运算与一次函数评估）。
- 空间复杂度：
  - 保留完整历史：`O(T)`；
  - 仅保留当前三点：`O(1)`。

## R08

边界与异常处理：
- 若 `x0, x1, x2` 有重复导致 `h1` 或 `h2` 为 0，直接报错；
- 若 `h1+h2=0` 使二阶差商不可计算，报错；
- 若稳定分母接近 0，报错并提示数值退化；
- 达到 `max_iter` 仍未满足阈值时，返回 `converged=False`，不静默伪装成功。

## R09

MVP 取舍说明：
- 选用 `numpy` 做多项式评估和参考根验证；
- 不调用现成根求解器作为主流程，核心迭代完全手写；
- 仅做标量迭代，不做批量并行，以便把 Muller 核心公式清晰呈现。

## R14

关键参数建议：
- `tol`：步长停止阈值，默认 `1e-12`；
- `ftol`：函数值停止阈值，默认 `1e-12`；
- `max_iter`：最大迭代步，默认 `50`；
- `trace_limit`：仅打印前若干轮轨迹，避免输出过长。

工程中可先用较宽松阈值验证流程，再逐步收紧。

## R15

方法对比：
- 相比牛顿法：
  - 优点：不需要显式导数；
  - 缺点：每轮状态更多（三点而非一点），实现稍复杂。
- 相比割线法：
  - 优点：二次插值通常更快；
  - 缺点：稳定性更依赖分母分支处理。
- 相比二分法：
  - 优点：局部收敛快、可进入复域；
  - 缺点：缺少天然区间包络保证。

## R16

## R18

源码级算法流（对应 `demo.py`，8 步）：
1. `main` 构造多项式样例（系数 + 三初值 + 参数）并逐个调用 `muller`。  
2. `muller` 将 `x0,x1,x2` 与函数值 `f0,f1,f2` 转成复数域统一处理。  
3. 计算 `h1,h2,delta1,delta2,d`，得到二次插值参数 `b`。  
4. 计算判别项 `rad = sqrt(b*b - 4*f2*d)`。  
5. 比较 `|b+rad|` 与 `|b-rad|`，选取模更大的分母 `denom` 以减少消去误差。  
6. 计算 `step = -2*f2/denom`，更新 `x3 = x2 + step`，并评估 `f3 = f(x3)`。  
7. 将 `iter/x3/f3/|step|/|f3|` 记录进 `history`，按 `tol` 与 `ftol` 判断收敛。  
8. 若未收敛则窗口前移 `(x0,x1,x2) <- (x1,x2,x3)` 继续迭代，直到成功或超限返回。
