## R01

并查集（Disjoint Set Union, DSU / Union-Find）用于维护“元素分组”和“连通性查询”。

核心操作：
- `find(x)`：返回元素 `x` 所在集合的代表元（根）。
- `union(a, b)`：合并 `a` 与 `b` 所在集合。

本条目聚焦“路径压缩（path compression）”：在 `find` 过程中把访问路径上的节点直接挂到根节点，显著降低后续查询成本。

## R02

典型应用场景：
- 无向图动态连通性判断。
- Kruskal 最小生成树中的成环检测。
- 社交关系分组、账户聚合与标签归并。
- 图像连通域、网格联通块统计。
- 离线等价关系维护（“是否属于同一类”）。

## R03

数学模型：

给定全集 `U = {0, 1, ..., n-1}`，维护其不交划分 `P = {S1, S2, ...}`，满足：
- `Si` 两两不交；
- `S1 ∪ S2 ∪ ... = U`。

操作语义：
- `find(x)` 返回 `x` 所在集合的唯一代表元；
- `union(a, b)` 若二者分属不同集合则将其合并，否则保持不变。

## R04

路径压缩的直觉：

- 未压缩时，`find` 需要沿父指针一路追溯到根，路径可能很长。
- 路径压缩在一次 `find(x)` 完成后，把 `x -> root` 路径上的节点都直接指向 `root`。
- 同一路径上的后续 `find` 将接近 `O(1)`。

它不改变集合划分，只优化树形表示。

## R05

为何有效：

- 朴素并查集在极端合并顺序下会退化为链表，`find` 接近 `O(n)`。
- 路径压缩会不断把“高而瘦”的树压成“矮而胖”的树。
- 在长序列操作下，均摊复杂度显著下降。

若再结合按秩/按大小合并，理论均摊复杂度可达 `O(α(n))`（反 Ackermann 函数，工程上近似常数）。

## R06

正确性要点：

1. 每个集合始终以一棵有根树表示，根满足 `parent[root] = root`。  
2. `find` 追溯父指针一定到达根；路径压缩只改“父指针形状”，不改“所属集合”。  
3. `union` 仅连接两个根节点，避免破坏树结构。  
4. 若 `find(a) == find(b)`，则 `union(a,b)` 不应减少分量数。  
5. 任意时刻两元素同组当且仅当它们根相同。

## R07

复杂度分析：

- 仅使用路径压缩（不加按秩/按大小）时，最坏单次 `find` 仍可能较慢，但实践中通常很快。
- 路径压缩 + 合理合并策略能达到近似常数均摊复杂度 `O(α(n))`。
- 空间复杂度 `O(n)`，主要为 `parent` 与可选 `size` 数组。

## R09

接口约定（`UnionFindPathCompression`）：

- `UnionFindPathCompression(n: int)`：创建 `n` 个单元素集合。
- `find(x: int) -> int`：返回 `x` 的根，并执行路径压缩。
- `union(a: int, b: int) -> bool`：若发生真实合并返回 `True`。
- `connected(a: int, b: int) -> bool`：判断是否同组。
- `size(x: int) -> int`：返回 `x` 所在集合大小。
- `groups() -> list[list[int]]`：返回当前所有分组（便于验证和展示）。

## R10

边界处理：

- `n < 0`：抛 `ValueError`。
- 下标越界：抛 `IndexError`。
- `union(x, x)`：合法，返回 `False`（结构不变）。
- `n = 0`：允许创建空并查集，但不可查询元素。

## R11

与“按秩合并 / 按大小合并”的关系：

- 本题核心是路径压缩，重点在 `find` 的优化。
- 若只做路径压缩，不限制合并方向，仍可能出现短期偏斜树。
- 工程中通常把“路径压缩 + 按秩/按大小”同时启用，以获得更稳健性能。
- 本 MVP 保持合并策略简单，便于直接观察路径压缩带来的扁平化效果。

## R12

示例流程（链式合并）：

- 依次执行 `union(0,1), union(1,2), ..., union(8,9)`。
- 在该合并顺序下会形成较深路径（用于演示）。
- 对 `find(0)` 执行一次路径压缩后，`0` 到根的长链会被压平。
- `demo.py` 会输出压缩前后最大深度对比。

## R14

工程注意事项：

- `find` 建议迭代实现，避免递归栈风险。
- 路径压缩应在找到根后统一回写，避免中途写坏路径。
- `union` 必须基于根连接，不能直接连接原节点。
- 若未来需要“可回滚并查集”，需谨慎使用完全路径压缩。

## R18

`demo.py` 源码级算法流程（7 步）：

1. 初始化：`parent[i]=i`、`comp_size[i]=1`、`components=n`。  
2. 调用 `find(x)`：沿 `parent` 追溯到根 `r`，把访问链暂存。  
3. 路径压缩：把暂存链上每个节点的父指针直接改为 `r`。  
4. 调用 `union(a,b)`：分别求根 `ra, rb`；若相同则不合并并返回 `False`。  
5. 根合并：把一个根挂到另一个根（本实现为 `parent[ra] = rb`），并更新集合大小与分量计数。  
6. 查询操作：`connected(a,b)` 比较根是否相等，`size(x)` 读取根对应大小。  
7. 主程序验证：先做链式示例观察压缩效果，再做随机对拍确保与朴素实现行为一致。  

实现未调用第三方并查集黑盒，算法步骤与源码一一对应。
