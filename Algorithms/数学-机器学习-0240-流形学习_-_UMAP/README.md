## R01

UMAP（Uniform Manifold Approximation and Projection）是一类非线性降维方法，目标是在低维空间中尽量保留高维数据的局部邻域结构，并兼顾一定的全局可分性。它特别适合可视化高维样本（如图像特征、文本向量、生物组学矩阵）并做聚类前的结构探索。

## R02

问题建模：
- 输入：高维样本矩阵 `X in R^(n x d)`。
- 输出：低维表示 `Y in R^(n x m)`（通常 `m=2` 或 `m=3`）。
- 优化目标：让高维“邻近关系图”与低维“邻近关系图”在概率意义下尽量一致。

核心不是重构原始特征，而是重构“谁和谁是邻居、邻居强度有多大”。

## R03

UMAP 的直觉可分两层：
1. 在高维空间中，使用 kNN + 局部尺度校准（`rho_i`, `sigma_i`）把点云转为“模糊邻接图”（fuzzy simplicial set）。
2. 在低维空间中，用可微相似度函数定义边概率，通过交叉熵损失把高维图压到低维图。

因此 UMAP 不是“直接拟合坐标”，而是“拟合图结构概率”。

## R04

高维边权（简化写法）：
- 对点 `i` 的邻居 `j`，设距离 `d_ij`。
- 局部连通半径 `rho_i`：通常取 `i` 的最近非零邻居距离。
- 局部平滑参数 `sigma_i`：通过二分搜索，使
  `sum_j exp(-max(0, d_ij - rho_i) / sigma_i)` 接近目标常数（与 `n_neighbors` 相关）。
- 有向权重：
  `p_ij = exp(-max(0, d_ij - rho_i) / sigma_i)`。

无向融合（fuzzy union）：
`w_ij = p_ij + p_ji - p_ij * p_ji`。

## R05

低维相似度（本 MVP 采用简化版）：
- 令 `dist2_ij = ||y_i - y_j||^2`。
- 定义 `q_ij = 1 / (1 + dist2_ij)`。

训练损失分两部分：
- 正样本（图中存在边，权重 `w_ij`）：最大化 `q_ij`，即最小化 `-w_ij log(q_ij)`。
- 负样本（随机采样非邻居）：最小化 `q_ij`，即最小化 `-log(1 - q_ij)`。

这对应 UMAP 思路中的“吸引 + 排斥”。

## R06

与标准 `umap-learn` 的关系：
- 相同点：kNN 图、局部尺度、模糊并集、交叉熵式优化、负采样。
- 差异点：本项目为教学级 MVP，采用更紧凑的低维核函数和损失写法，未复刻全部工程细节（如更完整的采样调度、更多参数化核拟合）。

这样做的目的：保持可读性与可运行性，同时不把算法当黑箱调用。

## R07

复杂度（近似）：
- kNN 查询：`O(n log n)` 到 `O(n^2)`（取决于维度、索引结构和后端实现）。
- 构图与参数搜索：`O(n * k * T)`，`k=n_neighbors`，`T` 为二分迭代次数。
- 训练：`O(E * epochs * (1 + neg_samples))`，`E` 为图边数，通常约 `O(nk)`。

内存主要由邻接图稀疏矩阵与边列表占用。

## R08

关键超参数建议：
- `n_neighbors`：控制“局部 vs 全局”权衡。小值更强调局部团簇，大值更强调全局连续结构。
- `epochs`：轮数过低会欠拟合，过高收益递减。
- `neg_samples`：越大越能拉开簇间间隔，但训练更慢。
- `lr`：学习率过大可能震荡，过小收敛慢。

在样本规模不大时，优先先调 `n_neighbors` 和 `epochs`。

## R09

数值稳定性与工程处理：
- 所有 `log` 都加 `eps`，避免 `log(0)`。
- 每轮训练后对 `Y` 去中心化，减小坐标漂移。
- 使用稀疏矩阵保存图，避免稠密 `n x n` 内存爆炸。
- 固定随机种子，提升结果可复现性。

## R10

与 PCA / t-SNE 的简要对比：
- PCA：线性、速度快、可解释性强，但难保留复杂非线性流形。
- t-SNE：局部可视化强，但对全局结构、参数和随机性较敏感。
- UMAP：通常在“局部保持 + 全局可分 + 速度”之间取得更均衡折中。

本目录 `demo.py` 里也给了 `PCA_2D` 对照，并输出 trustworthiness 指标。

## R14

结果解释建议：
- 若 `UMAP_MVP` 的 trustworthiness 高于 `PCA_2D`，说明局部邻域保留更好。
- 若簇间距离很近，优先调大 `neg_samples` 或适度增大 `epochs`。
- 若簇结构碎裂，可增大 `n_neighbors`，让图连接更平滑。

这个演示更强调算法机制透明性，而不是追求与官方实现完全同数值。

## R15

常见失败模式：
- 样本极少或高度重复：`rho/sigma` 估计会退化，图边权区分度下降。
- 学习率过大：训练损失振荡，嵌入形状不稳定。
- 邻居数设置不当：过小导致碎片化，过大导致细节被抹平。

处理策略：先固定 `seed`，再逐项单参数扫描。

## R18

源码级算法流程（对应本目录 MVP）：
1. 读取并标准化高维数据 `X`，设定 `n_neighbors`、随机种子等参数。  
2. 用 `NearestNeighbors` 获取每个点的 k 近邻距离与索引，去掉自环邻居。  
3. 对每个点 `i`：计算局部阈值 `rho_i`，并用二分搜索求 `sigma_i`，得到有向边权 `p_ij`。  
4. 把所有有向边写入 `scipy.sparse` 稀疏矩阵，并通过 `p + p^T - p*p^T` 做模糊并集，得到无向权重图。  
5. 用 PCA 生成低维初始坐标 `Y0`，作为优化起点。  
6. 将边列表转为张量，在 PyTorch 中计算正样本吸引损失 `-w*log(q)`（`q=1/(1+||yi-yj||^2)`）。  
7. 对每条正边采样若干负点，计算排斥损失 `-log(1-q_neg)`，与正损失相加。  
8. 用 Adam 迭代更新 `Y`，每轮去中心化，直到达到设定 epoch。  
9. 输出最终二维嵌入，并用 trustworthiness 与 PCA 基线做定量比较。
