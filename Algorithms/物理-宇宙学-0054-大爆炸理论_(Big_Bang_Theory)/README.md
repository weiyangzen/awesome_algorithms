# 大爆炸理论 (Big Bang Theory)

- UID: `PHYS-0054`
- 学科: `物理`
- 分类: `宇宙学`
- 源序号: `54`
- 目标目录: `Algorithms/物理-宇宙学-0054-大爆炸理论_(Big_Bang_Theory)`

## R01

大爆炸理论的核心不是“某个瞬间的爆炸画面”，而是一个可计算的宇宙演化框架：
- 宇宙从高温高密状态膨胀并冷却；
- 尺度因子 `a(t)` 随时间增长；
- 不同能量成分（辐射、物质、暗能量）在不同时代主导膨胀动力学。

本条目将该理论压缩为一个可运行 MVP：从 `ΛCDM` 背景方程构建宇宙时间线，并提取关键里程碑（BBN、平衡点、复合、加速转折、今天）。

## R02

目标是实现一个“透明且可审计”的最小算法管线，而不是调用黑箱宇宙学软件包：
1. 数值积分得到 `a(t)` 与 `t(a)`；
2. 推导温度历史 `T(a)=T0/a`；
3. 计算关键事件对应的 `z`、`T`、`t`；
4. 在辐射/物质/暗能量时代分别拟合 `H(a)` 的幂律斜率；
5. 用内置物理阈值自动输出 `Validation: PASS/FAIL`。

## R03

MVP 采用一组近似 Planck 风格参数：
- `H0 = 67.4 km/s/Mpc`
- `Omega_m = 0.315`
- `Omega_r = 9e-5`
- `Omega_k = 0`
- `Omega_lambda = 1 - Omega_m - Omega_r - Omega_k`

时间线网格：
- `a_min = 1e-12`（覆盖极早期高温阶段）
- `a_max = 1`
- 对数网格 `n_grid = 120000`

事件锚点：
- BBN 温度尺度 `T ~ 1e9 K`
- 复合时代 `z ~ 1089`
- 物质-辐射平衡 `rho_m = rho_r`
- 加速转折 `q(z)=0`

## R04

本实现使用的关键方程：

1. 第一弗里德曼方程（无量纲）：

`E(a)^2 = H(a)^2/H0^2 = Omega_r a^{-4} + Omega_m a^{-3} + Omega_k a^{-2} + Omega_lambda`

2. 时间演化积分：
`dt/da = 1 / [a * H(a)]`

3. 红移与尺度因子关系：
`z = 1/a - 1`

4. 温度演化（绝热近似）：
`T(a) = T0/a = T0 * (1+z)`

5. 减速参数：
`q(a) = [Omega_m a^{-3} + 2 Omega_r a^{-4} - 2 Omega_lambda] / [2 E(a)^2]`

6. 物质-辐射平衡红移：
`z_eq = Omega_m/Omega_r - 1`

## R05

关键建模假设：
- 宇宙在背景层面满足 FRW 均匀各向同性。
- 仅处理背景演化，不包含密度扰动增长。
- 温度标度采用 `T ∝ 1/a` 的理想绝热近似。
- BBN/复合仅以温度或典型红移作为“时间线锚点”，不做核反应网络或辐射转移细解。

因此该 MVP 适合“算法化理解大爆炸理论”，不用于高精度观测拟合。

## R06

`demo.py` 输入输出约定：
- 输入：脚本内置 `BigBangConfig`（无命令行参数，无交互）。
- 输出：
1. 历史样本表（`a,z,t,T,H,q,dominant_component`）；
2. 里程碑表（事件名、红移、温度、时间）；
3. 时代幂律拟合表（辐射/物质/暗能量时代斜率）；
4. 一组一致性检查与 `Validation: PASS/FAIL`。

## R07

高层流程：
1. 解析宇宙学参数并补全 `Omega_lambda`。
2. 在对数 `a` 网格上计算 `E(a)` 与 `H(a)`。
3. 对 `dt/da` 做累计积分，得到 `t(a)`。
4. 派生 `z(a)`、`T(a)`、`q(a)` 与主导能量成分标签。
5. 由温度/红移锚点插值得到里程碑时刻。
6. 用 `scikit-learn` 对三个时代做 `log H` vs `log a` 线性拟合。
7. 用 `torch.trapz` 做宇宙年龄交叉校验。
8. 执行阈值检查并输出 PASS/FAIL。

## R08

设网格点数为 `N`：
- 历史构建与积分：`O(N)`
- 里程碑插值：`O(1)` 每事件（总计常数级）
- 时代回归：`O(N)`（子区间线性回归）
- 加速转折搜索：`O(M)`（固定密度红移网格，M 为常数级）

总体时间复杂度 `O(N)`，空间复杂度 `O(N)`。

## R09

数值稳定性策略：
- 使用对数 `a` 网格覆盖从 `1e-12` 到 `1` 的多尺度区间。
- 在计算前检查 `E(a)^2 > 0`，避免不物理参数导致开方错误。
- `q(z)=0` 先网格找符号翻转，再用 `scipy.optimize.brentq` 精化。
- 通过 `numpy` 与 `torch` 两套积分对同一宇宙年龄做交叉验证，抑制静默数值偏差。

## R10

MVP 使用的最小工具栈：
- `numpy`：向量化方程计算
- `scipy`：累计积分与根求解
- `pandas`：结果表格化输出
- `scikit-learn`：时代幂律线性回归
- `PyTorch`：独立积分交叉校验

核心宇宙学公式均在源码中显式展开，没有把关键流程外包给黑箱求解器。

## R12

主要输出字段说明：
- `a`: 尺度因子。
- `z`: 红移。
- `t_sec`: 自 `a_min` 积分到当前 `a` 的宇宙时间（秒）。
- `temperature_k`: 背景辐射温度（K）。
- `H_km_s_Mpc`: 哈勃参数。
- `q`: 减速参数（`q<0` 表示加速膨胀）。
- `dominant_component`: 当下主导项（辐射/物质/曲率/暗能量）。
- `milestone` 表中给出事件对应 `z,T,t`。
- `epoch fit` 表中给出拟合斜率与理论斜率对比、`R^2`。

## R13

内置验收检查：
1. 今天宇宙年龄在 `13-15 Gyr`。
2. 物质-辐射平衡红移在 `2500-4500`。
3. 复合时代时间在 `2e5-6e5` 年。
4. 加速转折红移在 `0.4-1.0`。
5. 辐射/物质/暗能量时代的 `H(a)` 幂律斜率在合理区间。
6. 时代回归通过分层阈值：辐射时代 `R^2>0.999`、物质时代 `R^2>0.995`、暗能量时代 `R^2>0.99`。
7. `torch` 与 `numpy` 的年龄积分差小于 `0.03 Gyr`。

全部满足则输出 `Validation: PASS`，否则 `FAIL` 并返回非零退出码。

## R18

`demo.py` 源码级算法流程（9 步）：
1. `main` 创建 `BigBangConfig`，调用 `build_background_history` 生成从 `a_min` 到 `1` 的背景演化表。
2. `resolved_densities` 补全密度参数，`e2_of_a` 计算 `E(a)^2`，并在历史构建时检查其正定性。
3. 在 `build_background_history` 中计算 `H(a)`，并用 `dt/da = 1/(aH)` 通过 `cumulative_trapezoid` 得到 `t(a)`。
4. 同函数派生 `z(a)`、`T(a)`、`q(a)`，并通过各分量贡献 `Omega_i a^{-n}` 判定 `dominant_component`。
5. `build_milestones` 计算 `z_eq`、`z_bbn`，并用 `interpolate_by_z` 把每个事件映射到宇宙时间；`find_acceleration_transition_z` 用“符号翻转 + `brentq`”求 `q(z)=0`。
6. `fit_epoch_power_laws` 在辐射/物质/暗能量三个红移区间上构造 `log10(H)` 对 `log10(a)` 数据，并用 `LinearRegression` 拟合斜率。
7. `torch_age_crosscheck_gyr` 复用同一积分表达式，使用 `torch.trapz` 独立计算宇宙年龄。
8. `main` 打印历史样本、里程碑表、时代拟合表，并汇总关键物理量（年龄、平衡点、复合时刻、加速转折）。
9. 执行 9 条一致性检查（物理区间 + 斜率 + `torch` 交叉验证），全部通过输出 `Validation: PASS`，否则 `FAIL` 并退出非零状态。
