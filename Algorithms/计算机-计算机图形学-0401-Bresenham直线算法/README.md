## R01

Bresenham 直线算法用于在离散像素栅格上绘制“尽量接近理想直线”的像素序列。它的关键价值是全程只使用整数加减和符号判断，避免浮点乘除，因此在光栅化、嵌入式显示、实时渲染等场景里非常常见。

## R02

目标问题：给定整数端点 `(x0, y0)` 与 `(x1, y1)`，在二维整数网格上生成一条 8 连通像素路径，使路径从起点到终点单调推进，并尽量贴近欧式直线段。

本实现输出一个点列表 `[(x, y), ...]`，首点为起点，尾点为终点。

## R03

核心思想是维护一个“误差累积量” `err`，用它决定下一步应当只走主轴，还是主轴和副轴同时走：

- `dx = |x1 - x0|`，`dy = |y1 - y0|`
- `err = dx - dy`
- 每轮计算 `e2 = 2 * err`
- 若 `e2 > -dy`：沿 `x` 方向走一步，并把 `err -= dy`
- 若 `e2 < dx`：沿 `y` 方向走一步，并把 `err += dx`

这样就把“距离理想线最近”的判定，转化为整数阈值比较。

## R04

本目录实现采用统一的“全象限”写法（而不是按斜率分 8 个分支）：

- `sx` 表示 `x` 方向步进符号（`-1 / 0 / 1`）
- `sy` 表示 `y` 方向步进符号（`-1 / 0 / 1`）
- 循环直到 `(x, y)` 抵达终点

该写法同时覆盖：

- 水平线、竖直线
- 斜率绝对值 `< 1` 和 `> 1`
- 正斜率与负斜率
- 任意起止方向（从左到右、从右到左等）

## R05

MVP 约定：

- 输入端点必须是整数。
- 输出点序列长度恒为 `max(dx, dy) + 1`。
- 邻接点步长满足 `Δx ∈ {-1, 0, 1}`、`Δy ∈ {-1, 0, 1}` 且不同时为 0。
- 算法不依赖图形库，仅在 `demo.py` 中用 `numpy` 做 ASCII 栅格展示。

## R06

伪代码：

```text
BRESENHAM(x0, y0, x1, y1):
    dx = abs(x1 - x0)
    dy = abs(y1 - y0)
    sx = sign(x1 - x0)
    sy = sign(y1 - y0)
    err = dx - dy

    x, y = x0, y0
    points = []

    while True:
        points.append((x, y))
        if x == x1 and y == y1:
            break

        e2 = 2 * err

        if e2 > -dy:
            err = err - dy
            x = x + sx

        if e2 < dx:
            err = err + dx
            y = y + sy

    return points
```

## R07

正确性直觉：

- 每轮至少推进一个坐标，因此循环必然终止。
- 坐标推进方向由 `sx/sy` 决定，不会反向回退。
- `err` 累积了“当前离理想直线的偏差趋势”，阈值比较控制何时补偿副轴步进。
- 对于主轴长度 `max(dx, dy)`，每步至多前进一步，故总点数稳定为 `max(dx, dy) + 1`。

## R08

复杂度：

- 时间复杂度：`O(max(dx, dy))`
- 空间复杂度：`O(max(dx, dy))`（若把结果存列表）；若直接写像素缓冲可降为 `O(1)` 额外空间。

## R09

工程细节：

- 该算法天然适配整数像素坐标，避免浮点误差累积。
- 与 DDA（数字微分分析）相比，Bresenham 在整数硬件或低成本设备上更友好。
- 在现代 GPU 管线中直线光栅化通常已硬件化，但 CPU 端离散几何处理仍常复用该思路。

## R10

与 DDA 的差异：

- DDA 通常每步执行浮点增量并四舍五入。
- Bresenham 用误差项替代浮点步进，分支决策只依赖整数比较。
- 两者都能输出合理像素线，但 Bresenham 在“速度、可预测性、整数稳定性”上更经典。

## R11

边界条件：

- 起点等于终点：输出单点列表。
- `dx = 0`（竖线）与 `dy = 0`（横线）直接被统一逻辑覆盖。
- 负坐标输入同样成立。
- 反向绘制（终点在起点“左下”或“右上”）由 `sx/sy` 自动处理。

## R12

常见错误：

- 误把条件写成 `>=` 或 `<=`，导致 tie-break 行为变化、路径偏移。
- 漏掉“每轮先记录当前点”导致丢起点或丢终点。
- `sx/sy` 只写 `±1` 忽略 `0`，会在水平/竖直线时越界。
- 用浮点误差变量替代整数误差，失去算法本意。

## R13

`demo.py` 结构：

- `bresenham_line`：全象限整数版主算法。
- `render_ascii`：把点集映射到字符栅格，便于肉眼检查。

## R18

`demo.py` 的源码级算法流（8 步）：

1. `main` 先调用 `run_fixed_regression_cases()`，逐条读取固定端点与期望像素路径。  
2. 每个样例都进入 `bresenham_line(x0, y0, x1, y1)`：初始化 `dx/dy/sx/sy/err`。  
3. 在 `while` 循环中先写入当前像素，再检查是否到达终点。  
4. 计算 `e2 = 2 * err`，依据 `e2 > -dy` 与 `e2 < dx` 两个整数条件决定是否推进 `x`、`y`。  
5. 坐标推进后继续下一轮，直到命中终点，返回整条离散像素链。  
6. `validate_path_properties` 对返回链做结构性校验：端点一致、长度为 `max(dx,dy)+1`、步长合法且不倒退。  
7. `run_random_property_tests()` 用随机端点重复执行主算法与属性断言，覆盖不同象限和退化情形。  
8. `main` 打印一条示例线的 ASCII 栅格并在全部断言通过后输出 `All checks passed.`。  

这 8 步完整对应了源码中的非黑箱流程：误差累积、分支决策、像素产出与自动验证都能逐行追踪。
