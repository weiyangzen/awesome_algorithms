# 树同构

- UID: `MATH-0505`
- 学科: `数学`
- 分类: `图论`
- 源序号: `505`
- 目标目录: `Algorithms/数学-图论-0505-树同构`

## R01

树同构（Tree Isomorphism）关注的问题是：给定两棵无根、无标号树，判断它们在“仅节点命名不同”的意义下是否是同一结构。  
若存在一个节点双射，使每条边在映射后仍对应为边，则两树同构。

## R02

本条目解决的是“判定问题”而不是“枚举所有同构映射”：
1. 输入两棵树；
2. 输出 `True/False` 判断是否同构；
3. 附带输出规范编码（canonical code）用于解释判定原因。

## R03

MVP 输入输出约定（`demo.py` 内固定样例，无交互）：
- 输入：若干测试用例，每个用例包含两棵树的 `n` 和边集 `edges`；
- 输出：
  - 每个用例的 `expected` 与 `actual`；
  - 两棵树的中心点、候选根编码与最终规范编码；
  - 最终 `ALL_CASES_PASSED` 汇总结果。

## R04

核心思路使用 AHU（Aho-Hopcroft-Ullman）树同构编码框架：
- 无根树先转“中心点根树”（一个中心或两个中心）；
- 根树通过“子树编码排序 + 括号包裹”得到自底向上的结构编码；
- 无根树规范编码取中心候选编码中的字典序最小值；
- 两树规范编码相同当且仅当同构。

## R05

前置条件与约束：
- 图必须是树：`n >= 1`、边数 `m = n-1`、连通、无自环、无重边；
- 节点编号采用 `0..n-1`；
- MVP 不处理带权树、带标签树和有向树。

## R06

算法高层流程：
1. 校验输入并构建邻接表；
2. 通过“剥叶子”算法求树中心；
3. 对每个中心执行根树编码；
4. 取最小编码作为无根树规范编码；
5. 比较两棵树的规范编码是否相同。

## R07

正确性直觉：
- 树中心是无根树到根树的标准化入口，避免任意选根造成编码不稳定；
- 根树编码中对子编码排序，消除了子节点排列顺序差异；
- 同构树在每个层级的子结构多重集合一致，最终得到相同编码；
- 非同构树必在某层结构上出现差异，编码会不同。

## R08

时间复杂度（单棵树，`n` 为节点数）：
- 构建与校验：`O(n)`；
- 求中心：`O(n)`；
- 根树编码：对子编码排序，整体最坏 `O(n log n)`；
- 总体：`O(n log n)`。

## R09

空间复杂度：
- 邻接表、父节点数组、编码数组均为 `O(n)`；
- 额外队列/栈也是 `O(n)`；
- 总体额外空间 `O(n)`。

## R10

本目录实现策略：
- 语言：Python 3；
- 依赖：仅标准库（`collections`, `dataclasses`, `typing`）；
- 关键函数：
  - `validate_and_build_tree`：树合法性校验；
  - `find_tree_centers`：剥叶子找中心；
  - `rooted_canonical_code`：根树规范编码；
  - `unrooted_canonical_code`：无根树规范编码；
  - `are_isomorphic`：同构判定主函数。

## R11

运行方式：

```bash
python3 Algorithms/数学-图论-0505-树同构/demo.py
```

脚本不需要命令行参数，也不读取交互输入。

## R12

输出字段说明：
- `expected`：该用例的预期同构结果；
- `actual`：算法计算结果；
- `tree1_centers` / `tree2_centers`：树中心点集合；
- `tree*_candidate_codes`：各中心作为根时的编码；
- `tree*_canonical`：最终无根规范编码（最小候选）。

## R13

边界与鲁棒性处理：
- 对 `n<=0`、边数不等于 `n-1` 直接报错；
- 对非法节点编号、自环、重边直接报错；
- 对非连通图直接报错；
- `n=1` 的单点树可正常处理并返回编码 `()`。

## R14

当前 MVP 局限：
- 仅提供判定，不输出具体节点映射；
- 编码使用字符串，面对超大树时会有额外常数开销；
- 未针对百万级节点做工程优化（如整数哈希压缩、外存流式处理）。

## R15

可扩展方向：
- 输出同构映射（节点对应关系）；
- 将字符串编码替换为整数哈希以降低内存与排序成本；
- 扩展到“带标签树同构”（标签进入编码）；
- 结合 NetworkX 做大规模随机回归测试与可视化。

## R16

最小测试建议：
- 同构测试：同一棵树重标号后应为 `True`；
- 非同构测试：同节点数的路径树 vs 星型树应为 `False`；
- 边界测试：单点树与单点树应为 `True`；
- 非法输入测试：重边、自环、非连通图应抛异常。

## R17

与替代方法对比：
- 与暴力节点排列 `O(n!)` 相比，AHU 编码可在多项式时间内完成；
- 与通用图同构算法相比，树结构可利用“无环 + 分层”性质，算法更简单、可解释性更高；
- 与中心哈希法相比，AHU 的括号编码更直观，教学和调试都更友好。

## R18

`demo.py` 源码级算法流程（8 步）：
1. `main()` 构造固定测试用例（同构、非同构、单点树）。
2. `run_case()` 调用 `validate_and_build_tree()` 分别校验并构建两棵树的邻接表。
3. `validate_and_build_tree()` 检查边数、编号范围、自环、重边和连通性，确保输入真的是树。
4. `run_case()` 对每棵树调用 `unrooted_canonical_code()` 生成无根规范编码。
5. `unrooted_canonical_code()` 先用 `find_tree_centers()` 通过剥叶子法求 1~2 个中心点。
6. 对每个中心调用 `rooted_canonical_code()`，以“自底向上收集子编码并排序”的方式构造括号编码。
7. 取候选编码中字典序最小值作为该树最终规范编码，并比较两棵树编码是否相同。
8. `main()` 打印每个用例的中心、候选编码、最终判定，并在任何结果不符预期时抛出异常。
