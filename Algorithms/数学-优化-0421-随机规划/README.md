## R01

随机规划（Stochastic Programming）研究的是“决策发生在随机量 realization 之前或过程中”的优化问题。典型二阶段形式为：

- `min_x c^T x + E_xi[Q(x, xi)]`

其中 `x` 是先验决策（第一阶段），`xi` 是随机变量（需求、价格、产能等），`Q(x, xi)` 是随机量揭示后可调整的补救（第二阶段 recourse）成本。

本条目的 MVP 采用样本平均近似（SAA）处理期望项，并用投影次梯度法直接求解一个带预算约束的二产品库存/备货随机规划。

## R02

随机规划的发展主线可概括为：

- Dantzig 在 1950 年代提出两阶段随机线性规划；
- 随后形成 L-shaped/Benders 分解、场景树、多阶段随机规划；
- 在工程与数据科学中，SAA 成为最常见落地范式：用有限场景把期望问题转为确定性近似问题，再用数值优化求解。

本目录刻意选择“小而完整”的 SAA + 一阶法实现，强调可读性、可复现和源码可追踪。

## R03

本问题建模为二产品备货决策：

- 第一阶段决定 `x in R^2_+`（两类商品备货量），并满足预算 `a^T x <= B`；
- 第二阶段在需求 `d_s` 实现后，支付缺货惩罚与积压惩罚：
  `p^T(d_s - x)_+ + h^T(x - d_s)_+`。

SAA 目标：

- `min_x c^T x + (1/N) * sum_s [p^T(d_s - x)_+ + h^T(x - d_s)_+]`
- `s.t. x >= 0, a^T x <= B`

这是凸但非光滑优化；我们在 `demo.py` 中显式写出次梯度与投影算子。

## R04

本目录采用的算法组合：

1. 用训练场景集合 `{d_s}` 构造 SAA 目标；
2. 通过次梯度 `g(x)` 做迭代更新：`x_{t+1/2} = x_t - eta_t g_t`；
3. 将中间点投影回可行域 `C={x>=0, a^T x<=B}`：`x_{t+1}=Proj_C(x_{t+1/2})`；
4. 步长采用 `eta_t = lr0 / sqrt(t)`；
5. 记录历史最优点作为输出。

可行域投影不是黑盒：代码使用 KKT 形式 `x_i(lambda)=max(0, z_i-lambda a_i)`，再对 `lambda` 做二分搜索满足预算约束。

## R05

记：

- `N` 为场景数；
- `m` 为产品维度（本 demo 中 `m=2`）；
- `T` 为迭代轮数。

复杂度：

- 单轮目标+次梯度计算：`O(Nm)`；
- 单轮投影（二分）约 `O(m * log(1/eps))`；
- 总时间：`O(TNm + Tm log(1/eps))`；
- 空间：`O(Nm)`（存场景）+ `O(m)`（参数与梯度）。

## R06

直观例子（单产品）：

- 若当前 `x=80`，某场景需求 `d=100`，则缺货 `20`、积压 `0`；
- 场景成本贡献为 `p*20`；
- 次梯度在该场景下对 `x` 的贡献约为 `-p`（增加备货会降低缺货损失）。

若改为 `x=120, d=100`：

- 缺货 `0`、积压 `20`；
- 成本贡献为 `h*20`；
- 次梯度贡献变为 `+h`（继续增大备货只会增加积压损失）。

这就是随机规划里“在随机场景上折中”的核心机制。

## R07

优点：

- 直接建模不确定性与 recourse，不必假设单一确定需求；
- 比“只看均值”的确定性优化更稳健；
- SAA + 一阶法实现简单，易扩展到更大样本。

局限：

- 结果依赖场景质量，样本偏差会传导到策略偏差；
- 非光滑目标收敛速度通常慢于光滑二阶法；
- 多阶段大规模问题通常需要分解法或专用求解器。

## R09

适用场景：

- 库存、产能、采购等“先决策后观测”问题；
- 需要同时考虑缺货成本与过量成本；
- 可获得历史样本或可模拟场景。

谨慎场景：

- 样本极少且分布重尾严重，SAA 方差可能很大；
- 约束非常复杂（整数决策、网络流大规模耦合）时，纯一阶法效率可能不足；
- 必须满足概率安全约束时，可能更适合 chance constraints 或 CVaR 变体。

## R10

实现正确性检查点：

1. 目标函数是否严格对应 `c^T x + E[p^T(d-x)_+ + h^T(x-d)_+]`；
2. 次梯度符号是否正确：`x<d` 时应偏向增加库存（负梯度分量）；
3. 投影后是否始终满足 `x>=0` 与 `a^T x<=B`；
4. SAA 训练目标在迭代中是否总体下降；
5. SAA 解在训练样本上的目标不应劣于“均值替代”基线。

`demo.py` 对上述点均做了可执行断言。

## R11

数值稳定策略：

- 步长衰减 `lr0/sqrt(t)`，降低非光滑震荡；
- 投影二分使用固定迭代次数与容差，避免约束漂移；
- 所有成本与需求使用 `float64`；
- 需求先 `clip(min=0)`，避免生成负需求导致语义异常。

## R12

调参建议：

- `train_scenarios`：越大统计误差越小，但每轮成本线性上升；
- `epochs`：增加可提高收敛，但边际收益递减；
- `lr0`：过大易震荡，过小收敛慢；
- `budget`：越紧，投影约束越常激活，解更偏向高价值商品。

实务中建议固定随机种子，先扫 `lr0` 与 `epochs`，再增加场景数做稳健性复核。

## R13

理论要点：

- 本问题是凸优化（线性项 + 分段线性凸项 + 凸约束集）；
- 经典次梯度法对凸 Lipschitz 目标可达 `O(1/sqrt(T))` 优化误差量级；
- SAA 在常规条件下对真期望问题具有一致收敛，统计误差通常是 `O_p(1/sqrt(N))` 量级；
- 本条目不是近似比算法，故“近似比”不适用。

## R14

常见失效模式与防护：

- 场景代表性不足：
  训练很好、线上失效。防护：增加场景覆盖和分层采样。
- 步长过大：
  目标剧烈震荡。防护：减小 `lr0` 或提高衰减速度。
- 预算过紧且成本参数失衡：
  解退化到单一商品。防护：检查单位收益/惩罚设定是否符合业务。
- 投影实现错误：
  迭代后解不可行。防护：每轮断言可行性。

## R15

工程落地建议：

- 先从少量核心变量构建可解释模型，再逐步加入复杂约束；
- 训练、验证、压力场景分开报告，不只看单一平均指标；
- 参数和随机种子入库，保证回放可复现；
- 需要更高求解精度时，可在 MVP 验证后替换为 LP/MIP 求解器。

## R16

与相关方法关系：

- 确定性优化：把随机量替换为均值，计算便宜但常低估风险；
- 鲁棒优化：优化最坏情形，不依赖概率；随机规划优化的是期望或风险度量；
- 分布鲁棒优化（DRO）：在“分布集合”上做最坏期望；
- 机会约束规划：直接约束违约概率，而非仅在目标中惩罚。

本 demo 对比的是 SAA 随机规划与“均值替代”确定性基线。

## R17

`demo.py` 的 MVP 做了以下事情：

- 构造训练/同分布测试/分布偏移测试三组需求场景；
- 用投影次梯度法训练 SAA 随机规划策略；
- 用“均值需求替代”的确定性策略作为基线；
- 输出三个分布下的平均成本、解向量与可行性信息，并执行断言。

脚本无交互输入，运行结束后会显示对比结果与校验状态。

## R18

`demo.py` 源码级流程拆解（8 步）：

1. `build_datasets` 采样多元需求场景，得到 `train/id/shift` 三个集合。  
2. `stochastic_objective` 显式计算 SAA 目标：采购成本 + 场景平均缺货/积压成本。  
3. `stochastic_subgradient` 根据 `x` 与各场景 `d_s` 的逐维大小关系构造次梯度。  
4. `project_to_budget_nonnegative` 先做非负截断，再用二分搜索拉格朗日乘子 `lambda`，完成到 `x>=0, a^T x<=B` 的欧式投影。  
5. `fit_saa_subgradient` 按 `eta_t=lr0/sqrt(t)` 迭代“次梯度步 + 投影步”，并持续记录历史最优解。  
6. `solve_expected_value_baseline` 把全部场景压成均值场景，再跑同一求解器得到确定性基线。  
7. `evaluate_policy` 在 `train/id/shift` 三套样本上计算平均总成本，比较 SAA 与基线。  
8. `main` 打印关键指标并执行断言（可行性、训练集优越性、优化过程有效），确保本次运行结果可信。  

该实现没有把随机规划交给黑盒库“一键求解”：目标、次梯度、投影和训练循环都在源码中可直接追踪。
