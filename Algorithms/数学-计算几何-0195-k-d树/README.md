# k-d树

- UID: `MATH-0195`
- 学科: `数学`
- 分类: `计算几何`
- 源序号: `195`
- 目标目录: `Algorithms/数学-计算几何-0195-k-d树`

## R01

算法目标: 在二维静态点集中高效回答“给定查询点，最近邻是谁”问题。  
本条目实现一个可运行的 `k-d tree` MVP，并用暴力法做逐次正确性对照。

## R02

问题形式化:
- 输入: 点集 `P = {p_i | p_i in R^2}`，查询点 `q in R^2`
- 输出: 最近邻索引 `i* = argmin_i ||p_i - q||_2`
- 约束: 支持多次查询；点集在查询阶段不更新（静态索引）

## R03

核心思想:
- 构建阶段按轴交替切分（`x -> y -> x -> ...`），每层选中位点做节点，得到近似平衡二叉树。
- 查询阶段先沿“更接近查询点”的子树下探，维护当前最优距离。
- 当“切分平面到查询点距离”小于当前最优距离时，才回溯探索另一侧子树（分支定界剪枝）。

## R04

数据结构设计:
- `KDNode(split_index, axis, left, right)`
- `split_index`: 当前节点对应的原始点索引
- `axis`: 当前层切分轴（二维中 `0/1`）
- `left/right`: 左右子树

## R05

构建流程（递归）:
1. 若候选索引为空，返回空节点。
2. 根据 `depth % dim` 选择切分轴。
3. 在该轴上稳定排序候选索引。
4. 取中位索引作为当前节点。
5. 中位左半构建左子树，右半构建右子树。

## R06

最近邻查询流程（递归）:
1. 访问当前节点，计算节点点与查询点的平方距离并更新最优解。
2. 按查询点与切分值的相对位置，先递归“近侧”子树。
3. 计算 `diff^2`（查询点到切分平面距离平方）。
4. 仅当 `diff^2 < best_dist2` 时，再递归“远侧”子树。
5. 回溯后得到全局最近邻。

## R07

复杂度分析:
- 构建: 平均 `O(n log n)`（当前实现每层排序）
- 单次查询: 平均 `O(log n)`，最坏 `O(n)`
- 空间复杂度: `O(n)`（树节点）

## R08

正确性要点:
- 最近邻候选不会漏掉: 若远侧子树可能包含更优点，则必须满足“切分平面距离 < 当前最优距离”，代码据此决定是否回溯。
- 最优值单调改进: 每访问一个节点都用真实欧氏距离更新 `best_dist2`。
- 等价验证: `demo.py` 对固定查询 + 200 次随机查询，与暴力最近邻逐次比较最小距离。

## R09

MVP 范围:
- 支持二维静态点集最近邻查询。
- 不包含动态插入/删除、批量并行查询优化、近似查询。
- 目标是“实现清晰、可复现、可验证”。

## R10

实现文件说明:
- `demo.py`
- `build_kdtree(...)`: 构建索引树
- `_nn_search(...)`: 递归分支定界搜索
- `nearest_neighbor(...)`: 单次查询接口
- `brute_force_nearest(...)`: 暴力基线
- `main()`: 构建、校验、性能统计

## R11

运行方式:
```bash
python3 demo.py
```
预期行为:
- 输出固定查询最近邻索引与距离
- 输出“200 次随机查询正确性校验: 通过”
- 输出 k-d tree 与暴力法同批查询耗时对比

## R12

样例输出解读:
- `固定查询最近邻索引/距离`: 单点查询结果
- `累计距离`: 多查询结果的距离和，用于检查两方法一致性
- 若实现错误，断言会在随机校验阶段抛异常并停止程序

## R13

与暴力法对比:
- 暴力法: 每次查询扫描全部点，复杂度 `O(n)`
- k-d 树: 利用空间分割与剪枝，在典型分布下显著减少访问节点数量
- 当维度升高或数据分布极端时，剪枝效果会下降

## R14

工程注意事项:
- 使用稳定排序（`mergesort`）保证构建可复现。
- 比较距离时优先使用平方距离，避免重复开方。
- 当前示例默认连续随机点，几乎不出现等距并列；并列场景可增加稳定 tie-break 规则。

## R15

测试策略:
- 固定随机种子，确保每次运行结果一致。
- 固定查询与随机查询双路径校验。
- 每次查询都将 k-d 树结果与暴力法最小距离进行一致性断言。

## R16

已知局限:
- 仅二维；高维可扩展但会受“维度灾难”影响。
- 当前构建策略每层排序，代码简洁但非最优构建实现。
- Python 递归在极端退化树形下受递归深度限制。

## R17

可扩展方向:
- 支持 `k` 近邻（维护有界优先队列）
- 支持动态数据（增删后局部/全局重建）
- 引入近似最近邻策略提升大规模吞吐
- 扩展到半径查询、范围查询等统一空间索引接口

## R18

源码级算法流（`demo.py`，非黑盒拆解）:
1. `main()` 生成二维随机点集 `points`，并构建索引数组 `0..n-1`。
2. 调用 `build_kdtree(points, indices)`，按 `depth % dim` 在不同层切换切分轴。
3. 每层在对应轴上排序并取中位点作为 `KDNode.split_index`，左右两半递归成子树。
4. 查询时调用 `nearest_neighbor(root, points, target)`，先用根节点初始化当前最优距离。
5. `nearest_neighbor` 进入 `_nn_search`：访问节点后计算 `dist2`，若更小则更新最优候选。
6. 根据 `target[axis] - pivot[axis]` 决定先访问近侧子树，再考虑远侧子树。
7. 仅当 `diff^2 < best_dist2` 时才访问远侧子树，实现分支定界剪枝。
8. 查询结束后返回最近邻索引与距离；`main()` 再用 `brute_force_nearest` 做 200 次随机对照断言，并输出性能统计。
