## R01

戳气球问题（Burst Balloons）定义如下：
- 给定数组 `nums[0..n-1]`，每个元素表示一个气球上的数值；
- 每次选择一个尚未被戳破的气球 `i`；
- 本次获得硬币数为 `left * nums[i] * right`，其中 `left/right` 是 `i` 左右两侧“当前仍存在”的最近气球值；
- 若某一侧不存在气球，则该侧按 `1` 计算；
- 目标是在戳完所有气球后，使总硬币数最大。

MVP 聚焦：计算最大硬币数，并给出一组可复现该最优值的戳破顺序。

## R02

MVP 输入输出约定：
- 输入：
  - `nums`：一维非负整数序列。
- 输出：
  - `max_coins`：可获得的最大硬币数；
  - `burst_order`：一个戳破顺序（原数组下标序列）；
  - `n`：气球数量。

约束约定：
- `nums` 必须是一维有限数值；
- 值必须为非负且可解释为整数；
- `n=0` 时返回 `0` 和空顺序。

## R03

状态定义（区间 DP，按“最后戳破”建模）：
- 设扩展数组 `arr = [1] + nums + [1]`，长度 `m = n + 2`；
- `dp[l][r]`：在开区间 `(l, r)` 内的气球全部戳破后，可获得的最大硬币数；
- `choice[l][r]`：达到 `dp[l][r]` 的最优“最后戳破气球”位置 `k`（`l < k < r`）。

边界：
- 当 `r <= l + 1` 时，区间内无气球，`dp[l][r] = 0`。

## R04

状态转移方程：

`dp[l][r] = max_{l<k<r} (dp[l][k] + dp[k][r] + arr[l] * arr[k] * arr[r])`

解释：
- 假设 `k` 是区间 `(l, r)` 里最后被戳破的气球；
- 那么在戳 `k` 的瞬间，`l` 与 `r` 必然还在，因此本次收益固定为 `arr[l] * arr[k] * arr[r]`；
- 左右子区间 `(l, k)` 与 `(k, r)` 相互独立，可分别取最优值。

## R05

最优子结构成立：
- 若 `(l, r)` 的最优策略最后戳 `k`；
- 则其左子区间 `(l, k)` 必须采用最优策略，右子区间 `(k, r)` 也必须采用最优策略；
- 否则用更优子策略替换会得到更大的总体收益，与“全局最优”矛盾。

因此可用区间长度递增方式自底向上填表。

## R06

填表顺序：
- 先处理最短有效区间（`gap=2`，即仅 1 个可戳气球）；
- 再逐步增加区间跨度 `gap` 到 `m-1`；
- 对每个 `(l, r)` 枚举 `k in (l, r)` 更新最优值与 `choice[l][r]`。

该顺序保证转移所依赖的 `dp[l][k]` 和 `dp[k][r]` 已先计算完成。

## R07

复杂度分析：
- 主算法（区间 DP）：
  - 时间复杂度：`O(n^3)`；
  - 空间复杂度：`O(n^2)`。
- 记忆化递归基线：
  - 时间复杂度同阶 `O(n^3)`；
  - 空间复杂度 `O(n^2)`（缓存状态）。
- 暴力基线（仅小规模校验）：
  - 时间复杂度约 `O(n!)`；
  - 空间复杂度与递归深度相关，约 `O(n)`（不含缓存字典常数）。

## R08

## R09

核心接口：
- `burst_balloons_dp(nums) -> BurstBalloonsResult`
  - 返回 `n / max_coins / burst_order`。
- `burst_balloons_top_down(nums) -> int`
  - 独立基线，返回最大硬币数。
- `burst_balloons_bruteforce(nums) -> int`
  - 暴力枚举（仅小规模校验）。
- `simulate_order_gain(nums, order) -> int`
  - 回放顺序并计算真实收益，用于验证重建顺序正确性。

## R12

输出解读：
- `max_coins`：区间 DP 算出的最大收益；
- `order`：一组可实现最优值的戳破顺序（原始下标）；
- `simulate_gain`：按 `order` 回放得到的收益；
- `cross-check`：
  - `top_down` 为记忆化递归结果；
  - `bruteforce` 为小规模暴力最优值。

程序会断言：
- `max_coins == simulate_gain`；
- `max_coins == top_down`；
- 当规模允许时，`max_coins == bruteforce`。

## R13

随机对拍策略：
- 默认 `250` 轮；
- 每轮随机生成 `n ∈ [0, 7]`，值域 `0..9`；
- 逐轮断言：
  - 区间 DP == 记忆化递归；
  - 区间 DP == 暴力最优；
  - 重建顺序回放收益 == 区间 DP 值。

该策略在小规模范围内提供较强正确性证据。

## R18

`demo.py` 源码级算法流程（8 步）：

1. `main` 调用 `run_case` 执行固定样例，再执行 `randomized_cross_check`。  
2. `burst_balloons_dp` 将输入标准化为整型数组，构造 `arr=[1]+nums+[1]`。  
3. 初始化 `dp` 与 `choice`，其中 `dp[l][r]` 表示开区间 `(l,r)` 的最优收益。  
4. 按区间跨度递增遍历 `(l,r)`，枚举最后戳破位置 `k`，计算候选值 `dp[l][k]+dp[k][r]+arr[l]*arr[k]*arr[r]`。  
5. 用最大候选更新 `dp[l][r]` 与 `choice[l][r]`，最终得到 `dp[0][m-1]`。  
6. 根据 `choice` 递归重建顺序：先重建左区间，再右区间，最后加入 `k-1`（映射回原下标）。  
7. `simulate_order_gain` 按重建顺序在“当前存活列表”上逐步回放，计算真实总收益。  
8. `run_case` 与 `randomized_cross_check` 分别将主算法结果和 `top_down`、`bruteforce`、回放收益对齐断言，全部通过即打印成功信息。
