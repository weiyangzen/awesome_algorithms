## R01

感知机（Perceptron）是最早的神经网络模型之一，也是线性二分类器的经典代表。

核心思想：
- 用线性函数 `f(x)=sign(w·x+b)` 对样本做二分类；
- 当样本被误分时，按规则修正参数 `w,b`；
- 通过反复遍历训练集，逐步找到能分开两类样本的超平面（若数据线性可分）。

本条目 MVP 使用 NumPy 从零实现二分类感知机，不调用黑箱训练器。

## R02

本条目问题定义（离散计算版）：

- 输入：
  - 训练数据 `X_train in R^(n*d)`；
  - 标签 `y_train in {-1,+1}^n`；
  - 超参数：学习率 `eta`、最大轮数 `max_epochs`。
- 输出：
  - 训练后的参数 `w in R^d, b in R`；
  - 每轮误分类数与训练/测试精度日志；
  - 收敛与正确性检查结果。

## R03

感知机模型与更新公式：

- 预测函数：

`y_hat = sign(w·x + b)`

- 样本 `(x_i, y_i)` 的分类间隔（功能间隔）：

`gamma_i = y_i * (w·x_i + b)`

- 若 `gamma_i <= 0`（误分或在分界面上），执行更新：

`w <- w + eta * y_i * x_i`

`b <- b + eta * y_i`

这就是感知机的核心在线学习规则。

## R04

时间复杂度（设样本数 `n`、特征维数 `d`、迭代轮数 `E`）：

- 单样本判断 + 可能更新：`O(d)`；
- 每轮遍历全部样本：`O(n*d)`；
- 总训练复杂度：`O(E*n*d)`。

若数据线性可分，`E` 通常在有限轮内收敛；不可分时，算法可能在不同超平面之间持续震荡。

## R05

空间复杂度：

- 数据矩阵 `X`：`O(n*d)`；
- 参数 `w` 与临时向量：`O(d)`；
- 训练日志：`O(E/report_every)`。

核心额外工作内存接近 `O(d)`（不含数据本体）。

## R06

一次误分类更新的微例：

假设当前参数：
- `w = [0.2, -0.4]`
- `b = 0.1`
- `eta = 1.0`

遇到样本：
- `x = [2.0, 1.0]`
- `y = +1`

先算间隔：
`gamma = y*(w·x+b) = 1*(0.2*2 + (-0.4)*1 + 0.1) = 0.1`

`gamma>0`，分类正确，不更新。

若换成 `x=[-2.0, 1.0], y=+1`，则 `gamma = -0.7 <= 0`，更新后：
- `w_new = [0.2, -0.4] + 1*(+1)*[-2.0, 1.0] = [-1.8, 0.6]`
- `b_new = 0.1 + 1*(+1) = 1.1`

## R07

感知机的重要性：

- 是“神经网络”历史上的起点模型；
- 奠定了“误分驱动参数更新”的在线学习思想；
- 连接线性分类、最大间隔思想与后续深度学习训练流程；
- 代码非常短小，适合作为机器学习入门与可解释基线。

## R08

理论基础（工程化摘要）：

1. 分类器是一个线性超平面，目标是把正负样本分到两侧。  
2. 误分样本满足 `y_i*(w·x_i+b)<=0`。  
3. 对误分点按 `w += eta*y_i*x_i` 更新，会提升该样本的功能间隔。  
4. 对线性可分数据，感知机收敛定理保证有限步内可找到一个可分超平面。  
5. 对线性不可分数据，标准感知机不保证收敛，通常需口袋算法/平均感知机等改进。

## R09

适用条件与局限：

- 适用于线性可分或近似线性可分的二分类任务；
- 适合在线学习、流式更新和轻量级部署；
- 不直接输出概率；
- 对不可分数据敏感，可能持续更新不稳定；
- 分类边界线性，表达能力有限（相比多层神经网络）。

## R10

正确性框架（本实现）：

1. 标签显式编码为 `{-1,+1}`，与感知机公式严格一致。  
2. 以 `margin = y*(w·x+b)` 判断误分条件（`margin<=0`）。  
3. 每个误分样本触发一次标准更新规则。  
4. 每轮统计 `epoch_errors`，若为 0 则触发收敛退出。  
5. 训练后断言：
   - 发生过至少一次更新；
   - 训练精度达到阈值；
   - 测试精度达到阈值；
   - 参数均为有限值。

## R11

误差与稳定性：

- 样本遍历顺序会影响最终可分超平面，随机打乱可降低顺序偏置；
- 特征标准化有助于更新步长在各维更均衡；
- 线性可分时，误分类数通常随 epoch 下降并趋于 0；
- 若训练误差长期不降，常见原因是数据不可分或学习率设置不当。

## R14

鲁棒性与常见失效模式：

- 数据非线性可分：误分类数无法归零，收敛判据不触发；
- 特征尺度差异过大：某些维度主导更新方向；
- 标签噪声较多：标准感知机震荡明显；
- 学习率过大：在边界附近反复来回更新；
- 类别极度不均衡：决策边界可能偏向多数类。

## R18

源码级算法流程拆解（`train_perceptron` 主流程，8 步）：

1. 初始化：读取配置，设置随机种子，令 `w=0, b=0`。  
2. 基线评估：计算未训练时的训练/测试精度，作为对照。  
3. epoch 循环：每轮先随机打乱训练样本索引。  
4. 单样本判定：对样本 `x_i` 计算 `margin = y_i*(w·x_i+b)`。  
5. 误分更新：若 `margin<=0`，执行 `w += eta*y_i*x_i`, `b += eta*y_i`，并累计误分计数。  
6. 轮末记录：计算并记录该轮 `errors/train_acc/test_acc`。  
7. 收敛判定：若该轮 `errors==0`，提前停止训练并记录收敛轮次。  
8. 验收断言：检查更新次数、最终精度阈值和参数有限性，通过后输出成功信息。

说明：本实现没有调用第三方黑箱训练函数；感知机的误分条件、参数更新与收敛判定都在源码中逐行展开。
