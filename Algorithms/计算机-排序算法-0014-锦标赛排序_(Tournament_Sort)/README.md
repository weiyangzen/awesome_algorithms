# 锦标赛排序 (Tournament Sort)

## R01

锦标赛排序（Tournament Sort）是一种基于“淘汰赛树”的选择式排序方法。  
它把待排序元素放在叶子节点，两两比较后逐层向上产生“胜者”，根节点就是当前最小值（或最大值）。  
每取出一个胜者后，只需沿其叶子到根的路径重算，就能得到下一个胜者。

本目录 MVP 目标：
- 实现一个可运行、可审计的最小可用版本（Python）；
- 明确展示“建树 -> 反复取冠军 -> 路径更新”的核心机制；
- 给出与 `sorted()` 的一致性校验和基础复杂度统计。

## R02

问题定义（升序）：

- 输入：长度为 `n` 的数值序列 `A = [a1, a2, ..., an]`；
- 输出：升序序列 `B`，满足：
  - `B` 是 `A` 的重排；
  - `B[0] <= B[1] <= ... <= B[n-1]`。

MVP 约束：
- 不依赖第三方排序黑箱；
- 支持重复值、负数、空数组；
- 结果可与 Python 内置 `sorted()` 自动对比验证。

## R03

选择锦标赛排序作为示例的原因：

- 它把“反复找最小值”的过程结构化为树更新，易于教学演示；
- 与堆排序类似都可视作树结构选择，但锦标赛树的“冠军路径”概念更直观；
- 适合解释外部排序中的“多路归并/败者树”思想前置概念。

## R04

核心数据结构（本实现）：

1. 叶子层
- 将输入扩展到不小于 `n` 的 2 的幂 `m`；
- 前 `n` 个叶子存真实元素 `(value, original_index)`；
- 其余叶子填充哨兵 `(+inf, +inf)`。

2. 内部节点
- 二叉树数组存储（1-based 概念，代码里用 Python list 模拟）；
- 每个内部节点存左右子树“较小者”（胜者）。

3. 稳定性处理
- 比较规则为 `(value, original_index)` 的字典序；
- 值相等时按原始下标决定先后，保证稳定输出。

## R05

算法流程（高层）：

1. 计算 `m = next_power_of_two(n)`，初始化树数组；
2. 把元素写入叶子区，底向上建树得到首个冠军；
3. 重复 `n` 次：
   - 读取根节点冠军并输出；
   - 将该冠军对应叶子置为哨兵；
   - 仅沿该叶子到根的路径重算胜者；
4. 收集得到完整升序结果。

## R06

正确性要点（不变式）：

- 不变式 1：任一节点始终保存其子树内的最小键（`(value, index)`）。
- 不变式 2：根节点始终是当前“尚未输出元素集合”的全局最小键。
- 不变式 3：每次输出后，把该叶子替换为哨兵，再路径重算后，不变式 1/2 继续成立。

由数学归纳可得：第 `k` 次输出的是剩余元素第 `k` 小值，因此整体输出为非降序；  
结合 `(value, original_index)` 比较，重复值保持原顺序，稳定性成立。

## R07

复杂度分析（`n` 为输入长度，`m` 为补齐后的 2 的幂，`n <= m < 2n`）：

- 建树：`O(m)`；
- 每次取冠军并更新路径：`O(log m)`；
- 共进行 `n` 次提取：`O(n log m)`，等价 `O(n log n)`；
- 额外空间：树与叶子存储约 `O(m)`，即 `O(n)`。

## R08

边界与异常处理策略：

- 空输入：直接返回空列表；
- 单元素：直接返回自身；
- 含重复值/负数/浮点值：统一按比较器处理；
- 非法输入（不可比较对象混入）：由 Python 比较阶段抛错，MVP 不做隐式吞错。

实现中使用哨兵 `(+inf, +inf)`，确保补齐叶子不会干扰真实最小值选择。

## R09

MVP 取舍说明：

- 采用“胜者树 + 叶子回溯更新”的直接实现，优先可读性与可验证性；
- 不实现复杂变体（败者树、外存块管理、多路并行）；
- 不追求极致常数性能，重点是机制透明；
- 保留比较次数、更新次数等指标，便于后续优化时做基线。

## R14

关键实现参数：

- `m`（补齐容量）：决定树高度与更新路径长度；
- 比较键 `(value, original_index)`：决定稳定性；
- 哨兵值：必须严格“大于任何真实键”；
- 随机样例规模（`n_random`）：用于演示时可调大观察统计趋势。

调优方向通常不是改算法阶数，而是减少对象创建与 Python 层循环开销。

## R15

与相关排序思路对比：

- 与选择排序：都重复取最小值；锦标赛排序把“全表扫描”改为“树路径更新”，从 `O(n^2)` 降为 `O(n log n)`；
- 与堆排序：复杂度同为 `O(n log n)`，都可在线维护最小值；堆更常用、实现更紧凑；
- 与归并排序：归并稳定且适合链表/外排；锦标赛树在“重复最小值选择”解释上更直观。

## R16

典型应用场景：

- 需要稳定选择过程且希望显式追踪比较路径的实验环境。

## R18

`demo.py` 源码级算法流程（8 步）：

1. `main` 构造测试用例（固定样例 + 随机样例），逐个调用 `run_case`。
2. `run_case` 创建 `TournamentTreeSorter(values)`，内部将每个元素包装为 `(value, original_index)`。
3. 初始化时用 `next_power_of_two` 计算叶子容量 `m`，把真实叶子与哨兵叶子写入树底层。
4. `_build_tree` 自底向上比较左右孩子，把胜者写入父节点，最终根节点成为全局最小值。
5. `sort` 循环 `n` 次：读取根冠军并写入输出列表。
6. 依据冠军的 `original_index` 定位到对应叶子，将该叶子替换为哨兵，表示“该元素已出局”。
7. 从该叶子向上到根调用 `_recompute_path`，只重算一条路径即可恢复新的全局冠军。
8. 循环结束后返回 `TournamentSortResult`，`run_case` 用 `sorted(input)` 校验并输出统计信息。
