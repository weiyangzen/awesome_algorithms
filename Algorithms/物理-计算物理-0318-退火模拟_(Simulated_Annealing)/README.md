# 退火模拟 (Simulated Annealing)

- UID: `PHYS-0315`
- 学科: `物理`
- 分类: `计算物理`
- 源序号: `318`
- 目标目录: `Algorithms/物理-计算物理-0318-退火模拟_(Simulated_Annealing)`

## R01

退火模拟（Simulated Annealing, SA）是一种受统计物理启发的全局优化方法：
通过在高温阶段允许“爬坡”（接受更差解），低温阶段逐步收敛到低能态，从而降低陷入局部极小的风险。

本条目提供一个可运行 MVP：在二维 Rastrigin 粗糙能量地形上执行 SA，目标是逼近全局最小能量 `E*=0`。

## R02

为什么选这个问题作为计算物理演示：

1. Rastrigin 地形具有大量局部极小，能直观体现“爬坡-降温-收敛”的核心机制。
2. 目标函数有明确全局最优点 `(0,0)`，便于做正确性验证。
3. 搜索空间和预算可控，适合作为无交互、可复现实验脚本。

## R03

本条目中的能量函数：

- 维度 `d=2`；
- 定义域 `x_i in [-5.12, 5.12]`；
- 能量
  `E(x) = 10d + Σ_i [x_i^2 - 10 cos(2πx_i)]`。

性质：
- 全局最小值 `E(0,0)=0`；
- 局部极小点众多，随机初始化时直接贪心下降容易被困住。

## R04

Metropolis 接受准则（SA 核心）：

- 若候选态能量更低（`ΔE <= 0`），总是接受；
- 若候选态更差（`ΔE > 0`），按概率
  `p = exp(-ΔE / T)` 接受；
- 温度 `T` 越高，越容易接受上坡步，探索更强；
- 温度 `T` 降低后，上坡接受概率收缩，逐步转向利用（收敛）。

## R05

降温策略：指数冷却（Exponential Cooling）

- `T_k = max(T_min, T0 * alpha^k)`；
- 本实现参数：
  - `T0 = 5.0`
  - `alpha = 0.9993`
  - `T_min = 1e-8`

该设置在 12000 步内能保持前期探索与后期收敛的平衡。

## R06

邻域提案（Proposal）与边界处理：

- 从当前状态 `x` 生成候选 `x' = x + Normal(0, sigma)`；
- 使用逐维截断 `clip(x', low, high)` 保证可行性；
- 本实现 `sigma = 0.4`。

说明：`sigma` 过小会导致扩散慢、易停在局部盆地；过大则会降低可接受率。

## R07

复杂度分析：

- 单步成本：`O(d)`（一次提案、一次能量计算）；
- 总时间复杂度：`O(n_steps * d)`；
- 空间复杂度：`O(d + n_trace)`，其中 `n_trace` 是诊断采样点数量。

本 MVP 的 `d=2`，计算量主要由步数 `n_steps` 决定。

## R08

`demo.py` 的模块结构：

1. `SAConfig`：统一管理维度、边界、温度与随机种子。
2. `rastrigin_energy`：能量函数实现。
3. `_temperature`：指数冷却调度。
4. `simulated_annealing`：主循环（提案、接受、更新、追踪）。
5. `random_search_baseline`：同预算随机搜索基线。
6. `run_multi_seed_benchmark`：多随机种子对比实验。
7. `summarize_benchmark`：生成均值/中位数等统计。
8. `main`：打印结果并执行断言验证。

## R09

可复现与稳定性策略：

- 全部随机过程使用 `numpy.random.default_rng(seed)`；
- 每轮 benchmark 通过可追踪种子偏移生成；
- 无需交互输入；
- 输出字段固定，适合自动校验。

## R11

脚本输出包含三部分：

1. **Single Run Summary**：单次退火的最优能量、最终能量、接受率、最优状态。
2. **Annealing Trace**：按固定步长采样的温度与能量轨迹。
3. **Benchmark Stats**：30 次多种子实验中，SA 与随机搜索的 `mean/median/std/min/max` 对比。

## R12

本条目的关键评价指标：

- `best_energy`：全程最优能量（越低越好）；
- `final_energy`：最后一步所在状态能量；
- `accepted_ratio`：总接受率；
- `uphill_accept_ratio`：已接受步中“上坡步”的比例；
- 多次试验的 `median(best_energy)`：鲁棒性能指标。

## R13

断言策略（自动验收用）：

1. 单次运行 `best_energy < 0.1`；
2. 接受率落在合理区间：`0.005 < accepted_ratio < 0.2`；
3. 多次试验中，SA 中位数优于随机搜索：
   `median_SA < 0.25 * median_random`。

这三条覆盖了“收敛到低能态 + 非退化动力学 + 相对基线优势”。

## R14

常见失效模式与排查：

1. 冷却过快（`alpha` 太小）导致过早冻结；
2. 初温过低（`T0` 太小）导致无法跨越能垒；
3. 步长不合适（`sigma` 太大/太小）导致接受率异常；
4. 忽略多种子统计，只看单次结果造成误判；
5. 忘记边界裁剪导致提案越界。

## R15

适用与不适用场景：

适用：
- 粗糙非凸能量面；
- 梯度不可用或代价高；
- 需要低工程复杂度的全局搜索基线。

不适用：
- 高维连续优化且梯度易得（梯度法通常更高效）；
- 需要严格最优性证明；
- 强实时约束下的大规模在线优化。

## R16

与相关方法关系：

- 与贪心局部搜索相比：SA 允许受控上坡，跳出局部极小能力更强；
- 与随机搜索相比：SA 同预算下利用局部结构，通常收敛更快；
- 与遗传算法/粒子群相比：SA 单轨迹、参数更少，实现更轻量；
- 与并行温度交换（Parallel Tempering）相比：本实现是单温链的最小版本。

## R17

可扩展方向：

1. 自适应温度与自适应步长（把接受率控制在目标区间）；
2. 引入重热（reheating）应对停滞；
3. 混合局部精修器（如末端局部二次近似）；
4. 扩展到离散系统（如 Ising 自旋翻转）；
5. 并行多链与温度交换，提高复杂地形下鲁棒性。

## R18

`demo.py` 的源码级算法流程（8 步）：

1. `main()` 构造 `SAConfig`（`n_steps=12000, T0=5.0, alpha=0.9993, sigma=0.4`）。
2. `simulated_annealing()` 用种子初始化 `x0 ~ Uniform([low, high]^d)`，计算起始能量。
3. 每步 `k` 调用 `_temperature(k, cfg)` 得到 `T_k`，执行指数降温。
4. 生成候选 `x' = clip(x + Normal(0, sigma), low, high)`，计算 `ΔE = E(x') - E(x)`。
5. 按 Metropolis 准则决定是否接受：`ΔE<=0` 必接收，否则按 `exp(-ΔE/T_k)` 概率接收。
6. 若接受则更新当前状态；同时持续维护历史最优 `best_x, best_e`，并统计接受率与上坡接受率。
7. 按 `trace_every` 记录诊断表（步数、温度、当前能量、最优能量、接受率）。
8. `run_multi_seed_benchmark()` 在 30 个种子上重复 SA，并与同预算 `random_search_baseline()` 对比，`main()` 汇总统计并用断言自动验证性能。

说明：实现未调用任何“黑盒 SA”库函数，温度调度、提案机制、Metropolis 接受逻辑、统计汇总与校验均在源码中显式展开。
