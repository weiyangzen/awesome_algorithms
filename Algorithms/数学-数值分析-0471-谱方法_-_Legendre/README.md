# 谱方法 - Legendre

## R01

这是一个 **Legendre 谱 Galerkin 方法** 的最小可运行实现，用来求解一维椭圆方程
`-u''(x) + u(x) = f(x)`（区间 `[-1,1]`，齐次 Dirichlet 边界）。

目标是展示谱方法的核心价值：在解足够光滑时，用较少自由度获得高精度近似。

## R02

问题设定：

- 定义域：`x ∈ [-1,1]`
- 方程：`-u'' + u = f`
- 边界：`u(-1)=u(1)=0`
- 验证用真解：`u*(x)=exp(x)(1-x^2)`
- 对应右端项：`f(x)=exp(x)(4x+2)`

这样可以直接评估数值解误差，而不是只看残差。

## R03

Legendre 多项式 `P_k(x)` 在 `[-1,1]` 上正交，适合做全局高阶近似。

为了天然满足边界条件，选取试探/检验基：

`φ_k(x) = P_k(x) - P_{k+2}(x),  k=0,...,N-1`

因为 `P_k(±1)=(±1)^k`，所以 `φ_k(±1)=0`。

## R04

弱形式：求 `u_N = Σ c_k φ_k`，使得对任意 `φ_j`：

`∫_{-1}^1 (u_N' φ_j' + u_N φ_j) dx = ∫_{-1}^1 f φ_j dx`

离散后得到线性系统：

`A c = b`

其中 `A = K + M`，

- `K_{jk} = ∫ φ_k' φ_j' dx`（刚度矩阵）
- `M_{jk} = ∫ φ_k φ_j dx`（质量矩阵）
- `b_j = ∫ f φ_j dx`

## R05

积分采用 Gauss-Legendre 求积（`numpy.polynomial.legendre.leggauss`）：

`∫_{-1}^1 g(x) dx ≈ Σ w_q g(x_q)`

在 `demo.py` 中，这一步用于组装 `K`、`M`、`b` 和计算 `L2` 误差。

## R06

实现策略（MVP）：

- 使用 `numpy` 做向量化矩阵组装与线性代数求解
- 可选使用 `pandas` 打印更清晰的收敛表（不可用时自动回退为纯文本）
- 不依赖交互输入，不写外部文件，直接命令行运行

## R07

时间复杂度（主导项）：

- 预计算基函数值/导数：`O(N * Q)`
- 矩阵组装：`O(N^2 * Q)`
- 线性求解（致密）：`O(N^3)`

其中 `N` 为谱模态数，`Q` 为求积点数。对于小中规模 `N`，实现简洁、精度高。

## R08

数值稳定性要点：

- 采用满足边界条件的基，避免额外罚项/边界处理误差
- 方程含 `+u` 项，系统矩阵更稳健（相较纯 Poisson 仅刚度项）
- 使用足够求积点，避免积分欠采样引入的离散误差

## R09

`demo.py` 关键函数：

- `exact_solution` / `forcing_term`：构造可验证问题
- `legendre_values_and_derivatives`：评估 `P_k` 与 `P_k'`
- `solve_legendre_galerkin`：组装并求解 `A c = b`
- `evaluate_legendre_series`：在任意网格重构近似解
- `compute_errors`：计算 `L_inf` 与 `L2` 误差
- `convergence_study`：输出不同 `N` 的误差趋势

## R11

结果解读建议：

- `n_modes=14` 时误差应已较小（通常显著优于低阶网格法同等自由度）
- 随 `n_modes` 增大，`L2_error` 应总体下降
- 对光滑真解，谱法常表现为“快于代数阶”的收敛趋势

## R12

正确性检查清单：

- 检查边界值：`u_N(-1), u_N(1)` 应接近 0
- 检查误差：`L_inf`, `L2` 随 `n_modes` 增大下降
- 检查可复现性：同参数下多次运行结果一致

## R13

参数建议：

- 入门演示：`n_modes = 8~14`
- 更高精度：适当增大 `n_modes` 与求积点数
- 如果出现不稳定/误差异常，优先检查 `quad_n` 是否过小

## R14

常见坑：

- 直接使用 `P_k` 作为基会破坏齐次边界条件
- 求积点过少会导致矩阵项积分不准确
- 把“残差小”误当“真误差小”，应优先对照解析解或高精参考

## R15

与常见方法对比（简述）：

- 有限差分：实现最简单，但同精度下通常需要更多网格点
- 低阶有限元：几何/边界灵活，但高精度往往需显著加密或升阶
- Legendre 谱法：对光滑问题非常高效，但更适合规则区间与全局近似

## R18

`demo.py` 的源码级算法流（8 步）：

1. 在 `exact_solution` 和 `forcing_term` 中固定一个有解析解的问题，保证误差可直接评估。  
2. 在 `solve_legendre_galerkin` 里调用 `leggauss` 生成 Gauss-Legendre 节点与权重。  
3. 在 `legendre_values_and_derivatives` 中逐阶构造 `Legendre.basis(k)`，并计算 `P_k(x_q)` 与 `P'_k(x_q)`。  
4. 组合 `φ_k=P_k-P_{k+2}` 与 `φ'_k=P'_k-P'_{k+2}`，把边界条件内嵌到基函数层。  
5. 通过加权矩阵乘法 `(dphi*w)@dphi.T` 与 `(phi*w)@phi.T` 分别组装刚度矩阵和质量矩阵。  
6. 用 `(phi*w)@f(x_q)` 组装右端，形成线性系统 `A c = b` 并由 `np.linalg.solve` 求系数。  
7. 在 `evaluate_legendre_series` 中按同一基函数重构 `u_N(x)`，并在 `compute_errors` 中计算 `L_inf` 与 `L2`。  
8. 在 `convergence_study` 中循环不同 `n_modes`，输出误差表验证谱近似收敛趋势。  

以上流程把第三方函数（如 `leggauss`、`Legendre.basis`）放在可追踪的装配链路中，而不是黑箱“一步到位”。
