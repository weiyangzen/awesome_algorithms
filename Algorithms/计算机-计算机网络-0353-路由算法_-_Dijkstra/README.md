## R01

Dijkstra 是经典的单源最短路径算法，适用于边权非负图。在计算机网络中，它常作为链路状态路由协议（如 OSPF、IS-IS）中 SPF（Shortest Path First）计算的核心。

## R02

路由语境下可把网络看作带权图：
- 顶点表示路由器。
- 边表示链路。
- 权重表示链路代价（可由带宽、时延、管理策略等综合得到）。
算法目标是：给定本地路由器 `source`，求到所有可达节点的最小总代价与下一跳。

## R03

本 MVP 输入：
- 一个非负权重拓扑 `graph: dict[str, list[tuple[str, float]]]`
- 源节点 `source`

输出：
- 每个目标节点的最短距离 `distance`
- 前驱映射 `predecessor`（用于回溯完整路径）
- 路由表项（目标、下一跳、总代价、完整路径）

## R04

问题约束与适用边界：
- 边权必须 `>= 0`，否则 Dijkstra 不保证正确。
- 支持连通或非连通图；不可达节点距离为 `inf`。
- 本实现使用邻接表和最小堆，适合稀疏图与教学演示。

## R05

核心状态变量：
- `distances[v]`：当前已知从 `source` 到 `v` 的最短上界。
- `predecessors[v]`：当前最优路径上 `v` 的前驱节点。
- `priority_queue`：候选节点最小堆（按当前距离排序）。
- `visited`：已“定型”（最短路已确定）的节点集合。

## R06

算法主循环（高层）：
1. 初始化 `source` 距离为 0，其余为无穷大。
2. 每次弹出堆顶最小距离节点 `u`。
3. 若 `u` 已定型或为过期条目则跳过。
4. 对 `u` 的每个邻居 `v` 做松弛：若 `dist[u] + w(u,v)` 更小，则更新 `dist[v]` 和 `pred[v]` 并入堆。
5. 堆空后结束。

## R07

正确性直觉：
- 在非负权条件下，最小堆每次取出的未定型节点 `u`，其当前距离已不可能再被更短路径改写。
- 因此可安全将 `u` 加入 `visited`，并只对其出边做一次“有效扩展”。
- 重复此过程直至候选集耗尽，可得到从源点到所有可达点的最短路。

## R08

复杂度分析：
- 时间复杂度：`O((V + E) log V)`（二叉堆实现）。
- 空间复杂度：`O(V + E)`（邻接表、距离表、前驱表、堆）。

## R09

在路由系统中的落地含义：
- `distances` 对应最短路径树上的代价。
- `predecessors` 可还原 SPF 树。
- 将目标路径的第二个节点作为 `next_hop`，即可形成本地路由转发表的核心字段。

## R10

本目录实现选择：
- 使用 Python 标准库（`heapq`、`dataclasses`、`typing`），避免额外依赖。
- 拓扑使用小规模静态样例，保证可读性和可重复性。
- 输出同时给出“距离表 + 下一跳路由表 + 路径明细”。

输出字段说明：

## R13

结果解读建议：
- 若存在多条等长最短路，本实现会保留“首次达到最优值”的那条（取决于边遍历顺序和堆弹出顺序）。
- 某个目标若不可达，会在距离表中保持 `inf`，并不会出现在最终路由表中。

## R14

边界与异常处理：
- 若 `source` 不在图中，抛出 `ValueError`。
- 若检测到负权边，抛出 `ValueError`（防止错误使用）。
- 路径回溯时若前驱链断裂，返回空路径表示不可达或数据不一致。

## R15

与 Bellman-Ford 对比：
- Dijkstra 更快（典型 `O((V+E)logV)`），但要求非负权。
- Bellman-Ford 可处理负权边并检测负环，代价更高（`O(VE)`）。
- 网络路由中通常通过代价设计保证非负，因此 Dijkstra 很常见。

## R16

可扩展方向：
- 支持 ECMP：保留多个等价前驱，生成多下一跳。
- 将静态拓扑改为链路状态数据库输入，模拟 LSA 变化后重算 SPF。
- 引入增量 SPF 或分层区域，贴近真实 OSPF 实现。

## R18

源码级流程拆解（对应 `demo.py`）：
1. `build_sample_topology()` 构造带非负权重的无向网络图（邻接表）。
2. `main()` 设定源节点 `R1`，调用 `dijkstra(graph, source)` 计算最短路。
3. `dijkstra()` 初始化 `distances` 为无穷、`source=0`，并把 `(0, source)` 压入最小堆。
4. 循环弹出堆顶 `(current_distance, node)`，若该条目过期或节点已定型则跳过。
5. 遍历 `node` 的邻居，执行松弛：若 `current_distance + weight < distances[neighbor]`，则更新 `distances` 与 `predecessors`，并把新候选压入堆。
6. 堆处理完成后，`distances/predecessors` 即形成最短路径树信息。
7. `build_routing_table()` 对每个目标调用 `reconstruct_path()` 回溯路径，并取路径第二跳作为 `next_hop`。
8. `print_distance_table()` 与 `print_routing_table()` 输出距离和路由表，形成可直接核对的 MVP 结果。
