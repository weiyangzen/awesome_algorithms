# 哈勃定律 (Hubble's Law)

- UID: `PHYS-0051`

- 分类: `宇宙学`

## R01

哈勃定律描述低红移宇宙中的线性关系：天体退行速度 `v` 与距离 `d` 近似满足

\[
v = H_0 d
\]

其中 `H0` 为哈勃常数（单位常写为 `km/s/Mpc`）。在低红移近似下，`v ≈ c z`，因此也可写为 `cz ≈ H0 d`。

本条目的目标是给出一个可运行、可审计的最小算法 MVP：从带噪声的“距离-红移”样本估计 `H0` 并做基础统计检验。

## R02

MVP 场景采用合成观测目录（synthetic catalog）：
- 距离 `d` 在给定区间内抽样（单位 Mpc）。
- 真实速度 `v_true = H0_true * d`。
- 观测速度 `v_obs` 叠加本动速度散射和测量噪声。
- 观测红移 `z_obs = v_obs / c`。

核心任务：
1. 用加权最小二乘估计 `H0`（原点约束模型）。
2. 计算置信区间与拟合质量指标。
3. 用带截距模型做诊断，检查是否存在异常系统偏移。

## R03

数学建模分两层：

1. 观测模型（低红移）：
\[
v_i = H_0 d_i + \epsilon_i,
\quad z_i = \frac{v_i}{c}
\]
其中 `ε_i` 代表本动速度和测量误差。

2. 异方差噪声：
\[
\sigma_{v,i} = \sqrt{\sigma_{\text{pec}}^2 + (f_{\text{meas}}\,v_{\text{true},i})^2}
\]
故每个样本具有不同权重 `w_i = 1/σ_{v,i}^2`。

该模型保留了教学上最关键的物理关系，同时避免过重的宇宙学参数化。

## R04

主估计器使用“过原点”的加权最小二乘（WLS）：
\[
\hat H_0 = \arg\min_H \sum_i w_i (v_i - H d_i)^2
\]
其闭式解为
\[
\hat H_0 = \frac{\sum_i w_i d_i v_i}{\sum_i w_i d_i^2}
\]

并计算：
- `chi2 = Σ((v_i - Ĥ0 d_i)/σ_i)^2`
- `reduced_chi2 = chi2 / (N-1)`
- 标准误差与基于 t 分布的置信区间
- `R²` 作为线性解释度的辅助指标

另外实现诊断模型 `v = H0 d + b`，检查截距 `b` 是否异常偏离 0。

## R05

关键假设：
- 仅考虑低红移区间，忽略广义相对论下高红移非线性效应。
- 采用欧式距离量纲（Mpc）并假设距离估计无系统偏差。
- 噪声独立，且可由 `σ_pec` 与比例测量误差近似。
- 合成样本不代表真实巡天选择函数，仅用于算法演示与验证。

## R06

`demo.py` 输入输出约定：
- 输入：脚本内置配置 `HubbleConfig`（样本数、真值 `H0`、噪声规模、随机种子等）。
- 输出：
1. 样本头部表（`distance_mpc`, `redshift_obs`, `velocity_obs_kms`, `sigma_velocity_kms`）；
2. 拟合汇总表（原点模型与带截距模型）；
3. 阈值检查项与最终 `Validation: PASS/FAIL`。

脚本无交互输入，直接运行可复现结果。

## R07

算法主流程（高层）：
1. 读取 `HubbleConfig`。
2. 生成合成目录：`d -> v_true -> v_obs -> z_obs`。
3. 对 `v = H0 d` 执行显式 WLS，得到 `H0`、SE、CI、`chi2`、`R²`。
4. 对 `v = H0 d + b` 执行 WLS 诊断拟合，得到 `b` 与其误差。
5. 汇总结果为 `pandas.DataFrame`。
6. 执行阈值检查（低红移、真值覆盖、截距约束等）。
7. 输出 PASS/FAIL，并在失败时返回非零退出码。

## R08

设样本量为 `N`：
- 合成数据生成：`O(N)`
- 原点 WLS：`O(N)`（仅需要几项加权求和）
- 带截距 WLS：`O(N)`（`2x2` 线性系统求解，常数开销）
- 汇总与打印：`O(N)`

总体时间复杂度 `O(N)`，空间复杂度 `O(N)`。

## R09

数值与统计细节：
- 对 `σ` 使用 `clip(σ, EPS, None)` 防止除零。
- 采用固定随机种子，保证可复现实验。
- 使用 `reduced_chi2` 放缩参数协方差，更贴近“噪声规模未知但可估计”的实践。
- 置信区间使用 `scipy.stats.t.ppf` 而非正态近似，避免小样本低估不确定性。
- 检查 `max|z| < 0.2`，确保使用 `v≈cz` 的前提大体成立。

## R12

输出字段解释：
- `distance_mpc`: 星系距离（Mpc）。
- `redshift_obs`: 观测红移 `z = v/c`。
- `velocity_obs_kms`: 观测退行速度（km/s）。
- `sigma_velocity_kms`: 每个样本的速度不确定度。
- `H0_est_km_s_Mpc`: 拟合得到的哈勃常数。
- `H0_se`: `H0` 标准误差。
- `CI_low`, `CI_high`: 置信区间边界（仅主模型输出）。
- `chi2_reduced`: 约化卡方。
- `R2`: 线性拟合解释度（主模型输出）。
- `intercept_km_s`: 诊断模型截距。

## R13

脚本内置正确性检查：
1. `max |z| < 0.2`（低红移前提）。
2. `|H0_est - H0_true| < 3 * H0_se`（估计稳定）。
3. `H0_true` 落在给定置信区间内（区间覆盖性）。
4. 诊断截距 `|b| < 500 km/s`（无显著整体偏移）。

以上全部满足时输出 `Validation: PASS`，否则 `FAIL` 并退出非零状态。

## R17

方案比较：
- 直接调用高阶天体物理工具库：实现快，但估计链路不够透明。
- 只做简单 OLS：忽略异方差，参数不确定性往往失真。
- 本方案：以最小代码显式实现 WLS、CI 与阈值校验，兼顾可解释性与可复现实验。

因此它适合作为“哈勃定律算法化”的第一步实现。

## R18

`demo.py` 源码级算法流程（8 步）：
1. `main` 创建 `HubbleConfig`，并调用 `make_synthetic_catalog` 生成样本。
2. `make_synthetic_catalog` 先采样 `distance_mpc`，计算 `velocity_true_kms = H0_true * d`。
3. 同函数内构造异方差噪声 `sigma_velocity_kms`，叠加高斯扰动得到 `velocity_obs_kms`，再计算 `redshift_obs = v/c`。
4. `fit_hubble_through_origin` 读取 `(d, v, sigma)`，计算权重 `w=1/sigma^2`。
5. 依据闭式解 `H0 = Σ(w d v)/Σ(w d^2)` 得到主估计值，并计算残差、`chi2`、`reduced_chi2`、`R²`。
6. 同函数用 `stats.t.ppf` 计算临界值，给出 `H0` 的置信区间。
7. `fit_hubble_with_intercept` 构造 `X=[d, 1]`，显式求解 `(X^T W X)^(-1)X^T W y` 得到诊断截距 `b`。
8. `main` 汇总 `DataFrame`，执行四项阈值检查并输出 `Validation: PASS/FAIL`。
