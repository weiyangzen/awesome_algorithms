# 连续性方程 (Continuity Equation)

- UID: `PHYS-0083`
- 学科: `物理`
- 分类: `流体力学`
- 源序号: `83`
- 目标目录: `Algorithms/物理-流体力学-0083-连续性方程_(Continuity_Equation)`

## R01

连续性方程表达的是“守恒量不会凭空产生或消失”的局部形式。对流体质量守恒，常见写法为：

`d(rho)/dt + div(rho * u) = 0`

其中 `rho` 是密度，`u` 是速度场。对不可压缩流且 `rho` 常数，可退化为 `div(u)=0`。

## R02

它是流体力学最基础的方程之一，与动量方程（Navier-Stokes / Euler）共同构成流动求解核心。无论是管流、气动、海洋环流还是数值仿真，连续性方程都用于保证“质量账本”平衡。

## R03

本条目 MVP 选择 1D 周期边界场景，求解保守形式：

`d(rho)/dt + d(rho*u)/dx = 0`

输入：
- 计算域长度 `L`、网格数 `N`；
- 常速 `u`；
- CFL 数和终止时间；
- 稳定且正值的初始密度波形。

输出：
- 数值解 `rho_numerical`；
- 解析平移解 `rho_exact`（用于校验）；
- 质量守恒漂移、`L2` 相对误差、`L_inf` 误差、离散残差。

## R04

时间复杂度（`N` 个网格、`T` 个时间步）：

- 每步只做若干次数组移位与逐元素运算，复杂度 `O(N)`；
- 总复杂度 `O(N*T)`。

本实现中 `T` 由 CFL 自动确定，且随 `N` 线性增大（固定物理时间和速度下）。

## R05

空间复杂度：

- 主要存储 `rho`、通量和参考解等一维数组；
- 总体 `O(N)`。

没有使用稀疏矩阵、线性求解器或大型缓存结构。

## R06

微型直观例子：

- 若速度 `u > 0`，密度峰会向右平移；
- 在周期边界下，峰离开右端后会从左端回卷；
- 若算法正确且保守，则总质量 `sum(rho_i * dx)` 在整个仿真中几乎不变。

MVP 正是用这个“可验证、可对照”的场景来验证连续性方程离散实现。

## R07

算法意义：

- 把偏微分方程守恒律转成可执行数值更新；
- 明确展示“通量差 = 单元质量变化”；
- 为后续扩展到变速场、二维有限体积、可压缩流模型提供最小基线。

## R08

本实现使用的核心关系：

1. 守恒型 PDE：`d(rho)/dt + dF/dx = 0`，其中 `F=rho*u`。
2. 单元平均有限体积更新：`rho_i^{n+1} = rho_i^n - (dt/dx)(F_{i+1/2} - F_{i-1/2})`。
3. 上风通量（`u>=0`）：`F_{i+1/2} = u * rho_i`。
4. 上风通量（`u<0`）：`F_{i+1/2} = u * rho_{i+1}`。
5. 周期边界通过 `roll` 处理邻接关系。
6. CFL 约束：`dt <= CFL * dx / |u|`。

## R09

适用条件：

- 对流主导、守恒律教学或原型验证；
- 周期边界问题；
- 需要清晰可解释而非高阶复杂格式。

局限：

- 一阶上风数值耗散明显；
- 当前仅 1D 且速度为常数；
- 未包含冲击捕捉、极限器、Riemann 求解等更高阶技术。

## R10

正确性要点：

- 更新公式来自守恒积分形式，因此质量守恒是结构性保证；
- 周期边界下，全域通量望远镜求和抵消，理论上总质量不变；
- 常速平流存在解析解（初值平移），可直接比较数值误差；
- 代码同时检查离散残差、质量漂移和解析误差，形成闭环验证。

## R11

数值稳定与精度实践：

- 通过 CFL 自动选步长，避免显式格式失稳；
- 使用正值且光滑初始密度，减小离散振荡风险；
- 误差评估同时给出 `L2` 与 `L_inf`，避免单指标盲区；
- 对保守律最关键的质量误差设置严格断言阈值。

## R12

性能视角：

- 热点是 NumPy 向量化数组运算和 `np.roll`；
- 单步计算是纯 `O(N)` 内存顺序访问，开销很轻；
- 在更大规模上可进一步考虑：
- Numba/JAX 加速；
- 二维/三维时的 cache-friendly 数据布局。

## R14

常见失效模式：

1. CFL 过大，显式格式可能失稳或误差急剧增大；
2. 把边界从周期改为开边界却未同步修改通量处理；
3. 初值含负密度或异常值，导致非物理状态；
4. 把强间断问题直接交给一阶格式，数值扩散过重。

## R15

`demo.py` 模块设计：

- `ContinuityConfig`：管理网格、速度、CFL、终止时间；
- `ContinuityResult`：集中返回结果与诊断指标；
- `validate_config`：参数合法性检查；
- `make_grid`：构建单元中心网格；
- `initial_density`：生成可解析平移的初值；
- `upwind_face_flux`：上风界面通量；
- `advance_one_step`：有限体积单步更新；
- `exact_density`：解析参考解；

## R18

`demo.py` 源码级算法流程（8 步，非黑盒）如下：

1. `main` 构造 `ContinuityConfig`，调用 `solve_continuity` 进入主流程。  
2. `solve_continuity` 先做参数校验，随后由 `make_grid` 生成 1D 周期网格与 `dx`。  
3. `initial_density` 生成初始密度 `rho^0`，并记录初始总质量。  
4. 按 CFL 计算 `dt` 和时间步数 `num_steps`，确保显式更新稳定。  
5. 每个时间步调用 `advance_one_step`：先用 `upwind_face_flux` 构造 `F_{i+1/2}`，再用通量差更新 `rho_i^{n+1}`。  
6. 同步计算离散残差 `((rho^{n+1}-rho^n)/dt + (F_{i+1/2}-F_{i-1/2})/dx)` 的无穷范数，检查公式实现一致性。  
7. 时间推进结束后，`exact_density` 用平移解析式生成 `rho_exact`，并计算质量漂移、`L2` 相对误差、`L_inf` 误差。  
8. `run_checks` 对守恒性、误差阈值、正密度和离散残差做断言；全部通过后打印 `All checks passed.`。  

说明：第三方库只用于数组存储与向量化算子；连续性方程离散、通量构造、校验逻辑均在源码中显式实现。
