# 算术级数中的素数算法

- UID: `MATH-0049`
- 学科: `数学`
- 分类: `解析数论`
- 源序号: `49`
- 目标目录: `Algorithms/数学-解析数论-0049-算术级数中的素数算法`

## R01

本题关注的问题是：在固定同余类 `p ≡ a (mod q)` 中，如何高效找出并统计所有不超过 `N` 的素数。它是解析数论中“狄利克雷素数定理”和“算术级数中素数分布”最常见的计算实验入口。

本目录提供一个可直接运行的 MVP：
- 用埃拉托斯特尼筛一次性建立 `0..N` 的素性表；
- 在步长为 `q` 的算术级数上提取满足 `p ≡ a (mod q)` 的素数；
- 对 `gcd(a,q)=1` 的同余类，给出 `pi(N; q, a) ≈ li(N)/phi(q)` 的简化估计并与实测对照。

## R02

有限计算版问题定义：
- 输入：上界 `N`、模数 `q>0`、剩余类 `a`。
- 输出：
  - 列表 `P = {p <= N | p 为素数, p ≡ a (mod q)}`；
  - 精确计数 `pi(N; q, a)`；
  - 当 `gcd(a,q)=1` 时的估计计数 `li(N)/phi(q)`（离散近似）。

## R03

数学背景（与实现直接相关）：
- 狄利克雷定理：若 `gcd(a,q)=1`，则同余类 `a mod q` 中有无穷多素数。
- 素数定理在算术级数中的形式（PNT-AP 直觉）：`pi(x; q, a)` 在不同互素剩余类之间渐近均衡。
- 工程上常用粗近似：
  - `pi(x; q, a) ≈ li(x)/phi(q)`，
  - 其中 `li(x)` 用离散和 `sum(1/log n)` 近似。

## R04

算法总流程：
1. 读入固定实验参数 `N`、`q` 与若干 `a`。
2. 运行筛法得到 `is_prime[0..N]`。
3. 对每个 `(a,q)` 归一化 `a <- a mod q`。
4. 从该同余类中第一个 `>=2` 的项开始，以步长 `q` 扫描。
5. 使用 `is_prime[n]` 进行 O(1) 素性判定并收集结果。
6. 统计计数并输出前若干样例素数。
7. 对 `gcd(a,q)=1` 的类计算估计值 `li(N)/phi(q)` 做对照。

## R05

核心数据结构：
- `bytearray is_prime`：筛法素性表，节省内存且查询快。
- `list[int]`：某个同余类下的素数列表。
- `dict[int, int]`：`q` 的各互素剩余类计数字典（用于分布对比）。

这些结构都可序列化、可复用，也便于后续替换成分段筛版本。

## R06

正确性要点：
- 筛法正确性：埃拉托斯特尼筛删除所有合数并保留全部素数。
- 同余筛选正确性：扫描序列严格为 `a + kq`，因此收集结果必然满足 `p ≡ a (mod q)`。
- 完备性：从该同余类第一个 `>=2` 的元素扫到 `N`，不会漏掉任何满足条件的素数。

## R07

复杂度分析（单次上界 `N`）：
- 建筛：时间 `O(N log log N)`，空间 `O(N)`。
- 单同余类扫描：约 `O(N/q)` 次检查。
- 若同一 `q` 下扫描多个 `a`：总计约 `O(kN/q)`（`k` 为类数）。
- `li` 离散近似：`O(N)`（演示规模下可接受）。

## R08

边界与异常处理：
- `q <= 0`：抛出 `ValueError`。
- `N < 2`：直接返回空结果。
- 外部传入的 `prime_table` 长度不足：抛出 `ValueError`。
- `gcd(a,q) != 1`：仍可精确统计该类素数，但不使用 `li/phi(q)` 估计（输出 `N/A`）。

## R09

MVP 取舍说明：
- 选择“全局筛 + 同余过滤”，而不是更复杂的分段同余筛，保证代码短小可读。
- 估计公式仅用于解析数论直觉对照，不作为严格误差界。
- 不依赖大型第三方数值框架，`python3 demo.py` 即可运行。

## R10

`demo.py` 函数职责：
- `sieve_eratosthenes`：构建素性表。
- `euler_phi`：计算 `phi(q)`。
- `logarithmic_integral_discrete`：离散近似 `li(x)`。
- `primes_in_arithmetic_progression`：提取 `p ≡ a (mod q)` 素数列表。
- `count_by_residue_classes`：统计 `q` 的所有互素剩余类计数。
- `run_single_experiment`：单组参数打印“实测 vs 估计”。
- `main`：固定参数跑完整演示。

## R11

运行方式：

脚本无交互输入，默认使用 `N=200000`，演示 `q=4/10/12` 的多个剩余类。

## R12

输出解读：
- `count`：精确计数 `pi(N; q, a)`。
- `estimate`：仅在 `gcd(a,q)=1` 时给出 `li(N)/phi(q)`。
- `first_10`：该同余类前 10 个素数样例。
- 最后一段 `q=10` 分布展示互素剩余类 `{1,3,7,9}` 的计数是否接近。

## R13

建议最小测试集：
- 基本正确性：`q=4, a=1/3`，检查输出均为奇素数并满足同余。
- 归一化：`a=-1` 与 `a=q-1` 应得到同一结果。
- 非互素类：`q=12, a=6`，应仅可能出现极少特例，估计列为 `N/A`。
- 极小上界：`N=1`、`N=2` 不应报错。

## R14

可调参数：
- `limit`：搜索上界，决定精度与耗时。
- `q`：模数，影响每个类的候选密度。
- `residues`：要观察的剩余类。

实践建议：先用小 `N` 校验逻辑，再放大 `N` 观察分布趋于均衡的趋势。

## R15

- 相比逐数试除：筛法批量判素更高效。
- 相比“先生成素数列表再按模过滤”：本实现本质相同，但提供了统一 API 与估计对照。
- 相比高性能分段筛/并行筛：当前版本更偏教学与验证，结构清晰但非极限性能。

## R18

源码级算法流（对应 `demo.py`，8 步）：
1. `main` 设定 `limit=200000`，并调用 `sieve_eratosthenes(limit)` 生成全局素性表。  
2. `run_single_experiment` 读取一组 `(q, residues)`，先用 `euler_phi(q)` 和 `prime_count_estimate_ap` 计算理论参考值。  
3. 每个 `raw_a` 先经 `normalize_residue(raw_a, q)` 归一化到 `[0, q-1]`。  
4. `primes_in_arithmetic_progression` 计算该同余类首个 `>=2` 的项 `first`，随后按步长 `q` 枚举 `first, first+q, ...`。  
5. 枚举过程中仅通过 `is_prime[n]` 布尔位判断素性；命中则加入结果列表。  
6. 返回后由 `len(primes)` 得到精确计数，并输出 `first_10` 样例用于人工校验。  
7. 若 `gcd(a,q)=1`，打印 `li(limit)/phi(q)` 近似；否则标记为 `N/A`，避免误用解析近似。  
8. 全部实验后，`count_by_residue_classes(limit, 10, prime_table)` 汇总 `q=10` 的互素类分布，形成最终对照输出。
