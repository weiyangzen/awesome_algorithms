# 指数积分法 (Index Calculus)

- UID: `MATH-0021`
- 学科: `数学`
- 分类: `数论`
- 源序号: `21`
- 目标目录: `Algorithms/数学-数论-0021-指数积分法_(Index_Calculus)`

## R01

问题定义：在素数域乘法群 `Z_p^*` 中，给定 `g, h, p`，求离散对数 `x`，满足

`g^x ≡ h (mod p)`。

本条目实现的是教学型 Index Calculus（指数积分法）最小可运行版本。

## R02

输入输出约定（`demo.py`）：
- 输入：`p`（素数）、`g`（群元素）、`h`（目标元素）。
- 输出：整数 `x`，满足 `pow(g, x, p) == h`。
- 失败时抛异常（例如关系不足、目标平滑分解失败、验证失败）。

## R03

核心思想：先把“少量基元的离散对数”求出来，再拼出目标对数。

1. 选一个因子基（factor base）`B = {q1, q2, ..., qt}`（通常是小素数）。
2. 随机找指数 `k`，若 `g^k mod p` 能完全分解到因子基上（`B`-smooth），就得到关系：

`k ≡ e1*log_g(q1) + e2*log_g(q2) + ... + et*log_g(qt) (mod p-1)`。

3. 收集足够关系后，解线性方程组得到 `log_g(qi)`。
4. 再找 `h*g^t` 的平滑分解，回推出 `log_g(h)`。

## R04

因子基与平滑性定义（本 MVP）：
- 因子基：`<= smooth_bound` 的所有素数（且 `< p`）。
- `factor_over_base(value, factor_base)`：
  - 若 `value` 完全由因子基素数构成，返回指数向量；
  - 否则返回 `None`。

这一步决定关系收集是否高效，也是 Index Calculus 成败关键。

## R05

关系收集方式（`collect_relations`）：
- 随机采样 `k`，计算 `y = g^k mod p`。
- 如果 `y` 可在因子基上完全分解，则记录关系 `(k, exponent_vector)`。
- 用 `seen_vectors` 去重，避免重复关系拖慢秩增长。

本实现要求收集到 `factor_base_size + relation_surplus` 条关系作为起点，不够则报错。

## R06

线性方程求解策略：

严格来说方程在模 `n = p-1` 上，而 `n` 常为合数。为保持 MVP 简洁，代码采用：

1. 将 `n` 分解为互异素因子 `n = r1 * r2 * ... * rk`（要求平方自由）。
2. 对每个素数 `ri`，在 `GF(ri)` 上解一套线性系统。
3. 通过 `gauss_jordan_mod` 做模素数高斯-约旦消元，得到每个因子基对数在 `mod ri` 下的值。

## R07

CRT 合并：
- 对目标离散对数同样先得到 `x mod ri`。
- 用 `crt_pairwise` 把多个同余类合并为 `x mod (p-1)`。

因为本 MVP 限制 `p-1` 为平方自由，模数两两互素，CRT 合并路径简单且稳定。

## R08

本 MVP 的范围与限制：
- 覆盖：素数域 `Z_p^*` 的教学规模离散对数。
- 限制：`p-1` 必须平方自由（例如 `1018 = 2 * 509`）。
- 限制：不实现 prime-power 模数上的 lifting 版本，也不做大规模性能优化。
- 工具栈：Python 3 标准库（`dataclasses`、`random` 等），不依赖重型第三方库。

## R09

算法总流程（高层）：
1. 构造因子基。
2. 收集平滑关系。
3. 分别在 `mod r` 上解出因子基对数。
4. 找到 `h*g^t` 的平滑表示。
5. 计算 `x mod r`。
6. CRT 合并得 `x mod (p-1)`。
7. 用 `pow(g, x, p) == h` 做最终校验。

## R10

复杂度（概念层面）：
- 关系收集：取决于 `B`-smooth 概率，主导成本通常在这里。
- 线性代数：设因子基大小为 `t`，消元近似 `O(t^3)`（每个模素数一次）。
- 目标分解：需要额外尝试直到命中平滑表示。

对于教学参数（`p≈10^3`），该 MVP 运行很快；对密码学大参数则远不足够。

## R11

正确性要点：
- 每条关系都来自同余恒等式，线性化后合法。
- 在每个 `GF(r)` 上求解得到的是正确的模 `r` 约束。
- 目标方程 `log(h)+t ≡ Σ e_i log(q_i)` 与关系构造同源。
- CRT 合并后再做幂模验证，确保结果不是伪解。

## R12

`demo.py` 固定示例：
- `p = 1019`
- `g = 2`
- `secret_x = 321`
- `h = g^secret_x mod p`

脚本会恢复 `x` 并打印校验值，保证无交互、可复现。

## R14

测试与验证策略：
- 主路径：固定参数恢复已知 `secret_x`。
- 终验：统一执行 `pow(g, recovered_x, p) == h`。
- 失败路径：通过异常明确提示是“关系不足”“秩不足”还是“目标不平滑”。

## R15

常见失败原因：
- `smooth_bound` 太小，导致平滑关系太少。
- 关系矩阵秩不够，无法恢复全部因子基对数。
- `p-1` 含平方因子（超出本 MVP 能力范围）。
- 找不到 `h*g^t` 的平滑分解（可增大平滑界或尝试次数）。

## R16

与同类离散对数算法对比：
- BSGS：时间 `O(sqrt(n))`，但内存开销大，确定性较强。
- Pollard's Rho：低内存、随机化，期望平方根复杂度。
- Index Calculus：在适用群里有更优渐近趋势，但实现复杂、预处理成本高。

本条目强调“可解释的最小实现”，而非大规模破解性能。

## R17

工程化扩展方向：
- 支持 `p-1` 含素数幂的通用模方程求解。
- 更智能的因子基选择和关系筛选（提高秩增长效率）。
- 并行关系收集与稀疏线性代数优化。
- 引入磁盘缓存，复用关系矩阵做多目标对数查询。

## R18

`demo.py` 源码级流程拆解（8 步）：
1. `index_calculus_discrete_log` 校验参数，设置 `n = p-1`，并分解 `n`。
2. 通过 `prime_sieve` 生成因子基，准备关系数量下限。
3. `collect_relations` 随机采样 `k`，对 `g^k mod p` 做 `factor_over_base`，收集平滑关系。
4. 对 `n` 的每个素因子 `r`，构造矩阵并调用 `gauss_jordan_mod`，求出因子基对数在 `mod r` 下的解。
5. `find_target_smooth_multiple` 搜索 `t`，使 `h*g^t mod p` 可在因子基上分解。
6. 按 `x ≡ Σ e_i*log_g(q_i) - t (mod r)` 计算每个 `x mod r`。
7. `crt_pairwise` 合并所有同余得到 `x mod (p-1)`。
8. 用 `pow(g, x, p) == h` 做最终验算，`main` 打印结果与关键中间统计。
