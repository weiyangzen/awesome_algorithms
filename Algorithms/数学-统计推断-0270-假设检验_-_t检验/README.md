## R01

t 检验（t-test）是用于均值推断的经典假设检验方法，典型场景是总体方差未知、样本量中小。核心思想是把“样本均值偏离原假设均值的程度”标准化成 t 统计量，再用 t 分布计算 p 值。

常见任务：

- 单样本 t 检验：检验样本均值是否等于给定值 `mu0`；
- 双样本 t 检验：比较两个总体均值是否相等；
- 配对 t 检验：比较同一对象处理前后的均值差异。

## R02

历史与定位（简要）：

- William S. Gosset（笔名 Student）提出 t 分布与 t 检验；
- 相比 z 检验，t 检验不要求已知总体标准差；
- 在 A/B 测试、小样本实验、医学对照试验和工程质量抽检中仍被广泛使用。

## R03

本条目 MVP 聚焦两个最常用版本：

- 单样本双侧 t 检验（手写公式 + SciPy 对照）；
- Welch 双样本双侧 t 检验（不假设方差齐性）。

输入：数值样本数组。  
输出：t 统计量、自由度、p 值、置信区间、是否拒绝原假设。

## R04

单样本 t 检验定义：

- 原假设：`H0: mu = mu0`
- 备择假设（双侧）：`H1: mu != mu0`
- 统计量：`t = (x_bar - mu0) / (s / sqrt(n))`
- 自由度：`df = n - 1`

其中：

- `x_bar` 为样本均值；
- `s` 为样本标准差（`ddof=1`）；
- 在 `H0` 下，统计量服从 `t(df)`。

## R05

Welch 双样本 t 检验（推荐默认）：

- 原假设：`H0: mu1 = mu2`
- 统计量：`t = (x_bar1 - x_bar2) / sqrt(s1^2/n1 + s2^2/n2)`
- 自由度（Welch-Satterthwaite）：

`df = (v1 + v2)^2 / ( v1^2/(n1-1) + v2^2/(n2-1) )`，其中 `v1=s1^2/n1, v2=s2^2/n2`。

该检验不要求方差相等，比经典 pooled t 检验更稳健。

## R06

复杂度：

- 单样本 t 检验：`O(n)` 时间，`O(1)` 额外空间；
- Welch 双样本：`O(n1+n2)` 时间，`O(1)` 额外空间。

本算法主要是统计量聚合与分布函数查询，不涉及高阶迭代优化。

## R07

简化手算示例（单样本）：`x=[10,12,9,11,13]`，检验 `mu0=10`。

1. `x_bar=11`，`n=5`。  
2. 样本方差 `s^2=((0)^2+2^2+(-1)^2+1^2+2^2)/(5-1)=2.5`，`s≈1.5811`。  
3. 标准误 `SE=s/sqrt(5)≈0.7071`。  
4. `t=(11-10)/0.7071≈1.4142`，`df=4`。  
5. 双侧 p 值约为 `0.23`，通常在 `alpha=0.05` 下不拒绝 `H0`。

## R08

适用前提：

- 样本近似独立；
- 数据近似正态（小样本时更重要）；
- 单样本/配对场景里观测应来自同一机制；
- 双样本场景里两组独立抽样。

实践中，样本量稍大时 t 检验对轻度非正态较鲁棒，但对极端重尾和异常值敏感。

## R09

检验流程标准模板：

1. 指定 `H0/H1` 与显著性水平 `alpha`；
2. 计算 t 统计量与自由度；
3. 根据备择方向计算 p 值；
4. 比较 `p` 与 `alpha` 得出拒绝/不拒绝结论；
5. 同时报告效应量与置信区间，避免只报 p 值。

## R10

正确性检查（本 MVP 的可执行验证）：

- 手写公式结果与 `scipy.stats.ttest_1samp/ttest_ind(equal_var=False)` 对齐；
- t 值与 p 值差异应在数值误差范围内；
- 输出自由度、置信区间和拒绝决策，便于人工复核。

`demo.py` 会在运行时自动断言手写实现和 SciPy 结果一致。

## R11

数值稳定性要点：

- 方差接近 0 时应抛错，避免除零；
- 使用 `float64` 计算，减少累计误差；
- p 值通过 `stats.t.cdf/sf` 计算，避免手写积分；
- 显式检查 `n>=2`，否则 `df<=0` 不可检验。

## R12

解释输出时的常见误区：

- “不拒绝 H0”不等于“证明 H0 为真”；
- p 值不是“原假设为真的概率”；
- 显著不等于实际重要，应结合效应量（例如 Cohen's d）；
- 多重检验时需控制家族错误率（Bonferroni/FDR 等）。

## R13

与其他方法关系：

- z 检验：总体标准差已知或大样本近似场景；
- Mann-Whitney U：非参数替代（比较分布位置，非直接比较均值）；
- Bootstrap 置信区间：弱分布假设下的替代推断路径；
- Bayesian t-test：输出 Bayes 因子或后验概率。

## R14

常见失败模式与防护：

- 失败：把明显偏态/重尾数据直接做 t 检验。  
  防护：先做分布诊断，必要时用稳健或非参数方法。
- 失败：双样本方差不齐却用 pooled t。  
  防护：默认用 Welch。
- 失败：把离群点当正常波动。  
  防护：先做异常值审计并记录处理策略。
- 失败：只给 p 值不给区间和效应量。  
  防护：统一输出 `p + CI + effect size`。

## R15

本实现（`demo.py`）包含：

- `one_sample_t_test_manual`：单样本 t 检验手写实现；
- `welch_t_test_manual`：Welch 双样本手写实现；

## R16

工程实践建议：

- 默认 `alpha=0.05`，但应按业务代价调整（例如医学可更严格）；
- 对实验平台保留完整审计信息：样本量、缺失值处理、异常值策略；
- 若连续上线 AB 测试，注意序贯检验或 alpha-spending；
- 固定随机种子保证复现实验输出。

## R18

`demo.py` 源码级算法流程（7 步，非黑盒）：

1. `main` 固定随机种子，生成一组单样本数据和两组独立样本数据。  
2. `one_sample_t_test_manual` 先计算均值、样本标准差和标准误，再得到 `t` 与 `df=n-1`。  
3. `_p_value_from_t` 根据双侧规则调用 t 分布尾概率，得到 p 值；随后构造均值差的置信区间。  
4. `welch_t_test_manual` 分别计算两组方差贡献 `v1,v2`，据此得到 Welch t 统计量和 Satterthwaite 自由度。  
5. 同样通过 `_p_value_from_t` 计算 Welch 检验 p 值，并给出均值差置信区间。  
6. `compare_with_scipy` 分别调用 `scipy.stats.ttest_1samp` 与 `ttest_ind(equal_var=False)`，对 t 值和 p 值做误差断言。  
7. `main` 将统计量、p 值、自由度、效应量、置信区间、拒绝结论汇总成 DataFrame 输出，形成可复核结果。  

说明：MVP 里 SciPy 仅用于“结果校验”，核心统计量与决策流程均已在源码中手写实现。
