## R01

ECDH（Elliptic Curve Diffie-Hellman，椭圆曲线 Diffie-Hellman）是经典密钥交换协议：
- 双方仅交换公钥，不直接传输私钥；
- 各自用“自己的私钥 + 对方公钥”计算共享点；
- 共享点经过 KDF（本实现用 `SHA-256`）得到会话密钥。

本目录提供一个可运行、可审计的最小 MVP：
- 手写短 Weierstrass 曲线算术（点加、倍点、标量乘）；
- 使用 `secp256k1` 真实参数演示 ECDH；
- 包含自动化一致性与异常路径校验（无需交互输入）。

## R02

问题定义（单轮密钥交换）：
- 输入：
  - 椭圆曲线参数 `E(F_p): y^2 = x^3 + ax + b (mod p)`；
  - 基点 `G` 与阶 `n`；
  - Alice 私钥 `d_A`、Bob 私钥 `d_B`（满足 `1 <= d < n`）。
- 过程：
  - Alice 公钥 `Q_A = d_A * G`；
  - Bob 公钥 `Q_B = d_B * G`；
  - Alice 共享点 `S_A = d_A * Q_B`；
  - Bob 共享点 `S_B = d_B * Q_A`。
- 输出：
  - 共享点 `S`（理论上 `S_A = S_B = d_A d_B G`）；
  - 会话密钥 `K = SHA256(x(S))`。

## R03

核心数学关系：
1. 椭圆曲线群运算在有限域 `F_p` 上进行。  
2. 点加与倍点通过斜率公式计算，所有除法都转化为模逆。  
3. 标量乘 `kP` 采用 double-and-add（二进制展开）：
   - 若当前位为 1：`result = result + addend`；
   - 每轮：`addend = 2 * addend`。  
4. ECDH 正确性：
   - `S_A = d_A(d_B G)`；
   - `S_B = d_B(d_A G)`；
   - 由交换律得 `S_A = S_B`。  
5. 安全性依赖 ECDLP（椭圆曲线离散对数难题），不是依赖保密算法流程。

## R04

算法流程（高层）：
1. 加载曲线参数（本实现固定 `secp256k1`）。  
2. 校验基点在曲线上，并验证 `n*G = O`（无穷远点）。  
3. 用私钥分别计算公钥 `Q_A, Q_B`。  
4. 对输入公钥执行合法性检查（在曲线上、非无穷远点、通过子群检查）。  
5. 双方分别执行一次标量乘得到共享点 `S_A, S_B`。  
6. 比较 `S_A` 与 `S_B`，若不同立即失败。  
7. 对共享点 `x` 坐标做 `SHA-256`，得到会话密钥摘要。  
8. 输出操作统计与汇总指标，并执行阈值校验。

## R05

核心数据结构：
- `Curve`：封装曲线参数 `p,a,b,g,n,h`。  
- `Point`：`Tuple[int, int]` 或 `None`（`None` 表示无穷远点）。  
- `OpCounter`：记录运算成本：
  - `additions`：普通点加次数；
  - `doublings`：倍点次数；
  - `inversions`：模逆次数；
  - `total`：总计。
- `run_case` 返回字典统计，用于 `numpy` 汇总。

## R06

正确性与安全相关要点：
- 模逆使用扩展欧几里得算法，避免把“除法”当作普通整数除法。  
- 点加规则覆盖特殊情形：
  - `P + O = P`；
  - `P + (-P) = O`；
  - `P = Q` 时走倍点公式。  
- 公钥校验包含 `n*Q == O` 子群检查，避免明显无效点输入。  
- 演示中验证群不变量：
  - `n*G = O`；
  - `(n+1)*G = G`；
  - `G + (-G) = O`。  
- 双方共享点必须严格相等，否则直接抛异常终止。

## R07

复杂度分析（令标量位数为 `L`，这里约 256 位）：
- 单次标量乘（double-and-add）：
  - 约 `L` 次倍点；
  - 约 `L/2` 次点加（平均）；
  - 每次点运算包含一次模逆与若干模乘。  
- 时间复杂度：
  - 标量乘 `O(L)` 次椭圆曲线点运算；
  - 一次 ECDH（已知公钥）需要 1 次标量乘；
  - 完整握手（两边公钥生成 + 共享点）在演示里约 4 次标量乘。  
- 空间复杂度：
  - 常数级（若不保存历史轨迹），`O(1)`。

## R08

边界与异常处理：
- 私钥必须满足 `1 <= d < n`，否则 `ValueError`。  
- 公钥若为无穷远点、不在曲线上、或子群检查失败，直接拒绝。  
- 模逆输入为 0 时抛 `ZeroDivisionError`。  
- 共享点若落在无穷远点（理论异常/恶意输入），立即失败。  
- `demo.py` 内置“畸形公钥”负测试，确认拒绝逻辑生效。

## R09

MVP 取舍说明：
- 只依赖 `numpy` + Python 标准库（`hashlib/dataclasses`），保持工具栈最小化。  
- 不调用第三方密码库黑盒 ECDH API，全部关键步骤源代码可见。  
- 采用固定私钥样例，保证可复现输出。  
- 这是教学/审计级实现，不包含常数时间防侧信道、内存擦除、协议认证等生产防护。

## R10

`demo.py` 主要函数职责：
- `secp256k1`：返回曲线参数。  
- `mod_inv`：扩展欧几里得求模逆。  
- `is_on_curve` / `point_neg`：点合法性与取负。  
- `point_add`：统一实现点加与倍点。  
- `scalar_mult`：double-and-add 标量乘。  
- `validate_private_key` / `validate_public_key`：密钥输入检查。  
- `derive_public_key`：私钥到公钥。  
- `ecdh_shared_point`：计算共享点。  
- `kdf_from_x_sha256`：共享点 `x` 坐标做 KDF。  
- `run_case`：执行单案例并打印统计。  
- `check_group_invariants`：群性质自检。  
- `main`：执行全部案例与汇总断言。

## R12

输出字段说明：
- `Curve`：当前曲线名称。  
- `Group invariants: PASS`：群不变量检查通过。  
- 每个 `Case`：
  - `Alice/Bob pubkey x`：公钥 `x` 坐标（缩写）；
  - `Shared x`：共享点 `x` 坐标（缩写）；
  - `KDF(SHA-256)`：会话密钥摘要（十六进制）；
  - `Ops`：公钥生成与共享点计算的运算次数。  
- `Summary`：
  - 平均操作数；
  - 共享点位长范围；
  - 确定性指纹；
  - `All ECDH checks passed.` 成功标识。

## R13

最小测试集（内置在 `main`）：
1. 群不变量测试：
- `n*G == O`
- `(n+1)*G == G`
- `G + (-G) == O`

2. 两组固定私钥案例：
- 验证 `d_A * Q_B == d_B * Q_A`；
- 验证 KDF 可稳定生成摘要；
- 统计操作成本。

3. 负向测试：
- 构造篡改后的公钥 `Q'`（通常不在曲线上）；
- 期望 `validate_public_key` 拒绝。

## R14

关键参数与工程注意：
- 曲线参数：`p,a,b,G,n,h` 决定安全级别与互通性。  
- 私钥范围：必须严格在 `[1, n-1]`。  
- KDF 输入：本实现使用共享点 `x` 坐标并固定字节长度。  
- 真实协议应补充：
  - 身份认证（防中间人攻击）；
  - 会话上下文绑定（如 transcript）；
  - 密钥确认与轮换策略。

## R15

方法对比：
- 对比经典 DH（模幂）：
  - ECDH 在同等安全级别下密钥更短、通信更省。  
- 对比 X25519：
  - X25519 在实现安全性与工程易用性上通常更友好；
  - 本实现选择 short-Weierstrass 主要用于展示通用 ECC 算法细节。  
- 对比黑盒库调用：
  - 黑盒更安全、成熟；
  - 手写实现更适合教学、审计与理解算法源流程。

## R18

`demo.py` 源码级算法流程（8 步）：
1. `main` 调用 `secp256k1()` 加载曲线，并先执行 `check_group_invariants`。  
2. `run_case` 为 Alice/Bob 各创建 `OpCounter`，并通过 `derive_public_key` 计算公钥。  
3. `derive_public_key` 内部调用 `scalar_mult(d, G)`；`scalar_mult` 使用二进制 double-and-add。  
4. `scalar_mult` 每一位通过 `point_add` 完成“按位加”与“倍点”，`point_add` 用 `mod_inv` 处理有限域除法。  
5. `run_case` 调用 `ecdh_shared_point(d_A, Q_B)` 与 `ecdh_shared_point(d_B, Q_A)`；该函数先 `validate_public_key` 再做标量乘。  
6. 若两个共享点不一致则失败；一致时把共享点 `x` 坐标传给 `kdf_from_x_sha256` 得到会话密钥摘要。  
7. `run_case` 额外构造一个篡改公钥触发负向测试，验证非法输入会被拒绝。  
8. `main` 用 `numpy` 汇总全部案例操作统计与位长区间，执行阈值断言后打印 `All ECDH checks passed.`。
