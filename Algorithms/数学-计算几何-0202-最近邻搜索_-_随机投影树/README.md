# 最近邻搜索 - 随机投影树

- UID: `MATH-0202`
- 学科: `数学`
- 分类: `计算几何`
- 源序号: `202`
- 目标目录: `Algorithms/数学-计算几何-0202-最近邻搜索_-_随机投影树`

## R01

问题定义：给定点集 `X={x_i}` 与查询点 `q`，希望快速返回最近邻 `argmin_i ||x_i-q||_2`。随机投影树（Random Projection Tree, RP Tree）通过“随机超平面递归划分 + 有界回溯”实现近似最近邻搜索，在中高维下常比暴力法更快。

## R02

核心思想：
1. 建树时随机采样一个方向向量 `w`。
2. 用投影值 `x·w` 的中位数切分数据，生成左右子集。
3. 递归直到叶子足够小。
4. 查询时优先走“与查询点同侧”的分支，同时把“异侧分支”按下界距离加入优先队列，做有限次回溯。

## R03

数据结构（MVP）包含：
- `RPTreeNode.indices`: 叶节点持有样本索引。
- `RPTreeNode.normal`: 当前分割超平面的法向量 `w`。
- `RPTreeNode.threshold`: 分割阈值（投影中位数）。
- `left/right`: 子节点引用。
- `RandomProjectionTree`: 持有数据矩阵、随机数发生器、树根与参数（`leaf_size`, `max_depth`）。

## R04

建树流程：
- 输入索引集合 `I`。
- 若 `|I| <= leaf_size` 或深度达到 `max_depth`，停止并构造叶子。
- 否则随机生成单位向量 `w`，计算 `P = X[I] @ w`。
- 取 `median(P)` 作为阈值，将 `I` 分为左右两部分。
- 若切分退化（某一侧为空），按排序后中点强制二分，仍失败则退化为叶子。

## R05

查询流程（Best-Bin-First 近似版）：
- 使用最小堆维护待访问节点，键值是该节点的下界距离平方 `lb`。
- 初始将根节点以 `lb=0` 入堆。
- 每次弹出 `lb` 最小节点：
- 若是叶子，线性扫描叶子内点并更新当前最优。
- 若是内部节点，先压入“同侧子树”（`lb` 不变），再压入“异侧子树”（`lb=max(lb, margin^2)`）。
- 达到 `max_leaves` 或堆空时停止，返回当前最优。

## R06

正确性直觉：
- 精确最近邻一定在某个叶子中。
- “同侧优先 + 异侧按下界排序”可以优先探索更可能含有近邻的区域。
- 当回溯预算足够大时，结果逼近精确搜索；预算有限时得到可控精度-速度折中。

## R07

时间复杂度（平均视角）：
- 建树：每层做一次投影，若树较平衡，约 `O(n log n)`。
- 单次查询：`O(log n + m * leaf_size)`，其中 `m` 是实际访问叶子数（受 `max_leaves` 控制）。
- 最坏情况（高维退化或预算极大）可接近 `O(n)`。

## R08

空间复杂度：
- 数据矩阵 `O(n*d)`。
- 树结构与索引总体 `O(n)` 级别。
- 查询堆空间约 `O(number_of_visited_nodes)`。

## R09

适用场景：
- 中高维向量检索（embedding、特征向量检索）。
- 允许“近似最近邻”而非严格精确解。
- 数据静态或低频更新（本 MVP 偏离线建树）。

## R10

局限与退化点：
- 维度极高且数据分布接近各向同性时，随机切分区分度会下降。
- 如果 `leaf_size` 过大，叶内线扫成本上升。
- 如果 `max_leaves` 太小，召回率下降。
- 本实现未做动态插入/删除，在线更新能力弱。

## R11

参数建议：
- `leaf_size`: 建议 `16~64`。小叶子更精确但树更深。
- `max_depth`: 可设为 `ceil(log2(n))` 到 `2*ceil(log2(n))`。
- `max_leaves`（查询预算）: 常见取 `8~64`；预算越大，召回越高、耗时越大。
- `random_state`: 固定后可复现实验结果。

## R12

与 KD-Tree 对比（简要）：
- KD-Tree按坐标轴切分，RP Tree按随机方向切分。
- 在高维下，轴对齐切分常更快退化；随机方向有时更稳健。
- KD-Tree更适合低维精确检索，RP Tree更偏中高维近似检索。

## R13

本目录 `demo.py` 的 MVP 特点：
- 仅依赖 `numpy` + Python 标准库。
- 从零实现建树与查询，不依赖黑盒 ANN 包。
- 自动生成数据集，比较“RP Tree 近似查询”与“暴力精确查询”。
- 输出构建时间、查询时间、召回率与示例结果。

## R14

运行方式：

```bash
python3 demo.py
```

输出解读：
- `build_time`: 建树耗时。
- `rp_query_time`: 随机投影树查询总耗时。
- `brute_query_time`: 暴力法查询总耗时。
- `recall@1`: RP Tree 返回的最近邻索引与精确解一致的比例。
- `speedup`: 暴力法耗时 / RP 查询耗时。

## R15

健壮性处理：
- 方向向量范数过小时直接退化成叶子，避免除零。
- 中位数切分导致空子集时，采用排序中点强制二分。
- 查询堆中使用单调计数器作为次关键字，避免节点对象比较异常。
- 所有距离计算使用平方距离，避免不必要开方。

## R16

可扩展方向：
- 多棵 RP Tree 组成森林并做候选集合合并（常见于 ANN 系统）。
- 叶内使用 SIMD/BLAS 加速批量距离计算。
- 增加批量查询接口与并行查询。
- 与产品量化（PQ）或图索引（HNSW）做两阶段检索。

## R17

最小测试清单：
- 功能测试：随机小数据上与暴力解逐点比对。
- 稳定性：不同随机种子下召回率分布。
- 参数敏感性：`leaf_size`、`max_leaves` 对速度与召回影响曲线。
- 边界测试：`n < leaf_size`、重复点、退化切分场景。

## R18

`demo.py` 的源码级算法流（8 步）：
1. `fit` 接收矩阵 `X`，创建 `0..n-1` 索引并调用 `_build`。
2. `_build` 在每个节点采样高斯随机向量并归一化成分割法向量 `w`。
3. 计算节点内样本的投影 `X[I]@w`，取中位数 `threshold` 形成左右分裂。
4. 若分裂退化，按投影排序并在中点强制二分，确保树继续生长。
5. `query` 用最小堆执行 best-bin-first：根节点以下界 `0` 入堆。
6. 弹出节点后，叶子节点做线性扫描更新最优 `(best_idx, best_dist2)`。
7. 内部节点根据 `margin=q·w-threshold` 判定同侧/异侧；同侧以原下界入堆，异侧以下界 `max(lb, margin^2)` 入堆。
8. 达到叶访问预算后停止，返回近似最近邻；主程序再与暴力法对比得到 `recall@1` 与加速比。
