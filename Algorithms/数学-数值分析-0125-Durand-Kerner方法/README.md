## R01

Durand-Kerner 方法（也称 Weierstrass 方法）用于**同时**求解一元 `n` 次多项式的全部复根。  
给定多项式 `P(z)` 与当前根近似 `z_1,...,z_n`，其同步迭代形式为：

- `z_k^{(m+1)} = z_k^{(m)} - P(z_k^{(m)}) / Π_{j≠k}(z_k^{(m)} - z_j^{(m)})`

核心特点：

- 一次迭代更新所有根估计，而不是“找一个根再降阶”。
- 在简单根条件下具有良好局部收敛性，且实现相对直接。
- 算法天然工作在复数域，对实根与复根统一处理。

## R02

- 该方法可追溯到 Weierstrass 的相关思想，后由 Durand 与 Kerner 推广为常见工程迭代格式。
- 在“多项式全根求解”场景里，它是经典教学与工程基线算法之一。
- 相比更复杂的工业级方案（如 Jenkins-Traub 等），Durand-Kerner 的优势是实现短小、便于审计与验证。

## R03

为什么这个算法重要：

- 控制系统稳定性分析常需多项式极点（根）的位置。
- 信号处理、振动分析、数值线性代数里，特征问题常可化为多项式求根。
- 需要全部根时，逐根牛顿法 + 降阶对初值敏感；Durand-Kerner 直接并行更新全部根，思路更统一。

## R04

设多项式次数为 `n`，迭代轮数为 `T`。

- 每轮需对 `n` 个点求 `P(z_k)`（每个 `O(n)`），共约 `O(n^2)`。
- 每轮还需计算 `Π_{j≠k}(z_k-z_j)`，直接实现也是 `O(n^2)`。
- 故总时间复杂度通常写为 `O(T * n^2)`。

在中小规模（例如 `n <= 100`）时，该复杂度通常可接受。

## R05

空间复杂度：

- 需要保存 `n` 个当前根、`n` 个函数值、`n` 个分母项，外加系数数组。
- 额外空间为 `O(n)`（不含输入多项式系数本身）。

## R06

示例多项式：

`P(x) = (x-1)(x+2)(x^2+1) = x^4 + x^3 - x^2 + x - 2`

真实根：

- `1`
- `-2`
- `i`
- `-i`

Durand-Kerner 会从一组复平面初值（通常放在圆周上）出发，经过若干轮同步更新，收敛到这 4 个根。

## R07

常见应用场景：

- 一般复系数/实系数多项式全根求解。
- 控制系统极点计算与稳定域判断。
- 作为教学示例，用于说明“全根并行迭代”的思路。
- 在需要可读、可审计、轻依赖的求根组件时作为基线实现。

## R08

前置知识：

- 多项式表示与 Horner 求值。
- 复数运算（模、辐角、共轭）。
- 迭代法终止判据（步长阈值、残差阈值）。
- 基础数值误差意识（舍入误差、近重根敏感性）。

## R09

适用前提与边界：

- 输入是一元多项式系数，且最高次项系数非零。
- 算法对简单根（非重根）表现更稳定；重根或近重根会显著拖慢收敛。
- 若某些迭代点过于接近，会导致分母 `Π_{j≠k}(z_k-z_j)` 过小，需做扰动/重启。
- 实系数多项式的复根通常成共轭对出现，但算法内部统一按复数处理。

## R10

正确性直觉（工程视角）：

1. 若当前 `z_k` 已靠近真实根 `r_k`，则 `P(z_k)` 会较小。
2. 分母项 `Π_{j≠k}(z_k-z_j)` 对应“去除其他根影响后的局部尺度”。
3. 迭代量 `P(z_k)/Π_{j≠k}(z_k-z_j)` 可以视为针对 `r_k` 的校正量。
4. 所有 `k` 同步更新后，根集合整体逐步逼近真实根集合。
5. 当步长与残差同时足够小，可认为已达到可用精度。

## R11

数值稳定性要点：

- 风险 1：分母接近 0（根估计彼此过近）会放大更新步长。
- 风险 2：系数尺度过大或过小会导致 `P(z)` 求值时舍入误差放大。
- 风险 3：近重根时收敛速度下降，迭代可能出现长平台。

工程缓解策略：

- 对多项式做首项归一化（monic 化）。
- 初值放在 Cauchy 半径附近并加小扰动，避免完全对称。
- 同时使用步长阈值与残差阈值，避免“步长小但残差仍大”的假收敛。

## R12

真实开销（超越渐近记号）：

- 热点计算是复数乘加与 `numpy.polyval`。
- Python + NumPy 对教学规模和中小规模多项式通常足够。
- 当次数很高或要求超高精度时，更常见做法是转向专门库或高精度算法。

本条目 MVP 重点是：实现透明、结果可复现、运行门槛低。

## R13

收敛与误差观察：

- 对简单根，Durand-Kerner 常表现为稳定收敛。
- 常用终止条件：
  - `max_k |Δz_k| < tol_step`
  - `max_k |P(z_k)| < tol_residual`
- 最终质量可用：
  - 最大残差 `max |P(r_k)|`
  - 与 `numpy.roots` 的最大匹配误差（用于对照，不作为唯一真值）。

## R14

鲁棒性与失效模式：

- 失效模式 A：分母过小导致步长爆炸。
  - 处理：对冲突点施加微扰，再继续迭代。
- 失效模式 B：到达最大迭代数仍未满足阈值。
  - 处理：提高迭代上限、调整初值半径或容差。
- 失效模式 C：近重根导致慢收敛。
  - 处理：放宽容差并增加最大迭代预算，必要时改用更强方法。

## R15

工程实现建议：

- 保持“同步更新”语义：每轮都基于旧根数组计算全部分母，再统一更新。
- 系数先 monic 化，降低尺度不平衡影响。
- 输出诊断信息：迭代次数、最大步长、最大残差、与 `numpy.roots` 对比误差。
- 采用确定性随机种子或固定初值策略，保证复现实验结果。

## R18

源码级流程（对应本目录 `demo.py`，非黑箱）可拆为 8 步：

1. 读取并检查系数数组，去除前导零并执行首项归一化（monic）。
2. 用 Cauchy 半径 `R=1+max|a_i|` 在复平面圆周上生成 `n` 个初始猜测点，并加入轻微相位偏移。
3. 进入迭代：在当前根数组上计算每个 `P(z_k)`。
4. 计算每个分母 `Π_{j≠k}(z_k-z_j)`；若某项过小，执行确定性微扰以避免除零和数值爆炸。
5. 得到校正量 `Δz_k = P(z_k) / Π_{j≠k}(z_k-z_j)`，按同步方式更新 `z_k <- z_k - Δz_k`。
6. 在新根上评估 `max_step=max|Δz_k|` 与 `max_residual=max|P(z_k)|`，用于停止判据。
7. 若满足阈值则判定收敛；否则继续，直到 `max_iter`。
8. 收敛后将根排序并输出，同时与 `numpy.roots` 做匹配误差对照，形成可审计结果。
