# 强连通分量 - Tarjan

- UID: `MATH-0476`

- 分类: `图论`

## R01

Tarjan 算法用于在线性时间内求解有向图的所有强连通分量（Strongly Connected Components, SCC）。
强连通分量是指：在该点集内部，任意两点都可互相到达。算法输出是一个分组，每组节点构成一个 SCC。

## R02

问题定义（面向本任务 MVP）：
给定 `n` 个点、`m` 条有向边，求所有强连通分量。

- 输入: `n`（节点数，编号 `0..n-1`），以及边列表 `edges=[(u,v), ...]`
- 输出:
1. `sccs`: `List[List[int]]`，每个子列表是一个 SCC 的节点集合
2. `comp_id`: `List[int]`，`comp_id[u]` 表示节点 `u` 所属 SCC 编号

## R03

Tarjan 的核心思想是一次 DFS 同时维护两个时间戳量：

- `dfn[u]`: 节点 `u` 在 DFS 中首次被访问的时间戳
- `low[u]`: 在“只允许经过 DFS 子树边 + 至多一条指向栈内祖先的返祖边”的条件下，`u` 能回溯到的最小 `dfn`

当 `low[u] == dfn[u]` 时，`u` 是一个 SCC 根；从栈顶弹出直到 `u` 的全部节点正好构成该 SCC。

## R04

为什么是线性复杂度：

- 每个节点恰好入栈一次、出栈一次
- 每条边在 DFS 中最多被检查一次
- 所有 `min` 更新都是常数操作

因此总时间复杂度是 `O(n + m)`，适合大规模稀疏图。

## R05

MVP 使用的数据结构：

- 邻接表 `adj: List[List[int]]`
- `dfn, low: List[int]`
- `on_stack: List[bool]`
- 显式栈 `stack: List[int]`
- 结果容器 `sccs` 和 `comp_id`

不依赖第三方图算法库，保证实现可读和可追踪。

## R06

算法流程概览：

1. 初始化所有节点 `dfn=-1`（未访问）
2. 依次对未访问节点发起 DFS
3. 访问节点时设置 `dfn=low=timestamp` 并压栈
4. 遍历出边 `(u,v)`:
1. 若 `v` 未访问，递归后 `low[u]=min(low[u], low[v])`
2. 若 `v` 在栈中，`low[u]=min(low[u], dfn[v])`
5. 若 `low[u]==dfn[u]`，从栈顶持续弹出直到 `u`，形成一个 SCC

## R07

关键不变量：

- 栈中节点尚未归属任何 SCC
- 若节点仍在栈中，则它属于当前 DFS 路径相关的“未闭合”子问题
- `low[u]` 始终不大于 `dfn[u]`
- 一旦某 SCC 被弹栈确定，其内部节点不会再次入栈或被重新分配

## R08

正确性直觉：

- `low[u]==dfn[u]` 说明 `u` 的子树无法回到 `u` 之前的更早祖先
- 栈顶到 `u` 这一段节点相互可达，且不能与栈外节点共同形成更大 SCC
- 提前弹出会破坏互达性，延后弹出会混入不该同组的节点

因此该弹栈边界恰好对应一个强连通分量。

## R09

时间复杂度：`O(n + m)`。

- 节点处理: `O(n)`
- 边扫描: `O(m)`

## R10

空间复杂度：`O(n + m)`。

- 邻接表占 `O(n + m)`
- 栈与辅助数组占 `O(n)`

## R11

边界情况处理：

- 空图 `n=0`: 返回空 SCC 列表
- 单节点无边: 该节点自身构成 SCC
- 自环 `(u,u)`: 仍是合法 SCC（通常单点 SCC）
- 重边: 不影响正确性，仅增加常数级扫描开销
- 非连通图: 外层循环会覆盖所有连通块

## R12

实现中的常见错误：

- 把“访问过”误写成“已出栈”，导致 `low` 更新错误
- 忘记 `on_stack` 判断，错误地使用所有回边更新 `low`
- 弹栈时未到根就停止，造成 SCC 被拆碎
- 递归返回后忘记 `low[u]=min(low[u], low[v])`

## R13

与常见替代算法对比：

- Kosaraju: 两次 DFS + 反图，思路直观但需要反图与两遍遍历
- Tarjan: 一次 DFS 完成，空间/实现都较紧凑
- Gabow: 也是单次 DFS，使用双栈策略

在教学与工程中，Tarjan 常作为 SCC 的标准线性解法。

## R18

`demo.py` 中 Tarjan 源码级流程（8 步）：

1. 由边列表构建邻接表 `adj`，并校验端点范围。
2. 初始化 `dfn/low/on_stack/stack/index/sccs/comp_id`。
3. 对每个未访问节点 `u` 调用 `dfs(u)`。
4. 在 `dfs(u)` 入口处设置 `dfn[u]=low[u]=index`，随后 `index += 1`，并将 `u` 入栈。
5. 遍历 `u` 的每条出边到 `v`：
1. `v` 未访问: 递归 `dfs(v)`，回溯后用 `low[v]` 更新 `low[u]`。
2. `v` 在栈内: 说明找到可回溯路径，用 `dfn[v]` 更新 `low[u]`。
6. 出边处理结束后，若 `low[u] != dfn[u]`，直接返回上层继续回溯。
7. 若 `low[u] == dfn[u]`，持续从栈顶弹出节点，直到弹出 `u`；弹出的整段即一个 SCC，并写入 `comp_id`。
8. 全部 DFS 完成后，返回 `sccs` 与 `comp_id`；示例中再做编号规范化和缩点 DAG 构建用于展示。
