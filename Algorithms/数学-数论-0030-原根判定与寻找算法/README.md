# 原根判定与寻找算法

## R01

原根判定与寻找算法解决两个核心问题：

1. 给定正整数 `n`，判断模 `n` 的乘法群（严格说是单位群 `(Z/nZ)^*`）是否是循环群，即是否存在原根。
2. 若存在原根，构造一个具体的生成元 `g`，使得 `g` 的幂能够生成全部与 `n` 互素的剩余类。

本目录 `demo.py` 给出一个可运行 MVP：
- 使用经典判定定理先判断“是否存在原根”；
- 再利用 `phi(n)` 的质因子做生成元检验并搜索最小可行 `g`。

## R02

历史脉络（简要）：

- 欧拉与高斯时期逐步建立了模运算与原根理论。
- 19 世纪后，关于“哪些 `n` 存在原根”得到完整刻画。
- 现代密码学与计算数论将该理论转化为可执行算法，广泛用于离散对数、指数群构造与教学验证。

## R03

问题动机：

- 在离散对数相关系统中，需要可控的循环群与生成元。
- 在构造测试样例、教学演示时，常需快速回答“这个模数有没有原根”。
- 与其枚举全部单位元并逐个求阶，不如先用存在性定理剪枝，再进行针对性搜索，效率更高。

## R04

时间复杂度（以本 MVP 的试除分解为准）：

- `factorize(n)`：最坏 `O(sqrt(n))`。
- `phi(n)` 计算：受分解主导，约 `O(sqrt(n))`。
- 原根判定 `has_primitive_root(n)`：一次或少量分解，约 `O(sqrt(n))`。
- 寻找原根 `find_primitive_root(n)`：
  - 设 `m = phi(n)`，`m` 的不同质因子个数为 `t`；
  - 每个候选 `g` 需要做 `t` 次模幂判定；
  - 模幂复杂度约 `O(log m)` 次乘模；
  - 总体可写为 `O(search_space * t * log m)`（忽略大整数乘法常数）。

## R05

空间复杂度：

- 因子表、临时变量与少量集合为主；
- 额外空间约 `O(k)`，`k` 为参与计算的质因子数量和展示用序列长度；
- 属于低内存占用的算法。

## R06

手算示例：`n = 17`。

1. `phi(17) = 16 = 2^4`，其质因子集合是 `{2}`。
2. 设候选 `g = 3`，检查 `3^(16/2) = 3^8 (mod 17)`。
3. `3^8 mod 17 = 16 != 1`，因此 `g=3` 满足原根判据。
4. `3^k mod 17 (k=0..15)` 恰好覆盖 `1..16` 的全部非零剩余类。

因此，`3` 是模 `17` 的一个原根。

## R08

所需前置知识：

- 欧拉函数 `phi(n)`。
- 模幂计算 `pow(a, e, n)`。
- 乘法群阶与元素阶的关系。
- 质因数分解（本实现用试除法）。

## R09

适用前提与边界：

- 输入为正整数 `n`。
- 原根存在当且仅当：
  - `n = 2`，或
  - `n = 4`，或
  - `n = p^k`（`p` 为奇素数），或
  - `n = 2*p^k`（`p` 为奇素数）。
- 对于不满足上式的 `n`，应直接返回“不存在原根”，而不是强行搜索。

## R10

正确性依据（核心）：

1. 存在性判定定理给出必要且充分条件（见 R09）。
2. 若原根存在，则单位群 `(Z/nZ)^*` 为循环群，群阶为 `phi(n)`。
3. 在有限群中，元素 `g` 的阶为 `phi(n)` 当且仅当对所有 `q | phi(n)`（`q` 为质因子），都有：
   `g^(phi(n)/q) != 1 (mod n)`。
4. `find_primitive_root` 正是按上述充要条件逐候选检查，因此找到即正确；不存在时由定理保证应先被过滤。

## R11

数值稳定性：

- 全部为整数运算，无浮点误差传播。
- Python 内置大整数是精确算术，结果在数值上稳定。
- 风险主要来自算法复杂度（慢），不是数值不稳定。

## R12

真实成本与性能瓶颈：

- 试除分解在大 `n` 时会成为主瓶颈。
- 搜索原根时，对多个候选执行多次模幂，CPU 时间显著增长。
- 若扩展到很大整数，应把分解器替换为更快方法（如 Pollard Rho 等）。

## R13

与概率算法的关系：

- 本实现是确定性流程，不含随机化概率误判。
- 只要分解与模幂实现正确，判定和搜索结论是确定的。
- 代价是速度依赖分解效率，尤其在大整数场景下明显。

## R14

常见失效模式：

- 忘记先做“存在性判定”，导致在无解模数上长时间无效搜索。
- 仅验证 `g^phi(n) = 1` 就误判为原根（这是必要不充分条件）。
- 没有对 `gcd(g, n) = 1` 做过滤，导致无效候选进入检验。

## R15

工程实现要点：

- 先剪枝：`n` 形态不合法则直接返回。
- 复用 `phi(n)` 与其质因子，避免重复计算。
- 利用 Python 三参数 `pow(base, exp, mod)`，高效且稳健。
- 演示中加入自动验证：检查 `g^k` 生成集合是否等于简化剩余系。

## R16

- 后续算法：离散对数（BSGS、Pollard Rho DLP）、原根相关指数方程。
- 典型应用：密码学教学、数论实验平台、竞赛题原型验证。

## R18

源码级算法流程（对应 `demo.py`）可拆成 8 步：

1. 输入 `n` 后先调用 `has_primitive_root(n)`，按定理判定是否有解。
2. 若无解，直接返回失败或跳过搜索分支。
3. 若有解，计算 `phi(n)`，并分解 `phi(n)` 得到其质因子集合 `Q`。
4. 从小到大枚举候选 `g`（并先过滤 `gcd(g, n) != 1`）。
5. 对每个 `q in Q` 计算 `pow(g, phi(n)//q, n)`。
6. 只要某个结果为 `1`，则 `g` 阶不满 `phi(n)`，该候选淘汰。
7. 若对所有 `q` 都不为 `1`，则该 `g` 阶为 `phi(n)`，确认为原根并返回。
8. 演示阶段再用 `g^0..g^(phi(n)-1)` 的像集与简化剩余系对比，做结果自验证。

本实现未调用黑箱第三方数论包，流程可在源码中逐行追踪。
