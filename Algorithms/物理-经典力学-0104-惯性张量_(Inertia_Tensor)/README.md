# 惯性张量 (Inertia Tensor)

## R01

惯性张量是刚体转动惯量在三维空间中的矩阵表达，它把角速度 `ω` 与角动量 `L` 关联为：

`L = I ω`

与单轴转动的标量转动惯量不同，惯性张量能同时描述任意轴旋转下的耦合关系，是刚体动力学、姿态控制、碰撞响应中的基础对象。

## R02

对离散质点系（`m_i, r_i`）相对于某一参考原点 `O`，惯性张量定义为：

`I = Σ m_i (||r_i||^2 E - r_i r_i^T)`

其中 `r_i` 是第 `i` 个质点相对 `O` 的位置向量，`E` 为 `3x3` 单位阵。该定义天然给出对称矩阵，且在物理上应为半正定。

## R03

本条目 MVP 采用“均匀长方体的离散点云近似”来演示完整流程：

- 在盒体内部构造规则网格点并赋予均匀质量。
- 在世界坐标原点与质心坐标分别计算惯性张量。
- 验证平行轴定理、坐标旋转协变性、主惯量分解。
- 与均匀长方体解析解对角元做误差对比。

该设置小而完整，能覆盖从定义到验证的核心链路。

## R04

平行轴定理（矩阵形式）在本实现中使用为：

`I_O = I_C + M (||d||^2 E - d d^T)`

- `I_C`：质心处惯性张量。
- `I_O`：相对新原点 `O` 的惯性张量。
- `M`：总质量。
- `d`：从 `O` 指向质心 `C` 的位移向量。

`demo.py` 同时做正向与反向检查（由 `I_C` 推 `I_O`，由 `I_O` 反推 `I_C`）。

## R05

坐标变换性质：若坐标系做正交旋转 `R`，同一物理刚体的惯性张量满足

`I' = R I R^T`

这不是“近似关系”而是张量协变性。MVP 里将点云直接旋转后重算 `I'`，并与 `R I R^T` 对比，作为实现正确性的关键检查之一。

## R06

主惯量与主轴通过特征分解获得：

`I v_k = λ_k v_k`

对实对称矩阵 `I`，`λ_k` 实数、`v_k` 可取正交归一基。主轴坐标下惯性张量对角化，`diag(λ1, λ2, λ3)` 即主惯量。`demo.py` 使用 `numpy.linalg.eigh` 进行该分解，并打印主轴方向。

## R07

本目录 `demo.py` 的输入输出约定：

- 输入：代码内置参数（盒体尺寸、网格分辨率、总质量、平移量、旋转角、验证阈值）。
- 输出：
1. `I_origin` 与 `I_com_direct` 矩阵。
2. 主惯量/主轴表。
3. 解析解与离散近似对角元误差表。
4. 多项验证指标与 `Validation: PASS/FAIL`。

脚本无需交互输入，直接运行。

## R08

算法主流程（高层）：

1. 构建均匀长方体离散点云与质量数组。
2. 计算质心。
3. 依据定义式在世界原点、质心两处计算惯性张量。
4. 用平行轴定理进行双向一致性校验。
5. 旋转点云后重算惯性张量，并与 `R I R^T` 对比。
6. 对质心惯性张量做特征分解，得到主惯量与主轴。
7. 与长方体解析对角惯量比较误差。
8. 汇总指标并给出 PASS/FAIL。

## R09

复杂度（`N` 为离散质点数）：

- 质心计算：`O(N)`。
- 惯性张量计算：`O(N)`（每个点常数次向量/矩阵运算）。
- 旋转校验额外一次张量重算：`O(N)`。
- `3x3` 特征分解：`O(1)`（常数规模）。

总复杂度由点云规模主导，整体为 `O(N)`。

## R10

数值稳定性与误差控制：

- 张量计算后做 `(I + I^T)/2` 对称化，抑制浮点非对称噪声。
- 使用多个彼此独立的物理不变量/关系做交叉验证：
  - 对称性
  - 平行轴一致性
  - 旋转协变性
  - 主惯量非负
- 解析对角元比较采用相对误差，避免绝对尺度依赖。

## R13

关键输出字段说明：

- `principal_moment`：主惯量（特征值）。
- `axis_x/axis_y/axis_z`：对应主轴在当前坐标系中的方向余弦。
- `I_discrete`：离散点云计算得到的质心惯量对角元。
- `I_analytic`：均匀长方体理论对角元。
- `relative_error`：二者相对误差。
- `parallel_axis_error_*`：平行轴定理数值一致性误差。
- `rotation_covariance_error`：旋转协变性误差。

## R14

正确性验证（demo 内置）要求同时满足：

1. 原点与质心惯性张量对称性误差小于阈值。
2. 平行轴定理正向/反向误差小于阈值。
3. 旋转协变性误差小于阈值。
4. 与解析长方体惯量对角元最大相对误差小于阈值。
5. 最小主惯量为正。

全部满足时输出 `Validation: PASS`，否则抛出异常。

## R15

## R18

`demo.py` 源码级流程拆解（8 步）：

1. `build_uniform_box_cloud` 生成轴对齐、质心在局部原点的均匀点云，并分配等质量。  
2. `main` 将点云平移到世界坐标，调用 `center_of_mass` 得到真实质心位置。  
3. `inertia_tensor` 依据 `Σ m_i (||r_i||^2E - r_i r_i^T)` 分别计算 `I_origin` 与 `I_com_direct`。  
4. `parallel_axis_term` 构造 `M(||d||^2E - dd^T)`，在 `main` 中完成平行轴定理正向和反向一致性检查。  
5. `rotation_matrix_xyz` 生成正交旋转矩阵，点云旋转后再次调用 `inertia_tensor` 得到 `I_origin_rot_direct`。  
6. `main` 计算 `I_origin_rot_expected = R I_origin R^T`，与直接重算结果比较，验证张量协变性。  
7. `principal_moments_axes` 调用 `numpy.linalg.eigh` 对 `I_com_direct` 做特征分解，提取主惯量与主轴；`analytic_box_inertia_at_com` 生成解析对角解用于误差评估。  
8. `main` 将各项指标整理为 `pandas` 表，按阈值判定并输出 `Validation: PASS/FAIL`（失败则抛异常）。

这里唯一的线性代数库黑盒是 `eigh` 的特征分解本身；建模、张量构造、坐标变换与物理一致性验证均在源码中显式展开。
