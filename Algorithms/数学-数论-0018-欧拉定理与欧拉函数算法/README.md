# 欧拉定理与欧拉函数算法

## R01

欧拉定理与欧拉函数算法的核心是两部分：

- 欧拉函数 `phi(n)`：统计 `1..n` 中与 `n` 互素的正整数个数。
- 欧拉定理：若 `gcd(a, n) = 1`，则 `a^phi(n) ≡ 1 (mod n)`。

本条目 MVP 覆盖两种 `phi` 计算方式：

1. 单点计算（试除分解）  
   利用公式 `phi(n) = n * Π(1 - 1/p)`（`p` 为 `n` 的不同素因子），通过试除找到素因子并迭代更新结果。
2. 区间计算（线性筛）  
   在 `1..N` 上一次性求出全部 `phi(i)`，复杂度线性，适合批量查询。

输入：
- 正整数 `n`（或上界 `N`）
- 可选底数 `a`（用于验证欧拉定理）

输出：
- `phi(n)` 或数组 `phi[1..N]`
- 欧拉定理是否成立的布尔验证结果（在互素前提下）

## R02

关键历史时间点：

- 1736 年：欧拉发表欧拉乘积公式等重要结果，奠定解析与算术函数联系。
- 1760-1763 年前后：欧拉提出并发展费马小定理的推广形式，即后来的欧拉定理。
- 1801 年：高斯在《算术研究》中系统使用并推广 `phi` 记号与相关理论。

因此本算法的理论根基在 18 世纪形成，19 世纪被系统化并纳入现代数论框架。

## R03

提出背景：

- 费马小定理只覆盖模素数情形（`a^(p-1) ≡ 1 mod p`），无法直接处理合数模数。
- 密码、同余方程、群结构分析都需要“合数模下指数循环规律”。
- 欧拉函数提供了“模 `n` 可逆元素数量”的精确定量，欧拉定理给出统一指数周期上界。

欧拉定理解决了“从素数模推广到一般模”的关键缺口，是模算术从特例走向一般理论的里程碑。

## R04

时间复杂度（按实现路径区分）：

- 试除分解求单个 `phi(n)`：`O(sqrt(n))` 最坏复杂度。  
  原因：最多试除到 `sqrt(n)`，并在找到因子后持续整除缩小 `n`。
- 线性筛求 `1..N` 全部 `phi`：`O(N)`。  
  每个合数仅被其最小素因子触达一次。
- 验证 `a^phi(n) mod n`（快速幂）：`O(log phi(n))` 次模乘。

## R05

空间复杂度：

- 试除分解单点计算：`O(1)` 额外空间（不计输入输出）。
- 线性筛批量计算：`O(N)`，主要来自 `phi` 数组、合数标记和素数表。
- 欧拉定理单次验证：`O(1)`（Python 内置大整数对象开销按数值位数增长，但不需额外线性数组）。

## R06

示例：`n = 36`

1. 质因数分解：`36 = 2^2 * 3^2`  
2. 欧拉函数：
   - `phi(36) = 36 * (1 - 1/2) * (1 - 1/3)`
   - `= 36 * 1/2 * 2/3 = 12`
3. 选 `a = 5`，`gcd(5, 36) = 1`
4. 欧拉定理断言：`5^12 ≡ 1 (mod 36)`
5. 计算可得：`5^12 mod 36 = 1`，与理论一致。

额外应用：`5^12345 mod 36 = 5^(12345 mod 12) mod 36 = 5^9 mod 36 = 17`。

## R07

算法意义：

- 理论上：把“模素数”的指数规律推广到“模任意正整数”，并与乘法群结构直接对应。
- 工程上：是 RSA、模逆元求解、循环节分析、快速同余计算中的基础模块。
- 教学上：连接了质因数分解、互素计数、同余、群论四个关键主题，属于数论核心知识节点。

## R08

路径依赖（直接前置知识/算法）：

- 欧几里得算法：用于 `gcd(a, n)` 判定互素前提。
- 质因数分解（至少试除版本）：用于计算 `phi(n)` 的乘积公式。
- 快速模幂（二进制幂）：用于高效验证 `a^phi(n) mod n`。
- 同余运算基本性质：支撑定理表述与实现正确性。
- 可选：线性筛思想，用于 `phi` 批量预处理。

## R09

适用前提与边界条件：

- `phi(n)` 定义域是正整数 `n >= 1`；约定 `phi(1) = 1`。
- 欧拉定理要求：
  - `n >= 2`
  - `gcd(a, n) = 1`
- 若 `a` 与 `n` 不互素，则 `a^phi(n) ≡ 1 (mod n)` 通常不成立。
- 对超大 `n`（如数百位整数），朴素试除分解不再实用，应使用更高阶分解算法。

## R10

正确性依据：

- `phi(n)` 公式正确性：  
  设 `n` 的不同素因子集合为 `{p1, p2, ...}`，在 `1..n` 中，不与 `n` 互素的数恰是被这些素因子整除的数。  
  通过容斥可得保留比例为 `Π(1 - 1/p)`，故 `phi(n) = n * Π(1 - 1/p)`。
- 欧拉定理正确性：  
  设 `R = {r1, ..., r_phi(n)}` 为模 `n` 的既约剩余系。对与 `n` 互素的 `a`，集合 `aR mod n` 只是 `R` 的一个置换。  
  因此 `a^phi(n) * Πri ≡ Πri (mod n)`。又 `Πri` 与 `n` 互素，可约去，得到 `a^phi(n) ≡ 1 (mod n)`。

## R11

数值稳定性与误差传播：

- 该算法是整数离散计算，不涉及浮点迭代收敛，因此不存在传统浮点误差放大问题。
- 主要风险来自实现层：
  - 固定宽度整数语言中的溢出（Python 大整数可规避）。
  - 错误地用浮点表达 `n * (1 - 1/p)` 可能引入截断误差，工程上应坚持整数写法 `result -= result // p`。
- 结论：在整数实现下可视为“数值稳定”；错误来源主要是代码实现失误而非数学病态。

## R12

真实计算成本（超越渐近记号）：

- 单点 `phi(n)`：
  - 常数项小，实现简单。
  - 当 `n` 含大素因子时，试除会接近 `sqrt(n)` 上限，延迟明显。
- 批量 `phi(1..N)`：
  - 线性筛 CPU 友好，顺序访问较多，缓存行为较好。
  - 内存占用随 `N` 线性增长，`N` 很大时会受 RAM 限制。
- 欧拉定理验证：
  - Python `pow(a, e, n)` 已做快速模幂优化，实践常数表现优于手写循环。
  - 若要验证很多 `(a, n)` 对，瓶颈常在 `gcd` 与 `phi` 的重复计算，可做缓存或批处理。

## R14

鲁棒性与失效模式：

- 输入非法：
  - `n <= 0` 时 `phi(n)` 无定义，应抛出异常。
- 误用欧拉定理：
  - 若未检查 `gcd(a, n) = 1` 就套用定理，会得到错误结论。
- 性能失效：
  - 对超大整数仍用试除分解，会出现不可接受的运行时间。
- 实现失效：
  - 若在其他语言中使用 32/64 位整数并发生溢出，会破坏正确性。

## R15

工程实现要点：

- `phi(1)` 作为特判直接返回 `1`。
- 试除时发现因子 `p` 后要“整除到底”，确保每个不同素因子只更新一次：
  - `result -= result // p`
- 循环结束后若剩余 `n > 1`，说明它是最后一个大素因子，也需更新一次。
- 欧拉定理验证前必须先做 `gcd(a, n)` 过滤。
- 批量任务优先用线性筛，不要对每个 `n` 单独试除（会重复工作）。

## R18

源码级算法拆解（本 MVP 未依赖黑箱第三方包）：

1. 输入校验：若 `n <= 0` 抛错；若 `n == 1` 直接返回 `phi(1)=1`。  
2. 初始化：令 `result = n`，`x = n`，从最小候选因子 `p=2` 开始试除。  
3. 因子命中：当 `x % p == 0`，先执行一次 `result -= result // p`，再循环 `x //= p` 直到不能整除，保证“每个不同素因子只处理一次”。  
4. 因子推进：若当前 `p` 不能整除，则继续下一个候选因子；循环条件保持 `p*p <= x`。  
5. 尾因子处理：循环结束若 `x > 1`，说明存在一个大素因子 `x`，执行 `result -= result // x`。  
6. 得到 `phi(n)`：返回 `result`，这就是 `n * Π(1 - 1/p)` 的整数实现版本。  
7. 欧拉定理验证：对 `a=1..n-1`，先检查 `gcd(a, n)==1`，再计算 `pow(a, phi(n), n)`，若出现非 `1` 即判失败。  
8. 批量版本优化：若需要 `1..N` 全部 `phi`，用线性筛维护素数表与最小素因子传播关系，在 `O(N)` 时间内生成整张 `phi` 表。
