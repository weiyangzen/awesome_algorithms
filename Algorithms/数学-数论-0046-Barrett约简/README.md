## R01

Barrett约简（Barrett Reduction）是一种用于快速计算 `x mod n` 的方法。它的核心思想是：

- 当模数 `n` 固定时，先离线预计算一个“倒数近似”参数 `mu`
- 在线阶段用乘法和位移近似商 `x / n`
- 再通过少量修正得到精确余数

它常用于需要大量重复模运算的场景（如模幂、密码学内核、大整数库底层）。

## R02

典型应用场景：

- 固定模数下的大量模乘与模幂
- RSA / Diffie-Hellman / ECC 等密码学实现中的基础算子
- 大整数库（bignum）或多精度算术中的模约简优化
- 需要避免“每次都做完整除法”的硬件/嵌入式环境

## R03

数学定义（以二进制基数为例）：

给定模数 `n > 1`，设：

- `k = bit_length(n)`
- `B = 2^k`
- 预计算 `mu = floor(B^2 / n) = floor(2^(2k) / n)`

对输入 `x`（通常满足 `0 <= x < B^2`），估计商：

`q_hat = floor((x * mu) / B^2)`

再计算：

`r = x - q_hat * n`

最后执行修正（`r` 可能略偏大或偏小）：

- 当 `r >= n` 时重复 `r -= n`
- 当 `r < 0` 时重复 `r += n`

最终得到 `r = x mod n`。

## R04

直观理解：

- 真正困难的是“除以 `n`”
- Barrett 用 `mu ≈ B^2 / n` 把除法转化为“乘法 + 右移”
- 近似商 `q_hat` 与真实商只差很小范围
- 因此余数 `r = x - q_hat*n` 只需要少量加减即可落回 `[0, n-1]`

## R05

正确性要点：

1. `mu` 是 `B^2 / n` 的向下取整，提供了可控误差的倒数近似。
2. `q_hat = floor(x * mu / B^2)` 对 `floor(x/n)` 做近似。
3. 对于 `0 <= x < B^2`，该近似误差有上界，`r = x - q_hat*n` 会落在一个有限邻域内。
4. 因为误差有限，经过有限次 `+/- n` 修正后可得到唯一标准余数区间 `[0, n-1]`。
5. `demo.py` 用随机测试将 `barrett_reduce(x)` 与 Python 真值 `x % n` 对比，验证实现正确性。

## R06

复杂度分析：

- 预计算：一次大整数除法（计算 `mu`）
- 在线约简：
  - 1 次大整数乘法（`x * mu`）
  - 1 次右移（相当于除 `2^(2k)`）
  - 1 次乘法和若干次常数级修正
- 在固定模数、重复调用场景中，摊销成本通常低于“每次完整除法”策略（尤其在底层语言实现中）

## R07

标准实现步骤：

1. 初始化 `BarrettReducer(n)`：计算 `k`、`B2 = 2^(2k)`、`mu = B2 // n`。
2. 约简输入 `x` 时先估计 `q_hat = (x * mu) >> (2k)`。
3. 计算 `r = x - q_hat * n`。
4. 用循环修正 `r` 到 `[0, n-1]`。
5. 在模乘中使用 `reduce(a*b)`，在模幂中用平方-乘法反复调用模乘。

## R09

`BarrettReducer` 接口约定：

- `BarrettReducer(modulus: int)`
  - 约束：`modulus > 1`
- `reduce(x: int) -> int`
  - 当前 MVP 约束：`0 <= x < 2^(2k)`（符合典型单次约简输入范围）
- `mul_mod(a: int, b: int) -> int`
  - 返回 `(a * b) % modulus`，内部走 Barrett 约简
- `pow_mod(base: int, exp: int) -> int`
  - 返回 `base^exp % modulus`，使用平方-乘并调用 `mul_mod`

## R10

测试策略：

- 约简正确性：随机生成 `x`，检验 `reduce(x) == x % n`
- 模乘正确性：随机 `a,b`，检验 `mul_mod(a,b) == (a*b) % n`
- 模幂正确性：随机 `base,exp`，检验 `pow_mod(base,exp) == pow(base,exp,n)`
- 使用固定随机种子保证复现性

## R11

边界条件与异常处理：

- `modulus <= 1`：抛出 `ValueError`
- `exp < 0`（模幂负指数）在本 MVP 中不支持，抛出 `ValueError`
- `x < 0` 或 `x >= 2^(2k)`：`reduce` 抛出 `ValueError`
- `a`、`b`、`base` 允许任意整数，`mul_mod/pow_mod` 内部先归一化到 `[0, n-1]`

## R12

与直接 `%` 的关系：

- 数学结果完全相同
- Python 内置 `%` 在 C 层高度优化，纯 Python Barrett 未必更快
- Barrett 的优势更常出现在可控底层实现中：
  - 避免除法指令
  - 更利于硬件流水化
  - 固定模数时预计算可摊销

## R18

源码级算法流程拆解（对应 `demo.py`，共 8 步）：

1. **预计算阶段**  
   在 `BarrettReducer.__post_init__` 中读取 `n`，计算 `k = bit_length(n)`、`shift = 2k`、`mu = floor(2^(2k)/n)`。

2. **输入范围校验**  
   `reduce(x)` 检查 `0 <= x < 2^(2k)`，保证当前近似公式的误差控制前提成立。

3. **近似商计算**  
   使用 `q_hat = (x * mu) >> (2k)` 近似 `floor(x / n)`，避免直接整除 `n`。

4. **候选余数生成**  
   计算 `r = x - q_hat * n`，得到可能偏大或偏小的候选余数。

5. **修正到标准区间**  
   循环执行 `while r >= n: r -= n` 与 `while r < 0: r += n`，最终落入 `[0, n-1]`。

6. **封装模乘**  
   `mul_mod(a,b)` 先把 `a,b` 归一化到模空间，再调用 `reduce(a*b)` 完成模乘。

7. **封装模幂**  
   `pow_mod(base,exp)` 用平方-乘法：按位扫描指数，循环调用 `mul_mod` 维护结果与底数。

8. **结果验证与基准**  
   `main()` 通过随机测试对比 `%` 和 `pow(..., mod)`，并做简要计时，输出正确性与性能观察。
