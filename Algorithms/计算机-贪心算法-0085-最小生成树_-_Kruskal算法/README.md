## R01

Kruskal 算法用于在无向带权图中构建最小生成树（Minimum Spanning Tree, MST）。
它的贪心策略非常直接：

1. 按边权从小到大排序。
2. 依次尝试加入当前最小边。
3. 若该边连接的是两个不同连通分量，则保留；否则会成环，跳过。
4. 当已选边数达到 `|V|-1` 时停止。

典型实现组合是“排序 + 并查集（Disjoint Set Union, DSU）”。

## R02

历史与定位：

- Kruskal 算法由 Joseph Kruskal 在 1956 年提出。
- 它与 Prim 算法并列为求解 MST 的两种经典基础算法。
- Kruskal 更偏“边驱动”的视角，Prim 更偏“点扩展”的视角。

在现代工程中，Kruskal 依然是教学、面试与中小规模图任务中的高频方案。

## R03

问题背景与动机：

- 许多网络建设问题本质上是“全部节点连通且总成本最小”，例如布线、路网、电网、管网。
- 若节点代表站点、边权代表造价，MST 就是最低成本骨架网络。
- Kruskal 的实现简单、证明清晰、调试容易，适合快速落地。

## R04

时间复杂度：

- 排序边集：`O(E log E)`。
- 并查集操作：每条边最多执行常数次 `find/union`，均摊 `O(alpha(V))`。
- 总体复杂度：`O(E log E)`（主导项来自排序）。

其中 `alpha` 为反 Ackermann 函数，增长极慢，在工程规模下近似常数。

## R05

空间复杂度：

- 边存储与排序需要 `O(E)`。
- 并查集父节点和秩数组需要 `O(V)`。
- 额外空间总计 `O(E + V)`，通常由边集主导。

## R06

手工示例（6 个节点）：

边集（`u-v:w`）：
`0-1:4, 0-2:4, 1-2:2, 1-3:5, 2-3:5, 2-4:11, 3-4:2, 3-5:6, 4-5:1`

按权重升序扫描：

1. 选 `4-5:1`
2. 选 `1-2:2`
3. 选 `3-4:2`
4. 选 `0-1:4`
5. `0-2:4` 成环，跳过
6. 选 `1-3:5`

最终选中 `5 = |V|-1` 条边，MST 总权重为 `14`。

## R07

算法意义：

- Kruskal 是“贪心可证最优”的经典范式。
- 它把“全局最优树”问题分解为“局部安全边选择”问题。
- 在课程体系中，它通常与并查集一起出现，是数据结构与算法结合的典型案例。

## R08

核心前置知识：

- 图论基础：连通图、森林、生成树、环。
- 贪心策略与正确性证明思路（切分性质、交换论证）。
- 并查集：路径压缩、按秩合并。
- 排序复杂度与实现细节。

## R09

适用条件与边界：

- 输入应为无向图；有向图不直接适用。
- 图可含负权、零权、正权，Kruskal 均成立。
- 图不连通时，输出的是最小生成森林（MSF），不是单棵 MST。
- 自环不会被选中，应在实现中过滤。

## R10

正确性简述：

- 切分性质：对任意割，跨割最小边是安全边，可属于某棵 MST。
- Kruskal 每轮都选当前可加入的最小安全边。
- 若某最优解不含该安全边，可通过交换一条不更优的边得到不劣解。
- 反复执行直到结束，得到全局最优 MST（或在不连通图上得到 MSF）。

## R11

精度与数值问题：

- 算法核心是比较与求和，不涉及迭代逼近，数值稳定性通常良好。
- 使用浮点权重时，接近相等的边可能因比较细节触发不同 tie-break 顺序。
- 多棵 MST 并存是正常现象，只要总权重一致且满足树约束即可。

## R12

真实开销分析：

- 实际耗时主要集中在边排序。
- 并查集几乎是常数开销，但仍受内存访问局部性影响。
- Python 中对象化边结构便于阅读，但会引入对象分配与属性访问成本。
- 超大图场景通常会迁移到连续内存结构或编译型实现以提升吞吐。

## R13

结果性质与可复现性：

- 设定固定排序键 `(weight, u, v)` 可提高输出可复现性。
- 同权边可能导致边集不同，但总权重应一致。
- 验证时应优先检查：
  - 是否无环；
  - 是否覆盖全部节点（连通图时）；
  - 总权重是否最小。

## R14

常见失误与风险点：

- 忘记做环检测，结果含环。
- 节点编号越界，导致并查集合并错误。
- 把不连通图当作连通图解释，业务语义错误。
- 没有过滤自环或非法边，造成结果污染或异常。

## R15

工程实现建议：

- 排序键固定为 `(weight, u, v)`，提升调试一致性。
- 并查集务必启用路径压缩 + 按秩合并。
- 当已选边到 `V-1` 时提前终止，避免无效扫描。
- 输入层先做合法性检查，核心循环保持简洁。

## R16

相关算法与应用：

- 相关算法：Prim、Boruvka、Reverse-Delete。
- 典型应用：
  - 网络骨干设计（通信/交通/电力）
  - 聚类预处理（如单链接层次聚类）
  - 图结构压缩与近似算法子步骤
- 可拓展方向：最大生成树、动态 MST、受约束 MST。

## R18

`demo.py` 的源码级算法流程可分为 8 步：

1. **输入清洗**：`kruskal_mst` 检查节点数与边端点范围，忽略自环边。
2. **并查集初始化**：为每个节点创建独立集合，准备连通分量管理。
3. **边排序**：按 `(weight, u, v)` 升序排列，建立确定性贪心顺序。
4. **逐边扫描**：从最小边开始遍历，调用 `union(u, v)` 判断是否跨分量。
5. **安全边接纳**：`union` 成功时加入结果边集并累加权重；失败说明会成环，跳过。
6. **提前终止**：一旦已选边达到 `num_nodes - 1`，立即结束循环。
7. **连通性判定**：用 `len(chosen) == num_nodes - 1` 判断是否得到完整 MST，否则为 MSF。
8. **结果验证**：`main` 通过固定样例断言、随机小图穷举对拍、非连通样例检查三部分构成可运行闭环。
