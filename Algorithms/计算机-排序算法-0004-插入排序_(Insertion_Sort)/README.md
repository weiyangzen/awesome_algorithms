# 插入排序 (Insertion Sort)

- UID: `CS-0004`
- 学科: `计算机`
- 分类: `排序算法`

## R01

插入排序（Insertion Sort）是一种经典的原地稳定排序算法，核心思想是：

- 把序列划分为“已排序区间”和“未排序区间”；
- 每轮从未排序区间取出一个元素（`key`）；
- 在已排序区间中从右向左找到合适位置并插入。

该算法结构短小、逻辑透明，适合教学、验证稳定性以及小规模数据排序。

## R02

问题定义（升序）：

- 输入：长度为 `n` 的可比较序列 `A = [a1, a2, ..., an]`；
- 输出：升序序列 `B`，满足：
  - `B` 是 `A` 的重排；
  - `B[0] <= B[1] <= ... <= B[n-1]`。

本目录 MVP 目标：

- 给出不依赖第三方排序黑箱的插入排序实现；
- 可复现地验证结果与 `sorted()` 一致；
- 用记录型样例验证稳定性。

## R03

为什么选插入排序作为排序算法条目中的基础项：

- 它是许多混合排序（如 Timsort 小分段处理）的基础构件；
- 具有稳定、原地、实现成本低的工程特征；
- 在“近乎有序”数据上有较好实际表现；
- 便于逐行追踪算法正确性与复杂度来源。

## R04

核心机制（本实现）：

1. 外层循环 `i = 1..n-1`：每轮把 `arr[i]` 视作待插入元素 `key`。  
2. 内层循环从 `j = i-1` 向左扫描：当 `arr[j] > key` 时右移 `arr[j]`。  
3. 当扫描结束（`j < 0` 或 `arr[j] <= key`）时，把 `key` 写入 `j+1`。

稳定性的关键点：

- 比较条件使用严格大于 `>`；
- 当 `arr[j] == key` 时不会继续右移，因此相同元素相对顺序不变。

## R05

高层流程：

1. 复制输入序列到可变数组 `out`；
2. 依次处理索引 `i`，将 `out[i]` 插入到 `out[0:i]` 的合适位置；
3. 统计比较次数与移动次数，便于观测行为；
4. 返回排序结果与统计信息；
5. 通过固定样例和随机样例执行自动校验。

## R06

正确性要点（循环不变式）：

- 外层第 `i` 轮开始前，不变式：`out[0:i]` 已非降序排列；
- 内层右移结束后，`j+1` 是 `key` 的正确插入位置；
- 写回 `key` 后，`out[0:i+1]` 仍非降序。

由归纳可得外层循环结束后 `out[0:n]` 全局有序；
再结合“严格大于时才右移”，可得算法稳定。

## R07

复杂度分析：

- 最好情况（已升序）：
  - 时间复杂度 `O(n)`（每轮仅一次比较，无元素移动）；
- 最坏情况（逆序）：
  - 时间复杂度 `O(n^2)`（每轮都需要大量右移）；
- 平均情况：`O(n^2)`；
- 额外空间：`O(1)`（原地修改，不计输出副本时）。

## R08

边界与异常策略：

- 空数组：返回空数组；
- 单元素：直接返回；
- 含负数、重复值、浮点值：统一按比较规则处理；
- 含不可比较混合类型：Python 运行期会抛出类型异常，MVP 不吞错。

## R09

MVP 范围与取舍：

- 保持最小可用，不引入复杂优化（哨兵、二分插入、链表变体）；
- 以“可读、可验证、可运行”为优先；
- 引入基础统计（比较/移动/插入次数）帮助理解成本来源；
- 不把第三方排序库当作算法主体。

## R10

## R15

与其他排序算法对比：

- 与冒泡排序：同为 `O(n^2)` 级，但插入排序通常移动更聚焦，常数更优；
- 与选择排序：选择排序交换次数少但不稳定；插入排序稳定；
- 与归并/堆排序：后者具备 `O(n log n)` 上界，适合大规模数据；
- 插入排序常用于“短数组”或“近乎有序”场景的局部优化。

## R18

`demo.py` 源码级流程拆解（6 步）：

1. `main` 生成固定样例与 `numpy` 随机样例，并逐个调用 `run_case`。  
2. `run_case` 调用 `insertion_sort`，后者复制输入并进入 `insertion_sort_in_place`。  
3. 在 `insertion_sort_in_place` 外层循环中，读取当前 `key = arr[i]`。  
4. 内层 `while` 从右向左比较：当 `arr[j] > key` 时把 `arr[j]` 右移到 `arr[j+1]`，并更新统计。  
5. 内层结束后把 `key` 写回 `arr[j+1]`，此时前缀 `arr[0:i+1]` 恢复有序。  
6. 全部轮次结束后返回结果与统计；`run_case` 用 `sorted()` 对照并汇总通过状态。

说明：排序核心完全由源码显式实现，第三方库仅用于生成可复现随机测试数据，不作为排序黑箱。
