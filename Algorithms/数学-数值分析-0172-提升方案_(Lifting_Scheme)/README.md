## R01

提升方案（Lifting Scheme）是一种把小波变换分解为“分裂-预测-更新”三个原子步骤的构造方法。  
它既能表达经典离散小波变换（DWT），也能在非均匀网格或几何数据上构造“第二代小波”。

对 1D 信号 `x`，一层提升的核心流程是：

1. Split：按奇偶位拆分为 `s`（偶位）和 `d`（奇位）。
2. Predict：用 `s` 预测 `d`，得到细节 `d <- d - P(s)`。
3. Update：用细节回馈 `s`，得到近似 `s <- s + U(d)`。

## R02

历史定位：

- 提升方案由 Wim Sweldens 在 1990s 系统化提出。
- 它把卷积+下采样形式的小波滤波器组，改写为一串可逆局部操作。
- JPEG 2000 中可逆 5/3 小波与不可逆 9/7 小波都可用提升结构实现。
- 在“需要原地计算、低内存、可逆整数变换”的工程中非常常见。

## R03

它解决的核心问题：

- 经典滤波器组实现通常需要更长卷积核与额外缓存。
- 直接设计满足双正交条件的滤波器并不直观。
- 在整数数据场景（图像无损压缩）中，需要严格可逆的整数映射。

提升方案的工程价值在于：

- 以局部预测/校正方式构造小波，结构清晰。
- 天然支持原地计算（in-place）。
- 容易做可逆整数版本（通过取整策略）。

## R04

时间复杂度（长度 `N`，分解层数 `L`）：

- 一层提升对每个样本做常数次算术运算，复杂度 `O(N)`。
- 多层分解长度依次 `N, N/2, N/4, ...`，总复杂度：
  `O(N + N/2 + ... + N/2^(L-1)) = O(N)`。
- 逆变换与正变换同阶，亦为 `O(N)`。

## R05

空间复杂度：

- 若采用原地实现，额外工作区可做到 `O(1)`（不计输入输出本身）。
- 本目录 MVP 为了可读性，局部使用短向量临时副本，整体仍是线性规模存储 `O(N)`。
- 多层系数打包在同一数组（前段近似、后段细节），便于后续阈值化与重构。

## R06

CDF 5/3 一层手算示例（浮点版，边界复制）：

原信号：`x = [4, 6, 10, 12]`

1. Split：
   - `s = [4, 10]`（偶位）
   - `o = [6, 12]`（奇位）
2. Predict（`d_i = o_i - 0.5*(s_i + s_{i+1})`，末端 `s_{i+1}=s_i`）：
   - `d0 = 6 - 0.5*(4+10) = -1`
   - `d1 = 12 - 0.5*(10+10) = 2`
3. Update（`s_i = s_i + 0.25*(d_{i-1}+d_i)`，左端 `d_{-1}=d_0`）：
   - `s0 = 4 + 0.25*(-1 + -1) = 3.5`
   - `s1 = 10 + 0.25*(-1 + 2) = 10.25`

得到一层系数：近似 `[3.5, 10.25]`，细节 `[-1, 2]`。  
逆过程按“先反更新、后反预测”可精确回到 `[4, 6, 10, 12]`（仅有浮点舍入误差）。

## R07

重要性：

- 是现代小波工程实现的“通用骨架”。
- 将滤波器设计转化为局部预测器与更新器设计，便于解释和调试。
- 对压缩、去噪、多分辨率分析、图像与几何处理都很关键。
- 在硬件实现中，原地与局部依赖特性有利于流式处理。

## R08

理论基础（以双通道情形概述）：

- 小波分解本质是把信号空间分为低频近似空间 `V` 与高频细节空间 `W`。
- 提升将多相矩阵（polyphase matrix）分解为若干个三角矩阵乘积：
  - 预测步对应一个上三角变换；
  - 更新步对应一个下三角变换；
  - 可选尺度归一化对应对角矩阵。
- 每一步都可逆，因此整体可逆。

本 MVP 采用 CDF 5/3 的常见浮点形式：

- Predict：`d_i <- d_i - 0.5*(s_i + s_{i+1})`
- Update：`s_i <- s_i + 0.25*(d_{i-1} + d_i)`

## R09

适用条件与限制：

- 输入长度应能被 `2^L` 整除（`L` 为分解层数）。
- 边界处理必须正逆一致（本实现用复制边界）。
- 对强噪声与非平稳信号，阈值策略决定去噪效果。
- 本 MVP 采用浮点提升，不提供严格整数可逆（若需无损，可加入 floor/round 规则）。

## R10

正确性框架（工程可验版）：

1. 一层变换由 `Split -> Predict -> Update` 组成。
2. 逆变换按反序执行：`Undo Update -> Undo Predict -> Merge`。
3. 若边界规则一致，且每步使用同一参数，则单层映射为双射。
4. 多层结构是“对子段重复同样双射”，因此整体仍可逆。
5. 用 `||x_hat - x|| / ||x||` 进行重构误差检查即可验证实现一致性。

## R11

数值误差与稳定性：

- 浮点版本主要误差来自舍入；多层后会有小量累计。
- 本实现重构误差通常在 `1e-12 ~ 1e-14`（双精度、随机信号）。
- 采用边界复制时，边缘样本会有轻微偏置风险，但实现简单且可逆。
- 阈值压缩/去噪引入的是“有意损失”，误差可通过 MSE 或相对范数衡量。

## R12

性能视角：

- 相比“卷积+下采样”直接实现，提升法可减少乘加与缓存访问。
- 原地布局提升了缓存友好性，适合大规模数据流。
- 多层复杂度线性，适合实时或批量处理。
- 本 MVP 用 NumPy + 显式步骤，重点是透明性而非极限速度。

## R13

可验证保证（本目录 `demo.py`）：

- 去噪验证：对含噪信号做小波软阈值，断言去噪后 MSE 低于原始噪声 MSE。
- 稀疏压缩验证：仅保留部分大系数细节，报告稀疏率与重构误差。
- 全流程无交互输入，运行结束输出 `All checks passed.`。

## R14

鲁棒性与常见失效点：

- `levels < 1`、输入为空、长度非偶数或不能整除 `2^levels` 时应拒绝。
- 正逆边界规则不一致会导致明显重构偏差。
- 阈值过大可能过度平滑，导致细节丢失。
- 极短信号配合过多分解层数会触发非法维度。

## R18

源码级算法流程拆解（`demo.py`，8 步）：

1. 读取输入向量并检查长度、层数约束（必须满足可多层二分）。
2. 在 `forward_lifting_cdf53_one_level` 中执行奇偶拆分，得到 `approx/detail`。  
3. 对每个细节点执行预测步 `detail <- detail - P(approx)`，提取局部高频残差。  
4. 对每个近似点执行更新步 `approx <- approx + U(detail)`，补偿低频偏移。  
5. 将一层结果按 `[approx | detail]` 原地回写，并对子段递归重复直至目标层数。  
6. 在逆变换中按相反层序取出 `[approx | detail]`，先执行反更新再反预测。  
7. 将恢复的偶位/奇位交错合并，逐层扩展直到重建整段信号。  
8. 在应用侧对细节系数做软阈值或保留大系数，再逆变换得到去噪/压缩结果并用 MSE、相对误差验收。

本 MVP 没有调用黑箱小波库；正逆更新公式在源码中逐步展开，可直接审计每一步数值流。
