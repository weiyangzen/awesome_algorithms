## R01

算法主题是“一元多项式结式（Resultant）计算”。  
给定两个一元多项式 `p(x), q(x)`，结式 `Res(p, q)` 是一个仅由系数决定的标量，关键性质是：

- `Res(p, q) = 0` 当且仅当 `p, q` 有公共根（在代数闭包中）。

这个性质使它在“公共根检测、消元、代数几何前处理、符号计算”里非常常用。

## R02

问题输入与输出（本 MVP 约定）：

- 输入：两个按降幂排列的整数系数数组。  
  例：`[1, -3, 2]` 表示 `x^2 - 3x + 2`。
- 输出：`Res(p, q)` 的整数值（Python `int`，可大整数）。

额外约定：

- 自动去除首部多余零系数。
- 处理常数多项式的简化规则：  
  `deg(p)=0` 时 `Res(p,q)=p0^(deg q)`；  
  `deg(q)=0` 时 `Res(p,q)=q0^(deg p)`。

## R03

数学定义（根表示）：

设

- `p(x) = a_m ∏_{i=1..m}(x-α_i)`
- `q(x) = b_n ∏_{j=1..n}(x-β_j)`

则

- `Res(p, q) = a_m^n ∏_{i=1..m} q(α_i)`
- 等价地 `Res(p, q) = (-1)^{mn} b_n^m ∏_{j=1..n} p(β_j)`

因此当且仅当存在 `α_i=β_j` 时，乘积出现零因子，结式为零。

## R04

计算实现采用 Sylvester 矩阵定义：

- 若 `deg(p)=m, deg(q)=n`，构造 `(m+n) x (m+n)` 的 Sylvester 矩阵 `S(p,q)`。
- 前 `n` 行是 `p` 的系数向量按行右移填充。
- 后 `m` 行是 `q` 的系数向量按行右移填充。
- `Res(p,q) = det(S(p,q))`。

本实现完全按这个定义编码，不调用外部 CAS 黑盒 `resultant()`。

## R05

数值稳定性与精确性策略：

- 不用浮点高斯消元，避免舍入误差导致“接近 0 但不为 0”的误判。
- 采用 Bareiss fraction-free 消元计算整数矩阵行列式：
  - 中间量保持整数；
  - 每轮消元后做整除约化；
  - 精确得到整数行列式。

这保证了“公共根判零”在整数输入下是严格准确的。

## R06

核心函数（`demo.py`）：

- `_trim_leading_zeros(coeffs)`：规范化多项式表示。
- `sylvester_matrix(p, q)`：构造 Sylvester 矩阵。
- `bareiss_det_int(A)`：整数行列式。
- `resultant(p, q)`：统一入口，含常数特判。
- `eval_poly(p, x)`：验证用 Horner 求值。

代码组织目标是“最小但可追踪”：每个数学步骤对应一个函数。

## R07

时间复杂度：

- 构造矩阵：`O((m+n)^2)`。
- Bareiss 消元主开销：`O((m+n)^3)` 算术操作。

空间复杂度：

- 存储矩阵 `O((m+n)^2)`。

备注：若多项式次数很高，结果整数位数增长明显，实际耗时会受大整数乘除成本影响。

## R08

与其他路线的对比：

- 与欧几里得算法 + 子结果式（subresultant PRS）相比：
  - Sylvester + Bareiss 实现更直接、教学性更强；
  - 高次数时，PRS 通常更高效。
- 与数值方法（特征值/根近似）相比：
  - 本法不依赖根求解，能避免数值不稳定。

本项目定位为 MVP，优先“可读、可验证、无黑盒”。

## R09

正确性检查（demo 内置）：

- 案例1：`(x-1)(x-2)` 与 `(x-2)(x-3)` 共享根 2，结果应为 0。
- 案例2：`x^2-1` 与 `x-2` 无公共根，结果应为 `p(2)=3`。
- 案例3：`2x^2-3x+1` 与 `3x+1` 无公共根，按公式 `3^2*p(-1/3)=14`。
- 额外线性因子检验：验证 `Res(p, x-a)=(-1)^{deg(p)}p(a)`（与本实现采用的 Sylvester 排布一致）。

运行 `python3 demo.py` 会逐条打印 `OK/MISMATCH`。

## R10

边界条件处理：

- 首部零系数：自动裁剪，不影响次数定义。
- 全零“多项式”未支持（数学上不适合作常规结式输入），会触发错误。
- 常数-常数输入：返回 1 作为约定化值（便于代码流程闭合）。
- 零主元情况：Bareiss 中通过换行（行交换）继续；若整列都为零，行列式为 0。

## R11

输出解释建议：

- `Res(p,q)=0`：判定为“存在公共根”。
- `Res(p,q)≠0`：判定为“无公共根”。
- 数值绝对值大小本身不是“接近程度”的稳定指标（尤其大次数），主要用于符号/离散判定。

若后续需要“近公共根”分析，应另配数值条件数或近似 GCD 方法。

  - 再考虑切换更高效的 subresultant 算法。

## R17

结式在计算代数中的定位：

- 它是“判公共根”和“变量消元”的基础代数工具；
- 常作为 Gröbner 基、代数曲线交点分析、参数消元的前置模块；
- 在符号推导中，结式提供了比直接求根更结构化的路径。

因此该算法条目属于“基础但高杠杆”组件。

## R18

`demo.py` 的源级算法流程（7 步）：

1. 读取 `p, q` 系数并用 `_trim_leading_zeros` 规范化输入。  
2. 在 `resultant()` 中判断常数特例，直接返回幂值，避免不必要矩阵化。  
3. 调用 `sylvester_matrix(p,q)` 构造 `(m+n)×(m+n)` Sylvester 矩阵：前 `n` 行放 `p` 的平移，后 `m` 行放 `q` 的平移。  
4. 将矩阵交给 `bareiss_det_int()`，执行 fraction-free 消元。  
5. 每一轮以 `a[k][k]` 为主元，若主元为 0 则换行修复；若整列为 0，立即返回 0。  
6. 用 `a[i][j] = (pivot*a[i][j] - a[i][k]*a[k][j]) / denom` 更新右下子块，并保持整除，确保全程整数精确。  
7. 最终返回最后对角元（乘行交换符号）作为 `det(S)`，即 `Res(p,q)`；`main()` 用已知公式案例做一致性核验并打印 `OK`。  

该流程与结式的定义完全对应，不依赖外部黑盒求解器。
