## R01

试位法（False Position / Regula Falsi）是**带区间约束的一维非线性方程求根算法**。

目标：给定连续函数 `f(x)`，在区间 `[a, b]` 上满足 `f(a) * f(b) < 0`，求方程 `f(x)=0` 的一个根 `x*`。

核心更新：用端点 `(a, f(a))` 与 `(b, f(b))` 的割线与 `x` 轴交点作为新估计值。

`x_r = (a f(b) - b f(a)) / (f(b) - f(a))`

与普通割线法不同，试位法每次都保持“异号包围区间”，因此有明确的区间安全性。

## R02

历史上，Regula Falsi 是早期数值计算中非常经典的“假位法/试位法”思想：

- 在牛顿法普及前，它是手工求根的重要方法。
- 它把线性插值（割线）与“符号包围”结合，兼顾一定速度与稳定性。
- 现代教材常把它放在二分法与割线法之间：比二分法常快、比割线法更稳。

## R03

提出背景：

- 二分法只用区间中点，稳定但收敛慢（线性收敛且固定缩半）。
- 牛顿法需要导数，且初值不佳时可能发散。
- 割线法不需导数，但不保区间，可能跳出根附近。

试位法在工程上常被用于：

- 导数难写或不可靠时的标量方程求根。
- 需要“始终保持根被包围”这一安全约束的场景。
- 作为更复杂混合求根器（如 Brent 思想）的基础组件。

## R04

时间复杂度（迭代视角）：

- 单次迭代仅做常数次函数计算与四则运算，单步代价 `O(1)`（不计 `f` 本身复杂度）。
- 总复杂度可写为 `O(k)`，`k` 为达到容差所需迭代次数。
- 经典试位法在某些函数上会出现“单端点长期不动”的停滞，导致 `k` 明显增大。

若把函数求值成本记为 `C_f`，总成本约为 `O(k * C_f)`。

## R05

空间复杂度：

- 基础实现只需保存当前端点、函数值和当前估计，额外空间 `O(1)`。
- 若像本目录 MVP 一样记录迭代历史用于分析，则为 `O(k)`。

## R06

手算示例：求 `f(x)=x^2-2` 在 `[1,2]` 的根。

- 初值：`f(1)=-1, f(2)=2`。
- 第 1 次：
  - `x1 = (1*2 - 2*(-1)) / (2-(-1)) = 4/3 = 1.333333...`
  - `f(x1) = -0.2222...`，与 `f(2)` 异号，更新区间为 `[x1, 2]`。
- 第 2 次：
  - 用 `[1.3333..., 2]` 再插值得 `x2 ≈ 1.4`
  - `f(x2)=-0.04`，区间更新为 `[1.4, 2]`。
- 第 3 次：
  - `x3 ≈ 1.4117647`
  - `f(x3)≈-0.00692`。

可见根逐步逼近 `sqrt(2)≈1.41421356`，且始终保持异号区间。

## R07

算法意义：

- 在“无导数 + 要稳定包围”的前提下，提供比纯二分更激进的插值更新。
- 是理解现代鲁棒根求解器的重要台阶。
- 展示了“几何插值 + 拓扑符号约束”这一经典数值分析组合思想。

## R08

理论基础：

- 介值定理：连续函数在异号区间上至少有一个零点。
- 线性插值：用端点割线近似函数局部行为。
- 区间不变性：每次更新只替换与新点同号的一端，保持 `f(a)*f(b)<=0`。
- 收敛直觉：当函数在根附近较平滑且不出现严重端点偏置时，插值点会向根推进。

## R09

适用前提与边界：

- `f` 在 `[a,b]` 上应连续。
- 必须有 `f(a)*f(b) <= 0`，且通常要求严格异号以保证“包围根”。
- 若某端点正好是根（`f(a)=0` 或 `f(b)=0`），应立即返回。
- 对多重根或极端不对称曲线，经典试位法可能很慢。

## R10

正确性要点（简述）：

1. 初始化时 `f(a)*f(b) < 0`，依据介值定理，根在 `[a,b]` 内。
2. 计算割线零点 `x_r` 后，检查 `f(x_r)` 符号：
   - 若 `f(a)*f(x_r) < 0`，根在 `[a, x_r]`；
   - 否则根在 `[x_r, b]`。
3. 每轮都保持“新区间仍包围根”，因此不会像无约束割线法那样失去根。
4. 在常见条件下（连续且单根），序列会收敛到某个零点；即使速度不快，区间安全性始终成立。

## R11

数值稳定性与误差：

- 优点：始终维持包围区间，避免大幅发散。
- 误差衡量常用两类：
  - 残差 `|f(x_r)|`
  - 区间宽度 `|b-a|`
- 风险：当某一端函数值绝对值长期远大于另一端时，插值点会长期贴近同一侧，出现停滞。
- 浮点实现中需注意 `f(b)-f(a)` 过小导致的数值放大；应设置分母保护阈值。

## R12

真实工程成本：

- 对“函数求值昂贵”的问题，迭代次数比单步算术成本更关键。
- 试位法通常每轮只新增 1 次函数求值（端点值可复用），这在工程上很划算。
- 但若发生停滞，整体 wall-clock 仍可能被拉长。
- 实际工程常用改进版（Illinois/Pegasus）或与二分法混合以抑制停滞。

## R14

鲁棒性与失效模式：

- 失效模式 1：初始区间不异号，算法无数学保证。
- 失效模式 2：`f(b)-f(a)` 接近 0，公式会不稳定。
- 失效模式 3：经典试位法停滞（一个端点几乎不动），收敛非常慢。
- 失效模式 4：函数不连续或含奇点，介值定理前提被破坏。

应对策略：

- 启动前校验符号与有限值。
- 加入 `max_iter` 与多终止准则（残差 + 区间宽度）。
- 监控端点长期不更新情况，必要时切换改进法或混合法。

## R15

工程实现要点：

- 缓存端点函数值，避免重复计算。
- 每轮更新后只重算被替换端点的函数值。
- 终止准则同时看 `|f(x)|` 与区间宽度，降低“伪收敛”风险。
- 提供结构化迭代日志（迭代号、区间、估计点、残差），便于调参与排障。
- demo 中加入二分法对照，验证结果合理性而非盲信单算法输出。

## R16

前驱、后继与应用：

- 前驱知识：介值定理、二分法、割线插值。
- 后继方法：Illinois/Pegasus 改进试位法、Brent-Dekker 混合法。
- 应用场景：
  - 物理/工程标量闭式方程求根。
  - 参数反解（由输出量反推输入量）。
  - 作为更大数值流程中的一维内核求解器。

## R18

源码级算法流追踪（`demo.py`，非黑箱）：

1. `main` 定义多个固定测试函数与区间，并设定统一容差与最大迭代数。  
2. `false_position` 启动时先算 `f(a), f(b)` 并做合法性检查：端点有限、异号包围、端点是否已是根。  
3. 每轮根据割线公式计算 `x_r = (a f(b) - b f(a)) / (f(b)-f(a))`，这是试位法的核心“假位”步骤。  
4. 评估 `f(x_r)` 后记录日志（区间宽度、残差等），再按符号规则替换同号端点，保持包围不变。  
5. 每次迭代检查双终止条件：`|f(x_r)| <= tol` 或区间宽度足够小，满足即返回收敛结果。  
6. 若达到 `max_iter` 仍未满足条件，返回 `converged=False`，同时保留最后状态给上层诊断。  
7. `main` 再调用 `bisection` 作为基线，对比两个方法得到的根并执行断言，形成最小可验证闭环。  
8. 可选 `numpy` 仅用于统计迭代残差均值（若环境缺失则自动回退纯 Python），不参与求根核心逻辑。
