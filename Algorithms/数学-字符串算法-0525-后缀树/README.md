# 后缀树

- UID: `MATH-0525`
- 学科: `数学`
- 分类: `字符串算法`
- 源序号: `525`
- 目标目录: `Algorithms/数学-字符串算法-0525-后缀树`

## R01

本条目实现“后缀树”最小可运行 MVP，采用压缩后缀 Trie 的朴素构建方式：
- 对每个后缀依次插入；
- 在首个不匹配处进行边分裂；
- 形成边标签为字符串区间的压缩树结构。

MVP 目标：
- 可构建后缀树并执行 `contains(pattern)` 子串查询；
- 可从树叶导出后缀数组并与朴素结果对照；
- 内置固定样例，直接运行，无需交互输入。

## R02

问题定义（本实现范围）：
- 输入：任意 Python 字符串 `text`。
- 预处理：自动追加一个不在原串中的唯一终止符 `terminal`。
- 输出能力：
  - `contains(pattern)`：判断模式串是否为 `text` 子串；
  - `suffix_array()`：返回不含终止符后缀的字典序起点下标；
  - `leaf_count()`：返回叶子数量（理论值 `len(text)+1`，包含仅终止符后缀）。

## R03

核心思想：
1. 后缀树可视作所有后缀构成的 Trie 的“路径压缩”结果。  
2. 每条边存储为原文本中的区间 `[start, end)`，不复制子串。  
3. 插入一个新后缀时，沿边字符比较直到发生分歧。  
4. 若边上出现首个不匹配字符，则把该边拆成“公共前缀 + 两条分支”。  
5. 追加唯一终止符后，各后缀互不为前缀，叶子与后缀一一对应。

## R04

构建流程（高层）：
1. 选择唯一终止符并构造 `full_text = text + terminal`。  
2. 建立根节点。  
3. 依次插入 `full_text[i:]`（`i = 0..n`）。  
4. 插入时按首字符查找出边。  
5. 若无对应出边，直接挂新叶子边。  
6. 若有对应边，计算最长公共前缀长度 `lcp`。  
7. 当 `lcp` 小于边长时执行边分裂，重挂旧子树并新增新叶。  
8. 全部后缀插入后得到压缩后缀树。

## R05

核心数据结构（`demo.py`）：
- `Edge(start, end, child)`：边标签为 `full_text[start:end]`。  
- `Node(children, suffix_index)`：
  - `children`：`首字符 -> Edge`；
  - `suffix_index`：仅叶子节点保存该叶对应后缀起点。  
- `SuffixTree`：封装构建、查询、后缀数组提取与校验辅助。

## R06

正确性要点：
- 完整性：每个后缀都被插入并终止于某个叶子。  
- 无歧义：终止符不在原串中，确保后缀路径唯一终止。  
- 分裂保持不变量：
  - 分裂后公共前缀仍在父边上；
  - 原后缀剩余部分与新后缀剩余部分分别成为两条分支。  
- 查询正确性：`contains` 沿树按字符匹配，失败即无该子串，完整匹配即存在。

## R07

复杂度：
- 朴素构建：最坏 `O(n^2)` 时间，`O(n^2)` 空间（节点/边数量与重复结构相关）。  
- 子串查询：`O(m)`，`m` 为模式串长度（沿边逐字符比较）。  
- 后缀数组导出：DFS `O(n)` 级别（忽略字符排序常数）。

说明：该条目强调可读和可验证，不追求 Ukkonen 的线性构建；线性在线版本见相邻条目 `MATH-0526`。

## R08

边界与异常：
- 空串：树中只有终止符后缀，`leaf_count() == 1`。  
- 空模式：按约定 `contains("") == True`。  
- 模式含终止符：直接返回 `False`（用户查询只面向原始文本）。  
- 输入可能包含常见符号：实现会扫描候选字符并自动选择未出现的终止符。

## R09

MVP 取舍：
- 不依赖第三方后缀树库，避免黑盒。  
- 只做单字符串后缀树，不扩展到广义后缀树。  
- 不实现动态增删文本。  
- 使用“后缀数组对照 + 子串对照 + 叶子数断言”建立最小可信验证闭环。

## R10

`demo.py` 主要函数职责：
- `SuffixTree.__init__`：初始化并插入所有后缀。  
- `_insert_suffix`：逐后缀插入与边分裂逻辑。  
- `contains`：子串存在性查询。  
- `suffix_array`：按字典序 DFS 叶子得到后缀数组。  
- `leaf_count`：统计叶子数。  
- `run_case`：单样例断言校验。  
- `main`：组织固定测试样例并打印结果。

## R12

输出解释：
- `Text`：当前样例字符串。  
- `Nodes`：总节点数（根 + 内部节点 + 叶子）。  
- `Leaves`：叶子数与理论值对照。  
- `Suffix Array`：由后缀树导出的后缀数组。  
- `Pattern Checks`：`pattern:True/False`，并已在程序内与 `pattern in text` 做一致性断言。

若任一断言失败，程序会抛异常并退出。

## R13

内置测试集：
- `banana`：经典教学串；  
- `mississippi`：重复模式密集；  
- `xabxac`：常见教材示例；  
- `aaaaa`：高度重复边界；  
- `""`：空串边界。

每个样例都会校验：

- 叶子数等于 `len(text)+1`；
- 每个后缀都能被 `contains` 命中；
- 多个模式串查询结果与 Python `in` 一致。

## R15

方法对比：
- 后缀树（本实现）：
  - 优点：一次构建后可高效多次查询；
  - 缺点：实现复杂度与内存常数较高。  
- 后缀数组：
  - 优点：结构更紧凑；
  - 缺点：查询常需二分和额外比较。  
- 朴素扫描：
  - 优点：实现最简单；
  - 缺点：多查询时总成本高。

## R16

典型应用：
- 文本索引与高频子串查询。

## R18

源码级算法流（`demo.py`，8 步）：
1. `main` 定义固定文本样例与模式串集合。  
2. `run_case` 为每个样例构造 `SuffixTree(text)`。  
3. 构造函数选择唯一终止符，生成 `full_text`，并按起点枚举所有后缀。  
4. `_insert_suffix` 从根出发，按首字符选择边并计算当前后缀与边标签的最长公共前缀。  
5. 若边完全匹配则继续向下；若无边则直接创建叶子边；若中途失配则创建分裂节点并重挂旧边与新叶。  
6. 构建后通过 `contains` 在压缩边上逐字符匹配，实现子串查询。  
7. `suffix_array` 对树做字典序 DFS，收集叶子 `suffix_index`（过滤掉仅终止符后缀）。  
8. `run_case` 把树结果与朴素后缀数组、`pattern in text`、叶子数量逐项断言并输出报告。
