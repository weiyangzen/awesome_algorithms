## R01

核 SVM（Kernel Support Vector Machine）是把线性 SVM 扩展到非线性分类的经典方法。核心思想是：
- 在原始空间中难以线性分开的样本，通过核函数 `K(x_i, x_j)` 隐式映射到高维特征空间；
- 在高维空间中仍然做“最大间隔”分类；
- 最终决策只依赖支持向量与核函数值，不需要显式构造高维特征。

本条目实现一个最小可运行 MVP：二分类核 SVM，训练采用简化版 SMO（Sequential Minimal Optimization）更新对偶变量。

## R02

问题定义（二分类，标签 `y_i ∈ {-1,+1}`）：

原始软间隔形式：

`min_{w,b,ξ} 1/2 ||w||^2 + C Σ ξ_i`

约束：

`y_i (w^T φ(x_i) + b) >= 1 - ξ_i, ξ_i >= 0`

核技巧下的对偶问题：

`max_α Σ α_i - 1/2 Σ_i Σ_j α_i α_j y_i y_j K(x_i, x_j)`

约束：

`0 <= α_i <= C, Σ α_i y_i = 0`

预测函数：

`f(x) = Σ_i α_i y_i K(x_i, x) + b`

`sign(f(x))` 即类别。

## R03

实现策略选择：
- 不把 `sklearn.svm.SVC` 当黑盒直接调用；
- 在 `demo.py` 中显式实现核矩阵、KKT 违反检查、双变量更新、阈值 `b` 更新；
- 这样可以直接看到“哪些样本成为支持向量、哪些样本推动优化”。

这更符合“算法条目”而不是“API 展示”。

## R04

MVP 用到的关键数学关系：

1. 决策值与误差：
`f_i = Σ_j α_j y_j K(x_j, x_i) + b`
`E_i = f_i - y_i`

2. KKT 近似违反条件（SMO 常见判据）：
- 若 `y_i E_i < -tol` 且 `α_i < C`，或
- `y_i E_i > tol` 且 `α_i > 0`，
则样本 `i` 可作为更新候选。

3. 成对更新边界（`α_i, α_j`）：
- 若 `y_i != y_j`：
`L = max(0, α_j - α_i), H = min(C, C + α_j - α_i)`
- 若 `y_i == y_j`：
`L = max(0, α_i + α_j - C), H = min(C, α_i + α_j)`

4. 更新核心量：
`η = 2K_ij - K_ii - K_jj`
`α_j <- clip(α_j - y_j(E_i - E_j)/η, L, H)`
`α_i <- α_i + y_i y_j (α_j_old - α_j_new)`

5. 阈值更新：
由 `b1/b2` 公式计算；若某个新 `α` 在 `(0,C)` 内优先使用对应阈值，否则取平均。

## R05

算法高层流程：

1. 校验输入、把标签映射到 `{-1,+1}`。
2. 对特征做标准化，减少核参数敏感性。
3. 计算训练核矩阵 `K(X, X)`。
4. 循环扫描样本，找 KKT 违反点 `i`，随机挑选 `j` 组成更新对。
5. 按 SMO 规则更新 `α_i, α_j, b`。
6. 连续若干轮无更新后停止（`max_passes`），或达到 `max_iter`。
7. 以 `α > alpha_tol` 识别支持向量。
8. 用支持向量构建预测函数并评估训练/测试指标。

## R06

正确性直觉：
- 对偶变量 `α_i` 非零意味着样本进入决策函数，即成为支持向量或边界点；
- SMO 每步在可行域内优化两个变量，保持约束可行性；
- 通过核函数，分类边界在原空间可非线性；
- 使用 RBF 核时，模型能拟合 XOR 这类线性不可分结构。

该 MVP 是教学导向的简化 SMO，不追求工业级收敛速度，但优化目标与核 SVM 理论一致。

## R07

复杂度（设样本数 `n`、特征维度 `d`）：
- 构建核矩阵：`O(n^2 d)`；
- 存储核矩阵：`O(n^2)`；
- 每次样本扫描涉及若干内积，近似 `O(n^2)` 级；
- 总训练成本与迭代次数相关，通常可写为 `O(T n^2)` 量级（此处 `T` 为扫描轮次/有效更新规模）。

因此核 SVM 更适合中小规模数据；大规模时通常需要近似核或线性化方法。

## R08

边界与异常处理：
- `x` 必须二维、`y` 必须一维且样本数一致；
- 输入必须是有限值（无 `nan/inf`）；
- 仅支持二分类；
- 约束 `C>0, gamma>0, degree>=1, tol/eps/alpha_tol>0`；
- `predict/decision_function` 必须在 `fit` 之后调用；
- 推理时特征维度必须与训练一致。

## R09

MVP 范围：
- 已实现：`linear/poly/rbf` 三种核、二分类、简化 SMO、支持向量统计、基础指标输出。
- 未实现：多分类（OvR/OvO）、启发式 `j` 选择、误差缓存加速、概率输出、交叉验证调参。

取舍重点是“机制透明 + 可运行”，不是最优工程性能。

## R14

关键超参数与调参建议：
- `C`：越大越强调训练拟合，越小正则越强；
- `gamma`（RBF/Poly）：越大边界越“局部”，过大易过拟合；
- `degree`（Poly）：多项式阶数，越高边界越复杂；
- `tol`：KKT 判断容忍度；
- `eps`：判定 `α_j` 更新是否有效的最小步长；
- `max_passes/max_iter`：停止条件。

实务上常先固定 `C`，扫描 `gamma`（如对数网格），再联合细调。

## R15

与相关模型对比：
- 对比线性 SVM：核 SVM 能处理明显非线性边界，但计算更重；
- 对比逻辑回归：逻辑回归更易解释概率，核 SVM 更强调间隔与边界质量；
- 对比树模型：树模型在特征交互上天然灵活，核 SVM 在中小数据和边界学习上常更稳定；
- 对比神经网络：神经网络容量更强，但训练与调参成本通常更高。

## R16

典型应用场景：
- 中小样本但边界非线性的二分类任务；
- 生物信息、文本特征映射后的分类；
- 需要“支持向量 + 核函数”这类经典可解释结构时；
- 作为复杂模型前的高质量非线性基线。

## R18

`demo.py` 源码级算法流程（8 步）：

1. `main` 调用 `make_xor_dataset` 与 `train_test_split`，得到可复现训练/测试集。
2. `KernelSVMMVP.fit` 先做 `_validate_xy` 与 `_to_signed_labels`，将标签转成 `{-1,+1}`。
3. `StandardScalerMVP.fit_transform` 对训练特征标准化，再由 `_kernel_matrix` 计算 `K(X, X)`。
4. 训练循环中对每个样本 `i` 计算 `f_i`、`E_i`，按 KKT 违反条件决定是否参与更新。
5. 若需要更新，随机选 `j`，基于 `L/H` 约束与 `η` 公式更新 `α_j`，再反推 `α_i`，并更新阈值 `b`。
6. 当连续 `max_passes` 轮无有效更新或达到 `max_iter` 时停止，并以 `alpha_tol` 提取支持向量。
7. `decision_function` 在推理时仅对支持向量计算核值并加权求和：`K(x, x_sv) @ (α_sv * y_sv) + b`。
8. `predict` 将分数过零阈值映射回原始类别，`summarize_metrics` 输出准确率、精确率、召回率、F1 与混淆矩阵。
