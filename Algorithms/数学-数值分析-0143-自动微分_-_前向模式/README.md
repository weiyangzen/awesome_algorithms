# 自动微分 - 前向模式

- UID: `MATH-0143`
- 学科: `数学`

## R01

本条目实现“自动微分（Automatic Differentiation, AD）前向模式”的最小可运行 MVP。  
目标不是调用黑箱库一键求导，而是用 `Dual Number`（对偶数）把链式法则嵌入运算符，直接在代码层展示“值 + 导数”同步传播。

## R02

前向模式 AD 的核心对象是 `x + x'ε`，其中 `ε^2 = 0`。  
若把每个中间变量都写成对偶数形式，普通算术和初等函数在传播实部（函数值）的同时，会自动传播 `ε` 系数（导数或方向导数），这就是“程序即求导器”的关键。

## R03

本 MVP 覆盖两类输入输出：

- 标量函数 `f: R^3 -> R`：输出 `f(x)`、方向导数 `df(x)[v]`、梯度 `∇f(x)`。
- 向量函数 `g: R^2 -> R^2`：输出雅可比矩阵 `J_g(x)`。

`demo.py` 内固定样例点，无需任何交互输入。

## R04

数学规则（对偶数微分代数）如下：

- 加法：`(a+a'ε)+(b+b'ε)=(a+b)+(a'+b')ε`
- 乘法：`(a+a'ε)(b+b'ε)=ab+(ab'+a'b)ε`
- 除法：`(a+a'ε)/(b+b'ε)=a/b + (a'b-ab')/b^2 * ε`
- 复合函数：`h(u+u'ε)=h(u)+h'(u)u'ε`

因此链式法则并非额外写出来，而是体现在每个算子重载里。

## R05

`demo.py` 中的高层流程：

1. 定义 `Dual(real, dual)` 及四则运算、幂运算。
2. 定义 `sin/cos/exp/log1p/sigmoid` 的对偶版本。
3. 对给定方向 `v`，一次前向传播得到 `f(x)` 与 `df(x)[v]`。
4. 用标准基方向重复 `n` 次得到梯度。
5. 对向量函数重复列种子，拼出雅可比矩阵。
6. 用中心差分与（可选）PyTorch autograd 做数值对照。

## R06

正确性直觉：

- 每个基础运算都满足微分代数规则，等价于局部链式法则。
- 程序由这些基础运算组合而成，因此整体自动满足全局链式法则。
- 方向种子 `v` 放在输入对偶部，就会在线性传播后得到 `df(x)[v]`。
- 以单位基向量做多次前向传播，即得到全部偏导数列。

## R07

时间复杂度：

- 单次方向导数：与原函数一次前向计算同阶，记为 `O(C_f)`。
- 梯度（`n` 维输入，标量输出）：`n` 次前向传播，`O(n * C_f)`。
- 雅可比（`m` 输出、`n` 输入）：通常也是按输入列做 `n` 次前向传播，复杂度 `O(n * C_g)`。

这体现了前向模式更适合“输入维度不大、输出维度较大”的场景。

## R08

空间复杂度：

- 每个标量由 `real + dual` 两个浮点表示，常数开销约为原始计算的两倍量级。
- 对单条计算图做前向传播时，额外空间主要是中间对偶变量缓存，总体与原前向同阶。
- 本实现不构建反向图，因此内存模式比典型反向模式更直接。

## R09

边界与数值注意点：

- 除法需要分母实部非零。
- `log1p(x)` 要求 `x > -1`。
- 对非整数幂，底数若非正可能无定义。
- 方向导数与有限差分比较时，差分步长过小会受舍入误差影响。

`demo.py` 中样例函数已选在安全定义域内。

## R10

与其他求导策略对比：

- 符号微分：可得闭式表达，但易表达式膨胀。
- 有限差分：实现简单，但存在截断误差与步长调参问题。
- 反向模式 AD：对 `R^n -> R` 梯度常更高效。
- 前向模式 AD：实现透明、方向导数自然，适合小中维输入或需要雅可比列的任务。

## R11

实现细节（`demo.py`）：

- `Dual` 类型重载 `+ - * / **`，是链式法则落地核心。
- `forward_directional_derivative`：一次传播给出函数值和方向导数。
- `forward_gradient`：循环单位基向量获得梯度。
- `forward_jacobian`：对向量函数逐列播种，得到雅可比。
- `finite_difference_grad` 与 `torch_gradient_reference` 用于交叉验证正确性。

## R15

常见实现错误：

- 忘记在乘法中使用乘法法则（`ab'+a'b`）。
- 把方向导数当作梯度，未做单位基循环。
- 复合函数只计算实部，不传播对偶部。
- 使用 `log(1+x)` 而非 `log1p` 导致小量精度不稳。

## R16

可扩展方向：

- 把标量对偶数扩展为“向量对偶数”（一次传播拿到整梯度）。
- 支持更多特殊函数（`tanh`, `atan2`, `erf` 等）。
- 与稀疏雅可比结构结合，减少多余方向传播。
- 面向机器学习场景封装为轻量前向 AD 模块。

## R18

`demo.py` 源码级算法流（8 步）：

1. `Dual` 定义数值对 `(real, dual)`，在 `__add__/__mul__/__truediv__/__pow__` 内写入微分传播公式。  
2. `d_sin/d_cos/d_exp/d_log1p/d_sigmoid` 把常见初等函数扩展为对偶版本，保证每个原子算子都返回“值+导数”。
3. `scalar_objective` 以普通表达式组合这些算子；当输入是 `Dual` 时，表达式自动执行链式传播。
4. `forward_directional_derivative` 把输入 `x` 与方向 `v` 配对为 `Dual(x_i, v_i)`，一次前向计算同时得到 `f(x)` 与 `df(x)[v]`。
5. `forward_gradient` 对每个坐标方向 `e_i` 重复第 4 步，收集 `df(x)[e_i]` 形成梯度向量。
6. `forward_jacobian` 对向量函数 `g` 逐列播种：第 `j` 列种子设为 `e_j`，读取每个输出分量的对偶部，拼接为 `J[:, j]`。
7. `finite_difference_grad` 用中心差分给出数值参考；`torch_gradient_reference`（可选）用 PyTorch autograd 给出第三方参考。
8. `main` 打印前向模式结果与误差范数，并用断言检查 `AD` 与差分/解析解（以及可用时的 PyTorch）一致，确认实现正确。
