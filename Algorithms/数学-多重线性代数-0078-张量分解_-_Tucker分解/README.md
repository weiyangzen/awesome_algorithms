## R01

Tucker 分解是把高阶张量 `X` 写成“低维核心张量 + 多个模态因子矩阵”的分解方式：

`X ≈ G ×_1 U^(1) ×_2 U^(2) ... ×_N U^(N)`

其中：
- `X` 是 `N` 阶原张量；
- `G` 是核心张量（尺寸更小，刻画跨模态交互）；
- `U^(n)` 是第 `n` 个模态的因子矩阵（通常列正交）。

它可视为矩阵 SVD 在高阶张量上的推广，适合压缩、降噪、特征抽取与可解释建模。

## R02

本条目 MVP 的目标是实现一个可运行、可验证、非黑盒的 Tucker 分解流程：
- 使用 `numpy` 手写基础张量算子（unfold/fold、n-mode product）；
- 用 HOSVD 提供初始化；
- 用 HOOI（Higher-Order Orthogonal Iteration）做交替优化；
- 在合成低秩三阶张量上输出误差和迭代轨迹。

`demo.py` 不依赖交互输入，直接运行即可得到可复现实验结果。

## R03

数学形式（以三阶张量为例）：
- `X ∈ R^(I1×I2×I3)`
- `U1 ∈ R^(I1×R1), U2 ∈ R^(I2×R2), U3 ∈ R^(I3×R3)`
- `G ∈ R^(R1×R2×R3)`

目标是最小化 Frobenius 误差：

`min ||X - G ×_1 U1 ×_2 U2 ×_3 U3||_F`

在因子矩阵列正交约束下，可用“固定其他因子、更新单个因子”的交替迭代求解；核心张量通过投影得到：

`G = X ×_1 U1^T ×_2 U2^T ×_3 U3^T`

## R04

算法策略：`HOSVD 初始化 + HOOI 精化`。

- HOSVD 初始化：
  对每个模态 `n`，把张量按该模态展开为矩阵 `X_(n)`，取其前 `R_n` 个左奇异向量作为 `U^(n)` 初值。
- HOOI 迭代更新：
  固定其余因子时，把张量投影到其余子空间，再对目标模态做 SVD，更新该模态因子。
- 每轮迭代后重建张量并计算相对误差，按误差改变量阈值停机。

该流程在教学与工程原型中都非常常见，兼顾可解释性与实现简洁度。

## R05

`demo.py` 的核心数据结构：
- `Array = np.ndarray`：统一张量/矩阵类型别名；
- `TuckerResult(core, factors, errors)`：保存分解结果与误差轨迹；
- `factors: List[Array]`：每个模态一个因子矩阵；
- `errors: List[float]`：每轮 HOOI 的相对重构误差。

这种“轻量 dataclass + numpy 数组”的结构足够支撑 MVP，也便于后续扩展。

## R06

正确性直觉：
- 当其它模态固定时，目标模态的最优子空间可由相应投影矩阵的主左奇异向量给出；
- HOOI 正是在各模态上交替做这个“局部最优子空间更新”；
- 因子更新后，核心张量由正交投影直接得到；
- 由于每步都在当前条件下提升或保持拟合质量，误差通常会下降并收敛到局部最优。

因此，HOSVD 给出合理起点，HOOI 进一步精化，通常比单独 HOSVD 有更低重构误差。

## R07

复杂度（`N` 阶张量，维度 `I_n`，秩 `R_n`）：
- 一次模态更新的主成本来自 SVD，约为 `O(I_n * J_n * min(I_n, J_n))`，其中 `J_n = prod_{m!=n} R_m`（在投影后空间）；
- 每轮 HOOI 需要更新全部模态，成本为各模态之和；
- 空间主要用于存储原张量、核心张量和各因子矩阵。

本 MVP 聚焦三阶张量，规模设置为 `(30,25,20)` 与秩 `(3,4,2)`，能在普通 CPU 上快速完成。

## R08

数值与实现细节：
- 使用 `np.linalg.svd(..., full_matrices=False)` 避免不必要的大矩阵；
- 随机正交因子由 QR 生成，提高合成数据稳定性；
- 相对误差使用 `||X-X_hat||/||X||`，便于跨规模比较；
- 停机条件用“相邻两轮误差差值”控制，避免固定轮数过长。

这套设置优先保证结果稳定和代码可读。

## R09

MVP 取舍：
- 仅依赖 `numpy`，不引入 `tensorly` 等高级张量库；
- 不调用黑盒 `tucker()` API，关键算子全部手写；
- 只实现三阶示例数据生成，但 `hooi` 本身支持任意阶（通过 `x.ndim`）；
- 不做 GPU/并行优化，强调算法主干可验证。

目标是“最小可行 + 算法透明”。

## R10

`demo.py` 函数职责：
- `unfold` / `fold`：模态展开与逆展开；
- `n_mode_product`：实现 `×_n` 张量-矩阵乘；
- `hosvd_init`：生成初始化因子；
- `project_to_core`：把原张量投影到因子子空间；
- `reconstruct`：由核心和因子重建张量；
- `hooi`：执行交替迭代优化；
- `build_synthetic_tensor`：构造低秩 + 噪声测试样本；
- `main`：串联完整实验并打印结果。

## R12

输出字段解释：
- `Tensor shape`：观测张量尺寸；
- `Target ranks`：Tucker 目标秩；
- `HOOI iterations`：实际迭代轮数；
- `HOSVD init error`：仅初始化分解的相对误差；
- `HOOI final error`：HOOI 精化后的相对误差；
- `iter k: ...`：第 `k` 轮 HOOI 后误差。

通常会看到 `HOOI final error <= HOSVD init error`。

## R13

预期现象：
- 在观测张量误差上，HOOI 会比 HOSVD 更低或持平；
- 在“干净低秩真值”误差上，HOOI 往往同样更优；
- 误差轨迹前几轮下降较快，后续趋于平稳，最后触发阈值停机。

如果秩设置偏小，误差会停在较高水平；若秩过大，可能过拟合噪声。

## R14

常见坑：
- `unfold/fold` 维度顺序写错，导致重建形状正确但数值错误；
- `n_mode_product` 的矩阵乘维度方向写反（应为 `(J, I_mode) @ (I_mode, -1)`）；
- 忽略 rank 合法性检查（`rank > dimension` 会崩溃）；
- 把“对观测误差下降”误解成“对真实信号一定更好”，在高噪声下需谨慎。

代码中已加入基本参数检查和统一误差评估。

## R15

与相关分解方法比较：
- 与 CP 分解：
  CP 约束更强（rank-1 分量求和），参数更少但优化常更敏感；Tucker 更灵活、表达力更强。
- 与矩阵 SVD：
  SVD 处理二阶数据；Tucker 直接保留多模态结构，避免粗暴展平带来的信息混叠。
- 与纯 HOSVD：
  HOSVD 是一次性谱截断；HOOI 在其基础上迭代精化，通常误差更低。

## R16

适用场景：
- 多维传感数据压缩（时间×空间×通道）；
- 视频/超光谱数据降维与降噪；
- 推荐与知识图谱中的多关系张量建模；
- 需要“可解释低秩结构”的科研原型。

当数据天然是高阶结构时，Tucker 通常优于先展平再做矩阵方法。

## R17

可扩展方向：
- 加入缺失值掩码，做加权/不完整张量分解；
- 替换全 SVD 为 randomized SVD 提升大规模效率；
- 引入非负约束，构造 NTD（Nonnegative Tucker Decomposition）；
- 增加验证集误差监控，减少噪声过拟合；
- 支持 PyTorch 后端以利用 GPU。

本 MVP 的函数边界已为这些扩展预留了清晰接口。

## R18

`demo.py` 的源码级算法流程（9 步）：
1. `main` 调用 `build_synthetic_tensor`：先随机生成正交因子与小核心，再重建出低秩干净张量并叠加高斯噪声。  
2. `hosvd_init` 对每个模态执行 `unfold -> SVD -> 截断 U`，得到初始因子矩阵列表。  
3. `project_to_core` 用 `n_mode_product(..., U^T, mode)` 逐模态投影原张量，得到初始化核心张量。  
4. `reconstruct` 用 `n_mode_product(..., U, mode)` 把核心映射回原空间，计算 HOSVD 初始误差。  
5. `hooi` 进入迭代：固定当前因子后，针对每个模态 `mode`，先把其余模态全部投影到低维子空间。  
6. 对该模态的投影张量做 `unfold`，再取主左奇异向量，更新该模态因子矩阵。  
7. 一轮模态更新结束后，再次 `project_to_core + reconstruct`，计算当前相对 Frobenius 误差并追加到 `errors`。  
8. 若相邻两轮误差差值小于 `tol`，提前停止；否则继续下一轮直到 `max_iters`。  
9. `main` 打印 HOSVD 与 HOOI 的误差对比及逐轮误差轨迹，形成可复现实验结论。
