# KMP算法

## R01

KMP（Knuth-Morris-Pratt）算法用于在主串 `text` 中查找模式串 `pattern` 的出现位置。其核心思想是：
- 失配时不回退主串指针；
- 利用模式串自身的前后缀重合信息（LPS / 前缀函数）决定模式串回退位置。

因此 KMP 把朴素匹配在最坏情况下的 `O(nm)` 优化为 `O(n+m)`。

## R02

本条目 MVP 的问题定义：
- 输入：字符串 `text` 与 `pattern`；
- 输出：`pattern` 在 `text` 中的全部匹配起点下标（支持重叠匹配）；
- 约定：空模式串匹配到 `0..len(text)` 的所有位置。

`demo.py` 提供确定性样例与随机回归测试，确保实现正确、可复现。

## R03

关键数学/字符串对象是 `lps`（longest proper prefix which is also suffix）数组：
- `lps[i]` 表示 `pattern[0:i+1]` 这个前缀中，最长“真前缀=后缀”的长度；
- 例如 `pattern="ababaca"` 时，`lps=[0,0,1,2,3,0,1]`。

该数组让失配时可直接跳到下一个候选位置，而不是从头比较。

## R04

KMP 主流程：
1. 预处理 `pattern`，构造 `lps`；
2. 设主串指针 `i`、模式串指针 `j`；
3. 若 `text[i]==pattern[j]`，同时右移；
4. 若 `j==m`，记录一次匹配，并将 `j` 回退到 `lps[j-1]` 继续找重叠匹配；
5. 若失配且 `j>0`，令 `j=lps[j-1]`；
6. 若失配且 `j==0`，仅移动 `i`。

整个过程每个字符最多被常数次处理。

## R05

时间复杂度：
- 构建 `lps`：`O(m)`；
- 匹配扫描：`O(n)`；
- 总体：`O(n+m)`。

其中 `n=len(text)`，`m=len(pattern)`。

## R06

空间复杂度：
- 主要额外空间是 `lps` 数组，大小 `m`；
- 其余为常数个指针与结果列表。

故额外空间为 `O(m)`（不计输出结果本身）。

## R07

正确性直觉（不回退主串仍不漏解）：
- `lps` 保证失配后跳转到“已匹配前缀的最长可复用边界”；
- 被跳过的位置在逻辑上已被证明不可能成为新匹配起点；
- 每次成功匹配后回退到 `lps[m-1]`，能继续发现重叠出现。

因此算法既不重复无效比较，也不会丢失合法匹配。

## R08

典型应用：
- 文本检索、日志关键字定位；
- DNA/蛋白序列模式匹配；
- 代码分析中的词法片段搜索；
- 作为更复杂字符串算法（如自动机方法）的基础模块。

## R09

适用条件与局限：
- 适合单模式串精确匹配；
- 对 Unicode 字符串同样有效（按 Python 字符比较）；
- 若需要多模式同时匹配，Aho-Corasick 往往更优；
- 若需要模糊匹配/编辑距离，KMP 不直接适用。

## R12

`demo.py` 函数分工：
- `build_lps(pattern)`：线性构建前缀表；
- `kmp_search_all(text, pattern)`：返回所有匹配起点；
- `naive_search_all(text, pattern)`：朴素匹配基线；
- `run_deterministic_cases()`：固定案例断言；
- `run_randomized_regression()`：随机回归测试。

`main()` 串联这些步骤并输出可读结果。

若任一断言失败，脚本会抛出异常并中止。

## R14

常见错误模式：
- `lps` 回退写错导致死循环或漏匹配；
- 成功匹配后将 `j` 清零，导致重叠匹配丢失；
- 未定义空模式串语义，造成边界行为不一致；
- 误把字符下标与字节偏移混用。

MVP 通过基线对照与随机测试覆盖这些风险点。

## R15

与相邻方法对比：
- 朴素匹配：实现最简单，但最坏 `O(nm)`；
- Rabin-Karp：平均快、支持多模式哈希，但存在碰撞处理问题；
- Boyer-Moore：在大字符集英文文本中常很快，但实现更复杂；
- KMP：最坏复杂度有严格上界，且实现相对稳定。

## R16

本 MVP 的实验设计：
- 固定样例覆盖：单次命中、无命中、重叠命中、Unicode、空串；
- 随机样例覆盖：`text` 长度 `0..40`、`pattern` 长度 `0..8`；
- 每个随机样例均与朴素结果逐一比对。

这样可在小成本下获得较高实现可信度。

## R18

`demo.py` 源码级算法流程拆解（8 步）：
1. `main()` 定义演示模式串，先调用 `build_lps` 打印前缀表。  
2. `build_lps` 从左到右扫描 `pattern`，维护当前边界长度 `length`。  
3. 若 `pattern[i]==pattern[length]`，边界扩展并写入 `lps[i]`；否则按 `lps[length-1]` 回退直到可扩展或归零。  
4. `kmp_search_all` 初始化主串指针 `i` 与模式串指针 `j`，并处理空模式串特例。  
5. 匹配成功时同步推进 `i,j`；当 `j==m` 时记录起点 `i-m`，再令 `j=lps[j-1]` 支持重叠匹配。  
6. 匹配失配时，若 `j>0` 则回退 `j=lps[j-1]`，否则仅推进 `i`，保证主串不回退。  
7. `run_deterministic_cases` 与 `run_randomized_regression` 分别对固定样例和随机样例执行 `KMP` vs `Naive` 对照断言。  
8. 全部断言通过后打印 `All KMP checks passed.`，形成可复现验收闭环。  

说明：本实现未调用第三方字符串匹配黑盒 API，KMP 的构表与匹配细节均在源码中显式实现。
