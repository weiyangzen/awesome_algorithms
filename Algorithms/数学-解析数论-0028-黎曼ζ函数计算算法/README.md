# 黎曼ζ函数计算算法

- UID: `MATH-0028`
- 学科: `数学`
- 分类: `解析数论`
- 源序号: `28`
- 目标目录: `Algorithms/数学-解析数论-0028-黎曼ζ函数计算算法`

## R01

问题定义：实现一个可运行的黎曼 `ζ(s)` 数值计算 MVP，并给出临界线 `Re(s)=1/2` 上非平凡零点的近似搜索结果。

## R02

数学背景：
- 黎曼ζ函数在 `s=1` 有极点，在负偶整数有平凡零点，在临界带 `0<Re(s)<1` 的零点称为非平凡零点。
- 非平凡零点分布与素数分布深度耦合，是解析数论核心问题。

## R03

输入输出约定（本目录 MVP）：
- 输入：无交互输入，`demo.py` 内置测试点与搜索区间。
- 输出：
  - 多个已知 `ζ(s)` 值的误差对照；
  - 前若干个临界线非平凡零点的近似 `t` 值与残差 `|ζ(1/2+it)|`。

## R04

基线方法与瓶颈：
- 直接用狄利克雷级数 `ζ(s)=Σ 1/n^s` 仅在 `Re(s)>1` 收敛，无法直接覆盖临界带。
- 若依赖外部黑盒 special function 虽实现快，但算法细节不可审计，不符合本任务“可追踪”要求。

## R05

MVP 选型：采用 **Hasse 级数 + 临界线极小值搜索**。
- 先计算 Dirichlet eta 函数 `η(s)` 的全局收敛表示；
- 再由 `ζ(s)=η(s)/(1-2^(1-s))`（`s!=1`）恢复 `ζ(s)`；
- 对 `|ζ(1/2+it)|` 进行粗扫与黄金分割细化，得到零点近似。

## R06

核心公式：
- Hasse 级数（用于 `η(s)`）：
  `η(s)=Σ_{n>=0} (1/2^{n+1}) Σ_{k=0..n} (-1)^k C(n,k)/(k+1)^s`
- 恢复关系：`ζ(s)=η(s)/(1-2^(1-s))`，避开 `s=1` 极点。

## R07

正确性要点：
- `η(s)` 的 Hasse 表示在复平面上（除恢复分母奇异点外）数值上稳定可用；
- 通过已知常数值（如 `ζ(2)=π^2/6`、`ζ(-1)=-1/12`）检验计算链条；
- 非平凡零点验证采用“参考零点 `t` + 残差 `|ζ|`”双指标，而非仅看单一坐标接近。

## R08

复杂度（定性）：
- 单次 `ζ(s)` 评估：双层求和，时间 `O(N^2)`，空间 `O(N^2)`（预存二项式）+ `O(N)`（幂缓存），`N=n_terms`。
- 零点搜索：若粗扫点数为 `M`、候选细化次数为 `Q`，总体约 `O((M+Q)N^2)`。

## R09

实现结构（`demo.py`）：
- `golden_section_minimize`：一维区间极小值搜索。
- `RiemannZetaHasse.eta`：Hasse 级数计算 `η(s)`。
- `RiemannZetaHasse.zeta`：由 `η` 恢复 `ζ`，并做缓存。
- `find_critical_line_zeros`：粗扫局部极小值 + 黄金分割细化。
- `main`：运行数值校验与零点搜索展示。

## R10

工具栈说明：
- 仅使用 Python 标准库（`cmath`、`math`、`dataclasses`、`typing`）。
- 不依赖 SciPy/mpmath 等特殊函数库，保证目录自包含、可直接运行。

## R11

与“黑盒库调用”边界：
- 未调用 `scipy.special.zeta`、`mpmath.zeta` 等现成黑盒函数。
- 代码显式展开：二项式系数构造、Hasse 双层求和、`η→ζ` 转换、临界线搜索流程，便于逐步审计。

## R14

工程取舍：
- 该 MVP 选择“公式可解释 + 代码短小”优先，牺牲了超大规模高精度场景的极致性能。
- 相比直接接黑盒 special function，当前实现更适合作为算法教学、审计与后续扩展基础。

## R15

风险与边界：
- `n_terms` 过小会导致临界带精度下降，零点定位漂移。
- 搜索过程依赖局部极小值，若扫描步长过粗可能漏检候选。
- `s=1` 为极点，代码显式抛错；靠近恢复分母奇异点时需要更高精度策略。

## R16

可验证性：
- `demo.py` 同时做两层验证：
  1) 实轴闭式参考值误差阈值；
  2) 临界线已知零点坐标偏差 `|Δt|` 与残差 `|ζ|` 阈值。
- 验证通过与 R08 的复杂度设计一致：在中等 `N` 下即可获得稳定、可复现实验结果。

## R17

可扩展方向：
- 在保持 Hasse 主流程不变的前提下，引入自适应 `n_terms` 与误差估计；
- 将“粗扫+细化”升级为符号变化辅助或 Argument Principle 计数，提高零点搜索完备性；
- 用并行批处理复用缓存，支持更大区间零点扫描。

## R18

源码级算法流程（非黑盒，8 步）：
1. 预计算 `log(k+1)` 与二项式系数行 `C(n,k)`，作为 Hasse 级数基础表。
2. 对目标复数 `s`，构造幂项 `(k+1)^(-s)=exp(-s*log(k+1))`。
3. 按 `Σ (-1)^k C(n,k)/(k+1)^s` 计算每个 `n` 的内层和。
4. 按权重 `1/2^(n+1)` 累加内层和，得到 `η(s)` 的近似值。
5. 使用 `ζ(s)=η(s)/(1-2^(1-s))` 恢复 `ζ(s)`，并处理 `s=1` 极点与分母近零异常。
6. 在临界线选取粗网格 `t`，批量评估 `|ζ(1/2+it)|`，筛出局部极小值候选区间。
7. 对每个候选区间运行黄金分割搜索，细化得到 `t*` 与最小残差 `|ζ(1/2+it*)|`。
8. 对细化结果做去重、排序并与已知零点对照，输出近似零点与误差报告。
