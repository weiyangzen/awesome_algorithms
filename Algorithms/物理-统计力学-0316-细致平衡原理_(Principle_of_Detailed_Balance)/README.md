## R01

细致平衡原理描述平衡态下“逐跃迁通量对消”的条件。对离散状态马尔可夫过程，若平稳分布为 `pi`、转移核为 `P`，则对任意状态对 `(i, j)` 有：

`pi_i * P_ij = pi_j * P_ji`

它比“整体平稳”`pi P = pi`更强：不仅总流入=总流出，而且每一对互逆跃迁都精确对消，因此不会存在环流概率流。

## R02

本条目 MVP 的目标是做一个可运行、可审计的最小例子来验证细致平衡，而不是只停留在公式陈述：

- 构造一个具有玻尔兹曼目标分布的离散能级系统；
- 用对称提议 + Metropolis 接受率建立转移核；
- 在解析层面验证 `pi_i P_ij = pi_j P_ji`；
- 在 Monte Carlo 样本层面验证经验通量也近似满足细致平衡。

## R03

采用有限状态空间 `S={0,...,N-1}`，每个状态有能量 `E_i`，给定逆温 `beta`，目标分布为：

`pi_i = exp(-beta*E_i) / Z`

其中 `Z = sum_k exp(-beta*E_k)`。

提议机制采用“环形邻居”对称随机游走：

- `q(i->j)=1/2`（若 `j` 是 `i` 的左/右邻居）；
- 否则 `q(i->j)=0`。

接受率采用 Metropolis 规则：

`a(i->j)=min(1, exp(-beta*(E_j-E_i)))`

于是 `P_ij = q(i->j)*a(i->j)`（`i!=j`），`P_ii = 1 - sum_{j!=i} P_ij`。

## R04

实现分两条验证链路：

1. 解析链路：直接从 `(E, beta)` 构造 `pi` 与 `P`，计算细致平衡残差矩阵 `|pi_iP_ij-pi_jP_ji|`；
2. 采样链路：运行长马尔可夫链，统计转移计数 `C_ij` 与状态访问频率；
3. 由 `C_ij` 行归一化得到经验转移核 `P_hat`，由访问频率得到 `pi_hat`；
4. 计算经验残差 `|pi_hat_i P_hat_ij - pi_hat_j P_hat_ji|`；
5. 同时检查 `pi` 的平稳性残差 `|piP-pi|` 与 `||pi_hat-pi||_1`。

## R10

正确性直觉：

1. 对称提议 `q(i->j)=q(j->i)` 让方向差异只来自能量差；
2. Metropolis 接受率通过 `min(1, exp(-beta*DeltaE))` 精确补偿能量不对称；
3. 因而每对互逆边上概率通量天然可对消，得到细致平衡；
4. 细致平衡成立则必然平稳，所以 `piP=pi` 也应同时成立；
5. 有限步采样只会产生统计波动，因此经验残差应“小但非零”。

## R11

数值稳健性措施：

- 计算 `exp(-beta*E)` 前先做能量平移 `E - min(E)`，避免指数下溢；
- 设置 `burn_in` 降低初始状态偏置；
- 用足够长链长（默认 300000）降低经验噪声；
- 同时检查多个指标（通量残差、平稳残差、`L1` 误差），避免单指标误判。

## R14

常见失效模式：

1. 提议分布不对称却仍沿用对称场景接受率，导致不可逆；
2. 忘记把拒绝提议计入 `i->i` 自环转移，统计核行和不为 1；
3. 能量符号写反（把 `E_j-E_i` 写成 `E_i-E_j`），分布会偏向高能态；
4. 采样步数过短导致经验残差较大，被误判为原理失效；
5. 只检查 `piP=pi` 而不检查成对通量，无法区分“平稳但有环流”的情况。

## R18

`demo.py` 源码级算法流程（8 步）：

1. `main` 创建 `DetailedBalanceConfig`，调用 `run_detailed_balance_demo`。  
2. `run_detailed_balance_demo` 先调用 `boltzmann_distribution` 计算理论分布 `pi`。  
3. 调用 `build_metropolis_kernel`：在环形邻居图上按“对称提议 + Metropolis 接受率”逐项构造转移矩阵 `P`。  
4. 调用 `detailed_balance_residual(pi, P)` 计算理论细致平衡残差，并计算 `piP-pi` 的平稳残差。  
5. 调用 `simulate_chain` 执行 Monte Carlo 采样：每一步提议邻居、按接受率判定、记录 `transition_counts` 与访问频率。  
6. 通过 `row_normalize_counts` 把计数转成经验核 `P_hat`，结合经验分布 `pi_hat` 再次调用 `detailed_balance_residual` 得到经验残差。  
7. 汇总 `summary/state_df/db_df` 三个 `pandas.DataFrame`，分别展示全局指标、状态概率对比、边通量对比。  
8. `main` 打印表格并执行断言（理论残差、经验残差、`L1` 误差、接受率范围），全部通过后输出 `All checks passed.`。

第三方库边界说明：`numpy` 仅用于数值与随机数，`pandas` 仅用于结果展示；核心算法逻辑（转移核构造、采样、细致平衡残差计算）均在源码中显式实现，不依赖黑盒 MCMC 工具。
