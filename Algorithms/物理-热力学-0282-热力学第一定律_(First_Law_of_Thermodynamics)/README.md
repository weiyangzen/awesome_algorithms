## R01

热力学第一定律（First Law of Thermodynamics）是能量守恒在热过程中的表述。  
对封闭系统，采用本条目中的符号约定：

- `Q_in > 0`：外界向系统输入热量；
- `W_by > 0`：系统对外做功。

则第一定律写为：

`delta_U = Q_in - W_by`。

其中 `delta_U` 是系统内能变化。该式说明：系统吸收的热量，一部分转化为内能增加，另一部分以功的形式输出。

## R02

工程与教学意义：

- 给出热-功-内能三者的统一记账规则，避免符号混乱；
- 是热机、制冷、化工能量平衡、动力系统建模的基础约束；
- 能直接变成代码中的可验证守恒关系（残差应接近 0）。

本 MVP 目标是“把第一定律落成可运行且可断言验证的最小脚本”，而不是复杂传热/流动耦合仿真。

## R03

`demo.py` 输入输出（无交互，参数写在脚本里）：

- 输入：
1. 初始状态 `ThermoState(t, v, n_mol)`；
2. 常数摩尔定容热容 `cv_molar`；
3. 两条到同一终点的过程路径：
   `A = 等容升温 -> 等压升温 -> 等温压缩`；
   `B = 等容升温 -> 等温压缩`。
- 输出：
1. 分段能量表（每段 `Q_in/W_by/delta_U/residual`）；
2. 路径总能量表（总热量、总功、总内能变化、总残差）；
3. 自动断言结果，最后打印 `All checks passed.`。

## R04

MVP 使用的核心公式（理想气体，封闭系统）：

1. 状态方程：`pV = nRT`；
2. 第一定律：`delta_U = Q_in - W_by`；
3. 理想气体内能（`Cv` 近似常数）：`delta_U = n * Cv * delta_T`；
4. 等容过程：`W_by = 0`，故 `Q_in = delta_U`；
5. 等压过程：`W_by = p * (V_out - V_in)`；
6. 等温过程（理想气体）：`delta_U = 0`，`W_by = nRT ln(V_out/V_in)`，故 `Q_in = W_by`。

脚本还用 `numpy` 数值积分 `∫p(V)dV`（`np.trapezoid`）与解析式对照，验证等温功计算。

## R05

复杂度（单次脚本运行）：

- 过程段计算为常数次代数运算；
- 数值积分主耗时来自 `n_steps` 个网格点；
- 时间复杂度：`O(n_steps)`；
- 空间复杂度：`O(n_steps)`（用于存储积分网格）。

若不做数值积分对照，仅解析计算则为 `O(1)` 时间、`O(1)` 空间。

## R06

`demo.py` 的最小闭环：

- 闭环 A（过程求解）：
  `step_isochoric/step_isobaric/step_isothermal` 显式计算每段 `Q/W/delta_U`。
- 闭环 B（守恒校验）：
  `ProcessResult.residual` 检查 `delta_U - (Q_in - W_by)`。
- 闭环 C（物理含义校验）：
  两条不同路径到同一终点，验证 `delta_U` 路径无关，而 `Q/W` 路径相关。

## R07

优点：

- 公式透明，没有黑盒热力学求解器；
- 分段、总量、残差三层输出，便于审计；
- 用“同起终点双路径”直接展示状态函数与路径函数差异。

局限：

- 仅适用于理想气体和封闭系统；
- `Cv` 视为常数，未考虑温度依赖；
- 未建模不可逆损失（摩擦、有限温差传热、泄漏）。

## R10

正确性直觉：

1. 每个过程段都按物理定义直接给出 `Q_in`、`W_by`、`delta_U`；
2. 对等容过程，体积不变所以 `W_by=0`，自然得到 `Q_in=delta_U`；
3. 对等温理想气体，内能只与温度有关，所以 `delta_U=0`，自然得到 `Q_in=W_by`；
4. 把段结果相加，相当于对整条路径做能量守恒；
5. 对同一起终点，`delta_U` 只由 `delta_T` 决定，因此两路径总 `delta_U` 一致；
6. 但路径不同导致积分功不同，因此总 `Q` 与总 `W` 可以不同，这与第一定律兼容。

## R11

数值稳定性策略：

- 对温度、体积、物质的量、`Cv` 做正值检查；
- 使用 `float64` 计算；
- 等式比较均用容差断言（`tol=1e-9`）；
- 等温功额外做解析式 vs `numpy.trapezoid` 的交叉验证，降低公式抄写错误风险。

## R12

关键参数与影响：

- `cv_molar`：决定同一 `delta_T` 下 `delta_U` 的幅值；
- `n_mol`：线性缩放热量、功和内能变化；
- `t_out`（各段目标温度）：直接影响 `delta_U` 与部分功项；
- `v_out`（等温段终体积）：通过 `ln(V_out/V_in)` 控制等温功；
- `n_steps`（积分网格）：越大则数值积分越精细，但耗时更高。

调参建议：

- 想看“路径差异”可固定起终点，改变中间段设计；
- 想看“能量量级变化”可增大 `n_mol` 或温升；
- 想看“数值误差变化”可调整 `n_steps`。

## R14

常见失效模式：

1. 符号约定混用（把 `W_on` 和 `W_by` 搞反）；
2. 温度输入为摄氏度而非开尔文；
3. 体积取非正值导致 `ln(V_out/V_in)` 无意义；
4. 在等压段误用初末压力不一致的计算；
5. 把理想气体结论直接外推到高压/相变工况。

## R18

`demo.py` 源码级算法流程（8 步）：

1. `main` 设置常数与初态，定义 `cv_molar` 和初始 `ThermoState`。  
2. `build_path_a` / `build_path_b` 分别构造两条过程路径，调用分段函数得到 `ProcessResult` 列表。  
3. 每个分段函数（等容/等压/等温）都先做输入合法性检查，再按对应物理公式计算 `Q_in`、`W_by`、`delta_U`。  
4. `ProcessResult.residual` 统一计算 `delta_U - (Q_in - W_by)`，作为第一定律局部残差。  
5. `path_to_frame` 把分段结果转成表格，包含起终温度、起终体积、能量项和残差。  
6. `summarize_path` 对每条路径求和得到 `Q_total/W_total/delta_U_total` 与总残差。  
7. `main` 做四组断言：段级守恒、路径级守恒、同终点路径 `delta_U` 一致、不同路径 `Q/W` 不同。  
8. 对每个等温段调用 `numerical_isothermal_work`（`numpy.trapezoid` 积分）与解析式对照，最终打印 `All checks passed.`。  

说明：脚本只使用 `numpy/pandas` 做数值与展示，第一定律与各过程公式都在源码中逐项展开，没有第三方黑盒热工求解器。
