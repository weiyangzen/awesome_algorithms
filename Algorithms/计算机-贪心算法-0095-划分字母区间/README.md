# 划分字母区间

- UID: `CS-0075`

- 分类: `贪心算法`

- 目标目录: `Algorithms/计算机-贪心算法-0095-划分字母区间`

## R01

“划分字母区间”（Partition Labels）要求把字符串切成尽可能多的连续片段，并满足：每个字符最多只出现在一个片段中。它是经典的一次扫描贪心问题。

## R02

给定字符串 `s`，输出一个整数数组 `ans`，其中每个元素是一个片段长度，满足：

- 所有片段顺序拼接后恰好等于 `s`。
- 对任意字符 `c`，`c` 的所有出现位置都落在同一个片段里。
- 在满足上面约束前提下，片段数量最大。

## R03

核心贪心观察：

- 若我们知道每个字符在 `s` 中的最后出现位置 `last[c]`，就能在扫描时维护“当前片段最远必须覆盖到哪里”。
- 扫描到位置 `i` 时，把当前片段右边界更新为 `end = max(end, last[s[i]])`。
- 只有当 `i == end` 时，当前片段内所有字符都已经“闭合”，此处切分才合法。

## R04

算法流程（两趟/一趟半）：

1. 预处理 `last`：记录每个字符最后出现的下标。
2. 线性扫描字符串，维护 `start`（当前片段起点）和 `end`（当前片段必须到达的最远终点）。
3. 每读到一个字符，更新 `end`。
4. 当扫描指针 `i` 到达 `end`，立即切分，长度为 `end - start + 1`，并令 `start = i + 1` 进入下一个片段。

## R05

本目录 MVP 的实现约定：

- 使用 Python 标准库实现，不依赖第三方黑盒函数。
- 支持任意 Python `str` 字符（不限定只含 26 个小写字母）。
- 空字符串返回空数组 `[]`。
- `demo.py` 内置固定样例与随机对拍，运行时不需要任何交互输入。

## R06

伪代码：

```text
function partition_labels(s):
    last = map from char -> last index in s
    ans = []
    start = 0
    end = 0

    for i from 0 to len(s)-1:
        end = max(end, last[s[i]])
        if i == end:
            ans.append(end - start + 1)
            start = i + 1

    return ans
```

## R07

正确性直觉（贪心不变量）：

- 不变量 1：在当前片段扫描过程中，`end` 始终等于“该片段内已见字符的最后出现位置最大值”。
- 推论 1：当 `i < end` 时，若强行切分，一定会把某个字符拆到后续片段，违反约束，因此不能提前切。
- 推论 2：当 `i == end` 时，当前片段内所有字符的全部出现都已包含在 `[start, end]`，此时切分合法。
- 贪心最优性：每次都在“最早合法位置”切分，才能给后续留出最大切分空间，因此最终片段数最大。

## R08

复杂度：

- 时间复杂度：`O(n)`，`n` 为字符串长度（一次建 `last` + 一次扫描）。
- 空间复杂度：`O(k)`，`k` 为字符串中不同字符数（`last` 哈希表）。

## R09

工程实现注意点：

- 下标与长度换算要用 `end - start + 1`。
- `start` 必须在切分后更新为 `i + 1`。
- 若字符串很长，算法依旧线性；瓶颈主要在字符访问和哈希查表。
- 该实现对 Unicode 字符同样成立，因为字典键是通用 `str`。

## R10

典型样例：`s = "ababcbacadefegdehijhklij"`

- 扫描前半段会把 `end` 推到 `8`，因此第一段长度 `9`（`ababcbaca`）。
- 第二段从 `9` 开始，最远闭合到 `15`，长度 `7`（`defegde`）。
- 第三段长度 `8`（`hijhklij`）。

输出：`[9, 7, 8]`。

## R11

边界条件：

- `s = ""`：返回 `[]`。
- `s = "a"`：返回 `[1]`。
- 所有字符都相同（如 `"aaaa"`）：返回 `[4]`。
- 所有字符都互异（如 `"abcd"`）：返回 `[1, 1, 1, 1]`。

## R12

常见错误：

- 错把“字符第一次出现位置”当作边界依据，导致错误切分。
- 在 `i == end` 时忘记追加结果或忘记重置 `start`。
- 用固定 26 长度数组但输入并非仅小写字母，导致索引逻辑不安全。
- 仅做样例测试，未覆盖随机短串与边界串。

## R13

`demo.py` 结构：

- `partition_labels`：`O(n)` 贪心主算法。
- `lengths_to_segments`：把长度数组映射为实际切片，便于展示。
- `is_valid_partition`：校验“每个字符只出现在一个片段”。
- `brute_force_max_parts`：小规模暴力求最大可切片段数，用于对拍。
- `run_known_examples` / `run_random_verification`：固定用例与随机验证。
- `main`：串联执行并输出结果。

## R18

`demo.py` 的源码级算法流（7 步）：

1. `main` 先调用 `run_known_examples()`，对固定字符串逐一执行 `partition_labels`。  
2. `partition_labels` 首先构建 `last = {ch: idx}`（每个字符最后出现下标）。  
3. 线性扫描 `s`，对每个位置执行 `end = max(end, last[ch])`，动态维护当前片段最远右边界。  
4. 当 `i == end` 时，把 `end - start + 1` 加入结果，并更新 `start = i + 1`。  
5. `run_known_examples` 用 `is_valid_partition` 检查合法性，并用 `lengths_to_segments` 打印对应文本片段。  
6. `run_random_verification` 生成小规模随机字符串；对每个字符串，用 `brute_force_max_parts` 穷举切分上限，并断言贪心结果片段数等于该上限。  
7. 所有断言通过后输出 `All checks passed.`，完成“算法实现 + 正确性对拍”的闭环。  

这 7 步把“最后位置统计 -> 边界扩张 -> 到点切分 -> 暴力对拍验证”完整展开，算法流程可直接在源码中逐行追踪。
