## R01

Oslo算法用于 **B样条在节点细化（knot refinement）时的系数转换**。给定旧节点向量与新节点向量（新向量是旧向量的细化，包含更多节点），它构造一个稀疏变换，使得：
- 曲线几何形状保持不变；
- 仅控制点表示发生变化；
- 新控制点可由旧控制点线性变换得到。

它是CAD/CAM、几何建模与等几何分析中非常经典的“重参数表示转换”算法。

## R02

历史与脉络可简述为：
- de Boor / Cox 的B样条递推奠定了稳定计算基础；
- Boehm节点插入给出了“单个节点插入时控制点如何更新”；
- Oslo算法把“多节点细化”组织成统一、可批量执行的系数计算框架。

工程上，Oslo算法常被看作“多次节点插入的系统化版本”，核心价值是把局部插值权重变成可复用的变换结构。

## R03

设B样条次数为 `p`，旧节点向量为 `U`，旧控制点为 `P_i`，细化后节点向量为 `V`（满足 `U ⊆ V`，按重数计）。

曲线表示：
- 旧表示：`C(u) = Σ_i P_i N_{i,p}(u; U)`
- 新表示：`C(u) = Σ_j Q_j N_{j,p}(u; V)`

Oslo目标是求 `Q = T P`（`T` 为细化变换矩阵）。

本MVP采用与Oslo等价的构造法：重复执行单节点插入。单次插入 `ξ`（位于 `U[k] <= ξ < U[k+1]`）时，新控制点 `Q_i` 为：
- `Q_i = P_i`，`i = 0..k-p`
- `Q_i = (1-α_i)P_{i-1} + α_i P_i`，`i = k-p+1..k`
- `Q_i = P_{i-1}`，`i = k+1..n+1`

其中 `α_i = (ξ - U_i) / (U_{i+p} - U_i)`（分母为0时按0处理）。该更新是局部凸组合。

## R04

输入输出定义：
- 输入：
  - 次数 `p`
  - 粗节点向量 `U`
  - 目标细化节点向量 `V`
  - 粗控制点矩阵 `P`（`n x d`）
- 输出：
  - 细化控制点 `Q`
  - 变换矩阵 `T`（满足 `Q = T @ P`）
  - 几何不变性验证指标（细化前后曲线采样误差）

MVP选择二维控制点，便于可读验证。

## R05

核心思想：
- 细化不改变曲线，只改变基函数展开坐标；
- 每次插入只影响局部 `p+1` 个控制点，故矩阵天然稀疏；
- 多次插入可连乘得到总变换：`T = A_r ... A_2 A_1`。

因此Oslo类方法兼顾“几何精确保持”和“局部、稳定、可组合”的数值特性。

## R06

标准计算流程：
1. 校验 `V` 是 `U` 的细化（多重集包含关系）。
2. 求出需插入的新增节点序列（含重数）。
3. 初始化 `T = I`。
4. 对每个新增节点 `ξ`：
   - 计算单次插入矩阵 `A`；
   - 更新 `T <- A @ T`；
   - 更新当前节点向量。
5. 得到最终 `Q = T @ P`。
6. 在采样点上比较旧表示与新表示，验证几何误差应接近机器精度。

## R07

伪代码（Oslo等价构造版本）：

```text
function oslo_refine(U, V, p, P):
    extras = multiset_difference(V, U)
    T = I(size(P,0))
    U_cur = U

    for xi in extras:
        U_cur, A = single_insert_matrix(U_cur, p, xi)
        T = A @ T

    Q = T @ P
    return U_cur, T, Q
```

单次插入矩阵 `A` 的每行最多2个非零（边界行为1个），这就是稀疏局部性的来源。

## R08

正确性直觉：
- 单次节点插入是经典结论：插入后曲线几何完全不变；
- Oslo多节点细化可视为多次“几何不变操作”的复合；
- 几何不变操作复合后仍几何不变。

因此，只要每次插入公式正确、节点顺序一致，最终 `Q` 与原曲线严格等价（仅浮点误差存在）。

## R09

复杂度（`n` 为旧控制点数，`r` 为新增节点数，`d` 为控制点维度）：
- 变换构造：约 `O(r * n)`（每步插入矩阵与累乘）。
- 控制点更新：`O((n+r) * n * d)` 若按稠密乘法实现；
  - 若利用稀疏结构可降至近线性局部更新。
- 曲线采样验证：`O(M * n * p)`（`M` 为采样点数，显式基函数求值）。

MVP偏教学可读性，采用直观实现。

## R10

数值稳定性与实现细节：
- 插值权重是凸组合，通常数值稳定；
- 当 `U_{i+p} - U_i` 很小或为0（重复节点）时，需安全分支避免除零；
- 节点比较使用非降序和容差策略，避免浮点键误差；
- 端点处基函数需单独处理（`x == U[-1]` 归入最后基函数）。

本MVP已覆盖以上细节。

## R11

典型应用：
- CAD中的局部加密（局部增加控制自由度但不改形状）；
- 曲线/曲面拼接前的节点对齐；
- NURBS/样条建模中的重参数表达转换；
- 等几何分析中的网格细化（h-refinement）前处理。

## R12

不太适合或需谨慎的场景：
- 目标不是“保持几何”而是“拟合新数据”（应使用最小二乘拟合等）；
- 大规模重复细化但仍用稠密矩阵实现，内存/计算开销会偏大；
- 节点序列非法（非单调、重数不一致）时算法前提不成立。

## R13

关键参数与经验：
- `degree p`：常见 `2/3`，`p` 越高局部影响带宽越大；
- `refined_knots`：必须是 `coarse_knots` 的细化；
- `ndigits`（多重集比较精度）：过低会误判，过高会受浮点噪声影响；
- 采样点数 `M`：验证几何不变性时建议至少几百点。

工程上应优先保证节点合法性，再谈性能优化。

## R14

与相近算法对比：
- 对比Boehm单节点插入：
  - Boehm处理一次插入；
  - Oslo处理“从旧节点到新节点”的整体细化流程。
- 对比de Boor求值算法：
  - de Boor是“给定参数值求曲线点”；
  - Oslo是“变换控制表示而不改曲线”。
- 对比黑箱库调用：
  - 库接口快但难审计；
  - 本MVP显式展开每个步骤，可直接追踪矩阵与权重来源。

## R18

`demo.py` 源码级流程可拆为 8 步：
1. 准备粗节点 `coarse_knots`、目标细化节点 `refined_knots`、粗控制点 `coarse_ctrl`。
2. 在 `multiset_difference_sorted` 中比较两节点多重集，提取待插入节点序列 `extras`。
3. 对每个 `xi` 调用 `single_knot_insertion_matrix`：按Boehm公式生成单次插入矩阵 `A` 与新节点向量。
4. 在 `oslo_refinement_matrix` 中累乘 `T <- A @ T`，得到从粗控制点到细化控制点的总变换 `T`。
5. 执行 `refined_ctrl = T @ coarse_ctrl` 获得新控制点表示。
6. 用 `bspline_basis_all` + `evaluate_bspline_curve` 显式计算细化前后两条曲线在同一采样网格上的点值。
7. 计算 `max_geom_error`、`mean_geom_error`，并统计 `row_sum` 与 `nnz_per_row` 验证凸组合和稀疏局部性。
8. 输出摘要表、稀疏度表和细化控制点表，形成“可运行 + 可验证 + 可审计”的闭环。
