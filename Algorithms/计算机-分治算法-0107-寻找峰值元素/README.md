# 寻找峰值元素

- UID: `CS-0086`
- 学科: `计算机`
- 分类: `分治算法`
- 源序号: `107`
- 目标目录: `Algorithms/计算机-分治算法-0107-寻找峰值元素`

## R01

“寻找峰值元素”问题：给定一维数组 `nums`，找任意一个峰值下标 `i`，使得
- `nums[i] > nums[i-1]`（若 `i=0` 则左侧视为 `-∞`），
- `nums[i] > nums[i+1]`（若 `i=n-1` 则右侧视为 `-∞`）。

目标不是找到“全局最大值”，而是找到任意一个局部峰值。

## R02

该题常见于分治/二分教学（例如在线题库中的 `Find Peak Element`）。

它的代表性在于：
- 虽然问题看起来像“局部搜索”，但可用二分式分治在 `O(log n)` 内完成；
- 思想与“根据中点方向信息砍半搜索区间”的二分框架一致；
- 是从线性扫描 `O(n)` 过渡到对数复杂度的重要案例。

## R03

形式化定义：
- 输入：长度为 `n (n>=1)` 的数组 `nums`；经典版本通常假设 `nums[i] != nums[i+1]`。
- 输出：一个下标 `idx`，满足峰值条件。

示例：
- `nums = [1, 2, 3, 1]`，可返回 `2`；
- `nums = [1, 2, 1, 3, 5, 6, 4]`，可返回 `1` 或 `5`。

## R04

分治核心：观察中点 `mid` 与 `mid+1`。

- 若 `nums[mid] > nums[mid+1]`，说明在区间 `[left, mid]` 内必存在峰值；
- 否则（`nums[mid] < nums[mid+1]`），在区间 `[mid+1, right]` 内必存在峰值。

因此每轮都可丢弃一半区间，递归或迭代直到 `left == right`。

## R05

MVP 的关键状态：
- `left, right`：当前分治区间；
- `mid`：二分中点；
- `arr[mid]` 与 `arr[mid+1]`：决定向左或向右收缩；
- `DivideConquerStats`：记录递归调用次数、比较次数、最大深度；
- `PeakResult`：返回峰值下标、峰值数值和统计信息。

## R06

示例 `nums = [1, 2, 1, 3, 5, 6, 4]`：

1. 初始区间 `[0, 6]`，`mid=3`，比较 `nums[3]=3` 与 `nums[4]=5`。
2. 因 `3 < 5`，峰值一定在右半段，区间变为 `[4, 6]`。
3. 现在 `mid=5`，比较 `nums[5]=6` 与 `nums[6]=4`。
4. 因 `6 > 4`，峰值一定在左半段，区间变为 `[4, 5]`。
5. 现在 `mid=4`，比较 `nums[4]=5` 与 `nums[5]=6`。
6. 因 `5 < 6`，区间变为 `[5, 5]`，返回下标 `5`，值为 `6`。

## R07

算法价值：
- 展示“局部比较信息足以做全局区间裁剪”的分治思想；
- 在无序数组中实现 `O(log n)` 查找（不是排序后查找）；
- 与单调性不完全等价，但仍可通过结构性质做二分决策。

## R08

前置知识：
- 二分法的区间不变量维护；
- 递归终止条件设计；
- 边界处理（首尾元素与虚拟 `-∞`）；
- 基础复杂度分析（`T(n)=T(n/2)+O(1)`）。

## R09

复杂度：
- 时间复杂度：`O(log n)`（每轮缩小一半区间）；
- 空间复杂度：
  - 递归实现为 `O(log n)` 调用栈；
  - 若改为迭代可降到 `O(1)`。

对比线性扫描基线：时间 `O(n)`，实现更直观但在大规模输入下更慢。

## R10

正确性要点：

1. 维护不变量：当前区间 `[left, right]` 内至少存在一个峰值。
2. 当 `nums[mid] > nums[mid+1]` 时，右侧出现下降趋势；从 `mid` 向左看，必能在某处到达峰值（可在 `mid` 或更左），故左半区保留不变量。
3. 当 `nums[mid] < nums[mid+1]` 时，对称地可知右半区必有峰值。
4. 每轮严格缩小区间长度，最终到 `left==right`。
5. 此时唯一元素即为不变量保证的峰值位置。

因此算法总能返回合法峰值下标。

## R11

数值稳定性：
- 本题主要是大小比较，不涉及浮点迭代误差积累；
- 关键风险来自语义约束，而非数值精度：
  - 若输入包含 `NaN`，比较会失真；
  - 若允许大量相邻相等，需要把“严格峰值”定义改为“非严格峰值（>=）”并同步修改判定逻辑。

本目录 MVP 采用经典严格峰值语义，并在随机测试中使用相邻不等数据。

## R12

工程成本（超越大 O）：
- 递归版本有函数调用开销，但实现清晰；
- 线性基线更易读，适合作为验证器；
- 实际线上场景若追求极限性能，通常改为迭代二分并减少对象创建；
- 大头成本通常不在单次查找，而在批量调用与数据准备（I/O、转换）。

## R14

常见失效模式：
- 空数组未拦截，导致越界；
- 递归边界写错（如 `mid` 更新不当）造成死循环；
- 把问题误写成“全局最大值”而不是“任意峰值”；
- 忽略边界虚拟 `-∞` 语义，导致首尾判定错误；
- 测试只校验“返回固定索引”，没有按“返回任意合法峰值”验收。

## R15

实现建议：
- 定义 `is_peak(nums, idx)` 统一验收条件；
- 分治函数只负责返回一个合法索引，不绑定唯一答案；
- 保留线性扫描版本用于随机对拍；
- 固定用例 + 随机回归同时覆盖：
  - 单元素；
  - 严格上升/下降；
  - 多峰数组。

## R18

`demo.py` 源码级流程（8 步）：

1. `main()` 依次调用 `run_fixed_cases()`、`run_random_regression()`、`run_benchmark()`，全程无交互输入。  
2. `find_peak_element_divide_conquer(nums)` 先把输入转成 `numpy` 数组并校验非空，初始化统计对象。  
3. 内部递归 `solve(left, right, depth)` 先更新调用计数和最大深度。  
4. 若 `left == right`，命中基线情形，直接返回该索引。  
5. 否则计算 `mid=(left+right)//2`，比较 `arr[mid]` 与 `arr[mid+1]`。  
6. 若左值更大，递归进入 `[left, mid]`；否则递归进入 `[mid+1, right]`，从而每轮砍半。  
7. 递归返回后封装为 `PeakResult(index, value, stats)`，并在 `run_fixed_cases` / `run_random_regression` 中用 `is_peak` 验证合法性。  
8. `run_benchmark()` 对不同规模重复采样，分别计时分治与线性基线，最后用 `pandas.DataFrame` 打印对比表。
