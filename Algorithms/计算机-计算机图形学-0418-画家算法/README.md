## R01

画家算法（Painter's Algorithm）是一种经典的隐藏面消除思想：  
先绘制“远处物体”，再绘制“近处物体”，让后绘制的像素自然覆盖先前像素。

它的核心规则是对象级排序而非像素级深度测试，因此实现简单，但在物体互相穿插时会出现错误。

## R02

本条目的计算目标：
- 输入：若干三角形（顶点 `x,y,z` 在 NDC 中）、颜色、屏幕分辨率；
- 输出：
  - 画家算法渲染得到的 `color/owner/depth` 缓冲；
  - Z-Buffer 参考结果（用于对照）；
  - 每三角形统计与全局指标；
  - 两种算法的像素差异量。

`demo.py` 为固定场景、无交互输入。

## R03

画家算法在本 MVP 中采用以下排序准则：
- 约定 `z` 越小越近、`z` 越大越远；
- 对每个三角形计算深度键 `depth_key = mean(z0, z1, z2)`；
- 按 `depth_key` 从大到小排序（远到近）并依次绘制。

该准则对“整体可比较深度”的场景有效，对穿插场景仅是近似。

## R04

MVP 主流程分 8 步：
1. 校验三角形数据合法性（形状、范围、颜色）；
2. 将 NDC 顶点映射到屏幕坐标；
3. 对三角形求包围盒并逐像素采样；
4. 用边函数 + 重心坐标筛出三角形内部片元并插值 `z`；
5. 画家算法：按远到近顺序直接覆盖写入颜色；
6. Z-Buffer：逐片元执行 `z < depth_buffer` 后再写入；
7. 汇总三角形级统计与全局覆盖率等指标；
8. 比较两者结果并做自动断言。

## R05

关键数据结构：
- `Triangle(name, vertices_ndc, color)`：输入图元；
- `color_buffer: uint8[H,W,3]`：颜色缓冲；
- `owner_buffer: int32[H,W]`：每像素最终归属三角形 ID；
- `depth_buffer: float[H,W]`：
  - 画家算法中记录最后写入片元深度；
  - Z-Buffer 中记录当前最近深度；
- `tri_stats/global_metrics: pandas.DataFrame`：统计报表。

## R06

`demo.py` 核心函数职责：
- `_validate_triangle`：检查顶点与颜色合法性；
- `ndc_to_screen`：坐标映射；
- `edge_function`：二维有向面积（inside-test 基础）；
- `rasterize_triangle`：输出包围盒候选像素与内部片元；
- `render_painter`：按深度键排序并覆盖绘制；
- `render_zbuffer`：参考实现，用于逐像素深度测试；
- `compare_buffers`：统计两种算法像素差异；
- `main`：构造两个场景并执行断言与打印结果。

## R07

复杂度（`T` 个三角形，`A_i` 为第 `i` 个包围盒像素面积）：
- 光栅化时间：`O(sum(A_i))`；
- 排序开销：`O(T log T)`（画家算法特有）；
- 总时间：`O(T log T + sum(A_i))`；
- 空间：`O(W*H)`（颜色/深度/归属缓冲）。

当分辨率高且图元密集时，瓶颈仍主要在片元遍历。

## R08

正确性依赖以下前提：
1. 场景能被“单一远近顺序”较好描述（无循环遮挡）；
2. 同一三角形内深度变化不导致跨图元顺序反转；
3. 排序键能近似代表该图元整体深度；
4. 绘制顺序严格执行“远先近后”。

若上述前提被破坏，画家算法会产生可见错误。

## R09

数值注意点：
- 边函数判定使用 `eps`，减少边界浮点误差漏填；
- 面积过小视为退化三角形并跳过；
- 顶点 `xy` 约束在 `[-1,1]`，`z` 约束在 `[0,1]`；
- 平均深度排序是启发式，不保证逐像素正确。

## R10

`demo.py` 使用两个场景：
- 场景 A（`monotonic_depth`）：
  - 三角形深度整体可比较；
  - 预期：画家算法与 Z-Buffer 结果一致。
- 场景 B（`interpenetrating_depth`）：
  - 两个三角形 2D 上高度重叠，且深度梯度相反；
  - 预期：画家算法出现像素错误，Z-Buffer 正确。

## R12

关键输出指标：
- `candidate_pixels`：包围盒内被检查的像素数量；
- `inside_fragments`：实际落在三角形内部的片元；
- `depth_pass`：
  - 画家算法中等于写入次数；
  - Z-Buffer 中为深度测试通过次数；
- `depth_fail`：仅 Z-Buffer 中有意义，表示被遮挡丢弃次数；
- `overwritten`：写入时覆盖已有像素次数；
- `owner_mismatch_pixels`：画家算法与 Z-Buffer 的像素归属差异数。

## R13

自动断言包含：
1. 场景 A 中 `owner_mismatch_pixels == 0`；
2. 场景 B 中 `owner_mismatch_pixels > 0`；
3. 场景 B 差异比例应大于 0（输出数值可直接核验）。

这保证脚本不仅运行成功，还能展示画家算法“何时有效、何时失效”。

## R14

边界与异常处理：
- 非法分辨率（`width<=0` 或 `height<=0`）抛 `ValueError`；
- 顶点数组形状不是 `(3,3)` 或出现非有限值抛 `ValueError`；
- 顶点越界、颜色不在 `0..255` 抛 `ValueError`；
- 退化三角形会被安全跳过，不中断整体渲染流程。

## R15

画家算法的典型应用：
- 教学场景中解释遮挡与渲染顺序；
- 图元较少、可提前排序的 2D/伪 3D 系统；
- 对性能要求不高且场景深度关系简单的可视化原型；
- 作为 Z-Buffer 管线的对照基线。

## R16

本 MVP 的限制：
- 仅支持三角形，不含线段/点规则；
- 未实现裁剪、透视正确插值、抗锯齿等完整管线；
- 画家算法排序键固定为平均深度，未做平面分割；
- 输出为终端统计，不保存图像文件。

## R17

可扩展方向：
1. 增加多种排序键（`max_z`、`min_z`、包围盒深度区间）并比较误差；
2. 引入 BSP / 平面分割，缓解循环遮挡问题；
3. 加入图像导出（PNG）和回归测试；
4. 增加随机场景批量评估画家算法失效率；
5. 与背面剔除、裁剪模块联合成简化渲染管线。

## R18

`demo.py` 源码级算法链路（9 步）：
1. `main()` 构造两个测试场景：单调深度和穿插深度。  
2. `render_painter()` 先对每个三角形计算 `depth_key=mean(z)` 并按远到近排序。  
3. 每个三角形通过 `ndc_to_screen()` 映射到像素坐标。  
4. `rasterize_triangle()` 在包围盒内用 `edge_function` 进行 inside-test，并用重心坐标插值得到片元深度。  
5. 画家算法对每个片元直接覆盖写 `color/owner/depth`，不做深度比较。  
6. `render_zbuffer()` 复用同一片元生成逻辑，但执行 `z < depth_buffer[y,x]` 决定是否写入。  
7. 两个渲染函数都累计 `candidate/inside/pass/fail/overwritten` 到 `tri_stats`。  
8. `compare_buffers()` 计算两算法 owner 缓冲差异像素与差异比例。  
9. `main()` 对场景 A 断言“差异为 0”，对场景 B 断言“差异大于 0”，并打印完整报表。
