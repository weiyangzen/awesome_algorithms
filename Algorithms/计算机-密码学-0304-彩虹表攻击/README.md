# 彩虹表攻击

## R01

`彩虹表攻击（Rainbow Table Attack）`是一种针对**无盐哈希口令**的时间-空间折中攻击。  
核心目标是：通过预计算大量“哈希-约简链”并仅保存链首/链尾，在线阶段快速反查目标哈希对应的明文口令。

## R02

本条目要交付一个可运行的最小 MVP，覆盖完整流程：
- 构造多条彩虹链（每列使用不同约简函数 `R_i`）；
- 仅存储链首与链尾，形成端点索引表；
- 对目标哈希执行在线回溯，命中后重建链并定位原口令；
- 输出成功率与碰撞统计，展示“压缩存储 vs 覆盖率损失”的权衡。

## R03

攻击模型与前提：
- 目标系统使用快速哈希（本例为 `SHA-1`）存储口令；
- 哈希值不含每用户唯一盐（salt）；
- 攻击者已知哈希算法、口令字符集与长度范围；
- 攻击者允许离线预计算，并在在线阶段尽可能减少搜索时间。

## R04

核心思想（时间-空间折中）：
- 朴素预计算：存“明文->哈希”全表，空间过大；
- 暴力破解：几乎不占空间，但在线时间高；
- 彩虹表：将大量映射压缩成链，仅存 `(start, end)`；
- 在线时通过“逆向对齐 + 尾段推进”尝试命中端点，再重放命中链确认真实前像。

## R05

链构造定义（长度为 `t`）：
- `P_0` 为链首明文；
- `H_i = hash(P_i)`；
- `P_{i+1} = R_i(H_i)`，其中 `R_i` 随列号变化；
- 最终仅保存 `(P_0, P_t)`。

在线破解目标哈希 `H*` 时，从 `pos=t-1...0` 枚举可能列位置，执行尾段推进，得到候选端点并查表命中。

## R06

`demo.py` 的输入输出约定：
- 输入：代码内固定配置与样本（零交互）；
- 配置参数：字符集、口令长度、链长、链数；
- 输出：建表耗时、端点冲突数、样本破解结果、总体成功率；
- 目标：可直接 `uv run python demo.py`，无需命令行参数与人工输入。

## R07

实现结构（单文件、可追踪）：
- `RainbowConfig`：集中管理参数；
- `sha1_hex`：哈希函数；
- `int_to_password`：整数到口令空间映射；
- `reduce_hash`：按列变化的约简函数族 `R_i`；
- `chain_end` / `build_rainbow_table`：离线链生成与表构建；
- `crack_hash`：在线候选端点推导与命中验证；
- `regenerate_chain_find_target`：重放链确认真实命中，过滤误报。

## R08

复杂度（记链数 `m`，链长 `t`）：
- 离线建表：约 `O(m * t)` 次哈希+约简操作；
- 存储：约 `O(m)`（仅存链首/链尾，不存全路径）；
- 单次在线查询：最坏约 `O(t^2 + k*t)`，`k` 为命中端点后的候选链重放数量；
- 实践中命中率受覆盖率、链合并（merge）与端点碰撞影响。

## R09

正确性要点：
- 若目标哈希确实出现在某条已存链的第 `j` 列，则从 `j` 到链尾的尾段推进会导出该链端点；
- 查表命中端点后，重放链可在某一步得到目标哈希，从而恢复对应明文；
- 为避免“伪命中”，必须做重放验证，不能仅凭端点命中直接返回结果。

## R12

边界与限制：
- 本演示固定口令长度（4 位）与固定字符集，超出空间无法命中；
- 采用 `SHA-1` 仅为教学演示，真实系统常见更强或更慢的 KDF；
- 若目标系统使用独立盐（salt）或 pepper，本表几乎失效；
- 链合并/碰撞会降低有效覆盖率，导致漏报（无法破解）。

## R13

安全防护建议（面向工程）：
- 禁止使用快速无盐哈希存储口令；
- 必须使用每用户唯一随机盐；
- 采用慢哈希/KDF（如 `Argon2id`、`scrypt`、`bcrypt`、`PBKDF2`）；
- 配合登录限速、MFA、异常检测与泄露响应机制。

## R14

与相近攻击方式对比：
- 对比暴力破解：暴力几乎不占空间，但时间高；彩虹表用空间换时间；
- 对比字典攻击：字典依赖弱口令分布，彩虹表更偏向“结构化覆盖固定空间”；
- 对比全预计算表：全表查询最快但存储最贵，彩虹表通过链压缩显著降存储。

## R18

源码级算法流程拆解（对应 `demo.py`，未用第三方黑盒破解）：
1. 读取配置 `alphabet/password_length/chain_length/chain_count`，定义有限口令空间。  
2. 通过 `build_rainbow_table` 随机选取链首 `P_0`，对每列执行 `H_i = SHA1(P_i)` 与 `P_{i+1}=R_i(H_i)`，直到链尾 `P_t`。  
3. 仅保存 `endpoint -> [start...]` 映射，形成压缩索引并统计端点碰撞。  
4. 给定目标哈希 `H*`，`crack_hash` 从最后一列向前枚举潜在位置 `pos`。  
5. 对每个 `pos`，执行尾段推进：反复应用 `R_j` 和 `hash`，推导出候选链尾 `endpoint*`。  
6. 若 `endpoint*` 在表中命中，取出该端点下所有链首候选，逐条重放整链。  
7. 在重放过程中比较每一步 `hash(plain)` 与 `H*`，命中即返回对应明文口令。  
8. 若全部 `pos` 与候选链都未命中，返回失败；最终在 `main` 中汇总成功率与失败样本。
