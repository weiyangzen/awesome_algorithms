## R01

Burnside 引理（也称 Cauchy-Frobenius 轨道计数引理）用于计算“在对称作用下本质不同对象”的数量。

设有限群 `G` 作用在有限集合 `X` 上，则轨道数满足：

`|X / G| = (1 / |G|) * sum_{g in G} |Fix(g)|`

其中：

- `X / G` 表示轨道集合（即去重后的本质类别）。
- `Fix(g) = {x in X : g·x = x}` 是被群元素 `g` 固定的元素集合。

## R02

- 历史背景:
  - 该结论源自 19 世纪的群作用计数理论，由 Cauchy 和 Frobenius 体系化，后在组合数学中常称 Burnside 引理。
- 方法定位:
  - 这是“先数固定点、再取平均”的轨道计数定理。
- 实际意义:
  - 把“按同构/对称去重”的困难分类问题，转成每个群元素下的固定点计数问题。

## R03

核心动机是避免重复计数。

- 直接枚举 `X` 常会把同一轨道中的对象重复计算。
- Burnside 引理不直接枚举轨道代表，而是统计每个对称变换固定了多少对象。
- 这些固定点数量的平均值恰好等于轨道数。

因此它把“分类”问题转化为更可计算的“固定点”问题。

## R04

若群大小为 `|G|`，并且每个 `g` 的固定点数计算成本是 `T_fix`，则：

- 时间复杂度: `O(|G| * T_fix)`。
- 对常见着色模型（长度 `n`、颜色数 `m`）里，
  - 若用循环分解算固定点，单个 `g` 成本 `O(n)`，
  - 总体约 `O(|G| * n)`。

本目录 MVP 选用循环群 `C_n` 的着色计数示例，总体复杂度 `O(n^2)`。

## R05

空间复杂度取决于是否预存群与中间结构。

- 若预存全部置换，空间约 `O(|G| * n)`。
- 单个置换的循环分解额外空间 `O(n)`。
- 本 MVP 同时保留群元素与小规模验证数据，便于可读性和对拍，整体仍属轻量。

## R06

示例：长度 `n=6` 的环形项链，`m=3` 种颜色，只按旋转视为等价。

- 群 `G = C_6`，共有 6 个旋转。
- 每个旋转对应一个置换，其固定着色数为 `3^{c(g)}`，`c(g)` 是循环个数。
- 6 个旋转的循环个数分别为 `6,1,2,3,2,1`。

所以：

`|X/G| = (1/6) * (3^6 + 3^1 + 3^2 + 3^3 + 3^2 + 3^1)`
`= (1/6) * 780 = 130`。

即本质不同 3 色长度 6 项链数为 `130`。

## R07

Burnside 引理的算法价值：

- 通用去重计数:
  - 对任意有限群作用都适用。
- 可组合性强:
  - 可与置换循环分解、Polya 计数、图自同构等工具结合。
- 可验证性好:
  - 每个群元素的固定点都有明确含义，便于调试与数学检查。

## R09

适用条件与边界：

- 适用:
  - 群 `G` 有限，作用集合 `X` 有限。
- 不适用:
  - 连续群或无限状态空间（需测度或其他工具）。
- 本 MVP 输入边界:
  - `n >= 1`、`m >= 1`；
  - 群元素需是合法置换。

## R10

正确性思路（双计数）：

1. 统计集合
   `S = {(g, x) in G x X : g·x = x}`。
2. 按 `g` 分组：
   `|S| = sum_{g in G} |Fix(g)|`。
3. 按 `x` 分组：
   若 `x` 所在轨道为 `Orb(x)`，其稳定子大小 `|Stab(x)| = |G| / |Orb(x)|`。
4. 同一轨道内所有元素对 `|S|` 的贡献和等于 `|G|`。
5. 所有轨道求和得 `|S| = |G| * |X/G|`。
6. 两式相等并除以 `|G|`，得 Burnside 引理。

## R11

数值稳定性：

- 该条目是离散整数精确计数，不涉及浮点误差。
- 主要工程风险是大整数幂计算变慢，不是数值不稳定。
- Python 整数是任意精度，保证数学结果精确。

## R12

性能特征：

- `|G|` 增长是主导因子。
- 对着色计数，固定点数通常由循环个数给出，计算快且结构清晰。
- 暴力轨道枚举复杂度约 `O(m^n * |G| * n)`，只适合小规模验算。

MVP 中保留小规模暴力对拍，兼顾正确性与可运行性。

## R14

鲁棒性与失效模式：

- 非法置换:
  - 若群元素不是 `0..n-1` 的双射，会导致错误结果；MVP 会做校验。
- 群不封闭或漏单位元:
  - 理论假设破坏会使“平均固定点数”失效。
- 输入过大:
  - 大 `n` 与大 `m` 时幂运算与暴力校验会变慢。

## R18

源码级算法流程（`demo.py`，核心逻辑全部自行展开，不依赖第三方黑箱）：

1. 校验 `n,m` 与置换合法性，确保输入满足有限群作用前提。
2. 构造循环群 `C_n` 的全部旋转置换，作为群元素列表。
3. 对每个置换做循环分解，统计循环个数 `c(g)`。
4. 用公式 `|Fix(g)| = m^{c(g)}` 计算该群元素固定的着色数。
5. 累加全部 `|Fix(g)|`，再除以 `|G|`，得到 Burnside 轨道数。
6. 通过暴力轨道枚举（小规模）独立计算真值：遍历全部着色并按群作用求最小代表。
7. 比较 Burnside 结果与暴力结果，断言一致后输出示例与通过信息。
