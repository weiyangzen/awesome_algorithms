# 线性筛/欧拉筛 (Linear Sieve)

- UID: `MATH-0004`
- 学科: `数学`
- 分类: `数论`
- 源序号: `4`
- 目标目录: `Algorithms/数学-数论-0004-线性筛／欧拉筛_(Linear_Sieve)`

## R01

线性筛（Euler Sieve）用于一次性求出 `1..n` 内全部质数，并在同一遍历中维护每个整数的最小质因子（least prime factor, LPF）。它的核心目标是把“筛选合数”的总工作量压到与 `n` 同阶。

## R02

适用场景：
- 需要批量查询质数（如多次判断 `x` 是否为质数）。
- 需要快速分解 `<= n` 的整数（借助 LPF 表）。
- 需要在数论题中进一步推导欧拉函数、莫比乌斯函数等积性函数。

## R03

核心思想：
- 维护一个质数列表 `primes`。
- 枚举每个 `i` 时，用已有质数 `p` 去标记 `i * p`。
- 一旦 `p` 整除 `i`，立即 `break`，保证每个合数只被其最小质因子路径标记一次。

## R04

输入输出约定（对应 `demo.py`）：
- 输入：非负整数上界 `n`。
- 输出：
  - `primes`：升序质数列表。
  - `least_prime_factor`：`least_prime_factor[x]` 为 `x` 的最小质因子（`x>=2`）。
- 对 `n < 2` 返回空质数表和长度 `n+1` 的零表。

## R05

伪代码：

```text
primes = []
mark = [False] * (n + 1)
lpf = [0] * (n + 1)
for i in [2..n]:
    if not mark[i]:
        primes.push(i)
        lpf[i] = i
    for p in primes:
        x = i * p
        if x > n: break
        mark[x] = True
        lpf[x] = p
        if i % p == 0: break
```

## R06

正确性要点：
- 若 `i` 未被标记，则它不含更小质因子，故必为质数。
- 对任意合数 `x`，设其最小质因子是 `p`，可写成 `x = i * p` 且 `p` 不大于 `i` 的任一质因子。算法在处理该 `i` 时会首次标记 `x`。
- `if i % p == 0: break` 阻止更大质因子重复标记同一合数，因此每个合数只被写入一次。

## R07

复杂度：
- 时间复杂度：`O(n)`（线性筛的关键性质）。
- 空间复杂度：`O(n)`（标记数组 + LPF 数组 + 质数表）。

## R08

与埃氏筛（Eratosthenes）对比：
- 埃氏筛通常实现是 `O(n log log n)`，常数较小、实现也很简洁。
- 线性筛是 `O(n)`，并且更自然地产出最小质因子表，适合“筛 + 分解”联动任务。
- 当只需要质数列表且实现简洁优先时，可选埃氏筛；当还要 LPF/积性函数时，线性筛更合适。

## R09

数据结构设计：
- `is_composite[i]`：是否已被判定为合数。
- `primes`：按发现顺序（自然升序）保存质数。
- `least_prime_factor[i]`：`i` 的最小质因子；对质数 `i` 有 `lpf[i] = i`。

## R10

边界与异常：
- `n < 0`：抛出 `ValueError`，避免无意义范围。
- `n = 0/1`：返回空质数列表。
- 因子分解函数仅支持 `1 <= x <= n`，越界时抛出异常。

## R14

常见实现错误：
- 忘记在 `i` 为质数时写入 `lpf[i] = i`。
- 把 `if i % p == 0: break` 写错或漏掉，导致退化为重复标记。
- `x = i * p` 越界判断放错位置，引发下标错误。

## R18

源码级算法流程（本实现，无黑箱第三方）：
1. 初始化 `is_composite`、`least_prime_factor` 和空 `primes` 列表。  
2. 从 `i=2` 遍历到 `n`；若 `i` 尚未被标记，则把 `i` 追加到 `primes`，并设 `lpf[i]=i`。  
3. 对当前 `i`，按 `primes` 的升序尝试每个 `p`。  
4. 计算 `x = i * p`；若 `x > n`，结束当前 `i` 的内层循环。  
5. 标记 `x` 为合数，并设置 `lpf[x] = p`。  
6. 若 `i % p == 0`，立刻中断内层循环，保证 `x` 由其最小质因子路径唯一生成。  
7. 外层循环结束后，`primes` 即完整质数表，`lpf` 可用于 `O(log n)` 级别分解单个整数。  
8. `demo.py` 再用朴素判素做小范围交叉校验，并验证 `pi(100000)=9592`，确保实现正确。
