# 布伦特方法 (Brent's Method)

- UID: `MATH-0120`
- 学科: `数学`
- 分类: `数值分析`
- 源序号: `120`
- 目标目录: `Algorithms/数学-数值分析-0120-布伦特方法_(Brent's_Method)`

## R01

本条目实现一维非线性方程求根的 Brent 方法最小可运行版本（MVP）。

Brent 方法的核心目标是同时满足：
- 像二分法一样稳健（始终保持根被区间夹住）；
- 像割线法/逆二次插值一样快（在可行时用高阶插值加速）。

`demo.py` 给出从零实现，不依赖现成黑盒求根接口。

## R02

问题定义（标量有界区间版）：
- 输入：
  - 连续函数 `f(x)`；
  - 区间端点 `a, b`，满足 `f(a) * f(b) < 0`（或某端点正好为根）；
  - 收敛参数 `tol`、`max_iter`。
- 输出：
  - 近似根 `x*`；
  - `f(x*)`；
  - 是否收敛、迭代步数、每轮迭代轨迹。

## R03

方法原理（混合策略）：
- 二分法：保证区间收缩与全局稳健性；
- 割线法：基于两点线性插值，通常比纯二分快；
- 逆二次插值（IQI）：基于三点构造更高阶插值，常在光滑情形下更快。

Brent 方法在每一步都先尝试插值步；若该步不安全（例如可能越界或改进不足），立即回退到二分步。这样兼顾“稳定性下界”和“速度上界”。

## R04

算法总览：
1. 校验输入与有根区间条件；
2. 初始化三个点 `a,b,c`（其中 `b` 是当前最好估计，`[b,c]` 保持夹根）；
3. 根据函数值大小重排点，优先让 `|f(b)|` 更小；
4. 计算停止阈值与当前半区间长度；
5. 若已满足收敛条件则结束；
6. 尝试割线或 IQI 产生候选步长；
7. 若候选步不满足接受条件，回退为二分步；
8. 更新点与函数值，记录轨迹并继续。

## R05

核心数据结构：
- `IterationRecord`（`dataclass`）：
  - 记录 `iteration, a, b, c, fa, fb, fc, half_interval, step`；
  - 用于复盘每轮用了哪种步（`bisection / secant / iqi`）。
- `BrentResult`（`dataclass`）：
  - 聚合 `root, f_at_root, iterations, converged, history`。

这种结构便于后续做统计、画收敛曲线或对比不同停止阈值。

## R06

正确性要点：
- 当 `f(a)` 与 `f(b)` 异号且函数连续，初始区间包含至少一个根；
- 每轮都维护夹根关系（必要时重置 `c`），因此不会失去根；
- 插值步只有在满足安全判据时才接受，否则退回二分；
- 二分回退保证最坏情况下仍可收敛。

因此该实现具备工程上常用的“先保正确，再争速度”的性质。

## R07

复杂度分析（单次求根）：
- 时间复杂度：`O(T)`，`T` 为迭代轮数；
- 空间复杂度：
  - 记录完整轨迹时 `O(T)`；
  - 若仅保留当前状态可降为 `O(1)`。

相较纯二分法，Brent 方法通常在相同精度下需要更少迭代步。

## R08

边界与异常处理：
- `a == b`：抛出 `ValueError`；
- `tol <= 0` 或 `max_iter < 1`：抛出 `ValueError`；
- `a/b/f(a)/f(b)` 非有限数：抛出异常；
- `f(a)` 与 `f(b)` 同号：抛出 `ValueError`（不满足夹根前提）；
- 迭代中出现非有限函数值：抛出 `RuntimeError`。

## R12

输出解读：
- `estimate root`：算法输出根；
- `reference`：已知参考根；
- `abs error / rel error`：误差指标；
- `|f(root)|`：函数残差；
- `step counts`：每种步型被采用的次数；
- 表格列 `step`：每轮采用 `bisection / secant / iqi` 或带 `+guard` 的保护步。

## R13

最小测试集建议：
- `f(x)=cos(x)-x, [0,1]`（经典固定点方程）；
- `f(x)=x^3-x-2, [1,2]`（多项式非线性）；
- `f(x)=exp(-x)-x, [0,1]`（指数与线性耦合）；
- 异常用例：同号端点、`tol<=0`、`a==b`、非有限输入。

这些用例覆盖正常收敛与主要防御分支。

## R14

可调参数：
- `tol`：收敛阈值（默认 `1e-12`）；
- `max_iter`：最大迭代次数（默认 `100`）；
- `print_history(..., max_lines=8)`：轨迹打印行数。

工程实践中可按性能/精度要求调整 `tol` 与 `max_iter`。

## R15

方法对比：
- 对比二分法：
  - 二分法稳定但线性收敛；
  - Brent 在稳定性近似不变的前提下通常更快。
- 对比牛顿法：
  - 牛顿法需要导数且初值敏感；
  - Brent 仅需函数值和夹根区间，更稳健。
- 对比割线法：
  - 割线法可能失去根区间；
  - Brent 用区间维护机制避免失控。

## R16

- 作为更复杂数值流程中的一维子问题求解器（例如参数反演中的内循环）。

## R18

源码级算法流（对应 `demo.py`，8 步）：
1. `main` 定义 3 个方程与区间，逐个调用 `run_case`。  
2. `run_case` 调用 `brent_root(f, a, b, tol, max_iter)` 并接收 `BrentResult`。  
3. `brent_root` 先做输入检查与变号检查，初始化 `a,b,c,fa,fb,fc,d,e`，写入初始轨迹。  
4. 每轮先修复夹根关系：若 `fb*fc>0`，则把 `c` 重置到 `a`，确保 `[b,c]` 继续夹根。  
5. 通过交换点保证 `|fb|` 更小，使 `b` 始终代表当前更优根估计。  
6. 满足条件时尝试插值步：`a==c` 用割线，否则用 IQI；若判据失败则回退二分步。  
7. 用选定步长更新 `b`，必要时加 `guard` 最小步防止停滞，再计算新 `fb` 并记录 `IterationRecord`。  
8. 当 `|c-b|/2` 足够小或 `fb==0` 时收敛返回；`run_case` 再输出误差、步型统计与轨迹。
