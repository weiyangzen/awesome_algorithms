## R01

Gibbs 采样是一类基于条件分布逐坐标更新的 MCMC 方法。  
核心思想是：如果目标联合分布 `p(x1, ..., xd)` 的每个全条件分布
`p(xi | x_{-i})` 可直接采样，就可以通过轮流更新各维来构造马尔可夫链，
其平稳分布即目标分布。

本目录的 MVP 选择“多元高斯目标分布”作为教学载体：
- 不依赖黑盒采样库；
- 直接用精度矩阵写出条件正态分布；
- 通过样本均值、协方差和 ESS 做数值验证。

## R02

Gibbs 采样主要解决的问题：
- 目标分布联合采样困难，但条件分布容易采样；
- Random-Walk 类 MH 在相关分布上混合慢、自相关高；
- 需要一个实现简单、可解释性强的后验采样基线方法。

对比很多“整向量提议”的方法，Gibbs 的单步计算轻量，
在中低维且条件分布可解析时非常实用。

## R03

问题定义（本实现）：
- 输入：
  - 目标高斯分布均值 `mean` 和协方差 `cov`；
  - 初始状态 `initial_state`；
  - 采样配置 `num_warmup / num_samples / thinning / seed`。
- 输出：
  - 后验样本矩阵 `samples`；
  - 诊断信息（lag-1 自相关、ESS）；
  - 与真值均值/协方差的误差对比。

## R04

数学建模（多元高斯）：
- 目标分布：`x ~ N(mu, Sigma)`，精度矩阵 `Q = Sigma^{-1}`；
- 对第 `i` 维，条件分布满足
  `x_i | x_-i ~ N(m_i, v_i)`，
  其中
  `v_i = 1 / Q_ii`，
  `m_i = mu_i - (Q_i,-i (x_-i - mu_-i)) / Q_ii`。

这意味着每次更新只需读取 `Q` 的一行并结合当前其他维状态，
即可采样新的 `x_i`。

## R05

本目录算法流程（单链）：
1. 由协方差矩阵求精度矩阵 `Q`；
2. 初始化状态向量 `x`；
3. 对每次迭代执行一次“整轮 Gibbs sweep”：
   按 `i=1..d` 顺序逐维采样 `x_i | x_-i`；
4. 丢弃 warmup 阶段；
5. 按 thinning 规则保留样本；
6. 对保留样本计算统计量并断言检查质量。

Gibbs 采样每步“天然接受”，不需要额外的 MH 接受拒绝步骤。

## R08

优点：
- 实现简单，逐维更新逻辑清晰；
- 在“条件分布可直接采样”场景中非常高效；
- 源码容易扩展到分层贝叶斯模型的分块 Gibbs。

局限：
- 变量强相关时链自相关可能较高、混合变慢；
- 顺序扫描 Gibbs 对坐标顺序有一定敏感性；
- 若条件分布不可解析，则需额外嵌套 MH/HMC 等子步骤。

## R09

适用场景：
- 共轭贝叶斯模型；
- 条件后验有标准分布（高斯、Gamma、Beta 等）可直接抽样；
- 需要快速搭建可解释的 MCMC 原型。

不适用场景：
- 条件分布难采样且强多峰；
- 高维复杂几何、强相关严重时，单坐标 Gibbs 可能效率不足；
- 对工业级诊断和鲁棒性要求极高时，通常要配合更成熟推断框架。

## R10

数值与工程注意点（本实现）：
- `cov` 必须正定（否则 `inv(cov)` 不可用）；
- 精度矩阵对角元 `Q_ii` 必须为正，条件方差 `1/Q_ii` 才合法；
- 使用固定随机种子确保可复现；
- warmup 用于削弱初值影响；
- 输出 ACF 与 ESS 以发现“看似收敛但信息量不足”的问题。

## R11

关键参数（见 `GibbsConfig`）：
- `num_warmup`：预热迭代数，不纳入最终统计；
- `num_samples`：最终保留样本数；
- `thinning`：抽稀间隔（每 `thinning` 个后验迭代保留 1 个）；
- `seed`：随机种子。

调参建议：
- 若 ESS 偏低，可增加 `num_samples`；
- 若初始点离目标较远，可提高 `num_warmup`；
- 若存储受限或自相关较高，可适度加大 `thinning`（但会损失样本总量）。

## R12

复杂度分析：
- 记维度为 `d`、后验迭代数为 `T`；
- 单次整轮 sweep 需对每维做一次长度 `d` 的点积，约 `O(d^2)`；
- 总时间复杂度约 `O(T * d^2)`；
- 空间复杂度约 `O(d + num_samples * d)`（状态与样本缓存）。

在本条目的小维度教学场景中，该复杂度足够轻量。

## R13

正确性直觉：
1. 每个坐标更新都来自该坐标在“其余坐标固定”下的真实条件分布；
2. 顺序组合这些条件更新可构成以目标联合分布为不变分布的马尔可夫核；
3. 在常见遍历条件下，链的经验均值会收敛到目标期望；
4. 通过已知真值高斯的均值/协方差回归，可经验验证实现正确。

本目录不做严格测度论证明，重点是可运行与可审计实现。

## R14

常见失效模式与排查：
1. `cov` 非正定：求逆失败或数值不稳定；
2. 条件均值公式实现错误（符号或索引错）导致样本明显偏移；
3. warmup 太短导致初值偏差残留；
4. 样本数不足导致误差和 ESS 波动较大。

建议排查顺序：
- 先检查矩阵正定和对角元正性；
- 再检查条件更新公式与代码索引；
- 最后看 ESS 与误差阈值是否合理。

## R15

可扩展方向：
- 分块 Gibbs（一次更新一组相关参数）以缓解强相关慢混合；
- 系统扫描/随机扫描策略比较；
- 在条件分布不可直接采样时嵌入 MH 子步骤（Metropolis-within-Gibbs）；
- 多链并行 + `R-hat` 诊断；
- 与 HMC/NUTS 在相同目标上的 ESS/秒对比基准。

## R16

相关方法对比：
- Random-Walk MH：通用但随机游走显著；
- Gibbs：条件可采样时实现最直接，单步成本低；
- HMC：利用梯度做长跳，通常 ESS 更高，但实现和调参更复杂；
- NUTS：自动路径长度的 HMC，工程复杂度高于 Gibbs。

因此 Gibbs 往往是“可解释基线”或“共轭模型首选”，
而非所有问题上的最优效率解。

## R18

`demo.py` 源码级算法流程（8 步）：

1. `main` 定义三维高斯真值参数与 `GibbsConfig`，设置初始状态后调用 `run_gibbs_gaussian`。  
2. `run_gibbs_gaussian` 先计算精度矩阵 `precision = inv(cov)`，初始化链状态 `x` 和样本缓存。  
3. 每次迭代调用 `gibbs_update_gaussian` 执行一次整轮坐标更新（一个 sweep）。  
4. 在 `gibbs_update_gaussian` 中，对每个维度 `i` 用 `Q` 的第 `i` 行计算条件均值 `m_i` 与条件标准差 `sqrt(1/Q_ii)`。  
5. 通过 `rng.normal(m_i, std_i)` 直接采样新 `x_i`，并立即写回状态用于后续维度更新。  
6. 迭代完成后，`run_gibbs_gaussian` 按 warmup 与 thinning 规则收集后验样本矩阵。  
7. `main` 调用 `empirical_covariance`、`lag1_autocorr`、`estimate_ess_per_dim` 计算统计诊断与误差指标。  
8. 通过均值误差、协方差误差、最大 lag-1 自相关和最小 ESS 断言验证采样质量，最后输出 `All checks passed.`。

说明：实现没有把第三方 MCMC 库当黑盒，条件分布推导到逐维采样流程都可在源码中逐行追踪。
