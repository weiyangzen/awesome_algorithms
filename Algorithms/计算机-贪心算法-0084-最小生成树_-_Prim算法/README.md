# 最小生成树 - Prim算法

- UID: `CS-0064`
- 学科: `计算机`
- 分类: `贪心算法`
- 源序号: `84`
- 目标目录: `Algorithms/计算机-贪心算法-0084-最小生成树_-_Prim算法`

## R01

问题定义：给定一个连通无向带权图 `G=(V,E)`，在所有覆盖全部顶点且不含环的子图中，找到总边权最小的一棵树，即最小生成树（MST）。

Prim 算法的视角是“从一个点开始不断扩张树”，每次都选择连接“已在树内顶点集合”和“未在树内顶点集合”的最小权重边。

## R02

输入（MVP 约定）：
- `num_nodes`：顶点个数，编号为 `0..num_nodes-1`；
- `edges`：边列表，单条边格式为 `(u, v, w)`；
- `start`：Prim 的起始顶点编号。

输出：
- `mst_edges`：MST 边集合（`n-1` 条）；
- `total_weight`：MST 总权重。

异常策略：
- 起点越界、边端点越界、图不连通时抛 `ValueError`。

## R03

贪心策略：
1. 维护已纳入生成树的顶点集合 `S`（初始 `S={start}`）；
2. 在所有“跨越割 `(S, V-S)`”的边中选择权重最小边；
3. 将该边和新顶点加入生成树；
4. 重复直到覆盖全部顶点。

这一步步选择局部最优边，最终达到全局最优 MST。

## R04

正确性依据是 MST 的割性质（Cut Property）：
- 对任意割 `(S, V-S)`，跨越该割的最小权边一定属于某棵 MST；
- Prim 在每一步恰好做这个选择；
- 每一步都不破坏最优性，累计到 `n-1` 条边后得到最小生成树。

## R05

MVP 使用的数据结构：
- 邻接表：表示无向带权图；
- 最小堆（`heapq`）：维护当前候选跨割边；
- `visited` 数组：标记顶点是否进入树；
- `mst_edges` 列表：收集最终边。

同时提供 `UnionFind + kruskal_mst` 用于交叉校验 Prim 结果。

## R06

算法流程（堆优化 Prim）：
1. 构建无向图邻接表；
2. 初始化最小堆 `[(0.0, start, -1)]`（`-1` 为虚拟父节点）；
3. 弹出堆顶最小候选边；
4. 若目标顶点已访问则跳过；
5. 否则将该点并入生成树，记录父边并累加权重；
6. 把该点发出的可扩展边压入堆；
7. 循环直到所有顶点被访问；
8. 若无法访问全部顶点，说明图不连通并抛异常。

## R07

复杂度：
- 建图：`O(E)`；
- 堆操作：每条边至多进入堆常数次，单次代价 `O(log V)`；
- 总时间复杂度：`O(E log V)`；
- 空间复杂度：`O(V + E)`。

## R08

适用前提：
- 图是无向图；
- 图应连通（否则不存在覆盖全图的单棵 MST）；
- 允许负边权与平行边。

限制说明：
- 本实现遇到非连通图时抛异常，不返回最小生成森林。

## R09

与常见 MST 算法对比：
- Prim：点扩张 + 候选边最小堆，适合邻接表场景；
- Kruskal：全局边排序 + 并查集，适合边列表驱动；
- Boruvka：分量并行扩张，常用于并行/混合实现。

本条目主算法是 Prim，Kruskal 仅作为验证对照。

## R10

工程细节：
- 无向边必须双向写入邻接表；
- `visited` 判定避免生成环；
- 起点以虚拟父边入堆，减少分支判断；
- 用断言校验 `|mst_edges| = n-1` 与总权重一致性。

## R11

`demo.py` 模块拆分：
- `build_adjacency`：构建邻接表并做端点合法性检查；
- `prim_mst`：Prim 主实现；
- `UnionFind` / `kruskal_mst`：交叉验证实现；
- `normalize_edges`：无向边标准化输出；
- `main`：固定样例运行 + 断言 + 异常分支测试。

## R12

MVP 特性：
- 仅依赖 Python 标准库（`heapq`、`dataclasses`、`typing`）；
- 无需交互输入，直接运行得到稳定输出；
- 内置连通图样例和不连通异常样例；
- 覆盖“正确性 + 异常行为”两个维度。

## R13

内置样例图包含 6 个点、10 条边，预期 MST 总权重是 `11.0`。

脚本会验证：
- Prim 输出边数为 `n-1`；
- Prim 总权重为 `11.0`；
- Prim 与 Kruskal 的总权重一致。

## R14

实现不变量：
- 已访问顶点集始终连通；
- 每次只把一个新顶点通过最小跨割边接入，故不产生环；
- 每一步选边满足割性质；
- 最终边数为 `n-1` 且覆盖所有顶点，得到 MST。

## R15

边界与异常处理：
- `num_nodes == 0`：返回空边集和 `0.0`；
- `start` 不在合法范围：抛 `ValueError`；
- 任一边端点越界：抛 `ValueError`；
- 图不连通：抛 `ValueError`。

`main` 中显式覆盖了不连通分支。

## R18

源码级流程拆解（`demo.py`，8 步）：
1. `main` 定义固定无向带权图 `edges` 与 `num_nodes=6`。  
2. 调用 `prim_mst`，其内部先通过 `build_adjacency` 将每条无向边写入双向邻接表。  
3. `prim_mst` 初始化 `visited`、最小堆 `heap=[(0.0, start, -1)]`、`mst_edges` 与 `total_weight`。  
4. 循环弹出堆顶候选 `(weight, node, parent)`；若 `node` 已访问则丢弃。  
5. 对未访问顶点：标记访问、累加权重、记录父边，然后将其可扩展邻边压入堆。  
6. 当访问点数达到 `num_nodes` 时结束；若堆空但未覆盖全部顶点则抛出“图不连通”。  
7. `main` 再调用 `kruskal_mst`（并查集 + 边排序）得到参考总权重，执行与 Prim 的一致性断言。  
8. `main` 打印结果，并执行不连通分支测试，最终输出 `All assertions passed.`。
