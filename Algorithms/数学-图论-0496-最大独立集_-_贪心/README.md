# 最大独立集 - 贪心

- UID: `MATH-0496`
- 学科: `数学`
- 分类: `图论`
- 源序号: `496`
- 目标目录: `Algorithms/数学-图论-0496-最大独立集_-_贪心`

## R01

最大独立集（Maximum Independent Set, MIS）问题：
在无向图 `G=(V,E)` 中，寻找一个顶点集合 `S ⊆ V`，使得：
- 任意 `u,v ∈ S` 之间都没有边相连（独立性）；
- `|S|` 尽可能大（最大化）。

本条目采用**贪心启发式**做可运行 MVP：每轮从当前剩余子图中选择“诱导度最小”的顶点加入解集，再删除该点及其邻居。

## R02

问题性质与定位：
- MIS 是经典 NP-hard 组合优化问题；
- 大规模图上通常不能指望多项式时间精确求解；
- 因此工程上常用启发式得到“可行且较好”的近似解；
- 本实现明确目标是：快速得到**极大独立集**（maximal independent set），并用小规模精确解做对照评估。

## R03

术语区分（非常关键）：
- `最大独立集`（maximum）：在所有独立集中规模最大；
- `极大独立集`（maximal）：不能再加入新顶点，否则破坏独立性。

贪心法天然保证“极大”，但不保证“最大”。
`demo.py` 会展示一个固定图实例：贪心得到规模 5，而精确最优为 6。

## R04

本条目贪心策略（最小度优先）：
1. 在当前活跃顶点集合 `A` 上计算诱导子图度数；
2. 选取度最小顶点 `v`（并列时取编号最小，保证确定性）；
3. 将 `v` 放入独立集 `S`；
4. 从 `A` 删除 `v` 及其所有邻居；
5. 重复直到 `A` 为空。

直觉：优先选低度点，通常能减少“占坑成本”，给后续留下更多可选点。

## R05

复杂度（邻接矩阵实现）：
- 设顶点数为 `n`；
- 每轮需要在活跃子图上计算度数，最坏约 `O(n^2)`；
- 最多进行 `n` 轮；
- 整体最坏时间复杂度约 `O(n^3)`；
- 空间复杂度 `O(n^2)`（邻接矩阵主导）。

注：本实现偏重清晰可验证，而非极致性能。

## R06

微型示意：
若当前活跃子图中某点 `v` 度最小，选中 `v` 后，`v` 与其邻居都被移除，保证后续不会选到与 `v` 相邻的点。
最终得到的 `S` 自动满足：
- 任意两点不相邻（独立性）；
- 图中剩余点为空，或每个未选点都与 `S` 某点相邻（极大性）。

## R07

正确性保证（本贪心部分）：
- 独立性：每次把 `v` 放入 `S` 时，已删除其邻居，因此后续不会加入与 `v` 相邻顶点；
- 极大性：算法终止时没有活跃点，意味着任一未选顶点在某轮已作为“被选点的邻居”被删除，故无法再加入 `S`。

这两点在 `demo.py` 中由 `is_independent_set` 与 `is_maximal_independent_set` 断言验证。

## R08

MVP 额外加入“小规模精确基线”用于对照：
- 使用 `branch-and-bound + bitmask` 精确求解 MIS；
- 仅在 `n <= 28` 时启用（避免指数爆炸）；
- 不是黑盒库调用，源码中完整展开了分支、剪枝与回溯逻辑。

作用：
- 验证贪心输出合法；
- 量化贪心与最优解差距（`approximation_ratio = |S_greedy| / |S_opt|`）。

## R09

适用场景：
- 图规模较大，需要快速得到可行解；
- 可接受“次优但快”的策略；
- 需要稳定、可复现（本实现有确定性 tie-break）。

不适用场景：
- 必须拿到全局最优且图规模不小；
- 需要近似比有严格理论保证（本启发式未提供固定近似因子）。

## R10

实现里的输入/图模型假设：
- 无向简单图；
- 不允许自环；
- 邻接矩阵必须方阵且对称；
- 顶点使用 `0..n-1` 编号。

若输入非法，相关函数会抛 `ValueError`。

## R11

稳定性与可复现设计：
- 贪心并列时固定取最小编号，避免随机波动；
- 固定样例图写死在 `main`，每次运行输出一致；
- 关键性质使用断言：
  - 贪心结果独立；
  - 贪心结果极大；
  - 精确解独立；
  - 贪心大小不超过最优大小。

## R12

性能观察建议：
- 贪心部分通常很快，可用于较大图的初始解生成；
- 精确分支限界仅用于小图评估，不应直接用于大图生产场景；
- 若要扩展到大图，可改为邻接表 + 增量维护度数，降低常数开销。

## R13

本目录 MVP 的“可验证输出”包含：
- 图规模与密度；
- 贪心独立集顶点及规模；
- 精确最优独立集顶点及规模；
- 近似比；
- 结论提示（贪心可能次优）。

脚本最后打印 `All checks passed.` 表示断言全部通过。

## R14

常见失效模式与应对：
- 图不是无向简单图：输入校验直接报错；
- 图太大仍调用精确解：`max_n_for_exact` 限制保护运行时间；
- 误把“极大”当“最大”：报告中明确同时给出 greedy 与 exact 规模，避免误解。

## R15

`demo.py` 模块划分：
- `build_undirected_adjacency`：构图与合法性校验；
- `greedy_maximal_independent_set_min_degree`：核心贪心；
- `is_independent_set` / `is_maximal_independent_set`：性质检查；
- `exact_maximum_independent_set_branch_and_bound`：精确基线；
- `edge_count`：图统计；
- `main`：组织样例、断言与打印报告。

## R16

相关算法链路：
- 精确算法：分支限界、回溯、补图最大团法、整数规划；
- 启发式：随机贪心、多次重启、局部搜索（交换/扰动）；
- 特殊图类：二分图可借助 König 定理间接求解（通过最小点覆盖转换）。

## R18

`demo.py` 的源码级算法流程（9 步）如下：
1. `main` 中定义固定无向图边集，调用 `build_undirected_adjacency` 生成布尔邻接矩阵。  
2. 进入 `greedy_maximal_independent_set_min_degree`，初始化活跃顶点掩码 `active` 与解集 `chosen`。  
3. 每轮在活跃诱导子图中计算各顶点度数，取最小度顶点（并列取最小编号）作为本轮选择 `v`。  
4. 把 `v` 加入解集，并从 `active` 一次性删除 `v` 及其所有活跃邻居。  
5. 重复步骤 3-4 直到 `active` 为空，得到一个极大独立集（贪心输出）。  
6. 调用 `exact_maximum_independent_set_branch_and_bound`：把每个顶点邻居编码为位掩码，启动 DFS。  
7. DFS 在“选 pivot 顶点/不选 pivot 顶点”两分支间递归，并用 `|current| + |candidate|` 上界做剪枝。  
8. DFS 结束后得到最优位掩码并还原为最优顶点集合（精确输出）。  
9. `main` 用断言校验贪心独立性与极大性，再打印 greedy/exact 规模与近似比，形成可复现实验报告。
