## R01

B样条（B-spline）是分段多项式曲线/曲面表示方法。  
给定控制点、次数和节点向量，可在参数域上稳定地生成光滑几何对象。  
本条目目标：给出一个可运行 MVP，同时覆盖 `B样条曲线` 与 `张量积B样条曲面` 的数值计算。

## R02

核心思想：
1. 用节点向量把参数域分段，每段由次数为 `p` 的多项式描述。
2. 每个基函数 `N_{i,p}(u)` 只在局部节点区间非零，形成局部支撑。
3. 曲线是控制点的加权和；曲面是二维基函数的张量积加权和。
4. 数值求值常用 de Boor 算法，稳定性优于直接展开高次多项式。

## R03

输入输出约定（当前 MVP 为固定样例、无交互）：
- 输入：代码内固定的曲线控制点、曲面控制网格、次数、均匀夹持节点向量、采样参数网格。
- 输出：终端报告，包含
  - 自实现 de Boor 与 `scipy.interpolate.BSpline` 的曲线结果差异；
  - 基函数分片统一性（partition of unity）误差；
  - 局部支撑实验；
  - 曲面采样统计量（运行耗时、`z` 范围、中心点）。

## R04

典型应用：
- CAD/CAE/CAM 中的自由曲线与曲面建模。
- 机器人轨迹平滑、路径生成与约束插值。
- 计算机图形学中的曲面近似、动画路径和形状编辑。
- 数值分析中的样条拟合与几何参数化问题。

## R05

数学定义（非有理 B 样条）：
- 曲线：
  `C(u) = Σ_{i=0}^{n-1} N_{i,p}(u) P_i`
- 曲面（张量积形式）：
  `S(u,v) = Σ_{i=0}^{n_u-1} Σ_{j=0}^{n_v-1} N_{i,p}(u) M_{j,q}(v) P_{ij}`
- 基函数由 Cox-de Boor 递推给出，de Boor 算法是在给定 `u` 时对上述表达式的稳定求值实现。
- 夹持节点向量下，曲线通过首末控制点（端点插值性质）。

## R06

高层算法步骤：
1. 由控制点个数与次数构造夹持均匀节点向量。
2. 使用二分查找定位参数 `u` 所在节点区间（knot span）。
3. 对当前区间执行 de Boor 递推，得到单点曲线值。
4. 在参数序列上重复单点评估，得到整条曲线采样点。
5. 曲面计算时先沿 `u` 方向对每列控制点做 de Boor，形成临时控制列；
6. 再沿 `v` 方向对临时控制列做 de Boor，得到 `S(u,v)`。
7. 用 SciPy `BSpline` 做曲线侧对照，验证实现正确性。

## R07

正确性直觉：
- de Boor 本质是对活跃控制点做分层线性插值，等价于样条基函数加权和。
- 每一步插值权重由节点区间比例 `alpha` 决定，最终结果落在局部控制点凸包附近。
- 局部支撑保证“改一个控制点，只影响有限参数区间”。
- 基函数和为 1（partition of unity）保证仿射不变性和数值一致性。

## R08

时间复杂度（设曲线次数 `p`，控制点数 `n`，采样点数 `m`）：
- 单点 de Boor：`O(p^2)`（严格来说 span 查找 `O(log n)`，通常被 `p^2` 主导）。
- 曲线采样：`O(m * p^2)`。
- 曲面采样（`m_u x m_v`，次数 `p,q`，控制网 `n_u x n_v`）在当前直接实现下约为  
  `O(m_u * (n_v * p^2 + m_v * q^2))`。

## R09

空间复杂度：
- 曲线单点评估额外需要 `O(p)` 临时点。
- 曲线批量采样输出为 `O(m)`。
- 曲面采样输出为 `O(m_u * m_v)`，外加每个 `u` 切片 `O(n_v)` 的临时控制列。

## R10

MVP 实现策略：

- 依赖：`numpy`（数组与线性代数）；若环境有 `scipy`，则额外使用 `BSpline` 做外部对照。

## R13

边界与鲁棒性设计：
- `n_ctrl <= degree` 时抛出异常，避免非法样条配置。
- `find_span` 对边界参数（`u=0`、`u=1`）做显式处理。
- de Boor 递推中若出现零分母（重复节点）则按 `alpha=0` 处理，避免 NaN。
- 全部比较使用误差容差，不做浮点严格相等判断。

- 性质测试：基函数和是否接近 1；扰动局部性是否满足理论支撑区间。

## R17

方案对比：
- 纯库函数一行求值：开发快，但不透明，不利于算法教学与审计。
- 全手写基函数递推+求和：可解释，但实现细节多，性能和稳定性要额外小心。
- 本方案（de Boor 手写 + SciPy 对照）：兼顾可解释性、数值稳定和工程可验证性，适合 MVP 阶段。

## R18

`demo.py` 源码级算法流程（8 步）：
1. `make_clamped_uniform_knots` 根据控制点数量与次数生成夹持均匀节点向量，保证端点重节点。
2. `find_span` 对每个参数执行二分查找，找到 de Boor 递推所需的活动节点区间。
3. `de_boor_point` 取 `p+1` 个局部控制点，按层递推 `r=1..p` 进行线性插值，得到单个曲线点。
4. `evaluate_curve_de_boor` 在参数数组上重复步骤 2-3，形成整条曲线采样点集。
5. `curve_demo` 若检测到 SciPy，则用 `BSpline`（同节点/同控制点）生成对照曲线并计算最大差异；无 SciPy 时走纯 de Boor 路径，仍构造基函数矩阵验证 partition of unity。
6. `curve_demo` 扰动单个控制点，重新采样并比较差分，验证“局部支撑区间”与理论区间 `[t_i, t_{i+p+1}]` 对齐。
7. `evaluate_surface_de_boor` 对每个 `u` 先沿 `u` 方向做一组 de Boor，得到临时控制列；再沿 `v` 方向做 de Boor，得到 `S(u,v)`，即张量积曲面点。
8. `main` 汇总曲线/曲面误差与统计量，进行阈值判定并输出 `Validation: PASS/FAIL`。
