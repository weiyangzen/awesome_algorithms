## R01

分支定界（Branch and Bound, B&B）是求解整数规划的经典精确算法。
核心思想是：
- 用“分支”把原问题拆成更小的子问题；
- 用“定界”给每个子问题计算上界（或下界）并做剪枝；
- 持续更新当前最优整数可行解（incumbent），直到所有节点都被处理或剪掉。

本目录实现的是一个可审计的 MVP：
- 问题类型：`0-1` 整数线性规划（binary ILP）；
- 搜索框架：手写 best-bound B&B；
- 节点上界：优先使用 LP 松弛（`scipy.optimize.linprog`），无 SciPy 时退化到保守上界，保证脚本可运行。

## R02

MVP 求解模型：

- 决策变量：`x_i in {0,1}`，`i=1..n`
- 目标：`max c^T x`
- 约束：`A x <= b`

输入：
- `c`：目标系数向量，长度 `n`
- `A`：约束矩阵，形状 `m x n`
- `b`：约束右端，长度 `m`

输出：
- `best_x`：最优二元解（若存在）
- `best_obj`：对应最优目标值
- `stats`：节点访问与剪枝统计

## R03

数学要点：

1. 对最大化整数规划，任意节点的 LP 松弛最优值都是该节点整数最优值的上界。  
2. 若节点上界 `UB <= incumbent`，该节点及其后代可整体剪枝（bound pruning）。  
3. 若 LP 松弛解已全整数，该解即该节点最优整数解，可直接更新 incumbent。  
4. 若 LP 松弛解有分数变量，选择一个变量 `x_k` 分支：`x_k=0` 与 `x_k=1`。  
5. 对二元变量 + `A>=0` + `Ax<=b`，若某节点的 `A@lb > b`，则该节点必 infeasible，可提前剪枝。

## R04

算法流程（best-bound 版本）：

1. 构建根节点（`lb=0, ub=1`）。
2. 计算根节点上界（LP 松弛或保守上界）。
3. 用优先队列按“上界最高优先”取出节点。
4. 若节点上界不可能优于 incumbent，则剪枝。
5. 若节点已全变量固定，检查可行性并尝试更新 incumbent。
6. 若 LP 解已整数化，直接更新 incumbent 并停止展开该节点。
7. 否则按分支变量生成两个子节点（`x_k=0/1`）。
8. 对子节点做可行性与上界检查，保留有潜力节点继续搜索。
9. 队列为空时结束，返回全局最优整数解。

## R05

核心数据结构：

- `Node`：
  - `lb, ub`：节点变量上下界向量
  - `upper_bound`：节点上界
  - `relax_x`：LP 松弛解（可选）
  - `depth, node_id, bound_source`
- `heap`：优先队列，键为 `-upper_bound`（实现最大上界优先）。
- `stats`：统计量
  - `visited_nodes`
  - `generated_nodes`
  - `pruned_by_bound`
  - `pruned_infeasible`
  - `pruned_integral_lp`
  - `incumbent_updates`

## R06

正确性说明（MVP 级）：

- 完备性：
  - 分支把二元变量空间按赋值划分为互不重叠子空间；
  - 不剪枝时最终会覆盖全部可行二元解。
- 安全剪枝：
  - `UB <= incumbent` 时剪枝不会丢失更优整数解；
  - `A@lb > b` 时节点无可行扩展，剪枝安全。
- 最优性：
  - 搜索终止后，所有未处理节点都已被安全剪枝或处理完；
  - incumbent 即全局最优整数解。

## R07

复杂度：

- 最坏时间复杂度：指数级，`O(2^n)`（B&B 理论上界）。
- 节点 LP 松弛代价：取决于 LP 求解器，约为多项式时间。
- 实际表现依赖：
  - 上界紧度（LP 松弛越紧，剪枝越多）；
  - 分支变量选择策略；
  - 问题结构稀疏性与约束强度。
- 空间复杂度：
  - 活跃节点队列 + 节点边界副本，通常小于全树规模。

## R08

边界与异常处理：

- 输入维度不匹配：抛 `ValueError`。
- `c/A/b` 含 `nan` 或 `inf`：抛 `ValueError`。
- 本 MVP 要求 `A>=0`（用于快速下界可行性剪枝），不满足则抛 `ValueError`。
- LP 求解器不可用：自动使用保守上界，算法仍可运行。
- 节点数超过 `max_nodes`：返回 `max_nodes_reached`，表示提前停止而非错误。

## R09

MVP 取舍：

- 仅覆盖 `0-1` ILP，不做一般整数/混合整数变量类型。
- 仅实现 `Ax<=b` 形式，不含等式约束与显式大 M 建模助手。
- 分支策略使用“最分数变量优先（有 LP）/目标系数优先（无 LP）”，保持实现简洁。
- 无 SciPy 时使用保守上界（忽略约束）以保证可运行，剪枝会更弱但结果仍正确。

## R10

`demo.py` 函数职责：

- `validate_binary_ilp`：输入校验。
- `is_integral_vector`：判断 LP 解是否整数化。
- `partial_infeasible`：快速检查节点下界是否已违反约束。
- `solve_lp_relaxation`：调用 `linprog` 求节点 LP 松弛上界。
- `build_node`：统一构造节点与上界计算。
- `choose_branch_variable`：选择分支变量。
- `branch_and_bound_binary_ilp`：主求解循环。
- `brute_force_binary_ilp`：小规模暴力校验器。
- `main`：固定样例运行与结果打印。

## R12

输出字段说明：

- `Status`：`optimal` 或 `max_nodes_reached`。
- `LP relaxation available`：当前环境是否启用 `linprog` 上界。
- `Best objective`：B&B 找到的最优目标值。
- `Best x`：对应二元向量。
- `Resource usage A@x`：解在各约束上的资源占用。
- `Node statistics`：搜索规模与各类剪枝次数。
- `Brute-force best objective / x`：小样例精确对照结果。
- `Objective gap`：B&B 与暴力校验的目标值差。

## R13

内置样例（固定）：

- 变量数：8（项目是否选择）
- 约束数：3（多预算上限）
- 目标 `c = [20, 24, 15, 40, 25, 30, 18, 12]`
- 约束矩阵：
  - `[2,3,1,4,2,3,2,1] <= 10`
  - `[3,1,2,5,3,2,1,2] <= 11`
  - `[1,2,2,1,2,1,1,2] <= 7`

该样例规模小，便于用暴力枚举 `2^8=256` 进行交叉校验。

## R14

关键参数：

- `max_nodes`：最大访问节点数（默认 `5000`）。
- `tol`：目标比较和可行性容差（默认 `1e-9`）。
- `INT_TOL`：整数化判定容差（默认 `1e-9`）。

调参建议：

- 若提前到达 `max_nodes_reached`，增大 `max_nodes`。
- 若数值抖动导致“几乎整数”无法识别，可适当放宽 `INT_TOL`。
- 若约束规模增大，建议保留 LP 上界（安装 SciPy）以增强剪枝。

## R15

与其他方法对比：

- 对比纯暴力枚举：
  - 暴力法简单但指数爆炸；
  - B&B 借助上界可大量剪枝。
- 对比动态规划：
  - DP 对某些特殊结构（如单背包）高效；
  - B&B 对一般线性约束更通用。
- 对比商用 MIP 求解器：
  - 商用品质更强（割平面、启发式、并行）；
  - 本实现胜在透明、可教学、可追踪源码逻辑。

## R16

典型应用：

- 二元选址与设施启停决策。
- 生产计划中的开关型约束决策。

## R18

`demo.py` 源码级流程（9 步）：

1. `main` 定义固定 `0-1` ILP 实例并调用 `branch_and_bound_binary_ilp`。  
2. `validate_binary_ilp` 校验维度、数值有效性与 `A>=0` 的 MVP 假设。  
3. `build_node` 创建根节点：`lb=0, ub=1`，并计算节点上界。  
4. 若环境可用 SciPy，`solve_lp_relaxation` 调 `linprog` 求 LP 松弛最优值作为上界；否则用保守上界替代。  
5. 主循环从优先队列取“上界最高”节点，先做 `upper_bound <= incumbent` 的定界剪枝。  
6. 若节点已全变量固定，直接验证 `Ax<=b` 并更新最优整数解。  
7. 若 LP 松弛解已整数化，直接作为该节点最优整数候选更新 incumbent，并停止展开该节点。  
8. 否则 `choose_branch_variable` 选择分支变量，生成 `x_k=0` 与 `x_k=1` 两子节点，再调用 `build_node` 做可行性与上界过滤。  
9. 队列耗尽后返回最优解和统计信息，并用 `brute_force_binary_ilp` 对小样例做精确交叉校验。
