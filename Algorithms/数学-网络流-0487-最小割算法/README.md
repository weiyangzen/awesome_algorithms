# 最小割算法

- UID: `MATH-0487`
- 学科: `数学`
- 分类: `网络流`
- 源序号: `487`
- 目标目录: `Algorithms/数学-网络流-0487-最小割算法`

## R01

最小割（Minimum Cut）是网络流中的核心问题：在有向带容量图中，给定源点 `s` 和汇点 `t`，把点集划分为 `S` 与 `T`（`s in S, t in T`），使从 `S` 指向 `T` 的边容量总和最小。

它与最大流满足最大流最小割定理：
- 任意可行流值 `<=` 任意 `s-t` 割容量；
- 存在一组流与割使二者相等。

因此工程上通常通过求最大流来得到最小割。

## R02

本条目 MVP 目标：
- 实现 Edmonds-Karp（BFS 增广路）求 `s-t` 最大流；
- 从最终残量网络提取一个最小割 `S/T`；
- 输出割边集合与割容量；
- 用小规模穷举割做交叉校验，验证 `max_flow == min_cut`。

范围限定：
- 图为有向图；
- 容量为非负整数；
- 固定内置样例，无需交互输入。

## R03

问题形式化：
- 输入：`G=(V,E)`，容量函数 `c(u,v) >= 0`，源点 `s`，汇点 `t`。
- 割定义：`(S,T)` 满足 `S U T = V`，`S ∩ T = 空`，`s in S`，`t in T`。
- 割容量：
  `cap(S,T) = sum_{u in S, v in T} c(u,v)`。
- 目标：最小化 `cap(S,T)`。

MVP 输出：
- 最大流值；
- 最小割容量；
- 一个可解释的最小割划分与割边列表。

## R04

Edmonds-Karp 核心思想：
- 在残量网络中反复用 BFS 找最短（按边数）增广路；
- 每次沿路径增加可行流量（瓶颈容量）；
- 同步更新正向残量与反向残量；
- 当 BFS 无法从 `s` 到达 `t` 时停止，此时已是最大流。

在这个终止状态下：
- 从 `s` 出发在残量网络可达的点集合记为 `S`；
- 其余点为 `T`；
- 原图中所有 `S -> T` 边构成一个最小割。

## R05

本实现的数据结构：
- `capacity[n][n]`：原图容量矩阵（支持重边自动累加）；
- `residual[n][n]`：残量容量矩阵；
- `parent[n]`：BFS 记录增广路前驱；
- `visited[n]`：在残量网络中标记可达点；
- `MinCutResult`（`dataclass`）：统一保存最大流/最小割结果。

选用矩阵是为了 MVP 代码直观、步骤可追踪；复杂场景可替换为邻接表边结构。

## R06

增广步骤（单轮）如下：
1. 在 `residual` 上 BFS 找 `s -> t` 路径，记录 `parent`；
2. 沿 `parent` 回溯到 `s`，取路径最小残量作为瓶颈 `delta`；
3. 正向边执行 `residual[u][v] -= delta`；
4. 反向边执行 `residual[v][u] += delta`；
5. 累加 `max_flow += delta`。

该循环直到找不到增广路。

## R07

最小割提取：
- 在终止残量网络中，从 `s` 只沿 `residual[u][v] > 0` 的边做一次 BFS/DFS；
- 可达点为 `S`，不可达点为 `T`；
- 遍历原容量矩阵，收集所有满足 `u in S, v in T, capacity[u][v] > 0` 的边；
- 这些边容量之和即割容量。

该割容量应等于前一步得到的最大流值（定理保证）。

## R08

复杂度分析（设点数 `V`，边数 `E`）：
- Edmonds-Karp：`O(V * E^2)`；
- 提取可达集：`O(E)`（矩阵实现下写成 `O(V^2)` 扫描）；
- 穷举校验（仅演示小图）：`O(2^(V-2) * E)`。

MVP 中穷举仅用于小规模样例自检，不用于大图生产环境。

## R09

正确性要点：
- 每轮增广都保持流守恒与容量约束；
- 无增广路时，残量网络不存在从 `s` 到 `t` 的可达路径，表示当前流已不可再提升；
- 由残量可达集构造的 `S/T` 割，其容量等于当前流值；
- 因流值从不超过任意割容量，故该割必为最小割。

脚本通过“最大流值 == 残量提取割值 == 穷举最小割值”三重一致性断言，增强可验证性。

## R10

边界与实现细节：
- 若 `s == t`，最小割定义会退化，MVP 直接视为非法输入；
- 容量必须非负；
- 可存在孤立点、零容量边、平行边（通过矩阵累加处理）；
- 图可以不强连通，只要有定义良好的 `s` 与 `t`；
- 为简洁起见，本实现使用整数容量，浮点容量不在本 MVP 范围。

## R11

`demo.py` 模块划分：
- `build_capacity_matrix`：把边列表转成容量矩阵；
- `bfs_augmenting_path`：在残量网络中找增广路并返回瓶颈；
- `edmonds_karp_max_flow`：主流程，返回最大流与最终残量；
- `extract_min_cut_from_residual`：从残量网络恢复 `S/T` 与割边；
- `brute_force_min_cut`：小图穷举最小割，用于交叉验证；
- `run_case`：执行单个样例并断言一致性；
- `main`：组织固定测试样例。

## R12

内置样例设计：
- `classic_clrs`：经典 6 节点网络，已知最优值为 `23`；
- `parallel_edges`：包含平行边，验证容量累加与割提取；
- `sparse_chain_like`：较稀疏结构，验证非完全图场景。

每个样例都会输出：
- 最大流值；
- `S/T` 划分；
- 割边列表；
- 残量提取最小割值与穷举最小割值。

## R14

输出解读：
- `max_flow`：Edmonds-Karp 得到的最大流值；
- `min_cut_by_residual`：由残量可达集提取的割容量；
- `min_cut_by_bruteforce`：穷举所有 `s-t` 划分得到的最小割容量；
- `cut_edges`：对应 `S -> T` 的边与容量。

三者数值一致，说明实现符合最大流最小割定理。

## R15

常见错误：
- 更新残量时忘记加反向边容量；
- BFS 只找路径不记录前驱，导致无法回溯瓶颈；
- 把“原图容量”与“残量容量”混用；
- 提取割边时错误地从残量矩阵取边，而不是从原容量矩阵取 `S -> T` 边；
- 误把无向图直接当有向图处理，导致语义偏差。

## R16

与其他算法对比：
- Ford-Fulkerson（DFS 增广）实现更短，但复杂度受路径选择影响，最坏情况不稳定；
- Edmonds-Karp 用 BFS 固定增广策略，复杂度上界清晰，适合教学与中小规模应用；
- Dinic 在大图上通常更快（分层图 + 阻塞流），但实现复杂度更高。

本条目选择 Edmonds-Karp，优先可读性与可验证性。

## R17

可扩展方向：
- 切换到 Dinic / Push-Relabel 以提升性能；
- 支持多源多汇（通过超级源/超级汇转换）；
- 支持最小割边集之外的割点恢复与可视化；
- 扩展到最小割在图像分割、可靠性分析中的应用示例；
- 增加随机图回归测试与性能基准。

## R18

`demo.py` 源码级流程可拆为 8 步：
1. `main` 构造固定测试图（点数、边集、源汇点）并逐例调用 `run_case`。
2. `run_case` 先用 `build_capacity_matrix` 生成原容量矩阵，重边容量自动累加。
3. 调用 `edmonds_karp_max_flow`：循环执行 `bfs_augmenting_path` 查找增广路。
4. 每次找到增广路后，沿 `parent` 回溯瓶颈值 `delta`，并同步更新正反向残量，累加最大流。
5. 无法再到达汇点时停止增广，得到最终残量网络与 `max_flow`。
6. 调用 `extract_min_cut_from_residual`：从源点在残量网络做可达搜索，得到 `S/T`，并在原容量矩阵中提取 `S->T` 割边与割容量。
7. 调用 `brute_force_min_cut` 穷举全部 `s-t` 划分，得到理论最小割值与一个最优划分。
8. 对三者做断言：`max_flow == min_cut_by_residual == min_cut_by_bruteforce`，全部样例通过后输出完成信息。
