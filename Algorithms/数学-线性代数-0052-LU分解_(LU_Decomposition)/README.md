# LU分解 (LU Decomposition)

- UID: `MATH-0052`
- 学科: `数学`
- 分类: `线性代数`
- 源序号: `52`
- 目标目录: `Algorithms/数学-线性代数-0052-LU分解_(LU_Decomposition)`

## R01

LU 分解把矩阵分解为下三角矩阵与上三角矩阵的乘积，是线性代数中求解线性方程组、行列式计算和矩阵逆求解的核心工具。  
在工程场景中通常使用带部分主元选取（partial pivoting）的形式：
\[
PA = LU
\]
其中 `P` 为置换矩阵，`L` 为单位下三角矩阵，`U` 为上三角矩阵。

## R02

本条目解决的问题：
- 输入：实数方阵 `A`，以及右端向量 `b`（用于演示线性方程求解）。
- 输出：`P, L, U` 满足 `P @ A = L @ U`，并给出 `Ax=b` 的数值解 `x`。

`demo.py` 使用固定样例矩阵，无需交互输入，可直接运行。

## R03

核心数学关系：
- 通过高斯消元把 `A` 逐步化为上三角矩阵 `U`。
- 消元乘子写入 `L` 的下三角部分，并令 `diag(L)=1`。
- 为避免零主元或极小主元造成不稳定，在每一列选择绝对值最大的主元并交换行，形成置换矩阵 `P`。

因此得到：
\[
PA = LU
\]
进一步，线性方程 `Ax=b` 可转写为：
\[
LUx = Pb
\]
先解 `Ly=Pb`（前代回），再解 `Ux=y`（后代回）。

## R04

算法高层流程：
1. 检查 `A` 是否是有限实数方阵。
2. 初始化 `P=I, L=I, U=A`。
3. 对每一列 `k` 做部分主元选取，确定并交换主元行。
4. 用主元行消去其下方元素，把消元倍数写入 `L[i,k]`。
5. 全部列处理后得到 `P, L, U`。
6. 给定 `b` 时，先求 `Ly=Pb`，再求 `Ux=y`。
7. 输出重构误差、求解残差以及与 `numpy.linalg.solve` 的解差。

## R05

核心数据结构：
- `a: np.ndarray (n, n)`：输入矩阵。
- `p: np.ndarray (n, n)`：行交换记录矩阵。
- `l: np.ndarray (n, n)`：单位下三角矩阵。
- `u: np.ndarray (n, n)`：上三角矩阵。
- `swap_count: int`：交换次数，用于估算行列式符号。

MVP 只处理二维密集实矩阵，避免引入稀疏矩阵框架复杂度。

## R06

正确性要点：
- 每次消元均满足“当前列主元以下元素被置零”，因此 `U` 保持上三角结构。
- 记录的消元倍数与实际消元操作一致，故 `L` 精确反映消元过程。
- 行交换同步更新 `P` 与 `L` 已完成列，保证等式始终保持 `P @ A = L @ U`。
- 线性系统求解由三角系统组成：`Ly=Pb` 与 `Ux=y`，在非奇异前提下解唯一。

## R07

复杂度分析（`n x n`）：
- LU 分解时间复杂度：`O(n^3)`。
- 前代回与后代回总复杂度：`O(n^2)`。
- 空间复杂度：`O(n^2)`（存储 `P, L, U`）。

## R08

边界与异常处理：
- 非方阵或含 `nan/inf`：抛出 `ValueError`。
- `tol <= 0`：抛出 `ValueError`。
- 某列主元绝对值不超过 `tol`：判定为数值奇异，抛出 `ValueError`。
- 三角求解阶段若出现接近零对角元：抛出 `ValueError`。
- `b` 维度不匹配：抛出 `ValueError`。

## R09

MVP 取舍：
- 采用手写“Doolittle + 部分主元”流程，不把 LU 本体交给第三方黑箱。
- 仅使用 `numpy`，保持依赖最小化。
- 使用 `numpy.linalg.solve` 只作校验，不替代主算法实现。

## R10

`demo.py` 函数职责：
- `validate_square_matrix`：校验并标准化输入矩阵。
- `lu_decompose_partial_pivot`：执行 `PA=LU` 分解并返回交换次数。
- `forward_substitution`：求解 `Ly=b`。
- `backward_substitution`：求解 `Ux=y`。
- `solve_with_lu`：组合前后代回求 `Ax=b`。
- `main`：构造样例、打印分解结果和质量指标、演示异常路径。

## R12

输出字段说明：
- `P / L / U`：分解结果矩阵。
- `||P@A - L@U||_F`：分解重构误差。
- `||A@x - b||_2`：方程求解残差。
- `||x_lu - x_numpy||_2`：手写 LU 解与 NumPy 参考解差距。
- `det(A) from LU` 与 `det(A) from NumPy`：行列式一致性检查。

## R13

建议最小测试集：
- 一般可逆矩阵（本脚本默认样例）。
- 首列首元为 0 的矩阵，验证主元交换是否生效。
- 奇异矩阵（如两行线性相关）应触发失败。
- 随机多组矩阵，对照 `numpy.linalg.solve` 检查误差水平。

## R14

可调参数：
- `tol`：主元与对角元零判定阈值，默认 `1e-12`。
- 样例矩阵规模 `n`：可从 `4` 扩展到更大规模观察复杂度。
- 打印精度：`np.set_printoptions` 控制展示格式。

工程建议：若数据噪声较大，可适当放宽 `tol`，但应同步检查误判风险。

## R15

与相关方法对比：
- 与 Cholesky：Cholesky 仅适用于对称正定矩阵，LU 适用范围更广。
- 与 QR：QR 通常更稳定但代价更高，LU 在方程组求解上更高效。
- 与无主元 LU：部分主元选取显著提升数值稳定性，是工程默认选项。

## R16

典型应用：
- 大量线性方程组求解（同一 `A`、不同 `b`）。
- 行列式计算（由 `U` 对角积和置换奇偶性得到）。
- 作为更复杂算法的子模块（隐式积分、优化、控制、PDE 离散求解）。

## R17

可扩展方向：
- 完全主元或 rook pivoting，进一步增强稳定性。
- 块 LU（block LU）提升缓存友好性。
- 稀疏 LU 与重排序策略，适配大规模稀疏系统。
- 并行/GPU LU，支持更大规模数值计算。

## R18

`demo.py` 源码级算法流程（8 步）：
1. `main` 构造固定矩阵 `A` 与向量 `b`，设置打印精度并调用 LU 分解。  
2. `lu_decompose_partial_pivot` 先校验 `A`，初始化 `P=I, L=I, U=A.copy()`。  
3. 对每一列 `k`，在 `U[k:,k]` 中选绝对值最大元素作为主元；若主元过小则报奇异错误。  
4. 若主元行不是 `k`，同步交换 `U` 的整行、`P` 的整行，以及 `L` 的前 `k` 列已完成部分。  
5. 对 `i=k+1..n-1`，计算消元乘子 `L[i,k]=U[i,k]/U[k,k]`，并执行 `U[i,k:]-=L[i,k]*U[k,k:]`。  
6. 分解完成后，`solve_with_lu` 先计算 `pb = P @ b`，再用 `forward_substitution` 求 `Ly=pb`。  
7. 用 `backward_substitution` 求 `Ux=y` 得到最终 `x`，并做对角元近零检查。  
8. `main` 计算并打印 `||P@A-L@U||_F`、`||A@x-b||_2`、与 `numpy.linalg.solve` 的解差，以及奇异矩阵失败演示。
