# 多点求值

- UID: `MATH-0566`
- 学科: `数学`
- 分类: `算法`
- 源序号: `566`
- 目标目录: `Algorithms/数学-算法-0566-多点求值`

## R01

多点求值（Multipoint Evaluation）问题：给定一个多项式 `P(x)` 和一组点 `x_1, x_2, ..., x_m`，要求一次性计算所有 `P(x_i)`。  
本条目的目标是实现一个可运行 MVP，并展示比“逐点 Horner”更结构化的算法：乘积树 + 余数树。

## R02

核心思想分两层：
1. 乘积树（Product Tree）：把每个点对应的线性因子 `(x - x_i)` 两两相乘，递归构成一棵树，根节点是 `M(x)=∏(x-x_i)`。
2. 余数树（Remainder Tree）：从根开始，把 `P(x)` 对子树多项式取余并向下传播，叶子处得到对 `(x-x_i)` 的余数常数，即 `P(x_i)`。

## R03

输入输出约定（当前 MVP 为固定样例、无交互）：
- 输入：代码内固定多项式系数（升幂存储）与 25 个等间隔点。
- 输出：终端报告，包括误差对比（fast vs naive / numpy）、耗时、若干样本值和 PASS/FAIL 校验。

## R04

典型应用：
- 计算机代数系统中的批量求值子模块。
- 多项式插值、快速卷积、编码理论中的中间步骤。
- 需要同时在大量采样点上评估同一多项式的工程场景。

## R05

数学要点：
- 单点 Horner 复杂度 `O(n)`（`n` 为多项式次数），`m` 个点总复杂度 `O(nm)`。
- 多点求值可用“构树 + 取余”实现；若多项式乘法/除法使用快速算法，可逼近 `O((n+m) log^2 m)` 等级。
- 本 MVP 使用朴素多项式乘法和长除法，理论复杂度未达 FFT 级最优，但流程完整且易验证。

## R06

高层算法步骤：
1. 为每个点构造叶子多项式 `L_i(x)=x-x_i`。
2. 自底向上构造乘积树，得到每个内部节点的乘积多项式。
3. 从根节点开始，把当前余式对左右子多项式分别取模。
4. 递归下推到叶子。
5. 叶子余式应为常数，该常数就是对应点的函数值。

## R07

正确性直觉：
- 在任意节点，传递的余式 `R(x)` 与原多项式 `P(x)` 在该节点模多项式意义下同余。
- 叶子节点模多项式为 `(x-x_i)`，任何多项式对其取余都等价于常数 `P(x_i)`（余式定理）。
- 因此叶子处收集到的常数就是全部目标值，且每个点恰好对应一个叶子，无遗漏无重复。

## R08

时间复杂度（本实现使用朴素多项式运算）：
- 构建乘积树：大量朴素乘法，整体可近似看作 `O(m^2)` 到 `O(m^2 log m)` 量级（与树形分布相关）。
- 余数下推：每层包含若干次朴素取模，整体复杂度同样受朴素长除法影响，通常高于理想快速算法。
- 对于中小规模演示，此开销可接受，且比逐点独立求值更贴近标准多点求值框架。

## R09

空间复杂度：
- 乘积树存储每个节点多项式，主导空间消耗。
- 结果数组 `O(m)`。
- 综上大致为 `O(m log m)` 节点结构 + 多项式系数存储；在朴素实现中常数较大但规模可控。

## R10

MVP 实现策略：
- 语言：Python 3。
- 依赖：`numpy`（用于生成测试点与第三方结果对照），其余为标准库。
- 原则：算法透明、代码短小、结果可复核，不依赖黑箱一行求值接口作为主实现。

## R12

输出字段说明：
- `Polynomial degree`: 多项式次数。
- `Point count`: 评估点数量。
- `Max |fast-naive|`: 多点求值结果与逐点 Horner 的最大绝对误差。
- `Max |fast-numpy|`: 多点求值结果与 `numpy.polyval` 的最大绝对误差。
- `Fast/Naive method time`: 两种实现的粗略耗时对比。
- `Validation`: 误差阈值校验结果。

## R13

边界与鲁棒性：
- 空点集时直接返回空列表。
- 多项式系数通过 `poly_trim` 清理高位近零项，减少数值噪声。
- `poly_mod` 对零多项式除数做显式保护（抛出异常）。
- 浮点运算存在舍入误差，因此校验使用容差阈值而非严格相等。

## R14

当前局限：
- 乘法与取模都为朴素实现，未使用 FFT/NTT 或半 GCD 等快速代数算法。
- 主要用于方法演示，不适合超大规模高精度场景。
- 使用浮点系数时会受到条件数与舍入误差影响，极端输入下稳定性有限。

## R15

可扩展方向：
- 用 FFT 多项式乘法替换 `poly_mul`。
- 将长除法优化为分治余式计算，降低整体复杂度。
- 支持模素数域（如 `mod p`）运算，便于竞赛和密码学应用。
- 增加随机化测试与基准脚本，形成小型性能回归体系。

## R16

最小测试建议：
- 功能测试：对低次数多项式和少量点，手工可验结果。
- 一致性测试：`fast` 与 `naive` 逐项比较，误差需在阈值内。
- 基准测试：固定次数与点数，重复运行比较两种方法耗时趋势。
- 边界测试：空点集、常数多项式、重复点、包含负数/小数点坐标。

## R17

方案对比：
- 逐点 Horner：实现最简单，复杂度 `O(nm)`，适合 `m` 很小场景。
- 黑箱库批量求值：开发快但难解释内部代数流程，不利于算法学习。
- 本方案（乘积树+余数树）：结构清晰、可扩展到快速代数版本，教学与工程过渡价值更高。

## R18

`demo.py` 源码级流程（8 步）：
1. `main()` 定义测试多项式 `poly`（升幂系数）与点集 `points`，并调用 `multipoint_eval`。
2. `multipoint_eval` 先用 `build_product_tree` 构造二叉乘积树；叶子为 `[-x_i, 1]`，内部节点为左右子多项式乘积。
3. 递归函数 `descend(node, rem)` 从根开始下推余式，根节点初始余式为原多项式。
4. 在每个内部节点，分别计算 `left_rem = rem mod left.poly`、`right_rem = rem mod right.poly`。
5. `poly_mod` 使用朴素多项式长除法：按最高次项消元，最终保留低于除式次数的部分作为余式。
6. 递归到叶子后，叶子多项式是 `(x-x_i)`，余式应为常数，取 `rem[0]` 写入对应结果槽位。
7. `main()` 同时调用 `naive_eval`（逐点 Horner）和 `numpy.polyval` 做双重对照，计算最大绝对误差。
8. 打印次数、点数、误差、耗时和样本点值，并依据误差阈值输出 `PASS/FAIL`。
