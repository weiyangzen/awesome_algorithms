# 自适应Gauss-Kronrod积分

- 分类: `数值分析`

## R01

自适应 Gauss-Kronrod 积分用于高精度近似
`I = \int_a^b f(x)\,dx`。

本条目使用经典嵌套对 `G7-K15`：

- 先用 15 点 Kronrod 规则得到 `K15(a,b)`。
- 同一批采样点中包含 7 点 Gauss 子规则，得到 `G7(a,b)`。
- 用 `|K15-G7|` 作为局部误差估计。
- 若误差超阈值，则把区间二分并递归/迭代细分。

核心思想是“把采样预算集中到难积分的局部区间”，而不是全区间均匀加密。

## R02

- 高斯求积（Gauss quadrature）通过优化节点与权重，在固定采样点数下提高多项式积分精度。
- Kronrod 扩展（1960s）在 Gauss 节点上插入额外节点，构成嵌套求积对，使一次采样同时给出“高阶估计 + 低阶估计”。
- 该结构后来成为 QUADPACK 等数值积分实现的核心构件之一（如 `QAG` 家族思想）。

## R03

很多积分场景存在“局部困难”而非“全局困难”，例如：

- 端点附近弱奇异（如 `x^{-1/2}`）。
- 局部尖峰或快速变化。
- 不同区间平滑度差异明显。

如果用统一步长，通常会在平滑区浪费大量采样。自适应 Gauss-Kronrod 通过局部误差驱动细分，能在相同函数评估预算下显著提升精度/稳定性。

## R04

设最终被处理的子区间数量为 `m`。

- 每个子区间使用一次 `G7-K15`，需要 15 次函数评估（中心点 1 次 + 对称点 7 对）。
- 总函数评估约为 `15m`，时间复杂度可写为 `O(m)`。
- `m` 由函数难度与容差共同决定，不是预先固定值。

相比固定网格法，复杂度是“数据依赖”的：平滑问题 `m` 小，复杂问题 `m` 自动增大。

## R05

空间复杂度主要来自待处理区间栈。

- 迭代实现下，额外空间为栈大小 `O(s)`。
- 最坏情况下 `s` 可到 `O(m)`；常见情况下更接近 `O(depth)`。
- 除栈外只维护常量个累加变量（积分值、误差估计、计数器）。

## R06

示例：
`I = \int_0^1 \frac{1}{1+10000(x-0.05)^2}\,dx`。

该函数在 `x=0.05` 附近有尖峰，初始整段误差估计通常偏大；算法会优先细分尖峰附近区间。

典型行为：

- 远离尖峰的区间很快通过误差门限。
- 尖峰附近区间被多次二分，投入更多采样。
- 最终在较少总体采样下达到目标精度。

这正是“自适应积分”相对均匀剖分的核心优势。

## R07

方法重要性体现在：

- 作为工程数值积分的通用主力算法之一。
- 对“平滑 + 局部困难”混合场景有稳定表现。
- 容易给出误差估计（虽然是估计量，不是严格上界）。
- 可作为更高维自适应积分、概率计算和科学仿真的基础模块。

## R08

前置知识：

- 定积分与数值求积基础。
- Gauss-Legendre 求积节点/权重思想。
- Kronrod 嵌套扩展思想（同一采样复用得到两阶估计）。
- 绝对误差/相对误差容差机制。
- 区间二分与栈式递归展开。

## R09

适用条件与边界：

- 目标函数在区间内部应可稳定评估（本实现不支持主值积分、无穷区间变换等高级场景）。
- 允许弱奇异，但若奇异性太强、容差过严，可能触发 `max_depth` 或 `max_intervals` 上限。
- 对极端振荡积分，单纯二分可能效率有限，实际工程常结合振荡专用变换。

## R10

正确性直觉（工程版）：

1. `K15` 与 `G7` 都是对同一区间积分的合法求积近似。
2. 两者差值 `|K15-G7|` 可反映局部离散误差量级。
3. 当某区间误差不达标时，二分后每段函数行为更接近局部低阶多项式，误差通常下降。
4. 反复细分并累计“已达标子区间”结果，可得到全局近似。
5. 在函数可积且数值评估稳定时，该流程通常收敛到目标值。

## R11

数值稳定性要点：

- 使用中心点和对称点配对求和，能减轻部分舍入误差。
- 误差估计采用 `|K15-G7|`，对光滑函数通常可靠；对强抵消问题可能偏乐观或偏保守。
- `demo.py` 中加入了一个机器精度保护下界，避免出现不合理的“零误差”判断。
- 通过 `max_depth` 和 `max_intervals` 避免无限细分。

## R12

实际性能观察：

- 热点在函数评估，不在权重线性组合。
- 如果 `f(x)` 计算很贵（如仿真调用），自适应策略的采样节省非常关键。
- 若 `f(x)` 很便宜，Python 层控制流和栈操作也会成为可见开销。
- 本 MVP 优先可审计与稳健，不做并行化与缓存复用优化。

## R13

误差与收敛说明：

- 局部误差代理：`e_local = |K15 - G7|`。
- 局部通过条件：`e_local <= abs_tol_local + rel_tol * |K15|`。
- 全局误差估计：各已接受区间 `e_local` 的和（启发式，可用于工程监控）。

注意：该误差并非严格数学上界，但在常见平滑/弱奇异问题中，通常有较高实用可信度。

## R14

鲁棒性与失效模式：

- 失效 1：`max_depth` 达到后仍误差超阈值。
  - 处理：提高 `max_depth`，或放宽容差，或做变量代换。
- 失效 2：区间数量超过 `max_intervals`。
  - 处理：检查被积函数是否有尖点/奇异，必要时分段积分。
- 失效 3：函数在某些点返回 NaN/Inf。
  - 处理：预处理定义域，或在调用前做安全包装。

## R15

实现细节（对应 `demo.py`）：

- `qk15_pair`：在单区间上计算 `K15`、`G7`、局部误差估计与评估次数。
- `adaptive_gauss_kronrod`：使用显式栈做自适应细分，累计积分与误差。
- 容差策略：绝对容差沿二分树等分；相对容差按局部 `|K15|` 评估。
- 防护策略：`max_depth`、`max_intervals`、机器精度误差下界。

## R18

源码级流程拆解（`demo.py`，非第三方黑箱）:

1. 初始化 `G7-K15` 的节点与权重常量，并建立“Gauss 节点是 Kronrod 子集”的索引映射。
2. 在 `qk15_pair` 中把 `[a,b]` 仿射映射到中心点 `c` 与半长度 `h`，按对称节点采样并累积 `K15` 与 `G7`。
3. 计算单区间误差代理 `e = |K15-G7|`，再施加机器精度下界保护。
4. 在 `adaptive_gauss_kronrod` 中创建待处理栈，初始仅含整段区间与根绝对容差。
5. 弹出一个区间后执行 `qk15_pair`，得到该段积分估计和误差估计。
6. 若 `e <= abs_tol_local + rel_tol*|K15|`，则接受该段并累加到全局结果；否则进入细分。
7. 对不达标区间做二分，把左右子区间压栈，并将该段绝对容差平分给两个子区间。
8. 重复第 5-7 步直到栈为空，或触发深度/区间数上限保护。
9. 汇总全局积分值、误差估计、评估统计与收敛标记，供主程序打印和断言。
