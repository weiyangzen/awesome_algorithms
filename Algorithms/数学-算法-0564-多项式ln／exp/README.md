# 多项式ln/exp

- UID: `MATH-0564`
- 学科: `数学`
- 分类: `算法`
- 源序号: `564`
- 目标目录: `Algorithms/数学-算法-0564-多项式ln／exp`

## R01

本条目讨论形式幂级数上的多项式（严格说是截断幂级数）`ln/exp` 运算：
- `ln(A(x))`：要求常数项 `A(0)=1`。
- `exp(F(x))`：要求常数项 `F(0)=0`。

目标是提供一个可运行、可复核的 Python MVP，展示经典实现路径，而不是把第三方库当黑箱直接调用。

## R02

核心恒等式：
- `ln(A) = ∫ (A'/A) dx`
- `exp(F)` 可通过 Newton 迭代解方程 `ln(G)=F`。

因此需要的基础模块包括：
1. 截断卷积（多项式乘法 mod `x^n`）
2. 多项式求导/积分
3. 多项式逆元 `A^{-1}`（Newton 迭代）
4. 组合得到 `poly_ln` 与 `poly_exp`

## R03

输入输出约定（MVP 无交互输入）：
- 输入：代码中固定两组系数
1. `f`（常数项为 0）用于 `exp(f)` 与 `ln(exp(f))` 回归
2. `a`（常数项为 1）用于 `ln(a)` 与 `exp(ln(a))` 回归
- 输出：
1. 两组系数级最大误差
2. 小区间数值抽样误差（与 `numpy.exp` 对照）
3. 耗时和部分系数打印
4. `PASS/FAIL` 校验结果

## R04

应用场景：
- NTT/FFT 多项式算法体系中的基础算子（竞赛与工程库常见）。
- 组合数学中的生成函数变换。
- 形式幂级数求逆、开方、指数生成函数相关计算。
- 需要在 `mod x^n` 语义下做高阶代数运算的场景。

## R05

数学前提与约束：
- `ln(A)` 在形式幂级数下要求 `A(0)=1`，否则常数项不满足对数定义。
- `exp(F)` 要求 `F(0)=0`，使得结果常数项为 `1`。
- 本实现在实数浮点上运行，语义是“截断到前 `n` 项”，即所有运算都按 `mod x^n` 处理。

## R06

`poly_ln` 高层步骤：
1. 计算 `A'(x)`。
2. 计算 `A(x)` 的逆元 `A^{-1}(x)`（截断到 `n` 项）。
3. 计算 `(A' * A^{-1}) mod x^(n-1)`。
4. 对结果积分并设置积分常数为 0，得到前 `n` 项。

## R07

`poly_exp` 高层步骤（Newton doubling）：
1. 初值 `G=1`（1 项正确）。
2. 每轮把正确项数从 `m` 扩到 `2m`（上限 `n`）。
3. 计算 `ln(G)`（截断到当前目标阶数）。
4. 构造校正项 `Δ = 1 - ln(G) + F`。
5. 更新 `G <- G * Δ mod x^k`。
6. 迭代直到达到 `n` 项。

## R08

复杂度（当前实现为朴素卷积）：
- 单次截断卷积约 `O(n^2)`。
- `poly_inv` 采用倍增 Newton，整体约 `O(n^2 log n)`。
- `poly_ln` 由一次逆元 + 少量线性/卷积步骤组成，量级约 `O(n^2 log n)`。
- `poly_exp` 每轮调用 `poly_ln` 并更新，整体也约 `O(n^2 log n)`。

若将卷积替换为 FFT/NTT，复杂度可进一步下降。

## R09

空间复杂度：
- 主要由若干长度 `n` 的系数数组组成。
- Newton 迭代中会保留中间向量，空间量级为 `O(n)`（不计 Python 对象常数开销）。
- MVP 强调可读性，因此临时数组分配较多，但规模可控。

## R12

输出字段说明：
- `Truncation order n`: 截断阶数（保留 `x^0` 到 `x^(n-1)`）。
- `Max coeff error |ln(exp(f)) - f|`: 组合映射回归误差。
- `Max coeff error |exp(ln(a)) - a|`: 逆向组合回归误差。
- `Max numeric error ...`: 在小区间采样点上，与 `exp(f(x))` 的数值误差。
- `Time for ...`: 两组测试的粗略耗时。
- `Validation`: 基于阈值的通过/失败结论。

## R13

边界与鲁棒性处理：
- `poly_ln` 显式检查 `a0=1`。
- `poly_exp` 显式检查 `f0=0`。
- `poly_inv` 检查常数项非 0，避免不可逆情况。
- 所有核心运算都按 `x^n` 截断，避免无界增长。
- 使用小阈值 `EPS` 跳过近零项，减少浮点噪声放大。

## R14

当前局限：
- 卷积是朴素 `O(n^2)`，不适合超大 `n`。
- 采用浮点而非模数域，极端条件下存在数值稳定性问题。
- 未实现更高阶算子（如多项式开方、幂函数、组合链式变换）。

## R15

可扩展方向：
- 用 FFT/NTT 卷积替换 `poly_mul_trunc`。
- 引入模素数域版本，适配竞赛高精度需求。
- 增加随机化 property test（例如多组 `ln(exp(f))` / `exp(ln(a))`）。
- 将当前接口封装成可复用的 `FormalPowerSeries` 小库。

## R16

最小测试建议：
- 正确性：固定样例验证两条恒等关系。
- 边界性：`n=1`、仅常数项、稀疏高次项等输入。
- 非法输入：`ln` 的 `a0!=1`、`exp` 的 `f0!=0` 应抛出异常。
- 数值性：在小 `|x|` 区间采样，与实函数数值对照。

## R17

方案对比：
- 黑箱库调用：开发快，但算法细节不可见，不利于学习与审计。
- 直接幂级数递推：可行但在高阶拓展时组织性较弱。
- 本方案（逆元 Newton + `ln/exp` Newton）：结构标准、便于升级到快速卷积，是工程上常见路线。

## R18

`demo.py` 源码级流程（9 步）：
1. `main()` 设定截断阶 `n`，并构造满足约束的 `f`（`f0=0`）与 `a`（`a0=1`）。
2. `poly_exp(f, n)` 从 `g=[1]` 开始做倍增 Newton，逐轮扩大已正确阶数。
3. 每轮内部调用 `poly_ln(g, m2)`，把当前 `g` 转成对数域误差量。
4. `poly_ln` 先经 `poly_derivative` 得到 `g'`，再通过 `poly_inv(g, m2)` 求逆。
5. `poly_inv` 使用 Newton 公式 `B <- B*(2-AB)` 迭代，并在每轮截断到当前阶数。
6. `poly_ln` 用 `poly_mul_trunc(g', g^{-1})` 得到商并 `poly_integral` 回到对数结果。
7. `poly_exp` 组装校正项 `delta = 1 - ln(g) + f`，再更新 `g <- g*delta mod x^m2`。
8. 达到 `n` 阶后，`main()` 分别计算 `ln(exp(f))` 与 `exp(ln(a))` 的系数误差，并做 `numpy.exp` 数值抽样对照。
9. 最后打印误差、耗时、前几项系数和 `Validation`，形成可直接验收的 MVP 输出。
