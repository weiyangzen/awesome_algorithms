## R01

Clock（时钟）页面置换算法是对 LRU 思想的近似实现，用较低开销在“命中率”和“维护成本”之间做平衡。

核心目标：
- 输入页面访问序列与物理页框数；
- 在每次缺页时决定淘汰页；
- 统计命中次数、缺页次数与过程状态轨迹。

本题关注的是操作系统内存管理中的经典页面置换策略，而非通用缓存库调用。

## R02

问题输入与输出定义：

- 输入：
  - 页面访问序列 `reference = [p0, p1, ...]`（非负整数）；
  - 页框数 `frame_count > 0`。
- 输出：
  - `hits`：命中次数；
  - `faults`：缺页次数；
  - 命中率 / 缺页率；
  - 逐步事件日志（每步页框状态、引用位、时钟指针移动、替换详情）。

`demo.py` 使用固定样例直接运行，无需交互输入。

## R03

Clock 的关键思想：

- 每个页框维护一位“引用位”（use bit）；
- 维护一个循环指针（hand），像时钟指针一样转动；
- 缺页时从指针开始检查：
  - 若当前页框引用位为 `0`，直接淘汰；
  - 若为 `1`，先清零并给予“二次机会”，指针继续前进；
- 命中时将该页框引用位置为 `1`。

因此它通过“最近被访问过”这一近似信号来避免过早淘汰热点页。

## R04

本实现中的状态结构：

- `frames: list[int | None]`：页框内容（`None` 表示空框）；
- `use_bits: list[int]`：引用位数组，取值 `0/1`；
- `hand: int`：当前检查位置；
- 统计量：`hits/faults/evictions/second_chance_clears`；
- `events`：每步日志，记录动作类型、扫描槽位数、替换页、指针前后位置等。

这些状态完全在源码中显式维护，便于逐步审计。

## R05

`demo.py` 中 Clock 的高层流程：

1. 初始化 `frames/use_bits/hand`；
2. 逐页处理访问请求；
3. 若命中：只置位引用位为 `1`；
4. 若缺页：启动指针循环扫描；
5. 遇空框直接填充；
6. 遇引用位 `1` 则清零并前进（记一次二次机会）；
7. 遇引用位 `0` 则替换该页并前进；
8. 记录本步事件并继续下一页。

## R06

固定演示样例：

- `reference = [7, 0, 1, 2, 0, 3, 0, 4, 2, 3, 0, 3, 2]`
- `frame_count = 3`

在当前实现下输出结果：

- Clock：`faults = 9`, `hits = 4`
- FIFO：`faults = 10`, `hits = 3`
- LRU：`faults = 9`, `hits = 4`

并额外输出 Clock 前 10 步事件追踪，展示“清零-前进-替换”的真实路径。

## R07

复杂度分析（设访问长度为 `n`，页框数为 `m`）：

- 时间复杂度：
  - 命中检测在本 MVP 中用线性查找，单步 `O(m)`；
  - 缺页扫描最坏可绕环，单步 `O(m)`；
  - 总体最坏 `O(nm)`。
- 空间复杂度：
  - 核心状态 `O(m)`；
  - 若保存全部事件日志则为 `O(n)`。

教学场景下该复杂度足够清晰且可验证。

## R08

边界与输入约束：

- `frame_count <= 0`：抛出 `ValueError`；
- 访问序列含负页号：抛出 `ValueError`；
- 空访问序列：命中/缺页都为 `0`（实现可兼容）；
- 页框数为 `1`：退化为几乎每次换入换出（除连续重复访问外）。

这些规则由 `_validate_inputs` 与统计逻辑共同保障。

## R09

Clock 与其他策略的工程特性对比：

- 对比 FIFO：
  - Clock 会利用引用位回避一部分“刚用过就淘汰”的情况；
  - 常见局部性场景下通常优于或接近 FIFO。
- 对比精确 LRU：
  - Clock 维护成本更低（单 bit + 指针）；
  - 命中率可能略差于 LRU，但实现简单、开销小。

因此 Clock 常被视为“低成本近似 LRU”。

## R10

正确性不变量（实现层面）：

1. 每一步恰有一次动作：`hit` / `fault_fill` / `fault_replace`；
2. 任意时刻 `len(frames) == frame_count` 且 `use_bits` 同长度；
3. 命中时不会替换页面，仅更新引用位；
4. 缺页替换时被替换槽位一定是在扫描时遇到的首个“空框或引用位 0”槽位；
5. 全流程满足 `hits + faults == len(reference)`。

`main()` 中断言会自动校验关键项。

## R11

本 MVP 附带 FIFO/LRU 手写对照实现，目的：

- 不是把第三方包当黑盒求解；
- 而是把 Clock 放在同一输入条件下进行可解释对比；

这样可快速验证 Clock 的行为是否符合预期趋势。

## R14

常见实现失误与防护：

- 失误：命中不置 `use_bit=1`，导致热点页被过早淘汰。  
  防护：命中分支强制置位。

- 失误：扫描时忘记“清零后前进”，会卡住或错误替换。  
  防护：清零与 `hand=(hand+1)%m` 绑定实现。

- 失误：替换后指针不前进，导致偏置替换。  
  防护：`fault_fill` 与 `fault_replace` 均统一前进。

- 失误：统计不一致。  
  防护：断言 `hits + faults == len(reference)`。

## R18

`demo.py` 中 Clock 算法的源码级流程可拆为 8 步：

1. `main()` 构造固定访问序列与页框数，调用 `clock_page_replacement`。  
2. `clock_page_replacement` 初始化 `frames/use_bits/hand` 与统计计数器。  
3. 对每个访问页先尝试命中：若命中则定位页框并把对应 `use_bits[idx] = 1`。  
4. 若缺页，进入 `while` 扫描：读取 `hand` 指向槽位。  
5. 若槽位为空，直接装入新页并置引用位为 `1`，指针前进一格。  
6. 若槽位非空且引用位为 `0`，执行替换（记录 `replaced`、`evictions`），然后指针前进。  
7. 若槽位非空且引用位为 `1`，先清零并记 `second_chance_clears`，指针前进继续扫描。  
8. 每步将动作与快照写入 `events`；处理完成后返回结果，由 `main()` 汇总并断言。

第三方库使用边界：
- `numpy` 仅用于生成随机访问序列与计算均值；
- `pandas` 仅用于表格化展示；
- 页面置换决策逻辑（命中/扫描/二次机会/替换）全部为源码手写实现，不是黑盒调用。
