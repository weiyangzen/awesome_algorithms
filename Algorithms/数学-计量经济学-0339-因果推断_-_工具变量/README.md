## R01

工具变量（Instrumental Variables, IV）用于处理回归中的内生性问题：当解释变量 `x` 与误差项相关时，普通最小二乘（OLS）会有偏且不一致。  
本条目实现一个最小可运行的 **2SLS（Two-Stage Least Squares）MVP**，演示如何用外生工具变量 `z` 识别 `x` 对结果 `y` 的因果效应。

## R02

核心识别逻辑：
- **相关性（Relevance）**：`Cov(z, x) != 0`，工具变量能显著解释内生变量。
- **外生性（Exogeneity）**：`Cov(z, u) = 0`，工具变量与结构误差独立。
- **排除限制（Exclusion）**：`z` 只通过 `x` 影响 `y`，不存在 `z -> y` 的直接通道。

若上述条件成立，2SLS 可恢复结构参数的一致估计。

## R03

MVP 的输入输出约定：
- 输入：`demo.py` 内部合成数据（无需交互）。
- 输出：
1. OLS 与 2SLS 的系数表（含 HC1 稳健标准误、t、p、95% CI）
2. 第一阶段回归结果 `x ~ z + w`
3. 弱工具变量诊断（排除工具变量 F 统计量）
4. DWH 内生性检验（残差纳入法）
5. `PASS/FAIL` 验证结论（相关性、内生性、IV 改善偏差）

## R04

结构方程与第一阶段方程：

- 结构方程：`y = beta0 + beta_x * x + beta_w * w + u`
- 第一阶段：`x = pi0 + pi_z * z + pi_w * w + v`

其中 `x` 为内生变量，`w` 为外生控制变量，`z` 为排除型工具变量。  
在实现中，`x` 与潜在混杂 `u` 同时相关，故 OLS 会偏；2SLS 借助 `z` 的外生变化识别 `beta_x`。

## R05

2SLS 的矩阵形式（显式实现，不依赖黑箱计量包）：

- `X = [W, x]`，`Z = [W, z]`
- `Pz = Z (Z'Z)^(-1) Z'`
- `beta_2sls = (X'PzX)^(-1) X'Pz y`

脚本中直接按上式构造并求解。若矩阵接近奇异，使用带微小 ridge 的逆与 `pinv` 回退，保证数值稳健。

## R06

第一阶段诊断：
- 回归 `x ~ z + w`，检查 `z` 的显著性与拟合能力。
- 计算经典排除变量 F 统计量：
  - 受限模型：`x ~ w`
  - 非受限模型：`x ~ z + w`
  - `F = ((SSR_r - SSR_u)/q) / (SSR_u/(n-k_u))`

经验规则：`F > 10` 通常说明弱工具变量风险较低（仅作启发式门槛）。

## R07

不确定性估计：
- OLS 使用 HC1 sandwich 协方差。
- 2SLS 使用线性 IV 的 HC1 sandwich 协方差：
  - `A = X'Z (Z'Z)^(-1) Z'X`
  - `S = Z' diag(u_hat^2) Z`
  - `Var(beta) = A^(-1) [X'Z (Z'Z)^(-1) S (Z'Z)^(-1) Z'X] A^(-1)`（含有限样本修正）

因此输出的标准误、t、p 更适配异方差场景。

## R08

脚本总体流程：
1. 生成含内生性的合成数据（`x` 与未观测混杂相关）。
2. 拟合 OLS：`y ~ x + w`。
3. 拟合 2SLS：`x` 由 `z` 工具化。
4. 执行第一阶段 F 检验与拟合诊断（R²、MSE）。
5. 执行 DWH 内生性检验。
6. 对比 `beta_x` 的真值偏差并输出 PASS/FAIL。

## R09

复杂度分析（`n` 为样本数，`k` 为参数维度）：
- OLS：`O(n*k^2 + k^3)`
- 2SLS：核心矩阵乘法为 `O(n*k^2)` 量级，求逆 `O(k^3)`
- 本 MVP 中 `k` 很小（常数级），总体近似线性随 `n` 增长。

## R10

`demo.py` 的数据生成策略：
- `z, w, u, v` 由随机过程生成；`u` 代表不可观测混杂。
- `x = 0.9z + 0.6w + 0.95u + v`
- `y = 1.0 + beta_x_true*x + 0.7w + 1.1u + eps`
- `eps` 使用 `torch.randn` 生成并设置异方差尺度。

因为 `u` 同时进入 `x` 和 `y`，OLS 对 `beta_x` 的估计偏离真值；2SLS 利用 `z` 的外生变化纠偏。

## R12

输出字段解释：
- `coef`：参数点估计
- `std_err`：HC1 稳健标准误
- `t / p_value`：显著性检验统计量
- `ci95_low / ci95_high`：95% 置信区间
- `Relevance F-stat`：工具变量相关性诊断
- `DWH p-value`：内生性检验（越小越支持“x 确有内生性”）
- `|est - true|`：与真值误差，衡量 OLS 与 IV 的偏差对比

## R13

边界处理与工程稳健性：
- 线性代数求逆统一走 `_safe_inv`，避免奇异矩阵直接报错。
- 协方差对角线若非正，标准误返回 `NaN`，避免虚假推断。
- 随机种子固定（`numpy` + `torch`），保证结果可复现。
- 全流程只依赖常见科学计算栈，无外部数据与交互输入。

## R14

当前 MVP 的局限：
- 仅演示线性、单内生变量、单排除工具变量场景。
- 未实现 LIML、GMM-IV、Kleibergen-Paap 等高级弱工具变量稳健检验。
- DWH 使用残差纳入法的简化版本，未展开更完整的检验族。
- 数据为合成示例，不替代真实实证中的识别论证与制度背景分析。

## R16

最小验证清单：

2. 第一阶段 `F > 10`（相关性基本通过）。
3. DWH `p < 0.05`（能检出内生性）。
4. `|beta_x_2sls - true| < |beta_x_ols - true|`（IV 比 OLS 更接近真值）。
5. 结果表中标准误、置信区间均为合理数值。

## R17

方法对比（实现视角）：
- 仅 OLS：
  - 优点：实现最简单
  - 缺点：存在内生性时偏差不可接受
- 直接调用高级计量黑箱库：
  - 优点：开发快
  - 缺点：细节不透明，难教学与排障
- 本实现（显式 2SLS + 诊断）：
  - 优点：公式与代码一一对应，便于审计
  - 缺点：功能覆盖面小于完整计量软件

## R18

`demo.py` 源码级算法流程（8 步）：
1. `simulate_iv_data` 构造 `z,w,u,v,eps`，并按结构方程生成 `x,y`，其中 `u` 同时影响 `x` 和 `y` 以制造内生性。
2. `ols_hc1` 调用 `_safe_inv` 和 `_hc1_covariance`，估计基线 OLS（`y~x+w`）与稳健推断统计。
3. `two_stage_least_squares_hc1` 构造 `X=[W,x]`、`Z=[W,z]`，按 `beta=(X'PzX)^(-1)X'Pz y` 显式求 2SLS 系数。
4. 在同一函数里用 `u_hat` 构造 `S=Z' diag(u_hat^2) Z`，按 sandwich 公式计算 IV-HC1 协方差与标准误。
5. `first_stage_f_test` 分别拟合受限/非受限第一阶段，基于 SSR 差异计算排除变量 F 统计量与 p 值。
6. `main` 中对第一阶段拟合结果额外计算 `r2_score` 与 `mean_squared_error`，补充工具变量相关性强弱信息。
7. `durbin_wu_hausman_test` 先取第一阶段残差 `v_hat`，再在 `y~W+x+v_hat` 中检验 `coef(v_hat)=0`，得到 DWH 统计结论。
8. `main` 汇总 OLS/2SLS 表、诊断统计和偏差对比，输出三项 `PASS/FAIL` 与总体 `OVERALL`。
