## R01

倾向得分（Propensity Score）是处理分配在协变量条件下的概率：

\[
e(x) = P(T=1 \mid X=x)
\]

其中 `T` 是是否接受处理（0/1），`X` 是处理前协变量。核心思想是把高维混杂压缩为一个“平衡得分”，在给定 `e(X)` 后，处理组与对照组的协变量分布应当可比。

常见输入输出与步骤：
- 输入：观测数据 `(X, T, Y)`，其中 `Y` 为结局变量。
- 输出：平均处理效应（ATE/ATT）估计值，及可选的平衡性诊断。
- 主要步骤：
1. 用分类模型估计 `e(X)`（常用逻辑回归、GBDT 等）。
2. 用倾向得分做匹配、分层、逆概率加权（IPTW）或协变量调整。
3. 计算处理效应，并检查平衡性（如 SMD）。

## R02

现代倾向得分框架由 Rosenbaum 与 Rubin 在 1983 年系统提出：
- 1983: Rosenbaum, P. R., & Rubin, D. B., *The central role of the propensity score in observational studies for causal effects*.

之后在 1990s-2020s 中扩展出稳定权重、双重稳健（AIPW）、TMLE、机器学习倾向得分等分支。

## R03

背景问题：在随机对照试验不可行时，观测数据中的处理分配往往与协变量相关，直接比较 `E[Y|T=1]-E[Y|T=0]` 会有混杂偏差。

前人不足：
- 仅做回归控制对模型设定敏感；
- 高维协变量下直接匹配困难（维度灾难）；
- 简单分层难以同时平衡多变量。

倾向得分的历史意义在于：把“高维平衡问题”转成“标量概率上的平衡问题”，使匹配与加权可工程化落地。

## R04

若用逻辑回归估计倾向得分（样本数 `n`、特征维度 `d`、迭代次数 `k`）：
- 拟合复杂度（典型）：`O(knd)`。
- 预测倾向得分：`O(nd)`。
- 基于权重计算 ATE：`O(n)`。

整体典型复杂度可写为 `O(knd + nd + n)`，主项通常是 `O(knd)`。

## R05

空间开销（密集矩阵实现）：
- 数据矩阵 `X`：`O(nd)`。
- 模型参数：`O(d)`。
- 倾向得分与权重向量：`O(n)`。

额外工作内存通常不超过 `O(nd + n + d)`，主导项为存放输入数据的 `O(nd)`。

## R06

可复现实例（与 `demo.py` 一致）：
- 构造 `n=6000` 的观测数据，`X=(x1,x2)` 同时影响处理分配 `T` 与结局 `Y`，制造混杂；
- 设真实处理效应 `true_ate=2.0`；
- 先计算 naive 均值差，再用逻辑回归估计 `\hat e(X)` 并做 IPTW：

\[
\hat\tau_{IPTW}=
\frac{\sum_i T_i Y_i / \hat e_i}{\sum_i T_i/\hat e_i}
-
\frac{\sum_i (1-T_i) Y_i / (1-\hat e_i)}{\sum_i (1-T_i)/(1-\hat e_i)}
\]

典型现象：
- naive 估计偏离真值（混杂导致）；
- IPTW 估计更接近真值；
- 协变量平衡指标（SMD）在加权后明显下降。

## R07

算法意义：
- 因果推断里连接“设计阶段（平衡）”与“估计阶段（效应）”的核心桥梁。
- 让大量医疗、经济、教育、政策评估中的观测研究具备可解释的准实验分析路径。
- 推动了后续双重稳健与机器学习因果估计方法的发展。

## R08

路径依赖的关键理论/算法：
- 潜在结果框架（Rubin Causal Model）。
- 条件可忽略性（Unconfoundedness）与重叠性（Positivity）。
- 二元分类概率建模（常用逻辑回归）用于估计 `P(T=1|X)`。
- 加权估计（Horvitz-Thompson/Hájek 类型）。
- 平衡性诊断（标准化均值差 SMD）。

## R09

适用前提与边界：
- `SUTVA`：个体处理定义清晰且无干扰。
- 条件可忽略性：`(Y(1),Y(0)) ⫫ T | X`。
- 重叠性：对所有协变量区域有 `0 < P(T=1|X) < 1`。
- 协变量应为处理前变量，不能使用处理后的中介变量。

不适用场景：
- 存在强未观测混杂；
- 处理几乎确定（倾向得分极端接近 0 或 1）。

## R10

正确性依据（以 IPTW 估计 ATE 为例）：
- 在可忽略性与重叠性成立时，倾向得分是平衡得分，条件于 `e(X)` 后处理与协变量分布可比。
- 关键恒等式：

\[
E\!\left[\frac{T Y}{e(X)}\right]=E[Y(1)],\quad
E\!\left[\frac{(1-T)Y}{1-e(X)}\right]=E[Y(0)]
\]

因此：

\[
ATE = E[Y(1)-Y(0)]
=
E\!\left[\frac{TY}{e(X)}-\frac{(1-T)Y}{1-e(X)}\right]
\]

样本版估计在样本量增大时收敛于目标量。

## R11

数值稳定性与误差传播：
- 倾向得分接近 0/1 时，权重 `1/e` 或 `1/(1-e)` 爆炸，方差急剧上升。
- 倾向模型错设会带来系统性偏差（非仅方差问题）。
- 有限样本下极端权重会让少量样本主导估计。

常见缓解：
- 对 `\hat e` 做 clipping/trimming（例如 `[1e-3, 1-1e-3]`）。
- 使用稳定权重（stabilized weights）。
- 检查并报告加权后的协变量平衡性。

## R12

真实计算成本（超越渐近符号）：
- 主耗时在倾向模型训练（优化器迭代、梯度计算）。
- 对 `n` 大、`d` 中等的数据，CPU 向量化足够；GPU 不是必须。
- I/O 与数据清洗常常比“权重计算”更耗时。
- 内存局部性会影响逻辑回归速度（连续数组通常更快）。

在本条目 MVP 中，`n=6000,d=2`，训练与估计均为秒级内完成。

## R13

误差界/概率保证（统计层面）：
- 在可忽略性、重叠性和正确模型设定下，IPTW 估计量一致；
- 在常规正则条件下具有渐近正态性，可构造置信区间；
- 若倾向模型错设，IPTW 可能偏；这也是双重稳健估计（AIPW/TMLE）被广泛采用的原因。

本 MVP 仅演示点估计，不实现渐近方差或 bootstrap 区间。

## R14

鲁棒性与失效模式：
- 失效模式 1：重叠性差，权重极端，估计方差失控。
- 失效模式 2：遗漏关键混杂变量，结果仍有偏。
- 失效模式 3：把处理后变量纳入 `X`，引入后门偏差。
- 失效模式 4：小样本 + 高维时倾向模型过拟合，平衡性变差。

工程上应至少做：
- 权重分布检查（最大值、分位数）；
- 平衡性报告（加权前后 SMD）；
- 敏感性分析（不同模型、不同截尾阈值）。

## R15

工程实现要点：
- 明确协变量时间顺序，仅保留处理前变量。
- 倾向模型建议先从可解释模型（逻辑回归）起步，再尝试更复杂模型。
- 设置足够迭代次数并检查收敛（`max_iter`）。
- 对概率做最小截断，避免除零和极端权重。
- 必做平衡诊断：若加权后 SMD 仍大，说明设计阶段未达标，不能直接解释效应。

## R16

前驱、后继与应用：
- 前驱：回归调整、分层比较、抽样加权思想。
- 同代方法：倾向得分匹配（PSM）、分层（stratification）、协变量调整。
- 后继：稳定 IPTW、双重稳健 AIPW、TMLE、因果森林与表示学习因果方法。

典型应用：
- 医疗真实世界证据（药物效果与安全性评估）；
- 计量经济与政策评估；
- 广告投放和推荐策略离线评估（在满足前提时）。

## R18

源码追踪（以 `scikit-learn` 逻辑回归 + 手写 IPTW 为例）：
- 倾向模型核心源码：`.venv/lib/python3.13/site-packages/sklearn/linear_model/_logistic.py`
  - `LogisticRegression.fit`
  - `_logistic_regression_path`
- 概率输出路径：
  - `LogisticRegression.predict_proba`（`_logistic.py`）
  - `LinearClassifierMixin._predict_proba_lr`（`.venv/.../sklearn/linear_model/_base.py`）

对应 8 步算法拆解（非黑箱）：
1. **数据校验与编码**：`fit` 内检查输入形状/类型，并把标签编码为二元目标。
2. **构造损失函数**：二分类走 `HalfBinomialLoss`，形成带正则项的逻辑损失目标。
3. **选择求解器**：`_check_solver` 确定 `lbfgs` 与惩罚项组合是否合法。
4. **数值优化参数**：进入 `_logistic_regression_path`，按 `C` 与样本权重计算 `l2_reg_strength`。
5. **L-BFGS 迭代求解**：调用 `scipy.optimize.minimize(method="L-BFGS-B")` 优化系数与截距。
6. **概率预测**：`predict_proba` 调用 `_predict_proba_lr`，对线性打分做 `sigmoid` 得到 `\hat e_i`。
7. **权重构造**：在 MVP 代码中执行 `w_i = T_i/\hat e_i + (1-T_i)/(1-\hat e_i)`，并对 `\hat e_i` 做 clipping。
8. **ATE 估计**：按加权均值差（Hájek 归一化形式）计算 `\hat\tau = \hat\mu_1-\hat\mu_0`，再结合 SMD 检查平衡性。
