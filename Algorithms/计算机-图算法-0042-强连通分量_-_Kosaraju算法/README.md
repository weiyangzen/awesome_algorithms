## R01

强连通分量（Strongly Connected Component, SCC）定义于有向图：

- 若一个顶点集合内部任意两点 `u, v` 互相可达（`u -> v` 且 `v -> u`），则该集合是强连通的；
- 极大强连通子图称为一个强连通分量。

Kosaraju 算法是经典 SCC 求解方法，核心思想是“两次 DFS + 反图”。

## R02

本条目解决的问题定义：

- 输入：有向图 `G=(V,E)`，顶点编号 `0..n-1`，边列表 `[(u,v), ...]`；
- 输出：
1. 所有 SCC 的顶点集合；
2. 每个顶点所属的 SCC 编号；
3. 第一次 DFS 的完成序列（finish order）；
4. 缩点图（condensation DAG）边集合及其拓扑序。

`demo.py` 使用固定样例，无交互输入。

## R03

Kosaraju 的理论要点：

1. 在原图上做 DFS，按“递归回溯完成时间”记录顶点序列；
2. 将图反向得到转置图 `G^T`；
3. 按第一步完成时间逆序，在 `G^T` 上进行 DFS；
4. 每次 DFS 扫到的一组顶点正好对应一个 SCC。

直觉上，第一遍 DFS 把“源 SCC 到汇 SCC”的依赖关系编码进完成时间，第二遍在反图中按逆序展开即可一块块剥离 SCC。

## R04

本 MVP 的算法流程：

1. 构建原图邻接表 `adj` 与反图邻接表 `rev_adj`；
2. 在 `adj` 上执行 DFS1，后序压入 `finish_order`；
3. 将遍历顺序设为 `reversed(finish_order)`；
4. 在 `rev_adj` 上执行 DFS2，收集每次搜索得到的一个分量；
5. 对分量做稳定排序，得到可复现的 `sccs` 与 `component_id_of_vertex`；
6. 遍历原始边，若端点落在不同分量，则加入缩点边；
7. 对缩点图执行 Kahn 拓扑排序，输出 `condensation_topo_order`。

## R05

设 `V=|V|`，`E=|E|`：

- 时间复杂度：`O(V + E)`；
- 空间复杂度：`O(V + E)`（邻接表、访问标记、结果容器）。

两次 DFS 都是线性扫描，是该算法的主要优势。

## R06

`demo.py` 的核心样例（Case-1）：

- `0 -> 1 -> 2 -> 0` 构成 SCC-A；
- `3 -> 4 -> 5 -> 3` 构成 SCC-B；
- `6 <-> 7` 构成 SCC-C；
- 再加边 `2 -> 3`、`5 -> 6`，形成 SCC 链。

输出应为：

- SCC：`{0,1,2}`、`{3,4,5}`、`{6,7}`；
- 缩点边：`A -> B`、`B -> C`。

## R07

优点：

- 实现简洁，结构清晰；
- 复杂度线性，适合静态大图；
- 结果可解释性强（两次 DFS 的顺序可追踪）。

局限：

- 需要完整反图（额外内存）；
- 对高频动态增删边场景不友好；
- Python 递归版在超深图上需要关注递归深度。

## R09

适用场景：

- 编译依赖环分析（模块循环引用）；
- 社交/引用网络中的强关系团识别；
- 工作流或状态机缩点预处理。

不适用场景：

- 在线动态图实时维护 SCC；
- 只关心最短路/最大流等其他图目标。

## R10

正确性直觉：

1. SCC 缩点后得到 DAG；
2. 第一遍 DFS 的完成顺序会让“更靠源头”的 SCC 在序列后部出现；
3. 在反图按该逆序做 DFS，会优先从某个 SCC 的“入口”出发；
4. 由于跨 SCC 边方向在反图中反向，搜索不会越界吞并尚未处理的前驱块；
5. 因此每次 DFS2 恰好恢复一个完整 SCC，且分量互不重叠、覆盖全部顶点。

## R11

实现细节与防坑：

- 输入边端点做范围校验，防止越界顶点污染结构；
- 第一遍 DFS 必须在“回溯时”记录顶点，不能在首次访问时记录；
- 第二遍 DFS 必须跑在反图上，否则分量会被错误合并；
- 分量输出做稳定排序，避免因邻接遍历次序导致测试不稳定；
- 缩点边使用集合去重，避免重复跨分量边。

## R12

工程成本评估：

- 代码体量小（百行级）；
- 调试路径明确（先看 `finish_order`，再看分量）；
- 单元测试重点：
1. 多 SCC 链结构；
2. 非连通图与孤立点；
3. 本身为 DAG 的输入（每点单独成 SCC）。

## R13

算法性质：

- 精确算法（非近似）；
- 确定性（给定同一输入、同一遍历规则，输出稳定）；
- 可解释性高（两个 DFS 序列 + 缩点图）；
- 结果可直接作为后续 DAG 分析输入。

## R14

常见失效模式与防护：

- 失效：把第一遍 DFS 记录写成前序，导致第二遍顺序错误。  
  防护：明确只在 DFS1 回溯结束后 `append(u)`。

- 失效：第二遍仍在原图跑 DFS，得到错误大分量。  
  防护：独立构建 `rev_adj`，DFS2 仅访问 `rev_adj`。

- 失效：输出随邻接插入顺序抖动，断言不稳定。  
  防护：分量内部排序、分量列表排序。

- 失效：缩点图重复边过多影响后续流程。  
  防护：缩点边先入 `set` 再排序输出。

## R15

实践建议：

- 先验证 SCC 集合，再验证缩点边，定位更快；
- 打印 `finish_order` 与 `second_pass_order` 能快速发现 DFS 顺序错误；
- 若要支持超大深链图，可改为显式栈 DFS，避免递归限制；
- 若上层需要业务可读输出，可把 SCC 映射回实体名称而非整数编号。

## R16

相关算法与扩展：

- Tarjan SCC：单次 DFS + low-link；
- Gabow SCC：双栈策略；
- 双连通分量（无向图）与 SCC（有向图）在问题定义上不同；
- SCC 缩点后可进行 DAG DP、拓扑优化、关键路径分析。

## R17

本目录 `demo.py` 的主要函数：

- `build_directed_graph`：构建原图/反图邻接表；
- `kosaraju_scc`：两遍 DFS 提取 SCC，并生成缩点图信息；
- `topological_sort_dag`：对缩点 DAG 做 Kahn 拓扑排序；
- `assert_case`：校验 SCC 与缩点边；
- `run_demo_cases`：运行 3 个固定测试样例。


脚本无交互输入，运行后输出每个样例的分量结果并最终打印 `All demo cases passed.`。

## R18

`demo.py` 的源码级流程可拆为 9 步（非黑盒）：

1. `run_demo_cases` 用 `numpy` 构造固定有向边数组，并转换为 Python `list[tuple[int,int]]`。  
2. `kosaraju_scc` 先调用 `build_directed_graph` 同时构建 `adj` 与 `rev_adj`。  
3. 在原图上执行 DFS1，按后序把顶点写入 `finish_order`。  
4. 把访问顺序设为 `second_pass_order = reversed(finish_order)`。  
5. 按该顺序在 `rev_adj` 上执行 DFS2，每次新搜索收集一个 `raw_component`。  
6. 对分量做稳定排序，建立 `old_to_new` 映射并生成 `component_id_of_vertex`。  
7. 扫描原图边 `(u,v)`，把跨分量边 `(comp[u], comp[v])` 加入集合形成缩点边。  
8. 调用 `topological_sort_dag` 对缩点 DAG 求拓扑序，验证其无环性质。  
9. `assert_case` 对 SCC 集合、缩点边、顶点覆盖率做断言，保证实现行为正确。  

第三方库拆解说明：

- `numpy` 仅用于样例数据组织；
- SCC 提取、分量编号、缩点构建、拓扑排序全部在源码中手写实现。
