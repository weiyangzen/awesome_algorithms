# AKS素性测试 (AKS Primality Test)

- UID: `MATH-0006`
- 学科: `数学`
- 分类: `数论`
- 源序号: `6`
- 目标目录: `Algorithms/数学-数论-0006-AKS素性测试_(AKS_Primality_Test)`

## R01

本条目实现 `AKS Primality Test`（AKS 素性测试）的最小可运行 MVP，目标是给出一个**确定性、无需随机性**的素数判定流程。  
给定整数 `n`，输出 `n` 是否为素数，并在 `demo.py` 中直接演示若干样例（含 Carmichael 数）。

## R02

AKS 的核心思想是把素性判定转化为多项式同余验证：

- 对素数 `n`，在合适的模空间中有  
  `(x + a)^n ≡ x^n + a (mod n)`。
- 对合数，这个关系会在受控范围内被检测出反例。
- 算法通过寻找满足条件的 `r`，把验证空间限制到 `(x^r - 1, n)`，从而可计算。

## R03

输入：单个整数 `n`（`n >= 0`）。  
输出：

- `True` / `False`（是否为素数）
- 附带文字原因（例如：`perfect power`、`non-trivial gcd`、`congruence fails`）

`demo.py` 中以 `AKSResult` 数据类承载输出：`(n, is_prime, reason)`。

## R04

算法使用的关键数学对象：

- `ord_r(n)`：`n` 在模 `r` 乘法群中的阶。
- `φ(r)`：欧拉函数（`1..r` 中与 `r` 互素的整数个数）。
- 完全幂检测：若 `n = a^b (a>1,b>1)`，则 `n` 一定是合数。
- 多项式环：在 `Z_n[x] / (x^r - 1)` 上做运算，利用 `x^r ≡ 1` 做循环降幂。

## R05

实现流程（对应 `demo.py`）：

1. 若 `n < 2`：合数；`n in {2,3}`：素数；偶数：合数。
2. 检查 `n` 是否为完全幂，若是则合数。
3. 找最小 `r`，使 `ord_r(n) > (log2 n)^2`。
4. 对 `a=2..r` 计算 `gcd(a,n)`，若出现 `1 < gcd < n` 则合数。
5. 若 `n <= r`，直接判素数。
6. 令 `limit = floor(sqrt(φ(r)) * log2 n)`。
7. 对 `a=1..limit` 验证  
   `(x+a)^n ≡ x^n + a (mod x^r - 1, n)`。若失败则合数。
8. 全部通过则素数。

## R06

正确性直觉：

- 完全幂和非平凡 `gcd` 检测会提前剔除大量合数。
- 关键的多项式同余是 AKS 证明中的“素数判定签名”。
- `r` 的选取保证验证范围足够强，不会把大量合数漏判为素数。
- 因此流程是确定性的，不依赖概率或随机基。

## R07

理论上 AKS 属于多项式时间算法。  
本 MVP 采用朴素循环卷积做多项式乘法，工程复杂度不追求最优：

- 多项式乘法：`O(r^2)`
- 多项式快速幂：`O(log n)` 次乘法
- 同余验证外层循环：`O(sqrt(φ(r)) log n)`

整体更偏教学可读性，适合中小规模 `n` 的演示。

## R08

空间复杂度主要来自长度为 `r` 的系数数组：

- 单次多项式表示：`O(r)`
- 乘法临时数组：`O(r)`
- 快速幂过程保持常数个多项式缓存，总体 `O(r)` 级别（不计 Python 大整数内部开销）。

## R09

边界与适用范围：

- 对 `n < 2` 返回合数。
- 对 `2, 3` 正确返回素数。
- 对偶数快速返回合数。
- 对 Carmichael 数（如 `561`）不会误判为素数（区别于某些费马型测试）。
- 该实现为教学版，面对特别大的 `n` 会较慢。

## R10

与常见素性测试对比：

- 试除法：简单但慢，通常只适合很小的 `n`。
- Miller-Rabin：实践极快，但常见版本是概率型（可选确定性基集合特化）。
- AKS：理论上确定性多项式时间，数学完备，但工程常数较大。

本条目选择 AKS 是为了“确定性 + 理论完整性”的教学价值。

## R11

`demo.py` 的关键实现点：

- `is_perfect_power`：二分幂检查，避免浮点误差主导结论。
- `find_smallest_r`：按定义搜索满足阶条件的最小 `r`。
- `poly_mul_mod`：在 `(x^r - 1, n)` 上进行循环卷积。
- `poly_pow_mod`：多项式快速幂。
- `aks_primality_test`：按 AKS 顺序串联全部步骤并返回结构化原因。

## R15

常见实现错误：

- 把 `x^r - 1` 的约束写错成普通截断（应使用循环降幂）。
- 忘记所有系数都要 `mod n`。
- `r` 的搜索没有按“阶大于 `log^2 n`”条件实现。
- 完全幂检测仅依赖浮点开根，导致边界误判。

## R16

可扩展方向：

- 用 FFT/NTT 或更快卷积替换朴素 `O(r^2)` 乘法。
- 对大整数场景加入更高效的完美幂检测与阶估计优化。
- 增加命令行参数（当前版本故意保持无交互 MVP）。
- 增加与 Miller-Rabin 的基准对照脚本（准确性与性能双维度）。

## R18

源码级算法流（`demo.py`，8 步）：

1. `aks_primality_test(n)` 先处理平凡分支：`n<2`、`2/3`、偶数。
2. 调用 `is_perfect_power(n)`，通过“指数枚举 + 底数二分”检测 `n=a^b`。
3. 调用 `find_smallest_r(n)`：逐个尝试 `r`，检查是否不存在 `k<=log^2 n` 使 `n^k ≡ 1 (mod r)`。
4. 对 `a=2..r` 执行 `gcd(a,n)`，若发现非平凡因子立即返回合数。
5. 若 `n<=r`，依据 AKS 定理可直接判定为素数并返回。
6. 计算 `limit=floor(sqrt(phi(r))*log2(n))`，其中 `phi(r)` 由 `euler_phi` 因式分解得到。
7. 对每个 `a in [1, limit]`：  
   `poly_pow_mod(x+a, n, r, n)` 计算左侧；构造 `x^n+a` 在 `x^r-1` 下的右侧表达并比较。
8. 若任一 `a` 比较失败则合数；全部通过则返回素数。
