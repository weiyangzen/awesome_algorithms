## R01

QuickHull 是二维凸包问题的分治算法。给定平面点集 `P`，目标是找到最小凸多边形 `CH(P)`，使得 `P` 中所有点都在该多边形内部或边界上。

核心思想：
- 先找左右极点 `A` 与 `B`，把点集按有向线段 `AB` 分成上下两侧。
- 在某一侧中选出距离直线 `AB` 最远的点 `C`，则 `C` 必是凸包顶点。
- 递归处理三角形 `A-C-B` 外侧的两块子区域，直到没有候选点。

二维叉积判定公式：
- `cross((b-a), (p-a)) > 0`：`p` 在有向边 `a->b` 左侧。
- 叉积绝对值与点到直线距离成正比（差一个常数比例），可用于“最远点”选择。

## R02

QuickHull 的工程来源可追溯到 Barber, Dobkin, Huhdanpaa 的 Qhull 系列工作（1990 年代中期）。它在实践中常被用于：
- 点云预处理与边界抽取；
- GIS 空间包围区域计算；
- 计算几何库中的凸包基础模块。

与 Graham Scan、Monotonic Chain 相比，QuickHull 以“分治+几何剪枝”见长，随机分布数据上通常表现不错。

## R03

要解决的核心问题：如何从大量点中只保留边界极点，剔除内部点。

QuickHull 的关键洞察：
1. 左右极点一定在凸包上。
2. 对一条候选边 `AB`，若存在点在其外侧，则离 `AB` 最远点 `C` 一定是凸包点。
3. 点落在三角形 `A-C-B` 内部或边上时，不可能成为该分支后续凸包点，可直接剪掉。

因此算法每一步都在“确认一个凸包顶点 + 丢弃一批内部点”。

## R04

二维 QuickHull 的标准流程：
1. 对输入去重，找到 `x` 最小点 `A` 与 `x` 最大点 `B`。
2. 用 `AB` 把点集划分为上侧集合 `S_up` 和下侧集合 `S_down`。
3. 递归函数 `ExtendHull(A, B, S)`：
- 若 `S` 为空，结束；
- 取 `S` 中使 `cross(AB, AP)` 最大的点 `P`；
- 在 `S` 中分别筛出位于 `A->P` 左侧和 `P->B` 左侧的点集；
- 递归处理这两个子集。
4. 组合顺序为：`A` + 上链 + `B` + 下链，得到闭合凸包顶点序列（本实现不重复首点）。

## R05

复杂度（二维）：
- 平均时间复杂度：常见情况下接近 `O(n log n)`。
- 最坏时间复杂度：`O(n^2)`（例如点分布接近圆周且每次剪枝效果差）。
- 空间复杂度：
- 需要存储点集与递归候选子集，约 `O(n)` 额外空间；
- 递归深度平均较浅，最坏可达 `O(n)`。

## R06

简化示例：
- 输入点：`{(-2,0), (0,3), (2,0), (0,1), (0,-1), (1,1)}`。
- 首先 `A=(-2,0)`, `B=(2,0)`。
- 上侧最远点是 `(0,3)`，下侧最远点是 `(0,-1)`。
- `(0,1)`、`(1,1)` 落在已形成凸边内侧，被剪除。

最终凸包顶点（逆时针）可写为：
`(-2,0) -> (0,-1) -> (2,0) -> (0,3)`。

## R07

主要优点：
- 分治结构清晰，几何意义直观。
- 对随机点云通常速度较好。
- 通过“最远点 + 左侧筛选”能快速剪掉大量内部点。

主要局限：
- 最坏复杂度会退化到 `O(n^2)`。
- 浮点误差会影响共线判定与边界点归属。
- 递归实现在极端输入上可能有深度压力（工程上可改迭代栈）。

## R09

适用场景：
- 二维静态点集凸包；
- 需解释性较强、可视化友好的几何流程；
- 中等规模数据，且点分布并非专门构造的最坏情况。

不适用或需谨慎：
- 强实时增量更新场景（更适合动态凸包结构）；
- 高维凸包（复杂度与实现复杂性显著上升）；
- 对最坏时间界非常敏感的任务（可考虑 `O(n log n)` 上界更稳的算法实现）。

## R10

正确性关键点（实现层面）：
1. 端点选择必须覆盖全局左右极点。
2. `cross > eps` 作为“严格在左侧”判定，避免把共线点反复递归。
3. 递归时选取“当前边外侧最远点”，该点可确认为凸包点。
4. 子问题只保留位于新边外侧的点，保证内部点不会误入。
5. 最终顶点序列需统一方向（本实现标准化为逆时针）并固定起点，方便复现对比。

## R11

数值稳定性处理：
- 使用 `float64` 存储几何坐标。
- 所有“左侧”判断使用 `eps=1e-12`，减轻接近共线时的抖动。
- 先 `np.unique` 去重，避免重复点导致零长度边与冗余递归。
- 面积和周长计算单独封装，退化情形（0/1/2 点）做显式分支处理。

## R12

实际成本与调参建议：
- `eps` 是主要阈值：
- 太小：噪声点可能被误判为严格左侧，增加递归与伪顶点；
- 太大：可能误删真实边界近共线点。
- 对工程数据通常从 `1e-12 ~ 1e-9` 试验，视坐标尺度调整。
- 若存在大量近共线或恶意输入，可加入：
- 递归深度监控；
- 小集合分支切换到单调链等稳健子程序。

## R14

常见失效模式与防护：
- 失效：输入全共线，得到异常多顶点或顺序混乱。
- 防护：严格左侧筛选 + 退化分支（本实现返回两端点）。
- 失效：重复点很多导致性能下降。
- 防护：入口去重。
- 失效：数值噪声下参考实现对比失败。
- 防护：比较时对顶点坐标做小数位舍入签名。
- 失效：极端分布递归过深。
- 防护：可改为显式栈迭代（本 MVP 未展开，但结构可平移）。

## R16

关系与变体：
- 同类二维凸包算法：Graham Scan、Monotonic Chain、Jarvis March。
- QuickHull 可视作“几何分治”路径，Monotonic Chain 更偏排序+栈。
- 三维及更高维常见实现会用到 Qhull 思路，但复杂度与鲁棒性要求更高。
- 在线/动态场景通常不直接用静态 QuickHull，而用专门动态数据结构。

## R18

源码级算法流程拆解（对应 `demo.py`，8 步）：
1. `build_demo_points` 生成测试点集（含噪声、重复、共线、极值），模拟真实脏数据。
2. `quickhull` 首先通过 `unique_points` 去重，并选取左右极点 `A/B` 作为初始分治边界。 
3. 通过 `cross_values(A,B,pts)` 与 `cross_values(B,A,pts)` 把候选点分为上链与下链两组。 
4. 在 `extend_hull(A,B,S)` 内，计算每个候选点相对 `AB` 的叉积，选择最远点 `P`（叉积最大）。 
5. 用 `A->P` 与 `P->B` 两条新边再次筛选“严格在左侧”的点，形成两个更小子问题。 
6. 递归拼接返回顺序：`Extend(A,P)` + `[P]` + `Extend(P,B)`，逐步生成该侧凸链。 
7. 主流程把上链和下链拼接成完整凸包序列，并做去重、逆时针方向统一、起点标准化。 
8. `main` 中再调用 `monotonic_chain_hull` 进行独立交叉验证，最后输出面积/周长与顶点摘要，确保 MVP 可执行且结果可信。

本实现没有把第三方库当黑盒，几何判定、最远点选择、递归拆分和结果组装都可在源码中逐行追踪。
