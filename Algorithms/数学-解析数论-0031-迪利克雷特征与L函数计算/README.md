# 迪利克雷特征与L函数计算

- UID: `MATH-0031`
- 学科: `数学`
- 分类: `解析数论`
- 源序号: `31`
- 目标目录: `Algorithms/数学-解析数论-0031-迪利克雷特征与L函数计算`

## R01

问题定义：给定模数 `q`（本 MVP 取奇素数 `p`）与复参数 `s>1`，构造迪利克雷特征 `χ`，并计算截断的迪利克雷 `L` 级数：

`L_N(s, χ) = Σ_{n=1..N} χ(n)/n^s`。

本目录目标是把“特征构造 + L 函数数值求和”做成可运行、可校验的最小实现，而不是只给公式。

## R02

数学背景（解析数论语境）：

- 迪利克雷特征是模 `q` 的完全积性、周期性算术函数。
- `L(s,χ)` 在 `Re(s)>1` 由绝对收敛级数定义，是研究素数分布（尤其算术级数中素数分布）的核心对象。
- 在可计算层面，先要“显式构造 χ”，再要“稳定计算部分和/近似值”。

## R03

本 MVP 的范围与输入输出约定：

- 范围：`q = p` 为奇素数；构造模 `p` 的全部 `p-1` 个特征 `χ_k`。
- 输入方式：`demo.py` 内置参数（`p=7`, `N=80000`, `s∈{2,3}`），无需交互输入。
- 输出内容：
  - 各特征在剩余系上的取值表；
  - 正交性数值验证；
  - 各 `χ_k` 的 `L_N(s,χ_k)`；
  - 主特征恒等式残差。

## R04

算法选型：

- 利用 `p` 为素数时 `(Z/pZ)^*` 是循环群，先找原根 `g`。
- 通过离散对数表 `log_g(a)` 把每个非零剩余类编码为指数。
- 定义特征：
  `χ_k(a)=exp(2π i k log_g(a)/(p-1))`，且 `p|n` 时 `χ_k(n)=0`。
- `L_N` 计算采用“按周期取值 + 幂级数求和”，可选 NumPy 向量化。

这样能完整暴露算法链路，避免黑盒调用。

## R05

复杂度（设模数为 `p`，级数截断项数为 `N`）：

- 原根搜索：约 `O(p * ω(p-1) * log p)`（`ω` 为不同质因子个数，模幂测试主导）。
- 离散对数表构造：`O(p)`。
- 单个 `L_N(s,χ_k)` 求值：`O(N)`。
- 计算全部 `p-1` 个特征的 `L_N`：`O(pN)`。
- 空间复杂度：`O(p)`（周期表、对数表、少量缓存）。

## R06

关键公式：

1. 特征构造（`a∈{1,...,p-1}`）：
   `χ_k(a) = exp(2π i k log_g(a)/(p-1))`。
2. 扩展到整数：
   `χ_k(n) = χ_k(n mod p)`；若 `p|n` 则 `χ_k(n)=0`。
3. 截断 `L` 级数：
   `L_N(s,χ_k)=Σ_{n=1..N} χ_k(n)/n^s`。
4. 主特征截断恒等式：
   `Σ_{n≤N, p∤n} 1/n^s = Σ_{n≤N}1/n^s - p^{-s} Σ_{m≤N/p}1/m^s`。

## R07

正确性要点：

- 原根 `g` 的存在性（`p` 为素数）保证离散对数表示覆盖全部非零剩余类。
- `χ_k` 的定义来自循环群角色（character）标准构造，天然满足周期性与完全积性（在与 `p` 互素部分）。
- `L_N` 只是定义级数前 `N` 项求和，无额外近似假设。
- 正交性检验与主特征恒等式残差为代码内自验证锚点。

## R08

前置知识：

- 模运算与原根。
- 群同态/角色（character）概念。
- 复指数 `exp(iθ)` 与共轭。
- `s>1` 时 Dirichlet 级数收敛性基础。

## R09

适用边界：

- 当前实现只覆盖“奇素数模数 `p`”。
- 未扩展到一般合数模 `q` 的全部特征分解（那需要处理更一般的单位群结构）。
- `s` 目前按实数 `s>1` 计算；未实现复平面的解析延拓。

## R10

数值稳定性与误差来源：

- 主要运算为复数求和，误差来自浮点舍入与求和顺序。
- 正交矩阵的非零误差、恒等式残差通常在 `1e-12 ~ 1e-15` 量级（与环境相关）。
- 未使用高精度库，故本 MVP 目标是“教学/验证级”而非高精度研究级。

## R11

性能瓶颈：

- 当 `N` 增大时，`L_N` 求和是线性主耗时。
- 当 `p` 增大时，计算全部角色的总成本随 `pN` 增长。
- NumPy 可将主求和向量化；无 NumPy 时会回退纯 Python 循环，速度较慢但仍可运行。

## R12

工程实现选择：

- 依赖栈最小化：标准库 + 可选 `numpy`。
- 未调用现成 `L` 函数黑盒库；`demo.py` 显式实现：
  - 素性检查与分解；
  - 原根搜索；
  - 离散对数表；
  - 特征值构造；
  - `L_N` 累加与两类校验。

## R13

常见失效模式：

- 把“模 `p` 周期性”误写成普通数组下标，导致 `χ(n)` 错位。
- 忘记设置 `p|n` 时 `χ(n)=0`，会破坏主特征恒等式。
- 对原根判定不完整，导致离散对数表不满射，进而角色定义错误。
- `s<=1` 时继续做同样级数求和，可能出现收敛性问题。

## R14

可验证性设计：

- 验证 A：角色正交关系
  `Σ χ_k(a) conj(χ_l(a)) = p-1 (k=l), 0 (k≠l)`。
- 验证 B：主特征截断恒等式残差接近 0。
- 这两项分别约束“角色构造正确性”和“级数求和正确性”。

## R18

源码级算法流程（8 步，非黑盒）：

1. 检查模数 `p` 是否为奇素数，并分解 `p-1` 的质因子。
2. 枚举候选 `g`，用 `g^{(p-1)/q} != 1 (mod p)` 判定并找到一个原根。
3. 迭代 `x <- x*g mod p` 构造离散对数表 `log_g(a)`（`a=1..p-1`）。
4. 对每个角色编号 `k` 构造一周期取值：
   `χ_k(a)=exp(2π i k log_g(a)/(p-1))`，并设置 `χ_k(0)=0`。
5. 计算 `L_N(s,χ_k)` 时，将 `n` 映射到 `n mod p`，按周期取 `χ_k(n)` 并累加 `χ_k(n)/n^s`。
6. 对全部 `k,l` 计算 `Σ_{a=1}^{p-1} χ_k(a)conj(χ_l(a))`，检查正交矩阵的对角/非对角误差。
7. 对主特征 `χ_0`，并行计算右侧 `Σ_{n≤N}1/n^s - p^{-s}Σ_{m≤N/p}1/m^s`，得到恒等式残差。
8. 输出 `L_N` 数值与两类残差，形成“构造正确 + 求和正确”的闭环验证。

本实现即使使用 NumPy，也只用于向量化数组运算；核心数学步骤均在源码中显式实现和可追踪。
