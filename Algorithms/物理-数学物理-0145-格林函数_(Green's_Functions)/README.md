# 格林函数 (Green's Functions)

- UID: `PHYS-0145`
- 学科: `物理`
- 分类: `数学物理`
- 源序号: `145`
- 目标目录: `Algorithms/物理-数学物理-0145-格林函数_(Green's_Functions)`

## R01

格林函数方法把线性微分算子方程
\[
L u(x) = f(x)
\]
转化为核积分
\[
u(x)=\int G(x,\xi)f(\xi)\,d\xi.
\]
它的核心价值是把“求函数解”分解为两步：先求算子对单位脉冲响应的核 \(G\)，再对任意源项 \(f\) 做叠加。

## R02

本条目实现一个最小可运行实例：一维泊松方程的 Dirichlet 边值问题
\[
-u''(x)=f(x),\quad x\in[0,1],\quad u(0)=u(1)=0.
\]
示例源项取
\[
f(x)=\sin(\pi x)+0.25\cos(2\pi x).
\]
该问题有解析 Green 核，便于演示“理论-实现-验证”闭环。

## R03

算子定义为
\[
L=-\frac{d^2}{dx^2}.
\]
对应的格林函数满足
\[
L_x G(x,\xi)=\delta(x-\xi),\quad G(0,\xi)=G(1,\xi)=0.
\]
其中 \(\delta\) 是狄拉克 delta，表示单位点源。

## R04

对区间 \([0,1]\) 与上述边界条件，Green 核为分段函数
\[
G(x,\xi)=
\begin{cases}
x(1-\xi), & x\le \xi,\\
\xi(1-x), & x>\xi.
\end{cases}
\]
它连续，但一阶导数在 \(x=\xi\) 处有单位跳跃，正是 delta 源导致的特征。

## R05

由核表示可得解
\[
u(x)=\int_0^1 G(x,\xi)f(\xi)\,d\xi.
\]
这一步把二阶微分方程化为一维积分，数值上可通过标准求积法稳定实现。

## R06

MVP 中的数值策略：
1. 在 \(\xi\) 方向用均匀网格 + 梯形求积近似积分。
2. 构造矩阵 \(G_{ij}=G(x_i,\xi_j)\)。
3. 计算向量化乘积 \(u \approx G\,(w\odot f)\)。
该实现只依赖 `numpy` 的广播和矩阵乘法，代码短且可读。

## R07

为了校验 Green 解的正确性，demo 同时用 `scipy.integrate.solve_bvp` 求同一边值问题的参考数值解 \(u_{\text{bvp}}\)，并比较
\[
\|u_{\text{green}}-u_{\text{bvp}}\|_{\infty},\quad \text{RMSE}.
\]
这让结果不依赖“肉眼看曲线”，而是有明确误差指标。

## R08

`demo.py` 还计算 PDE 残差
\[
r(x)=-u''(x)-f(x),
\]
用二阶中心差分估计 \(u''\)，并报告 \(\|r\|_\infty\)。当 Green 积分和离散精度足够时，残差应显著小于解的量级。

## R09

时间复杂度（设解网格点数为 \(N\)，求积点数为 \(M\)）：
- 构造核矩阵：\(O(NM)\)
- 矩阵-向量乘：\(O(NM)\)
- `solve_bvp` 参考解：通常高于线性复杂度，取决于自适应网格与线性化迭代次数
空间复杂度主导项是核矩阵 \(O(NM)\)。

## R10

数值稳定性要点：
1. Green 核在本问题中非奇异（仅导数有跳跃），梯形求积表现稳定。
2. 残差计算用二阶差分，边界两端不直接做二阶导估计，避免端点误差放大。
3. 参考求解器设置较严格容差 `tol=1e-7`，便于做可靠对照。

## R11

最小工具栈与职责：
- `numpy`: 网格、广播、核矩阵与线性代数
- `scipy`: `solve_bvp` 构建独立参考解
- `pandas`: 输出样例点结果表，便于人工检查
- `scikit-learn`: `mean_squared_error` 计算 RMSE
- `PyTorch`（可选可用时启用）: 与 `numpy` 结果做一次张量乘法一致性对照

## R12

代码结构（`demo.py`）：
1. `forcing`: 定义源项 \(f(x)\)
2. `green_kernel`: 生成分段 Green 核
3. `solve_with_green`: 梯形求积实现核积分解
4. `solve_with_bvp`: `solve_bvp` 参考解
5. `pde_residual_linf`: 残差上界估计
6. `main`: 指标汇总与表格打印

## R14

预期输出包含以下字段：
- `L_inf(u_green - u_bvp)`: Green 解与 BVP 参考解的最大绝对误差
- `RMSE(u_green, u_bvp)`: 均方根误差

以及一个 `x/u_green/u_bvp/abs_err` 的样例表。

## R16

常见坑：
1. 算子符号混淆：`-u''=f` 与 `u''=f` 的核不同，不能混用。
2. 忘记积分权重端点 `0.5`，会引入系统偏差。
3. 直接把 `solve_bvp` 当黑箱，不校验边界与残差。
4. 网格过粗时，残差与解误差可能都被低估或高估，需要联合观察。

## R18

以下给出本 MVP 的“源码级算法流”（避免把第三方库当黑箱）：

1. 生成离散网格：在解点 \(x_i\) 与积分点 \(\xi_j\) 上建立均匀网格，并构造梯形权重 \(w_j\)。
2. 组装 Green 核矩阵：按分段定义计算 \(G_{ij}=G(x_i,\xi_j)\)，使用 `numpy.where` 完成向量化分支。
3. 形成离散右端：计算 \(b_j=f(\xi_j)w_j\)。
4. 线性代数求积分解：执行矩阵向量乘法 \(u_i=\sum_j G_{ij}b_j\)，得到 \(u_{\text{green}}\)。
5. 构造 BVP 一阶系统：把二阶方程写成 `y[0]=u, y[1]=u'` 与 `y'=[y1, -f(x)]`，并写边界残差 `bc=[u(0),u(1)]`。
6. `solve_bvp` 内部流程（SciPy）：在当前网格上做分段多项式配点离散，线性化非线性残差方程，解修正量并迭代；若误差超阈值则自适应加密网格，直到满足容差或触发节点上限。
7. 在同一 \(x_i\) 上评估 \(u_{\text{bvp}}\)，并计算 \(\|u_{\text{green}}-u_{\text{bvp}}\|_\infty\) 与 RMSE。
8. 对 \(u_{\text{green}}\) 施加中心差分估计 \(u''\)，得到 PDE 残差 \(-u''-f\) 的 \(\infty\)-范数，作为额外正确性证据。

以上 8 步完整覆盖了“核构造 -> 数值积分 -> 独立求解器校验 -> 残差验证”的可执行链路。
