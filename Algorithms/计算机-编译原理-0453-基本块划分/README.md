## R01

基本块划分（Basic Block Partition）是控制流分析的起点：把线性中间代码切分为“单入口、单出口、内部顺序执行”的最大连续片段。

本条目提供一个最小可运行 Python MVP，完整覆盖：
- leader 识别；
- 基本块切分；
- 块级 CFG 建立；
- 固定样例断言验证。

## R02

基本块定义（经典教材口径）：
- 只有第一条指令可作为入口（外部跳转只能跳到块首）；
- 除最后一条外，不会发生控制流转移；
- 块内控制流是顺序执行。

因此，基本块是后续数据流分析、优化、寄存器分配等阶段的统一计算粒度。

## R03

本实现采用经典 leader 规则：
1. 程序第一条指令是 leader；
2. 任意跳转目标指令是 leader；
3. 任意 `jump/cjump` 的下一条指令是 leader（若存在）。

对 leader 序列排序后，相邻 leader 间的区间就是基本块。

## R04

`demo.py` 的示例 TAC 程序：
- `L0:`
- `t1 = a + b`
- `if t1 > 0 goto L1 else L2`
- `L1:`
- `x = t1 * 2`
- `goto L3`
- `L2:`
- `x = t1 - 1`
- `L3:`
- `y = x + 4`
- `return y`

这是一条典型“分支 + 汇合”结构，便于验证基本块边界。

## R05

基于上述程序，leader 结果应为：
- 指令 `0`（程序入口）
- 指令 `3`（`L1`，条件跳转目标）
- 指令 `6`（`L2`，条件跳转目标）
- 指令 `8`（`L3`，无条件跳转目标）

即 leader 索引序列：`[0, 3, 6, 8]`。

## R06

按 leader 划分得到 4 个基本块：
- `B1 = [0, 2]`
- `B2 = [3, 5]`
- `B3 = [6, 7]`
- `B4 = [8, 10]`

块边界满足最大化原则：同一块内不存在新的 leader。

## R07

块级 CFG 构建规则（看每个块最后一条指令）：
- 末指令是 `jump`：后继仅为其目标块；
- 末指令是 `cjump`：后继为真分支目标块与假分支目标块；
- 末指令是 `return`：无后继；
- 其他普通指令：后继为文本上的下一个基本块（fall-through）。

该规则与编译器教材中的 basic-block CFG 建模一致。

## R08

`demo.py` 的核心数据结构：
- `TACInstruction`：最小 IR 指令对象（`op/text/label/target` 等）；
- `BasicBlock`：块名、起止指令索引、块内指令序列；
- `label_to_index`：标签到指令索引映射；
- `index_to_block`：指令索引到块名映射。

这样可以把“线性指令地址空间”与“块图空间”稳定关联。

## R09

时间复杂度（设指令数 `N`，基本块数 `B`）：
- 构建标签表：`O(N)`
- leader 扫描：`O(N)`
- 基本块切分：`O(B)`（切片总访问仍受 `N` 约束）
- 建立 `index_to_block`：`O(N)`
- 构建 CFG：`O(B)`

总体为线性量级 `O(N)`。

## R10

空间复杂度：
- 标签表与索引映射约 `O(N)`；
- 块对象与块内指令引用总量约 `O(N)`；
- CFG 邻接表约 `O(B + E)`（`E` 为块间边数）。

整体空间开销为 `O(N)`。

## R11

运行环境与依赖：
- Python 3.10+；
- 核心算法仅用标准库；
- `numpy` 为可选依赖，仅用于块长度统计输出（缺失时自动回退纯 Python 统计）。

## R12

脚本输出包括：
- 原始 TAC 指令列表（带索引）；
- leader 索引列表；
- 每个基本块的索引区间与指令内容；
- 块级 CFG 边；
- 块大小统计（count/mean/min/max/std）；
- 断言结果 `Validation: PASS`。

输出是确定性的，便于回归检查。

## R13

正确性保障策略：
- 在 `assert_expected` 中固定校验 leader、块区间、CFG 三组结果；
- 若任一规则实现偏差，立即抛出 `AssertionError`；
- 通过“白盒预期值 + 自动断言”保证算法语义不漂移。

这比仅打印结果更稳健。

## R14

实现中处理的边界情况：
- 空程序返回空 leader/空块；
- 缺失目标标签、重复标签会抛出 `ValueError`；
- `cjump` 若未提供 `false_target`，回退到顺序后继（若存在）；
- CFG 后继集合按出现顺序去重，避免重复边。

这些机制保证 MVP 在教学级输入上可稳定运行。

## R15

## R18

`demo.py` 源码级算法流程（8 步）：
1. `build_sample_program()` 构造固定 TAC 指令序列，包含标签、条件跳转、无条件跳转与返回。
2. `build_label_index()` 扫描指令，建立 `label -> instruction_index` 映射并检查重复标签。
3. `find_leader_indices()` 按 3 条经典规则计算 leader 集合并排序。
4. `partition_basic_blocks()` 用相邻 leader 切片，得到 `B1..Bk` 的起止区间与块内指令。
5. `build_index_to_block()` 建立 `instruction_index -> block_name` 映射，连接线性地址与块空间。
6. `build_block_cfg()` 查看每个块末指令类型（`jump/cjump/return/fall-through`）并生成块级后继表。
7. `summarize_block_sizes()` 统计块大小分布（优先 `numpy`，缺失则纯 Python 回退）。
8. `assert_expected()` 对 leader、块区间、CFG 做确定性断言，全部通过后输出 `Validation: PASS`。
