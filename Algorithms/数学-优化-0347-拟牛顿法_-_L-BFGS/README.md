## R01

L-BFGS（Limited-memory BFGS）是大规模无约束优化中的经典拟牛顿法。它不显式构造完整 Hessian 矩阵，而是仅保留最近 `m` 轮迭代的向量对 `(s_k, y_k)` 来近似逆 Hessian 作用，从而在保持较快收敛速度的同时显著降低内存开销。

本目录提供一个“可运行、可审计”的最小 MVP：手写两层循环递归（two-loop recursion）+ Armijo 回溯线搜索，避免把核心算法交给黑盒接口。

## R02

本条目解决的问题是：
- 输入：可微目标函数 `f(x)`、梯度 `∇f(x)`、初值 `x0`，以及 L-BFGS 超参数（`m`、`gtol`、`max_iter` 等）。
- 输出：近似最优解 `x*`、最终函数值、梯度范数、每轮迭代轨迹（函数值/步长/记忆长度）。

`demo.py` 内置两组固定案例，无需交互输入：
1. Rosenbrock 函数（2 维，非凸）
2. SPD 二次函数（30 维，凸）

## R03

核心思想：
- 通过最近 `m` 组信息
  - `s_k = x_{k+1} - x_k`
  - `y_k = g_{k+1} - g_k`（`g_k = ∇f(x_k)`）
  来近似逆 Hessian `H_k`。
- 每轮并不形成 `H_k` 矩阵，而是用 two-loop recursion 直接计算方向 `p_k = -H_k g_k`。
- 再用线搜索确定步长 `α_k`，更新 `x_{k+1} = x_k + α_k p_k`。

这让 L-BFGS 在中高维问题中比完整 BFGS 更省内存，且通常比纯梯度下降收敛更快。

## R04

关键数学关系：

1. 记忆向量定义：
- `s_i = x_{i+1} - x_i`
- `y_i = g_{i+1} - g_i`
- `ρ_i = 1 / (y_i^T s_i)`

2. two-loop recursion（求 `H_k g_k`）
- 第一层（逆序）：
  - `α_i = ρ_i s_i^T q`
  - `q <- q - α_i y_i`
- 初始缩放：`r <- γ_k q`，常用
  - `γ_k = (s_{k-1}^T y_{k-1}) / (y_{k-1}^T y_{k-1})`
- 第二层（正序）：
  - `β_i = ρ_i y_i^T r`
  - `r <- r + s_i (α_i - β_i)`
- 下降方向：`p_k = -r`

3. Armijo 条件（回溯线搜索）：
- `f(x_k + α p_k) <= f(x_k) + c1 * α * g_k^T p_k`

4. 曲率条件（更新记忆前检查）：
- 需要 `y_k^T s_k > 0`（实现中用小阈值过滤数值噪声）。

## R05

算法高层流程：
1. 初始化 `x <- x0`，清空历史 `S/Y/RHO`。
2. 计算 `f(x)` 与 `g=∇f(x)`，若 `||g|| <= gtol` 则结束。
3. 用 two-loop recursion 计算方向 `p`。
4. 若 `g^T p >= 0`（非下降方向），回退到最速下降 `p=-g`。
5. 用 Armijo 回溯线搜索求步长 `α`。
6. 更新 `x_next = x + α p`，并计算 `g_next`。
7. 构造 `s = x_next - x`、`y = g_next - g`，满足曲率条件则写入有限记忆队列（超长则弹出最旧项）。
8. 记录轨迹并继续，直到收敛或到达停止条件。

## R06

MVP 主要数据结构：
- `s_hist: list[np.ndarray]`：保存最近 `m` 个 `s_k`。
- `y_hist: list[np.ndarray]`：保存最近 `m` 个 `y_k`。
- `rho_hist: list[float]`：保存对应 `ρ_k`。
- `history: list[HistoryItem]`：每轮记录 `(iter, f(x), ||grad||, step, mem_size)`。

这种结构直接体现 L-BFGS 的“有限记忆”特性：空间复杂度与 `m` 成正比，而不是与维度平方成正比。

## R07

正确性与数值稳定要点：
- two-loop recursion 的两层遍历顺序必须严格区分：第一层逆序、第二层正序。
- 必须校验曲率 `y^T s`，否则会破坏逆 Hessian 近似的正定性。
- 必须保证搜索方向为下降方向；若方向退化，使用 `-g` 兜底。
- 线搜索用 Armijo 约束可确保目标函数单调下降（在可接受范围内）。
- 对非有限数值（`nan/inf`）立即中止，避免污染后续迭代。

## R08

复杂度分析（设变量维度为 `n`、保留记忆长度为 `m`、迭代轮数为 `T`）：
- 每轮 two-loop recursion 代价约 `O(mn)`。
- 每次梯度计算代价记为 `C_grad`，函数值计算代价记为 `C_f`。
- 线搜索若平均回溯 `B` 次，则总时间近似：
  - `O(T * (mn + C_grad + B*C_f))`
- 空间复杂度：
  - `O(mn)`（核心优势，相比完整 BFGS 的 `O(n^2)`）。

## R09

边界与异常处理策略：
- 输入检查：`x0` 必须是一维有限向量；`m/max_iter/gtol/min_step` 均需合法。
- 初值异常：若 `f(x0)` 非有限，立即报错。
- 线搜索失败：若步长衰减到 `min_step` 以下仍不满足 Armijo，停止并返回 `line search failed`。
- 梯度异常：若更新点梯度出现非有限值，停止并返回失败信息。
- 曲率过小：`y^T s` 低于阈值时跳过该记忆对，不强行写入历史。

## R10

MVP 取舍：
- 选择手写 L-BFGS，而不是直接调用 `scipy.optimize.minimize(method="L-BFGS-B")`，以便逐步追踪算法细节。
- 仅覆盖“无约束、可微”场景，不引入盒约束与投影算子。
- 仅依赖 `numpy`，避免引入不必要框架；重点放在算法透明度与可复现。

## R12

输出字段说明：
- `iter`：迭代轮数（从 1 开始）。
- `f(x)`：该轮更新后的目标函数值。
- `||grad||`：该轮更新后的梯度二范数。
- `step`：线搜索最终接受的步长 `α`。
- `mem`：当前已保存的 `(s,y)` 记忆对个数。
- `converged`：是否因 `||grad|| <= gtol` 收敛。
- `message`：停止原因（收敛/线搜索失败/达到最大迭代等）。
- `abs_x_error / rel_x_error`：相对参考解的误差指标。
- `Summary`：两组案例的聚合统计。

## R14

关键参数与调参建议：
- `m`（默认 10）：记忆长度。维度越高可适当增大，但会增加每轮代价。
- `gtol`（默认 `1e-6`）：梯度收敛阈值，越小越严格。
- `max_iter`：最大迭代次数。
- `c1`（默认 `1e-4`）：Armijo 常数，控制充分下降条件。
- `line_search_shrink`（默认 `0.7`）：回溯衰减率。
- `min_step`（默认 `1e-12`）：最小可接受步长。

经验上，先固定 `m=10` 与较保守线搜索，再根据收敛速度和稳定性微调。

## R15

与相关方法对比：
- 对比梯度下降：L-BFGS 利用曲率信息，通常显著减少迭代步数。
- 对比完整 BFGS：L-BFGS 牺牲部分二阶近似精度，换取 `O(mn)` 内存优势，适合大规模变量。
- 对比牛顿法：牛顿法需要 Hessian 或线性系统求解，单步昂贵；L-BFGS 只需一阶信息，工程上更常用。
- 对比共轭梯度：CG 在特定二次问题高效；L-BFGS 对一般平滑目标更通用。

## R16

典型应用场景：
- 大规模机器学习参数优化（无约束平滑目标）。
- 科学计算中的参数反演与最小二乘变体。
- 需要比 SGD/纯 GD 更快收敛、但又不想构造 Hessian 的场景。

## R18

`demo.py` 源码级算法流程（8 步）：
1. `main` 构造两个固定案例（Rosenbrock 与 SPD 二次函数），并调用 `run_case` 逐个执行。  
2. `run_case` 调用 `lbfgs_optimize`，统一传入 `m/gtol/max_iter/line search` 参数。  
3. `lbfgs_optimize` 初始化 `x`、历史记忆 `s_hist/y_hist/rho_hist`，计算初始 `f(x)` 与 `g(x)`。  
4. 每轮先检查 `||g||` 是否达到 `gtol`；若未达到，进入 `lbfgs_two_loop` 计算近似牛顿方向。  
5. `lbfgs_two_loop` 先逆序扫历史计算 `alpha` 并修正 `q`，再按缩放 `gamma` 初始化 `r`，最后正序回代得到方向 `p=-r`。  
6. 若 `g^T p` 非负则回退到最速下降方向 `-g`；随后 `armijo_backtracking` 通过回溯衰减步长直到满足 Armijo 条件。  
7. 接受步长后更新 `x/g/f`，计算 `s_k/y_k`，仅在 `y_k^T s_k` 足够大时写入有限记忆队列（超出 `m` 则弹出最旧元素）。  
8. 记录 `(iter, f, ||grad||, step, mem)` 到 `history`；循环结束后返回结果，`run_case` 输出误差与收敛统计，`main` 汇总整体指标。
