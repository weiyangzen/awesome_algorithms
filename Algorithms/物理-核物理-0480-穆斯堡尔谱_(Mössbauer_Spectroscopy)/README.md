# 穆斯堡尔谱 (Mössbauer Spectroscopy)

- UID: `PHYS-0457`
- 学科: `物理`
- 分类: `核物理`
- 源序号: `480`
- 目标目录: `Algorithms/物理-核物理-0480-穆斯堡尔谱_(Mössbauer_Spectroscopy)`

## R01

穆斯堡尔谱算法的核心任务是：
- 输入速度扫描轴 `v`（mm/s）与透射谱 `T(v)`
- 用可解释的线型模型（本 MVP 采用 Fe-57 常见六重峰 Lorentzian 叠加）拟合超精细参数
- 输出 `isomer_shift(δ)`、`magnetic hyperfine step(h)`、`linewidth(Γ)`、吸收深度与基线

本目录中的 `demo.py` 采用“先合成再反演”的最小闭环：
- 先生成带噪声的合成谱（可复现）
- 再用非线性最小二乘反演参数
- 用 `MSE` 和 `R²` 评估拟合质量

## R02

相关理论与方法的时间线可简述为：
- `1958`: Rudolf Mössbauer 发现无反冲共振吸收（穆斯堡尔效应）
- `1960s` 起：穆斯堡尔谱成为核能级超精细相互作用（同质异能位移、四极分裂、磁超精细分裂）测量工具
- 现代工程实践：速度驱动 + 多峰线型拟合 + 统计指标验证，形成标准数据分析流程

因此该算法的历史起点通常锚定在 `1958`。

## R03

提出背景是“如何从实验谱线反推出核局域环境信息”：
- 原始观测是计数率/透射率随多普勒速度变化的曲线
- 物理解释参数（δ、ΔEQ、Bhf 等）并不直接可见
- 需要把“峰位、峰宽、峰深”映射为可比较的超精细参数

算法化拟合把这一过程从人工读图升级为可重复的数值反演。

## R04

设光谱点数为 `N`，峰线数为 `K`（本实现 `K=6`），迭代步数为 `I`：
- 单次模型评估复杂度约 `O(N*K)`
- 非线性拟合总复杂度约 `O(I*N*K)`
- 在本 MVP 固定 `K=6` 时，可视为 `O(I*N)`

## R05

空间复杂度：
- 速度轴、观测谱、预测谱等向量均为 `O(N)`
- 参数向量和协方差矩阵规模常数级（5 维）
- 因而总体空间复杂度为 `O(N)`

## R06

`demo.py` 内置一个 Fe-57 风格示例：
- 真值参数：`baseline=1.0, depth=0.24, δ=0.18 mm/s, h=0.82 mm/s, Γ=0.21 mm/s`
- 速度范围：`[-6, 6] mm/s`，共 401 个采样点
- 噪声：高斯噪声 `σ=0.004`
- 输出：
  - 参数对比表（真值 vs SciPy 拟合 vs 可选 Torch 微调）
  - 指标表（`MSE`, `R²`）
  - 抽样光谱点表（观测值与模型预测值）

## R07

算法意义：
- 物理上：把穆斯堡尔谱“形状”转化为核超精细参数
- 方法上：将实验数据处理规范化、可复现化
- 工程上：可作为更复杂模型（含四极双峰、非对称展宽、多组分相分解）的 baseline

## R08

本 MVP 直接使用的理论/数值组件：
- 线型函数：Lorentzian 共振线
- 六重峰叠加：`3:2:1:1:2:3` 相对强度（粉末近似）
- 非线性拟合：`scipy.optimize.curve_fit`
- 误差评估：`sklearn.metrics.mean_squared_error`, `r2_score`
- 表格输出：`pandas.DataFrame`
- 可选梯度微调：`PyTorch` 自动求导 + `Adam`

## R09

适用前提与边界：
- 当前模型假设单一对称六重峰组分
- 未显式拟合四极分裂双峰、纹理效应、厚吸收体传输效应
- 线型固定为 Lorentzian，未引入 Voigt/Gaussian 展宽
- 适合教学、验证与管线搭建，不等价于高精度材料定量分析软件

## R10

正确性来自三层一致性：
- 结构一致：模型参数与可观测谱形一一对应（峰位/峰宽/峰深）
- 目标一致：最小化残差平方和，直接对应“拟合观测谱”
- 验证一致：用 `MSE` 与 `R²` 对反演结果做独立量化校验

## R11

数值稳定性要点：
- 给参数设置上下界，避免非物理解（如负线宽）
- 初值选在可行物理区间，降低局部极小风险
- 噪声较大时，`δ` 与 `h` 可能存在耦合，需要更强先验或多组分约束
- Torch 微调中通过 `softplus` 参数化保证正值参数可微可训练

## R12

真实成本（超越大 O）：
- 401 点谱线拟合在 CPU 上通常毫秒到秒级
- 常数项开销主要来自重复模型评估与 Jacobian 近似
- 输出 DataFrame 的开销相对可忽略
- 若扩展到批量样本，主要瓶颈在“每条谱线的非线性拟合次数”

## R13

误差与不确定度来源：
- 实验噪声（计数统计噪声）
- 模型误差（真实谱线可能非单一六重峰）
- 参数相关性（宽度与峰深、位移与分裂尺度）

MVP 中通过以下方式控制：
- 固定随机种子保证可复现实验
- 输出拟合指标用于跨版本回归
- 保留 `pcov`（协方差）接口以便后续扩展置信区间

## R14

鲁棒性与失效模式：
- 若初值偏离过远，拟合可能收敛到次优局部解
- 若样本实际是双峰/多相混合，单六峰模型会系统偏差
- 若速度范围过窄，外侧峰信息不足，参数退化会加重
- 若数据点过少，参数可辨识性下降

## R15

工程实现要点：
- 用 `dataclass` 固化真值参数，减少字段错位
- 把模型函数拆成 `numpy` 与 `torch` 两套同构实现，便于交叉验证
- 拟合、评估、表格展示分层实现，方便替换任一组件
- 提供“无 PyTorch 兜底路径”，保证脚本在最小依赖环境可运行

## R16

前驱/后继关系：
- 前驱：核共振吸收、超精细相互作用理论、谱线形函数
- 同层：非线性最小二乘、参数估计与误差传播
- 后继：
  - 四极双峰与多组分全谱联合拟合
  - 温度/磁场扫描下的参数轨迹分析
  - 与分类模型结合做物相自动识别

## R18

源码级算法流程（非黑箱，8 步）：
1. 读取 `SextetParams` 真值参数，按固定随机种子生成速度轴与带噪声合成谱。
2. 调用 `mossbauer_sextet_numpy`：根据 `δ + h*offset` 生成 6 个峰心，并按 `3:2:1:1:2:3` 权重叠加 Lorentzian。
3. 构造残差函数 `r_i = T_obs(v_i) - T_model(v_i, θ)`，其中 `θ=[baseline, depth, δ, h, Γ]`。
4. 将 `θ` 初值与上下界交给 `scipy.optimize.curve_fit`；在有界条件下它转入 `least_squares`（TRF 框架）求解。
5. 每次迭代中，求解器会评估当前残差，并用有限差分近似 Jacobian，形成局部线性化子问题。
6. TRF 子问题给出参数增量 `Δθ`，更新后再次评估残差平方和，直到步长/梯度/迭代条件满足终止准则。
7. 拟合完成后，用 `sklearn` 计算 `MSE` 与 `R²`，并将真值、拟合值与抽样谱点整理成 `pandas` 表格。
8. 若 `torch` 可用，脚本把 SciPy 解作为初始化，用 `softplus` 约束正参数并执行 Adam 梯度微调，输出对比指标。

第 4-6 步强调：`curve_fit` 在这里是“可展开的数值优化流程封装”，不是不可解释的黑箱模型。
