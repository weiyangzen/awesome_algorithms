# 素数定理证明相关算法

- UID: `MATH-0047`
- 学科: `数学`
- 分类: `解析数论`
- 源序号: `47`
- 目标目录: `Algorithms/数学-解析数论-0047-素数定理证明相关算法`

## R01

本条目实现“素数定理证明相关算法”的最小可运行版本：围绕
`pi(x)`、Chebyshev 函数 `theta(x)=sum_{p<=x} log p`、`psi(x)=sum_{n<=x} Lambda(n)`
做数值计算与对比，展示素数定理常见等价刻画的经验行为（例如 `psi(x) ~ x`）。

## R02

MVP 的核心思想是“先构造可复用前缀数组，再做近似对比”:
1. 用埃拉托斯特尼筛法生成 `<= N` 的全部素数。
2. 构造 `pi_prefix[x]`、`theta_prefix[x]`、`psi_prefix[x]`。
3. 在若干采样点比较 `pi(x)` 与 `x/log x`、`li(x)`，并比较 `theta(x)/x`、`psi(x)/x` 与 1 的接近程度。

## R03

输入输出约定（无交互）：
- 输入: 代码内固定 `N=100000` 与固定采样点集合。
- 输出: 终端报告，包括 `pi(N)`、`theta(N)`、`psi(N)`、误差摘要，以及逐采样点的对比表。

## R04

应用场景：
- 解析数论课程中演示“素数定理等价命题”的数值直觉。
- 作为后续实现 Riemann 显式公式或零点相关实验前的基线模块。
- 用于快速检查筛法与 von Mangoldt 累积实现是否一致。

## R05

数学背景（与证明相关）：
- 素数定理（PNT）陈述 `pi(x) ~ x/log x`。
- 经典等价形式之一是 `psi(x) ~ x`（或 `theta(x) ~ x`）。
- 因此在算法层面，计算 `psi(x)/x`、`theta(x)/x` 的趋近行为，是“证明相关”数值实验的直接抓手。

## R06

高层算法步骤：
1. 用筛法得到有序素数表 `primes`。
2. 构造 prime indicator 并前缀和得到 `pi_prefix`。
3. 在素数位置写入 `log p` 并前缀和得到 `theta_prefix`。
4. 枚举每个素数幂 `p^k<=N`，在该位置写入 `log p`（即 `Lambda(n)` 非零点），前缀和得到 `psi_prefix`。
5. 数值近似 `li(x)=int_2^x dt/log t`，与 `pi(x)` 对比。
6. 输出误差表与摘要统计。

## R07

正确性直觉：
- 筛法正确性来自“任一合数存在不超过平方根的素因子”。
- `pi_prefix[x]` 是 prime indicator 的前缀和，按定义即 `pi(x)`。
- `theta_prefix[x]` 在每个素数位置累计 `log p`，与定义一致。
- `psi_prefix[x]` 通过在每个素数幂位置填入 `log p` 来构造 von Mangoldt 和，恰好等于 `sum_{n<=x} Lambda(n)`。

## R08

时间复杂度：
- 筛法: `O(N log log N)`。
- 构造 `pi/theta` 前缀: `O(N)`（加上遍历素数写值）。
- 构造 `psi` 时枚举素数幂，总体低于线性级常数倍，主导仍是筛法。
- 近似 `li(x)` 采用一次统一网格积分并插值，多采样点共享，整体近似 `O(G)`（`G` 为网格点数）。

## R09

空间复杂度：
- 筛数组与前缀数组都为 `O(N)`。
- 额外采样表规模远小于 `N`。
- 总空间复杂度 `O(N)`。

## R12

关键输出字段：
- `pi(N)`: 不超过 `N` 的素数个数。
- `x/log(x)`、`li(x)`: PNT 常见近似基准。
- `theta(x)`、`psi(x)`: 与 PNT 等价刻画紧密相关的函数。
- `psi(x)/x`、`theta(x)/x`: 接近 1 的程度。
- `rel_err_*`: 相对误差，衡量近似质量。

## R13

边界与鲁棒性：
- `N<2` 时返回空素数表和零前缀，避免越界。
- `x/log x` 在 `x<=2` 时无意义，本实现采样点仅取 `>=10`。
- `pandas` 不可用时自动走纯文本打印，保证 `python3 demo.py` 仍可运行。

## R14

局限性：
- 数值实验不能替代严格证明，只能提供经验支持。
- `li(x)` 使用数值积分近似，不是高精度特殊函数求值器。
- `N=100000` 仅用于演示趋势，不足以观察极远区间细节。

## R16

最小测试建议：
- 功能测试: `N=30` 时 `pi(30)=10`。
- 一致性测试: `pi_prefix` 单调不减，`pi_prefix[N] == len(primes)`。
- 定义测试: `theta(10)` 应为 `log 2 + log 3 + log 5 + log 7`。
- 结构测试: `psi_prefix` 在非素数幂处不应产生新增跳跃。

## R17

与替代实现对比：
- 直接逐点试除: 更直观但在 `N` 稍大时明显更慢。
- 直接调用黑盒库函数: 开发快但不利于理解证明相关量的构造细节。
- 本实现（筛 + 前缀构造 + 数值积分）在“可读性、可复核性、运行效率”间较平衡，适合作为教学型 MVP。

## R18

`demo.py` 源码级算法流程（8 步）：
1. `main()` 固定 `limit=100000` 与采样点，调用 `sieve_primes(limit)` 得到素数表。
2. `sieve_primes` 建立布尔数组并从 `p^2` 开始按步长 `p` 清零合数位，最终提取所有素数下标。
3. `build_prefix_tables` 用 prime indicator 的前缀和构造 `pi_prefix`。
4. 同函数在每个素数位置写入 `log(p)`，前缀和得到 `theta_prefix`。
5. 同函数枚举每个素数的幂 `p, p^2, ...`，在对应下标写入 `log(p)` 形成 `Lambda(n)` 稀疏表，再前缀和得到 `psi_prefix`。
6. `approximate_li_on_points` 仅在 `[2, max(sample)]` 建一条积分网格，对 `1/log t` 做梯形积分累计后插值到各采样点，得到 `li(x)` 近似。
7. `evaluate_pnt_records` 逐采样点读取 `pi/theta/psi` 与 `x/log x`、`li(x)`，计算相对误差和比值，生成结构化记录。
8. `render_report` 输出摘要和明细表（优先 `pandas`，否则纯文本），形成可直接复核的非交互报告。
