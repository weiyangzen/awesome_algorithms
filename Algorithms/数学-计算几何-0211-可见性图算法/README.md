## R01

可见性图算法（Visibility Graph）用于求解二维平面中“多边形障碍物 + 起点 + 终点”的最短无碰路径问题。

核心目标是把连续几何空间离散成图：
- 节点: 起点、终点、所有障碍物顶点
- 边: 两节点之间直线段若不穿过障碍物内部，则连边
- 权重: 欧氏距离

最终在该图上运行最短路（Dijkstra）得到路径。

## R02

问题输入与前提约束:
- 障碍物为简单多边形（顶点按顺时针或逆时针给出，不能自交）
- 起点与终点不在障碍物严格内部
- 可允许沿障碍物边界移动

问题输出:
- 可见性图节点集合与边集合
- 起点到终点最短路径长度
- 对应路径节点序列与坐标序列

## R03

算法思想分两层:
- 几何层: 判断“两个点是否可见”（连线不穿障碍内部）
- 图论层: 在可见图上做单源最短路

为什么只需考虑障碍顶点:
- 在折线最短路径中，转折点可归约到障碍物顶点（或起终点）
- 因而搜索空间可由无限连续空间缩小到有限离散图

## R04

本目录 MVP 的数据结构:
- `Node` 数据类: `idx`, `point`, `kind`, `obstacle_id`, `vertex_id`
- `obstacles: list[np.ndarray]`: 每个障碍物是 `N x 2` 浮点数组
- `adjacency: dict[int, list[tuple[int, float]]]`: 邻接表存图

关键几何函数:
- `orient(a, b, c)`: 叉积方向
- `on_segment(a, p, b)`: 点在线段上判定
- `segments_intersect(...)`: 线段相交判定
- `point_in_polygon_strict(...)`: 点在多边形严格内部判定
- `are_visible(p, q, obstacles)`: 可见性判定

## R05

MVP 主流程:
1. 构造固定示例场景（起点、终点、多个障碍物）
2. 组装节点集合（起终点 + 所有障碍顶点）
3. 枚举节点对并进行可见性判定，建立无向加权图
4. 用 Dijkstra 求起点到终点最短路
5. 打印图规模、最短距离、路径节点和路径坐标

## R06

正确性直觉:
- 若两点连线与任意障碍边发生非端点交叉，则该边必穿障碍区域，不可行
- 若线段中点落在某障碍严格内部，则该边穿过内部，不可行
- 经过上述过滤后得到的边均是可行通道
- 在该可行边图上用 Dijkstra，得到离散图上的全局最短路径

注意: 这是经典可见性图范式的工程化近似实现，适配一般多边形障碍路径规划。

## R07

设总节点数为 `V`（约等于障碍物顶点总数 + 2），总障碍边数为 `E_obs`。

复杂度:
- 建图: 枚举点对 `O(V^2)`，每次可见性检测最坏扫全部障碍边 `O(E_obs)`，总体约 `O(V^2 * E_obs)`
- 最短路: `O((V + E_graph) log V)`，其中 `E_graph` 为可见性图边数

空间复杂度:
- 邻接表 `O(V + E_graph)`

## R08

数值与鲁棒性处理:
- 使用 `EPS = 1e-9` 做浮点比较阈值
- 相交判定包含端点接触和共线特判
- `point_in_polygon_strict` 将边界点视为“非严格内部”，避免误杀贴边可行路径
- 共线重叠增加轴向投影重叠判断，避免“贴边穿透”

## R12

边界情况清单:
- 起点或终点在障碍物内部: 直接抛出异常
- 起点与终点无可行通道: 返回空路径
- 线段仅在障碍顶点处接触: 允许（共享端点情形）
- 线段与障碍边共线且有实质重叠: 禁止
- 点重合: 不建立自环边

## R13

与常见替代方案对比:
- 栅格 + A*: 实现简单，但精度依赖栅格分辨率，路径有离散化误差
- PRM/RRT: 适合高维或复杂空间，但最短性与稳定性依赖采样
- 可见性图: 在二维多边形障碍场景中通常更直接，路径质量高，解释性强

本题目标是“可解释 + 可验证 + 小而完整的 MVP”，因此选可见性图。

## R18

`demo.py` 的源码级算法流（8 步）:
1. `demo_scene()` 生成固定起点、终点与 3 个多边形障碍。
2. `build_visibility_graph()` 先用 `point_in_polygon_strict()` 校验起终点不在障碍物内部。
3. 将起点、终点、所有障碍顶点封装为 `Node`，建立候选节点集合。
4. 双重循环枚举每一对节点 `(i, j)`，调用 `are_visible(points[i], points[j], obstacles)`。
5. `are_visible()` 内部逐障碍逐边调用 `segments_intersect()`，过滤非共享端点交叉与共线重叠。
6. `are_visible()` 再取线段中点，用 `point_in_polygon_strict()` 拒绝“穿障碍内部”的连边。
7. 对可见节点对，按 `euclidean()` 计算权重并加入邻接表，形成无向可见性图。
8. `dijkstra()` 在该图上求 `start_idx=0` 到 `goal_idx=1` 的最短路并回溯输出路径。

这 8 步对应本实现的完整执行链条，没有把核心逻辑交给黑箱库函数。
