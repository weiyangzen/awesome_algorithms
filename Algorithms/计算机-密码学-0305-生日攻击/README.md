# 生日攻击

- UID: `CS-0160`

## R01

`生日攻击（Birthday Attack）`是利用“生日悖论”思想寻找哈希碰撞的通用方法。  
其结论是：对一个 `n` 位哈希输出，找到任意一对碰撞样本通常只需约 `O(2^(n/2))` 次尝试，而不是 `O(2^n)`。

## R02

本条目目标是提供一个最小可运行 MVP，直观展示：
- 如何对截断哈希输出执行碰撞搜索；
- 碰撞样本数与理论近似值 `sqrt(pi/2) * 2^(n/2)` 的量级一致；
- 为什么哈希长度每增加 1 bit，抗碰撞成本会按平方根指数级上升。

## R03

攻击模型与前提：
- 攻击者无需控制输入语义，只需能不断生成候选消息；
- 目标是“任意两条不同消息同哈希”（碰撞），不是特定原像；
- 这里使用 `SHA-256` 后取前 `n` 位作为教学目标哈希，便于在有限时间内观察碰撞；
- 真实密码系统要求完整输出位数足够大，避免该攻击在可行预算内完成。

## R04

数学基础（设哈希空间大小 `N = 2^n`）：
- 抽取 `k` 个样本无碰撞概率近似为：`exp(-k(k-1)/(2N))`；
- 因此出现至少一次碰撞概率约：`1 - exp(-k(k-1)/(2N))`；
- 当碰撞概率接近 0.5 时，`k` 约为 `sqrt(2 ln 2) * 2^(n/2)`；
- 若看“首次碰撞期望尝试次数”，常用近似是 `E[k] ≈ sqrt(pi/2) * 2^(n/2)`。

## R05

`demo.py` 输入输出定义：
- 输入：脚本内固定参数（位数列表、试验次数、随机种子），无交互输入；
- 核心输入变量：`bits`（截断位数）、`trials`（重复实验次数）、`max_attempts`（安全上限）；
- 输出：每个 `bits` 的理论值、经验均值/中位数、比值，以及一组真实碰撞样本。

## R06

数据结构设计：
- `dict[int, bytes]`：键为截断哈希值，值为首次出现该值的消息；
- `CollisionResult`：记录一次成功碰撞的位数、尝试次数、碰撞哈希、两条消息与耗时；
- `numpy.ndarray`：存放多轮尝试次数并计算均值、标准差、分位数。

## R07

实现模块划分：
- `truncated_sha256_bits`：`SHA-256` 后截取前 `n` 位；
- `build_candidate_message`：构造确定性可复现实验样本；
- `find_first_collision`：哈希表增量搜索首次碰撞；
- `expected_attempts`：给出理论期望值；
- `run_trials`：多轮实验统计；
- `main`：按多个位数执行实验并打印结论。

## R08

复杂度分析：
- 单轮首次碰撞搜索，若实际尝试次数为 `k`，时间复杂度 `O(k)`；
- 空间复杂度 `O(k)`（需保存已见哈希桶）；
- 理论上 `k` 的量级约为 `2^(n/2)`，因此生日攻击总体成本是 `O(2^(n/2))`。

## R09

正确性说明：
- 哈希表记录“某哈希值第一次出现时对应消息”；
- 当新消息产生哈希值 `h` 且 `h` 已存在，且两条消息不同，即得到碰撞对；
- `demo.py` 在返回前会再次计算并断言两消息截断哈希相等；
- 同时断言消息字节串不同，避免把同一输入重复计为伪碰撞。

预期输出特征：

## R12

边界与限制：
- 本演示使用“截断哈希”（20/24/28 位）以便快速复现实验；
- 这不是对完整 `SHA-256` 的现实可行攻击规模；
- 若 `max_attempts` 设得过小，极端情况下可能在上限前未命中；
- 哈希函数被假设为近似均匀映射，真实实现可能存在细微偏差，但不影响教学结论。

## R13

安全启示：
- 抗碰撞安全强度约为输出位数的一半；
- 若希望达到 `s` 位碰撞安全，哈希输出应至少约 `2s` 位；
- 在签名、证书、去重指纹等场景，必须避免使用过短摘要；
- 对于可被攻击者自由构造输入的系统，碰撞风险评估尤为关键。

## R14

与相邻攻击目标对比：
- 生日攻击（碰撞）：找任意 `m1 != m2` 且 `H(m1)=H(m2)`，复杂度约 `2^(n/2)`；
- 原像攻击：给定 `h` 找 `m` 使 `H(m)=h`，复杂度约 `2^n`；
- 次原像攻击：给定 `m1` 找 `m2` 使 `H(m2)=H(m1)`，常见复杂度也接近 `2^n`；
- 因此“抗碰撞”通常先成为哈希长度设计的关键门槛。

## R18

源码级算法流程拆解（对应 `demo.py`）：
1. `main` 定义测试位数集合（20/24/28）和每个位数的试验次数，然后逐组调用 `run_trials`。  
2. `run_trials` 为每轮生成确定性种子，调用 `find_first_collision(bits, seed, max_attempts)` 执行一次完整碰撞搜索。  
3. `find_first_collision` 初始化哈希表 `seen`，并在循环中不断调用 `build_candidate_message` 生成新消息。  
4. 每条消息经 `truncated_sha256_bits` 计算“SHA-256 前 `bits` 位”得到桶编号 `digest_value`。  
5. 若该桶首次出现，则写入 `seen[digest_value] = message`；若已存在旧消息且不同，则判定碰撞命中。  
6. 命中后执行断言：两条消息不同、两者截断哈希相同，然后打包成 `CollisionResult` 返回。  
7. `run_trials` 将多轮 `attempts` 汇总为 `numpy` 数组，计算均值、中位数、标准差，并与 `expected_attempts(bits)` 对比。  
8. `main` 打印每个位数的理论-经验对照与示例碰撞，形成“攻击成本约为 `2^(n/2)`”的可观察证据。
