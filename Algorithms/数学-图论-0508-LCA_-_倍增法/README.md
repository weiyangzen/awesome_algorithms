## R01

LCA（Lowest Common Ancestor，最近公共祖先）用于回答树上两点 `u, v` 的“最近公共祖先是谁”。

倍增法（Binary Lifting）是最常见的 LCA 预处理方案：
- 预处理阶段构建 `up[k][v]`，表示节点 `v` 的 `2^k` 级祖先；
- 查询时先把深度对齐，再从高位到低位同时上跳；
- 在 `O(log n)` 时间得到 LCA。

本目录提供一个纯 Python、可直接运行、带随机回归校验的最小 MVP。

## R02

问题定义（本实现）：
- 输入：
  - 节点数 `n`；
  - 无向树边集 `edges`（节点编号 `0..n-1`）；
  - 根节点 `root`（默认 `0`）；
  - 查询点对 `(u, v)`。
- 输出：
  - `lca(u, v)`；
  - 可选派生量：两点距离 `dist(u, v)`；
  - 回归测试结果（与朴素 LCA 对照的一致性）。

`demo.py` 中内置固定样例 + 随机样例，无需交互输入。

## R03

核心数学与性质：

1. 祖先跳表定义：
   `up[0][v] = parent(v)`，
   `up[k][v] = up[k-1][ up[k-1][v] ]`。
2. 深度对齐：
   若 `depth[u] > depth[v]`，可按二进制拆分 `depth[u]-depth[v]`，让 `u` 逐级上跳到同深度。
3. 同步上跳：
   对 `k` 从大到小，若 `up[k][u] != up[k][v]`，则同步更新 `u = up[k][u], v = up[k][v]`。
4. 终止结论：
   当二者再跳一步即相同父节点时，`up[0][u]`（或 `up[0][v]`）就是 LCA。

## R04

算法流程（高层）：
1. 构建邻接表，校验输入边合法。
2. 从 `root` BFS，得到每个节点的 `depth` 与 `up[0]`（直接父亲）。
3. 基于递推式填充整张 `up` 表（`k=1..log-1`）。
4. 查询时先做深度对齐（把更深节点上跳）。
5. 若对齐后相等，直接返回该节点。
6. 否则从最高幂到最低幂检查，找到仍可“不同步上跳”的层。
7. 循环结束后返回两者父节点 `up[0][u]`。
8. 距离可由公式 `depth[u] + depth[v] - 2*depth[lca]` 得到。

## R05

核心数据结构：
- `adj: List[List[int]]`：树的邻接表。
- `depth: List[int]`：每个节点深度。
- `up: List[List[int]]`：倍增祖先表，形状约为 `log2(n) * n`。
- `BinaryLiftingLCA`：封装预处理和查询 API：
  - `kth_ancestor(node, k)`；
  - `lca(u, v)`；
  - `distance(u, v)`。

## R06

正确性要点：
- BFS 构建的 `depth` 与 `up[0]` 定义了根树结构。  
- `up[k][v]` 递推严格对应“跳 `2^k` 层祖先”。  
- 深度对齐后，`u, v` 位于同一层；若已相同，该点即最近公共祖先。  
- 从高位到低位同步上跳保证：
  - 不会越过 LCA；
  - 最终停在 LCA 的两个不同子节点上。  
- 因此返回其父节点 `up[0][u]` 必为且仅为 LCA。

## R07

复杂度分析：
- 预处理：
  - BFS 建树 `O(n)`；
  - 填充倍增表 `O(n log n)`。
- 单次查询：`O(log n)`。
- 空间复杂度：`O(n log n)`（主要来自 `up` 表）。

## R08

边界与异常处理：
- `n <= 0`、`root` 越界：抛 `ValueError`。
- 边数不等于 `n-1`：抛 `ValueError`。
- 边端点越界或自环：抛 `ValueError`。
- 图不连通（不是一棵树）：抛 `ValueError`。
- 查询节点越界或 `k < 0`：抛 `ValueError`。

备注：本 MVP 假设输入是无向树，不处理带权图和一般图。

## R15

方法对比：
- 对比朴素“向上爬父节点”：
  - 朴素法单次常为 `O(height)`；
  - 倍增法单次稳定 `O(log n)`。
- 对比 Euler Tour + RMQ：
  - RMQ 查询可达 `O(1)`（视实现）；
  - 但构建与实现复杂度更高。
- 对比 Tarjan 离线 LCA：
  - Tarjan 适合“离线一次性批量查询”；
  - 倍增法更适合在线多次查询。

## R18

`demo.py` 源码级算法流程（9 步）：
1. 在 `BinaryLiftingLCA.__init__` 中检查 `n/root/edges` 的合法性，并构建无向邻接表。
2. 使用 BFS 从 `root` 出发，计算每个节点深度 `depth`，同时记录直接父亲 `up[0]`。
3. 若 BFS 后存在未访问节点，立即报错，阻断“非树输入”。
4. 令 `log = bit_length(n)`，分配 `up[log][n]` 并按递推式填表。
5. 查询 `lca(u, v)` 时，先调用 `kth_ancestor` 把较深节点上跳到同一深度。
6. 若对齐后两节点已相同，直接返回该节点作为 LCA。
7. 否则从大到小遍历幂次 `k`，在 `up[k][u] != up[k][v]` 时同步上跳。
8. 循环结束后返回 `up[0][u]` 作为最近公共祖先，并可据此计算距离。
9. `run_random_regression` 用 `naive_lca` 做 1000 次随机对拍（5 个规模 * 200 次）验证实现正确性。
