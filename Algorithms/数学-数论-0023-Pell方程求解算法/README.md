# Pell方程求解算法

- UID: `MATH-0023`

## R01

问题定义: 对非平方正整数 `D`，求 Pell 方程

`x^2 - D y^2 = 1`

的整数解，尤其是最小正解（基本解）`(x1, y1)`，其中 `x1 > 0, y1 > 0`。

## R02

核心思想: 利用 `sqrt(D)` 的连分数周期展开。

关键事实:
1. `sqrt(D)` 的简单连分数必为周期形式。
2. 该周期的某个收敛分数 `p/q` 正好给出 Pell 方程的基本解。
3. 得到基本解后，所有正解可由
   - `x_n + y_n*sqrt(D) = (x1 + y1*sqrt(D))^n`
   递推生成。

## R03

输入:
- 一个非平方正整数 `D`。
- 可选参数 `count`，表示需要生成前多少组正解。

输出:
- 基本解 `(x1, y1)`。
- 前 `count` 组正解 `[(x1, y1), (x2, y2), ...]`。

约定:
- 当 `D <= 0` 或 `D` 是完全平方数时，问题不满足标准 Pell 方程前提，抛出异常。

## R04

正确性直观说明:
1. 连分数理论保证 `sqrt(D)` 的周期收敛分数中存在满足 `p^2 - D q^2 = ±1` 的项。
2. 根据周期长度奇偶性选取对应收敛分数，可得到 `+1`（而不是 `-1`）右端的最小正解。
3. 单位元乘法递推
   - `x' = x1*x + D*y1*y`
   - `y' = x1*y + y1*x`
   保持不变量 `x'^2 - D*y'^2 = 1`。
4. 因为 `x1 + y1*sqrt(D) > 1`，幂次递增会严格产生新的更大正解，且覆盖全部正解。

## R05

伪代码:

```text
FUNDAMENTAL_PELL(D):
    require D > 0 and D is not a perfect square

    a0 = floor(sqrt(D))
    m, d, a = 0, 1, a0
    period = []

    repeat:
        m = d*a - m
        d = (D - m*m) / d
        a = floor((a0 + m) / d)
        append a to period
    until a == 2*a0

    L = len(period)
    if L is even:
        target_index = L - 1
    else:
        target_index = 2*L - 1

    coeffs = [a0] + first target_index terms of repeated period
    (p, q) = convergent(coeffs)
    return (p, q)

GENERATE_SOLUTIONS(D, x1, y1, count):
    x, y = x1, y1
    repeat count times:
        output (x, y)
        (x, y) = (x1*x + D*y1*y, x1*y + y1*x)
```

## R06

本目录 `demo.py` 的实现采用:
1. `math.isqrt` 做整数平方根，避免浮点误差。
2. 显式实现周期连分数生成，不依赖外部数论黑箱库。
3. 显式实现收敛分数递推（分子分母二阶线性递推）。
4. 通过整型大数直接运算，天然支持超 64 位范围的解。

## R07

时间复杂度（高层）:
- 周期展开长度记为 `L`，求基本解主流程约 `O(L)` 次迭代。
- 生成前 `k` 个解约 `O(k)` 次递推。

补充:
- 严格位复杂度依赖大整数乘法成本；当解很大时，单步乘法不再是常数时间。

## R08

空间复杂度:
- 存储一个周期数组 `period`，约 `O(L)`。
- 递推生成解时仅需常数个大整数状态（若不保存全部输出则为 `O(1)` 额外空间）。

## R09

边界与特殊情况:
1. `D <= 0`：非法输入。
2. `D` 为完全平方数：`x^2 - D y^2 = 1` 不属于标准 Pell 非平凡情形。
3. 某些 `D`（如 `61`）的最小解非常大，但 Python 大整数可以正确承载。
4. 若仅需基本解，不必生成后续解，避免不必要的大数增长。

## R10

适用场景:
1. 数论教学中展示“连分数 <-> Pell 方程”对应关系。
2. 需要显式构造二次域单位元的基础实验。
3. 竞赛或算法题中求指定 `D` 的最小解。
4. 需要一批可验证的大整数解样本用于测试。

## R11

局限与不适用场景:
1. 对超大 `D`，虽迭代步数可接受，但大整数运算会变重。
2. 本 MVP 聚焦标准方程 `x^2 - D y^2 = 1`，不直接覆盖 `= N` 的一般 Pell 型方程。
3. 未做并行优化或分布式大数加速，定位是教学与中小规模验证。

## R12

与相关方法对比:
1. 连分数法:
   - 理论完整、实现紧凑、可直接得到最小正解。
2. Chakravala（婆罗摩笈多-巴斯卡拉方法）:
   - 也很高效，但实现细节与参数选择更复杂。
3. 暴力枚举 `y`:
   - 对小 `D` 可验证正确性，但对实际求解几乎不可扩展。

## R13

示例 (`D = 13`):
1. `sqrt(13) = [3; (1,1,1,1,6)]`，周期长度 `L = 5`（奇数）。
2. 因 `L` 为奇数，应取索引到 `2L-1` 的收敛分数。
3. 得到基本解 `(x1, y1) = (649, 180)`。
4. 验证: `649^2 - 13*180^2 = 421201 - 421200 = 1`。
5. 下一组解由递推得到 `(842401, 233640)`，同样满足方程。

## R14

工程实现要点:
1. 连分数循环终止条件使用 `a == 2*a0`（标准 `sqrt(D)` 周期判据）。
2. 收敛分数递推使用整型公式，避免分数对象反复约分。
3. 在 `demo.py` 中保留 `brute_force_min_solution`，用于小规模交叉验证。
4. 打印时给出方程残差 `x^2 - D*y^2`，便于人工快速确认。

## R15

最小测试清单:
1. 输入合法性: `D=1,4,0,-3` 应抛异常。
2. 小规模已知解:
   - `D=2` -> `(3,2)`
   - `D=3` -> `(2,1)`
   - `D=5` -> `(9,4)`
   - `D=13` -> `(649,180)`
3. 中等规模已知解:
   - `D=61` -> `(1766319049, 226153980)`
4. 随机小 `D` 通过暴力法对拍基本解。

`demo.py` 已包含自动断言与对拍步骤。

MVP 输出解读:

2. `fundamental` 是最小正解 `(x1, y1)`。

## R18

源码级算法流程拆解（对应 `demo.py`，非黑箱）:
1. `continued_fraction_sqrt_period(D)` 用 `(m,d,a)` 递推生成 `sqrt(D)` 的周期项序列。
2. `fundamental_solution_pell(D)` 读取周期长度奇偶性，决定目标收敛项索引（`L-1` 或 `2L-1`）。
3. `convergent_from_coeffs(coeffs)` 以二阶递推累积得到目标收敛分数 `(p,q)`。
4. 将 `(p,q)` 作为基本解返回，并在 `verify_pell` 中直接检查 `p^2 - D*q^2 == 1`。
5. `generate_solutions(D, x1, y1, count)` 使用单位元乘法递推，逐个构造后续正解。
6. `brute_force_min_solution` 在小范围扫描 `y` 做交叉验证，确保基本解最小性与正确性。
7. `main()` 先跑断言测试，再打印多个样例，形成可复现实验输出。
