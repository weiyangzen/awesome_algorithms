# 二阶锥规划 (SOCP)

## R01

二阶锥规划（Second-Order Cone Programming, SOCP）是凸优化中的重要问题类，标准形式可写为：

`min c^T x`  
`s.t. ||A_i x + b_i||_2 <= d_i^T x + e_i, i=1,...,m`  
`Fx = g`

其中每个约束都表示变量落在一个二阶锥（Lorentz cone）内。SOCP严格包含线性规划（LP），并可表达大量工程问题（鲁棒回归、波束形成、投资组合风险控制等）。

## R02

本目录 MVP 选择一个最小但真实的 SOCP 实例：

`min_{x,t} t`  
`s.t. ||A x - b||_2 <= t`  
`t >= 0`

这是典型 SOCP：把 `u = A x - b, v = t`，约束就是 `||u||_2 <= v`。其物理意义是最小化残差上界；在该例中也等价于最小二乘残差范数最小化。

## R03

选择该实例的原因：

- 结构足够简单，方便直接展示 SOCP 的锥约束本质。
- 仍保留真实算法要素：线性目标 + 非线性锥不等式 + 约束可行性检验。
- 可用 `numpy.linalg.lstsq` 作为基准，验证 SOCP 求解结果是否可信。

## R04

数学建模细节：

- 决策变量：`z = [x, t] in R^(n+1)`。
- 目标函数：`f(z) = t`。
- 锥约束函数：`g(z) = t - ||A x - b||_2 >= 0`。
- 边界条件：`t >= 0`。

该约束定义了一个凸可行域；目标函数是线性的，因此整体是凸优化问题。

## R05

MVP 求解策略：使用 `scipy.optimize.minimize` 的 `SLSQP` 方法求解。

实现层面传入：

- 目标函数及其梯度：`grad_f = [0,...,0,1]`
- 不等式约束 `g(z) >= 0` 及其雅可比
- 变量边界（仅 `t` 下界为 `0`）

虽然 `SLSQP` 是通用约束优化器，但本示例中它直接处理了 SOCP 的核心锥约束。

## R06

正确性与可行性检查：

- 约束残差（cone slack）：`t - ||A x - b||_2` 应非负（允许微小数值误差）。
- 与最小二乘基准一致性：
  - `||A x_socp - b||_2` 应接近 `||A x_ls - b||_2`
  - `x_socp` 与 `x_ls` 的差应很小

`demo.py` 中提供自动断言，若偏差超过阈值将抛出异常。

## R07

复杂度（单次函数/梯度计算）：

- 计算 `A x - b` 成本 `O(mn)`。
- 计算范数与目标为 `O(m)`。
- 计算约束雅可比核心项 `A^T r` 成本 `O(mn)`。

若迭代 `K` 次，总体约为 `O(Kmn)` 量级（忽略 `SLSQP` 内部线性代数常数项）。

## R08

边界和异常处理：

- 输入 `A` 必须是二维，`b` 必须是一维，且 `A.shape[0] == b.shape[0]`。
- 禁止 `nan/inf`。
- 初值构造为严格可行：`x0=0, t0=||b||+1`。
- 若优化器返回失败状态，直接抛出 `RuntimeError` 并带状态信息。

## R09

MVP 取舍说明：

- 只依赖 `numpy + scipy`，不引入额外 SOCP 专用库。
- 不实现通用多锥、多等式约束解析器，只实现一个清晰案例。
- 重点放在“可运行 + 可验证 + 可解释”，而非工程级高性能求解框架。

## R10

`demo.py` 的模块职责：

- `validate_problem_data`：校验输入维度和数值有效性。
- `build_socp_problem`：生成可复现实验数据。
- `soc_constraint_value` / `soc_constraint_jacobian`：定义锥约束与雅可比。
- `solve_socp_least_residual`：调用 SLSQP 完成求解。
- `compute_least_squares_reference`：计算最小二乘基准解。
- `summarize_and_assert`：打印关键指标并做数值一致性断言。
- `main`：串联整个流程。

## R12

输出字段说明：

- `success/status`：优化器是否正常收敛及状态码。
- `objective t*`：最优上界变量。
- `||A x* - b||_2`：SOCP 解对应残差范数。
- `cone slack`：可行性裕量，理论应 `>=0`。
- `LS residual norm`：最小二乘基准残差范数。
- `|t* - LS|`：SOCP 与基准目标差。
- `||x_socp - x_ls||_2`：变量解偏差。

## R13

最小测试覆盖（脚本内置）：

- 生成一组过定约随机线性系统（`m > n`）并加噪声。
- 求解 SOCP。
- 计算并对比最小二乘参考解。
- 执行可行性与一致性断言，确保结果可靠。

## R14

关键参数建议：

- `ftol`：收敛精度阈值，默认 `1e-12`（较严格）。
- `maxiter`：最大迭代次数，默认 `500`。
- 规模参数 `m, n`：建议先用小规模验证，再扩大。
- 断言容差 `feas_tol/value_tol`：用于数值误差窗口控制。

## R15

与相关问题关系：

- 与 LP 的关系：LP 可视为 SOCP 的特例（无二范数锥项）。
- 与 QP 的关系：许多凸二次约束/鲁棒变体可重写为 SOCP。
- 与最小二乘关系：本案例展示了最小残差范数问题的 SOCP 等价表达。

## R16

典型应用场景：

- 鲁棒线性回归与不确定性约束建模。
- 金融风险控制（如均值-方差约束的锥化形式）。
- 信号处理/通信中的波束形成与功率约束。
- 结构设计中的应力/范数型安全约束。

## R18

`demo.py` 源码级算法流程（8 步）：

1. `build_socp_problem` 固定随机种子生成 `A,b`，并确保数据类型为 `float`。
2. `solve_socp_least_residual` 构造变量 `z=[x,t]`，其中目标是最小化最后一维 `t`。
3. 在同一函数中定义 `objective` 和常量梯度 `objective_grad`，将问题转为标准 NLP 接口。
4. 通过 `soc_constraint_value` 实现锥约束 `t-||Ax-b||_2 >= 0`，并用 `soc_constraint_jacobian` 提供解析雅可比。
5. 设置初值 `x0=0, t0=||b||+1`（严格可行）与边界 `t>=0`，交给 `scipy.optimize.minimize(method="SLSQP")`。
6. SciPy 的 SLSQP 在每次迭代里线性化约束并构造二次子问题，更新 `z`，直到满足一阶最优性和可行性容差。
7. 迭代结束后提取 `x*`、`t*`，并由 `compute_least_squares_reference` 计算 `x_ls` 与最小二乘残差范数作为独立基准。
8. `summarize_and_assert` 打印收敛信息、锥可行性、目标差、解差，并执行阈值断言，确保该 SOCP MVP 输出可信。
