## R01

公共子表达式消除（Common Subexpression Elimination, CSE）用于删除重复计算。
如果在某个程序点再次出现表达式 `x op y`，且 `x/y` 自上次计算后没有被重新定义，就可以复用已有结果，而不是再次执行算术运算。

本条目实现的是可运行、可审计的最小版本：**块内 CSE**。

## R02

本 MVP 的目标是把“重复二元表达式”改写成复制语句，直接减少 IR 中的算术指令条数。

实现范围：
- 输入：手工构造的三地址码风格基本块序列；
- 输出：优化前后 IR、命中追踪日志、统计信息、语义一致性验证结果；
- 约束：不依赖 LLVM 之类黑盒优化器，核心逻辑全部在 `demo.py` 内展开。

## R03

本实现使用的 IR 指令类型：
- `BinOp(target, op, left, right)`
- `Assign(target, source)`
- `Const(target, value)`

优化语义：
- 若当前 `BinOp` 对应的值编号表达式在块内已出现且仍有可用持有变量，改写为 `Assign(target, holder)`；
- 否则保留原 `BinOp` 并登记为新的表达式值类。

## R04

核心判等机制使用**局部值编号（LVN）**：
- 变量绑定到值编号（VN）；
- 表达式键为 `(op, vn_left, vn_right)`；
- 对交换律算子（`+`、`*`）做规范化（按 VN 排序）；
- 每次变量被重定义时，变量会绑定到新 VN，天然避免旧值误复用。

因此不需要手写复杂 kill 集，也能正确处理重定义。

## R05

`demo.py` 的关键状态结构：
- `var_vn: variable -> VN`：当前变量值属于哪个等价类；
- `expr_vn: (op, vn1, vn2) -> VN`：表达式值类表；
- `vn_holders: VN -> {variables}`：当前仍持有该值的变量集合；
- `const_vn: const -> VN`：常量值编号复用。

其中 `vn_holders` 的作用很重要：即使表达式类已知，如果没有任何变量还持有该值，也不能直接复制，必须重新计算。

## R06

示例程序含两个基本块：
- `B1` 展示重复 `a+b`、交换律 `b+a`、以及 `a` 重定义后的“旧表达式失效”；
- `B2` 展示 `x*y` 与 `y*x` 的复用，以及复制传播后的二次命中。

这组样例覆盖了 CSE 的三个关键点：
1. 直接重复表达式；
2. 交换律归一化；
3. 变量重定义导致的等价类切换。

## R07

复杂度（单基本块，指令数 `n`）：
- 时间复杂度：均摊近似 `O(n)`（哈希表查询/更新）；
- 空间复杂度：`O(n)`（最坏每条表达式都形成新值类）。

全程序按块独立执行时，总复杂度是所有块复杂度之和。

## R08

正确性直观说明：
- 若两条表达式映射到同一个 LVN 键，说明它们依赖的操作符和操作数值类一致；
- 当变量重定义时，`var_vn` 会更新，后续表达式键自然变化，不会误命中旧值；
- 命中时仅做“值复制”，不改变控制流与可见副作用（示例 IR 无副作用算子）。

因此优化前后应保持语义等价。

## R09

边界和限制：
- 当前是**块内** CSE，不跨块做全局可用表达式分析；
- 仅支持 `+/-/*` 三种二元算术；
- 假设表达式无副作用（不含函数调用、内存写、异常行为）；
- 浮点下 NaN/舍入等细节未建模，示例使用整数语义。

## R10

源码职责映射：
- `local_cse_block`：单块 LVN + CSE 主流程；
- `optimize_program`：遍历所有块并汇总统计；
- `execute_program`：解释执行 IR；
- `verify_semantics`：随机输入对比优化前后结果；
- `build_sample_program`：构造演示用 IR；
- `main`：串联输出、统计与校验。

## R13

验证策略采用“执行语义对拍”：
- 自动推导程序输入变量集合；
- 固定随机种子 `2026` 生成 200 组整数输入；

- 任意差异立即抛异常并打印反例。

该策略可快速发现错误复用或重定义处理错误。

## R14

CSE 常见错误与本实现规避方式：
- 错误 1：仅按变量名比较表达式，忽略重定义。
  - 规避：按值编号而非变量名做表达式键。
- 错误 2：表达式类命中后默认可复制，忽略结果变量可能已被覆盖。
  - 规避：通过 `vn_holders` 检查是否存在仍持有值的变量。
- 错误 3：未处理交换律，导致 `a+b` 与 `b+a` 无法复用。
  - 规避：对 `+/*` 做键规范化。

## R15

## R18

`demo.py` 源码级算法流程（8 步）：
1. `build_sample_program` 构造两块三地址码样例，包含重复表达式、交换律场景和变量重定义场景。  
2. `main` 调用 `optimize_program`，逐块进入 `local_cse_block`。  
3. `local_cse_block` 为每条指令维护 `LVNState`：`var_vn`、`expr_vn`、`vn_holders`、`const_vn`。  
4. 遇到 `BinOp` 时，先把两个操作数映射到 VN，再用 `canonical_expr_key` 生成规范化表达式键（`+/*` 会排序）。  
5. 若表达式键已存在且 `pick_holder` 找到可用持有变量，则把该 `BinOp` 改写为 `Assign(target, holder)`，记一次 CSE 命中。  
6. 若未命中，则保留原 `BinOp`，并把该表达式登记到 `expr_vn`，同时更新目标变量绑定到对应 VN。  
7. 优化完成后，`main` 使用 `numpy` 统计每块二元表达式削减量与总削减比例（`numpy` 只用于统计展示，算法逻辑不依赖黑盒库）。  
8. `verify_semantics` 以固定随机种子生成 200 组输入，分别执行优化前后 IR 并逐例比较结果，全部一致则报告通过。
