# 箍缩效应 (Pinch Effect)

- UID: `PHYS-0192`
- 学科: `物理`
- 分类: `等离子体物理`
- 源序号: `193`
- 目标目录: `Algorithms/物理-等离子体物理-0193-箍缩效应_(Pinch_Effect)`

## R01

箍缩效应（Pinch Effect）指载流等离子体因自身电流产生的环向磁场而受到径向内压，进而发生收缩。  
在最常见的 Z-pinch 场景中，轴向电流 `I` 诱导环向磁场 `B_theta`，磁压力与等离子体热压力竞争，决定半径演化与平衡位置。

## R02

本条目实现“最小可运行”的柱对称零维径向模型：

- 只跟踪等离子体柱半径 `R(t)` 与径向速度 `v_r(t)`；
- 电流 `I` 视为给定常数；
- 热压力按绝热压缩律随半径变化；
- 用阻尼项代表能量耗散与未解析机制；
- 目标是展示“磁压驱动收缩并趋于平衡半径”的核心机制。

## R04

`demo.py` 使用的连续模型（圆柱近似、单位长度分析）：

1. 环向磁场（安培定律）  
   `B_theta(R) = mu0 * I / (2*pi*R)`

2. 磁压力（向内）  
   `p_B(R) = B_theta^2/(2*mu0) = mu0*I^2/(8*pi^2*R^2)`

3. 热压力（向外，绝热）  
   `p_th(R) = p0 * (R0/R)^(2*gamma)`

4. 径向动力学方程  
   `dR/dt = v_r`  
   `dv_r/dt = (2*pi*R/m_l) * (p_th - p_B) - nu*v_r`

其中 `m_l` 为单位长度质量（line mass），`nu` 为线性阻尼系数。

平衡半径由 `p_th(R_eq) = p_B(R_eq)` 给出：

`R_eq = [p0*R0^(2*gamma) / (mu0*I^2/(8*pi^2))]^(1/(2*gamma-2))`（要求 `gamma > 1`）。

## R05

离散与实现策略：

- 时间离散：显式半隐式欧拉（先更新速度，再更新半径）；

- 状态变量：`R, v_r`；

## R10

正确性直觉：

1. 当 `p_B > p_th` 时，`dv_r/dt < 0`，半径应收缩；
2. 收缩后 `p_th ~ R^(-2*gamma)` 上升更快，可与 `p_B ~ R^(-2)` 竞争；
3. 在 `gamma > 1` 情况下存在稳定平衡半径 `R_eq`；
4. 加入阻尼后，`R(t)` 应从初值收敛到 `R_eq` 附近；
5. 因此“收缩 + 力平衡残差减小”是最核心的数值验收信号。

## R14

常见失效模式：

1. `dt` 过大：显式推进误差放大，可能出现半径越界；
2. `I` 过大且阻尼不足：快速塌缩触发半径下限异常；
3. `gamma <= 1`：平衡半径公式失效，不满足当前模型假设；
4. `m_l` 过小：同压差下加速度过大，导致数值刚性上升；
5. 把本模型用于不稳定性主导问题，会得到“看似平滑但物理不充分”的结果。

## R18

`demo.py` 的源码级算法流程（8 步）：

1. `PinchConfig` 固化电流、初始半径、线密度、绝热指数、阻尼和时间步等参数。  
2. `magnetic_field_theta` 与 `magnetic_pressure` 根据安培定律计算 `B_theta(R)` 与 `p_B(R)`。  
3. `thermal_pressure` 按 `p_th = p0*(R0/R)^(2*gamma)` 计算绝热热压力。  
4. `equilibrium_radius` 由 `p_th = p_B` 的解析式给出理论平衡半径，用于后验核验。  
5. `simulate_pinch` 在每步先算压差与加速度，再更新 `v_r` 和 `R`，并记录完整时序。  
6. 时间循环结束后，构造包含 `time/radius/velocity/pressures/B_theta` 的 `pandas` 表。  
7. `main` 计算关键诊断：压缩比、平衡半径误差、末态力平衡残差、有限值与边界条件。  
8. 打印检查项并输出 `Validation: PASS/FAIL`；若失败则以非零状态退出。

第三方库没有作为黑盒求解器：

- `numpy` 仅用于数值数组与基础运算；
- `pandas` 仅用于整理输出表格；
- 箍缩动力学方程、离散推进与验收逻辑均在源码中显式实现。
