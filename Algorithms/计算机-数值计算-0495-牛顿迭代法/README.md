# 牛顿迭代法

- UID: `CS-0325`
- 学科: `计算机`
- 分类: `数值计算`
- 源序号: `495`
- 目标目录: `Algorithms/计算机-数值计算-0495-牛顿迭代法`

## R01

本条目实现牛顿迭代法（Newton's Method）的最小可运行 MVP，用于求解一元非线性方程 `f(x)=0`。

MVP 范围：
- 显式实现迭代公式，而不是调用黑盒求根器；
- 支持“解析导数”和“中心差分近似导数”两种模式；
- 提供收敛日志（每步 `x, f(x), f'(x), step`）；
- 用多个确定性测试函数验证精度和鲁棒性。

## R02

问题定义：
- 输入：
  - 目标函数 `f: R -> R`；
  - 初值 `x0`；
  - 导数函数 `df`（可选）；
  - 收敛阈值 `tol`，最大迭代次数 `max_iter`；
  - 阻尼系数 `damping`（`0 < damping <= 1`）。
- 输出：
  - 近似根 `x*`；
  - 是否收敛 `converged`；
  - 迭代步明细与停止原因。

目标是让脚本在无交互输入条件下稳定运行并输出可核查的数值结果。

## R03

核心公式：

1. 牛顿更新公式

`x_{k+1} = x_k - f(x_k)/f'(x_k)`

2. 阻尼牛顿（本实现默认可选）

`x_{k+1} = x_k - damping * f(x_k)/f'(x_k)`

3. 若未提供解析导数，使用中心差分近似

`f'(x) ~= (f(x+h)-f(x-h)) / (2h)`，其中 `h = diff_eps * max(1, |x|)`。

## R04

算法流程：
1. 检查参数合法性（`tol > 0`、`max_iter >= 1`、`0 < damping <= 1`）。
2. 从 `x0` 开始，计算 `f(x_k)`。
3. 若 `|f(x_k)| <= tol`，判定残差收敛并终止。
4. 计算导数 `f'(x_k)`（优先解析导数，否则中心差分）。
5. 若导数过小或非有限，判定失败并终止（避免除零/数值爆炸）。
6. 计算步长 `step = -damping * f/df`，更新 `x_{k+1}`。
7. 若步长范数 `|step| <= tol * max(1, |x_{k+1}|)`，判定步长收敛并终止。
8. 否则继续迭代，直到达到 `max_iter`。

## R05

核心数据结构：
- `NewtonStep`：记录单步迭代信息：
  - `iter_id`、`x`、`fx`、`dfx`、`step`。
- `NewtonResult`：记录运行结果：
  - `root`、`converged`、`iterations`、`reason`。
- `Case`：实验配置：
  - `name`、`func`、`deriv`、`x0`、`expected`。

这些结构使输出可追踪，便于教学演示和调试。

## R06

正确性要点：
- 若 `f` 在根附近可微且 `f'(x*) != 0`，牛顿法在足够好初值下局部二次收敛；
- 本实现每步都显式记录 `(x_k, f_k, f'_k)`，可以直接检查迭代是否符合公式；
- 通过“残差收敛 + 步长收敛”双标准，降低误判概率；
- 对有解析解的问题（如 `x^2-2=0`）可直接比对误差验证结果。

## R07

复杂度分析（记迭代步数为 `K`）：
- 时间复杂度：`O(K)` 次函数评估；
  - 若有解析导数：每步约 1 次 `f` + 1 次 `df`；
  - 若用差分导数：每步额外 2 次 `f`。
- 空间复杂度：
  - 仅求根可做到 `O(1)`；
  - 本 MVP 为保留日志使用 `O(K)`。

## R08

边界与异常处理：
- 导数过小（`|df| < min_deriv`）直接终止，返回失败原因；
- `f` 或 `df` 出现 `nan/inf` 时立即终止；
- `max_iter` 到达仍未满足终止准则时返回未收敛；
- 对多重根（如 `(x-1)^2`）会出现退化为线性收敛，脚本通过日志暴露该现象。

## R09

MVP 取舍：
- 只实现“一元方程”而不扩展到多维 Jacobian 系统；
- 使用 `numpy` + 标准库，不引入大型框架；
- 不调用 `scipy.optimize.newton` 作为主求解，保持算法透明；
- 增加“有限差分导数后备”以覆盖没有解析导数的函数。

## R10

`demo.py` 主要函数职责：
- `finite_diff_derivative`：中心差分导数近似；
- `safe_call`：统一函数值有限性检查；
- `newton_solve`：牛顿主循环与终止逻辑；
- `print_trace`：打印前若干步迭代轨迹；
- `run_case`：执行单案例并汇总误差；
- `main`：组织配置并批量运行全部测试。

## R11

运行方式：


脚本无交互输入，运行后会输出每个案例的收敛状态、误差和迭代轨迹。

## R12

输出字段解读：
- `converged`：是否满足终止条件；
- `reason`：停止原因（残差收敛、步长收敛、导数过小、达到迭代上限等）；
- `root`：最终近似根；
- `expected`：参考真值（由解析值或高精度常数给出）；
- `abs_error`：`|root-expected|`；
- `iterations`：已执行的迭代步数；
- `trace`：每步 `(x, f(x), f'(x), step)`。

## R13

最小测试集（已在 `demo.py` 中覆盖）：
- `x^2 - 2 = 0`（验证经典平方根场景）；
- `cos(x) - x = 0`（非多项式，验证一般光滑函数）；
- `x^3 - 7 = 0`（立方根场景）；
- `(x-1)^2 = 0`（多重根，观察收敛降阶）；
- `exp(x) - 3 = 0`（不提供解析导数，走差分导数路径）。

## R14

可调参数：
- `tol`：收敛阈值，默认 `1e-12`；
- `max_iter`：最大迭代次数，默认 `40`；
- `damping`：阻尼系数，默认 `1.0`；
- `diff_eps`：差分步长比例，默认 `1e-6`；
- `min_deriv`：导数最小安全阈值，默认 `1e-14`。

实践建议：
- 初值远离根时可适当降低 `damping` 增强稳定性；
- 若函数量级大、差分噪声明显，可调大 `diff_eps`；
- 先看 `trace` 再调参，避免盲目增大 `max_iter`。

## R15

方法对比：
- 相比二分法：
  - 牛顿法通常更快（局部二次收敛）；
  - 但不具备二分法那种区间保守收敛保障。
- 相比割线法：
  - 牛顿法依赖导数信息，单步质量更高；
  - 割线法不需要导数，实现更轻。
- 相比库函数黑盒求根：
  - 本实现更可解释，便于教学和故障定位；
  - 通用性和容错策略不如成熟数值库丰富。

## R18

源码级算法流（对应 `demo.py`，8 步）：
1. `main` 定义统一配置（`tol/max_iter/damping`）和测试用例列表。  
2. 每个案例进入 `run_case`，调用 `newton_solve` 执行求根。  
3. `newton_solve` 先在当前 `x` 上计算 `f(x)`，并检查有限性与残差阈值。  
4. 若提供 `deriv`，直接求 `f'(x)`；否则走 `finite_diff_derivative` 做中心差分。  
5. 对导数做安全检查（非有限或过小则终止），避免除零与不稳定更新。  
6. 计算牛顿步 `step = -damping * f/df` 并更新 `x <- x + step`。  
7. 记录 `NewtonStep` 到日志，按步长阈值判断是否可终止。  
8. 返回 `NewtonResult`，`run_case` 再计算误差并打印收敛摘要与迭代轨迹。  

补充：本实现没有把求根交给 `scipy.optimize.newton` 等黑盒 API，核心迭代与终止判断均在本地源码中逐行展开。
