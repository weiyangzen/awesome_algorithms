# 蒙哥马利模乘 (Montgomery Multiplication)

- UID: `MATH-0016`
- 学科: `数学`
- 分类: `数论`

- 目标目录: `Algorithms/数学-数论-0016-蒙哥马利模乘_(Montgomery_Multiplication)`

## R01

蒙哥马利模乘用于高效计算 `a * b mod n`，核心思想是把数映射到 Montgomery 域，在该域内把“模除法”替换为按位移位与加法。

- 输入: 奇模数 `n`，操作数 `a, b`，以及基数 `R = 2^k`（满足 `R > n` 且 `gcd(R, n) = 1`）。
- 输出: `a * b mod n`。
- 关键机制: 定义 `REDC(T) = T * R^{-1} mod n`。对 `a_bar = aR mod n`、`b_bar = bR mod n`，有 `REDC(a_bar * b_bar) = abR mod n`，再一次 `REDC` 即回到标准域。

## R02

该方法由 Peter L. Montgomery 在 1985 年提出，代表论文为 *Modular Multiplication Without Trial Division*（发表于 1985 年的 *Mathematics of Computation*）。

## R03

历史背景是大整数密码学计算（尤其 RSA）需要大量模乘。传统 `(a*b) mod n` 在大整数上包含昂贵的试除或整除步骤，硬件和早期软件中都成为瓶颈。蒙哥马利方法通过引入 `R=2^k`，把“除以 `n`”转为“截断低位 + 右移”，显著降低了每次模乘的常数成本。

## R04

按 `k` 位整数计，设乘法复杂度为 `M(k)`。

- 单次 Montgomery 约化/模乘: `O(M(k))`。
- 若用学校乘法，`M(k)=O(k^2)`，则单次模乘为 `O(k^2)`。
- 模幂 `a^e mod n`（二进制幂）总复杂度约为 `O(log e * M(k))`。

## R05

空间开销主要来自多精度中间变量。

- 额外工作空间: `O(k)` 字（或 `O(k)` bit 级块的常数倍）。
- 预计算存储: `n' = -n^{-1} mod R`、`R^2 mod n`，均为 `O(k)`。

## R06

可复现实例（小数值）：

- 取 `n=17`，`R=32`（`2^5`，且 `R>n`）。
- `n^{-1} mod R = 17`，故 `n'=-17 mod 32 = 15`。
- `a=13`，`b=11`。
- 映射到 Montgomery 域: `a_bar = 13*32 mod 17 = 8`，`b_bar = 11*32 mod 17 = 12`。
- 域内乘法: `t=8*12=96`。
- 约化: `m=(t*n') mod R=(96*15) mod 32 = 0`，`u=(t+m*n)/R = 96/32 = 3`。
- 反映射: 再做一次 `REDC(3)` 得到 `7`。
- 校验: `13*11 mod 17 = 143 mod 17 = 7`，一致。

## R07

该算法是现代公钥密码实现的基础组件之一。

- 理论意义: 提供了“避免模除法”的标准化路径。
- 工程价值: 在重复模乘场景（模幂、椭圆曲线标量乘）中极其关键。
- 产业影响: OpenSSL、GMP 类库及大量硬件密码核均采用其变体。

## R08

直接依赖的先导知识与算法：

- 模运算与同余环 `Z_n`。
- 扩展欧几里得算法（用于求 `n^{-1} mod R`）。
- 二进制位运算（当 `R=2^k` 时，`mod R` 与 `/R` 可由掩码和右移实现）。
- 多精度整数乘法。
- 在应用层常与二进制幂算法配合形成快速模幂流程。

## R09

适用前提与边界条件：

- 典型实现要求 `n` 为奇数，使 `gcd(n, 2^k)=1`。
- 需选 `R=2^k` 且 `R>n`。
- 输入 `a,b` 可先做 `mod n` 归一化。
- 当模数固定、需要大量重复模乘时收益最大；单次计算不一定划算。

## R10

正确性依据来自同余不变量：

- 设 `n' = -n^{-1} mod R`。
- 对任意 `T`，令 `m = (T*n') mod R`，则 `T + m*n ≡ 0 (mod R)`，因此可被 `R` 整除。
- 定义 `u = (T + m*n)/R`，可证明 `u ≡ T*R^{-1} (mod n)`。
- 再结合 `0 <= u < 2n`，最多一次条件减法即可得到 `[0,n)` 内唯一代表元。

## R11

数值稳定性方面，本算法在整数环上工作，不涉及浮点舍入。

- 舍入误差: `N/A`（整数精确运算）。
- 主要风险: 固定字长实现中的溢出、符号扩展错误、条件分支引起侧信道。
- 在 Python 大整数环境下，数值误差不传播，但常数时间属性并不自动成立。

## R12

真实计算成本与实现细节密切相关：

- 预处理一次性成本: 求逆与 `R^2 mod n` 预计算。
- 热路径成本: 每次模乘约为“1 次大整数乘法 + 若干低位掩码/加法/移位 + 最多 1 次减法”。
- 与直接 `% n` 相比，是否更快取决于语言与实现层级。
- 在 Python 中，内建 `%` 在 C 层高度优化，纯 Python Montgomery 常用于教学与结构验证；在 C/汇编/硬件实现里优势更明显。

## R13

近似比/误差界/概率保证：

- 本算法为确定性精确算法，不是近似算法。
- 误差界: 恒为 0（在无溢出、实现正确前提下）。
- 成功概率: 1（满足前提条件时）。

## R14

鲁棒性与失效模式：

- 模数为偶数时 `n^{-1} mod 2^k` 不存在，算法参数构造失败。
- `R <= n` 或 `gcd(R,n) != 1` 会破坏正确性。
- 少做/多做域转换会得到系统性错误结果。
- 在低级语言若中间乘法溢出，会出现静默错误。
- 非常关注安全时，分支式 `if u>=n` 需改为常数时间条件减法。

## R15

工程实现要点：

- 固定模数场景先缓存 `k`、`R`、`mask`、`n'`、`R^2 mod n`。
- `REDC` 实现中优先使用按位与和右移替代 `% R` 与 `/ R`。
- 对外接口建议区分标准域和 Montgomery 域，避免误用。
- 批量运算（如模幂）尽量在域内完成，仅在输入和最终输出时转换。
- 安全实现需关注常数时间与内存访问模式。

## R16

前驱/后继与应用谱系：

- 前驱思路: 传统长除法取模、Barrett reduction（同属“减少除法”的路线）。
- 后继扩展: 多种词级实现（CIOS/FIOS/FIPS 等）、Montgomery ladder（用于抗侧信道幂/标量运算流程）。
- 典型应用: RSA 模幂、Diffie-Hellman、ECC 有限域运算、多精度算术库基元。
- 与 R08 一致性: 都依赖模运算、求逆、位运算和幂算法组合。

## R17

Python MVP 实现方案（`demo.py`）：

- 使用工具栈: Python 3 标准库（`dataclasses`、`random`、`time`）。
- 组件:
  - `MontgomeryContext` 保存 `n, R, n', R^2, mask`。
  - `redc`、`to_mont`、`from_mont`、`mont_mul`、`mul_mod`、`pow_mod`。
  - `self_test` 对比 Python 内建 `%` 与 `pow(..., mod)`。
  - `benchmark` 给出直接模乘、完整 Montgomery、域内 Montgomery 三种路径的耗时对比。
- 运行方式: `python3 demo.py`，无需交互输入。

## R18

本条目未把第三方包当黑箱，`demo.py` 直接实现了核心流程。按源码逻辑可拆为 8 步：

1. 参数选择: 设定奇模数 `n`，取 `k = bit_length(n)`，`R = 2^k`，并令 `R > n`。
2. 求逆预处理: 用扩展欧几里得求 `n^{-1} mod R`，得到 `n' = -n^{-1} mod R`。
3. 预计算转换常量: 计算 `R^2 mod n`，用于把标准域元素高效映射到 Montgomery 域。
4. 实现 `REDC(T)`: 计算 `m = (T*n') mod R`，再算 `u = (T + m*n) / R`。
5. 归一化结果: 若 `u >= n`，执行一次减法 `u -= n`，得到 `[0,n)` 的结果。
6. 域内乘法: 对 `a_bar,b_bar` 执行 `mont_mul(a_bar,b_bar)=REDC(a_bar*b_bar)`，仍留在 Montgomery 域。
7. 域转换: `to_mont(x)=REDC(x*R^2 mod n)`，`from_mont(x_bar)=REDC(x_bar)`。
8. 组合成应用: `mul_mod` 和 `pow_mod` 在域内循环调用 `mont_mul`，最后再 `from_mont` 输出标准模结果。
