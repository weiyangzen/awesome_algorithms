## R01

Z 函数（Z-algorithm）与扩展 KMP（ExKMP / Extended KMP）是一组紧密相关的线性字符串算法。

- `Z` 数组解决的问题：对一个串 `s`，计算每个位置 `i` 处后缀 `s[i:]` 与整个串前缀 `s` 的最长公共前缀长度。
- `extend` 数组（扩展 KMP）解决的问题：对文本 `text` 与模式 `pattern`，计算每个位置 `i` 处 `text[i:]` 与 `pattern` 的最长公共前缀长度。

它们的核心价值是：把大量重复比较压缩为 `O(n)`，避免朴素匹配中的重复回退。

## R02

形式化定义：

给定字符串 `s`，长度为 `n`，定义 `Z[i]` 为最大整数 `k`，满足：

- `0 <= k <= n - i`
- `s[0:k] == s[i:i+k]`

通常约定 `Z[0] = n`（本目录采用该约定，便于调试与展示）。

给定文本 `T`（长度 `N`）和模式 `P`（长度 `M`），定义扩展数组：

- `extend[i] = LCP(T[i:], P)`，即 `T` 从 `i` 开始后缀与 `P` 的最长公共前缀长度。

当 `extend[i] == M` 时，表示模式串在 `T` 的位置 `i` 发生一次完整匹配。

## R03

两者的共同思想是维护一个“当前最右匹配区间”（Z-box）：

- 记区间为 `[l, r]`，表示当前已知 `s[l:r+1] == s[0:r-l+1]`（或 ExKMP 中 `text[l:r+1] == pattern[0:r-l+1]`）。
- 对于新区间内的位置 `i`，可以直接复用历史信息得到一个下界，减少比较次数。
- 只有在“碰到区间右边界”时才继续显式比较扩展。

因此每个字符最多被常数次触达，整体达到线性复杂度。

## R04

`Z` 函数的线性流程（单串场景）：

1. 初始化 `z[0]=n`，`l=r=0`。
2. 从左到右遍历 `i=1..n-1`。
3. 若 `i<=r`，先令 `z[i]=min(r-i+1, z[i-l])`，表示可复用区间内已知信息。
4. 再从该初值继续比较：`s[z[i]]` 与 `s[i+z[i]]`，直到失配或越界。
5. 若新匹配到的右端超过 `r`，更新 `[l, r]`。

该流程只用整数下标与字符比较，不依赖额外复杂结构。

## R05

扩展 KMP（ExKMP）可以看作“把 Z 思路从同串推广到双串”：

1. 先对 `pattern` 计算 `next = Z(pattern)`。
2. 计算 `extend[0] = LCP(text, pattern)`。
3. 维护 `text` 上的最右匹配区间 `[l, r]`，表示 `text[l:r+1] == pattern[0:r-l+1]`。
4. 对每个 `i>0`：
- 若 `i<=r`，可用 `k=i-l` 与 `next[k]` 给出复用初值：`extend[i] = min(r-i+1, next[k])`。
- 然后从初值继续显式比较扩展。
- 如右端变大则更新 `[l, r]`。

最终一次扫描即可得到所有 `extend[i]`。

## R06

复杂度分析：

- `Z(pattern)`：`O(M)`。
- `extend(text, pattern)`：`O(N)`。
- 合计：`O(N+M)`。

空间复杂度：

- `next` 长度 `M`，`extend` 长度 `N`。
- 附加变量常数级。
- 总空间 `O(N+M)`（若只输出匹配位置，可视需求减少持久化数组）。

## R07

小例子：

- `pattern = "aba"`
- `text = "abacabaaba"`

`extend` 结果会在位置 `0、4、7` 取值 `3`，对应三次完整命中：

- `text[0:3] = "aba"`
- `text[4:7] = "aba"`
- `text[7:10] = "aba"`

这展示了扩展 KMP 在重叠与重复结构场景下的稳定线性表现。

## R08

正确性关键不变量：

1. `Z/extend` 在区间复用时只给“保守下界”，不会夸大匹配长度。
2. 每次显式扩展都从当前下界继续，直到首个失配，因此得到真实最大 LCP。
3. 维护的 `[l, r]` 始终表示一个已验证匹配区间。
4. 当 `i<=r` 时，复用来源索引 `i-l` 与已有信息语义一致。

这些不变量共同保证：每个位置输出值既不漏也不错。

## R09

边界条件与约定：

- 空串 `s=""`：`Z(s)=[]`。
- 空文本 `text=""` 且非空模式：`extend=[]`。
- 空模式 `pattern=""`：本 MVP 明确禁止并抛异常（避免“每个位置都命中”的歧义语义）。
- 单字符串：`Z=[1]`（按 `Z[0]=n` 约定）。

这些规则在 `demo.py` 中都有显式处理。

## R10

应用场景：

- 单模式精确匹配（可用 `extend[i]==len(pattern)` 判定）。
- 字符串周期性分析（利用 `Z` 快速验证前后缀结构）。
- 与哈希联合做快速过滤后精确比对。
- 竞赛与工程中的批量 LCP 查询前处理。

相比 KMP，ExKMP 直接给每个位置的完整 LCP 长度，在某些统计任务上更直接。

## R11

与 KMP 的关系：

- KMP 的 `pi`/前缀函数关注“模式内部失配跳转”。
- Z 关注“每个后缀与整体前缀的 LCP”。
- ExKMP 用 `Z(pattern)` 作为复用底座，把“线性失配处理”扩展到 `text` 的每个起点。

三者本质都在避免重复比较，只是状态表示不同。

## R12

常数项与实现细节：

- 统一使用 Python 字符串随机访问与整数下标，常数较低。
- 尽量减少切片创建，比较时用下标推进。
- 使用局部变量缓存长度与数组引用，可减轻解释器开销。
- 随机回归时固定种子，保证可复现。

本 MVP 不依赖第三方库，便于最小化部署与复现。

## R14

常见错误模式与防护：

- 错误：`i<=r` 时复用索引写成 `z[i]` 而非 `z[i-l]`。
- 防护：代码中明确写出 `k=i-l` 再访问。

- 错误：`while` 扩展边界条件漏掉 `pattern` 长度检查，导致越界。
- 防护：ExKMP 扩展使用双边界 `i+extend[i] < n` 且 `extend[i] < m`。

- 错误：`Z[0]` 约定混乱导致测试比较不一致。
- 防护：README 与代码统一采用 `Z[0]=n`。

- 错误：把空模式当普通输入，匹配语义不明确。
- 防护：构造时直接 `ValueError`。

## R18

源码级算法流程拆解（`demo.py`，非黑箱，8 步）：

1. 在 `z_function` 中初始化 `z[0]=n` 与当前最右匹配区间 `[l, r]`。
2. 遍历每个位置 `i`，若 `i` 落在 `[l, r]` 内，先用 `min(r-i+1, z[i-l])` 复用历史信息。
3. 从该初值继续逐字符比较，扩展 `z[i]` 到真实最大值，并按需更新 `[l, r]`。
4. 在 `exkmp_extend` 中先对 `pattern` 调用 `z_function` 得到 `next` 数组。
5. 计算 `extend[0]`（`text` 与 `pattern` 的整体 LCP），并建立文本侧匹配区间 `[l, r]`。
6. 对每个 `i>0`，若 `i<=r`，令 `k=i-l`，用 `min(r-i+1, next[k])` 得到 `extend[i]` 的复用初值。
7. 从初值继续显式比较 `text[i+extend[i]]` 与 `pattern[extend[i]]`，必要时更新 `[l, r]`。
8. 最后用 `extend[i] == len(pattern)` 提取匹配起点，并用朴素实现做随机回归对照，验证正确性。
