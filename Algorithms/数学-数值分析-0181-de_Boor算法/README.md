## R01

de Boor 算法用于在给定 B 样条阶次、节点向量与控制点后，稳定地计算曲线在参数 `u` 处的函数值。  
它本质上是 de Casteljau 在 B 样条上的推广：

- 输入：次数 `p`、节点向量 `U`、控制点 `P_i`、参数 `u`
- 输出：`C(u)`，即样条曲线上对应点

相比直接计算全局基函数后求和，de Boor 只在局部 `p+1` 个控制点上递推，数值稳定且计算局部化。

## R02

为什么会有 de Boor：

- B 样条定义是 `C(u)=sum(N_{i,p}(u) * P_i)`，但直接算全部 `N_{i,p}` 在工程上不够高效。
- de Boor 利用“节点区间定位 + 局部递推”，把问题缩小到一个 knot span 内。
- 对 CAD/CAM、字体轮廓、动画曲线等应用，这是工业标准的曲线求值核心步骤。

## R03

核心思想（局部递推）：

1. 先找到 `u` 所在区间 `U_k <= u < U_{k+1}`。
2. 只取局部控制点 `P_{k-p}, ..., P_k`。
3. 进行 `p` 轮线性插值递推，最终得到 `C(u)`。

递推系数为：
`alpha = (u - U_i) / (U_{i+p-r+1} - U_i)`

每轮把控制多边形进一步“收缩”，最后收敛到曲线点。

## R04

复杂度（单个参数点）：

- 时间复杂度：`O(p^2)`（三角递推，约 `(p+1)*p/2` 次向量更新）
- 空间复杂度：`O(p)`（仅需一个长度 `p+1` 的工作数组）

若要评估 `m` 个参数点，总时间约 `O(m * p^2)`。在固定小次数（常见 `p=2/3/4`）下非常高效。

## R05

与“基函数全量求和”的对比：

- 全量法通常显式计算全部 `N_{i,p}(u)`，单点评估成本偏高。
- de Boor 只处理命中区间附近的 `p+1` 个控制点，天然稀疏、局部。
- 当控制点数量很大但次数较小（实践中很常见）时，de Boor 更有优势。

## R06

前置知识：

- B 样条节点向量（knot vector）与次数 `p`
- 基函数递推（Cox-de Boor）
- 参数域与节点区间 `span` 的关系
- 线性插值与浮点误差基本认知

## R07

输入有效性要求：

- 控制点数 `n_ctrl` 需要满足 `n_ctrl >= p + 1`
- 节点向量长度必须为 `n_ctrl + p + 1`
- 节点向量需非降序（允许重复节点）
- 参数 `u` 通常取在 `[U_p, U_{n_ctrl}]`（开区间边界需特殊处理 `u=U_{n_ctrl}`）

## R08

正确性直觉：

- B 样条基函数具备局部支撑；对固定 `u`，只有少数基函数非零。
- de Boor 递推只在这些非零基函数对应的控制点上进行等价变换。
- 每轮插值等价于“把一次基函数递推吸收到控制点组合里”。
- 因而最终结果与 `sum(N_{i,p}(u)P_i)` 一致，但计算路径更局部、更稳。

## R09

数值稳定性要点：

- 分母 `U_{i+p-r+1} - U_i` 可能因重节点变小甚至为 0，需要安全分支。
- `u` 落在边界（尤其 `u=U_{n_ctrl}`）时，span 查找要做右端夹取。
- 浮点误差下建议使用 `eps` 判断并做参数夹取，避免越界索引。

## R10

常见应用：

- CAD/NURBS 求值中的基础步骤（NURBS 分子分母都可用 de Boor 思路评估）
- 机器人/运动规划中的平滑轨迹生成
- 字形和矢量图形路径插值
- 数据可视化中的光滑曲线构造

## R11

`demo.py` 的 MVP 范围：

- 手写实现 de Boor（非黑盒）
- 同时手写 Cox-de Boor 基函数求值作为交叉验证
- 使用二维控制点构造三次开区间均匀 B 样条

不包含：NURBS 权重、曲面张量积、弧长参数化。

## R12

本实现的数据结构：

- `control_points: np.ndarray`，形状 `(n_ctrl, dim)`
- `knots: np.ndarray`，形状 `(n_ctrl + p + 1,)`
- `u_values: np.ndarray`，待评估参数集合
- 返回值保持向量形式，可统一支持 1D/2D/3D 控制点

这种数组化表示足够透明，也方便后续扩展到批量评估。

## R13

实验设计与验收口径：

1. 对一组三次 B 样条控制点，在 `[0,1]` 上采样多个 `u`。
2. 分别用 de Boor 和基函数求和计算点值。
3. 统计 `max ||C_deboor(u)-C_basis(u)||_2` 作为一致性误差。
4. 检查端点：`u=0` 接近首控制点、`u=1` 接近末控制点（开区间均匀节点）。

若最大误差在机器精度量级（如 `1e-12` 到 `1e-14`），说明实现正确。

## R14

失效模式与缓解：

- 失效 A：节点向量长度不匹配。
  - 缓解：初始化阶段严格检查 `len(U)==n_ctrl+p+1`。
- 失效 B：参数越界导致 span 查找失败。
  - 缓解：将 `u` 夹到合法参数域，右端点单独处理。
- 失效 C：重复节点导致 0 分母。
  - 缓解：当分母接近 0 时将该轮 `alpha` 置 0，避免 NaN。

## R15

工程建议：

- 高频场景可缓存 `span` 或分块处理 `u`，减少重复搜索。
- 对固定节点向量可预计算部分结构，提高吞吐。
- 若后续扩展到 NURBS，可先在齐次坐标中做 de Boor，再做透视除法。
- 测试用例应覆盖：边界参数、重复节点、高低阶次、小样本与大样本。

## R16

相关算法关系：

- 前驱：Cox-de Boor 基函数递推（定义层）

## R18

`demo.py` 中 de Boor 的源码级流程（8 步）：

1. `build_open_uniform_knot_vector` 根据控制点数与次数构造开区间均匀节点向量。  
2. `find_knot_span` 对每个参数 `u` 二分定位区间 `U_k <= u < U_{k+1}`，并处理右端点夹取。  
3. `de_boor_point` 读取局部 `p+1` 个控制点到工作数组 `d[0..p]`。  
4. 在 `r=1..p` 的外层循环中逐层递推，内层 `j=p..r` 反向更新，确保不覆盖尚未使用的旧值。  
5. 每次更新用 `alpha=(u-U_i)/(U_{i+p-r+1}-U_i)`，分母近 0 时走保护分支。  
6. 第 `p` 轮结束后 `d[p]` 即曲线点 `C(u)`；`evaluate_curve_de_boor` 对全部 `u` 重复此过程。  
7. `basis_functions_at` 与 `evaluate_curve_by_basis` 独立实现 Cox-de Boor 基函数求和值，作为交叉真值。  
8. `run_demo` 对两种结果做范数误差统计并打印样例点，形成可复核、非黑盒的最小闭环。
