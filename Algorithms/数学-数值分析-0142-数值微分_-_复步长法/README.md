# 数值微分 - 复步长法

## R01

复步长法（Complex-Step Differentiation）用于近似实函数的一阶导数，核心公式是：

`f'(x) ≈ Im(f(x + i h)) / h`

其中 `h` 是很小的实数步长，`Im(·)` 取虚部。  
和前向差分 `[(f(x+h)-f(x))/h]` 最大区别是它不做“两个近似相等实数相减”，因此显著减轻消去误差（catastrophic cancellation）。

## R02

复步长思想来自把实函数延拓到复平面后再做泰勒展开。  
在现代数值优化与计算科学中，它在 20 世纪末到 21 世纪初被广泛工程化，用于“低成本高精度”的导数近似，尤其适合做梯度校验与敏感性分析。

## R03

经典数值微分长期受一个矛盾约束：

1. 步长 `h` 取大了，截断误差大。
2. 步长 `h` 取小了，差分相减会放大舍入误差。

复步长法通过“取虚部而非相减”绕开第二点，因此可以把 `h` 设得非常小（如 `1e-20` 量级），通常仍保持稳定。

## R04

时间复杂度取决于函数评估成本 `C_f`：

1. 标量一阶导: `O(C_f)`，只需一次复数函数评估。
2. `n` 维梯度: `O(n * C_f)`，每个坐标轴做一次复步长扰动。

与中心差分梯度 `O(2n * C_f)` 相比，复步长通常少一半函数调用。

## R05

空间复杂度通常为：

1. 标量导数: `O(1)` 附加空间。
2. `n` 维梯度: `O(n)`（保存梯度与一个复向量工作副本）。

## R06

示例函数：

`f(x) = exp(x) * sin(x) + x^3`

解析导数：

`f'(x) = exp(x) * (sin(x) + cos(x)) + 3x^2`

在 `x=1` 处，复步长估计值与解析值通常可达到机器精度附近；而把前向差分步长降到极小值时，误差反而会明显上升。

## R07

算法意义：

1. 在不引入自动微分框架的前提下，提供接近解析导数精度的数值导数。
2. 是优化器、反问题求解、科学计算中“梯度验算”的实用基线方法。
3. 对已有函数代码侵入小，通常只要求函数支持复数输入。

## R08

前置知识与依赖条件：

1. 复数运算与泰勒展开。
2. 函数在目标点附近可解析延拓（至少在局部复邻域行为良好）。
3. 实现语言/库支持复数标量或复数数组（本目录用 `numpy`）。

## R09

适用条件与边界：

1. 适用: 平滑、可复延拓的函数，如多项式、指数、三角函数及其组合。
2. 谨慎: 含分段/不可解析操作的函数，如 `abs`、`max`、`ReLU`、显式比较分支。
3. 不适合: 目标函数内部强制转实数或丢弃虚部时，复步长信号会被破坏。

## R10

正确性直觉来自复泰勒展开：

`f(x + i h) = f(x) + i h f'(x) - h^2/2 f''(x) - i h^3/6 f'''(x) + ...`

取虚部后：

`Im(f(x + i h)) / h = f'(x) - h^2/6 f'''(x) + O(h^4)`

因此截断误差是 `O(h^2)`，且没有差分相减导致的严重消去误差。

## R11

数值稳定性要点：

1. 前向差分需要做 `f(x+h)-f(x)`，当 `h` 很小会损失有效数字。
2. 复步长法直接提取虚部，通常能在极小 `h` 下维持高精度。
3. 实际误差主要来自函数本身的复数实现质量与浮点运算舍入，而非差分相减。

## R12

真实运行成本由以下因素主导：

1. 一次函数调用是否昂贵（模型推理、仿真、积分器等）。
2. 多变量梯度维度 `n`。
3. 函数内部是否向量化与是否产生大量临时复数组。

在“函数调用昂贵”场景中，复步长相比中心差分的调用次数优势通常非常明显。

## R13

保证类型说明：

1. 该方法是确定性数值算法，不涉及随机化成功概率。
2. 对满足解析性条件的函数，导数近似通常具有高精度与稳定性。
3. 若函数不解析或实现中丢失虚部，则该保证不成立。

## R14

常见失效模式：

1. `f` 内部使用 `abs`、条件分支或类型转换导致虚部被截断。
2. 使用不支持复数的第三方黑箱函数，返回值不含正确虚部响应。
3. 步长 `h` 过大时，`O(h^2)` 截断误差可见；步长过小虽通常安全，但仍受实现细节限制。

## R15

工程建议：

1. 先用简单解析函数做单元校验，确认复数链路正确。
2. 标量导数优先尝试 `h=1e-20` 到 `1e-30` 的量级。
3. 对梯度计算，逐坐标复扰动并避免在循环中频繁分配大对象。
4. 与解析导数或高精度参考值做对照，而不是只看单次输出。

## R16

方法关系：

1. 前驱: 前向差分、后向差分、中心差分。
2. 同类替代: 自动微分（AD）、符号微分。
3. 组合方式: 在工程中常用复步长做“梯度验算”，再用 AD 做大规模训练与优化。

## R17

本目录 `demo.py` 的 MVP 覆盖：

1. `complex_step_derivative`：标量一阶导复步长近似。
2. `complex_step_gradient`：多变量梯度复步长近似。
3. `forward_difference` 与 `central_difference`：作为对照基线。
4. 固定示例函数（标量 + 二元），打印估计值、真值与误差。

运行方式：

```bash
python3 demo.py
```

## R18

`demo.py` 的源码级算法流程（8 步）：

1. 定义标量目标函数 `f_scalar(x)` 与解析导数 `f_scalar_prime_exact(x)`。
2. `complex_step_derivative` 计算 `z = x + i h`，调用 `f(z)`，返回 `imag(f(z)) / h`。
3. `forward_difference` 与 `central_difference` 分别计算差分近似，作为对照。
4. 在 `run_scalar_demo` 中，对多个步长逐一计算三种方法误差并打印表格。
5. 定义二元函数 `f_vector(x)` 与解析梯度 `grad_vector_exact(x)`。
6. `complex_step_gradient` 对每个坐标构造复扰动向量 `x + i h e_k`，提取虚部得到第 `k` 个偏导。
7. 在 `run_vector_demo` 中输出复步长梯度、解析梯度及 `L2` 误差。
8. `main` 依次运行标量与向量演示，全程无交互输入，直接可复现结果。
