## R01

数据流分析（Data Flow Analysis）是在控制流图（CFG）上求解程序性质的固定点方法。
本条目给出一个最小但完整的 Python MVP：
- 抽象出通用迭代框架（前向/后向皆可）
- 实例化两类经典问题：到达定义（Reaching Definitions）与活跃变量（Live Variables）
- 打印中间与最终结果，并做断言验证

## R02

核心思想是“格 + 传递函数 + 迭代到不动点”：
- 每个基本块关联一个数据流集合（例如定义集合、变量集合）
- 在 CFG 边上传播，并通过 `meet` 运算合并多条路径信息
- 每个基本块应用本地 `transfer` 函数更新集合
- 对有限高度格进行单调迭代，最终收敛到固定点

## R03

MVP 中的示例程序（按基本块划分）如下：
- `B1`: `a = 1; b = 0`
- `B2`: `if a > b goto B3 else B4`
- `B3`: `b = a + 2`
- `B4`: `a = b + 3`
- `B5`: `c = a + b`
- `B6`: `return c`

CFG 边：
- `B1 -> B2`
- `B2 -> B3, B4`
- `B3 -> B5`
- `B4 -> B5`
- `B5 -> B6`

## R04

到达定义（前向 may 分析）定义：
- 语义：某条赋值定义 `d` 是否可能在某块入口处“到达”且未被覆盖
- 合流运算：并集（`union`）
- 方程：
  - `IN[B] = ⋃ OUT[P], P ∈ pred(B)`
  - `OUT[B] = GEN[B] ∪ (IN[B] - KILL[B])`
- 边界：入口块 `IN[entry] = ∅`

## R05

活跃变量（后向 may 分析）定义：
- 语义：变量在某点之后是否可能被使用且在使用前未重定义
- 合流运算：并集（`union`）
- 方程：
  - `OUT[B] = ⋃ IN[S], S ∈ succ(B)`
  - `IN[B] = USE[B] ∪ (OUT[B] - DEF[B])`
- 边界：出口块 `OUT[exit] = ∅`

## R06

到达定义示例中的定义域（Universe）设置为：
- `d1: a@B1`
- `d2: b@B1`
- `d3: b@B3`
- `d4: a@B4`
- `d5: c@B5`

`GEN/KILL` 规则：
- `GEN[B]`：块内最后一次赋值对应定义（本示例每块最多一条对应定义）
- `KILL[B]`：同变量在其它块上的定义集合

## R07

活跃变量示例中的 `USE/DEF`：
- `B1: USE=∅, DEF={a,b}`
- `B2: USE={a,b}, DEF=∅`
- `B3: USE={a}, DEF={b}`
- `B4: USE={b}, DEF={a}`
- `B5: USE={a,b}, DEF={c}`
- `B6: USE={c}, DEF=∅`

这些集合被注入同一个通用求解器，只是方向和 transfer 函数不同。

## R08

通用求解器实现要点：
- 统一维护 `in_map` 与 `out_map`
- 支持 `direction in {forward, backward}`
- 支持可插拔 `meet` 与 `transfer`
- 每轮按方向扫描所有块，直到全图不再变化
- 返回 `迭代轮数 + 最终 IN/OUT`

该设计与教材中的 Kildall 框架一致，便于扩展到其它数据流问题。

## R09

复杂度分析：
- 设基本块数 `N`，边数 `E`，数据流域大小 `D`
- 单轮迭代成本约 `O(E * D)`（集合并/差操作折算到集合大小）
- 总成本约 `O(I * E * D)`，`I` 为收敛轮数
- 本示例 `N=6`，规模很小，毫秒级完成

## R10

空间复杂度：
- 主要存储 `IN/OUT` 两张映射表，量级 `O(N * D)`
- 额外存储 `GEN/KILL` 或 `USE/DEF`，同样是 `O(N * D)`
- Python `set` 有额外哈希开销，但对教学级 MVP 可接受

## R11

技术栈：
- Python 3 标准库（`dataclasses`, `typing`）
- 不依赖交互输入，不依赖大型框架
- 设计重点是算法透明度与可读性，而不是性能极限

## R12

- 活跃变量分析：每个块的 `IN/OUT`

## R13

正确性保障：
- transfer 函数保持单调（集合并/差构成单调框架）
- 数据流域有限，迭代终止可保证
- 用预期结果做“白盒回归断言”，防止实现与定义偏离
- 输出按块顺序展示，便于逐项对照教材方程

## R18

`demo.py` 源码级算法流程（8 步）：
1. 在 `build_cfg()` 中定义块序、前驱、后继，形成固定 CFG。
2. 在 `build_reaching_definitions_problem()` 中构造 `GEN/KILL`，并创建前向分析问题。
3. 在 `build_live_variables_problem()` 中构造 `USE/DEF`，并创建后向分析问题。
4. `solve_data_flow()` 初始化 `IN/OUT`，按方向循环扫描所有块。
5. 每个块先做 `meet`（前向汇总前驱 `OUT`；后向汇总后继 `IN`），再做 `transfer` 更新。
6. 若某轮扫描没有任何块变化，则达到不动点并返回结果与迭代次数。
7. `main()` 打印两类分析的 `IN/OUT` 表，并与手工预期集合逐块比较。
8. 全部断言通过后输出 `PASS`，保证结果可复核、可回归。
