## R01

B+树索引是数据库中最核心的磁盘友好型有序索引结构之一。它通过“多路平衡树 + 叶子链表”的设计，把点查询、范围查询与顺序扫描统一在同一个数据结构里。

本条目的 MVP 实现聚焦三个能力：

- `insert(key, value)`：写入索引；
- `search(key)`：点查；
- `range_query(low, high)`：范围扫描。

## R02

典型应用场景：

- 关系型数据库主键/二级索引（如 InnoDB B+Tree 索引思想）；
- 文件系统目录项组织；
- 需要“有序 + 范围查询”能力的 KV 索引层；
- 时序数据按时间键做区间扫描与分页读取。

相比哈希索引，B+树在范围检索、排序输出、前缀扫描方面更有优势。

## R03

问题定义（本 MVP 版本）：

- 输入：一组键值对 `(key: int, value: Any)`，允许同键多值；
- 操作：
  - 插入键值对；
  - 查询某个键对应的全部值；
  - 查询键区间 `[low, high]` 的有序结果；
- 输出：
  - 点查返回该键的值列表或 `None`；
  - 范围查返回按键非降序排列的 `(key, value)` 列表。

实现约束：树保持平衡，所有叶子位于同一深度。

## R04

算法核心思想：

1. 内部节点只存分隔键（separator keys）和子指针，不存业务值；
2. 叶子节点存真实键和值列表，并通过 `next` 指针串成有序链表；
3. 插入时递归下降到叶子，必要时分裂节点并把分裂键向上传播；
4. 根分裂时创建新根，树高加一；
5. 范围查询从 `low` 对应叶子起步，沿叶链顺序扫描到 `high` 截止。

## R05

复杂度（设树阶通过 `max_keys` 控制，树高为 `h`）：

- 点查询：`O(h)`；
- 插入：`O(h)`，最坏包含沿途分裂；
- 范围查询：`O(h + k)`，`k` 为输出记录数；
- 空间复杂度：`O(n)`。

在工程中，B+树扇出大、`h` 较小，因此具备良好的可扩展性。

## R06

`demo.py` 的实验设计：

- 用固定随机种子生成 48 条 `(key, row_id)` 记录；
- 键空间为 `[0, 19]`，故意制造重复键，验证“一键多值”；
- 先构建 Python 参考索引（`dict[int, list[str]]`）；
- 再构建 B+树并逐条插入；
- 对点查与多个范围查做一致性断言；
- 最后检查结构不变式并打印树统计信息。

整个流程无交互输入，`uv run python demo.py` 可直接执行。

## R07

优点：

- 同时支持高效点查与范围查；
- 叶链天然支持顺序遍历；
- 树高低、查询路径短，适合大规模索引。

局限：

- 本 MVP 只实现插入、查询、范围扫描，未实现删除与重分配/合并；
- 节点容量使用“内存简化参数 `max_keys`”，未模拟页大小与磁盘 I/O；
- 未实现并发控制与崩溃恢复。

## R08

前置知识：

- 平衡多路搜索树基础；
- 二分定位（`bisect_left`/`bisect_right`）；
- 递归与不变式检查。

运行环境：

- Python 3.10+；
- `numpy`（用于生成可复现实验数据；若不可用会自动回退到标准库随机数）。

## R09

适用：

- 读写混合且需要范围查询的索引系统；
- 有序主键访问、分页、区间聚合；
- 需要可预测延迟的树索引层。

不适用：

- 只做精确匹配且无需顺序遍历（哈希可能更简洁）；
- 超高并发事务场景但未配套锁/闩与 WAL；
- 需要持久化页管理的生产存储引擎（本实现是教学版内存结构）。

## R10

正确性要点（非形式化证明）：

- 路径选择正确：内部节点用分隔键与 `bisect_right` 决定子树，确保查找不会错过目标键；
- 分裂保持有序：叶分裂后提升右叶最小键，内部分裂后提升中位键，保持搜索边界；
- 平衡性保持：每次分裂只在当前层局部修正，必要时向上递归，最终确保所有叶子同深度；
- 范围查询正确：先定位 `low` 所在叶，再沿叶链顺序读，遇到 `key > high` 即可安全停止。

## R11

边界与鲁棒性处理：

- `max_keys < 3` 直接拒绝，避免退化配置；
- `range_query(low, high)` 在 `low > high` 时抛 `ValueError`；
- 查询不存在键时返回 `None`；
- 结构校验函数检查：
  - 叶子深度一致；
  - 内部节点满足 `children = keys + 1`；
  - 叶链全局有序。

## R12

实现细节选择：

- 重复键策略：叶子中每个键对应 `list[value]`，避免在树中散布同键副本；
- 路由规则：内部节点使用 `bisect_right`，与“分隔键归右子树”策略配套；
- 叶分裂策略：中点切分，提升右叶首键；
- 根节点处理：根分裂后创建新根，统一维护树高增长逻辑。

这些选择使代码简短且行为可预测，便于教学与调试。

## R13

理论性质（教学版）：

- 搜索、插入复杂度受树高 `h` 控制，通常远小于线性扫描；
- 范围查询只需一次树下降，后续沿叶链顺序输出，避免反复回树；
- 输出顺序天然按键有序，不需要额外排序；
- 对比黑盒库调用，本实现可逐行追踪分裂与提升过程。

## R14

常见错误：

1. 内部节点插入新分隔键后忘记插入对应右孩子指针；
2. 叶分裂后未维护 `next` 链，导致范围查询断链；
3. 把 `bisect_left`/`bisect_right` 混用到错误位置，造成边界键路由错误；
4. 仅检查局部有序，忽略“所有叶子同深度”这一 B+树关键不变式。

## R15

可扩展方向：

- 增加删除操作（借位、合并、父节点回收）；
- 加入页大小模型，按磁盘页组织节点；
- 支持并发控制（闩锁耦合）与恢复机制（WAL）；

## R16

相关数据结构：

- B树：内部节点也存业务值，范围扫描通常不如 B+树直接；
- LSM-Tree：写放大与读放大权衡不同，常用于写密集场景；
- 哈希索引：点查快但不擅长范围检索；
- 跳表：实现简单，常用于内存结构与并发场景。

## R18

`demo.py` 的源码级算法流程（8 步）：

1. `main` 生成固定随机数据，并用 `build_reference` 构造字典基准结果。  
2. 初始化 `BPlusTree(max_keys=4)`，逐条执行 `insert`。  
3. `insert` 调用 `_insert_recursive` 自顶向下定位目标叶；在叶子中用 `bisect_left` 定位键位。  
4. 若键已存在，把值追加到该键的值列表；否则在叶节点有序插入新键值。  
5. 若叶子溢出，`_split_leaf` 按中点切分，更新叶链 `next`，并把“右叶首键”上传父节点。  
6. 若内部节点因接收上传键而溢出，`_split_internal` 提升中位分隔键继续向上传播；根溢出则新建根。  
7. 点查 `search` 使用 `_find_leaf + bisect_left`；范围查 `range_query` 先定位 `low` 再沿叶链顺序收集到 `high`。  
8. `assert_tree_matches_reference` 与 `validate_structure` 双重校验结果与结构，最后打印统计信息。

本 MVP 没有把第三方库当作黑盒索引实现；核心 B+树插入/分裂/查询流程均在源码中完整展开并可直接追踪。
