# 最短路径 - Bellman-Ford

- UID: `MATH-0215`
- 学科: `数学`
- 分类: `图论`
- 源序号: `215`
- 目标目录: `Algorithms/数学-图论-0215-最短路径_-_Bellman-Ford`

## R01

本条目实现 `Bellman-Ford` 的最小可运行版本（MVP），用于**有向带权图的单源最短路径**计算。目标是：
- 支持负边权；
- 输出源点到所有节点的最短距离；
- 输出前驱数组并可重建路径；
- 检测“源点可达”的负权回路，并标记受其影响的节点；
- 提供固定样例，`python3 demo.py` 直接可运行。

## R02

问题定义（MVP 范围）：
- 输入：
  - 节点数 `n`；
  - 边集 `edges = (u, v, w)`，默认有向图；
  - 源点 `source`。
- 输出：
  - `dist[v]`：从 `source` 到 `v` 的最短距离（不可达为 `+inf`）；
  - `pred[v]`：最短路上的前驱节点（无前驱为 `-1`）；
  - `has_negative_cycle`：是否存在源点可达负环；
  - `negative_cycle_affected[v]`：节点 `v` 是否受该负环影响（最短路在数学上未定义）。

## R03

数学基础与松弛方程：

1. 初始化：`dist[source] = 0`，其余点为 `+inf`。  
2. 对任意边 `(u, v, w)`，若 `dist[u] + w < dist[v]`，则可进行一次松弛：
   `dist[v] = dist[u] + w`，并记录 `pred[v] = u`。  
3. 若图中不存在源点可达负环，最多执行 `n-1` 轮全边松弛后收敛。  
4. 第 `n` 轮若仍可松弛，说明存在源点可达负环。  
5. 从第 `n` 轮可继续松弛的节点出发沿有向边传播，可得到全部受负环影响的节点集合。

## R04

算法流程（MVP）：
1. 校验输入（`n`、`source`、边端点、权重有限性）。
2. 初始化 `dist` 与 `pred`。
3. 执行最多 `n-1` 轮“遍历全部边”的松弛。
4. 若某一轮无更新则提前停止（early stop）。
5. 额外再扫描一轮边，收集仍可松弛的“种子节点”。
6. 以种子节点为起点，沿邻接表做 DFS/BFS 传播，标记负环影响区。
7. 输出距离、前驱、负环标志、影响区标志。
8. 需要展示具体路径时，用 `pred` 从目标节点回溯到源点。

## R05

核心数据结构：
- `edges: List[Tuple[int, int, float]]`：有向加权边集；
- `dist: numpy.ndarray(float)`：单源最短距离；
- `pred: numpy.ndarray(int)`：路径回溯前驱；
- `adj: List[List[int]]`：用于负环影响传播的邻接表；
- `BellmanFordResult`（`dataclass`）：统一封装算法输出。

## R06

正确性要点：
- 第 `k` 轮松弛后，`dist[v]` 不大于所有“至多 `k` 条边路径”中的最小代价；
- 任意简单路径最多含 `n-1` 条边，因此无负环时 `n-1` 轮后达到最优；
- 第 `n` 轮仍可松弛等价于存在可使路径继续变短的可达回路，即可达负环；
- 从该回路继续可达的节点其最短距离可无限减小，因此需标记为“受负环影响”。

## R07

复杂度：
- 时间复杂度：
  - 主过程 `O(n * m)`（`m` 为边数）；
  - 负环影响传播 `O(n + m)`；
  - 总计仍以 `O(n * m)` 为主。
- 空间复杂度：`O(n + m)`（距离、前驱、邻接表与标记数组）。

## R08

边界与异常处理：
- `n <= 0`：抛出 `ValueError`；
- `source` 越界：抛出 `ValueError`；
- 边端点越界：抛出 `ValueError`；
- 非有限权重（`nan/inf`）：抛出 `ValueError`；
- 路径重建时：
  - 目标点不可达返回 `None`；
  - 目标点受负环影响返回 `None`；
  - 回溯超步数保护触发时返回 `None`（防御式处理）。

## R09

MVP 取舍说明：
- 只依赖 `numpy`，不引入图算法框架黑盒；
- 保留“路径重建 + 负环影响传播”两项关键可解释能力；
- 演示两个场景：
  - 含负边但无负环；
  - 存在源点可达负环；
- 不实现 Johnson 重标定、多源批处理、并行松弛等高级优化。

## R10

`demo.py` 职责划分：
- `validate_input`：统一输入合法性检查；
- `build_adjacency`：构建邻接表用于传播标记；
- `bellman_ford`：核心最短路与负环检测；
- `reconstruct_path`：基于前驱重建一条源到目标路径；
- `format_distance_row`：将普通距离/`inf`/负环影响做可读格式化；
- `run_case_without_negative_cycle` / `run_case_with_negative_cycle`：固定案例演示；
- `main`：无交互运行入口。

## R12

输出内容解读：
- `distance summary from source`：每个节点的距离状态；
- `reachable`：源点是否可达该点；
- `negative-cycle-affected`：该点最短路是否未定义；
- `path 0->t`：若可定义，输出一条最短路径与总代价。

说明：当 `negative-cycle-affected=True` 时，打印为 `-inf(*)`，表示可通过重复负环让代价无限下降。

## R15

方法对比：
- 对比 Dijkstra：
  - Dijkstra 要求非负边权；
  - Bellman-Ford 支持负边并可检测负环。
- 对比 Floyd-Warshall：
  - Floyd-Warshall 是全源最短路，复杂度 `O(n^3)`；
  - Bellman-Ford 是单源，稀疏图常更实用。
- 对比 SPFA：
  - SPFA 常见于工程优化，但最坏复杂度仍可退化；
  - Bellman-Ford 结构更稳定、分析更直接。

## R16

典型应用场景：
- 含负成本边的金融/套利路径分析；
- 路由协议中的路径代价迭代（思想层面）；
- 需要同时输出“最短路 + 负环判定”的教学与审计任务；
- 作为 Johnson 算法中的子过程（势能预处理）。

## R18

源码级算法流（对应 `demo.py`，8 步）：
1. `run_case_*` 准备 `n`、`edges`、`source`，调用 `bellman_ford`。  
2. `validate_input` 检查节点规模、源点、边端点与权重合法性。  
3. `bellman_ford` 初始化 `dist[source]=0`、其余 `inf`，并将 `pred` 设为 `-1`。  
4. 进行最多 `n-1` 轮全边松弛：若 `dist[u] + w < dist[v]`，更新 `dist[v]` 与 `pred[v]`。  
5. 若某轮无任何更新，提前结束循环，减少不必要计算。  
6. 再扫描一轮边，收集仍可松弛的节点作为负环影响“种子”。  
7. 对种子节点沿邻接表做 DFS，得到 `negative_cycle_affected` 标记。  
8. 演示层打印距离摘要，并对未受负环影响且可达节点调用 `reconstruct_path` 输出路径。
