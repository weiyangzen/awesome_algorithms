## R01

零知识证明（Zero-Knowledge Proof, ZKP）的目标是在不泄露秘密本身的前提下，证明“我知道某个秘密”。

本条目给出一个教学型 MVP：
- 采用 `Schnorr Σ-protocol`（离散对数知识证明）；
- 显式演示三件核心事情：
  - 完备性（诚实证明者一定通过）；
  - 可靠性直觉（不知道秘密的伪造者成功率随轮数指数下降）；
  - 零知识模拟（不知秘密也可生成可验证 transcript）。

说明：本实现强调“可跑 + 可审计”，不是生产级密码系统。

## R02

问题定义（离散对数知识证明）：

- 公共参数：素数 `p`、子群阶 `q`、生成元 `g`（满足 `q | p-1`）。
- 公共输入：`y = g^x mod p`。
- 私有见证：`x`（`0 < x < q`）。
- 目标：证明者向验证者证明“我知道 `x`”，但不泄露 `x`。

本 MVP 使用挑战位 `e ∈ {0,1}` 的三步协议（承诺-挑战-响应）。

## R03

Schnorr Σ-protocol 的核心关系：

1. 证明者选随机 `r ∈ Z_q`，发送承诺 `t = g^r mod p`。  
2. 验证者发送挑战 `e ∈ {0,1}`。  
3. 证明者回复 `s = r + e*x (mod q)`。  
4. 验证者检查：

`g^s ?= t * y^e (mod p)`

因为 `y = g^x`，当证明者诚实时：
`g^s = g^(r+ex) = g^r * (g^x)^e = t * y^e`，所以必然通过。

## R04

时间复杂度（记交互轮数为 `k`）：

- 诚实单轮验证：常数次模幂/模乘，`O(1)`；
- `k` 轮会话：`O(k)`；
- 伪造成功率估计（`T` 次试验）：`O(T*k)`；
- transcript 分布比较（`N` 条样本）：`O(N)`。

`demo.py` 使用小参数（`p=23, q=11`）和固定样本数，运行开销很小。

## R05

空间复杂度：

- 单轮只需保存 `(t,e,s)` 与少量临时变量，`O(1)`；
- 分布比较阶段若保留 `N` 条 transcript，为 `O(N)`；
- 频率统计向量长度为 `p`，额外 `O(p)`。

总空间复杂度由采样规模主导，可视为 `O(N + p)`。

## R06

微型手算示例（与代码参数一致）：

- 取 `p=23, q=11, g=2, x=7`，则 `y = 2^7 mod 23 = 13`。
- 取随机 `r=4`，承诺 `t=2^4 mod 23 = 16`。
- 若挑战 `e=1`，响应 `s=(4+1*7) mod 11 = 0`。
- 验证：
  - 左侧 `g^s = 2^0 mod 23 = 1`；
  - 右侧 `t*y = 16*13 mod 23 = 1`；
  - 左右相等，通过。

这体现了协议正确性，不暴露 `x` 的具体值。

## R07

工程价值与应用语境：

- 身份认证：证明“知道私钥”而不发送私钥；
- 协议构件：很多高级 ZK 系统（如 SNARK/STARK）可看作更复杂的“证明关系”扩展；
- 安全设计思路：把“正确性”和“隐私性”拆成可验证数学条件。

该 MVP 的定位是“理解最小闭环”，不是部署级组件。

## R08

理论基础（本实现对应）：

1. 完备性（Completeness）：诚实证明者一定能通过。  
2. 特殊可靠性（Special Soundness）：若同一承诺 `t` 能回答两种挑战，可提取见证 `x`。  
3. 诚实验证者零知识（HVZK）：存在模拟器，在不知道 `x` 时也能生成与真实分布一致的可验证 transcript。

`demo.py` 分别用：
- `run_honest_session`（完备性）；
- `extract_secret_from_two_challenges`（特殊可靠性）；
- `simulated_transcript + 分布距离`（HVZK 直观检验）。

## R09

适用条件与局限：

- 条件：参数需满足子群结构约束（`q | p-1` 且 `g^q = 1 mod p`）。
- 局限：
  - 使用极小素数，纯教学参数，不具备安全强度；
  - 挑战空间仅 `{0,1}`，单轮可靠性误差为 `1/2`；
  - 未覆盖网络协议层、重放防护、会话绑定、侧信道防护。

因此该实现仅用于算法学习与单元验证。

## R10

正确性检查框架（`main` 中自动执行）：

1. 诚实证明者 `k=8` 轮全部通过。  
2. 不知道见证的伪造者：
- 单轮成功率接近 `0.5`；
- 8 轮会话成功率接近 `2^-8`。  
3. 真实 transcript 与模拟 transcript 都能独立通过验证。  
4. 同一承诺下的双挑战可提取秘密，验证提取器逻辑。

任一条件不满足都会触发 `assert`。

## R11

数值稳定性与实现细节：

- 全部在有限域 `mod p` 上做整数运算，无浮点误差；
- 模逆用 `pow(a, p-2, p)`（费马小定理，适用于素域）；
- 随机性来自 `numpy.random.default_rng`，并固定 seed 保证复现；
- 对 `e, s, t` 范围做显式约束，避免无效 transcript 混入统计。

## R12

性能视角（本地教学规模）：

- 主要成本是重复模幂与大样本循环；
- 样本规模 `4000` 量级时可快速完成；
- `numpy` 在本实现中仅用于随机采样和统计向量操作，核心密码学等式均为源码显式实现，不依赖黑盒密码库。

## R14

鲁棒性与常见失败模式：

- 若参数不满足子群关系，`SchnorrParams` 初始化直接报错；
- 若 transcript 字段越界（例如 `e=2` 或 `s>=q`），验证失败；
- 若伪造者只预计算一种挑战，应在另一挑战下失败；
- 若代码误写验证方程（左右式次序/指数错误），统计行为会偏离理论并触发断言。

## R18

源码级算法流程拆解（`main` 主路径，8 步）：

1. 初始化 `SchnorrParams(p=23,q=11,g=2)` 并设定秘密 `x`，计算公钥 `y=g^x mod p`。  
2. 用 `run_honest_session` 连续生成并验证 `k=8` 轮诚实 transcript，检验完备性。  
3. 用 `cheating_round_without_secret` 构造“先猜挑战再造承诺”的攻击者模型。  
4. `estimate_cheating_rates` 统计伪造者单轮成功率与多轮会话成功率，展示 `1/2` 与 `2^-k` 规律。  
5. 用 `honest_transcript` 批量生成真实 transcript 样本。  
6. 用 `simulated_transcript` 在未知 `x` 条件下生成模拟 transcript，并逐条调用 `verify_transcript` 验证可接受性。  
7. 通过 `t_distribution + l1_distance` 比较真实/模拟分布差异，给出 HVZK 的经验性证据。  
8. 用 `extract_secret_from_two_challenges` 从同一承诺的双挑战响应提取 `x`，并用断言收束整个证明闭环。

说明：第三方库 `numpy` 仅用于随机抽样与统计；协议核心公式（承诺、响应、验证、提取）全部在源码中显式展开，没有把算法委托给黑盒函数。
