## R01

Polya 计数定理用于计算“在对称变换下视为同一种”的着色方案数。设有限群 `G` 作用在 `n` 个位置上，颜色数为 `m`。

- Burnside 形式:
  - 不同轨道（本质不同着色）数量
    `N = (1/|G|) * sum_{g in G} m^{c(g)}`。
  - 其中 `c(g)` 是置换 `g` 的循环个数。
- Polya 循环指数形式:
  - `Z_G(a_1,...,a_n) = (1/|G|) * sum_{g in G} prod_{j=1..n} a_j^{c_j(g)}`。
  - `c_j(g)` 是长度为 `j` 的循环数。
  - 当每个位置可用 `m` 种颜色且等权时，代入 `a_j = m` 即得到 Burnside 结果。

本条目 MVP 聚焦最常见的两类：

- `C_n`（旋转群）: 计算 necklace（项链）着色。
- `D_n`（二面体对称）: 计算 bracelet（手链，允许翻转）着色。

## R02

- 理论来源:
  - Cauchy-Frobenius 引理（又称 Burnside 引理）给出轨道计数框架。
  - George Pólya 在 1930 年代系统化“循环指数 + 代换”的计数方法。
- 历史定位:
  - 把“逐个分类讨论”提升为“群作用统一计数”，极大降低人工枚举复杂度。
- 现代地位:
  - 是组合计数、化学同构体计数、图案设计和离散对称约化的基础工具。

## R03

很多计数问题存在“视觉上相同、编码上不同”的重复。

- 例如长度 `n` 的环形珠串着色，线性字符串会把同一图案按旋转重复计数 `n` 次。
- 若还允许镜像翻转，重复更严重。

Polya 定理把问题转写为:

1. 先描述对称群对位置集合的作用。
2. 再统计每个群元素固定了多少着色。
3. 最后取平均（Burnside）。

这样无需枚举“本质类”本身，只需处理置换循环结构。

## R04

设位置数为 `n`，群大小为 `|G|`。

- 单个置换的循环分解: `O(n)`。
- Burnside 主循环: `|G|` 个置换，各做一次循环计数。
- 总时间复杂度: `O(|G| * n)`。

对常见群:

- 项链 `C_n`: `|G| = n`，总计 `O(n^2)`。
- 手链 `D_n`: `|G| = 2n`（在小 `n` 顶点作用上可能有重合映射），总计仍为 `O(n^2)` 量级。

若做暴力校验（仅用于小规模验证）:

- `m^n` 个着色，每个需遍历群作用，复杂度约 `O(m^n * |G| * n)`。

## R05

- 主算法额外空间:
  - 仅循环分解状态 `visited`，为 `O(n)`。
- 若预先保存全部群元素:
  - 需要 `O(|G| * n)`。
- 本目录 `demo.py` 选择“先构群再计数”，因为:
  - 代码更直观；
  - 可复用同一群做闭式校验与暴力校验。

## R06

示例: `n=6` 个位置，`m=3` 种颜色。

1. 只考虑旋转（项链，`C_6`）:

- 六个旋转的循环数分别是 `6,1,2,3,2,1`。
- 固定着色数分别为 `3^6,3^1,3^2,3^3,3^2,3^1`。
- 总和 `= 729+3+9+27+9+3 = 780`。
- 平均后 `N = 780 / 6 = 130`。

2. 允许翻转（手链，`D_6`）:

- 旋转部分仍贡献 `780`。
- 反射有两种类型（`n` 偶数）:
  - 3 个反射具有 4 个循环，固定数 `3^4=81`。
  - 3 个反射具有 3 个循环，固定数 `3^3=27`。
- 总和 `= 780 + 3*81 + 3*27 = 1104`。
- 平均后 `N = 1104 / 12 = 92`。

即:

- 3 色长度 6 项链数 = `130`
- 3 色长度 6 手链数 = `92`

## R07

Polya 计数定理的价值主要在三点。

- 结构性:
  - 把“去重计数”统一成“群作用 + 轨道计数”。
- 可扩展性:
  - 从简单项链到图着色、化学异构体、对称图案都可迁移。
- 可实现性:
  - 只需置换循环分解即可编码实现，适合教学与工程自动化计数。

## R08

最小理论依赖。

- 有限群作用与轨道概念。
- 置换分解为不交循环。
- Burnside 引理:
  - 轨道数等于固定点数平均值。
- Polya 循环指数:
  - 用循环长度统计统一表达不同群元素贡献。

## R09

适用条件与边界。

- 适用:
  - 有限位置集合、有限对称群、颜色等权独立赋值。
- 不直接适用:
  - 连续对称群或无限状态空间。
  - 带位置权重、邻接约束、全局配额约束的着色（需扩展生成函数或约束计数方法）。
- 输入边界（本 MVP 约定）:
  - `n >= 1`，`m >= 1`。

## R10

正确性依据可拆为两层。

1. Burnside 正确性:

- 轨道计数定理保证
  `|X/G| = (1/|G|) * sum_{g in G} |Fix(g)|`。

2. 固定点计数正确性:

- 置换 `g` 的每个循环中，循环内位置必须同色才能被固定。
- 每个循环可独立选 `m` 种颜色。
- 若 `g` 有 `c(g)` 个循环，则 `|Fix(g)| = m^{c(g)}`。

两者合并即得到 Polya/Burnside 计数公式。

## R11

数值稳定性与误差传播（离散整数计数视角）。

- 浮点误差: N/A，核心计算是整数幂与整数求和。
- 主要风险:
  - 固定宽度整数语言会溢出。
  - Python 使用任意精度整数，结果精确但大数运算会变慢。
- 结论:
  - 数学精度问题很小，性能与内存才是主要工程约束。

## R12

真实成本特征。

- 小中规模 (`n <= 10^3` 且只做项链/手链计数): Python 版本通常很快。
- 大规模瓶颈:
  - 幂运算的超大整数位宽增长。
  - 大量置换循环遍历的 Python 解释器开销。
- 并行性:
  - 不同群元素的贡献可并行累计（任务级并行）。
  - 单个置换的循环分解内部数据依赖较强。

## R14

鲁棒性与常见失效模式。

- 置换语义错误:
  - 若“位置映射方向”与“着色变换”不一致，会导致轨道校验失败。
- 群元素重复:
  - 在 `n=1,2` 等退化情况下，二面体作用在顶点上的置换可能重合。
  - 本实现会去重，避免重复计入同一作用。
- 输入非法:
  - `n<1` 或 `m<1` 会直接报错。
- 暴力验证超时:
  - `m^n` 增长极快，只能用于小规模自测。

## R15

工程实现建议。

- 优先写通用“置换循环计数”函数，再复用到所有群。
- 保持动作定义统一:
  - 同一份置换既用于循环分解，也用于着色变换。
- 为 closed-form（闭式）保留交叉校验:
  - 项链与手链都可与 `gcd` 公式比对。
- 保留小规模暴力真值:
  - 是排查群定义和下标方向错误的最有效手段。

## R18

源码级算法流程拆解（`demo.py`，核心算法自行实现，未把第三方包当黑箱）。

1. 参数校验: 检查 `n` 与 `m` 为正整数。
2. 构造群作用:
   - 旋转置换 `i -> (i + k) mod n`。
   - 反射置换 `i -> (k - i) mod n`。
3. 置换去重:
   - 处理小 `n` 退化时的重复作用映射。
4. 循环分解:
   - 对每个置换做 DFS/迭代追踪，获得所有不交循环长度。
5. Burnside 累加:
   - 对每个群元素累加 `m^{cycle_count}`。
6. 平均得到轨道数:
   - 用总和除以群元素数量，得到本质不同着色数。
7. 双重校验:
   - 与闭式 `gcd` 公式比对；
   - 对小规模与暴力轨道枚举结果比对。
8. 输出样例报告:
   - 打印代表性案例（如 `n=6,m=3`）与校验通过信息，保证脚本可自动执行完成。
