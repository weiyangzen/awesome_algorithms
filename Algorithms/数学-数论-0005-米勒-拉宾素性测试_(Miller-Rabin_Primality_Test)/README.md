## R01

米勒-拉宾素性测试是一个“随机化（或在有限整数范围上可确定化）”的素性判定算法。输入是整数 `n >= 0`，输出是“合数”或“可能是素数（probable prime）”。核心思想：

1. 把 `n-1` 分解为 `2^s * d`（`d` 为奇数）。
2. 随机或按固定规则选择若干底数 `a`。
3. 检查 `a^d mod n`，以及连续平方链 `a^(2^r d) mod n`（`0 <= r < s`）是否出现 `1` 或 `n-1` 的强伪素数条件。
4. 若某个底数直接给出反例（witness），则 `n` 必为合数；若所有底数都通过，则判定“可能是素数”。

该目录中的 `demo.py` 采用两段式机制：
- `n < 2^64` 时使用已知确定性底数集，结果可视为确定性判定。
- `n >= 2^64` 时采用随机底数并给出概率保证。

## R02

提出与完善时间线：

- 1976 年：Gary L. Miller 基于广义黎曼猜想（GRH）给出多项式时间判定思路。
- 1980 年：Michael O. Rabin 将其随机化，得到不依赖 GRH 的概率型多项式时间测试，即常称的 Miller-Rabin 测试。

## R03

历史背景与动机：

- 大整数密码学（尤其 RSA）需要频繁生成大素数。
- 朴素试除在大整数上过慢；早期费马测试又会被 Carmichael 数欺骗。
- 需要一种“足够快 + 错判概率可控”的算法，米勒-拉宾正好满足这一工程需求。

## R04

时间复杂度（按位复杂度）通常写为：

- 单轮测试：`O(log n)` 次模乘。
- 若使用学校乘法，单次模乘约 `O((log n)^2)`，则单轮约 `O((log n)^3)`。
- `k` 轮总成本：`O(k * (log n)^3)`（或写作 `O(k * M(log n) * log n)`，其中 `M(t)` 是 `t` 位乘法复杂度）。

实践中常数因子由大整数乘法实现、CPU 指令与缓存行为主导。

## R05

空间复杂度：

- 额外工作内存约为 `O(log n)` 位（主要是若干中间大整数寄存）。
- 不需要维护大规模表结构；流式处理每个底数即可。

## R06

示例：`n = 221 = 13 * 17`。

1. `n-1 = 220 = 2^2 * 55`，所以 `s = 2, d = 55`。
2. 取底数 `a = 2`。
3. 计算 `x0 = 2^55 mod 221 = 128`，既不是 `1` 也不是 `220`。
4. 做一次平方（因为 `s-1 = 1`）：`x1 = 128^2 mod 221 = 30`，仍不是 `220`。
5. 因此 `a=2` 是见证者，立即判定 `221` 为合数。

## R07

算法意义：

- 理论上：展示了随机化可在素性判定中提供强有力保证。
- 工程上：是现代公钥密码中大素数筛选的标准组件之一。
- 生态上：常作为“快速初筛”，后接更强测试（例如 Lucas/Baillie-PSW 或证明型方法）。

## R08

依赖的关键前置知识与算法：

- 模运算与同余理论。
- 快速模幂（二进制幂 / repeated squaring）。
- 费马小定理与乘法群结构性质。
- 随机数采样（在概率模式下选择底数）。
- 大整数乘法与取模实现（影响常数性能）。

## R09

适用前提与边界条件：

- 输入应为整数；`n < 2` 直接非素数。
- 偶数（除 `2`）直接非素数。
- 对 `n >= 2^64` 的随机版本，输出是“可能是素数”，不是绝对证明。
- 当需要“严格证明素数”时，需改用 ECPP、AKS 或证书型方法。

## R10

正确性依据（简述）：

- 若 `n` 为奇素数，`(Z/nZ)^*` 是群，米勒-拉宾判据对任意合法底数都成立，因此不会把素数误判为合数。
- 若 `n` 为奇合数，可证明使其“通过测试”的底数集合（strong liars）占比至多 `1/4`。
- 所以每增加一轮独立测试，漏判合数概率最多再乘 `1/4`。

## R11

数值稳定性与误差传播：

- 算法基于整数运算，不存在浮点舍入误差传播问题。
- 主要“误差”来自随机化统计风险（把合数判为 probable prime）。
- 若随机源质量差或种子固定不当，实际风险可能偏离理想独立采样假设。

## R12

真实计算成本（超越渐近符号）：

- Python 大整数在不同位宽下会切换不同乘法策略，常数差异显著。
- 模幂是热点，循环体包含大量 `%` 与乘法。
- 小素数预筛可显著减少昂贵模幂调用次数。
- 在批量候选场景中，候选生成与内存分配也会影响吞吐。

## R13

概率保证：

- 对奇合数 `n`，若每轮独立均匀采样 `a in [2, n-2]`，则 `k` 轮后漏判概率上界为 `(1/4)^k`。
- 本目录实现对 `n < 2^64` 使用固定底数集 `(2, 325, 9375, 28178, 450775, 9780504, 1795265022)`，该范围内可做确定性判定，不再是概率结论。

## R14

鲁棒性与失效模式：

- 轮数过少：会提高合数漏判概率。
- 随机底数分布差：可能集中采到 liar，降低可靠性。
- 忽略小素数/偶数快速分支：性能会明显退化。
- 把“probable prime”当作“证明素数”使用：属于语义层面的错误用法。

## R15

工程实现要点：

- 先做 `n < 2`、偶数、小素数整除等快速剪枝。
- 只分解一次 `n-1 = 2^s * d`，在所有底数轮次复用。
- 使用二进制模幂减少幂运算成本。
- 为 `64` 位范围配置确定性底数，兼顾速度与可靠性。
- 对超大整数，轮数建议按安全等级配置（例如 `k=12`、`k=16` 或更高）。

## R16

前驱/后继与典型应用：

- 前驱：费马素性测试（更快但更弱，易被 Carmichael 数欺骗）。
- 并行/后继方向：Solovay-Strassen、Baillie-PSW、AKS、ECPP 等。
- 典型应用：RSA/DSA/Diffie-Hellman 参数生成中的素数候选筛选与验证。

## R17

Python MVP 方案（见 `demo.py`）：

- 依赖：仅 Python 标准库（`random`、`typing`），无交互输入。

## R18

源码级流程追踪（对应 `demo.py` 实现）可拆为 8 步：

1. **输入过滤**：`is_probable_prime` 先处理 `n<2`、`n in {2,3}`、偶数与小素数整除分支。
2. **分解阶段**：`decompose_n_minus_one(n)` 计算 `n-1 = 2^s * d`。
3. **底数策略选择**：`_candidate_bases` 根据 `n` 位宽选择固定底数集（`<2^64`）或随机底数流（`>=2^64`）。
4. **轮次预处理**：每个底数先做 `a = base % n`，跳过 `a in {0,1}` 的退化情况。
5. **首次幂检验**：`is_strong_probable_prime_to_base` 中计算 `x = a^d mod n`（调用 `mod_pow`）。
6. **提前通过条件**：若 `x==1` 或 `x==n-1`，该底数轮次通过。
7. **平方链检验**：重复最多 `s-1` 次 `x = x^2 mod n`，若出现 `n-1` 则该轮通过；否则该底数成为见证者并立即返回合数。
8. **总判定输出**：若所有底数轮次均通过，则返回 `True`（`<2^64` 时等价确定性素性；更大整数时为 probable prime）。

这个流程没有依赖黑箱第三方库；核心判据与模幂细节在当前源码里可直接追踪。
