## R01

边界填充算法（Boundary Fill）是计算机图形学中的区域填充方法：
从一个种子点出发，向邻域扩展，把“不是边界色、也不是填充色”的像素改成目标填充色，直到遇到边界为止。

它与“泛洪填充（Flood Fill）”的核心区别是：
- 泛洪填充基于“原始区域颜色”扩展；
- 边界填充基于“边界颜色”停止，边界本身定义了可填充范围。

## R02

本条目解决的具体问题：

给定二维离散画布 `canvas`（整数栅格）、边界值 `boundary_value`、填充值 `fill_value` 和种子点 `(row, col)`，实现一个可运行 MVP：
- 在不穿越边界的前提下完成区域填充；
- 支持 `4` 邻域与 `8` 邻域；
- 输出填充前后画布、填充像素数量及合法性检查结果。

## R03

选择该问题作为 MVP 的原因：
- 边界填充是图形学入门中的基础区域操作，概念清晰、可视化直观；
- 纯 `numpy + Python` 即可实现，不依赖图像 GUI 或交互输入；
- 可直接展示“邻域定义（4/8 连通）对结果的影响”。

## R04

算法规则（离散栅格版本）：

设当前像素值为 `v = canvas[r, c]`，边界值为 `B`，填充值为 `F`。
若满足 `v != B` 且 `v != F`，则该像素可被填充并向邻域继续扩展。

邻域定义：
- `4` 邻域：`(±1,0), (0,±1)`
- `8` 邻域：`4` 邻域再加 `(\u00b11,\u00b11)` 四个对角方向

停止条件：
- 越界；
- 命中边界值 `B`；
- 命中已填充值 `F`（避免重复处理）。

## R05

MVP 高层流程：

1. 校验输入画布、seed、边界值与填充值。
2. 拷贝画布作为可写结果，避免修改原输入。
3. 将 seed 入栈（或队列）作为待处理像素。
4. 循环弹出像素并判定是否可填充。
5. 若可填充则写入 `fill_value`，并将邻域像素继续入栈。
6. 栈清空后结束，返回填充结果与统计信息。

## R06

正确性要点：
- 不越界：每次访问前先做索引范围检查；
- 不越边界：命中 `boundary_value` 立刻停止该方向扩展；
- 不重复填：命中 `fill_value` 不再重复扩展；
- 可达性保证：从 seed 出发、按邻域规则可达且非边界的像素最终都会被访问并填充（迭代式 DFS/BFS 的标准性质）。

## R07

复杂度分析：

记可达区域像素数为 `K`，画布总像素数为 `H*W`。
- 时间复杂度：`O(K)`（每个被填充像素最多处理一次，邻域常数级）
- 空间复杂度：`O(K)`（显式栈在最坏情况下容纳整个可达区域）

若 seed 在无效位置或直接落在边界/已填充位置，复杂度退化为 `O(1)`。

## R08

边界与异常处理策略：
- `canvas` 必须是二维 `numpy.ndarray`；
- `connectivity` 仅允许 `4` 或 `8`；
- `seed` 必须是画布内合法坐标；
- `boundary_value == fill_value` 直接报错（否则“边界判定”与“填充判定”冲突）；
- 若 seed 位于边界上或本身已是填充值，返回“0 像素填充”的稳定结果。

## R09

MVP 取舍说明：
- 仅使用 `numpy` 与标准库，保持最小依赖；
- 不接入 OpenCV/PIL 的黑盒填充 API；
- 不实现扫描线优化、多线程、GPU 版本；
- 优先保证“可读、可运行、可验证”，而非工程极限性能。

## R10

`demo.py` 函数职责：
- `FillStats`：记录填充像素数、访问步数、邻域类型和 seed 信息。
- `neighbor_offsets`：返回 `4/8` 邻域偏移。
- `boundary_fill`：核心边界填充实现（显式栈，非递归）。
- `render_canvas`：把数值画布渲染为终端 ASCII 图（`#` 边界、`.` 背景、`*` 填充）。
- `build_demo_canvas`：构造固定测试画布（外边框+内部障碍边界）。
- `run_case`：执行单次实验并打印结果与统计。
- `main`：依次运行 4 邻域与 8 邻域示例，输出摘要。

## R12

输出字段说明：
- `seed`：填充起点；
- `connectivity`：邻域连通方式（`4` 或 `8`）；
- `filled_pixels`：实际被改写为填充值的像素数；
- `visited_steps`：弹栈处理的总步骤数；
- `boundary_intact`：边界像素是否保持不变（基本正确性检查）；
- 终端图像中：
  - `#` 表示边界；
  - `.` 表示未填充背景；
  - `*` 表示填充区域。

## R13

最小测试集（脚本内置）：
- 画布尺寸：`16 x 28`；
- 元素构成：
  - 外矩形边框（封闭边界）；
  - 内部小矩形障碍（额外边界）；
- 测试 seed：`(2, 2)`；
- 测试配置：
  - `connectivity=4`
  - `connectivity=8`

目标：验证填充不穿越任何边界，且两种连通定义均可正常运行。

## R14

关键参数与调参建议：
- `boundary_value`：边界编码，需与背景/填充值不同；
- `fill_value`：填充编码；
- `seed`：建议选在目标封闭区域内部；
- `connectivity`：
  - 用 `4` 邻域可避免某些“对角漏填/串联”；
  - 用 `8` 邻域可让斜向接触区域视作连通。

工程建议：先明确图像连通性定义，再决定使用 4 邻域或 8 邻域，避免语义偏差。

## R15

与相关算法对比：
- 对比泛洪填充（Flood Fill）：
  - Flood Fill 依赖“起点原始颜色相同”；
  - Boundary Fill 依赖“边界颜色阻挡”。
- 对比扫描线填充（Scanline Fill）：
  - 扫描线填充在多边形栅格化中更高效；
  - 边界填充实现更直接，适合教学和小型区域处理。
- 对比递归实现：
  - 递归写法短，但大区域容易触发递归深度限制；
  - 本 MVP 用显式栈，稳定性更好。

## R18

`demo.py` 源码级算法流程（8 步）：

1. `main` 调用 `build_demo_canvas` 生成固定测试画布（含外边框和内部障碍边界）。
2. `main` 依次调用 `run_case`，分别执行 `connectivity=4` 和 `connectivity=8`。
3. `run_case` 先打印输入画布，然后调用 `boundary_fill` 执行核心算法。
4. `boundary_fill` 校验输入合法性（二维数组、seed 范围、`boundary_value != fill_value`、连通方式）。
5. 初始化显式栈 `stack=[seed]`，循环弹出像素；若越界则跳过。
6. 对每个候选像素读取当前值：若是边界值或已填充值则跳过，否则写入 `fill_value` 并计数。
7. 将该像素的邻域偏移（由 `neighbor_offsets` 根据 4/8 连通返回）压入栈，继续扩展，直到栈清空。
8. `boundary_fill` 返回填充后画布与 `FillStats`；`run_case` 再做 `boundary_intact` 检查并输出摘要与 ASCII 结果图。
