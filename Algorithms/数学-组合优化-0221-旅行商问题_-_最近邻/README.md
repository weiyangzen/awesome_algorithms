# 旅行商问题 - 最近邻

- UID: `MATH-0221`
- 学科: `数学`
- 分类: `组合优化`
- 源序号: `221`
- 目标目录: `Algorithms/数学-组合优化-0221-旅行商问题_-_最近邻`

## R01

本条目实现旅行商问题（TSP）的最近邻（Nearest Neighbor, NN）启发式最小可运行版本（MVP）。
目标是：
- 给定城市坐标后快速构造一条可行回路；
- 展示最近邻的核心贪心逻辑与复杂度；
- 用一个小规模精确对照（暴力枚举）说明“快但不保最优”的性质。

## R02

问题输入与输出定义：
- 输入：`n` 个城市二维坐标 `coords in R^(n x 2)`，或由其构造的距离矩阵 `dist`；
- 输出：一个长度为 `n` 的排列 `tour`（每个城市恰好访问一次），闭环总长度 `L(tour)`。

`demo.py` 默认行为：
- 固定随机种子 `221` 生成 `20` 个城市；
- 先运行 `start=0` 的单起点最近邻；
- 再运行“所有起点”的多起点最近邻并选择最优结果。

## R03

最近邻启发式规则：
- 从起点城市出发；
- 每一步在“尚未访问城市”中选择与当前城市距离最短者；
- 直到所有城市都被访问后回到起点。

本实现 tie-break 规则：
- 若多个候选距离相同，选择城市编号更小者（由 `numpy.argmin` 在有序候选上的行为保证）。

## R04

优化目标：
- 最小化闭环长度
  `L(tour) = sum_i dist[tour[i], tour[(i+1) mod n]]`。

最近邻属于构造型启发式，而非精确算法：
- 每一步都是局部最优决策；
- 全局最优性一般不成立。

## R05

单起点最近邻算法流程：
1. 初始化 `visited[start]=True`，`tour=[start]`；
2. 令 `current=start`；
3. 在未访问集合中取 `argmin dist[current, j]` 得到 `next`；
4. 将 `next` 加入 `tour` 并标记访问；
5. 重复步骤 3-4 直到访问完全部城市；
6. 用 `tour_length` 按闭环方式计算总长度。

## R06

多起点增强（仍属于最近邻范式）：
- 分别以每个城市 `s=0..n-1` 作为起点运行一次最近邻；
- 比较 `n` 条回路长度，保留最短者。

意义：
- 缓解“单一起点敏感”问题；
- 计算代价从 `O(n^2)` 增加到 `O(n^3)`，但实现仍然简单且可解释。

## R07

复杂度分析（`n` 为城市数）：
- 距离矩阵构造：`O(n^2)` 时间，`O(n^2)` 空间；
- 单起点最近邻：`O(n^2)` 时间，`O(n)` 额外空间；
- 多起点最近邻：`O(n^3)` 时间，`O(n)` 额外空间（不含距离矩阵）。

## R08

适用场景：
- 需要秒级得到可行路线的工程预估；
- 作为 2-Opt/3-Opt、Lin-Kernighan 等局部搜索的初始解生成器；
- 作为教学用例展示“贪心决策 vs 全局最优”的差异。

## R09

优点与局限：
- 优点：实现短小、速度快、可复现、可解释。
- 局限：不保证全局最优；对起点与点集分布敏感；容易形成“早期贪心导致后期绕远”。

## R10

`demo.py` 中的模块分工：
- `build_cities`：生成固定随机城市坐标；
- `pairwise_distance_matrix`：用 `numpy` 广播构造欧氏距离矩阵；
- `nearest_neighbor_tour`：单起点最近邻构造；
- `multi_start_nearest_neighbor`：遍历所有起点并选最优；
- `tour_length` / `validate_tour`：长度计算与合法性校验；
- `brute_force_tsp`：小规模精确解对照（仅用于验证）。

## R11

正确性与一致性检查：
- `validate_tour` 保证结果是 `0..n-1` 的排列；
- 断言 `multi_start_cost <= start0_cost`，验证多起点选择逻辑；
- 小规模 `n=9` 上用暴力枚举得到精确最优，报告最近邻与最优解差距。

## R12

MVP 技术栈与可复现性：
- 语言：`Python 3`；
- 依赖：`numpy`（最小依赖，未调用黑盒 TSP 求解库）；
- 无交互输入、无网络依赖；
- 固定随机种子，结果可重复。

## R13

与精确方法的关系：
- 最近邻是启发式近似方法，目标是快；
- `demo.py` 用 `brute_force_tsp` 在小规模上给出精确最优值；
- 输出中的 `Heuristic gap (%)` 直观反映启发式误差。

这能避免把最近邻误认为“总是最优”的黑箱。

## R14

边界与异常处理：
- `coords` 维度不是 `(n,2)` 时抛出 `ValueError`；
- `dist` 非方阵或 `start` 越界时抛出 `ValueError`；
- 若最近邻扩展遇到不可达边（非有限值），抛出异常；
- `brute_force_tsp` 仅允许 `n<=10`，防止阶乘爆炸。

## R18

`demo.py` 源码级算法流程（8 步）：
1. `build_cities` 生成固定随机二维城市坐标，`pairwise_distance_matrix` 计算 `n x n` 欧氏距离矩阵。  
2. `nearest_neighbor_tour(dist, start=0)` 初始化访问标记，从起点开始迭代选择最近未访问城市。  
3. 在每一步中，先取未访问候选 `candidates`，再用 `argmin(dist[current, candidates])` 得到下一个城市。  
4. 重复扩展直到得到长度为 `n` 的排列 `tour`，再通过 `tour_length` 按闭环公式计算总路长。  
5. `multi_start_nearest_neighbor` 对所有起点重复步骤 2-4，比较各自长度并保留最短回路。  
6. `validate_tour` 对单起点与多起点结果做排列合法性检查，并在 `main` 中断言多起点结果不劣。  
7. `brute_force_tsp` 在 `n_small=9` 子问题上枚举全排列，得到精确最优值作为对照基线。  
8. `main` 打印两类最近邻结果、耗时、启发式误差和校验结论，形成无交互可复现的完整 MVP。  

该实现没有调用第三方 TSP 黑盒求解器，最近邻决策和多起点比较均可在源码中逐行追踪。
