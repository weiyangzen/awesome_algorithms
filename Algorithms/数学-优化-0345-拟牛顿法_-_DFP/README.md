## R01

DFP（Davidon-Fletcher-Powell）是早期经典拟牛顿法，用于求解无约束光滑优化问题：

\[
\min_{x \in \mathbb{R}^n} f(x)
\]

它不显式计算 Hessian 矩阵，而是在迭代中维护逆 Hessian 近似矩阵 \(H_k \approx (\nabla^2 f(x_k))^{-1}\)。相比最速下降，DFP 能利用曲率信息，通常显著减少迭代轮数。

## R02

适用场景：
- 目标函数可微，梯度可计算，但完整 Hessian 成本高或数值不稳定。
- 中低维无约束优化（如参数拟合、数值最优化教学示例、科学计算中的平滑目标）。
- 需要“比梯度下降更快，但比牛顿法更便宜”的折中方案。

不适用场景：
- 非光滑目标（如大量不可导点）。
- 超高维问题（完整 DFP 需要 `O(n^2)` 存储）。

## R03

DFP 的核心更新量：
- \(g_k = \nabla f(x_k)\)
- \(p_k = -H_k g_k\)
- \(x_{k+1} = x_k + \alpha_k p_k\)
- \(s_k = x_{k+1} - x_k\)
- \(y_k = g_{k+1} - g_k\)

逆 Hessian 近似的 DFP 更新公式：

\[
H_{k+1} = H_k + \frac{s_k s_k^\top}{s_k^\top y_k}
- \frac{H_k y_k y_k^\top H_k}{y_k^\top H_k y_k}
\]

工程实现中会检查 `s_k^T y_k > 0` 且 `y_k^T H_k y_k > 0`，避免除零和失去正定性。

## R04

`demo.py` 输入输出约定：
- 输入：固定初始点 `x0=[-1.2, 1.0]`，以及 `tol/max_iter`。
- 输出：
  - 最终点 `x`、最终目标值 `f(x)`、梯度范数。
  - 迭代次数、是否收敛、函数/梯度评估次数。
  - 部分迭代轨迹（包含步长与 `y^T s`）。

示例目标函数使用 Rosenbrock（理论最优点 `[1, 1]`）。

## R05

伪代码：

```text
x <- x0
H <- I
for k in 0..max_iter-1:
    g <- grad(x)
    if ||g|| < tol: stop

    p <- -H g
    if g^T p >= 0:
        p <- -g
        H <- I

    alpha <- Armijo_backtracking(f, x, g, p)
    x_new <- x + alpha p
    g_new <- grad(x_new)

    s <- x_new - x
    y <- g_new - g
    ys <- y^T s
    yHy <- y^T H y

    if ys > eps and yHy > eps:
        H <- H + (s s^T)/ys - (H y y^T H)/yHy
        H <- (H + H^T)/2
    else:
        H <- I

    x <- x_new
return x
```

## R06

正确性要点（工程视角）：
- 当 `H_k` 对称正定时，`p_k = -H_k g_k` 为下降方向。
- Armijo 回溯线搜索保证每步至少满足充分下降条件，避免盲目大步导致发散。
- DFP 更新通过割线条件编码局部曲率信息，较纯梯度法更快接近最优点。
- 通过曲率阈值检查与退化重置，可以抑制数值病态传播。

## R07

复杂度（每次迭代）：
- 时间复杂度：`O(n^2)`（矩阵-向量乘与 rank-2 更新）。
- 空间复杂度：`O(n^2)`（存储 `H_k`）。
- 线搜索额外代价：若平均回溯 `m` 次，则增加约 `m` 次函数评估。

## R08

与相关方法对比：
- 梯度下降：实现最简单但病态问题常收敛慢。
- 牛顿法：局部收敛快但依赖 Hessian/线性系统求解。
- BFGS：同属拟牛顿，实践中通常比 DFP 更鲁棒。
- L-BFGS：以内存换速度，适合高维；DFP 更适合教学与中小规模问题。

## R09

`demo.py` 关键数据结构：
- `x: np.ndarray` 当前参数向量。
- `H: np.ndarray` 逆 Hessian 近似矩阵。
- `DFPResult`：收敛信息打包（`x/f/grad_norm/iterations/...`）。
- `history`：迭代日志列表，记录 `(iter, f(x), ||grad||, step, y^T s)`。

## R10

边界与异常处理：
- `x0` 必须是一维非空向量。
- 若 `g^T p >= 0`（方向非下降），回退到 `-g` 并重置 `H=I`。
- 若 `y^T s` 或 `y^T H y` 过小，跳过 DFP 更新并重置 `H`，防止除零和矩阵污染。
- 回溯线搜索到达最大次数后返回最小候选步长，保证流程可继续。

## R14

常见实现错误：
- 把 DFP 公式写成错误的矩阵乘法顺序，导致维度或数值崩坏。
- 忽略 `y^T s` / `y^T H y` 检查，出现除零与非正定问题。
- 线搜索不验证 Armijo 条件，直接固定步长，容易震荡。
- 未处理 `g^T p >= 0` 的退化方向，导致“上坡”更新。

## R15

最小测试清单：
- 功能测试：默认初值能收敛到 Rosenbrock 最优点附近。
- 稳健性测试：替换初值（如 `[-1.5, 1.5]`、`[0, 0]`）应整体下降。

## R17

局限与取舍：
- 完整 DFP 需要 `O(n^2)` 内存，不适合超高维参数空间。
- 在强非凸区域，曲率信息可能不稳定，需要更强的保护策略。
- 本实现偏“教学可读 + 可运行验证”，未覆盖工业级线搜索细节（如插值型 strong Wolfe 全流程）。

## R18

`demo.py` 源码级流程（8 步，非黑箱）：
1. `main` 固定初值 `x0`，调用 `dfp_optimize` 执行优化。  
2. `dfp_optimize` 初始化 `x/H=I`，计算初始 `f(x)` 与 `g=∇f(x)`。  
3. 每轮先检查 `||g|| < tol`；若满足则直接收敛结束。  
4. 计算方向 `p=-H@g`，若 `g^T p >= 0` 则回退到最速下降并重置 `H`。  
5. `backtracking_line_search` 对 `p` 做 Armijo 回溯，找到满足充分下降的 `alpha`。  
6. 更新 `x_new/g_new` 后构造 `s=x_new-x`、`y=g_new-g`，并计算 `y^T s`、`y^T H y`。  
7. 若曲率条件有效，按 DFP 公式更新 `H` 并做对称化；否则重置 `H` 防止病态传播。  
8. 写入 `history` 并继续迭代，最终输出收敛指标和头尾迭代轨迹。
