# 引力时间延迟 (Shapiro Delay)

## R01

引力时间延迟（Shapiro Delay）描述的是：电磁信号经过大质量天体附近时，传播时间会比平直时空中的几何光程更长。对工程算法来说，它本质上是“额外时间项”的建模与反演问题，而不是直接求解全量爱因斯坦场方程。

本目录 MVP 聚焦一条可执行链路：

1. 构造近合日（conjunction）观测几何。
2. 用弱场近似公式生成 Shapiro 延迟。
3. 叠加测距噪声形成“观测”。
4. 反演 PPN 参数 `gamma` 并做可重复校验。

## R02

Shapiro 延迟是广义相对论的重要经典检验之一，典型场景是太阳系无线电测距（地球-行星/探测器链路）。在日掩附近，信号路径掠过太阳附近区域，时间延迟显著增强，便于统计估计。

这个算法在现代任务中的价值是：

- 为深空测控中的光行时修正提供基础模块；
- 在参数估计层面对引力理论（PPN 框架）给出可量化约束；
- 作为“物理可解释”的标定工具，避免纯黑箱回归。

## R03

MVP 使用的一程（one-way）弱场近似为：

`Δt = (1 + gamma) * (G*M/c^3) * ln(4*r1*r2 / b^2)`

其中：

- `M` 为中心引力体质量（本例取太阳）；
- `r1, r2` 为发射端和接收端到中心天体距离；
- `b` 为光线冲量参数（impact parameter）；
- `gamma` 为 PPN 参数，广义相对论预言 `gamma = 1`。

当 `b` 变小（靠近合日）时，对数项变大，延迟升高。

## R04

为了让估计步骤透明，`demo.py` 将模型写成线性形式：

`y_i = beta * x_i + noise_i`, 其中 `beta = 1 + gamma`

定义：

`x_i = (G*M/c^3) * ln(4*r1*r2 / b_i^2)`

于是通过零截距最小二乘可以直接得到 `beta_hat`，再换算 `gamma_hat = beta_hat - 1`。该写法把“物理公式 + 参数反演”拆成清晰的数值步骤，避免黑箱拟合器。

## R05

算法主流程（高层）：

1. 生成合日前后观测时刻 `day`。
2. 生成每个时刻的视向角 `alpha(day)`，并计算 `b(day)`。
3. 用真值 `gamma_true` 计算无噪声 `true_delay`。
4. 叠加高斯噪声得到 `observed_delay`。
5. 构造 `basis = (G*M/c^3)*ln(...)`。
6. 通过最小二乘估计 `beta_hat`、`gamma_hat`。
7. 计算拟合残差、RMSE、置信区间。
8. 执行阈值检查并输出 `Validation: PASS/FAIL`。

## R06

复杂度分析（`N` 为观测点数）：

- 几何生成：`O(N)`
- 延迟计算：`O(N)`
- 参数估计（内积与残差）：`O(N)`
- 汇总输出（DataFrame）：`O(N)`

总复杂度 `O(N)`，内存复杂度 `O(N)`。对教学和快速验证场景，代价很低。

## R07

物理与统计前提：

- 弱场、静态中心体近似（不含太阳自转拖拽等高阶效应）；
- 路径弯曲对几何量的二阶反馈被忽略（保留主导对数项）；
- 观测噪声近似为独立同分布高斯；
- `r1, r2` 在观测窗口中视作常量（MVP 级简化）。

因此该实现是“可解释的最小工作原型”，不是任务级高精度轨道动力学软件。

## R08

本目录默认参数：

- `r1 = 1.00 AU`（地球轨道尺度）
- `r2 = 1.52 AU`（火星轨道尺度）
- `M = 1.0 Msun`
- `gamma_true = 1.0`
- 观测窗口：`[-30, 30]` 天，共 `121` 个采样
- `alpha_min = 0.35 deg`
- 测量噪声标准差：`1.2 us`

在该设置下，峰值 Shapiro 延迟约百微秒量级，能稳定支撑参数回收。

## R09

前置知识建议：

- 广义相对论弱场极限与 PPN 参数 `gamma` 的含义；
- 对数函数在误差传播中的灵敏度特征；
- 最小二乘估计与标准误差（standard error）；
- 测距/测时单位换算（秒、微秒、纳秒）。

## R10

适用边界：

- 适用：教学、算法原型、参数可解释性验证。
- 谨慎：希望达到探测器级别系统误差控制时。
- 不适用：需要联合轨道积分、日冕等离子体色散、多体摄动和仪器系统误差联合建模的生产任务。

若用于真实任务，必须接入更完整的动力学与观测模型。

## R11

`demo.py` 内置正确性检查点：

1. `|gamma_hat - gamma_true|` 在阈值内；
2. `SE(gamma_hat)` 足够小；
3. 拟合 RMSE 不显著大于噪声设定；
4. 拟合模型相对 `gamma=0` 基线有明显改进；
5. 合日附近峰值延迟显著高于窗口边缘均值。

所有检查显式打印，任一失败会以非零退出码结束。

## R12

数值稳定性设计：

- 冲量参数 `b` 设置下界（`1.05 * R_sun`），避免对数奇异；
- 对 `b^2` 使用 `EPS` 下限，防止极端除零；
- 显式校验 `ln` 参数 `> 1`，若违反立即抛错；
- 固定随机种子，保证输出可复现。

这些措施保证 MVP 在多次执行中稳定。

## R13

与其他做法对比：

- 对比黑箱非线性拟合：本实现可直接追踪每个物理量如何影响估计量。
- 对比全轨道高精度仿真：本实现覆盖主导机制，复杂度和开发成本低得多。
- 对比“只做曲线拟合”：本实现把参数约束写回物理公式，便于审计和扩展。

## R14

常见失效模式与防护：

- `b` 采样区间过大（离合日太远），延迟动态范围不足。
  - 防护：增加近合日采样密度。
- 噪声过大导致 `gamma` 估计方差膨胀。
  - 防护：增大样本量或提升测距精度。
- 几何模型过简（真实任务中 `r1, r2` 非常量）。
  - 防护：将 `r1(t), r2(t)` 替换为历表或动力学积分结果。

## R15

调参建议：

- `alpha_min_deg`：越小峰值延迟越大，但需满足物理遮挡与传播约束。
- `n_observations`：越大估计更稳，但计算与数据管理成本增。
- `noise_std_s`：控制可辨识度，建议先用合成数据做灵敏度扫描。
- `half_window_days`：窗口太宽会稀释近合日信息，太窄则统计样本不足。

## R16

工程化扩展方向：

## R18

`demo.py` 源码级算法流程拆解（8 步）：

1. `DelayConfig` 固化几何、噪声和真值参数，统一实验配置。
2. `conjunction_geometry` 生成 `day -> alpha -> b` 的近合日冲量参数轨迹，并对 `b` 加下界。
3. `shapiro_log_term` 计算 `ln(4*r1*r2/b^2)` 并做物理合法性检查（参数必须大于 1）。
4. `shapiro_delay_one_way_s` 用 `(1+gamma) * (G*M/c^3)` 与对数项相乘，生成理论延迟。
5. `main` 在理论延迟上叠加高斯噪声，形成模拟观测序列 `observed_delay_s`。
6. `estimate_gamma` 对线性模型 `y=beta*x` 做零截距最小二乘，得到 `beta_hat`、`gamma_hat`、标准误和拟合序列。
7. `main` 计算 RMSE、95% 置信区间、与 `gamma=0` 基线对比，并提取合日峰值与边缘均值差。
8. 将结果写入 `DataFrame` 打印，同时执行 5 个布尔检查；全部通过才输出 `Validation: PASS`。

实现中未把核心逻辑交给黑箱三方函数：几何构造、Shapiro 公式、参数估计与校验全部在源码内逐步展开。
