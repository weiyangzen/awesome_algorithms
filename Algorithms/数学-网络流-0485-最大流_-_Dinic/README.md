## R01

Dinic 算法用于求解有向容量网络中的最大流问题：给定源点 `s`、汇点 `t` 和每条边容量 `c(u,v)`，求从 `s` 到 `t` 的最大可行流量。

它的核心不是“每次只找一条增广路”，而是：
- 先用 BFS 构建分层图（level graph）；
- 再在分层图里通过 DFS 一次性推送阻塞流（blocking flow）；
- 重复上述阶段直到 `t` 不可达。

## R02

形式化定义：

- 容量约束：`0 <= f(u,v) <= c(u,v)`
- 反对称：`f(u,v) = -f(v,u)`
- 流守恒：对任意 `u != s,t`，`sum_x f(x,u) = sum_x f(u,x)`
- 目标：最大化 `|f| = sum_v f(s,v)`

Dinic 通过维护残量网络 `G_f` 迭代改进 `f`，直到不存在 `s -> t` 的增广路径。

## R03

相对常见算法的定位：

- Ford-Fulkerson：思想简单，但路径选择不当时效率不稳定。
- Edmonds-Karp：用 BFS 找最短增广路，复杂度 `O(VE^2)`。
- Dinic：按“分层 + 阻塞流”批量增广，通常更快，工程上常作为最大流基线实现。

在中等规模图上，Dinic 往往能明显优于 Edmonds-Karp。

## R04

时间复杂度（一般容量网络）：

- 单次 BFS 建层图：`O(E)`
- 一轮阻塞流 DFS（结合当前弧优化）：摊还 `O(VE)`
- 分层轮数不超过 `O(V)`
- 总复杂度：`O(V^2 E)`

补充：在若干特殊图类（如单位容量网络）上可得到更优界，但本 MVP 使用通用版本结论。

## R05

空间复杂度：

- 残量图采用邻接表 + 反向边索引，存储 `2E` 条边：`O(E)`
- `level`、`it`、BFS 队列等辅助结构：`O(V)`

总空间复杂度：`O(V + E)`。

## R06

手工微型示例：

图：
- `s -> a (3)`
- `s -> b (2)`
- `a -> b (1)`
- `a -> t (2)`
- `b -> t (3)`

第一轮分层图中可找到两条可行路径：
- `s -> a -> t` 推 2
- `s -> b -> t` 推 2

当前流量为 4。此后残量网络还能通过 `s -> a -> b -> t` 再推 1，最终最大流为 5。

## R07

为什么 Dinic 重要：

- 是网络流经典算法之一，理论与竞赛/工程都高频出现；
- 结构清晰（BFS 层次 + DFS 推流），适合教学和审计；
- 是更复杂流模型（最小费用流、上下界流）实践中的基础组件。

## R08

理论关键点：

1. 残量网络中，边容量表示“还能继续推送的量”。
2. BFS 层次保证只沿最短层数方向走，避免无效回退。
3. 阻塞流含义：在当前分层图中，任意 `s->t` 路径都至少有一条边被饱和。
4. 每完成一轮阻塞流，残量网络中 `s->t` 的最短距离会严格增加。
5. 最短距离最多增加到 `V-1`，所以阶段数有上界。

## R09

适用条件与限制：

- 适用于单源单汇最大流；
- 容量需非负（本实现按整数容量处理）；
- 图允许重边、反向边、反平行边；
- 当顶点非常多且递归深度很大时，DFS 递归实现可能受 Python 递归深度限制（本示例规模较小，不触发）。

## R10

正确性框架（简版）：

1. 每次 DFS 推流都只在残量网络合法边上推进，故始终满足容量约束。
2. 反向边同步增减，保证流量反对称与流守恒不被破坏。
3. 某轮结束后得到阻塞流，意味着当前分层图内没有可继续增广的 `s->t` 路径。
4. 当 BFS 无法到达 `t` 时，残量网络不存在任何增广路。
5. 由最大流最小割定理，此时当前流即最大流。

## R11

数值与实现注意点：

- 整数容量下，算法全过程保持整数流量；
- 若容量很大，需避免 32 位整型溢出；本实现使用 Python `int`，不受该限制；
- 随机测试时使用固定随机种子，保证可复现；
- 基准对拍采用 Edmonds-Karp（参考实现）做一致性校验。

## R12

性能实践要点：

- “当前弧优化”（`it[u]`）避免同一阶段反复扫描失效边，是 Dinic 性能关键之一；
- 邻接表 + 反向边下标比邻接矩阵更节省空间、更新更快；
- 本 `demo.py` 输出了 `phase_count / augmentations / dfs_calls`，可直观看阶段与搜索开销。

## R14

鲁棒性处理（代码中已覆盖）：

- 顶点数 `n <= 1` 报错；
- 边端点越界报错；
- 负容量报错；

## R15

实现结构（对应 `demo.py`）：

- `Edge`：残量边结构，包含 `to/rev/cap`。

- `Dinic.max_flow`：外层阶段循环。
- `Dinic.min_cut_reachable`：在最终残量图取 `s` 可达集。
- `edmonds_karp_max_flow`：参考算法，仅用于对拍验证。
- `run_classic_case` / `random_graph_case`：固定样例 + 随机样例测试入口。

## R16

相关算法对比：

- Edmonds-Karp：更易证明与讲解，但一般更慢；
- Push-Relabel：在大规模稠密图上常表现很好，但实现复杂度更高；
- ISAP / HLPP：工程性能很强，代码和调参复杂度也更高。

Dinic 处于“实现复杂度与性能”较均衡的位置。

## R18

源码级算法流程拆解（`Dinic.max_flow` 主路径，8 步）：

1. 初始化总流量 `flow=0`，并准备 `inf` 作为单次 DFS 可推上界。
2. 调用 `_bfs_level_graph(s,t)` 在残量图上分层，得到每个点的最短层号。
3. 若 `t` 不可达，终止循环并返回 `flow`（不存在增广路，当前即最大流）。
4. 重置当前弧数组 `it`，确保本轮阻塞流从每个点的第一条候选边开始扫描。
5. 反复调用 `_dfs_blocking_flow(s,t,inf)`，仅沿“容量>0 且层号+1”边前进。
6. DFS 成功到达 `t` 时返回瓶颈流量 `pushed`，并在回溯中同步更新正反向残量边。
7. 若某次 DFS 返回 `0`，表示当前层次图已无法继续增广，本轮阻塞流完成，回到步骤 2 重建分层图。
8. 所有轮次结束后输出 `flow`；`min_cut_reachable` 可进一步恢复最小割的 `S/T` 划分。

本实现没有调用第三方最大流黑盒 API，Dinic 的分层与阻塞流过程在源码中完整展开。
