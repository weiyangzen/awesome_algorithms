# Cohen-Sutherland裁剪

## R01

Cohen-Sutherland 裁剪用于把一条二维线段裁剪到轴对齐矩形窗口内。

- 输入:
  - 裁剪窗口 `W = [xmin, xmax] x [ymin, ymax]`
  - 线段端点 `P1(x1,y1), P2(x2,y2)`
- 输出:
  - 若线段与窗口有可见部分: 返回裁剪后的线段端点
  - 若完全不可见: 返回空
- 目标:
  - 用快速位运算完成“快速接受/快速拒绝/必要时求交”的裁剪流程。

## R02

该算法由 Danny Cohen 与 Ivan Sutherland 在早期计算机图形学体系中提出，是经典线段裁剪方法之一。

- 背景: 光栅化和窗口系统需要先裁剪再绘制，避免无效像素工作。
- 特点: 通过 4 位区域码（outcode）表达点相对窗口的位置关系。
- 地位: 与 Liang-Barsky、Cyrus-Beck 一起构成“线段裁剪”核心知识点。

## R03

问题动机是减少渲染管线中的无效处理。

- 若不裁剪，窗口外大量线段仍可能进入后续绘制阶段，浪费计算。
- Cohen-Sutherland 利用区域码先做逻辑判断:
  - `code1 | code2 == 0` 时可直接接受；
  - `code1 & code2 != 0` 时可直接拒绝。
- 仅在“既不能直接接受也不能直接拒绝”时才计算边界交点，整体流程非常轻量。

## R04

可对照的朴素思路是“逐边求交 + 点内测试”:

1. 计算线段与矩形四条边的交点候选；
2. 筛掉不在线段参数范围内的交点；
3. 结合端点是否在窗口内，拼出最终可见段。

该方法也能得到正确答案，但分支复杂、重复判断较多。Cohen-Sutherland 用位编码把判定过程结构化，更适合作为工程基线实现。

## R05

核心算法结构:

1. 为每个端点计算 4 位区域码（LEFT/RIGHT/BOTTOM/TOP）。
2. 循环执行:
   - 若 `code1 | code2 == 0`，整段在窗口内，接受；
   - 若 `code1 & code2 != 0`，整段在同侧外部，拒绝；
   - 否则选取一个外点，根据其区域码命中的边界求交，并用交点替换该外点。
3. 重复直到接受或拒绝。

区域码定义（按位）:

- `LEFT = 1`（`x < xmin`）
- `RIGHT = 2`（`x > xmax`）
- `BOTTOM = 4`（`y < ymin`）
- `TOP = 8`（`y > ymax`）

## R06

示例: 窗口 `x:[0,10], y:[0,8]`，线段 `(-4,2) -> (12,6)`。

1. 起点在左侧，`code1=LEFT`；终点在右侧，`code2=RIGHT`。
2. 不能直接接受/拒绝，先处理左侧外点，与 `x=0` 求交，得 `(0,3)`。
3. 更新后线段变为 `(0,3) -> (12,6)`，终点仍在右侧。
4. 再与 `x=10` 求交，得 `(10,5.5)`。
5. 两端点都在窗口内，接受，结果是 `(0,3) -> (10,5.5)`。

## R07

算法价值:

- 图形学管线: 经典“裁剪阶段”基础算法，易与扫描转换连接。
- 软件工程: 位运算判定清晰，便于调试与单元测试。
- 教学价值: 展示“编码状态 + 布尔代数”如何减少几何分支复杂度。

## R08

前置知识:

- 二维解析几何（直线参数方程、边界求交）；
- 位运算（按位与、按位或）；
- 浮点比较与容差处理；
- 基础边界条件设计（水平线、垂直线、零长度线段）。

## R09

适用条件与边界:

- 适用窗口: 轴对齐矩形（AABB）。
- 适用对象: 二维线段（非无限直线）。
- 零长度线段: 允许；若点在窗口内则保留，否则拒绝。
- 若要处理旋转窗口或凸多边形，可考虑 Cyrus-Beck 等更一般方法。

## R10

正确性要点（简述）:

1. 每次循环保持不变式: 当前线段与原线段共线，且可见部分不会被错误删除。
2. `code1 | code2 == 0` 等价于两端点都在窗口内，因此整段可见。
3. `code1 & code2 != 0` 说明两端点在某同一外半平面，线段不可能穿过窗口，拒绝正确。
4. 在其余情况下，必有一端点在窗口外；按该端点命中的边界求交并替换，不会丢失潜在可见段。
5. 每次替换都会去掉一个外侧部分，有限步后一定进入“接受”或“拒绝”分支，算法终止。

## R11

数值与稳健性:

- 求交使用浮点运算，建议设置 `eps` 防止“几乎为零”分母造成数值爆炸。
- 点在边界上通常按“在内”处理（比较使用 `<` 与 `>`，而非 `<=` 与 `>=`）。
- 测试时对浮点结果使用容差比较（如 `1e-9`）。

## R12

工程成本（非大 O 维度）:

- 算法本身是常数时间流程，但实现质量高度依赖边界条件覆盖。
- 调试成本主要来自:
  - 交点公式分支（四条边）；
  - 水平/垂直线段的分母处理；
  - 端点更新顺序与区域码重算时机。
- 在批量线段场景中，可通过向量化预筛（先 outcode）进一步提速。

- 近似比/概率保证: N/A。
- 原因: Cohen-Sutherland 是确定性精确裁剪算法，不属于近似算法。

## R14

常见失效模式:

- 区域码位定义写错（如 TOP/BOTTOM 混淆），导致错误接受/拒绝。
- 更新端点后忘记重算区域码，可能死循环。
- 对水平线求 top/bottom 交点时分母为零未处理，导致异常或 NaN。
- 浮点容差过严，边界点被误判到外侧。

## R15

实现建议:

- 单独实现 `compute_outcode()`，不要把逻辑分散在主循环中。
- 交点计算封装为清晰分支，并对分母接近 0 做保护。
- 用一个参考实现（如 Liang-Barsky）做随机对拍，提高可信度。
- 固定样例至少覆盖: 全内、全外同侧、穿越左右边界、穿越上下边界、端点在边界。

## R16

前驱与后继关系:

- 前驱思路: 直接几何求交 + 过滤的朴素实现。
- 同代方法: Liang-Barsky（参数区间裁剪，通常分支更少）。
- 后继扩展:
  - Sutherland-Hodgman（多边形裁剪）；
  - Cyrus-Beck（任意凸裁剪窗口）；
  - 在现代 GPU 管线中对应更一般的裁剪阶段思想。

## R17

本目录 `demo.py` 的 MVP 设计:

- 依赖: `numpy`（随机回归与数据生成）；核心裁剪逻辑纯手写。
- 核心函数:
  - `compute_outcode()`：计算端点区域码。
  - `cohen_sutherland_clip()`：主算法，返回裁剪结果与迭代次数。
  - `liang_barsky_clip()`：作为对照实现用于随机回归验证。
- 运行内容:
  - 固定样例验证（含可视结果断言）；
  - 随机 2000 线段对拍（两种算法结果一致）；
  - 小型性能快照（输出耗时与接受数量）。
- 运行命令: `uv run python demo.py`。

## R18

`demo.py` 源码级算法流（8 步）:

1. `main()` 定义矩形窗口，执行固定样例、随机回归和性能快照三组实验。
2. `cohen_sutherland_clip()` 先对两端点调用 `compute_outcode()` 生成 4 位区域码。
3. 若 `code1 | code2 == 0`，直接返回原线段（完全可见）。
4. 若 `code1 & code2 != 0`，直接返回 `None`（完全不可见）。
5. 否则选择一个外点，根据其命中的位（TOP/BOTTOM/RIGHT/LEFT）与对应边界求交。
6. 用交点替换该外点，并重新计算其区域码；循环继续，直到进入第 3 或第 4 步。
7. `liang_barsky_clip()` 独立计算参数区间 `[u1,u2]`，作为结果对照，不参与 Cohen-Sutherland 主逻辑。
8. 随机回归中比较两算法的“是否可见”和“裁剪端点”，若不一致立即抛异常；全部通过后打印汇总。
