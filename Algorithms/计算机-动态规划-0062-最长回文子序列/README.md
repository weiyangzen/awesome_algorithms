# 最长回文子序列

- UID: `CS-0045`
- 学科: `计算机`
- 分类: `动态规划`
- 源序号: `62`
- 目标目录: `Algorithms/计算机-动态规划-0062-最长回文子序列`

## R01

“最长回文子序列（LPS, Longest Palindromic Subsequence）”要求在给定字符串中，选取若干字符（保持原相对顺序、可不连续）形成回文串，并使长度最大。  
它和“最长回文子串”不同：子串必须连续，而子序列不要求连续。

## R02

问题定义：
- 输入：字符串 `s`（长度记为 `n`）。
- 输出：
  - `L`：最长回文子序列长度；
  - `p`：一个长度为 `L` 的具体回文子序列（用于可解释验证）。

示例：
- `s = "bbbab"`，可得 `L=4`，一个可行 `p="bbbb"`。
- `s = "cbbd"`，可得 `L=2`，可行 `p="bb"`。

## R03

核心 DP 建模（区间 DP）：
- 状态：`dp[i][j]` 表示子串区间 `s[i..j]` 的最长回文子序列长度。
- 边界：`dp[i][i] = 1`（单字符天然回文）。
- 转移：
  - 若 `s[i] == s[j]`，则 `dp[i][j] = dp[i+1][j-1] + 2`（两端字符配对）。
  - 若 `s[i] != s[j]`，则 `dp[i][j] = max(dp[i+1][j], dp[i][j-1])`（去掉一端取更优）。
- 计算顺序：按区间长度从短到长递推。

## R04

正确性直觉：
- 任意区间 `s[i..j]` 的最优解，必然落在三种结构之一：
  - 使用两端字符（仅当两端相等）；
  - 不使用左端字符；
  - 不使用右端字符。
- 这三种选择覆盖了所有可行最优方案，因此取对应最优值即可。
- 子问题规模严格缩小（`(i+1,j-1)`、`(i+1,j)`、`(i,j-1)`），满足动态规划条件。

## R05

复杂度：
- 时间复杂度：`O(n^2)`（共有 `O(n^2)` 个区间状态，每个状态 `O(1)` 转移）。
- 空间复杂度：`O(n^2)`（保存二维 DP 表，便于后续重建具体序列）。

在 `n` 中等规模时，这个复杂度是该问题的标准解法。

## R06

`demo.py` 的实现内容：
- `lps_dp_with_reconstruction`：主算法，返回 `长度 + 一个可行最优子序列`。
- `lps_via_lcs_length`：使用 `LCS(s, reverse(s))` 计算长度做交叉校验。
- `bruteforce_lps_length`：小规模穷举精确解，仅用于验证。
- `run_case`：单用例运行与断言。
- `randomized_regression`：随机回归测试，提升正确性信心。

## R07

最小工具栈：
- `numpy`：用于构建二维 `dp` 数组（`np.int32`）和基础数值容器能力。
- Python 标准库（`dataclasses`, `random`）：结果组织与随机回归。

未使用大型框架，保持 MVP 轻量且可读。

## R08

输入与边界处理：
- 输入必须是 `str`，否则抛 `TypeError`。
- 空串 `""` 合法，结果应为长度 `0`、子序列 `""`。
- 单字符串 `"a"` 结果为长度 `1`、子序列 `"a"`。
- 对重复字符、无重复字符、混合字符都可统一处理。

## R09

序列重建策略：
- 基于已算好的 `dp`，用双指针 `i/j` 从区间两端向中间移动。
- 若两端字符匹配且满足最优值关系，则两端加入结果。
- 否则根据 `dp[i+1][j]` 与 `dp[i][j-1]` 的大小移动一端。
- 当两侧值相同，固定采用“先移动 `i`”的 tie-break，保证输出稳定可复现。

## R10

为什么要做多重校验：
- 单一 DP 实现可能存在边界 bug（如 `i+1 > j-1` 处理）。
- `LCS` 校验从不同实现路径验证“长度”正确性。
- 暴力穷举在小规模字符串上提供“精确真值”对照。
- 三者一致时，MVP 可信度显著提升。

## R12

输出字段说明：
- `dp_result -> length, subsequence`：主 DP 求得的长度与一个最优回文子序列。
- `lcs_cross_check -> length`：`LCS(s, reverse(s))` 计算出的长度。
- `bruteforce_check -> length`：短串穷举得到的最优长度。
- `checks`：
  - `palindrome`：输出序列是否回文；
  - `is_subsequence`：输出序列是否真为原串子序列；
  - `length_match`：输出序列长度与声明长度是否一致；
  - `lcs_match / bruteforce_match`：与校验实现是否一致。

## R13

建议测试样例：
- 典型案例：`"bbbab"`、`"cbbd"`。
- 边界案例：`""`、`"a"`。
- 无重复字符：`"abcd"`（结果应为 1）。
- 含多种可行解：`"character"`（长度固定但可有多答案）。
- 随机短串回归：验证大批输入稳定一致。

## R14

工程注意点：
- 区间 DP 的循环顺序必须按区间长度递增，否则依赖状态尚未计算。
- `s[i]==s[j]` 时不能盲目 `+2`，需结合内层区间值。
- 重建阶段若无固定平局策略，输出可能不稳定，不利于自动化比较。
- 暴力法仅用于小输入验证，不能用于大规模求解。

## R15

与其他方法对比：
- 与暴力枚举相比：DP 从指数复杂度降到 `O(n^2)`。
- 与仅记忆化 DFS 相比：自底向上 DP 更直观可控，且便于重建序列。
- 与“直接调用第三方现成函数”相比：本实现完整展示状态定义、递推和重建，算法透明。

## R16

应用场景：
- 生物序列、日志模式等场景中的对称结构挖掘。
- 文本分析中“保序压缩后的回文特征”提取。
- 动态规划教学示例（区间 DP + 路径重建）。
- 算法面试与竞赛中的经典字符串 DP 题。

## R17

可扩展方向：
- 在保持 `O(n^2)` 时间下做空间压缩（仅求长度时可滚动数组优化）。
- 输出“所有”最优回文子序列（需额外去重策略）。
- 增加字符集规模与长度分布的性能基准。
- 结合启发式剪枝做近似解，服务超长文本场景。

## R18

`demo.py` 源码级流程（8 步）：
1. `main` 准备固定字符串样例，逐个调用 `run_case`。  
2. `run_case` 调 `lps_dp_with_reconstruction`，先建 `n x n` DP 表并按区间长度递推。  
3. 在 `lps_dp_with_reconstruction` 中执行转移：匹配两端时取 `dp[i+1][j-1]+2`，否则取左右子区间最大值。  
4. 同一函数随后用双指针基于 `dp` 回溯，构造一个具体最优回文子序列。  
5. `run_case` 再调用 `lps_via_lcs_length`，通过 `LCS(s, reverse(s))` 独立计算长度做交叉验证。  
6. 对长度不超过 16 的字符串，`run_case` 调 `bruteforce_lps_length` 穷举全部子序列求精确最优长度。  
7. `run_case` 检查“回文性、子序列合法性、长度一致性、两种校验一致性”，并打印结果。  
8. 固定样例通过后，`randomized_regression` 运行 200 轮随机短串回归，确认实现稳定正确。
