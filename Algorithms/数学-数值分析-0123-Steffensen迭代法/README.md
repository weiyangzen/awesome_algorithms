# Steffensen迭代法

- UID: `MATH-0123`
- 学科: `数学`
- 分类: `数值分析`
- 源序号: `123`
- 目标目录: `Algorithms/数学-数值分析-0123-Steffensen迭代法`

## R01

本条目实现 Steffensen 迭代法（导数自由的非线性方程求根方法）最小可运行版本：
- 输入目标函数 `f(x)` 与初值 `x0`；
- 迭代逼近方程 `f(x)=0` 的实根；
- 输出根估计、收敛状态和迭代轨迹。

MVP 重点是把公式、停止条件与数值稳定性检查明确落地，而不是调用黑盒求根器。

## R02

问题定义（标量实数域）：
- 输入：
  - 连续函数 `f: R -> R`；
  - 初值 `x0`；
  - 控制参数 `max_iter`、`tol`、`denom_tol`。
- 输出：
  - 根的近似值 `x*`；
  - `converged`（是否达到收敛准则）；
  - 迭代历史（每一步的 `x_k`、`f(x_k)`、分母与步长信息）。

## R03

Steffensen 迭代核心公式（用于 `f(x)=0`）：

`x_{k+1} = x_k - f(x_k)^2 / (f(x_k + f(x_k)) - f(x_k))`

记：
- `f_k = f(x_k)`
- `d_k = f(x_k + f_k) - f_k`

则更新可写成：
`x_{k+1} = x_k - f_k^2 / d_k`

当根为单根、函数足够光滑且初值在吸引域内时，Steffensen 通常具有局部二次收敛。

## R04

算法流程：
1. 校验参数合法性（有限数、正阈值、正迭代上限）。
2. 设 `x = x0`。
3. 计算 `f(x)`；若 `|f(x)| <= tol`，直接收敛。
4. 计算 `f(x + f(x))` 并构造分母 `d = f(x + f(x)) - f(x)`。
5. 若 `|d|` 过小，停止并标记为“分母退化”。
6. 计算步长 `step = f(x)^2 / d`，更新 `x_next = x - step`。
7. 用“相邻迭代差值”与“残差”联合判停。
8. 记录轨迹并继续，直至收敛或达到 `max_iter`。

## R05

核心数据结构：
- `IterationRecord`（`dataclass`）：
  - `k`：迭代编号；
  - `x`：当前估计；
  - `fx`：当前残差值 `f(x)`；
  - `denom`：本轮分母 `f(x+f(x))-f(x)`；
  - `step`：本轮更新量 `f(x)^2/denom`。
- `SteffensenResult`（`dataclass`）：
  - `root`、`converged`、`iterations`、`final_residual`、`stop_reason`、`history`。

## R06

正确性与收敛说明（工程视角）：
- 该更新来自 Aitken `Δ^2` 加速思想在固定点迭代上的导数自由实现；
- 在单根附近，其收敛速度可达到与牛顿法同阶（二次）；
- 与牛顿法相比，避免显式计算导数，但每轮需要额外函数评估；
- 演示脚本给出多组样例与高精度参考根对照，可直接检验实现正确性。

## R07

复杂度分析（单个样例）：
- 每轮常数次函数评估，设迭代步数为 `T`；
- 时间复杂度：`O(T)`；
- 空间复杂度：
  - 记录完整轨迹为 `O(T)`；
  - 仅保留当前状态时为 `O(1)`。

## R08

边界与异常处理：
- 参数错误：`max_iter <= 0`、`tol <= 0`、`denom_tol <= 0` 抛出 `ValueError`；
- 数值错误：`x`、`f(x)`、`x_next` 非有限数时抛出 `RuntimeError`；
- 退化分母：`|denom|` 过小则安全停止，避免除零或极端放大；
- 达到最大迭代步仍未达标时，返回 `converged=False` 与停止原因。

## R09

MVP 取舍：
- 采用 Python 标准库（`math` + `dataclasses`）实现，确保脚本在基础环境可直接运行；
- 不依赖 `scipy.optimize` 等现成求根 API，避免算法步骤被黑盒隐藏；
- 通过自写二分法生成参考根，只用于验证，不参与主算法更新。

## R10

`demo.py` 结构与职责：
- `check_finite`：统一有限性检查；
- `steffensen`：核心迭代器；
- `bisection_reference`：生成对照真值（用于误差评估）；
- `relative_error`：计算相对误差；
- `print_history`：打印前若干步迭代轨迹；
- `run_case`：执行单个样例并输出结果；
- `main`：批量运行多个测试函数，无交互输入。

- `reference`：二分法高精度参考根；
- `residual`：`|f(estimate)|`；

## R13

最小测试集（脚本内置）：
- `f(x) = cos(x) - x`，初值 `x0=1.0`，根约 `0.739...`；
- `f(x) = x^3 - x - 2`，初值 `x0=1.5`，根约 `1.521...`；

这些样例覆盖超越方程与多项式方程，便于观察方法稳定性。

## R14

关键参数建议：
- `max_iter`：默认 `30`，一般足够达到双精度下高精度；
- `tol`：默认 `1e-12`，用于残差和步长判停；
- `denom_tol`：默认 `1e-14`，用于分母退化防护。

在噪声函数或病态函数上可适度放宽 `tol` 与 `denom_tol`。

## R15

方法对比：
- 对比牛顿法：
  - 牛顿法每轮需要导数 `f'(x)`；
  - Steffensen 无需导数，更适合“函数可评估但导数不便/不稳”的场景。
- 对比割线法：
  - 割线法需要两个初值；
  - Steffensen 使用一个初值，但对分母稳定性更敏感。
- 对比二分法：
  - 二分法全局稳健但线性收敛；
  - Steffensen 局部更快，但依赖初值与函数形态。

## R16

适用场景：
- 导数难以显式获取的非线性求根；
- 需要在简单实现中获得接近牛顿法的局部收敛速度；
- 数值分析教学中演示“导数自由二次收敛”思想。

## R17

可扩展方向：
- 向量化批量求解（`numpy`）并统计不同初值下迭代分布；
- 加入阻尼或回溯策略，提高远离根时的鲁棒性；

- 将迭代历史导出为 `pandas` 数据表，便于画收敛曲线。

## R18

源码级算法流（对应 `demo.py`，8 步）：
1. `main` 构造样例列表（函数、初值、参考区间）并逐个调用 `run_case`。  
2. `run_case` 先调用 `steffensen`，输入 `f, x0, max_iter, tol, denom_tol`。  
3. `steffensen` 在循环中计算 `fx = f(x)`，若残差达标则返回。  
4. 继续计算 `fx_shift = f(x + fx)`，构造 `denom = fx_shift - fx`。  
5. 若 `denom` 小于阈值，立即停止并返回“分母退化”原因。  
6. 计算 `step = fx*fx/denom`，执行更新 `x_next = x - step`，并记录 `IterationRecord`。  
7. 若步长足够小，则再做一次残差确认后返回；否则继续下一轮。  
8. `run_case` 用 `bisection_reference` 给出参考根，计算相对误差并打印轨迹摘要。
