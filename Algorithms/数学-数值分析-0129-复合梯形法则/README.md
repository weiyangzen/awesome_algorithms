# 复合梯形法则

- UID: `MATH-0129`

## R01

复合梯形法则（Composite Trapezoidal Rule）用于数值近似定积分：

`I = ∫[a,b] f(x) dx`

它把区间 `[a,b]` 等分为 `n` 个小区间，在每个小区间上用线性函数近似 `f(x)`，再把所有小梯形面积相加。该方法实现简单、稳定，常作为更高阶积分算法（如 Simpson、Romberg）的基础。

## R02

本条目 MVP 的目标是：

- 输入：可计算函数 `f`、积分区间 `[a,b]`、等分数 `n`。
- 输出：复合梯形近似值 `I_n`，并在演示中给出与解析值的误差对照。
- 验证：通过 `n` 逐步加倍，观察误差衰减与收敛阶（理论上对光滑函数是二阶）。

## R03

数学公式：

1. 令步长 `h = (b-a)/n`，网格点 `x_k = a + k h, k=0..n`。
2. 复合梯形法则为

`I_n = h * [ 0.5 f(x_0) + sum_{k=1}^{n-1} f(x_k) + 0.5 f(x_n) ]`

3. 若 `f` 在区间上二阶连续，则截断误差满足

`I - I_n = O(h^2)`，

即当 `n` 翻倍（`h` 减半）时，主导误差通常约缩小到原来的 `1/4`。

## R04

`demo.py` 的算法流程（单次积分）：

1. 检查输入端点与 `n` 的合法性（有限数、`n>0`）。
2. 生成均匀网格 `x = linspace(a,b,n+1)`。
3. 在网格点评估 `y_k = f(x_k)`（优先向量化，必要时回退逐点计算）。
4. 计算加权和 `S = y_0 + y_n + 2 * sum(y_1..y_{n-1})`。
5. 返回 `I_n = 0.5 * h * S`。

## R05

核心数据结构：

- `IntegralCase`：单个测试问题，包含函数、区间、解析真值。
- `n_values`：网格分段数序列（如 `4, 8, 16, ...`）。
- 逐轮统计列：`estimate`、`abs_error`、`rel_error`、`richardson_est`、`observed_order`。

该结构让“算法本体”和“实验输出”解耦，便于以后扩展自适应策略。

## R06

正确性要点：

- 复合梯形法来源于分段线性插值积分，公式有明确数学推导。
- 当 `f` 足够光滑时，误差应呈 `O(h^2)` 收敛，这可由实验中的 `observed_order` 接近 `2` 验证。
- 使用解析可积函数（如 `sin(x)`、`exp(-x^2)` 的 `erf` 形式、`1/(1+x^2)`）可直接检查实现正确性。

## R07

复杂度分析（单次积分）：

- 时间复杂度：`O(n)`（`n+1` 次函数评估与一次线性求和）。
- 空间复杂度：`O(n)`（显式保存网格与函数值）。

对多数工程任务而言，这是低门槛、可预测的数值积分基线方案。

## R08

边界与异常处理：

- `n <= 0`：抛出 `ValueError`；
- `a` 或 `b` 非有限数（`nan/inf`）：抛出 `ValueError`；
- `a == b`：直接返回 `0.0`；
- 函数若不能直接向量化计算，代码会自动回退到逐点评估路径。

该策略保证脚本在常见输入下行为可控、错误可诊断。

## R09

MVP 取舍说明：

- 选择“均匀网格 + 复合梯形”而非一次集成更复杂自适应框架；
- 不依赖黑盒积分 API（例如直接调用现成积分器），而是手写权重求和逻辑；
- 通过少量代表性函数做验证，强调可解释性与可复现性。

## R10

`demo.py` 函数职责：

- `_check_finite`：输入有限性校验；
- `_evaluate_on_grid`：函数值评估（向量化优先，逐点回退）；
- `composite_trapezoidal`：复合梯形积分核心实现；
- `richardson_error_estimate`：两层网格的误差估计；
- `relative_error`：相对误差计算；
- `run_case`：执行单个测试问题并打印收敛表；
- `main`：组织案例与统一运行入口。

## R11

运行方式：

```bash
cd Algorithms/数学-数值分析-0129-复合梯形法则
python3 demo.py
```

脚本无需交互输入，会自动输出三组积分问题在多种 `n` 下的误差表格。

## R12

输出字段解释（与 R04 的计算流程一一对应）：

- `estimate`：由复合梯形权重求和得到的 `I_n`；
- `abs_error`：`|I_n - I_exact|`；
- `rel_error`：相对误差；
- `richardson_est`：由 `n` 与 `2n` 结果估计的截断误差量级；
- `observed_order`：相邻两层误差计算得到的经验收敛阶。

若实现无误且函数够光滑，`observed_order` 通常会向 `2` 靠近。

## R13

建议最小测试集：

- `f(x)=sin(x), [0,pi], exact=2`
- `f(x)=exp(-x^2), [0,1], exact=0.5*sqrt(pi)*erf(1)`
- `f(x)=1/(1+x^2), [0,1], exact=atan(1)=pi/4`
- `n` 建议用 `4,8,16,32,64,128,256` 观察收敛趋势。

这些样例覆盖三角函数、指数函数与有理函数三类典型形态。

## R14

可调参数：

- `n_values`：离散粒度序列，影响精度与计算量；
- `order`（在 Richardson 估计中）：当前固定为 `2`，对应复合梯形理论阶；
- 测试案例集合 `cases`：可替换为业务中的被积函数。

调参建议：先用较小 `n` 验证流程，再逐步加密网格检查误差斜率。

## R15

方法对比：

- 相比单区间梯形法：复合梯形通过分段显著提高精度；
- 相比 Simpson 法：复合梯形实现更简洁，但同等精度通常需要更细网格；
- 相比高斯求积：复合梯形通用性好、易实现，但在高精度需求下效率可能较低。

## R16

应用场景：

- 缺少解析原函数时的快速定积分近似；
- 作为自适应积分、Romberg 外推的基线模块；
- 教学场景中的“误差阶验证”实验；
- 需要稳定、低复杂度、易审核实现的小型工程任务。

## R17

后续扩展方向：

- 加入自适应分段（按局部曲率细化网格）；
- 叠加 Romberg 外推，提高精度并复用已有 `n -> 2n` 结果；
- 增加对非光滑函数（如尖点、间断）的专门策略；
- 支持批量函数或批量区间的并行计算。

## R18

`demo.py` 源码级算法流（8 步，非黑盒）：

1. `main` 构造 `IntegralCase` 列表与统一的 `n_values` 网格序列。  
2. 每个案例进入 `run_case`，先打印区间与解析真值作为对照基准。  
3. `run_case` 循环调用 `composite_trapezoidal(func,a,b,n)` 计算各离散层结果。  
4. `composite_trapezoidal` 内先执行 `_check_finite` 与 `n>0` 校验，再生成均匀网格 `x`。  
5. `_evaluate_on_grid` 先尝试向量化评估 `f(x)`；若失败则逐点回退，保证函数兼容性。  
6. 对函数值应用梯形权重：两端点各乘 `1/2`，内部点系数为 `1`（通过 `2*sum` 统一实现）。  
7. `run_case` 用解析值计算 `abs_error/rel_error`，并用 `richardson_error_estimate` 给出误差量级估计与经验收敛阶。  
8. 表格输出 `n`、`estimate`、误差与阶数，直接验证复合梯形法的二阶收敛行为。
