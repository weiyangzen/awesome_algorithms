# 纳维-斯托克斯方程 (Navier-Stokes Equations)

- UID: `PHYS-0081`
- 学科: `物理`
- 分类: `流体力学`
- 源序号: `81`
- 目标目录: `Algorithms/物理-流体力学-0081-纳维-斯托克斯方程_(Navier-Stokes_Equations)`

## R01

纳维-斯托克斯方程描述黏性流体的动量守恒与质量守恒。二维不可压形式为：

- 动量方程：
  \[
  \frac{\partial \mathbf{u}}{\partial t} + (\mathbf{u}\cdot\nabla)\mathbf{u}
  = -\frac{1}{\rho}\nabla p + \nu\nabla^2\mathbf{u}
  \]
- 不可压约束：
  \[
  \nabla\cdot\mathbf{u}=0
  \]

本条目 MVP 采用投影法（projection method）求解二维方腔顶盖驱动流（lid-driven cavity）：

- 输入：流体参数 `rho, nu`，网格参数 `nx, ny`，时间推进参数 `nt`，压力迭代步数 `nit`，边界速度 `lid_velocity`。
- 输出：离散速度场 `u(x,y), v(x,y)`、压力场 `p(x,y)`，以及散度残差与采样点诊断值。
- 主步骤：
  1. 用当前速度构造压力泊松方程右端项 `b`；
  2. 迭代求解压力 `p`；
  3. 用压力梯度修正速度并加入对流/扩散项；
  4. 施加壁面无滑移与顶盖恒速边界；
  5. 重复至 `nt` 步，输出场量与诊断。

## R02

提出时间与关键里程碑：

- 1822 年：Claude-Louis Navier 给出黏性流体动量方程早期形式。
- 1845 年：George Gabriel Stokes 系统化黏性项，形成现代 Navier-Stokes 方程基础。
- 20 世纪中后期：随数值计算发展，形成 MAC、SIMPLE、PISO、投影法等 CFD 主流离散求解框架。

## R03

历史背景与问题来源：

- 欧拉方程忽略黏性，无法解释边界层、压降、涡耗散等关键现象。
- 工程上（管道、机翼、燃烧室、气象）需要同时处理对流、压强梯度、黏性扩散与边界条件。
- Navier-Stokes 方程将牛顿第二定律应用于连续介质微元，统一了“惯性-压力-黏性”的动力学平衡。

## R04

对本 MVP（二维规则网格 + 显式推进 + 压力迭代）可写成：

- 设网格总点数 `N = nx * ny`。
- 每个时间步：
  - 构造 RHS `b`：`O(N)`
  - 压力泊松 Jacobi 迭代 `nit` 次：`O(nit * N)`
  - 速度更新与边界处理：`O(N)`
- 总时间复杂度：
  \[
  O\bigl(nt\cdot (nit+1)\cdot N\bigr)
  \]

在本代码默认参数（`nx=ny=41, nt=300, nit=50`）下，主耗时来自压力泊松迭代。

## R05

空间复杂度（双精度数组）主要由网格场组成：

- 主状态：`u, v, p`，共 `3N`
- 临时数组：`un, vn, pn, b`（分阶段出现），约 `4N`
- 总体量级：`O(N)`

若按 `N=41*41=1681` 粗估，数组级内存开销很小，瓶颈更多是内存带宽与重复拷贝而非容量上限。

## R06

可复现实例（对应 `demo.py` 默认配置）：

- 物理参数：`rho=1.0, nu=0.1`
- 网格：`41 x 41`，`dx=dy=1/(41-1)=0.025`
- 时间步：`dt=min(cfl*dx/U_lid, 0.25*dx^2/nu)=0.00125`（默认 `cfl=0.05`）
- 边界条件：顶部 `u=1`，其余壁面 `u=v=0`
- 推进：`nt=300`，每步压力迭代 `nit=50`

运行 `uv run python demo.py` 后会打印：

- `max(|div u|) global` 与 `max(|div u|) core`：分别是全域残差与去除角点 2 层网格后的核心区残差；
- `max speed`、`mean speed`：速度场统计；
- 5 个探针点 `(x,y)` 的 `u,v,p,speed`。

## R07

算法意义：

- 理论上：是连续介质力学与偏微分方程中的核心方程组；3D 光滑解的整体存在唯一性仍是著名开放问题（千禧年难题之一）。
- 工程上：是 CFD、空气动力学、水动力学、热流耦合问题的基础模型。
- 方法论上：推动了离散化、线性求解器、湍流建模和高性能计算的发展。

## R08

该算法路径依赖的关键知识/工具：

- 质量守恒与动量守恒（连续介质控制方程）
- 向量微积分算子（梯度、散度、拉普拉斯）
- 有限差分离散（中心差分、显式时间推进）
- 压力泊松方程（由不可压条件导出）
- 线性迭代求解（Jacobi/Gauss-Seidel/SOR）
- CFL 与扩散稳定性约束

## R09

适用前提与边界条件：

- 本实现针对二维、不可压、牛顿流体、常密度、常黏度。
- 采用结构化笛卡尔网格，不适合复杂曲面几何直接贴体。
- 未包含湍流模型，仅对应层流或低雷诺数教学场景。
- 当高雷诺数、强瞬态或复杂边界层问题出现时，需要更高阶离散与更强求解器。

## R10

正确性依据（数值层面）：

- 投影法把“先预测速度、再用压力校正到无散度子空间”分解为两个子问题。
- 压力泊松方程来源于对动量方程取散度并施加 `∇·u=0` 约束。
- 在离散层面，若泊松方程解充分收敛，则校正后速度场散度趋近 0。
- 该离散方案对平滑解是一致的（consistent），在 `dt, dx, dy -> 0` 条件下具备收敛基础。

## R11

数值稳定性与误差传播：

- 时间离散为显式一阶，时间截断误差约 `O(dt)`。
- 黏性项与压力梯度是二阶中心差分（`O(dx^2), O(dy^2)`），对流项采用一阶迎风样式差分，整体精度受一阶对流项限制。
- 若 `dt` 过大，会出现速度振荡甚至发散；本实现用
  `dt=min(cfl*dx/U, 0.25*dx^2/nu)` 约束。
- 腔流顶角存在速度边界不连续（角点奇异），全域 `max(|div u|)` 往往被角点附近主导，因此代码同时报告核心区残差 `max(|div u|) core` 作为更稳定的比较指标。
- 压力迭代不足会导致不可压约束未充分满足，表现为残差偏大并污染速度场。

## R12

真实计算成本（超越渐近符号）：

- 压力迭代是主耗时：每个时间步要扫描网格 `nit` 次。
- 每步存在 `un/vn/pn` 拷贝，内存读写开销明显。
- Numpy 向量化降低了 Python 循环开销，但仍受限于内存带宽。
- 对更大网格，常见优化路径：
  - 使用更高效泊松解法（SOR、多重网格、FFT、稀疏线性代数）；
  - 减少全量拷贝；
  - 并行化（Numba/CuPy/PyTorch/JAX/GPU）。

## R13

近似比/误差界/概率保证：

- 本算法是确定性 PDE 数值求解，不涉及随机成功概率。
- 严格误差上界依赖解的正则性、边界条件与离散细节；MVP 未给出形式化常数界。
- 工程上通常用三类可观测指标做“实用误差控制”：
  - 散度残差 `||∇·u||_∞`；
  - 网格加密下关键观测量收敛（grid convergence）；
  - 与基准算例（如 Ghia cavity 数据）对比误差。

## R14

鲁棒性与失效模式：

- 失效 1：`dt` 太大导致显式推进不稳定，速度爆炸或出现 NaN。
- 失效 2：`nit` 太小导致压力场欠收敛，散度残差居高不下。
- 失效 3：高雷诺数下网格过粗，出现非物理振荡或涡结构失真。
- 失效 4：边界条件施加顺序错误，会造成壁面泄漏或虚假回流。
- 防护策略：保守 `dt`、监控残差、增加 `nit`、网格加密、必要时改用更稳定离散格式。

## R15

工程实现要点：

- 数据结构：使用 `numpy.ndarray` 存储 `u,v,p`，便于切片向量化。
- 初始化：零场 + 顶盖速度边界，保证初始状态与边界一致。
- 停止准则：MVP 使用固定 `nt`；工程版建议增加残差阈值提前停止。
- 调参优先级：`dt`（稳定性） > `nit`（不可压约束） > `nx,ny`（分辨率）。
- 常见陷阱：
  - 忘记每步重施加边界条件；
  - 使用错误索引导致梯度方向反号；
  - 误把 `dx` 和 `dy` 混用造成各向异性伪误差。

## R16

前驱/后继算法与应用：

- 前驱基础：欧拉方程、有限差分法、泊松方程迭代法。
- 经典后继与扩展：MAC、SIMPLE、PISO、分数步法、LES/RANS 湍流模型。
- 典型应用：
  - 腔流、管道流、绕流；
  - 飞行器外流场初步分析；
  - 微流控与热流耦合系统中的速度-压力场求解。
- 典型 benchmark：lid-driven cavity（本示例）、channel flow、backward-facing step。

## R17

Python MVP 方案（已在 `demo.py` 实现）：

- 依赖：`numpy`（数值计算）、`pandas`（探针结果表格化输出）。
- 核心函数：
  - `build_rhs`：构造压力泊松 RHS；
  - `pressure_poisson`：Jacobi 迭代解压力；
  - `cavity_flow`：执行 `nt` 次投影法时间推进；
  - `compute_divergence_linf`：评估不可压约束残差。
- 可运行命令：
  - `uv run python demo.py`
- 预期输出：网格与物理参数、`max(|div u|)`、速度统计和 5 个探针点值。
- 其中散度会同时报告 `global` 与 `core(trim 2 cells)` 两个版本。

## R18

源码追踪与算法拆解（对应本目录 `demo.py`，非黑箱调用）：

1. **配置与网格生成**：`SimulationConfig` 给出 `nx,ny,nt,nit,rho,nu`，在 `cavity_flow` 中计算 `dx,dy,dt`。
2. **状态初始化**：创建 `u,v,p` 全零场，作为初始速度与压力。
3. **构造压力源项**：`build_rhs` 用当前速度梯度组装 `b`，把不可压约束转化为压力驱动。
4. **压力泊松迭代**：`pressure_poisson` 对 `p` 做 `nit` 次 Jacobi 更新，并施加 Neumann/Dirichlet 混合边界。
5. **速度预测与校正**：在 `cavity_flow` 中显式更新 `u,v`，包含对流、黏性扩散、压力梯度三部分。
6. **施加速度边界条件**：每步强制壁面 `u=v=0`，顶盖 `u=lid_velocity`，防止边界漂移。
7. **时间推进循环**：重复步骤 3-6 共 `nt` 步，得到终态流场。
8. **后处理与验证**：`compute_divergence_linf` 评估 `||∇·u||_∞`，并输出探针点 `u,v,p` 诊断表。

以上 8 步覆盖了从 PDE 离散、线性子问题求解到不可压约束验证的完整最小实现链路。
