# 加速近端梯度法

## R01

加速近端梯度法（Accelerated Proximal Gradient, 常见实现为 FISTA）用于求解“光滑项 + 非光滑项”的复合凸优化问题：

`min_x F(x) = f(x) + g(x)`

其中 `f` 是可微且梯度 Lipschitz 连续，`g` 是凸但可非光滑（例如 `L1` 范数、指示函数）。该方法在保持每步计算便宜（梯度 + 近端算子）的同时，通过 Nesterov 动量加速收敛。

## R02

本目录的 `demo.py` 选择经典 LASSO 作为实验问题：

`min_x 0.5/n * ||Ax - b||_2^2 + lambda * ||x||_1`

- `f(x) = 0.5/n * ||Ax-b||_2^2`（光滑）
- `g(x) = lambda * ||x||_1`（非光滑）

该问题有封闭形式的近端算子（软阈值），适合用最小代码展示 APG/FISTA 的核心流程。

## R03

近端梯度基本迭代（ISTA）为：

`x_{k+1} = prox_{g/L}(x_k - (1/L) * grad f(x_k))`

其中 `L` 是 `grad f` 的 Lipschitz 常数。对 `g(x)=lambda||x||_1`，近端映射为逐元素软阈值：

`prox_{(lambda/L)||.||_1}(v)_i = sign(v_i) * max(|v_i| - lambda/L, 0)`

## R04

FISTA 在 ISTA 基础上加入外推变量：

- `t_{k+1} = (1 + sqrt(1 + 4 t_k^2)) / 2`
- `y_{k+1} = x_{k+1} + ((t_k - 1)/t_{k+1}) * (x_{k+1} - x_k)`
- 梯度与近端在 `y_k` 上计算，而不是直接在 `x_k` 上计算

直观上，`y_k` 把历史下降方向编码进当前迭代，从而减少达到同等精度所需的迭代次数。

## R05

理论收敛速度（目标函数误差）常见结论：

- ISTA：`O(1/k)`
- FISTA：`O(1/k^2)`

二者每步成本同阶（一次梯度 + 一次近端），因此 FISTA 通常在相同计算预算下达到更小目标函数值。

## R06

本实现的数值设定：

- 合成高维稀疏线性回归数据（`n=220, d=500, sparsity=28`）
- 高斯设计矩阵并按 `1/sqrt(n)` 缩放
- 稀疏真值向量 `x_true` + 小高斯噪声
- 正则强度 `lambda=0.01`

这样可以同时观察“收敛速度差异”和“稀疏恢复质量（support F1）”。

## R07

Lipschitz 常数采用：

`L = ||A||_2^2 / n`

在示例规模下用 SVD 直接计算谱范数，代码简单、稳定；再用步长 `1/L` 做固定步长迭代。对更大规模场景可替换为幂迭代或回溯线搜索。

## R08

FISTA 伪代码（与 `demo.py` 一致）：

```text
x0 = 0, y0 = x0, t0 = 1
for k = 0..K-1:
  x_{k+1} = prox_{g/L}( y_k - (1/L) * grad f(y_k) )
  t_{k+1} = (1 + sqrt(1 + 4 t_k^2))/2
  y_{k+1} = x_{k+1} + ((t_k-1)/t_{k+1}) * (x_{k+1} - x_k)
```

本实现还加了一个简单重启：若外推点目标值不降，则重置 `y=x, t=1`。

## R09

复杂度分析（每次迭代）：

- 梯度计算 `A^T(Ay-b)`：`O(nd)`
- 软阈值：`O(d)`
- 动量更新：`O(d)`

因此总复杂度由矩阵-向量乘主导，为 `O(nd)`；内存主要是 `A`（`O(nd)`）和若干长度 `d` 向量。

## R10

`demo.py` 不依赖黑箱优化器：

- 未调用 `scikit-learn` 的 `Lasso` 或 `cvxpy` 求解器
- 直接实现 `objective / grad_f / prox(soft_threshold) / ista / fista`

这样可以清晰展示“加速近端梯度法”本身，而不是封装后的 API 调用。

## R11

收敛对比策略：

- 先运行高精度长迭代 FISTA（`max_iter=5000`）作为参考最优值 `f_ref`
- 再用相同迭代上限（`400`）比较 ISTA 与 FISTA
- 用同一目标阈值 `target_obj = f_ref + 1e-3*(f(0)-f_ref)` 统计“首次达到目标迭代数”

该对比可直接量化加速收益，而不仅是看最终一个点。

## R12

评估指标：

- `final objective` 与 `objective gap to ref`
- 参数误差 `||x - x_true||_2`
- 稀疏支持集恢复 `support F1`
- `estimated nonzeros`
- `first iter <= target obj`

这些指标结合使用，可同时观察优化收敛与统计恢复表现。

## R14

预期现象：

- FISTA 在相同迭代预算下通常达到更小目标函数值
- 在“达到同一目标值”的指标上，FISTA 常显著少于 ISTA
- 两者都能恢复一定稀疏结构，但恢复质量受 `lambda`、噪声和样本规模影响

由于数据随机生成，数值会随随机种子变化，但整体趋势稳定。

## R15

常见问题与排查：

- 若出现震荡或偶发上升：可开启/保留重启策略
- 若收敛很慢：检查 `L` 是否低估（步长过大）
- 若解过稀或过密：调整 `lambda`
- 若支持集 F1 偏低：增加样本量、降低噪声或改进特征条件数

## R16

可扩展方向：

- 自适应重启（基于梯度映射或函数值判据）
- 回溯线搜索 FISTA（不显式依赖精确 `L`）
- 组稀疏/核范数等更复杂近端项
- 与坐标下降、ADMM、L-BFGS（仅光滑部分）进行系统对比

## R17

适用边界：

- 适用于 `f` 可微且 `g` 近端算子可高效计算的复合凸问题
- 若 `g` 近端无闭式且求解昂贵，单步成本会显著上升
- 若问题强非凸，标准 FISTA 理论保证不再直接成立

因此工程上应先确认问题结构是否匹配“可近端化”的假设。

## R18

`demo.py` 的源码级算法流程（9 步）：

1. `make_problem()` 生成稀疏线性回归数据 `(A, b, x_true)`，构造可控的复合凸优化测试集。  
2. `lipschitz_constant()` 用 SVD 计算 `L = ||A||_2^2 / n`，确定固定步长 `1/L`。  
3. `objective()` 定义 `F(x)=0.5/n||Ax-b||^2 + lambda||x||_1`，作为统一比较指标。  
4. `grad_f()` 实现光滑项梯度 `A^T(Ax-b)/n`，供 ISTA/FISTA 更新使用。  
5. `soft_threshold()` 实现 `L1` 近端算子，把梯度步结果做逐坐标收缩得到稀疏解。  
6. `fista(..., max_iter=5000)` 先求高精度参考解，得到 `f_ref` 作为近似最优目标值。  
7. `ista()` 与 `fista()` 在同一预算下分别迭代，记录每轮目标函数历史 `history`。  
8. `report_solver()` 计算并打印 `gap / ||x-x_true||_2 / support F1 / 达标迭代数`，形成结构化对照。  
9. `main()` 汇总速度提升（`speedup`）和最后 5 个目标值，验证“加速近端梯度法优于普通近端梯度”的核心结论。
