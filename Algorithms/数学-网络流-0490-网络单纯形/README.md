## R01

本条目实现“网络单纯形（Network Simplex）”的一个最小可运行版本：
- 问题类型：平衡运输问题（Transportation Problem）的最小费用流；
- 算法内核：树基可行解 + 势函数（对偶变量）+ 负检验数入基 + 回路换基；
- 目标：在 `demo.py` 中可直接运行并输出最优流量矩阵与目标值。

说明：运输问题是网络单纯形的经典特例，结构清晰，便于把“入基-成环-出基”的核心流程完整落地。

## R02

数学模型（最小化总运输成本）：

- 决策变量：`x_{ij} >= 0`，表示供应点 `i` 到需求点 `j` 的运输量。
- 目标函数：
  `min \sum_i \sum_j c_{ij} x_{ij}`
- 约束：
  - 行约束（供应守恒）：`\sum_j x_{ij} = s_i`
  - 列约束（需求满足）：`\sum_i x_{ij} = d_j`
  - 平衡条件：`\sum_i s_i = \sum_j d_j`

这对应一个二部网络上的最小费用可行流问题。

## R03

网络单纯形在本实现中的关键判据：

1. 基可行解：
- 基变量集合（`basis`）对应一棵生成树，大小为 `m + n - 1`。
- 初始基由西北角法生成，并处理退化情形。

2. 节点势（对偶变量）：
- 对基弧满足 `u_i + v_j = c_{ij}`。
- 通过 `_compute_potentials` 在树上传播求得 `u, v`。

3. 检验数（约化成本）：
- 对非基弧 `(i,j)`：`r_{ij} = c_{ij} - u_i - v_j`。
- 若存在 `r_{ij} < 0`，可通过入基继续降低目标函数。
- 若所有非基弧 `r_{ij} >= 0`，满足最优性条件。

## R04

单次换基（pivot）步骤：

1. 入基弧选择：
- 在全部非基弧中选取负检验数弧；实现采用 Bland 风格的字典序选择，减小循环风险。

2. 成环：
- 将入基弧加入树后会形成唯一回路。
- 在运输问题中用“行节点/列节点”的树路径恢复该回路。

3. 正负号分配：
- 入基弧标记为 `+`。
- 回路上其余弧按交替 `- + - + ...` 分配。

4. 步长与出基：
- `theta = min(所有 '-' 弧当前流量)`。
- `+` 弧加 `theta`，`-` 弧减 `theta`。
- 达到最小值的 `-` 弧中，按字典序选一条出基弧，保证基大小保持 `m+n-1`。

## R05

`demo.py` 的核心数据结构：

- `NetworkSimplexResult`：
  - `flow`：最终流量矩阵；
  - `objective`：目标函数值；
  - `iterations`：迭代轮数；
  - `optimal`：是否在 `max_iter` 内达到最优；
  - `basis_size`：最终基弧数量。
- `basis: set[(i,j)]`：当前基弧集合（树结构）。
- `flow: np.ndarray[m,n]`：每条弧的流量值。

## R06

正确性直观说明：

- 可行性保持：
  - 初始解由西北角法构造，满足行列守恒；
  - 每次回路换基在 `+/-` 交替上做同量增减，不破坏任一节点平衡。

- 最优性推进：
  - 负检验数入基意味着沿该方向可下降目标值；
  - 当不存在负检验数时，满足运输单纯形的对偶可行与互补条件，对应最优。

- 终止性：
  - 有限基组合 + Bland 风格选边策略，降低退化循环风险；
  - 同时提供 `max_iter` 上限作为工程保护。

## R07

复杂度（`m` 个供应点，`n` 个需求点，迭代次数 `K`）：

- 单次迭代：
  - 计算势：`O(m+n)` 到 `O(mn)`（取决于实现扫描方式）；
  - 检验数扫描：`O(mn)`；
  - 找回路与换基：`O(m+n)`。
- 总体：`O(K * mn)` 量级（本 MVP 以清晰实现为主，不做高阶数据结构优化）。

## R08

边界与异常处理：

- 成本矩阵维度、供需向量长度不匹配时抛出 `ValueError`。
- 供需出现负数时抛出 `ValueError`。
- 非平衡问题（`sum(supply) != sum(demand)`）抛出 `ValueError`。
- 初始基大小不为 `m+n-1` 或树不连通时抛出 `RuntimeError`。
- 迭代结束后会检查行/列残差，若超阈值抛出 `RuntimeError`。

## R09

MVP 取舍说明：

- 实现“运输问题版网络单纯形”，不覆盖一般有向图的上下界弧与超源超汇建模。
- 只依赖 `numpy`，避免调用现成最小费用流黑盒求解器。
- 重点保证：
  - 可运行；
  - 可读；
  - 核心换基逻辑完整（入基、成环、出基）。

## R10

函数职责映射：

- `_assert_balanced_transport`：输入合法性校验。
- `_northwest_corner_init`：构造初始基可行解（含退化处理）。
- `_compute_potentials`：在基弧方程上求节点势 `u,v`。
- `_choose_entering_arc`：按负检验数选择入基弧。
- `_basis_path_nodes` / `_path_edges_from_nodes`：恢复树路径与回路弧序列。
- `transportation_network_simplex`：主循环（定价、换基、终止判定）。
- `_run_case` / `main`：固定样例执行与结果打印。

## R18

源码级算法流程（对应 `demo.py`，8 步）：

1. `main` 构造成本矩阵与供需数据，调用 `_run_case` 执行样例。  
2. `transportation_network_simplex` 将输入转为 `numpy` 数组并做平衡性校验。  
3. `_northwest_corner_init` 生成初始可行流与 `m+n-1` 条基弧（处理退化零流基弧）。  
4. 每轮迭代先用 `_compute_potentials` 求基弧对偶势 `u,v`。  
5. `_choose_entering_arc` 扫描非基弧，选择一条负检验数弧作为入基弧；若不存在则判最优并终止。  
6. `_basis_path_nodes` + `_path_edges_from_nodes` 在当前基树中恢复回路，得到 `+/-` 交替弧集合。  
7. 计算 `theta`，对 `+` 弧加流、对 `-` 弧减流，并选一条达到下界的 `-` 弧出基完成换基。  
8. 循环结束后校验行列守恒，封装 `NetworkSimplexResult` 并输出目标值与流量矩阵。
