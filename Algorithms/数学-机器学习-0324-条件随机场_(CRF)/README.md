## R01

条件随机场（Conditional Random Field, CRF）是判别式概率图模型，用于建模条件分布 `P(y|x)`，常见于序列标注任务（如分词、词性标注、命名实体识别）。

本目录实现的是一阶线性链 CRF：
- 标签链为 `y_1,...,y_T`；
- 观测序列为 `x_1,...,x_T`；
- 条件概率由“路径得分 + 配分函数归一化”定义。

核心形式：
- `s(x,y) = start(y_1) + sum_t emit(y_t, x_t) + sum_{t>1} trans(y_{t-1}, y_t) + end(y_T)`
- `P(y|x) = exp(s(x,y)) / Z(x)`，其中 `Z(x)=sum_{y'} exp(s(x,y'))`

## R02

CRF 由 John Lafferty、Andrew McCallum、Fernando Pereira 在 2001 年系统提出，用于解决序列标注中“全局归一化”和“特征灵活表达”的问题。

相对生成式模型（如 HMM），CRF 直接学习 `P(y|x)`，因此：
- 可使用更丰富的观测特征；
- 缓解局部归一化带来的标签偏置问题；
- 在结构化预测任务中兼顾概率解释与预测性能。

## R03

CRF 主要解决的问题：
- 标签之间存在强依赖（例如 BIO 标签约束）时，逐位置独立分类会产生不一致预测；
- 只看局部概率会忽略整条路径的全局最优。

线性链 CRF 的目标是：
1. 训练时最大化条件对数似然 `sum log P(y|x)`；
2. 推理时用 Viterbi 找最优标签路径 `argmax_y s(x,y)`；
3. 用前向-后向算法高效计算 `Z(x)` 与边缘概率。

## R04

本实现算法流程：
1. 用 `forward_log` 在 log-space 递推前向量 `alpha`，得到 `log Z(x)`。
2. 用 `backward_log` 递推后向量 `beta`。
3. 由 `alpha/beta/logZ` 计算节点边缘 `gamma_t(k)=P(y_t=k|x)` 与边边缘 `xi_t(i,j)`。
4. 梯度采用“经验计数 - 模型期望计数”：
   - 起止标签、转移特征、发射特征都这样构造。
5. 对整批序列做全批次梯度上升，并加 L2 正则项。
6. 预测阶段使用 `viterbi_decode` 输出最优标签路径。

## R05

设：
- 序列长度 `T`，标签数 `K`，观测字典大小 `V`。

单条序列的主要复杂度：
- 前向：`O(T*K^2)`
- 后向：`O(T*K^2)`
- 边缘 `xi`：`O(T*K^2)`
- Viterbi：`O(T*K^2)`

若训练集有 `N` 条序列，则每个 epoch 约 `O(N*T*K^2)`；
参数空间约 `O(K + K + K^2 + K*V)`（start/end/trans/emit）。

## R06

一个极简直觉例子（`K=2`，`T=3`）：
- 若局部发射更偏向标签 `A`，但转移 `A->A` 很弱、`A->B` 很强，
- 则 Viterbi 可能输出路径 `A,B,B` 而不是逐点独立分类的 `A,A,A`。

这体现了 CRF 的本质：
- 不是仅看每个位置“谁分高”；
- 而是联合考虑整条路径总分 `s(x,y)` 的全局最优。

## R07

优点：
- 对序列结构建模自然，能显式编码转移约束；
- 概率形式清晰，可解释边缘概率；
- 在结构化标注中通常比独立分类器更稳健。

局限：
- 训练需要重复做前向-后向，计算开销高于独立分类；
- 线性链只建模一阶邻接，长程依赖表达受限；
- 若特征非常高维且数据不足，容易过拟合（需正则与验证）。

## R08

前置知识建议：
- 概率论：条件概率、对数似然、归一化常数；
- 动态规划：前向递推、后向递推、Viterbi 回溯；
- 数值计算：`log-sum-exp` 稳定技巧；
- 机器学习：梯度上升/下降、L2 正则。

## R09

适用场景：
- NLP 序列标注（分词、词性、NER）；
- 需要标签间约束的结构化预测任务；
- 希望同时得到最优路径和边缘置信度。

不太适用：
- 长程依赖主导但只能用一阶标签链时；
- 超大标签空间、超长序列且实时性要求极高时；
- 端到端深度模型已内含更强上下文表示且无需显式结构层时。

## R10

正确性检查点（`demo.py` 已覆盖）：
1. `gamma` 每个时刻行和应为 1。
2. `xi` 每个时刻张量总和应为 1。
3. 边缘一致性：
   - `sum_j xi_t(i,j) = gamma_t(i)`
   - `sum_i xi_t(i,j) = gamma_{t+1}(j)`
4. 训练后平均 NLL 应下降。
5. 测试集 token accuracy 达到合理阈值（脚本中设为 `> 0.78`）。

## R11

数值稳定性策略：
- 所有前向/后向递推都在 log-domain 完成；
- 使用自实现 `logsumexp`，避免直接 `exp` 后求和溢出；
- 概率归一化处加入极小量 `1e-300` 防止 `log(0)`；
- 训练步长使用轻微衰减，减少后期振荡。

## R12

工程成本与调参建议：
- `lr`（学习率）是最敏感参数，过大易震荡，过小收敛慢；
- `l2` 控制泛化与参数幅度，数据少时建议适当增大；
- `epochs` 建议先看 NLL 曲线，再决定是否继续训练；
- `K`（标签数）增加会导致 `K^2` 成本上升，应评估实时预算。

## R13

理论说明：
- 线性链 CRF 在给定参数下的推断（前向-后向、Viterbi）是精确动态规划，不是近似采样；
- 这里没有“近似比”概念；
- 算法本身是确定性的（固定数据与初始化后结果可复现），随机性仅来自合成数据采样和初始化随机种子。

## R14

常见失效模式与防护：
- 失效：训练 NLL 不降。
  - 防护：降低 `lr`，检查梯度符号（本实现使用梯度上升）。
- 失效：边缘概率不归一。
  - 防护：优先检查 `logZ`、`alpha/beta` 递推与 `exp` 归一化步骤。
- 失效：过拟合导致测试精度下滑。
  - 防护：增大 `l2`、减少 epoch、扩大训练集。
- 失效：序列极长时耗时高。
  - 防护：裁剪长度、批处理并行、或采用更高效后端。

## R15

实践建议：
- 先在小型可控数据上验证“公式正确 + 指标可提升”，再接入真实数据；
- 保留概率一致性断言，避免后续重构引入隐蔽错误；
- 训练日志至少输出 `initial/middle/final NLL` 与测试精度；
- 若接入真实文本特征，可先保留相同 CRF 核心，只替换发射特征构造。

## R16

相关算法关系：
- HMM：生成式 `P(x,y)`，CRF：判别式 `P(y|x)`；
- MEMM：局部归一化，可能有 label bias；CRF 全局归一化；
- 前向-后向：在 CRF 中用于 `logZ` 和边缘期望；
- Viterbi：在 CRF 中用于最优路径解码。

## R17

本目录 MVP（`demo.py`）做了什么：
- 用 `numpy` 从零实现一阶线性链 CRF，不调用第三方 CRF 黑盒；
- 生成可复现实验数据（3 个标签、6 个观测符号）；
- 训练：全批次条件对数似然梯度上升 + L2；
- 推理：Viterbi 解码；
- 验证：边缘归一化/一致性断言 + NLL 下降 + 测试精度阈值。

## R18

`demo.py` 的源码级流程（8 步）：
1. `make_synthetic_sequence_dataset` 采样带标签的序列数据（马尔可夫标签转移 + 标签条件发射）。
2. `train_crf` 初始化 `start/end/trans/emit` 参数，进入 epoch 训练循环。
3. 对每条样本调用 `sequence_gradients`：先算 `forward_backward` 得到 `alpha/beta/logZ`。
4. `marginals` 用 `alpha/beta/logZ` 计算 `gamma` 与 `xi`，得到模型期望特征计数。
5. 在 `sequence_gradients` 中用“经验计数 - 期望计数”形成单样本梯度，并累积对数似然。
6. `train_crf` 对全批梯度做均值、叠加 L2 梯度修正，用学习率衰减执行参数更新。
7. 训练后 `run_consistency_checks` 验证 `gamma/xi` 归一化及边缘恒等式，确保推断实现正确。
8. `main` 调用 `viterbi_decode` 评估 token accuracy 与 exact-match，打印 NLL 曲线和样例预测并执行阈值断言。

上述步骤完整覆盖了 CRF 的训练、推断、解码与数值正确性校验闭环。
