# 割线法 (Secant Method)

## R01

割线法用于求解一元非线性方程 `f(x)=0` 的近似根，是一种不显式使用导数的开型迭代法。

- 输入:
  - 连续函数 `f(x)`（至少在根附近可计算）；
  - 两个初始点 `x0, x1`（通常取在目标根附近且 `f(x0) != f(x1)`）。
- 输出:
  - 近似根 `x*`；
  - 迭代信息（是否收敛、迭代次数、残差 `|f(x*)|`）。
- 迭代公式:
  - `x_{k+1} = x_k - f(x_k) * (x_k - x_{k-1}) / (f(x_k) - f(x_{k-1}))`
- 直观解释:
  - 用两点 `(x_{k-1}, f(x_{k-1}))` 与 `(x_k, f(x_k))` 连成割线；
  - 取该割线与 `x` 轴的交点作为下一次迭代值。

## R02

割线法属于经典求根方法族，历史上与牛顿法、弦截法（Regula Falsi）在 17-18 世纪逐步形成并系统化。其核心思想是“用差商近似导数”，可看作牛顿法
`x_{k+1}=x_k-f(x_k)/f'(x_k)` 中 `f'(x_k)` 的数值替代。

## R03

提出动机是解决“导数难求或代价高”的求根问题。

- 牛顿法收敛快，但需要可用且稳定的导数。
- 在工程模型中，`f(x)` 常来自仿真器、查表或黑盒计算，导数不可直接获得。
- 割线法仅依赖函数值，不需要解析导数与自动微分，就能在很多场景达到接近牛顿法的效率。

## R04

时间复杂度通常按“迭代次数 `k` 与单步函数计算成本 `Cf`”衡量。

- 单次迭代成本: `O(Cf)`（1 次新函数值 + 常数次标量运算）。
- 总成本: `O(k * Cf)`。
- 局部收敛阶:
  - 对简单根（`f'(r) != 0`）且初值足够接近时，误差满足超线性收敛，阶约为黄金分割数
    `phi ≈ 1.618`；
  - 通常慢于牛顿法（二阶），快于线性收敛方法（如固定点线性迭代）。

## R05

- 原地迭代实现只需维护最近两点及函数值，额外空间 `O(1)`。
- 若保存完整迭代轨迹用于分析/可视化，则空间 `O(k)`。
- 本目录 `demo.py` 默认保存历史记录以便打印与估算收敛阶。

## R06

示例：求 `x^2-2=0` 的正根，取 `x0=1, x1=2`。

1. `x2 = 1.3333333333`
2. `x3 = 1.4000000000`
3. `x4 = 1.4146341463`
4. `x5 = 1.4142114385`
5. `x6 = 1.4142135621`
6. `x7 = 1.4142135624`

结果快速逼近 `sqrt(2) ≈ 1.4142135623730951`，对应残差接近机器精度。

## R07

割线法的价值在于“低门槛 + 高效率折中”。

- 理论层面: 展示了用差商替代导数后仍可保持超线性收敛。
- 工程层面: 只要能评估 `f(x)` 就可直接应用，适合黑盒函数求根。
- 实践层面: 常作为牛顿法不可用时的首选替代，或与二分法组合成保守-快速混合法。

## R08

算法依赖的数学基础。

- 连续函数与零点存在性（用于理解初值区间的可行性）。
- 线性插值/差商思想（割线就是两点线性近似）。
- 误差递推分析（局部收敛阶结论）。
- 与牛顿法关系:
  - 牛顿法用导数；
  - 割线法用 `f'(x_k) ≈ (f(x_k)-f(x_{k-1}))/(x_k-x_{k-1})`。

## R09

适用前提与边界。

- 函数在迭代路径上应可计算且尽量平滑。
- 初值需避免 `f(x0) == f(x1)`（分母为 0）。
- 对多重根或平坦区域，收敛可能明显变慢。
- 割线法是开型方法，不像二分法那样天然保区间，可能跳出期望范围甚至发散。

## R10

正确性依据（简述）。

1. 每步用 `(x_{k-1}, f_{k-1})` 与 `(x_k, f_k)` 构造线性近似 `L_k(x)`。
2. `x_{k+1}` 被定义为 `L_k(x)=0` 的解，因此是“局部线性模型”的零点。
3. 当 `x_k` 接近简单根 `r` 且 `f` 足够光滑时，差商逼近导数，割线迭代逼近牛顿迭代。
4. 在标准局部条件下可证明收敛到 `r`，并具有超线性收敛阶 `phi`。

## R11

数值稳定性与误差传播要点。

- 主要风险不是舍入噪声本身，而是分母 `f(x_k)-f(x_{k-1})` 过小导致步长爆炸。
- 当函数值非常接近时，会出现“差分消去”，放大浮点误差。
- 常见缓解手段:
  - 设置 `tiny` 阈值，分母过小时提前终止并报告失败；
  - 增加步长/区间保护（如退回二分法）；
  - 同时使用 `|f(x)|` 与 `|dx|` 双重停止准则。

## R12

真实计算成本（超越渐近记号）。

- 真正主项通常是函数评估 `f(x)`，而非公式本身的加减乘除。
- 若 `f(x)` 来自 PDE/蒙特卡洛/复杂仿真，单步代价会非常高，此时减少函数评估次数最关键。
- 相比牛顿法:
  - 省去导数评估（或数值差分导数）成本；
  - 但可能需要更多迭代。
- 因此实际效率取决于“导数成本 vs 额外迭代次数”的权衡。

## R13

收敛与误差性质。

- 不是全局收敛算法，无法给出“任意初值必收敛”的统一保证。
- 局部理论（简单根附近）:
  - 误差满足 `e_{k+1} ≈ C * e_k * e_{k-1}`；
  - 渐近收敛阶 `p = (1 + sqrt(5)) / 2 ≈ 1.618`。
- 对重根（`f'(r)=0`）或病态函数，收敛阶会下降，甚至可能停滞/失败。

## R14

鲁棒性与失效模式。

- 分母塌陷: `f_k - f_{k-1} ≈ 0` 导致数值不稳定。
- 初值不当: 初始点离目标根过远，可能振荡或发散。
- 多根竞争: 迭代可能跳向非预期根。
- 非光滑函数: 局部线性近似质量差，收敛性变差。
- 防护建议:
  - 设置 `max_iter`、`tiny` 与双停止准则；
  - 失败时切换到区间法（如二分法）做兜底。

## R15

工程实现建议。

- API 至少返回: `root / converged / iterations / residual / reason`。
- 建议保留 `history`，便于诊断“分母塌陷、步长爆炸、慢收敛”。
- 在生产中常用“割线 + 二分”混合策略：
  - 能走割线就快速推进；
  - 不安全时回退到区间收缩保证稳定。
- 测试覆盖:
  - 正常简单根；
  - 多重根慢收敛；
  - 分母为零失败分支；
  - 不同量级初值。

## R16

前驱/后继关系与应用。

- 前驱基础:
  - 二分法（保守但稳定）；
  - 牛顿法（快但依赖导数）。
- 后继/相关方法:
  - Regula Falsi/Illinois/Pegasus（带区间约束的割线变体）；
  - Brent-Dekker（工程常用鲁棒求根器，结合二分、割线与反二次插值）。
- 典型应用:
  - 控制参数整定中的标量方程求解；
  - 物理/化学模型的平衡点求解；
  - 金融定价中隐含参数反解。

## R18

源码级流程拆解（对应本目录 `demo.py`，非第三方黑箱）。

1. 初始化 `x0, x1` 与函数值 `f0, f1`，检查是否为有限数。
2. 进入迭代循环，先计算分母 `denom = f1 - f0`。
3. 若 `|denom| <= tiny`，立即返回失败结果（防止除零与爆步长）。
4. 按割线公式计算新点 `x2 = x1 - f1*(x1-x0)/denom`。
5. 计算 `f2 = f(x2)`，记录步长 `|x2-x1|` 与完整迭代日志。
6. 以“双停止准则”判断收敛：`|f2| <= tol_f` 或 `|x2-x1| <= tol_x*(1+|x2|)`。
7. 若未收敛，状态平移为 `(x0,f0) <- (x1,f1)`、`(x1,f1) <- (x2,f2)`，继续下一轮。
8. 达到 `max_iter` 仍未满足条件时返回非收敛结果，并附带最后残差与原因。

该流程完整体现了割线法的数学机制、工程防护与可观测性设计。
