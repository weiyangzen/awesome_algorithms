## R01

加权法（Weighted Sum Method）是多目标优化里最常用、最易工程化的标量化方案之一。

核心思想是把多个目标函数按权重线性组合成一个单目标：

`J_w(x) = w1*f1(x) + w2*f2(x) + ... + wm*fm(x)`

其中 `wi >= 0` 且 `sum(wi)=1`。改变权重向量 `w`，就能得到不同偏好的折中解，并近似 Pareto 前沿。

## R02

在双目标最小化 `min (f1(x), f2(x))` 中，通常不存在“两个目标同时最优”的单点解。

- 只优化 `f1` 会牺牲 `f2`；
- 只优化 `f2` 会牺牲 `f1`；
- 真实需求往往是“按偏好权衡”。

加权法把“偏好”显式参数化为权重，便于做灵敏度分析和业务解释。

## R03

本 MVP 的任务定义：

- 决策变量：`x in [0, 2]`；
- 双目标（均最小化）：
  `f1(x) = x^2`，
  `f2(x) = (x-2)^2`；
- 加权子问题：
  `min_x J_w(x) = w1*f1(x) + w2*f2(x)`，
  `s.t. x in [0,2], w1+w2=1`。

该构造具备清晰冲突关系：当 `x` 增大时，`f1` 变差而 `f2` 变好。

## R04

数学形式：

- 原问题：`min_x (f1(x), f2(x)), x in X`
- 加权标量化：
  `min_x J_w(x) = sum_i wi*fi(x)`
  `s.t. x in X, wi>=0, sum_i wi=1`

本例 `X=[0,2]` 且目标为凸函数，因此每个权重子问题是 1 维凸优化，可由数值方法稳定求解。

## R05

高层流程：

1. 定义双目标问题与变量边界；
2. 生成一组权重 `w1 in [0,1]`（`w2=1-w1`）；
3. 对每个权重构造加权目标 `J_w(x)`；
4. 用有界一维优化求解 `argmin J_w(x)`；
5. 记录 `x, f1, f2, J_w, nfev`；
6. 对结果做非支配筛选与一致性校验；
7. 输出完整扫描表和 Pareto 近似点。

## R06

默认参数设置：

- 权重网格：`w1 = 0, 1/16, ..., 1`（共 17 点）；
- 求解器：`scipy.optimize.minimize_scalar(method="bounded")`；
- 数值容差：`xatol=1e-10`；
- 输出内容：全量权重扫描结果、非支配点、两个锚点（`w1=0` 与 `w1=1`）。

## R07

复杂度分析（设权重点数为 `K`）：

- 外层扫描：`K` 次子问题；
- 每次子问题：1 维有界标量优化，迭代步数记为 `T`；
- 总体复杂度：`O(K*T)`；
- 非支配筛选（朴素实现）：`O(K^2)`。

本例规模很小，计算成本可忽略，重点在算法可解释性。

## R08

数值与工程细节：

- 权重检查：要求 `w1,w2 >= 0` 且 `w1+w2=1`；
- 明确边界：所有求解都限制在 `[0,2]`，避免无界行为；
- 保存求解状态：包括 `success`、`nit`、`nfev` 与消息；
- 使用容差断言（`1e-7 ~ 1e-6`）处理浮点误差；
- 同时给出解析解 `x*=2*(1-w1)` 作为 sanity check。

## R09

结果解读：

- `w1` 越大，越重视 `f1`，解会向 `x=0` 侧移动；
- `w2` 越大，越重视 `f2`，解会向 `x=2` 侧移动；
- 在本例中，权重扫描会给出一条平滑折中曲线。

这展示了加权法的优势：偏好可调、实现简单、输出直观。

## R10

`demo.py` 模块职责：

- `WeightedProblem1D`：问题数据容器（区间、目标函数）；
- `build_problem`：构造示例问题；
- `solve_weighted_subproblem`：求单个权重下的最优解；
- `weight_sweep`：执行权重扫描并形成表格；
- `pareto_filter_min`：非支配点筛选；
- `validate_results`：执行完整门禁校验；
- `main`：运行、打印、汇总锚点。

## R12

输出字段说明：

- `w1, w2`：两个目标的权重（和为 1）；
- `x_star`：该权重下最优决策；
- `f1, f2`：对应双目标值；
- `weighted_obj`：加权目标最优值 `J_w(x_star)`；
- `x_closed_form`：解析解（用于交叉校验）；
- `abs_err_closed_form`：数值解与解析解绝对误差；
- `status`：求解状态（期望全为 `optimal`）；
- `nit, nfev`：优化迭代/函数评估次数。

## R13

正确性门禁（代码内已实现）：

1. 结果表不能为空；
2. 所有权重子问题必须求解成功；
3. `w1+w2` 必须近似等于 1；
4. `x_star` 必须位于 `[0,2]`；
5. 数值解需与解析解 `x*=2*(1-w1)` 高精度一致；
6. 随 `w1` 增大，`x_star` 与 `f1` 应单调不增，`f2` 单调不减；
7. 非支配点数量必须大于 0。

## R14

常见失效模式与对策：

- 失效 1：权重未归一化（`w1+w2!=1`）。
  对策：统一做归一化或直接拒绝非法输入。

- 失效 2：权重含负值导致偏好失真。
  对策：强制 `wi>=0` 并在入口断言。

- 失效 3：完全依赖黑箱结果，无可解释检查。
  对策：增加解析校验、单调性检查、状态字段输出。

- 失效 4：误认为加权法能覆盖所有 Pareto 点。
  对策：说明其主要覆盖“支撑型（supported）”前沿点，非凸前沿可能漏解。

## R15

实践建议：

- 先做目标量纲归一化，再设权重；
- 用“粗网格扫描 + 局部加密”提高效率；
- 业务汇报时把权重翻译为偏好语言（如“成本权重 70%”）；
- 保留锚点解与中间解，便于做策略对比。

## R16

与其他多目标方法对比：

- 加权法：实现最简单、便于接入现有单目标求解器；
- ε-约束法：更容易覆盖复杂前沿结构，但要处理可行性与阈值设计；
- 目标规划/理想点法：偏好表达更灵活，但参数解释成本更高。

本任务选择加权法，是为了给出最小可运行、最容易落地的多目标优化入口。

## R17

可扩展方向：

- 从 1 维变量扩展到高维约束优化（线性/非线性）；
- 加入目标标准化、自动权重采样和结果可视化；
- 对非凸场景引入 ε-约束法或进化算法（如 NSGA-II）补点；
- 在工业场景加入不确定性（鲁棒/随机优化）处理。

## R18

`demo.py` 源码级算法流（8 步，包含第三方求解链路）：

1. `build_problem` 定义区间 `[0,2]` 与目标函数 `f1,f2`，形成 `WeightedProblem1D`。  
2. `weight_sweep` 生成权重网格 `w1 in [0,1]`，并计算 `w2=1-w1`。  
3. 每个权重调用 `solve_weighted_subproblem`，先校验 `w1,w2` 的非负与归一性。  
4. 在子问题中构造闭包目标 `phi(x)=w1*f1(x)+w2*f2(x)`，并调用 `scipy.optimize.minimize_scalar(method="bounded")`。  
5. SciPy 的 `bounded` 分支进入有界 Brent 流程：先在区间内维护候选点，再在“抛物线插值步”和“黄金分割步”之间自适应切换，逐步收缩区间直到满足 `xatol`。  
6. 求解完成后返回 `x_star`、`fun`、`nit`、`nfev`、`success`，脚本再计算 `f1,f2` 与解析解 `x*=2*(1-w1)` 误差。  
7. `validate_results` 执行门禁：成功率、边界合法性、单调性、解析一致性。  
8. `pareto_filter_min` 做非支配筛选，`main` 打印全表、Pareto 近似点与锚点。  

这 8 步形成了“权重参数化 -> 单目标求解 -> 结果验证 -> Pareto 提取”的完整可追踪实现链路。
