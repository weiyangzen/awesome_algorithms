## R01

指数积分法（Exponential Integrators）用于求解如下“线性刚性 + 非线性”常微分方程：

\[
u'(t)=Au(t)+g(u(t),t),
\]

其中 `A` 通常是刚性的线性算子。核心思想是把线性部分精确积分到指数传播子里，减轻刚性约束。  
本目录 MVP 采用最基础的 ETD Euler（指数欧拉）：

\[
u_{n+1}=e^{hA}u_n+h\varphi_1(hA)g(u_n,t_n),\quad \varphi_1(z)=\frac{e^z-1}{z}.
\]

## R02

指数积分法的现代系统化发展主要在 1990s-2000s（Kassam, Trefethen, Hochbruck, Ostermann 等工作影响较大）。  
背景是刚性 ODE/PDE 时间推进中，传统显式法受稳定性限制、隐式法需要昂贵非线性求解，指数法在两者之间给出了一条工程上很实用的路线。

## R03

动机是“线性刚性项导致显式步长极小”：

- 显式 Euler / RK：稳定域有限，步长受最左特征值限制，刚性下很慢。
- 全隐式方法：稳定但每步需解（可能非线性）方程，复杂度和实现成本高。
- 指数积分法：把 `A` 的作用通过 `exp(hA)` 吸收，非线性仅显式处理，兼顾稳定性和实现简洁性。

## R04

以本目录 ETD Euler 的稠密矩阵实现为例（状态维数 `n`，步数 `N_t`）：

- 预计算 `exp(hA)` 与 `h*phi1(hA)`：一次 `O(n^3)`。
- 每个时间步两次矩阵向量乘（`E@u` 与 `H@g(u)`）：`O(n^2)`。
- 总复杂度约 `O(n^3 + N_t n^2)`。

若利用稀疏结构或 Krylov 近似，可进一步把每步成本降到更低。

## R05

空间复杂度（当前实现）：

- 存储 `A, E, H` 三个 `n x n` 矩阵，主导为 `O(n^2)`。
- 向量状态与临时量为 `O(n)`。

因此总体空间复杂度为 `O(n^2)`。

## R06

示例模型：1D Allen-Cahn 方程半离散后得到

\[
u'(t)=Au-u^3,\quad A=\varepsilon L+I,
\]

`L` 为二阶差分 Laplacian（Dirichlet 边界）。  
本目录 `demo.py` 设 `n=60, eps=0.01, T=1.0`，比较：

1. ETD Euler（目标算法）
2. 显式 Euler（基线）
3. Radau 高精度参考解（仅用于验算）

## R07

算法意义：

- 对刚性问题可用更大步长而不立即失稳。
- 避免每步求解大型非线性方程（相对全隐式更轻量）。
- 在半离散 PDE（反应扩散、薛定谔、KdV 变体等）中非常常见。

## R08

前置知识与依赖：

- 线性代数：矩阵指数 `exp(hA)`、矩阵向量乘。
- 数值时间积分：显式 Euler 与稳定性概念。
- 刚性 ODE 基础认知。
- 代码依赖：`numpy`、`scipy.linalg.expm`、`scipy.integrate.solve_ivp`。

## R09

适用范围与边界：

- 适用：可写成 `u'=Au+g(u,t)` 且 `A` 线性刚性主导的系统。
- 不适用：若系统难以拆分线性/非线性结构，或 `A` 随时间剧烈变化且难处理，直接 ETD Euler 效果有限。
- 风险边界：
  - 非线性项极强时，低阶 ETD Euler 精度可能不足。
  - 大规模系统若直接稠密 `expm`，会有 `O(n^3)` 代价和内存压力。

## R10

正确性直觉（针对 ETD Euler）：

1. 变参数公式给出精确关系：
\[
u(t+h)=e^{hA}u(t)+\int_0^h e^{(h-\tau)A}g(u(t+\tau),t+\tau)\,d\tau.
\]
2. 用 `g(u(t),t)` 近似积分核中的非线性，得到
\[
u_{n+1}=e^{hA}u_n+h\varphi_1(hA)g(u_n,t_n).
\]
3. 当 `h -> 0` 时该近似一致且是一阶时间精度方法。

因此在步长充分小且问题满足光滑性时，数值解收敛到真解。

## R11

稳定性要点：

- 线性刚性部分被指数传播子精确处理，比纯显式法稳健。
- 但整体仍受非线性显式处理影响，不是“无条件全局稳定”。
- 实务上常用 ETD2/ETDRK4 提升精度与鲁棒性；本目录刻意保留 ETD Euler 作为最小可解释实现。

## R12

实践性能瓶颈：

- 主要前置成本在 `expm`（本实现通过块矩阵指数一次性获得 `E` 和 `H`）。
- 时间循环本身主要是 BLAS 级矩阵向量乘。
- 当 `n` 中小规模时，MVP 方案简单可靠；当 `n` 大时建议改为 Krylov `expmv` 思路。

## R13

保证类型说明：

- 本算法不是随机算法，不涉及概率成功界。
- 对当前 ETD Euler，可给出“在常规光滑条件下的一阶收敛”。
- 代码输出中的误差对照是经验验证，不是形式化误差上界证明。

## R14

常见失效模式：

- 步长过大：非线性显式项导致精度劣化，甚至出现非物理振荡。
- 稠密指数代价过高：高维时运行时间和内存迅速上升。
- 参考解误用：若把外部求解器当主算法，就失去“算法可解释性”；本目录避免该问题，Radau 仅作验算。

## R15

工程建议：

- 中小规模先用 ETD Euler 快速打通，再升级到 ETD2/ETDRK4。
- 线性算子固定时，应预计算并复用指数相关矩阵。
- 必做数值体检：残差/误差对照、步长收敛测试、有限性检查（`isfinite`）。

## R16

关联关系：

- 前驱方法：显式 Euler、隐式 Euler、IMEX 分裂法。
- 同类方法：Lawson 方法、Magnus 展开、积分因子法。
- 后续增强：Krylov 子空间近似矩阵函数、ETDRK4、指数 Rosenbrock 方法。

## R17

`demo.py` 实现说明：

- 问题：`u' = A u - u^3`（由 1D Allen-Cahn 半离散得到）。
- 主算法：`etd_euler`，核心更新 `u <- E@u + H@(-u^3)`。
- 关键预计算：`precompute_etd_matrices` 用块矩阵指数同时求 `E=exp(hA)` 与 `H=h*phi1(hA)`。
- 验证：
  - 与 `solve_ivp(method="Radau")` 对比相对 `L2` 误差。
  - 与显式 Euler 对比稳定性。
  - 进行步长收敛表（`dt` 减半）检验一阶趋势。

运行方式：

```bash
uv run python demo.py
```

## R18

源码级算法流程（对应 `demo.py`，8 步）：

1. `build_linear_operator` 构建离散 Laplacian，并形成线性刚性算子 `A=eps*L+I`。  
2. `precompute_etd_matrices` 组装块矩阵  
   \[
   B=\begin{bmatrix}hA&hI\\0&0\end{bmatrix},
   \]
   然后调用 `scipy.linalg.expm(B)`。  
3. 从 `expm(B)` 的左上块抽取 `E=exp(hA)`，右上块抽取 `H=h*phi1(hA)`，避免手工求逆。  
4. `etd_euler` 时间推进：每步计算非线性 `g(u)=-u^3`。  
5. 执行更新 `u_{n+1}=E u_n + H g(u_n)`，完成指数积分一步。  
6. `reference_solution` 用 `solve_ivp(..., method="Radau")` 求高精度参考末态，仅用于误差评估。  
7. `relative_l2_error` 计算 ETD 与显式 Euler 相对参考解的误差，并检查数值是否有限。  
8. `convergence_check` 用更小步长 ETD 作为内部参考，输出 `dt` 递减时的误差表，验证一阶收敛趋势。  

补充（非黑箱说明）：`scipy.linalg.expm` 内部采用“缩放-平方 + Padé 逼近”路线来近似矩阵指数，本实现只把它作为底层线性代数模块，时间推进逻辑与非线性处理均在本地代码中显式实现。
