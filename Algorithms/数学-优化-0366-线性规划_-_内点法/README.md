## R01

线性规划内点法（Interior-Point Method for LP）是一类通过“始终在可行域内部迭代”逼近最优边界点的优化方法。  
与单纯形法沿多面体顶点移动不同，内点法在每次迭代里求解一个带障碍项（barrier）的平滑子问题，通过牛顿方向推进并逐步减弱障碍强度。

本目录给出一个可运行、可审计的最小 MVP：
- 问题采用标准型 `min c^T x, s.t. A x = b, x > 0`；
- 使用对数障碍 + 牛顿法构成路径跟随（path-following）；
- 输出每轮外层障碍参数、对偶间隙近似和目标函数值。

## R02

本实现求解的数学模型：
- 原始线性规划标准型：
  - `min c^T x`
  - `s.t. A x = b, x >= 0`
- 内点阶段使用严格可行域：
  - `x > 0`

对应障碍子问题（给定 `t > 0`）：
- `min f_t(x) = t * c^T x - sum(log(x_i))`
- `s.t. A x = b`

其中 `1/t` 控制障碍强度；`t` 越大，解越接近原 LP 的边界最优点。

## R03

核心数学关系：

1. 障碍子问题的梯度与 Hessian：
   - `grad f_t(x) = t c - 1/x`
   - `H(x) = diag(1/x^2)`
2. 含等式约束的牛顿 KKT 线性系统：
   - `[H  A^T] [dx ] = -[r_dual ]`
   - `[A   0 ] [dnu]   [r_primal]`
   - 其中 `r_dual = t c - 1/x + A^T nu`，`r_primal = A x - b`
3. 路径跟随停止判据（对偶间隙近似）：
   - `gap ~= n / t`（`n` 为变量维度）
   - 当 `n/t < tol` 时认为外层收敛。

## R04

算法流程（高层）：
1. 准备严格可行初值 `x0`，满足 `A x0 = b` 且 `x0 > 0`。  
2. 初始化障碍参数 `t = 1`，设放大系数 `mu > 1`。  
3. 固定当前 `t`，解一次障碍子问题（内层牛顿迭代）。  
4. 内层每轮构造 KKT 系统并求解 `(dx, dnu)`。  
5. 通过回溯线搜索保证 `x + alpha * dx > 0` 且残差下降。  
6. 内层收敛后更新外层统计，计算 `gap = n/t`。  
7. 若 `gap < tol` 则停止；否则令 `t <- mu * t`。  
8. 返回近似最优解、轨迹与终止信息。

## R05

核心数据结构（`demo.py`）：
- `OuterIterRecord = (outer_iter, t, objective, gap_est, inner_iters, min_x)`：
  - `outer_iter`：外层路径跟随迭代编号；
  - `t`：当前障碍参数；
  - `objective`：当前原始目标 `c^T x`；
  - `gap_est`：近似对偶间隙 `n/t`；
  - `inner_iters`：当前外层对应的牛顿步次数；
  - `min_x`：当前最小分量，观察是否逼近边界。
- `history: list[OuterIterRecord]`：完整外层收敛轨迹。
- `LPModel`（字典）：存放 `A, b, c, x0, description` 的固定样例配置。

## R06

正确性要点：
- 可行性保持：起点严格可行时，KKT 步配合线搜索可维持 `x > 0` 且控制约束残差。  
- 下降性：内层回溯线搜索强制 KKT 残差下降，避免牛顿步发散。  
- 渐近逼近边界：`t` 递增会减弱障碍项，解沿中心路径逼近 LP 最优边界点。  
- 可验证性：脚本附带一个小规模 LP，并与 `scipy.optimize.linprog`（HiGHS）做结果对照（若 SciPy 可用）。  
- 终止合理：外层使用 `n/t < tol`，内层使用残差范数阈值与最大迭代数双条件。

## R07

复杂度分析（`A` 稠密、`A in R^(m*n)`）：
- 单次牛顿步需解尺寸 `(n+m) x (n+m)` 的线性系统，代价约 `O((n+m)^3)`。  
- 若每个外层迭代内层牛顿步数为 `K`，外层迭代数为 `T`，则总复杂度约：
  - `O(T * K * (n+m)^3)`。  
- 空间复杂度：
  - 主要存储 KKT 矩阵 `O((n+m)^2)`；
  - 轨迹存储 `O(T)`。

在中小规模稠密问题中该 MVP 足够演示方法机制；大规模稀疏问题通常需稀疏分解与更专业线性代数后端。

## R08

边界与异常处理：
- `A` 非二维、`b/c/x0` 非一维或维度不匹配：抛 `ValueError`。  
- 输入含 `nan/inf`：抛 `ValueError`。  
- `x0` 不严格正或 `A x0` 与 `b` 不一致（超过容差）：抛 `ValueError`。  
- 参数非法（`mu<=1`、`tol<=0`、迭代上限非正）：抛 `ValueError`。  
- KKT 线性系统奇异或病态导致求解失败：抛 `RuntimeError`。  
- 达到迭代上限仍未收敛：抛 `RuntimeError`。

## R09

MVP 取舍说明：
- 采用 `numpy` 手写核心内点流程，避免仅调用黑盒求解器。  
- 只实现“等式 + 非负”的标准型，未覆盖通用不等式输入解析。  
- 使用稠密线性代数，代码短而直观，但不追求大规模性能。  
- 将 SciPy 对照作为“验证器”而非“主求解器”，确保算法逻辑仍可源级追踪。

## R10

`demo.py` 主要函数职责：
- `validate_lp_inputs`：检查 `A,b,c,x0` 的形状、有限性、严格可行性。  
- `kkt_residual`：计算当前 `(x, nu)` 在给定 `t` 下的 KKT 残差。  
- `solve_kkt_step`：构造并求解牛顿方向 `(dx, dnu)`。  
- `centering_newton`：固定 `t` 的内层牛顿-回溯求解。  
- `barrier_path_following_lp`：外层路径跟随循环与收敛判据。  
- `build_demo_lp_model`：构造固定 LP 样例（由不等式转标准型并给出严格可行初值）。  
- `try_scipy_reference`：可选调用 SciPy HiGHS 作为参考最优解。  
- `print_outer_history`：格式化输出外层轨迹。  
- `main`：组织执行、打印结果、进行数值一致性检查。

## R14

可调参数（`barrier_path_following_lp`）：
- `mu`：外层障碍参数放大倍数（默认 `12.0`）。  
- `outer_tol`：外层停止阈值（默认 `1e-9`，判据 `n/t < outer_tol`）。  
- `newton_tol`：内层 KKT 残差阈值（默认 `1e-10`）。  
- `max_outer`：最大外层轮数。  
- `max_newton_iter`：每个外层最大牛顿步数。  
- `backtrack_beta`：回溯缩放系数（默认 `0.5`）。  
- `residual_c1`：残差下降充分性系数（默认 `1e-2`）。

调参建议：
- 收敛慢：增大 `mu` 或放宽 `newton_tol`；  
- 数值不稳：减小 `mu`，增加 `max_newton_iter`，并保持较保守的回溯参数。

## R15

方法对比：
- 对比单纯形法：
  - 单纯形沿顶点移动，常在工程上很快；  
  - 内点法在大规模稀疏问题中常具更稳定的多项式时间行为。  
- 对比“直接调用 LP 黑盒”：
  - 黑盒更省代码，但难解释内部机制；  
  - 本条目源级展示障碍、KKT、牛顿与回溯的核心链路。  
- 对比罚函数法：
  - 罚函数可从不可行点出发，但常面临罚参数病态；  
  - 内点法通过严格可行路径更自然地逼近边界约束。

## R18

`demo.py` 源码级算法流程（9 步）：
1. `build_demo_lp_model` 构造一个二维不等式 LP，并引入 3 个松弛变量得到标准型 `A x = b, x >= 0`，同时给出严格可行初值 `x0`。  
2. `main` 调用 `barrier_path_following_lp` 开始路径跟随，初始障碍参数 `t=1`。  
3. 每个外层迭代进入 `centering_newton`，固定当前 `t` 求障碍子问题中心点。  
4. `kkt_residual` 计算当前 `r_dual, r_primal` 及其范数，作为内层收敛判据。  
5. `solve_kkt_step` 组装增广 KKT 矩阵 `[H, A^T; A, 0]` 并解出牛顿方向 `(dx, dnu)`。  
6. 回溯线搜索先保证 `x + alpha*dx` 严格正，再要求新残差满足充分下降条件。  
7. 更新 `(x, nu)` 后继续内层，直到残差低于 `newton_tol` 或达到内层上限。  
8. 外层记录 `(t, c^T x, n/t, inner_iters, min(x))`，若 `n/t < outer_tol` 则停止；否则 `t <- mu*t` 继续。  
9. `main` 打印最终解、约束残差与目标值，并尝试 `SciPy linprog(HiGHS)` 做数值交叉验证。
