## R01

约束 Delaunay 三角剖分（Constrained Delaunay Triangulation, CDT）是对平面直线图（PSLG）的三角剖分方法。

给定顶点集与必须保留的线段约束 `E_c`，CDT 要求：

- 所有约束边都出现在最终三角网中；
- 对所有非约束内部边，满足局部 Delaunay 条件（可通过翻边消除非法边）；
- 三角形覆盖目标区域（本目录 MVP 采用单连通简单多边形区域）。

它是“受几何约束的 Delaunay 化”：不是无约束 Delaunay，也不是任意三角化。

## R02

CDT 在计算几何、有限元网格、地理信息与路径规划中是常用基础结构：

- 对比普通三角剖分：CDT 能严格保留边界和内部断裂线；
- 对比纯 Delaunay：CDT 允许“违背空圆性质”的约束边存在，但会让其余边尽量 Delaunay；
- 在工程里常作为网格前处理步骤，为 PDE 离散、插值、碰撞检测提供质量更稳的单元。

历史上，约束恢复（constraint recovery）与 Lawson 翻边是最常见的构建思路之一。

## R03

本条目要解决的问题是：

- 输入：一个简单多边形（顶点按环顺序给出）+ 若干内部非交叉约束对角线（端点均来自顶点集）；
- 输出：同时满足“约束边保留 + 非约束边局部 Delaunay”的三角网。

MVP 目标不是覆盖所有 PSLG 变体（例如孔洞、相交约束、Steiner 点自动插入），而是提供可运行、可验证、可追踪源码流程的最小版本。

## R04

本目录实现采用两阶段流程：

1. **构造初始受约束三角化**
   - 先将多边形按每条约束对角线逐步切分为多个子面；
   - 每个子面使用 ear clipping（三角耳切分）得到初始三角形集合；
   - 此时已保证所有约束边存在，但网格未必满足局部 Delaunay。
2. **约束下 Delaunay 化**
   - 对每条非约束内部边，检查其相邻两三角是否形成非法边；
   - 若非法且可翻（对应对角线相交于四边形内部），执行 Lawson edge flip；
   - 迭代到无可翻非法边，得到 CDT（局部意义）。

## R05

记多边形顶点数为 `n`，最终三角形数量近似 `O(n)`。

- 约束插入与面切分：在本 MVP 的线性扫描实现下，约为 `O(n * k)` 到 `O(n^2)`（`k` 为约束数）；
- Ear clipping：单个 `m` 边形最坏 `O(m^2)`，总和在本规模下可接受；
- Lawson 翻边：每轮扫描边集，翻边次数与网格规模相关，经验上接近线性到次二次。

因此该实现更偏“教学与可验证 MVP”，非大规模最优复杂度版本。

## R06

## R10

正确性直觉（MVP 版本）：

1. 面切分阶段仅使用合法对角线切面，保持拓扑不交叉；
2. 每个子面 ear clipping 产生无交叉三角剖分，子面并起来覆盖原域；
3. 约束边作为切分边天然保留在三角网中；
4. 翻边阶段只翻非约束内部边，不会破坏约束边集合；
5. 对可翻四边形持续消除非法边，直到局部最优，即满足局部 constrained Delaunay 条件。

因此最终网格满足“约束保留 + 非约束边局部 Delaunay”。

## R11

数值与鲁棒性策略：

- 使用统一 `EPS = 1e-10` 处理方向、相交、空圆判定；
- 只接受非交叉约束，避免复杂约束恢复失败分支；
- 翻边设置 `max_flips` 上限，防止异常输入导致死循环；
- 对面积与约束完整性做断言，尽早暴露错误。

若输入点接近共线或重复，建议先做几何清洗（去重点、微扰或精确谓词替代）。

## R12

关键工程参数：

- `EPS`：几何容忍度。坐标量级越大，常需按比例放大；
- `max_flips`：翻边上限。输入异常时可快速失败；
- 约束顺序：当前为顺序插入，建议传入互不相交且拓扑可恢复的序列。

调参建议：

- 先固定 `EPS=1e-10` 跑通；
- 若出现误判（例如本应可翻却不翻），可测试 `1e-9 ~ 1e-8`；
- 若误把合法结构判成相交，适当减小 `EPS` 并检查坐标尺度。

## R14

常见失效模式：

1. 约束线段相互穿越或穿越边界；
2. 约束端点不在当前面边界顶点集中；
3. 多点近共线导致 ear 识别与 in-circle 抖动；
4. 输入多边形自交，导致“点在面内”逻辑失效。

诊断方式：

- 查看 `Constraint (...) cannot be inserted` 异常；
- 查看 `Ear clipping failed` 异常；
- 查看最终断言：缺失约束、非法边未消除、面积不一致。

## R18

`demo.py` 的源码级算法流程（8 步）：

1. `build_initial_constrained_triangulation` 先规范外多边形方向（必要时反转为 CCW），并初始化单面列表。  
2. 对每条约束，调用 `is_valid_diagonal` 检查其在某个当前面中是否合法，然后 `split_polygon_with_diagonal` 切分该面。  
3. 对切分得到的每个子面，执行 `triangulate_polygon_ear_clipping`，得到初始受约束三角网。  
4. 收集 `constrained_edges`（外边界 + 约束对角线），这些边在后续翻边中被锁定不可变。  
5. `constrained_delaunay_refine` 构建边到三角形邻接，扫描每条非约束内部边。  
6. 对候选边用 `edge_is_illegal`（包含对角线相交性 + `incircle` 判定）识别是否违反局部 Delaunay 条件。  
7. 对非法边执行一次 Lawson flip（两三角共享边替换为另一条对角线），并循环直到无可翻非法边。  
8. `run_demo_case` 做结果验证：`count_illegal_unconstrained_edges == 0`、约束边完整、面积一致，然后打印最终三角列表。

该实现没有把第三方几何库当黑盒：面切分、耳切、局部空圆测试与翻边都在源码中完整可追踪。
