## R01

本条目实现二分图最大基数匹配的经典 `匈牙利算法（Kuhn DFS 增广路版本）` 最小可运行版本（MVP）。目标是：
- 输入二分图 `G=(U,V,E)`；
- 输出最大匹配大小与匹配边；
- 明确“增广路 + 递归重排”的核心机制；
- 不依赖图算法黑盒，核心流程在 `demo.py` 中完整展开；
- 可直接 `uv run python demo.py` 无交互运行。

## R02

问题定义（本 MVP 范围）：
- 图为二分图，边仅从左部 `U` 连向右部 `V`；
- 匹配 `M` 中任意两条边不共享端点；
- 目标是最大化 `|M|`（最大匹配边数）；
- 不涉及权重与最小费用，不是“带权指派问题”的 Hungarian 矩阵版本；
- 输出 `pair_left`、`pair_right` 与 `matching_size`。

## R03

核心思想：增广路。
- 若存在一条从“未匹配左点”到“未匹配右点”的交替路（未匹配边/已匹配边交替），就可沿路翻转匹配状态，使总匹配数 `+1`。
- Kuhn 版本每次固定一个左点，做一次 DFS 尝试找到增广路。
- DFS 在访问某个右点后，若该右点已被左点 `u2` 占用，则递归尝试为 `u2` 重新安排匹配；成功则“腾位置”。
- 对全部左点重复即可得到最大匹配。

## R04

数学与记号：
- 左点集合 `U={0,1,...,n_left-1}`，右点集合 `V={0,1,...,n_right-1}`。
- 邻接表 `adj[u]` 记录左点 `u` 可连接的右点集合。
- `pair_left[u] = v` 表示 `u` 当前匹配到 `v`，否则 `-1`。
- `pair_right[v] = u` 表示 `v` 当前匹配到 `u`，否则 `-1`。
- 每次增广尝试维护 `seen_right[v]`，避免一次 DFS 内重复访问右点导致死循环。

## R05

`demo.py` 中的数据结构：
- `HungarianKuhnResult`：封装匹配结果与统计量（`dfs_calls`、`edge_scans` 等）。
- `validate_input`：输入校验 + 去重重复边。
- `build_adjacency`：构造左到右邻接表。
- `hungarian_kuhn`：核心算法实现。
- `is_valid_matching`：匹配合法性检查。
- `brute_force_maximum_size`：小规模暴力最优值参考，用于交叉验证。

## R06

正确性要点（直观版）：
1. 每次 `try_augment(u)` 成功时，都沿一条增广路翻转匹配，匹配规模恰增 `1`。  
2. 翻转过程中仅在“同一路径”上替换匹配，不会让一个右点同时匹配多个左点。  
3. 算法结束后，若仍存在增广路，则至少有一个未匹配左点可继续成功增广，和“无法再增广”矛盾。  
4. 依据 Berge 定理，“不存在增广路”当且仅当“当前匹配为最大匹配”。

## R07

复杂度（Kuhn DFS 版本）：
- 设总点数 `V=n_left+n_right`，边数 `E=|E|`。
- 单次 DFS 最坏扫描 `O(E)`（更精确是沿可达子图边数）。
- 每个左点做一次增广尝试，整体常见上界写作 `O(VE)`（或 `O(n_left * E)`）。
- 空间复杂度 `O(V + E)`：邻接表 `O(E)`，匹配数组和访问数组 `O(V)`。

## R08

边界与异常处理：
- `n_left` 或 `n_right` 为负时抛 `ValueError`；
- 边端点越界时抛 `ValueError`；
- 重复边会被去重，不影响匹配结果；
- 允许空图、孤立点、左右规模不等；
- `n_right=0` 时算法仍可运行，结果匹配大小为 `0`。

## R09

与 Hopcroft-Karp 的关系：
- 二者都解二分图最大匹配；
- Kuhn（本条目）实现更短、更直观，适合教学与小中规模图；
- Hopcroft-Karp 用 BFS 分层 + DFS 批量增广，复杂度更优 `O(E*sqrt(V))`；
- 工程上可先用 Kuhn 验证逻辑，再切换到 Hopcroft-Karp 处理更大规模数据。

## R10

## R14

实现细节与常见坑：
- 每次以某个左点发起 DFS 时，`seen_right` 必须重新初始化；
- `pair_right[v]` 已占用时，递归“挪位”必须在同一次 DFS 的 `seen_right` 约束下进行；
- 更新匹配时必须同步维护 `pair_left` 与 `pair_right`；
- 重复边不去重会导致额外扫描，影响统计但不影响最优值。

## R18

`demo.py` 源码级算法流程（8 步）：
1. `run_demo_case` 构造固定二分图，调用 `validate_input` 做范围校验并去重边。  
2. `hungarian_kuhn` 初始化 `pair_left/pair_right=-1`，表示全部点未匹配。  
3. 依次遍历每个左点 `u`，为本次尝试创建新的 `seen_right` 布尔数组。  
4. `try_augment(u, seen_right)` 深度优先扫描 `adj[u]` 的右邻居 `v`。  
5. 若 `v` 尚未访问，则标记 `seen_right[v]=True`，避免本轮 DFS 重复绕回。  
6. 若 `v` 未被占用（`pair_right[v]==-1`），或可递归为当前占用者 `matched_left` 找到替代匹配，则把 `u` 匹配到 `v`。  
7. 一次 `try_augment` 成功即完成一条增广路翻转，`matching_size` 增加 `1`。  
8. 全部左点尝试结束后返回结果，再用 `is_valid_matching` 与 `brute_force_maximum_size` 做合法性和最优性校验。
