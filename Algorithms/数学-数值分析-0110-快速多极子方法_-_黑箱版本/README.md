## R01

快速多极子方法（Fast Multipole Method, FMM）的“黑箱版本”核心思想是：

- 不依赖特定核函数的解析多极展开（比如手写球谐展开）。
- 只依赖核函数数值评估 `K(x, y)` 与插值/等效源构造。
- 把“远场作用”压缩成少量代理节点上的等效系数。

本条目在 `demo.py` 中实现了一个 2D MVP：

- 核函数：`K(x, y) = 1 / ||x - y||`
- 空间结构：四叉树
- 远场近似：Chebyshev 插值节点 + 上行聚合等效源
- 近场处理：叶子盒内直接求和

这是一种 kernel-independent（核无关）/black-box FMM 风格实现。

## R02

历史脉络（简述）：

- 1987 年，Greengard 与 Rokhlin 提出经典 FMM，奠定 `O(N)` 或近线性复杂度的快速 N 体计算框架。
- 经典 FMM 往往依赖核函数解析展开（例如 Laplace、Helmholtz 的专用展开）。
- 随后发展出 kernel-independent / black-box FMM：用插值、等效源、离散翻译矩阵替代手工解析展开，使方法更通用。

黑箱版本的重要意义在于：面对新核函数时，不必重新推导复杂解析公式，也能构建可用的快速算法。

## R03

问题背景：

给定 `N` 个源点 `y_j` 与权重 `q_j`，以及目标点 `x_i`，要计算

`phi(x_i) = Σ_{j!=i} q_j * K(x_i, y_j)`。

直接法复杂度是 `O(N^2)`，当 `N` 变大时成本迅速上升。黑箱 FMM 的动机是：

- 把距离很远的一簇源点对目标的影响，压缩成少量参数；
- 把每个目标的访问成本从“遍历全部源”降到“只看少量近场 + 少量远场代理”；
- 在可控误差下实现显著降复杂度。

## R04

复杂度（针对本 MVP）：

- 建树：平均 `O(N log N)`。
- 上行聚合（每个节点 `p^2` 个系数）：约 `O(#nodes * p^4)`（`p` 为单轴插值阶数）。
- 求值：每个目标通过树遍历，典型 `O(log N)` 到 `O(常数 * log N)`，整体近似 `O(N log N)`。

与直接法 `O(N^2)` 比，理论上在大规模更有优势。需要注意：本 MVP 是教学版 Python 实现，常数项与解释器开销较大，小规模下不一定比 NumPy 向量化直接法更快。

## R05

空间复杂度：

- 点与权重：`O(N)`。
- 树节点：`O(N)`（与叶容量相关）。
- 每节点保存 `p^2` 个等效系数 + `p^2` 个代理点，约 `O(#nodes * p^2)`。
- 总体可视为 `O(N + #nodes * p^2)`，在固定 `p` 下近似线性。

## R06

一个简化手算示意（1D 观念迁移到 2D）：

1. 设某盒内有多个源点，把盒映射到局部区间 `[-1,1]`。
2. 选 4 个 Chebyshev 节点 `t_k`。
3. 对每个真实源点 `y_j`，计算其在节点基函数上的插值权重 `L_k(y_j)`。
4. 聚合出节点等效系数 `alpha_k = Σ_j q_j * L_k(y_j)`。
5. 对远处目标 `x`，近似
   `Σ_j q_j K(x, y_j) ≈ Σ_k alpha_k K(x, y_k_proxy)`。

2D 情况就是做张量积，节点数从 `p` 变成 `p^2`，思想相同。

## R07

算法意义：

- 数值分析意义：把“核求和”转化为“层次化低秩近似 + 局部精确修正”。
- 工程意义：统一了多类核函数的快速求和实现路径。
- 方法论意义：将“解析展开驱动”升级为“离散算子/插值驱动”，更易模块化和复用。

## R09

适用前提与边界：

- 核函数在远场区域应足够光滑，便于低秩近似。
- 点分布越“空间可分簇”，层次方法越有效。
- 本实现是 2D 标量势、单核 `1/r` 的教学 MVP，不包含并行、SIMD、GPU、周期边界等工程特性。
- 近场奇异性通过直接求和处理，避免在奇异区做插值近似。

## R10

正确性依据（MVP 层面）：

- 叶子近场采用直接求和，因此近邻贡献是精确的。
- 远场误差来自插值逼近：
  - 用 Chebyshev 节点构造的插值基函数逼近源分布影响；
  - 节点等效系数在上行阶段保持“对远处可见势场”的一致性近似。
- 最终误差可通过与直接法对比评估（`demo.py` 输出 `Relative L2 error` 与 `Max abs error`）。

## R11

数值稳定性：

- 使用 barycentric 形式评估 Lagrange 基函数，避免直接高次多项式系数导致的不稳定。
- Chebyshev-Lobatto 节点比等距节点更稳健。
- 奇异核 `1/r` 在 `r -> 0` 处数值风险通过：
  - 自作用项剔除（`j != i`）；
  - `eps` 下界保护避免除零。

## R12

真实成本分析（实践层面）：

- Python 递归与逐点遍历常数项较高；
- `p` 从 4 提升到 6 时，精度通常提升，但每节点代价近似按 `p^2 ~ p^4` 项增长；
- `theta` 越大，远场判定越严格，通常精度更好但访问树更深、计算更多；
- `leaf_size` 太小会导致节点数膨胀，太大又会让近场直接和代价上升。

因此参数需要按规模和精度目标联合调参。

## R13

近似性与误差控制：

本算法是近似算法，误差主要受三类参数控制：

- 插值阶数 `interp_order`：高阶通常更准。
- 分离阈值 `theta`：阈值大，远场近似更保守。
- 树深度/叶容量：决定近场与远场分工。

没有单一固定误差常数；在给定核与分布下，通常通过基准对比（direct sum）做经验校准。

## R14

鲁棒性与失效模式：

- 失效模式 1：`theta` 太小导致“并不够远的盒子”被当远场，误差放大。
- 失效模式 2：`interp_order` 太低，远场压缩不足。
- 失效模式 3：极端非均匀分布导致树不平衡，性能退化。
- 失效模式 4：对强奇异/高振荡核直接复用参数，可能出现不稳定。

应对策略：增大 `theta`、提高 `interp_order`、限制 `max_depth`、为特定核重设插值与分离准则。

## R18

源码级流程拆解（`demo.py`，非第三方黑盒，8 步）：

1. **初始化插值基**：在 `BlackBoxFMM2D.__init__` 中生成 1D Chebyshev-Lobatto 节点、barycentric 权重与 2D 张量节点网格。
2. **构建空间树**：`_build_tree` 递归按中心点把样本划入四个子象限，直到达到 `leaf_size` 或 `max_depth`。
3. **叶节点等效化**：`_upward_pass` 在叶子上把真实源 `q_j` 投影到 `p^2` 插值节点，得到叶子 `alpha`。
4. **子到父转移**：`_transfer_matrix` 计算（并缓存）4 种相对方位的离散转移矩阵；父节点 `alpha += T * alpha_child` 完成上行聚合。
5. **分离判定**：`_well_separated` 用无穷范数距离与 `theta * half_size` 判断目标点与当前盒子是否可做远场近似。
6. **远场快速求值**：若满足分离，`_eval_point_recursive` 直接用当前盒子的代理点与 `alpha` 评估势贡献 `Σ alpha_k / r_k`。
7. **近场精确求值**：若不满足分离并到达叶子，`_direct_leaf_eval` 对叶内源点做直接求和并排除自作用项。
8. **基准校验与报告**：`main` 中同时计算 direct sum，对比相对 `L2` 误差与最大绝对误差，并打印时间统计。
