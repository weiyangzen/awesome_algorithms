# Fürer算法/Harvey-van der Hoeven算法

- UID: `MATH-0044`
- 学科: `数学`
- 分类: `算法`
- 源序号: `44`
- 目标目录: `Algorithms/数学-算法-0044-Fürer算法／Harvey-van_der_Hoeven算法`

## R01

该条目对应“超快速大整数乘法”谱系中的两代核心结果：

- Fürer 算法：把 `n` 比特整数乘法转化为特殊环上的快速傅里叶变换（FFT）问题，达到 `n log n` 级别并只带极慢增长因子。
- Harvey-van der Hoeven（HvdH）算法：在 Fürer 路线基础上进一步消除该慢增长因子，首次达到严格的 `O(n log n)` 位复杂度上界。

统一抽象如下：

- 输入：两个 `n` 比特大整数 `A, B`。
- 输出：乘积 `C = A * B`。
- 核心机制：
  1. 按基数 `B0` 分块，将整数编码为多项式系数。
  2. 把“整数乘法”降为“多项式卷积”。
  3. 在可用单位根的环中做 FFT/NTT 评估与插值。
  4. 执行点值乘法。
  5. 逆变换后做进位归并，恢复整数结果。

## R02

关键时间线可概括为：

- 1971：Schönhage-Strassen 给出 `O(n log n log log n)`，长期是理论上界基准。
- 2007（后续期刊版本 2009）：Martin Fürer 提出更快算法，复杂度写作 `n log n 2^{O(log* n)}`（或等价的 `n log n K^{log* n}` 形式）。
- 2019：David Harvey 与 Joris van der Hoeven 公布 `O(n log n)` 整数乘法结果（后续正式发表）。

## R03

提出背景是“把整数乘法复杂度尽量压近信息论下界 `Omega(n)`”：

- 朴素竖式乘法为 `O(n^2)`。
- Karatsuba/Toom-Cook 把复杂度降到亚二次，但仍远高于 `n log n`。
- Schönhage-Strassen 达到 `O(n log n log log n)`，但 `log log n` 因子仍存在。
- Fürer 通过更精细的变换环与递归结构，把额外因子显著弱化到 `2^{O(log* n)}`。
- HvdH 再进一步把这部分“超慢因子”消去，得到理论最优阶（差一个常数因子）的 `O(n log n)`。

## R04

令 `n` 为输入整数比特长度，典型位复杂度结论：

- Fürer 算法：`M(n) = n log n 2^{O(log* n)}`。
- HvdH 算法：`M(n) = O(n log n)`。

说明：

- 这里是渐近上界，关注 `n -> infinity` 的理论行为。
- 对实际中等规模（例如几千到几百万比特），“常数项 + 工程细节”往往比渐近阶更决定实际速度。

## R05

空间复杂度通常为准线性量级（以位计）：

- 需要存放分块系数数组、频域数组、临时缓冲区。
- 实现形态不同，上界常写为 `O(n)` 到 `O(n log n)` 位之间。
- 工程实现里常见表现是“与输入规模同阶的多块工作内存”，并非常数空间。

## R06

一个可复现的分块卷积示例（演示主路径，不是完整 Fürer/HvdH 细节）：

取基数 `B0=100`，计算 `1234 * 5678`。

1. 分块：
   - `1234 = 34 + 12 * 100`，系数向量 `a=[34,12]`
   - `5678 = 78 + 56 * 100`，系数向量 `b=[78,56]`
2. 多项式卷积系数：
   - `c0=34*78=2652`
   - `c1=34*56 + 12*78 = 2840`
   - `c2=12*56=672`
3. 基数 100 进位归并：
   - `2652 -> 52` 进位 `26`
   - `2840+26=2866 -> 66` 进位 `28`
   - `672+28=700 -> 00` 进位 `7`
4. 得到分块 `[52,66,0,7]`，还原为 `7006652`。

即 `1234 * 5678 = 7006652`。

## R07

算法意义：

- 理论意义：把整数乘法复杂度上界推进到 `O(n log n)`，是复杂度理论里程碑。
- 方法意义：巩固“整数乘法 -> 多项式卷积 -> 快速变换”的统一范式。
- 工程意义：虽然完整 HvdH 并不直接用于日常库实现，但其思想持续影响高性能大整数/多项式乘法设计。

## R08

该路径依赖的关键先导技术：

- 快速傅里叶变换（Cooley-Tukey 系列分解）。
- 数论变换/模环变换（在可控误差或精确模运算中做卷积）。
- 中国剩余定理（在多模数表示与重构中常见）。
- 快速多项式乘法与分治递归。
- 大整数分块编码与进位恢复。

## R09

适用前提与边界：

- 输入应为任意精度整数乘法问题，且关注非常大规模 `n` 的渐近行为。
- 对小规模/中等规模整数，Karatsuba、Toom-Cook、经典 FFT 实现通常更实用。
- 理论算法要求变换长度、环结构、参数配比满足可用单位根与误差控制条件。
- 若仅使用浮点 FFT 且参数不当，可能出现舍入导致的系数误差，需要额外校验。

## R10

正确性依据（高层）：

1. 基数分块给出整数与系数向量的一一对应（在固定基数下）。
2. 系数卷积对应整数乘法的无进位结果（多项式乘法同构）。
3. FFT/NTT 在满足条件时可正确完成卷积（评估-点乘-插值）。
4. 进位归并把卷积系数还原为唯一的基数表示。
5. 因此还原整数等于原整数乘积。

## R11

数值稳定性与误差传播：

- 纯理论版本可通过精确环算术避免浮点误差（HvdH 路线强调理论可证界）。
- 若采用复数浮点 FFT（工程常见），误差源主要是：
  - 浮点舍入；
  - 频域乘法后逆变换的小量噪声。
- 典型缓解：
  - 选择较小分块基数；
  - 长度补零到 2 的幂；
  - 逆变换后四舍五入并做整型进位；
  - 结果与内置乘法做抽样比对。

## R12

真实计算成本（超越渐近式）：

- 常数项极大：高阶理论算法在可实现范围内通常不占优。
- 内存带宽与缓存行为：大规模 FFT 常被内存访存主导。
- 并行化/向量化：FFT 天然可并行，但通信与重排成本也明显。
- 参数选型成本：基数、变换长度、模数链、递归阈值都会改变真实性能。
- 对 Python MVP 而言，`numpy.fft` 调用、数组分配和进位循环是主要时间项。

## R13

`N/A + 原因`：该类算法目标是“精确整数乘法”，不是近似优化或随机近似算法，通常不以“近似比/概率成功率”定义主保证。

补充：工程实现如果使用浮点 FFT，可能会加入“结果校验与重试”，但这属于实现层鲁棒性策略，不改变其目标问题的“精确输出”性质。

## R14

鲁棒性与失效模式：

- 失效模式 1：分块基数过大，卷积系数超出浮点安全范围，导致逆变换舍入错误。
- 失效模式 2：变换长度不足（未正确补零），出现循环卷积混叠。
- 失效模式 3：进位逻辑错误（负号、末尾进位、前导零处理不当）。
- 失效模式 4：超大输入下内存不足或频繁分配导致性能劣化。

对应防护：

- 约束基数和测试规模；
- 统一用 `len(a)+len(b)-1` 决定最小线性卷积长度；
- 对关键样例做 `x*y` 一致性校验；
- 增加随机回归测试。

## R15

工程实现要点：

- 数据表示：采用小端分块数组，便于卷积和进位。
- FFT 长度：取不小于输出长度的最小 2 幂，便于高效 FFT。
- 变换流程：`rfft -> 频域逐点乘 -> irfft`。
- 精度处理：逆变换结果先 `round` 再进入整数进位。
- 符号处理：绝对值乘法 + 末端恢复符号。
- 验证机制：示例用例 + 随机 bit 长基准对照 Python 内置乘法。

## R16

前驱/后继与应用：

- 前驱：Karatsuba、Toom-Cook、Schönhage-Strassen。
- 中间里程碑：Fürer（`n log n 2^{O(log* n)}`）。
- 后继里程碑：Harvey-van der Hoeven（`O(n log n)`）。

典型应用场景：

- 任意精度整数库的高位乘法内核设计参考。
- 计算机代数系统中的大规模多项式/整数运算。
- 密码学与高精度科学计算中的基础算术模块。

现实 benchmark 结论通常是：理论最优不等于工程最快，实际库会分段选择不同算法阈值。

## R17

Python MVP（见同目录 `demo.py`）设计边界：

- 目标：用最小可运行代码复现“Fürer/HvdH 的共同主干思想”，不是复刻论文全部细节。
- 依赖：`numpy`（FFT），其余使用 Python 标准库。
- 实现组件：
  1. `int_to_chunks` / `chunks_to_int`：整数与分块互转。
  2. `fft_convolution`：用 `numpy.fft.rfft/irfft` 做线性卷积。
  3. `normalize_carries`：把卷积系数转回合法基数位。
  4. `fft_bigint_multiply`：完整乘法管线（含符号处理）。
  5. `benchmark`：多组 bit 长测试并与 Python 内置乘法核对。
- 运行方式：`python3 demo.py`，无需交互输入。

## R18

针对 `demo.py` 中第三方 FFT 调用的源码级流程拆解（8 步）：

1. `fft_bigint_multiply` 把输入整数拆成基数块，调用 `fft_convolution(a, b)`。
2. `fft_convolution` 调用 `np.fft.rfft(arr, n)`；NumPy 的公开入口在 `numpy/fft/_pocketfft.py`。
3. `numpy.fft._pocketfft.fft/rfft` 经调度器进入 `_raw_fft(...)`，在这里完成长度检查、归一化参数和输出 dtype 选择。
4. `_raw_fft` 根据变换类型选择底层 ufunc（如 `pfu.rfft_n_even`、`pfu.fft`），其中 `pfu` 指向 C 扩展模块 `numpy.fft._pocketfft_umath`。
5. C 扩展执行 pocketfft 内核，完成蝶形分解、旋转因子应用与频域结果写回。
6. Python 层拿到 `spec_a`、`spec_b` 后做逐点复数乘法 `spec_c = spec_a * spec_b`（卷积定理）。
7. 再调用 `np.fft.irfft(spec_c, n)`，走 `_raw_fft -> pfu.irfft` 反向路径返回时域卷积系数；随后 `round` 到最近整数。
8. `normalize_carries` 执行逐位进位，`chunks_to_int` 复原最终整数，完成“变换域卷积 -> 精确整数乘积”的闭环。

说明：本 MVP 显式实现了整数编码、卷积、进位恢复和校验；仅把“FFT 内核”交给 NumPy，并已按模块级路径展开其调用链，不作为黑箱省略。
