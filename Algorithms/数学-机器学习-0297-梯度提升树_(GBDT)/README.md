## R01

梯度提升树（Gradient Boosting Decision Tree, GBDT）是一类加法模型：
- 用多个弱学习器（通常是回归树）逐轮叠加；
- 每一轮都去拟合当前模型在损失函数下的负梯度（也叫伪残差）；
- 通过“小步更新 + 多轮迭代”把整体损失逐步压低。

本条目实现的是二分类版本（log-loss），用手写的 `RegressionStump`（深度 1 回归树）作为基学习器，显式展开 GBDT 主训练循环。

## R02

本条目问题定义：

- 输入：
  - 训练样本 `X in R^(n*d)`，标签 `y in {0,1}^n`；
  - 超参数：`n_estimators`、`learning_rate`、`n_thresholds`、`min_samples_leaf`。
- 输出：
  - 初始常数预测 `F0`；
  - 一组按顺序训练得到的回归树 `{h_1, ..., h_M}`；
  - 对任意样本的最终打分 `F_M(x)` 与概率 `sigmoid(F_M(x))`。

目标是最小化二分类对数损失（log-loss）。

## R03

二分类 GBDT（log-loss）核心公式：

- 加法模型：

`F_m(x) = F_{m-1}(x) + eta * h_m(x)`

- 概率映射：

`p(x) = sigmoid(F(x)) = 1 / (1 + exp(-F(x)))`

- 每轮伪残差（负梯度）：

`r_i^(m) = y_i - p_{m-1}(x_i)`

- 基学习器拟合目标：

`h_m = argmin_h sum_i (r_i^(m) - h(x_i))^2`

也就是用回归树去拟合“当前还没解释掉的方向信息”。

## R04

算法流程（工程版）：

1. 用训练集正类比例初始化 `F0 = log(pos_rate/(1-pos_rate))`。  
2. 对每个样本设置初始 raw score `F_i = F0`。  
3. 进入迭代第 `m` 轮：计算 `p_i = sigmoid(F_i)`。  
4. 计算伪残差 `r_i = y_i - p_i`。  
5. 训练一棵回归树 `h_m` 拟合 `(X, r)`。  
6. 得到该树在训练集上的预测增量 `u_i = h_m(x_i)`。  
7. 更新 `F_i <- F_i + eta * u_i`。  
8. 保存该树并进入下一轮，最终把所有树的输出累加。

## R05

复杂度分析（`M` 棵树，样本数 `n`，特征数 `d`）：

- 训练复杂度：约 `M * Cost(TreeFit)`。
- 若单棵 CART 回归树训练近似 `O(n*d*log n)`，则总训练约 `O(M*n*d*log n)`（依具体实现有差异）。
- 推理复杂度：单样本约 `O(M * depth)`。
- 额外模型存储：`O(M * nodes_per_tree)`。

GBDT 典型策略是“浅树 + 多轮”，靠 boosting 累积表达能力。

## R06

单轮更新的微例（直观）：

假设当前某样本的：
- 标签 `y=1`；
- 当前打分 `F=0.2`，则 `p=sigmoid(0.2)≈0.55`；
- 伪残差 `r=y-p≈0.45`。

如果该样本在新树上预测增量 `h(x)=0.30`，学习率 `eta=0.1`，
则更新后：
- `F_new = 0.2 + 0.1*0.30 = 0.23`；
- `p_new = sigmoid(0.23)≈0.557`。

概率朝正确方向（更接近 1）小幅推进。

## R07

GBDT 的工程价值：

- 对结构化表格数据通常很强；
- 能自动建模非线性与特征交互；
- 对特征缩放不敏感，数据预处理成本较低；
- 训练与推理逻辑清晰，便于解释（特征重要性、树路径分析）。

## R08

理论与实践前置：

1. 梯度下降思想：沿损失负梯度方向迭代下降。  
2. 加法模型思想：复杂函数由多个简单函数相加逼近。  
3. 决策树回归：把目标映射到分段常数函数。  
4. logistic link：把实值打分映射为概率。  
5. shrinkage（学习率）控制每轮步幅，提高泛化稳定性。

## R09

适用条件与边界：

- 适用：中小到中大型表格数据、异质特征、非线性关系明显任务。  
- 不适用或需谨慎：
  - 极高维稀疏文本（线性模型或专门方法可能更优）；
  - 超大规模低延迟在线训练（需用专门并行框架）；
  - 噪声标签很重且树深过大时，易过拟合。

## R10

正确性检查框架（本实现）：

1. 初始化常数项来自训练集正例比例，避免不合理起点。  
2. 每轮显式按 `r = y - sigmoid(F)` 计算伪残差。  
3. 每轮回归树只拟合当前残差，不直接看未来轮信息。  
4. 每轮更新严格执行 `F += learning_rate * tree_pred`。  
5. 训练后校验：树数正确、预测有限值、loss 下降、精度优于 baseline。

## R11

数值稳定性：

- `sigmoid` 输入裁剪到 `[-50, 50]` 防止指数溢出；
- `log-loss` 计算时对概率做 `clip(eps, 1-eps)` 防止 `log(0)`；
- 初始正例比例 `pos_rate` 做边界裁剪，避免 `log(0)`；
- 用较小学习率配合多轮迭代，减少训练振荡。

## R12

性能与调参要点：

- `n_estimators` 增大：偏差下降，但训练时间和过拟合风险上升；
- `learning_rate` 降低：每步更稳，常需更多树配合；
- `n_thresholds` 增大：单轮分裂候选更细，拟合更强但训练更慢；
- `min_samples_leaf` 增大：可抑制过深分裂，改善泛化；
- 通常用“低学习率 + 更多树”取得稳定效果。

## R13

理论保证与本条目定位：

- 近似比保证：N/A（不是组合优化近似算法）。
- 概率成功保证：N/A（不是 Las Vegas / Monte Carlo 解法）。

GBDT 的强项在经验风险优化与泛化性能，不以近似比界为主要评价指标。

## R14

常见失效模式：

- 树太深 + 轮数太多：训练集很好但测试集退化（过拟合）；
- 学习率过大：loss 抖动大、后期不稳定；
- 类别极度不平衡：初始偏置和阈值未处理好会影响召回；
- 样本过少：树模型方差高，结果对随机种子敏感。

防护思路：限制树深、增大叶子最小样本、降低学习率、做早停或交叉验证。

## R18

源码级流程拆解（对应 `demo.py`，8 步）：

1. `make_dataset` 用显式可控的非线性规则 + 噪声生成二分类数据，并固定随机种子做分层切分。  
2. `fit` 读取标签均值 `pos_rate`，计算初始 raw score `F0 = log(pos/(1-pos))`。  
3. 用长度为 `n` 的向量 `raw_scores` 存每个样本当前打分，初始全为 `F0`。  
4. 每轮先调用 `sigmoid(raw_scores)` 得到当前概率，再算伪残差 `residual = y - p`。  
5. 训练 `RegressionStump` 去拟合 `(X, residual)`，在各特征候选阈值中搜索最小平方误差分裂。  
6. 用树预测训练集增量 `update = tree.predict(X)`，并执行 `raw_scores += eta * update`。  
7. 将该树按顺序存入 `self.trees_`；推理时 `decision_function` 逐棵树累加相同更新。  
8. `main` 计算 baseline 与最终 `log-loss/accuracy`，并断言“损失下降 + 精度提升 + 数值有限”，完成可验证闭环。

说明：本实现没有调用任何现成 GBDT 黑箱训练器；GBDT 的关键迭代链路（残差构造、树桩拟合、加法更新）均在源码中显式展开。
