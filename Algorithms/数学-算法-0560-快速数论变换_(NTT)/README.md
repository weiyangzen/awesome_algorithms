## R01

快速数论变换（Number Theoretic Transform, NTT）是“在有限域上做 FFT”的离散变换，用于把多项式卷积从 `O(n^2)` 降到 `O(n log n)`。

- 输入:
  - 两个多项式系数序列 `A(x), B(x)`（整数或模 `p` 的系数）
  - 适配 NTT 的素数模 `p` 与原根 `g`
- 输出:
  - 卷积系数 `C(x) = A(x) * B(x) (mod p)`
- 核心机制:
  - 选长度 `N`（通常为 2 的幂），并满足 `N | (p - 1)`
  - 在模 `p` 下存在 `N` 次单位根 `omega`
  - 正变换把系数域转到点值域，逐点相乘后做逆变换返回系数域
  - 全流程只做整数模运算，避免浮点误差

## R02

NTT 的思想来源于 Cooley-Tukey FFT（1965）的分治结构，但把复数单位根替换为有限域单位根。  
在计算机代数与竞赛编程中，NTT 随着“友好素数”（如 `998244353 = 119 * 2^23 + 1`）的普及而成为标准工具。  
现代密码学（如格密码的环上多项式运算）也广泛使用 NTT 及其变体。

## R03

提出背景是“高效计算多项式乘法/卷积”。

- 朴素卷积复杂度为 `O(n^2)`，当 `n` 上万时成本过高
- 复数 FFT 可做到 `O(n log n)`，但会引入浮点舍入与精度管理问题
- NTT 的回答是：在模素数域里复用 FFT 的蝶形结构，同时保持严格离散精确

## R04

设变换长度为 `N`（补零后 2 的幂）。

- 单次 NTT 或 INTT 时间复杂度: `O(N log N)`
- 两个多项式卷积（2 次正变换 + 1 次逆变换 + 逐点乘）:
  - 总复杂度: `O(N log N)`
- 相比朴素卷积 `O(nm)`，当输入规模增大时优势显著

## R05

空间复杂度通常为 `O(N)`。

- 需要保存补零后的两个长度 `N` 数组
- 迭代就地 NTT 本身可在数组内原地完成
- 若保留输入不变，工程上常额外拷贝一份，仍为线性空间

## R06

示例：`A=[1,2,3]`，`B=[4,5]`，目标卷积长度为 `4`。

1. 朴素结果应为 `[4,13,22,15]`
2. NTT 选 `N=4`（最小满足 `N >= 3+2-1` 的 2 次幂）
3. 把 `A,B` 补零到长度 4：`[1,2,3,0]`、`[4,5,0,0]`
4. 对两者做 NTT，得到频域表示 `A_hat, B_hat`
5. 逐点相乘 `C_hat[i] = A_hat[i] * B_hat[i] mod p`
6. 对 `C_hat` 做 INTT，取前 4 项，得到 `[4,13,22,15]`

该结果与朴素卷积一致，说明流程正确。

## R07

NTT 的价值在于“把大规模离散卷积变成可工程化的标准组件”。

- 理论价值: 保留 FFT 级别复杂度，同时提供模域精确性
- 工程价值: 避免浮点误差和复杂的误差补偿
- 应用价值: 大整数乘法、形式幂级数、计数型 DP、密码学多项式运算

## R08

先导知识/算法依赖：

- 模运算、快速幂、费马小定理（求逆元）
- 原根与单位根概念（需要 `N | (p-1)`）
- FFT 的分治/蝶形结构（bit-reversal + 分层合并）
- 多项式卷积定义与补零策略

## R09

适用前提与边界：

- 变换长度 `N` 必须是 2 的幂（本 MVP 实现约束）
- 且 `N` 必须整除 `p-1`，否则不存在所需 `N` 次单位根
- 输入系数应先规约到 `mod p`
- 若结果可能超过单模范围（例如需要整数精确恢复），要使用多模 NTT + CRT
- 本条目使用 `p=998244353`、`g=3`，支持 `2^23` 以内长度

## R10

正确性依据可分三层：

1. **存在性**：当 `N | (p-1)` 时，有限域中存在 `N` 次原单位根 `omega`
2. **线性变换可逆性**：NTT 矩阵是 Vandermonde 型，节点两两不同，因此可逆
3. **卷积定理**：系数域卷积对应频域逐点乘积；即  
   `INTT(NTT(A) .* NTT(B)) = A * B (mod p)`

算法实现上，逆变换最后乘 `N^{-1} mod p`，与理论定义一致。

## R11

`N/A + 原因`：经典 NTT 在整数模域内计算，不经过浮点数，不存在 FFT 常见的舍入误差累积问题。  
补充：工程风险主要是“模数/长度不匹配”与整数溢出（在低级语言中），而非数值稳定性问题。

## R12

真实成本（超越渐近阶）：

- 主成本是蝶形中的模加/模减/模乘
- 常数项来源：
  - bit-reversal 重排
  - 每一层 `w_len` 的幂更新
  - Python 大整数 `%` 的解释器开销
- 实务优化：
  - 固定友好模数与原根
  - 迭代原地实现，减少临时对象
  - 批量任务时可复用预计算旋转因子表（twiddle factors）

## R13

`N/A + 原因`：NTT 是确定性精确算法，不是近似算法。  
给定相同输入、模数、长度与实现，输出唯一且可重复，不涉及概率正确性或近似比。

## R14

鲁棒性与常见失效模式：

- 失效模式 1：`N` 不是 2 的幂（当前实现不支持）  
  - 缓解：自动向上补到 2 的幂
- 失效模式 2：`N` 不整除 `p-1`  
  - 缓解：更换模数/长度，或改用多模方案
- 失效模式 3：跨模语义混淆（输入未规约）  
  - 缓解：统一执行 `x % p`
- 失效模式 4：目标为“整数精确卷积”，但单模发生模环绕  
  - 缓解：多模 NTT + CRT 恢复

## R18

源码追踪与算法拆解（本条目未使用第三方黑箱；流程可直接映射到 `demo.py`）：

1. **卷积尺寸确定**：`convolution_ntt` 先计算输出长度 `len(a)+len(b)-1`，并选最小 2 次幂 `N`。  
2. **输入补零与规约**：把两组系数映射到 `mod MOD`，扩展到长度 `N`。  
3. **位逆序重排**：`_bit_reverse_permute` 就地交换元素，为迭代蝶形做布局准备。  
4. **逐层蝶形（正变换）**：`ntt(..., invert=False)` 从长度 2 开始翻倍，按层执行模加/模减/模乘。  
5. **频域逐点乘**：两组频域系数做 `fa[i] = fa[i] * fb[i] mod MOD`。  
6. **逐层蝶形（逆变换）**：`ntt(..., invert=True)` 使用逆单位根重复同样结构。  
7. **归一化**：逆变换末尾统一乘 `N^{-1} mod MOD`，得到正确系数。  
8. **裁剪输出**：只保留前 `out_size` 项，丢弃补零引入的尾部。  
9. **正确性与性能验证**：`correctness_demo` 对拍 `convolution_naive`，`scaling_demo` 输出耗时与蝶形计数。
