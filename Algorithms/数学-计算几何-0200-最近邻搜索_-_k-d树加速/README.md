# 最近邻搜索 - k-d树加速

- UID: `MATH-0200`
- 学科: `数学`
- 分类: `计算几何`
- 源序号: `200`

## R01

算法目标: 在二维点集中，对给定查询点 `q` 快速返回最近点 `p*`，使欧氏距离 `||p* - q||` 最小。  
本条目使用 `k-d tree` 构建空间索引，通过“先走近侧、再按超平面距离回溯”的剪枝策略加速最近邻搜索。

## R02

问题形式化:
- 输入: 点集 `P = {p_i | p_i in R^2}`，查询点 `q in R^2`
- 输出: `argmin_{p_i in P} ||p_i - q||_2`
- 距离度量: 欧氏距离（实现中比较平方距离，避免重复开方）
- 场景: 静态点集，多次查询（先建树，再重复检索）

## R03

核心思想:
- 构建阶段: 按层交替选择切分轴（x/y），每层选中位点作为当前节点，形成近似平衡的二叉空间划分树。
- 查询阶段: 先进入与查询点同侧的子树获取当前最优解；然后判断另一侧是否可能存在更优点。
- 剪枝条件: 若查询点到切分超平面的平方距离 `axis_diff^2 >= best_dist2`，则远侧子树不可能改进当前最优解，可直接剪枝。

## R04

数据结构设计:
- `KDNode(index, axis, left, right)`
- `index`: 节点对应点在原数组中的索引
- `axis`: 当前节点切分轴（`0 -> x`, `1 -> y`）
- `left/right`: 左右子树
- 点坐标不冗余存储在节点内，而是统一存于 `points`，减少对象体积并方便向量化基线比对

## R05

构建流程（递归）:
1. 输入当前候选索引集合 `indices`，若为空返回 `None`。
2. 令 `axis = depth % 2`。
3. 按 `points[indices, axis]` 排序并取中位索引作为当前根。
4. 左半索引递归构建左子树，右半索引递归构建右子树。
5. 返回根节点。

## R06

最近邻查询流程（递归）:
1. 访问当前节点并计算 `dist2 = ||point - query||^2`，若更小则更新最优解。
2. 根据 `axis_diff = query[axis] - point[axis]` 判断近侧/远侧子树。
3. 先递归近侧子树，以尽快缩小 `best_dist2`。
4. 再判断 `axis_diff^2 < best_dist2`：成立则递归远侧，否则剪枝。
5. 递归结束后返回全局最优索引与最小平方距离。

## R07

复杂度分析:
- 构建复杂度（当前实现）: `O(n log^2 n)`  
  原因: 每层对子问题做排序，递推 `T(n)=2T(n/2)+O(n log n)`。
- 查询复杂度: 平均约 `O(log n)`，最坏 `O(n)`。
- 空间复杂度: `O(n)`（节点 + 递归栈）。

## R08

正确性要点:
- 最优性维护: 每访问一个节点都与当前最优距离比较，确保 `best_dist2` 单调不增。
- 剪枝安全性: 远侧子树任一点在当前轴向与查询点的距离下界为 `|axis_diff|`；若该下界已不小于当前最优距离，则远侧不可能产生更优解。
- 等价验证: `demo.py` 中固定查询 + 200 次随机查询与暴力法逐次对照，断言最近距离一致。

## R09

MVP 范围:

- 单点最近邻（1-NN）查询
- 包含构建、正确性校验、粗粒度性能对比
- 不包含删除/插入、近似最近邻、高维自适应优化

## R13

与暴力法对比:
- 暴力法: 每次查询计算与全部点的距离，复杂度 `O(n)`，实现简单且鲁棒。
- k-d tree: 通过空间划分显著减少访问节点数，在低维与点分布较均匀时通常更快。
- 退化情形: 高维、点分布极端、或查询点位置导致剪枝失效时，k-d tree 优势减弱。

## R14

工程注意事项:
- 使用固定随机种子保证实验可复现。
- 采用平方距离比较，减少 `sqrt` 开销与浮点噪声。
- 对“并列最近点”场景，验证时以最近距离一致为主，不强制索引一致。
- 排序使用稳定 `mergesort`，在重复坐标时行为更可预测。

## R15

测试策略:
- 固定单查询: 快速检查路径逻辑与剪枝分支。
- 200 次随机查询: 与暴力法逐次对比最近距离，保证正确性。
- 500 次批量查询: 比较耗时并检查距离校验和一致，避免只测功能不测统计。

## R16

已知局限:
- 当前仅二维；维度升高时 k-d tree 剪枝效率明显下降（维度灾难）。
- 构建过程每层排序，复杂度不是最优工程版本。
- Python 递归在极端退化树上可能受递归深度限制。

## R17

可扩展方向:
- 支持 `k` 近邻（k-NN）: 维护大小为 `k` 的候选堆并修改剪枝阈值。
- 支持动态更新: 周期性重建或使用批量插入后重平衡。
- 高性能实现: 使用 `scipy.spatial.cKDTree` / `sklearn.neighbors.KDTree` 做工程加速。
- 高维替代: Ball Tree、HNSW、FAISS（近似）等更适合高维向量检索。

## R18

源码级算法流（`demo.py`，非黑盒拆解）:
1. `main()` 生成二维点集 `points` 与索引数组 `0..n-1`，调用 `build_kdtree` 构建索引树。
2. `build_kdtree` 在每个递归层按 `depth % 2` 选择切分轴，对当前索引子集按该轴排序并取中位索引建节点。
3. 同一函数分别对中位点左右两侧索引递归，形成近似平衡的二叉空间分割结构。
4. 查询由 `kd_nearest_neighbor` 启动，先用根节点初始化 `(best_idx, best_dist2)`，再调用 `_nearest_search`。
5. `_nearest_search` 每访问一个节点就计算查询点到该节点的平方距离，若更优则更新当前最优解。
6. 根据 `axis_diff = query[axis] - node_point[axis]` 先递归“近侧子树”，尽快缩小 `best_dist2`。
7. 用 `axis_diff^2 < best_dist2` 判断是否需要访问“远侧子树”；不满足则安全剪枝。
8. 递归返回后得到最近邻；`main()` 以 `brute_force_nearest` 做随机查询对照断言，并输出访问节点比例与耗时对比。
