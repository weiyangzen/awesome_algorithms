# 虚功原理 (Principle of Virtual Work)

- UID: `PHYS-0110`
- 学科: `物理`
- 分类: `分析力学`
- 源序号: `110`
- 目标目录: `Algorithms/物理-分析力学-0110-虚功原理_(Principle_of_Virtual_Work)`

## R01

虚功原理用于处理带约束的力学系统静力平衡：在所有满足约束的任意虚位移 `δr` 上，理想约束力不做虚功，系统平衡当且仅当外力（非约束力）总虚功为零。该方法的价值是把“未知约束反力”从方程中消去，只在可行位移方向上做判定。

## R02

本目录 MVP 的问题定义：
- 系统：长度 `l` 的单摆，摆球质量 `m`，受重力 `g` 与水平外力 `F_x`。
- 约束：摆球始终在圆弧上，`|r| = l`。
- 求解目标：求静力平衡角 `θ*`，并验证约束力（拉力）在虚位移上的虚功为 0。
- 输出：
  1. 数值求根得到的 `θ*`；
  2. 解析解 `atan2(F_x, m g)`；
  3. 在平衡点与扰动点的虚功分解。

## R03

符号与约定：
- 坐标：`r(θ) = [l sinθ, -l cosθ]`，`θ=0` 表示竖直向下。
- 虚位移：`δr = (∂r/∂θ) δθ`，其中 `∂r/∂θ = [l cosθ, l sinθ]`。
- 力：
  - 重力 `F_g = [0, -m g]`
  - 水平外力 `F_x_vec = [F_x, 0]`
  - 约束拉力 `T_vec = T[-sinθ, cosθ]`

## R04

虚功原理在该 1 自由度系统上的形式：

`δW = (F_g + F_x_vec + T_vec) · δr = 0`

由于理想约束满足 `T_vec · δr = 0`，故平衡条件化为：

`(F_g + F_x_vec) · (∂r/∂θ) = 0`

也就是广义力 `Q_θ = 0`。

## R05

代入可得：

`Q_θ = (F_g + F_x_vec) · [l cosθ, l sinθ]`

`Q_θ = l(F_x cosθ - m g sinθ)`

平衡角满足：

`F_x cosθ* - m g sinθ* = 0`

`tanθ* = F_x / (m g)`，解析解可写为 `θ* = atan2(F_x, m g)`。

## R06

MVP 算法（数值版）
1. 定义函数 `Q_θ(θ) = l(F_x cosθ - m g sinθ)`。
2. 用解析解附近两点作为 secant 初值，调用 `scipy.optimize.root_scalar` 求 `Q_θ(θ)=0`。
3. 在 `θ*` 处计算虚位移 `δr`，分别计算 `W_g, W_x, W_T`。
4. 检查 `W_T≈0` 且 `W_g+W_x≈0`。
5. 在 `θ*+Δ` 处重复计算，展示偏离平衡后总虚功不再为零。

## R07

正确性要点：
- 几何正交性：`T_vec` 与切向虚位移方向正交，保证约束力虚功恒为零。
- 平衡判据：`Q_θ=0` 与“任意 `δθ` 下总虚功为零”等价。
- 数值与解析对照：`root_scalar` 结果应与 `atan2(F_x, m g)` 一致到机器精度量级。

## R08

复杂度分析：
- 单次 `Q_θ` 计算为 `O(1)`。
- secant 求根迭代步数通常 < 20，因此单场景复杂度近似 `O(1)`。
- 空间复杂度 `O(1)`。

## R09

数值稳定性与工程处理：
- 使用 `atan2` 生成初值可避免 `mg≈0` 时直接做比值导致不稳。
- secant 法不依赖导数，减少手写导数误差。
- 输出时给出残差 `Q_θ(θ*)` 与拉力虚功 `W_T`，便于快速诊断。

## R10

`demo.py` 的函数划分：
- `pendulum_kinematics`：返回位置、切向导数、径向单位向量。
- `generalized_force`：实现 `Q_θ`。
- `virtual_work_breakdown`：分解重力/外力/拉力虚功。
- `solve_equilibrium_theta`：数值求解平衡角。
- `run_case`：单参数场景执行并打印结果。

## R11

## R13

输出解读：
- `theta_numeric` 与 `theta_analytic`：数值解与解析解对比。
- `Q(theta_eq)`：平衡点广义力残差，应接近 0。
- `W_tension(eq)`：平衡点约束力虚功，应接近 0。
- `W_total(eq)`：平衡点总虚功，应接近 0。
- `W_total(perturbed)`：扰动点总虚功，通常非 0。

## R14

自检清单：
- `F_x = 0` 时，`θ*` 应接近 0。
- `F_x > 0` 时，`θ* > 0`；`F_x < 0` 时，`θ* < 0`。
- 任意场景下 `W_tension` 数值量级应远小于外力虚功分量。
- 数值解与解析解差值应在 `1e-10 ~ 1e-8` 量级（与机器精度和迭代终止准则相关）。

## R15

边界与局限：
- 本例是静力、单自由度、理想约束。
- 未覆盖非理想约束（如干摩擦约束）与非完整约束。
- 对极端参数（如 `g→0`）时，平衡判据物理意义会退化，需额外建模处理。

## R16

可扩展方向：
- 扩展到多自由度系统，使用 `Q_i = Σ_j F_j · ∂r_j/∂q_i`。
- 联合拉格朗日乘子法同时求解状态与约束反力。
- 从静力扩展到达朗贝尔原理，处理动力学平衡（惯性力并入虚功项）。

## R17

与相邻方法比较：
- 牛顿受力平衡：直接列力平衡，通常需要显式求约束反力。
- 虚功原理：只在可行虚位移方向判据，天然消去理想约束力。
- 拉格朗日方程：可视为虚功/达朗贝尔原理在广义坐标下的系统化结果。

## R18

本实现的源码级算法流（8 步）：
1. `run_case` 读取参数并调用 `solve_equilibrium_theta`。
2. `solve_equilibrium_theta` 内部构造闭包 `q(theta)`，其核心调用 `generalized_force`。
3. `generalized_force` 直接计算 `l(F_x cosθ - m g sinθ)`，这是虚功原理约化后的平衡方程。
4. `root_scalar(method="secant")` 用解析解附近两点迭代，返回 `θ*`。
5. `run_case` 在 `θ*` 处调用 `virtual_work_breakdown`，令 `δr = (∂r/∂θ)δθ`。
6. `virtual_work_breakdown` 调用 `pendulum_kinematics` 得到切向与径向，再分别计算 `W_g, W_x, W_tension`。
7. 由于 `T_vec` 与切向正交，代码中 `W_tension = T_vec·δr` 自动接近零，形成对“理想约束不做虚功”的数值验证。
8. `run_case` 再在 `θ*+0.15` 处重复虚功计算，展示偏离平衡时 `W_total ≠ 0`，最终打印可核验结果。
