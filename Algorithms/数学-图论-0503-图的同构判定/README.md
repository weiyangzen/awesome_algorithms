# 图的同构判定

- UID: `MATH-0503`
- 学科: `数学`
- 分类: `图论`
- 源序号: `503`
- 目标目录: `Algorithms/数学-图论-0503-图的同构判定`

## R01

图的同构判定（Graph Isomorphism, GI）问题：

- 输入两张图 `G1=(V1,E1)` 与 `G2=(V2,E2)`；
- 判断是否存在一个双射 `f: V1 -> V2`，使得任意点对 `(u,v)` 满足：
  - `(u,v) in E1` 当且仅当 `(f(u),f(v)) in E2`。

本目录实现的是**无向、简单、顶点带离散标签**图的同构判定 MVP。

## R02

`demo.py` 的固定输入由代码内构造，不需要交互：

- `Case-1`：同构图对（第二张图由第一张图顶点重标号得到）；
- `Case-2`：非同构图对（结构边集被破坏）；
- 以及一组随机小图对拍（与暴力解对照）。

输出包括：

- 是否同构（`True/False`）；
- 一组映射 `u -> v`（若存在）；
- 搜索统计（扩展节点、回溯次数、前向剪枝次数）。

## R03

复杂度背景：

- 朴素算法是枚举全部顶点排列，复杂度 `O(n!)`；
- 图同构问题已知属于 `NP`，但目前没有被证明是 `P`，也没有被证明是 `NP-Complete`；
- 理论上存在准多项式时间算法（Babai 2015 方向），但工程实现较复杂。

本 MVP 目标是教学可读性，不追求最强理论或工业极致性能。

## R04

本实现采用“三段式”流程：

1. **不变量过滤**：点数、边数、度序列、标签多重集不一致则立即失败；
2. **1-WL 成对颜色细化**：迭代压缩“当前颜色 + 邻居颜色多重集”；
3. **回溯搜索**：按颜色类候选做 MRV 选点，并用邻接一致性 + 前向检查剪枝。

该方案比纯暴力更稳健，同时保留源码可追踪性。

## R05

图结构定义（`Graph`）：

- `adj: numpy.ndarray`，`n x n` 邻接矩阵（`0/1`）；
- `labels: Tuple[str, ...]`，每个顶点一个离散标签；
- 默认无自环、无重边、无向图（矩阵对称）。

选择邻接矩阵是为了让“邻接一致性检查”和“重标号后矩阵比较”直接明了。

## R06

不变量过滤规则（必要条件）：

- `|V1| == |V2|`；
- `|E1| == |E2|`；
- `sorted(deg(G1)) == sorted(deg(G2))`；
- 顶点标签多重集相同。

任一不满足即可返回非同构，避免无谓搜索。

## R07

1-WL 成对细化（`pair_color_refinement`）：

- 初始颜色 token：`(label(u), degree(u))`；
- 迭代 token：`(color(u), multiset(color(neighbors(u))))`；
- 对两图 token **联合压缩**到统一 color id（保证可比）；
- 每轮比较颜色类计数直方图，若不一致则直接判非同构；
- 直到颜色稳定（不再变化）。

这一步通常能显著收缩回溯空间。

## R08

回溯策略（`backtrack_isomorphism`）：

- 候选只在同颜色类内选择；
- 选点使用 MRV：优先扩展“可行候选最少”的未映射顶点；
- 局部一致性：对已映射点 `mu -> mv`，要求  
  `adj1[u,mu] == adj2[v,mv]`（边与非边都保持）；
- 前向检查：每个未映射顶点至少还要保留一个可行候选，否则剪枝。

最终映射满覆盖时即找到同构。

## R09

正确性要点（简述）：

- 完备性：回溯遍历了所有满足颜色类与单射约束的候选分支，不会漏解；
- 可靠性：每次扩展都强制局部邻接保持，最终完整映射必满足图同构定义；
- 安全剪枝：前向检查仅在“未来无候选”时剪枝，不会删除可行解路径。

因此算法返回 `True` 当且仅当存在合法同构映射。

## R10

演示样例设计：

- 固定正例：`g2 = permute_graph(g1, perm)`，应返回同构；
- 固定反例：在相同标签集合下删改关键边，应返回非同构；
- 随机对拍：小规模图上与 `brute_force_isomorphic` 对照，检验实现正确性。

这样可同时覆盖“命中解、拒绝假解、随机鲁棒性”。

## R11

代码结构：

- `Graph`：图数据结构与构图；
- `invariant_check`：快速必要条件过滤；
- `pair_color_refinement`：成对 1-WL 细化；
- `choose_next_vertex`：MRV 选点；
- `locally_consistent`：局部邻接一致性；
- `forward_check`：前向可行性检查；
- `backtrack_isomorphism`：DFS 回溯；
- `solve_graph_isomorphism`：统一入口；
- `brute_force_isomorphic`：小图暴力校验；
- `run_fixed_demo` / `run_random_crosscheck`：演示与对拍。

## R13

输出解释：

- `result=True`：存在同构；
- `mapping={...}`：返回一组可行双射（不保证唯一）；
- `expanded`：尝试扩展的候选映射次数；
- `backtracks`：回溯触发次数；
- `pruned`：被前向检查提前剪掉的分支数。

这些统计便于观察剪枝效果和搜索代价。

## R14

MVP 边界：

- 支持：无向简单图、顶点离散标签、判定与单解映射；
- 不支持：
  - 边标签/边权；
  - 有向图、多重边、自环；
  - 大规模工业求解器优化（如 nauty/traces 级别 canonical labeling）。

目标是“短小、诚实、可运行”的学习型实现。

## R15

常见实现错误：

- 只校验“边保持”，忘记“非边也要保持”（这会退化成子图同构口径）；
- 回溯时遗漏单射约束（同一目标点被重复使用）；
- 仅靠度序列判同构，忽略结构细节导致误判；
- 缺少负例和随机对拍，错误难以暴露。

`demo.py` 通过暴力对照降低这些风险。

## R16

与常见方法关系：

- 本实现：`1-WL + Backtracking`，强调直观与可解释；
- VF2 / VF2++：状态维护更精细，工程中常见；
- nauty/traces：基于 canonical labeling / partition refinement，性能更强但实现复杂；
- 理论先进算法（如 GI 准多项式方向）不适合作为入门 MVP。

因此这里优先选择可读性与可验证性。

## R17

可扩展方向：

1. 加入边标签和属性一致性约束；
2. 引入更强剪枝（如更细 partition refinement、双向可行性规则）；
3. 支持枚举全部同构映射而非返回首解；
4. 切换到 canonical form 输出，做图去重与哈希索引；
5. 在更大图上加入位集优化或 C 扩展加速。

## R18

`demo.py` 源码级算法流程（8 步）：

1. `build_fixed_cases` 构造固定正例/反例图，`main` 触发整套演示。  
2. `solve_graph_isomorphism` 先调用 `invariant_check` 做点数、边数、度序列、标签多重集过滤。  
3. 若通过，进入 `pair_color_refinement`：按“当前颜色 + 邻居颜色多重集”对两图联合压缩颜色，并逐轮比较颜色类计数。  
4. 颜色细化稳定后，按 `colors2` 建立目标图颜色类桶 `class_to_v2`。  
5. `backtrack_isomorphism` 使用 `choose_next_vertex`（MRV）选取候选最少的未映射顶点 `u`。  
6. 对候选 `v`，`locally_consistent` 检查其与已映射点之间的边/非边关系是否完全一致。  
7. 临时加入 `u -> v` 后执行 `forward_check`，确认其余未映射顶点仍至少有一个合法候选；否则立即剪枝回溯。  
8. 若全部顶点映射完成则返回同构映射；`run_random_crosscheck` 再用 `brute_force_isomorphic` 在小图上随机对拍验证实现正确性。
