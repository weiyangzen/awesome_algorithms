## R01

GMRES（Generalized Minimal Residual）用于求解一般非对称线性方程组 `Ax=b`，核心目标是在 Krylov 子空间中直接最小化残差范数。

给定初值 `x0` 与初始残差 `r0=b-Ax0`，第 `m` 步 GMRES 在
`x0 + K_m(A,r0)` 中寻找近似解 `xm`，使：

`||b - A xm||_2` 最小。

它适合“矩阵可能非对称、但 matvec（矩阵-向量乘）可高效执行”的场景。

## R02

历史定位（简要）：

- GMRES 由 Saad 与 Schultz 在 1986 年提出；
- 它把 Arnoldi 子空间构造与最小二乘残差优化结合起来；
- 在非对称稀疏线性系统中是最常见的 Krylov 方法之一；
- 工程上常用重启版本 `GMRES(m)` 与预条件版本（左/右预条件）。

## R03

本条目 MVP 解决的问题：

- 输入：
  - 实方阵 `A`；
  - 右端向量 `b`；
  - 初值 `x0`（默认零向量）；
  - 最大 Krylov 维度 `max_iter`；
  - 收敛阈值 `tol`；
- 输出：
  - 近似解 `x`；
  - 收敛标记与迭代次数；
  - 相对残差 `||b-Ax||/||b||`；
  - 残差历史 `residual_history`；
  - Arnoldi 关系误差（验证用）。

## R04

时间复杂度（稠密矩阵）：

- 每步一次 `A @ v`，代价 `O(n^2)`；
- 每步正交化约 `O(jn)`（第 `j` 步）；
- 总体近似 `O(m n^2 + m^2 n)`。

稀疏矩阵时，matvec 代价可降为 `O(nnz(A))`，总计近似：
`O(m * nnz(A) + m^2 n)`。

## R05

空间复杂度：

- Krylov 基 `V` 存储 `n x (m+1)`，`O(nm)`；
- Hessenberg `Hbar` 大小 `(m+1) x m`，`O(m^2)`；
- 旋转系数与辅助向量 `O(m)`。

因此 GMRES 额外内存主要由 `V` 主导，约 `O(nm)`。

## R06

直觉示例（二维情形）：

- 第 1 步只在 `x0 + span{r0}` 上选点，等价于“沿一个方向找最小残差”；
- 第 2 步扩展到 `x0 + span{r0, Ar0}`，可调空间更大，残差不会变差；
- 继续扩展 Krylov 空间，直到达到目标精度或预算上限。

所以 GMRES 的本质是“每次多给一个自由方向，再做一次最优残差拟合”。

## R07

算法意义：

- 可处理非对称、非正定系统（相对 CG 的适用面更宽）；
- 每一步都显式最小化当前子空间残差，收敛过程可监控；
- 与 Arnoldi 深度绑定，便于扩展到重启和预条件框架。

## R08

核心公式：

1. `r0 = b - A x0`, `beta = ||r0||`, `v1 = r0 / beta`。
2. Arnoldi：`A V_m = V_{m+1} Hbar_m`。
3. 令 `xm = x0 + V_m y`，残差可写为：
   `||b - A xm|| = ||beta e1 - Hbar_m y||`。
4. 因而每一步转化为小规模最小二乘：
   `min_y ||beta e1 - Hbar_m y||_2`。
5. 用 Givens 旋转逐列消元，可在线更新残差并避免每步重做完整最小二乘。

## R09

适用条件与局限：

适用：

- 大规模稀疏线性系统；
- 非对称/非正定矩阵；
- 可以高效执行 matvec（或线性算子接口）。

局限：

- 未重启时内存与正交化代价随 `m` 上升；
- 谱性质差时可能收敛慢；
- 有限精度下正交性损失会影响残差估计质量。

## R10

正确性要点（工程可验证版）：

1. Arnoldi 生成近似正交基 `V`，并建立关系 `A V_m = V_{m+1} Hbar_m`。  
2. 近似解被约束为 `x0 + V_m y`，把原问题降成小规模最小二乘。  
3. Givens 旋转把 `Hbar_m` 逐步上三角化，同时同步更新右端项。  
4. 上三角回代得到 `y`，进而得到 `xm`。  
5. `demo.py` 同时检查真残差与 Arnoldi 关系误差，保证实现闭环正确。

## R11

数值稳定性：

- 使用 Modified Gram-Schmidt（MGS）并支持二次重正交选项；
- 使用 Givens 旋转逐步处理最小二乘，避免显式法方程放大条件数；
- 以 `tol * ||b||` 作为停止标准，兼顾尺度；
- 对接近 breakdown 的情况用阈值保护，避免除零。

## R12

性能视角：

- 主耗时在 `A @ v` 与正交化内积循环；
- 小 `m` 时计算和内存都较轻；
- 若要求长迭代，通常采用 `GMRES(m)`（重启）平衡收敛速度与资源；
- 预条件器质量往往决定实际收敛效率上限。

## R13

本目录 MVP 的可验证保证：

- 自动输出并检查：

## R14

常见失效模式：

- `A` 非方阵或 `b` 维度不匹配；
- `b=0`、`tol<=0`、`max_iter<=0` 等非法输入；
- 条件数高或谱分布不利导致收敛慢；
- 正交化不足时，残差历史与真实残差可能出现偏离。

## R15

实现设计（`demo.py`）：

- `GMRESResult`：封装解、残差、Krylov 信息与诊断指标；
- `validate_inputs`：参数与维度校验；
- `compute_givens`：构造二维旋转消去下元素；
- `gmres_solve`：显式实现 Arnoldi + Givens + 回代，不依赖黑盒 GMRES；
- `build_nonsymmetric_system`：生成可复现测试方程组；
- `run_checks`：统一断言与失败早停；
- `main`：执行实验、打印报告。

## R16

相关算法链路：

- Arnoldi：GMRES 的子空间构造内核；
- FOM：同样基于 Krylov，但优化目标不同；
- CG：对称正定特化，内存更省；
- BiCGSTAB：另一类非对称求解器，单步开销与稳定性权衡不同；
- 预条件 GMRES：工业实践的主流版本。

## R18

`demo.py` 的源码级算法流程（9 步，非黑盒）如下：

1. `main` 构造确定性非对称矩阵 `A` 与真解 `x_true`，计算 `b=A x_true`。  
2. `gmres_solve` 调用 `validate_inputs`，初始化 `x0`、`r0`、`beta`、Krylov 基 `V` 与 Hessenberg `H`。  
3. 每轮先做 `w = A @ v_j`，再用 MGS 把 `w` 对已有基向量逐个正交化，填充 `H(i,j)`。  
4. 可选二次重正交进一步削弱浮点误差导致的正交性退化。  
5. 把之前累计的 Givens 旋转作用到新列 `H(:,j)`，保持“逐步上三角化”的结构。  
6. 新建当前列的 Givens 旋转，消去 `H(j+1,j)`，并同步更新右端向量 `g`。  
7. 由 `|g(j+1)|/||b||` 在线得到相对残差，满足阈值则提前停止。  
8. 迭代结束后对上三角系统 `R y = g_head` 回代，得到 `x = x0 + V_m y`。  
9. 在 `main` 中计算真残差、真解误差与 Arnoldi 关系误差，并由 `run_checks` 统一验收。  

说明：本实现没有调用 `scipy.sparse.linalg.gmres` 等一行黑盒接口；GMRES 主流程（Arnoldi、旋转更新、回代）全部在源码中展开实现。
