## R01

ElGamal 加密是基于离散对数困难问题的公钥加密方案，1985 年由 Taher ElGamal 提出。  
它的核心特征是“随机化加密”：同一明文在同一公钥下多次加密会得到不同密文。

## R02

算法工作在有限域乘法群 `Z_p*`（`p` 为大素数）上。  
公开参数通常为 `(p, g)`，其中 `g` 是群生成元；私钥为 `x`，公钥为 `h = g^x mod p`。

## R03

安全性基础：若攻击者无法高效求解离散对数（由 `h = g^x mod p` 反推出 `x`），则难以恢复私钥。  
在随机 `k` 使用正确的前提下，语义安全可由 DDH（判定型 Diffie-Hellman）假设支撑。

## R04

密钥生成：
1. 选择大素数 `p` 与生成元 `g`。  
2. 随机选私钥 `x ∈ [1, p-2]`。  
3. 计算 `h = g^x mod p`。  
4. 公钥为 `(p, g, h)`，私钥为 `x`。

## R05

加密单个消息元素 `m`（要求 `0 <= m < p`）：
1. 随机选一次性随机数 `k ∈ [1, p-2]`。  
2. `c1 = g^k mod p`。  
3. `s = h^k mod p`。  
4. `c2 = m * s mod p`。  
5. 密文为 `(c1, c2)`。

## R06

解密 `(c1, c2)`：
1. 计算共享因子 `s = c1^x mod p`。  
2. 计算逆元 `s_inv = s^{-1} mod p`。  
3. 恢复 `m = c2 * s_inv mod p`。  
由费马小定理可用 `s^(p-2) mod p` 求逆（`p` 为素数且 `s != 0`）。

## R07

正确性说明：
`c2 = m * h^k = m * (g^x)^k = m * (g^k)^x = m * c1^x (mod p)`，  
所以 `c2 * (c1^x)^(-1) = m (mod p)`，解密可还原原文。

## R08

时间复杂度（单消息元素）：
- 加密：2 次模幂 + 1 次模乘，约 `O(log p)`。  
- 解密：1 次模幂 + 1 次逆元 + 1 次模乘，约 `O(log p)`。  
空间复杂度常数级，不含密钥存储。

## R12

MVP 输出预期：
- `Recovered == Original`。  
- 同一明文重复加密得到不同密文（概率型加密特性）。  
- 若断言失败会抛异常，便于自动化验证。

## R13

与 RSA 对比（简述）：
- 两者都是公钥体制；ElGamal 天生随机化，抗明文模式泄露更直接。  
- ElGamal 密文膨胀约 2 倍（每个块变成 `(c1,c2)`），RSA 通常 1 块 1 密文。  
- 现代实践中常将 ElGamal 思想用于 KEM/混合加密而非直接大消息加密。

## R14

工程注意点：
- 生产中必须使用足够大的安全参数（如 2048-bit 以上群）。  
- 严禁重复随机数 `k`；重复会泄漏私钥信息。  
- 需使用 CSPRNG、参数校验、侧信道防护与完整性保护（如 AEAD 或签名）。

## R15

常见错误：
- 把明文映射到 `m>=p` 且未做分块。  
- 误用弱随机源（`random` 而非 `secrets`）。  
- 忽略群参数合法性（`g` 不是生成元导致子群攻击风险）。

## R16

可扩展方向：
- 改成“大整数分块 + 填充编码”以支持任意长度消息。  
- 改为“混合加密”：ElGamal 仅加密会话密钥，数据走对称加密。  
- 增加性能基准，对不同 `p` 位数测试吞吐与延迟。

## R17

测试建议：
- 单元测试：字节边界值 `0/1/255`，以及空字节串。  
- 性质测试：多次随机消息加解密后恒等。  
- 统计测试：同明文多次加密，密文分布应呈随机离散变化。

## R18

源级算法流（对应 `demo.py`，6 步）：
1. 参数校验：检查 `p` 为素数、`g` 为 `Z_p*` 的生成元，避免退化群。  
2. 生成密钥：`x <- secrets`，计算 `h = pow(g, x, p)`，形成公私钥对。  
3. 加密字节：对每个 `m` 随机采样 `k`，计算 `c1 = g^k mod p` 与 `c2 = m*h^k mod p`。  
4. 解密字节：用私钥算 `s = c1^x mod p`，再求 `s_inv = s^(p-2) mod p`，恢复 `m`。  
5. 序列封装：把字节级函数映射到 `bytes` 序列，得到 `[(c1,c2), ...]`。  
6. 端到端验证：在 `main()` 中执行“加密->解密->断言相等”，并额外验证概率型加密特性。
