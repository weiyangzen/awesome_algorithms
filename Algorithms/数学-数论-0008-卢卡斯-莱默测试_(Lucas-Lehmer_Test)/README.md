# 卢卡斯-莱默测试 (Lucas-Lehmer Test)

## R01

卢卡斯-莱默测试用于判定梅森数 `M_p = 2^p - 1` 是否为素数（其中 `p` 通常要求是素数）。

- 核心思想: 构造递推序列 `s_0 = 4`，`s_{k+1} = s_k^2 - 2`，并在模 `M_p` 下迭代 `p-2` 次。
- 输入: 指数 `p`（整数，理论保证要求 `p` 为素数）。
- 输出: `M_p` 是否为素数的布尔结论。
- 关键判据: 对素数 `p`，`M_p` 为素数当且仅当 `s_{p-2} ≡ 0 (mod M_p)`。

## R02

该测试来源于 19-20 世纪关于梅森素数的研究脉络：

- 1870s: Édouard Lucas 给出针对梅森数的判定思想（常追溯到 1876/1878 年相关工作）。
- 1930: Derrick H. Lehmer 对该判定做了现代形式化与计算实现整理。
- 因此该方法通常称为 Lucas-Lehmer Test。

## R03

历史背景是“高效寻找大素数”。

- 直接对 `M_p` 做试除或通用素性测试，随着 `p` 增长会迅速不可承受。
- 梅森数有特殊结构 `2^p-1`，可用专门判定替代通用判定。
- Lucas-Lehmer 测试把问题压缩为固定形式的模平方迭代，显著提升了该类数的判定效率，是大规模梅森素数搜索（例如 GIMPS）的基础方法。

## R04

令 `n = p`（`M_p` 的比特长度约为 `p`）。

- 迭代次数: `p-2` 次。
- 每次迭代: 一次大整数平方 + 模约减。
- 复杂度写法: `O(p * M(p))`，其中 `M(p)` 是 `p` 比特乘法复杂度。
- 若用朴素乘法 `M(p)=O(p^2)`，则总复杂度约 `O(p^3)`。
- 若用 Karatsuba/Toom- Cook/FFT 等快速乘法，实际可优于 `O(p^3)`。

## R05

额外空间主要保存常数个 `p` 比特整数（`s`、`M_p`、临时乘积）。

- 渐近额外空间: `O(p)` 比特级别（若按字长计可记为 `O(p / word_size)` 字）。
- 若启用 trace 记录全部中间序列，则空间增加到 `O(p^2)` 比特量级（因为保存约 `p` 个 `p` 比特状态）。

## R06

示例一（素数情况）: `p=7`，`M_7=127`。

- `s_0=4`
- `s_1=(4^2-2) mod 127 = 14`
- `s_2=(14^2-2) mod 127 = 67`
- `s_3=(67^2-2) mod 127 = 42`
- `s_4=(42^2-2) mod 127 = 111`
- `s_5=(111^2-2) mod 127 = 0`
- 因为 `s_{p-2}=s_5=0`，故 `127` 为素数。

示例二（合数情况）: `p=11`，`M_11=2047`，迭代到 `s_9` 不为 0（demo 中可复现），因此 `2047` 为合数。

## R07

算法意义体现在“专用且强力”：

- 理论上: 给出梅森数素性的充要判据（在 `p` 为素数条件下）。
- 工程上: 是目前梅森素数大规模搜索的核心判定内核之一。
- 学科上: 连接了数论、代数结构与高性能大整数计算。

## R08

路径依赖的前置知识/工具：

- 基本数论: 若 `p` 合数，则 `2^p-1` 必合数（可先过滤指数）。
- 模算术与快速模乘: 支撑 `s = (s*s-2) mod M_p` 的核心迭代。
- Lucas 序列相关理论: 支撑最终“余数为 0 即素数”的判据。
- 指数素性检测: 实现中常先判断 `p` 是否为素数，避免无效 LL 迭代。

## R09

适用前提与边界：

- 只适用于梅森数 `M_p=2^p-1`，不是通用素性测试。
- 理论充要判据建立在 `p` 为素数时；实践中应先验证 `p` 素性。
- `p=2` 是边界特例（`M_2=3` 为素数），实现需单独处理。
- 对极大 `p`，时间与内存需求仍然很高，需要高性能实现与分布式环境。

## R10

正确性依据是经典 Lucas-Lehmer 定理：

- 对素数 `p>2`，定义 `s_0=4`，`s_{k+1}=s_k^2-2`。
- 有 `M_p` 为素数 `iff s_{p-2} ≡ 0 (mod M_p)`。
- 证明思路依赖有限域/代数整数上的序列性质与群结构阶性质。
- 工程实现只要忠实执行该递推与模运算，即可继承理论正确性。

## R11

该算法是整数模运算，不涉及浮点近似：

- 浮点舍入误差: 无。
- 误差传播: 无数值分析意义下的舍入传播，结果是离散且精确的。
- 主要风险转化为“实现正确性”问题（例如溢出、错误模约减、边界分支漏判），而非数值稳定性问题。

## R12

真实成本主要由大整数算术常数项决定：

- 每轮一次大整数平方与取模，常数成本高于普通整型运算。
- Python 的 `%` 会调用通用大整数除法路径；若针对 `2^p-1` 写专门约减，可进一步降常数。
- 缓存与内存带宽会影响超大 `p` 的实测性能。
- 并行性: 单个 `p` 的递推链基本串行；多 `p` 候选可任务级并行。
- 与 R04 一致: 渐近上是 `p` 轮大整数操作，实际瓶颈在乘法/约减实现。

## R13

该算法不是近似/随机算法：

- 在 `p` 为素数且实现正确时，结论是确定性、零误判（充要条件）。
- 不提供概率保证，因为不依赖随机化。
- 若跳过“`p` 必须为素数”的前置条件，则理论保证不成立；实现层面应先做指数素性过滤。

## R14

常见失效模式与鲁棒性问题：

- 漏掉 `p=2` 特判，可能误报 `M_2` 非素数。
- 在定长整数语言里发生溢出，导致错误结论（Python 大整数可规避）。
- 未先筛掉合数指数 `p`，会产生无意义计算甚至错误解释。
- 为追求性能引入自定义模约减时，若实现有 bug，会直接破坏正确性。

## R15

工程实现要点（对应 `demo.py`）：

- 先做 `p` 的素性筛查（小中规模可用试除法）。
- `M_p` 用位运算构造: `(1 << p) - 1`，避免幂函数开销。
- 主循环仅维护一个状态 `s`，空间占用低。
- 提供 trace 开关用于教学/调试，默认关闭以降低内存开销。
- 用一组已知指数（如 `2,3,5,7,11,13...`）做自检断言，防回归。

## R16

前驱、后继与应用：

- 前驱脉络: 梅森数试除、Lucas 序列理论、基础模算术优化。
- 后继扩展: Lucas-Lehmer-Riesel（广义形如 `k*2^n-1`）、其他专用素性测试与分布式实现框架。

## R17

Python MVP 方案（见 `demo.py`）：

- 工具栈: 仅 Python 3 标准库（`time`）；利用内建 arbitrary-precision int 完成大整数运算。
- 关键函数:
  - `is_prime_small(n)`: 指数素性筛查。
  - `lucas_lehmer_test(p, trace=False)`: 执行 LL 判定并可选返回中间序列。
  - `main()`: 批量跑若干 `p`，输出表格、耗时和 trace 示例。
- 运行方式: 在该目录执行 `python3 demo.py`，无需交互输入。
- MVP 目标: 小而诚实，覆盖正确性、自检、可解释输出三件事。

## R18

源码级流程拆解（对应 `demo.py`，共 6 步）：

1. 读取候选指数 `p`，先由 `is_prime_small` 做指数素性筛查。  
2. 若 `p < 2` 或 `p` 非素数，直接返回“非 LL 适用/非梅森素数”路径。  
3. 处理边界 `p=2`：直接给出 `M_2=3` 为素数的结果。  
4. 构造 `M_p = (1 << p) - 1`，初始化 `s=4`。  
5. 执行 `p-2` 轮递推 `s = (s*s - 2) % M_p`（可选记录 trace）。  
6. 检查末状态 `s==0`：为 0 则 `M_p` 素数，否则为合数，并在 `main()` 中汇总打印与断言校验。
