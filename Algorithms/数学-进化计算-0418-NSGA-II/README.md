## R01

`NSGA-II (Non-dominated Sorting Genetic Algorithm II)` 是经典多目标进化算法，目标是在一次优化中同时逼近多条相互冲突目标的 Pareto 前沿。
本目录提供一个最小可运行 MVP，聚焦二目标 `ZDT1` 测试问题，验证算法主流程可用。

## R02

要解决的问题是：在不把多个目标硬合成为单一加权和的前提下，同时优化多个目标函数，并得到一组互不支配（non-dominated）的候选解。
示例里目标为：
- `f1(x)` 越小越好
- `f2(x)` 越小越好

## R03

NSGA-II 的三个核心机制：
- 快速非支配排序：按 Pareto 层级（Front 0, Front 1, ...）给个体分级。
- 拥挤距离：在同一 Front 内估计邻域稀疏度，优先保留更“分散”的解。
- 精英保留：父代与子代合并后再截断，避免优良解在进化中丢失。

## R04

以最小化为例，若向量目标 `a` 满足：
- 对任意目标维度 `m`，`a_m <= b_m`
- 且至少存在一个维度 `k`，`a_k < b_k`
则称 `a` 支配 `b`。

ZDT1（本 demo 使用）定义：
- `f1(x) = x1`
- `g(x) = 1 + 9 * mean(x2...xn)`
- `f2(x) = g * (1 - sqrt(f1 / g))`
- 决策变量约束：`x_i in [0,1]`

## R05

高层伪代码：

```text
初始化种群 P
评估目标 F(P)
for t in 1..T:
  对 P 做非支配排序 + 拥挤距离
  用二元锦标赛选择父母
  通过 SBX 交叉 + 多项式变异生成子代 Q
  评估 F(Q)
  R = P ∪ Q
  对 R 做非支配排序
  依据 Front 优先 + 拥挤距离截断，得到新 P
输出 P 中的第一前沿作为近似 Pareto 集
```

## R06

时间复杂度（每代主导项）：
- 非支配排序：`O(MN^2)`，`M` 为目标维度，`N` 为种群规模。
- 拥挤距离计算：对每个 Front 每个目标排序，合计通常小于排序主项。
- 交叉/变异：约 `O(ND)`，`D` 为决策维度。

因此在典型设置中，主要瓶颈是 `O(MN^2)` 的排序与支配比较。

## R07

空间复杂度主要来自：
- 种群与子代：`O(ND)`
- 目标矩阵：`O(NM)`
- 排序中临时结构（支配集合、计数、front 列表）：约 `O(N^2)` 最坏

MVP 中通过 `numpy` 数组承载主要数据，结构直观、便于调试。

## R08

参数含义（demo 默认值）：
- `pop_size=80`：种群规模，越大覆盖越广但计算更慢。
- `generations=80`：迭代代数，越大前沿通常更接近真实 Pareto。
- `crossover_prob=0.9`：交叉概率。
- `eta_c=15`：SBX 交叉分布指数，越大越接近父母。
- `eta_m=20`：多项式变异分布指数。
- `mutation_prob=None`：自动使用 `1/n_var`。

## R09

实现边界与假设：

- 单线程 CPU 执行。

## R11

结果解读建议：
- 关注 `Approximate Pareto front size`，确认存在多解而非退化到单点。
- 查看输出的 `(f1, f2)`，应呈现互相权衡关系：`f1` 下降时 `f2` 常上升（或反向）。
- 多次更换种子可观测前沿采样分布变化。

## R12

正确性快速检查清单：
- 所有目标值应为有限实数（非 `NaN` / `Inf`）。
- 前沿中任意两点不应互相支配。
- 随迭代增加，前沿一般会更贴近 ZDT1 理论曲线 `f2 = 1 - sqrt(f1)`（当 `g=1` 时）。

## R13

常见失败模式：
- 种群过小：前沿离散且抖动明显。
- 变异过强：搜索像随机游走，前沿不稳定。
- 变异过弱：早熟收敛，覆盖范围窄。
- 只看单次运行：容易被随机性误导。

## R14

与相近方法对比：
- 相比 NSGA-I：NSGA-II 通过拥挤距离替代共享参数，调参更轻。
- 相比加权和法：可一次得到整条前沿，不依赖权重网格。
- 相比 MOEA/D：NSGA-II 更偏“排序+筛选”范式，MOEA/D 偏分解协同子问题。

## R15

工程实践建议：
- 先在小规模参数下验证流程正确，再放大种群和代数。
- 固定随机种子保证可复现，再增加多种子统计实验。
- 若要提速，可优先优化非支配排序（向量化/并行化/近似策略）。

## R18

`demo.py` 源码级算法流程（8 步）：

1. `main()` 构造 `NSGA2Config`，设置种群规模、代数、交叉/变异参数与随机种子。
2. `run_nsga2()` 随机初始化 `pop`（`[0,1]^n`），调用 `evaluate_population()` 计算初始目标矩阵 `objs`。
3. `evaluate_population()` 逐个体调用 `zdt1_objectives()`，得到每个解的 `(f1, f2)`。
4. 每一代先调用 `assign_rank_and_crowding()`：
   - `fast_non_dominated_sort()` 计算每个体 rank（Front 层级）
   - `crowding_distance()` 计算同层拥挤距离
5. `make_offspring()` 基于 `binary_tournament()` 进行父代选择；比较规则是“rank 小优先，rank 相同则 crowding 大优先”。
6. 对选中父母执行 `sbx_crossover()` 生成子代，并在 `polynomial_mutation()` 中做逐维扰动（边界裁剪到 `[0,1]`）。
7. 父代与子代合并后，`environmental_selection()` 再做一次非支配排序；按 Front 从优到次填充下一代，最后一层按拥挤距离降序截断。
8. 迭代结束后 `summarize_pareto()` 提取第一前沿并打印，形成可观察的近似 Pareto 结果。

这 8 步对应了 NSGA-II 的核心机制，不依赖第三方黑箱优化器调用。
