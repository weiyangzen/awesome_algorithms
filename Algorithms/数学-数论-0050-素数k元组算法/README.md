# 素数k元组算法

- UID: `MATH-0050`
- 学科: `数学`
- 分类: `数论`
- 源序号: `50`
- 目标目录: `Algorithms/数学-数论-0050-素数k元组算法`

## R01

素数 k 元组（Prime k-tuples）研究的是一组固定偏移量模式 `H={h1,...,hk}`，是否存在无穷多个整数 `n` 使得 `n+h1,...,n+hk` 全是素数。典型例子是孪生素数 `(p,p+2)`、三元组 `(p,p+2,p+6)`、四元组 `(p,p+2,p+6,p+8)`。

本目录的 MVP 给出一个“可运行、可解释、可扩展”的最小实现：
- 用筛法快速判素；
- 用“可容许模式（admissible pattern）”做先验合法性检查；
- 在有限上界内枚举所有 k 元组；
- 用 Hardy-Littlewood 风格近似计数做结果对照。

## R02

问题定义（有限计算版）：
- 输入：
  - 上界 `N`（要求元组中所有素数 `<= N`）；
  - 偏移模式 `H={h1,...,hk}`，其中 `hi >= 0` 且互异。
- 输出：
  - 所有满足 `n+hi` 全为素数的 k 元组；
  - 计数 `count(N,H)`；
  - 近似计数 `estimate(N,H)`（启发式）。

## R03

采用的数学前置：
- 可容许性：若某个素数模 `p` 下，`{-hi mod p}` 覆盖全部剩余类，则该模式不可能长期成立。
- 对于 `p > k`，`k` 个偏移无法覆盖 `p` 个剩余类，因此检查 `p <= k` 即可判定可容许性。
- Hardy-Littlewood 启发：
  - 计数近似为 `S(H) * Integral( dt / (log t)^k )`；
  - 其中 `S(H)` 是奇异级数（本实现做有限素数截断）。

## R04

算法总览：
1. 规范化偏移模式（排序、去重、非负性检查）。
2. 检查模式可容许性（小素数模约束）。
3. 构造 `0..N+max(H)` 的素数布尔表（埃拉托斯特尼筛）。
4. 扫描 `base` 从 `2` 到 `N-max(H)`，验证 `base+hi` 是否全为素数。
5. 汇总 k 元组并输出计数。
6. 计算奇异级数截断乘积与对数积分离散和，给出近似计数。

## R05

核心数据结构：
- `bytearray is_prime`：筛法生成的素性表，`is_prime[x] in {0,1}`。
- `tuple[int,...] pattern`：偏移模式的规范表示。
- `list[tuple[int,...]]`：枚举到的全部 k 元组。

这些结构保证代码简单、内存可控、便于后续替换为分段筛或并行版本。

## R06

正确性要点（MVP 级）：
- 筛法正确性：埃拉托斯特尼筛会删除所有合数并保留所有素数。
- 枚举正确性：对每个 `base` 精确检查 `base+hi` 的素性，满足条件即加入结果，不会漏报或误报。
- 可容许性判定正确性：按定义检查所有 `p <= k` 的覆盖情况；若出现满覆盖即判不可容许。

## R07

复杂度分析：
- 建筛：时间 `O(N log log N)`，空间 `O(N)`（这里实际为 `N+max(H)`）。
- 枚举：`O((N-max(H)) * k)`。
- 奇异级数近似：若截断到 `P`，约 `O(P log log P + pi(P)*k)`。

当 `k` 小、`N` 在几十万到几百万量级时，该实现能在普通 CPU 上快速完成。

## R08

边界与异常处理：
- 空模式、负偏移、重复偏移会抛出 `ValueError`。
- `upper_bound < 2` 返回空结果。
- 若外部传入的 `prime_table` 长度不足，会抛出 `ValueError`。
- 不可容许模式不会阻止运行，但会在输出中明确标记。

## R09

MVP 取舍说明：
- 选择纯 Python + 标准库，避免额外依赖，保证 `python3 demo.py` 即可运行。
- 近似计数仅做启发式对照，不作为严格证明。
- 当前主要展示算法结构和数论约束，不追求超大规模性能极限。

## R10

`demo.py` 的函数职责：
- `canonicalize_pattern`：模式合法化。
- `build_prime_table`：筛法生成素性表。
- `is_admissible_pattern`：可容许性判定。
- `find_prime_k_tuples`：在上界内枚举 k 元组。
- `singular_series_approx`：奇异级数截断乘积。
- `logarithmic_integral_k`：离散化对数积分。
- `hardy_littlewood_estimate`：综合近似计数。
- `main`：运行三种经典模式并打印结果。

## R12

输出解读：
- `admissible=True/False`：模式是否满足可容许条件；
- `实际计数`：枚举到的真实个数；
- `近似计数(HL截断)`：基于截断奇异级数 + 对数积分的启发式估计；
- `前8个样例`：用于人工快速校验。

通常会看到：
- 孪生素数数量最多；
- 随着 `k` 增大，元组显著稀疏；
- 近似值与真实值同量级，但不要求精确贴合。

## R13

建议最小测试集：
- 合法模式：`(0,2)`、`(0,2,6)`、`(0,2,6,8)`。
- 非法输入：空模式、负偏移、重复偏移。
- 不可容许模式：例如 `(0,1)`（模 2 覆盖全部剩余类）。
- 小上界回归：`N=30` 时可手工核对全部输出。

## R14

可调参数：
- `upper_bound`：控制搜索范围与运行时间。
- `prime_cutoff`：控制奇异级数截断深度。
- `patterns`：可替换为任意 k 元组偏移模式。

工程上建议：先小 `N` 做正确性回归，再扩大 `N` 做统计观察。

## R18

源码级算法流（对应 `demo.py`，8 步）：
1. `main` 定义 `upper_bound` 与模式列表，并计算所有模式的最大偏移量。  
2. `build_prime_table(upper_bound + max_offset)` 一次性生成素数布尔表，供多模式复用。  
3. 对每个模式调用 `is_admissible_pattern`：先规范化，再检查所有 `p <= k` 的模覆盖。  
4. 若模式不可容许，直接打印说明并跳过枚举（避免无意义搜索）。  
5. 调用 `find_prime_k_tuples` 扫描 `base in [2, upper_bound-max_offset]`，逐偏移检查 `is_prime[base+d]`。  
6. 将满足条件的 `(base+d1,...,base+dk)` 追加到结果列表，得到真实计数。  
7. 调用 `hardy_littlewood_estimate`：内部先算 `singular_series_approx`，再乘 `logarithmic_integral_k`。  
8. 输出可容许性、实际计数、近似计数和前若干样例，形成“理论-实验”对照。
