# 布隆过滤器

## R01

布隆过滤器（Bloom Filter）是一种空间高效的概率型集合结构，回答的问题是：
- 某个元素“可能在集合中”；
- 某个元素“一定不在集合中”。

它允许假阳性（False Positive），但不允许假阴性（False Negative）。
在数据库系统中，布隆过滤器常用于减少不必要的磁盘访问、网络传输或连接探测。

## R02

核心机制：
1. 维护长度为 `m` 的位数组（初始全 0）；
2. 使用 `k` 个哈希函数将元素映射到 `k` 个位置；
3. 插入时把这些位置设为 1；
4. 查询时若任一位置为 0，则元素一定不存在；若都为 1，则“可能存在”。

常用近似公式：
- 理论假阳性率：`p ~= (1 - exp(-k*n/m))^k`；
- 给定期望元素数 `n` 和目标假阳性率 `p`，可取：
  - `m = -n ln(p) / (ln 2)^2`
  - `k = (m/n) ln 2`

## R03

本条目 MVP 解决的问题：
- 输入：一批待插入键、一批负样本查询键；
- 输出：
1. 可运行的 Bloom Filter 实现（构建 + 查询）；
2. 理论与实测假阳性率对比；
3. 无假阴性校验与位数组密度校验；
4. 样例查询表。

实现目标是“源码透明可追踪”，而不是调用黑箱库。

## R04

`demo.py` 的实现策略：
1. 用 `numpy` 的 `uint8` 数组作为位图；
2. 用 `hashlib.blake2b` 生成双哈希基值 `h1/h2`；
3. 通过 `h_i = (h1 + i*h2 + i^2) mod m` 生成 `k` 个位置；
4. `add` 逐位写 1，`might_contain` 逐位检查；
5. 用公式自动选取 `m/k`；
6. 对固定数据集做断言和统计输出；
7. 使用 `pandas` 展示可读样例查询表。

## R05

关键数据结构：
- `BloomFilter.bits: np.ndarray[uint8]`：位数组；
- `BloomFilter.bit_size/hash_count`：结构参数 `m/k`；
- `QueryRecord`：单次查询结果记录（真值、预测、是否假阳性）；
- `pandas.DataFrame`：查询样例表。

## R06

正确性直觉：
- 插入阶段：每个元素都会把自身对应的 `k` 个位设为 1；
- 查询阶段：若某位为 0，说明该元素插入时不可能覆盖此位，因此可确定“不存在”；
- 所以不会出现假阴性；
- 但多个元素会共享位，可能把某个未插入元素的全部 `k` 位“碰巧”置 1，于是假阳性可出现。

## R07

复杂度（`k` 为哈希次数）：
- 插入：`O(k)`；
- 查询：`O(k)`；
- 空间：`O(m)`。

在固定 `m/k` 下，该复杂度与已插入元素数近似解耦，是布隆过滤器的主要工程价值。

## R08

边界与异常处理：
- `bit_size <= 0` 或 `hash_count <= 0`：抛 `ValueError`；
- `expected_items <= 0` 或 `target_fp_rate` 不在 `(0,1)`：抛 `ValueError`；
- 双哈希步长 `h2` 若取模后为 0，强制修正为 1，避免退化；
- 使用固定编码（UTF-8）确保字符串键哈希稳定。

## R09

MVP 设计取舍：
- 采用“位图 + 哈希”最小实现，不引入计数布隆过滤器或分层结构；
- 重点验证三件事：
1. 无假阴性；
2. 假阳性率数量级合理；
3. 位密度与理论接近。
- 不追求并发写、删除支持、分布式合并等工程特性。

## R10

`demo.py` 函数职责：
- `BloomFilter._hash_pair`：为键生成双哈希基值；
- `BloomFilter._positions`：生成 `k` 个位位置；
- `BloomFilter.add`：插入元素；
- `BloomFilter.might_contain`：成员查询；
- `choose_parameters`：按目标误判率反推 `m/k`；
- `theoretical_fp_rate`：计算理论假阳性率；
- `evaluate_queries`：生成样例查询记录；
- `main`：构造实验、断言、打印结果。

## R12

输出说明：
- `bit_size(m), hash_count(k)`：Bloom Filter 参数；
- `theoretical_fp_rate / observed_fp_rate`：理论与实测假阳性率；
- `expected_bit_density / observed_bit_density`：理论与实测位密度；
- `Sample query table`：样例查询真值与模型输出；
- `All checks passed...`：全部校验通过。

## R13

最小验证清单：
1. 所有已插入键查询结果必须为 `True`（无假阴性）；
2. 负样本查询的实测假阳性率不能显著高于理论值 + 容忍带；
3. 实测位密度应接近 `1-exp(-k*n/m)`；
4. 样例表中 `in_ground_truth=False 且 bloom_result=True` 的行仅作为假阳性，不应影响整体正确性结论。

## R14

当前固定实验参数：
- 期望元素数：`expected_items = 250`；
- 目标假阳性率：`target_fp_rate = 0.02`；
- 自动计算得到：`m` 与 `k`（运行时打印）；
- 插入键：`user:0000` 到 `user:0249`；
- 负样本键：`probe:0000` 到 `probe:2999`。

这组参数足以稳定观察假阳性现象，同时保持脚本运行轻量。

## R15

与相关结构对比：
- 哈希集合（`set`）：零误判但内存更高；
- 哈希索引：可返回真实 RID/记录指针，Bloom Filter 只做“存在性预筛”；
- 计数布隆过滤器：支持删除但占用更多空间；
- Cuckoo Filter：也支持近似成员测试，删除能力更好但实现更复杂。

## R16

适用场景：
- 数据库 Join 前过滤（减少无效探测）；
- LSM-Tree/SSTable 读路径预筛；
- 缓存系统的“可能命中”前置判定。

不适用场景：
- 必须返回精确成员集合或记录位置；
- 业务不能接受任何假阳性；
- 需要频繁删除且不想引入计数结构。

## R17

可扩展方向：
- 支持计数布隆过滤器（删除）；
- 支持分片位图与并发原子更新；
- 引入可序列化存储与版本管理；
- 对比不同哈希策略与负载下的误判曲线；
- 与数据库查询计划器联动，按统计信息动态启停 Bloom 过滤。

## R18

`demo.py` 源码级算法流（8 步，非黑箱）：
1. `main` 先调用 `choose_parameters`，根据 `expected_items` 和目标误判率计算位图长度 `m` 与哈希次数 `k`。  
2. 构造 `BloomFilter(m, k)`，内部创建 `numpy` 位数组并初始化为全 0。  
3. 对每个插入键调用 `add`：`_hash_pair` 先用两次 `blake2b` 求 `h1/h2`，`_positions` 再生成 `k` 个下标并把对应位写 1。  
4. 对全部已插入键调用 `might_contain`，逐个检查 `k` 位是否全为 1，并断言结果全为真，从而验证“无假阴性”。  
5. 对负样本键重复查询，统计假阳性个数并计算实测假阳性率。  
6. 调用 `theoretical_fp_rate` 计算理论误判率，比较“实测值 vs 理论值 + 容忍带”并做断言。  
7. 根据 `1-exp(-k*n/m)` 计算理论位密度，与 `bits.mean()` 的实测位密度做一致性校验。  
8. `evaluate_queries` 生成样例查询记录，`pandas` 打印结果表，最后输出 `All checks passed.`。
