## R01

树上 DSU（DSU on Tree，也常叫 Sack）是一种用于处理“子树统计类问题”的离线技巧。  
典型任务：对每个节点 `u`，统计其整棵子树中的某个频次指标，例如：
- 不同颜色个数；
- 出现次数最多的颜色；
- 颜色频次满足阈值的类别数。

本目录 MVP 选择最经典问题：**求每个节点子树内不同颜色数**。

## R02

核心思想是“小并大”（small-to-large）：
- 每个节点选一个重儿子（子树最大）；
- 先递归处理轻儿子并清空贡献；
- 再处理重儿子并保留贡献；
- 最后把轻儿子子树重新加回当前统计结构。

这样每个节点作为“轻子树节点”被搬运的次数是受限的，整体复杂度能控制在 `O(n log n)`（常见实现）或接近线性（依赖数据结构与常数优化）。

## R03

问题定义（本实现口径）：
- 输入：
  - 一棵 `n` 个点、根为 `0` 的无向树；
  - 每个节点一个颜色值 `color[u]`（非负整数）。
- 输出：
  - `ans[u]`，表示以 `u` 为根的子树中，不同颜色的数量。

这是一个典型“对子树批量求值”的场景，不适合对每个 `u` 都单独 DFS 统计（会退化到 `O(n^2)`）。

## R04

MVP 算法流程：
1. `dfs_size` 预处理：
   - 子树大小 `size[u]`；
   - 重儿子 `heavy[u]`；
   - Euler 序区间 `[tin[u], tout[u])`。
2. `dfs_sack(u, p, keep)` 执行树上 DSU：
   - 轻儿子先算并丢弃；
   - 重儿子后算并保留；
   - 把轻儿子整段 Euler 区间加回频次桶；
   - 加入当前节点自身；
   - 记录 `ans[u]`；
   - 若 `keep=False`，再把当前子树贡献整体清空。

频次桶用 `dict[color] -> count`，并维护 `distinct_colors` 作为实时不同颜色数。

## R05

复杂度分析：
- 预处理 `dfs_size`：`O(n)`；
- 主过程 `dfs_sack`：每个节点被“轻子树搬运”次数受限，整体 `O(n log n)`（实现常见上界）；
- 额外空间：
  - 树结构、Euler 序、辅助数组均为 `O(n)`；
  - 频次桶最坏 `O(n)`。

因此总空间 `O(n)`，时间通常写作 `O(n log n)`。

## R06

示例：
- 树边：`(0,1),(0,2),(1,3),(1,4),(2,5),(2,6)`
- 颜色：`[1,2,1,3,2,1,4]`

则：
- `subtree(0)` 颜色集合 `{1,2,3,4}`，答案 `4`；
- `subtree(1)` 颜色集合 `{2,3}`，答案 `2`；
- `subtree(2)` 颜色集合 `{1,4}`，答案 `2`；
- 叶子节点答案都为 `1`。

`demo.py` 会打印该固定样例并断言结果。

## R07

树上 DSU 的价值在于：
- 把大量“子树统计查询”从朴素 `O(n^2)` 压到 `O(n log n)`；
- 思路直接、代码长度适中，适合竞赛和工程基线实现；
- 可扩展到多种统计对象，不限于“不同颜色数”。

## R08

前置知识：
- DFS 与树形 DP 的基本递归框架；
- 重儿子定义（按子树大小最大）；
- Euler Tour 把子树映射为连续区间；
- 哈希表/计数桶维护频次。

## R09

适用边界：
- 适用：
  - 静态树；
  - 离线批量子树查询；
  - 查询函数可由“可加可减”的计数结构维护。
- 不适用：
  - 树结构频繁动态修改（增删边）；
  - 需要在线路径查询（更偏向 HLD、LCT、树剖线段树等）。

本 MVP 假设输入是一棵连通无环树，根固定为 `0`。

## R10

正确性直觉：
1. `dfs_size` 正确建立了每个节点的 Euler 子树区间，子树节点等价于 `order[tin[u]:tout[u])`。
2. `dfs_sack` 中，重儿子贡献被保留，轻儿子贡献通过区间重放补齐，因此在记录 `ans[u]` 时，频次桶恰好对应 `u` 的完整子树。
3. `keep=False` 时清空当前子树，保证返回父层后不污染兄弟分支。
4. 每次颜色计数变化同步维护 `distinct_colors`，因此 `ans[u]` 即为子树不同颜色数。

由递归结构可得全体节点答案正确。

## R11

实现稳健性要点：
- 输入边数必须等于 `n-1`，否则抛错；
- 顶点编号越界时抛错；
- 递归前设置 `sys.setrecursionlimit` 以避免较深树触发默认栈限制；
- 用“增减同一接口”更新频次，避免分支路径遗漏造成计数不一致。

## R12

性能调优建议：
- 颜色离散且范围小，可把 `dict` 改为数组计数，减少哈希常数；
- Euler 区间批量遍历是热点，可尽量减少 Python 层函数调用；
- 若只需单点答案，可只计算目标子树，避免全树遍历。

本目录先强调可读性与可验证性，不做激进微优化。

## R13

保证类型：
- 本算法是确定性算法，不依赖随机化；
- 在输入满足“树 + 合法编号”前提下，输出与朴素子树统计一致；
- 复杂度保证是渐进上界（常规实现 `O(n log n)`）。

## R14

常见失效模式：
- 把一般图误传为树（有环或不连通）；
- 在 `keep` 分支中忘记清空/误清空，导致答案串扰；
- Euler 区间边界写错（`tout` 开闭区间混淆）；
- 颜色更新逻辑未对“从 0 到 1 / 从 1 到 0”做对称处理。

本 MVP 通过输入检查、固定样例断言与随机对拍降低这些风险。

## R15

实践建议：
- 先用小树手算答案，再对照程序输出；
- 新增统计目标时，优先保持“add/remove 可逆”；
- 保留朴素解做随机对拍，是迭代树上 DSU 代码最有效的防回归手段。

## R16

相关方法对比：
- 朴素每点 DFS：实现最简单，但 `O(n^2)`；
- 树上 DSU：离线子树统计高性价比，`O(n log n)`；
- 树剖 + 线段树：更适合路径查询与在线更新；
- 莫队上树：适合更通用离线区间/路径统计，但实现复杂度更高。

## R17

## R18

`demo.py` 的源码级算法流程（8 步）：
1. `run_fixed_demo` 构造一棵 7 节点样例树与颜色数组，并给出手工期望答案。
2. `TreeDSUDistinctColors.__init__` 建图、校验输入、初始化 `size/heavy/tin/tout/order` 与频次桶。
3. `solve` 调用 `_dfs_size`：一次 DFS 计算子树大小、重儿子与 Euler 序区间。
4. `solve` 调用 `_dfs_sack(0, -1, True)` 进入树上 DSU 主递归。
5. 在 `_dfs_sack` 内，先处理所有轻儿子（`keep=False`，返回后贡献被清空），再处理重儿子（`keep=True`，贡献保留）。
6. 将每个轻儿子的 Euler 区间重新加入频次桶，再加入当前节点颜色；此时桶状态恰好对应当前节点完整子树。
7. 记录 `ans[u] = distinct_colors`；若 `keep=False`，通过当前节点 Euler 区间整体减回，实现回溯清理。
8. `run_random_crosscheck` 生成随机树做 60 轮对拍：树上 DSU 结果与朴素 DFS 结果逐轮比较，全部通过后打印成功信息。

整个流程只依赖标准库，没有调用第三方黑盒函数；核心 small-to-large 迁移过程在源码中可逐行追踪。
