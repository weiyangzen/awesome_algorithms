## R01

C4.5 算法是 ID3 的改进版，核心思想是：
- 用信息增益率（Gain Ratio）替代纯信息增益，缓解“高分支数特征偏好”；
- 支持连续特征阈值切分；
- 以递归方式构造决策树。

本目录实现一个可运行、可审计的 C4.5 风格分类 MVP：
- 划分准则：信息增益率；
- 特征类型：连续 + 类别；
- 输出：分类预测、树结构统计、规则样例；
- 取舍：不做后剪枝，保留最小闭环。

## R02

MVP 问题定义（多分类）：

给定训练数据 `D={(x_i,y_i)}_{i=1}^n`，学习决策树分类器 `f(x)`，其中 `y_i` 属于有限类别集合。

在当前节点样本集 `S` 上：
- 节点熵：`H(S) = -sum_k p_k log2 p_k`
- 若按某特征划分成子集 `S_v`，则
  - 信息增益：`Gain = H(S) - sum_v (|S_v|/|S|) H(S_v)`
  - 划分信息：`SplitInfo = -sum_v (|S_v|/|S|) log2(|S_v|/|S|)`
  - 增益率：`GainRatio = Gain / SplitInfo`

每次选择增益率最大的有效划分进行递归建树。

## R03

为什么不直接几行调用现成库：
- C4.5 的学习价值在“候选切分生成 + 增益率计算 + 停止条件”；
- 黑盒 API 难以看到连续特征阈值枚举、类别特征多叉划分、未知类别回退策略；
- 本实现把训练和推理路径都展开到源码层，便于验证和二次开发。

## R04

关键数学与实现细节：

1. 连续特征：
- 排序后在相邻取值中点构造阈值候选；
- 仅在“取值变化且标签变化”的位置尝试候选；
- 按 `<= threshold` 与 `> threshold` 二叉划分，计算增益率。

2. 类别特征：
- 按类别值做多叉划分；
- 每个子节点样本数需满足 `min_samples_leaf`；
- 增益率由该多叉划分一次性计算。

3. 选择规则：
- 先比较 `gain_ratio`；
- 平局时比较 `info_gain`；
- 再平局按特征索引较小者优先，保证确定性。

## R05

高层流程：

1. 校验输入维度、特征类型、连续特征有限性并编码标签。
2. 在节点上计算类别计数、熵和多数类预测。
3. 若满足停止条件（纯节点/深度上限/样本过少）则建叶子。
4. 枚举所有可用特征，计算最优切分及其增益率。
5. 若最优增益率低于阈值 `min_gain_ratio`，建叶子。
6. 连续特征按阈值分成左右子集递归。
7. 类别特征按取值分多叉子集递归，且该类别特征在子树内不再复用。
8. 预测时从根节点按规则下行到叶子输出类别。

## R06

正确性直觉：
- 对当前节点，算法显式比较该节点下的所有有效候选切分，贪心选择增益率最大者；
- 增益率 = 信息增益 / 划分信息，能抑制“取值极多”特征的不合理优势；
- 叶节点预测采用该节点经验分布的多数类，和决策树经验风险最小化思想一致。

注意：这是局部贪心，不保证全局最优树结构。

## R07

复杂度分析（单节点样本数 `m`，特征数 `d`）：
- 连续特征：排序 `O(m log m)`，扫描候选 `O(m)`；
- 类别特征：统计各取值子集并计熵，约 `O(m)`；
- 节点总成本约 `O(d * m log m)`（当连续特征占主导时）。

整棵树训练成本为各节点成本之和；树越深，节点样本数越小。
空间方面主要是树节点存储与局部掩码数组。

## R08

边界与异常处理：
- `X` 必须二维，`y` 必须一维，样本数一致；
- 特征类型只允许 `continuous/categorical`；
- 连续特征列必须全部为有限值；
- 类别数至少为 2；
- `max_depth/min_samples_split/min_samples_leaf/min_gain_ratio` 都做合法性检查；
- 预测时若遇到训练中未出现的类别取值，回退到当前节点多数类。

## R09

MVP 范围与取舍：
- 实现 C4.5 的核心建树机制（增益率 + 连续/类别切分）；
- 不实现后剪枝（如悲观剪枝）；
- 不处理缺失值权重分配；
- 不做并行化和大规模工程优化；
- 优先保证：可运行、可读、可追踪。

## R10

`demo.py` 主要模块职责：
- `SplitDecision`：记录候选切分信息（增益率、增益、阈值等）。
- `C45Node`：树节点结构（叶标记、预测类别、切分信息、子节点）。
- `C45ClassifierMVP.fit`：训练入口。
- `_best_continuous_split`：连续特征阈值候选扫描。
- `_best_categorical_split`：类别特征多叉划分评估。
- `_best_split`：跨特征选择最优切分。
- `_build_tree`：递归建树与停止条件控制。
- `_predict_encoded_one`：单样本路径推理。
- `export_rules`：导出可解释规则。
- `make_mixed_classification_data`：生成混合特征数据集。
- `main`：训练、评估、打印结果并执行质量门槛检查。

## R14

关键参数与调参建议：
- `max_depth`：控制树深，越大越容易过拟合；
- `min_samples_split`：节点继续分裂最低样本数；
- `min_samples_leaf`：每个叶子最少样本数；
- `min_gain_ratio`：增益率阈值，越大越保守。

建议调参顺序：
1. 先控制复杂度：`max_depth`、`min_samples_leaf`；
2. 再调 `min_samples_split`；
3. 最后细调 `min_gain_ratio`。

## R15

与相关方法对比：
- 对比 ID3：C4.5 用增益率替代增益，更稳健；
- 对比 CART：CART 常用 Gini 且严格二叉，C4.5 原生支持类别多叉；
- 对比随机森林/GBDT：单树可解释性更强，但泛化稳定性通常不如集成模型。

## R16

- 连续与类别特征混合的数据建模；
- 作为更复杂树模型前的教学/审计基线。

## R18

`demo.py` 源码级算法流程（9 步）：

1. `main` 调用 `make_mixed_classification_data` 生成混合特征三分类数据，并做分层训练/测试划分。
2. 初始化 `C45ClassifierMVP`，设置 `max_depth/min_samples_split/min_samples_leaf/min_gain_ratio`。
3. `fit` 在 `_validate_and_prepare` 中完成输入检查、特征类型规范化和标签编码。
4. `fit` 调用 `_build_tree`，在每个节点先计算类别计数、熵和默认多数类预测。
5. `_build_tree` 调用 `_best_split` 遍历特征；连续特征走 `_best_continuous_split`，类别特征走 `_best_categorical_split`。
6. `_best_continuous_split` 对列排序后线性扫描候选阈值，计算信息增益、划分信息和增益率，选最优阈值。
7. `_best_categorical_split` 按类别值分组计算多叉划分的加权熵与增益率；`_best_split` 决定全局最佳切分。
8. `_build_tree` 根据最佳切分递归生成子树（连续特征可复用，类别特征在子树内移除），直到触发停止条件。
9. `predict` 通过 `_predict_encoded_one` 沿树路径下行输出类别，`main` 汇总 accuracy/F1/混淆矩阵、打印规则和样本预测，并执行质量断言。
