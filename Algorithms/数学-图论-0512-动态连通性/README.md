# 动态连通性

- UID: `MATH-0512`
- 学科: `数学`
- 分类: `图论`
- 源序号: `512`
- 目标目录: `Algorithms/数学-图论-0512-动态连通性`

## R01

动态连通性（Dynamic Connectivity）关注这样一类在线/离线查询：

- 图的边会随时间被添加或删除；
- 在任意时刻，询问两个点 `u,v` 是否连通。

若操作只有加边，称增量连通性（incremental）；只有删边，称减量连通性（decremental）；二者都有则称全动态连通性（fully dynamic）。本目录实现的是离线全动态版本：先拿到完整操作序列，再统一计算所有查询答案。

## R02

历史脉络可以粗分为三条线：

- 经典并查集解决增量连通性（只加边）；
- Euler Tour Tree、Link-Cut Tree 等动态树结构支持更强在线动态维护；
- 离线场景中，逐步形成“时间分治 + 回滚并查集”套路，用较低实现复杂度处理中等规模全动态连通性问题。

本项目选择第三条路线，因为它实现小、可审计、适合作为算法入门与工程基线。

## R03

问题动机来自“拓扑变化频繁但查询也频繁”的系统：

- 网络链路上下线、故障恢复；
- 社交关系/协作关系的时间演化分析；
- 仿真系统中的接触图连通检测。

如果每次查询都重建图并 BFS/DFS，单次就是 `O(n+m)`，大量查询时成本很高。动态连通性算法的核心目标是把“更新 + 查询”总成本压到可接受范围。

## R04

本目录算法复杂度（`q` 为操作数，`m` 为边生命周期区间数）：

- 构建边活跃区间：`O(q)`；
- 把区间分配到线段树：`O(m log q)`；
- DFS 线段树时，每条区间边在若干节点做并查集合并，总计 `O(m log q)` 次合并；
- 每次 `find/union` 在“按 size 合并 + 无路径压缩”下，最坏高度 `O(log n)`。

因此总体可写为 `O(m log q log n + q)`。在竞赛/工程常数实践中，这个复杂度通常远优于逐查询重算连通性。

## R05

空间复杂度主要由三部分组成：

- 线段树桶数组：`O(q)` 个节点，每节点存若干边引用；
- 所有区间边总存储量：`O(m log q)`（每区间分解到若干线段树节点）；
- 回滚并查集：父节点、大小数组和历史栈，共 `O(n + m log q)` 量级。

总体空间复杂度可记为 `O(n + q + m log q)`。

## R06

一个最小示例（无向图）：

1. `add(0,1)`
2. `add(1,2)`
3. `query(0,2)` -> `True`
4. `remove(1,2)`
5. `query(0,2)` -> `False`

解释：第 3 步时路径 `0-1-2` 存在；第 4 步删掉关键边后，第 5 步不再连通。

## R07

该问题的重要性在于它把“图结构随时间演化”显式纳入算法模型：

- 理论层：连接静态图算法与动态数据结构；
- 工程层：支撑连续事件流上的实时连通性判断；
- 实践层：为更复杂动态图任务（动态 MST、动态割、动态桥）提供基础认知。

## R08

本目录 MVP 的核心思想有两点：

1. 时间轴线段树分治  
每条边只在其“活跃时间区间 `[l,r)`”有效。把这个区间挂到线段树对应节点，DFS 到某时间叶子时，路径上的边集合正是该时刻有效边集。

2. 可回滚并查集  
DFS 进入节点时批量 `union` 当前节点边；离开节点时把并查集回滚到进入前快照。这样无需复制整张并查集状态。

二者组合后就能在离线条件下高效回答任意时刻连通查询。

## R09

适用条件与边界：

- 适用：离线已知全部操作序列；无向图连通查询。
- 可处理：重复加边/删边（本实现用计数法处理多重边活跃度）。
- 不适用：
  - 强在线场景（操作到达时必须立即答复且未来未知）；
  - 需要维护有向图强连通等更复杂性质。

另外，本实现顶点编号要求在 `[0, n-1]`。

## R10

正确性直觉可分两层：

1. 区间覆盖正确  
边 `e` 只在 `[l,r)` 有效；线段树分配保证 `e` 被加入且仅加入覆盖该区间的节点集合。对任意时刻 `t`，根到叶路径恰好包含所有满足 `l <= t < r` 的边。

2. 并查集状态正确  
DFS 进入节点时加入该节点边，离开时回滚到快照。于是到达叶子 `t` 时，并查集连通分量恰是时刻 `t` 的真实连通分量。对 `query(u,v)` 返回 `find(u)==find(v)` 即正确答案。

由这两点可得整套离线求解正确。

## R11

稳定性与实现注意点：

- 回滚并查集不能做路径压缩，否则无法正确撤销；
- 采用“按 size 合并”控制树高，保证 `find` 不会退化太严重；
- 输入操作合法性要检查（如删除不存在边、顶点越界）；
- 自环边 `u==v` 不影响连通分量，可安全忽略合并操作。

本目录 `demo.py` 已覆盖以上防护。

## R12

实际性能受三个因素主导：

- 活跃区间数量与长度（决定线段树桶总边量）；
- 查询占比（查询越多，动态算法收益越明显）；
- 图规模与连边模式（影响并查集 find 深度常数）。

在“中大规模操作序列 + 高频 query”下，离线动态连通性通常比逐 query BFS/DFS 快一个量级以上。

## R13

保证类型：

- 这是确定性算法，不依赖随机化成功概率；
- 在输入合法前提下可保证输出与逐时刻真实连通性一致；
- 复杂度保证为上界型（见 R04），不承诺常数时间更新/查询。

本目录 demo 还用朴素在线 BFS 求解器做随机对拍，作为工程级正确性补强。

## R14

常见失效模式：

- `remove` 了不存在的边：输入语义错误；
- 忘记回滚或回滚粒度错误：会把兄弟区间状态污染；
- 对并查集使用路径压缩：回滚失效，答案可能随机错误；
- 区间右端点处理错（`[l,r)` 写成闭区间）：边生效时刻偏移。

这些都是实现中最容易踩坑的地方。

## R15

工程实现建议（已在 `demo.py` 采用）：

- 边统一规范化为 `(min(u,v), max(u,v))`；
- 用 `active_count + start_time` 支持重复加删同一边；
- 将求解器封装为 `OfflineDynamicConnectivity` 类，接口清晰；
- 用固定样例 + 随机对拍（和朴素解对比）做自测；
- 保持“无交互输入、直接运行即出结果”，便于自动验证。

## R16

方法谱系：

- 前驱：并查集（增量连通）、朴素 BFS/DFS 重算；
- 同类：Euler Tour Tree、Link-Cut Tree（更偏在线、实现更复杂）；
- 后继扩展：
  - 动态最小生成树；
  - 动态桥/割点检测；
  - 带权动态图上的更多查询任务。

离线时间分治 + 回滚 DSU 是进入动态图算法的高性价比起点。

## R17

本目录 `demo.py` 的 MVP 内容：

- 核心数据结构：
  - `RollbackDSU`：快照与回滚；
  - `OfflineDynamicConnectivity`：区间构建、线段树分配、DFS 求解。


运行后会打印固定样例结果和随机对拍通过信息，无需输入参数。

## R18

源码级流程拆解（对应 `demo.py`，8 步）：

1. `OfflineDynamicConnectivity._build_edge_intervals` 扫描操作序列，把每条边转换成若干活跃区间 `[l,r)`。
2. `solve` 创建时间线段树，把每个活跃区间通过 `add_interval` 挂到覆盖该区间的节点桶中。
3. 初始化 `RollbackDSU`，并在 DFS 前准备 `answers_by_time` 容器。
4. DFS 进入节点时记录 `checkpoint = dsu.snapshot()`，并将该节点桶中的边逐条 `union`。
5. 若到达叶子时间点 `t` 且该操作是 `query(u,v)`，直接读取 `dsu.connected(u,v)` 作为该时刻答案。
6. 若是内部节点，则递归处理左右子区间，复用当前已合并状态。
7. 节点处理完成后执行 `dsu.rollback(checkpoint)`，精确撤销本节点新增合并，避免污染其他分支。
8. DFS 结束后按 query 出现顺序收集答案；`main` 中再与 `naive_online_solver` 对拍验证实现正确性。
