## R01

中点椭圆算法（Midpoint Ellipse Algorithm）用于在离散像素网格上高效绘制椭圆边界。它是中点圆算法在椭圆场景下的扩展，核心目标是在每一步只做常数次加减乘，避免昂贵三角函数。

## R02

问题定义：给定椭圆中心 `(xc, yc)`、长半轴 `rx`、短半轴 `ry`（均为正整数），输出一组像素点，使这些点在栅格上逼近隐式方程：

`(x - xc)^2 / rx^2 + (y - yc)^2 / ry^2 = 1`

## R03

算法核心思想：

1. 仅计算第一象限（或第一卦限）上的点。
2. 用椭圆的四向对称性一次生成 4 个像素。
3. 使用“中点判别值”判断下一步应沿 `E`（x 增）还是 `SE`（x 增且 y 减）方向推进。
4. 由于斜率变化，分为两个区域分别迭代。

## R04

椭圆隐式函数可写为：

`F(x, y) = ry^2 * x^2 + rx^2 * y^2 - rx^2 * ry^2`

- 区域 1（斜率绝对值 < 1）：以 x 为主增量，终止条件常写为 `dx < dy`。
- 区域 2（斜率绝对值 >= 1）：以 y 为主减量，直到 `y < 0`。

判别值初始化常见写法：

- `d1 = ry^2 - rx^2 * ry + 0.25 * rx^2`
- `d2 = ry^2 * (x + 0.5)^2 + rx^2 * (y - 1)^2 - rx^2 * ry^2`

## R05

对称性生成规则（以 `(x, y)` 为第一象限点）：

- `( xc + x, yc + y )`
- `( xc - x, yc + y )`
- `( xc + x, yc - y )`
- `( xc - x, yc - y )`

这将每步的有效输出扩大 4 倍，并天然保证图形中心对称。

## R06

标准流程：

1. 初始化 `x = 0, y = ry`，并计算 `rx^2, ry^2, dx, dy, d1`。
2. 区域 1 循环：当 `dx < dy` 时，根据 `d1` 选择 `E` 或 `SE` 更新。
3. 计算区域 2 初值 `d2`。
4. 区域 2 循环：当 `y >= 0` 时，根据 `d2` 选择 `S` 或 `SE` 更新。
5. 每次状态变化后都写入四向对称点。

## R07

正确性直观依据：

- 判别值本质是在比较两个候选像素中心到真实椭圆的相对位置。
- 若判别值指向“椭圆内侧”，则优先沿 x 方向推进；若指向“外侧”，需要同时下移 y 纠偏。
- 分区处理使每段迭代都沿主变化轴单调推进，保证边界连续且不漏点。

## R08

复杂度：

- 时间复杂度：`O(rx + ry)`（两个区域分别单调推进 x 或 y）。
- 空间复杂度：`O(rx + ry)`（若存储所有点）；若直接流式绘制可视为 `O(1)` 额外空间。

## R09

数值实现注意：

- 判别值可用浮点（实现简单）或通过放大系数改写为整数递推（更稳定）。
- 像素集合建议用 `set` 去重，避免轴上点被重复写入。
- 输出排序仅用于展示，不影响算法正确性。

## R10

边界与异常处理：

- `rx <= 0` 或 `ry <= 0` 应直接报错。
- `rx = 1` 或 `ry = 1` 时仍可运行，会退化为细长椭圆近似。
- 当 `rx`、`ry` 很大时，ASCII 渲染不适合，建议改为图像缓冲区绘制。

## R11

## R14

与常见方法对比：

- 参数方程采样：实现直观，但需 `sin/cos` 且采样均匀性依赖步长。
- Bresenham 椭圆：同属增量法，思想相近。
- 中点椭圆：在教学与工程中都常见，速度快，易硬件化。

## R15

典型应用：

- 2D 光栅图元绘制（CAD、教学图形系统、嵌入式显示）。
- 作为椭圆填充、抗锯齿、扫描线算法的边界生成基础。
- 图形学课程中离散曲线光栅化的经典案例。

## R16

可扩展方向：

- 椭圆填充：在边界点基础上按扫描线填充内部。
- 抗锯齿：结合像素覆盖率或距离场改善边缘平滑性。
- 任意旋转椭圆：先生成轴对齐椭圆，再做坐标变换并重采样。

## R17

常见错误清单：

- 区域切换条件写反（`dx < dy` / `dx >= dy`）。
- `dx`、`dy` 更新与判别值更新顺序不一致。
- 忘记加中心偏移 `(xc, yc)`。
- 未处理重复点导致统计数量异常。

## R18

`demo.py` 的源码级算法流（非黑盒，8 步）：

1. 校验输入半轴，预计算 `rx2 = rx*rx`、`ry2 = ry*ry`。
2. 从顶点 `(x, y) = (0, ry)` 起步，把四向对称点写入集合。
3. 初始化区域 1 判别量 `d1`，以及 `dx = 2*ry2*x`、`dy = 2*rx2*y`。
4. 在 `dx < dy` 区间循环：
   - 若 `d1 < 0`，走 `E`：`x += 1`，更新 `dx` 与 `d1`。
   - 否则走 `SE`：`x += 1, y -= 1`，同步更新 `dx/dy/d1`。
   - 每步输出四向对称点。
5. 用当前 `(x, y)` 初始化区域 2 判别量 `d2`。
6. 在 `y >= 0` 区间循环：
   - 若 `d2 > 0`，走 `S`：`y -= 1`，更新 `dy` 与 `d2`。
   - 否则走 `SE`：`x += 1, y -= 1`，更新 `dx/dy/d2`。
   - 每步输出四向对称点。
7. 将点集排序后返回，便于稳定展示与测试。
8. 主函数再做两件事：ASCII 渲染（可视验证）和隐式方程残差统计（数值验证）。
