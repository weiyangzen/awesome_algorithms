## R01

子集构造法（Subset Construction）用于把**非确定有限自动机**（NFA，含 `ε` 转移）转换为等价的**确定有限自动机**（DFA）。

核心思想：
- DFA 的一个状态，不再对应 NFA 的单个状态；
- 而是对应“某一时刻 NFA 可能同时处于的状态集合”（即 NFA 状态子集）；
- 再通过 `move + ε-closure` 递推出新的子集状态。

这使得后续词法分析器可以用确定自动机做高效线性扫描。

## R02

本条目 MVP 的输入/输出语义：
- 输入：手工构造的一个 `ε-NFA`（示例语言为 `(a|b)*ab`）；
- 输出：
  - 构造得到的 DFA 状态集合与转移表；
  - NFA 与 DFA 在多组字符串上的识别结果一致性报告。

脚本不需要交互输入，运行即完成构造与验证。

## R03

形式化定义（本实现使用）：
- NFA：`(Q, Σ, δ, q0, F)`，其中 `δ: Q × (Σ ∪ {ε}) -> P(Q)`；
- DFA：`(Q', Σ, δ', q0', F')`，其中 `δ': Q' × Σ -> Q'`。

构造规则：
- `q0' = ε-closure({q0})`；
- 对任意 DFA 状态 `S ⊆ Q` 和符号 `a ∈ Σ`：
  - `T = ε-closure(move(S, a))`；
  - 令 `δ'(S, a) = T`；
- 若 `S ∩ F != ∅`，则 `S` 是 DFA 接受态。

## R04

子集构造法的工作流：
1. 求起始子集 `ε-closure({q0})`，命名为 `D0`；
2. 维护待处理队列 `worklist`；
3. 取出一个未处理子集 `S`；
4. 对每个输入符号 `a`，计算 `T = ε-closure(move(S, a))`；
5. 若 `T` 从未出现，分配新状态名并入队；
6. 记录 DFA 转移 `S --a--> T`；
7. 队列清空后停止。

本实现保留空集子集 `{}` 作为合法 DFA 状态（陷阱态）。

## R05

`ε-closure` 是正确构造的关键。

在 `demo.py` 中，`epsilon_closure` 用深度优先/栈式扩展：
- 初始把给定状态集放入 `closure` 和 `stack`；
- 反复弹出状态，沿 `ε` 边扩展；
- 新到达状态若未见过，继续入栈；
- 直到栈空，得到闭包。

如果忽略 `ε-closure`，会漏掉可达路径，导致 DFA 识别能力错误。

## R06

示例 NFA 采用 Thompson 风格，识别正则 `(a|b)*ab`：
- `q0..q9` 共 10 个状态；
- 含多条 `ε` 转移用于表达并联与闭包；
- 终态为 `q9`。

该例的价值：
- 同时覆盖“非确定分支 + ε 转移 + 重复结构”；
- 构造出的 DFA 状态数明显少于 `2^10`，体现“可达子集才会出现”。

## R07

复杂度分析（设 NFA 状态数 `|Q|`，字母表大小 `|Σ|`）：
- 最坏情况下可达子集数为 `2^|Q|`；
- 每个 DFA 状态需对每个符号计算一次 `move + ε-closure`；
- 单次闭包/转移在朴素实现下约 `O(|Q| + |E|)`（`E` 为转移边数）。

故最坏时间复杂度可写为：
- `O(2^|Q| * |Σ| * (|Q| + |E|))`。

空间复杂度主要来自已发现子集与转移表，最坏 `O(2^|Q|)` 级别。

## R08

正确性要点（直观）：
- DFA 状态 `S` 精确代表“读入某前缀后，NFA 可能所处状态集合”；
- `move(S, a)` 覆盖 NFA 在符号 `a` 上的全部并行后继；
- 再做 `ε-closure` 保证“读符号后可免费走的 ε 路径”被完整纳入；
- 因而每个前缀下 DFA 与 NFA 的可达语义一致；
- 接受态判定 `S ∩ F != ∅` 也与 NFA 接受定义一致。

所以转换前后语言等价。

## R09

边界与异常处理：
- 空字符串：直接由起始闭包是否含接受态决定；
- 未知输入符号：`simulate_nfa/simulate_dfa` 会显式抛 `ValueError`；
- 空集状态：在构造中保留，形成完整 DFA（常作陷阱态）；
- 随机测试：固定种子 `2026`，结果可复现。

输出重点解读：

## R13

验证策略：
- 固定样例覆盖典型正反例（如 `ab`、`aab`、`baba`）；
- 随机样例补充广度（长度 `0..8`）；
- 对每个字符串同时跑 `simulate_nfa` 与 `simulate_dfa`；
- 若有任一不一致，立即抛异常并列出样例。

这是一种“实现级回归测试”，可快速发现构造逻辑问题。

## R14

常见错误与本实现的规避方式：
- 错误 1：只对起点做一次 `ε-closure`。
  - 规避：每一步符号转移后都做 `ε-closure`。
- 错误 2：接受态判定写成“子集全是终态”。
  - 规避：使用“交集非空即接受”。
- 错误 3：漏处理空集子集，导致转移不全。
  - 规避：把空集当作普通 DFA 状态纳入构造。

## R15

与编译原理上下文的关系：
- 正则表达式 -> ε-NFA（Thompson 构造）；
- ε-NFA -> DFA（本条目子集构造法）；
- DFA 最小化（可选后处理）。

词法分析器通常在离线阶段完成这三步，再将 DFA 表部署到运行时扫描器中。

## R18

`demo.py` 源码级算法流程（8 步）：
1. `main` 调用 `build_sample_nfa` 构建 `(a|b)*ab` 的 ε-NFA。
2. `subset_construction` 先计算 `start_subset = epsilon_closure({q0})`，命名 `D0`。
3. 进入 `worklist` 循环，逐个取出未展开的子集状态。
4. 对每个输入符号执行 `move(nfa, subset, symbol)`，得到一步可达集。
5. 对上一步结果调用 `epsilon_closure`，得到下一 DFA 子集状态 `next_subset`。
6. 若 `next_subset` 首次出现，则分配新状态名、登记子集映射并加入队列；若其与 NFA 接受态相交，则标记为 DFA 接受态。
7. 记录确定化转移 `dfa_transitions[(current_name, symbol)] = next_name`，直到队列清空。
8. `verify_equivalence` 用固定样例与随机样例分别调用 `simulate_nfa/simulate_dfa` 对比结果，一致则报告通过。
