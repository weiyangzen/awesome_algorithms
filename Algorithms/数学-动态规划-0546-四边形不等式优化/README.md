## R01

本条目实现“区间合并最小代价”这一经典模型，并用它演示四边形不等式优化（常见实现形态是 Knuth Optimization）的最小可运行 MVP。

问题背景：
- 有一排相邻段（或文件）权重 `w[0..n-1]`；
- 每次只能合并相邻两段，代价等于被合并区间总权重；
- 重复直到合并成一段，目标是总合并代价最小。

`demo.py` 同时给出：
- 朴素 `O(n^3)` 区间 DP（基线真值）；
- 四边形不等式优化后的 `O(n^2)` 版本；
- 两者结果一致性校验、决策单调性校验、以及四边形不等式样例校验。

## R02

问题定义（MVP 范围）：
- 输入：一维非负数序列 `weights`，长度为 `n`。
- 输出：
  - `min_cost`：最小总代价；
  - `expression`：一种最优括号合并结构（如 `((A1+A2)+A3)`）；
  - `dp`：DP 代价表；
  - `opt`：每个区间的最优决策点（分割点）。

边界约定：
- `n=0` 或 `n=1` 时，无需合并，总代价为 `0`；
- 区间下标采用 0-based：`[i, j]` 表示第 `i..j` 段；
- 区间权重和 `S(i,j)=sum(weights[i..j])` 由前缀和 `O(1)` 计算。

## R03

状态与转移：

1. 定义 `dp[i][j]`：把区间 `[i,j]` 合并成一段的最小代价。  
2. 边界：`dp[i][i] = 0`。  
3. 转移（朴素）：
   `dp[i][j] = min_{k in [i, j-1]} (dp[i][k] + dp[k+1][j] + S(i,j))`。  
4. 令 `opt[i][j]` 为使上述式子取最小的 `k`。  
5. 若代价函数满足四边形不等式与区间单调性，可得决策单调：
   `opt[i][j-1] <= opt[i][j] <= opt[i+1][j]`。  
6. 因此在算 `[i,j]` 时，只需在更窄区间
   `[opt[i][j-1], opt[i+1][j]]` 内枚举 `k`，由 `O(n^3)` 降到 `O(n^2)`。

## R04

算法流程（本实现）：
1. 校验输入为一维、有限、非负数组。
2. 构建前缀和数组 `prefix`，支持 `S(i,j)` 常数时间查询。
3. 先运行朴素区间 DP（`O(n^3)`）得到基线最优值与决策表。
4. 再运行四边形不等式优化版本（Knuth 形式）：
   - 区间长度从小到大；
   - 每个 `[i,j]` 只在单调窗口内枚举分割点。
5. 从 `opt` 回溯生成一个最优括号表达式。
6. 校验优化结果与基线结果一致。
7. 校验决策单调性 `opt[i][j-1] <= opt[i][j] <= opt[i+1][j]`。
8. 对小规模样例额外校验 `S(i,j)` 的四边形不等式成立。

## R05

核心数据结构：
- `weights: np.ndarray(float)`：输入权重序列；
- `prefix: np.ndarray(float)`：前缀和，`prefix[t] = sum(weights[:t])`；
- `dp: np.ndarray(float)`：最优代价表，大小 `n x n`；
- `opt: np.ndarray(int)`：最优分割点表，大小 `n x n`；
- `MergeDPResult`（`dataclass`）：
  - `min_cost: float`
  - `dp: np.ndarray`
  - `opt: np.ndarray`
  - `expression: str`

## R06

正确性要点（简述）：
- 最优子结构：区间 `[i,j]` 的最优方案必经某个分割点 `k`，拆成 `[i,k]` 与 `[k+1,j]` 两个子问题最优解。
- 无后效性：一旦子区间最优代价确定，组合时只依赖这些数值，不依赖构造路径细节。
- 穷尽性：朴素版本枚举了所有 `k`，因此不会漏解。
- 单调窗口正确性：在四边形不等式条件下，最优决策点具有单调性；窗口裁剪不会排除最优 `k`。
- 回溯正确性：`opt[i][j]` 记录了每个区间的最优切分，递归展开即得到与 `dp` 一致的最优括号。

## R07

复杂度分析：
- 朴素区间 DP：
  - 时间：`O(n^3)`（长度 * 起点 * 分割点）
  - 空间：`O(n^2)`
- 四边形不等式优化（Knuth 形式）：
  - 时间：`O(n^2)`
  - 空间：`O(n^2)`
- 额外校验：
  - 决策单调性检查：`O(n^2)`
  - 四边形不等式检查（仅小样本）：`O(n^4)`，但仅用于演示。

## R08

边界与异常处理：
- 输入不是一维数组：抛出 `ValueError`；
- 输入含 `nan/inf`：抛出 `ValueError`；
- 输入含负值：抛出 `ValueError`（本模型按非负权重定义）；
- `n=0`：返回 `min_cost=0`、`expression="EMPTY"`；
- `n=1`：返回 `min_cost=0`、`expression="A1"`；
- 若单调窗口异常为空，代码会回退到 `[i, j-1]` 全窗口保证稳健性。

## R09

MVP 取舍：
- 仅用 `numpy + 标准库`，不依赖黑盒 DP 求解器；
- 同时保留 `O(n^3)` 基线，便于验证优化版正确性；
- 选择“相邻合并最小代价”作为载体问题，结构清晰、易展示四边形不等式优化；
- 不做并行化、GPU 化、超大规模工程封装，聚焦算法核心与可验证性。

## R10

`demo.py` 函数职责：
- `to_1d_nonnegative_array`：输入校验与标准化；
- `build_prefix_sums` / `interval_sum`：区间和计算；
- `merge_cost_cubic`：`O(n^3)` 基线 DP；
- `merge_cost_knuth`：四边形不等式优化版 `O(n^2)`；
- `reconstruct_expression`：根据 `opt` 恢复最优括号结构；
- `verify_opt_monotonicity`：校验决策单调性；
- `check_quadrangle_inequality_for_range_sum`：对小规模样例验证四边形不等式；
- `run_case` / `main`：组织样例、打印结果、断言一致性。

## R15

方法对比：
- 朴素区间 DP：
  - 优点：直接、通用、容易写对；
  - 缺点：`O(n^3)` 在大 `n` 时较慢。
- 四边形不等式优化（Knuth）：
  - 优点：利用结构性质降到 `O(n^2)`；
  - 缺点：并非所有区间 DP 都满足适用条件。
- 其他优化（如分治优化）：
  - 适用条件不同；
  - 本条目不展开，仅强调“四边形不等式 + 决策单调”这一路线。

## R16

典型应用：
- 最优文件合并（相邻合并代价最小）；

## R18

源码级算法流（对应 `demo.py`，8 步）：
1. `main` 构造固定样例并逐个调用 `run_case`。  
2. `run_case` 先将输入传入 `merge_cost_cubic`，用完整分割枚举求 `O(n^3)` 基线最优值。  
3. `merge_cost_cubic` 内部先做输入校验，再建立前缀和，通过三重循环计算 `dp[i][j]` 与 `opt[i][j]`。  
4. `run_case` 再调用 `merge_cost_knuth`，执行四边形不等式优化版本。  
5. `merge_cost_knuth` 在区间 `[i,j]` 只枚举 `k in [opt[i][j-1], opt[i+1][j]]`，更新最优代价和决策点。  
6. 两种算法都用 `reconstruct_expression` 从 `opt` 表回溯出一个最优括号表达式。  
7. `run_case` 调用 `verify_opt_monotonicity` 检查 `opt[i][j-1] <= opt[i][j] <= opt[i+1][j]`。  
8. `run_case` 对小规模样例调用 `check_quadrangle_inequality_for_range_sum`，并断言“基线代价 = 优化代价”后打印结果。
