# 快速选择 (QuickSelect)

- UID: `CS-0078`
- 学科: `计算机`
- 分类: `分治算法`
- 源序号: `98`
- 目标目录: `Algorithms/计算机-分治算法-0098-快速选择_(QuickSelect)`

## R01

快速选择（QuickSelect）用于在无序数组中寻找第 `k` 小元素（0-based 秩）。

- 输入: 可比较序列 `A`，秩 `k`。
- 输出: `A` 排序后下标为 `k` 的元素值。
- 核心思想: 借鉴快速排序的“分区（partition）”步骤，但每轮只递归/迭代进入一侧子问题。
- 目标: 不做完整排序，只定位一个秩位置。

## R02

历史背景:

- 提出者: C. A. R. Hoare（Tony Hoare）。
- 时间: 1961 年左右，与快速排序思想同源。
- 工程意义: 在只需要分位点/阈值元素时，QuickSelect 常比完整排序更省计算。

## R03

为什么需要它:

- 若直接排序再取第 `k` 个，典型复杂度为 `O(n log n)`。
- QuickSelect 平均只需 `O(n)`，尤其适合:
  - 中位数/分位数查询；
  - Top-K 前的阈值定位；
  - 大量数据中只关心某一秩值，而非完整有序序列。

## R04

时间复杂度:

- 平均复杂度: `O(n)`（随机 pivot 或“足够好”pivot 的期望结果）。
- 最坏复杂度: `O(n^2)`（每次都选到极差 pivot，问题规模只减少 1）。
- 备注: 本目录 `demo.py` 使用随机 pivot，强调平均表现与实现简洁性。

## R05

空间复杂度:

- 原地分区版本: 额外空间 `O(1)`。
- 递归实现通常有栈空间；本目录实现为迭代主循环 + 原地分区，避免递归栈增长。
- 因此示例实现总体额外空间接近 `O(1)`（不计输入拷贝为工作数组那一步）。

## R06

示例（找第 `k=3` 小，0-based）:

- 输入: `[12, 3, 5, 7, 4, 19, 26]`
- 第一次分区后，pivot 左边是较小元素，右边是较大元素。
- 若 `k` 落在左区间，则只继续处理左区间；若落在右区间，则只处理右区间。
- 反复执行后，最终定位到结果 `7`（排序结果为 `[3, 4, 5, 7, 12, 19, 26]`）。

## R07

算法价值:

- 分治思路清晰，易于实现和讲解。
- 在“只求一个秩”的问题上通常明显快于完整排序。
- 可作为更复杂选择器（如 introselect）的基础模块。
- 对重复值可配合三路划分提升鲁棒性。

## R08

前置知识:

- 比较模型与序关系。
- 分区操作（partition）与双指针思想。
- 分治复杂度分析（平均与最坏区分）。
- 0-based 秩与“第 k 小”定义转换。

## R09

适用条件与边界:

- 输入元素必须可比较。
- `k` 必须满足 `0 <= k < n`，空数组非法。
- QuickSelect 不保证输出整体有序，只保证目标秩正确。
- 若场景必须最坏线性保证，可考虑 BFPRT（中位数的中位数）替代 pivot 策略。

## R10

正确性要点（不变式）:

1. 分区后满足三段关系：`< pivot`、`== pivot`、`> pivot`。
2. 若 `k` 在 `< pivot` 段，答案必在该段；若在 `> pivot` 段，答案必在该段。
3. 若 `k` 落在 `== pivot` 段，答案直接是 `pivot`。
4. 每轮都严格缩小搜索区间，因此最终必终止并返回正确秩元素。

## R11

数值与比较稳定性说明:

- QuickSelect 是选择算法，不涉及迭代数值逼近，通常不存在“误差传播”问题。
- 主要风险来自比较语义:
  - 比较器不满足传递性会破坏结果正确性；
  - 混入不可比较类型会抛异常。
- “稳定性”在排序语义下通常不适用，因为该算法不输出完整稳定序列。

## R12

工程成本（超越大 O）:

- pivot 质量直接影响常数项与轮数。
- 随机 pivot 一般简单有效，但运行轨迹有随机性。
- 分区访问模式线性，缓存友好度通常较好。
- Python 实现的常数成本主要在解释器循环与交换操作。

## R13

保证类型:

- 这是精确算法，不是近似算法（无近似比概念）。
- 使用随机 pivot 时，给出的是期望线性时间保证，而非最坏线性保证。
- 若需要确定性最坏线性，可改用 BFPRT pivot 选择。

## R14

鲁棒性与常见失效模式:

- `k` 越界（负数或 `>= n`）。
- 空输入未处理。
- 分区实现错误导致死循环（边界指针不推进）。
- 重复值处理不当导致区间收缩异常。
- 解决策略: 输入校验 + 三路划分 + 多样化回归测试。

## R15

实现建议:

- 采用三路划分（Dutch National Flag）以正确处理大量重复值。
- 用迭代代替递归，减少栈风险。
- 将“选择核心”和“测试/打印逻辑”分离，便于复用。
- 测试至少覆盖:
  - 小样例可手算；
  - 大量重复值；
  - 含负数与混合分布；
  - 随机回归与排序结果对照。

## R16

前驱与后继关系:

- 前驱思想:
  - 快速排序中的 partition 机制。
- 相关后继/变体:
  - BFPRT（最坏 `O(n)`）；
  - Introselect（实践中常见的混合策略）；
  - Top-K / 分位数算法中的秩选择子过程。

## R17

本目录 `demo.py` 的 MVP 设计:

- 核心函数:
  - `_partition_three_way(...)`：对当前区间执行三路分区；
  - `quickselect(values, k, seed)`：迭代收缩区间定位目标秩；
  - `_run_case(...)`：按样例执行并做断言。
- 统计信息:
  - `partition_rounds`、`comparisons`、`swaps`、`pivots` 便于观察运行轨迹。
- 校验策略:
  - 用 `sorted(values)[k]` 作为主参考；
  - 若安装了 `numpy`，额外用 `np.partition` 交叉验证（仅验证，不参与核心决策）。
- 运行命令:
  - `uv run python demo.py`

## R18

`demo.py` 源码级流程拆解（8 步，非黑箱）:

1. `main()` 构造 3 组固定样例并运行 `_random_regression()` 随机回归。
2. 每个样例进入 `_run_case()`，先用 `sorted(values)[k]` 计算基准答案。
3. `_run_case()` 调用 `quickselect(values, k, seed=2026)`，进入快速选择主流程。
4. `quickselect` 先做输入校验，再复制工作数组并初始化左右边界 `[left, right]`。
5. 每轮随机选一个 `pivot`，调用 `_partition_three_way` 对当前区间做 `< / == / >` 三路分区。
6. 根据返回的 `(lt, gt)` 判断 `k` 落点: 在左段就收缩到左段，在右段就收缩到右段，在中段直接返回。
7. 主循环持续缩小搜索区间，直到命中中段或退化为单点 `left == right`。
8. 返回结果后，`_run_case()` 再与 `sorted`（以及可选 `numpy.partition`）对照断言，最后打印轮数与 pivot 轨迹摘要。
