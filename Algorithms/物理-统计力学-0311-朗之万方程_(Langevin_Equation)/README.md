## R01

朗之万方程用于描述“确定性耗散 + 随机热噪声”共同作用下的动力学，是统计力学与随机过程的核心桥梁之一。  
本目录实现的是一维过阻尼情形（布朗粒子处于谐势阱），其方程写作：

`dx_t = -(k/gamma) * x_t * dt + sqrt(2 k_B T / gamma) * dW_t`

这也是经典 Ornstein-Uhlenbeck (OU) 过程。

## R02

目标是构建一个最小、可运行、可核验的 Langevin MVP：

- 使用 Euler-Maruyama 显式离散模拟轨道；
- 用长期样本验证平衡分布二阶矩 `Var(x)=k_B T / k`；
- 验证自相关函数的指数衰减 `C(t)=exp(-t/tau)`，其中 `tau=gamma/k`；
- 输出数值与理论对照表，并内置断言用于自动验收。

## R03

采用的物理模型与记号：

- 粒子位置：`x(t)`
- 势能：`U(x)=0.5*k*x^2`
- 摩擦系数：`gamma`
- 温度：`T`
- 玻尔兹曼常数：`k_B`

过阻尼近似下，惯性项被消去，得到：

`dx = a(x) dt + b dW, a(x)=-(k/gamma)x, b=sqrt(2k_B T/gamma)`

该过程平衡态为高斯分布，均值 0，方差 `k_B T / k`。

## R04

理论可检验结论：

1. 平衡均值：`E[x]=0`
2. 平衡方差：`Var(x)=k_B T / k`
3. 归一化自相关：`rho(t)=exp(-(k/gamma)t)=exp(-t/tau)`
4. 松弛时间：`tau=gamma/k`

`demo.py` 将针对第 2、3 条给出数值-理论对比。

## R05

数值算法采用 Euler-Maruyama（非黑盒）：

`x_{n+1} = x_n - (k/gamma) x_n dt + sqrt(2k_B T/gamma) * sqrt(dt) * eta_n`

其中 `eta_n ~ N(0,1)` 独立同分布。实现上：

- 同时推进多条轨道以降低统计误差；
- 丢弃前 `burn_in_steps` 作为热化阶段；
- 在平稳段计算样本均值、方差、自相关并与理论比较。

## R10

正确性直觉：

1. 漂移项 `-(k/gamma)x` 把轨道拉回 0；
2. 扩散项持续注入热涨落；
3. 二者平衡后形成高斯平稳态，方差由 `k_B T / k` 固定；
4. 回归强度 `k/gamma` 决定相关衰减速度，故自相关呈指数衰减。

数值上若同时满足“方差匹配 + 自相关匹配”，通常说明 Langevin 离散实现正确。

## R11

数值稳定性与工程处理：

- 使用较小步长 `dt=1e-3`，控制显式离散误差；
- 通过 `burn_in_steps` 去除初值瞬态；
- 多轨道并行平均，降低统计噪声；
- 对输入做基本合法性检查（`burn_in < n_steps`）；
- 断言阈值保守设置，防止偶然噪声导致误判。

## R12

- `n_trajectories`：并行轨道数；

## R14

常见失败模式：

1. 把噪声项写成 `sqrt(dt)` 之外的尺度，导致方差严重偏离；
2. 忘记 `2 k_B T / gamma` 因子，破坏涨落-耗散一致性；
3. 未丢弃热化段，导致平稳统计被初值污染；
4. 步长过大，出现离散偏差；
5. 轨道数太少，相关函数抖动大。

本 MVP 分别通过显式公式、burn-in、小步长与多轨道统计来规避。

## R18

`demo.py` 源码级算法流程（8 步）：

1. `main` 构造 `LangevinConfig`，固定 `k, gamma, T, dt` 及轨道数、步数和随机种子。  
2. `run_mvp` 调用 `simulate_trajectories`，创建形状 `(n_steps, n_trajectories)` 的状态矩阵并推进时间。  
3. 在每个时间步，按 Euler-Maruyama 更新：上一时刻状态先做线性漂移，再加 `sqrt(dt)` 缩放高斯噪声。  
4. `stationary_statistics` 丢弃 burn-in 后把样本拉平成一维，计算样本均值和方差。  
5. 同函数计算理论方差 `k_B T / k`，并给出相对误差，形成第一组物理一致性检验。  
6. `autocorrelation_table` 在多个 lag 上计算经验归一化自相关，并与解析式 `exp(-(k/gamma)t)` 逐项比较。  
7. `main` 打印统计摘要和对照表，提取最大相关误差作为全局指标。  
8. 执行三条断言（均值接近 0、方差误差小、自相关误差小）；全部通过后输出验证成功。  

第三方库边界：`numpy` 用于随机数与向量化运算，`pandas` 仅用于结果表格展示；Langevin 离散推进、统计量计算、自相关比较均在源码中显式实现，不依赖黑盒 SDE 求解器。
