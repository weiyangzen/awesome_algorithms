# 量子门 (Quantum Gates)

## R01

量子门是作用在量子态上的幺正线性算子，是量子电路的基本构件。给定初始态与门序列，算法任务是按门顺序演化状态向量，并输出最终态与测量概率。

本条目聚焦“可追踪的门级线性代数实现”：不依赖量子 SDK 黑盒，而是直接用矩阵与向量运算实现单比特门、双比特门与组合门流程。

## R02

本目录 MVP 的目标：

1. 显式给出常见单量子比特门 `X/Y/Z/H/S/T` 与旋转门 `Rx/Rz`。
2. 构造任意 `n` 比特系统上的“单比特门扩展矩阵”。
3. 通过基矢索引映射显式构造 `CNOT`。
4. 演示门序列 `H(q0)` + `CNOT(q0,q1)` 制备 Bell 态。
5. 给出可复现实验输出与断言校验（幺正性、恒等式、保真度、概率分布）。

## R03

问题输入/输出抽象：

- 输入：
  - 初始态 `|psi0>`（本实现使用计算基 `|bits>` 生成）；
  - 门集合与参数（如 `Rx(theta)`）；
  - 门作用位置（`target`，以及 `CNOT` 的 `control/target`）。
- 输出：
  - 演化后的态向量 `|psi_out>`；
  - 计算基测量概率 `p_i = |<i|psi_out>|^2`；
  - 诊断结果（门是否幺正、标准恒等式残差、与目标态保真度）。

## R04

`demo.py` 中使用的核心门矩阵：

- Pauli 门：
  - `X = [[0,1],[1,0]]`
  - `Y = [[0,-i],[i,0]]`
  - `Z = [[1,0],[0,-1]]`
- Hadamard 门：
  - `H = (1/sqrt(2))*[[1,1],[1,-1]]`
- 相位门：
  - `S = diag(1, i)`
  - `T = diag(1, exp(i*pi/4))`
- 旋转门：
  - `Rx(theta) = cos(theta/2)I - i sin(theta/2)X`
  - `Rz(theta) = diag(exp(-i theta/2), exp(i theta/2))`

## R05

多比特门扩展规则：

- 若门 `G` 作用在 `n` 比特系统的第 `target` 位，完整算子为各位算子 Kronecker 积：
  - 目标位放 `G`，其余位放 `I`。
- 本实现采用 `q0` 为最高位（most significant bit）的位序约定。
- `single_qubit_gate_matrix` 负责该扩展，`kron_all` 负责逐项 Kronecker 乘积。

## R06

CNOT 的构造方式（非黑盒）：

- 不调用现成量子框架 API，而是遍历计算基列索引 `col`。
- 若 `control` 位为 1，则翻转 `target` 位得到 `row = col xor (1 << shift)`；否则 `row=col`。
- 将 `U[row, col] = 1` 写入，最终得到置换矩阵形式的 `CNOT`。

该方法直接体现了 CNOT 的逻辑定义，便于源码级审计。

## R07

复杂度分析（`n` 为量子比特数，态维度 `d=2^n`）：

- 构造单比特扩展矩阵：显式 Kronecker 积通常为 `O(d^2)`。
- 构造 CNOT 全矩阵：遍历基矢列索引，写矩阵元素，`O(d^2)`（稀疏结构可进一步优化）。
- 一次全矩阵乘态向量：`O(d^2)`。
- 存储全门矩阵：`O(d^2)`；存储态向量：`O(d)`。

本 MVP 只演示 `n=1,2`，所以计算开销极小，重点在正确性与可解释性。

## R08

正确性不变量：

1. 幺正门应满足 `U^dagger U = I`。
2. 幺正演化保持态归一化（`||psi||_2 = 1`）。
3. 测量概率应满足非负且和为 1。
4. 已知门恒等式应成立，例如：
   - `H^2 = I`
   - `HZH = X`
   - `S^2 = Z`
   - `T^2 = S`
   - `CNOT^2 = I`

`demo.py` 对以上条件逐项断言。

## R09

`demo.py` 主要函数职责：

- `basis_state(bits)`：生成计算基态 `|bits>`。
- `single_qubit_gate_matrix(...)`：将 `2x2` 门扩展到 `n` 比特。
- `cnot_matrix(...)`：按索引映射构造 CNOT。
- `apply_unitary(state, unitary)`：执行门作用并归一化。
- `probabilities(state)`：计算测量概率分布。
- `fidelity(psi, phi)`：纯态保真度。
- `run_single_qubit_examples()`：单比特演示。
- `run_bell_state_example()`：Bell 态制备演示。
- `run_identity_checks()`：门恒等式残差检查。

## R10

测试策略（脚本内置断言）：

1. 门库幺正性检查：`I/X/Y/Z/H/S/T/Rx/Rz/CNOT`。
2. 单比特行为检查：
   - `H|0> = |+>`
   - `X|0> = |1>`
   - `Z|+> = |->`
3. Bell 态检查：
   - 保真度 `F((H+CNOT)|00>, (|00>+|11>)/sqrt(2)) ≈ 1`
   - 概率分布 `[0.5, 0, 0, 0.5]`
4. 矩阵恒等式残差均小于阈值 `1e-10`。

## R11

边界与异常处理：

- 非法 bitstring（非二进制或空）会抛 `ValueError`。
- `target/control` 越界或重复（`control==target`）会抛 `ValueError`。
- 非方阵门或维度不匹配会抛 `ValueError`。
- 零范数态或零概率总和会抛 `ValueError`。

这些检查避免“静默错误”污染量子门验证结果。

## R12

单比特示例解释：

- 从 `|0>` 出发：
  - 经过 `H` 得到叠加态 `|+> = (|0>+|1>)/sqrt(2)`。
  - 经过 `X` 得到 `|1>`。
- 从 `|+>` 出发，经过 `Z` 得到 `|->`。
- `Rx(pi/3)|0>` 给出带复相位振幅的连续旋转结果，展示“离散门 + 参数化门”并存的量子门族。

## R13

Bell 态示例（两比特）：

1. 初态 `|00>`。
2. 在 `q0` 上施加 `H`，得到 `( |00> + |10> ) / sqrt(2)`。
3. 施加 `CNOT(control=q0, target=q1)`，得到 Bell 态
   `( |00> + |11> ) / sqrt(2)`。

预期测量概率：

- `P(00)=0.5`
- `P(11)=0.5`
- `P(01)=P(10)=0`

## R14

工程实现细节：

- 使用 `complex128` 提升数值稳定性。
- 所有比较通过容差阈值（`ATOL=1e-10`）判断，而非硬性“机器零误差”。
- `state_to_dirac` 仅输出显著非零振幅，便于人工审查门作用结果。
- `normalize_state` 在每次门作用后执行归一化，防止误差累积导致概率漂移。

## R15

MVP 边界与局限：

- 仅覆盖纯态、门级理想演化，不含噪声与退相干。
- 仅实现到两比特示例，未优化大规模多比特性能。
- 未引入密度矩阵、量子通道、误差校正与硬件拓扑约束。

因此它是“教学与回归验证基线”，不是硬件级仿真器。

## R18

`demo.py` 源码级算法流程（8 步）：

1. **定义门库与旋转门生成器**
   在文件顶部定义 `I/X/Y/Z/H/S/T`，并通过 `rx(theta)`、`rz(theta)` 生成参数化门矩阵。

2. **构造输入基态**
   `basis_state(bits)` 将 bitstring 映射到计算基向量（维度 `2^n`），作为电路输入态。

3. **将单比特门扩展到多比特**
   `single_qubit_gate_matrix` 先生成各位算子列表（目标位放 `G`，其余放 `I`），再用 `kron_all` 合成全系统门矩阵。

4. **显式构造 CNOT 置换矩阵**
   `cnot_matrix` 遍历每个基矢列索引，根据控制位是否为 1 决定是否翻转目标位，并写入 `U[row, col] = 1`。

5. **执行门作用并归一化**
   `apply_unitary` 完成矩阵-向量乘法 `|psi'> = U|psi>`，随后 `normalize_state` 保证数值归一。

6. **计算概率与保真度**
   `probabilities` 输出测量概率分布；`fidelity` 用 `|<psi|phi>|^2` 对比目标态与实际态。

7. **运行两组演示流程**
   `run_single_qubit_examples` 验证基础门作用；`run_bell_state_example` 运行 `H + CNOT` 并得到 Bell 态与概率分布。

8. **系统性断言与结果输出**
   `main` 依次执行幺正性检查、恒等式残差检查、示例保真度/概率断言；全部通过后输出 `All checks passed.`。

说明：本实现的核心算法流完全在源码中展开，仅使用 `numpy` 进行底层线性代数运算，不存在“几行黑盒 API 直接求结果”的不可解释步骤。
