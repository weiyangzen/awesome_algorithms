# 最大团 - Bron-Kerbosch

## R01

Bron-Kerbosch 算法用于在无向图中枚举极大团（maximal cliques），并可据此求解最大团（maximum clique）。

- 核心思想: 通过回溯维护三元组 `(R, P, X)`。
  1. `R`: 当前已选顶点集合（始终保持为团）。
  2. `P`: 仍可加入 `R` 且不破坏团性质的候选顶点。
  3. `X`: 已处理过、不能再用于当前分支扩展的顶点。
- 终止条件: 当 `P` 与 `X` 同时为空时，`R` 是一个极大团。
- 本条目目标: 在枚举逻辑上增加“记录当前最大团 + 上界剪枝”，得到实用的最大团求解 MVP。

## R02

- 提出时间: 1973 年。
- 提出者: Coen Bron 与 Joep Kerbosch。
- 代表论文: *Algorithm 457: finding all cliques of an undirected graph*（CACM, 1973）。
- 后续关键改进:
  - Tomita 等人的 pivot 选择优化（显著减少分支）。
  - Eppstein 等人提出结合退化序（degeneracy ordering）的高效枚举变体。

## R03

最大团问题是经典 NP-hard 问题，广泛出现在社交网络社区检测、蛋白质相互作用分析、冲突图调度、组合优化等场景。朴素穷举顶点子集需要 `O(2^n)` 级别搜索，规模稍大就不可用。

Bron-Kerbosch 的价值在于:

- 直接在“满足团约束的候选空间”搜索，而不是无约束遍历所有子集。
- 每次递归都通过邻接交集收缩候选集，能在稀疏图上显著减少实际分支。
- 结合 pivot 与剪枝后，可在中等规模图上快速得到最大团。

## R04

时间复杂度（最坏情况）:

- 枚举全部极大团的理论上界为 `O(3^(n/3))`（Moon-Moser 上界同阶）。
- 求最大团可视为“枚举极大团 + 取最大”，最坏同阶。
- 使用 pivot 后，常数因子显著降低，但不改变最坏指数级本质。

实践中影响性能的关键因素:

- 图密度（过稠或过稀都可能影响分支形态）。
- pivot 质量（优先选择使 `P \ N(u)` 最小的 `u`）。
- 剪枝强度（例如 `|R| + |P| <= best` 时提前返回）。

## R05

空间复杂度:

- 递归深度最多 `O(n)`。
- 每层维护集合 `R, P, X`，若直接复制集合，单层额外 `O(n)`。
- 整体工作空间可记为 `O(n^2)` 量级（取决于实现细节与临时集合分配）。
- 邻接表存储图本身需要 `O(n + m)`。

## R06

手工示例（求最大团）:

设图顶点 `{0,1,2,3,4}`，边集为:

- `0` 连 `1,2,3`
- `1` 连 `0,2,3`
- `2` 连 `0,1,3,4`
- `3` 连 `0,1,2`
- `4` 连 `2`

其中 `{0,1,2,3}` 两两相连，是一个 4 阶团；且无法再加入 `4`，因此它是极大团且也是最大团，最大团大小为 `4`。

## R07

算法意义:

- 理论层: 是“约束满足 + 回溯搜索 + 剪枝”范式在图论中的标志性算法。
- 复杂度研究层: 为极大团枚举、最大团分支限界、参数化算法提供基线框架。
- 工程层: 在实际图挖掘中常作为子程序，用于发现高密度子结构。

## R08

前置知识与算法依赖:

- 无向简单图表示（邻接表/邻接集合）。
- 团定义: 顶点子集内任意两点有边。
- 极大团与最大团区别:
  - 极大团: 不能再扩展。
  - 最大团: 顶点数最多。
- 集合运算: 交集、差集、并集是算法核心操作。
- 回溯正确性依赖于不变量: `R` 始终是团，且 `P ∪ X` 给出相对 `R` 的完整扩展边界。

## R09

适用前提与边界:

- 输入应为无向图；若有向图需先定义等价无向关系。
- 若存在自环或重边，需在建图时清洗为简单图语义。
- 对非常大且稠密的图，最坏情况仍会爆炸，需结合启发式或近似方法。
- 本 MVP 面向“中小规模 + 可验证”场景，不追求工业级分布式扩展。

## R10

正确性要点（简述）:

1. 初始时 `R=∅, P=V, X=∅`，覆盖了所有可能团。
2. 在分支点选顶点 `v` 时递归到 `(R∪{v}, P∩N(v), X∩N(v))`，保证新增后仍为团。
3. 当 `P` 与 `X` 皆空时，`R` 无法再扩展且未被历史分支覆盖，故为极大团。
4. 每个极大团恰被报告一次（`X` 负责去重语义）。
5. 在报告极大团时更新全局最优，即可得到最大团。

## R11

数值稳定性与误差:

- N/A（纯组合算法，不涉及浮点迭代或数值误差传播）。
- 主要风险来自实现正确性而非数值稳定性:
  - 集合更新次序错误导致漏解或重复。
  - 未做上界剪枝会引发性能问题而非精度问题。

## R12

真实运行成本（超越大 O）:

- Python 中热点在集合交集 `P & N(v)`、差集 `P - N(u)`。
- 顶点编号与访问局部性会影响常数项。
- 递归函数调用与频繁集合复制带来额外开销。
- pivot 选择若用 `max(P∪X, key=|P∩N(u)|)`，会增加单层计算但通常换来更少递归。
- 在小图上，暴力校验可作为正确性基准；在大图上不可行。

## R13

近似比/误差界/概率保证:

- N/A。Bron-Kerbosch 是确定性精确搜索框架，不是随机算法或近似算法。
- 输出最大团时，结果是精确最优值（若实现正确且遍历完整）。

## R14

鲁棒性与失效模式:

- 图不对称（`u` 邻接 `v` 但 `v` 不邻接 `u`）会破坏无向图假设，建图阶段必须强制双向写入。
- 递归深度在大图上可能触发 Python 递归限制。
- 若在迭代候选时直接修改同一集合，可能产生“跳点”错误；应迭代快照列表。
- 仅用“当前最优大小”剪枝是安全剪枝，不会影响正确性。

## R18

源码级流程拆解（`demo.py` 中 `bron_kerbosch_maximum_clique`，无第三方黑箱，8 步）:

1. 初始化 `best_clique = ∅`，并准备内部递归函数 `search(R, P, X)`。
2. 每次进入 `search` 先做安全上界剪枝: 若 `|R| + |P| <= |best_clique|`，当前分支不可能改写最优，立即返回。
3. 若 `P` 与 `X` 同时为空，说明 `R` 是一个极大团；若 `|R|` 更大，则更新 `best_clique`。
4. 选择 pivot `u ∈ P ∪ X`，使 `|P ∩ N(u)|` 尽量大，以减少待展开候选。
5. 仅对 `candidates = P \ N(u)` 中的顶点分支（Bron-Kerbosch pivot 核心）。
6. 对每个候选 `v`，递归调用 `search(R∪{v}, P∩N(v), X∩N(v))`，保证新状态仍满足团不变量。
7. 分支返回后执行回溯更新: `P.remove(v)` 与 `X.add(v)`，表示 `v` 在当前层已处理，避免重复枚举。
8. 全部递归结束后返回 `best_clique`，即最大团（若有多个同规模团，返回按遍历顺序遇到的一个）。
