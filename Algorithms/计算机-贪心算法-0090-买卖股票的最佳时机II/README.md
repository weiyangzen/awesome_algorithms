# 买卖股票的最佳时机II

- UID: `CS-0070`
- 学科: `计算机`
- 分类: `贪心算法`
- 源序号: `90`
- 目标目录: `Algorithms/计算机-贪心算法-0090-买卖股票的最佳时机II`

## R01

“买卖股票的最佳时机II”（Best Time to Buy and Sell Stock II）要求在给定每日股价序列下，进行不限次数交易（但同一时间最多持有一股），求可获得的最大总利润。

这是贪心算法中的经典题：局部上涨就应当吃到，不需要预测更远的“神奇卖点”。

## R02

问题约束可表述为：

- 输入：`prices[i]` 表示第 `i` 天价格；
- 操作：每天可选择买入、卖出或不操作；
- 限制：卖出前必须先买入，且不能同时持有多股；
- 输出：最大可实现总利润。

本题默认允许“当天买入、未来某天卖出”，并允许多次往复交易。

## R03

关键贪心观察：

- 若 `prices[i] > prices[i-1]`，这段上涨 `prices[i] - prices[i-1]` 一定可以被纳入最优利润；
- 整段上升趋势 `a -> b -> c` 的收益 `c-a`，等价于 `(b-a) + (c-b)`；
- 因此把所有“相邻正增量”求和，就是全局最优。

这避免了复杂择时，直接把问题分解成每天的局部决策。

## R04

算法流程（单次线性扫描）：

1. 初始化 `profit = 0`；
2. 从第 2 天开始遍历；
3. 对每个相邻差值 `delta = prices[i] - prices[i-1]`：
   - 若 `delta > 0`，则 `profit += delta`；
   - 否则跳过；
4. 扫描结束后返回 `profit`。

`demo.py` 中实现函数为 `max_profit_greedy`。

## R05

复杂度：

- 时间复杂度：`O(n)`，只扫描一次价格序列；
- 空间复杂度：`O(1)`（不计输入存储），只维护常数变量。

这是该题最常用、也最简洁的最优实现。

## R06

本目录 MVP 的验证设计：

- 固定样例覆盖：标准示例、单调上升、单调下降、单元素、空数组、锯齿序列；
- 引入两类基线对照：

## R07

优点：

- 代码短小、可读性高；
- 性能线性，适合大规模回测预处理；
- 数学等价关系明确，容易证明正确。

局限：

- 本题不含手续费、冷冻期、持仓上限等现实约束；
- 若加入交易成本，直接“正增量求和”可能不再最优；
- 不输出具体交易路径（仅输出最大利润）。

## R08

前置知识：

- 贪心算法基本思想；
- 相邻差分与累加；
- 动态规划状态机（用于交叉验证）。

运行环境：

- Python 3.10+；
- `numpy`（仅用于随机样例构造与数组输入桥接，不是黑盒求解器）。

## R09

适用场景：

- 无交易成本、无限次交易的一维价格序列；
- 需要快速估计“理论上可吃到的波动收益”；
- 作为更复杂策略的 baseline。

不适用场景：

- 含手续费/税费、冷冻期、做空限制等规则；
- 多资产组合联动优化；
- 需要最优交易明细路径而非仅利润值。

## R10

正确性直觉：

- 任何一次合法交易都可分解到相邻天差值的和；
- 相邻负差值不会提升利润，跳过总是安全；
- 相邻正差值若不计入，等价于主动放弃可实现收益；
- 因而“累加全部正差值”既不损失可行性，也不会遗漏最优收益。

`demo.py` 用 DP 与穷举结果做一致性断言，进一步支撑实现正确。

## R11

边界与鲁棒性处理：

- 空输入或单天价格返回 `0`；
- 连续下跌序列返回 `0`；
- 输入中若出现负价格，`demo.py` 会抛出 `ValueError`；
- `numpy` 桥接函数校验必须是一维数组。

这些约束让 MVP 在数据异常时尽早失败，避免静默错误。

## R12

实现细节选择：

- 使用 `_normalize_prices` 统一输入清洗与类型转换；
- `max_profit_greedy` 仅依赖相邻对 `zip(values, values[1:])`；
- `max_profit_dp` 采用经典 `cash/hold` 两状态转移；
- `max_profit_bruteforce` 用记忆化 DFS，仅用于小规模校验，不用于生产求解。

这种“主算法 + 两条验证链”结构比单一实现更易排错。

## R13

理论保证与边界：

- 在本题约束下，贪心与 DP 最优值严格一致；
- 该贪心是精确算法，不是近似；
- 结果与价格序列确定后唯一（利润值唯一，交易路径可不唯一）。

当问题扩展到手续费/冷冻期/限次交易时，应切换到对应 DP 模型。

## R14

常见错误：

1. 误把本题写成“只能交易一次”的解法（会低估结果）；
2. 错误地尝试寻找全局最低买点和最高卖点；
3. 在 DP 转移里用更新后的 `cash` 立即更新 `hold`（破坏同日状态语义）；
4. 忽略空数组与长度为 1 的边界；
5. 把负价格静默接受，造成业务语义不清。

## R15

相关题目与方法：

## R18

`demo.py` 的源码级算法流程（7 步）：

1. `main` 依次执行固定样例、随机验证、`numpy` 示例，形成完整执行闭环。  
2. `_normalize_prices` 将输入转为非负整数列表，提前过滤非法数据。  
3. `max_profit_greedy` 对相邻价格做差，累计所有正增量，得到候选最优利润。  
4. `max_profit_dp` 用 `cash/hold` 两状态逐日转移，计算同一问题的最优利润基线。  
5. `max_profit_bruteforce` 通过记忆化 DFS 枚举“持仓/不持仓”决策路径，给出小规模精确真值。  
6. `assert_consistency` 断言贪心结果与 DP 一致，且在小规模时与穷举一致。  
7. 所有断言通过后输出成功信息，证明 MVP 在样例与随机集上稳定正确。  

本实现没有调用第三方“股票最优交易黑盒函数”；`numpy` 只负责生成/承载测试数据，核心算法逻辑完全在源码中可追踪。
