# Delaunay三角剖分 - 分治法

- UID: `MATH-0191`
- 学科: `数学`
- 分类: `计算几何`
- 源序号: `191`
- 目标目录: `Algorithms/数学-计算几何-0191-Delaunay三角剖分_-_分治法`

## R01

本题目标是实现一个可运行的 Delaunay 三角剖分 MVP，并显式体现“分治法”结构。

- 输入：二维点集 `P={(x_i,y_i)}`；
- 输出：三角形集合（simplices）与边集合（edges）；
- 核心性质：任意三角形外接圆内部不包含其他输入点（空圆性质）。

## R02

为什么选 Delaunay：

- 对“瘦长三角形”有抑制作用，网格质量通常更好；
- 可自然导出 Voronoi 图（对偶关系）；
- 是许多数值几何、插值、有限元、路径规划前处理的基础步骤。

## R03

几何判定基础：

1. 方向测试（orientation）：判断三点顺逆时针。
2. 内接圆测试（in-circle）：判断点是否落在某三角形外接圆内。
3. Delaunay 条件：若所有三角形都满足“空圆”，则形成 Delaunay 三角剖分（在一般位置下唯一）。

## R04

经典分治法思想（Guibas-Stolfi 路线）：

1. 按 `x` 坐标排序并二分；
2. 递归构建左半与右半的 Delaunay；
3. 沿上下公切线逐步“拉链式”合并，维护局部空圆条件；
4. 合并完成后得到全局 Delaunay。

理论复杂度可达 `O(n log n)`。

## R05

本目录采用“教学型工程 MVP”：

- 保留了分治递归骨架（按 `x` 中位数切分、递归处理子问题）；
- 子问题和合并步骤调用 `scipy.spatial.Delaunay`（Qhull 后端）求三角化；
- 在顶层输出统一三角网，并执行几何性质与平面图不变量校验。

说明：该 MVP 的“合并”是“对子集重三角化”的稳健实现，不是手写公切线拉链，因此不是最优常数实现，但流程清晰、可验证、可运行。

## R06

`demo.py` 的主要数据结构：

- `NodeStat`：记录递归树节点（深度、点数、`x` 范围）；
- `simplices`：形状 `(m,3)` 的三角形顶点索引；
- `edges`：由三角形集合提取出的去重无向边；
- `depth_report`：按递归深度聚合的统计表（`pandas` 输出）。

## R07

核心流程（实现视角）：

1. 生成固定随机种子的二维点；
2. 按 `x` 递归二分，收集分治节点统计；
3. 每个子问题调用 Qhull 计算 Delaunay；
4. 父问题对左右并集执行一次三角化作为合并结果；
5. 根节点产出最终 triangulation；
6. 计算边集、三角形面积、空圆违例数；
7. 执行断言门禁并打印汇总表。

## R08

复杂度（本 MVP）：

- 若设每层都对各子问题重三角化，整体成本高于经典 `O(n log n)`；
- 在当前 `n=128` 的演示规模下可稳定秒级完成；
- 空圆验证为 `O(T*n)`（`T` 为三角形数），属于“验证成本”，用于确保结果可信。

因此该实现强调“可读与可验”，而非追求竞赛级最优常数。

## R09

数值稳定处理：

- 调用 `Delaunay(..., qhull_options="QJ")` 处理退化点集风险；
- 使用 `numpy` 双精度；
- 外接圆判定使用容差 `tol=1e-9`，避免浮点噪声导致误报；
- 三角面积断言要求严格大于 0，过滤退化三角形。

## R10

`demo.py` 的函数职责：

- `generate_points`：生成可复现实验点集；
- `triangulate_subset`：对子集调用 Qhull 并映射回全局索引；
- `recursive_dc_triangulate`：分治递归与父层合并；
- `canonicalize_simplices`：三角形标准化去重；
- `simplices_to_edges`、`triangle_areas`：拓扑和几何统计；
- `validate_empty_circumcircle`：空圆性质验证；
- `summarize_depth_stats`：分治树统计汇总；
- `main`：执行、打印、断言门禁。

## R12

输出指标说明：

- `triangles`：最终三角形个数；
- `edges`：去重后边数；
- `hull_vertices`：凸包顶点数；
- `area(min/mean/max)`：三角形面积统计；
- `violations`：空圆条件违反数量；
- `depth_report`：各递归层节点数与点规模分布。

## R13

正确性门禁（代码内断言）：

1. 递归树必须实际展开（节点数 > 1）；
2. 输出必须是三角形索引矩阵 `(m,3)`；
3. 满足平面三角剖分不变量：
   - `T = 2n - 2 - h`
   - `E = 3n - 3 - h`
4. 所有三角形面积必须正；
5. 空圆检查违例数必须为 0；
6. 分治结果与一次性全局三角化（同参数）一致。

## R14

常见失效模式与应对：

- 失效 1：点集近共线导致 Qhull 报错。  
  对策：`QJ` 抖动、输入去重、必要时预旋转坐标。

- 失效 2：浮点误差导致“伪违例”。  
  对策：空圆判断使用容差，并结合最坏 margin 诊断。

- 失效 3：重复点或极近点导致退化三角形。  
  对策：预处理去重，保留最小距离阈值。

## R15

工程实践建议：

- 大规模点集优先直接调用成熟库，先拿正确结果；
- 手写公切线合并适合在“性能瓶颈已确认”后再做；
- 输出阶段保留不变量和几何校验，避免“算出来但不可证”；
- 对下游任务（插值/网格）可同时导出邻接信息与边界边。

## R16

与相关算法对比：

- 与 Bowyer-Watson（增量法）相比：分治法理论复杂度和并行潜力更优，但实现复杂；
- 与扫描线/翻边法相比：分治法结构清晰，适合离线批处理；
- 与纯库黑盒调用相比：本目录补充了分治可解释层和显式验证层。

## R18

本实现的源码级算法链路（8 步）：

1. `main` 调用 `generate_points` 得到点集，并创建根索引 `0..n-1`。  
2. `recursive_dc_triangulate` 对当前索引按 `x` 稳定排序，中位数二分为左右子问题。  
3. 递归继续下钻，直到 `leaf_size` 触发叶子求解；每个节点都记录 `NodeStat`。  
4. 每个求解点调用 `triangulate_subset`，其内部执行 `scipy.spatial.Delaunay(...)`。  
5. 在 SciPy 内部，`Delaunay` 来自 `scipy.spatial._qhull`，构造 `_Qhull` 实例并进入 Qhull C 核心（`qhull_src`）。  
6. Qhull 在抛物面提升后的凸包框架下构造 Delaunay 单纯形，SciPy 将结果暴露为 `tri.simplices`。  
7. 脚本对 simplices 做标准化去重，提取边集合，计算面积，并用外接圆检查验证空圆性质。  
8. 最后用欧拉型不变量（`T=2n-2-h`、`E=3n-3-h`）和“一次性全局三角化对照”完成门禁，确保输出可信。  

这 8 步覆盖了“分治组织 -> 子问题求解 -> 结果合并 -> 几何校验”的完整最小闭环。
