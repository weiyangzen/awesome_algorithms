## R01

扫描线填充（Scanline Fill）是多边形光栅化中的经典算法：
按 `y` 方向逐行扫描，在每一条扫描线上计算多边形边与该扫描线的交点，把交点按 `x` 排序后两两配对并填充区间。

其核心不是“从种子向外扩散”，而是“基于边-扫描线交点的区间填充”。

## R02

本条目 MVP 目标：

给定二维画布尺寸与一个简单多边形顶点序列（整数顶点坐标），实现可直接运行的 Python 脚本：
- 生成扫描线填充掩码；
- 把掩码写入到输出画布；
- 打印 ASCII 可视化结果与统计信息；
- 用一个独立的逐像素奇偶规则参考实现做一致性校验。

## R03

选择该算法作为 MVP 的理由：
- 是图形学管线中“向像素网格落地”的基础步骤；
- 结构清晰，便于展示边表（ET）与活动边表（AET）的工程写法；
- 只依赖 `numpy` 即可实现可验证版本，不需要 GUI 或交互输入。

## R04

离散模型（奇偶规则）定义：

设像素中心为 `(x+0.5, y+0.5)`，多边形区域内部定义为：
从该点向 `+x` 发射射线，与多边形边相交次数为奇数则在内部，为偶数则在外部。

扫描线填充输出掩码 `M(y,x)` 满足：
- 若像素中心位于多边形内部，则 `M(y,x)=True`；
- 否则 `M(y,x)=False`。

最终填充画布 `B` 满足：
- `M(y,x)=True` 的位置写入 `fill_value`；
- 其余位置保持原值。

## R05

MVP 高层流程（ET + AET）：

1. 读取顶点环，按相邻顶点构造边，跳过水平边；
2. 为每条非水平边计算 `inv_slope=dx/dy`，并映射到其起始扫描线 `y_start`；
3. 扫描 `y` 从最小有效行到最大有效行；
4. 将 `y_start==y` 的边加入 AET；移除 `y>=y_end` 的边；
5. 按当前交点 `x` 对 AET 排序；
6. 两两配对交点并填充对应 `x` 区间；
7. 每处理完一行，把 AET 中每条边的 `x` 递增 `inv_slope`；
8. 输出填充掩码、填充结果与统计信息。

## R06

正确性要点：

- 不误填：只填充由交点配对确定的“内部区间”；
- 不漏填：每条扫描线都根据当前全部活动边计算内部区间；
- 终止性：扫描线范围有限，且每条边在 `y_end` 后被移出 AET，算法有限步结束。

在本实现中采用半开区间思想（边在 `y` 方向按 `[y_start, y_end)` 生效），减少顶点重复计数问题。

## R07

复杂度分析：

设边数为 `E`，扫描线高度为 `H`，第 `y` 行活动边数为 `A_y`。
- 构建边表：`O(E)`；
- 扫描阶段排序成本：`O(sum_y A_y log A_y)`；
- 填充写入成本与填充像素数 `K` 近似同阶。

最坏情况下可写为 `O(H * E log E + K)`，空间复杂度为 `O(E + H*W_mask)`（其中布尔掩码占 `H*W`）。

## R08

边界与异常处理策略：
- 顶点数少于 3：报错；
- 画布尺寸非正：报错；
- 水平边不进入 ET（避免重复交点）；
- 纵向有效区间裁剪到画布高度；
- 若某行交点数为奇数，记录统计字段 `odd_intersection_rows`，并按可配对部分继续执行。

## R09

MVP 取舍：
- 实现标准 CPU 版扫描线，不依赖 OpenCV/PIL 的黑盒填充；
- 不做自交多边形布尔处理、抗锯齿或子像素覆盖率估计；
- 不做 GPU 并行优化；
- 重点放在“源码可读 + 可跑 + 可校验”。

## R10

`demo.py` 主要函数职责：
- `EdgeRecord`：活动边记录（`y_end/x/inv_slope`）；
- `FillStats`：一次填充任务的统计摘要；
- `build_edge_table`：从顶点构造 ET，并给出扫描范围；
- `scanline_fill_mask`：核心扫描线填充，返回布尔掩码与统计；
- `fill_polygon`：把掩码写入画布；
- `point_in_polygon_even_odd` / `reference_mask_even_odd`：逐像素参考实现（用于校验）；
- `bresenham_line` / `draw_polygon_outline`：生成轮廓可视化；
- `run_case`：执行单例并打印结果；
- `main`：无交互自动跑两个案例。

## R12

输出字段说明：
- `edges_total`：参与填充的非水平边数量；
- `ignored_horizontal_edges`：被跳过的水平边数量；
- `scanlines_processed`：实际处理的扫描线行数；
- `filled_segments`：扫描线区间填充次数；
- `filled_pixels`：掩码内像素总数；
- `odd_intersection_rows`：交点数为奇数的扫描线数量；
- `matches_reference`：是否与逐像素奇偶规则参考实现完全一致。

## R14

关键参数与建议：
- `vertices`：建议使用按顺时针或逆时针给出的简单多边形；
- `height/width`：应覆盖多边形范围，避免可视化截断；
- `fill_value`：建议与背景值和轮廓值不同，输出更易读；
- 若需要处理复杂布尔几何（洞、多环、自交），应升级到更完整的多边形裁剪/布尔库。

## R15

与相关算法对比：
- 对比种子填充（Flood Fill）：
  - 扫描线填充按几何边界直接求内部区间；
  - 种子填充按连通分量扩张，依赖种子点与像素连通定义。
- 对比边界填充（Boundary Fill）：
  - 扫描线更适合多边形光栅化主流程；
  - 边界填充更常见于“已知边界色”的交互改色。
- 对比三角形硬件光栅化：
  - GPU 管线通常使用边函数/重心坐标并行实现；
  - 本 MVP 展示的是 CPU 端教学版本与数据流。

## R18

`demo.py` 源码级算法流程（8 步）：

1. `main` 定义两个多边形案例（凸与凹），并逐个调用 `run_case`。  
2. `run_case` 先创建空白画布并绘制轮廓，然后调用 `fill_polygon` 触发扫描线填充。  
3. `fill_polygon` 调用 `scanline_fill_mask`；该函数先用 `build_edge_table` 遍历顶点环，生成按 `y_start` 分桶的 ET。  
4. 在扫描循环中，每行先把 ET 中对应边加入 AET，再移除 `y>=y_end` 的过期边。  
5. 对 AET 按当前交点 `x` 排序，交点按偶奇配对得到内部区间。  
6. 把每个区间映射为离散列范围 `[x_start, x_end)`，并在布尔掩码该行置 `True`。  
7. 行处理结束后，对 AET 每条边执行 `x += inv_slope`，把交点推进到下一扫描线。  
8. `run_case` 再调用 `reference_mask_even_odd` 做逐像素奇偶规则校验，输出 `matches_reference` 和统计字段，完成可验证闭环。
