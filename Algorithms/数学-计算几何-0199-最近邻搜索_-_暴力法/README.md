# 最近邻搜索 - 暴力法

- UID: `MATH-0199`
- 学科: `数学`
- 分类: `计算几何`
- 源序号: `199`
- 目标目录: `Algorithms/数学-计算几何-0199-最近邻搜索_-_暴力法`

## R01

算法目标: 在二维点集 `P` 中，对任意查询点 `q` 返回最近邻点 `p*`，使得欧氏距离 `||p* - q||` 最小。  
本条目采用最直接的“全量扫描”策略: 对每个候选点都计算距离，不做空间剪枝。

## R02

问题形式化:
- 输入: 点集 `P = {p_i | p_i in R^2}`，查询点 `q in R^2`
- 输出: `i* = argmin_i ||p_i - q||_2`
- 距离度量: 欧氏距离（实现中比较平方距离 `||p_i - q||_2^2`）
- 数据前提: 点集可静态或动态更新，但单次查询都需要线性扫描全体点

## R03

核心思想:
- 不建立索引结构（如 k-d tree / R-tree）。
- 对每个点 `p_i` 计算 `dist2_i = (p_i.x - q.x)^2 + (p_i.y - q.y)^2`。
- 维护当前最优 `(best_idx, best_dist2)`，遍历结束即得到最近邻。
- 该方法实现最简单，适合作为正确性基线。

## R04

数据结构设计:
- `points: np.ndarray`，形状 `(n, 2)`，保存二维点。
- `query: np.ndarray`，形状 `(2,)`，单个查询点。
- 结果结构:
  - 单查询: `(nearest_index, nearest_dist2)`
  - 批查询: `indices: (m,)`, `dist2s: (m,)`

## R05

单点查询流程（暴力扫描）:
1. 初始化 `best_dist2 = +inf`，`best_idx = -1`。
2. 遍历 `points` 中每个候选点。
3. 计算当前点到查询点的平方距离 `dist2`。
4. 若 `dist2 < best_dist2`，更新最优解。
5. 遍历结束，返回最优索引与最小平方距离。

## R06

批量查询流程（向量化分块）:
1. 将 `queries` 按 `chunk_size` 分块，避免一次性构造超大三维张量。
2. 对每个块计算 `diff = queries_chunk[:, None, :] - points[None, :, :]`。
3. 用 `einsum` 计算每个查询到所有点的平方距离矩阵。
4. 按行 `argmin` 得到最近邻索引，并提取对应最小距离。
5. 合并各块结果，得到完整批量查询输出。

## R07

复杂度分析:
- 单次查询时间复杂度: `O(n)`
- `m` 次查询时间复杂度: `O(mn)`
- 额外空间复杂度:
  - 纯循环实现: `O(1)`（除输入外）
  - 向量化分块: `O(chunk_size * n)`

## R08

正确性要点:
- 完备性: 每个候选点都被访问，不会漏解。
- 最优性: `best_dist2` 在遍历过程中单调不增，最终即全局最小。
- 交叉验证: `demo.py` 使用“NumPy 向量化版本 vs 纯 Python 循环版本”逐次比对最近距离，确保实现一致。

## R09

MVP 范围:
- 二维欧氏距离最近邻（1-NN）
- 单查询与批查询
- 正确性断言 + 性能对照输出
- 不包含 k 近邻、近似搜索、空间索引构建

## R10

实现文件说明:
- `demo.py`
- `brute_force_nearest_python(...)`: 纯 Python 循环版暴力搜索
- `brute_force_nearest_numpy(...)`: NumPy 向量化单查询版本
- `brute_force_batch_numpy(...)`: NumPy 分块批查询版本
- `main()`: 数据生成、正确性校验、耗时统计、结果输出

## R12

样例输出解读:

- `随机查询一致性校验`: 若通过，说明两种实现在最小距离上保持一致
- `NumPy 批量查询耗时`: 面向工程的主执行路径
- `Python 循环耗时`: 作为可读性强、无黑盒的基线
- `同批加速比(python/numpy)`: 向量化收益的直观指标

## R13

与空间索引法对比:
- 暴力法优势:
  - 实现简单、调试友好
  - 无索引维护成本，适合小规模数据或查询次数较少场景
  - 常作为 k-d tree / ball tree / ANN 的正确性基线
- 暴力法劣势:
  - 查询复杂度线性增长，数据规模大时吞吐受限
  - 多次查询场景下，通常劣于可剪枝索引结构

## R14

工程注意事项:
- 距离比较优先使用平方距离，减少 `sqrt` 调用。
- 批量查询应控制块大小，避免内存峰值过高。
- 并列最近点场景下索引可能不同，验证应优先比较最小距离一致性。
- 固定随机种子可保证实验可复现。

## R15

测试策略:
- 固定查询: 快速检查实现是否工作。
- 随机查询对照: 使用 200 次随机查询，对比 NumPy 与 Python 版本的最近距离。
- 批量一致性: 在同一查询子集上比较两个实现的距离和，确保统计级别结果一致。
- 性能测试: 分别测量 NumPy 批量路径和 Python 循环路径耗时。

## R16

已知局限:
- 对大规模数据集和高频查询，`O(n)` 查询成本偏高。
- 当前示例聚焦二维欧氏距离，未扩展到其他度量或高维特化优化。
- NumPy 分块仍会产生中间张量，块过大时内存压力明显。

## R17

可扩展方向:
- 扩展到 `k` 近邻（返回前 `k` 个最小距离）。
- 增加半径查询（`range query`）。
- 在保证结果一致的前提下接入 `scipy/sklearn` 索引方法并做基线对照。
- 针对高维与超大规模引入近似最近邻（HNSW/FAISS）。

## R18

源码级算法流（`demo.py`，非黑盒拆解）:
1. `main()` 固定随机种子，生成二维点集 `points` 和查询集 `queries`。
2. 先对固定查询调用 `brute_force_nearest_numpy` 与 `brute_force_nearest_python`，比较最近距离是否一致。
3. `brute_force_nearest_python` 逐点循环: 计算 `(dx * dx + dy * dy)`，若更小则更新 `(best_idx, best_dist2)`。
4. `brute_force_nearest_numpy` 先做逐元素差 `diff = points - query`，再用 `einsum` 得到每个候选点平方距离。
5. NumPy 版本通过 `argmin` 找到最小距离对应索引，返回单查询最近邻。
6. 随机查询阶段重复调用两种单查询实现 200 次，逐次断言最近距离一致。
7. `brute_force_batch_numpy` 将查询分块，对每块构造 `(chunk, n, 2)` 的差分张量并计算距离矩阵，按行取最小值。
8. `main()` 统计 NumPy 批量路径和 Python 循环路径耗时，输出一致性校验与加速比，形成可复现实验结果。
