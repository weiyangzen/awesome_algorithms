## R01

工作集算法（Working Set Algorithm）用于估计进程“当前活跃页面集合”，并据此指导内存驻留与页面置换。  
定义上，时刻 `t` 的工作集 `W(t, Δ)` 是最近 `Δ` 次内存引用中出现过的页面集合。若物理内存容量不足，优先淘汰不在该集合内的页，以降低抖动风险。

## R02

在操作系统虚拟内存管理中，工作集思想由 Denning 提出，核心目标不是只追求单次最优命中，而是让进程在一个稳定的活跃页集合上运行。  
与纯 LRU/FIFO 相比，工作集方法更强调“阶段性局部性”：程序阶段切换时，工作集会变化，系统应随之调整驻留页。

## R03

本条目 MVP 的输入输出如下：
- 输入：
  - `frame_limit`：物理页框数上限。
  - `window_size(Δ)`：工作集统计窗口长度（按“最近引用次数”计）。
  - `reference_string`：页面访问序列（`demo.py` 内置固定数据）。
- 输出：
  - 每一步是否命中、是否淘汰及淘汰页号。
  - 每一步的驻留页（按访问新旧顺序）与 `WS(before/after)`。
  - 汇总统计：`hits/faults/evictions/hit_rate/fault_rate/avg_ws_size/max_ws_size`。

## R04

实现采用“引用窗口 + 最近访问时间戳”的组合模型：
- `_history + _history_counter`：维护最近 `Δ` 次引用，快速得到当前工作集。
- `resident(set)`：当前在内存中的页面。
- `last_access(dict)`：记录页面最近访问时刻，用于在候选中选择最旧页淘汰。

置换规则：
1. 缺页且有空闲帧：直接装入。
2. 缺页且帧已满：先从 `resident - W(t,Δ)` 中选最久未访问页；若为空则回退到全体 resident 中的最旧页（LRU 回退）。

## R05

算法流程（单次访问）：
1. 读取当前窗口形成 `ws_before`。
2. 若访问页在 `resident`，记一次命中。
3. 若不在 `resident`，记一次缺页；必要时触发淘汰。
4. 淘汰时优先选不在 `ws_before` 的驻留页；若没有，则选全局最旧驻留页。
5. 更新该页 `last_access`。
6. 将该页压入最近窗口（超过 `Δ` 时弹出最旧引用）。
7. 生成步骤快照用于可解释输出。

## R06

正确性直觉来自两个不变式：
- 不变式 A：`_history_counter` 始终等价于“最近 `Δ` 次引用的频次统计”，因此其键集合就是 `W(t,Δ)`。
- 不变式 B：`last_access` 始终记录每个页面最近一次访问时刻，因此在候选集合中取最小值即“最旧页面”。

因此，当发生缺页并需要置换时，算法一定会优先驱逐“当前工作集之外且最旧”的页面；当不存在此类页面时，回退策略与 LRU 一致。

## R07

记访问长度为 `n`，页框数为 `F`：
- 单次访问：
  - 命中处理与窗口更新为 `O(1)`。
  - 发生淘汰时，候选筛选需要扫描 `resident`，为 `O(F)`。
- 总体时间复杂度：`O(n * F)`（`F` 通常远小于 `n`）。
- 空间复杂度：
  - 驻留页与时间戳 `O(F)`；
  - 历史窗口 `O(Δ)`；
  - 合计 `O(F + Δ)`（不计输入序列本身）。

## R09

边界条件与处理：
- `frame_limit <= 0` 或 `window_size <= 0`：初始化直接抛 `ValueError`。
- 空访问序列：统计为 0，命中率/缺页率返回 0。
- `Δ` 大于序列长度：工作集会逐步扩大直到覆盖已访问不同页。
- `frame_limit = 1`：退化为单帧场景，频繁置换时可用于压力测试。

## R10

`demo.py` 的 MVP 结构：
- `StepRecord`：单步输出结构体，便于打印和验证。
- `WorkingSetPager`：
  - `_working_set()` 计算当前工作集；
  - `_push_history()` 维护滑动窗口；
  - `_choose_victim()` 执行“工作集优先 + LRU 回退”；
  - `access()` / `simulate()` 负责单步与整段模拟。
- `run_sanity_checks()`：两个最小断言用例，防止基本逻辑回归。

## R12

输出字段说明：
- `Result`：`HIT` 或 `FAULT`。
- `Evicted`：缺页且内存满时显示被淘汰页，否则为 `-`。
- `Resident(old->new)`：当前驻留页，按“最旧访问 -> 最新访问”排序。
- `WS(before)`：当前访问前的工作集 `W(t,Δ)`。
- `WS(after)`：处理本次引用并更新窗口后的工作集。
- `avg_ws_size/max_ws_size`：窗口工作集规模的平均值与峰值。

## R13

与常见页面置换策略对比：
- FIFO：实现简单，但不感知局部性，可能产生较差置换决策。
- LRU：强调最近性，效果通常好，但未显式建模阶段性工作集大小。
- 工作集算法：把“活跃页集合”作为一等对象，更适合解释抖动与内存分配压力；但实现上需要维护窗口统计与候选筛选。

## R14

常见实现错误：
- 用“最近 `Δ` 个不同页”替代“最近 `Δ` 次引用”，导致工作集定义错误。
- 窗口弹出时未正确维护计数，造成幽灵页面残留在工作集中。
- 淘汰时忽略“先看非工作集页”的优先级，退化成纯 LRU。
- 统计口径混乱（例如把换入但未满帧时也计为 eviction）。

## R15

可扩展方向：
- 引入每进程 `Δ` 自适应（依据缺页率或 CPU 等待时间动态调整）。
- 输出多组 `Δ` 的敏感性分析曲线，对比命中率与工作集大小波动。
- 和 PFF（Page Fault Frequency）联合，形成“缺页率驱动 + 工作集约束”的混合策略。
- 将模拟扩展为多进程全局页框竞争模型，观察局部/全局置换差异。

## R16

建议测试集：
- 基础命中测试：`[1,2,3,1]` 在 `frame_limit=3` 下应得到 1 次命中。
- 触发淘汰测试：`[1,2,3]` 在 `frame_limit=2` 下应出现 1 次淘汰。
- 局部性切换测试：前后两段访问不同页簇，检查工作集迁移是否合理。
- 退化测试：`frame_limit=1`、`Δ=1` 与高重复序列。
- 稳定性测试：随机长序列，核对统计总量守恒 `total = hits + faults`。

## R17

本条目完成了一个可运行、可追踪的工作集算法最小实现：
- 逻辑上严格遵循 `W(t,Δ)` 定义；
- 代码规模小，便于教学和验证；
- 输出提供了“访问结果 + 工作集变化 + 驻留集变化”的完整可解释链路。

## R18

`demo.py` 源码级算法流（8 步）：
1. `main()` 固定 `reference_string/frame_limit/window_size`，并先执行 `run_sanity_checks()` 做最小正确性保护。
2. `simulate()` 逐页遍历访问序列，每次调用 `access(page)` 处理一条引用。
3. `access()` 在处理前调用 `_working_set()`，从 `_history_counter` 读出 `ws_before`。
4. 若命中（`page in resident`），仅更新命中计数；若缺页，进入置换路径。
5. 缺页且帧满时，`_choose_victim(ws_before)` 先筛出 `resident` 中不在工作集的页面，再按 `last_access` 最小值选受害页；若筛空则回退到全 resident 的 LRU。
6. 完成换入后，更新 `last_access[page]` 并调用 `_push_history(page)` 维护最近 `Δ` 次引用窗口（必要时弹出最旧引用并更新计数）。
7. `access()` 生成 `StepRecord`（含 `hit/evicted/resident/ws_before/ws_after`）返回给 `simulate()` 汇总。
8. `simulate()` 用 `numpy` 计算 `avg_ws_size/max_ws_size`，`main()` 打印逐步表格与最终统计，形成可复现实验输出。
