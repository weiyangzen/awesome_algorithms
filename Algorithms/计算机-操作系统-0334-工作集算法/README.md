## R01

工作集算法（Working Set）是操作系统虚拟内存管理中的经典页面置换思想：
在时刻 `t`，进程的工作集 `W(t, Δ)` 定义为最近 `Δ` 次内存访问中出现过的页面集合。算法倾向于只保留这个集合中的页面，从而贴合程序的局部性特征。

## R02

核心目标是降低缺页率并缓解抖动（thrashing）：
- 如果页面近期被频繁访问，就应继续驻留内存。
- 如果页面长时间未被访问，应优先回收。
- 通过 `Δ` 这个“时间窗”在“保留历史”与“快速淘汰冷页”之间折中。

## R03

形式化定义（按引用次数计时）：
- 引用串：`R = [r1, r2, ..., rn]`
- 时间窗：`Δ > 0`
- 工作集：`W(t, Δ) = { p | p 在区间 [t-Δ, t-1] 中至少被引用一次 }`
- 缺页：`rt` 不在当前驻留集时触发。

在本 MVP 中，另加一个工程约束 `max_frames`（物理帧上限），用于模拟内存有限场景。

## R04

本目录 `demo.py` 使用的数据结构：
- `resident: set[int]`：当前驻留页面。
- `last_seen: dict[int, int]`：页面最近一次访问时间戳。
- `records: list[StepRecord]`：每一步日志（命中、过期回收、淘汰、当前驻留集、累计缺页）。

这些结构均为标准库实现，便于阅读和复现。

## R05

单步访问流程（高层）：
1. 时间 `t += 1`。
2. 回收所有 `last_seen[p] < t-Δ` 的页面（不在工作集窗口内）。
3. 若访问页在 `resident` 中：命中。
4. 否则：缺页；若超出 `max_frames`，再按“最老最近访问时间”淘汰 1 页。
5. 更新当前页 `last_seen`，记录日志。

## R06

正确性直觉：
- 第 2 步保证“近期未用页面”被剔除，驻留集始终近似工作集定义。
- 对缺页时的补充淘汰（仅在帧上限约束下触发）使用最老页，符合局部性近似。
- 每步都更新 `last_seen`，使“是否在窗口内”的判定可直接由时间戳完成。

## R07

复杂度（设引用串长度 `n`，驻留集平均大小 `m`）：
- 时间复杂度：`O(n * m)`（每次过期扫描驻留集；`m` 在工程上受 `max_frames` 限制）。
- 空间复杂度：`O(m)`（驻留集和最近访问表）。

若需要更高性能，可引入分层队列或近似工作集维护结构。

## R08

与常见策略对比：
- FIFO：实现简单，但不感知局部性，可能出现 Belady 异常。
- LRU：感知近期性，但不显式表达“时间窗内活跃集合”。
- Working Set：显式建模程序当前活跃页集合，更偏向系统级负载控制与抖动抑制。

## R09

MVP 设计选择：
- 使用固定引用串，保证示例可重复。
- 以“引用次数”作为时间单位，避免依赖真实时钟。
- 保留 `max_frames` 参数，既能演示工作集思想，也能体现内存压力下的行为。
- 提供 LRU 缺页数作为基线对照。

## R10

运行方式（无交互输入）：

```bash
uv run python Algorithms/计算机-操作系统-0334-工作集算法/demo.py
```

程序会打印逐步日志表和最终缺页统计。

## R11

输出字段说明：
- `step`: 第几次访问。
- `page`: 当前访问页号。
- `hit`: 是否命中（`Y/N`）。
- `expired`: 因超出 `Δ` 窗口被回收的页。
- `evicted`: 在帧上限下额外淘汰的页。
- `resident`: 当前驻留集。
- `faults`: 累计缺页数。

## R12

参数敏感性：
- `Δ` 变小：回收更激进，可能提升换入换出频率。
- `Δ` 变大：驻留集更稳定，但占用更多帧。
- `max_frames` 变小：额外淘汰增多，缺页率通常上升。

实践中应结合负载特征（顺序扫描、循环热点、多相位访问）联合调参。

## R13

边界与异常处理：
- `delta <= 0` 会抛出 `ValueError`。
- `max_frames <= 0`（提供时）会抛出 `ValueError`。
- 引用串为空时，算法可自然返回 0 缺页（当前 demo 使用非空固定串以展示行为）。

## R14

工程落地注意点：
- 真实 OS 中通常用“虚拟时间”与采样近似，而非每次访问都精确维护集合。
- 多进程场景下还涉及全局帧分配、公平性与缺页率控制（例如 PFF 联动）。
- 在高并发系统中，维护成本与准确性要权衡，常用近似工作集。

## R15

建议测试集：
- 小循环热点：如 `[1,2,3,1,2,3,...]`。
- 阶段切换：如 `A` 热点后切到 `B` 热点。
- 大范围扫描：如线性遍历大量新页。
- 混合模式：短循环 + 偶发冷页。

重点观察缺页数、过期回收频率、额外淘汰频率。

## R16

常见实现错误：
- 过期判定边界写错（`<` 与 `<=` 混淆）。
- 先更新当前页时间再过期扫描，导致窗口语义偏移。
- 帧上限触发时未考虑“先过期再淘汰”的顺序。
- 统计口径不统一（命中/缺页记录时机不一致）。

## R17

可扩展方向：
- 增加 `trace` 文件输入与批量实验。
- 统计每页驻留时长、重入间隔、抖动指标。
- 加入 Page Fault Frequency (PFF) 调节 `Δ`。
- 与 Clock、WSClock、LRU-K 做多策略对比。

## R18

`demo.py` 的源码级算法流（8 步）：
1. `main()` 定义引用串、`delta` 与 `max_frames`，调用 `simulate_working_set()`。
2. `simulate_working_set()` 创建 `WorkingSetPager`，逐页调用 `access(page)`。
3. `access()` 先将 `time += 1`，再调用 `_expire_stale_pages()`。
4. `_expire_stale_pages()` 计算阈值 `threshold = time - delta`，回收所有 `last_seen[p] < threshold` 的驻留页。
5. 回到 `access()`，若 `page in resident` 则命中；否则记一次缺页。
6. 缺页且达到 `max_frames` 时，按 `last_seen` 最小值选择牺牲页（最老页）并淘汰。
7. 将当前页加入 `resident`（若原本不在）并更新 `last_seen[page] = time`。
8. 组装 `StepRecord` 写入 `records`；`main()` 最后打印日志表，并用 `simulate_lru_faults()` 给出同容量 LRU 对照缺页数。
