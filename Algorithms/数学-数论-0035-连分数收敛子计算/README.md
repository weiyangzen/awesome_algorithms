# 连分数收敛子计算

- UID: `MATH-0035`
- 学科: `数学`
- 分类: `数论`
- 源序号: `35`
- 目标目录: `Algorithms/数学-数论-0035-连分数收敛子计算`

## R01

连分数收敛子计算用于把有限或无限简单连分数的前缀项
`[a0; a1, a2, ..., ak]` 转换为有理数近似 `p_k / q_k`。

核心递推：

- `p_{-2}=0, p_{-1}=1, p_k=a_k*p_{k-1}+p_{k-2}`
- `q_{-2}=1, q_{-1}=0, q_k=a_k*q_{k-1}+q_{k-2}`

输入通常是整数项序列 `a0..an`，输出是每个前缀的收敛子 `(p_k, q_k)`。

## R02

- 古希腊《几何原本》已包含连分数思想相关的辗转相除结构。
- 17-18 世纪，Brouncker、Euler、Lagrange 系统化了连分数与逼近理论。
- Lagrange 证明了二次无理数连分数展开最终周期性，使“收敛子计算”成为数论基础工具。

## R03

很多数（如 `sqrt(2)`、`pi`、`e`）不能被有限小数精确表达，但在工程中常需“分母较小且误差小”的有理近似。收敛子提供了这一需求下的标准方案：它不是任意截断，而是由连分数结构导出的最优逼近路径之一。

## R04

设输入长度为 `n`（即有 `n` 个连分数项）。

- 算术步数复杂度：`O(n)`。
- 若考虑大整数位复杂度，单步主要是大整数乘加，整体可写为 `O(n * M(B))`，`M(B)` 为 `B` 位整数乘法复杂度，`B` 与收敛子规模相关。

## R05

- 若只流式输出当前收敛子，不保留历史，额外空间 `O(1)`。
- 若保存全部 `n` 个收敛子，空间 `O(n)`（不含大整数对象本身增长的位宽成本）。

## R06

以 `pi` 的前几项连分数 `[3; 7, 15, 1, 292]` 为例：

1. `k=0`: `3/1`
2. `k=1`: `22/7`
3. `k=2`: `333/106`
4. `k=3`: `355/113`
5. `k=4`: `103993/33102`

可见 `355/113` 已给出很高精度近似，是著名的 `pi` 分数逼近。

## R07

收敛子计算在以下方向非常关键：

- 有理逼近与误差控制。
- Pell 方程与二次无理数研究。
- Wiener attack 等密码分析中的小分母近似判定。
- 数值算法中“把实数近似为可解释分数”的工程步骤。

## R08

前置知识依赖：

- 整数四则与辗转相除法。
- 简单连分数定义与前缀截断。
- 分数运算（分子分母互素化不是必须步骤，但常见）。
- 基本误差界结论（收敛子误差上界）。

## R09

适用前提与边界：

- 输入项应为整数；标准简单连分数通常要求 `k>=1` 时 `a_k > 0`。
- 序列为空时无定义，需显式报错。
- 若连分数项来自浮点反推，项值可能受浮点误差影响，不等于真实数学展开。
- 对负数可计算，但解释上需统一 floor 约定。

## R10

正确性可由递推不变量给出：

1. 初始化给出第 `-2`、`-1` 层基值。
2. 每步由 `a_k` 线性组合前两层得到 `p_k, q_k`，与连分数分式化简一致。
3. 可证明行列式恒等式：`p_k*q_{k-1} - p_{k-1}*q_k = (-1)^{k-1}`。
4. 上式蕴含相邻收敛子分数不同且分母递增（在标准简单连分数条件下）。
5. 因而迭代到 `k` 时得到的正是前缀 `[a0;...;ak]` 的精确值。

## R11

数值稳定性分析：

- 若仅用整数递推，结果是精确的离散值，不存在浮点舍入传播。
- 误差主要来自“输入项来源”：若项由浮点数分解得到，浮点近似会影响后续项。
- 在 Python 中 `int` 为任意精度，避免了固定宽度溢出，但超大项会增加时间与内存常数。

## R12

真实成本考虑：

- 热点在大整数乘法与加法，不在循环控制。
- 项 `a_k` 较大时，`p_k, q_k` 位宽增长更快，常数成本显著增加。
- 流式输出可减少内存峰值；若要做误差分析通常要保留至少相邻两个收敛子。

## R13

误差与逼近性质（对无理数 `x` 的标准连分数收敛子）：

- 交错性：`(x - p_k/q_k)` 的符号通常交替。
- 误差上界：`|x - p_k/q_k| < 1/(q_k * q_{k+1}) <= 1/q_k^2`。
- 最优性（经典结论之一）：在分母不超过 `q_k` 的范围内，收敛子通常给出最优或近最优有理逼近。

## R14

鲁棒性与失效模式：

- 空输入序列会导致无意义输出，应直接抛错。
- 非整数项破坏定义，应做类型检查。
- 对浮点反推得到的连分数，若 `tol` 或 `max_terms` 设置不当，可能提前终止或出现“异常大项”。
- 若使用固定宽度整型语言实现，可能发生溢出并导致错误收敛子。

## R15

工程实现建议：

- 使用迭代版递推，避免递归栈。
- 为输入做最小校验：非空、整型、必要时约束 `a_k` 正性。
- 同时提供“精确分数回代验证”与“收敛子递推验证”，便于测试。
- 若输入来自实数，提供 `max_terms` 与 `tol` 两个安全阈值。

## R16

前驱/后继关系：

- 前驱基础：欧几里得算法（连分数展开与辗转相除同源）。
- 后继方向：
  - 周期连分数（求二次无理数周期）。
  - Pell 方程求最小解。
  - 连分数在 Diophantine 逼近与密码分析中的应用（如 Wiener attack）。

## R18

源码级算法流程（对应 `demo.py`，无第三方黑箱）可拆为 8 步：

1. 读取连分数项 `terms`，检查非空且每项为 `int`。
2. 初始化递推基值 `p_{-2}, p_{-1}, q_{-2}, q_{-1}`。
3. 逐项读入 `a_k`，计算 `p_k=a_k*p_{k-1}+p_{k-2}`、`q_k=a_k*q_{k-1}+q_{k-2}`。
4. 追加当前 `(p_k, q_k)` 到收敛子列表，并滚动更新前两层状态。
5. 对每个前缀 `terms[:k+1]`，用 `Fraction` 回代，交叉验证递推结果一致。
6. 检查行列式恒等式 `p_k*q_{k-1} - p_{k-1}*q_k = (-1)^{k-1}`，确保结构正确。
7. 若给定目标实数 `x`，计算 `abs(x - p_k/q_k)`，并与 `1/(q_k*q_{k+1})` 做误差界核验。
8. 输出最终收敛子与误差报告，完成可复现实验。
