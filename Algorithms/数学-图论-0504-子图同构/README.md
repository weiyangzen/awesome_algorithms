# 子图同构

- UID: `MATH-0504`
- 学科: `数学`
- 分类: `图论`
- 源序号: `504`
- 目标目录: `Algorithms/数学-图论-0504-子图同构`

## R01

子图同构（Subgraph Isomorphism）问题：给定模式图 `P` 和目标图 `G`，判断是否存在一个**单射** `f: V(P) -> V(G)`，使得 `P` 中每条边在 `G` 中都被保持，即：

- 若 `(u,v) in E(P)`，则 `(f(u), f(v)) in E(G)`。

本条目实现的是**无向有标号图**上的“非诱导”子图同构最小 MVP（不要求 `P` 中不存在的边在 `G` 中也不存在）。

## R02

问题输入与输出在 `demo.py` 中固定构造：

- 输入：
  - 一个目标图 `target`；
  - 两个模式图：`pattern_exists`（应存在映射）与 `pattern_not_exists`（应不存在映射）。
- 输出：
  - `found=True/False`；
  - 一组映射 `p{i} -> t{j}`（若存在）；
  - 搜索统计：扩展节点数、回溯次数、约简剪枝次数。

脚本无需交互输入，直接运行即可复现实验。

## R03

复杂度背景：

- 子图同构判定是经典 NP-Complete 问题；
- 最坏情况下需要指数级搜索；
- 实际求解依赖剪枝与启发式（候选过滤、约束传播、顶点排序等）降低平均搜索量。

本实现强调“可读、可验证”，不追求工业级极限性能。

## R04

本 MVP 采用 Ullmann 风格框架：

1. 建立候选矩阵 `M`（模式点可映射到哪些目标点）；
2. 对 `M` 做邻接一致性约简（arc-consistency-like）；
3. 使用回溯搜索逐步固定映射；
4. 每次赋值后再次约简与剪枝；
5. 找到完整映射即返回。

这是一种“约束传播 + 深度优先搜索”的经典组合。

## R05

图表示方式：

- 使用 `Graph` 数据类；
- 结构：`adj: numpy.ndarray`（邻接矩阵）、`labels: Tuple[str, ...]`（顶点标签）；
- 约束：
  - 无向图（`adj[u,v] == adj[v,u]`）；
  - 不支持自环（MVP 有意简化）。

使用邻接矩阵便于实现局部一致性检查和候选矩阵约简。

## R06

初始候选矩阵 `M` 的构建规则（必要条件）：

- 标签匹配：`label_P(i) == label_G(j)`；
- 度约束：`deg_P(i) <= deg_G(j)`。

若某一模式顶点整行无候选（`M[i,:]` 全 False），可立即判定无解。

## R07

候选约简 `refine_candidates` 规则：

- 对每个候选对 `(i,j)`，模式图中 `i` 的每个邻点 `x`，都必须在目标图中 `j` 的邻居集合里找到至少一个候选 `y`，使 `M[x,y]=True`；
- 若条件不满足，删除该候选；
- 迭代至不再变化（不动点）。

这一步在进入深层回溯前可显著缩小搜索空间。

## R08

搜索策略：

- 选择下一个模式顶点时使用 MRV（Minimum Remaining Values）：优先选候选数最少的未映射顶点；
- 赋值时保持单射：目标顶点不能被复用；
- 局部一致性检查：已映射顶点之间，`P` 的边必须映射到 `G` 的边；
- 赋值后立即做一次候选约简，失败则回溯。

## R09

正确性要点（简述）：

- 完整性：回溯会枚举所有满足单射与局部约束的候选分支，因此不会漏掉可行解；
- 正确性：每次扩展都强制边保持与候选一致性，最终得到的完整映射必为合法子图同构；
- 安全剪枝：`refine_candidates` 仅删除不可能延拓为可行解的候选，不改变可行解集合。

## R10

`demo.py` 设计了两个确定性案例：

- `Case-1 Exists`：三角形加尾巴（带标签）在目标图中存在；
- `Case-2 NotExists`：同标签 `K4` 在目标图中不存在。

这样可以同时验证“能找到解”和“能正确判无解”。

## R11

代码结构：

- `Graph`：图构造与基础属性；
- `initial_candidate_matrix`：初始候选生成；
- `refine_candidates`：候选约简；
- `choose_next_pattern_vertex`：MRV 选点；
- `locally_consistent`：局部边保持检查；
- `backtrack_search`：DFS 回溯主流程；
- `solve_subgraph_isomorphism`：求解入口；
- `brute_force_subgraph_isomorphism`：小规模暴力校验器（仅用于演示自检）；
- `main`：运行样例并打印结果。

## R12

运行方式：

```bash

```

预期行为：打印两组案例结果，并在内部执行一次“回溯结果 vs 暴力校验”一致性断言。

## R13

输出解读：

- `found=True` 且有 `mapping={...}`：找到一个合法嵌入；
- `found=False`：不存在满足标签与边约束的嵌入；
- `nodes_expanded` 越大，表示搜索树访问越多；
- `pruned_by_refine` 越大，表示约简剪枝越有效。

MVP 只返回第一组可行映射，不枚举全部同构解。

## R14

MVP 边界：

- 支持：无向、离散标签、非诱导同构判定；
- 不支持：
  - 边标签/权重约束；
  - 有向图；
  - 多重边与自环；
  - 大规模工程优化（位运算、并行、索引结构）。

该取舍用于保证实现短小透明。

## R15

常见实现陷阱：

- 只做度过滤，不做邻接一致性，导致回溯爆炸；
- 忘记单射约束（多个模式点映射到同一目标点）；
- 在“非诱导”与“诱导”定义之间混淆，导致误判；
- 只看正例，不做负例和暴力对照校验。

`demo.py` 中通过负例和 brute-force 小图校验降低误实现风险。

## R16

与常见算法关系：

- Ullmann：候选矩阵 + 约简 + 回溯（本实现核心思想）；
- VF2/VF2++：通常使用更精细的状态与可行性规则，工程上更常见；
- LAD/RI 等：在某些图分布上有更强剪枝策略。

本条目选择 Ullmann 风格是为了更直观展示“候选传播”机制。

## R17

可扩展方向：

1. 切换到诱导子图同构（增加“非边也需保持”约束）；
2. 引入边标签与属性过滤；
3. 加入顶点排序策略（如按度/稀有标签优先）；
4. 用位集（bitset）重写候选运算提升性能；
5. 扩展为“枚举全部解”而非仅返回首解。

## R18

`demo.py` 的源码级算法流程（8 步）：

1. `build_demo_graphs` 构造目标图与两个模式图（存在/不存在案例），并附上顶点标签。  
2. `solve_subgraph_isomorphism` 调用 `initial_candidate_matrix`，按“标签匹配 + 度约束”生成候选矩阵 `M`。  
3. 对 `M` 执行 `refine_candidates`：若模式点邻居无法在目标点邻域中找到任何候选，则删除该候选并迭代到不动点。  
4. `backtrack_search` 进入 DFS，先通过 `choose_next_pattern_vertex`（MRV）选择候选最少的未映射模式顶点。  
5. 对每个候选目标点 `j`，先做 `locally_consistent`（已映射边保持）检查，不合法直接跳过。  
6. 若尝试 `i -> j`，则在 `M` 中固定该行并清除同列其他行（保证单射），随后再次调用 `refine_candidates` 进行前向剪枝。  
7. 若约简后出现空行则该分支立即失败并回溯；若所有模式点都已映射则返回该解。  
8. `run_case` 输出映射与统计，同时调用 `brute_force_subgraph_isomorphism` 做小规模真值对照，确保 MVP 求解器判定一致。
