# 筛法求积性函数前缀和

- UID: `MATH-0029`
- 学科: `数学`
- 分类: `数论`
- 源序号: `29`
- 目标目录: `Algorithms/数学-数论-0029-筛法求积性函数前缀和`

## R01

问题：给定上界 `N`，高效计算某个积性函数 `f(n)` 的前缀和

`S(N) = sum_{k=1..N} f(k)`。

目标不是逐个分解每个整数（那样通常是 `O(N sqrt(N))` 或更慢），而是用筛法把整体复杂度压到线性级别（`O(N)`）。

## R02

应用场景：

- 批量计算欧拉函数前缀和 `sum phi(k)`。
- 批量计算莫比乌斯函数前缀和 `sum mu(k)`（Mertens 函数）。
- 批量计算约数个数 `tau(n)`、约数和 `sigma(n)` 等积性函数前缀和。
- 作为数论分块、杜教筛、Min_25 等更高阶算法的基础数据层。

## R03

核心数学定义：

- 积性函数：若 `gcd(a,b)=1`，则 `f(ab)=f(a)f(b)`，且通常有 `f(1)=1`。
- 对于任意正整数 `n`，可唯一写成 `n = p^e * r`，其中 `p` 是 `n` 的最小质因子，`e>=1`，且 `gcd(p,r)=1`。
- 若已知 `f(p^e)` 的表达式（prime-power 形式），则 `f(n)=f(p^e)*f(r)`。

## R04

筛法总体思路：

1. 用线性筛一次性得到每个 `n` 的最小质因子信息。
2. 同时维护 `n` 的 `(p, e, r)` 分解参数：
   - `p = lp[n]`（least prime）
   - `e = exp[n]`（`p` 的幂次）
   - `r = rest[n]`
3. 利用积性：`f[n] = f[rest[n]] * f_p_power(p, exp[n])`。
4. 再做一遍前缀累加得到 `prefix[n] = prefix[n-1] + f[n]`。

## R05

为什么是线性复杂度：

- 线性筛的关键性质：每个合数只会在“被其最小质因子第一次筛到”时处理一次。
- 因此筛部分是 `O(N)`。
- 对 `f[n]` 的计算是每个 `n` 常数次操作，总计 `O(N)`。
- 前缀和再一趟 `O(N)`。
- 总复杂度 `O(N)`，空间 `O(N)`。

## R06

与“先分解再算”方法对比：

- 朴素法：对每个 `k<=N` 做试除分解，通常最坏接近 `O(N sqrt(N))`。
- 本方法：一次构建全量分解骨架，然后 `O(1)` 转移得到所有 `f[k]`。
- 当 `N` 到 `1e6`、`1e7` 量级时，筛法优势明显。

## R07

本题 MVP 设计（`demo.py`）：

- 提供一个通用函数 `sieve_prefix_multiplicative(limit, prime_power_value)`。
- 参数 `prime_power_value(p, e)` 返回 `f(p^e)`。
- 用同一框架演示三类函数：
  - `phi(n)`：`phi(p^e)=p^(e-1)*(p-1)`
  - `mu(n)`：`mu(p)= -1`，`mu(p^e)=0 (e>=2)`
  - `tau(n)`：`tau(p^e)=e+1`

## R08

关键数据结构：

- `lp[i]`：`i` 的最小质因子。
- `exp[i]`：`lp[i]` 在 `i` 中出现次数。
- `rest[i]`：`i / lp[i]^exp[i]`。
- `f[i]`：函数值 `f(i)`。
- `prefix[i]`：前缀和 `sum_{k<=i} f(k)`。
- `primes`：筛过程中收集到的质数表。

## R09

线性筛递推细节：

- 当 `i` 是新质数：`lp[i]=i, exp[i]=1, rest[i]=1`。
- 枚举质数 `p` 令 `x=i*p`：
  - 若 `p < lp[i]`（或 `p` 不整除 `i`）：`lp[x]=p, exp[x]=1, rest[x]=i`。
  - 若 `p == lp[i]`：`lp[x]=p, exp[x]=exp[i]+1, rest[x]=rest[i]`，并 `break`。
- 该 `break` 保证“每个合数只被最小质因子路径命中一次”。

## R10

函数值恢复公式：

对 `n>=2`：

- `p = lp[n]`
- `e = exp[n]`
- `r = rest[n]`
- `f[n] = f[r] * prime_power_value(p, e)`

初值：`f[1]=1`，`prefix[0]=0`。

## R11

正确性要点（简述）：

- 唯一分解：`n = p^e * r` 由最小质因子拆分唯一确定。
- 互素性：`gcd(p^e, r)=1`。
- 积性：`f(n)=f(p^e)f(r)`，递归到 `f(1)` 收敛。
- 筛的构建保证每个 `n` 都得到正确 `(p,e,r)`，故 `f[n]` 全部正确，前缀和自然正确。

## R12

边界与异常处理：

- `limit < 1` 抛出 `ValueError`。
- `limit = 1` 时返回 `f[1]=1`、`prefix[1]=1`。
- Python 整数自动大数，不会发生固定宽度整型溢出。
- 对 `mu` 前缀和会出现负值，属于正常现象。

## R13

示例输出（`N=100000`）会打印：

- `sum_{k<=N} phi(k)`
- `sum_{k<=N} mu(k)`
- `sum_{k<=N} tau(k)`
- 以及若干采样点（如 `1,10,100,1000,100000`）的前缀值。

并在小范围内用朴素分解交叉校验，确保 MVP 结果可信。

## R14

可扩展方向：

- 新增任意积性函数只需更换 `prime_power_value(p,e)`。
- 例如：
  - `sigma(p^e) = (p^(e+1)-1)/(p-1)`
  - 指示平方自由函数、Jordan 函数等。
- 框架不变，复用筛分解骨架即可。

## R15

性能建议：

- `N <= 1e6`：纯 Python 通常可接受。
- 若追求更大 `N`：
  - 优先使用 `PyPy`；
  - 或改写为 `C++/Rust`；
  - 或在 Python 中通过 `numba` 做 JIT（本 MVP 不强依赖第三方）。

## R16

常见错误：

- 忘记 `f[1]=1` 导致全表错误。
- 线性筛中 `p==lp[i]` 分支忘记 `break`，复杂度和正确性都可能出问题。
- 把“积性”误当“完全积性”（即错误套用 `f(p^e)=f(p)^e`）。
- `mu` 的 prime-power 规则写错：`e>=2` 必须为 `0`。

## R18

`demo.py` 源码级流程（8 步）：

1. 校验 `limit`，初始化数组 `lp/exp/rest/f/prefix` 与 `primes`。
2. 运行线性筛，逐步填充每个整数的最小质因子分解信息 `(lp,exp,rest)`。
3. 设置 `f[1]=1`，作为积性递推基点。
4. 对每个 `n>=2`，读取 `(p,e,r)`，调用 `prime_power_value(p,e)` 得到 `f(p^e)`。
5. 按 `f[n]=f[r]*f(p^e)` 计算完整函数值数组。
6. 一次线性扫描构建前缀和 `prefix[n]=prefix[n-1]+f[n]`。
7. 用三套 `prime_power_value`（`phi`/`mu`/`tau`）分别调用同一框架，演示通用性。
8. 在小范围用朴素分解计算做交叉校验，最后输出关键结果与采样点前缀和。
