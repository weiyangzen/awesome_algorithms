# 束方法 (Bundle Method)

- UID: `MATH-0378`
- 学科: `数学`
- 分类: `凸优化`
- 源序号: `378`
- 目标目录: `Algorithms/数学-凸优化-0378-束方法_(Bundle_Method)`

## R01

束方法（Bundle Method）是求解非光滑凸优化的重要算法。它不直接依赖单点梯度下降，而是维护一组历史切平面（bundle），构造函数的分段线性下界模型，再结合近端二次项求下一步。

直观上，束方法做的是：

- 用多个子梯度“拼”出当前对目标函数的局部近似；
- 通过近端项控制步长和稳定性；
- 用 serious/null step 机制区分“真正推进”与“仅更新模型”。

## R02

发展脉络可概括为：

- 切平面法（cutting-plane）提供了“线性下界 + 迭代逼近”的思想；
- 近端点方法（proximal point）提供了稳定性和收敛控制；
- 束方法把两者结合，成为工业界处理凸非光滑问题（如拉格朗日松弛、风险优化）的核心工具之一。

与早期纯切平面法相比，束方法最大的改进是“近端正则 + serious/null step 判别”，显著降低震荡与发散风险。

## R03

它解决的核心问题是：  
当目标函数非光滑（例如 `max`、`L1`、支持函数等）时，单一子梯度法往往收敛慢且路径抖动大，难以稳定利用历史信息。

束方法通过“多切面记忆 + 近端子问题”来缓解这一点：

- 历史切面累积了局部几何信息；
- 子问题优化的不是原函数，而是一个可控的下界模型；
- serious/null step 决策确保“下降不足时不贸然移动中心点”。

## R04

本目录 `demo.py` 的具体模型是二维凸函数：

- `f(x) = max_i (a_i^T x + b_i) + (mu/2)||x||_2^2`

其中前半部分是非光滑 max-affine，后半部分二次项保证解唯一。脚本中构造了 `x*=0` 为已知最优点。

束模型使用切面：

- `m_k(x) = max_j { f(y_j) + g_j^T (x - y_j) }`

近端束子问题：

- `x_trial = argmin_x m_k(x) + (1/(2t))||x - x_c||_2^2`

并通过 serious/null 判据更新中心点 `x_c`。

## R05

设：

- 变量维度 `d`；
- 当前束大小 `B`；
- 外层迭代轮数 `K`；
- 对偶投影梯度内循环轮数 `T_inner`。

则每次内循环主要成本是矩阵乘法 `O(B*d)`，单次外层复杂度约 `O(T_inner * B * d)`，总复杂度约：

- `O(K * T_inner * B * d)`

内存复杂度主要存储切面（`y,f,g`）与小矩阵，约 `O(B*d)`。

## R06

一个数值直觉：

- 若当前中心点 `f(x_c)=2.4`；
- 束子问题给出模型值（含近端项）`v_k=1.9`，则预测下降 `pred=0.5`；
- 若真实试探点 `f(x_trial)=2.2`，真实下降 `actual=0.2`，比值 `actual/pred=0.4`。

当阈值 `gamma=0.25` 时，该步可接受为 serious step；若比值更低，则作为 null step，仅保留新切面而不移动中心。

## R07

优点：

- 对非光滑凸函数比朴素子梯度法更稳定；
- 能系统利用历史子梯度信息；
- serious/null 机制在“下降不足”时避免坏步。

局限：

- 需要解近端子问题，单步成本高于子梯度法；
- 参数（如近端系数、接受阈值、束大小）需要调节；
- 在高维大规模场景下，束管理与子问题求解会带来工程负担。

## R08

建议前置知识：

- 凸函数与子梯度定义；
- 切平面下界 `f(x) >= f(y) + g^T(x-y)`；
- 近端正则（`(1/2t)||x-x_c||^2`）的稳定化作用；
- 简单的对偶与单纯形投影（本实现使用投影梯度法解对偶）。

## R09

适用场景：

- 目标函数是凸但不可微（`max`、`L1`、piecewise linear/quadratic）；
- 需要稳健下降而不是极致单步速度；
- 允许维护有限历史信息并解小型子问题。

不太适用：

- 光滑凸问题（L-BFGS/牛顿法通常更高效）；
- 目标函数无凸性保证；
- 资源非常紧张，无法承担子问题迭代。

## R10

本实现中的正确性检查点：

1. 子梯度 oracle 必须满足切平面下界性质；  
2. 束模型系数 `beta_j = f(y_j) - g_j^T y_j` 不能写错；  
3. 近端子问题目标要包含 `1/(2t)` 而非 `t/2`；  
4. serious/null 判断需基于 `actual/pred`；  
5. 每轮都要把试探点切面加入束（即使 null step）；  
6. 对偶变量需始终投影到概率单纯形。

`demo.py` 还包含一个平滑点有限差分梯度检查与最终断言门禁。

## R11

数值稳定策略：

- 对 `pred` 分母使用 `max(pred, 1e-12)` 防除零；
- 对偶步长用 Lipschitz 上界 `1/L`，避免内循环发散；
- 束大小限制 `bundle_cap`，防止历史切面无限增长；
- 通过 `np.allclose` 判定中心切面并固定保留，避免模型漂移。

## R12

调参建议（对应 `demo.py`）：

- `prox_t`：近端尺度。太小步子保守，太大可能 oscillation；
- `gamma`：serious step 接受阈值。常用 `0.1~0.3`；
- `bundle_cap`：束大小。太小信息不足，太大计算变慢；
- `inner_max_iter`：对偶子问题求解精度预算；
- `tol`：预测下降停止阈值，控制终止精细度。

实操上可先固定 `gamma=0.25`，再在 `prox_t` 与 `bundle_cap` 上做小网格扫描。

## R13

理论层面（凸问题）：

- 束模型由子梯度切面构成，是原函数的全局下界；
- 近端项使子问题强凸，从而试探点唯一；
- serious/null 策略在不充分下降时保留中心点，同时增强模型，通常改善全局收敛表现。

本条目是“确定性优化算法”，不涉及概率成功率或随机近似比。

## R14

常见失效模式与修复：

- 失效：`prox_t` 过大，试探点跳跃、null step 过多。  
  修复：减小 `prox_t` 或提高 `gamma`。

- 失效：`bundle_cap` 过小，模型信息不足。  
  修复：适当增大束容量（例如 8 -> 14）。

- 失效：内循环太少，子问题解太粗糙。  
  修复：提高 `inner_max_iter` 或改进步长。

- 失效：切面公式实现错误，导致预测下降失真。  
  修复：逐项核对 `f(y)+g^T(x-y)` 与 `beta` 推导。

## R15

工程落地建议：

- 将束方法作为“非光滑模块”的求解器，而不是全系统统一求解器；
- 给每轮记录 `f_center/pred/ratio/serious_or_null` 便于回放；
- 在长任务中增加 checkpoint，必要时重启时保留最近束；
- 若目标是可分块结构，可考虑 block-wise bundle 或并行切面评估。

## R16

与相关算法关系：

- 子梯度法：单切面、单点信息，成本低但震荡大；
- 切平面法：多切面但无近端稳定化，易出现不稳定大步；
- 近端点法：稳定但不显式利用多子梯度历史；
- 束方法：把“多切面记忆”和“近端稳定化”组合起来，是折中方案。

在凸非光滑优化中，束方法通常是比朴素子梯度更“工程可用”的基线。

## R17

`demo.py` 的实现内容：

- 构造一个二维 max-affine + quadratic 的凸非光滑目标；
- 实现 oracle（函数值 + 子梯度）；
- 实现束子问题的对偶投影梯度求解；
- 实现 serious/null step 与束裁剪；
- 提供子梯度下降基线对比；
- 输出最终值、步数统计，并执行断言验证。

## R18

`demo.py` 的源码级算法流程（8 步）：

1. `build_problem` 定义 max-affine 参数 `A,b` 与二次系数 `mu`，形成测试目标。  
2. `oracle` 在给定点计算 `f(x)` 并返回一个合法子梯度 `g(x)`。  
3. 每条切面保存为 `Cut(y,f,g)`，并转成线性模型项 `g^T x + beta`（`beta=f-g^T y`）。  
4. `solve_bundle_subproblem` 把近端子问题转到对偶单纯形上，用投影梯度法迭代 `alpha`。  
5. 由 `x = x_c - t * G^T alpha` 还原试探点，计算模型值与子问题最优值 `v_k`。  
6. `bundle_method` 计算 `pred = f(x_c)-v_k` 与 `ratio = actual/pred`，执行 serious/null 判定。  
7. 无论 serious 还是 null，都把试探点切面加入束，并用 `prune_bundle` 控制束大小且保留中心切面。  
8. `main` 汇总运行、打印 serious/null 统计，与子梯度基线比较，并用断言验证收敛质量。  

这 8 步完整覆盖了“子梯度建模 -> 近端子问题 -> 步型判定 -> 束更新 -> 收敛校验”的最小可运行束方法流程。
