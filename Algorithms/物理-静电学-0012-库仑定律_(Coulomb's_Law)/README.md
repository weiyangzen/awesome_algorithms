# 库仑定律 (Coulomb's Law)

- UID: `PHYS-0012`
- 学科: `物理`
- 分类: `静电学`
- 源序号: `12`
- 目标目录: `Algorithms/物理-静电学-0012-库仑定律_(Coulomb's_Law)`

## R01

库仑定律描述两个点电荷之间的静电作用力大小与方向。

- 标量形式（力的大小）:
  - `|F| = (1 / (4*pi*epsilon)) * |q1*q2| / r^2`
- 真空中 `epsilon = epsilon_0`，常写成:
  - `|F| = k * |q1*q2| / r^2`，其中 `k = 1/(4*pi*epsilon_0)`。
- 向量形式（作用在 `q1` 上，由 `q2` 产生）:
  - `F_12 = (1 / (4*pi*epsilon)) * q1*q2 * (r1-r2) / |r1-r2|^3`

输入通常为电荷量 `q1, q2`（单位 C）和两电荷间距离 `r`（单位 m），输出为力（单位 N）。

## R02

- 提出者: Charles-Augustin de Coulomb（库仑）。
- 关键时间: 1785 年左右，通过扭秤实验系统测量电荷间作用力。
- 历史定位: 静电学最核心定律之一，与万有引力的反平方结构类似，但适用于电荷相互作用。

## R03

在静电问题中，最基础的任务是确定“两个带电体之间作用力有多大、方向如何”。工程和科学中的大量模型都建立在这一点上，例如电容器场强估算、离子束近似分析、微电子器件中的电场相互作用等。

库仑定律把该问题降为一个低维、可闭式求解的表达式。对于多电荷系统，再结合叠加原理即可扩展为向量求和问题。

## R04

单次两电荷作用力计算的时间复杂度为 `O(1)`。

若对 `n` 对电荷批量计算（本目录 `demo.py` 使用 NumPy 向量化），总时间复杂度是 `O(n)`，其中常数项主要来自乘法、除法与平方运算。

对 `N` 个粒子的全对相互作用，若不做近似（如 Barnes-Hut/FMM），复杂度为 `O(N^2)`。

## R05

- 单次计算额外空间复杂度: `O(1)`。
- 向量化批量计算空间复杂度: `O(n)`（保存输入数组与输出数组）。
- 若扩展到全对矩阵（`N x N`）并显式存储，空间可能达到 `O(N^2)`。

## R06

示例: 在真空中，`q1 = 2e-6 C`，`q2 = -3e-6 C`，`r = 0.05 m`。

1. 常数 `k ~= 8.9875517923e9 N*m^2/C^2`
2. 电荷乘积绝对值 `|q1*q2| = 6e-12`
3. 距离平方 `r^2 = 2.5e-3`
4. `|F| = k * |q1*q2| / r^2 ~= 21.57 N`

因为电荷异号，方向为相互吸引。

## R07

库仑定律是静电学中的“最小原子规则”。

- 理论层: 连接电荷、场、势能等核心概念。
- 算法层: 提供可直接编码的基础算子，可嵌入粒子模拟与场求解。
- 工程层: 为 MEMS、电容传感、电喷雾、等离子体近似模型等提供第一性估算。

## R08

前置知识与公式依赖:

- 向量代数（方向、范数、单位向量）。
- SI 单位体系（C、m、N）。
- 介电常数关系:
  - `epsilon = epsilon_r * epsilon_0`
- 多体扩展依赖叠加原理:
  - `F_total = sum_i F_i`

## R09

适用前提与边界:

- 主要适用于静电或准静态场景。
- 理想模型默认点电荷，或带电体之间距离远大于其尺寸。
- 介质可用标量 `epsilon_r` 近似时成立；各向异性/非线性介质需更复杂模型。
- `r > 0` 必须成立；`r = 0` 存在奇异性。
- 高速相对运动、辐射效应明显时，需使用更完整电动力学模型（如 Maxwell-Lorentz 框架）。

## R10

正确性依据（简述）:

1. 实验事实: 扭秤实验支持力与电荷乘积成正比、与距离平方成反比。
2. 场论一致性: 该形式与点电荷电场 `E ~ 1/r^2` 一致。
3. 方向性一致: 同号斥、异号吸可由 `q1*q2` 的符号统一表达。
4. 维度一致性: `k*q^2/r^2` 的单位为牛顿（N），量纲闭合。

## R11

数值稳定性与误差传播:

- 该公式不是迭代算法，无迭代收敛不稳定问题。
- 主要风险在极端尺度:
  - `r` 很小时 `1/r^2` 会急剧放大，导致溢出或不合理物理近似。
  - `r` 很大时可能出现下溢或有效数字损失。
- 一阶相对误差传播近似:
  - `dF/F ~= dq1/q1 + dq2/q2 + 2*dr/r + depsilon/depsilon`
- 工程上应设置最小距离截断 `r_min` 并统一单位，以降低数值异常。

## R12

真实计算成本（超越渐近符号）:

- 标量计算极快，瓶颈通常不在公式本身。
- 批量任务的主要开销:
  - 内存带宽（数组读写）
  - 大规模广播中间数组分配
  - Python 循环开销（可由 NumPy 向量化降低）
- 若进入 `O(N^2)` 全对计算，真正瓶颈会迅速转为算力和内存规模，而非单公式复杂度。

## R13

误差界/近似比/概率保证:

- 库仑定律在“点电荷 + 静电近似 + 介质参数已知”条件下是确定性闭式关系，不属于随机或近似算法。
- 若将输入测量误差上界分别记为 `|dq1|, |dq2|, |dr|`，则一阶误差上界可用线性传播估计:
  - `|dF|/|F| <= |dq1|/|q1| + |dq2|/|q2| + 2|dr|/r + |depsilon|/epsilon`
- 该界不是概率保证，而是局部线性误差估计。

## R14

鲁棒性与失效模式:

- `r <= 0`: 数学非法或奇异，必须在代码中显式拒绝。
- 输入单位混乱（mm 与 m 混用）: 会引入数量级错误。
- 忽略介质影响（`epsilon_r`）: 在液体/高介电材料中会显著偏差。
- 把有限尺寸电极当点电荷: 在近场会系统性失真。
- 多体场景若只算单对力: 会遗漏叠加效应。

## R15

工程实现要点:

- 先做输入检查: `r > 0`, `epsilon_r > 0`。
- 统一 SI 单位与数量级，必要时在入口处做单位转换。
- 将“大小”和“向量方向”拆开实现，便于测试与复用。
- 批量计算优先 NumPy 向量化，避免 Python for 循环。
- 增加交叉实现校验（本目录 demo 使用 PyTorch 与 NumPy对照）以降低实现错误风险。

## R16

前驱/后继算法与应用链路:

- 前驱基础:
  - 点电荷模型、叠加原理、向量代数。
- 后继扩展:
  - 多体电荷相互作用（全对求和、邻域截断、FMM 等近似算法）。
  - 从力积分到势能: `U = k*q1*q2/r`。
  - 与 Poisson/Laplace 方程耦合的连续场求解。
- 典型应用:
  - 电容器与静电执行器估算。
  - 带电粒子初步轨道模拟。
  - 传感器和微纳器件中的静电力建模。

## R17

Python MVP 说明（见 `demo.py`）:

- 目标: 提供可运行、非交互、可复核的库仑定律最小实现。
- 依赖:
  - `numpy`: 向量化计算与数组运算。
  - `scipy.constants`: 使用 `epsilon_0` 构造物理常数 `k`。
  - `pandas`: 结构化展示批量样例结果。
  - `scikit-learn`: 对 `log(F)-log(r)` 做线性拟合，验证斜率接近 `-2`。
  - `torch`: 提供第二套张量实现，与 NumPy 结果交叉校验。
- 主要函数:
  - `coulomb_force_magnitude`: 标量/批量力大小。
  - `coulomb_force_vector`: 三维向量力（作用在 `q1` 上）。
  - `inverse_square_fit`: 反平方律回归检验。
  - `torch_force_magnitude`: 张量实现一致性检查。
- 运行方式:
  - 在仓库根目录执行 `uv run python Algorithms/物理-静电学-0012-库仑定律_(Coulomb's_Law)/demo.py`

## R18

源码级流程拆解（`demo.py`，避免把第三方包当黑箱）:

1. 用 `scipy.constants.epsilon_0` 读取真空介电常数，按 `k = 1/(4*pi*epsilon_0)` 显式计算库仑常数。
2. 在 `coulomb_force_magnitude` 中先做输入约束检查（`epsilon_r > 0`, `r > 0`），再按元素应用 `k*|q1*q2|/(epsilon_r*r^2)`。
3. 在 `coulomb_force_vector` 中计算 `diff = r1-r2` 与 `distance = ||diff||`，再按 `k*q1*q2*diff/distance^3` 得到方向正确的三维力向量。
4. 在 `build_demo_table` 中构造固定样本，使用 NumPy 批量计算并用 Pandas 输出结构化结果，便于人工核查。
5. 在 `inverse_square_fit` 中将 `distances` 与 `forces` 做对数变换，传入 `LinearRegression.fit`，并显式读取斜率、截距、`R^2` 三个可解释量。
6. 用 `torch_force_magnitude` 复现同一公式路径（张量版），与 NumPy 结果做逐元素误差比较，输出 `max |difference|`。
7. `main` 顺序执行以上模块，打印常数、批量结果、反平方拟合、框架一致性和三维向量示例，保证脚本一次运行即可完整验证。
