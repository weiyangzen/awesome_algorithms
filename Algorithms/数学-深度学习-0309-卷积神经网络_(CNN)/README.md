## R01

卷积神经网络（Convolutional Neural Network, CNN）是面向网格结构数据（如图像）的深度学习模型。

核心思想：
- 局部感受野：每个卷积核只看局部邻域，不直接连接全图；
- 权重共享：同一个卷积核在空间上滑动复用，显著减少参数量；
- 分层表征：浅层学边缘/纹理，深层学更抽象语义；
- 池化或步幅下采样：降低分辨率、增强平移鲁棒性。

本条目 MVP 在 `sklearn` 手写数字集（8x8 灰度图）上训练一个小型 CNN，完成 10 类分类任务。

## R02

本条目问题定义（监督学习分类）：

- 输入：
  - 图像 `X in R^(N x 1 x 8 x 8)`；
  - 标签 `y in {0,1,...,9}^N`；
  - 超参数：`batch_size, epochs, lr, weight_decay`。
- 输出：
  - 训练后的模型参数 `theta`；
  - 测试集分类预测 `y_pred`；
  - 指标日志（`train_loss/train_acc/test_loss/test_acc`）。

目标：最小化多分类交叉熵，并在测试集达到可验证精度阈值。

## R03

本 MVP 的数学与结构：

1. 卷积层（二维离散卷积，框架中常实现为相关运算）：

`z_k(i,j) = sum_{u,v,c} W_k(c,u,v) * x_c(i+u,j+v) + b_k`

2. 非线性激活：`a = ReLU(z) = max(0, z)`。

3. 下采样：第一层后使用 `MaxPool2d(2)` 将 `8x8` 降到 `4x4`。

4. 分类头：
- 特征抽取：`Conv(1->16) -> ReLU -> MaxPool -> Conv(16->32) -> ReLU -> Conv(32->32) -> ReLU`；
- 展平后接 `Linear(32*4*4 -> 64) -> ReLU -> Linear(64 -> 10)` 输出 logits。

5. 损失函数：

`L = - (1/B) * sum_{n=1}^B log( softmax(logits_n)[y_n] )`

6. 参数更新：使用 Adam（`torch.optim.Adam`）按反向传播梯度迭代。

## R04

时间复杂度（单次前向主项，批量 `B`）：

- 第一层卷积：约 `O(B * H * W * C_in * C_out * K^2)`；
- 第二层卷积：同理按其通道数计算；
- 池化/激活/全连接通常低于卷积主项。

本实现输入固定 `8x8`，网络很小，因此每 epoch 成本主要由两层卷积与数据遍历决定；总训练复杂度近似线性于 `epochs * N`。

## R05

空间复杂度：

- 参数存储：`O(P)`，其中 `P` 为 CNN 参数总数；
- 激活与梯度：与 batch 和中间特征图大小相关，约 `O(B * feature_maps)`；
- 数据集存储：`O(N * 8 * 8)`。

本 MVP 网络尺寸很小，内存占用主要来自 mini-batch 激活张量，而非参数本身。

## R06

卷积微例（单通道 3x3 卷积核）：

若输入局部块与卷积核分别是：

`X_patch = [[1,2,0],[0,1,3],[2,1,0]]`

`W = [[1,0,-1],[1,0,-1],[1,0,-1]]`

则该位置输出（省略偏置）为逐元素乘积求和：

`1*1 + 2*0 + 0*(-1) + 0*1 + 1*0 + 3*(-1) + 2*1 + 1*0 + 0*(-1) = 0`

这反映了 CNN 对局部结构（此处类似竖直边缘模板）响应的机制。

## R07

CNN 的工程价值：

- 对图像任务比全连接网络参数更经济；
- 具有局部结构归纳偏置，训练更高效；
- 可扩展到分类、检测、分割、OCR 等视觉任务；
- 与现代训练范式（批训练、自动微分、GPU）高度兼容。

## R08

理论基础（工程化摘要）：

1. 图像是二维局部相关信号，卷积适合捕捉局部模式。  
2. 权重共享将平移等价性编码进模型结构，减少样本复杂度。  
3. 多层堆叠实现“从低级到高级”的特征组合。  
4. 反向传播通过链式法则计算每个参数梯度。  
5. 交叉熵与 softmax 共同构成多分类概率学习目标。

## R09

适用条件与局限：

- 适用：图像/语音谱图等具有网格邻域结构的数据；
- 适用：希望利用局部平移先验而不是完全无结构建模；
- 局限：
  - 对旋转、尺度大幅变化仍可能敏感；
  - 小数据下可能过拟合；
  - 网络过深时需更复杂正则化和训练技巧。

## R10

正确性框架（本实现）：

1. 数据与标签由 `sklearn` 标准数据集提供，且分层划分训练/测试。  
2. `TinyCNN.forward` 明确定义从输入到 logits 的计算图。  
3. 训练循环显式执行 `forward -> loss -> backward -> step`。  
4. 每个 epoch 在训练集与测试集独立评估，避免只看训练批次。  
5. 结束时断言：
   - 训练损失低于初始损失；
   - 测试精度达到阈值（`>= 0.93`）；
   - 参数全为有限值（非 NaN/Inf）。

## R11

误差与稳定性来源：

- 随机初始化与批次打乱会带来波动；
- 学习率过高会导致损失振荡；
- 小 batch 方差大，收敛轨迹更噪声化；
- 数据预处理（本例高斯平滑 + 归一化）可改善训练稳定性。

本实现通过固定随机种子、保守学习率、权重衰减来提升可复现性。

## R12

性能视角：

- 主计算在卷积层，复杂度对通道数和核大小最敏感；
- 8x8 输入 + 小网络使 CPU 也可快速完成训练；
- 使用 `DataLoader` 做 mini-batch，避免一次性大张量训练；
- 该 MVP 以“算法链路可读性”优先，不追求极限吞吐。

## R14

鲁棒性与常见失效模式：

- 学习率过大：损失不降甚至发散；
- epoch 太少：准确率不足；
- 权重衰减过强：欠拟合；
- 数据分布变化：测试精度可能明显下降；
- 过度平滑预处理：可能抹除细节，损伤分类边界。

## R18

源码级算法流程拆解（`demo.py` 主链路，8 步）：

1. 参数初始化：`main` 构建 `CNNConfig` 并设置随机种子，确定运行设备。  
2. 数据准备：`load_preprocessed_digits` 读取 `load_digits`，做归一化与 `gaussian_filter` 平滑，再分层切分训练/测试集。  
3. 张量装载：`make_dataloader` 把 `numpy` 数据转为 `[N,1,8,8]` 张量并构造 `DataLoader`。  
4. 模型建图：`TinyCNN` 定义 `Conv->ReLU->Pool->Conv->ReLU->Conv->ReLU->FC->ReLU->FC` 的前向路径。  
5. 初始评估：训练前调用 `evaluate` 计算 epoch 0 基线损失与精度。  
6. 迭代训练：每个 epoch 在 `train_one_epoch` 中显式执行 `zero_grad -> forward -> cross_entropy -> backward -> optimizer.step`。  
7. 轮末验证：再次调用 `evaluate` 得到训练/测试损失与准确率，并写入 `pandas` 日志表。  
8. 结果验收：执行断言（损失下降、精度达标、参数有限），输出混淆矩阵与分类报告。

说明：虽然优化器来自 PyTorch，但 CNN 的核心算法流（卷积前向、损失定义、反向传播触发、参数更新时序、评估与验收）在源码中可逐步追踪，不是“几行黑盒训练器”封装。
