## R01

区块排序（Block Sort）是一类“先分块、后合并”的排序思想：

- 先把数组切成多个固定大小的块；
- 在每个块内部完成局部有序；
- 再把这些有序块按稳定归并的方式逐层合成为全局有序。

本条目给出的 MVP 是工程上易读、可验证的版本：`块内插入排序 + 块间迭代归并`。它保留了 Block Sort 的核心结构（block/run + merge），不追求复杂的原地块旋转优化。

## R02

问题定义（离散序列排序）：

- 输入：可比较元素序列 `A`，以及块大小 `B`（`B >= 1`）。
- 输出：同一批元素的非降序排列结果。
- 额外目标：在存在重复键时保持稳定性（相同键的相对先后不变）。

本实现还提供记录型数据 `(value, original_index)` 的稳定性验证。

## R03

MVP 的算法结构：

1. 将数组按 `block_size` 划分为若干连续区间。  
2. 对每个区间做块内排序（本实现用插入排序）。  
3. 将“块大小”视作初始有序 run 长度，从 `run = block_size` 开始两两归并。  
4. 每轮归并后 `run *= 2`，直到 `run >= n`。

归并过程使用稳定比较 `<=`，因此全局结果稳定。

## R04

时间复杂度（`n` 为元素数，`B` 为块大小）：

- 块内排序：约 `O(n * B)`（每块插入排序 `O(B^2)`，共 `n/B` 块）；
- 块间归并：`O(n log(n/B))`（自底向上的归并层数）；
- 总体：`O(n * B + n log(n/B))`。

当 `B` 取常数或小值时，复杂度可视作接近 `O(n log n)`。

## R05

空间复杂度：

- 使用一个长度为 `n` 的临时缓冲区用于归并写回；
- 额外空间 `O(n)`；
- 其余为常数级索引变量。

这比高级原地 Block Merge Sort 占用更多内存，但实现显著简化。

## R06

微型示例：

给定数组 `[7, 3, 9, 1, 6, 2, 8, 4]`，设 `B=2`。

- 分块后：`[7,3] [9,1] [6,2] [8,4]`
- 块内排完：`[3,7] [1,9] [2,6] [4,8]`
- 第 1 轮归并（run=2）：`[1,3,7,9] [2,4,6,8]`
- 第 2 轮归并（run=4）：`[1,2,3,4,6,7,8,9]`

得到全局有序结果。

## R07

算法价值：

- 体现“局部有序 -> 全局有序”的经典分治路径；
- 适合讲解外部排序、分块处理、归并稳定性；
- 在工程上可替换块内策略（插入排序、快速排序、小型网络排序等）；
- 能自然扩展到流式/分段数据处理场景。

## R08

理论基础（工程化摘要）：

1. 每个块完成排序后可视作一个有序 run。  
2. 两个有序 run 可在线性时间内稳定归并为一个更长有序 run。  
3. 归并操作保持“元素多重集不变”和“有序性不变式”。  
4. 反复将 run 长度翻倍，最终覆盖整个数组，得到全局有序结果。  
5. 若归并比较使用 `<=`（左侧优先），则稳定性可逐层传递到最终结果。

## R09

适用条件与局限：

- 适合可比较元素、可接受 `O(n)` 额外缓冲区的场景；
- 适合强调稳定性和可解释性的实现；
- 若需要严格原地排序，需更复杂的块旋转/缓冲复用技巧；
- 对极小数组，直接插入排序可能更快；
- 对 Python 原生数值列表，内建 `sorted`（Timsort）通常更快。

## R10

正确性框架（本实现）：

1. 块内排序后，每个块内有序。  
2. 归并函数 `merge_ranges` 假设左右区间已排序，并构造排序后的并集。  
3. 写回后新区间仍有序，且元素数量守恒。  
4. 归并层层扩大覆盖范围，直到整个数组成为一个有序区间。  
5. 通过与 `sorted` 对照、以及稳定性断言完成终态验证。

## R11

稳定性与误差讨论：

- 排序算法本身是确定性的，不涉及数值迭代误差；
- 主要风险来自实现错误（边界、拷贝区间、比较符号）；
- 归并中使用 `<=`（而非 `<`）是稳定性的关键；
- demo 通过重复键样本检查“同键原始索引非降序”，直接验证稳定性。

## R12

性能视角：

- 主要成本在两部分：块内排序与多轮归并；
- 块大小是折中参数：
  - 过小：块数量多，归并轮数增加；
  - 过大：块内 `O(B^2)` 成本上升；
- demo 默认 `block_size=16`，在可读性和运行耗时之间取平衡；
- 与库排序相比，该实现优先“可追踪算法流程”，不是极致性能版本。

## R14

常见失效模式：

- 归并边界 `left/mid/right` 计算错误导致越界或漏元素；
- 临时缓冲写回区间错误导致结果污染；
- 比较符号写错（`<`）会破坏稳定性；
- `block_size <= 0` 导致非法分块；
- 忽略尾块（长度不足 `B`）会造成局部无序。

代码中通过配置校验与断言覆盖以上高频问题。

## R15

实现设计（`demo.py` 对应关系）：

- `BlockSortConfig`：集中管理规模、块大小、随机种子。  
- `insertion_sort_range`：完成块内局部排序。  
- `merge_ranges`：将两个有序区间稳定归并到临时缓冲并写回。  
- `block_sort`：先分块排序，再按 run 长度翻倍迭代归并。  
- `is_sorted_non_decreasing`：有序性检查。  
- `check_stability`：重复键稳定性验证。

## R16

相关算法链路：

## R18

源码级流程拆解（`block_sort` + `merge_ranges`，8 步）：

1. 初始化：在 `block_sort` 中读取 `n=len(arr)`，构建 `temp=[None]*n`，处理 `n<=1` 的直接返回。  
2. 分块循环：按 `start=0, B, 2B...` 划分区间 `[start, end)`，每块调用 `insertion_sort_range`。  
3. 形成初始 runs：完成第 2 步后，每个长度至多 `B` 的区间都已局部有序。  
4. 设置归并步长：令 `run=B`，进入 `while run < n` 的层循环。  
5. 层内扫描：按 `left=0, 2run, 4run...` 计算 `mid=min(left+run,n)`、`right=min(left+2*run,n)`。  
6. 稳定归并：若 `mid < right`，调用 `merge_ranges(arr, left, mid, right, temp)`，在比较时左侧 `<=` 优先，保证稳定性。  
7. 写回与扩大：归并结果写回 `arr[left:right]` 后，令 `run *= 2`，进入下一层更大区间合并。  
8. 终态校验：`main` 对结果执行有序性、与 `sorted` 一致性、重复键稳定性三类断言，全部通过后结束。

说明：本实现没有把第三方排序接口当黑箱调用；核心排序与归并逻辑可在源码中逐行追踪。
