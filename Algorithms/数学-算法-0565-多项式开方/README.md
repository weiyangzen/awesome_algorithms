# 多项式开方

## R01

多项式开方问题是：给定多项式 `A(x)`，寻找 `B(x)` 使得 `B(x)^2 = A(x)`。在代数计算中，这既可以作为“完全平方判定”，也可作为形式幂级数运算的基础模块（例如牛顿迭代、代数函数展开、生成函数处理）。

本条目给出一个最小可运行 MVP，重点是：
- 用有理数精确算术（`fractions.Fraction`）避免浮点误差。
- 用按系数递推法求 `sqrt(A)` 的前若干项。
- 通过回代平方 `B(x)^2` 判定 `A(x)` 是否为完全平方多项式。

## R02

问题定义（本实现）：
- 输入：`A(x)=a0+a1*x+a2*x^2+...`，以系数升幂数组表示（如 `[a0,a1,a2,...]`）。
- 输出：
  - `B(x)` 的候选开方结果（按形式幂级数递推得到）；
  - `B(x)^2`；
  - 布尔值 `is_perfect_square`，表示是否严格满足 `B(x)^2 == A(x)`。

求解域设定为 `Q`（有理数）及其形式幂级数环 `Q[[x]]`。

## R03

数学基础：
- 若 `A(x)=sum a_i x^i`，`B(x)=sum b_i x^i`，则 `B(x)^2=A(x)` 的系数关系为：
  - `b0^2 = a0`
  - `2*b0*bm + sum_{i=1}^{m-1}(b_i*b_{m-i}) = a_m`（`m>=1`）
- 因而可递推：
  - `bm = (a_m - sum_{i=1}^{m-1} b_i*b_{m-i}) / (2*b0)`
- 若最低非零项次数 `v=ord_x(A)` 为奇数，则在 `Q[[x]]` 中不存在平方根。
- 若 `v` 为偶数，可写成 `A(x)=x^v*C(x)`，其中 `sqrt(A)=x^(v/2)*sqrt(C)`。

## R04

算法总览：
1. 规范化输入系数并去除高位无效零项。
2. 处理零多项式和最低非零阶（valuation）。
3. 若 valuation 为奇数，直接判定无解。
4. 提取 `x^v` 因子后，把问题降为“常数项非零”的子问题。
5. 用按系数递推公式逐项求出 `sqrt` 的前 `t` 项。
6. 将前面的 `x^(v/2)` 移位补回。
7. 得到候选根 `B(x)` 后做一次精确平方 `B(x)^2`。
8. 比较 `B(x)^2` 与原多项式，给出完全平方判定结果。

## R05

核心数据结构：
- `list[Fraction]`：多项式系数容器（升幂顺序）。
- `trim`：删除尾部零项，保证表示唯一性。
- `valuation`：返回最低非零项次数，驱动可开方性判断。
- `poly_mul`：朴素卷积乘法，供回验阶段使用。

选择这些结构的原因是实现简单、可读性高、足够支撑 MVP 正确性验证。

## R06

正确性要点：
- 递推步骤的正确性来自系数逐项匹配：在已知 `b0...b_{m-1}` 时，`bm` 由唯一线性方程给出（前提 `b0 != 0`）。
- valuation 偶奇性判据保证了 `x` 次数结构一致性：平方多项式的最低非零次数必为偶数。
- 回验 `B^2 == A` 能过滤掉“形式上可递推但有限阶不闭合”的情况，得到严格的完全平方结论。

## R07

复杂度分析（`n` 为输入多项式长度）：
- 递推开方：第 `m` 项求和成本 `O(m)`，总计 `O(n^2)`。
- 朴素平方回验：`O(n^2)`。
- 空间复杂度：`O(n)`（存储系数与中间结果）。

MVP 的目标是正确性与可解释性，不追求 FFT 级最优复杂度。

## R08

边界与异常处理：
- 零多项式：平方根定义为零多项式。
- valuation 为奇数：抛出异常并说明“形式幂级数中不存在平方根”。
- 常数项不是有理完全平方：抛出异常（当前实现使用精确有理开方，不引入代数扩域）。
- `terms <= 0`：抛出参数异常。

## R09

MVP 取舍：
- 工具栈只用 Python 标准库（`fractions`、`math`），避免引入重依赖。
- 乘法使用朴素卷积，保持代码短小直观。
- 不做符号因式分解，不调用黑盒 CAS；核心算法全部在源码中可追踪。

## R10

`demo.py` 函数职责：
- `to_fraction_poly`：输入系数转 `Fraction`。
- `trim`：去尾零，规范化表示。
- `poly_mul`：多项式乘法（回验用）。
- `valuation`：计算最低非零次数。
- `sqrt_fraction_exact`：常数项有理平方根判定。
- `sqrt_series_nonzero_constant`：常数非零场景的递推开方核心。
- `poly_sqrt_series`：处理 valuation 和移位后的统一开方入口。
- `exact_polynomial_sqrt`：生成候选根并完成精确平方回验。
- `format_poly` / `run_case` / `main`：展示结果。

## R12

输出解读：
- `A(x)`：输入多项式。
- `候选 B(x)`：递推得到的开方候选。
- `B(x)^2`：候选根回代后的平方。
- `是否完全平方`：当且仅当 `B(x)^2` 与 `A(x)` 完全相等时为 `True`。
- 对无解样例，会输出具体失败原因（如 valuation 奇数）。

## R13

最小测试集建议：
- 完全平方：`(x^2 + 2x + 3)^2`。
- 扰动非平方：在完全平方基础上修改非零次项（常数项保持可开方）。
- 常数非平方：例如常数项改成 `10`，可快速判定无解。
- 偶数 valuation：`x^2*(1+x)^2`。
- 有理系数平方：`(x + 1/2)^2`。
- 奇数 valuation 无解：例如 `x + x^3`。

`demo.py` 已覆盖以上类型。

## R14

可调参数：
- `poly_sqrt_series(..., terms=t)` 的 `t` 可控制输出的幂级数截断阶数。
- `main` 中的 `cases` 可替换为任意测试多项式。

建议先用小阶样例验证，再扩展到更高阶输入。

## R15

与相关方法比较：
- 相比直接数值开方：本实现是符号/精确系数层面，避免浮点误判。
- 相比依赖 CAS 一步求解：本实现能清晰展示每个系数如何被递推出来。
- 相比高阶 FFT + 牛顿迭代：本实现更易读，适合作为教学与基线实现。

## R18

源码级算法流（对应 `demo.py`，8 步）：
1. `main` 构造固定样例，并逐个调用 `run_case`。  
2. `run_case` 将输入系数统一转为 `Fraction`，打印 `A(x)`。  
3. `exact_polynomial_sqrt` 根据输入阶数设定候选根长度，并调用 `poly_sqrt_series`。  
4. `poly_sqrt_series` 先计算 valuation：若奇数直接报错；若偶数则剥离 `x^v` 并把问题降到常数非零子问题。  
5. `sqrt_series_nonzero_constant` 先通过 `sqrt_fraction_exact` 求 `b0`，再按递推式逐项求 `b1,b2,...`。  
6. 子问题结果返回后，`poly_sqrt_series` 把 `x^(v/2)` 移位补回，得到完整候选根。  
7. `exact_polynomial_sqrt` 调用 `poly_mul` 计算 `B(x)^2`，与原始 `A(x)` 做精确比较。  
8. `run_case` 输出候选根、回验平方和最终布尔判定，形成可解释结果。
