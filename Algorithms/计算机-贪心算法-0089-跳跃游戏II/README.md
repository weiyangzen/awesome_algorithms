# 跳跃游戏II

- UID: `CS-0069`
- 学科: `计算机`
- 分类: `贪心算法`

- 目标目录: `Algorithms/计算机-贪心算法-0089-跳跃游戏II`

## R01

“跳跃游戏II”（Jump Game II）要求：给定非负整数数组 `nums`，`nums[i]` 表示从下标 `i` 最多可向前跳的步数，求从下标 `0` 到达最后一个下标所需的最少跳跃次数。

本题核心不是“能不能到达”（Jump Game I），而是“最少跳几次”，属于典型贪心分层问题。

## R02

形式化描述：

- 输入：一维非负整数数组 `nums`；
- 状态：当前位置 `i` 可跳到区间 `(i, i + nums[i]]`；
- 目标：最小化跳跃次数，使路径到达 `n-1`；
- 输出：最少跳跃次数（本目录 MVP 额外支持不可达时返回 `None`）。

约定：

- `n <= 1` 时返回 `0`（已经在终点）；
- 若无法到达终点，返回 `None`，便于工程场景明确识别失败样本。

## R03

贪心关键观察：

- 一次跳跃可以视为在当前“可达层”里任选起跳点；
- 在同一层内，不需要立刻决定具体落点，只需统计“下一层最远可达位置 `farthest`”；
- 当扫描指针到达当前层边界 `current_end`，就必须做一次跳跃，并把层边界推进到 `farthest`。

这相当于对隐式图做按层扩展（BFS 思想），但无需显式队列，能把复杂度降到 `O(n)`。

## R04

主算法（`min_jumps_greedy`）流程：

1. 若数组长度 `n <= 1`，直接返回 `0`；
2. 初始化：`jumps = 0`、`current_end = 0`、`farthest = 0`；
3. 遍历 `i = 0 .. n-2`（终点无需再起跳）；
4. 更新 `farthest = max(farthest, i + nums[i])`；
5. 若 `i == current_end`，说明当前层扫描完：
   - `jumps += 1`；
   - `current_end = farthest`；
   - 若 `current_end >= n-1` 可提前返回；
6. 若出现 `i > farthest`，表示断层不可达，返回 `None`。

## R05

复杂度分析：

- 贪心主解：
  - 时间复杂度 `O(n)`；
  - 空间复杂度 `O(1)`（不计输入）；
- DP 对照解（`min_jumps_dp`）：
  - 时间复杂度 `O(n^2)`；
  - 空间复杂度 `O(n)`；
- 穷举校验（`min_jumps_bruteforce`）只用于小规模真值验证，不用于生产计算。

## R06

MVP 验证策略：

- 固定样例覆盖：标准可达、含 `0` 可达、不可达陷阱、单元素、空数组；
- 双基线交叉验证：
  - `min_jumps_dp`（确定性参考解）；
  - `min_jumps_bruteforce`（小规模精确解）；
- `numpy` 随机生成 300 组用例，断言贪心与基线一致；
- 整个脚本无交互输入，`uv run python demo.py` 可直接运行。

## R07

优点：

- 线性时间，适合大规模数组；
- 代码短、实现清晰，便于面试与工程复用；
- 与 BFS 分层直觉一致，易解释正确性。

局限：

- 当前实现仅输出“最小跳数”，不输出具体最优路径；
- 不处理加权代价、概率跳跃或障碍成本等复杂约束；
- 对扩展模型通常需改用 DP、最短路或启发式搜索。

## R09

适用场景：

- 一维位置序列上的最少步数覆盖问题；
- 需要快速评估“最少跳数”而非具体跳法；
- 作为更复杂路径优化模型的 baseline。

不适用场景：

- 需要完整最优路径明细；
- 跳跃有动态代价、惩罚或多维状态约束；
- 需要在随机环境下做风险最优控制。

## R10

正确性直觉：

- 每次跳跃前，当前层内所有起点都等价于“同样跳数下可选位置”；
- 在该层内选择能让下一层覆盖最远的位置，永远不会比更短覆盖差；
- 层结束时才增加 `jumps`，保证跳数最小化；
- 若扫描点超过 `farthest`，说明不存在继续前进的合法边，必须判不可达。

`demo.py` 通过 DP 与穷举一致性断言，为该实现提供额外可信度。

## R11

边界与鲁棒性处理：

- `[]` 和单元素数组返回 `0`；
- 出现不可达断层时返回 `None`；
- 输入含负数会抛出 `ValueError`；
- `numpy` 桥接函数要求一维数组输入。

这些约束让失败模式显式化，减少静默错误。

## R12

实现细节：

- `_normalize_nums` 统一做类型与非负校验；
- `min_jumps_greedy` 维护三元状态：`jumps/current_end/farthest`；
- `min_jumps_dp` 用“到达每个位置的最小步数”做参考解；
- `min_jumps_bruteforce` 用记忆化 DFS 给小规模精确真值；
- `assert_consistency` 在同一入口统一断言三种结果。

这种结构在保持 MVP 简洁的同时，具备可回归性。

## R13

理论保证：

- 在本题模型（非负步长、每步代价统一为 1）下，贪心分层算法是精确最优解；
- 不是近似算法，不依赖概率假设；
- 对可达输入返回最小跳数，对不可达输入返回 `None`。

若问题改为“不同跳跃代价”或“带障碍惩罚”，该贪心保证不再成立。

## R14

常见错误：

1. 遍历到 `n-1` 仍继续计步，导致多算一次跳跃；
2. 忘记处理 `i > farthest` 的不可达断层；
3. 在层内提前计步（而不是在 `i == current_end` 时计步）；
4. 把 Jump Game I（可达性）误当成 Jump Game II（最小跳数）；
5. 忽略空数组和单元素数组边界。

## R18

`demo.py` 的源码级算法流程（8 步）：

1. `main` 依次执行固定样例、随机验证、`numpy` 示例，构成最小可复现闭环。  
2. `_normalize_nums` 把输入转换为非负整数列表，并在出现负值时立即报错。  
3. `min_jumps_greedy` 从左到右扫描，持续维护当前层边界 `current_end` 和下一层最远点 `farthest`。  
4. 每当扫描到 `current_end`，说明一层遍历结束，跳数 `jumps` 加一并推进边界到 `farthest`。  
5. 若在扫描中出现 `i > farthest`，判定不可达并返回 `None`；若边界覆盖终点则提前返回最小跳数。  
6. `min_jumps_dp` 用二维可达关系压缩成一维最小步数转移，提供确定性参考答案。  
7. `min_jumps_bruteforce` 对小规模输入做记忆化 DFS，给出精确真值并校验贪心/DP。  
8. `assert_consistency` 在固定与随机样本上统一断言三者一致，全部通过后打印成功信息。  

本实现没有调用任何第三方“最少跳数黑盒函数”；`numpy` 仅用于生成样本与数组桥接，核心求解过程完全在源码中可追踪。
