## R01

Viterbi 算法用于在离散时间序列模型中求“最可能隐状态路径”。
在给定模型参数与观测序列后，它把原本指数级路径枚举问题转化为动态规划问题：

- 目标：`argmax_{z_1:T} P(z_1:T, o_1:T)`
- 典型模型：离散 HMM（隐马尔可夫模型）、线性链 CRF 的解码环节
- 核心思想：保存“到每个时刻、每个状态的最优前缀分数”，并记录回溯指针

本条目提供可运行 MVP，强调“可审计、可验证”，不把解码过程留给黑盒库。

## R02

问题定义（离散 HMM 版）：

- 隐状态集合大小 `N`
- 观测符号集合大小 `M`
- 初始分布 `pi[i] = P(z_0=i)`
- 转移矩阵 `A[i,j] = P(z_t=j | z_{t-1}=i)`
- 发射矩阵 `B[i,k] = P(o_t=k | z_t=i)`
- 观测序列 `o_0, o_1, ..., o_{T-1}`

输出：

- 最优路径 `z*_0:T-1`
- 对应路径对数概率 `log P(z*, o)`

## R03

动态规划状态与转移：

- `delta[t, j]`：在时刻 `t` 到达状态 `j` 的最优路径对数概率
- `psi[t, j]`：达到 `delta[t, j]` 时的最优前驱状态下标

对数域递推（防止连乘下溢）：

- 初始化：
  `delta[0, j] = log(pi[j]) + log(B[j, o_0])`
- 转移：
  `delta[t, j] = max_i (delta[t-1, i] + log(A[i, j])) + log(B[j, o_t])`
- 指针：
  `psi[t, j] = argmax_i (delta[t-1, i] + log(A[i, j]))`

终止与回溯：

- `z*_{T-1} = argmax_j delta[T-1, j]`
- 从 `t=T-2` 到 `0` 用 `psi[t+1, z*_{t+1}]` 逆推得到整条路径

## R04

为什么它是动态规划：

- 最优子结构：到达 `(t,j)` 的最优路径，必然由某个 `(t-1,i)` 的最优路径延展而来。
- 无后效性：`delta[t,j]` 只依赖上一层 `delta[t-1,:]`，不需要保存完整历史路径。
- 指数到多项式：
  - 枚举全部路径复杂度 `O(N^T)`
  - Viterbi 复杂度 `O(T*N^2)`

## R05

复杂度分析：

- 时间复杂度：`O(T*N^2)`
  - 每个时刻 `t`，每个当前状态 `j`，需在 `N` 个前驱状态中取 `max`
- 空间复杂度：`O(T*N)`
  - 存 `delta` 和 `psi` 便于回溯完整路径

若只关心最优分数而不关心完整路径，可把 `delta` 压缩为两行（本 MVP 保留全表，便于解释与调试）。

## R06

直观例子：

- 隐状态：`Rainy / Sunny / Cloudy`
- 观测：`walk / shop / clean`
- 给定固定概率参数和观测序列，例如：`walk, shop, clean, clean, walk`

Viterbi 的作用是输出一条“整体最一致”的天气路径，而不是逐点独立猜测天气。
这与序列标注场景（词性标注、行为阶段识别）中的“上下文一致性”需求完全一致。

## R07

数值稳定性与实现约束：

- 连乘概率很快接近 0，必须在对数域计算。
- 对 `0` 概率使用 `log(max(p, eps))`，避免 `log(0)`。
- 输入合法性检查必须包含：
  - 维度匹配（`pi` 一维，`A/B` 二维）
  - 非负性
  - 概率归一化（`pi.sum=1`，`A/B` 每行和为 1）
  - 观测索引范围合法

## R08

与前向算法的区别：

- 前向算法：求总概率 `P(o)`，使用 `sum` 聚合路径。
- Viterbi：求最优路径，使用 `max` 聚合路径。

二者递推骨架相似，但语义不同：

- 前向回答“观测整体有多可能”
- Viterbi 回答“最可能是哪条解释路径”

## R09

适用与不适用：

适用：

- 隐状态是离散且数量可控
- 序列长度中等
- 需要可解释、可追踪的最优路径

不适用或需改造：

- 状态数量极大导致 `N^2` 转移成本过高
- 需要输出路径分布而非单一路径（需前向后向或采样）
- 模型假设不是一阶链式依赖

## R10

本目录 MVP 的验证策略：

- 主算法：`viterbi_decode`（动态规划 + 回溯）
- 参照算法：`exhaustive_decode`（穷举全部路径）
- 校验方式：在小规模序列上比较最优对数概率是否一致

这样可以直接验证“DP 实现是否等价于数学定义”，而不是只做表面跑通。

## R11

`demo.py` 的函数职责：

- `assert_hmm_params`：参数维度与概率合法性检查
- `safe_log`：对数域安全变换
- `viterbi_decode`：核心动态规划与回溯
- `path_log_prob`：给定路径时计算联合对数概率
- `exhaustive_decode`：穷举对照（小规模验证）
- `main`：固定案例演示 + 一致性断言 + 随机回归测试

## R13

输出解读：

- `best_path_ids / best_path_names`：最优路径（状态下标与名称）
- `best_log_prob`：最优路径联合对数概率
- `bruteforce_log_prob`：穷举法最优联合对数概率
- `score_match=True`：表示 DP 结果与数学定义对齐

注意：当多条路径并列最优时，不同实现可能给出不同但等价的路径；此时应以最优分数一致为主。

## R14

常见失效模式与防护：

- 失效：概率矩阵某行未归一化。
  - 防护：`assert_hmm_params` 强制行和检查。
- 失效：含零概率导致 `-inf` 传播。
  - 防护：`safe_log` 中使用 `eps` 截断。
- 失效：观测索引越界。
  - 防护：递推前检查 `obs` 是否落在 `[0, M)`。
- 失效：实现错误但样例太简单没暴露。
  - 防护：加入随机小规模“Viterbi vs 穷举”自动回归。

## R15

工程实践建议：

- 将“建模参数检查”和“推理逻辑”解耦，减少隐性 bug。

## R16

可扩展方向：

- n-best Viterbi：返回前 `k` 条最优路径而非 1 条
- Beam Search：近似解码以降低复杂度
- 半马尔可夫/持续时长建模：显式建模状态持续时间

## R17

本实现的边界约定：

- 空观测序列返回空路径与 `0.0` 对数概率
- 输入必须是离散观测索引；文本观测需先映射到整数 ID
- 默认在 CPU 上用 `numpy` 执行，无其他框架依赖

MVP 目标是“最小可运行 + 可审计 + 可验证”，不追求大而全训练框架。

## R18

`demo.py` 源码级算法流（8 步）：

1. `main` 定义状态名、观测词表、`pi/A/B` 和固定观测序列，并编码为整数观测 ID。  
2. `viterbi_decode` 先调用 `assert_hmm_params` 校验概率参数形状、非负性与归一化。  
3. `viterbi_decode` 使用 `safe_log` 将 `pi/A/B` 转为对数域，初始化 `delta[0,:]`。  
4. 对每个时刻 `t` 与状态 `j`，计算 `scores = delta[t-1,:] + logA[:,j]`，取 `argmax` 得前驱并写入 `psi[t,j]`。  
5. 同时更新 `delta[t,j] = max(scores) + logB[j, obs_t]`，完成整张 DP 表填充。  
6. 在末时刻对 `delta[T-1,:]` 取最大值得到终点状态，再沿 `psi` 逆序回溯得到最优路径。  
7. `main` 调用 `path_log_prob` 复算该路径分数，并调用 `exhaustive_decode` 穷举全部路径求全局最优分数。  
8. 对比 Viterbi 与穷举最优分数并断言一致，再进行多组随机小样本回归，形成可重复正确性闭环。
