# 外部排序

- UID: `CS-0238`
- 学科: `计算机`
- 分类: `数据库`
- 源序号: `392`
- 目标目录: `Algorithms/计算机-数据库-0392-外部排序`

## R01

外部排序（External Sorting）指在“数据总量大于可用内存”时的排序方法族。  
其核心原则是：把排序拆成可落地的顺序磁盘 I/O 流程，而不是假设全部记录一次装入内存。

## R02

数据库里的典型触发场景：
- `ORDER BY` 结果集溢出内存（spill to disk）；
- `DISTINCT` / `GROUP BY` 采用排序路径且输入过大；
- 批量索引构建前需要按键排序；
- 离线日志、明细账单、事件流归档前的全局排序。

## R03

问题抽象：
- 输入：`N` 条记录（示例中是 `(key, seq)` 文本行，`seq` 表示原始到达顺序）；
- 约束：内存最多容纳 `M` 条（`M << N`）；
- 输出：按 `key` 非降序，且同键时按 `seq` 非降序（稳定语义）的新文件。

## R04

本条目采用“外部排序中最常见的外部归并框架”：
1. Run 生成：按 `chunk_size` 分块读入内存，块内排序后落盘，形成多个有序 run；
2. 多轮归并：每轮最多并 `fan_in` 个 run，直到只剩 1 个最终 run；
3. 输出与清理：将最终 run 移动为目标输出，删除临时文件。

## R05

外部排序不等于单一算法，它是工程策略集合。常见成员包括：
- 简单分块 + 多路归并；
- 替换选择（Replacement Selection）生成更长初始 run；
- 多阶段归并计划（控制每轮归并路数与临时文件数量）。

本 MVP 选择第一种，优点是实现小、行为清晰、便于验证。

## R06

正确性关键不变量：
- 块内有序：每个 run 文件内部都按 `(key, seq)` 排好序；
- 归并最小性：堆顶始终是所有 run 当前候选中的最小记录；
- 输出前缀有序：每写出一个堆顶，输出文件的已写部分仍保持全局有序。

由此可归纳得到：当堆清空时，输出包含全部记录且全局有序。

## R07

时间复杂度（比较操作视角）：
- 设初始 run 数 `R = ceil(N / M)`；
- Run 生成阶段近似 `O(N log M)`；
- 每轮归并为 `O(N log fan_in)`，轮数 `P = ceil(log_{fan_in}(R))`；
- 总体近似：`O(N log M + P * N log fan_in)`。

## R08

I/O 复杂度（记录读写次数）近似：
- 生成 run：读 `N` + 写 `N`；
- 每轮归并：再读 `N` + 写 `N`；
- 总计约 `2N * (1 + P)` 条记录级 I/O。

工程上优化重点通常是减少 `P` 与提升顺序读写吞吐。

## R09

稳定性说明：
- 记录使用 `(key, seq)` 比较；
- 同一 `key` 的先后由 `seq` 决定；
- 块内排序与堆归并都按同一二元键比较。

因此该实现对“按输入顺序标注 `seq`”的数据是稳定的。

## R10

数据库执行器中，外部排序一般作为物理算子：
- 优化器先估算内存是否够用；
- 内存不足时转入外部排序路径并创建临时文件；
- 在执行监控中可观察 spill 字节数、归并轮数、临时文件占用。

## R11

关键资源与约束：
- `chunk_size` 决定单次内存峰值；
- `fan_in` 决定单轮打开文件数与归并轮数；
- 临时目录空间需至少同阶于输入规模；
- 文件句柄上限会限制可用 `fan_in`。

## R12

本目录 MVP 设计：
- 仅用 Python 标准库（`heapq`、`pathlib`、`shutil`、`random`）；
- 输入输出均为文本文件，便于直接检查；
- 自动生成测试数据，不依赖交互输入；
- 提供正确性校验：全局有序 + 稳定性检查。

## R13

数据格式：
- 一行一条记录，格式 `key,seq`；
- `key` 可重复，可正可负；
- `seq` 为输入顺序编号（从 0 开始）。

示例：
```text
12,0
-5,1
12,2
```

## R14

参数建议：
- `chunk_size`：太小会产生大量 run；太大可能逼近内存上限；
- `fan_in`：太小会增加归并轮数；太大会增加单轮句柄/缓冲压力；
- 实践中可从 `chunk_size=1e4~1e6`、`fan_in=8~64` 试配（取决于机器与文件系统）。

## R15

运行方式（仓库根目录）：

```bash
uv run python Algorithms/计算机-数据库-0392-外部排序/demo.py
```

脚本会自动在本目录下创建 `mvp_data`，生成输入、执行外部排序、打印统计。

## R16

输出字段说明：
- `records_in / records_out`：输入和输出记录数（应一致）；
- `chunk_size / fan_in`：本次资源配置；
- `initial_runs`：第一阶段生成的 run 数；
- `merge_passes`：多轮归并轮数；
- `is_sorted_stable`：是否通过“有序 + 稳定”校验；
- `head_keys / tail_keys`：结果文件头尾样本键值。

## R17

可扩展方向：
- 替换选择生成更长 run，减少初始 run 数；
- 二进制块读写与页缓存，降低文本解析开销；
- 并行归并与异步 I/O；
- 记录格式扩展到 `(key, payload)` 并支持多字段排序规则。

## R18

`demo.py` 源码级流程（8 步）：
1. `generate_input_file` 生成 `(key,seq)` 输入文件，构造可复现实验数据。  
2. `external_sort` 校验参数并准备临时工作目录。  
3. `split_into_sorted_runs` 顺序读取输入，每攒满 `chunk_size` 条记录就调用 `_flush_chunk_to_run`。  
4. `_flush_chunk_to_run` 按 `(key, seq)` 排序并写出 `run_XXXX.txt`。  
5. `multi_pass_merge` 以 `fan_in` 为分组上限，逐轮组织 run 归并。  
6. `merge_run_group` 对每组 run 建最小堆，循环弹出最小记录写入新 run，并补充来源 run 的下一条记录。  
7. 当只剩一个 run 时移动为最终输出文件，并清理临时目录。  
8. `is_sorted_and_stable` 与 `sample_head_tail_keys` 验证结果并输出统计指标。
