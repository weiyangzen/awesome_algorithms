## R01

鲁棒主成分分析（Robust PCA, RPCA）要解决的问题是：当数据矩阵同时包含“低秩结构”与“稀疏异常值”时，如何把它们分离出来。

经典建模（Principal Component Pursuit, PCP）为：

- 观测矩阵：`M = L + S`
- `L`：低秩部分（主结构）
- `S`：稀疏部分（离群点、遮挡、脉冲噪声）

优化目标：

`min ||L||_* + λ||S||_1,  s.t. M = L + S`

其中 `||L||_*` 是核范数（奇异值和），`||S||_1` 是元素级 `L1` 范数。

## R02

历史与定位（简版）：

- 传统 PCA 假设噪声接近高斯且不含大幅离群，面对稀疏但幅值很大的异常值会明显失真。
- Candes、Li、Ma、Wright 等工作系统化提出 PCP/RPCA，使“低秩 + 稀疏”分解可由凸优化稳定求解。
- 在视频前景-背景分离、人脸遮挡恢复、异常检测等任务中，RPCA 已成为基础方法。

## R03

本条目 MVP 的目标：

- 输入：二维矩阵 `M`（无交互，脚本内自动生成可复现测试样本）。
- 输出：
- 低秩矩阵 `L_hat`
- 稀疏矩阵 `S_hat`
- 收敛状态、迭代次数、相对残差
- 估计秩与估计稀疏度
- 对比指标：低秩恢复误差、稀疏恢复误差、重构误差、稀疏支撑集精确率/召回率

## R04

时间复杂度（以 `m x n` 矩阵为例，`p=min(m,n)`）：

- IALM 每轮的主耗时来自一次 SVD（奇异值阈值化），通常为 `O(mnp)`。
- 软阈值和残差更新是逐元素或矩阵加减，约 `O(mn)`。
- 若迭代轮数为 `k`，总复杂度近似 `O(k * mnp)`。

因此，SVD 是绝对主导成本。

## R05

空间复杂度：

- 需要同时持有 `M, L, S, Y` 等若干 `m x n` 矩阵，量级 `O(mn)`。
- SVD 还需要 `U, s, V^T`，额外约 `O(mn + p)`。
- 总体仍可视为 `O(mn)` 级别（常数项较大）。

## R06

微型直观示例：

设

`M = [[1, 0, 0], [0, 1, 0], [0, 0, 20]]`

可理解为：

- 前两维是较平稳的“低秩结构”趋势（近似单位结构）
- 右下角 `20` 是一个显著离群脉冲

RPCA 的目标不是把 `M` 整体直接做 PCA，而是倾向于分解为：

- `L`：解释主体结构
- `S`：吸收这种稀疏大幅异常

这样得到的主成分更不受单点离群干扰。

## R07

算法意义：

- 对“少量坏点污染整体数据”的场景特别有效。
- 保留 PCA 的结构提取能力，同时显式建模离群项。
- 在工程上可解释性更强：`L` 表示主结构，`S` 直接定位异常位置与幅值。

## R08

核心数学关系：

1. 目标问题：`min ||L||_* + λ||S||_1, s.t. M=L+S`。
2. 引入拉格朗日乘子 `Y` 与罚参数 `μ`，构造增广拉格朗日。
3. 固定其他变量时，`L` 子问题对应奇异值阈值化（SVT）。
4. `S` 子问题对应元素级软阈值化（Soft Thresholding）。
5. 乘子更新：`Y <- Y + μ(M-L-S)`。
6. 停止准则：`||M-L-S||_F / ||M||_F < tol`。

## R09

适用条件与局限：

适用：

- 数据可近似表示为“低秩 + 稀疏异常”。
- 异常点数量占比不高，且位置较分散。

局限：

- 当异常不稀疏（接近稠密噪声）时，RPCA 假设被破坏。
- 大规模矩阵上全 SVD 成本高。
- 恢复质量依赖 `λ`、数据尺度与低秩/稀疏分离程度。

## R10

正确性要点（工程化说明）：

- PCP 是凸优化问题，目标函数与约束均为凸集合上的凸形式。
- IALM 通过交替求解两个可闭式阈值子问题，逐步逼近 KKT 条件。
- 当相对残差 `||M-L-S||_F/||M||_F` 足够小，可视作满足约束并接近最优解。
- 本 MVP 额外用“恢复误差 + 支撑集 F1 + 重构误差”多指标联合验证实现正确性。

## R11

数值稳定性与收敛实践：

- 数据预缩放有助于稳定 `μ` 与阈值尺度；本实现使用默认 `λ=1/sqrt(max(m,n))`。
- `μ` 采用逐轮放大（`rho>1`）并设置上界 `mu_bar`，避免过快或过慢更新。
- 终止准则采用相对残差，兼顾不同数据幅值。
- 若出现振荡或收敛慢，可调 `rho`、`tol`、`max_iter`。

## R12

性能视角：

- 主要热点在 `np.linalg.svd`。
- 软阈值、残差、乘子更新均是 NumPy 向量化操作，开销相对较小。
- 若迁移到更大规模任务，常见优化方向：
- 截断 SVD / 随机 SVD
- 稀疏存储
- GPU 线性代数后端

## R13

本目录 MVP 的可验证保证：

- 固定随机种子生成低秩真值 `L_true` 与稀疏真值 `S_true`。
- 运行后自动断言：
- `converged == True`
- `relative_residual <= 1e-6`
- 低秩与稀疏相对误差均不超过 `0.20`
- 重构误差 `<= 1e-6`
- 稀疏支撑集 `F1 >= 0.95`
- 任一指标失败会抛异常，便于自动化验证。

## R14

常见失效模式：

- 输入不是二维矩阵，或含 `NaN/Inf`。
- 异常比例过高（不再稀疏），导致 `S` 与 `L` 难以分离。
- `λ` 取值不合适：
- 过大时异常被过度压缩到 `L`
- 过小时噪声被过度吸收到 `S`
- 迭代上限过小导致未收敛。

## R15

`demo.py` 结构设计：

- `RPCAResult`：集中返回分解结果与诊断字段。
- `soft_threshold`：实现 `L1` 子问题近端算子。
- `svd_threshold`：实现核范数子问题近端算子（SVT）。
- `validate_inputs`：输入与超参数合法性检查。
- `robust_pca_ialm`：IALM 主循环。
- `build_synthetic_rpca_case`：构造可复现的低秩+稀疏样本。
- `support_metrics`：计算稀疏支撑集 precision/recall/F1。
- `run_checks`：统一断言质量门槛。
- `main`：一键运行、打印报告、无交互退出。

## R16

相关算法与方法链路：

- 标准 PCA：只建模低秩，不显式建模稀疏离群。
- RPCA / PCP：低秩 + 稀疏分解（本条目）。
- 矩阵补全（Matrix Completion）：关注缺失值恢复，不同于异常值分解。
- GoDec / OR-PCA：更偏工程速度的近似或在线变体。

## R18

`demo.py` 的源码级算法流程（9 步，非黑盒）如下：

1. `main` 固定随机种子，调用 `build_synthetic_rpca_case` 生成 `M=L_true+S_true`。  
2. `robust_pca_ialm` 根据矩阵尺寸设置 `λ=1/sqrt(max(m,n))`（或使用外部传入值）。  
3. 计算 `||M||_2` 与 `||M||_inf` 初始化对偶变量 `Y`，并设置 `μ`、`mu_bar`。  
4. 每轮先做 `L`-步：对 `M-S+Y/μ` 执行 SVD，再对奇异值做阈值化（SVT）得到 `L`。  
5. 再做 `S`-步：对 `M-L+Y/μ` 执行元素级软阈值，得到稀疏矩阵 `S`。  
6. 计算残差 `R=M-L-S` 与相对残差 `||R||_F/||M||_F` 作为收敛判据。  
7. 更新拉格朗日乘子 `Y <- Y + μR`，并按 `μ <- min(ρμ, mu_bar)` 调整罚参数。  
8. 达到阈值后返回 `L,S` 及迭代诊断；随后 `run_checks` 计算恢复误差与支撑集 F1。  
9. 若所有断言通过，打印 `All checks passed.`，说明 MVP 在该可复现实验上通过。  

说明：第三方库仅用于基础线性代数（SVD、范数、向量化）；低秩/稀疏分解流程由源码显式实现。
