## R01

归并排序（Merge Sort）是一种基于“分治法（Divide and Conquer）”的经典稳定排序算法，核心思想是：

- 把原数组递归地拆分为更小的子数组；
- 当子数组长度降到 1 时天然有序；
- 再把两个有序子数组线性归并成一个更大的有序数组。

它的上界时间复杂度稳定为 `O(n log n)`，因此在需要可预测性能时非常常用。

## R02

问题定义（升序）：

- 输入：长度为 `n` 的可比较序列 `A = [a1, a2, ..., an]`；
- 输出：升序序列 `B`，满足：
  - `B` 是 `A` 的重排；
  - `B[0] <= B[1] <= ... <= B[n-1]`。

本条目 MVP 目标：

- 给出“源码级可追踪”的归并排序实现（非第三方排序黑箱）；
- 自动验证结果与 `sorted()` 一致；
- 验证重复键情况下的稳定性。

## R03

归并排序在排序算法体系中的价值：

- 与数据初始分布关系较弱，性能上界稳定；
- 稳定排序，适合“主键+次键”多阶段排序流程；
- 结构清晰，易于并行化与外部排序扩展；
- 是理解分治递归与“线性归并”成本模型的典型教材算法。

## R04

核心机制（自顶向下版本）：

1. `sort(lo, hi)` 负责排序半开区间 `[lo, hi)`；
2. 当区间长度 `<= 1` 时直接返回；
3. 计算中点 `mid = (lo + hi) // 2`；
4. 递归排序左半区间 `[lo, mid)` 与右半区间 `[mid, hi)`；
5. 调用 `merge(lo, mid, hi)` 把两个有序段合并。

稳定性的关键在归并比较：

- 当左右当前元素“相等”时优先取左侧（`<=`），即可保持原相对顺序。

## R05

高层流程：

1. 将输入复制到工作数组 `arr`，并准备辅助数组 `aux`；
2. 递归拆分区间直至叶子区间（长度 1）；
3. 自底向上回溯执行归并；
4. 每次归并先写入 `aux`，再回写到 `arr`；
5. 返回最终有序结果与统计信息（比较次数、写入次数、归并调用次数、递归深度）。

## R06

正确性要点：

- 归纳基：长度 0/1 区间天然有序；
- 归纳假设：左右子区间经过递归后均有序；
- 归并阶段每次从两端最小候选里选更小者写入，因此新序列保持全局非降序；
- 当某一侧耗尽时，另一侧剩余部分整体追加，仍保持有序。

因此可归纳得到整个区间最终有序；配合“相等取左”策略可证明算法稳定。

## R07

复杂度分析：

- 时间复杂度：
  - 递推式 `T(n) = 2T(n/2) + O(n)`；
  - 解为 `O(n log n)`（最好/平均/最坏一致量级）。
- 空间复杂度：
  - 需要一个与输入同规模的辅助数组，额外空间 `O(n)`；
  - 递归栈深度 `O(log n)`。

## R08

边界与异常策略：

- 空数组：直接返回空数组；
- 单元素：直接返回；
- 含负数、重复值、浮点数：按比较规则正常处理；
- 不可比较混合类型：Python 会抛出运行期异常，MVP 保持显式失败（不吞错）。

## R15

与常见排序算法对比：

- 对比插入/冒泡/选择：归并排序在大规模下通常更稳健（`O(n log n)` 对 `O(n^2)`）；
- 对比快速排序：归并排序最坏时间上界更稳，但需要额外 `O(n)` 空间；
- 对比堆排序：两者都可达 `O(n log n)`，归并排序稳定而堆排序通常不稳定；
- 实务中归并排序常用于“稳定性优先”或“外部排序”场景。

## R16

典型应用：

- 多关键字排序中的稳定阶段（先次键后主键）；
- 链表排序（可利用链表结构低成本归并）；
- 外部排序（磁盘分块有序后多路归并）；
- 需要可预测上界的离线批处理排序任务。

## R17

可扩展方向：

- 自底向上（迭代）归并，减少递归调用开销；
- 小区间切换到插入排序，优化常数；
- 并行归并与多线程/多进程分治；
- 外部排序中的多路归并与缓冲策略优化；
- 对超大数组使用内存映射与分块流水线。

## R18

`demo.py` 源码级流程拆解（8 步）：

1. `main` 构造固定样例和 `numpy` 随机样例，并逐个调用 `run_case`。  
2. `run_case` 调用 `merge_sort(data)`，同时准备 `sorted(data)` 作为结果校验基准。  
3. `merge_sort` 初始化工作数组 `arr`、辅助数组 `aux` 与 `MergeSortStats`。  
4. 内部 `sort(lo, hi, depth)` 递归拆分区间：长度 `<=1` 直接返回，否则继续拆到左右子区间。  
5. 两侧递归返回后，`merge(lo, mid, hi)` 同时扫描左右有序段并执行比较计数。  
6. 每次比较后把较小元素写入 `aux`，若键相等优先取左侧（`<=`）保证稳定性。  
7. 当任一侧耗尽后，把另一侧剩余元素追加到 `aux`，再整体回写到 `arr[lo:hi]`。  
8. 全部归并完成后返回排序结果与统计；`main` 汇总样例通过状态和稳定性检查结果。

说明：排序主体完全由源码实现，第三方库仅用于随机样例生成，不构成算法黑箱。
