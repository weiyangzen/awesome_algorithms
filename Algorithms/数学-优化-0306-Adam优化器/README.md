## R01

Adam（Adaptive Moment Estimation）是一类一阶随机优化方法，用于迭代更新参数 `theta` 以最小化目标函数 `f(theta)`。其核心思想是同时维护梯度一阶矩估计（动量）和二阶矩估计（平方梯度的指数滑动平均），并做偏差修正后再更新参数。

标准更新写法：
- `m_t = beta1 * m_{t-1} + (1-beta1) * g_t`
- `v_t = beta2 * v_{t-1} + (1-beta2) * (g_t^2)`
- `m_hat_t = m_t / (1-beta1^t)`
- `v_hat_t = v_t / (1-beta2^t)`
- `theta_t = theta_{t-1} - alpha * m_hat_t / (sqrt(v_hat_t) + eps)`

其中 `g_t` 是当前梯度，`alpha` 是学习率。

## R02

Adam 由 Diederik P. Kingma 与 Jimmy Ba 在 2014 年提出（论文标题 *Adam: A Method for Stochastic Optimization*）。它融合了：
- Momentum 的一阶动量加速；
- RMSProp 的自适应步长缩放；
- 偏差修正以缓解初期动量估计偏小问题。

该方法在深度学习与一般随机优化任务中非常常见。

## R03

Adam 面向的问题是：在高维、噪声较大的优化场景下，固定学习率 SGD 往往收敛慢且对尺度敏感。

Adam 的改进点：
1. 利用 `m_t` 平滑随机梯度，降低抖动。
2. 利用 `v_t` 按坐标自适应缩放步长，缓解特征尺度不一致。
3. 用偏差修正改善前几步更新质量，避免初期更新过小。

## R04

算法流程（每个迭代步）可概括为：
1. 计算梯度 `g_t = ∇f(theta_{t-1})`。
2. 更新一阶矩 `m_t`。
3. 更新二阶矩 `v_t`。
4. 计算偏差修正 `m_hat_t, v_hat_t`。
5. 用归一化方向 `m_hat_t / (sqrt(v_hat_t)+eps)` 更新参数。

典型超参数：
- `alpha = 1e-3`
- `beta1 = 0.9`
- `beta2 = 0.999`
- `eps = 1e-8`

## R05

设参数维度为 `d`。
- 单次迭代时间复杂度：`O(d)`（逐坐标向量运算）。
- 额外空间复杂度：`O(d)`（存 `m` 与 `v` 两个状态向量）。
- 总训练复杂度取决于迭代次数 `T`，约 `O(T*d)`（不计梯度计算之外模型前向/反向成本）。

## R06

一个简化数值例子（单维）：
- 初始 `theta0=1.0, m0=0, v0=0`
- 梯度 `g1=0.2`
- 取 `alpha=0.1, beta1=0.9, beta2=0.999, eps=1e-8`

则：
- `m1 = 0.02`
- `v1 = 0.00004`
- `m_hat1 = 0.2`
- `v_hat1 = 0.04`
- 更新量 `delta = 0.1 * 0.2 / (sqrt(0.04)+eps) ≈ 0.1`
- `theta1 ≈ 0.9`

可见偏差修正后，第一步规模合理，不会被初始零向量状态严重压小。

## R07

Adam 的主要优点：
- 对稀疏梯度和噪声梯度鲁棒性较好。
- 逐坐标自适应步长，减少手工调参难度。
- 早期收敛速度通常快于普通 SGD。

常见局限：
- 泛化性能在部分任务上可能不如精细调参的 SGD+Momentum。
- 对学习率仍敏感，过大时会震荡或发散。

## R09

适用边界：
- 适用：可求梯度的一阶优化问题，尤其是高维、噪声或稀疏梯度任务。
- 不适用或需谨慎：
  - 非光滑且梯度信息极不稳定的问题；
  - 需要严格二阶信息的高精度优化阶段；
  - 极端小数据任务中，过快自适应可能带来过拟合风险。

## R10

正确性（实现层面）关键检查点：
1. `m`、`v` 与参数同维度。
2. 每步先更新 `m/v`，再做偏差修正。
3. 偏差修正指数 `t` 必须从 1 递增。
4. 分母使用 `sqrt(v_hat) + eps`，避免除零。
5. 参数更新方向为负梯度方向（最小化）。

若上述顺序错误，通常会导致收敛显著变差。

## R11

数值稳定性考虑：
- `eps` 防止 `v_hat` 很小时分母接近零。
- 对逻辑回归损失实现中使用 `log(p+eps)` 防止 `log(0)`。
- `sigmoid` 输入做裁剪（本实现裁剪到 `[-50, 50]`）防止指数溢出。
- 训练中可监控 loss 是否下降，作为异常检测信号。

## R12

真实成本与调参经验：
- `alpha` 决定收敛速度与稳定性，通常先调它。
- `beta1` 越大动量越平滑，但响应变化更慢。
- `beta2` 越大二阶矩更平滑，但早期可能过保守。
- `eps` 一般保持很小，主要做数值保护。

工程上常从 `alpha=1e-3`（深度学习）或更高（小模型）开始网格试验。

## R14

常见失效模式与防护：
- 失效：学习率过大导致 loss 震荡。
  - 防护：降低 `alpha`，或添加学习率衰减。
- 失效：`beta2` 过高导致步长过小、训练停滞。
  - 防护：适当下调 `beta2`。
- 失效：实现遗漏偏差修正，初期收敛异常慢。
  - 防护：单元测试对比首步更新量。
- 失效：梯度实现错误。
  - 防护：有限差分梯度检查（本 MVP 已包含）。

## R16

前后关系与常见变体：
- 前驱：SGD、SGD+Momentum、AdaGrad、RMSProp。
- 同类或后继：AdamW（解耦权重衰减）、Nadam、AMSGrad。
- 实践倾向：
  - Adam 常用于快速收敛阶段；
  - 某些任务后期切换 SGD 以追求泛化。

## R18

源码级算法流程拆解（对应 `demo.py`，8 步）：
1. `make_synthetic_binary_dataset` 生成两类高斯样本，并标准化特征后拼接偏置列。 
2. `split_train_test` 固定随机种子切分训练/测试集，确保可复现。 
3. `logistic_loss_and_grad` 计算逻辑回归 BCE 损失及其解析梯度，并加入可选 L2 正则。 
4. `finite_diff_gradient_check` 在初始点做有限差分，对比解析梯度，先验证导数实现正确。 
5. `train_logistic_with_adam` 初始化参数 `w` 与状态 `(m, v, t)`，进入迭代循环。 
6. 每轮先用当前 `w` 得到 `loss, grad`，再在 `adam_step` 内依次更新 `m`、`v`、偏差修正 `m_hat/v_hat`。 
7. `adam_step` 按 `w <- w - lr * m_hat / (sqrt(v_hat)+eps)` 完成参数更新并回传新状态。 
8. 训练结束后在 `main` 中计算训练/测试准确率、打印 loss 变化并做阈值断言，确认优化有效。 

本实现未依赖第三方黑盒 Adam；每个状态变量与更新公式都在源码中显式可追踪。
