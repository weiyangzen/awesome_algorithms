# 持久同调算法

- UID: `MATH-0242`
- 学科: `数学`
- 分类: `拓扑数据分析`
- 源序号: `242`
- 目标目录: `Algorithms/数学-拓扑数据分析-0242-持久同调算法`

## R01

问题定义（MVP 版本）：
给定点云 \(X=\{x_i\}_{i=1}^n \subset \mathbb{R}^d\)，构造其 Vietoris-Rips 过滤并计算持久同调区间，重点输出 \(H_0\) 与 \(H_1\)：
\[
\mathrm{VR}_\epsilon(X)=\{\sigma\subseteq X \mid \forall u,v\in \sigma,\ \|u-v\|_2\le \epsilon\}
\]
随着 \(\epsilon\) 增大，拓扑特征（连通分量、环）会出生与死亡。持久同调的输出是区间集合：
\[
(b, d),\quad b=\text{birth},\ d=\text{death}
\]
其中 \(d=\infty\) 表示在当前过滤范围内未被杀死。

## R02

数学背景与核心对象：
- 单纯复形：由顶点、边、三角形等闭合于取面的组合对象。
- 过滤（filtration）：一列嵌套复形
  \[
  K_0 \subseteq K_1 \subseteq \cdots \subseteq K_t
  \]
  可由阈值 \(\epsilon\) 驱动构造。
- 同调群 \(H_k\)：度量 \(k\) 维“洞”的代数对象。
- 持久同调：追踪同调类在过滤中的出生和死亡，常用持久度
  \[
  \mathrm{pers}=d-b
  \]
  区分“主结构”和“噪声结构”。

## R03

本实现的输入与输出：
- 输入（`demo.py` 内置）：
  - 使用 `sklearn.datasets.make_circles` 生成二维双环点云；
  - 过滤最大边阈值 `max_edge_length=1.6`；
  - 最高单形维度 `max_dim=2`（足够计算 \(H_0,H_1\)）。
- 输出：
  - VR 复形规模统计（边数、三角形数、总单形数）；
  - \(H_0/H_1\) 主要持久区间表（`birth/death/persistence`）；
  - 若干尺度下的 \(\beta_0,\beta_1\) 探针值。

## R04

算法总体思路：
1. 计算点对距离矩阵 \(D\)；
2. 用阈值构造 VR 复形（0/1/2-单形）并赋予过滤值；
3. 按 `(filtration, dim, vertices)` 排序，得到全局过滤顺序；
4. 构建边界矩阵 \(\partial\)（系数域取 \(\mathbb{Z}_2\)）；
5. 对矩阵做列约化，获得 birth-death 配对；
6. 提取持久区间并汇总。

## R05

过滤值定义（源码一致）：
- 顶点 \(v\)：\(f(v)=0\)
- 边 \((i,j)\)：\(f(i,j)=\|x_i-x_j\|_2\)
- 三角形 \((i,j,k)\)：
  \[
  f(i,j,k)=\max\{f(i,j),f(i,k),f(j,k)\}
  \]
这保证了“面先于体”的过滤一致性。即使出现同过滤值，也通过排序中的 `dim` 次关键字强制先处理低维面。

## R06

边界算子与系数域：
- 在 \(\mathbb{Z}_2\) 上，符号问题消失，边界是“面索引集合”的异或关系。
- 对 1-单形 \((i,j)\)：\(\partial(i,j)=\{(i),(j)\}\)
- 对 2-单形 \((i,j,k)\)：\(\partial(i,j,k)=\{(i,j),(i,k),(j,k)\}\)
- 代码中用 `set[int]` 存每列非零行，列加法用对称差 `^=` 实现。

## R07

正确性要点（MVP 级）：
- 过滤顺序正确：任意单形的所有面都不会晚于该单形出现。
- 列约化不变量：每个非零列有唯一 `low`，对应“该列杀死的出生单形”。
- 配对语义：
  - `pivot(low)=j` 表示 index=`low` 的出生类在 index=`j` 死亡；
  - 未被配对且约化后为空列的单形对应无限区间。
- 对 \(H_0/H_1\) 而言，只需构造到 2-单形即可得到完整配对信息。

## R08

复杂度分析：
- 距离矩阵：时间 \(O(n^2)\)，空间 \(O(n^2)\)。
- 边枚举：\(O(n^2)\)。
- 三角形枚举（朴素）：\(O(n^3)\)。
- 边界矩阵列约化（稀疏集合实现）：
  - 最坏可较高（与列冲突程度相关），
  - 在本 MVP 的点数规模（几十点）可在秒级内完成。
- 因此总体瓶颈通常在高阶单形数量增长（组合爆炸）。

## R09

边界条件与鲁棒性：
- `max_edge_length` 过小：图过稀，可能只有很多 \(H_0\) 出生、少量或无 \(H_1\)。
- `max_edge_length` 过大：单形数激增，计算变慢。
- 当无某维区间时，输出空表而非报错。
- 过滤值相等时，通过排序 tie-break（维度、顶点字典序）保证稳定可重复。

## R10

工程实现结构（`demo.py`）：
- `Simplex`：单形结构（顶点、维度、过滤值）。
- `Interval`：持久区间结构（维度、出生、死亡、持久度）。
- `build_vr_complex`：构建 0/1/2-单形过滤。

- `intervals_to_dataframe`：结果表格化（`pandas`）。
- `betti_at_scale`：在给定尺度估计 \(\beta_k\)。

## R13

输出解读建议：
- \(H_0\)：
  - 一般会有 1 条无限区间（最终连通分量）；
  - 其余有限区间对应“被并入主连通分量”的组件。
- \(H_1\)：
  - 持久度较大的区间通常对应主要环结构；
  - 短区间常可视作采样噪声或局部几何伪环。
- `Betti` 探针用于观察不同 \(\epsilon\) 下的拓扑状态变化。

## R14

MVP 范围声明：
- 仅实现 Vietoris-Rips 过滤，不含 Čech / Alpha complex。
- 仅构造到 2-单形，目标是稳定支持 \(H_0/H_1\)。
- 仅做 CPU 稀疏集合列约化，不做并行优化与块矩阵优化。
- 不输出图形条形码/持久图（可在后续版本补充）。

## R16

常见错误与规避：
- 错误 1：按“先维度后过滤值”排序，会破坏过滤语义。
- 错误 2：未保证面单形存在就构造体单形，导致边界矩阵不闭合。
- 错误 3：约化时使用“原列”而非“已约化列”做消元，结果错误。
- 错误 4：把无限区间误当成异常值过滤掉，丢失关键拓扑信息。
- 错误 5：阈值选太大导致组合爆炸，应先做小规模试跑。

## R18

`demo.py` 源码级流程拆解（8 步）：
1. `make_point_cloud`：用 `make_circles` 生成示例点云，并用 `StandardScaler` 标准化。
2. `pairwise_distance`：通过 `scipy.spatial.distance` 构造全点对欧氏距离矩阵。
3. `torch_distance_error`：用 `torch.cdist` 复算距离并输出与 scipy 的最大绝对误差（数值自检）。
4. `build_vr_complex`：按边阈值生成顶点、边、三角形，并计算各自过滤值。
5. `build_boundary_columns`：把每个单形映射为边界行索引集合，形成 Z2 稀疏列。
6. `reduce_boundary_matrix_z2`：逐列做 low-pivot 消元，得到唯一 pivot 配对关系。
7. `extract_intervals`：将 `birth_idx -> death_idx` 转为有限区间，并补上无限区间。
8. `intervals_to_dataframe` 与 `betti_at_scale`：按维度汇总主要区间并打印多尺度 Betti 探针。
