## R01

欧拉路径（Euler trail）是“每条边恰好经过一次”的路径；欧拉回路（Euler circuit）是起点和终点相同的欧拉路径。

本算法条目覆盖无向图与有向图两个版本的存在性判定与构造：

- 无向图欧拉回路：非零度顶点连通且所有顶点度数为偶数。
- 无向图欧拉路径（非回路）：非零度顶点连通且恰有 2 个奇度顶点。
- 有向图欧拉回路：所有非零度顶点同属一个弱连通分量，且每点入度=出度。
- 有向图欧拉路径（非回路）：恰有一个点 `out-in=1`、一个点 `in-out=1`，其余点入度=出度，且满足弱连通条件。

## R02

该问题的历史起点通常追溯到 1736 年欧拉对柯尼斯堡七桥问题的分析。其核心贡献是把“能否一笔画完”抽象为图上的结构判定问题，从而形成图算法中的“度数约束 + 连通性约束”范式。

## R03

工程上需要同时解决两个目标：

- 判定：图是否存在欧拉路径或欧拉回路。
- 构造：若存在，输出一个可执行的顶点序列，保证每条边恰好出现一次。

仅做判定不足以支撑自动化系统，因此本目录 MVP 采用可复现构造算法（Hierholzer）并配套正确性校验。

## R04

采用 Hierholzer 算法，核心机制是“走到无法继续时回退拼接”：

1. 先检查欧拉性条件（度数/入出度 + 连通性）。
2. 选择起点：优先奇度点或 `out-in=1` 点；若回路则选任一有边点。
3. 用栈维护当前探索路径。
4. 若栈顶有未用边，就沿一条未用边前进并压栈。
5. 若栈顶无未用边，则将该顶点弹出并追加到输出序列。
6. 所有边消费完后反转输出序列，得到欧拉路径/回路。

## R05

设顶点数为 `V`，边数为 `E`：

- 时间复杂度：`O(V + E)`。
- 空间复杂度：`O(V + E)`。

其中连通性检查与路径构造都在邻接表上线性完成，是该问题的经典高效实现。

## R06

无向图样例：`(0,1),(1,2),(2,0),(0,3),(3,2)`。

- 度数分别为 `deg(0)=3, deg(1)=2, deg(2)=3, deg(3)=2`。
- 奇度点为 `0` 和 `2`，恰好 2 个。
- 非零度顶点连通。

因此存在欧拉路径（非回路），例如 `0 -> 1 -> 2 -> 0 -> 3 -> 2`，路径长度为 `E+1`。

## R07

优点：

- 判定规则清晰，构造过程确定性强。
- 线性复杂度，适合大规模稀疏图。
- 支持重边（通过边 ID 区分）。

局限：

- 目标仅是“每条边一次”，不考虑最短或最小代价。
- 图结构变更后通常需要重新判定与构造。
- 若不满足欧拉条件，只能转向中国邮差等“允许重复边”的问题。

## R09

适用场景：

- 一笔画可行性检查。
- 巡检/扫边类任务（边覆盖一次）。
- 路由与图数据处理中的边级遍历子模块。

不适用场景：

- 最短路、最小花费巡回（应使用 Dijkstra、最小费用流、中国邮差、TSP 等）。
- 高频在线动态图（需增量维护结构）。

## R10

正确性要点：

1. 先做欧拉存在性判定，不能跳过连通性检查。
2. 每条边必须有唯一“已使用”标记，禁止重复消费。
3. 结果路径长度必须满足 `len(path)=E+1`。
4. 无向图按无序边计数；有向图按有序边计数，方向必须严格匹配。

## R11

实现稳定性策略：

- 用“边 ID + used 数组”而不是删除邻接边，避免重边误合并。
- 对每个顶点维护游标 `next_idx`，跳过已用边，避免重复扫描。
- 无边图返回平凡回路（单点）。
- 构造后使用验证器二次检查边计数，防止隐藏错误。

## R14

常见失效模式与防护：

- 只检查度数不检查连通性，导致“局部可走、全局不可覆盖”。
- 重边被错误去重，导致边数量守恒失效。
- 结果路径存在但未覆盖全部边。
- 有向图中忽略方向，导致错误把反向边当可替代。

对应防护已在 `demo.py` 的判定函数和 `verify_*` 中实现。

## R15

实践建议：

- 将“判定”和“构造”拆分，单测可分别定位故障。
- 在有向图上优先打印 `out-in` 分布，能快速发现无解原因。
- 小图先通过后再上大图，尤其验证重边与非连通边界。
- 保持数据结构简单（邻接表 + 边 ID）比追求抽象更稳健。

## R18

`demo.py` 源码级算法流程（9 步）：

1. `run_demo_cases` 用 `numpy` 定义固定测试图，并转换为 Python 边列表。  
2. `find_euler_undirected` 统计度数，调用 `is_connected_nonzero_undirected` 检查非零度连通性。  
3. 同函数依据奇度点数量判定 `none/path/circuit` 并选择起点。  
4. `hierholzer_undirected` 使用“邻接表 + 边 ID + used + 栈”消费每条无向边一次并回退拼接。  
5. `verify_undirected_path` 通过 `Counter` 比较目标边多重集与路径边多重集，确认无向边使用次数完全匹配。  
6. `find_euler_directed` 统计入度/出度，检查弱连通和 `out-in` 差分条件，再决定类型和起点。  
7. `hierholzer_directed` 在有向邻接表上执行同样的栈式回退流程，生成有向欧拉路径。  
8. `verify_directed_path` 校验有向边方向与次数，防止方向错误或漏边。  
9. `main` 汇总运行所有用例并断言预期类型，最终打印 `All demo cases passed.`。  

第三方库拆解说明：

- `numpy` 仅用于样例数据构造，不承担图算法逻辑；
- 图判定、路径构造与正确性验证均在源码中手写实现，非黑盒调用。
