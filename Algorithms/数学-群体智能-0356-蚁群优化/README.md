# 蚁群优化

- UID: `MATH-0356`

- 分类: `群体智能`

- 目标目录: `Algorithms/数学-群体智能-0356-蚁群优化`

## R01

蚁群优化（Ant Colony Optimization, ACO）是一类受蚂蚁觅食行为启发的随机优化方法。核心思想是：

- 用“信息素”表示历史经验（某些决策路径被反复选中）；
- 用“启发式信息”表示即时贪心偏好（例如距离越短越优先）；
- 通过群体并行采样 + 信息素更新，逐步把搜索重心推向高质量解。

本目录 MVP 采用 ACO 求解对称欧氏 TSP（旅行商问题）。

## R02

ACO 最早在组合优化场景中广泛成功，尤其是路径规划、调度、车辆路径、网络路由等“离散结构 + 全局最优难求”的问题。

相较于单一贪心，ACO 的优势在于：

- 同时保留探索（随机转移）和利用（信息素强化）；
- 可以持续迭代改进，而不是一次构造后就固定；
- 能在复杂搜索空间中找到较高质量可行解。

## R03

本条目问题定义如下：

- 输入：`n` 个二维城市坐标；
- 目标：找到一个 Hamilton 回路，使总路径长度最小；
- 输出：一条城市访问顺序（不重复访问，每个城市恰好一次）与对应回路长度。

MVP 不追求证明全局最优，而追求可运行、可复现、可解释的 ACO 最小实现。

## R04

实现采用经典 Ant System 框架并加入精英强化：

1. 初始化距离矩阵与信息素矩阵；
2. 每轮让多只蚂蚁按概率规则构造完整回路；
3. 根据回路质量做信息素沉积（路径越短，沉积越多）；
4. 执行全局挥发，防止早熟收敛；
5. 对当前全局最优回路追加精英沉积；
6. 迭代若干轮后返回最好结果。

状态转移概率使用 `tau^alpha * eta^beta`，其中 `tau` 是信息素，`eta=1/distance` 是启发式可见度。

## R05

设城市数 `n`、蚂蚁数 `m`、迭代轮数 `T`：

- 单只蚂蚁构造一条路径约 `O(n^2)`（逐步在剩余节点中计算概率）；
- 单轮成本约 `O(m * n^2)`；
- 总体成本约 `O(T * m * n^2)`。

本 MVP 选择中小规模实例（22 城市），可在秒级完成。

## R06

`demo.py` 的固定实验设置：

- 随机种子固定，生成 22 个二维城市点；
- ACO 参数默认：`ants=30`、`iterations=160`；
- 输出与两个基线对比：
  - 最佳最近邻路径；
  - 随机采样路径中的最佳值；
- 并输出收敛历史尾部，展示是否趋稳。

整个流程无需交互输入。

## R07

优点：

- 算法结构清晰，便于学习群体智能核心机制；
- 结果可复现（固定随机种子）；
- 代码没有依赖黑盒优化器，便于二次修改。

局限：

- 仍是启发式，不保证有限轮内达到全局最优；
- 参数较敏感，问题规模增大时需要重新调参；
- 当前只覆盖对称欧氏 TSP，不含时间窗/容量等扩展约束。

## R09

适用场景：

- 离散组合优化教学与原型验证；
- 中小规模路径规划实验；
- 需要快速构建可解释启发式求解器的场景。

不适用场景：

- 严格最优证明必须给出的场景；
- 超大规模且强实时约束的工业求解；
- 需要复杂业务约束（例如 VRP 多维约束）但尚未建模扩展的场景。

## R10

正确性直觉（MVP 级）：

1. 每只蚂蚁每步只从未访问城市中选点，保证最终是合法排列；
2. 回路长度直接由距离矩阵累加得到，评价函数与目标一致；
3. 信息素挥发抑制过时路径，避免单一路径永久垄断；
4. 短路径沉积更多信息素，使下一轮采样更偏向优解；
5. 精英强化提高已知最优路径的保留概率。

因此算法能在迭代中逐步提升“采样到好解”的概率。

## R11

数值与鲁棒性处理：

- 距离矩阵对角线设为 `inf`，显式禁止自环；
- 启发式矩阵对角线重置为 `0`，避免无意义转移概率；
- 若概率权重和为 0 或非有限值，退化为均匀随机选择；
- 保持信息素矩阵对称并清零对角，避免模型漂移。

这些策略使 MVP 在常规随机点集上稳定运行。

## R12

关键参数及影响：

- `alpha`：信息素影响强度，越大越偏历史经验；
- `beta`：启发式影响强度，越大越偏局部短边；
- `evaporation_rate`：挥发率，越大遗忘越快；
- `q`：沉积总量系数，越大更新越激进；
- `n_ants`、`n_iterations`：计算预算上限；
- `elite_weight`：精英路径额外强化强度。

调参建议：先固定 `beta` 较高（3~5）保证可行质量，再通过 `alpha` 和挥发率平衡收敛速度与多样性。

## R13

理论保证边界：

- 该实现属于启发式近似方法，不提供固定近似比保证；
- 在有限迭代与有限蚂蚁数下，不保证命中全局最优；
- 随迭代增长，经验上常能获得高质量解，但质量依赖参数、实例结构与随机种子。

因此它更适合作为“工程上可用的近似搜索器”，而非严格最优证明器。

## R14

常见失效模式：

1. 参数过度利用（`alpha` 过大、挥发过低）导致早熟收敛；
2. 参数过度探索（挥发过高、蚂蚁过少）导致收敛慢；
3. 城市分布含大量近重复点，概率权重数值不稳定；
4. 规模扩大后迭代预算不足，结果波动明显。

诊断信号：

- `history` 很早进入平台且质量一般；
- 与基线相比长期无提升；
- 多次不同种子运行方差过大。

## R15

工程化扩展建议：

- 增加 `2-opt` 局部搜索作为后处理（ACO + 局部改进）；
- 引入参数自适应策略（按轮次动态调整挥发率）；
- 对大规模实例加入候选邻域列表，降低构造复杂度；
- 做多种子批量评估并汇总统计分位数，而不是单次结果。

## R16

相关算法：

- 模拟退火（SA）：单轨迹随机爬山；
- 遗传算法（GA）：群体进化 + 交叉变异；
- 粒子群优化（PSO）：连续空间更常见；
- 禁忌搜索（Tabu）：显式记忆禁忌表；
- Lin-Kernighan / 2-opt：TSP 经典局部改进。

ACO 的特点是“构造式采样 + 轨迹记忆”。

## R18

`demo.py` 的源码级算法流程（8 步）：

1. `main` 生成固定随机城市坐标并读取 `ACOConfig`，确保实验可复现。  
2. `run_aco_tsp` 调用 `euclidean_distance_matrix` 构造距离矩阵，并把对角线设为 `inf` 防止自环。  
3. 在 `run_aco_tsp` 中构建启发式矩阵 `heuristic=1/distance`，并用最近邻结果估计初始信息素 `tau0`。  
4. 每轮迭代中，每只蚂蚁通过 `construct_ant_tour` 按 `tau^alpha * eta^beta` 概率规则逐步采样下一城市，形成完整回路。  
5. 对每条回路用 `cycle_length` 计算总长度，并维护全局最优 `best_tour / best_length`。  
6. 执行全局挥发 `pheromone *= (1 - evaporation_rate)`，随后对当轮各蚂蚁路径调用 `deposit_pheromone` 按 `q/length` 沉积。  
7. 若存在全局最优路径，再追加一次精英沉积（`elite_weight * q / best_length`），强化稳定优解。  
8. 迭代结束后返回最优结果；`main` 计算最近邻/随机基线并打印对比，同时通过断言验证回路合法与收敛记录单调性。

实现中没有把第三方包当黑盒优化器：核心 ACO 流程（构造、转移概率、挥发、沉积、精英强化）均在源码内可逐步追踪。
