## R01

FETI（Finite Element Tearing and Interconnecting）方法是一类非重叠区域分解算法。  
核心思想是把全局有限元系统按子域“撕开”（tearing），在界面上引入拉格朗日乘子，用对偶变量来重新“连接”（interconnecting）子域解。

本目录给出一个可运行的最小 MVP：  
- 问题是 1D Poisson 方程；  
- 区间被拆成两个子域；  
- 界面仅有 1 个连续性约束；  
- 用显式构造的对偶 Schur 补系统 + CG 完成求解。

## R02

求解模型：

`-u''(x) = 1 + 0.2*sin(2*pi*x),  x in (0,1)`  
`u(0)=0, u(1)=0`

解析解用于验证：

`u(x) = 0.5*x*(1-x) + (0.05/pi^2)*sin(2*pi*x)`

离散与划分：
- 全局均匀网格，默认 `80` 个小区间；
- 在 `x=0.5` 处分成两个非重叠子域；
- 每个子域保留自己的界面自由度拷贝；
- 通过 `B1*u1 + B2*u2 = 0` 强制界面位移连续。

## R03

MVP 的离散代数模型（与 `demo.py` 一致）：

1. 子域线性系统：
- `K1*u1 = f1 - B1^T*lambda`
- `K2*u2 = f2 - B2^T*lambda`

2. 代回连续性约束，得到对偶系统：
- `F*lambda = d`
- `F = B1*K1^{-1}*B1^T + B2*K2^{-1}*B2^T`
- `d = B1*K1^{-1}*f1 + B2*K2^{-1}*f2`

3. 解出 `lambda` 后回代得到 `u1,u2`，再拼接为全局 `u_feti`。

说明：  
为了和全局离散严格对应，界面对角刚度与界面载荷在两个子域之间做了“一分为二”的贡献分配。

## R04

时间复杂度（按当前 MVP 实现）：

- 组装 `K1,K2,f1,f2`：`O(n)`，`n` 为每个子域节点数；
- 本地线性求解（`numpy.linalg.solve`，稠密）：每次 `O(n^3)`；
- 对偶系统维度为 1，本例 CG 为 `O(1)`（通常 1 次迭代）；
- 额外做一次单体全局解用于校验（稠密）是 `O((2n)^3)`。

因此当前脚本总体由稠密求解主导，规模律可记为 `O(n^3)`。

## R05

空间复杂度（按当前 MVP）：

- 本地矩阵 `K1,K2` 稠密存储：各 `O(n^2)`；
- 验证用全局矩阵 `A_global` 稠密存储：`O((2n)^2)`；
- 向量级数据是 `O(n)`。

整体空间复杂度为 `O(n^2)`。  
工程版通常改为稀疏存储与迭代子域解算器。

## R06

一个界面微例（`n=4`）：

- 全局内点是 `1..7`，界面在全局点 `4`；
- 左子域未知含 `1..4`，右子域未知含 `4..7`（各自持有界面拷贝）；
- 界面约束是 `u1(4)-u2(4)=0`；
- 因为界面节点在全局矩阵中的对角贡献为 2，拆域后左/右各取 1；
- 对应载荷在界面节点也按 `1/2` 分摊。

按这个构造，FETI 拼接解与单体全局解可达到机器精度一致。

## R07

FETI 的价值在于：

- 可把大问题拆成多个子域并行处理；
- 把耦合集中到低维界面系统，便于做预条件；
- 适合多体/多材料/复杂几何的“局部求解 + 界面协调”模式；
- 在 HPC 场景中天然匹配分布式内存并行。

本 MVP 重点展示“算法骨架”，不是性能极限实现。

## R08

理论要点（对偶域分解视角）：

1. 子域变量满足本地平衡方程；  
2. 界面连续性通过拉格朗日乘子实现；  
3. 消去本地变量后得到界面 Schur 补系统；  
4. 若本地算子与约束构造正确，则对偶系统是可解且与原问题一致；  
5. 解出乘子后回代，得到满足连续性的全局位移场。

## R09

适用条件与局限：

- 适用于有限元离散后可明确提取界面自由度的线性系统；
- 适用于希望并行处理本地子问题、再做界面协调的任务；
- 本条目只覆盖“两子域、单界面约束、无粗空间”的教学最简形态；
- 真正大规模 FETI/FETI-DP 需要：
  - 稀疏线性代数；
  - 高质量界面预条件；
  - 处理“浮动子域”刚体模态的粗空间机制。

## R10

正确性框架（脚本中的可执行检查）：

1. `interface_jump = B1*u1 + B2*u2` 必须接近 0；  
2. `u_feti` 与单体全局解 `u_global` 的无穷范数误差需接近机器精度；  
3. 与解析解比较误差需落在阈值内；  
4. 对偶 CG 的残差应收敛到阈值以下。  

`demo.py` 的 `run_checks` 对这些条件均做了断言。

## R11

在默认配置（`half_nodes=40`）下，典型输出量级为：

- `lambda ≈ 3.98e-4`（非零，说明界面对偶变量确实参与求解）；  
- `interface_jump ≈ 1e-16`；  
- `||u_feti-u_global||_inf ≈ 1e-16`；  
- `||u_feti-u_exact||_inf ≈ 1e-6`；  
- CG 通常 1 次迭代达到阈值（本例对偶维度为 1）。

这说明实现既满足界面连续性，也与单体解一致。

## R14

鲁棒性与失效模式：

- 对配置参数做了合法性校验（网格规模、容差、迭代次数）；
- CG 检测到分母接近 0 时会抛出错误，防止静默失败；
- 若连续性、精度或残差不达标，`run_checks` 触发断言。

常见错误来源：
- 界面符号写反（`B1/B2` 号位错误）；
- 界面刚度或载荷没有正确“半分”；
- 拼接全局向量时重复或漏掉界面节点。

## R18

源码级算法流程拆解（`demo.py`，8 步）：

1. 读取默认配置，确定每个子域节点数 `n` 与网格步长 `h`。  
2. 分别组装 `K1,K2,f1,f2`，并把界面对角刚度与载荷贡献在左右子域中各取一半。  
3. 构造界面约束矩阵 `B1,B2`，表达连续性 `B1*u1 + B2*u2 = 0`。  
4. 通过 `K_i^{-1}` 消去本地未知，显式形成对偶系统 `F*lambda=d`。  
5. 用手写 CG 求解 `lambda`（本例对偶维度为 1，通常一步收敛）。  
6. 将 `lambda` 回代到本地方程，得到 `u1,u2` 并拼接出 `u_feti`。  
7. 独立构造单体全局系统 `A_global*u_global=rhs`，与 `u_feti` 做一致性对比，并和解析解计算误差。  
8. 运行断言检查（界面连续性、对单体一致性、误差阈值、残差阈值），通过后输出结果表。  

说明：本实现只把 `numpy.linalg.solve` 当作线性代数基础算子；FETI 的对偶构造、界面约束、回代流程都在源码中显式展开，不是黑盒函数调用。
