## R01

最大似然估计（Maximum Likelihood Estimation, MLE）是参数估计的核心方法。给定样本 `x_1,...,x_n` 与参数化概率模型 `p(x|theta)`，MLE 选择使观测样本“最可能出现”的参数：

- 似然函数：`L(theta)=∏_{i=1}^n p(x_i|theta)`
- 对数似然：`ell(theta)=sum_{i=1}^n log p(x_i|theta)`
- MLE 定义：`theta_hat = argmax_theta ell(theta)`

由于对数函数单调，最大化 `L(theta)` 与最大化 `ell(theta)` 等价；工程实现通常使用对数似然，便于数值稳定与求导。

## R02

历史背景（简要）：

- 20 世纪初，R. A. Fisher 系统化提出并推广 MLE 框架；
- MLE 连接了概率建模、优化和统计推断，成为现代统计学习的基石；
- 在广义线性模型、隐变量模型（如 EM）和深度学习概率头部中，MLE 都是默认训练准则之一。

## R03

本条目 MVP 聚焦问题：

- 模型：一维高斯分布 `N(mu, sigma^2)`，其中 `mu` 与 `sigma>0` 未知；
- 输入：独立同分布样本向量 `x`；
- 输出：`mu_hat`、`sigma_hat` 两个参数估计；
- 实现目标：
  - 给出高斯 MLE 的闭式解；
  - 用梯度上升从零实现数值优化求解；
  - 对比两者结果，验证算法正确性。

## R04

对一维高斯样本，log-likelihood 为：

`ell(mu,sigma) = -n*log(sigma) - (1/(2*sigma^2))*sum_i (x_i-mu)^2 - n/2*log(2*pi)`

其 MLE 闭式解：

- `mu_hat = mean(x)`
- `sigma_hat^2 = (1/n) * sum_i (x_i-mu_hat)^2`

注意这里是除以 `n`（MLE），而非无偏方差常用的 `n-1`。

## R05

复杂度分析：

- 闭式解：
  - 时间复杂度 `O(n)`（一次均值 + 一次方差统计）；
  - 空间复杂度 `O(1)`（不计输入样本存储）。
- 梯度上升（迭代 `T` 次）：
  - 时间复杂度 `O(T*n)`；
  - 空间复杂度 `O(1)`（仅维护标量参数与若干统计量）。

## R06

微型手算示例：`x = [1,2,4]`。

1. `mu_hat = (1+2+4)/3 = 7/3 ≈ 2.3333`
2. 偏差平方和：
   - `(1-7/3)^2 = 16/9`
   - `(2-7/3)^2 = 1/9`
   - `(4-7/3)^2 = 25/9`
   - 合计 `42/9 = 14/3`
3. `sigma_hat^2 = (1/3)*(14/3)=14/9≈1.5556`
4. `sigma_hat ≈ 1.2472`

这就是该样本在高斯模型下的 MLE。

## R07

MLE 的价值与优点：

- 统一：很多模型都能写成“最大化 log-likelihood”；
- 可解释：目标函数直接对应数据在模型下的概率；
- 可扩展：闭式解不可得时可转为数值优化；
- 理论性质好：在常规条件下具备一致性、渐近正态性。

局限：

- 对模型设定敏感（模型错设会导致偏差）；
- 对异常值可能不鲁棒（尤其高斯假设）；
- 小样本下可能方差较大。

## R08

高斯 MLE 的关键推导（简要）：

1. 写出 `ell(mu,sigma)`。
2. 对 `mu` 求偏导并设为 0：
   - `d ell / d mu = (1/sigma^2)*sum_i(x_i-mu)=0`
   - 得 `mu_hat = mean(x)`。
3. 对 `sigma^2` 或 `sigma` 求偏导并设为 0，得到
   - `sigma_hat^2 = (1/n)*sum_i(x_i-mu_hat)^2`。
4. 检查二阶导（或凸凹性）可确认是极大值点。

在 `demo.py` 里，我们用 `eta = log(sigma)` 重参数化，避免 `sigma` 取到非正值。

## R09

适用前提与边界：

- 样本近似独立同分布（i.i.d.）；
- 模型可辨识（不同参数对应不同分布）；
- 参数空间设置合理（如 `sigma>0`）；
- 数据量足够时，估计更稳定。

不满足时需谨慎：

- 强相关样本、重尾噪声或显著离群点；
- 模型明显错设（如用单高斯拟合多峰数据）。

## R10

正确性要点（本 MVP 的可验证版本）：

1. 闭式解是高斯 log-likelihood 的一阶条件解；
2. 梯度上升实现中，目标函数值应单调上升或总体上升；
3. 梯度数值检查（finite difference）应与解析梯度接近；
4. 梯度上升收敛点应与闭式解接近。

`demo.py` 对上述 2-4 给出自动断言。

## R11

数值稳定性策略：

- 使用 `log-likelihood` 替代 likelihood，避免乘积下溢；
- 使用 `eta=log(sigma)`，把 `sigma>0` 约束转为无约束优化；
- 采用每样本平均对数似然，减少随 `n` 放大的梯度尺度；
- 使用固定随机种子，避免验证时结果漂移。

## R12

性能与调参要点：

- 该问题有闭式解时优先闭式解，计算更快更稳；
- 数值优化用于通用场景（无闭式解）时，学习率 `lr` 是首要超参数；
- 重参数化（如 `log(sigma)`）比硬截断更稳定；
- 梯度范数和参数更新步长可作为收敛监控指标。

## R13

理论保证（统计意义，简述）：

- 一致性：当 `n->∞`，`theta_hat` 以概率收敛到真值（在正则条件下）；
- 渐近正态：`sqrt(n)(theta_hat-theta0)` 近似服从正态分布；
- 渐近有效：达到 Cramér-Rao 下界（在常规条件与正确模型下）。

这些是渐近性质，不是有限样本的绝对保证。

## R14

常见失败模式与防护：

- 失败：学习率过大，梯度上升震荡。
  - 防护：减小 `lr`，增加迭代上限。
- 失败：直接优化 `sigma` 导致出现非正值。
  - 防护：改用 `eta=log(sigma)`。
- 失败：把无偏方差 `1/(n-1)` 误当 MLE 方差。
  - 防护：明确 MLE 用 `1/n`。
- 失败：样本含明显离群点，高斯 MLE 被拉偏。
  - 防护：做异常值诊断，必要时改用鲁棒模型。

## R15

本实现（`demo.py`）的结构：

- `generate_normal_samples`：生成可复现的高斯样本；
- `closed_form_gaussian_mle`：计算 `mu_hat` 与 `sigma_hat` 闭式解；
- `avg_log_likelihood_and_grad`：给出平均对数似然及解析梯度；
- `gradient_ascent_mle`：从零实现梯度上升；
- `finite_difference_grad_check`：做数值梯度对照；
- `main`：执行完整实验、打印结果并做断言。

## R16

相关方法关系：

- MAP：在 MLE 目标上加入先验正则项（可视作惩罚项）；
- EM：含隐变量模型中常用于求解 MLE 的迭代框架；
- 贝叶斯推断：不做点估计，而输出参数后验分布；
- 鲁棒估计（如 Huber、t 分布）：用于减轻离群点影响。

## R18

`demo.py` 源码级流程拆解（8 步，非黑盒）：

1. `main` 固定随机种子并调用 `generate_normal_samples` 生成一维高斯样本。  
2. 调用 `closed_form_gaussian_mle`，用样本均值与 `1/n` 方差得到闭式 MLE。  
3. 在初始点 `(mu0, log_sigma0)` 调用 `finite_difference_grad_check`，验证解析梯度正确。  
4. `gradient_ascent_mle` 初始化参数并进入迭代：每轮调用 `avg_log_likelihood_and_grad` 计算目标与梯度。  
5. 按 `mu <- mu + lr*g_mu`、`log_sigma <- log_sigma + lr*g_log_sigma` 执行梯度上升更新。  
6. 每轮记录对数似然历史，并用参数更新量阈值判断是否收敛。  
7. 迭代结束后将 `log_sigma` 反变换为 `sigma=exp(log_sigma)`，得到数值优化估计。  
8. 在 `main` 中比较“闭式解 vs 梯度解”误差并断言足够接近，同时确认最终目标函数高于初值。  

该 MVP 未调用第三方黑盒估计器；MLE 主体计算与优化步骤均在源码中显式实现。
