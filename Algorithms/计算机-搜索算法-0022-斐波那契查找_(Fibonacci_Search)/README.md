# 斐波那契查找 (Fibonacci Search)

- UID: `CS-0022`
- 学科: `计算机`
- 分类: `搜索算法`
- 源序号: `22`
- 目标目录: `Algorithms/计算机-搜索算法-0022-斐波那契查找_(Fibonacci_Search)`

## R01

斐波那契查找（Fibonacci Search）用于在**有序数组**中定位目标值。它不按“二分中点”切分区间，而是使用斐波那契数列确定探测位置，逐步缩小搜索范围。

## R02

问题形式化：
- 输入：升序数组 `arr`（长度 `n`）与目标值 `target`
- 输出：若存在下标 `i` 使 `arr[i] == target`，返回某个合法下标；否则返回 `-1`

## R03

适用前提：
- 输入序列必须按非降序排列
- 元素类型支持比较操作（`<`, `==`, `>`）
- 若存在重复值，需要约定返回语义（本条目返回“任意一个命中位置”）

## R04

核心思想：
- 先找到最小的斐波那契数 `F_m`，使 `F_m >= n`
- 用三元组 `(F_m, F_{m-1}, F_{m-2})` 维护当前候选窗口规模
- 每次探测位置取 `i = min(offset + F_{m-2}, n - 1)`，并比较 `arr[i]` 与 `target`
- 根据比较结果，将窗口缩小到左侧或右侧子区间，并同步更新斐波那契三元组

## R05

简化伪代码：

```text
if n == 0:
    return -1

build smallest Fibonacci number fibM >= n
fibMm1 = previous Fibonacci of fibM
fibMm2 = one before fibMm1
offset = -1

while fibM > 1:
    i = min(offset + fibMm2, n - 1)
    if arr[i] < target:
        fibM = fibMm1
        fibMm1 = fibMm2
        fibMm2 = fibM - fibMm1
        offset = i
    elif arr[i] > target:
        fibM = fibMm2
        fibMm1 = fibMm1 - fibMm2
        fibMm2 = fibM - fibMm1
    else:
        return i

if fibMm1 and offset + 1 < n and arr[offset + 1] == target:
    return offset + 1

return -1
```

## R06

正确性要点（不变式直观）：
- 任意迭代时，目标若存在，必然位于“当前窗口”内
- `offset` 左侧元素都已经被证明 `< target`，无需再考虑
- 每次比较都会丢弃一段不可能包含目标的区间，并把窗口映射到更小的斐波那契规模
- 当窗口缩小到不可再分（`fib_m <= 1`）后，只需检查最后一个候选位置即可

## R07

复杂度：
- 时间复杂度：`O(log n)`（窗口规模按斐波那契级数收缩）
- 空间复杂度：`O(1)`（仅维护常数个变量）

## R08

边界与特殊情况：
- 空数组
- 单元素数组
- 目标小于最小值或大于最大值
- 探测下标可能越界，需通过 `min(..., n - 1)` 截断
- 重复值存在时返回任意命中位置

## R09

示例：
- `arr = [10, 22, 35, 40, 45, 50, 80, 82, 85, 90, 100]`, `target = 85`
  - 构建到 `F_m = 13`（`>= 11`）
  - 首次探测约在下标 `4`（值 `45`），目标更大，窗口右移
  - 再探测下标 `7`（值 `82`），继续右移
  - 再探测下标 `8`（值 `85`），命中
- `target = 101` 时，窗口最终耗尽，返回 `-1`

## R12

典型应用：
- 顺序存储且支持随机访问的有序表查找
- 对比二分查找时，作为“非均匀切分”思路的教学示例
- 某些存储模型下（历史上顺序介质）减少“跳跃模式”成本的查找策略参考

## R13

常见错误：
- 忘记有序性前提，直接用于无序数组
- 斐波那契三元组更新写错，导致死循环或漏检
- 漏掉最后候选位置 `offset + 1` 的补充检查
- 下标不做边界裁剪，触发越界异常

## R14

常见变体：
- 返回第一个等于目标的位置（命中后向左收缩）
- 仅返回布尔结果（是否存在）
- 对泛型可比较对象复用同样流程（字符串、时间戳等）

## R15

与二分/指数查找对比：
- 二分查找：固定按中点切分，逻辑最直接
- 指数查找：先指数扩张区间，再二分精搜，适合上界未知场景
- 斐波那契查找：按斐波那契比例切分区间，不依赖除法运算的中点公式（现代 CPU 上该优势通常不明显）
- 三者在有序静态数组中的渐进复杂度都可达 `O(log n)`

## R16

工程实践建议：
- API 文档中显式声明“输入必须非降序”
- 明确重复值返回语义（任意命中/最左/最右）
- 以表驱动测试覆盖边界样例，防止更新三元组时引入回归
- 对外层调用者暴露统一失败语义（如返回 `-1`）

## R17

最小测试清单：
- 空数组、单元素命中/未命中
- 命中首元素、中间元素、末元素
- 未命中（小于最小值、位于缺口、大于最大值）
- 重复值数组（验证“存在性”和返回索引合法）
- 非有序输入（应抛出 `ValueError`）

## R18

源码级算法流程拆解（以本目录 `demo.py` 的 `fibonacci_search` 为准）：
1. 接收 `arr` 与 `target`，先调用 `_ensure_non_decreasing` 校验数组是否非降序；不满足则抛出 `ValueError`。
2. 若 `arr` 为空直接返回 `-1`；随后初始化斐波那契三元组 `fib_mm2=0, fib_mm1=1, fib_m=1`。
3. 循环扩展三元组直到 `fib_m >= n`，得到能覆盖当前数组长度的最小斐波那契窗口。
4. 设置 `offset = -1`，表示当前已确认 `< target` 的最后位置；进入主循环 `while fib_m > 1`。
5. 按 `i = min(offset + fib_mm2, n - 1)` 计算探测点，读取 `arr[i]` 与 `target` 比较。
6. 若 `arr[i] < target`，丢弃左段并右移窗口：更新 `fib_m = fib_mm1`、`fib_mm1 = fib_mm2`、`fib_mm2 = fib_m - fib_mm1`，同时 `offset = i`。
7. 若 `arr[i] > target`，保留左段并收缩窗口：更新 `fib_m = fib_mm2`、`fib_mm1 = fib_mm1 - fib_mm2`、`fib_mm2 = fib_m - fib_mm1`。
8. 若 `arr[i] == target` 立即返回 `i`；循环结束后补查 `offset + 1`，命中则返回该下标，否则返回 `-1`。
