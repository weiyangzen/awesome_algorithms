# scrypt

## R01

`scrypt` 是一种面向口令的密钥派生函数（Password-Based KDF），核心目标是“让暴力破解在 CPU 与内存上都昂贵”。
它由 Colin Percival 提出，典型应用是口令哈希与基于口令的密钥生成。

## R02

与仅强调计算量的 PBKDF2 不同，scrypt 把“内存硬度（memory-hardness）”作为一等约束：
- 攻击者不能只靠堆更多 ALU 线性提速；
- 若试图减少内存，会显著增加重算成本；
- 更适合抵抗 GPU/ASIC 大规模并行口令猜测。

## R03

常用参数与符号：
- `P`：口令（password）
- `S`：盐（salt）
- `N`：CPU/内存成本参数（必须是 2 的幂）
- `r`：块大小参数（影响内存与带宽）
- `p`：并行参数（独立 ROMix 链路数）
- `dkLen`：输出密钥长度（字节）
- `DK`：最终导出密钥

## R04

可抽象为：

`DK = scrypt(P, S, N, r, p, dkLen)`

该函数是确定性的：同一组输入总是得到同一输出；任何一个输入位改变都应导致输出显著变化。

## R05

算法骨架（RFC 7914 思路）可分两层：
1. 外层使用 PBKDF2-HMAC-SHA256 把 `(P, S)` 扩展成初始缓冲区 `B`；
2. 对 `B` 的每个并行分块执行 `ROMix`（其内部大量使用 `BlockMix`）；
3. 把混合后的 `B'` 再送入 PBKDF2-HMAC-SHA256 得到 `DK`。

## R06

内存成本近似为：

`Memory ≈ 128 * r * N bytes`

这就是 scrypt 的关键防御面。示例参数 `N=2^14, r=8` 时，ROMix 仅核心缓存约为 `16 MiB`，已能显著抬高离线暴力成本。

## R07

参数选择是“安全性 vs 延迟”的工程权衡：
- 提高 `N`：最直接提高内存与时间成本；
- 提高 `r`：提升每轮块处理成本和内存占用；
- 提高 `p`：增加并行链路，通常线性提高总工作量；
- 生产环境应按硬件预算压测后定标，而非固定抄模板。

## R08

复杂度（量级）可概括为：
- 时间：约 `O(N * r * p)`（外加 PBKDF2 常数开销）；
- 空间：约 `O(N * r)`；
- 因为存在大内存随机访问，实际性能还受内存带宽和缓存行为显著影响。

## R09

安全性要点：
- scrypt 的目标是“延缓口令猜测”，不是替代随机高熵主密钥；
- 必须为每条口令使用唯一随机盐，防止彩虹表复用；
- 参数过低会让算法退化为“可快速并行枚举”；
- 输出若用于认证存储，应结合恒定时间比较与完整元数据存档。

## R10

本目录 MVP 目标：
- 用标准库 `hashlib.scrypt` 实现最小可运行样例；
- 给出参数校验、派生、记录构造、口令校验完整闭环；
- 通过断言演示确定性、口令敏感性、盐敏感性；
- 无交互输入，`uv run python demo.py` 可直接执行。

## R11

`demo.py` 函数划分：
- `ScryptParams`：参数容器；
- `_validate_params`：约束 `N/r/p/dkLen` 合法性；
- `derive_key`：调用 `hashlib.scrypt` 执行派生；
- `build_record`：生成可存储记录（含参数、salt、派生值）；
- `verify_password`：重算并用 `hmac.compare_digest` 恒定时间比较；
- `bit_distance`：衡量输出变化（雪崩效应的直观指标）。

## R13

预期输出特征：
- 打印参数 `N/r/p/dkLen`、估算内存占用、单次派生耗时；
- 同输入两次派生值完全相同（`deterministic check: True`）；
- 修改口令或盐后派生值不同（敏感性检查为 `True`）；
- 正确口令校验通过、错误口令校验失败。

## R14

边界与异常处理：
- `N` 非 2 的幂或小于 2 直接报错；
- `r/p/dkLen` 非正值直接报错；
- 空口令或空盐在本 MVP 中拒绝执行；
- `bit_distance` 要求两个输入长度一致。

## R18

源码级流程拆解（对应 `demo.py` 与 RFC 7914，8 步）：
1. `main()` 构造 `ScryptParams(N=2^14, r=8, p=1, dkLen=32)`，并准备固定口令和盐。  
2. `derive_key()` 先执行 `_validate_params()`，保证 `N` 为 2 的幂且各参数为正。  
3. `hashlib.scrypt` 内部先做一次 PBKDF2-HMAC-SHA256（`c=1`）生成初始缓冲区 `B`。  
4. `B` 被分成 `p` 个区块，每个区块进入 `ROMix`：先顺序填充向量 `V[0..N-1]`（保存历史状态）。  
5. `ROMix` 第二阶段按 `j = Integerify(X) mod N` 随机索引 `V[j]`，执行 `X = BlockMix(X xor V[j])`，重复 `N` 轮。  
6. `BlockMix` 以 64 字节块为粒度调用 `Salsa20/8` 核心，完成扩散与重排，形成新的区块状态。  
7. 所有区块混合完成后拼接成 `B'`，再经 PBKDF2-HMAC-SHA256（`c=1`）输出最终 `DK`。  
8. `demo.py` 用 `build_record()` 存储参数与摘要，用 `verify_password()` 重算并 `compare_digest` 比较，完成认证闭环。  

以上拆解把 `hashlib.scrypt` 的内部算法路径展开为可追踪步骤，而非仅停留在“单函数黑盒调用”。
