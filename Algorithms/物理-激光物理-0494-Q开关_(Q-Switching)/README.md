## R01

Q 开关（Q-Switching）是通过快速改变激光腔损耗（即品质因子 Q）来压缩能量释放时间的脉冲产生技术。

核心思想：

- 在“低 Q（高损耗）”阶段抑制激光振荡，让反转粒子数先积累；
- 在开关时刻切到“高 Q（低损耗）”，使腔内净增益瞬间变正；
- 储存的反转在很短时间内转化为高峰值脉冲。

## R02

Q 开关是固体激光器、脉冲打标、激光测距、非线性频率变换等场景的基础脉冲机制之一。

相比“连续波（CW）激光”：

- 平均功率未必更高，但峰值功率可显著提高；
- 脉冲持续时间更短，便于提升瞬时光强；
- 对材料加工或阈值效应过程更有效。

本条目实现的是“最小可运行速率方程 MVP”，强调机理可追踪而非器件级全细节仿真。

## R03

本任务要解决的问题：

- 输入：一组激光器简化参数（泵浦率、上能级寿命、增益系数、开关时刻、开关前后腔光子寿命等）；
- 过程：用耦合速率方程在时间网格上积分，得到反转数 `N(t)` 与光子密度 `Phi(t)`；
- 输出：脉冲峰值、峰值延迟、FWHM、脉冲面积等关键指标。

目标是用最小代码清晰复现“先储能，再突发释放”的 Q 开关行为。

## R04

本目录算法采用两变量速率方程：

- 反转方程：`dN/dt = R_p - N/tau_f - g*N*Phi`
- 光子方程：`dPhi/dt = (g*N - 1/tau_p(t))*Phi + beta_sp*N/tau_f`

其中 `tau_p(t)` 是分段函数：

- `t < t_switch` 时取 `tau_p_closed`（低 Q，高损耗）；
- `t >= t_switch` 时取 `tau_p_open`（高 Q，低损耗）。

数值积分采用显式欧拉法，配合非负截断保证 `N` 与 `Phi` 不出现非物理负值。

## R05

设时间步数为 `T = floor(t_end / dt) + 1`。

- 时间复杂度：`O(T)`，每步常数次代数运算；
- 空间复杂度：`O(T)`，存储 `t, tau_p, N, Phi` 四个数组。

在本 MVP 默认参数下，`T` 约为几十万量级，单机可快速完成。

## R06

`demo.py` 默认参数模拟一个主动 Q 开关单脉冲过程：

- 开关前：`tau_p_closed = 1 ns`，光子难以建立；
- 开关后：`tau_p_open = 10 ns`，腔内净增益提升；
- 在 `t_switch = 180 us` 打开 Q。

程序会打印：

- 开关前反转储能水平；
- 开关前最大光子密度（应接近 0）；
- 开关后脉冲峰值与峰值延迟；
- FWHM 与脉冲面积；
- 仿真末态。

## R10

正确性直觉：

1. 在低 Q 阶段，`1/tau_p` 很大，光子损耗主导，`Phi` 维持极低值；
2. 泵浦项 `R_p` 持续推动 `N` 增长，形成反转储能；
3. 开关后 `tau_p` 增大，损耗下降，`g*N` 可能超过 `1/tau_p`；
4. 一旦净增益转正，`Phi` 快速增长形成脉冲；
5. `g*N*Phi` 项会迅速耗尽反转，导致脉冲回落。

这正是 Q 开关“先积累再突发”的典型动力学链条。

## R11

数值稳定性与鲁棒性策略：

- 使用小时间步长（默认 `0.5 ns`）解析快脉冲段；
- 每步更新后对 `N, Phi` 做非负截断，避免非物理值累积；
- 输出前后区段统计并加入断言，及时暴露异常参数组合。

如果出现振荡或异常尖峰，可先减小 `dt`，再检查 `gain_coeff` 与 `tau_p` 的组合是否过激。

## R12

关键参数与调参方向：

- `pump_rate`：提高会增加储能，通常提升峰值；
- `t_switch`：越晚打开，通常反转更高，脉冲更强；
- `tau_p_closed`：越小越能压制开关前泄漏；
- `tau_p_open`：越大损耗越低，脉冲更易建立；
- `gain_coeff`：决定受激发射耦合强度；
- `dt`：控制精度与计算量权衡。

建议先固定默认值跑通，再单变量扫描观察指标变化。

## R14

常见失效模式：

1. `dt` 过大导致数值不稳定或脉冲形状畸变；
2. `gain_coeff` 过小或 `t_switch` 过早，导致不开脉冲；
3. `tau_p_closed` 过大导致开关前泄漏振荡；
4. 参数过激导致状态剧烈震荡并触发断言。

诊断方法：

- 先看 `max_photon_before_switch` 是否异常偏大；
- 再看 `peak_photon` 是否低于阈值；
- 最后检查 `peak_delay` 与 `fwhm` 是否超出预期量级。

## R17

`demo.py` MVP 完成项：

- 定义参数数据类 `QSwitchParams`；
- 构造 Q 开关分段光子寿命 `cavity_photon_lifetime`；
- 用 `simulate_q_switch` 积分得到 `N(t), Phi(t)`；
- 用 `extract_pulse_metrics` 提取峰值、延迟、FWHM、面积；
- 打印关键指标并执行 3 条自动断言（开关前抑制、峰值强度、时序正确）。

脚本无需交互输入，执行一次即给出完整统计。

## R18

`demo.py` 的源码级算法流程（8 步）：

1. `run_demo` 创建默认 `QSwitchParams`，定义泵浦、寿命、增益、开关时刻与时间步长。  
2. `simulate_q_switch` 生成时间网格 `t`，并调用 `cavity_photon_lifetime` 构建分段 `tau_p(t)`。  
3. 初始化状态数组 `N[0]=n0`、`Phi[0]=phi0`，作为速率方程初值。  
4. 在时间循环中计算 `dn_dt` 与 `dphi_dt`，对应反转积累/耗尽和光子增长/损耗。  
5. 用显式欧拉更新到下一步，并对 `N, Phi` 做 `>=0` 截断防止非物理负值。  
6. `extract_pulse_metrics` 在 `t >= t_switch` 区间内定位峰值，计算峰值时刻与延迟。  
7. 同函数调用 `estimate_fwhm`（半高宽）并用 `np.trapezoid` 计算脉冲面积，形成指标字典。  
8. `run_demo` 打印指标并执行断言，确认“开关前受抑制 + 开关后形成强脉冲”的 Q 开关行为。

本实现未把现成 Q 开关库当黑盒：速率方程、分段损耗、积分与指标提取都在源码里可直接追踪。
