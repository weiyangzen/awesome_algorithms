## R01

斜率优化 DP（Convex Hull Trick for DP）用于把一类形如
`dp[i] = min_j (A(j) * X(i) + B(j)) + C(i)`
的转移从 `O(n^2)` 降到 `O(n)` 或 `O(n log n)`。

本目录 MVP 选择了最常见的“前缀和 + 二次代价分段”模型：
- 给定非负序列 `a1..an` 和常数惩罚 `C`；
- 令 `S[i] = a1 + ... + ai`；
- 定义
  `dp[i] = min_{0<=j<i} { dp[j] + (S[i]-S[j])^2 + C }`。

当 `a[k] >= 0` 时，`S[i]` 单调不降，能够使用单调队列维护下凸壳，达到线性时间。

## R02

本条目中的输入/输出定义：
- 输入：
  - `values`: 非负整数序列；
  - `penalty`: 每开一个分段固定增加的代价 `C`。
- 输出：
  - `dp[n]`: 完成全部元素分段后的最小总代价；
  - `dp` 全表（长度 `n+1`）；
  - `prev` 决策数组（记录最优断点）；
  - 恢复后的最优分段区间。

索引约定：
- `dp[0]=0` 表示空前缀；
- 状态 `i` 对应前 `i` 个元素；
- 转移来自 `j<i`，含义是最后一段为 `(j+1..i)`。

## R03

核心代数展开：

原式：
`dp[i] = min_j { dp[j] + (S[i]-S[j])^2 + C }`

展开得：
`dp[i] = S[i]^2 + C + min_j { dp[j] + S[j]^2 - 2*S[i]*S[j] }`

把 `x = S[i]`，并把每个 `j` 视作一条直线：
- 斜率 `m_j = -2*S[j]`
- 截距 `b_j = dp[j] + S[j]^2`
- 线值 `y_j(x) = m_j*x + b_j`

则：
`dp[i] = S[i]^2 + C + min_j y_j(S[i])`

于是问题变成按顺序“加线 + 查最小值”的几何问题。

## R04

可用单调队列版 CHT 的前提：
- 查询点 `x=S[i]` 单调不降；
- 加入直线的斜率 `m_j=-2*S[j]` 单调不增（与上面等价）。

因此本 MVP 限定 `values` 为非负数。若存在负数，`S[i]` 可能非单调，就不再满足该线性版本假设，应改用 Li Chao Tree 或分治优化等更通用结构。

队列维护规则：
- 查询时：队首若不优于下一条线，则弹出队首；
- 插入时：若新线使队尾中间线永不可能最优，则弹出中间线。

## R05

`demo.py` 的核心数据结构：
- `Line(m, b, idx)`：
  - `m` 斜率，`b` 截距，`idx` 对应 DP 断点 `j`；
- `deque[Line]`：维护当前候选下凸壳；
- `prefix: np.ndarray[int64]`：前缀和；
- `dp: list[int]`：最优值表；
- `prev: list[int]`：最优转移来源（用于重建分段）。

## R06

正确性要点：
1. 最优子结构：`dp[i]` 的最优方案一定来自某个 `j<i` 的最优 `dp[j]` 加最后一段代价。
2. 代数等价：平方项展开后与原目标函数严格等价，无近似。
3. 下凸壳查询正确：对于固定 `x`，下凸壳返回 `m*x+b` 最小的线，恰好对应最优 `j`。
4. 单调弹队首正确：查询点单调增加时，被后继线永久压制的队首不可能再最优。
5. 单调删队尾正确：若中间线交点顺序被破坏，则该线对未来所有 `x` 都不可能成为最优。
6. 由 `prev` 回溯得到的分段，与 `dp[n]` 对应的最优转移链一致。

## R07

复杂度：
- `n = len(values)`。
- 单调 CHT 版：
  - 每条线最多入队/出队一次；
  - 总时间 `O(n)`，空间 `O(n)`。
- 暴力 DP 对照版：
  - 时间 `O(n^2)`，空间 `O(n)`。

## R08

边界与异常处理：
- `values` 非一维：报错；
- 存在负数：报错（因为破坏单调查询条件）；
- `penalty < 0`：报错（MVP 中不允许负惩罚）；
- 空序列：返回 `dp=[0]`，最小代价 `0`；
- 全 0 序列：允许，斜率相同分支由“同斜率保留更小截距”逻辑处理。

## R09

MVP 取舍说明：
- 选择最小可审计实现：只用 `numpy` + 标准库；
- 主算法手写，不依赖黑盒优化包；
- 保留暴力 DP 作为真值校验，避免“只跑快算法不验对错”；
- 不扩展到非单调 `x` 情况，避免把 Li Chao Tree 复杂度引入入门版本。

## R10

`demo.py` 主要函数：
- `validate_inputs(values, penalty)`：输入检查与标准化；
- `line_value(line, x)`：计算直线在 `x` 的取值；
- `is_redundant(l1, l2, l3)`：判断中间线是否应删；
- `slope_optimized_dp(values, penalty)`：主算法；
- `brute_force_dp(values, penalty)`：`O(n^2)` 对照实现；
- `reconstruct_segments(prev)`：回溯最优分段；
- `run_case(...)`：执行单案例、打印与断言；
- `main()`：运行全部内置案例。

## R15

方法对比：
- 朴素 DP：
  - 优点：直观；
  - 缺点：`O(n^2)`，中大规模慢。
- 单调 CHT（本实现）：
  - 优点：在单调前提下 `O(n)`；
  - 缺点：要求查询与斜率有序。
- Li Chao Tree：
  - 优点：支持无序查询；
  - 缺点：实现与常数更复杂，MVP 不引入。

## R16

典型应用：
- 分段最小化中的二次惩罚模型；
- 批处理/调度中的“切分 + 成本平方”目标；
- 竞赛与工程中的一维 DP 优化模板。

可迁移公式常见形态：
- `dp[i] = min_j { dp[j] + A[i]*B[j] + D[i] + E[j] }`。

## R18

`demo.py` 源码级算法流（8 步）：
1. `main` 构造固定测试集，逐个调用 `run_case`。
2. `run_case` 先执行 `slope_optimized_dp`，得到 `dp_cht` 与 `prev_cht`。
3. `slope_optimized_dp` 调用 `validate_inputs`，生成非负整数数组并计算前缀和 `S`。
4. 初始化 hull 为仅含 `j=0` 的直线 `m=0, b=0`，并置 `dp[0]=0`。
5. 对每个 `i=1..n`：在当前 `x=S[i]` 下，从队首弹掉不再最优的线，得到最优 `j`。
6. 用该 `j` 计算 `dp[i] = S[i]^2 + C + (m_j*S[i]+b_j)`，并写入 `prev[i]=j`。
7. 把由 `i` 生成的新线 `m=-2*S[i], b=dp[i]+S[i]^2` 入队；入队前用 `is_redundant` 反复删除无用队尾。
8. `run_case` 再跑 `brute_force_dp` 做真值对照，断言一致后用 `reconstruct_segments` 输出最优分段。
