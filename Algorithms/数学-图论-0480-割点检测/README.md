# 割点检测

- UID: `MATH-0480`
- 学科: `数学`
- 分类: `图论`
- 源序号: `480`
- 目标目录: `Algorithms/数学-图论-0480-割点检测`

## R01

割点（Articulation Point）是无向图中的关键顶点：
删除该顶点（及其关联边）后，图的连通分量数量会增加。

本目录实现的是 Tarjan 深度优先搜索（DFS）割点检测算法，属于图论中经典的线性时间算法。
它通过一次 DFS 同时维护每个节点的：
- `disc[u]`：首次被访问的时间戳
- `low[u]`：通过树边 + 至多一条返祖边可回溯到的最小时间戳

## R02

本 MVP 要解决的问题：

给定一个无向图 `G=(V,E)`（可包含多个连通分量），输出全部割点编号（升序）。

输入形式（`demo.py` 内部固定用例）：
- 顶点数 `n`
- 边列表 `edges = [(u,v), ...]`

输出形式：
- `List[int]` 类型的割点集合（升序）
- 每个用例附带 Tarjan 与暴力法对拍结果

## R03

选择 Tarjan 的原因：
- 与“删除每个点后做一次连通性检查”的暴力法相比，复杂度从 `O(V*(V+E))` 降到 `O(V+E)`；
- 算法逻辑可解释，适合作为图论基础算法条目；
- 不依赖图算法黑盒库，源码可直接审计。

## R04

割点判定条件（DFS 树视角）：

设 `u` 为 DFS 树上的节点，`v` 为 `u` 的一个子节点：

1. 若 `u` 是 DFS 根节点，且根有至少两个子树，则 `u` 是割点。  
2. 若 `u` 不是根，且存在子节点 `v` 满足 `low[v] >= disc[u]`，则 `u` 是割点。  

解释：
- `low[v] >= disc[u]` 表示以 `v` 为根的子树无法绕过 `u` 回到 `u` 的祖先；
- 因此删除 `u` 会把该子树与上层部分断开。

## R05

算法主流程：

1. 构建无向图邻接表（去重、忽略自环）。
2. 初始化数组 `disc/low/parent/is_ap`，全部节点初始未访问。
3. 对每个未访问节点执行 DFS（支持非连通图）。
4. DFS 中对树边递归并回溯更新 `low[u]`。
5. DFS 中对返祖边执行 `low[u] = min(low[u], disc[v])`。
6. 按 R04 的两个条件标记 `is_ap[u]`。
7. 扫描 `is_ap` 输出所有割点。

## R06

正确性要点：
- `disc[u]` 保证 DFS 访问顺序可比较；
- `low[u]` 精确表示 `u` 子树可回溯到的最早祖先访问时间；
- 根节点割点条件对应“子树数量决定根是否为连接枢纽”；
- 非根节点条件 `low[v] >= disc[u]` 对应“子树是否只能通过 `u` 与外界连通”。

`demo.py` 还用暴力删点法做逐用例对拍，作为实现层面的二次校验。

## R07

复杂度分析：

- 时间复杂度：`O(V+E)`  
  每个顶点访问一次，每条无向边在邻接表中最多被检查两次。
- 空间复杂度：`O(V+E)`  
  邻接表占 `O(V+E)`，辅助数组 `disc/low/parent/is_ap` 占 `O(V)`。

## R08

边界与异常处理：
- `n < 0` 或顶点编号越界：抛出 `ValueError`；
- `n=0` 时返回空割点集；
- 自环边 `(u,u)` 自动忽略（不影响割点判定）；
- 支持非连通图：从每个未访问起点发起 DFS；
- 非法边端点（非整数）直接报错，避免静默错误。

## R09

MVP 取舍说明：
- 仅使用 Python 标准库，实现尽量小而完整；
- 不引入 NetworkX 等图算法黑盒；
- 不做并行 DFS、动态图维护等高级优化；
- 增加暴力对拍而非追求极限性能，优先“可验证正确性”。

## R10

`demo.py` 函数职责：
- `_validate_graph`：检查 `n` 与边合法性。
- `build_adj_list`：构建邻接表（去重、忽略自环）。
- `articulation_points_tarjan`：Tarjan 割点主算法。
- `_components_count_with_removed`：删点后的连通分量计数（BFS）。
- `articulation_points_bruteforce`：暴力割点检测（对拍基线）。
- `run_case`：执行单个测试图并校验结果。
- `main`：组织内置测试用例并批量运行。

## R15

方法对比：
- 暴力删点法：
  - 优点：概念直观、容易验证；
  - 缺点：复杂度高，规模稍大就慢。
- Tarjan DFS 法：
  - 优点：线性复杂度、工程常用；
  - 缺点：实现细节（根节点判定、返祖边处理）更容易写错。

因此本条目同时保留两种实现：Tarjan 负责高效求解，暴力法负责正确性对拍。

## R16

典型应用场景：
- 网络鲁棒性分析（找单点故障节点）；
- 通信/交通拓扑中的关键枢纽识别；
- 图结构系统中的脆弱性评估；
- 与桥边（割边）检测配合做更全面的连通性诊断。

## R18

`demo.py` 源码级算法流程（8 步）：

1. `main` 构建四个固定图用例，逐个交给 `run_case`。  
2. `run_case` 先调用 `articulation_points_tarjan`，再调用 `articulation_points_bruteforce` 做对拍。  
3. `articulation_points_tarjan` 内先 `build_adj_list`，得到去重后的无向邻接表。  
4. 初始化 `disc/low/parent/is_ap` 与全局时间戳 `time`，并对每个未访问点启动 DFS（覆盖非连通图）。  
5. DFS 遇到树边 `u->v` 时递归访问 `v`，回溯后执行 `low[u] = min(low[u], low[v])`。  
6. DFS 遇到返祖边 `u->v (v != parent[u])` 时执行 `low[u] = min(low[u], disc[v])`。  
7. 在回溯阶段按两条规则判定割点：根节点 `children>1`，非根节点存在子节点使 `low[child] >= disc[u]`。  
8. 暴力法通过“删点后连通分量是否增加”复核 Tarjan 结果，`run_case` 断言三方一致后输出通过信息。
