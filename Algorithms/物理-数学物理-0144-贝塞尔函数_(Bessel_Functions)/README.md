# 贝塞尔函数 (Bessel Functions)

- UID: `PHYS-0144`
- 学科: `物理`
- 分类: `数学物理`
- 源序号: `144`
- 目标目录: `Algorithms/物理-数学物理-0144-贝塞尔函数_(Bessel_Functions)`

## R01

贝塞尔函数是圆柱/球对称问题中最核心的特殊函数之一，常见于波动方程、Helmholtz 方程、热传导与振动问题。

本条目关注第一类整数阶贝塞尔函数 `J_n(x)`，目标是给出一个可运行、可验证、非黑箱的最小实现：
- 用幂级数构造种子解；
- 用三项递推扩展到更高阶；
- 通过 ODE 残差、正交积分和库函数对照做一致性验证。

## R02

MVP 范围定义如下：
- 阶数：`n = 0..6`。
- 自变量区间：`x in [0, 20]`，均匀采样 `2001` 点。
- 核心实现：`demo.py` 显式实现 `J_n` 的级数与递推，不把 SciPy 当作唯一求值器。
- 参考用途：SciPy 仅用于对照误差（`special.jv`）和构造正交性测试的零点（`special.jn_zeros`）。

这样既保持算法透明，也便于自动化验收。

## R03

第一类贝塞尔函数满足微分方程：

`x^2 y'' + x y' + (x^2 - n^2) y = 0`

其中 `n` 为阶数（本 MVP 取非负整数）。对应的幂级数表示为：

`J_n(x) = sum_{m=0}^inf [(-1)^m / (m!(m+n)!)] * (x/2)^(2m+n)`

该表达在 `x=0` 附近数值稳定，并给出边界极限：
- `J_0(0)=1`
- `J_n(0)=0 (n>0)`

## R04

除幂级数外，`J_n` 还满足经典三项递推：

`J_{n+1}(x) = (2n/x) J_n(x) - J_{n-1}(x)`

据此可采用“种子 + 递推”策略：
1. 用级数先算 `J_0, J_1`。
2. 对 `n>=1` 用递推生成更高阶。

考虑到前向递推在“小 `x` + 高阶”时会放大误差，代码中使用了混合策略：
- 中大 `x` 区域采用递推；
- 小 `x` 区域回退到直接级数值，避免数值不稳定。

## R05

适用前提与假设：
- 只处理第一类贝塞尔函数 `J_n`。
- 阶数限定为非负整数。
- 关注实数自变量。
- 使用双精度浮点（`float64`）。

不在本 MVP 覆盖范围内：
- 第二类贝塞尔函数 `Y_n` 的奇点处理；
- 复参数/复自变量；
- 超高阶、超大区间下的统一渐近展开与误差控制。

## R06

`demo.py` 的输入输出约定：
- 输入：脚本内置参数（`max_order=6`、`x` 网格、正交性模式数等），无命令行参数。
- 输出：
1. 每阶对照误差表（相对 SciPy 参考）。
2. 递推残差。
3. Bessel ODE 有限差分残差。
4. 基于 `J_0` 零点的正交积分 Gram 矩阵与误差。
5. 阈值检查与最终 `Validation: PASS/FAIL`。

运行无需交互输入。

## R07

算法主流程（高层）：
1. 构建 `x` 网格，设置最大阶数。
2. 用幂级数递推公式计算 `J_0(x), J_1(x)`。
3. 通过三项递推生成 `J_2..J_6`。
4. 在小 `x` 区间用级数结果覆盖，抑制前向递推误差累积。
5. 以 `special.jv` 作为参考，计算逐阶误差统计。
6. 验证递推恒等式残差。
7. 验证 ODE 残差（数值微分）。
8. 用 `J_0` 零点做加权正交积分，检查正交性和对角项理论值。
9. 汇总阈值判断并输出 PASS/FAIL。

## R08

复杂度分析（`N` 为阶数上限，`M` 为 `x` 采样点数，`K` 为级数项上限）：
- 级数计算（单阶）：`O(K*M)`。
- 递推扩阶：`O(N*M)`。
- 对照误差：`O(N*M)`。
- ODE 残差（梯度两次）：`O(M)` 每个被测阶。
- 正交性 Gram（模式数 `p`）：`O(p^2*M)`。

本实现参数规模很小（`N=6`, `M=2001`, `p=4`），在普通 CPU 上可瞬时完成。

## R09

数值稳定性处理：
- 幂级数按“相邻项递推”更新，避免直接用大阶乘产生溢出。
- `x=0` 处强制写入解析极限 `J_0=1, J_{n>0}=0`。
- 对 `|x|` 很小区间使用级数覆盖递推结果，避免 `2n/x` 导致病态放大。
- ODE 残差只在内区间 `[0.25, 19.75]` 统计，以减弱边界差分误差与奇点附近噪声。

## R12

主要输出字段说明：
- `order`：阶数 `n`。
- `max_abs_error_vs_scipy`：该阶在全网格上的最大绝对误差。
- `rms_error_vs_scipy`：该阶均方根误差。
- `max_recurrence_residual`：递推公式残差上界。
- `max |x^2 y'' + x y' + (x^2-n^2) y|`：ODE 残差上界。
- `Gram matrix I_mk`：加权正交积分矩阵。
- `max |offdiag(I)|`：正交性非对角项最大值。
- `max relative diag error`：对角项与理论值的最大相对误差。

## R13

正确性验证由 5 条阈值联合判定：
1. `max abs error vs scipy < 2e-8`
2. `max recurrence residual < 1e-11`
3. `max ODE residual (FD) < 5e-3`
4. `orthogonality offdiag max < 1e-5`
5. `orthogonality diag relative error < 5e-4`

全部满足则输出 `Validation: PASS`；任一失败则退出码非 0，输出 `Validation: FAIL`。

## R14

当前实现局限：
- 阶数较高时仍未引入 Miller 反向递推或统一渐近展开。
- ODE 校验基于有限差分，不是严格符号验证。
- 正交性仅示范 `n=0` 的前 4 个零点。
- 未覆盖 `Y_n`、Hankel 函数及复参数情况。

## R16

典型应用场景：

- 圆形膜振动本征模与本征频率分析。

## R17

方案对比：
- 纯库函数调用：开发最快，但难以展示算法路径与误差来源。
- 纯级数全域计算：在大 `x` 可能收敛慢、项数多。
- 纯前向递推：在小 `x` + 高阶容易失稳。

本 MVP 采用“级数种子 + 前向递推 + 小 `x` 级数覆盖 + 多重验证”折中方案，兼顾：
- 可读性（公式到代码映射直接）
- 稳定性（显式补丁）
- 可审计性（误差、方程、正交三重证据）

## R18

`demo.py` 源码级流程拆解（9 步）：
1. `bessel_j_series_seed` 按幂级数项间递推计算给定阶数 `J_n(x)`，避免直接计算大阶乘比值。
2. `bessel_j_upward` 先求 `J_0, J_1`，再用三项递推得到更高阶 `J_n`。
3. 在 `bessel_j_upward` 内对小 `x` 区间执行“级数覆盖”修正，处理前向递推病态区域。
4. `run_mvp` 生成 `x` 网格并得到 `J_0..J_6` 的数值结果矩阵。
5. `run_mvp` 通过 `special.jv` 形成参考矩阵，计算逐阶绝对误差。
6. `recurrence_residuals` 评估递推公式离散残差，验证内部一致性。
7. `ode_residual` 用数值微分评估 Bessel ODE 残差，验证方程层正确性。
8. `orthogonality_check` 用 `special.jn_zeros` 构造基函数并计算 Gram 矩阵，检查正交性与理论对角项。
9. `main` 汇总误差表与阈值，输出 `Validation: PASS/FAIL`，供自动验收直接使用。

该流程保证即使使用了第三方库做参考，核心算法路径（求值、递推、稳定化、验证）仍在源码中完整展开。
