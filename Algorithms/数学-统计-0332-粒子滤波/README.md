## R01

粒子滤波（Particle Filter, PF）是一类用“带权随机样本”近似后验分布的序贯蒙特卡洛（SMC）方法，常用于非线性、非高斯状态空间模型的在线状态估计。

在离散时间下，目标是递推估计：

- 状态转移：`x_t ~ p(x_t | x_{t-1})`
- 观测模型：`y_t ~ p(y_t | x_t)`
- 目标后验：`p(x_t | y_1:t)`

它用 `N` 个粒子 `x_t^{(i)}` 与权重 `w_t^{(i)}` 近似：

`p(x_t|y_1:t) ≈ sum_i w_t^{(i)} delta(x_t - x_t^{(i)})`

## R02

发展脉络可简述为：

- 1950s-1970s：蒙特卡洛思想成熟，但序列贝叶斯估计主要由 Kalman 家族主导。
- 1990s：Gordon 等提出 Bootstrap Filter，使 PF 在工程界广泛落地。
- 2000s 以后：出现 Rao-Blackwellized PF、Auxiliary PF、SIR 等变体，提升稳定性与效率。

今天 PF 是“无法使用线性高斯闭式解”场景中的主力在线推断方法之一。

## R03

它主要解决的问题是：

- Kalman Filter 假设线性高斯，很多真实系统（目标跟踪、机器人定位、非线性金融时序）不满足；
- 扩展卡尔曼 / 无迹卡尔曼在强非线性、重尾噪声场景可能偏差明显；
- 我们需要在每个时刻保留后验分布形状（可能多峰），而不只是均值与协方差。

PF 通过采样近似整条后验，天然支持非线性与非高斯。

## R04

Bootstrap 粒子滤波（本目录 `demo.py` 实现）核心流程：

1. 预测：
- 对每个粒子采样 `x_t^{(i)} ~ p(x_t | x_{t-1}^{(i)})`

2. 更新权重：
- `w_t^{(i)} ∝ w_{t-1}^{(i)} * p(y_t | x_t^{(i)})`
- 归一化后满足 `sum_i w_t^{(i)} = 1`

3. 退化检测：
- 有效样本数 `ESS = 1 / sum_i (w_t^{(i)})^2`
- 当 `ESS` 过低，说明少数粒子垄断权重

4. 重采样（Systematic Resampling）：
- 按权重抽取新粒子集并将权重重置为均匀 `1/N`

5. 状态估计：
- 后验均值：`hat{x}_t = sum_i w_t^{(i)} x_t^{(i)}`

## R05

设粒子数 `N`、序列长度 `T`：

- 时间复杂度：`O(T*N)`（每个时刻对每个粒子做常数级运算）
- 空间复杂度：`O(N)`（在线估计）或 `O(T*N)`（若存完整轨迹）
- 重采样步骤通常也是 `O(N)`

PF 的主要成本由粒子数决定，精度与算力需要权衡。

## R06

`demo.py` 使用经典非线性状态空间模型做最小实验：

- 状态方程：`x_t = 0.5x_{t-1} + 25x_{t-1}/(1+x_{t-1}^2) + 8cos(1.2t) + v_t`
- 观测方程：`y_t = x_t^2 / 20 + n_t`
- 其中 `v_t ~ N(0, q)`，`n_t ~ N(0, r)`

该模型非线性明显、观测非一一映射（平方项会造成歧义），很适合展示 PF 的必要性。

## R07

优点：

- 处理非线性、非高斯模型；
- 可近似任意形状后验（包括多峰）；
- 在线递推，适合流式数据。

局限：

- 粒子退化与样本贫化（特别是高维状态）；
- 粒子数不足时方差大、估计抖动；
- 计算开销通常高于卡尔曼类闭式方法。

## R08

建议前置知识：

- 概率论：贝叶斯公式、条件概率、概率密度；
- 蒙特卡洛方法：采样、重要性权重；
- 状态空间模型（隐变量时序模型）；
- Python/Numpy 向量化基础。

## R09

适用场景：

- 机器人定位与 SLAM 中的位姿估计；
- 雷达/视觉目标跟踪；
- 非线性系统状态监测与故障诊断；
- 强非线性时序的在线隐变量估计。

不太适用：

- 维度极高且实时预算很紧（PF 可能需要过多粒子）；
- 模型本身近似线性高斯（Kalman 更高效）。

## R10

实现正确性可用以下检查：

1. 每时刻权重和应为 1；
2. `ESS` 应在 `[1, N]`；
3. 重采样后权重应回到均匀分布；
4. 估计序列无 `NaN/Inf`；
5. 在固定随机种子下结果可复现；
6. 合理粒子数下 RMSE 应显著低于随机猜测。

`demo.py` 内置了前 5 项断言，并打印 RMSE 做效果参考。

## R11

数值稳定性要点：

- 权重更新用对数域：`log w` 累加后再减去最大值做 `exp`；
- 归一化分母设置最小阈值 `eps` 防止除零；
- 似然计算使用高斯对数密度，避免直接连乘下溢；
- 定期重采样抑制权重塌缩。

## R12

关键参数与调优经验：

- 粒子数 `N`：越大越稳但越慢，先从 `500~2000` 试起；
- 重采样阈值 `ess_ratio`：常用 `0.3~0.7`，本实现用 `0.5`；
- 过程噪声 `q` 与观测噪声 `r`：决定先验动态与观测信任度；
- 初始分布：过窄会导致初期丢真值，过宽会降低收敛速度。

建议先固定模型噪声，再逐步增加 `N` 观察 RMSE 收敛趋势。

## R13

理论性质（针对 bootstrap PF）：

- 在合适条件下，当 `N -> ∞`，粒子近似收敛到真实后验；
- 有限粒子下是随机近似，误差通常随 `N` 增大而下降（方差级别约 `O(1/N)`）；
- 算法是“无偏估计某些积分量 + 有方差”的近似推断，不是精确闭式解。

因此 PF 的核心不是“找到唯一最优状态”，而是“近似完整后验分布”。

## R14

常见失效模式与应对：

- 权重瞬间塌缩（单粒子权重接近 1）。
  对策：提升 `N`、提高重采样频率、优化 proposal。
- 样本贫化（重采样后多样性不足）。
  对策：重采样后加入轻微抖动（jitter）或改进提议分布。
- 噪声参数失配导致系统性偏差。
  对策：离线估计 `q/r`，或在验证集上网格搜索。
- 高维状态性能骤降。
  对策：分块建模、Rao-Blackwell 化或改用更合适滤波框架。

## R15

工程落地建议：

- 将“模型函数（转移/观测）”与“滤波器框架”解耦，便于替换业务模型；
- 输出 ESS、重采样次数、RMSE 等监控指标；
- 保留固定随机种子回归测试，防止重构后数值漂移；
- 若需要更高性能，可把核心循环迁移到 `numba` / `PyTorch`。

## R16

与相关算法对比：

- Kalman Filter：线性高斯下最优且高效；PF 更通用但更耗算力。
- EKF/UKF：用局部线性化或 sigma 点近似；PF 用采样直接近似后验。
- MCMC：偏离线后验采样；PF 是在线序贯推断。

一句话：PF 牺牲部分计算效率，换取对复杂非线性/非高斯问题的鲁棒适应能力。

## R18

`demo.py` 的源码级算法流（8 步）：

1. `simulate_nonlinear_system` 按给定 `q/r` 采样真值状态和观测，构造可复现实验数据。  
2. `particle_filter` 初始化粒子 `x_0^{(i)}` 与均匀权重 `w_0^{(i)}=1/N`。  
3. 每个时刻先做预测：把每个粒子代入状态转移方程并加过程噪声，得到先验粒子。  
4. 计算每个粒子的观测对数似然 `log p(y_t|x_t^{(i)})`，与旧权重相加得到 `logw_raw`。  
5. 用“减最大值 + exp + 归一化”将 `logw_raw` 转为稳定权重，并累计近似 `log-evidence`。  
6. 根据当前权重计算后验均值/方差与 `ESS`，记录在线估计轨迹。  
7. 若 `ESS < ess_ratio * N`，执行系统重采样：按累计权重等距抽样粒子，并将权重重置为均匀。  
8. `main` 汇总 RMSE、重采样次数、平均 ESS，打印前若干时刻明细并执行断言形成闭环验证。  

本实现没有调用黑盒 PF 库函数，流程可直接映射到教科书中的 SIR/Bootstrap Particle Filter。
