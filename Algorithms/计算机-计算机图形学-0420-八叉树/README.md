## R01

八叉树（Octree）是三维空间的层次划分结构。  
它把一个立方体空间递归分成 8 个子立方体（八个象限体），直到达到最大深度或叶子容量限制。  
在计算机图形学中，八叉树常用于加速空间查询、碰撞检测、体素数据索引和光线追踪前的候选对象筛选。

## R02

本题 MVP 的计算目标：
- 输入：三维点云 `points (N,3)` 与若干轴对齐包围盒查询 `AABB`。
- 构建：基于点云建立八叉树，参数包含 `max_depth` 与 `bucket_size`。
- 输出：
  - 八叉树结构统计（节点数、叶子数、叶子负载分布、深度分布）；
  - 每个查询命中的点索引集合；
  - 八叉树查询与暴力查询一致性校验；
  - 访问节点数、叶子点测试数等性能指标。

`demo.py` 使用固定随机种子，无需交互输入。

## R03

空间划分与路由规则：

给定当前节点中心 `c=(cx,cy,cz)` 与点 `p=(x,y,z)`，子节点编号由 3 个二进制位决定：
- `bx = 1` 当 `x >= cx`，否则 `0`
- `by = 1` 当 `y >= cy`，否则 `0`
- `bz = 1` 当 `z >= cz`，否则 `0`
- 子编号 `child_id = bx + 2*by + 4*bz`，范围 `0..7`

查询时，若节点包围盒与查询盒不相交，则整棵子树可直接剪枝。

## R04

MVP 执行流程（8 阶段）：
1. 生成固定点云样本（簇状 + 均匀噪声）；
2. 计算全局立方体根边界（包住全部点）；
3. 逐点插入八叉树，叶子满载则细分；
4. 细分后将旧点重新分发到 8 个子节点；
5. 对每个查询盒执行递归范围查询；
6. 同时做暴力全扫描查询作为真值；
7. 断言两种查询返回索引集合完全一致；
8. 输出树结构统计与查询剪枝效果。

## R05

核心数据结构：
- `AABB(min_corner, max_corner)`：轴对齐包围盒，提供 `contains_point` 与 `intersects`。
- `OctreeNode`：
  - `bounds`：节点空间边界；
  - `points/indices`：叶子保存的点与原始索引；
  - `children`：8 个子节点（内部节点）；
  - `depth/max_depth/bucket_size`：细分控制参数。
- `QueryStats`：记录单次查询的 `visited_nodes`、`visited_leaves`、`point_tests`。

## R06

`demo.py` 中的函数职责：
- `build_octree`：从点云构建根节点并完成插入；
- `OctreeNode.insert`：插入点，必要时触发 `_subdivide`；
- `OctreeNode.query_range`：递归范围查询并统计访问量；
- `brute_force_range_query`：全量扫描基准；
- `build_demo_points`：构造可复现点云；
- `main`：组织实验、断言正确性、打印表格。

## R07

复杂度（`N` 点，树高 `H`，命中点数 `k`）：
- 建树平均时间：`O(N * H)`，均匀分布常近似 `O(N log N)`；
- 建树最坏时间：`O(N^2)`（极端退化分布，且不断落入同一路径）；
- 范围查询平均：`O(log N + k)`（受数据分布影响）；
- 最坏查询：`O(N)`；
- 空间复杂度：`O(N + M)`，`M` 为内部节点数量。

## R08

正确性依赖的不变量：
1. 任一点只存储于一条根到叶的唯一路径中；
2. 子节点空间并集覆盖父节点空间；
3. 任一叶子要么 `len(points) <= bucket_size`，要么 `depth == max_depth`；
4. 查询时只有与查询盒相交的节点会被访问；
5. 返回索引集合与暴力扫描结果一致。

## R09

实现细节与数值注意：
- 根节点使用“立方体边界”而非长方体，保证标准八分；
- 插入路由用中心比较（`>=`），处理边界点时有稳定归属；
- 子边界使用闭区间判断，边界点不丢失；
- 对非有限数值、非法形状、非法参数进行显式校验并抛错。

## R10

实验配置：
- 点数：`N = 2000`；
- 数据分布：4 个高斯簇 + 400 个均匀噪声点；
- 构建参数：`max_depth = 7`，`bucket_size = 24`；
- 查询盒数量：4 个（中心盒、x 正半轴薄片、局部簇探测、角落稀疏区）；
- 对每个查询都输出：命中数、访问节点、叶子访问数、点测试削减比例。

## R12

输出指标解释：
- `node_count / leaf_count`：树结构规模；
- `avg_leaf_load`：叶子平均装载点数；
- `visited_nodes`：单次查询中实际触达的节点数；
- `point_tests_octree`：叶子内逐点判定次数；
- `point_tests_bruteforce`：暴力法固定为 `N`；
- `prune_ratio`：`1 - point_tests_octree / N`，越大表示剪枝越有效。

## R13

自动校验逻辑：
1. 每个查询比较 `sorted(octree_hits)` 与 `sorted(brute_force_hits)`；
2. 若任一查询不一致，抛出 `AssertionError`；
3. 若全通过，打印 `All checks passed.`。

该断言保证不仅“能运行”，而且“结果正确”。

## R14

边界与异常处理：
- `build_octree` 对 `points` 维度、空输入、非有限数值进行校验；
- `AABB` 构造时校验 `min_corner <= max_corner`；
- `max_depth < 0` 或 `bucket_size <= 0` 直接抛错；
- 插入若发现点不在根包围盒，抛 `RuntimeError`（理论不应发生）。

## R15

八叉树在图形学中的典型用途：
- 大规模场景的近邻和区域检索；
- 碰撞检测中的候选对裁剪（broad-phase）；
- 稀疏体素/体渲染数据组织；
- 光线追踪中的空间加速预筛。

## R16

本 MVP 的范围限制：
- 仅处理点数据，不含三角网格/体素块对象；
- 查询类型只实现 `AABB` 范围查询；
- 未实现并行构建与并行查询；
- 未做内存池、SIMD、磁盘分页等工程级优化。

## R17

可扩展方向：
1. 支持球查询、kNN 查询和射线遍历；
2. 节点存储对象包围盒而非单点；
3. 增加并行批量查询（多线程或 GPU）；
4. 引入 Morton code / 线性八叉树以提升缓存局部性；
5. 支持动态更新（插入/删除）与重平衡策略。

## R18

`demo.py` 源码级算法链路（9 步）：
1. `main()` 调用 `build_demo_points` 生成固定随机点云。  
2. `build_octree()` 计算全局立方体边界，创建根节点 `OctreeNode`。  
3. 对每个点执行 `root.insert(point, idx)`；当叶子超容量且未达最大深度时触发 `_subdivide()`。  
4. `_subdivide()` 构造 8 个子 `AABB`，并把叶子已有点重新路由到对应子节点。  
5. `main()` 构造 4 个查询 `AABB`，逐个发起 `root.query_range(...)`。  
6. `query_range` 先做 `bounds.intersects(query)`，不相交则整子树剪枝返回。  
7. 对相交叶子逐点执行 `query.contains_point`，累积命中索引和 `QueryStats.point_tests`。  
8. 同一查询再调用 `brute_force_range_query` 全量扫描，断言与八叉树命中集合一致。  
9. 所有查询通过后，输出树统计、深度分布和查询对照表，打印 `All checks passed.`。
