## R01

大整数乘法关注的是“超出固定机器字长的整数”相乘问题。这里采用的核心算法是 Karatsuba 分治乘法。

- 输入: 两个十进制整数字符串（可带符号）。
- 输出: 乘积的十进制字符串。
- 分治思想:
  1. 将每个大整数在中间位拆成高位和低位两部分。
  2. 用 `ac`、`bd`、`(a+b)(c+d)` 三次递归乘法替代朴素四次乘法。
  3. 通过移位与加减重组得到最终结果。

## R02

- 主要提出者: Anatoly Karatsuba 与 Yuri Ofman。
- 时间: 1962 年发表相关论文，首次打破传统乘法 `O(n^2)` 复杂度壁垒。
- 历史意义: 开启了“快速大整数乘法”研究路线，后续 Toom-Cook、FFT/NTT 乘法都可视为这一方向的延展。

## R03

朴素按位乘法在 `n` 位输入上需要 `O(n^2)` 级别操作。对于密码学、计算代数、任意精度计算器等场景，位数上千甚至上万，`n^2` 成本很快成为瓶颈。

Karatsuba 的价值是: 在保持精确整数运算前提下，把递归复杂度降为 `O(n^{log2(3)}) ≈ O(n^1.585)`，在中大规模位数时通常优于朴素法。

## R04

设输入长度为 `n`（按十进制位计），Karatsuba 的递推关系:

- `T(n) = 3T(n/2) + O(n)`
- 由主定理得 `T(n) = O(n^{log2(3)}) ≈ O(n^1.585)`

对比:

- 朴素乘法: `O(n^2)`
- Karatsuba: `O(n^1.585)`（渐进更优）

## R05

空间复杂度主要来自递归与中间字符串:

- 递归深度: `O(log n)`
- 每层存在若干中间和与差，累计额外空间通常可记为 `O(n)` 到 `O(n log n)`（与实现细节相关）

本目录 MVP 为可读性优先，采用字符串运算，未做极限内存优化。

## R06

示例: `1234 × 5678`

1. 取 `m=2`，拆分:
   - `1234 = 12 * 10^2 + 34`
   - `5678 = 56 * 10^2 + 78`
2. 计算三项:
   - `z2 = 12 * 56 = 672`
   - `z0 = 34 * 78 = 2652`
   - `z1 = (12+34)*(56+78) = 46*134 = 6164`
3. 中间项:
   - `mid = z1 - z2 - z0 = 6164 - 672 - 2652 = 2840`
4. 重组:
   - `z2*10^(2m) + mid*10^m + z0`
   - `672*10^4 + 2840*10^2 + 2652 = 7006652`

结果与普通乘法一致。

## R07

算法意义:

- 理论层: 是分治优化乘法的里程碑。
- 工程层: 常作为大整数库中的中规模乘法策略（再往上可切换 Toom/FFT）。
- 教学层: 展示“用代数恒等式降低递归分支数”的经典方法。

## R08

前置知识:

- 进位制整数表示与按位加减。
- 分治递归与递推复杂度分析。
- Karatsuba 恒等式:
  - `(a*B^m+b)(c*B^m+d) = ac*B^(2m) + ((a+b)(c+d)-ac-bd)*B^m + bd`

## R09

适用条件与边界:

- 输入应为合法整数字符串（可含 `+/-` 符号）。
- 当位数较小，朴素法常数项更低，通常更快。
- 实践中常设置阈值 `threshold`:
  - 小于阈值走朴素乘法。
  - 大于阈值走 Karatsuba。

## R10

正确性要点:

1. 基础情形: 小规模直接用朴素乘法，结果正确。
2. 归纳假设: 递归调用能正确算出子问题乘积。
3. 代数重组:
   - `z2 = ac`
   - `z0 = bd`
   - `middle = (a+b)(c+d)-ac-bd = ad+bc`
4. 合成式 `ac*B^(2m) + (ad+bc)*B^m + bd` 与原乘积完全等价。
5. 因此递归终止后返回值正确。

## R11

数值稳定性与误差传播:

- 本题是整数精确计算，不涉及浮点舍入误差。
- 主要风险来自实现细节:
  - 前导零处理错误会破坏比较与减法前提。
  - 符号处理错误会导致结果正负号反转。
  - 借位/进位实现错误会导致局部错误扩散。

## R12

工程成本（超越大 O）:

- 字符串切片与拼接有常数开销。
- 递归函数调用会引入栈与调度成本。
- 阈值选取影响明显:
  - 阈值过小，递归开销偏大。
  - 阈值过大，朴素段过长，失去分治优势。
- 因此真实工程里会做平台相关基准来调阈值。

## R13

- 近似比/概率保证: N/A。
- 原因: Karatsuba 是确定性精确算法，不是随机近似算法。

## R14

鲁棒性与常见失效模式:

- 非法输入（空串、非数字字符）应明确抛错。
- `sub_abs(a,b)` 的前置条件 `a>=b` 若被破坏会直接出错。
- 极大规模输入可能受 Python 递归深度与内存限制影响。
- 未规范化 `-0` 可能造成符号不一致问题。

## R15

实现建议:

- 先把输入统一为“符号 + 规范化绝对值字符串”。
- 把加法、减法、比较、移位拆成独立函数，便于单测。
- 使用混合策略（Karatsuba + 朴素阈值）平衡常数与渐进复杂度。
- 在回归测试中使用 Python `int` 仅作 oracle，对照验证正确性。

## R16

前驱与后继关系:

- 前驱:
  - 竖式（学校）乘法 `O(n^2)`
- 后继:
  - Toom-Cook（更高阶分块）
  - Schönhage-Strassen / FFT 类快速乘法
  - NTT 乘法（模数域卷积）

在通用大整数实现中，常见策略是“按位数分段切换不同乘法内核”。

## R17

本目录 `demo.py` 的 MVP 设计:

  - 随机回归（与 Python `int` 乘法对照）。

## R18

源码级流程拆解（对应 `demo.py`）:

1. `main()` 依次执行固定样例、随机回归、性能对照，确保正确性和可运行性。
2. `multiply_bigint` 先调用 `parse_signed_decimal` 解析符号并规范化绝对值，再进入绝对值乘法。
3. `karatsuba_mul_abs` 接收两个非负字符串；若位数不大于阈值，直接走 `schoolbook_mul_abs`。
4. 否则把两个数左侧补零到等长偶数位，并按中点拆成 `high/low` 两半。
5. 递归计算 `z0 = low*low`、`z2 = high*high`、`z1 = (high+low)*(high+low)` 三项。
6. 通过 `middle = z1 - z2 - z0` 得到交叉项 `ad+bc`，其中减法由 `sub_abs` 完成。
7. 用十进制移位重组: `z2*10^(2m) + middle*10^m + z0`，通过 `add_abs` 合并并去前导零。
8. 回到 `multiply_bigint` 恢复结果符号并返回，测试代码再与 Python `int` oracle 对照断言。
