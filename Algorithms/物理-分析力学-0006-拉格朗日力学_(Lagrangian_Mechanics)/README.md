# 拉格朗日力学 (Lagrangian Mechanics)

- UID: `PHYS-0006`
- 学科: `物理`
- 分类: `分析力学`
- 源序号: `6`
- 目标目录: `Algorithms/物理-分析力学-0006-拉格朗日力学_(Lagrangian_Mechanics)`

## R01

问题定义：用拉格朗日力学建立并求解一个最小可运行的动力学系统。

本条目选取单摆（one-DOF pendulum）作为载体，用广义坐标 `theta` 建模，目标是：
- 从 `L = T - V` 出发给出欧拉-拉格朗日方程；
- 数值积分得到 `theta(t), omega(t)`；
- 验证守恒量与方程残差，形成可复现实验。

## R02

理论背景：拉格朗日力学将“受力分解”转化为“标量函数变分”。

关键优势：
- 对广义坐标友好，避免笛卡尔坐标下的约束反力显式求解；
- 对多自由度系统可系统化扩展；
- 与哈密顿力学、最小作用量原理天然衔接。

在工程和科研中，这一框架常用于机器人关节动力学、轨道动力学、分子振动等。

## R03

本 MVP 的计算任务：
1. 定义单摆拉格朗日量 `L(theta, omega)`；
2. 推导并实现欧拉-拉格朗日方程对应的 ODE；
3. 使用高精度 ODE 积分器求时域轨迹；
4. 输出可解释指标：
   - 能量漂移；
   - 周期与小角近似周期偏差；
   - 欧拉-拉格朗日离散残差（RMS 和 max）。

## R04

建模假设（MVP 级别）：
- 质点质量集中于摆端，摆杆刚性且无质量；
- 重力场均匀，重力加速度常数 `g`；
- 默认无阻尼（`damping=0`），系统近似保守；
- 运动平面固定，只有一个广义坐标 `theta`。

该假设适合教学和算法演示，不包含空气阻力、轴承摩擦、三维摆动耦合等复杂因素。

## R05

核心方程（`demo.py` 已实现）：

- 动能：`T = 0.5 * m * l^2 * omega^2`
- 势能：`V = m * g * l * (1 - cos(theta))`
- 拉格朗日量：`L = T - V`

欧拉-拉格朗日方程（含广义非保守力 `Q`）：
`d/dt(∂L/∂omega) - ∂L/∂theta = Q`

在本模型中 `Q = -c * omega`，得到：
`theta_ddot = -(g/l) * sin(theta) - (c/(m*l^2)) * omega`

默认 `c=0` 时为经典无阻尼单摆。

## R06

数值算法流程：
1. 设状态 `y = [theta, omega]`；
2. 在 `rhs` 中按上式计算 `alpha = theta_ddot`；
3. 调用 `solve_ivp(method="DOP853")` 做自适应积分；
4. 后处理计算 `L(t)` 与 `E(t)`；
5. 通过零点上穿估计仿真周期；
6. 用离散梯度计算欧拉-拉格朗日残差并统计误差量级。

## R07

复杂度分析（设采样点为 `N`）：
- 时间复杂度：`O(N)`，每个采样点只进行常数次三角函数和代数运算；
- 空间复杂度：`O(N)`，保存 `t/theta/omega` 及诊断数组。

瓶颈主要来自积分精度要求和采样密度，而不是线性代数求解。

## R08

数值稳定性策略：
- 使用 `DOP853`（8 阶显式 Runge-Kutta）并设置较紧容差：`rtol=1e-9, atol=1e-11`；
- 利用能量漂移作为主质量指标（保守系统应近似守恒）；
- 增加欧拉-拉格朗日残差诊断，避免“轨迹看起来合理但方程不闭合”的假稳定。

## R09

适用场景：
- 分析力学课程中从 `L` 到动力学方程的完整演示；
- 作为多自由度拉格朗日系统（双摆、机械臂）的起始模板；
- 数值积分器与物理守恒诊断的联合 sanity check。

不适用场景：
- 强阻尼、冲击、干摩擦主导系统；
- 高精度实验反演（需更完整误差模型和参数辨识）。

## R10

正确性检查点（脚本中已落地）：
1. ODE 必须求解成功，否则抛 `RuntimeError`；
2. `energy_rel_drift <= 5e-4`；
3. `period_rel_error <= 3%`（与小角周期的偏差上限）；
4. `el_residual_rms <= 3e-2`（离散欧拉-拉格朗日残差）。

这 4 个检查联合约束“数值正确 + 物理一致”。

## R11

默认参数（`PendulumParams`）：
- `mass=1.0 kg`
- `length=1.0 m`
- `gravity=9.81 m/s^2`
- `damping=0.0`
- `theta0=0.45 rad`（约 `25.783°`）
- `omega0=0.0 rad/s`
- `t_end=12.0 s`
- `num_steps=2000`

参数设计意图：
- 初角不太小，保留轻度非线性；
- 仍保持与小角近似周期在可比较范围。

解读：

## R13

理论与验证边界：
- 连续系统在 `c=0` 下理论上守恒机械能；
- 数值求解无法给出全局严格解析证明，但可通过误差控制与诊断量验证；
- 因此本条目属于“可追踪的数值物理验证”，而非符号证明系统。

## R14

常见失败模式与修复：
- 失败：步长过粗或容差过松导致能量漂移偏大。
  - 修复：收紧 `rtol/atol` 或提高采样密度。
- 失败：初值过大（接近倒立）导致周期估计不稳。
  - 修复：减小 `theta0` 或延长仿真时间增加过零样本。
- 失败：引入阻尼后仍用“能量守恒阈值”判断。
  - 修复：有阻尼时应改为单调耗散一致性检查。

## R15

工程实践建议：
- 把“模型方程残差”与“守恒漂移”同时记录，避免单一指标误判；
- 周期估计优先使用插值后的过零点，避免直接按离散采样计数；
- 扩展到多自由度时，先保证每个广义坐标的 `∂L/∂q` 与 `∂L/∂qdot` 计算一致，再集成求解器。

## R16

方法脉络与扩展方向：

- 从拉格朗日形式扩展到哈密顿正则方程；
- 引入约束可用拉格朗日乘子；

## R18

`demo.py` 源码级算法流拆解（8 步，避免黑盒）：
1. `PendulumParams` 固化物理参数与仿真配置，保证结果可复现。
2. `lagrangian` 与 `mechanical_energy` 显式计算 `L=T-V` 与 `E=T+V`，把物理量定义落地到代码。
3. `rhs` 直接实现欧拉-拉格朗日推导后的二阶方程，并转为一阶系统 `[theta_dot, omega_dot]`。
4. `simulate` 组织 `t_eval`、初值 `y0`，调用 `solve_ivp(DOP853)` 进入自适应时间推进。
5. 在 DOP853 内部，每个候选步会进行多阶段斜率评估（Runge-Kutta stage），生成高阶解与嵌入误差估计。
6. 积分器根据局部截断误差与 `rtol/atol` 自动接受/拒绝步长并调整下一步步长，再用致密插值回填 `t_eval`。
7. 轨迹返回后，代码计算 `E(t)`、周期估计、以及离散 `d/dt(∂L/∂omega)-∂L/∂theta-Q` 残差，形成方程一致性诊断。
8. `main` 执行阈值断言（能量漂移、周期偏差、残差 RMS），确保该 MVP 不是“只跑通”而是“有基本物理正确性保障”。

该实现未调用“单摆黑盒求解器”；动力学方程、数值求解控制与验证指标都可在源码逐行追踪。
