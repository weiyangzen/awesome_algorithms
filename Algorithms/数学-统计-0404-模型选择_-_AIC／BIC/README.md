## R01

AIC/BIC 是经典的统计模型选择准则，用来在“拟合好坏”和“模型复杂度”之间做权衡。

- AIC（Akaike Information Criterion）更偏向预测性能；
- BIC（Bayesian Information Criterion）更偏向结构简洁与大样本下一致选择。

本目录用一个最小可运行 MVP 演示如何在多项式回归候选集中用 AIC/BIC 选阶数。

## R02

模型选择的核心矛盾是：

- 参数太少会欠拟合，无法解释数据结构；
- 参数太多会过拟合，训练误差下降但泛化能力可能变差。

AIC/BIC 都在对数似然前面加上复杂度惩罚项，从而避免“只看训练误差”的单一准则。

## R03

本实现的任务定义：

- 输入：一维自变量 `x`、因变量 `y`，以及候选阶数集合 `degree in {1,...,8}`；
- 候选模型：多项式回归 `y = beta0 + beta1 x + ... + betad x^d + epsilon`；
- 噪声假设：`epsilon ~ N(0, sigma^2)`；
- 输出：每个候选模型的 `RSS`、`log-likelihood`、`AIC`、`BIC`，以及最优阶数。

## R04

本目录使用的公式（高斯误差，`sigma^2` 用 MLE）：

1. 残差平方和：`RSS = sum_i (y_i - yhat_i)^2`
2. 方差 MLE：`sigma2_hat = RSS / n`
3. 对数似然：
   `logL = -n/2 * [log(2*pi) + 1 + log(RSS/n)]`
4. 信息准则：
   `AIC = 2k - 2logL`
   `BIC = k*log(n) - 2logL`

其中 `k` 为参数个数（本实现含截距和噪声方差参数）。

## R05

高层流程：

1. 生成可复现实验数据（真模型为三次多项式 + 高斯噪声）；
2. 对每个候选阶数构造设计矩阵；
3. 用最小二乘解得到回归系数；
4. 计算 `RSS -> logL -> AIC/BIC`；
5. 汇总为表格，分别找出 AIC 最小和 BIC 最小模型；
6. 输出结果并做断言校验。

## R06

`demo.py` 的默认实验设置：

- 样本数：`n=220`
- 候选阶数：`1..8`
- 真模型阶数：`3`
- 数据生成：
  `y = 1.5 - 2.0x + 0.8x^2 - 0.55x^3 + noise`

这样可以观察到：

- 随阶数上升，训练 `RSS` 通常单调不增；
- AIC/BIC 不会盲目偏好最高阶，而是给出折中选择。

## R07

复杂度分析（候选数 `M`，样本数 `n`，最高阶约为 `d`）：

- 单个候选模型主要成本是线性最小二乘，约 `O(n*d^2 + d^3)`；
- 全部候选总成本约 `O(M*(n*d^2 + d^3))`；
- 空间复杂度主要来自设计矩阵与中间向量，约 `O(n*d)`。

本 MVP 里 `d<=8`，计算量很小，适合作为教学与验证脚本。

## R08

数值稳定与工程细节：

- 用 `numpy.linalg.lstsq` 求解 OLS，避免显式求逆；
- 对 `RSS` 使用下界 `EPS=1e-12`，避免 `log(0)`；
- 全流程固定随机种子，保证可复现；
- 所有输出使用 `pandas.DataFrame` 统一呈现，便于审阅。

## R09

准则解读要点：

- AIC/BIC 的“绝对值”通常不直接解释，重点看候选间相对大小；
- 同一批候选中，值越小越好；
- `n` 增大时，BIC 惩罚项 `k*log(n)` 增长更快，通常比 AIC 更保守；
- 二者不一致时，常见做法是结合业务目标（解释优先/预测优先）决定。

## R10

`demo.py` 函数职责：

- `simulate_polynomial_data`：生成合成数据；
- `build_design_matrix`：按阶数构造多项式特征；
- `fit_ols_gaussian_mle`：拟合 OLS 并计算 `RSS/logL/AIC/BIC`；
- `evaluate_candidates`：批量评估候选模型；
- `scores_to_dataframe`：转换为表格并格式化；
- `pick_best`：按指定准则选择最优模型；
- `main`：组织执行、打印结果与断言检查。

脚本无交互输入，运行后直接打印模型比较结果。

## R12

输出字段说明：

- `degree`：候选多项式阶数；
- `k`：参数个数（含截距与噪声方差）；
- `rss`：残差平方和；
- `sigma2_hat`：噪声方差的 MLE；
- `log_likelihood`：高斯对数似然；
- `aic`：AIC 分数；
- `bic`：BIC 分数。

随后脚本会单独打印 AIC 最优阶数和 BIC 最优阶数。

## R13

正确性门禁（脚本内断言）：

1. 结果表不允许 `NaN`；
2. 由于候选模型嵌套，`RSS` 应近似单调不增（允许微小数值误差）；
3. AIC/BIC 选出的最优阶数需落在合理区间（接近真阶数 3）；
4. AIC/BIC 最优值应不高于一次模型基线。

这些检查确保脚本不是“只打印不校验”。

## R14

常见失效模式与对策：

- 失效 1：候选集过窄（不包含合理模型）。
  对策：扩展候选结构，而不是只调准则。

- 失效 2：样本太小导致高方差选择。
  对策：减少参数自由度或用重复采样评估稳定性。

- 失效 3：误差分布严重偏离高斯，`logL` 失真。
  对策：改用更匹配的似然模型或使用稳健比较方法。

- 失效 4：多重共线性导致参数不稳。
  对策：正则化、正交基、或先做特征缩放。

## R15

实践建议：

- AIC/BIC 先用于“快速筛模型”，再结合交叉验证复核；
- 不要跨数据集直接比较 AIC/BIC 数值，只在同一数据与同一似然框架内比较；
- 报告时建议附上 `delta AIC / delta BIC` 与候选排名，而不仅给单一最优结论；
- 关注可解释性时可优先参考 BIC，关注预测时可优先参考 AIC。

## R16

与相关方法对比：

- 对比交叉验证：CV 更直接面向泛化误差，但计算成本更高；AIC/BIC 计算更快。
- 对比调整后 `R^2`：调整后 `R^2` 常用于线性回归，适用范围不如似然型准则通用。
- 对比 AICc：AICc 在小样本时对 AIC 有额外修正，本 MVP 先展示 AIC/BIC 的基础形态。

## R17

可扩展方向：

- 把候选从“多项式阶数”扩展到“特征子集/分布族”；
- 增加 AICc、WAIC 等准则；
- 用 `sklearn` 的管道候选（如正则回归）并统一输出信息准则；
- 在真实业务数据上增加 train/test 划分，对比信息准则与测试误差的一致性。

## R18

`demo.py` 的源码级算法流程（8 步）：

1. `main` 固定随机种子并调用 `simulate_polynomial_data` 生成三次真模型样本。  
2. `evaluate_candidates` 遍历候选阶数 `1..8`，逐个评估。  
3. `build_design_matrix` 将原始一维 `x` 展开为 `[x, x^2, ..., x^d]` 特征矩阵。  
4. `fit_ols_gaussian_mle` 在特征前拼接截距列，用 `numpy.linalg.lstsq` 求最小二乘参数。  
5. 同一函数内计算预测值与残差，得到 `RSS` 和 `sigma2_hat = RSS/n`。  
6. 将 `RSS` 代入高斯对数似然公式，计算 `log_likelihood`。  
7. 基于参数数 `k` 计算 `AIC = 2k-2logL` 与 `BIC = k*log(n)-2logL`，写入结果表。  
8. `pick_best` 分别按 AIC/BIC 取最小项，`main` 打印结论并执行断言门禁（数值有效性、RSS 单调性、最优阶数合理性）。

这 8 步完整覆盖了“候选生成 -> 拟合 -> 似然评估 -> 信息准则打分 -> 选择与验证”的最小实现链路。
