## R01

“不同二叉搜索树”问题是：给定 `n` 个互异键值（通常取 `1..n`），问能构造出多少种结构不同的二叉搜索树（BST）。

关键点是“只计结构，不计具体插入过程”。

## R02

该问题是 Catalan 数列在数据结构中的经典落点，常见于算法课程与面试题。

它与以下主题直接相关：
- 分治递推（枚举根节点后分解为左右子树子问题）；
- 动态规划（自底向上复用子规模结果）；
- 组合数学（闭式公式 `C_n = (1/(n+1)) * binom(2n, n)`）。

## R03

形式化定义：
- 输入：非负整数 `n`。
- 输出：整数 `f(n)`，表示由键集合 `{1,2,...,n}` 可形成的 BST 结构数量。

边界条件：
- `n = 0` 时，空树算一种结构，因此 `f(0) = 1`。

## R04

分治递推关系：
- 设根节点选择第 `r` 小元素（`1 <= r <= n`）。
- 左子树必须使用 `r-1` 个元素，右子树必须使用 `n-r` 个元素。
- 左右子树彼此独立，组合数相乘。

因此有：
- `f(0) = 1, f(1) = 1`
- `f(n) = sum_{r=1..n} f(r-1) * f(n-r)`

这就是本题的分治核心。

## R05

MVP 中的关键状态：
- `solve(nodes)`：返回 `nodes` 个节点能构造的 BST 数量。
- `memo[nodes]`：记忆化缓存，避免重复求解同一规模。
- `DivideConquerStats`：记录递归调用次数、缓存命中次数、切分评估次数。

## R06

以 `n = 3` 为例：
- 选根 `1`：左规模 `0`，右规模 `2`，贡献 `f(0)*f(2)=1*2=2`。
- 选根 `2`：左规模 `1`，右规模 `1`，贡献 `f(1)*f(1)=1*1=1`。
- 选根 `3`：左规模 `2`，右规模 `0`，贡献 `f(2)*f(0)=2*1=2`。

总数 `f(3)=2+1+2=5`。

## R07

该算法的价值：
- 理论：展示“按根切分”如何把结构计数问题转成规模递推；
- 实践：可迁移到括号化计数、树形结构枚举、语法树数量估算等任务；
- 工程：记忆化版本是小而稳健的可复用模板。

## R08

前置知识：
- BST 性质（左小右大）；
- 递归/分治基本写法；
- 动态规划中“重叠子问题 + 最优子结构”思想；
- Python 基础类型与整数计算。

## R09

复杂度（记 `n` 为节点数）：
- 状态数：`0..n` 共 `O(n)` 个子问题；
- 每个 `k` 需要枚举 `k` 个根切分；
- 总时间复杂度：`O(n^2)`；
- 额外空间复杂度：`O(n)`（缓存与 DP 数组）。

## R10

正确性（归纳证明要点）：

1. 基础情形：`f(0)=1`、`f(1)=1` 显然成立。  
2. 归纳假设：对所有 `< n` 的规模，`f` 已正确给出 BST 数量。  
3. 对规模 `n`，任一 BST 的根必对应某个 `r`，其左右子树规模唯一确定为 `r-1` 与 `n-r`。  
4. 左右子树结构可独立组合，方案数为 `f(r-1)*f(n-r)`。  
5. 枚举全部 `r` 且求和，既不重不漏地覆盖所有 BST 结构。  

故递推式正确，算法正确。

## R11

数值与语义稳定性：
- 本题纯整数计算，无浮点误差传播；
- Python `int` 为任意精度，避免了固定宽度整数溢出；
- 语义关键在于是否把“空树”计为 1，若处理错误会导致全局结果偏差。

## R12

工程成本（超越大 O）：
- 递归版本有函数调用开销；
- 记忆化显著减少重复计算，但会引入缓存存储；
- 当 `n` 增大时，结果值增长很快，打印/序列化的成本会高于核心循环；
- 在 Python 中，迭代 DP 常比递归更容易做性能追踪。

## R14

常见失效模式：
- 把 `f(0)` 写成 `0`（最常见逻辑错误）；
- 递推索引写错，出现 `left/right` 规模偏移；
- 未对负输入做校验，导致无意义递归；
- 仅做单点样例而缺少系统性回归（例如未验证到 `n=10`）。

## R15

- 基准时分开统计：算法耗时、递归统计、缓存命中。

## R18

`demo.py` 源码级流程（8 步）：

1. `main()` 依次执行固定样例、全量一致性检查、序列摘要、基准测试，整个流程无交互输入。  
2. `num_trees_divide_conquer(n)` 初始化 `memo={0:1,1:1}` 与统计器，并定义递归 `solve(nodes)`。  
3. `solve(nodes)` 先查 `memo`；命中则直接返回并累计 `memo_hits`。  
4. 若未命中，枚举 `left_nodes in [0, nodes-1]`，令 `right_nodes = nodes-1-left_nodes`。  
5. 对每个切分，递归求左右子问题并累加 `solve(left_nodes) * solve(right_nodes)`，得到当前规模答案。  
6. 把当前规模结果写回 `memo[nodes]`，逐层回传，最终得到 `f(n)`。  
7. `num_trees_dp` 以同一递推做自底向上计算，`num_trees_catalan_closed_form` 用组合数闭式给出第三路结果。  
8. 校验阶段断言三种实现完全一致，随后用 `pandas` 组织基准表、`numpy` 生成增长比统计并打印。
