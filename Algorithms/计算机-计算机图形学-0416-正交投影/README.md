## R01

正交投影（Orthographic Projection）是把三维点映射到二维屏幕时最基础的投影模型之一。  
它的关键特征是：投影射线彼此平行，不经过同一个视点，因此不会产生透视缩短（near 大、far 小）的视觉效果。

## R02

在图形学管线里，正交投影通常用于：
- CAD / 工程制图（强调真实尺寸关系）；
- 2D/3D 编辑器中的顶视图、正视图、侧视图；
- UI 叠加层、地图、等距/轴测风格中的近似投影；
- 阴影贴图等离屏渲染中对平行光视锥的投影步骤。

## R03

本题抽象输入输出如下：
- 输入：相机空间点集 `P = {(x_i, y_i, z_i)}`；
- 输入：轴对齐观察体 `box = [l,r] x [b,t] x [n,f]`（要求 `l<r, b<t, n<f`）；
- 输出：NDC 坐标 `p_ndc in [-1,1]^3`，以及可选屏幕坐标 `(u,v)`；
- 裁剪规则：仅 `|x_ndc|<=1 && |y_ndc|<=1 && |z_ndc|<=1` 的点视为在可见体内。

## R04

本目录 MVP 使用 `4x4` 齐次矩阵形式：

```text
M_ortho =
[ 2/(r-l)      0          0      -(r+l)/(r-l) ]
[    0      2/(t-b)       0      -(t+b)/(t-b) ]
[    0         0       2/(f-n)   -(f+n)/(f-n) ]
[    0         0          0             1      ]
```

对齐次点 `p_h=[x,y,z,1]^T` 有：`clip = M_ortho @ p_h`，再做 `ndc = clip[:3] / clip[3]`。

## R05

同一投影也可写为显式线性归一化公式：

- `x_ndc = (2x - (r+l)) / (r-l)`
- `y_ndc = (2y - (t+b)) / (t-b)`
- `z_ndc = (2z - (f+n)) / (f-n)`

由于正交投影矩阵最后一行固定为 `[0,0,0,1]`，故 `w'=1`，分母恒定，不引入透视除法的深度非线性。

## R06

从 NDC 到屏幕像素（顶点在左上角原点约定）采用：

- `u = (x_ndc * 0.5 + 0.5) * (W-1)`
- `v = (1 - (y_ndc * 0.5 + 0.5)) * (H-1)`

这一步是视口变换，与投影本身分离，便于调试“几何变换错误”还是“像素映射错误”。

## R07

复杂度分析（`N` 为点数）：
- 矩阵批量投影：`O(N)`；
- 直接公式投影：`O(N)`；
- 空间复杂度：`O(N)`（输出数组与中间齐次数组）。

因此瓶颈主要是点数量而非算法阶数。

## R08

正确性依赖的关键不变量：
1. 观察体边界严格有序（`l<r, b<t, n<f`）；
2. 点集必须是 `(N,3)` 且数值有限；
3. 投影矩阵必须是 `4x4`；
4. 正交投影后 `w'` 应恒等于 `1`（在浮点容差内）；
5. 矩阵法与直接公式法应数值一致（`allclose`）。

## R09

正交投影的几何性质（本 MVP 会实测）：
- 保平行：空间中平行线段投影后仍平行；
- 无透视缩短：相同长度、不同深度的线段投影后长度相同；
- 深度独立的 x/y 缩放：`x,y` 映射与 `z` 无耦合，只受 `l,r,b,t` 控制。

## R10

本目录 MVP 目标：
1. 手写正交投影矩阵构造函数；
2. 提供矩阵投影与直接公式投影双实现并交叉校验；
3. 提供 NDC 立方体内点判定；
4. 提供 NDC 到屏幕像素映射；
5. 输出可读表格与指标，形成“可运行 + 可验证”闭环。

## R11

`demo.py` 的核心函数分工：
- `orthographic_matrix`：生成 `4x4` 投影矩阵；
- `project_points_matrix`：齐次矩阵批量投影；
- `project_points_direct`：按坐标轴公式投影；
- `inside_ndc_cube`：做可见性判定；
- `ndc_to_screen`：映射到像素平面；
- `main`：构造样例、执行断言、统计性能与输出表格。

## R13

脚本输出的关键指标：
- `max_diff_matrix_vs_direct_sample`：样例点矩阵法与直接法最大差；
- `max_w_deviation`：`w'` 偏离 1 的最大值；
- `parallel_cross`：两条投影线方向向量叉积（应接近 0）；
- `near_far_length_ratio`：近远同长线段投影长度比（应接近 1）；
- `cloud_max_diff_matrix_vs_direct`：随机大样本双实现差异；
- `matrix_projection_ms / direct_projection_ms`：两种实现耗时对比。

## R14

`demo.py` 内置门禁断言：
1. 矩阵投影与直接公式投影必须一致；
2. `w'` 必须保持常数 1（容差 `1e-12`）；
3. 平行性检测必须通过（叉积接近 0）；
4. 无透视缩短检测必须通过（长度比接近 1）；
5. 随机点云双实现误差不得超过阈值。

## R18

`demo.py` 源码级算法链路（8 步）：
1. `main()` 定义观察体 `OrthoBox(l,r,b,t,n,f)` 并调用 `orthographic_matrix` 生成 `4x4` 投影矩阵。  
2. 对样例点集先走 `project_points_matrix`：先 `to_homogeneous` 扩展为 `(N,4)`，再做 `(M @ P^T)^T` 得到 `clip` 和 `ndc`。  
3. 同一批点走 `project_points_direct`，按三条显式归一化公式逐轴计算 `x_ndc/y_ndc/z_ndc`。  
4. 用 `np.allclose` 与最大绝对误差对比两条路径，并检查 `clip[:,3]` 偏离 `1` 的最大值，验证“正交投影无透视除法”。  
5. 调用 `inside_ndc_cube` 判定点是否落入 `[-1,1]^3`，再通过 `ndc_to_screen` 映射到屏幕像素坐标。  
6. 构造两条不同深度但平行的线段，投影到 NDC 平面后比较方向向量叉积，验证“平行保持”。  
7. 构造近处/远处等长线段，比较投影后长度比，验证“无透视缩短”（比例应约等于 1）。  
8. 在 `250000` 随机点云上分别执行矩阵法与直接公式法，统计耗时和最大误差，最终用 `pandas` 输出样例表与指标表。
