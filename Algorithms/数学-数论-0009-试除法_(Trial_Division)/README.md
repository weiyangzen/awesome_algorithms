# 试除法 (Trial Division)

- UID: `MATH-0009`
- 学科: `数学`
- 分类: `数论`
- 源序号: `9`
- 目标目录: `Algorithms/数学-数论-0009-试除法_(Trial_Division)`

## R01

试除法（Trial Division）是最直接的整数判定/分解方法：依次尝试用候选因子去整除目标整数。

- 输入:
  - 素性判定: 整数 `n`
  - 因式分解: 整数 `n`
- 输出:
  - 素性判定: `n` 是否为素数
  - 因式分解: `n = p1^e1 * p2^e2 * ...` 的素因子分解
- 核心机制:
  - 先处理小因子（如 `2`）
  - 对奇数因子 `d = 3, 5, 7, ...` 逐个测试
  - 仅需测试到 `d*d <= n`（等价于 `d <= sqrt(n)`）
  - 对分解任务，发现因子后持续除尽该因子，直到不能整除再尝试下一个因子

## R02

可追溯到古希腊时期的整数与素数研究传统；作为系统化的“算法描述”，通常与欧几里得时代（约公元前 300 年）数学文本中的整除与素数理论同源。  
现代算法教材把它作为素性测试和因子分解的起点方法，长期用于教学与小规模计算。

## R03

提出背景是“如何判断一个数是不是素数、如何把整数分解为素因子”这两个基础问题。

- 历史痛点: 缺少统一、可执行、可证明正确的通用流程
- 试除法的回答: 用“整除关系 + 有界搜索（到 `sqrt(n)`）”把问题离散化
- 前人不足: 仅靠经验判断或列举因子缺乏可证明边界；试除法补上了明确终止条件和正确性依据

## R04

设输入整数绝对值为 `N`。

- 素性判定:
  - 最坏时间复杂度: `O(sqrt(N))`
  - 在仅测试奇数因子时，模运算次数约减半，仍是 `O(sqrt(N))`
- 因式分解:
  - 最坏时间复杂度: `O(sqrt(N))`（例如 `N` 本身是大素数，或最小因子很大）
  - 常见输入上若存在小因子，会提前停止，实际耗时常显著低于最坏情况

## R05

额外空间复杂度通常为 `O(1)`（不计输入本身）。

- 素性判定: 仅维护当前试除因子和若干计数器
- 因式分解: 若要输出所有因子，输出容器占用 `O(k)`，`k` 为不同素因子个数（或按重数计的长度）

## R06

示例：分解 `n = 221`

1. 先测 `2`：`221 % 2 != 0`
2. 从奇数开始试除：`3, 5, 7, 9, 11, 13`
3. 到 `13` 时，`221 % 13 == 0`，得到一个因子 `13`
4. 商为 `17`，`17` 是素数
5. 输出分解: `221 = 13 * 17`

素性判定示例：`n = 97`

1. 测 `2` 不整除
2. 仅需测到 `sqrt(97) < 10`，尝试 `3,5,7,9`
3. 都不整除，因此 `97` 是素数

## R07

试除法的意义不在于处理超大整数，而在于“基准性”：

- 理论上: 它把素性与分解问题还原为最基础的整除测试
- 工程上: 是小规模输入的可靠基线（baseline），便于校验更复杂算法结果
- 教学上: 是理解 `sqrt(n)` 边界、循环不变量和算法复杂度的经典入口

## R08

依赖的先导知识/算法：

- 整数除法与取模运算（`a % b`）
- 基本数论事实：若 `n` 为合数，则存在不大于 `sqrt(n)` 的非平凡因子
- 奇偶性判定（先剔除因子 `2`）
- （实现层）循环与条件分支，不依赖高阶数值库

## R09

适用前提与边界：

- 输入应为整数；非整数需先定义转换规则
- 对 `n < 2`，按约定不是素数
- 对负数分解，通常先拆出符号因子 `-1`
- 适合中小规模整数；对密码学级大整数（数百/数千比特）通常不适用，耗时不可接受

## R10

正确性依据：

1. 若 `n = a*b` 且 `1 < a <= b`，则 `a <= sqrt(n)`（反证可得）
2. 因此合数 `n` 必有一个不大于 `sqrt(n)` 的因子
3. 试除法枚举该范围内候选因子，若找到整除则 `n` 合数
4. 若未找到任何因子，则不存在非平凡因子，`n` 为素数
5. 因式分解时，每找到因子就“除尽并递归/迭代处理剩余商”，可保证输出因子乘积还原原数

## R11

`N/A + 原因`：试除法是离散整数算法，不依赖浮点迭代，通常不存在浮点舍入误差传播问题。  
补充：若实现中使用浮点 `sqrt` 作为边界，可能引入极端大整数的边界误判；工程上更稳妥的是使用整数条件 `d*d <= n`。

## R12

真实成本（超越渐近阶）：

- 单次 `%` 运算是主成本；Python 大整数下该成本随位数增长
- 常数优化很关键：
  - 先处理 `2`，后续只测奇数
  - 使用 `d*d <= n` 避免反复调用 `sqrt`
  - 在分解中“除尽当前因子”可大幅减少后续测试次数
- 缓存/向量化收益有限：该算法分支依赖强、循环短、难以高效 SIMD 化
- 并行化空间有限：单个 `n` 的顺序依赖较强，但可对“多个待测整数”并行批处理

## R13

`N/A + 原因`：该算法为确定性精确算法，不是近似算法、随机算法或统计估计算法。  
结论是“正确/错误”或“精确因式分解”，不涉及近似比、概率保证或误差上界。

## R14

鲁棒性与失效模式：

- 失效模式 1：输入规模过大（尤其大素数）导致运行时间过长
- 失效模式 2：输入类型不规范（字符串、浮点、空值）导致异常或语义歧义
- 失效模式 3：对 `0`、`1`、负数等边界值处理不一致
- 缓解手段：
  - 明确输入校验
  - 约定边界值返回语义
  - 对超大输入设置超时/阈值，必要时切换到 Miller-Rabin 或 Pollard Rho

## R15

工程实现要点：

- 函数拆分：
  - `is_prime_trial(n)`：素性判定
  - `factorize_trial(n)`：因式分解
- 先处理边界值：`n < 2`、`n == 2`、偶数
- 循环条件用 `d*d <= n`，避免浮点边界问题

## R18

源码追踪与算法拆解（本条目未调用第三方黑箱；直接追踪 `demo.py` 实现）：

1. **输入归一化**：读取整数 `n`，处理 `n < 2`、符号位与特殊值。  
2. **快速剪枝**：先检查 `n` 是否为 `2` 或偶数，快速返回一批结果。  
3. **建立候选序列**：从 `d = 3` 开始，仅枚举奇数候选因子（`d += 2`）。  
4. **边界控制**：循环条件使用 `d*d <= n`，等价于只搜索到 `sqrt(n)`。  
5. **整除测试**：执行 `n % d`；若余数为 0，判定为合数（或记录因子）。  
6. **因子除尽**（分解分支）：当 `d` 可整除时，重复 `n //= d` 直到不可整除，记录指数。  
7. **残余处理**：循环结束后若剩余 `n > 1`，该残余即最后一个素因子。  
8. **结果组装**：输出布尔素性结论或格式化素因子分解，并附加统计信息（可选）。

该流程是试除法的“源码级最小闭环”：没有把逻辑委托给单行黑箱函数，步骤可逐条映射到代码分支。
