# Aitken加速法

- UID: `MATH-0122`
- 学科: `数学`
- 分类: `数值分析`
- 源序号: `122`
- 目标目录: `Algorithms/数学-数值分析-0122-Aitken加速法`

## R01

Aitken 加速法（Aitken's delta-squared process）用于提升线性收敛数列的收敛速度。  
给定一个逐步逼近极限 `x*` 的标量序列 `x_k`，Aitken 通过三项 `x_k, x_{k+1}, x_{k+2}` 构造一个新的估计 `x_hat_k`，常可把“慢线性收敛”显著加快到接近二次效果（在理想模型下可一步命中极限）。

## R02

本条目 MVP 聚焦一维标量序列加速，目标是回答三个问题：
- 如何从原始迭代序列构造 Aitken 加速序列；
- 在典型固定点迭代 `x_{k+1}=g(x_k)` 中是否明显更快；
- 在分母接近 0 等数值边界下如何稳定实现。

`demo.py` 提供两个无交互实验：
- 合成几何收敛序列（用于验证公式）；
- `x_{k+1}=cos(x_k)` 固定点迭代（用于观察真实加速效果）。

## R03

核心公式（Delta-Squared）：

- 一阶差分：`Delta x_k = x_{k+1} - x_k`
- 二阶差分：`Delta^2 x_k = x_{k+2} - 2x_{k+1} + x_k`
- Aitken 估计：
  `x_hat_k = x_k - (Delta x_k)^2 / (Delta^2 x_k)`

当误差模型近似 `x_k = x* + c q^k`（`|q|<1`）时，上式在精确算术下可直接给出 `x_hat_k = x*`，这解释了其对线性收敛序列的高效性。

## R04

算法流程（标量版）：
1. 先生成原序列 `x_0, x_1, ..., x_n`。
2. 对每个 `k=0..n-2` 计算 `Delta x_k` 与 `Delta^2 x_k`。
3. 若 `|Delta^2 x_k|` 足够大，按公式计算 `x_hat_k`。
4. 若分母过小则跳过该点（避免数值爆炸）。
5. 输出原序列误差与加速后误差，比较达到目标精度所需步数。

## R05

`demo.py` 中的主要数据对象：
- `raw_seq: np.ndarray`：原始迭代序列，长度 `N+1`；
- `aitken_seq: np.ndarray`：Aitken 估计序列，长度 `N-1`；
- `valid_mask: np.ndarray[bool]`：标记哪些 `k` 的 `Delta^2 x_k` 可用；
- `raw_errors / aitken_errors`：与参考极限的绝对误差。

这种“数组 + 掩码”结构比复杂类层次更适合 MVP，计算和打印都直接透明。

## R06

正确性要点：
- Aitken 并不改变原始迭代映射 `g`，而是对产生的数列做后处理；
- 对几何误差模型，它可解析消去主导线性误差项；
- 对一般问题，它不保证总是单调改进，但在线性主导区通常能显著减小误差；
- `demo.py` 的几何序列实验用于验证公式，`cos` 固定点实验用于验证实战效果。

## R07

复杂度分析（长度为 `n+1` 的原序列）：
- 生成原序列：`O(n)`；
- 构造 Aitken 序列：`O(n)`；
- 误差评估与打印：`O(n)`；
- 总时间复杂度：`O(n)`；
- 额外空间复杂度：`O(n)`（保存序列与误差）。

对单次在线更新也可降为 `O(1)` 额外空间，但 MVP 选择可读性更高的批处理数组写法。

## R08

数值稳定性与边界处理：
- 当 `Delta^2 x_k` 接近 0 时，公式会出现大幅放大误差，代码用阈值 `eps` 拦截；
- 被拦截的位置记为无效（`valid_mask[k]=False`），不参与收敛统计；
- 参考极限计算采用高精度停止准则，减少“基准值不准”导致的误判；
- 所有计算限定在标量实数范围，避免在 MVP 中引入复数分支。

## R09

MVP 取舍说明：
- 仅依赖 `numpy`（最小第三方栈）；
- 不调用黑盒“加速器 API”，Aitken 公式完全手写；
- 仅覆盖标量序列，不做向量序列或 Anderson acceleration；
- 强调“可运行 + 可验证 + 可解释”，不追求通用框架化。

## R10

`demo.py` 函数职责对应：
- `fixed_point_sequence`：生成原始固定点迭代序列；
- `aitken_delta_squared`：按 Delta-Squared 公式构造加速序列；
- `high_precision_fixed_point`：生成参考极限（用于误差评估）；
- `first_index_below`：统计首次达到误差阈值的位置；
- `run_geometric_validation`：几何模型正确性实验；
- `run_cos_fixed_point_demo`：真实固定点加速对比实验；
- `main`：统一入口，无交互执行。

## R12

输出字段解读：
- `x_k`：原始第 `k` 次迭代值；
- `|x_k-x*|`：原始误差；
- `x_hat_k`：Aitken 对 `x_k,x_{k+1},x_{k+2}` 的加速估计；
- `|x_hat_k-x*|`：加速误差；
- `valid`：该加速点是否通过分母阈值检查；
- “first step below tol”用于展示达到目标误差所需步数差异。

## R13

预期现象：
- 在几何序列实验中，首个有效 `x_hat_k` 应接近真实极限（通常机器精度级）；
- 在 `cos` 固定点实验中，Aitken 误差通常比同阶段原序列小得多；
- 达到相同误差阈值时，加速序列通常需要更少有效步。

若 `valid` 点较少，通常表示原序列后期已经趋于平坦，`Delta^2` 太小被保护逻辑过滤。

## R14

常见坑与规避：
- 误写二阶差分（符号错误）会直接导致错误加速；
- 忽略 `Delta^2` 近零检查会产生异常大值或 `inf`；
- 用过短序列评估会误判加速效果（建议至少 10+ 步）；
- 参考极限不够精确时，误差比较可能“看起来不加速”。

## R15

与相关方法对比：
- 与 Steffensen 法：Steffensen 实质把 Aitken 嵌入固定点迭代中直接构造下一步；本条目实现的是“先生成序列再后处理”的纯 Aitken 版本。
- 与 Richardson 外推：Richardson 更依赖步长与误差阶模型；Aitken 只需要连续三项序列值。
- 与 Anderson acceleration：Anderson 适合高维非线性问题，但实现更重；Aitken 轻量、适合一维或教学场景。

## R16

适用场景：
- 固定点迭代收敛慢但稳定时的廉价提速；
- 数值分析课程中展示“误差模型消去”思想；
- 迭代法原型调试阶段，快速判断是否存在可利用的线性误差主项；
- 需要低工程复杂度的后处理加速模块。

## R17

可扩展方向：
- 扩展为 Steffensen 法（把 Aitken 内嵌成新迭代器）；
- 对向量序列实现分量级或子空间加速；
- 在噪声环境中加入鲁棒阈值或正则项；
- 与停止准则联动：动态选择“原始步”或“加速步”；
- 增加多问题基准集，统计平均提速比。

## R18

`demo.py` 的源码级算法流程（8 步）：
1. `main` 先执行 `run_geometric_validation`，构造 `x_k = x* + c q^k` 的合成序列作为公式校验样本。  
2. `aitken_delta_squared` 逐个 `k` 计算 `Delta x_k` 与 `Delta^2 x_k`，并在分母不小于阈值时输出 `x_hat_k`。  
3. 几何样本中读取首个有效 `x_hat_k`，与真值 `x*` 对比，验证 Delta-Squared 的代数效果。  
4. `run_cos_fixed_point_demo` 调用 `fixed_point_sequence` 生成 `x_{k+1}=cos(x_k)` 的原始序列。  
5. `high_precision_fixed_point` 用严格阈值迭代得到参考极限 `x*`，作为统一误差基准。  
6. 再次调用 `aitken_delta_squared` 得到 `cos` 问题上的加速序列与有效掩码。  
7. 计算 `raw_errors` 与 `aitken_errors`，逐行打印 `x_k`、`x_hat_k` 及对应误差，直观看加速效果。  
8. `first_index_below` 统计两种序列首次达到目标容差的步号，输出“达到同精度所需步数”对比结论。
