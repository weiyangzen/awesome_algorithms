## R01

鲁棒优化（Robust Optimization, RO）研究“在不确定性存在时，如何做最坏情形可行且代价可控的决策”。标准形式是二层 min-max：

- `min_{x in X} max_{u in U} f(x, u)`

其中 `x` 是决策变量，`u` 是不确定参数，`U` 是不确定集合。  
与只优化平均性能的方法相比，鲁棒优化直接对冲“坏场景”，目标是降低部署阶段的失效风险。

## R02

历史主线可概括为三步：

- 早期 Soyster 形式：极保守，保证强但常损失过大；
- Ben-Tal / Nemirovski 等人系统化了鲁棒对偶与可解性条件；
- Bertsimas-Sim 预算不确定集在“保守性 vs 可行性”之间提供可调折中。

今天的鲁棒优化已广泛用于运筹、控制、金融和机器学习稳健训练。

## R03

鲁棒优化解决的核心矛盾是：

- 名义模型（只看估计值）在训练/规划阶段表现好；
- 真实环境存在参数偏移、测量噪声、分布漂移；
- 一旦偏移落在关键约束方向，名义最优解会快速失效。

RO 的做法不是“猜最准的未来”，而是“在可接受的不确定集合内都能扛住”。

## R04

本目录 MVP 采用一个可微且可解释的鲁棒回归模型：

- 原问题：`min_w max_{||Delta||_F <= rho} (1/(2n)) ||(X + Delta)w - y||_2^2 + (l2/2)||w||_2^2`
- `Delta` 表示特征矩阵测量误差，`rho` 是误差预算（Frobenius 范数球半径）。

关键等价式（源码中显式使用）：

- `max_{||Delta||_F <= rho} ||(X + Delta)w - y||_2 = ||Xw - y||_2 + rho||w||_2`

所以可转成单层问题：

- `f_rob(w) = (1/(2n)) (||Xw-y||_2 + rho||w||_2)^2 + (l2/2)||w||_2^2`

`demo.py` 直接实现该目标和梯度，不依赖黑盒鲁棒优化库。

## R05

设样本数 `n`、特征维度 `d`、迭代轮数 `T`：

- 单次前向与梯度：`O(n*d)`；
- 训练总复杂度：`O(T*n*d)`；
- 空间复杂度：`O(n + d)`（残差向量与参数/梯度）。

MVP 使用全批量梯度下降，结构简单、复杂度透明，便于验证推导正确性。

## R06

直观数值例子（单次目标评估）：

- 若 `||Xw-y||_2 = 5.0`，`||w||_2 = 1.2`，`rho = 1.6`；
- 最坏残差上界变为 `5.0 + 1.6*1.2 = 6.92`；
- 鲁棒项惩罚的是“残差 + 参数范数耦合”，不是仅残差。

这会抑制过大权重，从而提升对特征扰动的耐受性。

## R07

优点：

- 直接建模最坏情形，目标含义清晰；
- 相比仅做 `L2` 正则，更贴近“输入矩阵会漂”的风险假设；
- 本问题可化为单层可微优化，工程实现简单。

局限：

- 选择过大的 `rho` 会过度保守，ID 精度可能下降；
- 假设的不确定集合若与真实误差机制不匹配，收益会减弱；
- 依然需要调学习率、迭代轮数与正则强度。

## R08

前置知识建议：

- 线性回归与均方误差；
- 向量/矩阵范数（`L2` 与 Frobenius）；
- 链式法则和梯度下降；
- min-max 建模直觉（“先对手最坏，再自己最优”）。

## R09

适用场景：

- 传感器噪声会改变输入特征；
- 训练和部署特征质量不一致；
- 更重视坏场景稳定性，而不只看平均误差。

不太适用：

- 完全确定性、几乎无测量噪声的数据；
- 业务只关注名义平均性能且对失效容忍高；
- 对 `rho` 没有任何先验或验证渠道。

## R10

实现正确性的关键检查点：

1. 目标函数必须是 `(||r|| + rho||w||)^2` 结构，而不是简单相加罚项；
2. 梯度需正确包含 `d||r||/dw` 与 `d||w||/dw` 两部分；
3. `w=0` 或 `r=0` 附近要做数值平滑（除零保护）；
4. 需要有限差分梯度检查；
5. 训练应出现明确的 loss 下降。

`demo.py` 对名义与鲁棒目标都做了 finite-difference 校验。

## R11

数值稳定策略：

- 使用 `eps=1e-12` 保护 `||r||` 与 `||w||` 的分母；
- 采用轻度学习率衰减 `lr/(1+alpha*t)` 抑制后期震荡；
- 标准化特征时仅使用训练集统计量，避免数据泄漏；
- 使用固定随机种子保证可复现对比。

## R12

调参经验（对应本 MVP）：

- `rho`：鲁棒预算，越大越保守；建议从 `0.5 ~ 2.0` 网格扫描；
- `l2`：控制参数范数，常与 `rho` 联动；
- `lr` 与 `epochs`：先保证 loss 收敛，再比较 ID/shift 指标；
- 评估必须同时看 `ID MSE` 与 `Shift MSE`，否则容易误判。

## R13

理论性质（本实现对应模型）：

- 目标由范数与平方复合构成，是凸优化问题（线性模型下）；
- 不涉及近似算法，不存在“近似比”概念；
- 成功概率保证不适用（算法是确定性梯度法，随机性仅来自数据采样种子）。

因此该 MVP 重点是“稳健目标建模 + 数值可解性”，不是随机近似界。

## R14

常见失效模式与对策：

- `rho` 过大导致欠拟合。  
  对策：降低 `rho` 并联合观察 ID/Shift 曲线。
- 学习率过大导致 loss 抖动。  
  对策：减小初始 `lr` 或加大衰减。
- 梯度漏掉范数导数项。  
  对策：保留有限差分检查作为硬约束。
- 数据尺度失衡。  
  对策：先标准化，再训练。

## R15

工程落地建议：

- 把 `rho` 当作“风险预算”，与业务方按容错等级协商；
- 离线验证时固定随机种子和数据切分，保证实验可复现；
- 部署监控不只看平均误差，要单独跟踪尾部/坏场景误差；
- 先在线性可解释模型验证机制，再扩展到复杂模型。

## R16

与相关方法的关系：

- 名义最小二乘：`rho=0` 时退化到普通回归；
- 仅 `L2` 正则：控制参数大小，但不显式建模输入不确定性；
- 对抗训练：在样本级扰动上做 min-max，可视为另一类稳健框架；
- 分布鲁棒优化（DRO）：不确定对象是“分布”，而本项是“参数/数据矩阵扰动集合”。

二者都属于“最坏情形优化”，但不确定层级不同。

## R17

`demo.py` 做了以下事情：

- 合成“训练观测特征 / ID 测试 / Shift 测试”三套数据；
- 训练名义模型与鲁棒模型（同一优化框架、不同目标）；
- 输出梯度检查误差、loss 收敛、ID 与 Shift MSE；
- 显式构造一组最坏扰动 `Delta*`，验证鲁棒解的最坏残差更低。

运行方式：

```bash
cd Algorithms/数学-优化-0420-鲁棒优化
python3 demo.py
```

脚本无需交互输入，运行结束会执行断言检查。

## R18

`demo.py` 的源码级算法流程（8 步）：

1. `build_dataset` 生成潜在干净特征，再叠加不同强度测量噪声，得到训练、ID、Shift 三套观测输入。  
2. `standardize_from_train` 用训练集统计量标准化所有输入，避免尺度偏差和数据泄漏。  
3. `nominal_loss_grad` 实现名义最小二乘目标与梯度，作为基线。  
4. `robust_loss_grad` 实现鲁棒目标 `((||r||+rho||w||)^2)/(2n)+l2||w||^2/2` 及显式梯度。  
5. `finite_diff_gradient_check` 用中心差分逐维验证名义/鲁棒梯度实现正确。  
6. `fit_gradient_descent` 对两类目标分别做全批量梯度下降，得到参数与 loss 曲线。  
7. `worst_case_residual_norm` 构造闭式 `Delta* = rho * u v^T`（`u` 沿残差方向、`v` 沿参数方向）并计算最坏残差范数。  
8. `main` 汇总打印 ID/Shift 误差、鲁棒目标对比和断言，完成一次可复现实验闭环。  

这 8 步覆盖了“建模 -> 推导 -> 训练 -> 验证”的完整鲁棒优化最小实现路径。
