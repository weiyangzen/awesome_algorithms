## R01

堆排序（Heap Sort）是一种基于二叉堆的比较排序算法。它先把数组原地建成最大堆，再反复把堆顶（当前最大值）交换到尾部，逐步得到升序结果。

本目录 MVP 目标：
- 手写可审计的堆排序实现，而不是调用现成黑箱排序；
- 输出比较次数、交换次数、堆化调用次数与关键轨迹；
- 使用 `numpy`/`pandas` 做结果校验与可视化展示。

## R02

算法定位：
- 属于比较排序，理论最坏时间复杂度稳定在 `O(n log n)`；
- 与快速排序相比，它没有 `O(n^2)` 的最坏退化风险；
- 与归并排序相比，它是原地排序（额外空间 `O(1)`），但通常缓存局部性较差。

因此，堆排序常用于“需要最坏复杂度保证且希望低额外内存”的场景。

## R03

问题定义：
- 输入：长度为 `n` 的可比较序列 `A`（本实现限制为有限实数序列）；
- 输出：非降序序列 `A'`，满足：
  - 有序性：`A'[0] <= A'[1] <= ... <= A'[n-1]`；
  - 排列保持：`A'` 与 `A` 由相同元素多重集组成。

堆排序通过“最大堆 + 尾部固定”机制实现该目标。

## R04

核心思想：
1. 将数组重排为最大堆（父节点始终不小于子节点）。
2. 堆顶 `A[0]` 是当前堆中最大值，将其与末尾 `A[end]` 交换。
3. 末尾位置 `end` 变为已排序区，不再参与堆操作。
4. 对缩小后的堆区间 `[0, end-1]` 执行下滤（sift-down）恢复最大堆。
5. 重复直到 `end == 1`。

不变量：每轮提取后，后缀区间始终有序且包含当前全局最大的若干元素。

## R05

数据结构与状态量：
- 主数组：`arr`（列表，原地更新）；
- 堆边界：`end`（当前堆最后下标）；
- 下滤指针：`root`、`left`、`right`、`largest`；
- 统计量：
  - `comparisons`：元素大小比较次数；
  - `swaps`：交换次数；
  - `heapify_calls`：下滤函数调用次数；
- 轨迹：记录阶段（建堆/提取/恢复）、关键下标和数组快照。

## R06

正确性要点（不变量证明思路）：
- 建堆阶段：从最后一个非叶节点向前下滤，处理到节点 `i` 时，其子树已是最大堆，下滤后以 `i` 为根的子树也成为最大堆；最终整段区间成为最大堆。
- 提取阶段：
  - 交换 `A[0]` 与 `A[end]` 后，`A[end]` 等于当前堆最大值，应处于最终位置；
  - 对 `[0, end-1]` 下滤恢复最大堆，不影响已固定后缀。
- 归纳可得：当 `end` 递减到 `1` 时，整个数组升序。

因此算法满足有序性与排列保持。

## R07

时间复杂度：
- 建堆：`O(n)`（自底向上的堆化总成本线性）；
- 提取 `n-1` 轮，每轮下滤 `O(log n)`；
- 总体：最好/平均/最坏均为 `O(n log n)`。

这是堆排序在工程上最重要的复杂度保证。

## R08

空间复杂度：
- 算法主体只使用常数个辅助变量，额外空间 `O(1)`；
- 若启用轨迹记录（本 MVP 为可审计性保留快照），会产生额外 `O(k * n)` 观测开销，`k` 为记录条数。

去掉轨迹即可恢复经典原地空间特性。

## R09

算法性质：
- 原地排序：是；
- 稳定性：否（跨层交换会打乱相等元素相对次序）；
- 自适应：弱（对“接近有序”输入无明显收益）；
- 最坏复杂度：`O(n log n)`，优于快速排序的最坏界。

## R10

边界与输入约束：
- 空数组、单元素数组：直接返回；
- 重复值、负数、浮点数：支持；
- 非有限值（`NaN/Inf`）或非数值输入：拒绝；
- 非一维输入：拒绝；
- 为避免污染调用方数据，本实现先复制再排序。

## R11

伪代码：

```text
heap_sort(A):
  build_max_heap(A)
  for end from n-1 down to 1:
    swap A[0], A[end]
    sift_down(A, root=0, end=end-1)
  return A

sift_down(A, root, end):
  while left_child(root) <= end:
    largest <- root
    if A[left] > A[largest]: largest <- left
    if right <= end and A[right] > A[largest]: largest <- right
    if largest == root: break
    swap A[root], A[largest]
    root <- largest
```

## R15

与常见排序算法对比：
- 对比快速排序：
  - 快排平均常数因子常更优；
  - 堆排序最坏复杂度更稳。
- 对比归并排序：
  - 归并稳定且顺序访问友好；
  - 堆排序原地、内存开销更低。
- 对比选择排序：
  - 选择排序 `O(n^2)`，堆排序扩展到大规模数据更可用。

## R18

`demo.py` 源码级流程（9 步，非黑箱）：
1. `main` 构造固定样例与随机样例，逐个调用 `run_case`。
2. `run_case` 先调用 `validate_numeric_sequence`，把输入规范化为 1D 有限浮点列表。
3. `heap_sort` 复制输入到 `arr`，初始化 `comparisons/swaps/heapify_calls/trace`。
4. 建堆阶段从最后非叶节点向前遍历，每个节点调用一次 `_sift_down`，把区间调整为最大堆。
5. `_sift_down` 内部反复比较 `root` 与左右子节点，找出最大值位置 `largest`。
6. 若 `largest != root`，交换两者并继续向下；直到子树满足堆性质或到达叶层。
7. 提取阶段从 `end=n-1` 递减到 `1`：先交换 `arr[0]` 与 `arr[end]`，把当前最大值放入最终位置。
8. 对缩小后的堆区间 `[0, end-1]` 再次调用 `_sift_down` 恢复最大堆，然后继续下一轮提取。
9. 排序结束后，仅用 `sorted` 与 `np.sort` 做一致性核验，`pandas` 负责把轨迹表格化输出；第三方库不参与主算法决策。
