## R01

幂级数解法用于求解在普通点附近解析的常微分方程。核心思想是把未知解写成
`y(x)=\sum_{n\ge0} a_n (x-x_0)^n`，再把方程中的导数与系数函数都展开为幂级数，比较同次幂系数得到递推关系。

本目录实现的是二阶线性方程的最小可运行 MVP：

`y''(x) + p(x) y'(x) + q(x) y(x) = 0`

其中 `p(x), q(x)` 由泰勒系数给出。实现重点是手写递推，不依赖黑箱 ODE 求解器来“代替算法本体”。

## R02

求解任务定义：

- 输入：
  - `p_coeffs`：`p(x)=\sum p_k x^k` 的系数数组；
  - `q_coeffs`：`q(x)=\sum q_k x^k` 的系数数组；
  - 初值 `y(0)=a0`、`y'(0)=a1`；
  - 截断阶数 `n_terms`。
- 输出：
  - 截断解 `y_N(x)=\sum_{n=0}^{N-1} a_n x^n` 的系数 `a_n`；
  - 在采样网格上的近似值；
  - 与参考解的最大绝对误差；
  - ODE 残差 `y_N''+p y_N'+q y_N` 的数值检查。

## R03

设

- `y(x)=\sum_{n\ge0} a_n x^n`
- `p(x)=\sum_{k\ge0} p_k x^k`
- `q(x)=\sum_{k\ge0} q_k x^k`

则

- `y'(x)=\sum_{n\ge0}(n+1)a_{n+1}x^n`
- `y''(x)=\sum_{n\ge0}(n+2)(n+1)a_{n+2}x^n`

代入 `y'' + p y' + q y = 0` 并比较 `x^n` 系数，得到递推：

`(n+2)(n+1)a_{n+2} + \sum_{k=0}^{n} p_k (n-k+1)a_{n-k+1} + \sum_{k=0}^{n} q_k a_{n-k} = 0`

所以

`a_{n+2} = - (\sum p_k (n-k+1)a_{n-k+1} + \sum q_k a_{n-k}) / ((n+2)(n+1))`。

这就是 `demo.py` 的核心实现。

## R04

算法流程（高层）如下：

1. 检查输入系数、阶数、初值是否合法。
2. 将 `p_coeffs/q_coeffs` 补零到 `n_terms` 长度，便于统一索引。
3. 初始化 `a[0]=y0, a[1]=dy0`。
4. 对 `n=0..n_terms-3` 用上式递推求 `a[n+2]`。
5. 得到截断多项式后，在网格点评估 `y_N(x)`。
6. 计算导数系数并评估残差 `y_N'' + p y_N' + q y_N`。
7. 与参考解比较最大误差，若超阈值则报错。
8. 输出每个样例的表格与汇总统计。

## R05

`demo.py` 的核心数据结构：

- `ODECase`（`@dataclass`）：
  - `name`：样例名；
  - `p_coeffs/q_coeffs`：方程系数级数；
  - `y0/dy0`：初值；
  - `n_terms`：截断项数；
  - `x_grid`：评估点；
  - `reference`：参考解函数；
  - `error_tol`：误差阈值。
- `numpy.ndarray`：用于存储级数系数和采样值，保证计算简洁可复现。

## R06

正确性保障来自三层：

1. 公式层：递推关系直接来自代入与同次幂比较。
2. 代数层：`derivative_coeffs` 对幂级数系数做解析求导，避免数值差分误差。
3. 数值层：
- 与参考解（解析 `cos(x)` 或高精度 `solve_ivp`）做最大误差校验；
- 直接检查 ODE 残差，确认截断多项式确实满足方程到对应阶次。

## R07

设截断项数为 `N`、评估点数为 `M`。

- 系数递推：外层 `n` 到 `N`，内层卷积求和到 `n`，总计 `O(N^2)`。
- 多项式评估：每次 `polyval` 近似 `O(N)`，在 `M` 点上约 `O(MN)`。
- 总体时间复杂度：`O(N^2 + MN)`。
- 空间复杂度：`O(N + M)`（保存系数和评估向量）。

## R08

边界与异常处理：

- `n_terms < 2` 直接报错（至少要容纳 `a0,a1`）。
- 系数数组不是一维时抛 `ValueError`。
- 负导数阶次（`order < 0`）抛 `ValueError`。
- `airy_like_reference` 仅支持 `x>=0` 的示例区间，若传入负值会报错。
- 误差超过阈值时抛 `RuntimeError`，让自动验证可直接捕获失败。

## R09

MVP 取舍说明：

- 保留：
  - 手写递推（算法本体）；
  - 两个固定样例（常系数 + 变系数）；
  - 参考解与残差双重校验。
- 不做：
  - Frobenius 法（含奇点指数方程）；
  - 自适应阶数选择与收敛半径自动估计；
  - 高阶稀疏卷积优化。

目标是“实现短小、流程完整、可验证”。

## R10

`demo.py` 函数职责：

- `_pad_coeffs`：系数数组补零/截断。
- `power_series_coefficients`：按递推式生成 `a_n`。
- `eval_series`：评估幂级数。
- `derivative_coeffs`：生成一阶/二阶导的级数系数。
- `ode_residual`：计算 `y''+p y'+q y`。
- `airy_like_reference`：用高精度 IVP 构造参考解（仅用于验证）。
- `run_case`：单样例运行、打印、阈值断言。
- `build_cases`：构造固定样例集合。
- `main`：执行全部样例并汇总。

## R12

输出信息解释：

- `n_terms`：该样例采用的幂级数截断项数。
- `max|approx-ref|`：网格点上近似解与参考解的最大绝对误差。
- `max|ODE residual|`：网格点上方程残差最大值。
- 表格列：
  - `x`：采样点；
  - `approx`：幂级数近似；
  - `ref`：参考解；
  - `abs_err`：绝对误差。
- `Summary`：全部样例中的最差误差与最差残差。

## R13

内置最小测试集：

1. `y'' + y = 0, y(0)=1, y'(0)=0`
- 参考解 `cos(x)`；
- 目标：验证常系数方程下的高精度与系数递推正确性。

2. `y'' - x y = 0, y(0)=1, y'(0)=0`
- 变系数方程（Airy-like）；
- 参考解使用高精度 `solve_ivp(DOP853)` 数值积分；
- 目标：验证实现对非平凡 `q(x)` 的适用性。

## R14

关键参数与经验：

- `n_terms`：截断阶数，越大通常越准，但计算成本上升。
- `x_grid`：采样区间越远离展开中心 `x=0`，同样精度往往需要更高阶项。
- `error_tol`：验收阈值，建议与样例难度和参考解精度匹配。

经验上，本 demo 在 `x∈[0,1.2]` 内使用 `20~30` 项即可达到较高精度。

## R15

与其他 ODE 数值法对比：

- 对比 Runge-Kutta：
  - RK 在时间推进上灵活，适合大区间逐步积分；
  - 幂级数法在展开点邻域内能给出解析型多项式表达。
- 对比有限差分：
  - 差分离散化更偏边值问题网格求解；
  - 幂级数更适合局部解析结构和高阶导信息显式利用。
- 对比黑箱 `solve_ivp`：
  - `solve_ivp` 是通用数值积分器；
  - 本实现展示的是“如何从方程系数推导并构造级数解”的算法细节。

## R16

典型应用：

- 教学场景：展示 ODE 的解析构造法与递推推导。
- 特殊函数近似：在展开中心附近快速评估函数值。
- 作为数值法初始化：给射击法、边值法提供高质量初值。
- 系统辨识中的局部模型：在小区间内构造可微、可解释近似。

## R18

`demo.py` 源码级流程（8 步）：

1. `main()` 调用 `build_cases()`，生成两个固定 ODE 样例（简谐方程与 Airy-like 方程）。
2. `run_case()` 对每个样例先调用 `power_series_coefficients()`，依据递推式求出 `a_0..a_{N-1}`。
3. 在 `power_series_coefficients()` 内，循环 `n=0..N-3`，分别计算 `p*y'` 与 `q*y` 的卷积和，再解出 `a_{n+2}`。
4. `run_case()` 使用 `eval_series()` 在 `x_grid` 上计算截断解 `y_N(x)`。
5. `run_case()` 通过样例提供的 `reference` 计算参考值：简谐样例用 `cos`，Airy-like 样例用 `airy_like_reference()`。
6. `airy_like_reference()` 内部调用 `scipy.integrate.solve_ivp(DOP853)` 只做“参考真值”构造，不参与级数递推本体。
7. `run_case()` 调用 `ode_residual()`，后者通过 `derivative_coeffs()` 构造 `y_N'`、`y_N''` 并计算残差 `y_N''+p y_N'+q y_N`。
8. 若 `max|approx-ref|` 超过样例阈值则抛错；全部样例通过后，`main()` 打印汇总并结束。
