# 坡印廷矢量 (Poynting Vector)

- UID: `PHYS-0168`

## R01

坡印廷矢量用于描述电磁场的能流密度（单位面积单位时间的能量传输率）：

`S = (1 / mu0) * (E x B)`

在 SI 制下，`S` 的单位是 `W/m^2`。其方向给出电磁能传播方向，大小表示能流强度。

## R02

在电动力学里，坡印廷矢量把“场的局部值”直接转成“可计量的功率流”：
- 天线辐射功率评估；
- 波导/自由空间中的能量输运分析；
- 电磁仿真的能量守恒与边界通量校验。

它是从“场方程”走向“工程功率预算”的桥梁变量。

## R03

本条目 MVP 的目标是做一个最小闭环，而非大规模电磁求解器：
1. 构造真空中一维右行高斯电磁脉冲 `E(x,t), B(x,t)`；
2. 显式计算 `S=(1/mu0)E×B` 和能量密度 `u`；
3. 数值验证局部守恒 `du/dt + dSx/dx ~= 0`；
4. 数值验证全局守恒 `d/dt ∫u dx + (S_right - S_left) ~= 0`；
5. 在探测截面上积分能流，并与解析高斯积分对比。

## R04

`demo.py` 使用的核心方程：

1. 坡印廷矢量定义：`S = (1/mu0) (E x B)`。
2. 真空能量密度：`u = 0.5 * (eps0 * |E|^2 + |B|^2 / mu0)`。
3. 一维右行波构造：`E_y(x,t) = E0 * exp(-((x - c t - x0)^2)/(2 sigma^2))`。
4. 真空平面波关系：`B_z = E_y / c`。
5. 因而有：`S_x = E_y^2/(mu0 c)` 且 `S_x = c * u`。
6. 坡印廷定理（无源 1D 形式）：`du/dt + dS_x/dx = 0`。
7. 有限区域能量平衡：`d/dt ∫ u dx + [S_x(L)-S_x(0)] = 0`。

## R05

设空间网格 `nx`、时间网格 `nt`：
- 场构造：`O(nt * nx)`；
- `S`、`u` 计算：`O(nt * nx)`；
- 局部守恒差分与残差：`O(nt * nx)`；
- 全局能量与探针积分：`O(nt + nx)`（相对主项可忽略）。

总体时间复杂度 `O(nt * nx)`，空间复杂度 `O(nt * nx)`。

## R06

`demo.py` 的输出分四组指标：
- `pointwise`：`S_x = c*u` 点对点相对误差，以及主脉冲区平均能流方向指示；
- `local_continuity`：`du/dt + dSx/dx` 的相对 `L2` 残差和最大绝对残差；
- `global_balance`：`d/dt ∫u dx + (S_right-S_left)` 的相对 `L2` 残差；
- `probe_transport`：探测截面能量通量积分（数值）与解析值对比误差。

脚本通过 `assert` 直接给出通过/失败，不需要人工判读图像。

## R10

正确性直觉：
1. 对真空行波，`E` 与 `B` 正交且同步，`E×B` 必沿传播方向；
2. 平面波中 `S_x = c*u` 是标准解析恒等式；
3. 局部守恒 `du/dt + dSx/dx = 0` 是坡印廷定理在无源 1D 下的直接形式；
4. 把局部守恒在空间上积分，就得到全局边界通量平衡。

如果代码正确，三层检查应同时成立。

## R11

数值稳定与精度措施：
- 对 `x`、`t` 网格做严格均匀性检查；
- 导数采用中心差分，端点采用单边差分；
- 守恒残差只在内区统计（去掉两侧若干网格点）；
- 使用相对 `L2` 误差而非绝对量，避免单位尺度误导阈值。

## R12

关键参数（`PulseConfig`）：
- `e0`：电场峰值（`V/m`），决定能流幅值（平方关系）；
- `sigma_m`：脉冲空间宽度，决定能量包络宽度；
- `x0_m`：初始中心位置；
- `domain_length_m`、`nx`：空间区间与分辨率；
- `t_end_s`、`nt`：仿真时长与时间分辨率；
- `area_m2`：探测截面积；
- `probe_x_m`：探测面位置。

调参建议：
- 降低守恒残差优先增大 `nx` 和 `nt`；
- 若要看更完整穿越过程，可增加 `t_end_s`；
- 若 `probe_x_m` 太靠近边界，解析/数值比较会受截断影响。

## R14

常见失效模式：
1. 网格不均匀，却仍使用固定步长差分；
2. 分辨率不足导致守恒残差上升；
3. 误把 `B=E*c`（单位错误）写进代码；
4. 叉乘顺序写反（`B x E` 会改变方向）；
5. 把全局平衡的边界通量符号写错。

## R18

`demo.py` 源码级算法流程（8 步）：

1. `PulseConfig` 固定实验参数（网格、脉冲宽度、探针位置、截面积），定义可复现实验域。  
2. `_assert_uniform_grid` 检查 `x/t` 是否均匀，避免差分公式在非法网格上静默失真。  
3. `build_traveling_pulse` 构造 `E_y(x,t)=E0*exp(-((x-ct-x0)^2)/(2sigma^2))` 与 `B_z=E_y/c`，并组装三维向量场。  
4. 在同一函数中显式计算 `S=(1/mu0)E×B` 与 `u=0.5*(eps0|E|^2+|B|^2/mu0)`，不把关键物理关系交给黑盒库。  
5. `centered_difference` 对 `u` 和 `S_x` 做中心差分，`verify_local_energy_continuity` 计算 `du/dt + dSx/dx` 的相对残差。  
6. `verify_global_energy_balance` 先用 `np.trapezoid` 计算 `U(t)=∫u dx`，再检查 `dU/dt + (S_right-S_left)` 的残差。  
7. `gaussian_flux_integral_over_window` 用误差函数 `erf` 给出有限时间窗解析通量，`verify_probe_energy_transport` 与数值积分逐项比对。  
8. `run_demo/main` 汇总各检验指标到 `pandas.DataFrame` 并打印，同时由 `assert` 执行自动验收，保证 `uv run python demo.py` 一次性完成非交互校验。  

第三方库的角色是基础数值算子（数组、积分、表格），而算法主流程（波场构造、能流计算、守恒残差定义、解析对照）都在源码中逐步展开。
