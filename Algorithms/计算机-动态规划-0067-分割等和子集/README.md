# 分割等和子集

- UID: `CS-0050`
- 学科: `计算机`
- 分类: `动态规划`
- 源序号: `67`
- 目标目录: `Algorithms/计算机-动态规划-0067-分割等和子集`

## R01

题目目标：判断一个非负整数数组是否可以被分成两个子集，使两个子集元素和相等。

核心等价变形：
- 设总和为 `total`；
- 若 `total` 为奇数，则不可能平分；
- 若 `total` 为偶数，问题等价为是否存在子集和恰好等于 `target = total / 2`。

因此这是标准的 `0/1 子集和`（Subset Sum）动态规划问题。

## R02

形式化定义：
- 输入：一维非负整数序列 `nums`，长度为 `n`。
- 输出：
  - `can_partition`：是否可分割为两个等和子集；
  - 若可分割，给出一组可复现的划分方案（子集 A 与子集 B 的下标和值）。

约束说明：
- 元素只能使用一次（0/1 选择）；
- 子集 A 与子集 B 不重叠，且并集覆盖全部元素。

## R03

状态设计（二维 DP + 回溯）：
- 定义 `dp[i][s]`：只使用前 `i` 个元素（下标 `0..i-1`）时，是否能凑出和 `s`。
- 边界：
  - `dp[0][0] = True`；
  - `dp[0][s>0] = False`。
- 转移：
  - 不选第 `i-1` 个元素：`dp[i-1][s]`；
  - 选第 `i-1` 个元素（需 `s >= nums[i-1]`）：`dp[i-1][s-nums[i-1]]`；
  - 合并：`dp[i][s] = dp[i-1][s] or dp[i-1][s-nums[i-1]]`。

最终判定：`dp[n][target]`。

## R04

正确性直觉：
- 对每个元素只有“选/不选”两种互斥决策；
- `dp[i][s]` 穷尽了前 `i` 个元素形成和 `s` 的全部可能方式；
- 转移完全覆盖可行解空间且不遗漏；
- 因此 `dp[n][target]` 为真当且仅当存在目标子集，等价于可分割等和子集。

## R05

复杂度：
- 设 `target = total/2`。
- 二维 DP（可回溯方案）：
  - 时间复杂度 `O(n * target)`；
  - 空间复杂度 `O(n * target)`。
- 一维滚动 DP（仅判定可行性）：
  - 时间复杂度 `O(n * target)`；
  - 空间复杂度 `O(target)`。
- 暴力 DFS（仅小规模校验）为指数复杂度。

## R06

`demo.py` 提供三条实现路径：
- `partition_equal_subset_dp_trace`：主实现，二维 DP 并回溯出一组具体划分；
- `can_partition_dp_optimized`：一维滚动数组，仅返回可行性；
- `can_partition_bruteforce`：记忆化 DFS 精确判定，用于随机回归交叉验证。

脚本会自动断言三者结果一致（在小规模样本上）。

## R07

输入校验策略：
- 输入必须是一维序列；
- 拒绝 `nan`/`inf`；
- 数值需可安全解释为整数（非整数浮点报错）；
- 元素需非负；
- 空数组视为可平分（两个空子集，和都为 0）。

统一校验函数为 `to_1d_nonnegative_int_array`。

## R08

方案重建策略：
- 在 `dp[n][target] == True` 时，从 `(i=n, s=target)` 逆向回溯；
- 若 `dp[i-1][s] == True`，优先判定“不选第 i-1 个元素”；
- 否则若 `s >= nums[i-1]` 且 `dp[i-1][s-nums[i-1]] == True`，判定“选中”；
- 回溯结束后得到子集 A，下标补集即为子集 B。

平局时优先“不选”，用于稳定输出并保持可复现。

## R09

边界条件覆盖：
- 空数组 `[]`：可分割；
- 单元素 `[x]`：`x>0` 时不可分割；
- 总和奇数：立即不可分割；
- 含 `0` 数组（如 `[0,0,0]`）：可分割；
- 多解场景（如 `[1,5,11,5]`）：返回一组稳定可行解。

## R10

验证策略：
- 固定样例：覆盖可分/不可分、奇偶总和、含零、空数组；
- 随机回归：
  - 生成小规模随机数组；
  - 对比 `dp_trace`、`dp_optimized`、`bruteforce` 判定结果；
  - 对可分样例检查重建方案是否完整覆盖且两侧和相等。

## R12

`demo.py` 输出字段说明：
- `can_partition`：是否可分割；
- `target`：目标半和；
- `subset_a_indices / subset_b_indices`：两侧下标划分；
- `subset_a_values / subset_b_values`：两侧对应元素值；
- `checks`：
  - `valid_partition`：划分结构是否合法；
  - `optimized_match / bruteforce_match`：不同实现判定是否一致；
  - `balanced_sum`：可分场景下两侧元素和是否相等。

## R13

MVP 取舍：
- 采用 `numpy + 标准库` 的最小技术栈；
- 主算法自己实现，不依赖第三方黑盒求解器；
- 优先保证“可解释 + 可验证 + 可运行”；
- 不引入大型训练框架或复杂工程封装。

## R14

与相关问题的关系：
- 本题是 `0/1 背包可行性判定` 的直接实例；
- `目标和（Target Sum）`、`最小子集和差` 等问题可由类似 DP 思路扩展；
- 多维约束版本可推广到多目标背包，但状态维度会快速增长。

## R15

可扩展方向：
- 改为返回“字典序最小”或“元素个数最少”的等和划分；
- 输出所有可行划分数量（计数 DP）；
- 增加位运算 bitset 版本用于更快判定；
- 增加更大规模的性能基准测试。

## R16

工程注意点：
- 一维滚动 DP 必须倒序更新 `s`，否则会错误复用同一元素；
- 回溯逻辑需与 DP 定义严格一致，否则方案可能不合法；
- 暴力解仅用于小规模回归，避免误用到大输入；
- 输入校验应前置，减少隐式类型问题。

## R18

`demo.py` 源码级算法流程（8 步）：
1. `main` 构造固定样例，逐个调用 `run_case`。
2. `run_case` 先执行 `partition_equal_subset_dp_trace`：检查总和奇偶并构建 `dp[i][s]`。
3. 当 `dp[n][target]` 为真时，从 `(n, target)` 逆向回溯，恢复子集 A 的下标。
4. 通过下标补集构造子集 B，得到完整二分方案与两侧值列表。
5. `run_case` 再调用 `can_partition_dp_optimized`，用一维倒序 DP 复算可行性。
6. 对小规模输入，额外调用 `can_partition_bruteforce`（记忆化 DFS）做精确对照。
7. `validate_partition_result` 检查方案覆盖完整、无重复、两侧和一致。
8. 固定样例通过后，`randomized_regression` 执行多轮随机交叉验证，确认实现稳定正确。
