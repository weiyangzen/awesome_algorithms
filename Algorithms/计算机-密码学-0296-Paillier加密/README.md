## R01

Paillier 加密是 1999 年提出的概率型公钥加密方案，核心特点是“加法同态”：

- 不解密即可在密文域完成加法；
- 解密后结果等于明文和（模 `n`）；
- 同一明文每次加密都会因为随机因子不同而产生不同密文。

这使它非常适合隐私统计、匿名投票、联邦计数等“只需聚合、无需暴露单条明文”的场景。

## R02

Paillier 工作在模 `n^2` 的整数群上：

- 选两大素数 `p, q`，令 `n = p*q`；
- 明文空间是 `Z_n`；
- 密文空间是 `Z_{n^2}*` 的一个可计算子结构。

常见参数化中取 `g = n + 1`，可显著简化实现与正确性证明。

## R03

安全性基础是复合剩余类相关困难问题（Composite Residuosity Assumption）。

直观上，攻击者即便拿到公钥 `(n, g)` 与密文 `c`，也很难区分其对应的明文值，因为密文中含有随机项 `r^n`。只要 `r` 每次独立随机采样且与 `n` 互素，加密具有语义安全所需的随机化特征。

## R04

密钥生成（本条目实现）流程：

1. 生成两个奇素数 `p, q`（教学 MVP 用较小位数，便于运行）。
2. 计算 `n = p*q` 与 `n_sq = n*n`。
3. 计算 `lambda = lcm(p-1, q-1)`。
4. 取 `g = n + 1`。
5. 计算 `u = g^lambda mod n_sq`。
6. 计算 `L(u) = (u-1)/n`。
7. 计算 `mu = L(u)^(-1) mod n`。
8. 公钥：`(n, n_sq, g)`；私钥：`(lambda, mu)`。

## R05

加密公式（明文 `m` 满足 `0 <= m < n`）：

1. 随机采样 `r in [1, n-1]` 且 `gcd(r, n) = 1`；
2. 计算 `c1 = g^m mod n_sq`；
3. 计算 `c2 = r^n mod n_sq`；
4. 密文 `c = (c1 * c2) mod n_sq`。

由于 `r` 每次不同，`m` 相同也会得到不同密文。

## R06

解密公式：

1. 计算 `x = c^lambda mod n_sq`；
2. 计算 `L(x) = (x-1)/n`；
3. 还原 `m = (L(x) * mu) mod n`。

其中 `L(.)` 是 Paillier 的关键变换函数。

## R07

同态性质（加法）说明：

- 若 `c1 = Enc(m1)`、`c2 = Enc(m2)`，则
  `c_add = c1 * c2 mod n_sq = Enc((m1 + m2) mod n)`。
- 若 `k` 为常数，则
  `c_mul = c1^k mod n_sq = Enc((k*m1) mod n)`。

这就是“密文乘法对应明文加法，密文幂对应明文数乘”的来源。

## R08

复杂度（大整数模幂主导）：

- 单次加密：2 次模幂（`g^m` 与 `r^n`），约 `O(log n)` 次大整数乘法级别；
- 单次解密：1 次模幂（`c^lambda`），约 `O(log n)`；
- 同态加法：1 次模乘；
- 同态数乘：1 次模幂。

空间复杂度主要由大整数密钥与密文存储决定。

## R12

程序内置验证项：

1. `decrypt(encrypt(m)) == m`；
2. 同一明文多次加密密文通常不同（概率型特性）；
3. `decrypt(c1*c2 mod n^2) == (m1+m2) mod n`；
4. `decrypt(c1^k mod n^2) == (k*m1) mod n`；
5. 多条加密投票聚合后解密，等于明文求和。

任一条件不满足会触发 `assert`，便于自动化检查。

## R13

与 RSA 的差异（简述）：

- RSA 原生不具备 Paillier 这种“加法同态聚合”能力；
- Paillier 密文更适合隐私统计而非通用高吞吐数据加密；
- 两者都需要大整数运算，但安全假设与可同态操作类型不同。

## R14

工程落地注意点：

- 生产参数应使用至少 2048-bit 以上模数级别（或更高）；
- 必须使用密码学安全随机源采样 `r`；
- 明文通常要配合编码、范围约束与完整性保护机制；
- 需要防侧信道与实现层错误（时序、异常分支泄漏等）。

## R15

常见错误：

- 未确保 `gcd(r, n)=1`，导致密文退化或不可逆问题；
- 忘记在 `n^2` 下运算，错误地只在 `n` 下取模；
- 明文超出 `Z_n` 范围却未处理；
- 把教学参数误当生产安全参数。

## R18

`demo.py` 源码级算法流程（8 步）：

1. `main()` 调用 `generate_keypair(bits=32)`，先生成两个素数 `p, q`。  
2. 在密钥生成中计算 `n, n_sq, lambda=lcm(p-1,q-1)`，固定 `g=n+1`。  
3. 计算 `u = g^lambda mod n_sq`，再用 `L(u)=(u-1)//n` 与模逆得到 `mu`。  
4. `encrypt` 对明文 `m` 采样与 `n` 互素的随机 `r`，构造 `c = g^m * r^n mod n_sq`。  
5. `decrypt` 对密文计算 `x = c^lambda mod n_sq`，再执行 `m = L(x)*mu mod n` 还原明文。  
6. `add_ciphertexts` 直接做密文乘法 `c1*c2 mod n_sq`，实现明文加法同态。  
7. `mul_ciphertext_by_constant` 计算 `c^k mod n_sq`，实现明文常数倍同态。  
8. `main()` 通过断言验证：基本加解密、概率型加密、同态加法、同态数乘、加密投票聚合。  

第三方库说明：本 MVP 未调用现成 Paillier 实现库，所有关键算法路径均在 `demo.py` 手写展开，可逐行追踪。
