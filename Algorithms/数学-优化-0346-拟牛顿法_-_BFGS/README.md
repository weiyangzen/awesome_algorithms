## R01

BFGS（Broyden-Fletcher-Goldfarb-Shanno）是经典拟牛顿法，用于求解无约束光滑优化问题：

\[
\min_{x \in \mathbb{R}^n} f(x)
\]

它不直接计算 Hessian 矩阵 \(\nabla^2 f(x)\)，而是迭代维护其逆矩阵近似 \(H_k \approx (\nabla^2 f(x_k))^{-1}\)，从而在“每步成本”和“收敛速度”之间取得平衡。

## R02

适用场景：
- 目标函数可微，且梯度可计算，但 Hessian 昂贵或不稳定。
- 中等维度无约束优化（机器学习参数拟合、曲线拟合、统计估计）。
- 希望比梯度下降更快，又不想承担完整牛顿法二阶导开销。

## R03

核心思想：
- 搜索方向：\(p_k = -H_k g_k\)，其中 \(g_k = \nabla f(x_k)\)。
- 通过线搜索找步长 \(\alpha_k\)，更新 \(x_{k+1} = x_k + \alpha_k p_k\)。
- 定义 \(s_k = x_{k+1} - x_k\), \(y_k = g_{k+1} - g_k\)。
- 令逆 Hessian 近似满足割线条件（secant condition）：\(H_{k+1} y_k = s_k\)。
- BFGS 逆矩阵更新：

\[
\rho_k = \frac{1}{y_k^\top s_k},\quad
H_{k+1} = (I-\rho_k s_k y_k^\top)H_k(I-\rho_k y_k s_k^\top)+\rho_k s_k s_k^\top
\]

当 \(y_k^\top s_k > 0\) 且 \(H_k\) 对称正定时，\(H_{k+1}\) 仍保持对称正定。

## R04

输入输出约定（对应 `demo.py`）：
- 输入：
  - 初始点 `x0`（默认 `[-1.2, 1.0]`）。
  - 容差 `tol`、最大迭代数 `max_iter`。
- 输出：
  - 最优近似点 `x*`、最终函数值 `f(x*)`。
  - 迭代次数、是否收敛、梯度范数。
  - 函数/梯度评估次数与若干迭代轨迹。

示例问题使用 Rosenbrock 函数，理论最优点为 `[1, 1]`。

## R05

伪代码：

```text
x <- x0
H <- I
for k in 0..max_iter-1:
    g <- grad(x)
    if ||g|| < tol: stop

    p <- -H g
    if g^T p >= 0:           # 数值保护
        p <- -g
        H <- I

    alpha <- backtracking_line_search(f, x, g, p)
    x_new <- x + alpha p
    g_new <- grad(x_new)

    s <- x_new - x
    y <- g_new - g

    if y^T s > eps:
        rho <- 1 / (y^T s)
        V <- I - rho s y^T
        H <- V H V^T + rho s s^T
    else:
        H <- I               # 曲率异常时重置

    x <- x_new
return x
```

## R06

正确性要点（工程版）：
- 若 `H_k` 正定，则 `p_k=-H_k g_k` 是下降方向（`g_k^T p_k < 0`）。
- 回溯线搜索（Armijo 条件）保证每步有足够下降：

\[
f(x_k+\alpha p_k) \le f(x_k) + c_1\alpha g_k^\top p_k
\]

- 当 `y_k^T s_k > 0` 时，BFGS 更新保持 `H_{k+1}` 对称正定，下一步方向仍倾向下降。
- 在光滑、局部强凸等常见条件下，BFGS 具备良好局部收敛性质；实践中通常显著快于纯梯度下降。

## R07

复杂度（每次迭代）：
- 时间：`O(n^2)`（矩阵向量乘 + rank-2 更新）。
- 空间：`O(n^2)`（存储 `H_k`）。
- 额外线搜索会带来若干次函数评估，设回溯次数为 `m`，则额外成本约 `O(m)` 次 `f` 计算。

## R08

与相关方法对比：
- 梯度下降：`O(n)` 存储，单步便宜，但在病态问题上常慢。
- 牛顿法：收敛快，但需 Hessian/线性系统，单步代价高。
- DFP：也是拟牛顿更新，BFGS 通常在数值稳定性和实际表现上更常用。
- L-BFGS：只存最近若干 `(s,y)`，空间降到 `O(nm)`，适合高维大模型。

## R10

边界与异常处理：
- `x0` 必须是一维向量，且长度大于 0。
- 若出现 `g^T p >= 0`（方向非下降），回退到最速下降并重置 `H`。
- 若 `y^T s <= 1e-12`（曲率信息退化），跳过/重置更新，避免除零或非正定污染。
- 线搜索超过最大回溯次数后接受最小步长，保证算法可继续前进。

## R14

常见实现错误：
- 把更新式写错成 `H = H + ...` 的非标准形式，导致不稳定。
- 忘记检查 `y^T s`，导致除零或负曲率引发矩阵失真。
- 线搜索只做固定步长，不检查 Armijo 条件，容易震荡或发散。
- 把梯度向量形状写成列向量/行向量混乱，触发广播错误。

局限与取舍：
- 完整 BFGS 需 `O(n^2)` 内存，不适合超高维。
- 在非凸区域可能遇到负曲率，需额外保护策略。

## R18

源码级算法流程（`demo.py` 实现，非黑箱）:
1. 初始化 `x=x0`、`H=I`，计算 `f(x)` 与 `g=∇f(x)`。  
2. 若 `||g|| < tol` 直接收敛；否则计算候选方向 `p=-H@g`。  
3. 若方向异常（`g^T p >= 0`），回退为最速下降 `p=-g`，并将 `H` 重置为单位阵。  
4. 对方向 `p` 执行 Armijo 回溯线搜索，逐步缩小步长 `alpha`，直到满足充分下降。  
5. 更新点 `x_new = x + alpha p`，重新计算 `g_new = ∇f(x_new)`。  
6. 构造 `s = x_new - x` 与 `y = g_new - g`，检查曲率量 `y^T s`。  
7. 若 `y^T s` 足够大，按 BFGS 逆矩阵公式更新 `H`；否则将 `H` 重置，避免数值病态传播。  
8. 写入迭代日志并进入下一轮，直到达到梯度阈值或最大迭代次数，最后输出收敛信息与轨迹摘要。
