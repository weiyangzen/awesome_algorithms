# Schönhage-Strassen算法

- UID: `MATH-0043`
- 学科: `数学`
- 分类: `算法`
- 源序号: `43`
- 目标目录: `Algorithms/数学-算法-0043-Schönhage-Strassen算法`

## R01

Schönhage-Strassen 算法是用于超大整数乘法的快速算法，核心是把整数乘法转成卷积，再用快速变换计算卷积。

- 目标问题: 计算超大整数 `A * B`。
- 核心思想: 在环 `Z/(2^K + 1)Z` 中做 FFT/NTT 风格变换，把卷积从 `O(n^2)` 降到 `O(n log n)`（卷积层）。
- 位复杂度结论: 经典分析为 `O(n log n log log n)`（`n` 为位数）。
- 典型特征: 使用 2 的幂作为根，乘以根可被实现为“移位 + 符号处理”的廉价操作（理论实现里尤其重要）。

## R02

- 提出时间: 1971 年。
- 提出者: Arnold Schönhage 与 Volker Strassen。
- 历史地位: 是大整数乘法从 Karatsuba/Toom-Cook 迈向“准线性复杂度”时代的关键里程碑。
- 后续发展: 之后出现 Fürer 算法与 Harvey–van der Hoeven 结果，进一步改进渐近复杂度；但 Schönhage-Strassen 仍是工程与教学中的核心经典。

## R03

当整数位数非常大时，朴素乘法 `O(n^2)` 会成为瓶颈。核心痛点是“系数两两相乘”太多。Schönhage-Strassen 的突破在于把问题重写为多项式卷积，并把卷积交给快速变换框架处理，显著减少乘法次数。

适用场景包括高精度计算、计算机代数系统、密码学参数生成与大数论实验。

## R04

复杂度分层如下。

- 朴素乘法: `O(n^2)`。
- Karatsuba: `O(n^1.585)`。
- Toom-Cook（典型）：`O(n^1.46)` 左右（按分裂策略不同）。
- Schönhage-Strassen: `O(n log n log log n)`（位复杂度）。

说明: 本目录 `demo.py` 是教学型 MVP，常数因子和实现细节未做工业级优化，小规模输入上可能慢于 Python 内建乘法或简单双重循环。

## R05

- 额外空间主要来自变换数组与中间缓存。
- 典型空间量级: `O(n)`（按分块长度计）。
- 实际开销还受以下因素影响:
  1. 分块基数（`limb_bits`）
  2. 变换长度（补齐到 2 的幂）
  3. 模数位宽（`2^K + 1` 的 `K`）

## R06

一个简化示例（思路级）:

1. 设基数 `B = 2^m`，把整数写成 `A = Σ a_i B^i`、`B = Σ b_i B^i`。
2. 这等价于多项式 `P(x)=Σ a_i x^i`、`Q(x)=Σ b_i x^i` 在 `x=B` 处的取值。
3. 先求系数卷积 `c_k = Σ a_i b_{k-i}`。
4. 再做进位传播，恢复整数乘积。

Schönhage-Strassen 的关键是第 3 步不用 `O(n^2)`，而用快速变换把卷积降为 `O(n log n)`。

## R07

算法意义可分三层。

- 理论层: 把整数乘法复杂度推进到准线性范畴。
- 方法层: 展示“变换域计算 + 逆变换 + 进位重建”的通用范式。
- 工程层: 影响了多精度库在大规模区间的乘法策略设计（通常按阈值切换不同算法）。

## R08

核心数学与机制。

- 卷积定理: 时域卷积等价于频域点乘。
- 环选择: `R = Z/(2^K + 1)Z`。
- 根的构造: 利用 `2^K ≡ -1 (mod 2^K+1)`，从 2 的幂中构造高阶单位根。
- 可逆条件: 变换长度与模数互素，保证逆变换中 `n^{-1}` 存在。
- 系数上界控制: 选择足够大的模数，避免卷积系数在模意义下回绕。

## R09

前提与边界。

- 输入: 任意符号整数（本实现支持正负与零）。
- 适用规模: 中大规模整数更有意义。
- 不适用点:
  - 小整数时，算法常数开销较高。
  - 若模数过小或参数不当，会出现系数回绕导致结果错误。

## R10

正确性可拆成三段。

1. 表示正确: 分块表示与原整数等价。
2. 卷积正确: NTT + 点乘 + INTT 在选定环中返回正确卷积系数。
3. 重建正确: 进位传播把卷积系数映射回唯一整数值。

当参数满足“根可逆 + 模数足够大不回绕”时，最终结果与数学乘积一致。

## R11

数值稳定性与误差。

- 本实现是纯整数环运算，不依赖浮点 FFT。
- 因此不存在浮点舍入误差累积问题。
- 主要风险不是“数值不稳定”，而是“参数选择错误导致模回绕”。

## R12

真实成本来源。

- 主成本在 NTT/INTT 中的大整数模乘与模加。
- Python 大整数的对象分配与垃圾回收也会带来常数开销。
- 当规模不大时，`O(n^2)` 的 schoolbook 往往更快（常数更小）。
- 随规模增长，变换法的增长率优势逐步体现。

## R13

- 近似误差: 无（整数精确算法）。
- 随机失败概率: 无（确定性流程）。
- 保证形式: 只要参数选择满足约束，输出即为精确乘积。

## R14

常见失效模式。

- 模数不足: 卷积系数超过模数，产生回绕错误。
- 根不满足阶条件: 变换不可逆或结果错乱。
- 进位实现错误: 会在高位出现系统性偏差。
- 符号处理遗漏: 负数乘法结果符号可能错误。

本目录 `demo.py` 用断言对比 Python 内建乘法来防止上述实现错误悄悄通过。

## R15

工程实现建议。

- 用固定 `limb_bits` 做分块，平衡分块数量与单系数大小。
- 自动推导 `K`，同时满足“阶条件 + 系数上界条件”。
- 用统一的 `carry_propagate` 处理卷积到整数的重建。
- 始终保留与语言内建乘法的回归测试。
- 在生产环境通常采用“阈值切换”：小规模走 schoolbook/karatsuba，大规模再切到变换法。

## R16

前驱、并行与后继关系。

- 前驱:
  - Schoolbook multiplication
  - Karatsuba
  - Toom-Cook
- 同类路线:
  - 基于复数 FFT 的整数乘法
  - 基于 NTT 的多项式乘法
- 后继:
  - Fürer 系列改进
  - Harvey–van der Hoeven 的渐近改进

常见应用: 高精度库、CAS、密码学与数论实验中的大整数核心运算。

## R17

本目录 Python MVP（`demo.py`）说明。

- 定位: 教学型、可运行、可验证的 Schönhage-Strassen 风格实现。
- 依赖: Python 3 标准库（`random`、`time`、`typing`）。
- 主要函数:
  1. `_choose_ring_params`: 自动选择 `2^K+1` 模数与根阶。
  2. `_ntt` / `_intt`: 迭代式原地变换。
  3. `_cyclic_convolution_ring`: 变换域点乘得到卷积。
  4. `_carry_propagate`: 进位重建。
  5. `schonhage_strassen_multiply`: 对外乘法接口。
- 演示内容:
  - 固定样例与随机样例正确性验证。
  - 与 schoolbook 实现做简单计时对比。
- 运行方式:
  - `python3 demo.py`

## R18

源码级算法流（对应 `demo.py`，8 步）。

1. `schonhage_strassen_multiply` 接收 `x, y`，处理零值与符号。
2. `_to_limbs` 把整数拆成基数 `2^limb_bits` 的小端块序列。
3. `_cyclic_convolution_ring` 计算卷积长度 `n`（补齐到 2 的幂）。
4. `_choose_ring_params` 自动选择 `mod = 2^K + 1`，并验证 2 的阶可生成 `n` 阶单位根。
5. `_ntt` 对两个块序列做前向变换，得到频域表示。
6. 在频域逐点相乘，再用 `_intt` 逆变换回时域卷积系数。
7. `_carry_propagate` 对卷积系数做基数进位，得到规范块序列。
8. `_from_limbs` 把块序列还原为整数，并恢复符号返回最终乘积。

说明: 本实现没有把第三方库当作黑箱乘法调用，核心流程均在源码内显式展开。
