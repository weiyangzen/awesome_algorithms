## R01

问题定义：给出一个可运行的最小示例，验证“正则变换不会破坏哈密顿结构与动力学等价性”。

本条目选择 1 自由度谐振子：
- 原变量：`(q, p)`
- 变换后变量：`(Q, P)`
- 变换类型：由时间无关生成函数 `F2(q,P)` 诱导的线性剪切变换

MVP 的核心目标：
- 验证辛条件 `J^T Ω J = Ω`；
- 验证 Poisson 括号 `{Q,P}_{q,p} = 1`；
- 在两套变量下独立积分并比较轨迹映射误差。

## R02

正则变换是“保持哈密顿方程形式不变”的变量变换。它不是任意坐标替换，而要保持相空间辛结构。

在 1 自由度场景中，等价可从三条路径理解：
- 辛矩阵条件：Jacobian 满足 `J^T Ω J = Ω`；
- Poisson 结构：新变量仍满足基本括号关系；
- 动力学一致：由 `H(q,p)` 推到 `K(Q,P)` 后，哈密顿方程与原物理轨道一致。

## R03

本 MVP 的计算任务：
1. 定义谐振子哈密顿量 `H(q,p)`。
2. 定义生成函数 `F2(q,P)=qP-(alpha/2)q^2`。
3. 写出正则变换 `Q=q, P=p+alpha q` 及逆变换。
4. 写出变换后哈密顿量 `K(Q,P)=H(q(Q,P),p(Q,P))`。
5. 分别在 `(q,p)` 与 `(Q,P)` 中积分哈密顿方程。
6. 将 `(Q,P)` 轨迹逆映射回 `(q,p)`，比较最大误差。
7. 输出辛残差、Poisson 括号、能量漂移、链式法则残差等诊断量。

## R04

建模假设：
- 系统为无阻尼、无驱动的 1D 线性谐振子；
- 变换为时间无关正则变换（因此不额外引入 `∂F/∂t` 项）；
- 只验证最小可解释案例，不覆盖非线性多自由度复杂系统。

数值假设：
- 采用高精度 `DOP853` 作为积分器，目标是验证“变量框架等价性”，而非讨论辛积分长期误差优势。

## R05

原系统：
- `H(q,p) = p^2/(2m) + (k/2) q^2`
- `dq/dt = ∂H/∂p = p/m`
- `dp/dt = -∂H/∂q = -kq`

选取生成函数（II 类）：
- `F2(q,P)=qP-(alpha/2)q^2`
- `Q = ∂F2/∂P = q`
- `p = ∂F2/∂q = P-alpha q`，即 `P = p + alpha q`

新哈密顿量：
- `K(Q,P) = (P-alpha Q)^2/(2m) + (k/2)Q^2`

新变量中的哈密顿方程：
- `dQ/dt = ∂K/∂P = (P-alpha Q)/m`
- `dP/dt = -∂K/∂Q = (alpha/m)(P-alpha Q) - kQ`

## R06

`demo.py` 算法流程：
1. 校验参数合法性（正值、有限值、采样点数）。
2. 构造正逆变换函数与 Jacobian。
3. 计算辛条件残差与 Poisson 括号。
4. 在同一 `t_eval` 网格上积分原方程 `rhs_original`。
5. 用变换后的初值积分新方程 `rhs_transformed`。
6. 将新轨迹逆映射回旧变量，计算 `q/p` 轨迹最大绝对误差。
7. 分别计算 `H` 与 `K` 的相对漂移，并比较 `H-K`。
8. 用链式法则比对两套 RHS，输出最大残差并执行质量门禁断言。

## R07

设采样点数为 `N`：
- 两次积分总时间复杂度近似 `O(N)`（积分器内部常数不同）；
- 存储轨迹和诊断序列空间复杂度 `O(N)`；
- 额外结构检查（辛条件、Poisson、链式法则）均为 `O(1)` 或 `O(M)`（`M` 为抽样点数，远小于 `N`）。

## R08

数值策略：
- 两个系统使用同一 `t_eval`，消除重采样比较误差；
- 采用严格容差 `rtol=1e-11, atol=1e-13` 降低积分误差干扰；
- 误差指标使用“最大绝对误差 + 相对漂移”组合，兼顾状态一致性与守恒量稳定性；
- 加入链式法则 RHS 残差，避免仅凭轨迹接近造成误判。

## R09

适用场景：
- 分析力学教学中“正则变换不是形式技巧而是结构保持”演示；
- 哈密顿系统代码基线测试（变量替换前后结果一致性）；
- 后续扩展到更复杂生成函数和多自由度辛映射前的最小模板。

不适用场景：
- 强非线性、刚性或非哈密顿耗散系统；
- 需要长时间辛结构高保真的轨道模拟（应使用专门辛积分器）。

## R13

理论保证边界：
- 对时间无关生成函数，正则变换保证辛结构保持，哈密顿方程形式保持；
- 本例线性变换可解析验证 `J^TΩJ=Ω` 与 `{Q,P}=1`；
- 数值结果属于“理论结构 + 数值实现一致”的工程验证，不替代一般情形的形式证明。

## R14

常见失败模式与修复：
- 失败：把 `det(J)=1` 当作唯一判据。
  - 修复：必须同时检查 `J^TΩJ=Ω`。
- 失败：新哈密顿量 `K(Q,P)` 写错符号（尤其 `P-alpha Q` 项）。
  - 修复：从逆变换 `p=P-alpha Q` 重新代入推导。
- 失败：比较两轨迹时使用不同时间采样。
  - 修复：强制同一 `t_eval`。
- 失败：忽略链式法则，误以为“轨迹近似一致就足够”。
  - 修复：增加 RHS 级别一致性残差检查。

## R15

工程实践建议：
- 正则变换代码应同时保留 `forward/inverse`，避免手工代换散落在多处；
- 结构检查（辛条件、Poisson）应做成自动化门禁；
- 报告里同时给“结构指标 + 动力学指标”，便于定位错误属于几何层还是积分层；
- 多自由度扩展时先写块矩阵形式的 Jacobian 与 `Ω`，再做数值验证。

## R18

`demo.py` 源码级算法流拆解（8 步）：
1. `CanonicalParams` 定义物理参数与采样配置，`validate_params` 做输入合法性约束。
2. `forward_transform`/`inverse_transform` 显式实现 `Q=q, P=p+alpha q` 及逆映射。
3. `jacobian_matrix` 与 `symplectic_residual` 直接计算 `J^TΩJ-Ω`，验证辛结构。
4. `poisson_bracket_qp` 用偏导公式计算 `{Q,P}_{q,p}`，验证基本正则括号。
5. `rhs_original` 与 `rhs_transformed` 分别给出旧/新变量中的哈密顿方程右端。
6. `integrate_original` 与 `integrate_transformed` 用 `solve_ivp(DOP853)` 在同一网格独立积分。
7. `build_report` 将新轨迹逆映射回旧变量，统计轨迹误差、能量漂移、`H-K` 差与链式法则残差。
8. `main` 打印 `DataFrame` 报告并执行阈值断言，形成可自动验证的最小闭环。

第三方库角色说明：
- `numpy` 负责数组与矩阵运算；
- `scipy.integrate.solve_ivp` 仅负责通用 ODE 积分；
- `pandas` 仅用于结构化输出。

正则变换建模、哈密顿量构造、方程推导和一致性检验逻辑都在源码中显式实现，不依赖黑盒物理库。
