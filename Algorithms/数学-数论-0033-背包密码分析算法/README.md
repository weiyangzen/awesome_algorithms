## R01

背包密码分析算法通常指把公钥背包密码（典型是 Merkle-Hellman）还原为子集和问题后，对密文执行恢复攻击的方法。本文档对应的 MVP 聚焦“低密度子集和的格攻击（LLL）”。

- 输入: 公钥权重向量 `a=(a1,...,an)` 与密文和 `S`，满足 `S = sum(ai*xi), xi in {0,1}`。
- 输出: 估计的二进制明文向量 `x`。
- 核心思想: 通过整数格嵌入把“0/1 子集和”转换为“寻找很短向量”，再用 LLL 约化把候选短向量显式化。
- 本目录实现: `demo.py` 内含密钥生成、加密、私钥解密校验、LLL 攻击恢复。

## R02

- 1978 年: Merkle 与 Hellman提出基于“超递增背包 + 模变换”的公钥密码。
- 1982 年: Shamir 给出针对原始 Merkle-Hellman 的结构性破坏思路，说明其安全性不足。
- 1980s: Lagarias-Odlyzko、Coster 等工作把低密度子集和攻击与格基约化（LLL）系统化。
- 历史结论: 背包密码是现代密码学中“从设计到被破译”的经典教学案例。

## R03

背包密码最初被寄望于构造“不同于 RSA”的公钥体系：加密只需做加法求和，工程上看起来高效。然而其公开权重序列在低密度条件下，会暴露可被格算法利用的几何结构。密码分析算法的任务，就是在不知私钥的前提下，仅凭公钥和密文恢复明文，进而评估该方案是否满足现实安全需求。

## R04

以本实现的 LLL 方案看复杂度（设维度 `d=n+1`，输入整数位长为 `B`）。

- 理论上: LLL 在多项式时间内完成，常见上界可写成 `poly(d, B)`。
- 工程上（本脚本的小规模实现）:
  - Gram-Schmidt 与 size-reduction 反复执行，粗略可视为 `O(d^4)` 次标量运算量级。
  - 整体成本随维度增长较快，因此演示选择 `n=10` 的教学规模。
- 结论: 作为密码分析原型可运行；不是生产级高维攻击器。

## R05

- 基向量矩阵空间: `O(d^2)`。
- Gram-Schmidt 浮点副本与系数矩阵: `O(d^2)`。
- 候选向量池（含两两组合）: `O(d^3)` 数量级中的常数规模（本例维度小，内存压力低）。
- 总体: 典型演示规模下可视作低内存脚本。

## R06

示例流程（对应 `demo.py` 自动执行）。

1. 生成 Merkle-Hellman 密钥对: 私钥超递增序列 `w`，模数 `m`，乘子 `r`，公钥 `a_i=(r*w_i) mod m`。
2. 随机生成 10 位明文 `x`，密文计算 `S=sum(a_i*x_i)`。
3. 攻击端只拿到 `(a, S)`，构造 `(n+1)` 维整数格嵌入。
4. 对基执行 LLL，扫描短向量并识别形如 `(±1,...,±1,0)` 的候选。
5. 反解得到二进制向量并回代验证 `sum(a_i*x_i)==S`。

## R07

该算法的价值不在“加密”，而在“安全性审计”。

- 教学价值: 把密码学问题映射到数论与格几何，展示跨学科攻击链。
- 工程价值: 用于识别“看似随机但密度偏低”的背包构造风险。
- 方法价值: 呈现从代数表达式到格最短向量近似求解的标准套路。

## R08

关键理论依赖。

- 子集和问题（Subset Sum）与 0/1 约束。
- 整数格、基向量、短向量概念。
- LLL 约化的两个核心约束:
  1. Size reduction（系数约束）。
  2. Lovasz 条件（保证基“更正交、更短”）。
- 低密度条件经验结论: 密度 `d = n/log2(max(a_i))` 较小时，格攻击成功率更高。

## R09

适用前提与边界。

- 适用:
  - 教学规模的 Merkle-Hellman/低密度子集和实例。
  - 需要展示“从密文恢复明文”的可运行实验。
- 不适用:
  - 高密度、加固变种、或现代安全参数的格密码体系。
  - 需要严格最优攻击成功率评估的科研级实验（应使用更强 BKZ/枚举工具链）。

## R10

正确性依据（针对本脚本攻击路径）。

1. 若真实解 `x in {0,1}^n` 存在，嵌入构造保证向量
   `v = sum(x_i*row_i) - last_row = (2x1-1,...,2xn-1,0)` 属于该格。
2. 该向量前 `n` 维均为 `±1`，范数很小，是“短向量候选”。
3. LLL 倾向于把这类短向量或其线性相关形式推到约化基附近。
4. 一旦识别到末坐标为 `0` 且前缀为 `±1` 的向量，即可一一映射回比特并验证子集和方程。

## R11

数值稳定性与误差来源。

- 理论问题是整数格；本实现的 Gram-Schmidt 用 `float`，存在轻微舍入误差。
- 风险点:
  - 极端大整数或高维情况下，浮点误差可能影响约化路径。
  - 误差不改变“候选验证”步骤，因为最终回代验证是整数精确检验。
- 结论: 小维教学规模可接受；高维应使用高精度/有理数或成熟库。

## R12

真实运行成本（超越大 O）。

- 主要开销在 LLL 循环内的反复 Gram-Schmidt 与向量更新。
- Python 纯实现的常数项较大，但维度仅十余时速度可接受。
- 为确保演示稳定，脚本采用“多 seed 自动挑选可攻击实例”，换取少量额外时间。
- 本条目定位是可解释 MVP，不追求极限性能。

## R13

近似性/概率性说明。

- LLL 攻击不是“必定找最短向量”的精确算法，而是多项式时间近似约化。
- 因此恢复成功存在实例依赖性。
- 本脚本通过:
  - 低密度参数构造。
  - 候选向量池扩展（含行向量 ± 与两两组合）。
  - 最终整数回代验证。
  来提高可复现成功率。

## R14

鲁棒性与失效模式。

- 失效 1: 实例密度偏高，短向量信号被噪声淹没。
- 失效 2: LLL 约化得到的基中不直接出现可解码向量。
- 失效 3: 浮点误差导致约化质量下降。
- 应对:
  - 调整参数（更低密度、更小维度、不同 seed）。
  - 增强候选搜索（更多线性组合或枚举）。
  - 使用更稳健的 LLL/BKZ 实现。

## R15

工程实现注意事项。

- 不要把“成功一次”误判为“安全或不安全的充分证据”，应做批量统计。
- 攻击代码必须保留最终整数方程验证，避免浮点近似导致误判。
- 对同一算法报告结果时，至少给出维度、密度、参数来源与成功/失败条件。
- 若迁移到生产评估，建议替换为专业格库并记录随机种子确保可复现。

## R16

前驱、后继与关联算法。

- 前驱基础:
  - 超递增背包贪心解法。
  - 模逆、同余变换。
- 同类分析技术:
  - Meet-in-the-middle 子集和（指数级但精确）。
  - Lattice + BKZ/枚举的更强组合。
- 后继方向:
  - 现代格密码（如 LWE）刻意避开这类可直接还原结构。
  - 密码审计中常把背包体系当作“反例教材”。

## R18

源码级流程拆解（`demo.py`，无第三方黑盒）。

1. `find_attackable_instance` 固定起始种子，循环构造 Merkle-Hellman 实例与随机明文。
2. `encrypt_bits` 生成密文 `S=sum(a_i*x_i)`，攻击端仅保留 `(a,S)`。
3. `build_lattice_embedding` 生成 `(n+1)` 维整数基: 前 `n` 行是 `2e_i` 与 `2a_i` 绑定，最后一行为 `(1,...,1,2S)`。
4. `lll_reduce` 对该基执行 LLL: 反复做 size reduction、重算 Gram-Schmidt、检查 Lovasz 条件并交换向量。
5. `recover_bits_via_lll` 从约化基构造候选池（行向量、取反、两两加减），按范数从小到大扫描。
6. `decode_pm1_vector` 识别末坐标为 `0` 且前缀为 `±1` 的向量，并把 `±1` 映射回 `0/1` 比特。
7. 用整数方程 `sum(a_i*x_i)==S` 做精确验证，过滤伪候选。
8. 主程序打印攻击结果，并用私钥解密交叉校验，最终 `assert` 明文恢复成功。
