# 列生成算法

- UID: `MATH-0373`
- 学科: `数学`
- 分类: `优化`
- 源序号: `373`
- 目标目录: `Algorithms/数学-优化-0373-列生成算法`

## R01

列生成（Column Generation）用于求解“变量数量极大但约束数量相对较少”的线性规划/整数规划。核心思想是：不一次性把所有变量放进模型，而是先解一个受限主问题（Restricted Master Problem, RMP），再通过定价子问题（Pricing Problem）只生成“有价值”的新变量（列），循环迭代直到没有可改进列。

## R02

本目录的 MVP 采用经典的一维切割库存（Cutting Stock）建模：
- 给定母卷长度 `L`
- 有 `m` 类订单，每类长度 `l_i`、需求 `d_i`
- 一个“切割方案”就是一列 `a_{i,p}`，表示方案 `p` 中第 `i` 类切了多少件
- 目标是最少使用母卷数量

数学形式（LP 松弛）：
- `min Σ_p x_p`
- `s.t. Σ_p a_{i,p} x_p >= d_i,  i=1..m`
- `x_p >= 0`

## R03

困难点在于切割方案总数指数级，无法显式枚举。列生成通过“按需扩列”规避全枚举：
- RMP 当前只含少量方案列
- 解出 RMP 的对偶价格 `π_i`
- 用 `π_i` 定价：寻找 reduced cost 最小的新列

对切割库存来说，定价子问题是无界背包：
- `max Σ_i π_i a_i`
- `s.t. Σ_i l_i a_i <= L, a_i ∈ Z_+`
- 若最优值 `> 1`，则存在负 reduced cost 列（`1 - Σ_i π_i a_i < 0`）

## R04

算法直觉：
- 对偶价格高的订单类型，代表“覆盖它们更值钱”
- 定价子问题会自动把母卷容量优先分配给高性价比订单
- 只要还能找到负 reduced cost 列，就说明当前 RMP 还可改进

## R05

MVP 的初始列选择为“单品最大切割列”：
- 对每个订单类型 `i`，构造一列仅切该类型 `floor(L / l_i)` 件
- 这些列保证 RMP 初始可行（能覆盖所有需求）
- 虽然质量一般，但实现简单、稳定

## R06

RMP 求解细节（双路径实现）：
- 优先路径：`scipy.optimize.linprog` + HiGHS
- 兼容路径：当运行环境无 SciPy 时，自动切到 `numpy` 实现的修正单纯形（解 RMP 对偶）
- 两条路径都在求同一件事：得到 RMP 最优值与对偶价格，再交给定价子问题扩列

## R07

终止条件：
- 定价子问题最优 reduced cost `>= -tol`（例如 `tol=1e-8`）
- 或达到 `max_iter`
- 或生成到重复列（避免退化情况下死循环）

## R08

正确性要点（LP 层面）：
- RMP 是原问题的受限版本，目标值是原问题最优值的上界（最小化）
- 定价找的是“最负 reduced cost 列”，若不存在负值，则满足最优性条件
- 因而在无负 reduced cost 时，当前 RMP 解即原 LP 松弛最优解

## R09

复杂度（单次迭代）：
- RMP（HiGHS 或 `numpy` fallback 单纯形），规模约为 `m` 约束、`k` 列
- 定价子问题 DP 复杂度 `O(mL)`，`L` 为母卷长度
- 总体复杂度与迭代次数和新增列数相关，通常远小于全量枚举

## R10

数值与工程注意事项：
- reduced cost 判定要留容差，避免浮点误差造成抖动
- 对偶符号要与约束方向一致，否则会错误扩列
- 需要去重列，避免重复模式导致无效迭代

## R11

`demo.py` 暴露的关键参数：
- `stock_length`：母卷长度
- `item_lengths`：订单长度数组
- `demands`：订单需求数组
- `max_iter`：最大列生成迭代次数
- `tol`：最优性容差

## R12

最小技术栈：
- `numpy`：向量/矩阵运算、无 SciPy 时的修正单纯形 fallback
- `scipy.optimize.linprog`（可选）：可用时用于更稳健的 RMP 求解
- 不依赖大型建模框架（如 Pyomo/PuLP）以保持 MVP 轻量

## R13

运行方式：

```bash
python3 demo.py
```

脚本会打印：
- 每轮列生成迭代日志（RMP 目标、对偶价、新列、reduced cost）
- 最终 LP 解与由 LP 解上取整得到的整数可行解
- 每类订单的需求覆盖情况

## R14

输出解读：
- `LP objective`：母卷数量的下界（LP 松弛最优）
- `x_p`：各切割模式的连续使用量
- `ceil(x_p)`：简单整数化后的可行用量（不保证最优，仅保证易懂可复现）
- `coverage`：各订单类型最终覆盖件数，应满足 `coverage >= demands`

## R15

边界情况：
- 若某个订单长度 `l_i > L`，初始列不可构造，问题本身不可行
- 若环境无 `SciPy`，脚本会自动使用 `numpy` fallback；若 fallback 迭代上限触发会抛出明确错误
- 若达到 `max_iter` 仍有负 reduced cost，说明仅获得截断近似

## R16

可扩展方向：
- 用 `scipy.optimize.milp` 或分支定价实现整数最优
- 加入稳定化策略（如对偶平滑）缓解退化
- 支持多资源约束（二维切割、车辆路径等更一般列生成场景）

## R18

本目录源码级算法流程（`demo.py`）可拆为 8 步：
1. 读取内置样例实例（母卷长度、订单长度、需求）。
2. 构造初始列集：每个订单类型一个“单品最大切割”模式。
3. 求解当前 RMP：若有 SciPy 则走 `linprog`，否则走 `numpy` 修正单纯形 fallback。
4. 统一拿到对偶价格 `π`（SciPy 路径从 `marginals` 提取，fallback 路径由单纯形直接给出）。
5. 在定价子问题中执行无界背包 DP，最大化 `Σ π_i a_i`，恢复最佳新模式。
6. 计算 reduced cost `rc = 1 - Σ π_i a_i`，判断是否小于 `-tol`。
7. 若 `rc < -tol` 且新模式未重复，则将该列加入 RMP 并继续迭代；否则终止。
8. 输出 LP 最优值、各列使用量，并给出 `ceil(x)` 的整数可行近似与覆盖校验。

这 8 步完整展开了“主问题求对偶价 -> 子问题按对偶价找负 reduced cost 列 -> 扩列回主问题”的列生成闭环。
