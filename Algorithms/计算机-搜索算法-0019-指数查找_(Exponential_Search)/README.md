# 指数查找 (Exponential Search)

## R01

指数查找（Exponential Search）用于在**有序数组**中定位目标值。它先用指数级扩张快速找到一个可能包含目标的区间，再在该区间内执行二分查找。

## R02

问题形式化：
- 输入：升序数组 `arr`（长度 `n`）与目标值 `target`
- 输出：若存在下标 `i` 使 `arr[i] == target`，返回某个合法下标；否则返回 `-1`

## R03

适用前提：
- 输入序列必须按非降序排列
- 元素类型支持比较操作（`<`, `==`, `>`）
- 若存在重复值，需要明确返回策略（本条目返回“任意一个命中位置”）

## R04

核心思想分两段：
- 阶段 A（指数扩张）：从下标 `1` 开始，每轮将边界翻倍（`1, 2, 4, 8, ...`），直到边界越界或 `arr[bound] >= target`
- 阶段 B（区间二分）：在区间 `[bound/2, min(bound, n-1)]` 内做二分查找

## R05

简化伪代码：

```text
if n == 0:
    return -1
if arr[0] == target:
    return 0

bound = 1
while bound < n and arr[bound] < target:
    bound *= 2

left = bound // 2
right = min(bound, n - 1)
return binary_search(arr, target, left, right)
```

## R06

正确性要点（不变式直观）：
- 指数扩张结束时，目标若存在，不会落在 `left = bound//2` 左侧
- 结束条件保证右端 `right` 至少覆盖到首个 `>= target` 的候选区域（或数组末尾）
- 后续二分查找在该候选区间内是完备的，不会漏掉目标

## R07

复杂度：
- 时间复杂度：`O(log p)`，其中 `p` 是目标位置（最坏可视作 `O(log n)`）
  - 指数扩张约 `O(log p)` 次比较
  - 区间二分约 `O(log p)` 次比较
- 空间复杂度：`O(1)`（迭代实现）

## R08

边界与特殊情况：
- 空数组
- 单元素数组
- 目标等于首元素（可在扩张前快速返回）
- 目标小于首元素或大于末元素
- 重复值存在时返回任意命中下标

## R09

示例：
- `arr = [2, 4, 7, 9, 13, 21, 34]`, `target = 13`
  - 扩张：`bound=1(4) -> 2(7) -> 4(13)` 停止
  - 二分区间 `[2, 4]`，命中下标 `4`
- `arr = [2, 4, 7, 9, 13, 21, 34]`, `target = 8`
  - 扩张后区间 `[2, 4]`，二分未命中，返回 `-1`

## R12

典型应用：
- 无界/近似无界有序流中的目标定位（先“探测范围”再精搜）
- 大规模有序数组中靠前位置元素的快速命中
- 作为二分查找的“区间预定位”增强步骤

## R13

常见错误：
- 忘记有序性前提，直接对无序数据执行算法
- 扩张停止条件写错（如 `<=`/`<` 混用）导致越界或漏检
- 区间边界计算错误（`left = bound // 2`、`right = min(bound, n-1)`）
- 二分子过程使用错误区间定义导致死循环

## R14

常见变体：
- 查找第一个等于目标的位置（先指数扩张，再 lower-bound 二分）
- 仅返回是否存在（布尔接口）
- 在字符串有序表、时间戳索引等可比较对象上复用同一流程

## R15

与普通二分查找对比：
- 二分查找默认已知完整边界 `[0, n-1]`
- 指数查找先指数探测候选区间，再二分精搜
- 当目标位置较靠前或上界未知时，指数查找更自然
- 在普通静态数组且边界已知时，两者渐进复杂度同为 `O(log n)`

## R16

工程实践建议：
- 对外 API 明确“输入必须非降序”
- 将“扩张阶段”和“区间二分阶段”分函数实现，便于单测
- 对重复值场景文档化返回语义（任意命中/最左/最右）
- 覆盖空数组、首元素命中、越界目标、重复值等测试

## R17

最小测试清单：
- 空数组、单元素命中/未命中
- 目标命中首元素、中间元素、末元素
- 目标不存在且小于最小值、大于最大值、位于中间缺口
- 重复值数组（验证“存在性”和返回索引合法）
- 非有序输入（应抛出 `ValueError`）

## R18

源码级算法流程拆解（以本目录 `demo.py` 的 `exponential_search` 为准）：
1. 接收 `arr` 与 `target`，先调用 `_ensure_non_decreasing` 验证数组非降序，不满足则抛出 `ValueError`。
2. 若数组为空，直接返回 `-1`；若 `arr[0] == target`，直接返回 `0`。
3. 初始化 `bound = 1`，进入指数扩张循环：只要 `bound < n` 且 `arr[bound] < target`，就执行 `bound *= 2`。
4. 扩张结束后计算候选区间：`left = bound // 2`，`right = min(bound, n - 1)`。
5. 调用 `_binary_search_inclusive(arr, target, left, right)` 在闭区间内做标准二分。
6. 二分每轮取 `mid` 并比较 `arr[mid]` 与 `target`，相等即返回下标。
7. 若 `arr[mid] < target`，左边界右移到 `mid + 1`；否则右边界左移到 `mid - 1`。
8. 区间耗尽仍未命中时返回 `-1`；`main()` 再用 `bisect_left` 仅做存在性校验，验证实现行为。
