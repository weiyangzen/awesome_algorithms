## R01

多普勒效应（声波）描述了当声源和观察者相对运动时，观察到的频率会偏离声源本征频率的现象。
本条目聚焦一维匀速场景（介质静止），给出可运行的数值验证：

- 声源周期发射波峰；
- 波峰以声速 `c` 在介质中传播；
- 观察者记录波峰到达时间序列并反推接收频率。

## R02

MVP 的目标：把“教材公式”落到“可测量的到达时间序列”。

- 输入：`c, f0, x_s0, x_o0, v_s, v_o`；
- 输出：理论频率 `f_theory`、由到达时间反演的 `f_measured`、相对误差；
- 验证：多场景下统计 `R^2 / MAE / 最大相对误差` 并断言通过。

## R03

建模假设：

- 介质静止且均匀，声速 `c` 为常数；
- 声源与观察者都做一维匀速运动；
- 观察者始终在声源右侧（`x_o > x_s`），只考虑向右传播的波峰；
- 速度满足 `|v_s| < c, |v_o| < c`。

这些假设覆盖了声学多普勒的核心结构，同时便于做严格数值对照。

## R04

在本符号约定下（右方向为正方向），经典声学多普勒公式为：

`f' = f0 * (c - v_o) / (c - v_s)`

物理解释：

- `v_s > 0`（声源向观察者方向运动）使分母变小，频率升高；
- `v_o > 0`（观察者远离声源）使分子变小，频率降低。

## R05

单个波峰的时序关系：

- 第 `n` 个波峰发射时刻：`t_e(n) = n / f0`；
- 到达时刻 `t_a(n)` 满足传播方程：

`x_o(t_a) - x_s(t_e) = c * (t_a - t_e)`

其中

`x_s(t) = x_s0 + v_s * t`，`x_o(t) = x_o0 + v_o * t`。

脚本用数值求根求出每个 `t_a(n)`，再用 `1 / mean(diff(t_a))` 估计接收频率。

## R06

算法流程（高层）：

1. 生成若干波峰发射时刻 `t_e`；
2. 对每个 `t_e`，建立到达方程并数值求根得到 `t_a`；
3. 用相邻到达间隔 `Δt_a` 构造瞬时接收频率 `f_inst = 1/Δt_a`；
4. 统计 `f_measured` 并与 `f_theory` 对照；
5. 汇总全部场景并计算总体误差指标。

## R07

复杂度（设每个场景波峰数为 `N`，场景数为 `K`）：

- 求根次数：`K * N`；
- 每次求根为常数级开销（本模型是一维单根问题）；
- 总时间复杂度近似 `O(KN)`；
- 空间复杂度 `O(N)`（保存单场景时间序列）。

## R08

正确性直觉：

- 多普勒频移本质是“到达间隔”相对“发射间隔”的拉伸/压缩；
- 数值求根直接从传播几何约束求出到达时刻，避免把结论硬编码为输出；
- 若 `t_a` 序列正确，频率估计自然正确。

因此该实现以“可观测时序”为中间层，验证闭环更透明。

## R09

`demo.py` 使用的最小工具栈：

- `numpy`：向量化生成时间网格、误差统计；
- `scipy`：`optimize.brentq` 求波峰到达时刻根；
- `pandas`：场景结果表格化输出；
- `scikit-learn`：`r2_score` 和 `mean_absolute_error` 评估理论/测量一致性；
- `torch`：对瞬时频率序列做张量统计（均值、标准差、峰峰值）。

## R10

演示场景覆盖：

- 静止基准（`v_s=0, v_o=0`）；
- 仅声源运动（接近/远离）；
- 仅观察者运动（接近/远离）；
- 声源与观察者同时运动但方向不同。

每个场景均自动输出 `f_theory`, `f_measured`, `rel_error`。

## R11

主要输出指标：

- 场景级：`f_theory_hz`、`f_measured_hz`、`mean_inst_hz`、`std_inst_hz`、`rel_error`；
- 全局级：
  - `global_r2`：理论频率与测量频率的线性一致性；
  - `global_mae_hz`：平均绝对误差；
  - `max_rel_error`：最坏场景相对误差；
  - `torch_global_span_hz`：跨场景测量频率张量峰峰值。

## R12

## R14

常见失效模式：

1. 速度符号约定混乱，导致公式分子分母写反；
2. 使用超声速参数（`|v|>=c`）使传播模型失效；
3. 初始几何关系不满足（观察者不在声源前方）导致括号区间不含根；
4. 把电磁多普勒公式误用于声波介质模型。

## R18

`demo.py` 的源码级算法流程（8 步）：

1. `Scenario` 数据类定义单个场景的物理参数，并在 `validate` 中检查 `|v_s|, |v_o| < c` 与初始几何有效性。  
2. `theoretical_received_frequency` 按本条目符号约定计算 `f' = f0 * (c - v_o)/(c - v_s)`。  
3. `arrival_equation` 对每个发射时刻 `t_emit` 构造隐式方程 `x_o(t_arr)-x_s(t_emit)-c(t_arr-t_emit)=0`。  
4. `solve_arrival_time` 用 `scipy.optimize.brentq` 在保守区间内求根，得到单个波峰的到达时刻。  
5. `simulate_arrivals` 生成 `n_crests` 个发射时刻并逐个求根，得到完整到达时间序列 `arrivals`。  
6. `estimate_frequency_from_arrivals` 用 `np.diff(arrivals)` 得到瞬时频率序列，结合 `torch` 统计均值/标准差/峰峰值。  
7. `evaluate_scenarios` 对全部场景重复步骤 2-6，组装 `pandas` 结果表，并用 `sklearn` 计算全局 `R^2` 与 `MAE`。  
8. `main` 打印表格与汇总指标，并执行阈值断言，保证脚本能一键给出“通过/失败”结论。  

第三方库没有被当成黑箱算法：`scipy` 只负责单变量方程求根，Doppler 的物理建模、时序构造、频率反演和验收逻辑都在源码中逐步展开。
