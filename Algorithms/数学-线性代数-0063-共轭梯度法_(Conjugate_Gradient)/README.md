# 共轭梯度法 (Conjugate Gradient)

- UID: `MATH-0063`
- 学科: `数学`
- 分类: `线性代数`
- 源序号: `63`

## R01

共轭梯度法（Conjugate Gradient, CG）用于求解线性方程组 `Ax=b`，其中 `A` 要求为对称正定（SPD）矩阵。

它是“最速下降法 + Krylov 子空间 + A-共轭方向”结合后的结果：  
- 每一步只做一次矩阵向量乘法 `A @ p`；  
- 搜索方向两两满足 `p_i^T A p_j = 0 (i!=j)`；  
- 在无舍入误差时，`n` 维问题至多 `n` 步精确收敛。

## R02

历史定位（简要）：

- 1952 年 Hestenes 与 Stiefel 提出 CG，用来高效处理大规模 SPD 线性系统；  
- 它避免了高成本的直接分解（如稠密 `LU/Cholesky` 的 `O(n^3)`）；  
- 在现代科学计算中，CG 与预条件器（PCG）是 PDE 离散、优化子问题和图计算中的基础组件。

## R03

本条目 MVP 问题定义：

- 输入：
  - `A in R^{n x n}`：对称正定矩阵；
  - `b in R^n`：右端向量；
  - 初值 `x0`（可选，默认零向量）；
  - 最大迭代 `max_iters`；
  - 收敛阈值 `tol`（相对残差）。
- 输出：
  - 数值解 `x_hat`；
  - 残差范数 `||b-Ax_hat||` 与相对残差；
  - 迭代次数、收敛标志、残差历史。

## R04

时间复杂度：

- 每轮核心计算是一次 `A @ p`，若 `A` 稠密则约 `O(n^2)`；
- 其余向量点积与线性组合也是 `O(n)` 到 `O(n^2)` 量级内；
- 总复杂度约 `O(k n^2)`（`k` 为迭代轮数）。

与直接法对比：
- 稠密直接分解约 `O(n^3)`；
- 当 `k << n` 或 `A` 稀疏时，CG 通常更有优势。

## R05

空间复杂度：

- 矩阵存储：`O(n^2)`（稠密场景）；
- 迭代向量（`x, r, p, Ap`）与历史记录：`O(n + k)`；
- 因此额外迭代内存主要是 `O(n)`，总内存由矩阵存储主导。

## R06

2x2 微型示例（直观理解）：

令
`A = [[4, 1], [1, 3]]`（SPD），`b=[1,2]^T`，`x0=[0,0]^T`。  
初始残差 `r0 = b - A x0 = b`，令 `p0=r0`。

第一步：
- `alpha0 = (r0^T r0)/(p0^T A p0)`；
- `x1 = x0 + alpha0 p0`；
- `r1 = r0 - alpha0 A p0`。

第二步方向：
- `beta0 = (r1^T r1)/(r0^T r0)`；
- `p1 = r1 + beta0 p0`。

由于二维空间中最多两条 `A`-共轭方向，理论上 2 步内可达精确解（忽略浮点误差）。

## R07

算法意义：

- 线性系统求解中“只做 matvec、避免分解”的代表方法；
- 将优化观点（二次函数最小化）与线性代数观点（Krylov 子空间）统一；
- 是预条件共轭梯度法（PCG）、Truncated Newton 等方法的重要基础。

## R08

核心理论（简要）：

求解 `Ax=b` 等价于最小化二次函数  
`f(x)=0.5 x^T A x - b^T x`（`A` 为 SPD 时严格凸）。

CG 的关键性质：
- 第 `k` 步解位于仿射空间 `x0 + K_k(A, r0)`；
- 其中 `K_k(A,r0)=span{r0, A r0, ..., A^{k-1} r0}`；
- 方向 `p_k` 彼此 `A`-共轭，使得在该子空间内达到最优；
- 误差界与条件数 `kappa(A)` 相关，`kappa` 越小通常收敛越快。

## R09

适用条件与局限：

适用：
- `A` 为实对称正定；
- 可高效计算 `A @ v`；
- 大规模稀疏系统（不希望显式分解）。

局限：
- 非对称或不定矩阵不适用经典 CG（应考虑 GMRES / MINRES 等）；
- 条件数很大时收敛慢，常需预条件器；
- 浮点误差会破坏完美共轭性，实践中不保证精确 `n` 步收敛。

## R10

正确性说明（工程可验证版）：

1. 若 `A` 是 SPD，则每轮 `p_k^T A p_k > 0`，步长 `alpha_k` 有定义。  
2. 迭代保持 `r_k` 与已有方向的正交/共轭结构（精确算术下）。  
3. `x_k` 是当前 Krylov 子空间上的最优解。  
4. 残差 `||r_k||` 应总体下降并趋近 0。  
5. `demo.py` 通过与 `numpy.linalg.solve` 对照来验证数值正确性。

## R11

误差与稳定性：

- 截断误差由 `tol` 与 `max_iters` 决定；
- 舍入误差会逐步破坏正交性与共轭性，导致收敛速度偏离理想曲线；
- 条件数越高，数值误差放大越明显；
- 实务中常配合预条件器降低谱半径/条件数，提升稳健性。

## R12

性能视角：

- CG 每步主要成本是一次稀疏或稠密 matvec，硬件友好；
- 在稀疏场景可接近 `O(nnz(A))` 每步；
- 对超大规模问题可结合块存储、并行 matvec、预条件器获得明显加速；
- 本 MVP 以“透明可审计”优先，不引入复杂预条件模块。

## R13

本目录 MVP 可验证保证：

- `demo.py` 构造可复现 SPD 系统（固定随机种子）；
- 运行后输出迭代次数、残差、相对残差和误差对比；
- 自动断言：
  - 迭代已收敛；
  - 最终残差足够小；
  - 与 `numpy.linalg.solve` 解足够接近；
  - 残差历史长度和迭代次数一致。

## R14

鲁棒性与常见失效模式：

- 输入非法（维度错误、非有限值、非法阈值）会直接抛错；
- 非对称或非正定矩阵会在校验阶段拒绝执行；
- 若出现 `p^T A p <= 0`，代码会主动停止并报告数值异常；
- 若达到 `max_iters` 仍未满足阈值，则返回 `converged=False`（非静默失败）。

## R18

`demo.py` 源码级算法流程拆解（8 步）：

1. `main` 调用 `build_demo_system` 生成可复现 SPD 矩阵 `A`、右端 `b` 和真值 `x_true`。  
2. `conjugate_gradient` 先经 `validate_inputs` 检查方阵、维度匹配、有限值、对称性和 SPD（Cholesky）。  
3. 初始化 `x`（默认零向量）、`r=b-Ax`、`p=r`，并记录初始残差。  
4. 每轮先计算 `Ap=A@p` 与曲率 `p^T A p`，据此得到步长 `alpha=(r^T r)/(p^T A p)`。  
5. 更新解 `x <- x + alpha p`，更新残差 `r <- r - alpha Ap`。  
6. 计算新残差平方 `r_new^T r_new`，得到 `beta=(r_new^T r_new)/(r_old^T r_old)`，再更新方向 `p <- r + beta p`。  
7. 以相对残差 `||r||/max(||b||,1)` 与 `tol` 比较，满足则提前收敛返回；否则继续迭代。  
8. `run_checks` 用 `numpy.linalg.solve` 仅作验算对照，确认数值正确性后输出统计信息。

说明：MVP 的 CG 主过程是手写迭代，不依赖 `scipy.sparse.linalg.cg` 黑盒调用。
