## R01

Edmonds 开花算法（Blossom Algorithm）用于求解**一般无向图**中的最大基数匹配（maximum cardinality matching）。

给定无向图 `G=(V,E)`，匹配 `M` 是边集子集，满足任意两条边不共享端点。目标是最大化 `|M|`。

与二分图匹配不同，一般图会出现奇环，朴素增广路搜索会在奇环处“卡住”。Edmonds 的关键贡献是：
- 识别交替树中的奇环（blossom）；
- 将奇环收缩为一个超点继续找增广路；
- 找到增广路后再在原图中等价展开，从而保证正确性。

## R02

该算法由 Jack Edmonds 在 1965 年提出，是组合优化与匹配理论中的里程碑工作之一。它把“增广路法”从二分图推广到了任意无向图。

核心突破在于开花（blossom）收缩思想：
- 二分图中不存在奇环，BFS/DFS 增广路结构简单；
- 一般图中奇环会破坏层次结构；
- 通过收缩奇环，问题可临时转化为更“规整”的图继续搜索。

## R03

目标问题（本 MVP）是：
1. 输入：`n` 个点、无向边列表 `edges`（无权图）；
2. 输出：最大基数匹配 `M`，即匹配边数量最多；
3. 不要求完美匹配，不要求权重最优（不是最小权完美匹配）。

本实现重点覆盖“基数最大化”版本，这是开花算法最经典也最常用的教学入口。

## R04

算法主流程（增广路框架）如下：
1. 初始化匹配 `match[v] = -1`（全部未匹配）。
2. 对每个未匹配顶点 `root`，执行一次 BFS 交替树搜索。
3. 扩展边 `(v,u)` 时：
- 若发现可直接到达未匹配点，得到增广路；
- 若形成奇环，计算 LCA 基点并收缩 blossom；
- 在收缩图继续 BFS。
4. 一旦找到增广路，沿路径翻转匹配/非匹配边。
5. 重复直到不存在新的增广路。

根据 Berge 定理：不存在增广路时，当前匹配即为最大匹配。

## R05

复杂度（经典无权版本）：
- 时间复杂度：`O(V^3)`（常见实现上界）。
- 空间复杂度：`O(V + E)`。

本目录 `demo.py` 采用邻接表 + 数组状态（`match/parent/base/used/blossom`），符合该复杂度量级。

## R06

一个带奇环的示例：
- 奇环：`0-1-2-3-4-0`（长度 5）；
- 外部连接：`0-5`、`2-6`、`1-6`。

该图不是二分图。算法会在搜索中识别奇环并收缩，最终得到大小为 `3` 的最大匹配（例如 `[(0,5), (1,6), (3,4)]` 的某个等价结果）。

`demo.py` 会先跑这个固定样例，再用暴力枚举校验其最优性。

## R07

优点：
- 可处理一般图（含奇环）；
- 在最大基数匹配问题上有严格正确性保证；
- 工程上可作为更复杂匹配算法（带权、带约束版本）的基础。

局限：
- 实现复杂度明显高于二分图匹配（如 Hopcroft-Karp）；
- 常数项较大，小规模以外调试成本高；
- 若问题是带权匹配，需要更复杂的 Blossom V / primal-dual 变体。

## R08

前置知识建议：
- 匹配、增广路、交替路基本概念；
- Berge 定理（增广路存在性与最优性等价）；
- 并查集/LCA 的“祖先”思维（本实现用数组追溯实现 LCA）；
- BFS 队列与图遍历基础。

## R09

适用场景：
- 社交配对、任务配对、网络连边等可抽象为一般图最大配对数量问题；
- 图中存在奇环，二分图算法不可直接使用；
- 更关注“能配多少对”而不是边权收益。

不直接适用：
- 需要最小权完美匹配（应使用带权 blossom 变体）；
- 超大规模图且仅需近似解（可考虑近似/随机化策略）。

## R10

正确性关键点（实现层）：
1. `base` 数组必须维护当前收缩图中的基点映射。
2. 遇到奇环时要正确找到 LCA（blossom 基点）。
3. `mark_path` 需要同步标记 blossom 上两条路径并修正 `parent`。
4. 收缩后，所有属于 blossom 的点都应映射到同一 `base`。
5. 找到增广路后必须严格执行“沿路翻转”匹配关系。

`demo.py` 额外用小图暴力搜索做回归对照，辅助验证实现正确性。

## R11

此算法不涉及浮点数值稳定性，重点是**状态一致性稳定性**：
- 保证图是无向简单图（去重边、忽略自环）；
- 每轮 BFS 前重置 `used/parent/base`；
- 避免访问 `-1` 下标路径（通过条件分支保护）；
- 对输入顶点编号做边界检查。

这些约束比“数值精度”更关键。

## R12

工程成本与实践经验：
- 代码量不大，但状态变量之间耦合强，调试成本高；
- 推荐先做小图全枚举对拍，再扩大规模；
- 若只处理二分图，应优先选更简单且更快的 Hopcroft-Karp；
- 如需带权，建议直接选成熟实现并做源码审计。

## R14

常见失败模式与防护：
- `parent/base` 更新不一致，导致死循环或错误匹配。
- 未正确跳过 `base[v] == base[u]` 或 `match[v] == u` 的边，破坏交替树结构。
- 忘记在收缩后把新可达点入队，导致漏掉增广路。
- 增广阶段 `pv/nv` 交替更新顺序写错。

防护措施：
- 固定样例 + 随机小图 + 暴力最优对比三重验证。

## R15

实现建议：
- 用清晰命名（`match/parent/base/blossom`）代替过度缩写；
- 把 `lca`、`mark_path`、`find_augmenting_path` 拆成独立函数；
- 每个函数只维护单一职责，降低状态污染风险；
- 对小图做属性测试（本目录已实现随机回归测试）。

## R16

关联算法：
- 二分图最大匹配：Kuhn、Hopcroft-Karp（更简单，速度更快）。
- 一般图带权匹配：Edmonds 加权 blossom（primal-dual / Blossom V）。
- 特殊图上的匹配可利用结构性质进一步加速。

关系总结：Edmonds 开花算法是“一般图匹配”主干方法，许多后续算法都建立在它的收缩思想上。

## R18

`demo.py` 的源码级流程拆解（8 步）：
1. `EdmondsBlossom.__init__` 读取 `n, edges`，构建去重后的无向邻接表并初始化状态数组。
2. `solve` 外层循环扫描所有未匹配顶点，把它们作为交替树根尝试寻找新增广路。 
3. `_find_augmenting_path` 以根节点做 BFS，维护 `used/parent/base` 三类搜索状态。 
4. 扩展边 `(v,u)` 时，如果触发奇环条件，调用 `_lca(v,u)` 找 blossom 基点。 
5. `_mark_path` 沿两条路径把 blossom 顶点打标，并修正 `parent` 以保持收缩图中的交替树结构。 
6. BFS 中把所有处于 blossom 的点压缩到同一 `base`，继续在“收缩图”上搜索。 
7. 一旦命中未匹配端点，`_find_augmenting_path` 返回增广路终点，`solve` 沿 `parent` 链执行翻转更新 `match`。 
8. `main` 先跑固定奇环案例，再跑随机小图，并用 `brute_force_maximum_matching_size` 对拍保证结果确为最优。 

本实现未调用第三方黑盒匹配库，开花收缩与增广逻辑均可在源码中逐行追踪。
