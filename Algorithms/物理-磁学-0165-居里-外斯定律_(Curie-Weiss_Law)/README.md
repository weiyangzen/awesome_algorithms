## R01

居里-外斯定律描述顺磁材料在临界温度以上的磁化率温度依赖关系：

`chi(T) = C / (T - Theta)`

其中 `chi` 为磁化率，`C` 为居里常数，`Theta` 为外斯温度。`Theta` 的符号常用于判断交换相互作用倾向：

- `Theta > 0`：铁磁关联倾向；
- `Theta < 0`：反铁磁关联倾向；
- `Theta ~= 0`：近似理想居里行为。

## R02

本目录的目标是给出一个最小可运行 MVP：

- 生成一组可控噪声的 `chi-T` 数据；
- 用线性化方法估计 `C`、`Theta`；
- 用非线性最小二乘做参数精修；
- 输出拟合质量与物理解释，形成可验证的端到端流程。

## R03

模型与假设如下：

- 温度范围满足 `T > Theta`，避免模型奇点；
- 观测数据服从“真值 + 小幅乘性噪声”；
- 线性化关系：
  `1/chi = (1/C) * T - Theta/C`
- 参数识别分两阶段：
  先线性化求初值，再非线性拟合细化。

该 MVP 聚焦“参数反演算法”，不覆盖临界涨落修正和复杂多相模型。

## R04

关键推导：

1. 从 `chi = C/(T-Theta)` 取倒数，得到 `1/chi` 与 `T` 的线性关系。
2. 设线性模型 `y = a*T + b`，其中 `y=1/chi`，则
   - `a = 1/C`
   - `b = -Theta/C`
3. 因而可恢复：
   - `C = 1/a`
   - `Theta = -b/a`

这让问题可先用稳定的线性回归求近似，再用非线性拟合减少线性化误差。

## R05

`demo.py` 采用的数值流程：

- `numpy` 生成温度网格与合成数据；
- `scikit-learn` 在 `1/chi-T` 空间做线性回归，给出初值；
- `PyTorch` 用 `lstsq` 独立求同一个线性系统，做数值交叉校验；
- `scipy.optimize.curve_fit` 在原始 `chi(T)` 空间做有界非线性最小二乘；
- `pandas` 汇总输出表格和误差指标。

## R06

运行结果会输出：

- 真值参数 `C_true, Theta_true`；
- 线性化拟合参数与 `R2(1/chi~T)`；
- 非线性精修后的参数；
- `MAE/RMSE` 拟合误差；
- 数据头尾样本表（温度、观测值、两种拟合值）。

同时包含断言，保证流程在固定随机种子下可重复通过。

## R07

优点：

- 结构小、依赖少、易复现；
- 既有解析线性化直觉，也有非线性精修；
- 通过 `sklearn` 与 `torch` 双解法做一致性检查，降低静默错误风险。

局限：

- 未做加权拟合与不确定度传播；
- 靠近 `T=Theta` 时的临界行为未建模。

## R10

正确性直觉：

1. 线性化把非线性模型转为一次回归，先锁定近似参数区间。
2. 非线性拟合直接最小化 `chi` 空间残差，可修正线性化带来的偏差。
3. 两种线性求解器（`sklearn` 与 `torch`）结果一致，说明代数层没有实现错误。
4. 最终误差与参数断言共同验证模型闭环有效。

## R11

数值稳定性处理：

- 强制 `temperature_min > Theta_true`，避免分母接近 0；
- 对噪声后的 `chi` 做下限截断，避免 `1/chi` 爆炸；
- `curve_fit` 使用参数边界，限制 `Theta < min(T)`；
- 使用固定随机种子保证可复现；
- 设置断言检查回归质量与参数偏差。

## R12

核心参数（`CurieWeissConfig`）：

- `curie_constant_true`：真值 `C`；
- `weiss_temperature_true`：真值 `Theta`；
- `temperature_min/max`：温度扫描区间；
- `n_points`：采样点数量；
- `relative_noise_std`：乘性噪声标准差；
- `random_seed`：随机种子。

调参建议：若要增加挑战，可提高噪声或缩窄温度区间；若要增强稳定性，可提高采样点数。

## R14

常见失败模式：

1. 温度点过于接近 `Theta`，导致噪声被放大；
2. 数据噪声过大且采样过少，线性初值偏离严重；
3. 未设置边界时，非线性拟合可能探索到非物理解；
4. 忽略单位一致性，导致 `C` 和 `Theta` 物理解释错误；
5. 真实实验中的偏置项未建模，系统误差被误判为参数偏差。

## R18

`demo.py` 源码级算法流程（9 步）：

1. `main` 构造 `CurieWeissConfig`，固定真值参数、温度区间、噪声强度与随机种子。  
2. `generate_synthetic_data` 计算真值 `chi_clean=C/(T-Theta)`，注入乘性高斯噪声，生成 `chi_observed`。  
3. `fit_linearized_sklearn` 将观测值映射到 `y=1/chi`，拟合 `y=aT+b`，反推 `C=1/a`、`Theta=-b/a`。  
4. `fit_linearized_torch` 用 `torch.linalg.lstsq` 解同一线性系统，得到独立的 `a,b` 估计。  
5. 主流程比较 `sklearn` 与 `torch` 的斜率/截距差值，作为数值一致性校验。  
6. `fit_nonlinear_scipy` 以线性初值为起点，在参数边界内调用 `curve_fit` 最小化残差平方和：
   `sum_i [chi_obs_i - C/(T_i-Theta)]^2`。  
7. `curve_fit` 在有界条件下使用信赖域反射（TRF）迭代更新参数；每步通过局部线性化近似雅可比并求解子问题，不是一步黑盒常数替换。  
8. `summarize_results` 计算 `MAE/RMSE/R2`，输出头尾样本表并给出 `Theta` 符号的物理解释。  
9. 断言模块检查拟合质量阈值（线性 `R2`、参数误差、非线性优于线性），确保 MVP 可自动验证。  

第三方库职责边界：`numpy/pandas` 负责数据组织，`sklearn/torch` 负责线性代数求解，`scipy` 负责非线性优化；物理建模、参数反推、约束设计与验收断言均在源码中显式实现。
