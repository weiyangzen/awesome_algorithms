## R01

高斯朴素贝叶斯（Gaussian Naive Bayes, GNB）是朴素贝叶斯在“连续特征”场景下的经典变体。它假设：

- 类别先验 `P(y=c)` 可由样本频率估计；
- 在给定类别 `c` 条件下，各特征彼此条件独立；
- 每个特征服从一维高斯分布 `x_j | y=c ~ N(mu_cj, sigma2_cj)`。

由贝叶斯公式可得后验 `P(y=c|x)`，预测时取后验最大的类别。

## R03

概率模型定义如下（`d` 维特征，`K` 个类别）：

- 类先验：`pi_c = P(y=c)`
- 类条件分布：
  `P(x|y=c) = Π_{j=1..d} N(x_j ; mu_cj, sigma2_cj)`
- 后验（未归一化）：
  `P(y=c|x) ∝ pi_c * Π_j N(x_j ; mu_cj, sigma2_cj)`

为避免浮点下溢，实际计算采用对数空间。

## R04

核心公式：

1. 参数估计（极大似然）：
   `mu_cj = mean(x_j | y=c)`
   `sigma2_cj = var(x_j | y=c)`
2. 联合对数似然：
   `log P(y=c, x) = log pi_c + Σ_j [ -0.5*log(2π*sigma2_cj) - (x_j-mu_cj)^2/(2*sigma2_cj) ]`
3. 预测类别：
   `y_hat = argmax_c log P(y=c, x)`
4. 概率归一化：
   用 `log-sum-exp`/`softmax` 从联合对数似然恢复后验概率。

## R05

训练阶段算法（本实现显式编码）：

- 扫描每个类别 `c` 的训练子集 `X_c`；
- 计算 `class_count[c]` 与 `class_prior[c]`；
- 逐特征计算均值向量 `theta_[c, :]` 与方差向量 `var_[c, :]`；
- 对方差增加平滑项 `epsilon = var_smoothing * max(var(X_train, axis=0))`，防止零方差导致除零或 `log(0)`。

## R07

优点：

- 训练和推理都非常快，闭式估计，几乎无迭代成本；
- 对小数据集与高维稀疏统计特征常有稳健表现；
- 结果可解释，便于教学和基线构建。

局限：

- 条件独立假设通常不严格成立；
- 假设类条件高斯分布，对偏态/多峰特征不够鲁棒；
- 概率校准可能偏保守或偏激进，需要额外校准步骤。

## R09

适用场景：

- 连续特征分类的快速基线；
- 原型阶段需要低训练成本与高可解释性；
- 教学/面试场景下展示“概率模型 + 工程实现”。

不适用场景：

- 特征强相关且关系复杂（深度网络或树模型通常更优）；
- 分布显著非高斯且类别边界高度非线性；
- 对概率校准和极致精度有严格生产要求。

## R10

正确性直觉：

1. 在朴素独立与高斯假设成立时，后验最大化等价于比较联合对数似然；
2. 参数估计使用样本均值与方差，是高斯似然下的标准极大似然解；
3. 对数空间累加把“连乘”转为“求和”，减少数值下溢风险；
4. 因此只要参数估计与对数似然计算正确，分类决策就与理论一致。

## R11

数值稳定性策略：

- 方差平滑：`var += epsilon`，避免零方差；
- 在 `predict_proba` 中对每行对数似然减去最大值后再 `exp`，避免溢出；
- 所有核心计算向量化，减少 Python 循环累积误差；
- 输入检查保证 `fit` 后才允许 `predict`。

## R12

关键参数：

- `var_smoothing`：方差平滑强度，默认 `1e-9`；
- `test_size`：训练/测试划分比例，默认 `0.30`；
- `random_state`：随机种子，保证复现；
- `stratify=y`：分层抽样，防止类别比例漂移。

调参建议：

- 若出现方差过小导致极端概率，可适当增大 `var_smoothing`（如 `1e-8` 或 `1e-7`）。

## R13

复杂度与保证：

- 训练复杂度：`O(n*d + K*d)`，其中 `n` 为样本数，`d` 为特征数；
- 预测复杂度：`O(m*K*d)`，`m` 为待预测样本数；
- 空间复杂度：`O(K*d)`（存储均值、方差、先验）。

该问题不是近似优化问题，不涉及近似比；本实现保证在有限样本下稳定输出并可复现。

## R14

常见失败模式：

1. 忽略方差平滑，遇到常量特征直接数值崩溃；
2. 直接在概率空间连乘导致下溢，得到全 0；
3. 训练/测试未分层，导致评估波动过大；
4. 数据泄漏（先全量标准化再划分）；
5. 把 sklearn 一行调用当作实现，无法解释算法细节。

本目录通过手写训练与推理流程规避第 5 点。

## R18

`demo.py` 源码级算法流程（8 步）：

1. `main` 加载 Iris 连续特征数据，做分层训练/测试划分。  
2. 初始化 `GaussianNBScratch(var_smoothing=1e-9)`。  
3. `fit` 中按类别切分样本，计算每类先验 `class_prior_`。  
4. `fit` 中计算每类每特征均值 `theta_` 与方差 `var_`，并加入 `epsilon` 平滑。  
5. `predict` 调用 `_joint_log_likelihood`，按公式累计 `log pi_c + Σ log N(x_j|mu,sigma2)`。  
6. `predict_proba` 对联合对数似然做“减最大值 + exp + 归一化”，得到后验概率。  
7. 计算手写模型准确率，并与 `sklearn.GaussianNB` 的预测做一致率对照。  
8. 打印样例预测和混淆矩阵，并通过断言确保准确率与一致率达标。  

第三方库边界：`numpy/pandas` 只用于数值与展示，`sklearn` 仅用于数据集与基准对照；核心 GNB 参数估计与推断过程完全在源码中展开，不是黑盒调用。
