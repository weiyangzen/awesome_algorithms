## R01

Cholesky 分解用于把一个**对称正定矩阵** \(A\in\mathbb{R}^{n\times n}\) 分解为：
\[
A = LL^T
\]
其中 \(L\) 是下三角矩阵，且对角线元素均为正。

## R02

核心思想是把矩阵乘法逐项对齐：
- 先确定 \(L_{11}\)，再依次确定第一列。
- 第 \(i\) 行/列只依赖前面已经算好的元素。
- 通过“逐行递推 + 开平方”得到完整下三角矩阵 \(L\)。

## R03

输入：
- 一个实数方阵 \(A\)。

输出：
- 若 \(A\) 对称正定，则返回下三角矩阵 \(L\)，满足 \(A=LL^T\)。
- 若不满足条件（非方阵、非对称、非正定），应报错或返回失败状态。

## R04

适用前提：
- \(A\) 必须是方阵。
- \(A\) 应满足 \(A=A^T\)（数值计算中可用容差判断）。
- \(A\) 的特征值全部大于 0（正定）。

## R05

常用递推公式（\(i\ge j\)）：
\[
L_{ii} = \sqrt{A_{ii} - \sum_{k=1}^{i-1}L_{ik}^2}
\]
\[
L_{ij} = \frac{A_{ij} - \sum_{k=1}^{j-1}L_{ik}L_{jk}}{L_{jj}},\quad i>j
\]

当某一步出现 \(A_{ii} - \sum_{k=1}^{i-1}L_{ik}^2 \le 0\) 时，矩阵不是严格正定（或数值上已退化）。

## R06

伪代码：

```text
输入 A (n×n)
检查 A 是否方阵且近似对称
L ← zeros(n, n)
for i in [0..n-1]:
    for j in [0..i]:
        s ← dot(L[i, 0:j], L[j, 0:j])
        if i == j:
            v ← A[i, i] - s
            if v <= 0: 报错(非正定)
            L[i, i] ← sqrt(v)
        else:
            L[i, j] ← (A[i, j] - s) / L[j, j]
返回 L
```

## R07

正确性直觉：
- 每次计算 \(L_{ij}\) 时，所需的历史项都已经在更小列中确定。
- 对角元素通过平方根保证 \(L_{ii}>0\)，这与正定性匹配。
- 逐项构造后，\(LL^T\) 的每个元素都与 \(A\) 对齐。

## R08

复杂度：
- 时间复杂度：\(O(n^3)\)
- 空间复杂度：\(O(n^2)\)（存储 \(L\)）

相比通用 LU 分解，Cholesky 在对称正定条件下常数更小，通常更快。

## R09

数值性质：
- 对对称正定问题，Cholesky 通常稳定且高效。
- 若矩阵接近奇异（病态），对角项可能非常小，误差敏感。
- 工程中可做：
  - 对称化：\(A\leftarrow (A+A^T)/2\)
  - 加微小正则：\(A\leftarrow A+\lambda I\)

## R10

与其他分解对比：
- 与 LU：LU 对一般矩阵适用，常需主元选取；Cholesky 更专用于 SPD，开销更低。
- 与 QR：QR 更稳但更重，常用于最小二乘；SPD 线性方程优先 Cholesky。
- 与特征分解：特征分解可解释性强但代价更高，求解线性系统时通常不如 Cholesky 直接。

## R11

常见坑：
- 把“对称半正定”误当“正定”，导致开方出现 0 或负数。
- 忘记检查对称性，直接分解导致结果错误或库函数失败。
- 在浮点误差下用过严判定，需设置合理容差。

## R18

第三方实现不是黑箱，NumPy 路径可拆为以下 6 步：
1. `numpy.linalg.cholesky(A)` 先在 Python 层做参数整理与维度检查。
2. 输入被转换为底层可处理的连续数组，并确定是实数/复数路径。
3. 调用 NumPy 线性代数内部 ufunc（`_umath_linalg` 中对应 cholesky 入口）。
4. 该入口继续调用 LAPACK `xPOTRF` 系列例程（如双精度 `dpotrf`）。
5. `dpotrf` 进行块化/递推下三角分解：按列更新、计算对角开方、更新 trailing 子块。
6. 若 LAPACK 返回 `info>0`，说明某个主子式非正定并抛错；否则返回下三角因子给 Python。

手写版 `demo.py` 与上述流程在数学上等价，只是去掉了 LAPACK 的工程优化细节。
