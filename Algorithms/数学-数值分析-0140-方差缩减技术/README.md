# 方差缩减技术

- UID: `MATH-0140`
- 学科: `数学`
- 分类: `数值分析`
- 源序号: `140`
- 目标目录: `Algorithms/数学-数值分析-0140-方差缩减技术`

## R01

方差缩减技术（Variance Reduction）是 Monte Carlo 数值估计中的一类方法，目标是在不显著增加计算成本的前提下，降低估计量方差，从而更快达到给定精度。

## R02

本任务关注的核心问题是：估计

- `mu = E[f(X)]`

其中 `X` 服从已知分布。朴素 Monte Carlo 使用样本均值作为估计量，虽无偏但方差可能较大，导致收敛慢。

## R03

朴素 Monte Carlo 的标准误差约为 `O(1/sqrt(n))`，当函数波动大或需要高精度时，单纯增加样本数成本很高。方差缩减通过构造“更稳”的估计量，通常在同样 `n` 下显著降方差。

## R04

本算法条目的 MVP 以

- `f(x) = exp(x)`
- `X ~ N(0, 1)`
- 真值 `mu = exp(1/2)`

为演示目标，比较 4 种估计器：

- 朴素 MC（baseline）
- 对偶变量（Antithetic Variates）
- 控制变量（Control Variates）
- 重要性采样（Importance Sampling）

## R05

关键数学关系：

- 朴素估计：`mu_hat = (1/n) * sum f(X_i)`
- 估计方差：`Var(mu_hat) = Var(f(X))/n`
- 控制变量估计：`mu_cv = (1/n) * sum (f(X_i) - beta*(C_i - E[C]))`
- 最优系数：`beta* = Cov(f(X), C) / Var(C)`
- 重要性采样：`mu = E_q[f(X) * p(X)/q(X)]`

其中 `p` 是目标分布密度，`q` 是提议分布密度。

## R06

常见方差缩减技术族：

- 对偶变量：构造负相关样本对，抵消波动。
- 控制变量：引入已知期望且与目标强相关的变量。
- 分层采样：在样本空间分层后均匀覆盖。
- 重要性采样：将采样重心移向“贡献大”的区域。
- 条件 Monte Carlo / Rao-Blackwell：对部分随机性解析积分。

## R07

本 MVP 选择三种代表性技术并与 baseline 对比：

- 对偶变量：对每个 `Z` 同时使用 `Z` 与 `-Z`。
- 控制变量：选 `C=X`，利用 `E[X]=0`。
- 重要性采样：令 `q=N(theta,1)`，以 `theta=0.7` 做指数倾斜。

## R08

输入输出约定（`demo.py`）：

- 输入：无交互输入，参数写在 `main()` 中。
- 固定参数：`n_samples=2000`、`n_trials=500`。
- 输出：每种方法的 `mean/bias/var/mse/vrf_vs_crude` 表格，以及控制变量 `beta_hat` 的平均值。

## R09

实验流程：

- 对每种方法运行多次独立 trial。
- 每次 trial 用同样样本预算生成一个估计值。
- 在 trial 维度上计算经验方差和 MSE。
- 用 `VRF = Var(crude) / Var(method)` 量化降方差倍率。

## R10

正确性与无偏性要点：

- 朴素 MC：无偏。
- 对偶变量：样本对均值仍是 `E[f(X)]` 的无偏估计。
- 控制变量：当 `E[C]` 已知时，`f-beta(C-E[C])` 期望不变。
- 重要性采样：乘以权重 `p/q` 后，积分目标与采样分布解耦，估计保持无偏（要求 `q>0` 于 `p` 支持上）。

## R11

复杂度分析：

- 单次 trial 时间复杂度：`O(n)`。
- 单次 trial 空间复杂度：`O(n)`（向量化实现）。
- 全实验时间复杂度：`O(n_trials * n_samples)`。

## R12

数值稳定性注意事项：

- 重要性采样中优先使用对数权重 `log_weight` 再指数化，减少溢出风险。
- 控制变量的 `Var(C)` 若接近 0，`beta` 估计会不稳定；代码中做了零方差保护。
- 若估计目标函数重尾，建议增加稳健统计或截断策略。

## R13

参数建议：

- 先固定总预算 `n_samples`，比较不同技术的 VRF 再决定是否扩样。
- 控制变量应与目标函数高度相关，否则收益有限。
- 重要性采样的 `theta` 需要与被积函数形状匹配；偏差太大时可能导致权重方差反升。
- 若用于生产，可先做小规模 pilot 再锁定参数。

## R14

运行方式：

python3 Algorithms/数学-数值分析-0140-方差缩减技术/demo.py

运行后会打印结构化结果表，无需额外交互。

## R15

结果解读方式：

- `mean` 越接近真值越好。
- `bias` 越接近 0 越好。
- `var` 和 `mse` 越小越好。
- `vrf_vs_crude > 1` 表示相对朴素 MC 实现了降方差；数值越大越有效。

## R16

局限与失败模式：

- 对偶变量依赖函数结构，不是所有函数都能显著受益。
- 控制变量高度依赖相关性与 `E[C]` 的可得性。
- 重要性采样若提议分布选差，会出现权重退化，方差可能更高。
- 单一问题上的收益不应直接外推到全部任务。

## R17

可扩展方向：

- 多控制变量与岭回归式 `beta` 正则化。
- 分层采样与低差异序列（Sobol/Halton）结合。
- 自适应重要性采样（AIS/Cross-Entropy）。
- 与多层 Monte Carlo（MLMC）结合做多精度估计。

## R18

`demo.py` 的源码级算法流程（非黑箱）可拆为 8 步：

1. 定义目标 `mu = E[exp(X)]`，并给出解析真值 `exp(1/2)` 作为评估锚点。
2. 实现 baseline `crude_mc`：采样 `X~N(0,1)` 后直接取 `exp(X)` 样本均值。
3. 实现 `antithetic_mc`：采样 `Z`，构造配对 `(+Z,-Z)`，对每对先平均再总体平均。
4. 实现 `control_variate_mc`：用同一样本计算 `Y=exp(X)` 与 `C=X`，估计 `beta=Cov(Y,C)/Var(C)`，输出 `mean(Y-beta*C)`。
5. 实现 `importance_sampling_mc`：从 `q=N(theta,1)` 采样，显式计算 `log(p/q)=-theta*x+theta^2/2`，再算加权均值 `mean(exp(x+log_weight))`。
6. 用 `run_trials` 做多次独立重复试验，为每种方法生成一组估计值样本。
7. 用 `summarize` 计算 `mean/bias/var/mse`，再以 baseline 方差构造 `VRF`。
8. 将各方法指标格式化输出，附带控制变量平均 `beta_hat`，便于检查控制项是否稳定生效。
