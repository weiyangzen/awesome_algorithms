## R01

合并区间（Merge Intervals）是贪心算法中的经典基础题。给定若干闭区间 `[start, end]`，目标是把所有有重叠的区间合并成更少且互不重叠的区间，并覆盖原始区间并集。

核心思路是：

- 先按区间起点排序；
- 然后线性扫描，维护“当前正在合并的区间”；
- 如果新区间与当前区间重叠，则扩展右边界；否则开始一个新区间。

## R02

该问题常见于：

- 日程冲突整理（会议时间段合并）；
- 资源占用压缩（时间窗、带宽窗口）；
- 区间类索引预处理（减少后续查询复杂度）；
- 几何/统计数据中的一维区间并运算。

它是很多更复杂区间问题（插入区间、最少删除重叠区间、区间调度）的基础组件。

## R03

问题定义（闭区间版本）：

- 输入：区间集合 `intervals = [[s1,e1], [s2,e2], ...]`，且 `si <= ei`；
- 输出：一个新区间集合 `merged`，满足：
  - 两两不重叠；
  - 按起点有序；
  - `merged` 的并集与 `intervals` 的并集相同。

本条目 `demo.py` 以整数区间实现，但算法同样适用于实数区间。

## R04

贪心策略：

1. 按 `(start, end)` 升序排序；
2. 把第一个区间放入结果；
3. 依次处理后续区间 `[s, e]`：
   - 若 `s <= 当前结果最后区间的 end`，说明重叠，更新 `end = max(end, e)`；
   - 否则不重叠，直接把 `[s, e]` 作为新区间加入结果。

为何是贪心：每一步只需要保留当前可达到的最远右边界，不需要回溯历史决策。

## R05

复杂度：

- 排序：`O(n log n)`；
- 单次扫描：`O(n)`；
- 总时间复杂度：`O(n log n)`。

空间复杂度：

- 结果列表最坏 `O(n)`；
- 若排序在新数组上完成，额外空间 `O(n)`（Python `sorted`）。

## R06

`demo.py` 的实验设计：

## R07

优点：

- 性能稳定，适合工程预处理；

局限：

## R09

适用场景：

- 离线批处理区间去重与压缩；
- 日历、日志时间段聚合；
- 数据预处理阶段减少区间数量。

不适用场景：

- 在线动态插入/删除高频场景（更适合平衡树/线段树等）；
- 多维几何体并集（需更复杂数据结构）；
- 需要记录原始区间来源路径但未做额外索引时。

## R10

正确性要点（不做形式化全证明）：

- 排序后，任何可能与当前区间重叠的候选，都会在扫描时连续出现；
- 当 `s <= current_end`，把右边界更新为 `max(current_end, e)` 不会丢失并集覆盖；
- 当 `s > current_end`，新区间与当前区间不可能重叠，立即切分是必要且安全的；
- 扫描结束后得到的区间序列有序且互不重叠，并保持并集不变。

因此该贪心策略可得到正确的合并结果。

## R11

边界与鲁棒性处理：

- 空输入返回空列表；
- 单区间直接返回；
- 对每个输入区间校验长度必须为 2；
- 若出现 `start > end`，`demo.py` 抛出 `ValueError`，避免隐式纠错导致语义不明；
- 统一输出为 `List[Tuple[int, int]]`，便于后续断言与比较。

## R12

实现细节选择：

- 重叠判定使用 `s <= current_end`，即把边界相接视为可合并（闭区间语义）；
- 排序键使用 `(start, end)`，保证同起点区间按终点有序，便于稳定合并；
- 使用单遍扫描维护 `current_start/current_end`，避免复杂数据结构。

如果业务采用半开区间 `[start, end)`，则应把判定改为 `s < current_end`。

## R13

理论保证与边界：

- 在比较排序模型下，区间合并问题通常需要排序信息，`O(n log n)` 是该实现可达到的标准最优量级；
- 本实现不依赖近似或随机化，输出是确定性精确结果；
- 结果唯一性体现在“并集表示”上，区间顺序按起点规范化。

## R14

常见错误：

1. 忘记先排序，导致只能合并局部相邻项；
2. 误把重叠条件写成 `s < current_end`（在闭区间语义下会漏合并边界相接）；
3. 合并后只更新 `end = e`，而不是 `max(current_end, e)`；
4. 对非法区间 `start > end` 静默处理，造成后续结果难以解释。

## R17

`demo.py` 功能清单：

## R18

`demo.py` 的源码级算法流程（6 步）：

1. `main` 调用 `run_fixed_cases` 与 `run_numpy_case`，准备确定性样例并触发执行。  
2. `merge_intervals_numpy` 把 `numpy` 的 `n x 2` 区间数组转换为 Python 元组列表，再委托给核心函数。  
3. `merge_intervals` 先做输入校验（每个区间必须是 2 个端点且 `start <= end`），然后按 `(start, end)` 排序。  
4. 扫描排序后的区间：若新区间起点 `s <= current_end`，则执行 `current_end = max(current_end, e)`；否则把当前合并段写入结果并开启新区间。  
5. 扫描结束后补写最后一个合并段，得到最终不重叠区间列表。  
6. `assert_valid_merge` 对结果进行结构和覆盖断言，确保实现没有把区间错误拆分或遗漏。

本 MVP 没有使用第三方黑盒算法库来“直接合并区间”；`numpy` 仅用于样例数据构造与数组输入演示，核心贪心逻辑全部在源码中逐行可追踪。
