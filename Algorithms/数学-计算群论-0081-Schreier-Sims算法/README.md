## R01

Schreier-Sims 算法是计算群论中处理“置换群”的核心算法之一，用来构造稳定子链（stabilizer chain）及强生成元集（Strong Generating Set, SGS）。

给定一组置换生成元，算法可以高效得到：
- 群阶 `|G|`；
- 成员测试（某个置换是否属于该群）；
- 轨道与横截（orbit/transversal）结构。

本条目实现一个可运行、可审计的 MVP：显式写出 Schreier 树、Schreier 生成元、sifting（筛分）过程，不依赖黑箱群论库。

## R02

本条目问题定义（离散计算版）：

- 输入：
  - 置换作用点数 `n`（默认作用在 `{0,1,...,n-1}`）；
  - 生成元集合 `S = {g1, g2, ...}`，每个生成元是一个长度为 `n` 的置换；
  - 基（base）`B=[b0,b1,...,b_{n-1}]`（MVP 取全基 `0..n-1`）。
- 输出：
  - 构造出的强生成元集合 `SGS`；
  - 每层稳定子的轨道横截 `T_i`；
  - 由稳定子链得到的群阶 `|G| = Π_i |Orb_{G_i}(b_i)|`；
  - 可用于成员测试的 `sift` 机制。

## R03

MVP 核心思想：

1. 以基 `B` 定义稳定子链 `G = G0 >= G1 >= ... >= G_n = {e}`，其中 `G_i` 固定 `b0..b_{i-1}`。  
2. 对每层 `G_i`，用当前生成元构造 `b_i` 的轨道与 Schreier 横截。  
3. 由 Schreier 引理构造候选生成元  
   `u_alpha * g * u_{alpha^g}^{-1}`（属于 `G_{i+1}`）。  
4. 用 `sift` 将候选元沿稳定子链约化：  
   - 若完全约化到单位元，则它已被现有 SGS 覆盖；  
   - 若在某层失败，残差置换加入 SGS。  
5. 迭代直到没有新生成元加入，得到闭合的 SGS 与稳定子链数据。

## R04

复杂度（实现层面）：

- 设：
  - `n` 为作用点数；
  - `s` 为最终强生成元数量；
  - `r` 为外层迭代轮数；
  - 每层轨道规模至多 `n`。
- 一层 Schreier 生成元枚举约为 `O(n*s)` 个候选；
- 每次 `sift` 需经过 `n` 层，每层做置换合成/求逆（本 MVP 为显式数组操作），粗略计 `O(n^2)`；
- 总体可估计为 `O(r * s * n^4)` 量级（教学实现的保守上界）。

说明：经典 Schreier-Sims（含更精细数据结构与策略）有更优理论界，本 MVP 以可读性为优先。

## R05

空间复杂度：

- `SGS` 存储约 `O(s*n)`；
- 每层横截是点到置换的映射，最坏 `n` 层、每层 `n` 个点、每个置换长度 `n`，最坏 `O(n^3)`；
- 其他辅助队列、临时变量为低阶项。

因此本实现总体空间复杂度可按 `O(s*n + n^3)` 估计。

## R06

微型示例（`S3`）：

- 取生成元 `S = { (0 1), (0 1 2) }`，基 `B=[0,1,2]`。  
- 第 0 层：`Orb(0)={0,1,2}`，轨道大小 3。  
- Schreier 生成元会在第 1 层产生固定点 0 的稳定子生成元。  
- 第 1 层：`Orb_{G1}(1)={1,2}`，轨道大小 2。  
- 第 2 层轨道大小为 1。  
- 群阶得到 `|G|=3*2*1=6`，即 `S3`。

这正是“轨道乘积=群阶”的稳定子链计算方式。

## R07

算法价值：

- 置换群计算的基础设施算法，广泛用于计算代数系统（如 GAP/Magma）；
- 能把“指数级枚举全群”替换成“稳定子链结构计算”；
- 是群成员测试、群阶计算、轨道-稳定子分析的标准入口；
- 对组合优化、对称性约简、密码与代数计算都有实践意义。

## R08

理论基础（对应本 MVP）：

1. 轨道-稳定子定理：`|G_i| = |Orb_{G_i}(b_i)| * |G_{i+1}|`。  
2. 连乘得到：`|G| = Π_i |Orb_{G_i}(b_i)|`。  
3. Schreier 引理：若已知 `G_i` 的生成元与横截，可构造 `G_{i+1}` 的生成元。  
4. Sifting：将元素逐层右乘横截逆元，消去基点像；若可消尽即在群内。  
5. 当 Schreier 候选都可被 sift 消尽时，SGS 达到闭合，可用于阶与成员判定。

## R09

适用条件与局限：

- 适用于有限置换群（元素可表示为 `0..n-1` 上的置换）；
- 本 MVP 适合中小规模 `n` 的教学与验证场景；
- 未实现高级优化（基重排启发式、随机化变体、压缩表示）；
- 对超大规模群，需更优化的数据结构与策略，否则性能不足。

## R10

正确性框架（工程可验证版本）：

1. `orbit_with_transversal` 保证每个轨道点都对应一个把基点送到该点的代表元。  
2. `schreier_generators_for_level` 按 Schreier 引理生成稳定子候选元。  
3. `sift` 逐层消去基像：  
   - 若在某层找不到横截代表，则说明当前 SGS 尚不完备；  
   - 若最终化为单位元，则该元素已被链表示。  
4. 迭代加入“sift 失败残差”直到收敛，得到闭合 SGS。  
5. 收敛后用 `Π|orbit_i|` 求得的阶应与小规模暴力闭包一致（`demo.py` 内置交叉校验）。

## R11

终止性与可靠性：

- 每次迭代只会向 `SGS` 加入新置换；
- 在 `Sym(n)` 中可加入的不同置换总数有限，因此过程必停；
- 代码设置 `max_rounds` 作为额外保险，防止实现错误导致异常循环；
- 若超轮数未收敛会抛错，避免静默给出不可信结果。

## R18

源码级算法流程拆解（`schreier_sims` 主线，8 步）：

1. 预处理输入：检查 `n/base`，去重并移除单位生成元，得到初始 `strong_generators`。  
2. 调用 `build_transversals`：对每个层级 `i`，先筛出固定前缀基点的生成元，再构造 `b_i` 轨道横截。  
3. 在每一层调用 `schreier_generators_for_level`，枚举 Schreier 候选  
   `u_alpha * g * u_{alpha^g}^{-1}`。  
4. 对每个候选执行 `sift`：沿基逐层用横截逆元消去像，得到失败层或残差。  
5. 若 `sift` 在某层失败且残差非单位，则把该残差加入 `strong_generators`。  
6. 完成一轮所有层后，若有新元素加入则继续下一轮；否则宣布收敛。  
7. 收敛后重新构造最终横截，计算每层轨道大小并连乘得到群阶。  
8. 在 `run_case` 中用暴力闭包阶和成员测试交叉验证结果，确保实现行为与理论一致。  

说明：第三方群论库并未参与核心计算，`demo.py` 完整展开了 Schreier-Sims 的关键数据流与步骤。
