## R01

透视投影（Perspective Projection）是将三维场景映射到二维屏幕的核心几何变换。它满足“近大远小”，因此比正交投影更符合人眼视觉。

## R02

核心思想：

1. 先把 3D 点表示为齐次坐标 `(x, y, z, 1)`。
2. 使用 `4x4` 透视矩阵将点变到裁剪空间（clip space）。
3. 对 `w` 分量做透视除法（perspective divide），得到 NDC 坐标。
4. 把 NDC `[-1,1]` 映射到屏幕像素坐标。

## R03

常用右手系（相机朝向 `-Z`）透视矩阵：

设视场角为 `fov_y`，宽高比 `a = width/height`，近平面 `n`，远平面 `f`，且 `0 < n < f`。

- `s = 1 / tan(fov_y / 2)`
- 投影矩阵 `P`:

```text
[ s/a   0      0               0 ]
[ 0     s      0               0 ]
[ 0     0    (f+n)/(n-f)   2fn/(n-f) ]
[ 0     0     -1               0 ]
```

对 clip 坐标 `(x_c, y_c, z_c, w_c)` 做：

- `x_ndc = x_c / w_c`
- `y_ndc = y_c / w_c`
- `z_ndc = z_c / w_c`

## R04

输入：

- 相机空间点集 `N x 3`
- 视角参数：`fov_y, aspect, near, far`
- 视口大小：`width, height`

输出：

- 每个点的 clip 坐标
- 每个点的 NDC 坐标
- 每个点的屏幕坐标（像素）

## R05

MVP 使用的数据结构：

- `numpy.ndarray`（`float64`）
- 点集形状 `N x 3` / `N x 4`
- 投影矩阵 `4 x 4`

这样可以直接表达线性代数流程，避免额外框架开销。

## R06

算法流程（本目录 `demo.py` 的实现）：

1. 构建透视矩阵 `P`。
2. 将 `N x 3` 点扩展为齐次坐标 `N x 4`。
3. 计算 `clip = (P @ points_h^T)^T`。
4. 取 `w = clip[:, 3]` 并检查不接近 0。
5. 做透视除法得到 `ndc = clip[:, :3] / w`。
6. 将 `ndc.xy` 映射到屏幕像素坐标。
7. 做两个数值校验：
   - 同 `x` 不同深度验证“近大远小”
   - 检查点是否落入 NDC 立方体

## R07

正确性直觉：

- 齐次坐标线性变换可统一表示投影；
- 透视矩阵中 `w' = -z`（在该约定下），因此 `x_ndc = x' / w'` 与深度成反比；
- 当点更靠近相机（`|z|` 更小），`|x_ndc|` 与 `|y_ndc|` 更大，对应屏幕上“更大”。

## R08

复杂度：

- 时间复杂度：`O(N)`（每点常数次矩阵/向量运算）
- 空间复杂度：`O(N)`（存储 clip、ndc、screen）

## R09

边界与退化情况：

- `near <= 0` 或 `near >= far`：投影参数非法。
- `fov_y` 不在 `(0, 180)`：矩阵不可用或数值不稳定。
- `w` 接近 0：透视除法不稳定，需要丢弃或裁剪。
- 点在近远平面外：NDC 可能超出 `[-1, 1]`，应进入裁剪阶段。

## R12

结果解读要点：

- 同一立方体中，`z=-3` 的顶点对应的 `|ndc_x|`、`|ndc_y|` 大于 `z=-5` 的顶点。
- 屏幕坐标中，前面的面占据更大像素范围。
- 若所有顶点 `|ndc| <= 1`，表示都在标准裁剪立方体内。

## R14

工程中常见坐标约定差异：

- OpenGL 与 DirectX 在 `z` 范围、矩阵符号上可能不同；
- 左手/右手坐标系会改变部分符号；
- 屏幕 `y` 轴方向（向上或向下）依渲染管线而定。

本 MVP 明确采用：右手系 + 相机看向 `-Z` + NDC 映射到左上角为 `(0,0)` 的屏幕像素。

## R16

常见错误清单：

- 把 `near/far` 当作负值传入（和相机空间 `z<0` 混淆）。
- 漏掉齐次坐标 `w=1`。
- 透视除法用错分量（应除以 `w`）。
- 屏幕映射时忘记翻转 `y`。

## R18

`demo.py` 的源码级算法拆解（无黑盒）：

1. `perspective_matrix` 中根据 `fov/aspect/near/far` 手工写出 4x4 矩阵每个关键元素。  
2. `to_homogeneous` 把 `N x 3` 点扩展成 `N x 4`，显式追加齐次分量 `1`。  
3. `perspective_project` 用矩阵乘法得到 clip 坐标，并显式读取 `w`。  
4. 在透视除法前检查 `|w|` 是否过小，避免数值爆炸。  
5. 计算 `ndc = clip_xyz / w`，得到标准设备坐标。  
6. `ndc_to_screen` 把 `[-1,1]` 区间线性映射到像素坐标，并翻转 `y` 轴。  
7. 主程序打印每个点的 `camera -> clip -> ndc -> screen` 全链路结果，便于核对中间值。  
8. 使用两组不同深度点进行断言，验证“近大远小”确实由透视除法产生。  
9. 用 `|ndc|<=1` 检查点是否位于裁剪立方体内，形成最小正确性闭环。
