## R01

Čech 复形是拓扑数据分析（TDA）中非常经典的几何复形构造。给定点云与球半径 `r`，把每个点看作一个闭球 `B(x_i, r)`，当一组球有公共交时，就把对应顶点集合加入为一个单纯形。

本目录提供一个可运行、可审计的最小 MVP：
- 手写 2D Čech 复形（到 2 维）构造，不依赖 TDA 黑盒库
- 手写模 2 边界矩阵与高斯消元求秩
- 计算 `beta0/beta1/beta2` 并展示滤过半径变化下的拓扑特征

## R02

本条目解决的问题：
- 输入：二维点云 `X in R^{n x 2}`、半径序列 `r_1 < ... < r_m`、最大维度 `max_dim`（MVP 支持 `<=2`）
- 输出：每个半径 `r` 下
- 单纯形数量（`#V/#E/#T`）
- Betti 数（`beta0/beta1/beta2`）

`demo.py` 采用固定内置数据，无需任何交互输入：
- 案例 A：等边三角形，突出 Čech 与 clique 判定的差异
- 案例 B：双簇点云，观察连通分支数 `beta0` 随半径增大而下降

## R03

数学定义（Čech 复形）：
- 给定点集 `X={x_1,...,x_n}` 与半径 `r>=0`
- 对每个顶点 `i` 对应闭球 `B(x_i, r)`
- 顶点子集 `sigma subseteq {1,...,n}` 是一个单纯形，当且仅当
- `intersection_{i in sigma} B(x_i, r) != emptyset`

这个定义直接编码了“多球公共交”的几何信息。对于欧氏空间中的闭凸球，Čech 复形与球并集在同伦意义上等价（Nerve 定理语境）。

## R04

MVP 中使用的判定准则（`R^2`，到 2 维）：
- 0-单纯形：所有顶点都包含
- 1-单纯形 `(i,j)`：`||x_i - x_j|| <= 2r`
- 2-单纯形 `(i,j,k)`：三点最小包围圆半径 `R_min <= r`

三点最小包围圆半径计算要点：
- 若三角形钝角或直角，`R_min = 最长边/2`
- 若三角形锐角，`R_min = 外接圆半径 = abc/(4A)`
- 若接近退化（三点近共线），回退到 `最长边/2`

## R05

算法流程（单个半径 `r`）：
1. 校验输入点云并计算两两欧氏距离矩阵。
2. 构造 0-单纯形列表（全部顶点）。
3. 枚举所有顶点对，按 `d(i,j) <= 2r` 生成边集。
4. 枚举所有顶点三元组，先检查三条边都存在，再计算最小包围圆半径判定 2-单纯形。
5. 基于 `k` 维与 `k-1` 维单纯形构造边界矩阵 `d_k`（模 2）。
6. 对每个边界矩阵执行 `GF(2)` 高斯消元，得到秩 `rank(d_k)`。
7. 使用公式 `beta_k = n_k - rank(d_k) - rank(d_{k+1})` 得到 Betti 数。
8. 汇总并输出该半径下的统计行。

## R06

MVP 数据结构：
- `Simplex = Tuple[int, ...]`：单纯形用排序顶点索引元组表示
- `ComplexByDim = Dict[int, List[Simplex]]`：按维度组织单纯形
- `FiltrationRow`：保存 `radius, n0, n1, n2, beta0, beta1, beta2`
- 边界矩阵：`np.ndarray(dtype=np.uint8)`，元素只取 `{0,1}`

该结构紧凑透明，便于逐函数检查与单步调试。

## R07

正确性要点：
- Čech 复形核心是“公共交非空”，不是仅要求两两相交。
- 2-单纯形判定必须基于三球公共交；本实现用“三点最小包围圆半径 <= r”做等价判定（二维等半径情形）。
- 边界矩阵按“删去一个顶点得到面”构造，在模 2 系数下方向符号可忽略。
- Betti 公式中的秩和链群维数必须在同一系数域（本实现统一为 `GF(2)`）。

## R08

复杂度（`n` 为点数）：
- 距离矩阵：时间 `O(n^2)`，空间 `O(n^2)`
- 1-单纯形枚举：`O(C(n,2))`
- 2-单纯形枚举：`O(C(n,3))`，每项常数级几何计算
- 边界矩阵消元：设矩阵规模约 `m x p`，高斯消元约 `O(min(m,p)*m*p)`

由于组合爆炸，本 MVP 适用于中小规模教学与验证场景。

## R09

边界与异常处理：
- 点云必须是二维、非空、有限值数组，否则抛出 `ValueError`
- `radius` 必须 `>=0`，`max_dim` 必须 `>=0`
- MVP 仅支持 `max_dim <= 2`，更高维直接报错
- 空边界矩阵（如无高维单纯形）按秩 0 处理，保证 Betti 公式可统一执行
- 几何判定使用小容忍量 `1e-12`，减轻浮点误差影响

## R10

MVP 取舍说明：
- 不调用 Gudhi/Ripser 等库的即用接口，避免“几行黑盒”
- 重点是“从几何判定到 Betti 数”的源码链路可追踪
- 仅覆盖 2D 且到 2 维单纯形，保证实现小而完整
- 依赖最小化，仅使用 `numpy + Python 标准库`

## R12

输出字段解释：
- `r`：当前滤过半径
- `#V/#E/#T`：0/1/2 维单纯形数量
- `beta0`：连通分支数
- `beta1`：1 维环路数（模 2）
- `beta2`：2 维空腔数（模 2）

对于本 MVP（仅到 2 维），`beta2` 不会被 3 维边界进一步抵消，这是维度截断导致的预期现象。

## R13

内置测试场景：
1. Equilateral triangle
- 三个点组成边长 1 的等边三角形
- `r = [0.40, 0.55, 0.60]`
- `r=0.55` 时三条边都存在，但三球无公共交，出现 `beta1=1`

2. Two clusters
- 两个相距约 2.2 的小簇（每簇 6 点）
- `r = [0.08, 0.18, 1.20]`
- 观察 `beta0` 随半径增大而从多分支降到单分支

## R14

关键参数建议：

- `radii`：建议从小到大选 3-10 个半径观察拓扑变化

## R15

与相关复形对比：
- Vietoris-Rips 复形
- 只需两两距离阈值，计算更便宜，但是对 Čech 的近似
- Čech 复形
- 几何语义更直接（公共交），代价更高
- Alpha 复形
- 依赖 Delaunay 结构，通常更稀疏，几何解释强，但实现复杂度更高

## R16

## R18

`demo.py` 源码级算法流（9 步）：
1. `main` 构造两个固定点云与半径序列，触发滤过计算。  
2. `run_filtration` 对每个半径调用 `build_cech_complex_2d` 构建复形。  
3. 在 `build_cech_complex_2d` 中，先由 `ensure_points` 校验点云，再生成全部 0-单纯形。  
4. 通过两两距离判定 `d(i,j) <= 2r` 生成 1-单纯形，并缓存 `edge_set`。  
5. 对每个三元组先检查边存在，再调用 `triangle_min_enclosing_radius` 计算最小包围圆半径，判定是否加入 2-单纯形。  
6. `compute_betti_numbers` 逐维调用 `boundary_matrix_mod2` 构造 `d_1, d_2`。  
7. `gf2_rank` 对每个边界矩阵做模 2 高斯消元，得到边界秩。  
8. 按 `beta_k = n_k - rank(d_k) - rank(d_{k+1})` 计算 `beta0/beta1/beta2`。  
9. `print_rows` 输出每个半径下的单纯形计数与 Betti 数，并在结尾打印案例解释。
