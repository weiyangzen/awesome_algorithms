# 广度优先搜索 (BFS)

## R01

广度优先搜索（Breadth-First Search, BFS）是一种按“层”推进的图遍历算法：

- 从起点开始，先访问距离为 `0` 的顶点（起点本身）。
- 再访问距离为 `1` 的所有顶点。
- 再访问距离为 `2` 的所有顶点，依此类推。

BFS 的核心数据结构是先进先出队列（FIFO queue），它保证了“先发现的近层顶点先扩展”。

## R02

典型问题建模（无权图）：

- 输入：图 `G=(V,E)`，起点 `s`。
- 输出：
  - 访问顺序（遍历序）；
  - 起点到各顶点的最短边数距离 `dist[v]`；
  - 一棵最短路树前驱数组 `parent[v]`（可用于还原路径）。

若图是非连通图，BFS 一次只覆盖 `s` 所在连通分量；其他顶点距离保持不可达（常记为 `-1` 或 `+inf`）。

## R03

算法直觉：

- 队列里的顶点总是按“发现先后”排队。
- 某顶点 `u` 出队时，其未访问邻居 `v` 会被标记并入队，且 `dist[v] = dist[u] + 1`。
- 因为所有长度更短的路径对应的顶点会更早出队，所以第一次发现某顶点时就已经得到最短边数距离。

这使 BFS 成为无权最短路问题的标准解法。

## R04

时间复杂度：`O(|V| + |E|)`。

- 每个顶点最多入队/出队一次。
- 每条边最多被扫描常数次（无向图通常是 2 次，分别在两个端点邻接表中）。

因此 BFS 具有线性复杂度，是图算法中非常高效的基础操作。

## R05

空间复杂度：`O(|V|)`（不计输入图本身）。

- `visited`/`dist`/`parent` 等数组是线性空间。
- 队列最坏可达到 `O(|V|)`。
- 若计入邻接表存图，整体空间是 `O(|V| + |E|)`。

## R06

小例子（起点 `0`）：

- 边集：`(0,1),(0,2),(1,3),(2,3),(3,4)`。
- 分层结果：
  - 第 0 层：`{0}`
  - 第 1 层：`{1,2}`
  - 第 2 层：`{3}`
  - 第 3 层：`{4}`
- 距离：`dist=[0,1,1,2,3]`。

比如顶点 `4` 的最短边数是 `3`，路径可由 `parent` 反向恢复为 `0->1->3->4` 或 `0->2->3->4`（取决于邻接遍历顺序）。

## R07

BFS 的工程价值：

- 无权最短路径的基线算法。
- 连通性检查、层次划分、最少步数问题（状态图）都可直接使用。
- 也是很多高级算法的子过程，如 Dinic 分层图、双向搜索、最短路变体等。

前置知识：

- 图的表示方法（邻接表/邻接矩阵）。
- 队列（FIFO）语义。
- 基本复杂度分析。
- 路径还原的“前驱指针”思想。

## R09

适用边界：

- 适用于无权图，或所有边权相等（可视为单位权）时的最短路。
- 不适用于一般正权最短路（应使用 Dijkstra）或存在负权边的最短路（应考虑 Bellman-Ford/SPFA 等）。
- 对超大图可使用双向 BFS、外存 BFS 或并行 BFS 优化。

## R10

正确性要点（简述）：

- 不变式：队列中顶点按非递减层号出现，且出队顶点的 `dist` 已是最短边数。
- 初始化：起点 `s` 入队并设 `dist[s]=0`。
- 归纳：若 `u` 出队时 `dist[u]` 最优，则其未访问邻居 `v` 设为 `dist[u]+1` 必为首次可达的最短层；若存在更短路径，路径前缀对应顶点应更早出队并先发现 `v`，矛盾。
- 终止：队列为空时，所有可达顶点均已被最短距离标号。

## R11

鲁棒性与输入约束建议：

- 顶点编号应在 `[0, n-1]`。
- 图可有重边或自环：
  - 重边不影响正确性，但会增加扫描成本；
  - 自环对最短路无贡献，可保留或预清理。
- 若输入为空图或起点越界，应显式抛出异常。
- 需要可复现实验时，邻接表可排序保证遍历确定性。

## R12

性能与实现优化方向：

- 使用邻接表而非邻接矩阵（稀疏图更高效）。
- 使用 `collections.deque` 实现 `O(1)` 级出入队。
- 在多次查询场景中可复用图结构，仅重置 `dist/parent`。
- 对“全源层次”需求，可遍历每个未访问顶点触发一次 BFS（得到连通分量划分）。

## R13

与相关算法关系：

- 与 DFS：DFS 重深度路径探索，不保证最短边数；BFS 天然保证无权最短路。
- 与 Dijkstra：当所有边权都为 1 时，Dijkstra 退化到 BFS 思想。
- 与拓扑排序：都可借助队列，但目标不同；BFS 针对层次/最短步数，拓扑排序针对 DAG 依赖序。

## R14

常见错误模式：

- 忘记“入队时即标记访问”，导致重复入队和性能恶化。
- 错把栈当队列使用，结果变成 DFS 行为。
- 路径还原时未处理不可达顶点，导致死循环或错误输出。
- 在有权图上误用 BFS 计算最短路，得到错误答案。

## R18

源码级流程拆解（`demo.py`，8 步）：

1. **构图**：`GraphBFS.__init__` 根据 `n` 和边集建立邻接表，并检查顶点编号合法性。
2. **初始化状态**：`bfs(start)` 创建 `dist`（初始全 `-1`）、`parent`（初始全 `-1`）、空遍历序与队列。
3. **设置起点**：将 `start` 入队，并令 `dist[start]=0`。
4. **循环出队**：当队列非空时弹出队头 `u`，把 `u` 追加到访问序列。
5. **扫描邻居**：遍历 `u` 的所有邻居 `v`，仅对 `dist[v]==-1` 的未访问顶点处理。
6. **写入层次信息**：对首次发现的 `v`，设置 `dist[v]=dist[u]+1`、`parent[v]=u`，并入队。
7. **完成遍历**：队列耗尽后，返回 `BFSResult(order, distance, parent)`；不可达顶点保持 `dist=-1`。
8. **路径恢复**：`shortest_path` 从 `target` 按 `parent` 反向回溯到 `start`，再反转得到正向最短路径。
