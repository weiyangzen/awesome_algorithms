# 激光原理 (Laser Principle)

## R01

激光原理的核心是三件事同时成立：

1. 受激辐射提供与光场同相的放大；
2. 粒子数反转保证净增益为正；
3. 谐振腔反馈把放大过程闭环，形成可持续振荡。

从算法角度，最小可计算对象是“增益介质反转 + 腔内光子数”随时间演化，以及泵浦穿越阈值时的突变行为。

## R02

本条目要解决的最小问题：

给定一组归一化激光参数（增益、腔损耗、自发辐射耦合系数、泵浦），用常微分方程模拟稳态输出，并验证三条激光基本规律：

1. 阈值条件 `p_th = k/g`；
2. 阈值上方稳态光子数显著高于阈值下方；
3. 阈值上方 L-I（输出-泵浦）关系近似线性。

## R03

MVP 建模边界（半经典、可运行优先）：

- 用二维速率方程建模，不显式求解麦克斯韦方程组；
- 状态变量仅保留：
  `n(t)`（反转）和 `s(t)`（腔内光子数）；
- 使用归一化参数，强调机制而非材料细节；
- 不加入纵模竞争、横模、热效应、噪声谱等复杂工程因素。

## R04

`demo.py` 的方程如下：

- 反转演化：
  `dn/dt = p - n - g*n*s`
- 光子演化：
  `ds/dt = -k*s + g*n*s + beta*n`

符号说明：

- `p`：归一化泵浦强度；
- `g`：受激发射增益系数；
- `k`：腔损耗系数；
- `beta`：自发辐射耦合到激射模的比例。

在 `beta -> 0` 且 `s>0` 的稳态分支下，有阈值关系 `p_th = k/g`，并得到上阈值分支近似线性斜率 `ds/dp ≈ 1/k`。

## R05

脚本输出三类可核验结果：

- 阈值验证：理论阈值 vs 曲线估计阈值；
- 阈上/阈下对比：两个泵浦点的稳态光子数及其比值；
- 线性段验证：阈上拟合斜率与理论 `1/k` 的相对误差。

最终给出阈值检查与 `Validation: PASS/FAIL`。

## R06

实现策略：

- `LaserConfig` 统一管理模型与采样参数；
- `laser_rhs` 明确实现速率方程右端；
- `simulate_single_pump` 用 `scipy.integrate.solve_ivp` 求每个泵浦点的时域演化并取末值近似稳态；
- `sweep_pump` 构建 L-I 曲线；
- `make_validation_report` 生成阈值误差、阈上/阈下比值和斜率误差。

## R07

优点：

- 直接展示激光“阈值 + 受激放大 + 反馈损耗平衡”的核心机制；
- 代码短、参数清晰，适合作为教学与自动验证入口；
- 给出量化阈值检查，不是只做定性描述。

局限：

- 未建模多能级细节、泵浦动力学和饱和增益谱线；
- 未包含量子噪声、线宽、相干时间统计；
- 输出的是“腔内光子数 proxy”，不是绝对功率标定。

## R09

设泵浦采样点数为 `M`，每条轨迹时间步为 `T`（由 `t_eval` 给定）：

- 单泵浦积分复杂度近似 `O(T)`；
- 全扫泵复杂度 `O(M*T)`；
- 表格构建与统计 `O(M)`；
- 空间复杂度主要由单次 ODE 轨迹和结果表决定，近似 `O(T + M)`。

## R10

数值稳定性处理：

- 设置较严格 `rtol=1e-7`、`atol=1e-9`；
- 保留极小正初值 `s0` 以避免数值上长期停在零光子不动点；
- 对稳态光子数使用 `max(0, s_ss)` 抑制浮点负零；
- 在阈值估计中使用 `log1p(s)` 曲线，减小小信号区量级差影响。

## R11

默认参数（见 `demo.py`）：

- `g = 1.0`
- `k = 0.8`
- `beta = 1e-4`
- 泵浦扫描：`p in [0.2, 1.4]`，共 `25` 点
- 单点仿真时长：`t_end = 120`，采样 `3000` 点

对应理论阈值：`p_th = k/g = 0.8`。

## R12

内置正确性阈值：

1. 阈值估计绝对误差 `< 0.08`；
2. 阈上/阈下稳态光子数比值 `> 30`；
3. 阈上线性段斜率相对误差 `< 0.15`。

三项都满足才输出 `Validation: PASS`。

## R13

保证类型说明：

- 该条目属于物理机制数值验证，不是离散优化问题；
- 给定参数后是确定性算法，无随机抽样误差；
- “保证”体现为与理论关系的一致性阈值，而非概率保证或最优性保证。

## R14

常见失效模式：

1. `t_end` 太短导致未达稳态，阈值判断偏移；
2. `pump_points` 太少导致阈值定位粗糙；
3. `beta` 设得过大时阈值拐点被平滑，线性近似误差增大；
4. 把强工程参数直接套入归一化模型，易出现量纲混乱。

## R18

`demo.py` 的源码级算法流程（8 步）：

1. `LaserConfig` 定义增益 `g`、损耗 `k`、`beta`、泵浦扫描区间、积分时长和初值。  
2. `laser_rhs` 把激光原理写成二维速率方程：`dn/dt` 描述泵浦与受激耗尽，`ds/dt` 描述腔损耗、受激增益和自发注入。  
3. `simulate_single_pump` 对固定泵浦调用 `solve_ivp`，获得 `n(t), s(t)`，并取末值作为稳态近似。  
4. `sweep_pump` 逐点扫描泵浦，构建离散 L-I 数据（`pump, n_ss, s_ss`）。  
5. `estimate_threshold_from_curve` 对 `log1p(s_ss)` 曲线计算二阶导，取最大曲率点作为数值阈值估计。  
6. `estimate_slope_above_threshold` 在 `p >= p_th + 0.15` 区间做一阶拟合，得到阈上线性斜率。  
7. `make_validation_report` 汇总三项物理指标：阈值误差、阈上阈下光子比、斜率相对误差。  
8. `main` 打印抽样表和阈值检查，按三项阈值输出 `Validation: PASS/FAIL`，形成可自动化验收的最小闭环。

第三方库职责拆解：

- `numpy`：向量运算、导数与拟合；
- `scipy.integrate.solve_ivp`：只负责 ODE 数值积分；
- `pandas`：只负责结果表结构化与展示。

物理模型本体（方程、阈值关系、验证逻辑）都在源码显式实现，不是黑盒调用。
