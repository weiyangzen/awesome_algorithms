## R01

并查集（Disjoint Set Union, DSU / Union-Find）用于维护“元素分组 + 连通性查询”。

它支持两类核心操作：
- `find(x)`：返回元素 `x` 所在集合的代表元（根）。
- `union(a, b)`：把 `a` 与 `b` 所在集合合并。

本条目聚焦“按秩合并（union by rank）”，即合并时优先把“较矮树”挂到“较高树”下，以控制树高度。

## R02

典型应用：
- 无向图连通性维护（动态加边）。
- Kruskal 最小生成树（判断是否成环）。
- 社交网络/群组归并。
- 图像连通域标记与聚类后处理。
- 离线等价类管理（“是否属于同一集合”）。

## R03

模型定义：

给定全集 `U = {0,1,...,n-1}`，维护一个不交集合划分 `P = {S1, S2, ...}`，满足：
- `Si` 两两不交；
- 并集为 `U`。

操作语义：
- `find(x)` 返回唯一集合 `Si` 的代表元 `rep(Si)`；
- `union(a,b)` 把 `Sa` 和 `Sb` 合并为一个集合（若原本相同则不变）。

## R04

按秩合并的直觉：

- `rank` 不是“精确高度”，而是一个上界指标。
- 当两棵树合并时，把较小 `rank` 的根挂到较大 `rank` 的根上。
- 只有当两边 `rank` 相等时，新根 `rank += 1`。

这样可以避免总是把大树挂到小树导致退化链表，保持 `find` 路径较短。

## R05

路径压缩 + 按秩合并的配合：

- 按秩合并负责“结构性控制高度”。
- 路径压缩负责“查询时扁平化路径”。

二者组合后，`m` 次操作的均摊复杂度为 `O(m * α(n))`，其中 `α(n)` 是反 Ackermann 函数，增长极慢，工程上可视作近似常数。

## R06

正确性要点：

1. 每个集合以树表示，根节点满足 `parent[root] = root`。  
2. `find` 通过追溯父指针必定到达根，路径压缩不会改变集合归属。  
3. `union` 仅在根之间连边，且只连接不同根，因此集合划分始终合法。  
4. 若 `union(a,b)` 前 `find(a) == find(b)`，则返回“不发生合并”；否则分量数减 1。  
5. 整个过程中任意两点同组当且仅当根相同。

## R07

复杂度：

- 单次最坏（理论上）：接近树高。
- 均摊复杂度（路径压缩 + 按秩）：`find/union/connected` 都是 `O(α(n))`。
- 空间复杂度：`O(n)`，主要来自 `parent`、`rank`、`size` 数组。

## R09

接口约定（`UnionFindRank`）：

- `UnionFindRank(n: int)`：`n >= 0`。
- `find(x: int) -> int`：返回根。
- `union(a: int, b: int) -> bool`：若发生合并返回 `True`。
- `connected(a: int, b: int) -> bool`：是否同集合。
- `size(x: int) -> int`：`x` 所在集合大小。
- `groups() -> list[list[int]]`：返回当前所有分组（已排序，便于阅读）。

## R10

边界与异常：

- `n < 0`：构造时抛 `ValueError`。
- 越界下标：抛 `IndexError`。
- `union(x, x)`：合法但不改变结构，返回 `False`。
- `n = 0`：允许空结构，但不能执行带下标的查询。

## R11

与“按大小合并”的区别：

- 按秩合并依据 `rank`（树高上界）决策。
- 按大小合并依据子树大小决策。
- 两者都能保证极低均摊复杂度，实践性能接近。
- 本题指定“按秩合并”，因此实现以 `rank` 为主，并额外维护 `size` 供查询使用。

## R12

示例流程（8 个元素）：

- 初始：`{0},{1},{2},{3},{4},{5},{6},{7}`。
- 依次合并 `(0,1),(1,2),(3,4),(2,4),(5,6),(6,7),(4,7)`。

## R14

工程注意事项：

- `find` 建议用迭代实现，避免极端情况下递归栈问题。
- `union` 必须基于“根”判断和连接，不能直接改非根节点。
- 路径压缩与按秩合并一起使用时，`rank` 不再等于真实高度，这是正常现象。
- 若要支持回滚（rollback DSU），不能直接使用完全路径压缩，需要改造。

## R18

`demo.py` 的源码级算法流程（7 步）：

1. 初始化结构：`parent[i]=i`、`rank[i]=0`、`comp_size[i]=1`、`components=n`。  
2. 执行 `find(x)`：沿 `parent` 追到根 `r`，再把路径上节点直接挂到 `r`（路径压缩）。  
3. 执行 `union(a,b)`：先 `find` 出 `ra, rb`，若相同则返回 `False`。  
4. 比较秩：若 `rank[ra] < rank[rb]` 则交换，使 `ra` 成为高秩根。  
5. 合并根：`parent[rb] = ra`，并把 `comp_size[ra] += comp_size[rb]`，分量数减 1。  
6. 秩更新：仅当原先 `rank[ra] == rank[rb]` 时，执行 `rank[ra] += 1`。  
7. 正确性验证：主程序对确定性样例与随机对拍逐步断言 `connected/union/groups` 一致性。

这里没有调用第三方并查集黑盒，流程与代码一一对应。
