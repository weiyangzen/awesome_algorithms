## R01

快速傅里叶变换（Fast Fourier Transform, FFT）是离散傅里叶变换（DFT）的快速算法。  
它把长度为 `n` 的序列从时域映射到频域，把原本 `O(n^2)` 的 DFT 计算降到 `O(n log n)`，是数值计算、信号处理、计算机视觉、科学计算和竞赛算法中的基础组件。

## R02

本条目聚焦最常见的 `radix-2 Cooley-Tukey` 形式（长度为 2 的幂）。

给定复数序列 `x[0..n-1]`，DFT 定义为：

`X[k] = Σ_{t=0}^{n-1} x[t] * exp(-2πi * k * t / n)`。

逆变换为：

`x[t] = (1/n) * Σ_{k=0}^{n-1} X[k] * exp(2πi * k * t / n)`。

FFT 的目标是高效计算这两个变换，而不是改变变换定义本身。

## R03

核心分治思想：把原序列拆成“偶下标子序列”和“奇下标子序列”。

- 偶部：`x[0], x[2], x[4], ...`
- 奇部：`x[1], x[3], x[5], ...`

设 `E[k]`、`O[k]` 分别是它们长度 `n/2` 的 DFT，则可合并为：

- `X[k] = E[k] + W_n^k * O[k]`
- `X[k+n/2] = E[k] - W_n^k * O[k]`

其中 `W_n = exp(-2πi / n)`，这一对更新就是蝶形（butterfly）。

## R04

复杂度分析：

- 朴素 DFT：`n` 个频点，每个频点累加 `n` 项，时间 `O(n^2)`。
- FFT：递归/迭代共 `log2(n)` 层，每层处理 `n` 次常数级蝶形，时间 `O(n log n)`。
- 对于大规模数据，FFT 相比 DFT 有数量级优势。

## R05

空间复杂度与本实现约定：

- 本 MVP 使用迭代原地蝶形，额外空间约 `O(1)`（不计输入输出数组）。
- 为支持“线性卷积”，会把输入补零到长度 `N >= len(a)+len(b)-1` 的最小 2 次幂。
- 数据类型使用 `numpy.complex128`，在精度与可读性之间做平衡。

## R06

最常见应用是快速卷积。

多项式/序列线性卷积定义：

`c[k] = Σ a[i] * b[k-i]`。

流程：

1. 把 `a,b` 补零到统一长度 `N`（2 的幂）。
2. 分别做 FFT 得到 `A,B`。
3. 频域逐点乘 `C[t]=A[t]*B[t]`。
4. 对 `C` 做 IFFT 并截取前 `len(a)+len(b)-1` 项。

该流程把卷积从 `O(n^2)` 降到 `O(n log n)`。

## R07

算法价值：

- 信号处理：频谱分析、滤波、相关性计算。
- 科学计算：偏微分方程谱方法、快速积分。
- 计算几何与图像：卷积核加速、频域特征处理。
- 算法竞赛：大整数乘法、多项式乘法、计数卷积。

FFT 是“把卷积转化为逐点乘法”的关键桥梁。

## R08

关键理论依赖：

- 复根单位根性质：`W_n^n=1` 且 `W_n^{k+n/2} = -W_n^k`（`n` 为偶数）。
- DFT 线性性：可分解偶、奇子问题。
- 分治复用：子问题结果可在合并阶段共享，不必重复计算。
- 卷积定理：时域卷积对应频域逐点乘。

这些性质共同保证 FFT 的正确性与高效性。

## R09

适用前提与边界：

- radix-2 FFT 需要长度是 2 的幂；否则常用补零或混合基 FFT。
- 输入可为实数或复数，实数会自动提升为复数计算。
- 空数组卷积无定义，本 MVP 对空输入直接抛异常。
- 逆变换必须做 `1/n` 归一化，否则振幅会整体放大。

## R10

正确性直觉（简述）：

设 `n=2m`。对 DFT 求和按下标奇偶拆分：

`X[k] = Σ x[2r]W_n^{k(2r)} + Σ x[2r+1]W_n^{k(2r+1)}`。

利用 `W_n^2 = W_m` 得：

`X[k] = E[k] + W_n^k O[k]`，其中 `E,O` 分别是长度 `m` 的 DFT。  
再利用 `W_n^{k+m} = -W_n^k` 得到另一半频点：

`X[k+m] = E[k] - W_n^k O[k]`。

这正是蝶形公式。层层递推到长度 1 即完成证明框架；逆变换同理，仅指数符号与归一化不同。

## R11

数值稳定性与误差：

- FFT 基于浮点复数运算，会有舍入误差；验证时应使用 `np.allclose`。
- IFFT 后理论上实数卷积结果可能带极小虚部（如 `1e-12j`），通常取 `real` 即可。
- 长度越大、数值动态范围越大，误差累积越明显；工程上常结合缩放与容差控制。

## R12

工程真实成本：

- 大 O 相同不代表常数相同：小规模时 `numpy.convolve` 或朴素法可能更快。
- FFT 需要补零到 2 次幂，可能带来额外计算开销。
- 内存带宽、缓存行为、向量化能力会显著影响真实性能。
- 因此应按规模阈值选择内核，而不是盲目“全量 FFT”。

## R13

- 近似比/概率保证：N/A。  
- 原因：FFT 本身是确定性数值算法，不是随机近似优化算法。  
- 但在浮点环境中结果受舍入误差影响，应采用数值容差判定正确性。

## R14

鲁棒性与失效模式：

- 忘记位逆序重排：输出会完全错误。
- 蝶形中直接覆盖但未保存 `u,v`：会污染后续计算。
- IFFT 忘记除以 `n`：结果整体放大 `n` 倍。
- 卷积补零长度不足：会从线性卷积变成循环卷积（混叠错误）。

本目录 demo 针对以上点都给出显式实现与校验。

## R15

实现要点（`demo.py`）：

- `next_power_of_two`：计算补零目标长度。
- `bit_reverse_permute`：完成迭代 FFT 所需位逆序重排。
- `fft_iterative`：迭代 radix-2 蝶形，支持 `inverse=True`。
- `linear_convolution_fft`：FFT 卷积主流程。
- `linear_convolution_naive`：`O(n^2)` 参考实现用于对拍。

代码目标是“透明可读 + 可验证”，而非依赖黑箱一行 API。

## R16

前驱、后继与扩展：

- 前驱基础：复数运算、单位根、分治思想。
- 后继方向：NTT、实数 FFT、二维 FFT、Bluestein（非 2 次幂）。
- 实际应用：大整数乘法、谱分析、相关匹配、快速多项式运算。

FFT 常被视作“通用数值加速器”。

## R17

MVP 运行说明：

```bash
python3 demo.py
```

脚本无交互输入，会执行：

1. 固定样例卷积（FFT vs 朴素 vs `numpy.convolve`）。
2. 固定长度 DFT 对照（FFT vs 朴素 DFT vs `numpy.fft.fft`）。
3. 多组随机卷积对拍。

若全部通过，最后打印 `All FFT checks passed.`。

## R18

`demo.py` 源码级算法流（8 步）：

1. `main` 准备输入序列 `a,b`，并计算线性卷积目标长度 `need=len(a)+len(b)-1`。  
2. `next_power_of_two` 计算 `n`，把输入补零到 `n`，避免循环卷积混叠。  
3. `fft_iterative` 先调用 `bit_reverse_permute`，把数组重排为位逆序布局，便于原地蝶形分层访问。  
4. 从块长 `2` 开始逐层扩展到 `n`，每层用旋转因子 `wlen = exp(±2πi/len)` 执行蝶形：`u+v` 与 `u-v`。  
5. 对 `a`、`b` 分别完成正变换得到 `A,B`，然后频域逐点相乘 `C=A*B`。  
6. 对 `C` 执行 `inverse=True` 的同构蝶形流程，并在末尾统一除以 `n` 得到时域结果。  
7. 截取前 `need` 项得到线性卷积输出，并和 `linear_convolution_naive`、`numpy.convolve` 做数值比对。  
8. 额外执行朴素 DFT 与随机测试，确保“变换正确 + 卷积正确 + 代码非黑箱”三项同时成立。
