## R01

Galois群计算算法的目标是：给定域 `Q` 上多项式 `f(x)`，求其分裂域 `K` 的自同构群

`Gal(K/Q) = { sigma: K->K | sigma 是域同构且在 Q 上恒等 }`。

本目录 MVP 采用“低阶可解释版本”：
- 精确支持 `deg(f)=2` 与 `deg(f)=3` 的情形；
- 输出群类型（如 `TRIVIAL`、`C2`、`A3`、`S3`）和群阶；
- 对 `deg(f)>=4` 明确给出“不在当前 MVP 支持范围”。

## R02

历史上，Galois 理论把“方程可解性”与“群结构”连接起来。对于低阶多项式，群判定可由判别式、可约性和根结构直接给出：
- 二次不可约多项式对应 `C2`；
- 三次不可约多项式对应 `A3` 或 `S3`（由判别式是否平方决定）。

现代完整实现常用 CAS（如 Magma/Pari/Sage）做高阶群计算；本 MVP 则强调源码可追踪和基础判定逻辑。

## R03

待解决问题（本实现边界内）：
1. 输入整数系数多项式 `f(x) in Z[x]`（视作 `Q[x]`）。
2. 判断其在 `Q` 上的分解模式（是否有有理根、是否不可约）。
3. 对 2/3 次多项式输出可解释的 Galois 群类别。

本 MVP 不追求通用高阶群计算，而是提供一个可运行、可验证、可阅读的最小算法实现。

## R04

核心方法组合：
1. 有理根定理：枚举 `p/q` 候选根（`p|a0, q|an`）。
2. 精确 Horner + 合成除法（`Fraction`）提取线性因子。
3. 判别式规则：
   - 二次 `ax^2+bx+c`：`Delta=b^2-4ac`；
   - 三次 `ax^3+bx^2+cx+d`：标准判别式公式。
4. 群分类规则：
   - 二次：平方判别式 -> `TRIVIAL`，否则 `C2`；
   - 三次不可约：`Delta` 平方 -> `A3(≅C3)`，否则 `S3`；
   - 三次可约：按 `1+2` 或 `1+1+1` 分解模式给 `C2/TRIVIAL`。

## R05

设多项式次数为 `n`，首项系数 `an`，常数项 `a0`。

对本 MVP（主要 2/3 次）复杂度很低；一般化估计：
- 候选有理根个数约 `O(tau(|a0|) * tau(|an|))`，`tau` 为约数个数函数；
- 每个候选做一次 Horner 评估成本 `O(n)`；
- 每次命中根后做一次合成除法 `O(n)`。

因此总体可写为：
- 时间：`O(C*n + R*n)`，其中 `C` 是候选根数量，`R` 是提取到的线性因子数量；
- 空间：`O(n)`（保存当前系数与中间结果）。

## R06

三次不可约示例：`f(x)=x^3-2`
- 先做有理根测试：`±1, ±2` 都不是根 -> 在 `Q` 上不可约。
- 判别式：`Delta = -108`（非平方）。
- 结论：`Gal(f/Q) ≅ S3`，群阶 `6`。

三次可约示例：`f(x)=x^3-2x^2+x-2=(x-2)(x^2+1)`
- 有一个有理根 `2`，剩余二次因子 `x^2+1` 不可约。
- 结论：分裂域由该二次扩张给出，群为 `C2`。

## R07

优点：
- 不依赖黑盒 CAS，关键步骤都在源码中可追踪。
- 用精确有理数运算避免浮点误判（判根、除法均为精确）。
- 对二次/三次场景可稳定给出标准结论。

局限：
- 不支持四次及以上通用 Galois 群计算。
- 系数规模很大时，有理根枚举成本会增加。
- 对判别式为 0（重根）的三次，仅标注为非可分情形，不进一步细分群结构。

## R10

正确性依据（本 MVP）：
1. 有理根检测：由有理根定理给出完备候选集，且用 `Fraction` 精确验证 `f(r)=0`。
2. 因式提取：合成除法在有理域上精确进行，不引入数值误差。
3. 二次分类：`Delta` 平方与否等价于是否在 `Q` 中分裂。
4. 三次不可约分类：经典定理给出 `Delta` 平方 <-> 群嵌入 `A3`，否则 `S3`。
5. 三次可约分类：分解模式 `1+2` 对应二次扩张自同构 `C2`，`1+1+1` 对应平凡群。

## R11

实现层面的稳定性策略：
- 判根、除法、系数归一化使用 `fractions.Fraction`，避免浮点误差。
- 判平方采用整数 `isqrt` 精确判断，不做浮点近似。
- `numpy.roots` 仅用于“展示近似根”，不参与群判定核心逻辑。

这样可保证“分类决策”与“可视化展示”解耦。

## R12

## R14

常见失效模式与防护：
1. 输入零多项式。
   - 防护：`normalize_int_coeffs` 直接抛错。
2. 误把浮点根当作精确根。
   - 防护：核心逻辑不用浮点，统一 `Fraction` 判定。
3. 超出支持范围（`deg>=4`）被误用。
   - 防护：明确输出 `UNSUPPORTED` 并附边界说明。
4. 三次重根误作 `S3/A3`。
   - 防护：先检查 `Delta==0`，标注为 `NON_SEPARABLE_CASE`。

## R18

源码级流程拆解（`demo.py`，8 步）：
1. `normalize_int_coeffs` 清洗输入系数并确定次数，排除零多项式。
2. `extract_rational_linear_factors` 把当前多项式转为 `Fraction`，循环提取有理线性因子。
3. 每轮提取中，`rational_root_candidates` 基于有理根定理生成 `p/q` 候选集。
4. `eval_poly_fraction` 做精确 Horner 验证；命中后用 `synthetic_division_fraction` 精确降阶。
5. `classify_galois_group_q` 根据原次数（2/3）和分解模式进入对应分支。
6. 二次分支用 `discriminant_deg2` 与 `is_perfect_square` 判 `TRIVIAL` 或 `C2`。
7. 三次分支先用 `discriminant_deg3` 判重根，再按“不可约 + 判别式平方性”给 `A3/S3`，或按可约模式给 `C2/TRIVIAL`。
8. `main` 汇总结果为 `pandas.DataFrame`、打印判定理由，并用内置期望值做回归校验。

补充说明：`numpy.roots` 仅用于展示近似根，不参与第 2~7 步的群判定核心逻辑。
