# 牛顿运动定律 (Newton's Laws of Motion)

## R01

牛顿运动定律是经典力学的基础公理体系，用于描述宏观低速物体在惯性参考系中的运动规律。三条定律分别回答三个核心问题：

- 第一定律：在合外力为零时，物体如何运动；
- 第二定律：外力如何改变运动状态（加速度）；
- 第三定律：相互作用力如何成对出现。

在工程和科学计算里，三定律可统一写成“受力分析 -> 动力学方程 -> 数值积分 -> 状态更新”的算法流程。

## R02

历史上，牛顿在 1687 年《自然哲学的数学原理》中系统提出该体系。它把开普勒天文学、伽利略实验事实和几何推导统一到同一力学框架，形成了后续工程力学、航天动力学和数值仿真的共同语言。

在现代语境中，牛顿定律既是物理理论，也是大量仿真软件、游戏引擎和机器人控制器中的“底层更新规则”。

## R03

牛顿运动定律解决的问题可概括为：

- 给定质量、初始状态和作用力，预测未来位置与速度；
- 判断系统中相互作用是否满足力的守恒结构（作用-反作用）；
- 将定性“受力会改变运动”转为可计算的微分方程。

这类问题是轨迹预测、机械系统设计、碰撞分析和稳定性评估的核心。

## R04

基础数学形式：

- 第一定律：若 `sum(F)=0`，则 `a=0`，速度 `v` 保持常量；
- 第二定律：`sum(F)=m*a`，即 `a=sum(F)/m`；
- 第三定律：两物体相互作用力满足 `F_12 = -F_21`。

若用时间离散步长 `dt` 做最小 MVP，可用半隐式欧拉更新：

- `v_{t+1} = v_t + a_t * dt`
- `x_{t+1} = x_t + v_{t+1} * dt`

该形式简单、稳定性优于显式欧拉位置更新，适合教学和快速验证。

## R05

设仿真步数为 `T`，粒子数为 `N`，配对作用边数为 `E`：

- 无相互作用的独立粒子更新复杂度约为 `O(T*N)`；
- 若存在稀疏配对作用（如弹簧连接），复杂度约为 `O(T*(N+E))`；
- 空间复杂度约 `O(N)` 到 `O(N+E)`（取决于是否存全轨迹）。

本目录 `demo.py` 采用 1D 小系统，时间和空间成本都很低。

## R06

直观示例：质量 `m=2 kg` 的小车受恒力 `F=6 N`，则

- `a = F/m = 3 m/s^2`；
- 若初速度 `v0=1 m/s`，`2 s` 后理论速度 `v = v0 + a*t = 7 m/s`。

这就是第二定律最常见的工程计算模板，`demo.py` 也会通过拟合数值轨迹恢复该加速度并与理论值比较。

## R07

优点：

- 物理含义直观，变量可测量；
- 与数值积分天然兼容，易写成可执行代码；
- 在低速、宏观、弱相对论条件下精度高。

局限：

- 强非惯性系需要显式引入惯性力。

## R09

典型适用场景：

- 教学演示：从受力到轨迹的完整闭环；
- 机械/机器人：低速刚体与执行器估算；
- 游戏物理：角色与物体的基础动力学更新；
- 工程预估：对复杂仿真前的快速量级检查。

不适合直接使用的场景：

- 涉及相对论或量子效应主导的系统。

## R10

实现正确性的关键检查点：

1. 第一定律：`sum(F)=0` 时速度时间序列应近似常数；
2. 第二定律：由数值轨迹回归得到的 `a_hat` 应接近 `F/m`；
3. 第三定律：每个时间步都应满足 `F_12 + F_21 ~= 0`；
4. 封闭系统动量 `p = m1*v1 + m2*v2` 应近似守恒；
5. 代码中单位必须一致（`N, kg, m, s`）。

## R11

数值稳定策略：

- 采用较小时间步 `dt`，减小离散误差；
- 使用半隐式欧拉更新，降低能量发散风险；
- 对线性回归和阈值断言设置合理容忍度（浮点误差）；
- 全程使用 `float64`，避免累计误差过大。

## R12

本 MVP 的主要可调参数：

- `dt`：越小越精确，代价是步数增加；
- `steps`：决定仿真时长与拟合稳健性；
- `k`（弹簧刚度）：过大时需更小 `dt` 保稳定；
- 质量比 `m1/m2`：影响相互作用下速度变化幅度。

推荐先固定 `dt=1e-3 ~ 1e-2`，再根据误差目标调整其余参数。

## R13

理论性质（本离散实现对应）：

- 连续模型层面，牛顿定律是确定性常微分方程系统；
- 在满足 Lipschitz 条件的受力函数下，初值问题局部唯一；
- 本 MVP 不涉及随机近似，因此不存在“成功概率”或“近似比”指标；
- 收敛误差主要由时间离散方法阶数决定（欧拉法一阶）。

## R14

常见失效模式与对策：

- `dt` 过大导致轨迹震荡或漂移。  
  对策：减小 `dt`，必要时改更高阶积分器。
- 力方向写反导致第三定律检查失败。  
  对策：统一使用 `F_12 = -F_21` 的单源计算方式。
- 单位混用（如 g 与 kg）。  
  对策：在代码中显式注释单位并做量纲自检。
- 将非惯性系数据直接套用惯性系方程。  
  对策：补充惯性力项或改用惯性坐标系数据。

## R15

工程落地建议：

- 把受力项封装为可插拔函数，便于替换摩擦、阻尼、控制输入；
- 将“守恒量监控”作为仿真健康指标（动量/能量漂移）；
- 先用简化 1D/2D 案例验证，再扩展到高维和复杂约束；
- 为关键参数建立自动化回归测试，避免后续改动破坏物理一致性。

## R16

与相关方法关系：

- 拉格朗日力学：用能量函数与广义坐标推导运动方程，适合约束系统；
- 哈密顿力学：在相空间中描述系统演化，守恒结构更显式；
- 本项牛顿形式：最直接、最工程化，便于从受力图到代码更新。

在许多工程软件中，三者可互相推导，但牛顿形式通常是实现层的第一入口。

## R17

`demo.py` 完成三个可复现实验：

- 第一定律实验：零外力粒子，验证速度恒定；
- 第二定律实验：恒力驱动粒子，验证 `a_hat ≈ F/m`；
- 第三定律实验：双质量弹簧相互作用，验证力对和动量守恒。

运行方式：

```bash
cd "Algorithms/物理-经典力学-0001-牛顿运动定律_(Newton's_Laws_of_Motion)"
uv run python demo.py
```

脚本无需交互输入，会打印指标表并执行断言。

## R18

`demo.py` 的源码级算法流程（8 步）：

1. `simulate_first_law` 初始化单粒子状态，在 `net_force=0` 下按离散时间更新，得到 `t/x/v` 序列。  
2. `evaluate_first_law` 计算速度最大偏差 `max|v-v0|`，作为第一定律数值误差指标。  
3. `simulate_second_law` 对恒力系统执行半隐式欧拉积分，记录速度随时间变化。  
4. `evaluate_second_law` 用 `scipy.stats.linregress` 对 `v(t)` 做线性拟合，得到 `a_hat` 并与 `F/m` 比较。  
5. `simulate_third_law` 构造双质量弹簧系统，每步由同一 `force_12` 生成 `force_21=-force_12` 后更新状态。  
6. `evaluate_third_law` 统计 `|F_12+F_21|` 与总动量漂移 `|p(t)-p(0)|`，验证第三定律与封闭系统动量守恒。  
7. `build_report` 将三条定律的关键指标整理为 `pandas.DataFrame`，统一输出。  
8. `main` 串联三组实验、打印报告并执行阈值断言，形成“物理公理 -> 数值实验 -> 自动校验”的最小闭环。
