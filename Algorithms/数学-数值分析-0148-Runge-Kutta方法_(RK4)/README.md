# Runge-Kutta方法 (RK4)

- UID: `MATH-0148`
- 学科: `数学`
- 分类: `数值分析`
- 源序号: `148`
- 目标目录: `Algorithms/数学-数值分析-0148-Runge-Kutta方法_(RK4)`

## R01

本条目实现经典四阶 Runge-Kutta 方法（RK4）的最小可运行版本，用于求解常微分方程初值问题（IVP）：
- 方程：`y' = f(t, y)`
- 初值：`y(t0) = y0`

MVP 目标：
- 明确展示 RK4 的四次斜率采样与加权更新；
- 给出可复现的误差与收敛阶验证；
- 脚本可直接运行，无需交互输入。

## R02

问题定义（固定步长、标量状态版）：
- 输入：右端函数 `f(t, y)`、`t0`、`y0`、步长 `h > 0`、步数 `N`；
- 输出：离散网格 `t_n = t0 + n*h` 与近似解 `y_n`；
- 目标：在 `n=0..N` 上用 RK4 近似真解并评估误差。

本实现聚焦“算法透明性”，不引入自适应步长控制。

## R03

RK4 核心公式（一步从 `n` 到 `n+1`）：

- `k1 = f(t_n, y_n)`
- `k2 = f(t_n + h/2, y_n + h*k1/2)`
- `k3 = f(t_n + h/2, y_n + h*k2/2)`
- `k4 = f(t_n + h, y_n + h*k3)`

更新：
- `y_{n+1} = y_n + (h/6) * (k1 + 2*k2 + 2*k3 + k4)`
- `t_{n+1} = t_n + h`

性质：局部截断误差 `O(h^5)`，全局误差 `O(h^4)`。

## R04

算法流程：
1. 校验输入合法（有限数、`h>0`、`steps>0`）。
2. 初始化 `(t, y) = (t0, y0)`，写入轨迹数组。
3. 每步计算 `k1,k2,k3,k4` 四个斜率。
4. 按 RK4 加权公式得到 `y_next`。
5. 更新时间 `t <- t + h`，记录新状态。
6. 循环 `N` 次后返回完整轨迹。

## R05

核心数据结构：
- `t_values: list[float]`：网格点；
- `y_values: list[float]`：RK4 数值解；
- `RK4StepTrace(dataclass)`：保存每步 `k1..k4` 与 `y_next`，用于源级可解释输出；
- `results: list[tuple[h, steps, final_error, max_error]]`：收敛实验结果。

## R06

正确性要点：
- `demo.py` 严格按经典 RK4 公式计算四个阶段斜率和加权平均；
- 采用有解析解的测试方程 `y' = y - t^2 + 1, y(0)=0.5`，可逐点对照；
- 当 `h` 逐步减半时，最大误差应大致按 `h^4` 缩放，经验阶接近 4。

## R07

复杂度分析：
- 单次积分（`N` 步）：
  - 时间复杂度：`O(N)`；
  - 空间复杂度：
    - 保留完整轨迹与 trace 时 `O(N)`；
    - 若仅保留当前状态可降为 `O(1)`。
- 常数开销：RK4 每步需要 4 次 `f` 评估。

## R08

边界与异常处理：
- `h <= 0`：抛出 `ValueError`；
- `steps <= 0`：抛出 `ValueError`；
- `t0/y0/h` 非有限：抛出 `ValueError`；
- `k1..k4` 或 `y_next` 非有限：抛出 `RuntimeError`；
- `t_end / h` 非近整数（本 demo 用固定整步网格）：抛出 `ValueError`。

## R09

MVP 取舍：
- 使用 Python 标准库实现，避免黑盒 ODE 求解器；
- 只做标量方程与固定步长，先保障“公式-代码-输出”链路清晰；
- 提供内部斜率 trace，强调教学可解释性；
- 不覆盖刚性系统、自适应步长、事件检测等进阶特性。

## R10

`demo.py` 函数职责：
- `validate_inputs`：参数合法性检查；
- `rk4_step`：执行单步 RK4 更新并返回 `k1..k4`；
- `rk4_solve`：循环积分，输出轨迹与 step trace；
- `ode_rhs` / `ode_exact`：示例方程与解析解；
- `max_abs_error`：计算网格最大绝对误差；
- `run_single_resolution`：执行单个步长实验；
- `estimate_orders`：估计经验收敛阶；
- `print_trajectory_sample` / `print_step_trace`：展示结果与内部阶段量；
- `main`：组织批量实验并打印。

## R12

输出解读：
- `h`：步长；
- `steps`：对应积分步数；
- `final_abs_error`：终点误差；
- `max_abs_error`：全网格最大误差；
- `estimated order p`：相邻步长误差比估计的收敛阶；
- `k1..k4`：单步内部斜率，可用于检查 RK4 公式落地是否正确。

## R13

建议最小测试集：
- 收敛测试：`h in {0.2, 0.1, 0.05, 0.025}`；
- 参数异常：`h=0`、`h<0`、`steps=0`；
- 数值异常：`t0=nan`、`y0=inf`；
- 网格异常：`t_end=2.0, h=0.3`（不能整除区间）。

## R14

关键可调参数：
- `h_values`：步长列表，控制精度与计算成本；
- `t_end`：积分终点；
- `rows` / `trace_rows`：输出轨迹与内部斜率展示行数。

实践建议：先用较大步长快速验证流程，再逐步减小步长检查四阶收敛行为。

## R15

方法对比（同类单步法）：
- Euler：每步 1 次函数评估，`O(h)` 全局误差；
- Heun / 改进 Euler：每步 2 次评估，`O(h^2)`；
- RK4：每步 4 次评估，`O(h^4)`，通常在精度-成本上表现稳定。

在相同步长下 RK4 误差通常显著小于一阶和二阶方法。

## R16

应用场景：
- 非刚性 ODE 初值问题的工程近似；
- 课程教学中的高阶单步法示例；
- 需要稳定、可解释基线求解器的原型阶段；
- 作为更复杂求解器（自适应 RK、多步法）的对照基准。

## R18

源码级算法流（对应 `demo.py`，8 步）：
1. `main` 设定 `h_values`，逐个调用 `run_single_resolution` 执行 RK4 收敛实验。  
2. `run_single_resolution` 将 `t_end / h` 转为整数步数并检查网格一致性。  
3. `rk4_solve` 先调用 `validate_inputs`，拒绝非法步长、步数和非有限输入。  
4. 每步内 `rk4_step` 依次计算 `k1, k2, k3, k4` 四个阶段斜率。  
5. `rk4_step` 用加权组合 `y_next = y + (h/6)*(k1+2k2+2k3+k4)` 生成下一状态。  
6. `rk4_solve` 将新 `(t, y)` 写入轨迹，并把 `k1..k4` 存入 `RK4StepTrace`。  
7. 回到 `run_single_resolution` 后，以 `ode_exact` 计算终点误差与全局最大误差。  
8. `main` 汇总多步长误差，用 `estimate_orders` 估计阶数，并打印轨迹与内部斜率样例。
