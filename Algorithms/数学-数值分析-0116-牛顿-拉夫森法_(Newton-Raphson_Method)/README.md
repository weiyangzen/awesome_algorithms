# 牛顿-拉夫森法 (Newton-Raphson Method)

## R01

牛顿-拉夫森法用于求解非线性方程 `f(x)=0` 的数值根，是数值分析中最常用的一类迭代法。  
本条目给出一个“可直接运行、步骤透明”的标量 MVP：通过手写迭代公式与轨迹打印，展示牛顿法如何从初值收敛到目标根。

## R02

本目录要解决的问题是：
- 输入：目标函数 `f`、导数 `f'`、初值 `x0`、误差阈值 `tol`、最大迭代数 `max_iter`。
- 输出：方程根的近似值 `x*`，以及每一轮迭代轨迹（`x_k`、`f(x_k)`、步长 `|delta_k|`）。

`demo.py` 内置三组固定样例，无交互输入。

## R03

牛顿法数学更新式：

对于当前迭代点 `x_k`，在该点做一阶泰勒近似：
`f(x) ≈ f(x_k) + f'(x_k)(x - x_k)`。

令近似式为 0，可得下一步更新：
`x_(k+1) = x_k - f(x_k) / f'(x_k)`。

若 `f` 在根附近足够光滑且 `f'(x*) != 0`，并且初值落在收敛域内，牛顿法通常表现为局部二次收敛。

## R04

算法高层流程：
1. 检查输入合法性（有限数、`tol > 0`、`max_iter > 0`）。
2. 设当前值 `x = x0`。
3. 计算 `f(x)` 与 `f'(x)`。
4. 若 `|f'(x)|` 太小则终止并报错（防止除零/不稳定更新）。
5. 计算牛顿步长 `delta = f(x)/f'(x)`，更新 `x <- x - delta`。
6. 记录迭代轨迹并判断停止条件（残差或步长阈值）。
7. 达到停止条件返回结果；否则继续下一轮，直到 `max_iter`。

## R05

核心数据结构：
- `HistoryItem = (iter, x_k, f(x_k), |delta_k|)`：单轮迭代记录。
- `history: list[HistoryItem]`：完整轨迹。
- `cases: list[dict]`：`main` 中的固定演示任务（函数、导数、初值、参考解）。

MVP 保持单变量标量结构，避免不必要框架复杂度。

## R06

正确性要点：
- 更新公式严格来自牛顿法定义 `x_{k+1}=x_k-f/f'`。
- 每轮都重新计算当前点处的 `f` 与 `f'`，保证切线更新正确。
- 终止判据同时使用残差 `|f(x_k)|` 与步长 `|delta_k|`，可避免只依赖单一指标。
- 对照已知参考根时，若实现正确，应看到误差快速下降到接近机器精度。

## R07

复杂度分析（单个标量方程）：
- 设实际迭代轮数为 `T`。
- 时间复杂度：`O(T * (C_f + C_df))`，其中 `C_f/C_df` 为函数与导数单次计算开销。
- 空间复杂度：
  - 保留完整轨迹时：`O(T)`；
  - 仅保留当前值时：`O(1)`。

## R08

边界与异常处理：
- 初值非有限（`nan/inf`）直接抛出 `ValueError`。
- `tol <= 0` 或 `max_iter <= 0` 抛出 `ValueError`。
- 迭代中若 `f(x)`、`f'(x)`、`x_next` 非有限，抛出 `RuntimeError`。
- 若 `|f'(x)|` 低于 `derivative_floor`，抛出 `RuntimeError`，避免数值爆炸。

## R09

MVP 取舍说明：
- 采用“手写牛顿迭代”而非 `scipy.optimize.newton` 黑盒调用，便于核对每个数值步骤。
- 仅做标量问题，不扩展到 Jacobian 的多元方程组，保持最小可理解实现。
- 仅引入 `numpy` 做有限性判断与统计，不引入更重依赖。

## R10

`demo.py` 主要函数职责：

## R12

输出字段说明：
- `iter / x_k / f(x_k) / |delta|`：每轮迭代轨迹。
- `root_estimate`：牛顿法得到的近似根。
- `reference`：预置参考根（用于校验）。
- `abs_error / rel_error`：绝对与相对误差。
- `final_residual`：最终残差 `|f(root_estimate)|`。
- `Summary`：全样例最大相对误差、平均相对误差、通过标志。

## R14

可调参数：
- `tol`：停止阈值，默认 `1e-12`。
- `max_iter`：最大迭代轮数，默认 `30`。
- `derivative_floor`：导数下限，默认 `1e-14`。
- `max_lines`（打印层）：单个样例最多打印多少行迭代轨迹。

工程建议：先固定严格阈值验证正确性，再按性能或输出长度做适度放宽。

## R15

方法对比：
- 与二分法相比：牛顿法通常收敛更快（二次 vs 线性），但需要导数且对初值敏感。
- 与割线法相比：牛顿法每轮成本略高（需导数），但通常步数更少、更稳定。
- 与黑盒库接口相比：手写版本可追踪每步细节，更适合教学与算法审计。

## R16

典型应用场景：
- 科学计算中的非线性方程求根（物理、化学、工程参数反解）。
- 作为更复杂方法的核心子步骤（如隐式时间推进、最优化子问题）。
- 数值分析教学中用于展示“切线逼近 + 局部二次收敛”的经典例子。

## R18

`demo.py` 源码级算法流程（8 步）：
1. `main` 定义三组方程样例（函数、导数、初值、参考根），并逐个调用 `run_case`。  
2. `run_case` 把配置传入 `newton_raphson`，开始实际迭代，同时准备后续误差统计。  
3. `newton_raphson` 先做参数检查：`x0` 有限、`tol/max_iter/derivative_floor` 合法。  
4. 每轮先计算当前 `f(x)` 与 `f'(x)`；若导数太小或数值非有限则立即报错终止。  
5. 计算牛顿步长 `delta = f(x)/f'(x)`，执行更新 `x_next = x - delta`。  
6. 计算更新后残差 `f(x_next)`，把 `(iter, x_next, f(x_next), |delta|)` 追加到 `history`。  
7. 使用双停止条件判断是否收敛：`|f(x_next)| <= tol` 或 `|delta| <= tol*(1+|x_next|)`；否则继续循环。  
8. `run_case` 打印轨迹与误差，`main` 汇总全部样例的最大/平均相对误差与通过标记。
