# 选择排序 (Selection Sort)

## R01

选择排序（Selection Sort）是一种原地比较排序算法。它在第 `i` 轮从区间 `[i, n-1]` 中选出最小值，放到位置 `i`，逐步构建有序前缀。

本目录 MVP 的目标：
- 用可审计的 Python 实现完整选择排序流程；
- 输出比较次数、交换次数和每轮状态；
- 用 `numpy` 内置排序仅作结果校验，不作为主算法实现。

## R02

历史与定位：
- 选择排序是早期教学中最常见的基础排序算法之一；
- 与冒泡排序相比，选择排序交换次数更少（最多 `n-1` 次）；
- 与插入排序相比，它对“已部分有序”数据不敏感，复杂度通常不降。

因此它更适合教学“比较排序”和“原地交换”思想，而非高性能生产排序。

## R03

问题定义：
- 输入：长度为 `n` 的可比较序列 `A`（本实现为有限实数列表）。
- 输出：非降序序列 `A'`，满足：
  - 有序性：`A'[0] <= A'[1] <= ... <= A'[n-1]`
  - 排列保持：`A'` 与 `A` 含有完全相同的元素多重集。

选择排序在每一轮固定一个元素的最终位置，直到全局有序。

## R04

核心思想：
1. 设当前轮起点为 `i`。
2. 在线性扫描中找到最小元素下标 `min_idx`。
3. 交换 `A[i]` 与 `A[min_idx]`（若 `min_idx == i` 则不交换）。
4. `i` 递增，继续处理后续未排序区间。

不变量：第 `i` 轮结束后，前缀 `A[0:i+1]` 已有序，且是原数组中最小的 `i+1` 个元素。

## R05

数据结构与状态量：
- 主数组：`arr`（列表，原地更新）。
- 轮次变量：`i`（有序前缀终点）。
- 最小值指针：`min_idx`（当前轮最小元素下标）。
- 统计量：
  - `comparisons`：比较次数；
  - `swaps`：实际交换次数（`min_idx != i` 时加 1）。
- 轨迹记录：每轮的 `i/min_index/min_value/swapped/array_state`。

## R06

正确性说明（交换式归纳）：
- 基础步：`i=0` 时，算法将全局最小值放到 `A[0]`，命题成立。
- 归纳假设：第 `i-1` 轮后，`A[0:i]` 为最小 `i` 个元素且有序。
- 归纳推进：第 `i` 轮从 `A[i:n]` 选最小值交换到 `A[i]`，于是 `A[0:i+1]` 仍由最小 `i+1` 个元素组成并有序。
- 终止时 `i=n-1`，整个数组有序。

因此算法满足有序性与排列保持。

## R07

时间复杂度：
- 比较次数固定为：
  `sum_{i=0}^{n-2} (n-1-i) = n(n-1)/2`
- 最好/平均/最坏时间复杂度均为 `O(n^2)`。

即使输入已排序，选择排序仍需完整扫描未排序区间。

## R08

空间复杂度：
- 算法主体仅使用常数级辅助变量（`i/min_idx/temp`），为 `O(1)` 额外空间；
- 若保留轮次轨迹用于展示，轨迹存储约 `O(n^2)`（每轮拷贝一份数组状态），属于可选调试开销。

## R09

算法性质：
- 就地排序：是。
- 稳定性：否（远距离交换可能改变相等元素原始顺序）。
- 自适应：否（输入接近有序时复杂度不改善）。
- 交换次数：最多 `n-1`，通常少于冒泡排序。

## R10

边界与输入约束：
- 空数组、单元素数组：直接返回，视为已排序；
- 含重复值：支持；
- 含负数与浮点数：支持；
- 含 `NaN/Inf` 或不可比较对象：应拒绝（本实现做有限值校验）；
- 输入数据不应被就地污染：本实现先复制后排序，保证调用方原数据保留。

## R11

伪代码：

```text
selection_sort(A):
  n <- len(A)
  for i in [0 .. n-2]:
    min_idx <- i
    for j in [i+1 .. n-1]:
      if A[j] < A[min_idx]:
        min_idx <- j
    if min_idx != i:
      swap A[i], A[min_idx]
  return A
```

## R15

与常见排序算法对比：
- 对比冒泡排序：
  - 都是 `O(n^2)`；
  - 选择排序交换更少，但不稳定且不自适应。
- 对比插入排序：
  - 插入排序在“近乎有序”数据上常接近 `O(n)`；
  - 选择排序仍固定 `O(n^2)`。
- 对比快速/归并/堆排序：
  - 这些算法平均或最坏复杂度更优，适合工程场景；
  - 选择排序胜在实现最直接、分析门槛低。

## R16

适用场景：
- 教学：排序思想、循环不变量、复杂度推导；
- 小规模数据：数据量很小且实现简洁性优先；
- 交换代价高但比较代价可接受的场景（因交换次数上界较低）。

不建议用于大规模生产数据排序。

## R18

`demo.py` 源码级算法流程（8 步，非黑箱）：
1. `main` 构造固定样例与随机样例，逐个调用 `run_case`。
2. `run_case` 先执行 `validate_numeric_sequence`，拒绝空值、非数值与非有限值。
3. 将输入复制为列表 `arr`，初始化 `comparisons/swaps/trace`。
4. 外层循环按 `i=0..n-2` 推进有序前缀；每轮先令 `min_idx=i`。
5. 内层循环扫描 `j=i+1..n-1`，逐次比较 `arr[j] < arr[min_idx]`，更新 `min_idx` 并累计比较次数。
6. 内层结束后，若 `min_idx != i`，交换 `arr[i]` 与 `arr[min_idx]`，累计交换次数。
7. 记录当前轮轨迹（最小下标、最小值、是否交换、数组快照），继续下一轮。
8. 排序完成后，仅把 `np.sort` 和 `sorted` 用作结果对照；若不一致立即报错，否则输出统计和轨迹。

补充：第三方库在此实现中仅用于“校验与展示”，排序主过程完全由源码中的双循环与显式交换实现。
