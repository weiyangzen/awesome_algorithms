## R01

生成函数（Generating Function）是把一个数列编码为幂级数的方法。

给定序列 `a_0, a_1, a_2, ...`，其普通生成函数（OGF）定义为

`A(x) = \sum_{n\ge 0} a_n x^n`。

组合计数里，`a_n` 常表示“规模为 `n` 的对象数”，而“求计数”转化为“取系数 `[x^n]A(x)`”。

## R02

- 历史背景:
  - 生成函数思想可追溯到欧拉对整数分拆与无穷乘积的研究。
- 方法定位:
  - 是“离散计数问题的代数化工具”，把组合构造映射为多项式/幂级数运算。
- 现代应用:
  - 组合数学、概率、算法设计、随机过程、符号计算中都非常常见。

## R03

核心动机是“结构乘法对应计数卷积”。

- 若一个对象由独立部件拼接而成，部件生成函数相乘。
- 相乘后 `x^k` 的系数就是“总规模为 `k`”的方案数。
- 这让复杂枚举问题转化为可编程的卷积过程。

典型模型：

`G(x)=\prod_{i=1}^m (1+x+...+x^{c_i})`

其中 `c_i` 是第 `i` 类物品最多可选数量。`[x^k]G(x)` 就是总数为 `k` 的选法个数。

## R04

若目标次数截断到 `K`，并有 `m` 个因子：

- 朴素截断卷积复杂度：
  - 时间 `O(m*K^2)`（每次多项式乘法做双重循环）。
- 如果使用 FFT，可降到近似 `O(m*K log K)`，但实现与常数更复杂。

本目录 MVP 使用“手写截断卷积 + 可选 `numpy.convolve` 校验”，优先保证透明和可验证。

## R05

- 空间复杂度:
  - 只保留当前累计多项式与当前因子，额外空间 `O(K)`。
- 工程实践:
  - 每次卷积后立即截断到 `K` 次，避免无效高阶项膨胀。

## R06

示例：有 4 类物品，可选数量上界分别为 `2,3,1,4`，求总共选 `6` 个的方案数。

生成函数：

`G(x)=(1+x+x^2)(1+x+x^2+x^3)(1+x)(1+x+x^2+x^3+x^4)`。

答案是 `[x^6]G(x)`。

`demo.py` 会同时用：

- 生成函数卷积法
- 暴力枚举法

进行对拍，验证结果一致。

## R09

适用条件与边界：

- 适合“计数可按规模加法分解”的问题。
- 若需要的目标次数是 `K`，必须做截断，否则高阶项会造成不必要开销。
- 对巨大 `K`（如十万级以上），朴素卷积会变慢，应考虑 FFT/NTT 或专门优化。

## R10

正确性关键（系数乘法定理）：

设

`A(x)=\sum a_i x^i, B(x)=\sum b_j x^j`，

则

`A(x)B(x)=\sum_k (\sum_{i+j=k} a_i b_j)x^k`。

内层求和恰好枚举了“规模 `i` 的左部件 + 规模 `j` 的右部件”组成规模 `k` 的所有组合，因此系数就是方案总数。

多因子相乘时反复应用该结论即可。

## R11

数值稳定性：

- 本条目使用整数计数，不涉及浮点误差传播。
- 风险主要是“大整数变慢”，不是数值漂移。
- 若改用浮点 FFT，需要额外做四舍五入与误差防护。

## R12

- `K` 增大：朴素卷积近二次增长。

## R18

源码级算法流程（`demo.py`，拆解到可执行步骤，不把第三方库当黑箱）：

1. 先做输入校验，保证所有容量、面值、目标次数是非负整数。
2. 把每个组合组件转成“因子多项式系数数组”（例如上界 `c` 对应 `[1,1,...,1]`，长度 `c+1`）。
3. 用手写截断卷积逐因子累乘：
   - 双重循环实现 `c_k = \sum_i a_i b_{k-i}`；
   - 超过目标次数 `K` 的项直接丢弃。
4. 最终累计多项式在下标 `K` 位置的值就是 `[x^K]` 系数，也就是答案。
5. 若检测到 `numpy`，再用 `numpy.convolve` 做同样截断流程，验证与手写结果一致。
6. 对“有上界选取”案例，用 `itertools.product` 暴力枚举每类取值并计数，对拍答案。
7. 对“骰子分布”和“硬币找零”案例，用独立 DP 实现对拍，保证建模正确。
8. 统一打印三组结果并输出 `All checks passed.`，确保可被自动验证脚本直接运行。
