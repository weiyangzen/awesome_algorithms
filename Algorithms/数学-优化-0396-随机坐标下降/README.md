## R01

随机坐标下降（Randomized Coordinate Descent, RCD）是一类迭代优化方法：每轮只随机选择一个坐标（变量分量）更新，而不是一次更新整个向量。

本条目采用的目标函数为带 `L2` 正则的最小二乘（岭回归）：

`f(x) = (1/(2m)) * ||Ax - b||_2^2 + (lambda/2) * ||x||_2^2`。

对于该二次凸目标，按单坐标更新时可以写成闭式步长，因此非常适合作为可验证 MVP。

## R02

方法背景（简要）：

- 坐标下降思想可以追溯到早期分量最小化策略；
- 随机化版本在大规模机器学习中被广泛使用，因为每步计算便宜；
- 对凸、光滑目标，RCD 具有可证明的期望收敛性质，并且常用作高维优化基线方法。

## R03

本 MVP 要解决的问题：

- 输入：矩阵 `A in R^(m x p)`、向量 `b in R^m`、正则系数 `lambda`；
- 过程：随机选坐标 `j`，更新 `x_j`；
- 输出：
  - 近似解 `x`；
  - 迭代次数（坐标更新次数）与 epoch 数；
  - 最终目标值 `f(x)`；
  - 梯度无穷范数 `||grad f(x)||_inf`；
  - 与闭式解 `x*` 的目标函数间隙（objective gap）。

## R04

时间复杂度（稠密情形）：

- 预处理坐标 Lipschitz 常数：`O(mp)`；
- 单次坐标更新：
  - 计算单坐标梯度 `grad_j` 需要 `O(m)`；
  - 残差增量更新 `r <- r + step * A[:, j]` 需要 `O(m)`；
- 总计 `T` 次更新约为 `O(mp + Tm)`。

当 `p` 很大但每轮只改一个坐标时，单步成本远小于全梯度法 `O(mp)`。

## R05

空间复杂度：

- 存储 `A`：`O(mp)`；
- 存储 `x`、`b`、残差 `r`、Lipschitz 向量：`O(p + m)`；
- 若保留每个 checkpoint 目标值历史，额外 `O(K)`（`K` 为记录点数）。

算法附加空间（不计输入矩阵）主要是线性级别。

## R06

单步更新示例（概念型）：

设当前点为 `x`，残差 `r = Ax - b`，随机抽到坐标 `j`。

1. 计算该坐标梯度：
   `grad_j = (A[:,j]^T r)/m + lambda * x_j`。
2. 计算坐标二阶系数（Lipschitz）：
   `L_j = ||A[:,j]||_2^2 / m + lambda`。
3. 执行更新：
   `step = -grad_j / L_j`，`x_j <- x_j + step`。
4. 同步残差：
   `r <- r + step * A[:,j]`。

这样可避免每步重算 `Ax`，显著节约计算。

## R07

算法意义：

- 将高维优化拆成低成本局部修正，适合大规模问题；
- 随机抽样减少“固定循环坐标”可能带来的坏顺序效应；
- 对二次凸目标可做精确坐标步，工程实现简单、可复现、可诊断。

## R08

核心数学关系：

1. 目标函数：`f(x) = (1/(2m))||Ax-b||^2 + (lambda/2)||x||^2`。  
2. 全梯度：`grad f(x) = (A^T(Ax-b))/m + lambda x`。  
3. 第 `j` 个坐标梯度：`grad_j = (A[:,j]^T(Ax-b))/m + lambda x_j`。  
4. 第 `j` 坐标二阶项：`L_j = ||A[:,j]||^2/m + lambda`。  
5. 一维精确最小化步：`x_j <- x_j - grad_j / L_j`。  
6. 若 `lambda > 0`，目标为强凸，解唯一。

## R09

适用条件与局限：

适用：

- 目标函数可按坐标高效求偏导；
- 单坐标更新代价显著小于全梯度更新；
- 需要可扩展、实现简单、可在线迭代的优化器。

局限：

- 在强耦合变量场景下，单坐标推进可能慢于二阶/动量方法；
- 随机性导致单次运行迭代步数有波动；
- 非凸问题中只能保证到达某类驻点，不保证全局最优。

## R10

正确性要点（本实现对应）：

1. 目标是凸二次函数，沿任一坐标切片都是一元二次函数。  
2. 更新 `x_j <- x_j - grad_j/L_j` 是该切片的精确最小化。  
3. 每次坐标更新都不会增加目标值（数值误差可忽略）。  
4. 在 `lambda > 0` 时问题强凸，最优解唯一；随机坐标下降收敛到该唯一解。  
5. `demo.py` 用闭式解 `(A^T A / m + lambda I)^{-1}(A^T b/m)` 作为真值对照，验证 objective gap。

## R11

数值稳定性与收敛监控：

- 使用残差增量更新，避免反复构造 `Ax` 带来的重复误差和开销；
- 通过 `||grad f(x)||_inf` 作为停止指标，比仅看参数步长更直接；
- `lambda > 0` 提升 Hessian 条件，有助于稳定收敛；
- 实现中对输入有限值、形状和 Lipschitz 常数正性做了显式检查。

## R12

性能实现要点：

- 预先计算 `L_j`，避免循环内重复计算列范数；
- 每步只访问一列 `A[:,j]`，并用向量化做内积和残差更新；
- 每个 epoch 才计算一次全梯度范数，平衡诊断精度与性能；
- 保持最小依赖（仅 `numpy` + 标准库），降低运行门槛。

## R13

本目录 MVP 的可验证结果：

- 固定随机种子构造可复现实验；
- 程序自动断言：
  - 已在 `max_updates` 内收敛；
  - 目标函数明显低于初始值；
  - `||grad f(x)||_inf` 小于阈值；
  - 相对 objective gap 足够小；
  - checkpoint 目标值历史单调不增（允许极小浮点误差）。

任何断言失败会抛出异常并退出。

## R14

常见失效模式：

- `A` 与 `b` 维度不匹配或存在非有限值；
- `lambda < 0` 破坏目标定义；
- `max_updates` 过小导致未达收敛阈值；
- 特征尺度差异过大时，收敛速度可能下降（可先做标准化）。

## R16

相关算法对比：

- 全梯度下降（GD）：每步 `O(mp)`，方向全局但单步昂贵；
- 随机梯度下降（SGD）：按样本随机，适合大数据流式训练；
- 随机坐标下降（RCD）：按特征随机，适合高维特征优化；
- 二阶方法（如牛顿法）：迭代次数少但单步代价高、内存重。

在“特征很多、单列访问便宜”的条件下，RCD 是实用折中方案。

## R18

`demo.py` 的源码级算法流程（8 步，非黑盒）如下：

1. `main` 固定 `seed`，构造 `A,b`，并计算零向量初值下的初始目标 `f(x0)`。  
2. `randomized_coordinate_descent_ridge` 调用 `validate_inputs`，检查形状、有限值、`lambda`、阈值与迭代上限。  
3. 预计算每个坐标的 `L_j = ||A[:,j]||^2/m + lambda`，初始化 `x=0` 与残差 `r=-b`。  
4. 每次迭代随机采样坐标 `j`，计算 `grad_j = A[:,j]^T r / m + lambda*x_j`。  
5. 按 `step = -grad_j/L_j` 更新 `x_j`，并同步 `r <- r + step*A[:,j]`。  
6. 每 `p` 次更新（1 个 epoch）记录目标值，并计算 `||grad f(x)||_inf`；若小于 `tol` 则停止。  
7. 结束后调用 `solve_ridge_closed_form` 求闭式解 `x*`，计算 `objective_gap = f(x)-f(x*)` 与相对 gap。  
8. `run_checks` 验证收敛性、目标下降、梯度范数、gap 与历史单调性，通过后打印最终报告。

说明：第三方库仅用于基础线性代数运算（向量内积、矩阵乘法、线性方程求解）；随机坐标下降主逻辑在源码中逐步实现并可逐行追踪。
