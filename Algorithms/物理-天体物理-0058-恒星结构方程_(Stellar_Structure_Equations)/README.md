# 恒星结构方程 (Stellar Structure Equations)

- UID: `PHYS-0058`
- 学科: `物理`
- 分类: `天体物理`
- 源序号: `58`
- 目标目录: `Algorithms/物理-天体物理-0058-恒星结构方程_(Stellar_Structure_Equations)`

## R01

恒星结构方程描述了球对称恒星内部在半径方向上的平衡与能量传输。  
经典 1D 形式包含四条耦合微分方程：质量守恒、流体静力平衡、能量产生、温度梯度。  
本条目实现一个可运行的最小数值模型，用来展示“方程组如何被真正求解”，而不是只写理论公式。

## R02

目标是提供一个可验证的 MVP：

- 给定中心条件 `rho_c, T_c`，向外积分恒星结构 ODE；
- 用“表面压力阈值”事件自动停止积分，得到 `R, M, L, T_surface`；
- 输出径向剖面表，检查静力平衡残差；
- 用两种 ODE 求解器交叉验证结果一致性。

## R03

建模假设如下：

- 牛顿引力、球对称、静态（不考虑自转/磁场/广义相对论修正）；
- EOS 使用理想气体：`rho = P * mu * m_H / (k_B * T)`；
- 能量产生率使用 pp 链幂律近似：`epsilon ~ rho * X^2 * (T/1e6)^4`；
- 温度梯度采用辐射输运近似；
- 表面定义为 `P_surface = 1e-6 * P_center`。

该实现强调算法链路可追踪，不追求“高精度恒星演化代码”。

## R04

MVP 使用的方程（cgs 单位）：

1. 质量守恒

`dm/dr = 4 * pi * r^2 * rho`

2. 静力平衡

`dP/dr = -G * m * rho / r^2`

3. 能量方程

`dL/dr = 4 * pi * r^2 * rho * epsilon`

4. 辐射温度梯度

`dT/dr = -(3 * kappa * rho * L) / (16 * pi * a * c * T^3 * r^2)`

5. 闭合关系

`rho(P, T) = P * mu * m_H / (k_B * T)`

## R05

算法流程（高层）：

1. 由中心密度和中心温度计算 `P_c`，构造初值 `(m0, P_c, L0, T_c)`；
2. 写出 ODE 右端 `f(r, y)`，显式计算四个导数；
3. 构造事件函数 `P - P_surface = 0`；
4. 用主求解器 `DOP853` 积分得到主解；
5. 用次求解器 `RK45` 重复积分得到对照解；
6. 从主解构建径向剖面 DataFrame；
7. 计算静力平衡残差、单调性诊断和跨求解器误差；
8. 执行断言，确保结果满足基础物理与数值一致性。

## R06

若单次积分步数为 `S`：

- ODE 积分时间复杂度约 `O(S)`；
- 剖面后处理（梯度、误差、统计）约 `O(S)`；
- 主+次求解器总复杂度约 `O(2S)`；
- 空间复杂度约 `O(S)`（保存半径网格和状态序列）。

默认参数下 `S` 在千步量级，单次运行可在秒级完成。

## R07

正确性直觉：

- 质量方程保证质量随半径累积；
- 静力方程保证压力向外递减以平衡引力；
- 能量方程使光度随半径累积；
- 温度梯度由辐射输运决定，连接能量流与热结构。

若数值解满足“`m` 非减、`P` 非增、`dP/dr` 与理论 RHS 一致”，就说明方程联立求解链路是自洽的。

## R08

数值稳定策略：

- 从小但非零 `r0` 起步，避免 `r=0` 奇点；
- 在分母位置统一使用 `max(..., 1e-30)` 防止除零；
- 当 `P<=0` 或 `T<=0` 时 RHS 返回零导数保护；
- 用事件终止而非硬截断半径，减少尾部积分病态；
- 采用双求解器交叉检验，避免单方法偶然误差被忽略。

## R09

`demo.py` 的最小工具栈：

- `numpy`：数值数组、插值与梯度计算；
- `scipy`：`solve_ivp` 积分恒星结构 ODE；
- `pandas`：组织并展示径向剖面表；
- `scikit-learn`：`MAE/R2` 量化主次求解器一致性；
- `torch`：基于张量差分统计单调性违例数。

## R10

演示脚本输出四类结果：

- 主求解器表面参数（`R, M, L, T, rho`）；
- 采样后的径向剖面（半径、质量、压力、温度、密度、残差）；
- 主次求解器一致性指标；
- 单调性与静力平衡残差诊断，以及断言通过状态。

脚本无需交互输入，直接运行即可复现实验。

## R11

关键输出指标：

- `surface_radius_cm / R_sun`：积分终止半径；
- `surface_mass_g / M_sun`：恒星总质量；
- `surface_luminosity_erg_s / L_sun`：总光度；
- `surface_temperature_k`、`surface_density_g_cm3`：表面热力学状态；
- `hydro_rel_error` 的中位数与 `p95`：静力平衡方程残差；
- `mass_mae_msun`、`pressure_mae_fraction`、`temperature_mae_fraction`：跨求解器偏差；
- `mass_decrease_violations`、`pressure_increase_violations`：单调性违例次数。

## R12

默认参数（对应 `StellarConfig`）：

- 组分与微观参数：`mu=0.61`, `X=0.70`, `kappa=0.34 cm^2/g`, `epsilon0=1.07e-7`；
- 中心条件：`rho_c=150 g/cm^3`, `T_c=1.0e6 K`；
- 表面判据：`P_surface/P_center = 1e-6`；
- 积分区间：`r in [1e6, 2e12] cm`；
- 数值设置：`max_step=2e9 cm`, `rtol=1e-6`, `atol=1e-8`；
- 求解器：主 `DOP853`，次 `RK45`。

这是一组“求解稳定优先”的教学参数，不等价于精确的太阳标定模型。

- 质量/压力单调性必须满足基本物理方向；

- 跨求解器误差必须在阈值内，否则脚本失败。

## R14

常见失效模式：

1. 参数极端（如过高 `epsilon0`）导致温度梯度过陡，积分退化；
2. `r_max` 太小，事件无法触发，结果只到“截断半径”；
3. 用不合理 EOS 仍套用本断言，可能得到形式正确但物理错误的曲线；
4. 在 `r -> 0` 附近不做保护会产生除零/步长崩溃；
5. 将该教学模型误当作完整恒星演化代码（忽略对流、化学演化、不透明度表等）。

## R18

`demo.py` 源码级算法流程（8 步）：

1. `StellarConfig` 定义中心条件、物理常数缩放参数、积分区间与容差。  
2. `density_from_pressure_temperature` 和 `pp_chain_proxy_epsilon` 提供 EOS 与能量产生闭合关系。  
3. `stellar_structure_rhs` 逐项计算 `dm/dr, dP/dr, dL/dr, dT/dr`，把四条方程写成可积的 ODE 系统。  
4. `make_surface_event` 构造 `P-P_surface=0` 终止事件，给出“恒星表面”的算法定义。  
5. `solve_single_star` 调 `solve_ivp` 获得主/次解，并提取表面 `R, M, L, T, rho`。  
6. `build_profile_dataframe` 把轨迹转成结构化表，并用数值梯度与理论 RHS 计算静力残差。  
7. `compare_solvers` 在公共半径网格上插值比较两求解器，输出 `MAE/R2`；`torch_monotonicity_checks` 用张量差分统计单调性违例。  
8. `main` 打印摘要与剖面、执行断言并给出通过/失败结论。

第三方库没有被当作黑盒“直接给答案”：`scipy` 仅提供通用 ODE 积分器，恒星方程、闭合关系、事件定义、残差评估和验收标准都在源码中显式实现。
