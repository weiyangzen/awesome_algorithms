# 页面置换 - LRU

- UID: `CS-0168`
- 学科: `计算机`
- 分类: `操作系统`
- 源序号: `313`
- 目标目录: `Algorithms/计算机-操作系统-0313-页面置换_-_LRU`

## R01

LRU（Least Recently Used，最近最少使用）页面置换用于虚拟内存管理：当物理页框已满且发生缺页时，淘汰“最长时间未被访问”的页面，以期在具有时间局部性的访问序列下减少后续缺页。

## R02

该问题是操作系统内存管理中的经典策略选择题。与 FIFO 不同，LRU 试图利用“最近访问过的页面更可能再次访问”的经验规律，常在实际系统中作为近似策略（如 Clock）设计的理论基线。

## R03

输入通常包含两部分：
- 页框容量 `capacity`（正整数）。
- 页面访问序列 `reference_string`（整数列表）。

输出通常包含：
- 每一步是命中（hit）还是缺页（fault）。
- 缺页时被淘汰的页面。
- 访问完成后的累计命中数、缺页数、缺页率。

## R04

本实现采用“哈希表 + 双向链表”的标准组合：
- 哈希表 `page -> node`：O(1) 判断页面是否在内存。
- 双向链表维护访问新旧顺序：头部是 LRU，尾部是 MRU。

当页面被访问：
- 命中时，将对应节点移动到链表尾部。
- 缺页时，如容量已满，弹出链表头并从哈希表删除。

## R05

算法流程：
1. 初始化空页框状态。
2. 依次读取页面访问请求。
3. 若页面已在哈希表中：记一次 hit，并将节点移动到 MRU 端。
4. 否则记一次 fault；若已满，淘汰 LRU（链表头）。
5. 将新页面插入 MRU 端并写入哈希表。
6. 记录每一步状态用于可解释输出。

## R06

正确性直觉：
- 链表顺序始终表达“最近使用时间”的全序关系。
- 每次访问都会把该页更新为最新（MRU）。
- 每次淘汰都从最旧端（LRU）弹出，因此与 LRU 定义一致。

换句话说，数据结构不只存了“有哪些页”，还精确存了“谁最久没用”。

## R07

时间复杂度（`n` 为访问次数）：
- 单次访问：O(1)（哈希查找 + 链表常数次指针操作）。
- 全序列模拟：O(n)。

空间复杂度：
- O(capacity)，由页框中页面节点与哈希索引共同决定。

## R08

示例（与 `demo.py` 一致）：
- `capacity = 3`
- `reference_string = [7, 0, 1, 2, 0, 3, 0, 4, 2, 3, 0, 3, 2]`

结果：
- 命中 `4` 次
- 缺页 `9` 次
- 缺页率 `9/13 = 69.23%`

## R09

边界条件：
- `capacity <= 0`：算法无意义，应立即报错。
- 空访问序列：总访问数为 0，命中与缺页均为 0。
- 容量大于不同页面总数：前若干次缺页后基本全命中。
- 高度重复单页访问：首次缺页后几乎全命中。

## R12

输出字段说明：
- `Result`：`HIT` 表示命中；`FAULT` 表示缺页。
- `Evicted`：缺页且内存已满时显示被换出的页；否则为 `-`。
- `Frames (LRU -> MRU)`：当前页框从“最旧”到“最新”的顺序。

该顺序是理解 LRU 决策的关键可视化信息。

## R13

与常见策略对比：
- FIFO：实现更简单，但可能出现 Belady 异常，且不考虑最近访问性。
- OPT（最优置换）：理论最优但需未来信息，仅作上界参考。
- LRU：在现实中可实现且效果稳定，是常用折中方案。

## R14

实现中的常见错误：
- 命中后只计数，不更新最近性顺序，导致“伪 LRU”。
- 淘汰后忘记同步删除哈希项，产生脏引用。
- 链表删头删尾时未处理空表/单节点边界。
- 统计口径不一致（把首次装填误算成命中）。

## R18

源码级算法流（对应 `demo.py`）：
1. `main` 构造固定访问序列和页框容量，实例化 `LRUPageReplacement`。
2. `simulate` 逐个遍历页面请求，对每个页面调用 `access`。
3. `access` 首先在 `_nodes` 哈希表中判断是否命中。
4. 若命中：调用链表 `remove(node)` 把节点从原位置摘下，再 `append_mru(node)` 挂到尾部，表示“刚访问过”。
5. 若缺页且容量已满：调用 `pop_lru()` 从链表头弹出最旧页面，并在 `_nodes` 中删除该页映射。
6. 对缺页页面创建新节点，`append_mru(new_node)` 插入尾部，并写入 `_nodes[page] = new_node`。
7. 每次 `access` 返回本步快照（命中/淘汰页/当前页框顺序），`simulate` 汇总为 `steps`。
8. `simulate` 计算 `hits/faults/fault_rate`，`main` 以表格形式打印逐步结果与最终统计。
