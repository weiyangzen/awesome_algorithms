# 轨道摄动理论 (Orbital Perturbation Theory)

- UID: `PHYS-0122`
- 学科: `物理`
- 分类: `天体力学`
- 源序号: `122`
- 目标目录: `Algorithms/物理-天体力学-0122-轨道摄动理论_(Orbital_Perturbation_Theory)`

## R01

轨道摄动理论研究的是：在理想二体开普勒运动上叠加“小扰动力”后，轨道根数如何随时间缓慢漂移。  
本目录 MVP 选择最经典且工程上高频的摄动源 `J2`（地球扁率项），完成一个“数值传播 + 理论速率对照”的最小可运行流程。

核心目标：
- 用 Cowell 法直接积分状态方程，得到含 `J2` 的振荡轨道根数；
- 从数值结果拟合出 `RAAN` 与 `argp` 的长期漂移率；
- 与一阶摄动理论的解析长期平均公式比较，验证量级与符号正确。

## R02

MVP 具体计算任务：
- 输入一组初始轨道根数（`a, e, i, Ω, ω, ν`）；
- 分别传播两条轨道：
  - 仅中心引力（Kepler 基线）；
  - 中心引力 + `J2` 摄动；
- 输出：
  - 随时间演化的 `RAAN/argp`（解包角度后的趋势）；
  - 数值拟合漂移率（deg/day）；
  - 理论 `J2` 漂移率（deg/day）；
  - 二者相对误差与验收结论。

## R03

采用该设定的原因：
- `J2` 是近地轨道最主要保守摄动之一，物理意义明确；
- 解析摄动理论有可对照的闭式长期平均速率，便于“算出来还能验”；
- Cowell 直接积分实现简单透明，适合做教学级源码 MVP；
- 相比只给公式，本实现提供了从状态向量到轨道根数再到长期漂移拟合的完整链路。

## R04

核心模型公式：

1. 运动方程（惯性系）

`r¨ = a_2body + a_J2`

其中：

`a_2body = -mu * r / |r|^3`

`a_J2 = (3/2) * J2 * mu * Re^2 / |r|^5 * [ x(5z^2/r^2-1), y(5z^2/r^2-1), z(5z^2/r^2-3) ]`

2. 一阶长期平均摄动理论（`J2`）：

`n = sqrt(mu/a^3), p = a(1-e^2)`

`dΩ/dt = -(3/2) * n * J2 * (Re/p)^2 * cos(i)`

`dω/dt = (3/4) * n * J2 * (Re/p)^2 * (5cos^2(i)-1)`

3. 数值趋势提取：

对 `unwrap(Ω(t))` 与 `unwrap(ω(t))` 做一次线性拟合，斜率即数值长期漂移率。

## R05

算法流程（高层）：
1. 读取默认初始轨道参数（近地椭圆轨道）。
2. 将轨道根数 `COE -> (r0, v0)` 转为惯性系初值。
3. 用 `solve_ivp(DOP853)` 传播 Kepler 基线（无 `J2`）。
4. 用同样积分器传播含 `J2` 的摄动轨道。
5. 对每个时刻 `(r,v)` 反算振荡轨道根数 `(a,e,i,Ω,ω,ν)`。
6. 对 `Ω`、`ω` 做解包并线性拟合，得到数值漂移率。
7. 按理论公式计算解析漂移率并比较误差。
8. 输出表格、指标并执行阈值验收。

## R06

正确性与一致性说明：
- 动力学方程与 `J2` 加速度按标准矢量形式显式实现，非黑盒；
- 解析速率仅依赖 `a,e,i` 的一阶公式，物理上应给出 `Ω`、`ω` 的主导长期漂移；
- Kepler 基线应近似零漂移，作为数值积分误差基线；
- “数值摄动率 vs 解析摄动率”的一致性用于验证实现是否符合轨道摄动理论预期。

## R07

复杂度分析（设采样点数为 `N`，积分器内部步数为 `K`）：
- 数值积分：`O(K)`，每步常数维度（6 维状态）；
- 轨道根数反算：`O(N)`；
- 角度解包与线性拟合：`O(N)`；
- 总体时间复杂度约 `O(K + N)`，空间复杂度 `O(N)`。

在本脚本默认参数下，属于秒级可运行的轻量 MVP。

## R08

边界与异常处理：
- `a > 0`、`0 < e < 1`、传播时长 > 0、采样点 >= 3；
- 对接近奇异轨道（`e≈0` 或 `i≈0`）的 `argp/RAAN`，本 MVP 不作为主测试场景；
- 若 `solve_ivp` 不收敛，立即抛出异常；
- 若 Kepler 基线漂移过大或数值-理论误差超阈值，抛 `RuntimeError`，便于自动化验证。

## R09

MVP 取舍：
- 只实现 `J2` 一种摄动，不扩展到气动阻力、三体、SRP 等；
- 使用 Cowell 直接积分，不实现 Encke/高阶平均摄动解析器；
- 重点演示长期漂移率验证，不做轨道机动控制与最优设计；
- 选取单个代表轨道参数，优先保证“可运行 + 可解释 + 可验算”。

## R10

`demo.py` 主要函数职责：
- `validate_config`：输入参数合法性检查；
- `coe_to_rv / rv_to_coe`：轨道根数与状态向量双向转换；
- `two_body_acceleration / j2_acceleration`：动力学加速度模型；
- `dynamics`：统一状态方程右端；
- `propagate_orbit`：调用 `solve_ivp` 完成轨道传播并回算根数序列；
- `fit_secular_rate`：从角度时间序列拟合长期漂移率；
- `analytical_j2_secular_rates`：理论摄动率计算；
- `build_summary_table`：组织关键采样输出；
- `main`：执行双场景传播、指标比对与验收。

## R11

运行方式：

脚本无交互输入，运行后会打印代表性时刻表、速率对比与检查结果。

## R12

输出字段说明：
- `time_hr`：传播时间（小时）；
- `RAAN_kepler_deg`：无摄动基线的升交点赤经（解包后）；
- `RAAN_j2_deg`：含 `J2` 摄动的升交点赤经（解包后）；
- `argp_kepler_deg`：无摄动基线的近地点幅角；
- `argp_j2_deg`：含 `J2` 摄动的近地点幅角；
- `raan_rate_*_deg_day`、`argp_rate_*_deg_day`：对应长期漂移率（度/天）；
- `raan_rel_err`、`argp_rel_err`：数值率相对解析率误差；
- `mean_a_j2_km/std_a_j2_km`、`mean_e_j2/std_e_j2`：振荡根数统计量。

## R13

最小验收项（脚本已自动检查）：
- Kepler 基线漂移应接近 0（阈值 `0.03 deg/day`）；
- 数值 `J2` 漂移率与解析率相对误差：
  - `RAAN <= 8%`
  - `argp <= 12%`
- 若任一条件失败，脚本抛异常；全部通过则打印 `All checks passed.`。

## R14

关键参数与调参建议：
- `duration_hours`：越长越能抑制短周期噪声，提高长期拟合稳定性；
- `samples`：增大可提升拟合抗噪性，但会增大后处理开销；
- `rtol/atol/max_step`：积分精度与速度折中；
- 轨道选择建议避免 `e` 或 `i` 过小，以免 `argp/RAAN` 数值奇异；
- 若关注太阳同步设计，可把 `i` 调到特定值观察 `RAAN` 漂移目标匹配。

## R15

与替代方案对比：
- 纯解析平均摄动：速度快，但看不到短周期振荡细节；
- 纯数值传播：真实度高，但缺乏“可解释理论基线”；
- 本 MVP（数值 + 解析对照）：在保持实现简洁的同时兼顾可解释性与可验证性；
- 与直接调用商业轨道库相比，本实现更适合教学和源码审计。

## R16

典型应用场景：
- 近地卫星长期轨道设计中的节点回归估计；
- 太阳同步轨道初值筛选（依赖 `RAAN` 漂移率）；
- 教学中演示“振荡根数 vs 平均摄动率”关系；
- 作为更复杂摄动模型（拖曳、三体、SRP）的基线模块。

可扩展方向：

## R18

`demo.py` 源码级算法流程（8 步）：

1. `main` 构造 `OrbitConfig` 并调用 `validate_config`，锁定椭圆轨道参数与传播窗口。  
2. `propagate_orbit(with_j2=False)` 先执行 Kepler 基线：`coe_to_rv` 生成初值，`solve_ivp(DOP853)` 积分两体方程。  
3. `propagate_orbit(with_j2=True)` 再执行含摄动传播：在 `dynamics` 中把 `j2_acceleration` 叠加到中心引力。  
4. 对两次传播的每个采样状态 `rv_to_coe` 反算振荡轨道根数序列，得到 `Ω(t), ω(t)` 等。  
5. `fit_secular_rate` 对 `Ω(t), ω(t)` 先 `unwrap` 再做一次线性拟合，提取数值长期漂移率（rad/s）。  
6. `analytical_j2_secular_rates` 按 `n,p` 与 `J2` 一阶公式计算解析 `dΩ/dt`、`dω/dt`。  
7. 汇总 `deg/day` 指标、相对误差、`a/e` 振荡统计，并用 `build_summary_table` 生成代表性时刻表。  
8. 在 `main` 做阈值验收：Kepler 基线近零漂移、J2 数值率接近解析率；通过后输出 `All checks passed.`。
