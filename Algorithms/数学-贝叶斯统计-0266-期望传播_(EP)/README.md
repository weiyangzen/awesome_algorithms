## R01

期望传播（Expectation Propagation, EP）是一类近似贝叶斯推断算法。核心思想是：
把难处理的后验分布分解为若干因子后，用一个“易处理分布族”（通常为高斯族）逐个近似每个因子，并通过局部矩匹配（moment matching）反复更新，最终得到全局近似后验。

## R02

设后验为

`p(z|D) ∝ p0(z) * Π_i f_i(z)`，

其中 `p0(z)` 是先验，`f_i(z)` 是数据因子（通常非共轭）。
EP 构造近似

`q(z) ∝ p0(z) * Π_i t_i(z)`，

其中 `t_i(z)` 与 `f_i(z)` 对应，且 `t_i` 来自可计算分布族（本任务为一维高斯站点，作用在每个潜变量 `f_i` 上）。
目标不是最小化单个解析目标函数，而是通过局部匹配让 `q` 的边缘矩贴近真实 tilted 分布。

## R03

EP 的一次局部更新包含三件事：

1. Cavity：把当前第 `i` 个近似站点除掉，得到 `q\i`。
2. Tilted：用真实因子替换回来，构造 `p_tilt ∝ q\i * f_i`。
3. Projection：把 `p_tilt` 投影回选定族（匹配均值/方差），再反推新的站点参数。

反复扫过所有因子直到站点参数变化足够小，即认为收敛。

## R04

本 MVP 使用的是“GP 二分类 + probit 似然”的 EP 形式。

输入：

- 训练样本 `X` 与二分类标签 `y ∈ {0,1}`
- 核函数超参数：`lengthscale`、`kernel_variance`
- EP 超参数：`damping`、`max_sweeps`、`tol`

输出：

- 近似后验 `q(f)=N(m, Σ)`
- 站点参数 `tau, nu`
- 测试集预测概率 `p(y=1|x)` 与指标（Accuracy / NLL / Brier）

## R05

为什么用该设置做 EP 演示：

- Probit 似然 `Φ(y_i f_i)` 非共轭，必须近似，能体现 EP 的必要性。
- 高斯过程先验给出相关结构，EP 站点更新可写成明确可执行公式。
- 对中小样本，代码简洁且结果直观，适合教学与验证。

## R06

MVP 主流程：

1. 生成 two-moons 风格二分类数据并做分层切分。
2. 构造 RBF 核矩阵 `K` 并初始化 `tau=0, nu=0`。
3. 迭代 sweep：逐样本做 cavity/tilted/moment matching，更新站点。
4. 每个 sweep 后重建全局近似后验：`Σ=(K^{-1}+diag(tau))^{-1}`, `m=Σ*nu`。
5. 收敛后对测试集计算预测概率与分类指标。
6. 输出收敛轨迹与最不确定样本。

## R07

复杂度（`n` 为训练样本数）：

- 构造和求逆核相关矩阵主耗时约 `O(n^3)`。
- 每次 sweep 的逐点局部更新是 `O(n)`；但 sweep 结束需要一次矩阵逆（`O(n^3)`）。
- 总体约 `O(S * n^3)`（`S` 为 sweep 数）。

空间复杂度主要由 `K, Σ, K^{-1}` 决定，为 `O(n^2)`。

## R08

优点：

- 比简单拉普拉斯近似更强调局部因子匹配，常有更好的边缘概率质量。
- 可直接给出概率输出，适合不确定性评估。
- 对很多模型有可解释的“站点”视角。

局限：

- 不是全局凸优化，可能受初始化、阻尼、更新顺序影响。
- 在病态数据上可能出现数值不稳定，需要抖动项与参数裁剪。
- 中大样本下 `O(n^3)` 成本显著。

## R09

适用场景：

- 需要近似后验但 MCMC 太慢。
- 希望比点估计更好地表达预测概率和不确定性。
- 样本规模中等、允许离线训练。

不太适合：

- 超大规模在线系统且必须严格实时。
- 极端高维且核矩阵不可承受的场景（需稀疏近似或诱导点方案）。

## R10

本实现的关键建模选择：

- 先验：`f ~ N(0, K)`，`K` 来自 RBF 核。
- 似然：`p(y_i|f_i)=Φ(y_i f_i)`（`y_i` 转为 `{-1,+1}`）。
- 站点：`t_i(f_i) ∝ exp(-0.5*tau_i*f_i^2 + nu_i*f_i)`。
- 收敛：最大站点改变量 `< tol` 或 sweep 达上限。
- 稳定性：`cdf` 截断、方差下界、特征值截断+jitter 的 PSD 逆。

## R11

与其他近似推断对比：

- Laplace：围绕 MAP 做二阶近似，实现简单，但尾部和局部形状可能偏差较大。
- 变分推断（VI）：常优化 ELBO，框架统一，但局部因子贴合方式与 EP 不同。
- EP：更像“逐因子修正 + 局部投影”，在分类概率校准上常表现良好。

## R12

工程注意事项：

- `gaussian_cdf(z)` 必须防止接近 0，否则 `pdf/cdf` 比值会爆炸。
- `sigma2_hat` 需要正下界，避免反推站点精度时出现负值。
- `inverse_psd` 使用对称化 + 特征值下界裁剪，避免病态矩阵导致逆不稳定。
- 阻尼 `damping` 太小收敛慢，太大可能震荡；MVP 取 `0.65`。

## R15

结果解读建议：

- `EP sweeps executed` 与 `Final max site delta` 用于判断是否稳定收敛。
- `tau` 的 `min/mean/max` 反映各数据点对后验精度贡献。
- `Accuracy` 看分类正确率，`NLL` 与 `Brier` 更关注概率质量。
- “Most uncertain samples” 可用于主动学习或人工复核入口。

## R16

常见问题与排查：

- 若出现 `LinAlgError`：提高 `jitter`，或减小 `kernel_variance`。
- 若收敛慢或震荡：降低 `damping`、增大 `max_sweeps`。
- 若概率过于极端：检查 `lengthscale` 是否过小导致过拟合。

## R17

可扩展方向：

- 引入超参数学习（基于近似边际似然或验证集调参）。
- 扩展为稀疏 GP + EP（诱导点）降低 `O(n^3)` 成本。
- 对比 EP 与 Laplace/VI 在概率校准（ECE、Brier）上的差异。
- 尝试并行或随机顺序站点更新策略，提升速度与稳定性。

## R18

本仓 `demo.py` 的 EP 源码级执行流可拆成 8 步（非黑箱）：

1. 初始化模型与数据：`make_two_moons_like`、`stratified_train_test_split` 生成并切分二分类数据。
2. 构造先验协方差：`rbf_kernel(X, X, ...)` 得到 `K`，再由 `inverse_psd` 计算 `K^{-1}`。
3. 初始化站点：`fit()` 中把 `tau, nu` 设为 0，对应“无信息站点”。
4. 计算 cavity：对每个样本 `i`，由当前 `Σ, m, tau_i, nu_i` 推出 `tau_cav, mu_cav, sigma2_cav`。
5. 计算 tilted 矩：结合真实 probit 因子 `Φ(y_i f_i)`，通过 `z`、`pdf/cdf` 比值得到 `mu_hat, sigma2_hat`。
6. 回投影并更新站点：从 tilted 矩反推 `tau_new, nu_new`，并做阻尼更新（`damping`）。
7. 全局重组后验：每个 sweep 结束重算 `Σ=(K^{-1}+diag(tau))^{-1}` 与 `m=Σ*nu`，记录 `max_site_delta` 判断收敛。
8. 预测与评估：`predict_proba` 用 `q(f)` 推导测试点潜变量均值方差，再经 probit 积分得到 `p(y=1|x)`，最后由 `evaluate_binary` 输出 Accuracy/NLL/Brier。
