# Jacobi特征值算法

- UID: `MATH-0061`

## R01

Jacobi 特征值算法用于求解实对称矩阵的全部特征值（以及可选的特征向量）。

- 核心思想: 反复用 Givens/Jacobi 平面旋转消去非对角元，让矩阵逐步对角化。
- 输入: 实对称矩阵 `A in R^{n*n}`。
- 输出: 对角线近似特征值 `lambda_i`，以及累计旋转得到的正交矩阵 `V`（列为特征向量近似）。
- 适用重点: “全谱 + 可解释迭代过程”的中小规模对称问题。

## R02

历史背景（简要）:

- Carl Gustav Jacobi 在 19 世纪提出该类旋转消元思想。
- 这是最早可系统求全特征分解的数值算法之一，概念直观、证明清晰。
- 现代高性能库常用 QR/分治等更快方案，但 Jacobi 仍是教学和高精度正交特征向量场景的重要基线。

## R03

要解决的问题:

- 给定对称矩阵 `A`，希望得到全部特征值，而不是只取最大/最小一个。
- 与幂法等单特征值方法相比，Jacobi 直接面向“全谱”。
- 与黑盒库调用相比，Jacobi 便于追踪每步如何降低非对角耦合项，适合审计和教学。

## R04

复杂度概览:

- 单次旋转更新需要修改两列/两行，代价约 `O(n)`。
- 一个 sweep 约含 `n(n-1)/2` 次旋转，代价约 `O(n^3)`。
- 若需要 `s` 个 sweep 达到阈值，总体约 `O(s*n^3)`。
- 对中小规模、强调可解释性的任务通常可接受；大规模稠密问题常用更快的 QR 家族实现。

## R05

空间复杂度:

- 存储当前矩阵 `A_k`: `O(n^2)`。
- 若需要特征向量，再存累计旋转矩阵 `V`: `O(n^2)`。
- 额外标量与历史记录开销较小（`O(k)`，`k` 为迭代步数）。

## R06

示例（2x2 直观）:

设

`A = [[4, 1], [1, 3]]`

选择 `(p,q)=(0,1)` 做一次 Jacobi 旋转，可把 `a_01` 消成 0，得到对角矩阵（在 2x2 情形一次即可完全对角化），对角线就是特征值。

这说明 Jacobi 的本质是“用正交相似变换逐步去耦合”。

## R07

算法价值:

- 数学上: 每次更新都是正交相似变换，特征值不变。
- 数值上: 对称结构保持良好，特征向量正交性通常较好。
- 工程上: 逻辑透明，便于调试、教学和做可信验证。
- 实务上: 作为 QR/三对角化等方法的对照基线很有价值。

## R08

关键数学机制:

- 设旋转矩阵 `J(p,q,theta)`，更新
  - `A <- J^T A J`
  - `V <- V J`
- 角度 `theta` 选取使得更新后 `a_pq'=0`。
- 稳定写法常用
  - `tau = (a_qq - a_pp)/(2*a_pq)`
  - `t = sign(tau)/(abs(tau)+sqrt(1+tau^2))`
  - `c = 1/sqrt(1+t^2), s=t*c`
- 迭代重复后，`A` 的严格下三角范数趋于 0，对角线收敛到特征值。

## R09

适用条件与边界:

- 主要适用于实对称（或复 Hermitian）矩阵。
- 若输入非对称，经典 Jacobi 特征值版本不保证行为，建议直接拒绝或改用通用算法。
- 收敛阈值 `tol` 越小，结果更精细但迭代更多。
- 迭代预算不足时应返回“未收敛”而非静默给出结果。

## R10

正确性要点:

1. 每次更新是 `A_{k+1}=J_k^T A_k J_k`，与 `A_k` 相似，特征值集合不变。
2. 旋转角专门选择以消去当前主元对 `(p,q)` 的非对角项。
3. 若迭代使非对角部分收敛到 0，则 `A_k` 逼近对角阵。
4. 对角元素因此逼近原矩阵特征值；累计矩阵 `V` 列向量逼近对应特征向量。

## R11

数值稳定性:

- Jacobi 仅使用正交变换，数值稳定性通常较好。
- 通过 `tau/t` 形式计算旋转参数可避免直接 `tan(2theta)` 带来的不稳定。
- 代码中每次旋转后对称位置同步写回，减少浮点漂移。
- 可通过 `||A - V*diag(lambda)*V^T||` 与 `V^T V - I` 监控质量。

## R12

真实成本（超越大 O）:

- 虽然理论常写为 `O(s*n^3)`，但常数项和内存访问模式影响明显。
- Python 层逐元素循环会慢，MVP 只保留必要循环并使用 NumPy 数组运算。
- 主元选择“最大非对角元”需要扫描上三角，单步有额外代价，但能减少无效旋转。
- 对大规模问题，工业实现会改为块 Jacobi 或改选 QR/分治路线。

## R13

误差与保证:

- 在精确算术下，Jacobi 对称特征值问题可收敛到对角化结果。
- 浮点实现中，用以下量评估结果质量:
  - 特征值误差: `max|lambda_hat - lambda_ref|`（与 `eigvalsh` 对照）
  - 重建误差: `||A - V*diag(lambda)*V^T||_F`
  - 正交误差: `||V^T V - I||_F`
  - 非对角残量: `||tril(A_k,-1)||_F`
- 本目录 demo 同时检查上述指标并设置断言门限。

## R14

失效模式与鲁棒性:

- 非方阵或含 `NaN/Inf` 输入应直接报错。
- 非对称输入应拒绝（本 MVP 明确限制对称矩阵）。
- `tol<=0` 或迭代预算无效应报错。
- 未在预算内收敛时返回 `converged=False`，让调用方决定是否加大预算或改算法。

## R15

工程实现建议:

- 明确区分“算法计算”与“参考验算”（后者只用于测试）。
- 固定随机种子构造可复现样例，保证 CI 可稳定通过。
- 输出迭代历史前几项/后几项，方便观察收敛曲线。
- 对异常路径（如非对称）做显式测试，避免未来改动破坏输入约束。

## R16

前驱与后继关系:

- 前驱基础:
  - 对称矩阵性质
  - 正交矩阵与相似变换
  - Givens 旋转
- 后继/相关算法:
  - QR 特征值算法（更常用于大规模高性能实现）
  - 三对角化 + QR
  - 分治、MRRR 等对称特征值高性能方法
- Jacobi 的定位:
  - 作为全谱分解“可解释基线”非常合适。

## R17

`demo.py` MVP 设计:

- 依赖: `numpy` + 标准库 `dataclasses/typing`。
- 核心函数:
  - `jacobi_eigen_symmetric(...)`: 主迭代，返回特征值/特征向量与收敛信息。
  - `largest_offdiag_index(...)`: 选择当前绝对值最大的非对角元作为旋转主元。
  - `apply_jacobi_rotation_inplace(...)`: 原地执行一次 `(p,q)` 旋转并同步更新 `A` 与 `V`。
- 演示内容:
  - 对固定 6x6 对称矩阵求解。
  - 用 `numpy.linalg.eigvalsh` 仅作结果验算（不替代主算法）。
  - 检查特征值误差、重建误差、正交误差、收敛标志。
  - 演示非对称输入的预期报错路径。
- 运行方式:
  - `python3 demo.py`

## R18

源码级流程拆解（对应 `demo.py`，8 步）:

1. `build_demo_matrix()` 生成可复现的 `6x6` 实对称矩阵 `A`。  
2. `jacobi_eigen_symmetric()` 做输入校验：方阵、有限值、对称性、`tol/max_sweeps` 合法。  
3. 初始化 `ak=A.copy()`、`eigvecs=I`、历史数组与迭代计数。  
4. 每次迭代用 `largest_offdiag_index(ak)` 扫描上三角，找到绝对值最大非对角元 `(p,q)`。  
5. 若严格下三角 Frobenius 范数或最大非对角元已小于阈值，则判定收敛并退出。  
6. 否则在 `apply_jacobi_rotation_inplace(ak, eigvecs, p, q)` 中计算 `tau/t/c/s`，执行 `A <- J^T A J` 并更新 `V <- VJ`。  
7. 迭代结束后读取 `diag(ak)` 作为特征值，并按升序同步重排特征向量列。  
8. `main()` 调用 `run_checks()`：与 `eigvalsh` 比较误差、验证 `V` 正交性与重建误差，并打印收敛摘要。  

说明: `numpy.linalg.eigvalsh` 只用于验算，Jacobi 主迭代（主元选择、旋转、终止判据）全部由源码显式实现。
