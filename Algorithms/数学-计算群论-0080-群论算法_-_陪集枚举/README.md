## R01

陪集枚举（Coset Enumeration）是计算群论中的经典算法，最常见实现是 Todd-Coxeter 方法。  
它解决的问题是：给定群表示
`G = < X | R >` 和子群 `H <= G`（由若干词生成），枚举 `H` 在 `G` 中的右陪集，并给出生成元在陪集集合上的作用表。

本条目给出一个可运行 MVP：
- 使用显式“陪集表 + 关系扫描 + 重合并（coincidence）”流程；
- 不依赖黑箱群论库；
- 用小规模可验证案例（`S3`、`D4`）做断言。

## R02

问题定义（离散可计算版本）：

- 输入：
  - 生成元集合 `X = {x1, ..., xk}`；
  - 关系词集合 `R`（每个关系词代表 `= e`）；
  - 子群生成词集合 `W_H`（每个词代表 `h in H`，对应 `H` 的约束）；
  - 最大允许陪集数 `max_cosets`（防止无限群/异常配置导致失控）。
- 输出：
  - 完整或近完整的陪集表 `T`；
  - `|G:H|`（已枚举陪集数量）；
  - 生成元在陪集集上的置换作用（右作用）。

## R03

核心思路（Todd-Coxeter）：

1. 建立陪集表：行表示陪集，列表示生成元及其逆元。  
2. 先放入陪集 `1 = H`，并施加子群词约束（如 `1 * w = 1`）。  
3. 对每个已知陪集与每条关系词执行“前后扫描”，推导新表项。  
4. 若扫描遇到冲突，触发陪集重合并（coincidence / union）。  
5. 若仍有未定义表项，则引入新陪集继续填表。  
6. 当无未定义项且关系闭合时，得到稳定陪集表与指数。

## R04

时间复杂度（教学实现的保守估计）：

- 设最终陪集数为 `N`，符号列数（生成元+逆元）为 `S=2k`，关系词总长度为 `L`。
- 每轮闭合扫描大约访问 `O(N * L)` 词位置；
- 重合并时会做行合并与冲突传播，最坏可达较高多项式成本；
- 实际复杂度高度依赖关系结构与枚举策略（HLT/Felsch 等）。

因此该 MVP 只承诺中小规模实例的可读与可验证，不追求工业级性能上界。

## R05

空间复杂度：

- 陪集表主存储约 `O(N * S)`；
- 并查集父指针与秩数组约 `O(N)`；
- 关系词与扫描临时变量约 `O(L)`。

总体可记为 `O(N*S + L)`，主导项通常是陪集表。

## R06

微型示例：`S3 = < a,b | a^2, b^3, (ab)^2 >`，取子群 `H=<a>`。

- 理论上 `|S3|=6`，`|H|=2`，指数应为 `|G:H|=3`；
- 陪集枚举会得到 3 行稳定陪集；
- 由表中 `a,b` 的作用可恢复一个 3 点置换表示（同构于 `S3` 对 `H`-陪集的右作用）。

`demo.py` 中会对该案例断言 `index == 3`。

## R07

算法价值：

- 面向“有限表示群（fp-group）”时，常常无法直接枚举全体元素，但可以先从陪集结构入手；
- 可用于判断子群指数、构造置换表示、辅助 Reidemeister-Schreier 重写；
- 是计算群论系统中处理表示群问题的基础设施算法之一。

## R08

理论锚点：

1. 群作用观点：`G` 右作用在 `G/H` 上，生成元诱导为置换。  
2. 关系保持性：若 `r in R`，则任一陪集 `C` 满足 `C * r = C`。  
3. 子群约束：若 `h in H`，基陪集 `H` 满足 `H * h = H`。  
4. Todd-Coxeter 通过“局部一致性 + 冲突合并”逼近全局一致陪集图。  
5. 对有限指数情形，表可在有限步内完成。

## R09

适用条件与局限：

- 适用：有限表示群、目标子群指数有限且规模适中；
- 局限：
  - 若指数很大或无限，可能爆表或难以收敛；
  - 枚举策略对性能影响很大，本 MVP 用的是朴素闭合策略；
  - 未实现高级压缩、优先级启发式与 backtracking 优化。

## R10

正确性框架（工程可验证版）：

1. `define(c,s,d)` 同时写入逆向一致项：`d * s^{-1} = c`。  
2. `scan_and_fill(start, word, end)` 用前后扫描推导缺失边或触发并合。  
3. `merge(a,b)` 通过并查集合并重合陪集，并传播两行约束。  
4. `close_under_constraints()` 对所有活跃陪集反复施加关系词与子群词直到不再变化。  
5. 终态下检查每个活跃行是否列定义完整，从而得到可用陪集表。

## R11

终止性说明：

- 对有限指数问题，若 `max_cosets` 足够大，枚举通常会停在完整表；
- 实现里设置 `max_cosets` 硬上限，超过即抛出异常而非无限运行；
- 每次新增陪集或合并都会推进状态，且状态空间在上限下有限。

## R12

性能观察：

- 主要成本来自两部分：
  - 关系词反复扫描；
  - 并合时的冲突传播。
- 关系词越长、活跃陪集越多，闭合迭代越慢；
- 对教学样例（几十陪集内）运行非常快，且更利于审计每一步数据流。

## R14

鲁棒性与失败模式：

- 词中出现未知符号会抛错；
- 若问题规模超出上限会抛 `RuntimeError`；
- 若最终仍存在未定义单元，会抛“table incomplete”异常，避免静默返回错误结果；
- 该实现偏向可读性，不适合大规模生产计算。

## R15

实现结构（`demo.py`）：

- `ToddCoxeter`：
  - `_new_coset`：创建新行；

  - `_scan_and_fill`：关系扫描与推导；
  - `close_under_constraints`：全局闭合；
  - `enumerate`：主枚举循环；
  - `standardized_table`：压缩输出；
  - `generator_action_permutations`：提取置换作用。
- `run_case/main`：测试案例与断言输出。

## R16

相关算法链路：

- 前后环节：
  - 输入是有限表示群（presentation）；
  - 输出可作为置换群输入，继续做 Schreier-Sims、轨道稳定子计算等。

## R18

源码级算法流程拆解（`ToddCoxeter.enumerate` 主线，8 步）：

1. 初始化编码器与空陪集表，创建基陪集 `1`，并查集初值 `parent[i]=i`。  
2. 调用 `close_under_constraints`：对所有活跃陪集施加全部关系词 `R`，并对 `rep(1)` 施加子群词 `W_H`。  
3. 在每个 `scan_and_fill` 中先前向扫描再后向扫描，定位缺口或冲突。  
4. 若前后扫描在中间相遇，调用 `_define` 直接补边；若冲突则 `_merge` 两个陪集。  
5. `_merge` 合并两行并传播已有列信息，必要时递归触发二次合并。  
6. 若闭合后仍有未定义格，`_find_hole` 选取一个空格并创建新陪集补入，再次闭合。  
7. 重复“补洞 -> 闭合”直到无空格，得到完整（或足够完整）陪集表。  
8. `standardized_table` 将活跃代表重编号为 `1..N`，`generator_action_permutations` 输出生成元在 `N` 个陪集上的置换作用。  

说明：本实现没有把第三方库当黑箱，核心数据流（扫描、定义、并合、闭合）都在源码中显式展开。
