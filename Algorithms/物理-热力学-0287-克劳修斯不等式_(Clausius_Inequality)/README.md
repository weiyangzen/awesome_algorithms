## R01

克劳修斯不等式描述循环过程中的热交换约束：

`∮ (δQ / T_b) <= 0`

其中 `δQ` 是系统在边界处与外界交换的微元热量（本项目约定：流入系统为正），`T_b` 是该处边界温度（或热库温度）。

- 等号成立：可逆循环；
- 严格小于：不可逆循环；
- 大于 0：违反第二定律，不可实现。

本条目把该关系离散化为 `sum_i (Q_i/T_i)`，并给出一个最小可运行验证器。

## R02

该不等式是热力学第二定律的工程判据之一，常用于：

- 判断一个循环热机/制冷机方案是否物理可行；
- 将“熵产生”写成可计算量：`S_gen = -∮δQ/T_b >= 0`；
- 给效率、最小耗功等性能上界提供合法性检查。

MVP 目标不是做高精度流体仿真，而是给出可审计的“输入热交换步骤 -> 输出可行性结论”的算法骨架。

## R03

`demo.py` 的输入输出（全为脚本内置参数，无交互）：

- 输入：
1. 多个循环样例，每个循环由若干 `HeatStep(segment, q, t_boundary)` 组成；
2. 判定容差 `tol`；
3. 统一符号约定：`Q>0` 入系统，`Q<0` 出系统。
- 输出：
1. `clausius_integral = Σ(Q_i/T_i)`；
2. `entropy_generation = -clausius_integral`；
3. `q_in`、`q_out`、`work_out` 与效率估计；
4. `classification`（`reversible_limit` / `irreversible_allowed` / `violates_clausius`）；
5. 断言通过后打印 `All checks passed.`。

## R04

MVP 使用的核心关系：

1. 离散近似：
`I = ∮(δQ/T_b) ≈ Σ_i (Q_i / T_i)`。

2. 克劳修斯不等式：
`I <= 0`。

3. 熵产生：
`S_gen = -I >= 0`。

4. 循环第一定律闭合（本脚本用于报告净输出功）：
`W_out = Σ_i Q_i`。

5. 热效率（仅在 `Q_in>0` 时定义）：
`η = W_out / Q_in`。

注意：该 MVP 聚焦“不等式检查”，不是状态方程求解器。

## R05

设一个循环含 `N` 个热交换步骤，样例数量为 `M`：

- 每个循环的时间复杂度：`O(N)`；
- 全部样例的时间复杂度：`O(MN)`；
- 空间复杂度：`O(N)`（构造向量 `q`、`t`）；

## R06

`demo.py` 的三个最小闭环：

- **闭环 A：积分计算**
  `clausius_integral` 将循环步骤映射为 `Σ(Q/T)`。
- **闭环 B：物理解释量**
  `entropy_generation`、`cycle_work`、`efficiency` 由积分与热量直接导出。
- **闭环 C：可行性判定**

## R10

正确性直觉：

1. 若循环可逆，熵交换可完全补偿，`∮δQ/T = 0`。  
2. 若存在摩擦、有限温差传热等不可逆性，系统与环境总熵增，等价于 `∮δQ/T < 0`。  
3. 若某方案出现 `∮δQ/T > 0`，意味着总熵减少，直接违背第二定律。  
4. 因此把循环离散成有限步后，`Σ(Q_i/T_i)` 的符号即可作为可实现性的必要判据。

## R11

数值与实现稳定性策略：

- 对每一步检查 `t_boundary > 0`，避免除零和非物理输入；
- 使用 `float64` 统一数值精度；
- 分类时引入容差 `tol`，避免把机器舍入误差误判为物理违反；
- 通过三类构造样例（等号、严格负、正值）做回归断言。

## R12

关键参数：

- `q`：每步热量，符号决定热流方向；
- `t_boundary`：该步边界温度，必须为正；
- `tol`：分类阈值，决定“等号附近”的判定敏感度。

调参建议：

- 若输入来自实验数据，`tol` 可取测量不确定度同量级；
- 若输入来自精确模型，`tol` 可取 `1e-10 ~ 1e-12`；
- 对超大/超小热量量纲，先做量纲归一化再判定。

## R14

常见失效模式：

1. 温度单位混用（摄氏度误当开尔文）导致判定错误；
2. 热量符号约定混乱（把流出热误写为正）；
3. 将系统温度误作边界温度，破坏不等式适用前提；
4. 大量步骤中四舍五入过强，导致等号附近误分类；
5. 仅看效率不看 `Σ(Q/T)`，可能把不可能循环当成“高性能方案”。

## R18

`demo.py` 源码级算法流程（8 步）：

1. `HeatStep` 记录单个热交换段：段名、热量 `q`、边界温度 `t_boundary`。  
2. `ClausiusCycle` 聚合多个步骤，并在 `validate` 中检查步骤非空且每个温度严格大于 0。  
3. `clausius_integral` 读取全部步骤，构造 `q` 与 `t` 向量并计算 `I = Σ(q/t)`，对应离散 `∮δQ/T`。  
4. `entropy_generation` 由 `S_gen = -I` 得到循环熵产生；`cycle_work` 用 `W_out = ΣQ` 给净输出功。  
5. `classify_cycle` 依据 `I` 与容差 `tol` 分为三类：可逆极限、可行不可逆、违反不等式。  
6. `evaluate_cycle` 汇总每个循环的 `q_in/q_out/work_out/efficiency` 与分类，返回结构化字典。  
7. `build_demo_cycles` 构造三组示例：可逆卡诺型（`I=0`）、不可逆可行（`I<0`）、刻意违规（`I>0`）。  
8. `main` 批量评估并打印 `DataFrame`，随后对三类样例分别断言，确保判定逻辑稳定。  

说明：本实现未调用任何热力学黑盒库，核心流程（积分、熵产生、分类）均在源码中显式展开。
