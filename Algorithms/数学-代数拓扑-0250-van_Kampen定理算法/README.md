# van Kampen定理算法

- UID: `MATH-0250`
- 学科: `数学`
- 分类: `代数拓扑`
- 源序号: `250`

## R01

van Kampen 定理把空间并集的基本群计算，转成群论里的“推前（pushout）/并合自由积（amalgamated product）”问题：
\[
\pi_1(U\cup V) \cong \pi_1(U) *_{\pi_1(U\cap V)} \pi_1(V).
\]
本目录的 MVP 目标是：用可执行代码复现这个代数构造流程，而不是只写定理文字。

## R02

给定三组群表示与两个同态：
- `G_U = <A | R_A>` 表示 `\pi_1(U)`
- `G_V = <B | R_B>` 表示 `\pi_1(V)`
- `H = <T | R_T>` 表示 `\pi_1(U\cap V)`
- `phi_U: H -> G_U`
- `phi_V: H -> G_V`

MVP 输出 `\pi_1(U\cup V)` 的表示：
\[
\langle A, B \mid R_A, R_B, \phi_U(t)\phi_V(t)^{-1}\ (t\in T) \rangle.
\]

## R03

本实现是“表示级算法（presentation-level algorithm）”，输入不是几何网格，而是：
- 生成元列表
- 关系词列表
- 交集群生成元在两边的像（词）

输出包括：
- 推导得到的目标群表示
- 粘合关系列表（每个 `t in T` 对应一条）
- Abel 化关系矩阵（指数和矩阵）与自由秩近似检查

## R04

`demo.py` 的核心数据结构：
- `Word = Tuple[(generator, exponent), ...]`：压缩词表示
- `GroupPresentation(generators, relators)`：有限群表示
- `phi_u / phi_v`：`Dict[str, Word]`，记录交集生成元像

采用指数压缩词（如 `a^2 b^-1`）而不是逐字母数组，便于输出可读且实现简单。

## R05

算法主线：
1. 规范化群表示（去重生成元、约化关系词、丢弃空关系）。
2. 校验同态映射是否覆盖交集群全部生成元。
3. 校验映射像是否只使用目标群合法生成元。
4. 对每个 `t in T` 生成粘合关系 `phi_U(t) phi_V(t)^{-1}`。
5. 合并两侧生成元和关系，得到 pushout 表示。
6. 构建 Abel 化矩阵（关系对各生成元指数和）。
7. 计算矩阵秩，给出 `H_1` 自由秩近似值用于 sanity check。

## R06

正确性要点（MVP 级）：
- 关系构造遵循 van Kampen 的标准表示级公式。
- `relation_from_identification(left, right)` 显式把“等式约束”改写为 relator：`left * right^{-1} = 1`。
- 所有输入词先做自由约化，避免相邻可消去项影响结果。
- 同态合法性校验可拦截最常见输入错误（漏映射、越界生成元）。

## R07

设：
- `n` 为最终生成元数
- `m` 为最终关系数
- `L` 为关系词平均长度（压缩后）

则主要复杂度为：
- 粘合关系构造：`O(|T| * L)`
- 结果规范化：`O(m * L)`
- Abel 化矩阵构造：`O(m * L)`
- 浮点秩计算（`numpy.linalg.matrix_rank`）：约 `O(min(m,n) * m * n)`

该成本适合教学与中小规模表示验证。

## R08

实现中的工程约束：
- 不调用黑盒群论 CAS；核心步骤都在源码中显式展开。
- 不尝试一般词问题与 Tietze 完全化简，只做“诚实最小 MVP”。
- 允许关系词中出现高次幂，内部统一用 `(generator, exponent)` 存储。

## R09

MVP 边界：
- 需要用户给出覆盖诱导同态 `phi_U, phi_V`，不自动从几何对象推导。
- 假设 `U` 与 `V` 的生成元命名不重叠（脚本会检测并报错）。
- Abel 化检查只提供线性层面的佐证，不能替代完整群同构判定。
- 不实现一般 `R_T` 的同态一致性证明（示例里 `H` 都取自由群）。

## R10

`demo.py` 内置 4 个典型案例：
- `S^1` 的“两可缩开弧覆盖” -> 平凡群
- 8 字形（两个圆楔合） -> `F_2`
- 环面 `T^2`（`1`-骨架加一个 `2`-胞腔） -> `<a,b|[a,b]>`
- 实射影平面 `RP^2` -> `<a|a^2>`

这些案例覆盖了：平凡、自由、交换化约束、有限阶约束。

## R11

输出解读：
- `推导得到 pi1(U∪V) 表示`：算法主结果。
- `粘合关系`：直接对应 van Kampen 识别过程。
- `Abel 化关系矩阵`：每行是一个 relator 的指数和向量。
- `rank(H1_free)`：通过关系矩阵秩给出的自由部分维数近似。

## R14

与“直接调用第三方群论库一键求值”相比：
- 本实现把 van Kampen 关键中间量都打印出来（`U/V/U∩V` 表示、粘合关系、结果表示）。
- 有利于教学与调试：能看到哪一条粘合关系导致了最终约束。
- 代价是功能边界更小，不追求自动化处理复杂词问题。

## R18

`demo.py` 源码级算法流（8 步，非黑盒）：
1. `build_cases` 组装 `G_U, G_V, H` 及 `phi_U, phi_V` 的离散输入。
2. `validate_homomorphism` 检查每个交集生成元都有像，且像只用合法生成元。
3. `relation_from_identification` 把 `phi_U(t)=phi_V(t)` 变成 relator `phi_U(t)phi_V(t)^{-1}`。
4. `van_kampen_pushout` 合并两侧生成元和关系，追加所有粘合关系，形成目标表示。
5. `reduce_word`/`multiply_words`/`inverse_word` 在自由群层面对词做规范化，避免冗余。
6. `abelianization_matrix` 把每个 relator 转换成整数指数和矩阵（行=关系，列=生成元）。
7. `abelianization_free_rank` 用 `numpy.linalg.matrix_rank` 给出 `H_1` 自由秩近似检查值。
8. `run_case` 与 `main` 打印每个案例的输入表示、粘合关系、输出表示与汇总表，形成可审阅的端到端结果。
