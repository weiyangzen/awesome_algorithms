## R01

最小二乘有限元（Least-Squares Finite Element Method, LSFEM）是把偏微分方程离散问题转化为“最小化残差平方和”的方法。

核心思想：
- 先把原方程改写为一阶系统；
- 构造残差向量 `R(U)`；
- 最小化泛函 `J(U)=||R(U)||^2`；
- 在有限元空间中求使 `J` 最小的离散解。

本条目给出一个 1D Poisson 方程的最小可运行 MVP，展示从方程到法方程的完整链路。

## R02

本条目求解问题：

`-u''(x)=f(x), x∈(0,1), u(0)=u(1)=0`

选取解析解用于验证：
- `u(x)=sin(pi x)`
- `p(x)=u'(x)=pi cos(pi x)`
- `f(x)=pi^2 sin(pi x)`

输出内容：
- 各网格级别的离散解 `u_h, p_h`；
- `||u-u_h||_L2`、`||p-p_h||_L2`、`|u-u_h|_H1`；
- 一阶系统残差范数 `||r1||_L2, ||r2||_L2`；
- 网格加密阶数估计（`log2(e_h/e_{h/2})`）。

## R03

MVP 的数学模型：

1. 一阶系统重写：
- `r1 = p - u' = 0`
- `r2 = p' + f = 0`

2. 最小二乘泛函：
- `J(u_h,p_h) = ∫(r1^2 + r2^2) dx`

3. 离散策略：
- 区间 `[0,1]` 均匀剖分为 `n` 个单元；
- `u_h, p_h` 均采用 `P1` 线性元；
- `u(0),u(1)` 强制为 0（只把内部节点作为未知量）；
- 用 2 点 Gauss 积分组装 `J` 对应法方程 `M x = b`；
- 线性代数层用 `numpy.linalg.solve` 求解。

## R04

时间复杂度（按当前 MVP 实现）：

- 组装阶段：每个单元固定个数积分点，代价 `O(n)`；
- 求解阶段：当前使用稠密矩阵直接解，未知量规模约 `2n`，代价 `O(n^3)`；
- 后处理误差与残差计算：`O(n)`。

因此总复杂度由稠密求解主导，为 `O(n^3)`。

## R05

空间复杂度（按当前 MVP 实现）：

- 法矩阵 `M` 采用稠密存储，维度约 `2n x 2n`，占 `O(n^2)`；
- 向量和网格数据占 `O(n)`。

总空间复杂度为 `O(n^2)`。

说明：实际工程里通常改为稀疏组装+稀疏求解，将复杂度显著降低。

## R06

单元级残差构造微例（示意）：

在单元 `e=[x_i,x_{i+1}]` 上，令 `h=x_{i+1}-x_i`，局部线性形函数 `N1,N2`，导数 `dN1/dx=-1/h, dN2/dx=1/h`。

若局部自由度写成 `u_i,u_{i+1},p_i,p_{i+1}`，则在任一积分点：
- `p_h = N1*p_i + N2*p_{i+1}`
- `u_h' = dN1*u_i + dN2*u_{i+1}`
- `p_h' = dN1*p_i + dN2*p_{i+1}`

于是：
- `r1 = p_h - u_h'`
- `r2 = p_h' + f(x_q)`

每个积分点都向法矩阵贡献 `w * a a^T`，向右端贡献 `-w * a * b`，这正是 `demo.py` 里手工组装的核心。

## R07

LSFEM 的价值：

- 把 PDE 离散统一成最优化问题，结构清晰；
- 法矩阵天然对称正定（在满足条件时），数值上稳定；
- 可以用同一个框架处理不同类型残差（方程、边界、约束）；
- 对教学非常友好，便于逐层解释“方程 -> 残差 -> 泛函 -> 线性系统”。

## R08

理论基础要点：

1. 在 Hilbert 空间里最小化 `||R(U)||^2` 等价于正交投影思想。  
2. 离散后 `R(U_h)` 是有限维向量函数，最优点满足梯度为零。  
3. 线性问题下，梯度条件可写为法方程 `A^T A x = -A^T c`。  
4. 在本实现中并未显式构造全局 `A`，而是直接按积分点累加 `A^T A` 与 `A^T c`。  
5. 这种“先残差、后平方”路径使 PDE 与线性代数映射关系非常透明。

## R09

适用条件与局限：

- 适合需要统一处理多残差项、并希望获得稳定 SPD 系统的场景；
- 适合做原型验证、教学、以及复杂耦合模型的骨架实现；
- 本 MVP 采用 1D、线性元、均匀网格与稠密求解，重点是透明性而非大规模性能；
- 若直接走法方程，条件数会变差（相当于平方），高维大问题应优先用稀疏与预条件技术。

## R10

正确性框架（本目录可执行版本）：

1. 解析解与 forcing 保持严格一致：`-u''=f`。  
2. 离散解来自最小二乘泛函的离散法方程，而非黑盒 PDE 求解器。  
3. 误差通过独立积分流程计算（不是复用组装值直接“抄答案”）。  
4. 网格加密应导致 `L2` 误差下降。  
5. `demo.py` 中 `run_checks` 对误差下降与最终精度设有断言。

## R11

误差与收敛（本实现观测）：

在 `n=8,16,32,64` 的均匀加密下，运行结果显示：
- `||u-u_h||_L2` 与 `||p-p_h||_L2` 都稳定下降；
- 估计收敛阶接近 2（约 `1.99~2.00`）；
- `|u-u_h|_H1` 及残差范数也随网格细化明显减小。

这与当前测试问题的光滑解和离散设置一致。

## R12

性能视角：

- 这个 MVP 采用“最小但诚实”的实现策略：
  - 显式定义残差；
  - 显式积分组装；
  - 显式误差评估。
- 优点是可审计、易调试、易扩展；
- 缺点是未使用稀疏矩阵与专业求解器，大规模场景效率有限。

后续工程化可替换为：

- 共轭梯度 + 预条件；
- 自适应网格或高阶元。

## R14

鲁棒性与失效模式：

- 配置层会检查网格列表是否合法；
- 组装后检查矩阵/向量是否为有限值；
- 求解前检查条件数，过大即报错；
- 若误差不随网格下降，`run_checks` 会触发断言。

常见失效来源：

- 边界自由度处理错误；
- 误差积分点与单元映射写错。

## R18

源码级算法流程拆解（`demo.py`，8 步）：

1. 设定 Poisson 测试问题与解析解 `u,p,f`，并给出网格级别序列。  
2. 对每个网格，把 `u` 的内部节点和 `p` 的全部节点打包成全局未知向量。  
3. 在每个单元上用 2 点 Gauss 积分，构造 `r1=p_h-u_h'` 与 `r2=p_h'+f` 的线性系数向量。  
4. 把每个积分点的贡献累加到法矩阵与右端：`M += w*a*a^T`，`b += -w*a*b_const`。  
5. 解线性系统 `M x = b`，恢复节点解 `u_h`（含两端 Dirichlet 零值）和 `p_h`。  
6. 再次独立积分计算 `||u-u_h||_L2`、`||p-p_h||_L2`、`|u-u_h|_H1` 与 `||r1||_L2, ||r2||_L2`。  
7. 用相邻网格误差比值计算收敛阶 `log2(e_h/e_{h/2})`。  
8. 执行自动断言（误差递减、最终精度阈值、数值有限性），全部通过后输出成功标记。

说明：实现只把 `numpy.linalg.solve` 用作线性代数求解器；PDE 离散与最小二乘组装全部在源码中显式展开，没有调用黑箱式“现成 LSFEM 一键函数”。
