## R01

光线追踪（Ray Tracing）是“从相机向场景发射光线、通过几何求交与光照模型计算像素颜色”的成像方法。  
本条目实现一个基础版 MVP（非路径积分、非全局光照）：
1. 主光线与球体/平面求交；
2. 局部光照（环境光 + 漫反射 + 高光）；
3. 阴影光线可见性判断（硬阴影）；
4. 输出可直接查看的 PPM 图像。

## R02

问题定义（MVP 范围）：
1. 输入（在代码中固定，不需要交互）：
   - 相机参数：分辨率、视场角、相机原点；
   - 场景参数：两个球体 + 一个地面平面；
   - 材质参数：颜色、环境/漫反射/高光系数、高光指数；
   - 光源参数：两个点光源。
2. 输出：
   - 渲染图像：`render_ray_tracing.ppm`；
   - 统计信息：光线总数、阴影光线数、求交测试次数等；
   - 断言通过时输出 `All checks passed.`。

## R03

核心数学模型：
1. 光线参数方程：`p(t) = o + t d`，`t > 0`。
2. 球体求交：解 `||o + td - c||^2 = r^2`，取最小正根。
3. 平面求交：`t = dot(p0 - o, n) / dot(d, n)`，且 `t > eps`。
4. 漫反射（Lambert）：`I_diff = kd * max(dot(n, l), 0)`。
5. 高光（Phong）：`I_spec = ks * max(dot(v, r), 0)^shininess`。
6. 阴影判定：从命中点朝光源发阴影光线，若在到光源前先碰到物体，则该光源贡献为 0。

## R04

`demo.py` 执行流程：
1. 构造固定场景（2 球 + 1 平面 + 2 点光源）；
2. 初始化 `RayTracer`；
3. 对每个像素发射主光线；
4. 求最近命中（遍历几何体）；
5. 未命中返回背景渐变；
6. 命中后计算环境光、漫反射、高光；
7. 对每个光源发阴影光线判断遮挡；
8. 写出 PPM 图像并打印统计、摘要。

## R05

关键数据结构：
1. `Material`：`color / ambient / diffuse / specular / shininess`；
2. `Sphere`、`Plane`：几何体和 `intersect` 求交方法；
3. `HitRecord`：最近命中的距离、位置、法线、材质、对象名；
4. `PointLight`：位置、颜色、强度；
5. `RayTracer.stats`：`rays_cast / shadow_rays / intersection_tests / surface_hits / shadowed_light_samples`。

## R06

正确性直觉：
1. 最近正距离命中保证“可见表面”选择正确；
2. 阴影光线把“是否看到光源”显式化，能正确产生硬阴影；
3. 光照分项（环境+漫反射+高光）分别对应基础亮度、角度相关亮度和镜面高光；
4. 统一做颜色裁剪到 `[0,1]`，避免数值溢出导致非法像素。

## R07

复杂度分析（设像素数 `P=W*H`，物体数 `O`，光源数 `L`）：
1. 时间复杂度：`O(P * O * (1 + L))`  
   说明：每像素一次最近求交 `O(O)`；命中后每个光源一次阴影可见性测试 `O(O)`。
2. 空间复杂度：`O(P)`（图像缓存）+ `O(1)`（常量级临时变量）。

## R08

边界与鲁棒性处理：
1. 使用 `EPSILON` 避免自相交（shadow acne）；
2. 平面求交时若分母接近 0 直接判定平行；
3. 法线根据入射方向修正正反面，避免错误高光；
4. 对异常短向量做归一化保护；
5. 通过断言检查像素范围、平均亮度和关键统计量。

## R09

MVP 取舍：
1. 不做递归反射/折射，不做全局光照采样；
2. 不引入 BVH/KD-Tree，直接线性遍历物体，代码更短更透明；
3. 使用 `numpy` 处理向量，`pandas` 仅用于控制台摘要；
4. 输出 PPM，避免依赖图像处理库，便于最小可运行交付。

## R10

`demo.py` 主要函数职责：
1. `normalize / reflect`：基础向量运算；
2. `Sphere.intersect / Plane.intersect`：显式几何求交；
3. `RayTracer.find_nearest_hit`：最近命中搜索；
4. `RayTracer.is_shadowed`：阴影可见性检查；
5. `RayTracer.shade`：局部光照计算；
6. `RayTracer.trace_ray`：单光线颜色求值；
7. `RayTracer.render`：逐像素渲染；
8. `save_ppm`：保存图像；
9. `run_sanity_checks`：自动化正确性检查。

## R15

与相邻方法差异：
1. 与光栅化相比：光线追踪天然以“光线-几何求交”表达可见性，不依赖三角形扫描转换流程；
2. 与 Whitted 光线追踪相比：本实现不做递归反射/折射，属于更基础入门版；
3. 与路径追踪相比：本实现确定性、低噪声，但缺少间接光照与全局照明效果。

## R16

适用场景：
1. 图形学教学中的“第一次可运行 ray tracer”；
2. 快速验证求交、阴影、局部光照逻辑；
3. 作为后续 Whitted/路径追踪/BVH 优化的起点。

不适用场景：
1. 电影级真实感渲染（需要全局光照、软阴影、体积效果）；
2. 超大规模实时场景（需要空间加速结构和并行优化）。

## R17

可扩展方向：
1. 增加递归镜面反射与折射；
2. 引入 Fresnel 混合与折射率；
3. 使用 BVH/KD-Tree 降低求交复杂度；
4. 支持三角网格与纹理采样；
5. 迁移到 PyTorch/NumPy 批量化光线以提升吞吐。

## R18

`demo.py` 源码级算法流（8 步）：
1. `main` 固定分辨率、FOV、相机参数并构建默认场景。  
2. 初始化 `RayTracer`，准备几何集合、光源集合和统计计数器。  
3. `render` 对每个像素反投影得到主光线方向并调用 `trace_ray`。  
4. `trace_ray` 使用 `find_nearest_hit` 遍历 `Sphere/Plane.intersect`，选出最近正距离命中。  
5. 若无命中，则返回 `background` 渐变；若命中进入 `shade`。  
6. `shade` 先计算环境光，再对每个点光源计算 Lambert 漫反射与 Phong 高光。  
7. 每个光源调用 `is_shadowed` 发阴影光线；若在光源距离内提前碰撞，则该光源贡献被抑制。  
8. 将颜色裁剪后写入图像缓冲，最终 `save_ppm` 输出文件并由 `run_sanity_checks` 做断言验证。
