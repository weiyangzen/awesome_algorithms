## R01

ε-约束法（epsilon-constraint method）是多目标优化里最经典的标量化方法之一。

核心思想：

- 从多个目标里挑一个作为主优化目标；
- 其余目标转成约束，形如 `f_j(x) <= epsilon_j`；
- 通过扫一组 `epsilon` 值，得到一组可比解，近似 Pareto 前沿。

本目录给出一个线性双目标 MVP，并用 `scipy.optimize.linprog` 完整跑通。

## R02

问题背景：在双目标最小化 `min (f1(x), f2(x))` 中，单一最优通常不存在，只有“互不支配”的折中解。

- 若只优化 `f1`，常会牺牲 `f2`；
- 若只优化 `f2`，常会牺牲 `f1`。

ε-约束法把这种冲突显式化：`epsilon` 越宽松，`f1` 可优化空间越大；`epsilon` 越严格，解会向 `f2` 最优端点移动。

## R03

本实现任务定义：

- 决策变量：`x = (x1, x2)`；
- 目标函数：
  `f1(x) = x1 + 4x2`，`f2(x) = 4x1 + x2`；
- 基础约束：
  `x1 + x2 >= 1`，`2x1 + x2 <= 4`，`x1 + 2x2 <= 4`，`x1,x2 >= 0`。

对每个 `epsilon` 子问题求解：

`min f1(x)`

`s.t. 原始可行域约束`

`     f2(x) <= epsilon`

## R04

数学形式（双目标场景）：

- 原问题：`min (f1(x), f2(x)), x in X`
- ε-约束子问题：
  `min f1(x)`
  `s.t. x in X, f2(x) <= epsilon`

在本线性示例中，子问题是标准 LP，可直接由 `linprog` 求解。

可行 `epsilon` 区间由两个锚点给出：

- `epsilon_min = min_{x in X} f2(x)`；
- `epsilon_max = f2(x_f1*)`，其中 `x_f1* = argmin_{x in X} f1(x)`。

## R05

高层流程：

1. 先单独求 `f1` 与 `f2` 的锚点最优解；
2. 用锚点确定 `epsilon` 扫描范围；
3. 对每个 `epsilon` 解一次 LP 子问题；
4. 收集 `(x, f1, f2, epsilon)` 结果表；
5. 过滤非支配点作为 Pareto 近似；
6. 做断言校验（约束满足、单调性、几何一致性）。

## R06

本 MVP 的默认设置：

- `epsilon` 等距取样 13 个点；
- LP 求解器：`scipy.optimize.linprog(method="highs")`；
- 输出：完整扫描表 + 非支配点表 + 两个端点锚点。

示例被特意设计为“目标冲突清晰、可解释性强”的教学规模，不依赖外部数据。

## R07

复杂度分析（设 `K` 个 epsilon 点，`n` 变量，`m` 约束）：

- 外层：`K` 次子问题求解；
- 每个子问题：一次 LP（HiGHS 内部复杂度依问题结构而定，工程上通常近似多项式可扩展）；
- 总体：约为 `K * LP(n,m)`；
- 额外开销：非支配筛选 `O(K^2)`（本例 K 很小）。

## R08

数值与工程细节：

- 统一转成 `A_ub x <= b_ub` 格式，避免符号混乱；
- 每次 `epsilon` 解都记录 `constraint_slack = epsilon - f2` 便于调试；
- 校验 `f2 <= epsilon + tol`（`tol=1e-7`）处理浮点误差；
- 结果集中按 `epsilon` 排序后再做单调性检查。

## R09

结果解读要点：

- `epsilon` 小：`f2` 约束更紧，解更偏向 `f2` 最优端；
- `epsilon` 大：`f1` 可继续下降，直到达到 `f1` 锚点最优；
- 非支配点集用于描述“改进一个目标必然恶化另一个目标”的折中边界。

本例中会观察到近似线性 trade-off，便于直观看懂 ε-约束机制。

## R10

`demo.py` 模块职责：

- `build_problem`：构造双目标 LP；
- `solve_lp`：统一封装 `linprog`；
- `objective_values`：计算 `(f1,f2)`；
- `epsilon_constraint_sweep`：执行 epsilon 扫描；
- `pareto_filter_min`：做最小化意义下的非支配筛选；
- `validate_results`：执行断言门禁；
- `main`：总控、打印与端点摘要。

## R12

输出字段说明：

- `epsilon`：当前 ε-约束阈值；
- `x1, x2`：该子问题最优决策变量；
- `f1, f2`：对应双目标值；
- `constraint_slack`：`epsilon - f2`，越接近 0 说明约束越紧；
- `status`：求解状态（本例期望全为 `optimal`）。

## R13

正确性门禁（代码中已实现）：

1. 结果表不能为空；
2. 所有子问题必须成功求解；
3. 所有点必须满足 `f2 <= epsilon + tol`；
4. `epsilon` 放宽时，最优 `f1` 不能变差（单调不增）；
5. 本构造例中 `f1 + f2 ≈ 5`，用于几何一致性 sanity check。

## R14

常见失效模式与对策：

- 失效 1：`epsilon` 取值过小，子问题不可行。
  对策：先求 `f2` 锚点确定下界。

- 失效 2：只取很少 epsilon 点，前沿采样过粗。
  对策：增加采样密度或采用自适应采样。

- 失效 3：数值误差导致“看似违反约束”。
  对策：统一容差判定，不做硬等号比较。

- 失效 4：使用黑箱求解但缺少验证。
  对策：像本脚本一样加断言与可解释输出。

## R15

实践建议：

- 先算锚点再扫 epsilon，避免盲扫不可行区间；
- 输出不仅看目标值，也要看变量与约束 slack；
- 若目标尺度差异大，可先标准化再设置 epsilon 网格；
- 报告时建议同时给出端点解和中间折中解。

## R16

与其他多目标方法对比：

- 加权和法：实现更简单，但在非凸前沿上可能漏解；
- ε-约束法：对每个 epsilon 都是受约束单目标，通常更容易覆盖前沿结构；
- 理想点法/目标规划：依赖目标归一化和偏好参数，解释性取决于设定。

本任务选择 ε-约束，是为了直接展示“约束阈值变化 -> 折中解移动”的因果链。

## R17

可扩展方向：

- 双目标扩展到多目标：固定一个主目标，其余都加 epsilon 约束；
- 线性扩展到非线性：把 `linprog` 换成 `scipy.optimize.minimize`/NLP 求解器；
- 结合可视化：绘制 `(f1,f2)` 前沿曲线和 `epsilon`-解映射；
- 结合业务约束：将预算、风险、资源上限都并入 `X`。

## R18

`demo.py` 源码级算法流（8 步，含第三方求解链路）：

1. `build_problem` 定义 `c1,c2,A_ub,b_ub,bounds`，把双目标问题写成统一 LP 数据结构。  
2. `epsilon_constraint_sweep` 先调用 `solve_lp` 两次，分别解 `min f1` 与 `min f2` 锚点问题。  
3. 用锚点计算 `epsilon` 扫描区间 `[f2_min, f2(x_f1*)]`，并生成等距网格。  
4. 对每个 `epsilon`，在基础约束下额外拼接一行 `c2^T x <= epsilon`，构造当前子问题。  
5. `solve_lp` 调用 `scipy.optimize.linprog(method="highs")`，SciPy 将问题传给 HiGHS 做 presolve、选解法（单纯形/IPM）并返回最优解与状态。  
6. 成功时计算 `f1,f2,slack` 写入结果表；失败时写入错误状态，保持可追踪性。  
7. `validate_results` 对结果执行门禁：可行性、约束满足、`f1` 单调性、几何一致性。  
8. `pareto_filter_min` 对结果做非支配筛选，`main` 打印扫描表、Pareto 近似和两端锚点。  

这 8 步形成了“锚点定界 -> epsilon 扫描 -> LP 求解 -> 结果验证 -> Pareto 提取”的完整可运行链路。
