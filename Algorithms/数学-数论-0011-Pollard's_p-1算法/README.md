## R01

Pollard's p-1 算法是一种整数分解算法，利用某个素因子 `p` 满足 `p-1` 由小素数构成（即 `B`-smooth）的性质来提取因子。

- 输入: 合数 `n`，光滑界 `B`，底数 `a`（常取 2 或随机与 `n` 互素的整数）。
- 输出: `n` 的一个非平凡因子 `d`（满足 `1 < d < n`），若失败则返回失败状态。
- 核心机制: 在模 `n` 下计算 `a^M`，其中 `M` 是 `lcm(1,2,...,B)` 的等价构造；若 `p-1 | M`，则通常有 `a^M ≡ 1 (mod p)`，从而 `gcd(a^M-1, n)` 可能暴露因子 `p`。

Stage-1 标准流程:
1. 生成所有不超过 `B` 的素数 `q`。
2. 对每个 `q` 取最大幂 `q^e <= B`。
3. 累积更新 `x <- x^(q^e) mod n`（等价于 `x = a^M mod n`）。
4. 计算 `d = gcd(x-1, n)`。
5. 若 `1 < d < n` 则成功；若 `d=1` 或 `d=n` 则该组参数失败。

## R02

- 提出者: John M. Pollard。
- 时间: 1974 年（相关工作发表于 1974-1975 年间）。
- 历史定位: 与 Pollard's Rho 同属早期高影响随机/半随机分解方法，成为现代分解工具链中的经典组件。

## R03

提出背景是“比试除法和费马分解更快地处理大整数因式分解”的需求。

- 传统方法不足:
  - 试除法在大整数上成本过高。
  - 费马分解依赖因子接近，不具普适效率。
- 新思路:
  - 在乘法群结构上做文章，不直接搜索因子。
  - 借助 `p-1` 的光滑性，把因子发现转化为群阶整除与 `gcd` 提取。

## R04

设 `n` 的比特长度为 `L=log2(n)`，Stage-1 光滑界为 `B`。

- 筛素数成本: `O(B log log B)`。
- 模幂主成本: 对每个素数 `q<=B` 做一次模幂，指数规模约 `O(log B)` 位；
  典型可写为 `O(pi(B) * log B * M(L))`，其中 `M(L)` 是 `L` 位乘法复杂度。
- 若采用学校乘法 `M(L)=O(L^2)`，可近似理解为 `O(B * L^2)` 量级（忽略对数因子后）。

注意: 这是条件性算法，整体期望时间高度依赖“是否存在 `p-1` 足够光滑的因子”。

## R05

Stage-1 额外空间主要来自:

- 素数表: `O(pi(B))`，约 `O(B/log B)`。
- 大整数中间变量 `x`、`gcd` 计算缓存: `O(log n)` 字长。

综合可记为 `O(B/log B + log n)`（实现中通常由素数表主导）。

## R06

可复现实例（与 `demo.py` 一致）:

- 目标: `n = 11413 = 101 * 113`
- 取 `B=7`，`a=2`
- 不超过 7 的素数为 `2,3,5,7`，对应最大幂分别为 `4,3,5,7`
- 迭代计算后得到 `x = 2^M mod n`（`M` 等价于 `lcm(1..7)`）
- 计算 `d = gcd(x-1,11413) = 113`

得到非平凡因子 `113`，另一因子为 `101`，分解成功。

## R09

适用前提与边界:

- 目标通常是奇合数 `n`（偶数可直接提取 2）。
- 至少存在一个素因子 `p|n`，使 `p-1` 在给定 `B` 下足够光滑。
- 仅 Stage-1 时，对“存在大素因子的 `p-1`”成功率明显下降。

不适用/效果差场景:

- 所有素因子的 `p-1` 都不光滑（相对当前 `B`）。
- `B` 过小导致系统性失败。
- 某些参数下可能得到 `d=n`（平凡结果）。

## R10

正确性依据（简述）:

设 `n = p*q*...`，并取 `M` 为 Stage-1 构造指数。

1. 若 `p-1 | M` 且 `gcd(a,p)=1`，则由群阶整除得 `a^M ≡ 1 (mod p)`。
2. 因此 `p | (a^M - 1)`。
3. 若对另一因子 `r` 有 `a^M \not\equiv 1 (mod r)`，则 `r \nmid (a^M-1)`。
4. 所以 `d=gcd(a^M-1,n)` 会包含 `p` 但不包含全部因子，进而 `1<d<n`。

这说明当“目标因子满足光滑条件且非目标因子未同步满足”时，算法返回非平凡因子。

## R11

该算法本质是整数运算，数值稳定性问题与浮点算法不同。

- 无浮点舍入误差主导问题（核心运算是模幂与 `gcd`）。
- 误差风险主要来自实现细节:
  - 若用浮点 `log` 计算幂次，可能因舍入导致指数偏差。
  - 固定宽度整数语言可能溢出。
- `demo.py` 处理方式:
  - 用“重复乘法”求 `q^e<=B`，避免浮点误差。
  - 每步做模约简，避免中间值失控。

## R12

真实成本（超越渐近符号）:

- 热点在多次大整数模幂，常数项显著。
- `pow(base, exp, mod)` 的实现质量直接决定速度（CPython 内置实现较高效）。
- 若使用多组 `(B, a)` 调度，失败重试会线性增加实际耗时。
- 内存压力通常不大，但当 `B` 升高时，筛素数与幂操作次数会迅速增长。
- 并行策略常见于“多底数并行尝试”。

## R13

概率保证（随机底数版本）:

- 固定 `(n,B,a)` 时算法是确定性的。
- 随机性来自“随机采样底数 `a`”。

对半素数 `n=pq` 的直观保证:

- 若 `p-1 | M` 且 `q-1 \nmid M`，成功条件是 `a^M \not\equiv 1 (mod q)`。
- 在模 `q` 的乘法群中，满足 `x^M=1` 的元素个数是 `gcd(M, q-1)`。
- 因而随机 `a`（且与 `n` 互素）失败比例约为 `gcd(M,q-1)/(q-1)`，
  成功概率可写成近似下界 `1 - gcd(M,q-1)/(q-1)`。

这解释了“换底数重试”为什么能提升成功率。

## R14

鲁棒性与失效模式:

- `d=1`: 常见失败，通常是 `B` 太小或底数不合适。
- `d=n`: 平凡失败，表示多个因子同时满足同余条件。
- 对恶劣输入（如 `p-1` 含大素因子）需要更大 `B` 或改用 ECM。
- 若 `gcd(a,n)>1`，会提前直接得到因子（这是“意外成功”，不是异常）。

## R15

工程实现要点:

- 先处理平凡情形: `n<=1`、偶数、`gcd(a,n)>1`。
- 不显式构造巨大 `M`，而是按素数幂逐步更新 `x <- x^(q^e) mod n`。
- `B` 不宜一次拉太大，建议分层调度（如 1e2, 1e3, 1e4 ...）。
- 失败后优先换底数，再逐步加大 `B`。
- 为调试保留尝试日志（`B`、`a`、`gcd`），便于判断是“界限不足”还是“底数问题”。

## R17

Python MVP 方案（见 `demo.py`）:

- 核心函数:

## R18

源码追踪与 3-10 步算法拆解（对应 `demo.py`，非黑箱）：

1. 通过 `sieve_primes(B)` 生成 Stage-1 所需素数集合 `P={q | q<=B}`。
2. 对每个 `q in P`，用 `largest_power_leq(q,B)` 计算最大幂 `q^e <= B`。
3. 初始化 `x = a mod n`，并迭代执行 `x = pow(x, q^e, n)`。
4. 迭代结束后，`x` 等价于 `a^M mod n`（`M` 覆盖了所有 `q^e`）。
5. 计算 `d = gcd(x-1, n)` 并分类:
   - `1 < d < n`: 返回非平凡因子（成功）。
   - `d == 1` 或 `d == n`: 当前参数失败。
6. 外层 `factor_with_schedule` 对不同 `B` 与 `a` 组合重试，直到找到因子或耗尽预算。

这 6 步就是可执行、可检查、可复现实装流程。
