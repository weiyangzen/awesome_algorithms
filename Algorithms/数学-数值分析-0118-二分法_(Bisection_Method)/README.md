# 二分法 (Bisection Method)

- UID: `MATH-0118`
- 学科: `数学`
- 分类: `数值分析`
- 源序号: `118`
- 目标目录: `Algorithms/数学-数值分析-0118-二分法_(Bisection_Method)`

## R01

本条目实现二分法（Bisection Method）的一维求根最小可运行版本：
- 输入一个连续函数 `f(x)` 与区间 `[a, b]`；
- 要求区间端点异号（或端点即根）；
- 通过反复“折半缩小区间”逼近实根。

MVP 重点是把“区间不变式 + 收敛判据 + 可观测迭代轨迹”完整落地，而不是追求最短代码。

## R02

问题定义（有限精度计算版）：
- 输入：
  - 标量函数 `f: R -> R`；
  - 区间端点 `a < b`；
  - 迭代上限 `max_iter` 与误差阈值 `tol`。
- 前置条件：
  - `f(a)` 与 `f(b)` 为有限实数；
  - `f(a) * f(b) <= 0`（若 `< 0` 则区间包根，若 `= 0` 则端点为根）。
- 输出：
  - 根近似 `x*`；
  - 迭代历史（每轮 `a,b,mid,f(mid),half_width`）。

## R03

数学原理：

1. 介值定理保证存在性：若 `f` 在 `[a,b]` 连续且 `f(a)f(b) < 0`，则存在 `c in (a,b)` 使 `f(c)=0`。  
2. 每轮取中点 `m = (a+b)/2`，检查 `f(m)` 符号：
   - 若 `f(a)f(m) < 0`，根在 `[a,m]`；
   - 否则根在 `[m,b]`。  
3. 区间长度每次减半：`|I_k| = (b-a)/2^k`，因此中点误差上界满足
   `|x_k - r| <= (b-a)/2^(k+1)`（`r` 为真实根）。

二分法是线性收敛，但稳定性很强。

## R04

算法流程总览：
1. 校验输入：`a,b` 有限、`a < b`、`max_iter > 0`、`tol > 0`。  
2. 计算 `fa=f(a), fb=f(b)`，检查其有限性。  
3. 若 `fa==0` 或 `fb==0`，直接返回端点根。  
4. 若 `fa*fb > 0`，抛出异常（不满足包根条件）。  
5. 进入循环：计算中点 `mid` 与 `f(mid)`。  
6. 记录本轮轨迹 `(k,left,right,mid,f_mid,half_width)`。  
7. 若 `|f(mid)| <= tol` 或 `half_width <= tol`，停止并返回。  
8. 依据符号更新左右端点，继续下一轮。

## R05

核心数据结构（`demo.py`）：
- `IterationRecord = (k, left, right, mid, f_mid, half_width)`：
  - `k`：迭代编号；
  - `left/right`：当前包根区间；
  - `mid`：中点估计；
  - `f_mid`：中点函数值；
  - `half_width`：当前半区间宽度（误差上界指标之一）。
- `history: list[IterationRecord]`：保存完整收敛轨迹，便于教学与调试。

## R06

正确性要点（不变式视角）：
- 初始时区间满足“端点异号或端点是根”；
- 每次更新都保留根所在子区间，因此包根不变式保持；
- 区间长度单调减半，形成嵌套闭区间序列；
- 在连续性与有限精度判停条件下，返回值是可控误差内的根近似。

## R07

复杂度分析：
- 时间复杂度：`O(K)`，其中
  `K ~= ceil(log2((b-a)/tol))`（受 `max_iter` 截断）。
- 空间复杂度：
  - 保留轨迹时为 `O(K)`；
  - 若只保留当前状态可降为 `O(1)`。

相比牛顿法，二分法步数更多，但每步仅需一次函数值与符号判断，鲁棒性更高。

## R08

边界与异常处理：
- `a >= b`：抛 `ValueError`；
- `a/b` 非有限值：抛 `ValueError`；
- `f(a)`、`f(b)`、`f(mid)` 非有限值：抛 `ValueError`；
- `fa*fb > 0`（区间不包根）：抛 `ValueError`；
- 端点恰好为根：直接返回端点；
- 达到 `max_iter` 仍未触发阈值：返回最后一个中点近似。

## R09

MVP 取舍：
- 采用纯标准库（`math` + `typing`），减少依赖并保证可移植性；
- 保留完整迭代历史，牺牲少量内存换取可解释性；
- 不使用更复杂的混合算法（如 Brent），优先呈现二分法核心机制。

## R10

`demo.py` 函数职责：
- `check_finite_number`：统一有限性检查；
- `bisection`：二分法主实现（校验、迭代、判停、返回轨迹）；
- `relative_error`：计算相对误差；
- `print_trace`：格式化输出前若干轮迭代；
- `run_case`：运行单个方程案例；
- `run_invalid_interval_demo`：展示区间不包根时的预期异常；
- `main`：组织固定样例，无交互执行入口。

## R12

输出字段解读：
- `estimate`：算法输出的根近似；
- `reference`：用于对照的已知高精度值；
- `relative_error`：相对误差；
- `iter`：当前迭代编号；
- `a/b`：当前区间端点；
- `mid`：当前中点；
- `f(mid)`：当前残差；
- `half_width`：当前半区间宽度（可视作误差上界尺度）。

## R13

建议最小测试集：
- 典型非线性方程：
  - `x^3 - x - 2 = 0`，区间 `[1,2]`；
  - `cos(x) - x = 0`，区间 `[0,1]`；
  - `x^2 - 2 = 0`，区间 `[1,2]`；
  - `exp(x) - 3 = 0`，区间 `[0,2]`。
- 边界场景：
  - 端点即根：`x - 5 = 0`，区间 `[5,8]`；
  - 无包根区间（应报错）：`x^2 + 1`，区间 `[-1,1]`。

## R14

可调参数与建议：
- `tol`：停止阈值，默认 `1e-12`；
- `max_iter`：最大迭代轮数，默认 `100`；
- `max_lines`：轨迹打印行数，默认 `8`。

调参建议：
- 先固定较小 `tol` 验证正确性；
- 若函数计算昂贵，可适当放宽 `tol` 或减少 `max_iter`。

## R15

方法对比：
- 对比牛顿法：
  - 牛顿法通常更快（二次收敛），但依赖导数和初值；
  - 二分法仅依赖符号变化，收敛慢但更稳。
- 对比割线法：
  - 割线法不需导数但可能失稳；
  - 二分法有明确包根不变式，更容易给出误差上界。
- 工程实践中常把二分法作为“保底策略”或混合算法初始化阶段。

## R16

应用场景：
- 一维非线性方程的鲁棒求根；
- 数值分析教学中的收敛与误差上界演示；
- 更复杂求解器（Brent、混合迭代）中的安全后备步骤；
- 参数反解问题中“先定位区间再精细求解”的第一阶段。

## R18

源码级算法流（对应 `demo.py`，8 步）：
1. `main` 定义固定样例与全局参数 `max_iter/tol`，逐个调用 `run_case`。  
2. `run_case` 将方程与区间传给 `bisection`，并在返回后计算相对误差。  
3. `bisection` 首先通过 `check_finite_number` 校验 `a,b`，再检查 `a < b`、`max_iter`、`tol`。  
4. 计算 `fa=f(a), fb=f(b)` 并检查有限性；若任一端点已为根则直接返回。  
5. 若 `fa*fb > 0`，立即抛出 `ValueError`，阻止“无包根”错误输入继续迭代。  
6. 在循环内计算 `mid` 与 `f_mid`，记录 `IterationRecord(k,left,right,mid,f_mid,half_width)` 到 `history`。  
7. 若满足 `abs(f_mid) <= tol` 或 `half_width <= tol` 则停止；否则按符号规则更新左右端点，保持包根不变式。  
8. `run_case` 调用 `print_trace` 展示前几轮轨迹；`run_invalid_interval_demo` 单独演示异常分支输出。
