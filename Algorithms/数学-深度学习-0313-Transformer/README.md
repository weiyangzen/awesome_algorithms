## R01

Transformer 是一种以注意力机制为核心的序列建模架构。它用“自注意力 + 前馈网络 + 残差连接 + 层归一化”替代传统 RNN 的逐步递归计算，实现了更强的并行性与更长程依赖建模能力。

标准 Transformer（Vaswani et al., 2017）由编码器-解码器组成：
- 编码器把源序列映射为上下文表示；
- 解码器在因果掩码下自回归生成目标序列，并通过交叉注意力读取编码器输出。

## R02

在 Transformer 出现前，序列到序列任务主要依赖 RNN/LSTM/GRU。它们在长序列上存在路径长、并行度低、梯度传播困难等问题。Transformer 通过“任意位置直接注意任意位置”把有效路径长度缩短为常数级，并在 GPU 上高效并行。

该架构很快成为 NLP、语音、多模态甚至视觉任务的基础模块，并衍生出 BERT、GPT、ViT 等一系列模型。

## R03

Transformer 主要解决三个问题：
1. 长距离依赖难建模：RNN 信息需逐步传递，远距离关系易衰减；
2. 训练吞吐受限：RNN 时间步串行，难以大规模并行；
3. 可扩展性不足：随数据与模型增大，传统架构训练效率瓶颈明显。

Transformer 的核心策略是把序列关系表达为“注意力权重矩阵”，让每个位置动态汇聚全局上下文。

## R04

本目录 MVP 实现了一个源码透明的 Tiny Transformer（未调用 `nn.Transformer` 黑盒）并在“序列反转”任务上训练：
- 输入：数字序列（如 `[2, 7, 5, 1]`）；
- 输出：反转序列（`[1, 5, 7, 2]`）；
- 训练方式：教师强制（teacher forcing）+ 交叉熵；
- 推理方式：贪心自回归解码。

关键数学组件：
- 缩放点积注意力：`Attention(Q,K,V)=softmax(QK^T/sqrt(d_k))V`
- 多头机制：将 `d_model` 切分为 `h` 个子空间并行注意力，再拼接线性映射；
- 解码器自注意力使用因果掩码，禁止访问未来 token。

## R05

设：
- 批大小 `B`，序列长度 `L`，模型维度 `d`，头数 `h`，层数 `N`。

单层注意力主要复杂度约为 `O(B * L^2 * d)`，前馈层约为 `O(B * L * d^2)`，总训练复杂度可粗略写为：
- `O(N * (B * L^2 * d + B * L * d^2))`。

对本 MVP（`L<=8`）而言，计算量很小，CPU 即可快速训练。

## R06

以样本 `src=[2,7,5,1]` 为例：
- 编码器输入：`[2,7,5,1,<EOS>]`；
- 解码器训练输入：`[<BOS>,1,5,7,2]`；
- 监督标签：`[1,5,7,2,<EOS>]`。

训练时每一步都看见真实历史；推理时只依赖已生成 token，逐步产出下一个 token。

## R07

优点：
- 并行性强，训练吞吐高；
- 全局依赖建模能力强；
- 架构模块化，易扩展。

局限：
- 注意力 `L^2` 开销对超长序列不友好；
- 小数据场景易过拟合；
- 推理阶段自回归解码仍是串行。

## R08

实现本算法建议具备以下基础：
- 线性代数：矩阵乘法、向量点积、softmax；
- 概率与优化：交叉熵、梯度下降、反向传播；
- 深度学习工程：mask 机制、残差结构、layer norm；
- PyTorch 张量维度管理与训练循环。

## R09

适用场景：
- 机器翻译、摘要、对话生成等序列到序列任务；
- 需要建模长程依赖的 token 序列任务；
- 训练阶段重视并行吞吐。

不适用或需改造：
- 超长上下文但算力受限（需稀疏注意力/线性注意力/分块策略）；
- 实时超低延迟生成（需蒸馏或小模型部署优化）。

## R10

正确性检查要点：
1. 解码器因果掩码必须生效，否则会“偷看未来”；
2. `PAD` 位在注意力中必须被屏蔽；
3. 训练标签需与解码器输入严格右移对齐；
4. 预测结束条件（`<EOS>`）要正确处理；
5. loss 应有下降趋势，且反转任务 exact-match 应显著高于随机。

## R11

数值与训练稳定性策略（MVP 已实现）：
- 注意力打分使用 `1/sqrt(d_k)` 缩放，抑制 softmax 饱和；
- masked 位置填充大负值（`-1e9`）避免无效注意力泄漏；
- 使用 `LayerNorm + Residual` 稳定深层训练；
- 使用梯度裁剪（`clip_grad_norm_`）减少梯度爆炸风险；
- 固定随机种子确保可复现。

## R12

关键超参数及影响：
- `d_model`：表示容量；过小欠拟合，过大训练更慢；
- `num_heads`：多视角关系建模能力；需整除 `d_model`；
- `num_layers`：深度；更深表达更强但训练更难；
- `d_ff`：前馈层容量；
- `lr / batch_size / epochs`：优化速度与稳定性权衡。

本 MVP 默认值：`d_model=64, heads=4, layers=2, d_ff=128`，用于快速演示而非 SOTA 追求。

## R13

理论上，Transformer 训练目标是非凸优化，不保证全局最优。实践中通过随机梯度法可找到性能良好的局部解。

本任务（序列反转）是监督学习拟合问题，不涉及“近似比”定义；评估以经验指标（loss、exact-match）为主。若模型容量足够且训练稳定，Transformer 能学习该确定性映射。

## R14

常见失效模式与排查：
- 失效：loss 不降。  
  排查：学习率过大、掩码逻辑错误、标签错位。
- 失效：输出重复或过早终止。  
  排查：`<BOS>/<EOS>` 编码与解码循环边界。
- 失效：验证精度低但训练 loss 低。  
  排查：过拟合，需增数据或正则。
- 失效：训练不稳定。  
  排查：梯度裁剪、初始化、批大小。

## R15

工程落地建议：
- 先用可控 toy 任务验证数据流与 mask，再扩展真实数据；
- 明确 tokenizer 与 special token 规范，避免训练/推理不一致；
- 保留配置与随机种子，便于复现实验；
- 对生产系统，优先引入学习率调度、mixed precision、checkpoint 与监控指标。

## R16

相关模型对比：
- RNN/LSTM：时间递归，长依赖和并行性较弱；
- Transformer：全局注意力并行建模，成为主流基础架构；
- BERT：基于 Transformer Encoder 的双向预训练模型，偏理解任务；
- GPT：基于 Transformer Decoder 的自回归模型，偏生成任务。

因此 Transformer 更像“通用积木”，BERT/GPT 是在其结构约束与训练目标上的特化。

## R17

本目录 `demo.py` 提供的 MVP 能力：
- 从零实现多头注意力、编码器层、解码器层与位置编码；
- 构造序列反转数据集并进行监督训练；
- 执行贪心自回归解码并计算测试集 exact-match；
- 输出若干可读样例（源序列、预测序列、真值序列）。

## R18

`demo.py` 的源码级算法流程（8 步）：
1. `build_reverse_dataset` 生成随机数字序列及其反转标签，`tensorize_pairs` 组装 `src/tgt_in/tgt_out` 并做 padding。  
2. `PositionalEncoding` 预计算正弦/余弦位置编码，注入序列位置信息。  
3. `MultiHeadAttention` 对输入做 `Q/K/V` 线性投影，分头后计算 `softmax(QK^T/sqrt(d_k))V`，再拼接映射。  
4. `EncoderLayer` 执行“自注意力 + 残差归一化 + 前馈网络 + 残差归一化”，堆叠得到编码器表示。  
5. `DecoderLayer` 先做带因果掩码的自注意力，再对编码器输出做交叉注意力，最后经前馈层得到解码表示。  
6. `TransformerSeq2Seq.forward` 组合源/目标嵌入、位置编码、mask 构造、编码器与解码器，输出词表 logits。  
7. `train_model` 使用交叉熵（忽略 `PAD`）训练，执行反向传播、梯度裁剪与 Adam 参数更新，记录 epoch loss。  
8. `greedy_decode` 自回归逐 token 解码到 `<EOS>`；`evaluate_exact_match` 统计精确匹配率，并在 `main` 中做结果断言。
