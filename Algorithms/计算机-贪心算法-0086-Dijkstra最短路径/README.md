# Dijkstra最短路径

- UID: `CS-0066`
- 学科: `计算机`
- 分类: `贪心算法`
- 源序号: `86`
- 目标目录: `Algorithms/计算机-贪心算法-0086-Dijkstra最短路径`

## R01

问题定义：给定一个带权图 `G=(V,E)` 与源点 `s`，在边权均为非负数的前提下，求 `s` 到其余各顶点的最短路径长度。

Dijkstra 算法的贪心思想是：
- 每一步从“当前未确定顶点”中选出临时距离最小的顶点 `u`；
- 将 `u` 的最短路值“定型”；
- 用 `u` 去松弛邻边，尝试改进其他顶点距离。

## R02

MVP 输入约定：
- `num_nodes`：顶点总数，编号 `0..num_nodes-1`；
- `edges`：有向边列表，格式 `(u, v, w)`；
- `source`：源点编号。

MVP 输出：
- `dist`：长度为 `num_nodes` 的最短距离数组，不可达记为 `math.inf`；
- `prev`：前驱数组，`prev[v]` 表示最短路上 `v` 的前一个点，不存在时为 `-1`。

## R03

贪心策略细化：
1. 初始化 `dist[source]=0`，其他为 `inf`；
2. 将 `(0, source)` 放入最小堆；
3. 每次弹出堆顶最小临时距离顶点 `u`；
4. 若 `u` 已被定型则跳过；
5. 否则扫描 `u` 的出边 `(u, v, w)`，执行松弛：
   `if dist[u] + w < dist[v] -> 更新 dist[v] 与 prev[v]`。

## R04

正确性核心依赖“非负边权”条件：
- 当一个顶点以最小临时距离从堆顶弹出并首次被定型时，后续任何路径都不可能通过非负边把它变得更短；
- 因此定型是安全的；
- 重复该过程可得到源点到所有可达点的全局最短路。

若存在负权边，该性质失效，Dijkstra 不再保证正确。

## R05

`demo.py` 中的数据结构：
- 邻接表 `adj[u] = [(v, w), ...]`；
- 最小堆（`heapq`）存候选 `(distance, node)`；
- `settled` 布尔数组标记“最短路已定型”；
- `dist` 与 `prev` 记录结果。

另外实现 `bellman_ford_reference` 作为交叉校验，不替代主算法。

## R06

算法流程（堆优化 Dijkstra）：
1. 检查输入合法性（节点范围、非负边权）；
2. 构建邻接表；
3. 初始化 `dist/source/heap`；
4. 弹出堆顶候选；
5. 若已定型则继续下一轮；
6. 否则定型并松弛所有出边；
7. 堆空后结束，得到最短距离数组；
8. 通过 `prev` 可恢复任意目标点路径。

## R07

复杂度：
- 建图：`O(E)`；
- 每条边最多触发常数次有效松弛入堆，堆操作 `O(log V)`；
- 总时间复杂度：`O((V+E) log V)`；
- 空间复杂度：`O(V+E)`。

## R08

适用前提：
- 边权必须非负；
- 允许图不连通，不可达顶点距离保持 `inf`；
- 支持有向图，若要无向图可把每条边双向加入。

限制：
- 不处理负权边与负环；
- 若业务包含负权，应改用 Bellman-Ford 或 Johnson 等算法。

## R09

与常见最短路算法对比：
- Dijkstra：非负权单源最短路，效率高；
- Bellman-Ford：可处理负权，时间复杂度更高 `O(VE)`；
- Floyd-Warshall：全源最短路，复杂度 `O(V^3)`；
- A*：单对点启发式搜索，依赖可行启发函数。

## R10

工程实现要点：
- 用“惰性删除”处理堆中过期条目（通过 `settled` 跳过）；
- 使用 `math.inf` 表示不可达，避免魔法值；
- 对输入边端点越界、负权边提前抛出 `ValueError`；
- 路径恢复与最短路计算分离，便于复用。

## R11

`demo.py` 模块拆分：
- `build_adjacency`：建图 + 输入校验；
- `dijkstra_shortest_paths`：主算法；
- `reconstruct_path`：由 `prev` 恢复源点到目标点路径；
- `bellman_ford_reference`：小图交叉校验；
- `make_random_graph`：构造固定随机测试图；
- `main`：样例运行、断言、异常分支验证。

## R13

内置确定性样例（6 个点，双向边以两条有向边表示）从源点 `0` 出发，预期最短距离为：

- `dist[0] = 0`
- `dist[1] = 7`
- `dist[2] = 9`
- `dist[3] = 20`
- `dist[4] = 20`
- `dist[5] = 11`

并且到顶点 `4` 的一条最短路径为 `0 -> 2 -> 5 -> 4`。

## R14

实现不变量：
- 任何已 `settled` 的顶点，其 `dist` 不再被改写；
- 任意时刻 `dist[v]` 是“当前已发现路径”的最小上界；
- 松弛操作只会使 `dist` 下降，不会上升；
- 由 `prev` 追溯形成的路径长度与 `dist[target]` 一致。

## R15

边界与异常策略：
- `num_nodes == 0`：返回空数组；
- `source` 越界：抛 `ValueError`；
- 边端点越界：抛 `ValueError`；
- 存在负边权：抛 `ValueError`；
- 目标点不可达：`reconstruct_path` 返回空路径 `[]`。

## R18

`demo.py` 源码级流程拆解（8 步）：
1. `main` 定义基础无向边并扩展为双向有向边列表，准备源点 `0`。  
2. 调用 `dijkstra_shortest_paths`：先在 `build_adjacency` 中检查端点与非负权，再构建邻接表。  
3. 初始化 `dist`、`prev`、`settled` 与最小堆 `[(0.0, source)]`。  
4. 循环弹出堆顶候选 `(du, u)`；若 `u` 已定型则跳过（惰性删除过期项）。  
5. 对 `u` 的每条出边执行松弛：若 `du + w < dist[v]`，就更新 `dist[v]`、`prev[v]` 并把新候选压入堆。  
6. 堆清空后返回 `dist/prev`；`main` 用 `reconstruct_path` 还原 `0 -> 4` 路径并做断言。  
7. `main` 再生成 3 组固定随机图，分别计算 Dijkstra 与 `bellman_ford_reference`，逐点比较距离一致性。  
8. 最后执行异常分支：不可达点路径应为空、负权边输入应抛 `ValueError`，全部断言通过后打印 `All assertions passed.`。  

本 MVP 没有把第三方最短路函数当黑盒；核心贪心流程（选点、定型、松弛）全部在源码中可逐行追踪。
