# 拉曼光谱 (Raman Spectroscopy)

- UID: `PHYS-0462`

## R01

拉曼光谱通过测量样品对激发光的非弹性散射，得到强度 `I(ν)` 随拉曼位移 `ν`（单位 `cm^-1`）的曲线，用于识别分子振动模并进行定性/定量分析。

在定量任务里，核心目标是：

- 从含噪声、含荧光背景、含干扰组分重叠峰的光谱中，估计目标组分浓度 `c_target`。

## R02

本 MVP 聚焦一个常见工程问题：

1. 输入：一批拉曼光谱 `S(ν)`；
2. 输出：每条光谱对应的目标浓度 `c_target`；
3. 难点：
   - 荧光背景抬升导致峰高失真；
   - 宇宙射线尖峰（cosmic spikes）导致局部异常；
   - 干扰物峰与目标峰重叠，单峰法易偏差。

## R03

建模范围与假设：

- 数据使用可控的合成拉曼谱验证算法流程，不直接声称实验级绝对精度；
- 每条谱由“目标组分 + 干扰组分 + 背景 + 噪声 + 尖峰”构成；
- 目标是演示可复现的端到端算法链：预处理、选峰、回归、验证。

不覆盖：

- 仪器响应函数反卷积；
- 偏振拉曼/共振拉曼等专门实验配置；
- 真实样本复杂化学先验库拟合。

## R04

`demo.py` 中使用的主要数学形式：

- 混合谱模型：
  `S(ν) = c_t * R_t(ν) + c_i * R_i(ν) + B(ν) + N(ν) + C(ν)`
- `R_t, R_i`：目标/干扰参考峰形（高斯峰叠加）；
- `B(ν)`：低频基线（多项式 + 荧光指数项）；
- `N(ν)`：高斯噪声；
- `C(ν)`：稀疏尖峰噪声。

预处理使用 Asymmetric Least Squares (AsLS) 基线估计：

- `min_z Σ w_i(y_i-z_i)^2 + λ Σ (Δ^2 z_i)^2`
- 迭代更新权重：`w_i = p`（当 `y_i > z_i`）否则 `1-p`。

## R05

脚本输出内容：

- 自动选择的分析峰位 `ν*`；
- 三类模型在 ID 与 shifted 测试集上的 `MAE / RMSE / R2`；
- shifted 集上的样例预测对照（真值与各模型预测）；
- 阈值检查与 `Validation: PASS/FAIL`。

## R06

实现策略：

- `simulate_raman_dataset`：生成含背景漂移、峰位轻微漂移、噪声与尖峰的合成光谱；
- `despike_spectrum`：中值滤波 + MAD 阈值剔除尖峰；
- `asls_baseline`：显式实现 AsLS 基线校正，不交给黑盒；
- `select_analytical_peak`：按“目标峰高/干扰峰高”比值自动选最稳健分析峰；
- 建模对比：
  - 单峰线性标定（可解释基线）；
  - `StandardScaler + Ridge`（全谱线性多变量）；
  - 小型 PyTorch MLP（全谱非线性拟合）。

## R07

优点：

- 端到端流程完整，覆盖拉曼定量的关键数值步骤；
- 物理可解释模型与数据驱动模型并列，便于横向比较；
- 所有关键算法步骤都可在源码追踪。

局限：

- 合成数据无法代表全部真实仪器和样本域偏移；
- 未实现多组分反演置信区间估计；
- MLP 解释性弱于单峰标定与 Ridge。

## R09

设：

- 样本数为 `N`；
- 光谱点数为 `M`；
- AsLS 迭代次数为 `K`；
- MLP 训练轮数为 `E`，隐藏宽度为 `H`。

复杂度（主量级）：

- 预处理（去尖峰 + AsLS + 平滑）：约 `O(N * K * M)`（含稀疏线性方程求解）；
- 单峰模型训练/预测：`O(N)`；
- Ridge 训练：约 `O(N*M^2 + M^3)`（数值实现相关）；
- MLP 训练：约 `O(E * N * M * H)`。

## R10

数值稳定性措施：

- MAD 加 `1e-12` 下界，避免尖峰检测零除；
- AsLS 使用稀疏矩阵并限制迭代次数，避免不必要振荡；
- 峰选择分母加小常数，避免干扰谱极小值导致比值爆炸；
- 回归预测做 `clip(min=0)`，避免出现负浓度。

## R11

本 MVP 默认参数（见 `demo.py`）：

- 拉曼位移范围：`200 ~ 1998 cm^-1`，步长 `2 cm^-1`；
- 数据规模：训练 `240`，ID 测试 `80`，shifted 测试 `80`；
- AsLS：`lambda=2e5, p=0.01, n_iter=7`；
- Savitzky-Golay：窗口 `11`，阶次 `3`；
- Ridge：`alpha=1.0`；
- MLP：两层隐藏层 `128/64`，`epochs=220`，学习率 `8e-4`。

## R12

脚本内置阈值检查：

1. Ridge 在 ID 测试集 `R2 >= 0.90`；
2. MLP 在 ID 测试集 `R2 >= 0.88`；
3. Ridge 在 shifted 测试集 `R2 >= 0.70`。

三项都满足则输出 `Validation: PASS`，否则输出 `FAIL` 并以非零状态退出。

## R13

理论与保证边界：

- 单峰线性标定在峰不重叠、背景处理充分时具有较好可解释性；
- Ridge 对二次损失 + `L2` 正则是凸问题，存在唯一全局最优（`alpha > 0`）；
- MLP 训练为非凸优化，不保证全局最优；
- 本实现保证“流程可复现 + 阈值可检验”，不保证对所有真实实验数据都达同样精度。

## R14

常见失效模式与应对：

1. 失效：荧光背景极强，峰被淹没。  
   应对：提高基线模型灵活度，或引入更强先验约束。
2. 失效：峰位漂移超过选峰窗口。  
   应对：先做波数校准/动态对齐，再做定量。
3. 失效：干扰物谱型与目标高度重叠。  
   应对：改用多组分联合建模或引入化学先验。
4. 失效：训练分布覆盖不足。  
   应对：扩充标定集并保留 shifted 验证集作为上线门槛。

## R18

`demo.py` 的源码级算法流程（8 步）：

1. `build_reference_profiles` 构建目标组分与干扰组分的参考拉曼峰形（多高斯峰叠加）。  
2. `simulate_raman_dataset` 依据 `S(ν)=c_tR_t+c_iR_i+B+N+C` 生成训练/测试光谱，显式注入荧光背景、峰位漂移、噪声与尖峰。  
3. `despike_spectrum` 通过中值滤波与 MAD 阈值检测替换尖峰，抑制 cosmic spikes。  
4. `asls_baseline` 用稀疏线性方程迭代求解 AsLS 基线；`preprocess_spectra` 逐条完成去尖峰、扣基线、Savitzky-Golay 平滑。  
5. `select_analytical_peak` 在目标参考谱上 `find_peaks`，并按 `target/interferent` 分离度选分析峰 `ν*`。  
6. `fit_single_peak_calibration` 在 `ν*` 处对训练集做线性标定，得到可解释单峰浓度估计器。  
7. `train_ridge_model` 与 `train_torch_mlp` 分别训练全谱线性模型和非线性模型，`predict_torch_mlp` 输出浓度预测。  
8. `main` 统一计算 `MAE/RMSE/R2`、执行阈值验收并打印 `PASS/FAIL`。

第三方库职责拆解：

- `numpy`：谱生成、线性代数与向量化计算；
- `scipy`：`find_peaks`、`savgol_filter`、稀疏矩阵与 `spsolve`；
- `pandas`：结果表格化输出；
- `scikit-learn`：`StandardScaler` 与 `Ridge`；
- `torch`：小型 MLP 训练与推理。

核心算法步骤（合成、去尖峰、AsLS、选峰、标定、评估）都在源码显式实现，没有把任务留给黑盒一键函数。
