# 扩展欧几里得算法 (Extended Euclidean Algorithm)

- UID: `MATH-0002`
- 学科: `数学`
- 分类: `数论`
- 源序号: `2`
- 目标目录: `Algorithms/数学-数论-0002-扩展欧几里得算法_(Extended_Euclidean_Algorithm)`

## R01

扩展欧几里得算法是在欧几里得算法基础上的增强版：在求 `gcd(a, b)` 的同时，构造整数解 `x, y`，使得
`a*x + b*y = gcd(a, b)`（Bézout 等式）。

- 输入: 两个整数 `a, b`（允许负数，通常不同时为 0）。
- 输出: 三元组 `(g, x, y)`，其中 `g = gcd(a, b)` 且 `a*x + b*y = g`。
- 主要机制:
  - 先执行辗转相除得到余数链。
  - 再通过回代（或迭代维护系数）把 `g` 表示成 `a,b` 的线性组合。
- 常见派生能力: 模逆元计算、线性丢番图方程求解、CRT 合并步骤。

## R02

时间脉络可分为两层：

- 欧几里得算法本体约见于公元前 300 年左右《几何原本》。
- “扩展”形式（显式追踪 Bézout 系数）是后世数论中对该过程的标准化整理，没有单一现代“首创年份”，在近代代数学教材中被系统固定。

## R03

提出背景是“只知道最大公因数不够用”：

- 仅有 `gcd(a, b)` 不能直接回答 `ax + by = c` 是否可解，也不能直接给出解。
- 模运算里要算 `a^{-1} mod m`，本质需要求 `a*x + m*y = 1` 的整数 `x`。
- 中国剩余定理的构造法也需要快速求逆元与线性组合系数。

因此，扩展欧几里得算法把“求 gcd”升级为“求 gcd + 构造系数”，解决了从判定到构造的一整类基础问题。

## R04

在普通整数算术模型下，时间复杂度与欧几里得算法一致：

- 最坏与典型复杂度: `O(log min(|a|, |b|))` 次迭代（按除法步数计）。
- 每一步包含一次整除和常数次加减乘，故整体渐近同阶。
- 若考虑大整数位复杂度，总成本约为“若干次大整数除法”的累积，通常写作与 `M(n)`（乘法复杂度）相关的准线性或超线性表达。

## R05

空间复杂度：

- 迭代写法只维护常数个整数变量（余数与系数），额外空间 `O(1)`（不计输入与大整数对象本身）。
- 递归写法额外需要调用栈 `O(log min(|a|, |b|))`。
- 在大整数实现里，临时对象分配会引入实际内存峰值波动，但渐近仍由常数个大整数主导。

## R06

示例：`a = 240, b = 46`

辗转相除：

- `240 = 46*5 + 10`
- `46 = 10*4 + 6`
- `10 = 6*1 + 4`
- `6 = 4*1 + 2`
- `4 = 2*2 + 0`，故 `gcd = 2`

回代得到：

- `2 = 47*46 - 9*240`
- 所以一组系数为 `x = -9, y = 47`
- 验证：`240*(-9) + 46*47 = 2`

## R07

算法意义：

- 理论上，它把“整除结构”与“线性表示”连接起来，是初等数论与抽象代数中的基础工具。
- 工程上，它是模逆元、CRT、RSA 密钥流程、同余方程求解等模块的核心子程序。
- 教学上，它展示了“不变量 + 递推/回代”这一类算法设计方法，迁移价值高。

## R08

直接依赖与路径关系：

- 除法算法（Division Algorithm）：`a = bq + r`。
- 欧几里得算法：通过余数递减求 `gcd`。
- Bézout 定理：`gcd(a,b)` 可表示为 `ax + by`。

与 R16 一致性：R08 描述“前置依赖”，R16 描述“后续扩展与应用”。

## R09

适用前提与边界：

- 输入需为整数域元素（典型是 `int`）。
- 不建议输入 `(0,0)`：`gcd(0,0)` 约定不统一，且系数不唯一无实用意义。
- 若用于模逆元，必须满足 `gcd(a, m) = 1` 且通常要求 `m > 1`。
- 对负数输入也可工作，但实现需统一 `gcd` 非负、系数符号回填规则。

## R10

正确性依据（迭代不变量思路）：

- 在每一轮维护：
  - `old_r = a*old_s + b*old_t`
  - `r = a*s + b*t`
- 执行 `q = old_r // r` 后更新：
  - `new_r = old_r - q*r`
  - `new_s = old_s - q*s`
  - `new_t = old_t - q*t`
- 上述更新保持线性表示不变量成立。
- 当 `r = 0` 时，`old_r` 即 `gcd(a,b)`，对应 `old_s, old_t` 即 Bézout 系数。

## R11

数值稳定性与误差传播：

- 算法在整数环上进行，理论上不存在浮点舍入误差传播问题。
- 主要风险来自实现层面：
  - 固定宽度整数（如 C/C++ `int64`）可能溢出。
  - 语言对整除/取余的负数语义不一致会导致符号差异。
- 在 Python 大整数环境下，可避免溢出，但要关注性能和大数分配开销。

## R12

真实成本（超越渐近）：

- 常数项主要在“整数除法”而非加减法。
- 大整数场景中，除法的字长相关成本占主导。
- 递归版有函数调用开销；迭代版通常更快、更省栈。
- 与 I/O、缓存、GPU 并行关系不大：该算法通常是短小串行内核，向量化收益有限。

## R13

该算法是精确确定性算法，不是近似/随机算法：

- 误差界：理论误差为 0（整数精确运算语义下）。
- 概率保证：不涉及随机成功率，输出满足严格代数等式。
- 可保证：
  - 输出 `g = gcd(a,b)`；
  - 输出 `x,y` 满足 `ax + by = g`；
  - 当 `g | c` 时，可构造 `ax + by = c` 的解。

## R14

鲁棒性与失效模式：

- 失效输入：`(0,0)` 需要显式拒绝或单独约定。
- 业务失效：请求模逆元时若 `gcd(a,m) != 1`，逆元不存在。
- 实现失效：
  - 忘记处理负数符号回填导致验证式不成立。
  - 在定宽整数语言中溢出导致错误结果。
- 防护建议：输入校验 + 结果断言 `a*x + b*y == g`。

## R15

工程实现要点：

- 优先采用迭代写法，避免深递归和栈限制。
- 统一约定输出 `g >= 0`，并对负输入做符号回填。
- 封装三个小接口最实用：
  - `extended_gcd(a,b)` 返回 `(g,x,y)`；
  - `mod_inverse(a,m)`；
  - `solve_linear_diophantine(a,b,c)`。
- 给关键路径加自检断言，防止维护中破坏不变量。

## R16

前驱/后继与应用：

- 前驱基础：欧几里得算法、整除理论、Bézout 定理（对应 R08）。
- 后继扩展：
  - 中国剩余定理构造解；
  - RSA 中私钥参数与同余方程处理；
  - 一般线性丢番图方程求解器。
- 典型应用场景：密码学库、计算机代数系统、竞赛与工程中的模运算内核。

- 工具栈：仅 Python 标准库（`math`、`typing`），保持最小可运行。

## R18

源码追踪与算法拆解（对应本目录 `demo.py` 的实现流）：

1. 输入校验：若 `a=b=0`，直接报错，避免不定情形。
2. 状态初始化：`(old_r, r)` 跟踪余数链；`(old_s,s)`、`(old_t,t)` 跟踪线性系数。
3. 迭代取商：计算 `q = old_r // r`，对应当前辗转相除步。
4. 余数推进：更新 `old_r, r <- r, old_r - q*r`，保证余数严格下降到 0。
5. 系数同步推进：用同一 `q` 更新 `s,t` 两组系数，保持 Bézout 线性表示不变量。
6. 终止取值：当 `r=0` 时，`old_r` 即 gcd，`old_s,old_t` 即相对 `|a|,|b|` 的系数。
7. 符号回填：根据原始 `a,b` 符号调整 `x,y`，得到满足原输入的 `ax+by=g`。
8. 上层复用：`mod_inverse` 与丢番图求解仅复用 `(g,x,y)` 并做可解性判定。

这不是第三方黑箱调用，而是源码级显式实现；步骤与数学推导一一对应，可直接审计与测试。
