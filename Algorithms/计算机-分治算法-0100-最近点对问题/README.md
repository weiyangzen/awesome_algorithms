# 最近点对问题

- UID: `CS-0080`
- 学科: `计算机`
- 分类: `分治算法`
- 源序号: `100`
- 目标目录: `Algorithms/计算机-分治算法-0100-最近点对问题`

## R01

最近点对问题（Closest Pair of Points）要求在平面点集中找出欧氏距离最小的两点。

- 输入: `n` 个二维点 `P={(x_i,y_i)}`，`n >= 2`。
- 输出: 最小距离 `d*` 及其对应点对。
- 目标: 从朴素 `O(n^2)` 枚举，优化到分治 `O(n log n)`。

## R02

- 经典归属: 计算几何中的基础问题，常见于算法教材与课程。
- 历史脉络: 20 世纪中后期，随着分治与几何算法体系化，该问题成为标准范例。
- 方法地位: “按 `x` 递归划分 + 中间条带检查”的结构，是分治几何算法的代表模板。

## R03

问题动机是“规模放大后的可计算性”。

- 朴素方法需要检查 `n(n-1)/2` 对点，时间 `O(n^2)`。
- 在点数达到 `10^5` 级别时，二次复杂度通常不可接受。
- 分治法利用几何约束，把跨分区候选点限制在窄条带中，并在条带内仅检查常数个邻点，从而把总体复杂度降为 `O(n log n)`。

## R04

朴素基线（用于正确性对照）:

1. 双循环枚举每个点对 `(i,j)`。
2. 计算欧氏距离并维护全局最小值。

复杂度:

- 时间: `O(n^2)`
- 空间: `O(1)`（不计输入）

优势是实现最直接，缺点是无法应对大规模输入。

## R05

分治算法核心结构:

1. 将点按 `x` 坐标排序，取中线分为左右子集。
2. 递归求左、右子问题最小距离 `dL`、`dR`，取 `d=min(dL,dR)`。
3. 构造中线附近宽度 `2d` 的条带 `strip`（`|x-mid_x| < d`）。
4. 将 `strip` 按 `y` 有序扫描；对每个点只需检查其后最多常数个候选（经典结论可取 7 个）。

递推式:

- `T(n) = 2T(n/2) + O(n)`
- 解为 `T(n)=O(n log n)`

## R06

示例点集:

`[(2,3), (12,30), (40,50), (5,1), (12,10), (3,4)]`

流程概览:

1. 按 `x` 排序并二分。
2. 递归得到左右最小距离。
3. 合并阶段在中线条带中按 `y` 扫描。
4. 发现 `(2,3)` 与 `(3,4)` 的距离为 `sqrt(2)`。
5. 这是全局最小值，因此最近点对为 `((2,3),(3,4))`。

## R07

该问题的算法价值:

- 理论层: 体现“分治 + 几何打包性质”如何把二次问题降到 `n log n`。
- 工程层: 可直接用于聚类预处理、空间索引验证、轨迹异常检测等。
- 教学层: 是证明“局部候选常数上界”思想的经典入口。

## R08

前置知识:

- 平面几何与欧氏距离。
- 分治递归与复杂度递推分析。
- 排序与稳定扫描（按 `x`、按 `y` 的双序列维护）。
- 基础正确性证明方法（归纳 + 不变式）。

## R09

适用条件与边界:

- 点维度: 本 MVP 针对二维欧氏距离。
- 数据类型: 支持 `int/float` 坐标。
- 重复点: 合法；若存在重复点，最小距离可为 `0`。
- 输入规模: `n < 2` 时问题无定义，接口应抛出异常。

## R10

正确性证明要点（简述）:

1. 递归基: `n<=3` 时暴力枚举，结果正确。
2. 归纳假设: 左右子问题都能正确返回其最近点对距离。
3. 全局最优点对只有三类:
   - 完全在左侧
   - 完全在右侧
   - 跨中线
4. 前两类由递归结果覆盖；第三类必在宽度 `2d` 条带内。
5. 条带中按 `y` 扫描时，每点仅需比较常数个后继点，不会漏掉跨线最优解。
6. 因此合并后得到的最小值即全局最优。

## R11

数值稳定性与误差:

- 本实现内部比较采用“平方距离”，避免频繁 `sqrt` 带来的额外浮点开销。
- 最终展示时再开平方得到真实距离。
- 浮点输入下可能出现极小舍入误差，测试中使用 `np.isclose` 进行容差比较。

## R12

工程成本（超越大 O）:

- 每层需要维护按 `y` 的序列划分，常数实现细节影响性能。
- Python 递归调用有函数开销；中小规模通常可接受。
- 若点数极大，需考虑:
  - 迭代化或 C/NumPy 加速
  - 并行化分治
  - 外存/流式数据处理

## R13

- 近似比/随机保证: N/A。
- 原因: 最近点对分治法是确定性精确算法，不是近似算法。

## R14

常见失效模式与防护:

- 忘记在条带内按 `y` 顺序检查，导致复杂度退化。
- 条带候选比较上界写错，可能漏解或多做无效比较。
- 未区分点身份（仅按坐标去重）会在重复坐标场景出错。
- `n<2` 不做输入检查会产生无意义结果。

## R15

实现建议:

- 在点结构中保留唯一 `idx`，避免重复坐标导致集合划分歧义。
- 内部统一用平方距离比较，最后再 `sqrt`。
- 保留朴素 `O(n^2)` 实现用于小规模对拍验证。
- 测试至少覆盖:
  - 重复点（距离应为 0）
  - 随机点
  - 浮点坐标
  - 最小输入规模边界

## R16

前驱与后继关系:

- 前驱基线: 朴素两两枚举 `O(n^2)`。
- 相关数据结构方法:
  - k-d tree 最近邻查询（多次查询场景常见）
  - Delaunay 三角剖分（理论上最近点对是其边）
- 高维扩展: 维度升高后常数迅速增大，需改用近似最近邻或索引结构。

## R17

本目录 `demo.py` 的 MVP 设计:

- 依赖: `numpy`（随机样本与数值比较）；核心算法不依赖黑箱几何库。
- 核心函数:
  - `closest_pair_divide_conquer(points_xy)`: `O(n log n)` 主算法。
  - `closest_pair_naive(points_xy)`: `O(n^2)` 基线算法。
  - `_closest_pair_rec(px, py)`: 递归分治 + 条带合并。
- 运行内容:
  - 固定样例验证（含已知答案）。
  - 重复点案例验证（最短距离应为 `0`）。
  - 随机对拍（分治 vs 朴素）。
  - 小规模性能对比输出。
- 运行命令:
  - `uv run python demo.py`

## R18

`demo.py` 源码级算法流（9 步，不依赖第三方黑箱求解）:

1. `main()` 先构造固定样例与随机样例，调用分治与朴素函数进行对照。
2. `closest_pair_divide_conquer()` 将输入点封装为带 `idx` 的内部点结构，并生成按 `x`、按 `y` 排序的两个序列。
3. 进入 `_closest_pair_rec(px, py)`；当子问题点数 `<=3`，调用 `_brute_force_points()` 直接枚举得到局部最优。
4. 对一般情形按中点将 `px` 切成左右两半，利用左半 `idx` 集把 `py` 线性划分为 `left_y/right_y`。
5. 递归求左、右最优平方距离 `dL`、`dR`，并初始化 `delta=min(dL,dR)`。
6. 构造中线条带 `strip`（满足 `(x-mid_x)^2 < delta`），且 `strip` 保持按 `y` 有序。
7. 对 `strip` 中每个点，仅检查其后最多 7 个点；若纵向差的平方已 `>= delta` 则提前停止该点的后续比较。
8. 合并阶段若发现更小平方距离则更新 `delta` 和对应点对，递归返回到顶层得到全局最优点对。
9. `closest_pair_divide_conquer()` 把平方距离开方得到最终距离；`main()` 用 `closest_pair_naive()` 做随机对拍并打印结果摘要。
