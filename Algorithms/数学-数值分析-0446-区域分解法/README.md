## R01

区域分解法（Domain Decomposition Method, DDM）把一个大区域上的方程拆成多个子区域上的子问题，再通过边界/接口信息交换迭代得到全局解。

它的核心价值是：
- 大问题分小问题，局部求解更便宜；
- 子问题天然适合并行；
- 可作为迭代求解器本体，也可作为预条件器。

## R02

本条目采用最小可运行的基准问题：

- 连续方程：`-u''(x) = pi^2 sin(pi x), x in (0,1)`
- 边界条件：`u(0)=u(1)=0`
- 连续精确解：`u(x)=sin(pi x)`

选该问题是因为有解析解，便于同时验证：
- 区域分解迭代是否收敛到离散方程真解；
- 离散方程真解与连续精确解的离散化误差关系。

## R03

离散化方式（1D 二阶中心差分）：

- 内点数：`n`
- 网格步长：`h = 1/(n+1)`
- 线性系统：`A u = b`
- 其中：
  - `A` 为三对角矩阵，主对角 `2/h^2`，副对角 `-1/h^2`
  - `b_i = pi^2 sin(pi x_i)`，`x_i = i h`

这为区域分解迭代提供了清晰、可审计的代数目标。

## R04

子域划分策略（本 MVP）：

- 两个重叠子域；
- 若内点索引是 `0..n-1`，中点 `mid=n//2`，重叠宽度 `overlap`；
- 子域 1：`[0, mid+overlap-1]`
- 子域 2：`[mid-overlap, n-1]`

要求 `overlap >= 1`，这样两子域在中间有公共区，保证 Schwarz 迭代可有效传递信息。

## R05

迭代算法采用 **重叠乘法 Schwarz（Multiplicative Schwarz）**：

1. 先在子域 1 解局部问题并回写全局向量；
2. 再在子域 2 解局部问题（使用更新后的接口值）并回写；
3. 以上两步构成一次 Schwarz 迭代。

局部问题边界条件：
- 物理边界处用真实 Dirichlet（本题为 0）；
- 人工接口处用当前全局迭代向量上的邻点值作为 Dirichlet 条件。

## R06

局部线性系统仍是三对角结构。

`demo.py` 没有把局部求解交给黑箱 PDE 包，而是显式实现了 Thomas 三对角消元：
- 前向消元构造修正系数；
- 回代得到局部解。

这样可以直接看到“区域分解”和“局部线性代数求解”两个层面的算法细节。

## R07

收敛直觉：

- 子域重叠越充分，接口信息传播越快；
- 乘法 Schwarz（顺序更新）通常比加法 Schwarz（并行再混合）收敛更快；
- 对本类 1D 椭圆型问题，重叠 Schwarz 是稳定且经典的基线方法。

## R08

时间复杂度（本实现）设：
- 全局内点 `n`
- 迭代轮数 `k`
- 子域长度约 `m1, m2`

则单轮成本约为两个三对角求解：`O(m1 + m2)`，量级可视为 `O(n)`；
总成本约 `O(k n)`。

这也是 DDM 在大规模问题上常见的优势来源：每次只做局部线性工作。

## R09

空间复杂度：

- 全局解向量与 RHS：`O(n)`；
- 两个局部三对角系数与临时向量：`O(m1+m2)`，同阶 `O(n)`；
- 迭代历史表：`O(k)`。

整体空间复杂度为 `O(n+k)`，在固定 `k` 下近似 `O(n)`。

## R10

关键参数含义：

- `n_interior`：离散分辨率，越大离散误差越小，但线性系统更大。
- `overlap`：子域重叠宽度，通常越大迭代次数越少，但单轮子问题更大。
- `tolerance`：残差停止阈值，越小代表更严格的离散求解精度。
- `max_iterations`：安全上限，防止异常参数导致无限循环。

## R11

本目录的正确性检查分三层：

1. **代数层**：计算 `relative_residual = ||A u - b|| / ||b||`，验证离散方程是否被满足。  
2. **离散真解层**：用同一三对角求解器直接解全局 `A u*=b`，比较 `||u-u*||/||u*||`。  
3. **连续参考层**：比较 `||u-u_exact||/||u_exact||`，观察离散化误差量级。  

`demo.py` 内置断言保证前两层达到阈值。

## R12

稳定性与常见失效模式：

- 若 `overlap=0` 或过小，接口信息传播慢，收敛显著变差；
- 若子域切分越界或不重叠，算法前提不成立；
- 若参数设置导致迭代不足，残差会停留在较高量级；
- 若矩阵装配错误（例如系数符号错误），残差会异常震荡或不降。

本实现对参数合法性和零主元都做了防御性检查。

## R13

运行输出会打印每次迭代的三列核心指标：

- `rel_residual`
- `rel_err_discrete`
- `rel_err_exact`

典型趋势是：
- 前两者快速下降到很小量级；
- `rel_err_exact` 最终停在离散化误差附近（不会无限趋零）。

这正是“离散求解误差”和“模型离散误差”分离后的标准行为。

## R14

本 MVP 的边界：

- 仅 1D Poisson；
- 仅两子域、固定重叠；
- 仅乘法 Schwarz；
- 未实现并行通信、粗网格校正、Krylov 加速。

因此它是算法骨架演示，不是高性能并行 DDM 工程框架。

## R15

可扩展方向：

- 从 1D 推广到 2D/3D（有限差分/有限元）；
- 从两子域扩展到多子域并行；
- 将 Schwarz 作为 GMRES/CG 预条件器；
- 引入双层方法（Two-level Schwarz）改善强扩展性；
- 针对非重叠情形引入 Schur 补、FETI、BDDC 等接口方法。

## R16

与相近方法对比：

- 对比全局直接法：
  - 直接法迭代次数少但全局耦合重；
  - DDM 强调局部求解与并行组织。  
- 对比 Jacobi/Gauss-Seidel 点迭代：
  - 点迭代交换粒度细；
  - DDM 一次交换的是“子域级信息”，对大规模 PDE 更自然。  
- 对比黑箱库调用：
  - 本条目显式实现了子域拆分、接口处理、局部三对角求解和收敛判定，便于审计与教学。

## R18

`demo.py` 源码级算法流程（9 步）：

1. `DDConfig.validate` 检查 `n_interior / overlap / tolerance / max_iterations` 合法性。  
2. `build_problem` 构造网格、右端 `b` 和连续参考解 `u_exact`。  
3. `split_into_two_overlapping_subdomains` 根据 `overlap` 生成两个重叠索引区间。  
4. `solve_discrete_poisson_exact` 通过三对角求解得到全局离散真解 `u_discrete_exact`（用于验证，不参与 Schwarz 更新）。  
5. 初始化全局迭代向量 `u=0`，进入迭代循环。  
6. 调用 `solve_local_subproblem` 在子域 1 组装局部 RHS（含接口 Dirichlet 值），再由 `solve_tridiagonal`（Thomas）求局部解并回写。  
7. 同样方式处理子域 2，构成一次乘法 Schwarz 迭代。  
8. 每轮计算三项诊断：`relative_residual`、`relative_error_to_discrete_solution`、`relative_error_to_exact_solution`，记录到 `history`，达到阈值则停止。  
9. `print_report` 打印迭代表，`run_checks` 断言收敛和数值有效性，最后输出通过信息。  

以上流程完整展开了区域分解求解链路，没有把关键步骤留给黑箱接口。
