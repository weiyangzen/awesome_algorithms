## R01

锥规划（Conic Programming）是带有“变量属于凸锥”约束的一类凸优化问题。标准形式可写为：

`min c^T x`

`s.t. Ax = b, x in K`

其中 `K` 是凸锥（如非负正交锥、二阶锥、半正定锥）。
本目录实现一个最小可运行 SOCP（Second-Order Cone Program）示例，重点是：
- 明确“锥约束”不是黑箱；
- 通过显式二阶锥投影实现可解释求解；
- 保留可审计迭代轨迹。

## R02

`demo.py` 解决的具体问题：

`min_z  0.5 * ||M z - d||_2^2 + g^T z`

`s.t.  z in Q^n = {(t, u) | ||u||_2 <= t}`

其中 `z=[t;u]`（第一维是 `t`，其余维组成向量 `u`）。
输出包括：
- 迭代中的目标函数值；
- 更新量范数 `||dz||`；
- 投影梯度残差（一阶最优性指标）；
- 锥可行性残差 `max(||u||-t, 0)`；
- 最终解与参考可行点对比。

## R03

选择该问题与求解策略的原因：
- 二阶锥是锥规划里最常见、最容易做几何解释的一类约束；
- 该目标函数光滑凸，配合“投影到 SOC”可用投影梯度法直接求解；
- 相比直接调用现成求解器（几行函数调用），手写投影和迭代更适合作为算法教学 MVP。

## R04

核心数学组件：

1. 目标函数与梯度

`f(z) = 0.5||Mz-d||_2^2 + g^T z`

`grad f(z) = M^T(Mz-d) + g`

2. 梯度 Lipschitz 常数

`L = ||M||_2^2`

取步长 `eta = step_scale / L`（`0 < step_scale <= 1`）。

3. 二阶锥投影（闭式公式）

给定 `v=(t,u)`：
- 若 `||u|| <= t`，`Proj_Q(v)=v`
- 若 `||u|| <= -t`，`Proj_Q(v)=0`
- 否则 `Proj_Q(v)=(a, a/||u|| * u)`，`a=(||u||+t)/2`

4. 迭代更新

`z_{k+1} = Proj_Q(z_k - eta * grad f(z_k))`

## R05

算法流程（高层）：

1. 校验 `M/d/g` 维度、有限值与参数合法性。
2. 计算 `L=||M||_2^2` 和步长 `eta`。
3. 从可行点 `z0=0` 开始（原点总在 SOC 内）。
4. 每轮做一次“梯度步 + SOC 投影”得到 `z_{k+1}`。
5. 记录 `objective`、`||dz||`、`pg_res`、`feas_gap`。
6. 当 `||dz|| < tol` 且 `pg_res < 5*tol` 时停止。
7. 达到 `max_iters` 仍不满足则标记未收敛。

## R06

正确性与收敛依据（MVP 层面）：
- 目标函数是凸二次函数，SOC 约束集合是闭凸集；
- 投影梯度法（Forward-Backward）在 `eta <= 1/L` 时对该类问题收敛；
- 终止时使用投影梯度残差 `||G_eta(z)||` 作为一阶最优性近似证据；
- `feas_gap` 持续为 0（数值误差内）保证锥可行性。

## R07

复杂度分析：
- 记观测数 `m`、变量维度 `n`、迭代轮数 `T`；
- 每轮主要成本在 `Mz` 与 `M^T r`，时间复杂度 `O(mn)`；
- 总时间复杂度 `O(Tmn)`；
- 空间复杂度 `O(mn)`（存储 `M`）与 `O(n)`（向量状态）。

## R08

边界与异常处理：
- `M` 必须是 2D 且列数至少 2（需要 `t+u` 结构）；
- `d/g` 必须是一维并与 `M` 维度匹配；
- 输入不能含 `nan/inf`；
- `tol<=0`、`max_iters<=0`、`step_scale` 不在 `(0,1]` 会报错；
- 若出现非有限目标值，运行时抛出异常终止。

## R09

MVP 取舍：
- 仅使用 `numpy`，避免大规模依赖；
- 不实现内点法、ADMM、障碍函数等高阶策略；
- 不调用外部 conic solver 黑箱；
- 优先“可解释 + 可运行 + 可审计”的小实现。

## R10

`demo.py` 结构说明：
- `validate_problem`：数据与维度检查。
- `project_soc`：SOC 投影闭式实现。
- `objective` / `gradient`：目标函数与梯度。
- `lipschitz_constant`：计算 `||M||_2^2`。
- `projected_gradient_mapping`：投影梯度残差。
- `solve_socp_pgd`：求解主循环。
- `make_synthetic_socp`：构造可复现实验。
- `objective_monotone_check` / `print_history`：结果审计与展示。
- `main`：一键运行与汇总输出。

## R12

输出字段说明：
- `iter`：迭代编号。
- `objective`：当前目标函数值。
- `||dz||`：连续两次迭代解的差值范数。
- `pg_res`：投影梯度残差（越小越接近 KKT 一阶条件）。
- `feas_gap`：SOC 可行性残差（0 表示可行）。
- `converged`：是否满足停止准则。
- `distance to reference feasible point`：与构造参考可行点的距离（仅作对照，不代表全局误差上界）。

## R14

关键参数与建议：
- `step_scale`：步长比例，默认 `0.99`，越接近 1 越激进。
- `tol`：收敛阈值，默认 `1e-8`。
- `max_iters`：最大迭代轮数，默认 `4000`。
- `noise_std`：数据噪声强度，影响问题难度与最优解位置。

调参建议：
- 若收敛慢，先增大 `max_iters`；
- 若震荡或单调性变差，减小 `step_scale`；
- 若只做快速演示，可放宽 `tol` 到 `1e-6`。

## R15

与其他锥规划方法对比：
- 内点法：高精度、迭代次数少，但单次迭代代价高、实现复杂；
- ADMM/分裂法：易并行、适合大规模，但参数敏感；
- 本实现（投影梯度）：实现最简单、可解释性强，适合教学和小规模基线。

## R16

典型应用场景（SOCP 视角）：
- 鲁棒回归与鲁棒约束建模；
- 受二范数约束的信号处理问题；
- 组合优化连续松弛中的锥约束子问题；
- 作为更复杂锥求解器前的教学/原型验证。

## R18

`demo.py` 源码级算法流程（8 步）：

1. `main` 调用 `make_synthetic_socp` 生成 `M, d, g` 和参考可行点 `z_ref`。
2. `solve_socp_pgd` 先用 `validate_problem` 完成维度与数值合法性检查。
3. 通过 `lipschitz_constant` 计算 `L=||M||_2^2`，设置固定步长 `eta=step_scale/L`。
4. 以 `z=0` 作为初始可行点，并记录初始目标值与投影梯度残差。
5. 每次迭代计算梯度 `grad f(z)=M^T(Mz-d)+g`，先做前向梯度步 `z-eta*grad`。
6. 对梯度步结果调用 `project_soc` 做闭式二阶锥投影，得到新解 `z_next`。
7. 记录 `objective`、`||dz||`、`pg_res`、`feas_gap`，并用阈值判定是否收敛。
8. 返回结果后，`main` 打印轨迹摘要、单调性检查和可行性检查，形成完整审计输出。
