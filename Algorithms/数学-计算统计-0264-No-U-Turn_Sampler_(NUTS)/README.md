## R01

No-U-Turn Sampler（NUTS）是 Hamiltonian Monte Carlo（HMC）的自适应路径长度版本。  
它通过“何时停止沿哈密顿轨迹前进”的自动判定，避免手工调节 HMC 里最难设定的 `L`（leapfrog 步数）。

本条目给出一个可运行 MVP：
- 从零实现 leapfrog、二叉树扩展、U-turn 停止准则；
- 结合 dual averaging 做 warmup 步长适配；
- 在二维高斯分布上做数值验证（均值/协方差/ESS）。

## R02

NUTS 解决的核心痛点是：
- 传统 HMC 需要同时调 `step_size ε` 和轨迹长度 `L`；
- `L` 太小会随机游走，`L` 太大轨迹会“折返”浪费计算并增大拒绝率。

NUTS 的策略是：
- 不直接固定 `L`；
- 每次采样动态扩展轨迹树；
- 一旦出现“路径开始掉头”（No-U-Turn 条件不满足）就停止扩展。

## R03

问题定义（本目录版本）：
- 输入：
  - 目标分布 `π(q)` 的 `log π(q)` 与梯度 `∇log π(q)`；
  - 采样配置：`num_warmup`、`num_samples`、`max_depth`、`target_accept`。
- 输出：
  - 后验样本矩阵 `samples`；
  - 诊断信息：接受率、树深、发散计数、最终步长；
  - 与已知真值的误差校验（MVP 使用已知均值协方差的高斯）。

## R04

数学建模：
- 位置变量 `q` 为待采样参数；
- 引入动量变量 `p ~ N(0, I)`，联合密度满足
  `log π(q, p) = log π(q) - 1/2 p^T p`。

哈密顿系统：
- 势能 `U(q) = -log π(q)`；
- 动能 `K(p) = 1/2 p^T p`；
- 哈密顿量 `H(q,p) = U(q)+K(p)`。

离散推进采用 leapfrog（速度 Verlet）：
1. `p_{t+1/2} = p_t + (ε/2)∇logπ(q_t)`
2. `q_{t+1} = q_t + ε p_{t+1/2}`
3. `p_{t+1} = p_{t+1/2} + (ε/2)∇logπ(q_{t+1})`

## R05

NUTS 的核心结构是“对称二叉树 + U-turn 判据”：
- 每轮迭代从当前点 `q` 与新动量 `p0` 出发；
- 采样 slice 变量 `log_u`，仅保留能量壳内候选状态；
- 按随机方向（前/后）递归扩树，深度从 `0` 到 `max_depth-1`；
- 边界状态满足
  `dot(q_plus - q_minus, p_minus) >= 0` 且 `dot(q_plus - q_minus, p_plus) >= 0`
  才继续，否则触发 No-U-Turn 停止；
- 在所有有效候选中按权重抽取下一个 `q`。

## R07

优点：
- 相比固定 `L` 的 HMC，NUTS 显著降低轨迹长度调参负担；
- 基于梯度，通常比随机游走型 MCMC 混合更快；
- 本实现无黑盒采样库，关键步骤可直接追踪。

局限：
- 仍需要步长及 warmup 参数，且不同目标分布的稳定区间差异很大；
- 当前实现使用单位质量矩阵 `I`，未做质量矩阵自适应；
- 递归建树在高维复杂后验上可能需要更精细诊断与鲁棒控制。

## R09

适用场景：
- 可提供 `log π(q)` 与梯度的连续参数模型；
- 贝叶斯后验采样原型验证；

不适用场景：

## R10

正确性直觉（MVP 视角）：
1. leapfrog 近似保持辛结构，能量误差可控；
2. slice 变量把采样限制在联合密度阈值之上，维持可逆扩展采样框架；
3. 树扩展同时向前/向后构造候选区间，避免单向偏置；
4. No-U-Turn 判据阻止轨迹反向折返，减少无效探索；
5. 候选按有效状态计数概率选择，配合 Metropolis 权重统计维持不变性。

## R11

数值稳定与鲁棒策略：
- `delta_max` 限制极端能量偏差，标记发散跃迁；
- 步长在 warmup 中用 dual averaging 调整，避免固定步长过激；
- 步长加硬裁剪 `clip([1e-4, 2.0])`，防止异常漂移；
- 使用固定随机种子，保证 demo 可复现实验输出。

## R12

关键参数说明（见 `NutsConfig`）：
- `num_warmup`：步长自适应迭代数；
- `num_samples`：最终保留样本数；
- `max_depth`：树最大深度，单次最多约 `2^depth` leapfrog 节点；
- `target_accept`：dual averaging 的目标接受率（常见 0.65~0.9）；
- `delta_max`：发散阈值；
- `seed`：随机种子。

调参建议：
- 接受率太低时可提高 `target_accept`（使最终步长更保守）；
- 轨迹探索不足时可适当增加 `max_depth`；
- 发散频繁时优先减小步长或提升目标分布重参数化质量。

## R13

理论性质（标准假设下）：
- NUTS 属于 HMC 家族，目标分布 `π(q)` 为其不变分布；
- 样本均值对期望量是一致估计（链足够长、满足遍历条件时）；
- 这里不提供有限样本误差上界，但在 demo 中用已知高斯真值进行经验校验。

## R14

常见失效模式：
1. 步长过大导致发散率高、接受率低；
2. `max_depth` 太小导致轨迹过短、ESS 低；
3. 目标后验几何恶劣（强相关/尺度悬殊）而未做重参数化；
4. 梯度实现错误导致链偏移或完全不收敛。

排查顺序：
- 先看发散率与接受率；
- 再看样本均值/协方差偏差；
- 最后检查梯度与 leapfrog 实现的符号、尺度。

## R15

工程落地建议：
- 引入多链并行（不同种子）并报告 `R-hat`；
- 增加质量矩阵自适应（对角/稠密）提升病态后验表现；
- 记录 per-iteration 诊断（能量变化、树节点数、发散位置）；
- 对复杂模型优先做重参数化（如 centered/non-centered 切换）。

## R16

相关算法对比：
- Random Walk MH：实现简单但混合慢，维度提升时性能退化明显；
- HMC：高效但需手调轨迹长度 `L`；
- NUTS：自动停止轨迹扩展，通常在“易用性/效率”上优于手工 HMC；
- 变分推断（VI）：速度快但有近似偏差；NUTS 更偏“高保真后验采样”。

## R18

`demo.py` 源码级算法流程（8 步）：

1. `main` 构造二维高斯目标（真均值/协方差已知），配置 `NutsConfig` 并调用 `run_nuts`。  
2. `run_nuts` 先通过 `find_reasonable_step_size` 试探初始步长，再初始化 dual averaging 变量 `mu/h_bar/step_size_bar`。  
3. 每次迭代先重采样动量 `p0`，计算联合对数密度 `joint_log_density(logp, p0)`，并采样 slice 阈值 `log_u`。  
4. 在 `while depth < max_depth` 中随机选择扩展方向，调用 `build_tree` 递归构造深度为 `depth` 的子树。  
5. `build_tree` 的基例（`depth==0`）执行一次 `leapfrog`，计算有效样本标志、发散标志、以及局部接受率贡献。  
6. `build_tree` 的递归层合并左右子树：按有效节点数更新候选 `q_proposed`，并用 `stop_criterion` 检查 No-U-Turn 是否仍成立。  
7. 一次迭代结束后用子树累计的 `alpha/n_alpha` 计算平均接受率；warmup 阶段更新 dual averaging，采样阶段固定 `step_size_bar` 并收集样本。  
8. `main` 对样本计算均值、协方差、ESS，并用断言验证误差与采样质量阈值，最终打印 `All checks passed.`。

本实现没有把第三方 NUTS 当黑盒：轨迹积分、树扩展、停止准则、步长适配与诊断都在源码中可逐行追踪。
