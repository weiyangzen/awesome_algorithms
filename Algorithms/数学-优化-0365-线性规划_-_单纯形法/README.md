## R01

线性规划（Linear Programming, LP）研究在线性约束下优化线性目标函数。  
单纯形法（Simplex Method）通过在可行域多面体的顶点之间“枢轴移动”，逐步改进目标值，直到达到最优顶点。

本条目实现一个教学级 MVP：
- 问题类型：`max c^T x`，满足 `Ax <= b, x >= 0`；
- 算法形态：表法单纯形（tableau simplex）；
- 初始基：松弛变量构成的单位阵基；
- 终止条件：目标行不存在可改进入基变量（无负检验数）。

## R02

历史与定位（简版）：
- 单纯形法由 George Dantzig 在 20 世纪中期提出，是运筹优化最经典算法之一；
- 尽管理论最坏复杂度可指数级，但在大量工程问题上表现非常高效；
- 现代工业求解器常混合单纯形、对偶单纯形和内点法；
- 本目录侧重“可读、可审计、可运行”的最小闭环实现，而非追求大规模工业性能。

## R05

数学模型：
- 原问题（本实现）：
  `max c^T x`
  `s.t. Ax <= b, x >= 0`
- 通过加入松弛变量 `s >= 0` 转为等式：
  `Ax + Is = b`
- 在基可行解视角，固定一组基变量 `x_B`，解得 `x_B = B^{-1}b`，其余非基变量为 0。

单纯形法本质上是在基可行解之间移动，并保证每一步目标不下降（最大化问题）。

## R06

表法单纯形核心步骤：
1. 构造初始单纯形表：`[A | I | b]` 与目标行 `[-c | 0 | 0]`；
2. 选入基列（entering）：目标行中最负系数（本实现用 Bland 规则取最小索引）；
3. 选出基行（leaving）：对入基列做最小比值检验 `b_i / a_ij`（`a_ij>0`）；
4. 枢轴变换：把枢轴元素化为 1，并把该列其余元素消为 0；
5. 更新基变量集合，循环执行直到最优或判定无界。

## R07

退化与循环处理：
- 当多个候选入基/出基并列时，若选择不当可能循环；
- 本实现使用 Bland 规则（最小索引优先）降低循环风险；
- 若入基列在所有约束行都无正系数，则问题在该方向上无界，算法返回 `unbounded`。

## R08

时间复杂度（表法实现）：
- 设约束数为 `m`，变量数为 `n`（不含松弛）；
- 每次 pivot 需要对 `(m+1) x (n+m+1)` 表做行操作，约 `O(m(n+m))`；
- 总成本为 `O(T * m(n+m))`，`T` 为 pivot 次数；
- 理论最坏情况下 `T` 可指数增长，但实际中通常远好于最坏界。

## R09

空间复杂度：
- 主存储是单纯形表：`(m+1) * (n+m+1)`，即 `O(m(n+m))`；
- 额外存储包含基索引、历史目标值等，`O(m + T)`；
- 对于教学与中小规模问题，内存占用可控。

## R13

鲁棒性与输入约束：
- 维度检查：`A` 必须是二维，`c` 与 `A` 列匹配，`b` 与 `A` 行匹配；
- 数值检查：`A,c,b` 需为有限实数；
- 本 MVP 假设 `b >= 0`，以便松弛变量直接给出初始可行基；
- 若出现无界方向，显式返回 `unbounded`；
- 若超出 `max_iter`，返回 `max_iter_reached`，避免静默失败。

## R17

方法对比：
- `linprog(method="highs")`：工业级稳健、速度快，但流程对初学者较黑盒；
- 内点法：理论复杂度更平滑，适合超大规模稀疏 LP；
- 单纯形法：路径可解释、调试直观，适合教学与中小规模可解释求解；
- 本实现选择“可读优先”的表法单纯形，强调算法步骤透明。

## R18

`demo.py` 源码级算法流程（8 步）：
1. `build_demo_lp()` 定义一个固定 LP 样例（`c,A,b`），并给出可复现测试入口。  
2. `simplex_max_leq()` 做输入校验，要求维度一致、有限数值、`b>=0`。  
3. 在 `simplex_max_leq()` 中构造初始 tableau：约束块 `[A|I|b]`，目标行 `[-c|0|0]`，并记录初始基为松弛变量。  
4. `choose_entering_variable()` 从目标行挑选可改进入基列（负检验数，按 Bland 规则选最小索引）。  
5. `choose_leaving_row()` 对入基列执行最小比值检验，选出基行；若无可选行则判定无界。  
6. `pivot_in_place()` 执行枢轴消元，把入基列化为单位列并更新整个 tableau。  
7. 循环步骤 4-6，直到不存在负检验数（最优）或达到迭代上限（返回 `max_iter_reached`）。  
8. `main()` 解析最优解与目标值，执行可行性/一致性断言；若 SciPy 可用，调用 `linprog` 仅做结果对照验证。  

说明：第三方库不负责主求解路径，主算法是显式编码的单纯形迭代流程。
