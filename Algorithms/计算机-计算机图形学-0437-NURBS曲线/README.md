## R01

NURBS（Non-Uniform Rational B-Spline，非均匀有理 B 样条）是 CAD/CAM 与几何建模里最常用的曲线表示之一。
它同时具备 3 个关键能力：
- 非均匀节点向量：可做局部参数化控制；
- B 样条基函数：具有局部支撑、数值稳定；
- 有理权重：可精确表示圆弧、圆锥曲线等二次曲线。

本目录的 MVP 聚焦“从零实现可审计的 NURBS 曲线求值与采样”，不依赖图形界面。

## R02

MVP 解决的问题：

给定控制点 `P_i`、权重 `w_i`、次数 `p`、节点向量 `U`，计算参数 `u` 处曲线点：

`C(u) = (sum_i N_{i,p}(u) * w_i * P_i) / (sum_i N_{i,p}(u) * w_i)`

并在参数区间 `[U[p], U[n+1]]` 上做均匀采样，输出可验证数值结果。

## R03

为什么用 NURBS 而不是普通 Bézier：
- Bézier 全局控制较强，局部修改会影响整体；
- B 样条有局部支撑，但无权重时难以精确表达圆；
- NURBS 在 B 样条基础上加入权重，既保留局部可控性，又能精确表达工程常见曲线。

因此它是“工程可控性 + 几何表达力”兼顾的主流方案。

## R04

核心数学对象：

1. 参数与索引：
- 控制点个数 `n+1`；
- 次数 `p`；
- 节点向量长度 `m+1 = n+p+2`。

2. B 样条基函数（Cox-de Boor 递推）：
- `N_{i,0}(u) = 1` 当 `U_i <= u < U_{i+1}`，否则为 `0`；
- `N_{i,p}(u) = ((u-U_i)/(U_{i+p}-U_i))*N_{i,p-1}(u) + ((U_{i+p+1}-u)/(U_{i+p+1}-U_{i+1}))*N_{i+1,p-1}(u)`。

3. NURBS 有理形式：
- 分子：`sum_i N_{i,p}(u) * w_i * P_i`；
- 分母：`sum_i N_{i,p}(u) * w_i`；
- 曲线点：分子/分母。

## R05

算法高层流程：

1. 校验输入维度、节点向量单调性、权重正值、长度约束。
2. 对每个参数 `u`，在节点向量中二分查找其所在区间 `span`。
3. 用 Cox-de Boor 递推计算该 `span` 对应的 `p+1` 个非零基函数值。
4. 取局部 `p+1` 个控制点与权重，计算加权分子与分母。
5. 得到 `C(u)`，重复以上过程完成整条曲线采样。

## R06

正确性与实现一致性要点：
- `find_span` 在参数右端点做夹紧，避免 `u=u_max` 时越界；
- `basis_functions` 仅计算局部非零基函数，符合 B 样条局部支撑性质；
- 分母接近 0 时显式抛错，避免静默数值污染；
- 对参数范围外输入直接拒绝，保证结果定义域清晰。

## R07

复杂度分析（单次求值）：
- `find_span`：二分查找，`O(log n)`；
- 基函数递推：`O(p^2)`；
- 局部加权求和：`O(p * d)`，`d` 为坐标维度。

采样 `s` 个参数点的总体复杂度约为：
`O(s * (log n + p^2 + p*d))`。

在工程常见 `p<=3` 的情况下，主要开销通常是采样点数量 `s`。

## R08

边界与异常处理：
- 控制点必须是二维数组 `(n, d)`；
- 权重长度必须等于控制点个数，且全部 `>0`；
- 节点向量长度必须满足 `len(U)=n_ctrl+p+1`；
- 节点向量必须非降序；
- 有效参数域 `[U[p], U[n+1]]` 不能退化为零长度；
- `num_points < 2` 的采样请求直接报错。

## R09

MVP 取舍：
- 重点做“可运行、可读、可审计”的曲线求值，不引入曲面、导数、节点插入等扩展模块；
- 不依赖外部 NURBS 黑盒库，直接实现核心算法；
- 输出数值检查而非绘图，减少环境依赖。

这样可确保算法流程透明，便于后续验证与扩展。

## R10

`demo.py` 函数职责：
- `NURBSCurve._validate_inputs`：输入合法性校验；
- `NURBSCurve.find_span`：参数区间定位；
- `NURBSCurve.basis_functions`：Cox-de Boor 基函数递推；
- `NURBSCurve.evaluate`：单点 NURBS 求值；
- `NURBSCurve.sample`：参数区间均匀采样；
- `open_uniform_knot_vector`：生成开区间均匀节点向量；
- `quarter_circle_case`：构建可精确表示四分之一圆的二次 NURBS；
- `generic_curve_case`：构建通用三次 NURBS 示例；
- `main`：执行示例、打印结果与检查项。

## R12

输出字段说明：
- `parameter_domain`：曲线有效参数区间；
- `start_point/end_point`：采样首尾点；
- `mid_point(u=0.5)`：四分之一圆示例中点；
- `max_radius_error`：四分之一圆采样点到单位圆半径的最大误差；
- `knot_vector`：通用三次示例使用的节点向量；
- `finite_points`：采样点是否均为有限值；
- `monotone_parameters`：采样参数是否单调不降。

## R13

最小验收项：
- 脚本可直接运行，不需要输入；
- 四分之一圆示例首尾点应接近 `(1,0)` 与 `(0,1)`；
- `max_radius_error` 应接近机器精度量级（通常很小）；
- `finite_points=True` 且 `monotone_parameters=True`。

## R14

关键参数与调参建议：
- `degree`：常用 2 或 3，次数越高局部控制越弱、计算成本略增；
- `weights`：控制“拉力”，中间点权重增大时曲线更向该点靠拢；
- `knot_vector`：内节点分布控制参数密度与局部形状变化速度；
- `num_points`：仅影响采样分辨率，不改变曲线数学定义。

## R15

与相近曲线模型对比：
- Bézier：实现简单但全局耦合强；
- B 样条：局部性好，但无法精确表达圆锥曲线；
- NURBS：在 B 样条上引入权重，兼具局部性与精确几何表达。

因此 NURBS 在工业几何建模与数控路径中更常见。

## R16

典型应用场景：
- CAD 曲线与轮廓设计；
- CAM 刀具轨迹表示；
- 曲线驱动的动画路径与字体外形；
- 工程图形系统中的可编辑几何对象。

## R17

可扩展方向：
- 增加一阶/二阶导数与曲率计算；
- 支持节点插入、节点移除、次数提升；
- 扩展到 NURBS 曲面（双参数 `u,v`）；
- 加入弧长参数化与等弧长采样；
- 接入可视化绘图用于直观调试。

## R18

`demo.py` 的源码级算法流程（8 步）：

1. `main` 调用 `quarter_circle_case` 和 `generic_curve_case`，构造两组控制点/权重/节点向量。
2. `NURBSCurve.__init__` 执行 `_validate_inputs`，确保维度、长度、单调性和参数域合法。
3. 对每个采样参数 `u`，`evaluate` 先调用 `find_span` 在节点向量中用二分法定位活跃区间。
4. `basis_functions` 在该区间上用 Cox-de Boor 递推计算 `p+1` 个非零基函数值。
5. `evaluate` 按 `span-degree : span` 取局部控制点和权重，得到 `weighted_basis`。
6. 计算分子 `sum(weighted_basis * control_points)` 与分母 `sum(weighted_basis)`，做有理归一化得到 `C(u)`。
7. `sample` 在 `[u_min, u_max]` 生成均匀参数序列并重复单点评估，形成整条曲线离散点。
8. `main` 对四分之一圆结果计算半径误差、打印关键点与布尔检查项，作为最小可验证输出。
