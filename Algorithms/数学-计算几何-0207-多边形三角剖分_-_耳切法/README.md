# 多边形三角剖分 - 耳切法

- UID: `MATH-0207`
- 学科: `数学`
- 分类: `计算几何`

## R01

耳切法（Ear Clipping）用于把一个简单多边形剖分为若干不重叠三角形。核心思路是：
- 在当前多边形中找到一个“耳朵”三角形；
- 切掉耳朵顶点；
- 对剩余多边形重复，直到只剩 3 个顶点。

本目录提供可运行 MVP：`demo.py` 手写耳切法主流程，不调用黑盒三角剖分库。

## R02

问题定义（本实现）：
- 输入：二维简单多边形顶点序列 `P = [p0, p1, ..., p(n-1)]`，每个点为 `(x, y)`。
- 输出：三角形索引集合 `T = {(i, j, k)}`，满足：
  - 所有索引指向归一化后的多边形顶点；
  - 三角形方向统一为逆时针（CCW）；
  - 三角形数量为 `n - 2`（`n` 为有效顶点数）；
  - 三角形面积和等于原多边形面积。

脚本使用内置测试样例，无需交互输入。

## R03

数学基础：
1. 方向判定：
   - `orientation(a,b,c) > 0` 表示 `a->b->c` 为左转（CCW）。
2. 多边形有向面积：
   - 鞋带公式可判断顶点顺序；面积为负时可反转为 CCW。
3. 耳朵定义（CCW 情况）：
   - 顶点 `v_i` 为凸点（局部左转）；
   - 三角形 `(v_{i-1}, v_i, v_{i+1})` 内不含其它顶点。
4. 耳分解定理：
   - 任意简单多边形（顶点数 `>= 4`）至少存在两个耳。

## R04

算法流程（高层）：
1. 读取顶点并做合法性检查（形状、有限数值）。
2. 清理闭合重复点、相邻重复点、连续共线冗余点。
3. 用有向面积统一顶点顺序为 CCW。
4. 检测多边形是否简单（无自交）。
5. 维护“活动顶点列表” `active`。
6. 在 `active` 中扫描耳朵：
   - 先判凸性；
   - 再判三角形内是否包含其他活动顶点。
7. 找到耳后输出一个三角形并删除耳顶点。
8. 重复步骤 6-7，直到剩余 3 个顶点，输出最后一个三角形。

## R05

核心数据结构：
- `PointArray (n x 2)`：归一化后的二维顶点数组。
- `active: list[int]`：当前仍在多边形边界上的顶点索引。
- `Triangle = (i, j, k)`：一个三角形。
- `TriangulationResult`：
  - `points`: 归一化后的顶点；
  - `triangles`: 耳切生成的三角形列表。

## R06

正确性要点：
- 方向一致性：`normalize_polygon` 保证多边形是 CCW。
- 耳判定正确性：
  - `orientation(a,b,c) > eps` 确保耳为凸角；
  - `point_in_triangle` 保证耳三角形内部/边界无其它活动点。
- 每次剪耳后顶点数减 1，终止于 3 个顶点，过程可收敛。
- 输出校验：
  - 三角形数量检查 `n-2`；
  - 三角形面积和与多边形面积对齐；
  - 所有三角形方向应为 CCW。

## R07

复杂度分析（`n` 为多边形顶点数）：
- 每次剪耳最多扫描 `O(n)` 个候选顶点；
- 每个候选耳要做“点在三角形内”检查，最坏 `O(n)`；
- 共需剪 `n-3` 次。

综合最坏时间复杂度约 `O(n^3)`，空间复杂度 `O(n)`。

## R08

边界与异常处理：
- 输入不是 `n x 2`：抛 `ValueError`。
- 输入含 `nan/inf`：抛 `ValueError`。
- 有效顶点数小于 3：抛 `ValueError`。
- 面积接近 0（退化为线段/点）：抛 `ValueError`。
- 自交多边形：抛 `ValueError`。
- 若扫描一轮未找到耳（异常数值或非法输入）：抛 `ValueError`。

## R09

MVP 设计取舍：
- 只覆盖二维简单多边形，不处理带孔多边形。
- 不依赖专门几何库，仅用 `numpy` 完成向量与面积计算。
- 优先可读性与可审计性，未做高性能优化（如单调队列、空间索引）。
- 通过面积一致性和计数规则做基础正确性验证。

## R10

`demo.py` 主要函数：
- `check_polygon`：输入格式和有限值检查。
- `remove_closing_duplicate` / `remove_adjacent_duplicates`：去重清洗。
- `remove_collinear_vertices`：剔除连续共线冗余点。
- `is_simple_polygon`：线段相交检测，验证简单多边形条件。
- `normalize_polygon`：执行清洗、方向统一、合法性检查。
- `is_ear`：耳朵判定。
- `ear_clipping_triangulation`：耳切主循环。
- `triangulation_area`：面积一致性校验。
- `summarize_case` / `main`：组织样例并打印结果。

## R12

输出字段说明：

- `triangles`：三角形数量及理论值 `n-2`。
- `all triangles CCW`：所有三角形是否为逆时针。
- `polygon area`：原多边形面积。

## R14

可调参数：
- `eps`（默认 `1e-12`）：
  - 凸性判定阈值；
  - 点在线段/三角形边界判定阈值；
  - 去重与共线清理阈值。

调参建议：
- 坐标尺度很大时，可适当增大 `eps`。
- 若误删细小结构，减小 `eps` 并先做坐标归一化。

## R15

与其他方法对比：
- 对比单调多边形分解 + 线性三角化：
  - 后者理论更快，但实现复杂度更高。
- 对比受约束 Delaunay 三角剖分（CDT）：
  - CDT 质量更高、用途更广；
  - 耳切法更轻量，适合教学和原型。
- 对比库函数直接调用：
  - 库更稳健高效；
  - 本实现流程透明，便于理解和调试。

## R18

`demo.py` 的源码级流程（9 步）：
1. `main` 构造三组固定测试多边形并依次调用 `summarize_case`。
2. `summarize_case` 调用 `ear_clipping_triangulation` 得到剖分结果。
3. `ear_clipping_triangulation` 首先调用 `normalize_polygon` 做输入清洗、CCW 统一与简单性校验。
4. `normalize_polygon` 依次执行去闭合重复点、去相邻重复点、去共线冗余点，并检查面积非零。
5. 若多边形为顺时针，则顶点序反转；随后 `is_simple_polygon` 用线段相交检测确保无自交。
6. 主循环维护 `active` 顶点索引，在每轮扫描中调用 `is_ear` 判断某顶点是否可被剪掉。
7. `is_ear` 先用 `orientation` 判断局部凸性，再用 `point_in_triangle` 排除“耳内含其他顶点”的情况。
8. 找到耳后输出三角形并从 `active` 删除耳顶点，循环直到剩下 3 个顶点并输出最后一三角形。
9. `summarize_case` 用 `triangulation_area` 对比多边形面积与三角形面积和，打印计数、方向和面积一致性结果。
