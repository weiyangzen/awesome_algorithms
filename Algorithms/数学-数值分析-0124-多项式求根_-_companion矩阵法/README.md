## R01

Companion 矩阵法把“一元多项式求根”转化为“矩阵特征值问题”：

- 给定多项式  
  `p(x) = a0*x^n + a1*x^(n-1) + ... + an`, `a0 != 0`
- 先归一化为首一形式  
  `x^n + b1*x^(n-1) + ... + bn`
- 构造 `n x n` companion 矩阵 `C`
- `C` 的特征值就是 `p(x)` 的全部根（重根按代数重数计算）

本目录 MVP 使用 `numpy` 构造 companion 矩阵并调用线性代数特征值求解器完成求根，同时对结果做残差验证。

## R02

有限计算版问题定义：

- 输入：实数/复数系数数组 `coeffs=[a0,a1,...,an]`（从高次到常数项）
- 输出：
  - 根数组 `roots`（复数域）
  - 数值验证指标 `max |p(root_i)|`
- 约束：
  - `a0 != 0`
  - 次数 `n >= 1`

## R03

方法背景：

- 闭式公式只覆盖低次（如二三四次），高次一般依赖数值算法。
- companion 变换把多项式问题放进成熟的矩阵特征值计算框架（LAPACK 生态）。
- 在工程上，这是一种“实现短、可解释、通用性高”的求根基线方案。

## R04

companion 矩阵（首一多项式）定义：

对 `q(x)=x^n+b1*x^(n-1)+...+bn`，构造

```text
C = [ -b1  -b2  ...  -bn  ]
    [  1    0   ...   0   ]
    [  0    1   ...   0   ]
    [ ...               ...]
    [  0    0   ...   1  0]
```

满足特征方程：

`det(lambda*I - C) = q(lambda)`  
因此 `eigvals(C)` 就是 `q(x)=0` 的根。

## R05

核心数据结构（`demo.py`）：

- `np.ndarray[complex128] coeffs`：输入多项式系数。
- `np.ndarray[complex128] companion`：companion 矩阵。
- `np.ndarray[complex128] roots`：数值根。
- `Case` 数据类：封装测试用例（名称、系数、期望根）。

## R06

正确性要点（MVP 级）：

1. 把原多项式除以首项系数 `a0`，不改变根集合。
2. 对首一多项式构造 companion 矩阵 `C`。
3. 根据 companion 矩阵的标准结论，`charpoly(C)=q(x)`。
4. 所以特征值集合与多项式根集合一致（重数一致）。
5. 数值实现中再通过 `|p(root_i)|` 残差检查确认结果可信度。

## R07

时间复杂度：

- 构造 companion 矩阵：`O(n^2)`。
- 求 `n x n` 矩阵全部特征值：主导成本约 `O(n^3)`。
- 残差评估（Horner 对全部根）：`O(n^2)`。

综合复杂度由特征值求解主导，即 `O(n^3)`。

## R08

空间复杂度：

- companion 矩阵存储：`O(n^2)`。
- 根向量与中间数组：`O(n)` 到 `O(n^2)`（实现相关）。
- 总体：`O(n^2)`。

## R09

数值稳定性说明：

- 多项式求根通常是病态问题，尤其在高次或重根附近，小系数扰动会导致根显著变化。
- companion 路线本身依赖浮点特征值算法，结果受条件数影响。
- 本实现通过“残差 + 与已知根误差”双指标展示质量，不把单次输出当作绝对真值。

## R10

边界与异常处理：

- 全零或空系数数组：报错。
- 去掉前导零后次数为 0（常数多项式）：报错（无“求根集合”意义）。
- 线性多项式：companion 为 `1x1`，直接得到唯一根。
- 复系数输入：直接支持（`complex128`）。

## R12

输出解读：

- `degree`：多项式次数。
- `max_residual`：`max_i |p(root_i)|`，越接近 0 越好。
- `max_root_error`：与已知真根做排序后逐项比较的最大误差（示例用途）。
- `status=PASS`：残差在阈值内，说明数值结果与方程一致性良好。

## R13

最小测试集（已在 demo 覆盖）：

- 互异实根：`(x-1)(x-2)(x-3)`。
- 含共轭复根：`(x-(1+2j))(x-(1-2j))(x+0.5)`。
- 重根：`(x-1)^3`（用于观察病态场景）。
- 稍高次数混合根：验证通用性与稳定性。

## R14

可调参数与扩展方向：

- 可调整多项式次数、根分布、容差阈值。
- 可增加随机批量回归统计（成功率、残差分布）。
- 高精度需求可替换为 `mpmath`/符号工具做交叉验证（非本 MVP 范围）。

## R15

与其他求根方法对比：

- 相比牛顿法：牛顿法需要初值，且通常单根迭代；companion 一次得到全部根。
- 相比 Jenkins-Traub / Aberth：这些专用多项式算法常更强，但实现复杂。
- companion 的优势是复用成熟特征值求解器，代码短、教学清晰、工程可复现。

## R16

应用场景：

- 控制理论特征方程稳定性分析；
- 信号处理滤波器极点计算；
- 数值分析教学中“多项式 <-> 线性代数”桥接示例；
- 作为更复杂根求解框架的 baseline。

## R17

`demo.py` 关键函数：

- `normalize_to_monic`: 去前导零并归一化为首一多项式。
- `build_companion_matrix`: 按定义构造 companion 矩阵。
- `roots_via_companion`: 通过 `np.linalg.eigvals` 求特征值作为根。
- `evaluate_polynomial_horner`: Horner 法计算 `p(x)`。
- `run_case`: 执行单案例并计算残差/误差指标。
- `main`: 组织 4 个固定案例并打印结果、做断言。

## R18

源码级流程拆解（对应 `demo.py`，8 步）：

1. `main` 构造若干测试多项式（由已知根反推系数）并逐个调用 `run_case`。  
2. `run_case` 先调用 `roots_via_companion`，进入 companion 求根主流程。  
3. `normalize_to_monic` 删除前导零并将系数除以首项，得到首一多项式系数 `b1..bn`。  
4. `build_companion_matrix` 创建 `n x n` 复矩阵：首行为 `-b1..-bn`，下副对角置 1。  
5. `np.linalg.eigvals` 触发 LAPACK `geev` 路径（内部包含平衡/Hessenberg 化/QR 迭代），返回全部特征值。  
6. 把特征值当作根，调用 `evaluate_polynomial_horner` 逐根回代，得到残差向量 `p(root_i)`。  
7. 计算 `max_residual` 与（可选）`max_root_error`，并与容差阈值比较。  
8. 打印每个案例的根与指标，若残差超阈值则断言失败，确保脚本可用于自动验证。
