## R01

矩阵链乘法（Matrix Chain Multiplication）问题定义：
- 给定维度序列 `p = [p0, p1, ..., pn]`；
- 由此确定矩阵链 `A1..An`，其中 `Ai` 尺寸为 `p(i-1) x p(i)`；
- 目标不是改变乘法结果，而是选择最优加括号顺序，使标量乘法次数最少。

本目录实现一个“最小可运行 MVP”：输出最小乘法次数，并重建一种最优括号化方案。

## R02

MVP 输入输出约定：
- 输入：
  - `dims`：一维正整数序列，长度至少为 2；
  - 若 `dims` 长度为 `n+1`，则表示 `n` 个矩阵。
- 输出：
  - `min_scalar_multiplications`：最小标量乘法次数；
  - `parenthesization`：一种最优括号化字符串（如 `((A1 x A2) x A3)`）；
  - `steps`：后序重建出的合并步骤列表（含每步标量代价）。

约束：
- `dims` 必须是一维、有限、正、整数值；
- 本 MVP 不处理稀疏矩阵、块矩阵和并行成本模型。

## R03

状态定义（区间 DP）：
- 设矩阵下标为 `0..n-1`（对应 `A1..An`）；
- `cost[i][j]`：把子链 `Ai..Aj`（含端点）相乘的最小标量乘法次数；
- `split[i][j]`：达到最优值时的切分点 `k`，表示先算 `Ai..Ak`，再算 `A(k+1)..Aj`。

边界条件：
- `cost[i][i] = 0`，单个矩阵无需乘法。

## R04

状态转移方程：

对于 `i < j`，枚举 `k in [i, j-1]`：

`cost[i][j] = min(cost[i][k] + cost[k+1][j] + p[i] * p[k+1] * p[j+1])`

其中：
- `p[i] * p[k+1] * p[j+1]` 是把两个子结果矩阵相乘的额外代价。

## R05

最优子结构成立原因：
- 若 `Ai..Aj` 的最优解最后一次切分在 `k`；
- 则左子链 `Ai..Ak` 与右子链 `A(k+1)..Aj` 必须分别最优；
- 否则替换为更优子解会使总成本更低，和“最优解”矛盾。

因此可采用“区间长度递增”的自底向上 DP 填表。

## R06

最优方案重建：
- `split` 记录了每个区间最优切分点；
- 从 `split[0][n-1]` 递归展开：
  - 叶子 `i==j` 输出 `Ai`；
  - 内部节点输出 `(left x right)`。
- 同时可按后序遍历记录每一步实际乘法：
  - 左区间、右区间、合并后区间及该步标量代价。

## R07

复杂度分析：
- 自底向上主算法：
  - 时间复杂度 `O(n^3)`（区间长度、起点、切分点三层循环）；
  - 空间复杂度 `O(n^2)`（`cost` 与 `split`）。
- 记忆化递归基线：
  - 时间复杂度同阶 `O(n^3)`；
  - 空间复杂度 `O(n^2)`（状态缓存）。

## R08

`demo.py` 结构：
- `to_dims_array`：输入校验与标准化；
- `matrix_chain_order`：主算法（自底向上 DP + `split`）；
- `build_parenthesization` / `rebuild_steps`：重建括号化和步骤；
- `matrix_chain_top_down_cost`：记忆化递归基线；
- `matrix_chain_bruteforce_cost`：无缓存暴力（仅小规模校验）；
- `run_case`：单样例运行与一致性断言；
- `randomized_cross_check`：随机对拍；
- `main`：无交互统一入口。

## R09

核心接口：
- `matrix_chain_order(dims) -> MatrixChainResult`
  - 返回最优成本、括号化结果、切分表与步骤。
- `matrix_chain_top_down_cost(dims) -> int`
  - 仅返回最优成本，作为独立基线。
- `left_to_right_cost(dims) -> int`
  - 计算按“从左到右固定顺序”相乘的成本，用于对比最优性。

## R10

固定样例（`main()`）覆盖：

## R12

输出解读：
- `optimal`：主算法给出的最优成本与括号化；
- `steps`：按重建顺序列出每次区间合并和该步成本；
- `cross-check`：
  - `top_down`：记忆化递归结果；
  - `brute_force`：小规模时暴力枚举结果；
  - `sum_step_cost`：步骤成本求和；
  - `left_to_right_cost`：固定左结合成本（通常更高）。

程序会断言关键结果一致，否则抛出 `AssertionError`。

## R13

随机对拍策略：
- 默认 `200` 轮；
- 每轮随机生成：
  - 矩阵数量 `n ∈ [1, 7]`；
  - 维度值 `2..25`，形成长度 `n+1` 的 `dims`。
- 对每轮断言：
  - 自底向上 DP 结果 == 记忆化递归结果；
  - 若 `n <= 5`，再要求 == 无缓存暴力结果；
  - `steps` 成本和 == DP 最优成本。

## R14

边界与异常处理：
- `dims` 不是一维、长度小于 2、含 `NaN/Inf`：抛 `ValueError`；
- 维度含非正数或非整数值：抛 `ValueError`；
- 若 `split` 记录异常导致无法重建（理论上不应出现）：抛 `RuntimeError`。

## R18

`demo.py` 源码级算法流程（8 步）：

1. `main` 先组织 4 个固定样例，逐个调用 `run_case`，最后执行 `randomized_cross_check`。  
2. `run_case` 调用 `matrix_chain_order`：创建 `cost` 与 `split` 二维表，先设 `cost[i][i]=0`。  
3. 主算法按区间长度 `2..n` 递增，遍历每个子链 `[i,j]`。  
4. 对每个 `[i,j]` 枚举切分点 `k`，计算 `cost[i][k] + cost[k+1][j] + p[i]*p[k+1]*p[j+1]`，更新最优值与 `split[i][j]`。  
5. 填表后，`build_parenthesization` 基于 `split` 递归生成括号表达式；`rebuild_steps` 后序重建每一步合并及其代价。  
6. `run_case` 再调用 `matrix_chain_top_down_cost`（`lru_cache`）得到独立基线，并在小规模下调用 `matrix_chain_bruteforce_cost` 做无缓存穷举校验。  
7. `run_case` 断言“DP 成本 = 基线成本 =（适用时）暴力成本 = 步骤代价总和”，并与 `left_to_right_cost` 对比输出。  
8. 对可控维度样例，程序还会随机生成同形状矩阵，分别按最优括号化与左结合顺序计算，断言数值结果一致以验证重建方案正确性。
