## R01

不动点迭代法（Fixed-Point Iteration）用于求解形如 `x = g(x)` 的方程。

如果原问题是 `f(x)=0`，可先重排为 `x=g(x)`，再从初值 `x0` 出发执行
`x_{k+1} = g(x_k)`，直到迭代点稳定。

本条目给出一个可运行的最小 MVP：
- 实现通用标量不动点迭代器；
- 记录完整迭代轨迹；
- 演示收敛与不收敛两类情形；
- 不依赖黑箱求根器。

## R02

问题定义（标量版）：
- 输入：
  - 映射 `g: R -> R`；
  - 初值 `x0`；
  - 停止参数 `tol`（容差）、`max_iter`（最大迭代轮数）。
- 输出：
  - 近似不动点 `x*`（若收敛）；
  - 迭代轨迹（每轮 `x_k, x_{k+1}, step, residual`）；
  - 收敛标志 `converged`。

其中：
- `step = |x_{k+1} - x_k|`；
- `residual = |g(x_{k+1}) - x_{k+1}|`。

## R03

理论基础（压缩版）：

1. 不动点定义：`x* = g(x*)`。
2. 若 `g` 在区间 `I` 上把 `I` 映射到自身，且满足压缩条件
   `|g(x)-g(y)| <= q|x-y| (0<q<1)`，则存在唯一不动点且迭代收敛。
3. 误差递推（理想压缩场景）：
   `|x_{k+1}-x*| <= q |x_k-x*|`，为线性收敛。
4. 不同重排方式会改变 `g'(x*)`，因此也会显著改变收敛速度和是否收敛。

## R04

算法流程：
1. 检查 `x0`、`tol`、`max_iter` 的合法性与有限性。
2. 令 `x <- x0`。
3. 计算 `x_next = g(x)`。
4. 计算 `step = |x_next - x|`。
5. 计算 `residual = |g(x_next) - x_next|`。
6. 保存本轮轨迹。
7. 若 `step <= tol` 且 `residual <= tol`，判定收敛并停止。
8. 否则置 `x <- x_next` 进入下一轮，直到达到 `max_iter`。

## R05

核心数据结构（`demo.py`）：
- `IterRecord`（`dataclass`）：
  - `k`：迭代轮数；
  - `x_k`：本轮输入点；
  - `x_next`：映射后新点；
  - `step`：步长；
  - `residual`：不动点残差。
- `history: list[IterRecord]`：完整轨迹。

MVP 使用 `numpy` 做基础数值函数与有限性判断，不引入大型框架。

## R06

正确性要点：
- 迭代核心严格对应定义 `x_{k+1}=g(x_k)`；
- `residual = |g(x)-x|` 直接衡量“不动点条件”满足程度；
- 同时使用 `step` 和 `residual` 判停，避免仅看步长造成“假收敛”；
- 对收敛案例，结果与已知参考解对比验证实现正确。

## R07

复杂度分析（单个样例）：
- 若迭代 `T` 轮：
  - 时间复杂度 `O(T)`（每轮常数次函数评估）；
  - 空间复杂度：
    - 保存轨迹为 `O(T)`；
    - 若仅保留当前值可降为 `O(1)`。

## R08

边界与异常处理：
- `tol <= 0`：抛出 `ValueError`；
- `max_iter <= 0`：抛出 `ValueError`；
- 初值或迭代中出现 `nan/inf`：抛出 `ValueError` 或 `RuntimeError`；
- 达到 `max_iter` 仍不满足收敛条件：返回 `converged=False`（不强制抛错）。

## R09

MVP 取舍：
- 只做一维标量迭代，优先“透明 + 可验证”；
- 不调用 `scipy.optimize.fixed_point` 等黑箱接口；
- 用多个小案例覆盖常见行为：
  - 正常收敛；
  - 快速收敛；
  - 不收敛（导数绝对值大于 1 的典型）。

## R12

输出字段解释：
- `converged`：是否在 `max_iter` 内满足停止条件；
- `estimate`：最终近似不动点；
- `reference`：参考真值（若已知）；
- `abs_error / rel_error`：与参考值误差；
- `|g'(x*)|_est`：局部导数绝对值估计（用于解释收敛快慢）；
- 轨迹表：每轮 `x_k, x_next, step, residual`。

## R13

本条目最小测试集：
1. `x = cos(x)`：经典压缩映射，解约 `0.7390851332`。
2. `x = exp(-x)`：解约 `0.5671432904`。
3. `x = 0.5*(x + 2/x)`：对应 `sqrt(2)` 的不动点形式，收敛很快。
4. `x = 1.5x + 0.2`：`|g'|=1.5>1`，演示不收敛路径。

## R14

可调参数建议：
- `tol`：默认 `1e-12`；
- `max_iter`：默认 `80`；
- `print_limit`：轨迹打印条数上限，避免输出过长。

调参经验：
- 先用严格 `tol` 验证实现；
- 若只要工程近似，可放宽至 `1e-8` 并减少 `max_iter`。

## R15

方法对比：
- 对比二分法：
  - 二分法需要区间异号且收敛稳健；
  - 不动点法不需要异号区间，但重排不当会发散。
- 对比牛顿法：
  - 牛顿法通常更快（局部二次），但需要导数；
  - 不动点法实现更轻，可直接复用已有映射。
- 实践中常把牛顿迭代写成不动点形式统一框架处理。

## R16

应用场景：

- 隐式方程状态更新（例如某些时间离散格式）；
- 作为更复杂迭代法（Picard 迭代、松弛迭代）的基础构件。

## R18

源码级算法流（对应 `demo.py`，8 步）：
1. `main` 定义 4 个案例（3 个收敛、1 个不收敛）与验证阈值。  
2. `run_case` 对每个案例调用 `fixed_point_iteration(g, x0, tol, max_iter)`。  
3. 迭代器先做参数校验：`tol>0`、`max_iter>0`、`x0` 有限。  
4. 每轮计算 `x_next = g(x)`，并检查结果是否有限。  
5. 计算 `step = |x_next - x|`，再计算 `residual = |g(x_next)-x_next|`。  
6. 把 `k, x_k, x_next, step, residual` 存入 `history`。  
7. 若 `step` 与 `residual` 同时小于 `tol` 则收敛退出；否则继续迭代直到 `max_iter`。  
8. `run_case` 打印轨迹、误差和 `|g'(x*)|` 估计，并用断言校验预期行为。
