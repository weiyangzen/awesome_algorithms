# 量子纠错 (Quantum Error Correction)

- UID: `PHYS-0239`
- 学科: `物理`
- 分类: `量子信息`
- 源序号: `242`
- 目标目录: `Algorithms/物理-量子信息-0242-量子纠错_(Quantum_Error_Correction)`

## R01

量子纠错（Quantum Error Correction, QEC）解决的问题是：量子比特会受到退相干和噪声扰动，但量子态又不能被直接复制。  
本条目给出一个最小可运行的 3 比特重复码（bit-flip code）实现，用来纠正独立 `X` 翻转噪声。

逻辑思想是把一个逻辑比特编码到 3 个物理比特：

- `|0_L> = |000>`
- `|1_L> = |111>`

然后通过稳定子测量提取综合（syndrome），再做条件纠错。

## R02

历史上，量子纠错是容错量子计算的基础里程碑之一：

- 1995 年 Shor 码（9 比特）证明了量子信息可纠错；
- 随后 Steane 码、CSS 码把构造系统化；
- 现代工程主线是表面码（surface code）等拓扑容错方案。

本目录不追求“大码距工程体系”，而是先把核心闭环做成可审计 MVP：编码、噪声、综合、恢复、解码。

## R03

核心矛盾：

- 量子态对噪声高度敏感；
- 不能像经典比特那样直接复制备份；
- 若不做纠错，物理误差会在门序列中累积，导致算法失效。

QEC 的关键不是“消灭噪声”，而是把物理错误映射到可检测的综合，并在逻辑层把错误率压低。

## R04

本 MVP 的数学模型（对应 `demo.py`）是 3 比特 bit-flip 码：

1. 编码等距映射 `E (8x2)`：
   - `E|0> = |000>`
   - `E|1> = |111>`
2. 噪声通道（每个物理比特独立翻转，概率 `p`）：
   - `rho -> sum_{e in {0,1}^3} p^{wt(e)} (1-p)^{3-wt(e)} X^e rho X^e`
3. 稳定子：
   - `S1 = Z0 Z1`
   - `S2 = Z1 Z2`
4. 综合投影算符：
   - `P_{a,b} = 1/4 (I + a S1)(I + b S2)`, `a,b in {+1,-1}`
5. 纠错映射：
   - `(+,+)->I`, `(-,+)->X0`, `(-,-)->X1`, `(+,-)->X2`

这套流程对应“单比特 `X` 错误可纠正”的标准示例。

## R05

复杂度（以一般 `n` 比特状态向量/密度矩阵思路衡量）：

- 若直接矩阵表示，算符维度随 `2^n` 增长；
- 本实现在 3 比特上使用 `8x8` 密度矩阵，属于固定常数规模；
- 噪声求和含 `2^3=8` 个错误模式，恢复含 4 个综合分支。

因此当前 MVP 重点是“流程正确性与可解释性”，不是大规模性能优化。

## R06

3 比特 bit-flip 码在独立翻转噪声下的逻辑失败概率有闭式解：

- 未编码：`p_uncoded = p`
- 编码后（理想综合与纠错）：`p_logical = 3p^2 - 2p^3`

含义：

- 单个物理错误可被纠正；
- 主要失败来自“至少两个物理比特同时翻转”。

`demo.py` 会用精确线性代数仿真并逐点对比这个公式。

## R07

优点：

- 全流程显式实现，不依赖量子 SDK 黑箱；
- 既有可运行结果，也有解析公式可交叉验证；
- 适合作为更复杂量子纠错（Shor/Steane/surface code）之前的基线。

局限：

- 只能处理 bit-flip（`X`）噪声，不处理相位噪声 `Z`；
- 只演示码距为 3 的最小重复码，容错能力有限；
- 未包含门误差、测量误差、时序噪声与硬件拓扑约束。

## R08

前置知识建议：

- 单量子比特态与密度矩阵；
- Pauli 算符 `X/Z`；
- 稳定子与综合（syndrome）概念；
- Kronecker 积与基本线性代数。

运行依赖：

- Python `>=3.10`
- `numpy`
- `pandas`

## R09

适用场景：

- 教学中讲解“量子纠错为何可行”；
- 验证综合投影 + 条件恢复的实现正确性；
- 给后续复杂 QEC 仿真搭建可测试的最小基线。

不适用场景：

- 需要 realistic 噪声模型（振幅阻尼、相位阻尼、串扰等）；
- 需要大码距容错阈值评估；
- 需要直接面向真实量子硬件执行。

## R10

正确性检查点（`demo.py` 内已做断言）：

1. 四个综合投影算符正交且完备（和为单位阵）；
2. 综合概率和为 1；
3. 纠错后解码得到合法密度矩阵（Hermitian、trace=1）；
4. 数值逻辑失败率与理论 `3p^2-2p^3` 一致；
5. 在 `p<0.5` 区间，编码失败率不高于未编码失败率。

## R11

数值稳定策略：

- 全部使用 `complex128`；
- 解码后做 Hermitian 对称化 `rho=(rho+rho^\dagger)/2`；
- 显式归一化 trace，避免浮点累积误差；
- 使用严格断言阈值 `1e-12` 检查代数恒等式。

## R12

关键参数：

- `p_grid`：噪声概率扫描网格；
- `state_theta, state_phi`：演示态（Bloch 球参数）；
- `p_demo=0.10`：用于展示综合概率和编码/未编码保真度对比。

调参建议：

- 若要观察编码失效拐点，可把 `p_grid` 扩展到 `0.5~0.9`；
- 若要看状态依赖性，可替换演示态为 `|0>`, `|1>`, `|+>`, 复相位态。

## R13

该条目可给出明确理论保证（在理想综合与理想纠错门条件下）：

- 逻辑失败概率：`p_L = 3p^2 - 2p^3`；
- 当 `0 < p < 0.5` 时，`p_L < p`，编码优于未编码；
- 当 `p > 0.5` 时，噪声过强，重复码不再提供净收益。

本 MVP 用“解析式 + 精确仿真”做双重验证。

## R14

常见失效模式：

1. 量子位位序约定错乱（`q0` 是否最高位）导致综合映射错误；
2. 综合到纠错算符映射表写反；
3. 忽略投影完备性检查，导致 trace 漂移；
4. 把两比特/三比特错误也当成可纠正，误解码距能力边界。

对策：保留代数断言和理论曲线对比，避免“看起来能跑但逻辑错误”。

## R15

工程落地建议：

- 先在最小码（3 比特）上固定接口与测试规范；
- 再扩展到相位翻转码与 CSS 组合码（如 9 比特 Shor 码）；
- 引入测量错误后，应把恢复流程扩展为“重复测量 + 判决”；
- 最终面向硬件时需要加入门深约束和噪声标定数据。

## R16

与相关方法关系：

- 3 比特重复码：只纠正单 `X` 错误；
- 3 比特相位翻转码：只纠正单 `Z` 错误；
- 9 比特 Shor 码：把 bit-flip 与 phase-flip 保护结合；
- 表面码：工程主流拓扑方案，适合大规模容错。

本条目位于“概念到实现”的第一层：稳定子综合恢复机制的最小可执行原型。

## R17

`demo.py` 功能清单：

- 构造编码/解码映射 `E,D`；
- 精确实现三比特独立 bit-flip 噪声通道；
- 用稳定子投影算符做综合分解并执行条件纠错；
- 输出综合概率表、逻辑失败率曲线、理论误差差值；
- 运行断言，自动验证模型代数正确性和物理一致性。

运行方式：

```bash
cd Algorithms/物理-量子信息-0242-量子纠错_(Quantum_Error_Correction)
uv run python demo.py
```

## R18

`demo.py` 源码级算法流程（8 步）：

1. `build_encoder_decoder` 构造等距编码 `E` 与解码 `D=E^\dagger`，把逻辑 `|0>,|1>` 映射到 `|000>,|111>`。  
2. `build_stabilizer_projectors` 由 `S1=Z0Z1, S2=Z1Z2` 生成四个综合投影 `P_{a,b}`。  
3. `validate_projectors` 检查投影算符完备性、正交性与幂等性，保证综合分解合法。  
4. `bitflip_channel_three_qubits` 枚举 `e in {0,1}^3` 并加权求和，实现独立物理 `X` 噪声。  
5. `recover_density` 对每个综合分支计算 `P rho P`，并按综合映射应用 `I/X0/X1/X2` 条件纠错。  
6. `decode_density` 把恢复后的 3 比特态投回逻辑 1 比特空间，并做 trace 归一化。  
7. `logical_failure_probability_from_zero` 计算输入 `|0>` 时的逻辑失败概率，与理论 `3p^2-2p^3` 对照。  
8. `run_mvp/main` 输出综合概率、失败率曲线和诊断信息，并通过断言闭环验证。  

说明：`numpy/pandas` 仅用于线性代数与结果表格化；编码、噪声、综合恢复和理论对比均为源码显式实现，不依赖黑盒量子算法库。
