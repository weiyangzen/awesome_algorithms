## R01

洛伦兹变换描述两个匀速相对运动惯性系之间的时空坐标变换。  
在 1+1 维（仅考虑 `t-x`）且采用自然单位 `c=1` 时，标准形式为：

- `t' = gamma * (t - beta * x)`
- `x' = gamma * (x - beta * t)`
- `gamma = 1 / sqrt(1 - beta^2)`, `|beta| < 1`

矩阵写法：

`[t', x']^T = Lambda(beta) [t, x]^T`，其中  
`Lambda(beta) = [[gamma, -gamma*beta], [-gamma*beta, gamma]]`。

## R02

洛伦兹变换是狭义相对论中“坐标系统一”的基础工具，用来连接：

- 时间膨胀（time dilation）；
- 长度收缩（length contraction）；
- 同时性的相对性；
- 速度叠加公式与因果结构。

它不是“近似修正项”，而是保证光速不变与闵可夫斯基间隔不变的核心对称变换。

## R03

本条目 MVP 目标：

- 实现 1+1 维洛伦兹变换矩阵与事件批量变换；
- 数值验证闵可夫斯基间隔 `s^2 = t^2 - x^2` 在变换前后不变；
- 验证逆变换 `Lambda(-beta)` 可恢复原事件；
- 通过世界线变换验证相对论速度叠加公式；
- 给出“同一时刻事件在另一系不同步”的可计算示例。

## R04

本实现使用的核心方程：

1. 洛伦兹因子：`gamma(beta) = 1/sqrt(1-beta^2)`。  
2. 变换矩阵：`Lambda(beta)` 如 R01。  
3. 闵可夫斯基间隔：`s^2 = t^2 - x^2`。  
4. 间隔不变性：`s'^2 = s^2`。  
5. 逆变换：`Lambda(-beta) * Lambda(beta) = I`。  
6. 速度叠加（同方向）：`u' = (u - beta)/(1 - u*beta)`。

`demo.py` 在源码中显式实现以上关系，不依赖任何相对论黑盒库。

## R05

设事件数为 `N`，世界线采样点为 `M`：

- 批量事件变换：矩阵乘法 `O(N)`；
- 间隔计算：`O(N)`；
- 世界线速度估计：`O(M)`；
- 其余标量检查：`O(1)`。

总体时间复杂度 `O(N+M)`，空间复杂度 `O(N+M)`（为可读性保留全部时序）。

## R06

`demo.py` 包含三个非交互演示：

- **Demo A: 间隔不变性与逆变换校验**  
  对一组事件计算 `s^2`、`s'^2`、恢复误差、矩阵闭合误差。
- **Demo B: 速度叠加校验**

- **Demo C: 同时性相对性示例**  
  在 `S` 中同时发生的两事件（`Delta t=0`）经变换后出现 `Delta t' != 0`，并与解析表达式比较。

## R10

正确性直觉：

1. 若变换正确，`s^2=t^2-x^2` 对每个事件都应保持不变。  
2. 若 `Lambda(-beta)` 真是逆变换，先变换再逆变换应回到原点。  
3. 一条匀速直线世界线在另一惯性系仍是匀速直线，其斜率满足速度叠加公式。  
4. 在 `S` 同时（`Delta t=0`）的空间分离事件，在 `S'` 中一般不同步（`Delta t'=-gamma*beta*Delta x`）。

## R11

数值与实现策略：

- 使用 `float64` 保证矩阵与差分计算稳定；
- 对 `beta` 强制 `|beta|<1`，防止 `gamma` 无定义；
- 用“最大绝对误差 + 相对误差”联合评估；
- 速度由大量均匀采样点求平均斜率，降低离散噪声。

## R12

关键参数：

- `beta`：参考系 `S'` 相对 `S` 的速度（单位 `c`）；
- `events`：形状 `(N,2)` 的事件数组，每行 `[t, x]`；
- `u`：粒子在 `S` 中的速度；
- `n_points`：世界线采样点数；
- `x0`：世界线初始位置偏置。

调参建议：

- 若误差偏大，先提高 `n_points`；
- 避免 `beta` 或 `u` 过于接近 `±1` 导致数值病态；
- 调试符号错误时优先检查矩阵非对角项是否为 `-gamma*beta`。

## R14

常见失效模式：

1. 把 `x'` 公式写成 `gamma(x + beta t)`（符号错）；  
2. 行向量/列向量约定混淆，矩阵乘法方向颠倒；  
3. 忘记检查 `|beta|<1`，导致 `gamma` 非实数；  
4. 把欧氏范数当作时空间隔（应使用 `t^2-x^2` 而非 `t^2+x^2`）；  
5. 直接在很粗网格上估速度，导致与解析公式偏差较大。

## R18

`demo.py` 的源码级算法流程（8 步）：

1. `lorentz_gamma` 校验 `|beta|<1` 并计算 `gamma`。  
2. `lorentz_matrix` 基于 `gamma` 组装 `[[gamma,-gamma*beta],[-gamma*beta,gamma]]`。  
3. `transform_events` 对事件批量执行 `events @ Lambda(beta)^T`，得到 `S'` 坐标。  
4. `minkowski_interval_sq` 分别计算变换前后 `s^2=t^2-x^2`，在 `run_interval_invariance_demo` 中比较误差。  
5. 仍在 `run_interval_invariance_demo` 中调用 `Lambda(-beta)` 做逆变换恢复，并验证 `Lambda(-beta)Lambda(beta)≈I`。  
6. `build_uniform_worldline` 生成匀速世界线样本；`estimate_velocity_from_worldline` 用 `dx/dt` 平均斜率估计速度。  
7. `run_velocity_addition_demo` 将世界线变换到 `S'`，并与 `velocity_addition` 的解析值对比。  
8. `run_relativity_of_simultaneity_demo` 构造 `Delta t=0` 事件对，验证 `Delta t'=-gamma*beta*Delta x`，最后在 `main` 打印指标并断言通过。  

流程中没有把第三方包当黑盒：`numpy` 仅用于数组、矩阵和基础数值操作，物理公式与验证逻辑都在源码里逐步展开。
