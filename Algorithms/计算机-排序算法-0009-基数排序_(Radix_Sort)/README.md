## R01

基数排序（Radix Sort）是一类“按位分配”的非比较排序算法。它不直接比较两个元素大小，而是按数位（个位、十位、百位等）反复做稳定分配，逐步得到全局有序结果。

本目录 MVP 聚焦最常见的 `LSD`（Least Significant Digit）版本：
- 从最低位开始处理到最高位；
- 每一位都用稳定计数分配；
- 支持负整数（通过统一偏移转成非负数后排序，再还原）。

## R02

定位与价值：
- 对定长整数键，基数排序可达接近线性的时间复杂度；
- 与快排/堆排等比较排序不同，它不受 `O(n log n)` 比较下界约束；
- 工程中常用于整数键、字符串键、后缀数组等场景的高性能排序子步骤。

在教学层面，它能很好体现“稳定排序 + 多关键字逐位合成全序”的思想。

## R03

问题定义（本条目）
- 输入：一维整数序列 `A`（允许重复值、负数、0）。
- 输出：非降序序列 `B`，满足：
  - 有序性：`B[0] <= B[1] <= ... <= B[n-1]`
  - 排列保持：`B` 与 `A` 具有相同元素多重集。

实现目标：
- 主排序逻辑必须手写，不调用现成排序黑箱；
- 输出每轮按位分配轨迹，支持人工审计；
- 用 `sorted` 与 `numpy.sort` 做结果一致性校验。

## R04

核心机制（LSD 基数排序）：
1. 先把所有数平移到非负区间（若最小值 < 0，则加 `shift = -min(A)`）。
2. 从个位开始，按当前位数字 `digit = (x // exp) % base` 分桶。
3. 用稳定计数分配把同位数字从小到大重排。
4. `exp *= base`，继续处理下一位。
5. 所有位处理完后，减回 `shift` 得到最终结果。

关键点：每轮必须稳定，否则低位建立的相对次序会被破坏。

## R05

数据结构与状态量：
- `arr`：待排序数组（拷贝后原地替换）；
- `base`：基数，本实现取十进制 `10`；
- `exp`：当前位权（`1, 10, 100, ...`）；
- `counts[0..9]`：当前位频次统计；
- `output`：当前位稳定分配后的临时数组；
- `trace`：每一位处理后的轨迹（位权、桶频次、数组状态）。

统计指标：
- `digit_passes`：处理了多少个数位；
- `bucket_writes`：写回次数（每轮约 `n` 次）；
- `shift`：负数平移量。

## R06

正确性要点：
- 单轮正确性：稳定计数分配保证按当前位非降序，且同位元素相对次序不变；
- 归纳不变式：处理完第 `k` 位后，数组按“最低 `k` 位组成的键”有序；
- 终止结论：当最高有效位处理完成后，数组按完整数值有序。

关于负数：统一平移 `x -> x + shift` 是单调映射，不改变元素间大小关系，因此排序后再减回 `shift` 仍保持正确顺序。

## R07

时间复杂度：
- 设元素个数为 `n`，最大平移后值为 `M`，基数为 `b`；
- 位数 `d = floor(log_b(M)) + 1`（`M=0` 时取 `d=1`）；
- 每一位处理成本 `O(n + b)`；
- 总复杂度 `O(d * (n + b))`。

当 `b` 固定（如 10）且 `d` 相对可控时，常写作接近 `O(n*d)`。

## R08

空间复杂度：
- 主过程需要一个长度为 `n` 的临时数组 `output`；
- 桶计数数组长度为 `b`（常数）；
- 额外空间为 `O(n + b)`，常写作 `O(n)`。

如果保留完整轨迹用于展示，轨迹存储额外增加约 `O(d*n)`，属于可选审计成本。

## R09

算法性质：
- 非比较排序：是；
- 稳定性：是（本实现每轮从右向左回填，保证稳定）；
- 原地性：否（需要 `output` 临时数组）；
- 自适应：一般不明显（主要由位数决定）；
- 对键域有要求：需要可提取离散“位”的键（本实现为整数）。

## R10

边界与输入约束：
- 空数组、单元素：直接返回；
- 重复值：支持且保持稳定次序；
- 负数：通过 `shift` 机制支持；
- 非整数、`NaN`、`Inf`：拒绝（抛异常）；
- 高维输入：拒绝，必须是一维序列。

MVP 的输入验证由 `validate_integer_sequence` 统一处理。

## R11

伪代码（十进制 LSD）：

```text
radix_sort_lsd(A, base=10):
  validate A as 1D integer sequence
  if len(A) <= 1: return A

  shift <- max(0, -min(A))
  arr <- [x + shift for x in A]
  max_val <- max(arr)
  exp <- 1

  while max_val // exp > 0:
    counts <- [0]*base
    for x in arr:
      counts[(x // exp) % base] += 1

    prefix-sum counts
    output <- [0]*len(arr)
    for i from len(arr)-1 downto 0:
      digit <- (arr[i] // exp) % base
      counts[digit] -= 1
      output[counts[digit]] <- arr[i]

    arr <- output
    exp <- exp * base

  return [x - shift for x in arr]
```

## R15

与常见排序对比：
- 对比快速排序：快排平均快、通用性强；基数排序在整数键位数较短时可能更有优势；
- 对比归并排序：归并稳定且 `O(n log n)`，基数排序可在特定键域达到接近线性；
- 对比计数排序：计数排序依赖值域范围，基数排序按位分治，可缓解超大值域问题。

取舍核心：是否满足“可按位提取键 + 位数可控 + 稳定分配可接受额外内存”。

## R16

适用场景：
- 大量整数 ID、订单号、离散键排序；
- 字符串排序中的多关键字分位处理（扩展到字符位）；
- 后缀数组/字符串算法中的线性级子过程（基数分配）。

不建议场景：
- 通用可比较对象且无自然位表示；
- 内存极度受限、不能接受 `O(n)` 临时空间；
- 位数过大导致常数开销偏高。

## R18

`demo.py` 源码级流程拆解（8 步，非黑箱）：
1. `main` 生成固定与随机测试数据，并追加空数组、单元素边界样例。
2. `run_case` 调用 `validate_integer_sequence`，确保输入是一维、有限、整数序列。
3. `radix_sort_lsd` 计算 `shift`，把原数组平移到非负区间，避免负数位提取歧义。
4. 按 `exp=1,10,100...` 进入循环；每轮调用 `_stable_counting_pass` 处理当前位。
5. `_stable_counting_pass` 先统计当前位桶频次 `counts`，再做前缀和得到每桶结束位置。
6. 该函数从右向左扫描输入，把元素放入 `output[counts[digit]-1]`，保证稳定性。
7. 每轮结束后记录轨迹（`exp`、桶频次、当前数组），并累计 `digit_passes/bucket_writes`。
8. 全部位处理完成后减回 `shift` 得到最终结果，再用 `sorted` 与 `numpy.sort` 仅作对照校验并输出。

补充：第三方库只承担校验与展示职责，排序主流程完全在源码中显式实现。
