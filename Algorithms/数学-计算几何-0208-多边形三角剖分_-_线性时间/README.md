# 多边形三角剖分 - 线性时间

- UID: `MATH-0208`
- 学科: `数学`
- 分类: `计算几何`
- 源序号: `208`
- 目标目录: `Algorithms/数学-计算几何-0208-多边形三角剖分_-_线性时间`

## R01

多边形三角剖分的目标是：对一个简单多边形加入若干不相交对角线，把区域拆成 `n-2` 个三角形（`n` 为顶点数）。

“线性时间”在理论上对应一般简单多边形的复杂算法（代表性结果是 Chazelle 线性时间算法），实现难度高、工程可读性差。本条目 MVP 采用可审计的线性核心：**y-单调多边形**的栈式三角剖分。

## R02

`demo.py` 解决的问题范围：
- 输入：一个简单且 y-单调的多边形顶点序列（脚本内置固定样例）；
- 输出：对角线集合（三角剖分结果）、恢复出的三角形集合、一致性校验结果。

脚本不需要交互输入，直接运行即可看到完整过程与校验。

## R03

基础性质：
- 简单多边形三角剖分后一定有 `n-3` 条内部对角线、`n-2` 个三角形；
- y-单调多边形可拆成“左链 + 右链”，两条链都从最高点走到最低点且 y 不上升；
- 经典栈算法在已知链结构后可在线性时间完成三角剖分。

本实现首先把输入规范到 CCW，再做链分类与单调性检查。

## R04

线性时间核心思路：
1. 找到 top / bottom 顶点；
2. 沿正反方向各走一条 top->bottom 链；
3. 验证两条链 y 值非增；
4. 用水平参考线判定哪条是左链、哪条是右链；
5. 线性归并两条链，得到按 y 递减的处理顺序；
6. 按“同链/异链”规则用栈生成对角线。

这样避免了全量排序的 `O(n log n)`，保持三角剖分主流程 `O(n)`。

## R05

数据结构设计（`demo.py`）：
- `Point = Tuple[float, float]`：二维点；
- `Edge = Tuple[int, int]`：按索引记录无向边（对角线）；
- `chain_map: Dict[int, str]`：`left/right/top/bottom` 链标签；
- `stack: List[int]`：栈式可见性维护结构。

所有几何运算都在源码显式实现，没有调用第三方几何库。

## R06

正确性校验分三层：
- 计数校验：对角线数应为 `n-3`；
- 结构校验：由“边界边 + 对角线”恢复三角形后应为 `n-2`；
- 面积校验：三角形面积和应等于原多边形面积（容差 `1e-9`）。

`demo.py` 运行后会输出 `diagonal_count_ok / triangle_count_ok / area_match_ok`。

## R07

复杂度：
- 链提取、单调性检查、链归并、栈式剖分：`O(n)`；
- 额外空间：`O(n)`（链、顺序、栈、对角线）；
- 面积与计数校验：`O(n)`；
- 三角形恢复函数 `triangles_from_edges` 是演示用途，采用三元组枚举，复杂度 `O(n^3)`（不属于核心三角剖分算法）。

## R08

鲁棒性与异常处理：
- 顶点数 `<3`：直接报错；
- 非 y-单调输入：在链 y 非增检查阶段报错；
- 链左右判定失败（参考线重合等退化）：报错；
- 剖分结果若对角线数不等于 `n-3`：报错。

因此脚本会“尽早失败”，避免静默输出错误结果。

## R09

MVP 范围与边界（诚实约束）：
- 已实现：y-单调简单多边形的线性时间三角剖分核心；
- 未实现：一般简单多边形的完整线性时间管线（例如 Chazelle 全流程）；
- 未实现：带孔多边形、强退化输入（大量共线与重顶点）的完整工程处理。

该取舍保证了“算法透明 + 代码可读 + 可运行验证”。

## R10

`demo.py` 函数分工：
- `signed_area2 / orientation`：基础几何算子；
- `ensure_ccw`：统一顶点方向；
- `classify_y_monotone_chains`：链构造、单调检查、左右判定；
- `merge_chains_descending_y`：线性归并得到处理顺序；
- `triangulate_y_monotone_linear`：主栈算法，产出对角线；
- `triangles_from_edges`：由边集合恢复三角形（报告用途）；
- `run_demo / main`：执行样例与打印校验。

## R12

输出字段解释：
- `vertex_count`：顶点数 `n`；
- `triangles_expected / diagonals_expected`：理论应为 `n-2 / n-3`；
- `Merged top->bottom processing order`：线性归并后的处理序列；
- `Chain labels`：每个顶点所属链；
- `Diagonals`：三角剖分内部对角线；
- `Checks`：计数和面积一致性结论。

## R14

关键实现参数：
- `EPS = 1e-12`：几何比较阈值；
- 链归并并列规则：y 相等时按 x 决定先后；
- 面积匹配阈值：`1e-9`。

当输入坐标尺度非常大或非常小，可按量纲调整 `EPS`。

## R15

和常见替代方案对比：
- 耳切法（Ear Clipping）：通用、实现直观，但通常 `O(n^2)`；
- Delaunay 三角化：优化角质量但不是原多边形受限剖分的直接替代；
- 本实现：限定 y-单调输入，主流程 `O(n)`，适合教学和可审计实现。

## R18

`demo.py` 源码级算法流程（8 步）：
1. `main` 提供固定 y-单调样例并调用 `run_demo`。  
2. `triangulate_y_monotone_linear` 先用 `ensure_ccw` 统一为 CCW 顶点序。  
3. `classify_y_monotone_chains` 找到 top/bottom，构造正反两条链并验证 y 非增，再按参考线交点判左链/右链。  
4. `merge_chains_descending_y` 在线性时间归并两条链，得到 top->bottom 的全局处理顺序。  
5. 主循环按“当前点与栈顶是否同链”分支：异链时批量连对角线并重置栈；同链时按方向谓词（`orientation`）持续弹栈并连可见对角线。  
6. 循环结束后用 bottom 顶点收尾，补齐剩余对角线，并验证数量必须是 `n-3`。  
7. `triangles_from_edges` 用“边界边 + 对角线”恢复三角形面集合，用于报告与验证。  
8. `run_demo` 输出对角线、三角形与三项校验（对角线数、三角形数、面积守恒），给出可直接审计的结果。

说明：核心三角剖分逻辑完全在源码中展开，没有调用黑盒三角剖分库。
