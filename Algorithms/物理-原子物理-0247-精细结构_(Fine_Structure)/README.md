# 精细结构 (Fine Structure)

## R01

精细结构描述的是氢样原子能级在“非相对论主能级”之上出现的更细分裂。其主导来源通常可归纳为三项：

- 相对论动能修正；
- 自旋-轨道耦合；
- Darwin 项（对 `l=0` 态有效）。

在合并后的一阶结果里，能量修正主要依赖于 `(n, j)`，而不显式依赖 `l`（忽略 Lamb shift）。

## R02

该算法条目的价值在于把“可观测谱线微小分裂”映射到可计算公式：

- 解释为什么同一主量子数 `n` 下不同总角动量 `j` 会有不同能量；
- 给出分裂量级随 `Z^4` 增长的定量规律；
- 为塞曼效应、超精细结构等更复杂修正提供基线；
- 形成一个适合教学与自动化回归测试的最小可复现数值闭环。

## R03

本目录 `demo.py` 的输入输出约定（无交互）：

- 输入（脚本内置）：
1. 基本物理常数（`scipy.constants`）；
2. 氢原子与类氢离子参数（`Z=1,2`）；
3. 能级枚举上限 `n_max=4`。

- 输出：
1. `(n,l,j)` 能级表：非相对论能量、精细结构修正、近似总能量、Dirac 精确值；
2. `2p_{1/2}` 与 `2p_{3/2}` 分裂（eV 与 GHz）；
3. 近似公式与 Dirac 公式误差；
4. `Z^4` 标度验证；
5. `All checks passed.`。

## R04

MVP 使用的核心公式：

1. 非相对论束缚能（含可选约化质量 `mu`）：
`E_n^(0) = - mu*c^2*(Z*alpha)^2 / (2*n^2)`。

2. 精细结构一阶修正（按 `(n,j)`）：
`Delta E_fs = E_n^(0) * (Z*alpha)^2 / n * (1/(j+1/2) - 3/(4n))`。

3. 近似总能量：
`E_{n,j}^(approx) = E_n^(0) + Delta E_fs`。

4. Dirac 精确束缚能（扣除静质量）：
`E_{n,j}^(Dirac) = mu*c^2 / sqrt(1 + (Z*alpha)^2/(n-delta_j)^2) - mu*c^2`，
其中 `delta_j = j+1/2 - sqrt((j+1/2)^2 - (Z*alpha)^2)`。

5. 分裂频率换算：
`nu = |Delta E| * e / h`。

## R05

设最大主量子数为 `N`，总态数约 `sum_{n=1..N} (2n-1) = O(N^2)`：

- 枚举量子态：`O(N^2)`；
- 每态常数时间能量计算：`O(1)`；
- 全表构建：`O(N^2)`；
- 空间复杂度：`O(N^2)`。

该复杂度足以支持教学级扫描和轻量工程验证。

## R06

`demo.py` 覆盖三个最小闭环：

- **闭环 A（公式闭环）**：非相对论项 + 精细结构项得到 `E^(approx)`；
- **闭环 B（模型互证）**：`E^(approx)` 与 `E^(Dirac)` 做逐态误差对比；
- **闭环 C（物理规律）**：验证 `2p` 分裂与 `Z^4` 比例，以及 `j` 顺序关系。

## R07

优点：

- 代码与物理公式一一对应，便于审计；
- 不把第三方库当黑箱：常数读取之外，主算法都在源码显式实现；
- 同时给出近似与“更精确参考”（Dirac），可自动化验算。

局限：

- 未包含 Lamb shift（因此 `2s_{1/2}` 与 `2p_{1/2}` 在本模型中简并）；
- 未处理多电子原子中的屏蔽与电子相关；
- 未进入超精细结构、QED 高阶修正。

## R08

前置知识与运行环境：

- 角动量量子数关系：`l=0..n-1`，`j=l±1/2`（`l=0` 仅 `j=1/2`）；
- 对精细结构微扰展开与 Dirac 结果的对应关系有基础认知；
- Python `>=3.10`；
- 依赖：`numpy`, `pandas`, `scipy`。

## R09

适用场景：

- 原子物理教学中展示精细结构分裂来源与数量级；
- 作为谱线计算前的 sanity check；
- 在自动测试里做“公式实现正确性”回归。

不适用场景：

- 需要和高精度实验频率逐 MHz 拟合；
- 需要 Lamb shift/超精细结构主导的能级比较；
- 多电子复杂原子结构计算。

## R10

正确性直觉：

1. 对固定 `n`，较小 `j` 通常得到更大的下移（更束缚）；
2. `2p_{3/2}` 与 `2p_{1/2}` 应发生可测分裂；
3. 分裂量对核电荷数满足近似 `Z^4`；
4. 微扰近似与 Dirac 精确值在低 `Z` 下应高度接近。

## R11

数值稳定策略：

- 使用 `float64` 并统一 SI -> eV 转换；
- 对 `sqrt((j+1/2)^2-(Z*alpha)^2)` 做合法域检查，防止高 `Z` 非物理输入；
- 验证时使用“相对 + 绝对”阈值，避免极小量比较失真；
- 全流程无随机数，结果可复现。

## R12

- `n_max`：能级枚举上限；
- `Z`：核电荷数，直接控制分裂尺度；

- 断言阈值：Dirac 误差、`Z^4` 比例误差、分裂频率区间。

调参建议：

## R14

常见失效模式：

1. 把 `j` 与 `l` 的关系写错（尤其 `l=0` 分支）；
2. eV/J 单位混用导致频率换算错误；
3. 直接把 `E_n^(0)` 当正值代入，导致修正项符号反了；
4. 忽略模型边界，用该脚本解释 Lamb shift 实验差异。

## R18

`demo.py` 源码级算法流程（8 步）：

1. `reduced_mass_hydrogen` 与常数定义阶段：从 `scipy.constants` 读取 `alpha,c,h,m_e,m_p,electron_volt`，统一单位基础。  
2. `enumerate_hydrogenic_states` 显式枚举每个 `n` 的 `(l,j)` 合法组合，避免把量子数关系隐藏在黑箱里。  
3. `nonrel_binding_energy_eV` 计算 `E_n^(0)`，作为精细结构展开的基线能量。  
4. `fine_structure_shift_eV` 按 `(n,j)` 公式计算一阶修正 `Delta E_fs`，并和 `E_n^(0)` 叠加得到 `E^(approx)`。  
5. `dirac_binding_energy_eV` 使用 `delta_j` 形式实现 Dirac 精确束缚能，作为近似解的对照标准。  
6. `build_energy_table` 对每个态写入 `pandas.DataFrame`，包含近似值、精确值和差值列，便于审计和测试。  
7. `compute_2p_splitting_eV` 直接从能级函数计算 `2p_{1/2}` 与 `2p_{3/2}` 的能量差，并换算 GHz。  
8. `main` 汇总打印、执行断言（Dirac 误差、`j` 顺序、`Z^4` 比例、GHz 量级），全部通过后输出 `All checks passed.`。  

说明：第三方库在本实现中仅提供数值容器和物理常数；量子态枚举、能量公式、单位换算与验证逻辑都在源码中展开。
