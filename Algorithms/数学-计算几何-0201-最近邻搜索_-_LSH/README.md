# 最近邻搜索 - LSH

- UID: `MATH-0201`
- 学科: `数学`
- 分类: `计算几何`
- 源序号: `201`
- 目标目录: `Algorithms/数学-计算几何-0201-最近邻搜索_-_LSH`

## R01

算法目标: 在中高维向量集合中，快速返回“近似最近邻”。本条目使用 `LSH (Locality-Sensitive Hashing)` 的随机超平面方案，将向量映射到多个哈希桶中，再只在候选集中做精确重排，减少全量扫描成本。

## R02

问题形式化:
- 输入: 数据向量集合 `X = {x_i | x_i in R^d}`，查询向量 `q in R^d`
- 输出: 近似 `argmax_i cos(x_i, q)`（实现中先做 L2 归一化，再最大化内积）
- 距离/相似度: 余弦相似度（等价角距离排序）
- 任务类型: 静态数据集、批量查询的近似最近邻检索

## R03

核心思想:
- 使用 `L` 个独立哈希表，每个表由 `k` 个随机超平面组成。
- 单表中，向量在每个超平面上取符号位，得到 `k` 位二进制签名并映射为桶 ID。
- 查询时跨表收集同桶样本并取并集作为候选集，再在候选上做精确余弦比较。
- 候选过小时启用一跳 `multiprobe`（翻转 1 bit）补充邻近桶，平衡召回率与速度。

## R04

数据结构设计:
- `RandomHyperplaneLSH`:
  - `hyperplanes`: 形状 `(num_tables, num_bits, dim)` 的随机超平面参数
  - `tables`: `List[Dict[int, List[int]]]`，每个表是“桶 ID -> 数据索引列表”
  - `data_unit`: 归一化后的数据矩阵
  - `bit_weights`: 位权重数组，用于把 bit 向量编码成整数桶 ID
- 查询输出: `(best_idx, best_sim, candidate_count)`，用于同时观察命中质量与候选规模

## R05

构建流程（`fit`）:
1. 检查输入维度与参数合法性。
2. 对数据逐行做 L2 归一化，统一为余弦检索空间。
3. 为每个哈希表采样 `k` 个高斯随机超平面。
4. 计算 `data_unit @ hyperplanes[t].T` 得到符号位矩阵。
5. 将 bit 向量编码为整数桶键，写入该表 `buckets[key].append(idx)`。

## R06

查询流程（`query` / `query_unit`）:
1. 将查询向量归一化（若已归一化可直接走 `query_unit`）。
2. 在每个表计算主桶 ID，并收集同桶样本作为初始候选。
3. 若候选不足 `min_candidates`，执行一跳 `multiprobe` 补充邻桶。
4. 若仍为空，兜底回退到全量样本（保证总有返回结果）。
5. 在候选集上精确计算余弦相似度，返回最大者。

## R07

复杂度分析（设 `n` 为样本数，`d` 为维度，`L` 为表数，`k` 为每表 bit 数，`m` 为候选数）:
- 构建: 约 `O(L * n * d * k)`（矩阵乘与建桶）
- 查询哈希阶段: `O(L * d * k)`
- 查询重排阶段: `O(m * d)`
- 总体: `m << n` 时显著快于暴力 `O(n * d)`；最坏退化仍可能接近暴力
- 存储: 超平面 `O(L * k * d)` + 桶索引 `O(L * n)`

## R08

正确性与近似性要点:
- 同侧概率性质: 夹角更小的向量在随机超平面下更易获得相同符号，从而更易碰撞入同桶。
- 多表提高召回: 任一表发生碰撞即可进入候选，并集策略降低漏检概率。
- 精确重排保证“候选内最优”: 候选选出结果一定是该候选集合中的真实最优余弦邻居。
- `demo.py` 同时给出暴力精确结果作为参照，输出 `Recall@1` 与相似度损失。

## R09

MVP 范围:
- 支持静态数据集构建与批量查询
- 支持余弦相似度近似最近邻（Top-1）
- 支持基础 multiprobe（Hamming 距离 1）
- 包含与暴力法的速度、召回率、候选规模对比
- 不包含动态增删、Top-k、磁盘索引、分布式部署

## R12

样例输出解读:
- `固定探针(数据内)命中`: 自查询烟雾测试，验证基础可用性
- `固定查询结果 exact | lsh`: 单例查询质量对照
- `查询加速比(精确/LSH)`: 与暴力检索的速度收益
- `Recall@1`: LSH 返回与精确最近邻一致的比例
- `平均相似度损失`: 近似解相对精确解的质量损失
- `候选规模统计`: 反映哈希配置与 multiprobe 的筛选强度

## R13

与暴力法对比:
- 暴力法: 每次对全部 `n` 向量算相似度，结果精确但代价稳定为 `O(n*d)`。
- LSH: 先粗筛再精排，常见情况下可显著降低查询时间。
- 代价交换: 查询更快但可能错过真正最近邻，需用 `Recall@1` 评估可接受性。
- 参数敏感: `num_tables`、`num_bits`、`min_candidates` 决定速度与召回平衡点。

## R14

工程注意事项:
- 统一归一化是余弦检索正确性的前提。
- `num_bits` 过大导致桶稀疏、召回下降；过小导致候选过大、速度下降。
- 启用 multiprobe 可补召回，但会增大候选规模。
- 代码含“空候选全量兜底”，避免线上出现无结果异常。

## R15

测试策略:
- 烟雾测试: 查询数据内已知向量，验证能返回自身或相同向量。
- 有效性测试: 批量查询统计 `Recall@1` 与平均相似度损失。
- 性能测试: 对同一批查询分别跑暴力与 LSH，比较耗时和加速比。
- 稳定性测试: 固定随机种子，保证实验可复现与参数可比。

## R16

已知局限:
- 当前为 Top-1；未实现 Top-k 返回。
- multiprobe 仅实现 Hamming-1，未做更深半径或自适应策略。
- 该实现以内存字典为主，未针对超大规模做压缩与持久化。
- 高维极端分布下，LSH 召回和候选规模可能波动较大。

## R17

可扩展方向:
- 增加 Top-k: 对候选相似度排序后返回前 `k` 个。
- 自适应 multiprobe: 按候选不足程度动态扩展探测半径。
- 参数调优自动化: 网格搜索 `(L, k, min_candidates)` 的 Pareto 前沿。
- 工程替代: 迁移到 FAISS/HNSW/Annoy 做更高吞吐 ANN 服务。

## R18

源码级算法流（`demo.py`，非黑盒拆解）:
1. `main()` 生成随机数据与查询向量，创建 `RandomHyperplaneLSH` 并调用 `fit`。
2. `fit` 先归一化数据，再采样 `num_tables * num_bits` 个随机超平面参数。
3. 对每个表计算 `data_unit @ hyperplanes[t].T`，将符号位编码成整数桶键，构建 `Dict[bucket -> indices]`。
4. 查询时 `query/query_unit` 对输入向量归一化，并调用 `_collect_candidates`。
5. `_collect_candidates` 先取各表主桶并集；若候选过少，再对每表主桶做按位翻转（一跳 multiprobe）补桶。
6. 若候选仍为空则回退到全量索引，确保系统总能返回合法答案。
7. `query_unit` 在候选子集上做一次精确余弦相似度比较，返回最佳索引与相似度。
8. `main()` 用 `exact_nearest_cosine` 逐查询做暴力对照，计算 `Recall@1`、平均相似度损失、候选统计与加速比。
