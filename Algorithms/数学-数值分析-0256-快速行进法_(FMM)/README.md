## R01

快速行进法（Fast Marching Method, FMM）是求解 Eikonal 方程
`F(x,y)|∇T(x,y)| = 1` 的经典单调前沿推进算法。

它把“到达时间场”`T` 从已知源点向外扩展，保证每次固定（freeze）的点都是当前全局最小的候选时间，因此可稳定处理“首达时间、最短传播时间、波前到达顺序”等问题。

## R02

本目录实现的是二维笛卡尔网格上的各向同性情形：

- 未知量：每个网格点的到达时间 `T[i,j]`
- 已知量：速度场 `F[i,j] > 0`（`F=0` 视为不可通行障碍）
- 源条件：源点满足 `T=0`
- 目标：计算所有可达点的最早到达时间

备注：这里的 FMM 是 Fast Marching Method，不是“快速多极子方法（Fast Multipole Method）”。两者同缩写但问题类型完全不同。

## R03

算法动机可以理解为“连续版 Dijkstra”用于 PDE：

- Dijkstra 在图上按最小距离扩展
- FMM 在网格 PDE 上按最小到达时间扩展
- 通过局部离散方程更新候选值，而不是直接使用边权相加

因此 FMM 在几何距离场、路径规划、图像分割、层析重建、前沿传播等任务中非常常见。

## R04

本实现使用三态集合管理节点：

- `FAR`：尚未进入窄带，时间未知（`inf`）
- `NARROW`：候选前沿，已经有暂定值，放入最小堆
- `FROZEN`：时间已最终确定，不再改变

核心不变量：`FROZEN` 中的 `T` 按冻结顺序非递减，且一旦冻结即视作最优解。

## R05

局部更新采用一阶迎风离散。对节点 `(i,j)`，先取两个坐标方向上的最优冻结邻点时间：

- `a = min(T[i-1,j], T[i+1,j])`（仅统计 `FROZEN` 邻点）
- `b = min(T[i,j-1], T[i,j+1])`（仅统计 `FROZEN` 邻点）

然后求解局部方程：

`(max(T-a,0))^2 + (max(T-b,0))^2 = (h/F)^2`

在二维情况下可化为显式分段公式（`demo.py` 的 `solve_local_eikonal_2d`），避免黑盒方程求解器。

## R06

算法流程（高层）：

1. 初始化：源点置 `T=0` 并标记 `FROZEN`。
2. 将源点邻居通过局部更新放入 `NARROW` 最小堆。
3. 反复弹出堆顶最小候选。
4. 若候选已过期（堆中旧条目）则丢弃。
5. 将该点冻结为 `FROZEN`。
6. 用该新冻结点尝试改进其 4 邻居的候选值并回推入堆。
7. 堆空即结束。

## R07

复杂度（`N` 为可达网格点数）：

- 时间复杂度：`O(N log N)`（最小堆 push/pop 主导）
- 空间复杂度：`O(N)`（时间场、状态场、堆）

与直接逐点全局迭代相比，FMM 利用“因果有序推进”显著减少无效更新。

## R08

正确性直觉：

- 局部更新只依赖已冻结邻域（迎风、因果）。
- 堆顶保证当前全局最小候选先被冻结。
- 冻结后不会被更小值改写，避免来回震荡。

在速度正且离散满足单调性的前提下，这种“单调接受（acceptance）”机制可稳定收敛到粘性解的离散近似。

## R09

数值性质：

- 本 MVP 使用一阶 4 邻域迎风格式，整体精度约为一阶。
- 在常速单源测试中，误差与网格步长 `h` 同数量级。
- 近对角方向会有栅格各向异性误差，这是低阶网格离散的典型现象。

## R10

`demo.py` 内置两个无交互案例：

- Case A（可验证精度）：
  - `101x101` 网格，`F=1` 常速，中心单源。
  - 与解析欧氏距离比较，断言：`max_err < 2.5h`、`mean_err < 0.8h`。
- Case B（异质速度）：
  - `121x121` 网格，含慢速环带（`F=0.45`）和不可通行障碍（`F=0`）。
  - 输出可达点比例与到达时间统计分位。

## R11

MVP 设计取舍：

- 依赖最小化：仅 `numpy` + 标准库 `heapq`。
- 不调用现成 FMM 第三方黑盒。
- 把局部 Eikonal 更新公式显式写在源码里，便于检查每一步数据流。
- 优先可读性与可验证性，不追求高阶模板或并行优化。

## R12

代码结构与职责映射：

- `solve_local_eikonal_2d`：二维局部显式更新公式
- `axis_min_frozen`：提取某坐标轴上的冻结邻点最小时间
- `local_update`：组合局部更新输入并返回单点候选
- `fast_marching`：堆驱动主循环（初始化、弹堆、冻结、扩展）
- `build_piecewise_speed`：构造异质速度+障碍测试场景
- `run_constant_speed_check` / `run_heterogeneous_demo`：验证与演示

## R14

典型输出现象（一次实测）：

- Case A：
  - `max_abs_err` 约 `1e-2`
  - `mean_abs_err` 约 `1e-3 ~ 1e-2`
- Case B：
  - 障碍区域不可达（`T=inf`）
  - 慢速带使传播时间明显拉长（高分位时间上升）

这与“速度越慢、路径越受阻、到达时间越大”的物理直觉一致。

## R15

常见实现坑：

- 把 `NARROW` 直接当最终值，不做 `FROZEN` 固定步骤。
- 局部更新时错误使用未冻结邻点，破坏因果性。
- 处理堆中过期条目不当，导致重复冻结或错误回退。
- 忽略 `F=0` 特判，出现除零或把障碍误当可达点。
- 把网格步长 `h` 与坐标域尺度混淆，导致时间量纲不一致。

## R16

可扩展方向：

- 更高阶迎风格式（减小栅格各向异性）

## R17

与相关方法对比：

- 对比 Dijkstra：
  - Dijkstra 作用于离散图边权；FMM 作用于 Eikonal PDE 离散。
- 对比 Fast Sweeping：
  - Fast Sweeping 通过多方向扫描迭代；
  - FMM 通过堆维护前沿有序性，通常更适合单调首达型传播。
- 对比最短路网格化方法：
  - FMM 的局部更新保留了连续传播模型的信息，而非纯图近似。

## R18

`demo.py` 的源码级算法流程（8 步，非黑盒）：

1. `run_constant_speed_check` 或 `run_heterogeneous_demo` 先构造 `speed` 与 `source`，再调用 `fast_marching`。
2. `fast_marching` 初始化 `t=inf`、`state=FAR`，把源点设为 `T=0` 并标记 `FROZEN`。
3. 对源点 4 邻居调用 `local_update`，生成初始窄带条目并压入最小堆。
4. 主循环每次 `heapq.heappop` 取当前最小候选；若该条目比 `t[i,j]` 大则视为过期并跳过。
5. 对有效候选，将节点状态设为 `FROZEN`，表示该点时间被最终接受。
6. 对其每个非冻结邻点调用 `local_update`；其中 `axis_min_frozen` 只收集冻结邻点时间，再由 `solve_local_eikonal_2d` 用显式分段公式求新候选。
7. 若新候选更小，则更新 `t`，置邻点为 `NARROW`，并把新值再次压入堆。
8. 堆清空后返回完整到达时间场；外层例程计算误差或统计量并执行断言，验证实现正确性。

以上 8 步完整对应了本仓库 MVP 的数据结构、局部离散公式和全局推进顺序。
