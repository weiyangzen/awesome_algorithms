## R01

本条目实现“引力波探测器数据分析”的最小可运行 MVP，目标不是复刻 LIGO 全流程，而是保留核心算法骨架：
- 双探测器（H1/L1）时间序列。
- 含色噪声背景下的模板匹配滤波（matched filtering）。
- 双站巧合判据（SNR 阈值 + 到达时间差窗口）。
- 用 `scikit-learn` 与 `PyTorch` 对检测特征做可解释分类诊断。

## R02

问题背景（实验引力物理）：
- 引力波应变信号极弱，通常淹没在非白、非平稳噪声中。
- 单探测器峰值可能是噪声涨落，因此工程上依赖“跨探测器一致性”。
- 基础检索链路通常是：预处理 -> 匹配滤波 -> 峰值候选 -> 巧合检验 -> 统计判别。

本 MVP 对应这条链路的教学版与工程原型版。

## R03

物理/建模假设：
- 波形模板使用简化双星并合 chirp 近似（频率与振幅随时间上升）。
- 噪声采用频域整形的高斯色噪声，不包含真实仪器所有谱线和突发噪声。
- 只做单模板检索，不做模板库参数搜索。
- 以“检测判别演示”为主，不做天体参数精密反演。

## R04

时间复杂度（`N` 为单段采样点数，`E` 为事件数）：
- 频域匹配滤波单次约 `O(N log N)`（FFT 主导）。
- 每个事件双探测器处理仍是 `O(N log N)` 常数倍。
- 全数据集约 `O(E * N log N)`。
- `LogisticRegression`（小特征维度）和小型 MLP 在本规模上开销远小于滤波批处理。

## R05

空间复杂度：
- 单事件频域缓存（数据、模板、PSD、SNR 序列）约 `O(N)`。
- 整体结果表按事件存储，约 `O(E)`。
- 训练阶段仅保存低维特征矩阵，内存压力小。

## R06

`demo.py` 的最小实验流程：
1. 生成长度 2 秒、采样率 512 Hz 的 chirp 模板。
2. 构造 240 个事件，每个事件模拟 H1/L1 两路色噪声。
3. 随机一半事件注入同源信号，并加入毫秒级到达时延差。
4. 对每路数据做带通、Welch PSD 估计、匹配滤波，提取峰值 SNR。
5. 形成 `snr_h1/snr_l1/network_snr/lag_ms/corr/snr_balance` 特征。
6. 输出规则触发结果、sklearn 结果、PyTorch 结果并对比。

## R07

算法意义：
- 物理意义：将“弱信号埋噪声”的引力波探测核心难题转成可执行流程。
- 工程意义：展示多探测器巧合如何压低误报。
- 方法意义：同一特征可同时支持传统阈值法与学习器判别，方便做 A/B 比较。

## R08

直接使用的方法与组件：
- `scipy.signal.chirp`：构建简化 inspiral 模板。
- `scipy.signal.butter/sosfiltfilt`：带通预处理。
- `scipy.signal.welch`：PSD 估计。
- `numpy.fft.rfft/irfft`：频域匹配滤波。
- `pandas`：事件表和结果汇总。
- `scikit-learn`：`StandardScaler + LogisticRegression` 基线分类器。
- `PyTorch`：小型 MLP 二分类器。

## R09

适用边界：
- 适合教学、快速原型和流程回归测试。
- 不适合直接替代真实台站离线搜寻管线。
- 未覆盖模板库扫描、数据质量 veto、非平稳噪声自适应、参数估计与置信区间推断。

## R10

正确性依据（链路一致性）：
- 信号构造与注入在代码中显式可见，标签来源透明。
- 匹配滤波严格按 `d(f) * h*(f) / S_n(f)` 的频域形式计算并归一化。
- 巧合特征（双站 SNR、时延）与实验引力波检索原则一致。
- 分类器输入来自物理可解释特征，而非不可追踪隐变量。

## R11

数值稳定性设计：
- PSD 加下界裁剪，避免分母接近 0。
- 带通使用 `sosfiltfilt`，比直接高阶 `ba` 系数更稳定。
- 所有 SNR 与比例特征对极端值做防护（如除零保护）。
- 训练随机种子固定，保证结果可复现。

## R12

真实运行开销（超越大 O）：
- 主要时间花在每个事件的 Welch + FFT 匹配滤波。
- 数据规模小（240 段），CPU 上通常秒级完成。
- 机器学习部分输入仅 6 维，训练成本基本可忽略。

## R13

误差与不确定性来源：
- 主误差来自模型简化（toy 波形、toy 噪声），不是数值积分误差。
- 数据集标签由注入真值给出，不涉及人工标注偏差。
- 输出指标（F1/AUC）可用于比较方法相对优劣，不应解释为真实台站探测率。

## R14

鲁棒性与失效模式：
- 如果噪声与模板失配很强，匹配滤波峰值会下降，召回率降低。
- 当延迟超出巧合窗口，规则法会漏检。
- 若把阈值设得过低，噪声峰值会显著提高误报率。
- 学习器可能过拟合模拟分布，迁移到真实数据会性能退化。

## R15

工程实现要点：
- 使用 `dataclass` 统一配置，避免散落魔法数。
- 先做物理特征提取，再做分类，便于排错和扩展。
- 输出规则法与学习法并列指标，便于审阅。
- 全流程无交互输入，可直接纳入自动验证任务。

## R16

前驱/后继关系：
- 前驱：干涉测量、噪声谱估计、匹配滤波理论、统计检验。
- 同层：跨探测器巧合检验、触发候选排序。
- 后继：模板库大规模搜索、参数估计（质量/自旋/天空定位）、贝叶斯证据评估。
- 工程扩展：接入真实开源 strain 数据并替换模拟噪声。

## R18

`demo.py` 源码级算法流（8 步，非黑箱）：
1. `make_inspiral_template` 用 `scipy.signal.chirp` 生成随时间上升的频率，再叠加包络和 Tukey 窗，得到标准化模板。
2. `build_event` 调 `generate_colored_noise` 为 H1/L1 生成色噪声，并按随机标签决定是否注入同源 chirp 与毫秒级时延。
3. `feature_from_streams` 先对两路数据做 `butter + sosfiltfilt` 带通，再用 `welch` 分别估计 PSD。
4. `matched_filter_snr` 在频域显式计算 `irfft( d(f) h*(f) / S_n(f) )`，并按模板内积归一化得到 SNR 时序。
5. 从 SNR 时序提取峰值与峰值时刻，构造 `snr_h1/snr_l1/network_snr/lag_ms` 等双站巧合特征。
6. `rule_based_detect` 执行经典触发逻辑：双站 SNR 超阈值且时延在 10 ms 内，得到规则法基线。
7. `sklearn_detector` 用 `StandardScaler + LogisticRegression` 训练线性判别器，输出测试集概率和标签并计算 AUC/F1。
8. `torch_detector` 训练 3 层小型 MLP（`BCEWithLogitsLoss` + Adam），与规则法/逻辑回归并列输出，形成端到端对照。
