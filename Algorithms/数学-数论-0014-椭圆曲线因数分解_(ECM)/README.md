# 椭圆曲线因数分解 (ECM)

- UID: `MATH-0014`
- 学科: `数学`
- 分类: `数论`
- 源序号: `14`
- 目标目录: `Algorithms/数学-数论-0014-椭圆曲线因数分解_(ECM)`

## R01

ECM（Elliptic Curve Method）用于分解合数 `N`，目标是找到一个非平凡因子 `d`，满足 `1 < d < N` 且 `d | N`。它尤其适合“存在相对较小素因子”的大整数分解任务。

## R02

核心思想：在模 `N` 的椭圆曲线群上做标量乘法。若某一步需要计算分母的模逆，而该分母与 `N` 不互素，则无法求逆；此时通过 `gcd(denominator, N)` 直接暴露 `N` 的非平凡因子。

## R03

与常见分解法比较：
- 试除法：实现最简单，但复杂度高，只适合很小因子。
- Pollard Rho：对一般中小规模分解效果不错。
- Pollard p-1：依赖 `p-1` 的光滑性。
- ECM：依赖椭圆曲线群阶的光滑性，通常比 `p-1` 更“泛化”，在寻找中小素因子时非常实用。

## R04

数学对象（模 `N`）：
- 曲线：`y^2 = x^3 + ax + b (mod N)`
- 判别式：`Δ = 4a^3 + 27b^2 (mod N)`
- 若 `gcd(Δ, N)` 在 `(1, N)` 之间，立即得到因子。
- 点加/倍点都需要分母求逆，逆不存在时同样可以用 `gcd` 提取因子。

## R05

本目录 MVP 的输入输出定义：
- 输入：`demo.py` 内置合数 `N`（无需交互输入）。
- 输出：
  - 原始 `N`
  - ECM 找到的素因子列表（升序）
  - 乘积回验结果（确认因子乘积等于 `N`）

## R06

实现包含的最小组件：
- Miller-Rabin（64 位场景下的确定性底数集合）用于素性测试。
- 质数筛 `primes_up_to(B1)` 用于 Stage-1 指数构造。
- 椭圆曲线点运算：`ec_add`、`ec_mul`。
- Stage-1 主过程：`ecm_stage1`。
- 递归分解封装：`factorize_with_ecm`。

## R07

Stage-1 的流程：
1. 随机选曲线参数 `(a, b)` 与点 `P`。
2. 计算判别式与 `N` 的 gcd，若有非平凡因子直接返回。
3. 构造 `k = ∏ p^{⌊log_p(B1)⌋}`（通过逐个 prime power 乘到点上实现）。
4. 依次计算 `P <- [p^e]P`。
5. 任何一次点运算里，若分母与 `N` 的 gcd 在 `(1, N)`，得到因子。
6. 当前曲线失败则换下一条曲线重试。

## R08

MVP 采用“多曲线 + 多参数配置”的朴素策略：
- 先用较小 `B1` 快速试探；失败后增大 `B1` 和曲线数。
- 随机种子固定，保证示例可复现。
- 保持 Stage-1 版本，不引入 Stage-2，控制代码规模和可读性。

## R09

正确性直觉：
- 对目标素因子 `p | N`，曲线在 `mod p` 上形成有限阿贝尔群。
- 若构造的标量 `k` 被该群阶整除，则 `[k]P = O (mod p)`，运算分母会在 `mod p` 下退化。
- 这等价于分母与 `N` 共享非平凡因子，从而 `gcd` 能提取 `p`（或其倍数中的因子）。

## R10

复杂度（粗略）：
- 单条曲线 Stage-1 主要开销约为 `O(π(B1) * log N)` 次大整数模运算（省略常数）。
- 总复杂度取决于“找到成功曲线所需次数”。
- ECM 的优势是：期望时间更多受最小素因子大小影响，而非 `N` 全长。

## R11

`demo.py` 的工程约束：
- 纯 Python 标准库实现，避免重依赖。
- 无黑箱库调用；椭圆曲线点运算和 gcd 触发逻辑全部显式实现。
- 脚本一键运行：`python3 demo.py`。

## R12

预期运行现象：
- 脚本打印一个固定合数 `N`。
- ECM 递归找到若干素因子（通常是 2-4 个）。
- 最后打印回验：`product(factors) == N`。

## R13

常见失败原因与处理：
- 曲线过少：提高 `max_curves`。
- `B1` 太小：提高 Stage-1 上界。
- 目标因子偏大：需要更多曲线与更高 `B1`，或实现 Stage-2。
- `gcd == N`：表示该步信息无效，直接换曲线继续。

## R14

边界与鲁棒性处理：
- `N <= 1`：返回空因子列表。
- 偶数快速分解：先提取因子 2。
- 素数输入：直接返回该素数。
- ECM 多轮失败时，MVP 保底使用小范围试除（仅为演示完整闭环）。

## R15

自测建议：
- 小整数：`8051 = 83 * 97`。
- 含重复因子：`3 * 3 * 101 * 103`。
- 含一个较小素因子和一个较大余因子的半素数。
- 多次运行确认固定种子下输出稳定。

## R16

参数调优建议：
- 若目标是快速演示：`B1` 取 200~2000，曲线数 20~80。
- 若目标是更高成功率：提高 `B1`，并分层尝试多组 `(B1, max_curves)`。
- 若目标是实际更大整数：需要加入 Montgomery 形式和 Stage-2 才更有竞争力。

## R17

可扩展方向：
- 从仿射坐标升级到 Montgomery 曲线 + x-only ladder。
- 增加 Stage-2（如 Brent-Suyama 变体）。
- 并行多曲线（多进程/多线程）。
- 与 Pollard Rho、QS 做混合调度。

## R18

`demo.py` 的源码级算法流（非黑箱，8 步）：
1. `main` 设定固定合数 `N`，调用 `factorize_with_ecm(N)`。
2. `factorize_with_ecm` 先处理平凡情形（`N<=1`、偶数、素数）。
3. 对合数调用 `ecm_find_factor`，按照多组 `(B1, max_curves)` 配置逐轮尝试。
4. `ecm_find_factor` 每条曲线通过 `random_curve_and_point` 随机生成 `(a,b,P)`，并先检查判别式 gcd 是否直接泄露因子。
5. `ecm_stage1` 枚举 `B1` 内素数，构造每个 `p^e`，反复执行 `P <- [p^e]P`。
6. `ec_mul` 用二进制法做标量乘；内部调用 `ec_add` 做点加与倍点。
7. `ec_add` 在求分母逆前先做 `gcd(den, N)`：若在 `(1, N)`，立即返回因子；否则继续计算新点坐标。
8. 一旦找到因子 `d`，`factorize_with_ecm` 递归分解 `d` 和 `N//d`，汇总、排序并回验输出。
