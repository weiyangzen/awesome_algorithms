## R01

k-means 是一种无监督聚类算法，目标是把 `n` 个样本划分为 `k` 个簇，并最小化簇内平方和（WCSS / inertia）：

`J = sum_{i=1..n} ||x_i - mu_{c_i}||_2^2`

其中 `c_i` 是样本 `x_i` 的簇标签，`mu_j` 是第 `j` 个簇中心。算法通过“分配-更新”交替迭代近似求解。

## R02

k-means 的经典来源包括：
- MacQueen（1967）提出在线/迭代式聚类思想；
- Lloyd（1957 内部报告，1982 发表）给出常见的批处理迭代流程；
- Forgy 初始化与后续 k-means++ 初始化（Arthur & Vassilvitskii, 2007）提升了初始中心质量。

工程里通常说的 k-means，多指 Lloyd 风格的批处理版本。

## R03

该算法解决的问题是：在没有标签时，把数据自动分组，使组内样本尽量相似、组间尽量分离。

它本质是在优化一个非凸目标，难点是：
1. 同时求“簇分配”和“簇中心”；
2. 对初始化敏感，可能落入局部最优；
3. 需要用户提供 `k`。

## R04

Lloyd 迭代流程：
1. 初始化 `k` 个中心（随机或 k-means++）。
2. 分配步：每个样本指派到最近中心（最小欧氏距离）。
3. 更新步：每个簇中心更新为该簇样本均值。
4. 重复步骤 2-3，直到中心位移小于阈值或达到最大迭代轮数。
5. 输出标签、中心和最终 inertia。

## R05

设样本数 `n`、维度 `d`、簇数 `k`、迭代轮数 `T`：
- 单轮时间复杂度：`O(n*k*d)`（计算样本到各中心距离）。
- 总时间复杂度：`O(T*n*k*d)`。
- 空间复杂度：
  - 主要状态 `O(n)`（标签）与 `O(k*d)`（中心）；
  - 若显式存完整距离矩阵，则为 `O(n*k)`。

## R06

简化示例（一维）:
- 数据点：`[1, 2, 10, 11]`，`k=2`
- 初始中心：`mu1=1, mu2=11`

第一次分配：
- 簇 A: `[1,2]`
- 簇 B: `[10,11]`

第一次更新：
- `mu1=(1+2)/2=1.5`
- `mu2=(10+11)/2=10.5`

再次分配不变，算法收敛。最终把小值和大值自然分成两簇。

## R07

优点：
- 实现简单，速度快，适合大规模向量数据。
- 可解释性强，中心点与簇内方差直观。

局限：
- 只能拟合“近似球形、方差接近”的簇。
- 对异常值和初始化敏感。
- 需要预先给定 `k`。
- 目标非凸，只保证局部最优。

## R08

前置知识与实现基础：
- 欧氏距离与向量化计算。
- 均值作为平方误差下的最优代表点。
- 迭代优化与收敛判据（位移阈值/目标函数变化）。
- 随机数种子与可复现实验。

## R09

适用场景：
- 向量特征、簇形状近似凸/球形的数据。
- 需要快速基线聚类或做特征压缩（如词向量分桶）。

不适合或需谨慎：
- 强非凸簇、环形簇、密度差异大簇。
- 离群点很多、特征尺度差异大但未标准化的数据。
- 类别特征或非欧氏距离更合理的问题。

## R10

实现正确性的关键检查点：
1. 分配步确实按最近中心做 `argmin`。
2. 更新步中心确实是簇内样本均值。
3. 每轮 inertia 应非增（浮点误差下允许极小抖动）。
4. 空簇需要显式处理，避免均值 NaN。
5. 收敛条件应基于中心位移或目标变化，不应无限循环。

## R11

数值与工程稳定性要点：
- 使用平方距离，避免不必要的开方开销。
- 距离计算用向量化，减少循环误差和实现错误。
- 空簇重置策略（如重置到最差拟合样本）可避免崩溃。
- 固定随机种子，保证可复现。
- 若特征量纲差异大，应先标准化再聚类。

## R12

真实成本和调参经验：
- `k` 是最关键超参数，可结合肘部法、轮廓系数或业务先验。
- 初始化很关键：k-means++ 通常显著优于纯随机。
- `max_iters` 通常几十到几百足够。
- 多次重启（`n_init` 思路）可以降低落入差局部最优的概率。

## R13

理论保证与边界：
- 全局最优求解一般是困难问题（k-means 目标非凸，相关问题已知 NP-hard）。
- Lloyd 迭代会在有限步内收敛到一个局部最优或稳定划分。
- 近似比保证：对通用 Lloyd 实现通常不提供稳定常数近似比承诺。

## R14

常见失效模式与防护：
- 失效：初始化差，结果很差。
  - 防护：使用 k-means++，并做多次重启。
- 失效：出现空簇。
  - 防护：重置空簇中心到当前误差最大样本。
- 失效：异常值拖拽中心。
  - 防护：先做离群点处理，或改用 k-medoids / DBSCAN。
- 失效：尺度不统一导致距离失真。
  - 防护：标准化/归一化特征。

## R15

工程实践建议：
- 先做一个可复现 MVP（固定 seed、可打印中间指标）。
- 监控 inertia 下降曲线，快速发现实现错误。
- 对核心函数做小样本手算验证（分配和均值更新）。
- 在生产中记录每次训练的 `k/seed/inertia/n_iter` 便于回溯。

## R16

相关算法与演化：
- k-means++：改进初始化质量。
- MiniBatchKMeans：用小批次提升大数据训练速度。
- Bisecting k-means：层次化二分聚类策略。
- 与高斯混合模型（GMM）关系：k-means 可看作等方差极限下的硬分配近似。

## R17

本目录 MVP（`demo.py`）说明：
- 仅依赖 `numpy`，从零实现 Lloyd 版 k-means，不调用黑盒聚类器。
- 数据：脚本内生成 3 个高斯簇二维样本。
- 核心函数：
  - `kmeans_plus_plus_init`：k-means++ 初始化；
  - `pairwise_sq_dists`：向量化距离矩阵；
  - `kmeans_fit`：分配-更新主循环；
  - `cluster_purity`：基于真值标签计算聚类纯度；
  - `center_rmse_with_best_permutation`：在簇标签置换下评估中心误差。
- 运行方式：

```bash
cd Algorithms/数学-机器学习-0229-k-means聚类
python3 demo.py
```

脚本无需交互输入，会打印迭代结果并做断言校验（inertia 非增、纯度阈值、中心误差阈值）。

## R18

源码级算法流程拆解（对应 `demo.py`，8 步）：
1. `make_synthetic_blobs` 生成三簇二维高斯数据并打乱顺序，得到 `x` 与参考标签 `y`。
2. `kmeans_fit` 调用 `kmeans_plus_plus_init` 选择初始中心：先随机一点，再按与已选中心最近距离平方做概率采样。 
3. 每轮在 `pairwise_sq_dists` 中一次性计算 `x` 到全部中心的平方距离矩阵。 
4. 分配步用 `argmin` 得到每个样本的簇标签，并累计最小距离平方和作为当轮 inertia。 
5. 更新步对每个簇取样本均值作为新中心；若某簇为空，则把该中心重置到当前误差最大的样本。 
6. 计算新旧中心位移 `shift`，当 `shift <= tol` 时提前停止，否则继续迭代。 
7. 结束后再做一次最终分配，输出 `centers/labels/inertia/n_iter/inertia_history`。 
8. `main` 计算纯度与中心 RMSE，并断言 inertia 单调不增和质量阈值，确保 MVP 行为正确。 

该实现没有把第三方库当黑盒，分配、更新、收敛和异常分支都在源码中显式可追踪。
