## R01

CP 分解（CANDECOMP/PARAFAC）把一个高阶张量表示为若干个秩-1 张量的和。对三阶张量 `X \in R^(I x J x K)`，其 rank-`R` CP 形式为：

`X ≈ Σ_{r=1..R} lambda_r * a_r ⊗ b_r ⊗ c_r`

其中 `a_r,b_r,c_r` 分别来自三个因子矩阵 `A,B,C` 的第 `r` 列，`lambda_r` 为每个分量的权重。直观上，CP 分解把“多维相关结构”压缩成少量可解释的潜在成分。

## R02

本条目 MVP 处理三阶实数张量的近似分解问题。

- 输入：
  - 观测张量 `X`（形状 `I x J x K`）
  - 目标秩 `R`
  - ALS 参数：`max_iter`、`tol`、`ridge`
- 输出：
  - 分解权重 `lambda`（长度 `R`）
  - 三个因子矩阵 `A (I x R), B (J x R), C (K x R)`
  - 迭代误差曲线（相对重建误差）

目标函数是最小化 Frobenius 误差：

`min ||X - [[lambda; A,B,C]]||_F^2`。

## R03

本实现采用 ALS（Alternating Least Squares）求解：固定两个因子、更新另一个因子，循环迭代直到收敛。

以更新 `A` 为例：

`A <- MTTKRP_1(X,B,C) * ( (B^T B) * (C^T C) )^{-1}`

- `*`（这里）表示矩阵乘；
- `(...) * (...)`（Hadamard）表示逐元素乘；
- `MTTKRP_1` 是 mode-1 的 Matricized Tensor Times Khatri-Rao Product。

`B` 与 `C` 更新完全对称。

## R04

为什么 ALS 在这里可行：

- 固定 `B,C` 时，目标函数对 `A` 是标准线性最小二乘；
- 固定 `A,C` 时，目标函数对 `B` 也是线性最小二乘；
- 固定 `A,B` 时，对 `C` 同理。

因此每一步子问题都可闭式更新（在 Gram 矩阵可逆或加 ridge 可解的前提下），并保证误差非增（到数值精度为止）。

## R05

时间复杂度（3 阶张量，单次 ALS 迭代）：

- 三次 MTTKRP 的主开销约为 `O(I*J*K*R)`（常数因实现而异）；
- 三次 Gram/Hadamard 与线性方程求解约为 `O((I+J+K)*R^2 + R^3)`；

当 `R` 远小于张量维度时，主成本通常由 MTTKRP 主导。总复杂度约为：

`O(T * I*J*K*R)`，其中 `T` 为迭代轮数。

## R06

空间复杂度：

- 原始张量 `X` 占 `O(I*J*K)`；
- 因子矩阵占 `O((I+J+K)*R)`；
- Gram 与临时矩阵占 `O(R^2)`。

整体额外开销主要是因子矩阵，故额外空间约为 `O((I+J+K)*R)`（不计输入张量本身）。

## R07

一个简化更新直觉（只看 `A` 的一列）：

- `B,C` 当前给出了“模式 2 和模式 3 的解释方向”；
- `MTTKRP_1` 把张量沿着这些方向投影，得到模式 1 的最优回归右端；
- `((B^TB)*(C^TC))^{-1}` 对列间耦合进行校正；
- 最终得到新的 `A`，使固定 `B,C` 条件下误差最小。

这就是“交替最小二乘”的本质：每步都做当前最优局部回归。

## R08

CP 分解的重要性：

- 可解释建模：每个 rank-1 分量对应一个潜在主题/机制；
- 参数压缩：从 `I*J*K` 规模压缩到 `(I+J+K)*R`；
- 广泛应用：推荐系统、多模态统计、化学计量学、信号分离、神经数据分析。

CP 在“解释性”和“压缩性”之间提供了较好平衡。

## R09

适用条件与限制：

- 需要用户给定或搜索目标秩 `R`；
- ALS 对初始化敏感，可能陷入局部最优；
- 分量存在置换与缩放不唯一性（符号/比例可在各因子间转移）；
- 当因子列接近共线时，Gram 条件数变差，数值不稳定性会增强。

因此工程上通常结合多次随机初始化与正则化。

## R10

正确性（工程可验证版本）：

1. 每个子问题是线性最小二乘，更新公式来自正规方程。  
2. 在固定其余因子时，该更新给出对应模式的最小二乘解。  
3. 交替执行三种模式更新，目标值应整体下降或保持不变（数值误差下近似成立）。  
4. `demo.py` 用重建相对误差曲线验证“迭代收敛”而不是仅输出一次结果。  
5. 最终通过断言（误差阈值、单调性容忍、有限值检查）给出可执行验收。

## R11

收敛与可辨识性说明：

- ALS 通常收敛到驻点，但不保证全局最优；
- CP 分解在特定充分条件下可唯一（除置换/缩放外），但一般数据不一定满足；
- 噪声较大、秩设定不当或初始化差时，可能出现慢收敛、震荡或退化分量。

MVP 重点是“透明算法流程 + 可复现实验”，不宣称最优全局解保证。

## R12

数值稳定与性能策略（本实现）：

- 使用 `ridge * I` 稳定 Gram 求解，降低病态风险；
- 每轮对因子列做归一化，并把尺度吸收进 `lambda`，避免数值爆炸；
- 采用 `numpy.einsum(..., optimize=True)` 计算 MTTKRP，保持代码简洁且性能可接受；
- 以误差变化阈值 `tol` 提前停止，避免无效迭代。

## R13

本目录 MVP 的验收口径：

- `python3 demo.py` 可直接运行（无交互）；
- 输出迭代次数、最终相对误差、与真值张量误差、分量对齐分数；
- 内置断言：
  - 误差历史均为有限值；
  - 末次误差明显低于基线；
  - 误差下降趋势合理（允许轻微数值抖动）。

## R14

常见失效模式：

- `R` 过大：过拟合与退化分量风险上升；
- `R` 过小：系统性欠拟合，重建误差下不去；
- 初始化过差：迭代停在较差局部解；
- Gram 近奇异：解线性系统不稳定（本实现用 ridge 缓解）；
- 数据量级跨度极大：建议预标准化后再分解。

## R15

`demo.py` 的实现模块：

- `CPALSConfig`：集中管理 rank、迭代和噪声参数并做校验；
- `mttkrp_mode{0,1,2}`：显式给出三种模式的 MTTKRP；
- `hadamard_gram`：构造 ALS 更新需要的 Hadamard Gram；
- `cp_als`：主迭代循环（更新、归一化、误差记录、停止条件）；
- `reconstruct_cp`：把 `lambda + 因子矩阵` 重建回三阶张量；
- `best_alignment_score`：用全排列匹配评估恢复分量与真值分量的一致性。

## R16

相关算法链路：

- Tucker/HOSVD：核心张量 + 因子矩阵，表达更灵活；
- Tensor Train (TT)：面向高阶张量的链式分解；
- 非负 CP（NCP）：附加非负约束提高可解释性；
- 随机化/并行 CP：面向超大规模数据加速。

CP-ALS 是最基础、最常见、最便于落地审计的入门方案。

## R18

`demo.py` 源码级流程拆解（8 步）：

1. 构造 `CPALSConfig` 并校验参数，固定随机种子保证可复现。  
2. 生成真值 `lambda_true, A_true, B_true, C_true`，合成低秩张量并加入高斯噪声得到观测 `X_obs`。  
3. 随机初始化 `A,B,C`，进入 ALS 主循环。  
4. 用 `mttkrp_mode0/1/2` 计算三个模式的投影右端，同时用 `hadamard_gram` 构造正规方程左端。  
5. 逐模式解线性系统更新因子矩阵（含 `ridge` 正则）。  
6. 每轮调用 `normalize_factors` 把列范数吸收到 `lambda`，再重建 `X_hat` 并记录相对误差。  
7. 误差变化小于 `tol` 时提前停止，得到最终 `lambda,A,B,C` 与误差历史。  
8. 计算对观测张量/真值张量误差与分量对齐分数，执行断言验收并打印 `All CP-ALS checks passed.`。

说明：实现没有调用现成黑箱 CP 分解 API，ALS 更新、MTTKRP 与重建流程均在源码中显式展开。
