# 逐次超松弛法 (SOR)

- UID: `MATH-0068`
- 学科: `数学`
- 分类: `线性代数`
- 源序号: `68`
- 目标目录: `Algorithms/数学-线性代数-0068-逐次超松弛法_(SOR)`

## R01

逐次超松弛法（Successive Over-Relaxation, SOR）是求解线性方程组 `Ax=b` 的经典定常迭代法，可视作高斯-赛德尔（GS）迭代的松弛加速版本。

对第 `k` 轮迭代和第 `i` 个分量，SOR 更新写为：

`x_i^{(k+1)} = (1-omega)x_i^{(k)} + (omega/a_ii) * (b_i - sum_{j<i} a_ij x_j^{(k+1)} - sum_{j>i} a_ij x_j^{(k)})`

其中松弛参数 `omega` 满足 `0 < omega < 2`：

- `omega = 1` 退化为 GS；
- `omega > 1` 为超松弛（常用于加速）；
- `omega < 1` 为欠松弛（更保守、更稳）。

## R02

历史定位（简要）：

- SOR 来自 20 世纪中期对 GS 的系统加速研究，是数值线性代数中最常见的平稳迭代法之一；
- 在稀疏椭圆型 PDE 离散后线性系统中长期大量使用；
- 现代多重网格方法中，SOR/GS 仍常作为平滑器（smoother）使用。

## R03

本条目 MVP 解决的问题：

- 输入：
  - 系数矩阵 `A in R^{n x n}`（方阵，且 `a_ii != 0`）；
  - 右端向量 `b in R^n`；
  - 松弛参数 `omega`；
  - 迭代上限 `max_iters` 与容差 `tol`；
  - 可选初值 `x0`。
- 输出：
  - 近似解 `x_hat`；
  - 残差 `||A x_hat - b||_inf`；
  - 迭代次数与收敛标记；
  - 残差历史、步长历史（便于诊断）。

## R04

时间复杂度：

- 每次外层迭代要更新 `n` 个分量；
- 稠密矩阵下，每个分量需要 `O(n)` 点积，单轮 `O(n^2)`；
- 迭代 `k` 轮总体 `O(k n^2)`；
- 稀疏矩阵下可写成 `O(k * nnz(A))`。

## R05

空间复杂度：

- 存储输入矩阵与向量：`O(nnz(A) + n)`（稠密时为 `O(n^2)`）；
- 迭代过程额外需要当前解 `x`、上一轮拷贝 `x_old`：`O(n)`；
- 若记录全部历史，额外 `O(k)`。

因此迭代额外空间开销为 `O(n + k)`。

## R06

2x2 微型示例（演示更新机制）：

设
`A = [[4,-1],[-1,3]]`，`b = [15,10]`，取 `omega = 1.1`，初值 `x^(0)=[0,0]`。

第 1 轮：

1. `x_1^(1) = (1-1.1)*0 + 1.1*(15 - (-1)*0)/4 = 4.125`。
2. `x_2^(1) = (1-1.1)*0 + 1.1*(10 - (-1)*x_1^(1))/3 ≈ 5.1792`。

可以看到，`x_2` 的更新已使用了本轮最新的 `x_1`，这就是 GS/SOR 的“前代后新”特性。

## R07

算法意义：

- 是理解“矩阵分裂迭代 + 松弛参数调节”的关键入门算法；
- 在中等精度需求场景中，常用作快速、低内存求解器；
- 在大型迭代框架（多重网格、预条件设计）中常作为基础构件。

## R08

核心理论（矩阵分裂视角）：

令 `A = D - L - U`（`D` 为对角，`L/U` 为严格下/上三角相反号记法），SOR 可写成：

`(D - omega L) x^(k+1) = [(1-omega)D + omega U] x^(k) + omega b`

对应迭代矩阵：

`B_omega = (D - omega L)^(-1) * ((1-omega)D + omega U)`。

收敛判据是：`rho(B_omega) < 1`（谱半径小于 1）。

常用结论：若 `A` 对称正定（SPD），且 `0 < omega < 2`，SOR 收敛。

## R09

适用条件与局限：

适用：

- 线性系统规模较大，不希望直接 `O(n^3)` 分解；
- 矩阵稀疏、可高效做分量更新；
- 可接受“近似到容差”而非一次精确解。

局限：

- 速度强依赖 `omega`，参数选择不当会变慢甚至不收敛；
- 对病态系统可能收敛缓慢；
- 本质是串行前代后新更新，不像纯 matvec 那样易做粗粒度并行。

## R10

正确性说明（工程可验证版）：

1. 每个分量更新都来自把 `Ax=b` 的第 `i` 行方程对 `x_i` 显式解出。  
2. 对 `j<i` 使用新值、`j>i` 使用旧值，等价于一次三角系统风格的顺序消去。  
3. 若迭代收敛到 `x*`，将 `x^(k+1), x^(k)` 同时取极限代入更新式，可得 `x*` 满足 `Ax*=b`。  
4. 若 `rho(B_omega)<1`，则误差满足 `e^(k+1)=B_omega e^(k)` 并几何衰减到 0。  
5. `demo.py` 用残差与参考解双重校验上述结论。

## R11

误差与稳定性：

- 终止误差由 `tol` 与 `max_iters` 控制；
- 实践中同时看 `residual = ||Ax-b||` 与 `step = ||x^(k+1)-x^(k)||` 更稳健；
- `omega` 越接近最优值，通常收敛越快；过大可能振荡；
- 浮点舍入误差会累积，但对 SPD 且参数合理的系统通常可控。

## R12

性能视角：

- 单轮成本低、实现简单，适合做“快速可用”的 MVP 解法；
- 相比 Jacobi，SOR 利用最新分量，通常迭代次数更少；
- 相比 GS，合适的 `omega>1` 常可进一步降迭代次数；
- 工程上常通过小规模 `omega` 扫描（如 `0.8~1.8`）做经验选参。

## R13

可验证保证（本目录 MVP）：

- `demo.py` 构造 SPD 三对角系统并设定已知真解；
- 主算法显式实现 SOR 循环，不调用黑箱迭代器；
- 使用 `numpy.linalg.solve` 仅作为最终参考对照；
- 自动断言：
  - 必须收敛；
  - `||Ax-b||_inf` 足够小；
  - 与参考解的 `inf` 范数误差足够小。

## R14

鲁棒性与常见失效模式：

- `A` 非方阵、`b` 维度不匹配：立即报错；
- `A` 或 `b` 含 `nan/inf`：立即报错；
- `a_ii = 0`：无法按行解出 `x_i`，直接报错；
- `omega` 不在 `(0,2)`：直接报错；
- 达到 `max_iters` 未收敛：返回 `converged=False` 供上层决策。

## R15

实现设计（对应 `demo.py`）：

- `SORResult`：封装解向量、残差、迭代次数、历史曲线；
- `validate_inputs`：集中做输入合法性检查；
- `sor`：SOR 主过程（按分量更新，明确使用新旧值）；
- `run_checks`：自动校验收敛性和数值精度；
- `main`：构造样例、运行求解、打印报告与 `omega` 扫描对比。

## R16

相关算法链路：

- `omega = 1` 即高斯-赛德尔（GS）；
- `omega < 1` 对应欠松弛（Under-Relaxation）；
- Jacobi/GS/SOR 构成经典平稳迭代族；
- 更强迭代法包括共轭梯度（CG）、GMRES、BiCGSTAB；
- SOR 在多重网格中常作为平滑器而非最终解法。

## R18

源码级流程拆解（`demo.py` 中 `sor` 主路径，8 步）：

1. `validate_inputs` 检查 `A,b` 维度、有限性、对角非零、`omega` 区间与停止参数合法。  
2. 初始化 `x`（默认零向量或用户给定 `x0`），计算初始残差。  
3. 每轮先拷贝 `x_old`，用于保留“旧上三角分量”。  
4. 对每个分量 `i`：
   先算 `lower_sum = a[i,:i]·x[:i]`（新值）与 `upper_sum = a[i,i+1:]·x_old[i+1:]`（旧值）。  
5. 根据行方程解出 GS 值 `gs_value`，再做松弛更新 `x[i]=(1-omega)x_old[i]+omega*gs_value`。  
6. 一轮结束后计算 `residual=||Ax-b||_inf` 与 `step=||x-x_old||_inf`，记录到历史。  
7. 若 `residual < tol` 且 `step < tol`，立即返回 `converged=True` 的 `SORResult`。  
8. 若到达 `max_iters` 仍未满足阈值，返回 `converged=False`，由上层决定是否调整 `omega` 或改算法。  

说明：`numpy.linalg.solve` 仅在 `main` 中做验算，不参与 SOR 核心迭代。
