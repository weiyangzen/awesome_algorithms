## R01

扩散模型（Diffusion Model）是一类生成模型。其核心思想是：
先把真实数据逐步加噪到接近高斯噪声，再训练神经网络学习“如何逐步去噪”，最终从纯噪声反推得到新样本。

在离散时间版本（DDPM）里，时间步 `t=1..T`，前向过程是固定马尔可夫链，反向过程由模型近似。

## R02

要解决的问题是：给定真实数据分布 `x0 ~ q_data(x)`，构造可采样的模型分布 `p_theta(x0)`，使其尽量逼近真实分布。

DDPM 的训练常用等价目标：

`min_theta E_{x0, eps, t}[ || eps - eps_theta(x_t, t) ||^2 ]`

其中

`x_t = sqrt(alpha_bar_t) * x0 + sqrt(1 - alpha_bar_t) * eps`, `eps ~ N(0, I)`。

## R03

核心思想可概括为三点：

- 训练时把“学习数据分布”转成“学习每个噪声层级下的噪声预测”。
- 采样时从 `x_T ~ N(0, I)` 出发，按时间反向迭代，每一步去掉一部分噪声。
- 前向加噪是已知闭式公式，训练稳定；生成质量由反向网络能力决定。

## R04

输入：

- 训练样本 `x0`（本 MVP 使用一维双峰高斯混合数据）
- 扩散步数 `T`
- 噪声日程 `beta_t`
- 噪声预测网络 `eps_theta`

输出：

- 训练后的噪声预测网络参数 `theta`
- 从纯噪声采样得到的生成样本
- 训练损失曲线与生成分布统计指标

## R05

本任务选择经典 DDPM 的 `epsilon-prediction + MSE` 目标，原因：

- 推导直接、实现简洁，适合最小可运行版本。
- 与离散高斯扩散过程严格对应。
- 在工程上可稳定训练，不依赖复杂技巧。

噪声日程使用线性 `beta_t`（`beta_start=1e-4`, `beta_end=0.02`），可作为入门基线。

## R06

标准流程：

1. 构造训练数据 `x0`（本例是二维高斯混合的一维切片）。
2. 定义扩散日程并预计算 `alpha_t`、`alpha_bar_t`。
3. 训练阶段随机采样时间步 `t`，按闭式公式生成 `x_t`。
4. 用网络预测噪声 `eps_theta(x_t, t)`，最小化 `MSE(eps_pred, eps)`。
5. 采样阶段从 `x_T ~ N(0, 1)` 出发，按 `t=T..1` 迭代去噪。
6. 输出生成样本与真实样本统计对比。

## R07

复杂度（粗略）：

- 设样本数 `N`、训练轮数 `E`、网络前向成本 `C_f`。
- 训练复杂度约为 `O(E * N * C_f)`（忽略小常数）。
- 采样每生成 `M` 个样本需 `T` 步，复杂度约 `O(M * T * C_f)`。

扩散模型的典型代价瓶颈在采样阶段的多步迭代（`T` 较大时更明显）。

## R08

优点：

- 训练目标稳定，较少出现 GAN 式对抗不稳定。
- 生成质量上限高，覆盖模式能力强。
- 数学结构清晰，便于做条件控制与理论分析。

缺点：

- 采样慢（需要多步反向迭代）。
- 对时间步编码和噪声日程较敏感。
- 真实高维任务需要更大网络与算力。

## R09

适用场景：

- 图像/语音/轨迹等高质量生成任务。
- 需要可控生成（条件扩散、引导采样）的场景。
- 需要可解释训练目标而非对抗博弈的研究场景。

不太适用：

- 强实时生成（超低延迟）且不能使用蒸馏/加速采样技术的场景。

## R10

本 MVP 参数设置：

- 数据：一维高斯混合，样本数 `2048`
- 扩散步数：`T=100`
- 网络：`MLP(1 + time_embed_dim -> 64 -> 64 -> 1)`
- 时间编码：sin/cos 频率编码（`time_embed_dim=32`）
- 训练：`epochs=250`，`batch_size=256`，`lr=1e-3`，`Adam`
- 采样：生成 `2048` 个点，和真实分布做统计对比

## R11

与 VAE/GAN 对比（简述）：

- 对比 VAE：扩散模型通常样本更锐利，但采样更慢。
- 对比 GAN：扩散模型训练更稳，模式崩塌风险更低，但推理成本更高。
- 工程上常见策略：用扩散做高质量离线生成，用蒸馏或少步采样改进速度。

## R12

工程注意点：

- `beta_t` 必须保持在 `(0, 1)` 且递增平滑，避免数值不稳定。
- `alpha_bar_t` 在代码中应预计算，减少重复开销。
- 时间步 `t` 要随机覆盖整个区间，避免模型只学到局部噪声层级。
- 训练和采样必须使用同一套噪声日程定义。

## R13

`demo.py` 的 MVP 特性：

- 使用 `numpy` 生成可复现的一维双峰数据。
- 使用 `PyTorch` 实现噪声预测网络、训练与反向采样。
- 使用 `pandas` 输出分布统计表（均值、标准差、分位数）。
- 优先用 `scipy.stats` 计算 Wasserstein 距离与 KS 检验；缺失时自动降级。
- 脚本单文件、无需交互输入，可直接运行。

## R15

结果解读建议：

- 先看训练损失是否随 epoch 下降并趋于稳定。
- 再看统计表中 `mean/std/q10/q50/q90` 是否与真实样本接近。
- Wasserstein 距离越小越好；KS 检验的高 p 值通常意味着两分布差异不显著（仅作参考）。

## R16

常见问题：

- 损失不下降：检查学习率是否过高、时间编码是否正确拼接。
- 生成样本塌缩到单峰：可能训练轮数不足或网络容量过小。
- 采样非常慢：可减小 `T` 做调试，再评估质量-速度折中。
- 数值异常（NaN）：检查 `beta` 取值、开方项是否出现负数。

## R17

## R18

本实现在源码层面的算法流（8 步）如下：

1. `build_noise_schedule` 生成 `betas / alphas / alpha_bars`，作为前向与反向过程的统一物理参数。
2. 训练时随机采样时间步 `t`，用闭式公式 `x_t = sqrt(alpha_bar_t)x0 + sqrt(1-alpha_bar_t)eps` 构造带噪样本。
3. `timestep_embedding` 把离散时间步编码为连续向量，提供给网络区分不同噪声层级。
4. `NoisePredictor.forward` 接收 `(x_t, t)` 并输出 `eps_pred`，等价学习 score 的离散近似。
5. `train_ddpm` 用 `MSE(eps_pred, eps)` 回传梯度更新参数，使网络逼近真实噪声。
6. 采样时从 `x_T ~ N(0, I)` 初始化，在 `sample_ddpm` 中执行 `t=T..1` 的反向迭代。
7. 每一步按 DDPM 均值公式先做去噪，再在 `t>1` 时注入方差项随机噪声，得到 `x_{t-1}`。
8. 最终得到 `x_0` 生成样本；`evaluate_distribution` 用统计量与距离指标评估与真实分布的贴合程度。

这 8 步对应了“前向定义 -> 噪声学习 -> 反向生成 -> 结果评估”的完整闭环。
