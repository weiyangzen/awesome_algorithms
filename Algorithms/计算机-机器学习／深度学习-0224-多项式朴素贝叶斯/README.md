## R01

本条目实现一个可运行、可审计的 `Multinomial Naive Bayes`（多项式朴素贝叶斯）最小 MVP，聚焦文本词频分类场景。

- 输入：离散词频特征 `X`（非负计数矩阵）与类别标签 `y`。
- 输出：类别预测、评估指标、每类高概率词项。
- 目标：在不依赖纯黑盒的前提下，清晰展示“计数统计 -> 条件概率平滑 -> 对数后验判别”的全流程。

## R02

多项式朴素贝叶斯的建模假设：

- 条件独立假设：给定类别 `c`，文档中各词项出现次数相互独立；
- 生成分布：文档由类别条件下的多项式分布生成；
- 特征要求：输入为非负计数（词袋、n-gram 计数等），不适合负值特征。

它在高维稀疏文本上通常能以很低训练成本给出稳定基线。

## R03

核心公式（拉普拉斯平滑）：

- 类先验：`P(c) = N_c / N`
- 条件概率：`P(w_j|c) = (N_{c,j} + alpha) / (sum_j N_{c,j} + alpha * V)`
- 判别规则：
  `argmax_c [ log P(c) + sum_j x_j * log P(w_j|c) ]`

其中：

- `N_c`：类别 `c` 的文档数；
- `N_{c,j}`：类别 `c` 中词 `j` 的总计数；
- `V`：词表大小；
- `alpha`：平滑系数（本实现默认 `1.0`）。

## R04

本 MVP 的实现策略：

1. 先手写一个 `fit_manual_multinomial_nb`，显式计算 `class_log_prior_` 与 `feature_log_prob_`；
2. 再调用 `sklearn.naive_bayes.MultinomialNB` 作为对照；
3. 对比两者在同一测试集上的预测一致率和指标；
4. 输出可解释信息（每类 top token 概率）与混淆矩阵。

这样既保证透明度，又能用成熟实现做结果校验。

## R05

为何选择“手写 + 对照”而非只用库：

- 仅用库会掩盖关键统计量的来源；
- 手写版可直接观察平滑与对数空间计算细节；
- 与 sklearn 对齐可以快速验证实现是否正确，降低“自写实现悄悄出错”的风险。

## R06

`demo.py` 的数据实验设计：

- 构造 3 类主题文本（`sports/finance/tech`）；
- 每类文档由“类内高频词 + 通用词 + 少量跨类噪声词”组成；
- 用固定随机种子生成 210 条样本（每类 70 条）；
- 按 `70/30` 分层切分训练集与测试集。

该设计保留可控噪声，能体现朴素贝叶斯在词频分类上的实用性。

## R07

核心数据结构：

- `x_train/x_test`: `scipy.sparse.csr_matrix` 词频矩阵；
- `ManualMultinomialNB`: 保存 `classes_`、`class_log_prior_`、`feature_log_prob_`、`alpha`；
- `metrics_table`: 两模型对比指标表（accuracy/f1_macro）；
- `conf_df`: 手写模型混淆矩阵；
- `top_token_df`: 每类最高条件概率词项及对应概率。

## R08

复杂度分析（设样本数 `N`，词表大小 `V`，类别数 `C`）：

- 训练阶段：
  - 统计词频约 `O(nnz)`（`nnz` 为非零计数）
  - 参数归一化约 `O(C*V)`
- 推理阶段：
  - 单样本打分约 `O(C*nnz_sample)`

在稀疏文本上，该复杂度通常显著低于许多判别式深模型。

## R09

数值稳定与工程细节：

- 全程在对数域计算后验，避免连乘下溢；
- 通过 `alpha > 0` 防止零概率；
- 对输入做非负检查，避免违背多项式模型前提；
- 使用固定随机种子保障实验可复现。

## R10

函数职责划分：

- `build_synthetic_corpus`: 生成可复现实验语料；
- `fit_manual_multinomial_nb`: 手写训练参数估计；
- `manual_joint_log_likelihood`: 计算每类联合对数似然；
- `manual_predict`: 输出 argmax 类别；
- `top_tokens_per_class`: 生成可解释词项表；
- `main`: 完成切分、训练、对照、断言、打印。

## R12

输出指标解释：

- `accuracy`: 整体分类正确率；
- `f1_macro`: 各类别 F1 的简单平均，更关注类间平衡；
- `manual_vs_sklearn_agreement`: 手写版与 sklearn 预测一致比例；
- `manual_confusion_matrix`: 观察具体误分类方向；
- `manual_top_tokens`: 每类最具代表性的高概率词。

## R13

代码中的正确性门禁：

1. 训练集/测试集不能为空；
2. 词表大小至少达到基本阈值（防止异常向量化结果）；
3. 计数特征必须非负；
4. 手写与 sklearn 预测一致率必须 `>= 0.95`；
5. 手写模型准确率必须 `>= 0.85`。

这些断言让脚本在异常配置下尽早失败。

## R14

常见失效模式与处理：

- 失效 1：把 TF-IDF 或中心化特征直接喂给模型。  
  对策：确保输入是非负计数（CountVectorizer 输出）。

- 失效 2：词表极小或训练样本过少导致先验偏置。  
  对策：扩大语料、做分层切分、检查类别分布。

- 失效 3：某些词只在测试出现。  
  对策：仅用训练集拟合向量器，测试集只做 transform（本实现已遵循）。

## R15

与相关模型对比：

- 对比伯努利朴素贝叶斯：
  - 伯努利使用“是否出现”二值特征；
  - 多项式使用“出现次数”，更贴合词频场景。

- 对比逻辑回归：
  - 逻辑回归通常上限更高，但训练开销更大；
  - 多项式 NB 作为小数据/冷启动基线更高效。

- 对比深度文本模型：
  - 深模型表达力强，但部署和数据需求更高；
  - 多项式 NB 胜在轻量、可解释、快速迭代。

## R18

`demo.py` 的源码级算法流程（8 步）：

1. `build_synthetic_corpus` 生成三类主题文本，每篇文档由“类内词 + 通用词 + 跨类噪声词”组成，形成带噪监督数据。  
2. `main` 用 `train_test_split(..., stratify=labels)` 做分层切分，并用 `CountVectorizer` 将文本转成非负词频稀疏矩阵。  
3. `fit_manual_multinomial_nb` 遍历类别统计 `class_count` 与 `feature_count`，得到每个类别下各词项总计数。  
4. 在同一函数中执行拉普拉斯平滑：`smoothed = feature_count + alpha`，再归一化并取对数，得到 `feature_log_prob_`；同时计算 `class_log_prior_`。  
5. `manual_joint_log_likelihood` 对每个样本计算 `x @ feature_log_prob_.T + class_log_prior_`，对应各类别联合对数似然。  
6. `manual_predict` 对步骤 5 的结果做 `argmax` 输出类别，形成手写模型预测。  
7. `main` 再训练 `MultinomialNB` 作为对照，计算 accuracy、macro-F1、预测一致率、混淆矩阵与 top token 表。  
8. 最后执行断言门禁（非负计数、一致率阈值、准确率阈值等）并打印结果，完成“训练-验证-解释”闭环。  

这 8 步完整覆盖了多项式朴素贝叶斯从数据准备到参数估计、再到推理与验证的最小工程路径。
