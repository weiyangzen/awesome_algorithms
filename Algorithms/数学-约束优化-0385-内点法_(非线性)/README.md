## R01

内点法（非线性）是一类在可行域内部迭代推进的约束优化方法。对不等式约束 `g_i(x) <= 0`，常见做法是把它们转成对数障碍项 `-log(-g_i(x))`，从而把原问题转成一系列无约束（或更易处理）的“中心化子问题”。

本目录给出一个最小可运行 MVP：
- 采用 `log-barrier + Newton + backtracking` 的经典路径；
- 处理二维非线性目标 + 线性不等式约束；
- 输出外层 barrier 迭代日志与最终解。

## R02

原始问题（demo）定义为：

- 目标函数：`min f(x) = (x1 - 1)^2 + (x2 - 2)^2`
- 约束：
  - `g1(x) = x1 + x2 - 2 <= 0`
  - `g2(x) = -x1 <= 0`（即 `x1 >= 0`）
  - `g3(x) = -x2 <= 0`（即 `x2 >= 0`）

可行域是一个三角形，解析最优解为 `x* = [0.5, 1.5]`。

## R03

Barrier 子问题写作：

`phi_t(x) = t * f(x) - Σ log(-g_i(x))`，其中 `g_i(x) < 0` 必须严格成立。

算法思想：
- 固定 `t` 时，用 Newton 法求 `phi_t` 的近似最优（中心化步骤）。
- 然后增大 `t <- mu * t`，逐步减弱障碍影响，让解逼近原约束最优。
- 以 `m / t`（`m` 是不等式数量）作为对偶间隙估计，控制停止。

## R04

中心化步骤（Newton）包含两层关键机制：

- 牛顿方向：解线性系统 `H * p = -grad`。
- 回溯线搜索：
  - 先保证 `x + alpha p` 仍严格可行（所有 `g_i < 0`）；
  - 再用 Armijo 条件保证障碍目标充分下降。

这两个条件一起保证“在可行域内部稳定下降”，避免越界到障碍奇点。

## R05

高层伪代码：

```text
输入: 严格可行初值 x0, t0, mu, tol
x = x0, t = t0
while m / t > tol:
    # centering step
    repeat:
        计算 phi_t(x), grad, hess
        解 hess * p = -grad
        若 Newton decrement 足够小: break
        线搜索选 alpha:
            1) x + alpha p 必须严格可行
            2) 满足 Armijo 下降
        x = x + alpha p
    t = mu * t
返回 x
```

## R06

复杂度（每次 barrier 外循环）：

- 设变量维度为 `n`、不等式个数为 `m`、牛顿迭代次数为 `K`。
- 每次牛顿迭代主成本：
  - 构造梯度/Hessian：`O(m n^2)`（本例约束雅可比常量，实际更快）
  - 解线性系统：`O(n^3)`
- 因此单次外循环约 `O(K * (m n^2 + n^3))`。

本 demo 是 `n=2, m=3`，主要用于展示流程而非大规模性能。

## R07

收敛与数值性质（本实现语境）：

- 当起点严格可行且 `phi_t` 局部足够光滑，Newton 中心化通常快速收敛。
- `t` 递增会让解沿“中心路径”逼近原问题最优边界。
- 若 Hessian 病态，本实现使用小正则（`reg * I`）兜底线性求解。

注意：这是教学型 MVP，不包含工业级自适应步长、预条件器和稀疏线性代数优化。

## R08

`demo.py` 关键参数（`BarrierConfig`）：

- `t0=1.0`：初始 barrier 权重。
- `mu=8.0`：外循环放大倍数。
- `duality_gap_tol=1e-8`：停止阈值（`m/t < tol`）。
- `newton_tol=1e-12`：中心化停止阈值（牛顿减量）。
- `max_outer_iter=30`、`max_newton_iter=80`：迭代上限。
- `armijo_c1=1e-4`、`backtrack_beta=0.5`：回溯线搜索参数。

## R09

实现边界与假设：

- 仅处理不等式约束（`g_i(x) <= 0`），不含等式约束消元。
- demo 问题的约束 Jacobian 为常量矩阵；代码结构支持一般光滑 `g_i`，但未做自动微分。
- 需要严格可行初值（`g_i(x0) < 0`）；若仅有边界可行点，需要先做可行化步骤。

- 最优性近似：结果接近解析解 `[0.5, 1.5]`。

## R13

常见失败模式：

- 初值不严格可行：`log(-g_i)` 无定义，算法会直接报错。
- `mu` 过大：外层推进过猛，中心化内循环压力上升。
- `newton_tol` 过严 + 迭代上限过小：可能出现“未在上限内收敛”。
- 约束几何过尖或 Hessian 病态：需要更强正则或更保守线搜索。

## R14

与相近约束优化路线对比：

- 相比罚函数法：
  - 内点法始终在可行域内部推进，边界行为更可控；
  - 罚函数常需很大罚参数，条件数可能更差。
- 相比主动集法：
  - 主动集显式猜测激活约束；
  - 内点法通过障碍隐式处理激活边界，路径更平滑。
- 相比 SQP：
  - SQP 每步解近似 QP，局部模型更强；
  - 本 MVP 更强调“障碍 + Newton”核心骨架，代码更小。

## R18

`demo.py` 源码级算法流程（9 步）：

1. `main()` 构造严格可行初值 `x0=[0.2,0.2]` 和 `BarrierConfig`，调用 `interior_point()`。
2. `interior_point()` 检查 `x0` 是否满足所有 `g_i(x0)<0`，设置外循环参数 `t=t0`、`m=#constraints`。
3. 每个外循环调用 `centering_newton(x,t,cfg)` 解决当前 barrier 子问题，并记录 `(outer_iter, t, m/t, newton_iters, phi)`。
4. `centering_newton()` 每步先用 `barrier_value_grad_hess()` 计算 `phi_t(x)`、梯度和 Hessian：
   - `phi_t = t f(x) - Σ log(-g_i(x))`
   - `grad = t∇f + Σ ∇g_i/(-g_i)`
   - `hess = t∇²f + Σ (∇g_i ∇g_i^T)/(-g_i)^2`（本 demo 约束线性，省去 `∇²g_i` 项）
5. `solve_newton_direction()` 先尝试求解 `(H + reg I)p = -grad`；若奇异则逐步增大正则，最后可回退最小二乘。
6. 用牛顿减量 `-grad^T p` 判停；否则进入回溯线搜索。
7. 线搜索分两段：先缩步保证 `x+alpha p` 仍严格可行，再验证 Armijo 条件保证障碍目标下降，然后更新 `x`。
8. 中心化结束后回到外循环，若 `m/t < duality_gap_tol` 则停止，否则 `t <- mu * t` 继续逼近原问题。
9. `main()` 最后调用 `solve_with_scipy_reference()`（`SLSQP`）做参考对照，并打印 IPM 解、解析解、可行性误差和解差距。

以上 9 步完整对应本仓库实现，不依赖第三方“单函数黑箱”来替代 barrier 内点主流程。
