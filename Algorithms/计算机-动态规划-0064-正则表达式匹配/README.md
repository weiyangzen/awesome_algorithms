# 正则表达式匹配

- UID: `CS-0047`
- 学科: `计算机`
- 分类: `动态规划`
- 源序号: `64`
- 目标目录: `Algorithms/计算机-动态规划-0064-正则表达式匹配`

## R01

问题定义：给定字符串 `s` 和模式串 `p`，判断 `p` 是否能**完整匹配** `s`。其中：
- `.` 可匹配任意单个字符。
- `*` 表示其前一个字符可以重复 `0` 次或多次。
- 只接受“整串匹配”，不是子串匹配。

## R02

输入输出约定：
- 输入：`s: str`，`p: str`。
- 输出：`bool`，`True` 表示匹配成功，`False` 表示失败。

MVP 中以固定测试集方式演示，不依赖交互输入。

## R03

典型样例：
- `s="aa", p="a"` -> `False`
- `s="aa", p="a*"` -> `True`
- `s="ab", p=".*"` -> `True`
- `s="aab", p="c*a*b"` -> `True`
- `s="mississippi", p="mis*is*p*."` -> `False`

## R04

边界与约束：
- `s` 可能为空串。
- `p` 可能为空串。
- 当 `p` 中出现 `*` 时，需要访问其前一个字符；若 `*` 出现在首位可视为非法模式，MVP 中按“不匹配”处理。
- 目标是可解释、可验证的 DP 解法，不走库函数黑盒。

## R05

核心思路：二维动态规划。

设 `dp[i][j]` 表示：`s` 前 `i` 个字符（`s[:i]`）是否能被 `p` 前 `j` 个字符（`p[:j]`）匹配。最终答案是 `dp[len(s)][len(p)]`。

## R06

状态定义：
- `i` 取值范围 `0..m`，`m=len(s)`。
- `j` 取值范围 `0..n`，`n=len(p)`。
- `dp` 为 `(m+1) x (n+1)` 布尔表。

字符匹配谓词：
- `char_match(sc, pc) = (pc == '.' or sc == pc)`。

## R07

状态转移：

1. `p[j-1] != '*'`：
- 需要当前字符可匹配，且前一状态已匹配：
- `dp[i][j] = (i>0) and char_match(s[i-1], p[j-1]) and dp[i-1][j-1]`

2. `p[j-1] == '*'`（星号作用于 `p[j-2]`）：
- 零次使用：`dp[i][j] |= dp[i][j-2]`
- 一次或多次使用：若 `i>0` 且 `char_match(s[i-1], p[j-2])`，则
  `dp[i][j] |= dp[i-1][j]`

## R08

初始化：
- `dp[0][0] = True`（空串匹配空模式）。
- `dp[0][j]` 仅可能由若干 `x*` 组合匹配空串：
  当 `p[j-1]=='*'` 且 `j>=2` 时，`dp[0][j] = dp[0][j-2]`。
- 其余默认 `False`。

## R09

正确性要点（归纳）：
- 基础：`dp[0][0]=True` 正确。
- 归纳假设：更短前缀的状态均正确。
- 对任意 `dp[i][j]`：
  - 非 `*` 情况只可能由 `dp[i-1][j-1]` 推来，且需要当前字符匹配。
  - `*` 情况穷尽“用 0 次”与“用 >=1 次”两种互斥来源。
- 因此转移覆盖且仅覆盖所有合法匹配方式，最终结论正确。

## R10

复杂度分析：
- 时间复杂度：`O(m*n)`，每个状态常数次转移。
- 空间复杂度：`O(m*n)`，保存完整 DP 表。

可进一步压缩到 `O(n)`，但会降低可读性；MVP 保留二维表以便教学与调试。

## R11

与暴力回溯对比：
- 暴力法遇到 `*` 会分叉，最坏可能指数级。
- DP 将重复子问题记忆化，稳定为多项式时间。
- 在模式较长或 `*` 较多时，DP 可靠性更高。

## R12

## R16

常见坑点：
- 将“整串匹配”误写成“包含匹配”。
- 处理 `*` 时忘记 `j>=2` 的边界。
- 把 `*` 的“零次使用”分支漏掉。
- `dp[0][j]` 初始化不完整，导致空串匹配相关样例出错。

## R17

可扩展方向：
- 空间优化：二维表降为滚动数组。
- 输出匹配路径：记录转移来源以恢复解释链。
- 扩展语法：支持 `+`、`?`、字符集等（需重设计状态机/转移）。

## R18

`demo.py` 源码级算法流程（8 步）：
1. 读取输入串 `s` 与模式串 `p`，计算 `m,n`。
2. 创建 `dp` 布尔矩阵 `(m+1, n+1)`，并置 `dp[0][0]=True`。
3. 预处理首行：遍历 `j=2..n`，若 `p[j-1]=='*'`，则令 `dp[0][j]=dp[0][j-2]`。
4. 双层循环遍历 `i=1..m`、`j=1..n`。
5. 若 `p[j-1]` 不是 `*`，调用 `char_match` 判断当前字符可否对应，并结合 `dp[i-1][j-1]` 更新 `dp[i][j]`。
6. 若 `p[j-1]=='*'` 且 `j>=2`，先尝试零次使用：`dp[i][j] |= dp[i][j-2]`。
7. 在第 6 步基础上，若当前字符能匹配 `p[j-2]`，再尝试多次使用：`dp[i][j] |= dp[i-1][j]`。
8. 循环结束后返回 `dp[m][n]`，由 `run_demo_cases` 汇总展示每个测试是否通过。
