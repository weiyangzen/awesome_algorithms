## R01

分块（Square Root Decomposition，常称“平方分解”）是一种把长度为 `n` 的线性结构按块长 `B≈sqrt(n)` 切分为 `O(sqrt(n))` 个块的技巧。  
它用“块级摘要 + 端点暴力”折中时间复杂度，在实现复杂度远低于线段树的前提下，能显著快于纯暴力。

本目录给出最小可运行 MVP：
- 支持区间加：`range_add(l, r, delta)`；
- 支持区间和：`range_sum(l, r)`；
- 支持单点赋值：`point_set(i, value)`；
- 内置确定性随机对拍，运行即可验证正确性。

## R02

问题定义（本实现）：
- 输入：一个整数数组 `a[0..n-1]`，以及若干操作。
- 操作：
  - 区间增量更新：把 `a[l..r]` 每项都加上 `delta`；
  - 区间求和查询：返回 `sum(a[l..r])`；
  - 单点覆盖赋值：令 `a[i] = value`。
- 输出：每次查询的和、演示日志、断言通过结果。

索引约定：全部使用 0-based 且闭区间 `[l, r]`。

## R03

核心思想：
1. 令块长 `B = floor(sqrt(n))`（至少为 1）。
2. 把数组分成 `m = ceil(n/B)` 个块。
3. 每块维护：
   - `block_sum[b]`：块内元素和；
   - `lazy[b]`：对整块尚未下推的增量标记。
4. 查询/更新一个区间时：
   - 两端“不完整块”做逐元素处理；
   - 中间“完整块”直接用 `block_sum/lazy` 常数时间处理。

因此单次操作复杂度从 `O(n)` 降到 `O(sqrt(n))` 量级。

## R04

高层流程：
1. 初始化：分块，预计算每块边界与块和。
2. `range_add(l, r, delta)`：
   - 同块：下推后逐元素更新并重算块和；
   - 跨块：处理左右端块，中间整块只改 `lazy` 和 `block_sum`。
3. `range_sum(l, r)`：
   - 同块：下推后直接求和；
   - 跨块：左右端块求部分和，中间块直接累加 `block_sum`。
4. `point_set(i, value)`：
   - 先下推所在块，差分更新 `block_sum`，再写入元素。
5. 每个操作都做索引合法性检查。

## R05

数据结构设计（`demo.py`）：
- `SqrtDecomposition` 类字段：
  - `n`：数组长度；
  - `block_size`：块长；
  - `num_blocks`：块数；
  - `data`：真实数组存储；
  - `block_sum`：每块当前总和；
  - `lazy`：每块懒标记（待下推的整块增量）。
- 内部辅助方法：
  - `_block_left/_block_right`：块边界；
  - `_rebuild_block`：重算某块块和；
  - `_push`：将某块 `lazy` 下推到元素层。

## R06

正确性要点：
- 不变式 1：任意时刻 `block_sum[b]` 等于该块真实元素和（考虑过已累计的整块更新）。
- 不变式 2：若 `lazy[b] != 0`，表示该块所有元素逻辑上都应额外加 `lazy[b]`，但可能尚未写回 `data`。
- 不变式 3：在访问块内部分元素前先 `_push`，保证读写局部元素时 `data` 为最新。
- 区间整块更新时同步维护 `block_sum[b] += delta * block_len`，避免遗漏。

基于这些不变式，可保证查询与更新结果与朴素数组一致。

## R07

复杂度分析（`n` 为数组长度，`B≈sqrt(n)`）：
- 预处理建块：`O(n)` 时间，`O(n)` 空间。
- `range_add(l, r, delta)`：`O(B + n/B)`，取 `B≈sqrt(n)` 得 `O(sqrt(n))`。
- `range_sum(l, r)`：同上，`O(sqrt(n))`。
- `point_set(i, value)`：
  - 含一次 `_push`（最多一个块）与常数更新，摊还可视为 `O(B)`，即 `O(sqrt(n))`。

与线段树 `O(log n)` 相比，分块在理论上略慢，但实现显著更短更直观。

## R08

边界与异常处理：
- 空数组：构造时抛 `ValueError`；
- 非法区间（`l>r`）或越界索引：抛 `IndexError`；
- 非整数索引：抛 `TypeError`；
- 支持负数、零、正数混合以及负增量。

本 MVP 选择“显式报错”，避免静默纠正导致数据污染。

## R09

MVP 取舍说明：
- 仅实现最常见的“区间加 + 区间和 + 单点赋值”。
- 不实现更复杂扩展（如区间最值、区间赋值覆盖、多维分块）。
- 全程纯 Python 标准库，无第三方依赖黑盒，便于逐行理解。
- 用小规模随机对拍而非性能基准，优先保证正确性与可复现性。

## R10

`demo.py` 结构说明：
- `SqrtDecomposition.__init__`：建块并初始化摘要数组。
- `range_add`：区间增量更新。
- `range_sum`：区间求和。
- `point_set`：单点覆盖赋值。
- `materialize`：导出当前完整数组（用于检查）。
- `run_demo`：固定样例演示 + 确定性随机对拍。
- `main`：入口函数。

## R14

参数可调项：
- `block_size` 默认 `int(sqrt(n))`，可按场景手动调优。
- 随机测试规模可改：`n_random` 与 `ops`。
- 数值范围可调，便于做溢出风险评估（在其他语言实现时尤其重要）。

经验上：
- 更新多、查询多且实现预算有限时，分块是高性价比方案；
- 若追求更强上界，再考虑线段树/Fenwick/平衡树等。

## R15

与常见替代方案对比：
- 纯暴力数组：
  - 更新/查询最坏 `O(n)`；
  - 实现最简单但性能弱。
- 前缀和：
  - 查询 `O(1)`，但区间更新代价高；
  - 不适合高频更新。
- Fenwick 树（树状数组）：
  - 适合前缀型操作，常数小；
  - 区间加+区间和需要技巧化双树实现。
- 线段树：
  - 功能强、复杂度优；
  - 代码量和调试成本更高。
- 分块：
  - 复杂度与工程复杂度均居中，常作为教学与中等规模工程首选。

## R16

典型应用场景：
- 在线评测中的中等规模区间操作题；
- 原型系统中需要快速上线的范围统计模块；
- 离线日志窗口聚合（可将窗口映射成区间）；
- 作为线段树前的过渡方案，用于先验证业务语义。

## R17

可扩展方向：

## R18

`demo.py` 源码级算法流程（8 步）：
1. `main()` 调用 `run_demo()`，先构造初始数组并创建 `SqrtDecomposition` 对象。  
2. `__init__` 计算 `block_size` 与 `num_blocks`，按块把 `block_sum` 预处理出来，`lazy` 初始全 0。  
3. 执行 `range_add(l, r, delta)` 时，先判定左右端所属块：若同块，先 `_push` 后逐元素改值并 `_rebuild_block`。  
4. 若跨块，左右端块做“下推 + 局部逐元素更新 + 重算块和”，中间完整块直接 `lazy += delta` 且 `block_sum += delta * block_len`。  
5. 执行 `range_sum(l, r)` 时，同样分同块/跨块两种；端块通过 `_push` 后逐元素累加，中间整块直接加 `block_sum`。  
6. 执行 `point_set(i, value)` 时，先 `_push` 所在块，计算差值 `delta`，同步更新 `data[i]` 与 `block_sum[blk]`。  
7. `materialize()` 在输出或校验前把每个块 `_push`，得到无懒标记的完整数组快照。  
8. `run_demo()` 最后做随机对拍：同一批随机操作同时作用于分块结构与朴素数组，若所有断言通过则打印 `All assertions passed.`。
