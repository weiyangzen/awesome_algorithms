## R01

Nelder-Mead 单纯形法是一种**无导数**（derivative-free）的局部优化方法，用于求解无约束连续优化问题：

\[
\min_{x \in \mathbb{R}^n} f(x)
\]

它不需要梯度或 Hessian，只依赖函数值比较，通过维护 `n+1` 个顶点构成的单纯形（simplex）在搜索空间中迭代移动。

## R02

适用场景：
- 目标函数不可导、导数难算、导数噪声大。
- 维度中低（常见 `n < 20`）的黑箱优化。
- 需要一个实现简单、可快速落地的局部搜索基线。

不适合：
- 超高维优化；
- 对全局最优有强要求的多峰问题；
- 需要严格收敛理论保证的场景。

## R03

核心几何操作（标准系数）：
- 反射（reflection）`alpha=1`：
  \[
  x_r = c + \alpha(c - x_h)
  \]
- 扩张（expansion）`gamma=2`：
  \[
  x_e = c + \gamma(x_r - c)
  \]
- 收缩（contraction）`rho=0.5`：
  - 外收缩：`x_c = c + rho (x_r - c)`
  - 内收缩：`x_c = c + rho (x_h - c)`
- 缩放（shrink）`sigma=0.5`：
  \[
  x_i \leftarrow x_l + \sigma(x_i - x_l),\; i=1,\dots,n
  \]

其中：
- `x_l`：当前最优点；
- `x_h`：当前最差点；
- `c`：除最差点外其他 `n` 个点的质心。

## R04

`demo.py` 的输入输出约定：
- 输入：
  - 目标函数 `fun(x)`；
  - 初值 `x0`；
  - 算法参数（`alpha/gamma/rho/sigma`、`tol_x/tol_f`、迭代/评估上限）。
- 输出：`NelderMeadResult`：
  - `x`：最优近似点；
  - `fx`：最终函数值；
  - `iterations`：迭代轮数；
  - `function_evals`：函数评估次数；
  - `converged`：是否按容差判停；
  - `history`：轨迹日志。

## R05

算法伪代码：

```text
构造初始单纯形 S = {x_0, ..., x_n}
计算每个顶点函数值 f_i
for k = 1..max_iter:
    按 f_i 从小到大排序
    若 simplex_diameter < tol_x 且 f_spread < tol_f:
        收敛并退出

    计算质心 c（排除最差点 x_h）
    反射点 x_r, f_r

    if f_r < f_best:
        扩张 x_e
        在 x_e 和 x_r 中取更优替换最差点
    elif f_best <= f_r < f_second_worst:
        直接用 x_r 替换最差点
    else:
        尝试外/内收缩
        若收缩失败:
            对除最好点外全部顶点执行 shrink

返回当前最好点
```

## R06

正确性直觉（工程角度）：
- 反射：把最差点朝“更可能下降”的对侧移动。  
- 扩张：当反射已经优于最好点时，沿该方向进一步试探。  
- 收缩：当反射失败时，减小步幅以避免过冲。  
- 缩放：当局部几何信息不可信时，整体收缩单纯形，回到更保守搜索。  

这套规则不依赖导数，但能在大量平滑/弱非光滑问题上稳定下降到局部最优附近。

## R07

复杂度（每轮）：
- 排序 `n+1` 个顶点：`O(n log n)`（按顶点数计）。
- 几何更新与向量运算：`O(n^2)` 量级（主要来自多顶点向量处理）。
- 函数评估次数：
  - 常规轮通常 1-2 次；
  - 触发 shrink 时额外 `n` 次。

总体开销通常由目标函数评估成本主导。

## R08

与相关方法对比：
- 对比梯度下降：Nelder-Mead 无需梯度，但在高维通常更慢。  
- 对比牛顿/BFGS：实现更轻、依赖更少，但收敛速度和精度下界通常不如二阶/拟二阶法。  
- 对比模拟退火：Nelder-Mead 更偏局部连续优化，不强调全局跳出。

## R09

数据结构设计（`demo.py`）：
- `simplex: ndarray[(n+1), n]`：当前单纯形顶点。  
- `fvals: ndarray[(n+1)]`：顶点函数值。  
- `HistoryItem = (iter, best_fx, diameter, f_spread, action)`：迭代摘要。  
- `NelderMeadResult`：统一返回结构，方便自动校验与打印。

## R10

边界与异常处理：
- `x0` 必须是非空一维实向量。  
- `initial_step > 0`。  
- 参数约束：`alpha>0`、`gamma>1`、`0<rho<1`、`0<sigma<1`。  
- `tol_x/tol_f/max_iter/max_eval` 必须为正。  
- 初始单纯形上的函数值必须有限，否则抛出异常。

## R14

常见实现错误：
- 质心错误地包含最差点，导致更新方向偏移。  
- 外收缩/内收缩公式写反。  
- shrink 后忘记重算所有被缩放点的函数值。  
- 只用 `f` 判停，不检查单纯形直径，可能“假收敛”。

最小测试清单：

## R17

局限与工程取舍：
- 对维度较敏感，维度升高后评估开销显著增加。  
- 对初始单纯形形状较敏感，坏初始化会拖慢收敛。  
- 非凸多峰问题仅保证局部最优附近表现，不保证全局最优。  
- 本实现优先简单透明，不包含并行评估或噪声鲁棒增强。

## R18

`demo.py` 源码级流程拆解（8 步，非黑箱）：
1. `main()` 定义两个固定目标函数与初值，逐个调用 `run_case()`。  
2. `run_case()` 调用 `nelder_mead()`，并在返回后输出解、误差与迭代摘要。  
3. `nelder_mead()` 先通过 `make_initial_simplex()` 生成 `n+1` 个初始顶点并评估函数值。  
4. 每轮先对顶点按函数值排序，计算最优点、最差点、单纯形直径 `diameter` 与函数离散度 `f_spread`。  
5. 若 `diameter < tol_x` 且 `f_spread < tol_f`，判定收敛并退出。  
6. 计算排除最差点后的质心 `centroid`，先尝试反射；若明显改进则尝试扩张，否则进入中间反射或收缩分支。  
7. 当收缩仍失败时，执行 shrink：把其余顶点向最好点按 `sigma` 比例收拢并逐点重算函数值。  
8. 循环结束后返回 `NelderMeadResult`，包含最优点、目标值、评估次数、收敛标记和完整轨迹。
