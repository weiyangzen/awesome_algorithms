## R01

Minkowski 和是集合运算：对两个点集 `A, B ⊆ R^2`，定义

- `A ⊕ B = {a + b | a ∈ A, b ∈ B}`

在计算几何里，最常见输入是凸多边形。若 `P, Q` 为二维凸多边形，则 `P ⊕ Q` 仍是凸多边形。它常用于：
- 机器人路径规划里的障碍膨胀（configuration space obstacle）。
- 碰撞检测中的形体扩张。
- 形状组合与几何形态学运算。

## R02

该概念来自 Hermann Minkowski 的几何与数论研究（19 世纪末至 20 世纪初），后在计算几何中演化成核心工具。工程上最重要的结论之一是：

- 两个按逆时针顺序给出的凸多边形，可在 `O(n+m)` 时间内求其 Minkowski 和边界。

这个线性算法依赖“边向量按极角有序”的结构，而不是暴力枚举点对。

## R03

要解决的问题是：给定两个凸多边形 `P(n 个点)`、`Q(m 个点)`，高效构造 `P ⊕ Q` 的顶点序列。

直观暴力法：
- 枚举 `n*m` 个顶点和，再求一次凸包。
- 复杂度通常为 `O(n*m log(n*m))`。

更优目标：
- 直接利用凸多边形边界有序性，做到线性 `O(n+m)`。

## R04

线性算法（边归并）流程：

1. 预处理两个多边形：去掉重复收尾点，保证逆时针顺序。
2. 将每个多边形旋转到“最低且最左”顶点作为起点。
3. 写出边向量序列 `eP[i]=P[i+1]-P[i]`、`eQ[j]=Q[j+1]-Q[j]`。
4. 结果多边形首点设为 `P0 + Q0`。
5. 使用双指针归并两组边向量：
   - 若 `cross(eP, eQ) > 0`，前进 `P` 指针；
   - 若 `< 0`，前进 `Q` 指针；
   - 若 `= 0`（平行同向），两者一起前进并把边向量相加。
6. 每次前进一步，就把对应增量加到结果最后一个点。
7. 结束后清理重复点/共线冗余点，得到 Minkowski 和边界。

## R05

设输入顶点数分别为 `n, m`。

- 时间复杂度：`O(n+m)`（每条边最多被处理一次）。
- 空间复杂度：`O(n+m)`（输出和辅助结构）。

对比暴力法：
- 暴力点对 + 凸包约为 `O(n*m log(n*m))`，在大规模几何输入上差距显著。

## R06

一个手算例子：

- `P`：单位正方形顶点 `(0,0),(1,0),(1,1),(0,1)`。
- `Q`：直角三角形顶点 `(0,0),(2,0),(0,1)`。

`P ⊕ Q` 的边界会变成一个 6 边形（具体顶点顺序依起点旋转而异），直观上等于“把三角形沿正方形边界平移扫过”得到的凸形状。

本目录 `demo.py` 会给出该例的程序输出，并与暴力基线核对一致性。

## R07

优点：
- 对凸多边形是最经典、最简洁的线性做法。
- 几何意义直接，便于做碰撞检测前处理。
- 可组合进更大系统（障碍膨胀、路径规划、形状合成）。

局限：
- 输入必须是凸且顶点序正确；非凸输入需先分解或取凸包。
- 需要处理数值精度与共线退化，否则会出现重复/抖动顶点。
- 结果只给几何边界，不自动附带拓扑语义（洞、分量等）。

## R09

适用范围：
- 2D 凸多边形 Minkowski 和。
- 需要实时或批量执行的几何计算场景。

不直接适用：
- 任意非凸多边形（需先做凸分解或改用其他算法）。
- 高维多面体（可推广，但实现和复杂度管理明显更复杂）。
- 含大量噪声点且边界不稳定的数据（应先做几何清洗）。

## R10

正确性要点（证明思路简述）：

1. 凸多边形边向量按边界顺序，其极角单调环绕。
2. Minkowski 和边界的法向/切向变化，等价于两组边向量按极角的归并序列。
3. 归并时每次选“较小极角边”前进，保证结果边界沿逆时针连续推进。
4. 平行同向边可以合并（向量相加），不会改变边界外形。
5. 所有边都处理完后，得到闭合凸多边形且恰为 `P ⊕ Q`。

因此双指针边归并在几何与顺序上都与定义一致。

## R11

数值稳定性与鲁棒性处理：

- 叉积比较使用阈值 `eps`，把非常小的值视为共线。 
- 清理重复点：相邻点距离若小于阈值则合并。 
- 清理共线点：保留转折点，移除中间冗余点。 
- 面积绝对值过小的输入多边形视为退化（近似线段/点）并报错或提前处理。

`demo.py` 中这些步骤均显式实现。

## R12

工程成本与参数：

- 主要参数是几何容忍度 `eps`（如 `1e-9`）。
- `eps` 太小：易受浮点噪声影响，产生锯齿式伪转角。
- `eps` 太大：可能误删真实小拐角。

实践建议：
- 根据坐标量级设置 `eps`（坐标在 `1e3` 量级时可用更宽容阈值）。
- 用暴力基线（点对+凸包）对随机样例做回归测试。

## R14

常见失效模式：

1. 输入顺序是顺时针，导致边角归并方向错误。 
2. 输入含重复点/闭合点重复，导致零向量边干扰。 
3. 非凸输入直接套线性算法，输出错误。 
4. 共线边未合并，导致结果顶点冗余。 
5. 浮点噪声导致叉积符号抖动，边归并顺序异常。

防护策略：
- 预处理统一为 CCW；
- 去重并做凸性/面积检查；
- 对关键比较引入 `eps`；
- 用基线方法做断言回归。

## R18

`demo.py` 的源码级流程拆解（8 步）：

1. `normalize_polygon` 清洗输入顶点：去闭合重复点、去相邻重复点、统一 CCW。  
2. `rotate_to_lowest_left` 把每个凸多边形重排到“最低且最左”起点。  
3. `edge_vectors` 构造每个多边形的循环边向量序列。  
4. `minkowski_sum_convex` 用双指针归并两组边：根据叉积符号决定推进哪一侧，平行边则合并推进。  
5. 归并过程中把每次边增量累加到结果顶点，形成候选边界。  
6. `clean_polygon` 删除重复点和连续共线冗余点，得到紧凑的 Minkowski 和顶点序列。  
7. `bruteforce_minkowski_hull` 枚举输入顶点对并调用 `convex_hull_monotonic_chain` 生成基线结果。  
8. `run_case` 比较线性算法输出与基线的循环等价性并断言，再由 `main` 汇总打印，确保整体可复现通过。

整个实现没有把第三方几何函数当黑箱；核心边归并逻辑、凸包和等价性校验都在源码中可追踪。
