# 欧拉方程 (Euler Equations)

- UID: `PHYS-0082`
- 学科: `物理`
- 分类: `流体力学`
- 源序号: `82`
- 目标目录: `Algorithms/物理-流体力学-0082-欧拉方程_(Euler_Equations)`

## R01

欧拉方程是描述理想可压流体守恒律的经典方程组，忽略黏性与热传导，只保留质量、动量和总能量守恒。它是可压缩流动、激波管、喷流和爆轰等问题的基础模型，也是更复杂Navier-Stokes方程在“无黏极限”下的核心骨架。

## R02

本条目采用一维可压欧拉方程守恒形式：

`dU/dt + dF(U)/dx = 0`

其中守恒变量与通量为：

- `U = [rho, rho*u, E]^T`
- `F(U) = [rho*u, rho*u^2 + p, u*(E+p)]^T`

这里 `rho` 为密度，`u` 为速度，`E` 为单位体积总能量，`p` 为压强。

## R03

闭合关系采用理想气体状态方程：

`p = (gamma - 1) * (E - 0.5*rho*u^2)`

其中 `gamma` 为比热比。MVP默认 `gamma=1.4`（空气常用近似）。状态方程把守恒变量与热力学压力联系起来，使方程组闭合可解。

## R04

欧拉系统是双曲守恒律系统，其特征速度为：

- `lambda_1 = u - a`
- `lambda_2 = u`
- `lambda_3 = u + a`

其中声速 `a = sqrt(gamma*p/rho)`。这三个波速分别对应左行声波、接触间断、右行声波，决定了数值格式的稳定步长和耗散强度设计。

## R05

空间离散采用一阶有限体积法。对第 `i` 个网格单元：

`U_i^{n+1} = U_i^n - (dt/dx)*(F_{i+1/2} - F_{i-1/2})`

关键是界面数值通量 `F_{i+1/2}` 的构造。MVP选择鲁棒、实现简洁的 Rusanov（局部Lax-Friedrichs）通量。

## R06

Rusanov通量公式：

`F_num = 0.5*(F_L + F_R) - 0.5*s_max*(U_R - U_L)`

其中 `s_max = max(|u_L|+a_L, |u_R|+a_R)` 是界面两侧最大特征速度估计。它可理解为“中心通量 + 局部特征耗散”，对激波与间断具有较强稳健性。

## R07

时间推进采用显式前向Euler，步长由CFL条件自适应控制：

`dt = CFL * dx / max_i(|u_i| + a_i)`

并在最后一步截断 `dt`，保证时间精确到达 `t_final`。这保证了显式离散在双曲问题上的基本稳定性。

## R08

边界条件使用零梯度外流边界（ghost cell复制端点）：

- 左幽灵单元 = 第一个物理单元
- 右幽灵单元 = 最后一个物理单元

该设置简单且适合冲击管类短时演化演示。

## R09

初值采用 Sod shock-tube 经典Riemann问题：

- 左侧（`x < 0.5`）：`rho=1.0, u=0.0, p=1.0`
- 右侧（`x >= 0.5`）：`rho=0.125, u=0.0, p=0.1`

该初值会产生激波、接触间断和稀疏波，适合验证欧拉方程求解器是否具备正确波系分辨能力。

## R10

为了避免数值负密度/负压力导致崩溃，`demo.py` 中做了轻量物理裁剪：

- `rho >= rho_floor`
- `E >= 0.5*rho*u^2 + p_floor/(gamma-1)`

这属于工程鲁棒性处理，不改变算法主干，但能显著降低MVP在间断附近的不稳定概率。

## R11

复杂度分析（`N` 为网格单元数，`T` 为时间步数）：

- 单步复杂度：`O(N)`（逐界面通量 + 逐单元更新）
- 总复杂度：`O(N*T)`
- 存储复杂度：`O(N)`

向量化实现下常数较小，普通CPU可快速完成几百网格的演示算例。

## R12

`demo.py` 的MVP实现要点：

- `primitive_from_conservative`：守恒量转原始量（`rho,u,p`）
- `physical_flux`：计算欧拉物理通量 `F(U)`
- `rusanov_flux`：界面数值通量（核心算法）
- `finite_volume_step`：有限体积一步更新
- `simulate`：CFL时间循环
- `save_csv`：输出 `result.csv`（`x,rho,u,p,E`）

## R14

默认参数：

- 区间：`[0,1]`
- 网格：`400`
- `CFL=0.45`
- 终止时刻：`t=0.2`
- 比热比：`gamma=1.4`

运行后应能观察到典型三波结构：左稀疏波、中间接触间断、右行激波。

## R15

快速正确性检查建议：

- 查看控制台输出的密度/压力是否均为正
- 查看质量守恒误差与总能守恒误差是否保持在较小量级
- 打开 `result.csv`，检查 `rho/u/p` 曲线是否在间断附近呈物理可解释的分段结构

## R16

常见实现错误：

- 把 `E` 误当作内能而非总能量
- 忘记在压力计算中扣除动能项 `0.5*rho*u^2`
- `F_{i+1/2}` 与 `F_{i-1/2}` 下标错位导致更新符号反转
- 未按 `|u|+a` 约束步长，导致显式更新不稳定

## R17

可扩展方向：

- 二阶MUSCL重构 + TVD限制器（提升间断分辨率）
- SSP-RK2/RK3时间推进（提升时间精度）
- HLL/HLLC或Roe通量替换Rusanov（减少过度耗散）
- 扩展到二维欧拉方程与结构网格/非结构网格场景

## R18

`demo.py` 的源码级算法流程可拆为8步：

1. `initial_condition` 在单元中心构造Sod初值，并转成守恒量 `U=[rho, rho*u, E]`。
2. `simulate` 进入时间循环，调用 `max_wave_speed` 计算全局最大特征速度 `|u|+a`。
3. 按CFL公式计算 `dt`，若将越过终止时刻则裁剪到 `t_final-t`。
4. `finite_volume_step` 通过复制端点创建幽灵单元，形成所有界面的左右状态 `U_L/U_R`。
5. `rusanov_flux` 分别计算 `F(U_L)`、`F(U_R)`，再由 `s_max=max(|u|+a)` 组装界面数值通量。
6. `finite_volume_step` 用守恒更新式 `U^{n+1}=U^n-(dt/dx)*(F_{i+1/2}-F_{i-1/2})` 更新全体单元。
7. `apply_physical_floors` 对每个单元执行 `rho` 与最小总能量约束，保证 `rho>0, p>0`。
8. 达到终止时刻后，`save_csv` 输出 `x,rho,u,p,E` 到 `result.csv`，主程序打印范围和守恒误差统计。
