## R01

嵌套循环连接（Nested Loop Join, NLJ）是关系数据库中最经典的连接算法之一。

核心思想非常直接：

- 取外表（outer relation）的一行；
- 扫描内表（inner relation）的每一行；
- 对每一对候选行判断连接谓词；
- 条件成立就输出连接结果。

它几乎不依赖数据分布和索引结构，属于“最通用、最朴素”的连接方法。

## R02

算法直觉：把连接看成“笛卡尔积 + 过滤条件”。

- 笛卡尔积阶段枚举所有行对 `(l, r)`；
- 谓词阶段筛掉不满足 `ON` 条件的行对；
- 外连接场景下，为未匹配行补 `NULL`。

因此，NLJ 的优势是实现简单、适配条件广，代价是当两表都大时比较次数高。

## R03

本条目解决的问题：

- 输入：左表与右表（使用 Python 的 `list[dict]` 表示）以及连接键；
- 输出：连接结果（前缀化列名，避免冲突）与比较次数统计。

MVP 支持以下连接类型：

- `inner`（内连接）
- `left`（左外连接）
- `right`（右外连接）
- `full`（全外连接）

## R04

本目录的实现策略：

1. 统一连接键提取逻辑（单列或多列键）；
2. 以双重循环做候选行对枚举；
3. 每次比较前记录一次谓词评估计数；
4. 匹配时输出拼接行（`l_`、`r_` 前缀）；
5. 外连接时补齐未匹配侧的 `NULL` 模板；
6. 返回结果行与统计信息（比较次数、匹配次数、输出行数）。

这是一份教学向、可追踪源码的最小可运行实现。

## R05

复杂度分析（设左表 `N` 行，右表 `M` 行）：

- 时间复杂度：`O(N * M)`（核心双层循环）；
- 空间复杂度：`O(1)` 额外工作内存（不计输出结果）；
- 输出复杂度：最坏可达 `O(N * M)`（当谓词对大量行成立）。

如果使用索引嵌套循环连接（Index NLJ），可将内层扫描从 `M` 降到索引查找成本，但本 MVP 不使用索引黑盒。

## R07

优点：

- 通用性强：几乎任何连接谓词都可直接判定；
- 实现成本低：无需复杂预处理；
- 结果可解释性高：每次比较都可追踪。

局限：

- 大表场景性能差；
- 对缓存局部性与 I/O 不友好（内层反复扫描）；
- 在高并发分析负载下通常不如哈希连接或排序归并连接。

## R10

正确性直觉：

1. 双重循环确保所有行对都被检查一次；
2. 对每个行对仅当 `key(left)==key(right)` 时产出结果；
3. 外连接语义通过“匹配标记 + `NULL` 模板补齐”实现；
4. 因为没有跳过候选行对，故不会漏掉满足谓词的匹配。

因此，算法满足连接定义的“完整枚举 + 条件过滤”原则。

## R11

鲁棒性与实现细节：

- 缺失键使用 `dict.get`，默认 `None`，避免键不存在时报错；
- 复合键统一转为元组比较，避免键结构不一致；
- 统一列名前缀（`l_`、`r_`）避免列名冲突；
- 用断言验证输出规模与关键字段，防止静默错误。

## R12

- 数据规模 `N, M`：直接决定 `N*M` 比较成本。

## R14

常见失效模式：

1. 键列名写错，导致全表不匹配；
2. 左右键类型不一致（如 `"101"` vs `101`）；
3. 复合键顺序错位，逻辑上等值但实现不相等；
4. 外连接 `NULL` 列补齐不完整，影响下游计算。

排查方式：

- 先打印键提取结果样本；
- 核对比较次数是否等于 `N*M`；
- 对照手算的预期匹配行进行断言。

## R15

工程化建议：

- 将连接层与存储层解耦，连接函数只处理迭代器/行对象；
- 加入执行统计（比较次数、输出行数）便于优化器决策；
- 为不同连接算法（NLJ/Hash/Merge）统一接口，便于策略切换；
- 在 CI 中加入多组边界测试（空表、全不匹配、全匹配、重复键）。

## R16

相关连接算法对比：

- 索引嵌套循环连接（Index NLJ）：内表走索引查找，常优于纯 NLJ；
- 哈希连接（Hash Join）：等值连接常见高性能方案；
- 排序归并连接（Sort-Merge Join）：适合已排序或可排序的大数据场景；
- 块嵌套循环连接（Block NLJ）：通过分块减少内表重复扫描的 I/O 代价。

本条目聚焦纯 NLJ，用最少机制展示连接语义本体。

## R18

`demo.py` 源码级算法流程（7 步）：

1. `normalize_key_spec` 把单列或多列连接键规格统一成元组结构。  
2. `build_key` 对每行提取连接键，得到可直接比较的标量或元组。  
3. `nested_loop_join` 先构造左右列名前缀模板与 `NULL` 模板，准备输出格式。  
4. 进入双重循环：外层遍历左表每行，内层遍历右表每行，并累计一次比较计数。  
5. 若键相等则输出拼接行，同时标记该右行已匹配；未匹配左行在 `left/full` 下补 `NULL` 右侧。  
6. 双层循环结束后，在 `right/full` 下回填未匹配右行（左侧填 `NULL`）。  
7. 返回 `JoinResult`（结果行、比较次数、匹配次数），`main` 对四种连接做断言与打印。

该实现没有把第三方库当黑盒，连接判定、外连接补齐和统计逻辑都可在源码中逐行追踪。
