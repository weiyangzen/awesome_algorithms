# 双连通分量算法

## R01

双连通分量（Biconnected Component, BCC）通常指无向图中的点双连通分量：在该子图内删除任意一个顶点后，其余顶点仍连通。  
与之配套的关键结构是割点（articulation point）与桥（bridge）：

- 割点：删除该点会增加连通分量数量；
- 桥：删除该边会增加连通分量数量；
- 点双分量：由 DFS 过程中一组边弹栈形成的极大子结构。

## R02

本条目解决的问题定义如下：

- 输入：无向图 `G=(V,E)`，顶点编号 `0..n-1`，边列表 `[(u,v), ...]`；
- 输出：
1. 点双连通分量顶点集合列表；
2. 每个点双分量对应的边集合；
3. 割点集合；
4. 桥集合；
5. DFS 发现时间 `tin` 与 `low-link` 数组（便于调试与解释）。

`demo.py` 使用固定样例图，不依赖交互输入。

## R03

核心判据（Tarjan low-link 思想）：

- `tin[u]`：DFS 首次访问 `u` 的时间戳；
- `low[u]`：从 `u` 出发，经若干树边再至多一条返祖边可到达的最小 `tin`；
- 对树边 `u -> v`：
1. 若 `low[v] >= tin[u]`，则 `u` 是一个分量分割点，栈中从 `(u,v)` 往上的边构成一个点双分量；
2. 若 `low[v] > tin[u]`，则 `(u,v)` 是桥；
3. 若 `u` 是 DFS 根，则它在子树数 `>1` 时才是割点。

## R04

MVP 算法流程采用“DFS + 边栈”实现：

1. 建立无向邻接表，给每条边分配唯一 `edge_id`；
2. 对每个未访问顶点启动 DFS；
3. 遇到树边与返祖边时压入边栈；
4. 回溯时更新 `low[u]`；
5. 当触发 `low[v] >= tin[u]` 时，从边栈弹出直到树边 `(u,v)`，形成一个点双分量；
6. 记录割点与桥；
7. 对孤立点单独作为单点分量处理。

## R05

算法不变量：

- 每条边至多入栈一次、出栈一次；
- `low[u] <= tin[u]` 始终成立；
- 每个被弹出的边块都对应一个极大点双分量；
- 割点可属于多个点双分量，普通点通常只属于一个。

这些不变量保证了结果完整且不重不漏。

## R06

设顶点数 `V=|V|`，边数 `E=|E|`：

- 时间复杂度：`O(V + E)`；
- 空间复杂度：`O(V + E)`（邻接表、递归栈、边栈、结果容器）。

因此该算法适合稀疏图与中大规模静态图分析。

## R07

示例图（`demo.py` 的 Case-1）包含：

- 三角形 `0-1-2-0`；
- 连接边 `1-3`；
- 三角形 `3-4-5-3`；
- 尾链 `3-6-7`。

可得：

- 割点：`{1,3,6}`；
- 桥：`(1,3)、(3,6)、(6,7)`；
- 点双分量：`{0,1,2}`、`{1,3}`、`{3,4,5}`、`{3,6}`、`{6,7}`。

## R10

正确性直觉：

1. DFS 树边定义了主递归结构；返祖边把子树“回连”到祖先。  
2. `low[v]` 准确编码了 `v` 子树是否能绕开 `u` 回到更高祖先。  
3. 若 `low[v] >= tin[u]`，说明 `v` 子树无法通过返祖边越过 `u`，故 `u` 成为分割边界，栈顶边块即一个点双分量。  
4. 若进一步 `low[v] > tin[u]`，则连回 `u` 的唯一通道就是树边 `(u,v)`，该边为桥。  
5. 根节点没有父边，只有在拥有两棵以上 DFS 子树时才真正“切开”图。

## R11

实现细节与防坑：

- 用 `edge_id` 区分无向边，避免把“父边反向”误判为返祖边；
- 返祖边仅在 `tin[v] < tin[u]` 时入栈，防止重复压栈；
- 点双分量通过“弹到目标树边”为止，保证边块边界正确；
- 桥按 `low[v] > tin[u]` 判定，不能写成 `>=`；
- 孤立点显式输出为 `{u}`，避免结果遗漏顶点。

## R14

常见失效模式与防护：

- 失效：把父边当返祖边，导致 `low` 被错误压低。  
  防护：通过 `parent_edge_id` 精确跳过父边。

- 失效：根节点割点判定错误（把单子树根也当割点）。  
  防护：根节点必须 `child_count > 1`。

- 失效：边栈弹出边界错误，分量合并或截断。  
  防护：弹栈直到触发边 `eid`，而非按顶点条件。

- 失效：忽略孤立点。  
  防护：在主循环中对 `degree=0` 顶点单独建分量。

## R15

实践建议：

- 先验证割点和桥，再验证分量拆分，定位更快；
- 对结果做“集合级断言”（忽略顺序）提高测试稳健性；
- 输出 `tin/low` 以便排查 DFS 逻辑错误；
- 若要支撑更大图，可改为显式栈 DFS 以规避递归深度限制。

## R18

`demo.py` 的源码级算法流程可拆为 9 步（非黑盒）：

1. `run_demo_cases` 使用 `numpy` 构造固定无向图边数组，并转成 Python 边列表。  
2. `analyze_biconnected_components` 调用 `build_undirected_graph`，为每条边分配唯一 `edge_id`。  
3. 初始化 `tin/low/timer`、边栈 `edge_stack`、结果容器（分量/割点/桥）。  
4. 对每个未访问顶点启动 DFS；若顶点孤立，直接记录为单点分量。  
5. DFS 扫描邻接边：树边入栈并递归；返祖边（仅祖先方向）入栈并更新 `low`。  
6. 子递归返回后，用 `low[v]` 更新 `low[u]`，并依据 `low[v] >= tin[u]` 触发弹栈生成一个点双分量。  
7. 同时依据 `low[v] > tin[u]` 标记桥；根节点按子树数规则决定是否是割点。  
8. DFS 结束后统一整理输出顺序，得到稳定可比较的结果结构。  
9. `assert_case` 对割点、桥、点双分量做集合断言，确保算法行为与预期一致。  

第三方库拆解说明：

- `numpy` 仅用于演示样例数据组织；
- 双连通分量判定、分量弹栈、割点/桥识别全部为源码手写实现。
