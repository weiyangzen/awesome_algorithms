# 素数计数函数π(x)算法

- UID: `MATH-0027`
- 学科: `数学`
- 分类: `解析数论`
- 源序号: `27`
- 目标目录: `Algorithms/数学-解析数论-0027-素数计数函数π(x)算法`

## R01

问题定义：给定非负整数 `x`，计算不超过 `x` 的素数个数 `π(x)`。例如 `π(10)=4`（2,3,5,7）。

## R02

数学背景：`π(x)` 是解析数论中的基础对象，和素数分布（如素数定理）直接相关。工程上常需要在大 `x` 下快速、精确地求 `π(x)`，而不是仅用近似公式。

## R03

输入输出约定：
- 输入：单个整数 `x >= 0`
- 输出：整数 `π(x)`
- 本目录 `demo.py` 采用“内置测试 + 固定样例”方式，不依赖交互输入，直接运行即可看到结果。

## R04

基线方法与瓶颈：
- 朴素做法是筛到 `x` 后计数，时间约 `O(x log log x)`，空间 `O(x)`。
- 当 `x` 很大（如 `10^9` 以上）时，直接开到 `x` 的筛会带来显著内存和时间压力。

## R05

MVP 选型：采用 **Lehmer prime counting**（勒默算法）+ 小范围筛预处理。
- 先用埃拉托斯特尼筛预计算一定范围内的素数和 `π` 前缀表。
- 大数查询使用分解公式，把问题递归为更小规模的 `π(·)` 与 `φ(·,·)`。

## R06

核心公式：
- `φ(x, a)`：不超过 `x` 且不被前 `a` 个素数整除的正整数个数。
- Lehmer 思路：取 `a = π(x^(1/3))`，并结合 `π(√x)`，通过容斥与递归将 `π(x)` 还原为小问题组合。

## R07

正确性要点：
- `φ(x,a)` 递推：`φ(x,a) = φ(x,a-1) - φ(x/p_a, a-1)`，对应“去掉能被第 `a` 个素数整除的项”。
- Lehmer 公式中的求和项用于修正重复计数，确保每个合数的贡献被精确扣除。
- 对小 `x` 直接查表 `pi_table[x]`，与筛法结果一致。

## R08

复杂度（定性）：
- 预处理筛：时间 `O(n log log n)`，空间 `O(n)`（`n=sieve_limit`）。
- 单次大数查询：显著优于直接筛到 `x`，常见经验复杂度近似低于 `x^{2/3}` 量级（实现常数取决于缓存与递归命中）。

## R09

实现结构（`demo.py`）：
- `icbrt(n)`：整数立方根。
- `sieve_with_numpy` / `sieve_pure_python`：构建 `primes` 与 `pi_table`。
- `LehmerPrimeCounter.phi(x,a)`：带缓存的 `φ` 递推。
- `LehmerPrimeCounter.pi(x)`：Lehmer 主过程。
- `main()`：运行已知值校验并展示 `π(10^9)`。

## R18

源码级算法流程（非黑盒，8 步）：
1. 用筛法生成 `primes`（素数表）和 `pi_table`（前缀素数计数表）。
2. 若查询 `x` 很小（`x < len(pi_table)`），直接 `O(1)` 查 `pi_table[x]`。
3. 对大 `x`，先求 `a = π(⌊x^(1/3)⌋)` 与 `b = π(⌊√x⌋)`。
4. 递归计算 `φ(x, a)`：统计不被前 `a` 个素数整除的数。
5. 用主项初始化：`result = φ(x, a) + (a - 1)`。
6. 对每个 `i ∈ [a, b)`，取对应素数 `p_i`，递归求 `π(⌊x/p_i⌋)`。
7. 按 Lehmer 修正项累减：`result -= (π(⌊x/p_i⌋) - i)`，去除重复和超计。
8. 返回 `result` 作为精确 `π(x)`，并由缓存复用子问题结果减少重复计算。
