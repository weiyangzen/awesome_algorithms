# 最大流 - Ford-Fulkerson

## R01

Ford-Fulkerson 是最大流问题最经典的增广法框架：
- 从零流开始；
- 在残量网络里反复找一条 `s -> t` 增广路；
- 沿该路径按瓶颈容量增广；
- 直到找不到增广路为止。

本目录给出一个可运行、可审计的 Python MVP，采用 DFS 选择增广路，显式维护前向/反向残量边，并输出每条原始边的最终流量。

## R02

本实现的问题定义：
- 输入：
  - 节点数 `n`（编号 `0..n-1`）；
  - 有向边列表 `[(u, v, capacity), ...]`；
  - 源点 `source` 与汇点 `sink`。
- 输出：
  - 最大流值 `max_flow`；
  - 每条输入边的 `(u, v, capacity, flow)`；
  - 增广路径日志（每轮路径与增广量），便于核查算法过程。

`demo.py` 使用固定内置样例，无交互输入。

## R03

核心数学对象：

1. 容量约束：`0 <= f(u,v) <= c(u,v)`。
2. 中间点流守恒：除源汇外，入流 = 出流。
3. 残量网络：
   - 前向残量表示“还能继续推多少”；
   - 反向残量表示“已推流量中可回退多少”。
4. 增广路瓶颈：
   - 对路径 `P`，`delta = min residual_cap(e), e in P`。
5. 终止条件：
   - 残量网络不存在 `s -> t` 路径，即当前流为最大流。

## R04

算法流程（DFS 选路版本）：
1. 构建残量图：每条原始边加一条反向边（初始容量 0）。
2. 在残量图上做 DFS 搜索增广路。
3. 若未找到路径，停止。
4. 回溯路径计算瓶颈 `delta`。
5. 路径上每条边执行：
   - 前向残量减 `delta`；
   - 反向残量加 `delta`。
6. `max_flow += delta`，记录本轮路径日志。
7. 重复 2-6，直到无法增广。

## R05

`demo.py` 的核心数据结构：
- `ResidualEdge(to, rev, cap)`：残量边。
- `graph: List[List[ResidualEdge]]`：邻接表残量图。
- `original_refs`：保存每条原始边在残量图中的前向边位置，用于回读最终流量。
- `AugmentRecord(path_nodes, delta)`：单轮增广日志。
- `MaxFlowResult(max_flow, edge_flows, augmentations)`：结构化输出。

这种结构可以把“求解结果”和“求解轨迹”同时保留下来，方便教学与调试。

## R06

正确性要点（与代码对应）：
- 容量合法：每次增广都以瓶颈为上限，不会超过残量容量。
- 可回退性：更新反向残量，保证后续路径可撤销既有决策。
- 可行流恢复：增广循环结束后，原边流量由 `初始容量 - 当前前向残量` 还原，满足容量约束与守恒。
- 最优性：若残量网络中不存在 `s -> t` 路径，则由最大流理论可知当前流即最大流。

## R07

复杂度（DFS 选路，整数容量场景）：
- 每轮找路 `O(E)`；
- 设增广轮数为 `K`，总复杂度约 `O(K * E)`；
- 空间复杂度 `O(V + E)`。

若容量为整数，且每轮至少增广 1，则 `K <= max_flow`，常写为 `O(E * max_flow)`（伪多项式）。

## R08

边界与异常处理：
- `n < 2`、`source == sink`、节点越界：抛 `ValueError`。
- 边容量为负或非有限值：抛 `ValueError`。
- 支持重边（平行边），会作为独立边加入残量图。
- 示例默认处理有向图；无向边应拆成两条有向边再输入。

## R09

MVP 取舍：
- 选择纯标准库实现，避免图算法黑盒；
- 不引入 Dinic 分层图、Push-Relabel 高级启发式；
- 重点展示“增广路 + 残量回退”这一 Ford-Fulkerson 本体；
- 输出增广日志，便于对照手算步骤。

## R10

`demo.py` 主要函数：
- `_validate_input`：输入合法性校验。
- `_add_edge`：添加前向/反向残量边。
- `_find_augmenting_path_dfs`：在残量图中用 DFS 找到一条增广路并给出瓶颈。
- `ford_fulkerson_max_flow`：主循环，持续增广直到停机。
- `run_case`：执行单个样例并打印结果。
- `main`：组织固定测试集。

## R12

输出字段说明：
- `max_flow`：最终最大流值。
- `expected / check`：样例预期值与一致性检查。
- `augmentations`：每轮增广路径与增广量。
- `edge flows (u -> v | flow/capacity)`：原始边最终流量与容量。

通过这些字段可以快速检查：
- 总流值是否与预期一致；
- 每条边是否满足 `flow <= capacity`；
- 增广路径是否符合残量网络逻辑。

## R13

内置样例：
- `CLRS directed network`：经典 6 点网络，期望最大流 `23`。
- `Small bottleneck network`：多路径 + 瓶颈场景，期望最大流 `13`。

每个样例都会进行断言检查，若失败会抛出异常，便于自动验证。

## R14

实现细节与参数：
- `EPS = 1e-12`：浮点比较阈值，避免微小数值误差干扰停机判断。
- DFS 路径选择受邻接顺序影响，不同顺序可能产生不同增广轨迹，但最大流值应一致。
- 代码显式记录每轮路径，便于复现实验与排查。

## R15

与常见最大流算法对比：
- Ford-Fulkerson（本实现）：
  - 优点：实现最直观，适合教学；
  - 缺点：复杂度依赖增广路选择，可能较慢。
- Edmonds-Karp：
  - 用 BFS 选最短增广路，复杂度上界更清晰（`O(VE^2)`）。
- Dinic / Push-Relabel：
  - 工程性能通常更好，但实现复杂度更高。

## R16

应用场景：

## R18

`demo.py` 源码级流程（8 步）：
1. `main` 构造固定测试图并调用 `run_case`。
2. `run_case` 调用 `ford_fulkerson_max_flow` 获取结果对象。
3. `ford_fulkerson_max_flow` 先做 `_validate_input`，再用 `_add_edge` 构建残量图与反向边。
4. 主循环调用 `_find_augmenting_path_dfs`：在当前残量图中找一条 `source -> sink` 路径并计算瓶颈 `delta`。
5. 若找不到路径（`delta` 近零），循环结束，算法停机。
6. 若找到路径，则沿路径更新残量：前向减 `delta`、反向加 `delta`，并累加 `max_flow`。
7. 记录本轮 `AugmentRecord(path_nodes, delta)`，继续下一轮 DFS 增广。
8. 循环结束后，通过“初始容量 - 前向残量”回读每条原始边流量，返回并打印最大流、增广日志与边流量明细。
