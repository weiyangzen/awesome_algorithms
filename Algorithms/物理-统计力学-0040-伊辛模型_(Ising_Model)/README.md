# 伊辛模型 (Ising Model)

- UID: `PHYS-0040`
- 学科: `物理`
- 分类: `统计力学`
- 源序号: `40`
- 目标目录: `Algorithms/物理-统计力学-0040-伊辛模型_(Ising_Model)`

## R01

伊辛模型用于刻画自旋系统中的相变行为。每个格点只有两种自旋状态：`s_i ∈ {+1, -1}`。  
二维正方晶格上的最近邻伊辛模型是统计力学中研究临界现象最经典的离散模型之一。

## R02

给定一个 `L x L` 周期边界晶格，在固定温度 `T` 下按玻尔兹曼分布采样自旋构型：

`P(s) ∝ exp(-H(s) / T)`（下文取 `k_B = 1`）。

目标是估计平衡态观测量，如平均磁化强度、平均能量、比热和磁化率，并观察临界温度附近的变化。

## R03

哈密顿量（`J=1`、外场 `h=0`）：

`H(s) = - Σ_{<i,j>} s_i s_j`

其中 `Σ_{<i,j>}` 对最近邻配对求和。若 `h ≠ 0`，可扩展为：

`H(s) = -J Σ_{<i,j>} s_i s_j - h Σ_i s_i`。

## R04

状态空间规模为 `2^(L^2)`，随 `L` 指数增长，无法直接枚举。  
因此采用马尔可夫链蒙特卡洛（MCMC）近似采样，最常用的是 Metropolis 单点翻转。

## R05

单点翻转 `s_ij -> -s_ij` 的能量改变量可局部计算：

`ΔE = 2 * s_ij * (s_up + s_down + s_left + s_right)`

使用周期边界时，邻居索引按模 `L` 回绕。局部更新避免每次重算全局能量。

## R06

Metropolis 接受准则：

1. 若 `ΔE <= 0`，无条件接受。
2. 若 `ΔE > 0`，以概率 `exp(-ΔE / T)` 接受。

该准则满足细致平衡，平稳分布即目标玻尔兹曼分布。

## R07

一次 sweep 定义为 `L^2` 次随机提案翻转。  
算法流程通常分为：

1. 初始化自旋（全 +1 或随机 ±1）。
2. 运行 burn-in 使系统接近平衡。
3. 在采样阶段统计观测量均值和波动。

## R08

主要观测量（按每个格点归一化）：

1. 能量密度：`e = <H> / N`，`N = L^2`
2. 绝对磁化：`m = <|M|> / N`，`M = Σ_i s_i`
3. 比热：`C = ( <H^2> - <H>^2 ) / (N T^2)`
4. 磁化率：`χ = ( <M^2> - <M>^2 ) / (N T)`

## R09

复杂度（单温度）：

1. 每次翻转提案为 `O(1)`。
2. 每个 sweep 为 `O(L^2)`。
3. 总复杂度 `O((burn_in + sweeps) * L^2)`。
4. 内存复杂度 `O(L^2)`（仅保存当前格点和少量统计量）。

## R10

MVP 选型说明：

1. 仅依赖 `numpy`，避免大型框架。
2. 使用 `numpy.random.Generator` 保证可复现。
3. 采用直观的 Python 循环实现，便于教学与验证。

该实现优先保证“可读 + 可运行 + 物理意义正确”。

## R11

数值与统计注意点：

1. burn-in 不足会导致偏差。
2. 相邻 sweep 存在自相关，等效样本数低于观测次数。
3. 临界区（约 `T_c ≈ 2.269`）涨落增大，需要更多采样。
4. 对有限尺寸晶格，临界行为会被平滑化（有限尺寸效应）。

## R12

默认实验参数（demo）：

1. `L = 24`
2. `burn_in = 120` sweeps
3. `sweeps = 360`
4. 温度点：`T = 1.50 ... 3.50`（9 个点）
5. 随机种子固定，保证重复运行输出稳定。

## R13

结果预期：

1. 低温区：`|m|` 较大（自发磁化显著）。
2. 高温区：`|m|` 接近 0（无序相）。
3. `C` 与 `χ` 在临界附近出现峰值趋势（有限尺寸下为钝峰）。

## R14

与解析结果关系：

1. 二维零外场伊辛模型存在精确临界温度：
   `T_c = 2 / ln(1 + sqrt(2)) ≈ 2.269185...`
2. 有限尺寸 + 有限采样下，观测峰值位置会偏移并变宽。
3. demo 主要用于“定性重现相变迹象”，非高精度临界指数估计。

## R15

可扩展方向：

1. 使用棋盘分块（checkerboard）并行更新。
2. 用 Wolff/Swendsen-Wang 团簇算法缓解临界慢化。
3. 扫描不同 `L` 做有限尺寸标度分析（Finite-Size Scaling）。
4. 引入外场 `h` 研究磁滞与响应函数。

## R16

常见实现错误：

1. 周期边界索引写错（尤其负索引和取模）。
2. 能量重复计数（最近邻求和时需避免双计数，或统一除以 2）。
3. 比热/磁化率公式中漏掉 `T`、`N` 归一化因子。
4. 将 `|M|` 与 `M` 混用，导致物理解释混乱。

## R17

本目录交付内容：

1. `demo.py`：二维伊辛模型 Metropolis MVP，可直接运行。
2. 终端输出：各温度的 `e, |m|, C, χ` 汇总表。
3. 无交互输入，适用于自动化验证命令：
   `uv run python demo.py`

## R18

`demo.py` 的源码级算法流程（8 步）：

1. `init_spins(L, rng)` 生成 `L x L` 的随机 ±1 晶格作为初态。
2. `local_delta_energy(spins, i, j)` 读取四邻居并计算翻转该点的 `ΔE`。
3. `metropolis_sweep(spins, T, rng)` 执行 `L^2` 次随机单点翻转提案，并按 Metropolis 准则决定接受。
4. `total_energy(spins)` 通过“右邻 + 下邻”求和避免双计数，得到当前总能量 `H`。
5. `run_ising_at_temperature(...)` 先跑 `burn_in` sweeps，再在采样阶段每个 sweep 记录一次 `H` 与 `M`。
6. 基于样本数组计算 `<H>、<H^2>、<M>、<M^2>`，再换算 `e、|m|、C、χ`。
7. `main()` 在多个温度点重复第 1-6 步，收集结果列表。
8. 最后按表格格式打印每个温度的观测量，直观看到临界区附近的变化趋势。
