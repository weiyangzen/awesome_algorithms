## R01

H 矩阵（Hierarchical Matrix）与 H² 矩阵（Hierarchical-Hierarchical Matrix）是一类用于压缩离散积分算子/核矩阵的分层低秩方法。

- 问题对象: 典型为稠密矩阵 `A in R^(n x n)`，例如边界元法、N 体核函数离散后得到的矩阵。
- 核心思想:
  - 近场块（几何上相近）保留稠密表示；
  - 远场块（几何上分离）用低秩分解近似；
  - 用分层块树组织所有子块，避免全矩阵存储。
- H 与 H² 的区别:
  - H 矩阵: 每个远场块独立存低秩因子 `U_ij V_ij`；
  - H² 矩阵: 行/列簇共享嵌套基（nested basis）+ 小耦合矩阵，进一步降存储。
- 本目录 MVP:
  - 完整实现 H 矩阵分块+低秩压缩；
  - 实现一个“H² 风格共享基”重压缩版本（简化版，不包含完整 nested transfer 链）。

## R02

- 主要发展期: 1990 年代后期至 2000 年代。
- 代表性研究脉络:
  - W. Hackbusch 提出 H 矩阵框架并系统化其算法理论；
  - Hackbusch 与 Börm、Grasedyck 等推进 H²/快速多极化方向；
  - 在边界元、电磁仿真、积分方程快速求解中形成成熟工程体系。
- 历史定位:
  - 是“从稠密 `O(n^2)` 到近线性复杂度”的关键桥梁；
  - 与 FMM（Fast Multipole Method）在思想上同源，常互补或互相转化。

## R03

直接构造并存储核矩阵需要 `O(n^2)` 内存，矩阵向量乘也需 `O(n^2)` 时间。对大规模几何离散问题（如 BEM）这往往不可接受。

H/H² 矩阵通过“几何可分离 -> 数值低秩”把远场交互压缩：

1. 先对点/自由度做层次聚类（cluster tree）。
2. 再建立块树（block cluster tree）划分矩阵子块。
3. 根据可容许条件判断块是否可低秩化。
4. 对可容许块做截断 SVD 或 ACA 等近似。

结果是用可控误差换取显著内存与算力下降。

## R04

典型复杂度（固定空间维度、秩 `k` 受控、树形平衡）:

- H 矩阵:
  - 存储: `O(k n log n)`
  - matvec: `O(k n log n)`
  - 构建（含局部低秩近似）常见在 `O(k n log n)` 到 `O(k^2 n log n)` 区间，取决于近似器与块策略。
- H² 矩阵（理想 nested basis）:
  - 存储: `O(k n)`
  - matvec: `O(k n)`
  - 构建复杂度依赖基构建与传递矩阵计算，常可做到近线性。

本目录 demo 为教学型 MVP，使用直接 SVD，常数较大，不追求最优渐近常数。

## R05

空间复杂度分解:

- 原始稠密矩阵: `O(n^2)`。
- H 矩阵:
  - 近场稠密块总和 + 远场低秩因子总和；
  - 若平均秩稳定为 `k`，总存储近似 `O(k n log n)`。
- H² 矩阵:
  - 共享基 + 小耦合矩阵；
  - 在可嵌套基成立时下降到 `O(k n)`。

本 demo 还展示“H²-like 共享基重压缩”后的参数统计，用于对比趋势。

## R06

一个简化二维核矩阵示意（`n=8`）:

1. 将点集按几何位置递归二分得到簇树。
2. 根块 `(root, root)` 展开为 4 个子块。
3. 对角附近子块因距离近，不满足可容许条件，保留稠密。
4. 非对角远离子块满足
   `max(diam(t), diam(s)) <= eta * dist(t, s)`，
   对这些子块做截断 SVD，得到 `U_r V_r`。
5. matvec 时，近场用稠密乘，远场用两次小矩阵乘替代大块乘法。

这一流程在更深层递归上重复，形成多层次压缩结构。

## R07

理论与工程价值:

- 把很多“天然稠密”的算子计算推向可扩展规模。
- 提供统一框架连接:
  - 核矩阵近似；
  - 快速积分方程求解；
  - 分层直接/迭代求解器预条件。
- 为后续快速算法（HSS/HODLR/H²/FMM 变体）提供通用思想模板。
- 在中高精度场景往往比纯黑盒随机压缩更可控（几何结构可解释）。

## R08

前置知识与数学依赖:

- 线性代数: SVD、低秩逼近、矩阵范数。
- 数值分析: 误差控制、截断阈值、条件数影响。

## R09

适用前提:

- 矩阵条目可由几何点集/簇对关系解释（或可嵌入此结构）。
- 远场交互具备可压缩性（数值秩不高）。
- 允许近似误差（不是逐元素严格精确存储）。

不适用或效果差的场景:

- 矩阵无明显几何/层次结构；
- 远场块秩随规模快速增长（高频振荡核可能出现）；
- 需要严格精确代数操作且误差预算极小。

## R10

正确性可从“结构覆盖 + 局部误差”说明:

1. 块树递归保证所有索引对 `(i, j)` 被且仅被一个叶块覆盖。
2. 近场块在本实现中按原值稠密保存，因此这部分误差为 0。
3. 远场块以截断 SVD 得到最优（在 Frobenius/2-范数意义下）定秩近似。
4. 全矩阵误差由各远场块误差叠加得到上界，例如
   `||A - A_hat||_F^2 = sum_blocks ||E_block||_F^2`（分块互不重叠时）。

因此只要单块截断阈值受控，整体误差可控。

## R11

数值稳定性关注点:

- SVD 本身是稳定分解；但阈值选择会影响秩与误差平衡。
- 对角奇异核需平滑参数（demo 用 `delta`），避免极端数值范围。
- 过小容差会抬高秩并放大计算成本；过大容差会损失精度。
- H² 风格二次压缩会引入额外误差，应独立监控其误差与参数收益。

结论: 算法稳定性主要由分块策略与截断策略决定，而非单步线代算子本身不稳定。

## R12

真实成本来源:

- 构建阶段:
  - 递归分块与可容许判定开销较小；
  - 主成本在远场块 SVD（demo 直接用 `numpy.linalg.svd`）。
- 乘法阶段:
  - 稠密近场块仍占一部分成本；
  - 远场块转为 `U @ (V @ x)` 两级乘法，显著降 FLOPs。
- 内存阶段:
  - 参数量下降是关键收益指标；
  - 若簇划分不均衡或 admissibility 太严格，收益会下降。

demo 输出压缩比、重构误差与 matvec 误差，直接反映这一权衡。

## R13

该算法是近似算法，存在明确误差预算:

- 单块截断误差:
  - 对远场块 `B`，保留前 `r` 个奇异值后，
    `||B - B_r||_F^2 = sum_{k>r} sigma_k^2`。
- 全局 Frobenius 误差:
  - 分块不重叠时可按块误差平方和求和。
- 概率保证:
  - 本 demo 使用确定性 SVD，不是随机化近似，因此无概率失败项。

工程上常以“相对尾能量阈值 + 最大秩上限”双约束来控制误差与资源。

## R14

常见失效模式与鲁棒性风险:

- 可容许条件过松:
  - 会把本应近场的块误判为远场，导致秩过高或误差恶化。
- 可容许条件过严:
  - 低秩块数量下降，退化成大量稠密块，压缩比变差。
- 点集分布极不均匀:
  - 树严重不平衡，性能抖动。
- 高频核或弱平滑核:
  - 远场秩增长，H/H² 优势减弱。
- 实现层面:
  - 未固定随机种子会导致基准难复现；
  - 大规模下直接 SVD 内存/时间开销明显。

  - `eta` 控制可容许边界；

- 扩展路径:

  - 引入真正 nested basis 与 transfer matrix，升级为标准 H²。

## R17

`demo.py` 的 MVP 说明:

- 依赖: `numpy`（Python 3 环境即可运行）。
- 主要组件:
  - `ClusterNode`: 几何簇树节点；
  - `HMatrix`: 构建 H 块并支持 `matvec`/重构/统计；
  - `H2LikeMatrix`: 在 H 低秩块上做共享基二次压缩（简化 H² 风格）。
- 固定实验:
  1. 生成二维点集；
  2. 构造核矩阵 `K_ij = 1/sqrt(||xi-xj||^2 + delta^2)`；
  3. 构建 H 矩阵；
  4. 基于 H 构建 H²-like 压缩；
  5. 评估压缩比、Frobenius 重构误差与多向量 matvec 误差；
  6. 断言关键精度阈值。
- 运行:
  - `python3 demo.py`
- 设计取舍:
  - 追求“最小可验证实现”，不引入大型外部框架；
  - H² 部分是教学型共享基重压缩，不是工业级完整 nested-basis 实现。

## R18

源码级算法流程追踪（对应 `demo.py`）:

1. `build_cluster_tree(...)`
   - 对点集递归二分，生成带包围盒的簇树节点。
2. `HMatrix._build_blocks(...)`
   - 在簇树笛卡尔积上递归划分块树，并执行可容许判定。
3. `admissible(...)`
   - 用包围盒距离与簇直径判断远场可压缩块。
4. `HMatrix._try_low_rank(...)`
   - 对可压缩块取子矩阵并做 `numpy.linalg.svd`。
5. `choose_rank_from_singular_values(...)`
   - 按相对尾能量阈值选截断秩，再比较参数量决定是否采用低秩块。
6. `HMatrix.matvec(...)`
   - 近场块执行稠密乘法，远场块执行 `U @ (V @ x)`。
7. `H2LikeMatrix.from_hmatrix(...)`
   - 按簇聚合低秩因子，构建共享行/列基，再计算耦合矩阵（H² 风格重压缩）。
8. `main()`
   - 构造数据、运行 H 与 H²-like 两条路径、输出误差与压缩统计，并做断言校验。

上述流程把第三方线代调用（SVD）放在可追踪的分块框架中，而非黑箱“一步到位”。
