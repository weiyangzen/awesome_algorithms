# 刚体碰撞 (Rigid Body Collision)

- UID: `PHYS-0127`
- 学科: `物理`
- 分类: `经典力学`
- 源序号: `127`
- 目标目录: `Algorithms/物理-经典力学-0127-刚体碰撞_(Rigid_Body_Collision)`

## R01

问题定义：给定两个二维刚体（此处用“可转动圆盘”表示），在某一时刻发生单点接触碰撞，要求计算碰撞后的：
- 质心线速度 `v'`
- 角速度 `omega'`
- 接触冲量 `J = J_n n + J_t t`

并验证刚体碰撞的基础物理约束：
- 线动量守恒
- 关于固定原点的角动量守恒
- 非完全弹性与摩擦导致的动能不增

## R02

物理背景与定位：
- 刚体碰撞是游戏物理、机器人抓取、多体动力学和离散元模拟中的核心算子；
- 经典刚体碰撞常用“瞬时冲量模型”，把碰撞力在极短时间内积分成冲量；
- 本条目采用二维平面模型，保留平动与自转耦合，同时避免 3D 姿态积分复杂度。

本 MVP 是“可审计的最小实现”，重点是把公式逐步落实到代码，而不是调用黑盒引擎。

## R03

本目录解决的具体计算任务：
1. 基于两圆盘初始位置和线速度，求首次接触时刻 `TOI`（time of impact）；
2. 在接触点计算相对速度 `v_rel`，分解为法向与切向分量；
3. 根据恢复系数 `e` 和库仑摩擦 `mu` 计算 `J_n`、`J_t`；
4. 更新两刚体的 `v` 与 `omega`；
5. 输出并断言守恒/耗散指标。

## R04

建模假设（最小可运行版本）：
- 刚体为二维均匀圆盘，转动惯量 `I = 1/2 m r^2`；
- 碰撞前后外力忽略，碰撞过程视为瞬时冲量；
- 单接触点、单次碰撞，不处理持续接触与多点接触；
- 碰撞前做匀速直线运动（无外力），碰撞后继续匀速运动。

该模型适合教学与算法验证，不等价于完整刚体引擎。

## R05

核心公式（`demo.py` 直接实现）：

1. 接触点速度（二维）  
`v_c = v + omega x r`，其中 `omega x r = omega * [-r_y, r_x]`

2. 相对速度分解  
`v_rel = v_c2 - v_c1`  
`v_rel_n = dot(v_rel, n)`，`v_rel_t = dot(v_rel, t)`

3. 法向冲量  
`J_n = -(1+e) v_rel_n / K_n`  
`K_n = 1/m1 + 1/m2 + (r1 x n)^2 / I1 + (r2 x n)^2 / I2`

4. 切向冲量（摩擦）  
`J_t* = -v_rel_t / K_t`  
`J_t = clip(J_t*, -mu J_n, mu J_n)`

5. 速度更新  
`v1' = v1 - J/m1`, `v2' = v2 + J/m2`  
`omega1' = omega1 - (r1 x J)/I1`, `omega2' = omega2 + (r2 x J)/I2`

## R06

数值算法流程：
1. 读取场景参数（`e, mu, t_end`）与两圆盘初值；
2. 解二次方程得到首次接触时间 `TOI`；
3. 将两圆盘推进到 `TOI` 时刻；
4. 计算接触法向 `n`、切向 `t`、碰撞前接触相对速度；
5. 计算并截断切向冲量，得到总冲量 `J`；
6. 更新两圆盘线速度和角速度；
7. 统计碰撞前后动量、角动量、能量；
8. 生成诊断表并做断言。

## R07

复杂度分析：
- `TOI` 求解是常数规模二次方程，时间 `O(1)`；
- 单次碰撞冲量求解只含向量代数，时间 `O(1)`；
- 轨迹采样 `N` 个时刻仅做线性插值与范数，时间 `O(N)`、空间 `O(N)`。

本 MVP 的瓶颈不在算法复杂度，而在是否正确实现物理约束。

## R08

数值稳定性与鲁棒性策略：
- `TOI` 选择非负且“仍在接近”的根，避免选到分离根；
- 处理 `|v_t|` 很小时的切向方向退化（回退到 `perp(n)`）；
- 冲量分母 `K_n, K_t` 显式包含平动 + 转动项，避免漏项；
- 用严格阈值断言守恒误差，防止静默错误。

## R09

适用范围：
- 二维游戏物理中的单次刚体碰撞算子；
- 刚体冲量法教学与单元测试；
- 需要解释“恢复系数 + 摩擦 + 自转耦合”的快速原型。

不适用范围：
- 三维多体多点接触并发求解；
- 连续接触、堆叠稳定、约束求解器（LCP/NCP）级别问题；
- 形变材料、塑性、黏附等复杂接触模型。

## R10

正确性检查框架（脚本内断言）：
1. `TOI >= 0` 且场景内能检测到碰撞；
2. `J_n > 0`（确实产生法向冲量）；
3. `||p_after - p_before||` 足够小（线动量守恒）；
4. `|L_after - L_before|` 足够小（角动量守恒）；
5. `E_after <= E_before`（非增能）；
6. `v_rel_n_after + e * v_rel_n_before ≈ 0`（恢复系数关系）。

## R12

一次典型运行会输出：
- `toi`：首次接触时刻；
- `j_n, j_t, j_t_unc`：法向、切向、未截断切向冲量；
- `v_rel_n_before/after` 与 `v_rel_t_before/after`；
- `momentum_error`, `angular_error`；
- `energy_before`, `energy_after`, `energy_ratio`；
- 轨迹采样最小中心距与碰撞时中心距。

这些指标能直接判断模型是否符合冲量碰撞预期。

## R13

正确性依据：
- 冲量更新来自牛顿-欧拉动量定理，且两体受冲量大小相等方向相反；
- 恢复系数关系 `v_rel_n_after = -e v_rel_n_before` 直接约束法向速度跳变；
- 切向冲量由库仑约束 `|J_t| <= mu J_n` 限制，控制滑移减少程度；
- 因仅考虑内冲量且无外力矩，系统线动量与关于固定点角动量应守恒。

## R14

常见失败模式与修复：
- 失败：选到二次方程较大根，碰撞已过。  
  修复：只取最早非负且“接近中”的根。
- 失败：漏掉 `(r x n)^2 / I` 转动项，导致冲量偏大。  
  修复：在 `K_n`、`K_t` 中完整加入转动有效质量。
- 失败：切向方向在低滑移下数值不稳。  
  修复：`|v_t|` 很小时回退到 `perp(n)`。
- 失败：用位置修正替代冲量更新，破坏守恒检验。  
  修复：先做速度冲量，再根据阶段模型决定是否位置修正。

## R18

`demo.py` 源码级算法流程拆解（8 步）：
1. `run_simulation` 创建两刚体初始状态，并读取 `CollisionScenario`（`e, mu, t_end`）。
2. `time_of_impact` 对 `||p + vt|| = r1+r2` 建立二次方程，筛选首个非负且仍接近的根作为碰撞时刻。
3. 用 `advance_ballistic` 将两刚体推进到 `TOI`，记录碰撞前线动量、角动量、动能。
4. `resolve_collision` 计算接触法向 `n`、接触点臂长 `r_a/r_b`，并求碰撞前接触相对速度 `v_rel`。
5. 依据法向有效质量 `K_n` 计算 `J_n`，依据切向有效质量 `K_t` 计算 `J_t*`，再按 `|J_t|<=mu J_n` 截断得到 `J_t`。
6. 把总冲量 `J` 作用到两体：更新 `v` 与 `omega`，并重新计算碰撞后法向/切向相对速度。
7. 采样碰撞前后分段轨迹，统计 `momentum_error`、`angular_error`、`energy_ratio`、`restitution_residual` 等指标。
8. `main` 打印 `pandas` 报表并执行断言，确保实现不是黑盒调用，而是每个物理步骤都在源码中可追踪。
