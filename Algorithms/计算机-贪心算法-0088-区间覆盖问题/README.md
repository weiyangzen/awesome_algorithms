## R01

区间覆盖问题（Interval Covering）在本条目中的定义是：
给定目标区间 `[L, R]` 与一组候选区间，选出**数量最少**的一组，使其并集连续覆盖 `[L, R]`；若无法覆盖，则返回不可行。

这是贪心算法中的经典问题，和“合并区间”“活动选择”同属区间类核心题型。

## R02

形式化定义：

- 输入：候选集合 `I = {(s_i, e_i)}`，满足 `s_i <= e_i`；目标区间 `[L, R]`。
- 输出：索引集合 `S`，使得 `|S|` 最小，且 `∪_{i in S} [s_i, e_i] ⊇ [L, R]`。
- 不可行条件：不存在满足覆盖条件的 `S`。

本 MVP 使用闭区间语义，并允许浮点端点。

## R03

核心贪心策略：

1. 以 `start` 升序（同起点按 `end` 降序）排序；
2. 维护当前已覆盖右端 `cursor`（初始为 `L`）；
3. 在所有 `start <= cursor` 的候选中，选择 `end` 最远者；
4. 将 `cursor` 推进到该最远 `end`，并重复；
5. 若某轮不存在 `start <= cursor` 且能推进右端的区间，则不可行。

这个策略每一步都做“局部最优扩张”，并可证明得到全局最少区间数。

## R04

为什么该贪心正确（证明直觉）：

- 若当前覆盖到 `cursor`，下一段覆盖必须从某个 `start <= cursor` 的区间开始；
- 在这些可选项里，选 `end` 最大者不会比任何其他选择更差，因为它给后续留下最大剩余空间；
- 用交换论证可把任意最优解的“第一段”替换为这个最远区间而不增解规模；
- 递归应用同样论证，可得到整体最优。

因此“每轮取可达区间中最远右端”是最少覆盖数的充分策略。

## R05

复杂度：

- 排序：`O(n log n)`；
- 扫描：`O(n)`（每个区间最多被指针访问一次）；
- 总时间：`O(n log n)`；
- 额外空间：`O(n)`（排序后的副本与结果索引）。

## R06

`demo.py` 的实现包含四个层次：

- 核心算法：`greedy_interval_cover`；
- 覆盖判定：`subset_covers_target`；
- 小规模最优对照：`brute_force_min_cover`（指数复杂度，仅用于 spot-check）；
- 数据与统计：`run_fixed_cases / collect_experiments / run_stat_models`。

核心最优性来自源码中的贪心逻辑，不依赖第三方黑盒求解器。

## R07

边界与异常处理：

- 目标区间 `L > R`：抛出 `ValueError`；
- 退化目标 `L == R`：返回空解 `[]`；
- 输入区间非二元组或 `start > end`：抛出 `ValueError`；
- 无法推进覆盖右端：返回 `None` 表示不可行。

## R08

固定样例（`run_fixed_cases`）覆盖了：

- 普通可行实例；
- 单区间直接覆盖；
- 存在中间断点导致不可行；
- 同起点时需要优先选择“右端更远”候选。

每个样例都带有期望最优区间数并强校验。

## R09

随机验证（`run_optimality_spotcheck`）：

- 生成小规模可行实例；
- 贪心解与暴力最优解逐例比较；
- 强制断言 `len(greedy) == len(brute_force)`。

该步骤是“正确性抽检”，用于防止实现细节偏差。

## R10

实验数据（`collect_experiments`）对每轮记录：

- `n_intervals`：候选区间总数；
- `avg_len`：平均区间长度；
- `density`：区间对目标段的重叠密度（可大于 1）；
- `greedy_count`：贪心所选区间数。

这些指标用于解释“实例结构”与“最少覆盖数量”之间的关系。

## R14

数值与鲁棒性细节：

- 浮点比较使用 `eps=1e-12`，避免边界误差导致的误判；
- 覆盖判定以“连续可达右端”方式进行，而非离散采样；
- 统计模型前保持 deterministic seed，确保复现性。

## R15

当前 MVP 的范围与局限：

- 只处理一维区间最少覆盖；
- 不输出所有最优解，只输出一个可行最优索引序列；
- 不处理带权覆盖（例如每个区间有成本）；
- 不处理在线增删场景。

## R16

## R17

工具栈与职责（最小但完整）：

- `numpy`：随机实例构造与数值特征统计；
- `scipy`：`spearmanr` 相关性诊断；
- `pandas`：表格化展示验证与实验结果；
- `scikit-learn`：线性回归解释特征与覆盖数量关系；
- `PyTorch`：用线性层做可微回归，展示同一数据的梯度优化拟合。

这些库用于“验证和解释”，不替代核心贪心算法本身。

## R18

`demo.py` 的源码级算法流程（8 步）：

1. `main` 固定随机种子并设置目标区间 `[0, 100]`，保证可复现执行。  
2. `run_fixed_cases` 调用 `greedy_interval_cover`，在人工样例上验证可行/不可行与期望最优区间数。  
3. `greedy_interval_cover` 先排序区间，再用单指针扫描所有 `start <= cursor` 的候选，并选取 `end` 最远者推进覆盖。  
4. 若某轮不存在可推进区间，`greedy_interval_cover` 返回 `None`，表示目标区间无法连续覆盖。  
5. `run_optimality_spotcheck` 生成小规模实例，调用 `brute_force_min_cover` 穷举子集，断言贪心解规模等于最优解规模。  
6. `collect_experiments` 生成批量可行实例，记录 `n_intervals / avg_len / density / greedy_count` 到 `pandas.DataFrame`。  
7. `run_stat_models` 用 `scipy` 计算 Spearman 相关性，用 `sklearn` 做线性回归，并用 `torch` 线性层做梯度拟合。  
8. `main` 打印固定样例、实验摘要与统计诊断，形成可直接验收的最小端到端结果。
