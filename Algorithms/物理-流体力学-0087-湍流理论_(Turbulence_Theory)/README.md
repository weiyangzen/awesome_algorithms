## R01

湍流理论研究高雷诺数流动中由多尺度非线性耦合产生的无规则时空结构。其关键特征包括：

- 宽频谱（能量分布在多个波数/尺度）；
- 强非线性（对流项 `u·∇u` 主导）；
- 统计描述优先（均值、方差、谱、耗散率）；
- 对初值和扰动敏感。

本条目给出一个可运行、可核查的 2D 湍流最小实现，而不是黑箱 CFD 调用。

## R02

本 MVP 的任务目标：

- 在周期边界上实现 2D 不可压 Navier-Stokes 的涡量-流函数形式；
- 使用伪谱法（FFT）推进涡量场随时间演化；
- 加入低波数外力维持受迫湍动态；
- 输出动能、涡量平方（enstrophy）、耗散率、CFL、能谱等诊断；
- 提供内置断言，确保结果满足基本物理/数值一致性。

## R03

采用方程（2D 不可压，涡量形式）：

`dω/dt + u·∇ω = ν∇²ω - αω + f`

其中：

- `ω`：标量涡量；
- `ν`：运动黏度；
- `α`：线性大尺度阻尼（Ekman-like drag）；
- `f`：低波数受迫项。

速度通过流函数 `ψ` 恢复：

- `∇²ψ = -ω`
- `u = ∂ψ/∂y`
- `v = -∂ψ/∂x`

## R04

空间离散采用 2D 周期伪谱法：

- 用 FFT 在实空间/谱空间间切换；
- 导数在谱空间计算：`∂/∂x -> i*kx`，`∂/∂y -> i*ky`；
- 拉普拉斯在谱空间为 `-k²`，因此粘性项可写成 `-ν*k²*ω_hat`。

为抑制混叠误差，代码使用 2/3 截断：

- 只保留 `|kx|, |ky| <= (2/3)k_max` 模式；
- 每次 RHS 和时间推进后都应用 `dealias_mask`。

## R05

受迫与耗散建模：

- 受迫：在低波数（`k_f=4`）施加确定性外力场
  `f = A[sin(k_f x) + sin(k_f y) + 0.5 sin(k_f(x+y))]`；
- 小尺度耗散：`ν∇²ω`；
- 大尺度泄放：`-αω`。

这样可以在不引入复杂边界条件的前提下，维持非平凡多尺度结构并避免能量无限累积。

## R06

时间积分使用二阶显式 RK2（Heun 形式）：

1. `k1 = RHS(ω^n)`
2. `k2 = RHS(ω^n + dt*k1)`
3. `ω^{n+1} = ω^n + 0.5*dt*(k1 + k2)`

每步后做：

- 去混叠掩码；
- 零均值约束 `ω_hat[0,0]=0`（去除无物理意义的常数模）。

## R07

复杂度（设网格 `N x N`，时间步 `T`）：

- 每步主要成本来自若干次 2D FFT：`O(N^2 log N)`；
- 总复杂度：`O(T * N^2 log N)`；
- 内存复杂度：`O(N^2)`。

在默认 `N=64, T=1000` 下，CPU 上可快速完成，适合作为可验证 MVP。

## R08

脚本输出诊断量：

- `kinetic_energy = 0.5 <u^2+v^2>`
- `enstrophy = 0.5 <ω^2>`
- `dissipation_rate = 2ν*enstrophy`
- `u_rms`, `u_max`, `cfl`
- 壳层各向同性能谱 `E(k)`
- 基于受迫尺度 `L_f = L/k_f` 的 `Re_forcing = u_rms*L_f/ν`

这些量用于判断“是否进入非平凡受迫流动态”与“数值是否健康”。

## R10

正确性框架（本实现）：

1. 方程闭环：`ω -> ψ -> (u,v) -> advection -> RHS -> ω` 全链路显式可见。  
2. 谱导数一致：导数、拉普拉斯都由 `i*k` 和 `k²` 精确实现。  
3. Parseval 校验：`sum_k E(k)` 与物理空间动能一致（脚本内断言）。  
4. 稳定性监控：跟踪 CFL，限制 `max(cfl)` 在安全阈值内。  
5. 结果可复现：固定随机种子初始化。

## R11

数值误差与稳定性处理：

- 显式时间推进误差阶为 `O(dt^2)`；
- 伪谱非线性项潜在混叠通过 2/3 规则抑制；
- 粘性项与线性阻尼提供高/低波数双向稳定机制；
- 通过 `np.isfinite` 断言防止 NaN/Inf 静默传播。

默认参数下 `cfl` 明显小于 1，时间推进处于保守稳定区。

## R14

常见失败模式：

1. `dt` 过大导致显式推进发散（CFL 或扩散稳定条件被破坏）。
2. 忘记去混叠，出现谱污染和能量异常转移。
3. `k` 网格单位不一致，导致导数/拉普拉斯系数错误。
4. 受迫太弱时流场衰减为近层流；受迫太强时会造成过度刚性。
5. 将 2D 结果误当作 3D Kolmogorov 规律的定量替代。

## R15

实现结构（对应 `demo.py`）：

- `TurbulenceConfig`：统一管理参数并校验。
- `build_spectral_operators`：构造 `k` 网格、`k²`、去混叠掩码、外力场。
- `initialize_vorticity_hat`：生成可复现初始涡量谱。

- `isotropic_energy_spectrum`：计算壳层能谱。
- `main`：打印结果并执行最终断言。

## R18

`demo.py` 源码级算法流程（8 步）：

1. `main` 创建 `TurbulenceConfig` 并调用 `validate`，确定网格、`dt`、`ν`、`α`、受迫参数。  
2. `build_spectral_operators` 生成 `kx/ky/k²/inv_k²`、2/3 去混叠掩码和低波数受迫场 `f(x,y)`。  
3. `initialize_vorticity_hat` 用固定随机种子生成初始 `ω_hat`，并做低通滤波与零均值处理。  
4. 在 `simulate` 中进入时间循环：先用 `diagnostics` 记录当前动能、enstrophy、`u_rms/u_max/CFL`。  
5. `vorticity_rhs` 内部先由 `ω_hat -> (u,v)`，再算 `∂ω/∂x, ∂ω/∂y` 和对流项 `u·∇ω`，拼出 RHS。  
6. 按 RK2 更新 `ω_hat`：`k1=RHS(ω)`, `k2=RHS(ω+dt*k1)`, `ω<-ω+0.5*dt*(k1+k2)`，随后再次去混叠。  
7. 终态通过 `isotropic_energy_spectrum` 计算壳层能谱 `E(k)`，再求积分尺度、受迫尺度雷诺数和谱斜率。  
8. `main` 执行断言（有限值、能量增长、CFL、安全谱闭合、低/高波段关系、Re 下限），全部通过则打印 `All checks passed.`。  

第三方库边界说明：`numpy` 负责 FFT 和向量化数值计算，`pandas` 仅用于日志表打印；湍流方程离散与时间推进均由源码显式实现，无外部 CFD 黑盒调用。
