## R01

Pollard's Rho（整数分解版）是一个随机化整数分解算法，核心思想是：
在模 `n` 的有限状态空间里构造伪随机迭代序列 `x_{k+1}=f(x_k) mod n`，并利用“生日悖论”导致的循环碰撞，让两个序列在某个未知素因子 `p | n` 上先相遇，从而通过 `gcd(|x-y|, n)` 抽取出非平凡因子。

- 输入: 待分解整数 `n > 1`（通常针对大奇合数）。
- 输出: `n` 的一个非平凡因子 `d`（`1 < d < n`），或在上层流程中递归得到完整素因子分解。
- 主要步骤:
  1. 预处理小因子（如 2）。
  2. 选取随机参数 `x0, c`，定义 `f(x)=x^2+c mod n`。
  3. 用 Floyd 龟兔指针推进 `x=f(x)`, `y=f(f(y))`。
  4. 每轮计算 `d=gcd(|x-y|, n)`。
  5. 若 `d=1` 继续；若 `1<d<n` 返回因子；若 `d=n` 说明本轮退化，重启随机参数。
  6. 与素性检测（常用 Miller-Rabin）配合递归，直到全是素数。

## R02

提出时间通常记为 1975 年，提出者是 John M. Pollard。
经典参考文献为：

- J. M. Pollard, *A Monte Carlo method for factorization*, BIT Numerical Mathematics, 1975。

该算法与 Pollard 在 1974 年提出的 `p-1` 方法同属“轻量级随机分解法”体系，但 Rho 通过循环检测机制，适用面更广。

## R03

历史背景是“需要在通用合数上比试除法更快的分解方法”：

- 试除法复杂度约 `O(sqrt(n))`，对大整数不可用。
- Fermat 分解在因子相近时有效，但对一般合数不稳定。
- `p-1` 方法依赖某个素因子 `p` 使 `p-1` 足够光滑，命中条件窄。

Pollard's Rho 的突破点是：不强依赖特殊代数结构，而利用伪随机轨道在有限域上的碰撞行为，提供一个平均意义下更高效、实现非常简洁的 Monte Carlo 分解方案。

## R04

设 `n = p*q`（`p <= q`）且 `p` 是最小素因子。典型复杂度分析为：

- 期望迭代次数: `O(sqrt(p))` 次函数迭代（生日悖论量级）。
- 每次迭代成本: 一到两次模乘 + 一次 `gcd`，约 `O(log^2 n)` 到 `O(log^3 n)` 位复杂度（取决于乘法模型）。
- 整体典型成本（位复杂度粗略写法）: `~O(sqrt(p) * polylog(n))`。

最坏情况下没有严格多项式上界（随机算法可能多次重启）；工程上通过多 seed/多多项式重试可显著降低卡死概率。

## R05

额外空间很小，核心是常数级状态：

- 当前龟兔指针 `x, y`
- 多项式参数 `c`
- 若干临时整数与 `gcd` 变量

因此工作内存为 `O(1)`（不计输入输出与递归保存完整因子列表的空间）。
若递归分解整数字典，空间主要由因子结果表决定，通常可视为 `O(k)`，`k` 为素因子个数（含重数）。

## R06

示例：分解 `n = 8051 = 83 * 97`。

取 `f(x)=x^2+1 mod 8051`，初值 `x=y=2`，每轮计算 `d=gcd(|x-y|,8051)`：

1. `x=5, y=26, d=gcd(21,8051)=1`
2. `x=26, y=7474, d=gcd(7448,8051)=1`
3. `x=677, y=871, d=gcd(194,8051)=97`

得到非平凡因子 `97`，另一个因子为 `8051/97=83`，分解完成。

## R07

算法意义：

- 理论上：是随机化整数分解算法的里程碑，把“碰撞检测 + gcd 抽因子”范式固定下来。
- 工程上：实现短小、常数低，是很多分解工具链里的默认子程序（尤其用于先拆中小因子）。
- 方法论上：把循环检测（Floyd/Brent）与数论分解耦合，影响后续离散对数版 Pollard Rho、ECM 等随机算法设计。

## R08

路径依赖的核心知识/算法：

- 欧几里得算法（`gcd` 计算，直接决定能否抽出因子）
- 模运算与快速模乘（迭代函数 `f(x)` 的计算基础）
- Floyd 或 Brent 循环检测（检测轨道碰撞）
- Miller-Rabin 素性测试（在递归分解时判断“是否继续分解”）
- 随机数生成（重启策略与成功概率相关）

## R09

适用前提与边界：

- 目标是整数分解，输入需为 `n > 1` 的整数。
- 对偶数可直接取 2，不必进入 Rho 主循环。
- 对素数 `n`，Rho 不能给出非平凡因子，应先做素性检测并直接返回“素数”。
- 对极端构造输入，单次随机轨道可能长时间不给因子，需要重启或改多项式。
- 对超大整数（密码学位数）常作为分解流程中的子例程，单靠 Rho 不一定足够。

## R10

正确性依据要点：

1. 若 `p | n`，则序列在模 `p` 意义下落入有限集合，必有循环。
2. 龟兔指针在模 `p` 上碰撞后，出现 `x ≡ y (mod p)`，即 `p | (x-y)`。
3. 若同时 `x !≡ y (mod q)`（对另一因子 `q`），则 `q ∤ (x-y)`。
4. 此时 `gcd(|x-y|, n)` 会抽出包含 `p` 的真因子，满足 `1 < d < n`。
5. 当 `d=1` 说明尚未形成可分离碰撞；当 `d=n` 说明碰撞“同时发生在所有因子上”或轨道退化，需随机重启。

因此算法是 Monte Carlo 型：一旦返回 `1<d<n`，该结果确定正确；失败只体现在“本轮未成功”。

## R11

该算法属于离散整数算法，不涉及浮点舍入误差；“稳定性”主要是概率稳定性而非数值稳定性：

- 无浮点误差传播，只有整数模运算与 `gcd`。
- 误差源是随机轨道选择不佳（如陷入短坏环、`d=n` 退化）。
- 传播表现为运行时间方差较大，而不是结果偏差。
- 通过多次重启、多多项式参数、Brent 改进可降低时间波动。

结论：数值上稳定（精确整数运算），概率性能上有波动。

## R12

真实成本除了渐近复杂度，还包括：

- 模乘常数项：大整数乘法是主要热点。
- `gcd` 调用频率：每轮都算 `gcd` 会产生不小常数，Brent 变体常做批量 `gcd` 降成本。
- 分支与缓存：大整数对象分配、取模实现、Python 大整数内部路径会影响性能。
- 随机重启开销：失败重启越多，真实耗时波动越大。
- 并行化：可对不同 seed 并行尝试，几乎线性加速“首次命中因子”的墙钟时间。

在 Python MVP 中，`math.gcd` 与内置大整数运算是关键性能点；无需 GPU。

## R13

概率保证（经验型）：

- Pollard's Rho 是 Las Vegas / Monte Carlo 风格混合使用的实践算法：
  - 找到 `1<d<n` 时结果必然正确；
  - 单次尝试存在失败/超时可能，需要重启。
- 对随机映射模型，命中最小因子 `p` 的期望步数为 `Theta(sqrt(p))`。
- 多次独立重启后，失败概率按几何方式下降：若单次成功率为 `s`，`k` 次都失败概率约 `(1-s)^k`。

因此工程上可通过“限定每次最大迭代 + 多 seed 重试”给出可控成功概率。

## R14

鲁棒性与失效模式：

- 失效 1：`d=n`。原因是轨道退化或碰撞不可分离；处理方式是换 `c`/`x0` 重启。
- 失效 2：长时间 `d=1`。原因是当前轨道未触发有效碰撞；处理方式是设置迭代上限后重启。
- 失效 3：把素数误当合数继续 Rho。处理方式是先做 Miller-Rabin。
- 失效 4：递归深度与重复因子处理不当。处理方式是先抽出小因子并在递归中处理 `d` 与 `n//d`。

整体上算法鲁棒，但必须实现好“重启策略 + 素性判定 + 终止条件”。

## R15

工程实现要点：

- 先做快速分支：`n<2`、偶数、`is_probable_prime(n)`。
- Rho 内核使用 `f(x)=x^2+c mod n`，`c` 取随机且避免退化常值。
- 使用 Floyd 龟兔指针实现最小 MVP，逻辑清晰、代码短。
- 为避免无限循环，设置每轮最大迭代，超限即重启。
- 分解流程采用递归或栈：拿到因子 `d` 后分解 `d` 与 `n//d`。
- 输出前对素因子排序并验算乘积，作为自校验。

常见陷阱：未处理 `d==n`、未做素性测试、随机种子不可复现、或对 `n=1` 未显式处理。

## R16

前驱与后继关系：

- 前驱：
  - 欧几里得算法（基础）
  - Fermat 分解、试除法（更早但效率较低）
  - Pollard `p-1`（同作者前期方法，依赖光滑性）
- 后继/扩展：
  - Brent 版本 Pollard Rho（更少 `gcd`、常数更优）
  - ECM（椭圆曲线分解，处理大因子常更强）
  - 与 QS/NFS 组合成分层分解流水线

典型应用：

- 通用整数分解工具中的“中小因子拆分器”
- 密码学教学和 CTF 数论题
- 大规模分解前的预处理阶段（先用轻量算法筛掉易分部分）

## R17

Python MVP 方案（`demo.py`）：

- 工具栈：仅使用 Python 标准库（`math`, `random`, `time`, `typing`），不依赖外部包，保证开箱可运行。
- 组件设计：
  1. `is_probable_prime`：64 位范围确定性 Miller-Rabin（固定基集合）。
  2. `pollards_rho`：Floyd 龟兔 + `gcd` 抽因子 + 重启机制。
  3. `factorize`：递归分解，直到全为素数。
  4. `factor_list` 与 `verify_factorization`：结果整理和正确性校验。
  5. `main`：对多组预置整数运行演示，打印分解结果和耗时。
- 运行方式：`python3 demo.py`，无需交互输入。

## R18

基于本目录 `demo.py` 的源码级流程可拆为 8 步：

1. `main` 读取预置测试整数列表，逐个调用 `factor_list(n)`。
2. `factor_list` 创建因子计数字典，调用 `factorize(n, rng, factors)`。
3. `factorize` 先处理基线：
   - `n == 1` 直接返回；
   - `is_probable_prime(n)` 为真则把 `n` 计入结果并返回。
4. 若 `n` 仍为合数，则调用 `pollards_rho(n, rng)` 获取一个非平凡因子 `d`。
5. `pollards_rho` 初始化随机 `x, c`，定义迭代函数 `f(x)=x^2+c mod n`，用龟兔指针推进：
   - `x = f(x)`；
   - `y = f(f(y))`；
   - `d = gcd(abs(x-y), n)`。
6. 当 `d=1` 继续迭代；当 `1<d<n` 返回该因子；当 `d=n` 说明本轮失败，随机重启参数重试。
7. `factorize` 拿到 `d` 后递归处理 `d` 与 `n//d`，直到全部叶子都是素数。
8. 回到 `main`，将字典展开为有序素因子列表，执行乘积回验并输出耗时与结果。

这不是对第三方黑箱的调用，而是完整显式实现了 Pollard's Rho 的核心算法路径。
