## R01

Dantzig-Wolfe 分解用于求解具有块角结构（block-angular structure）的线性规划：
- 变量可拆为多个独立块 `x_b`；
- 每个块有自己的局部约束 `x_b ∈ X_b`；
- 块之间只通过少量“耦合约束/链接约束”相连。

它通过“列生成 + 主从分解”把大问题变为：
- 受限主问题（RMP）：只在当前列集上求最优；
- 定价子问题（Pricing）：在每个块里找是否存在负 reduced cost 的新列。

## R02

典型原始模型可写为：

- `min Σ_b c_b^T x_b`
- `s.t. Σ_b D_b x_b >= d`（链接需求）
- `x_b ∈ X_b = {x | A_b x <= b_b, x >= 0}`

Dantzig-Wolfe 的核心是把每个 `x_b` 表示成该块极点的凸组合：

- `x_b = Σ_k λ_{b,k} v_{b,k}`
- `Σ_k λ_{b,k} = 1, λ_{b,k} >= 0`

于是原问题转化为“列变量 `λ` 的主问题”，每一列就是一个块的极点（或极射线）。

## R03

本目录 `demo.py` 的输入/输出（无交互，内置实例）：
- 输入：两块局部多面体、两条链接需求、线性成本。
- 输出：
  - 迭代日志（每轮 RMP 目标值、对偶价、各块 reduced cost、是否加列）；
  - D-W 恢复的分块决策 `x_b`；
  - 与单体 LP（不分解直接求解）的目标值对比。

## R04

为什么要分解：
- 若直接做单体 LP，变量与约束会随块数快速膨胀；
- D-W 主问题只保留“当前少量有价值列”；
- 子问题按块独立，可并行，且常有结构（背包、最短路、网络流等）可利用。

因此 D-W 更像是“结构驱动的求解框架”，不是单一公式替换。

## R05

本 MVP 使用了带人工变量的稳定初始化：
- 链接约束是 `Σ cover + s >= demand`；
- 人工变量 `s >= 0` 以大罚项 `M` 进目标；
- 初始每块只给一个零列，仍可借 `s` 保证 RMP 可行。

优点：避免手工构造可行初始列，流程更稳健、可复现实验。

## R06

受限主问题（RMP）在代码中的形式：
- 决策变量：`λ_{b,k}` + 人工变量 `s_i`；
- 目标：`min Σ c_b(v_{b,k}) λ_{b,k} + M Σ s_i`；
- 约束：
  - `Σ cover_i(v_{b,k}) λ_{b,k} + s_i >= d_i`；
  - 每块 `Σ_k λ_{b,k} = 1`。

实现上用 `demo.py` 内置的 `solve_lp_small()`（活跃约束枚举）分别求解 RMP 的原始与对偶形式。

## R07

定价子问题（每块独立）：

给定主问题对偶 `π`（链接约束）和 `μ_b`（该块凸性约束），新列 `v` 的 reduced cost 为：

- `rc(v) = c_b^T v - π^T(D_b v) - μ_b`

若存在 `rc(v) < 0`，就把该列加入主问题；否则该块在当前对偶下已无改进列。

## R08

本实现里，块维度很小（二维），所以没有把子问题再交给黑盒 LP：
- 先用活跃约束组合枚举每块多面体顶点；
- 定价时在顶点集合上做线性目标最小化（取最小值顶点）；
- 用这个顶点作为候选新列。

这样源码可审计：你能直接看到“列从哪里来”。

## R09

终止条件：
- 所有块都找不到可加入的新列（即无 `rc < -tol` 的非重复列）；或
- 达到 `max_iter`。

在 LP 设定下，若无负 reduced cost 列，则当前 RMP 达到 D-W 意义下最优（对当前分解模型）。

## R10

正确性直觉：
1. RMP 是原问题列空间的受限版本；
2. 定价等价于在完整列空间里寻找最违反最优性条件的列（最负 reduced cost）；
3. 没有负列时，KKT/对偶最优性条件满足；
4. 因而当前主问题解可视作全列主问题最优解。

## R11

复杂度（本教学版）：
- 每轮 RMP：由 `solve_lp_small()` 求解，规模随已生成列数增长；
- 每块定价：在顶点集上线性扫描，`O(|V_b|)`；
- 总成本约为“迭代轮数 ×（主问题求解 + 各块定价）”。

工业版的效率关键通常在：
- 子问题是否可用专门算法（最短路/背包/网络流）；
- 列稳定化与并行定价。

## R12

数值与工程注意：
- `rc` 判定需容差 `tol`，避免浮点噪声导致抖动；
- 要去重列，否则退化时容易循环加无效列；
- 人工变量罚系数 `M` 要足够大但不宜极端夸张，避免病态放大；
- 对偶符号必须与约束方向一致（本代码显式做了 `π = -marginals` 转换）。

## R14

预期输出重点：
- `Iteration log`：显示每轮 `obj / art_sum / rcs / added / pi`；
- `Recovered block decisions`：D-W 从 `λ` 恢复出的 `x_b`；
- `Artificial sum in final RMP`：应接近 0，表示回到原问题可行域；
- `Objective gap |DW - monolithic|`：应接近 0，验证分解实现正确。

## R15

可应用场景：
- 大规模多商品流/多期计划中的块角 LP；
- 车辆路径、机组排班、切割库存等列生成框架；
- Benders 与 Branch-and-Price 体系中的主问题层。

D-W 常作为“底层引擎”，不是孤立算法。

## R16

最小测试建议：
- 改小 `demand` 验证人工变量快速归零；
- 增大 `demand` 到边界，观察是否出现人工变量残留（可能指示不可行）；
- 调整 `big_m` 与 `tol`，检查迭代稳定性；
- 对比单体 LP，确认目标值与可行性一致。

## R17

可扩展方向：
- 将每块定价从“顶点枚举”替换为块内专用求解器（如背包 DP）；
- 多块并行定价（线程/进程）；
- 引入对偶稳定化（bundle/penalty）减少震荡；
- 接入整数主问题，走 Branch-and-Price。

## R18

`demo.py` 的源码级算法流程可拆为 8 步：
1. 在 `build_demo_blocks()` 定义块角 LP：局部约束 `A_b x<=b_b`、成本 `c_b`、链接映射 `D_b`。
2. 在 `enumerate_polytope_vertices()` 枚举每块极点，得到可作为列的候选集合。
3. 以“每块零列 + 人工变量”构造初始 RMP（`run_dantzig_wolfe()`）。
4. 分别调用 `solve_rmp_primal()` 与 `solve_rmp_dual()`，读取 `λ`、人工变量、目标值与对偶 `π, μ`。
5. 在 `solve_pricing_for_block()` 对每块计算 `rc(v)=c_b^Tv-π^T D_b v-μ_b`，找最小 rc 顶点。
6. 若某块 `rc < -tol` 且列不重复，则把该顶点加入该块列集，进入下一轮。
7. 若所有块都无可加列，则停止迭代，说明当前列集已满足最优性条件。
8. 用 `recover_block_decisions()` 还原 `x_b`，再由 `solve_monolithic_lp()` 做单体 LP 对照，输出目标差。

这里第三方求解器只负责“当前 RMP LP 的数值解”；
D-W 的核心闭环（对偶提取、定价、扩列、终止判定）都在源码中显式实现。
