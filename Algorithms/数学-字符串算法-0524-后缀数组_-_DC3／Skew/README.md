## R01

本条目实现后缀数组的 `DC3/Skew` 线性时间构造算法（Difference Cover Mod 3）。

目标是给定字符串 `S`，输出后缀数组 `SA`：
- `SA[i]` 表示字典序第 `i` 小后缀的起始位置。

本目录提供一个可运行 MVP：
- 纯 Python 手写 `DC3/Skew` 核心，不依赖后缀数组黑盒库；
- 附带 `Kasai` 的 `LCP` 计算，便于结果解释；
- 自带固定样例与随机回归校验，执行一次即可验真。

## R02

问题定义：
- 输入：字符串 `text`，长度为 `n`。
- 输出：长度同为 `n` 的数组 `SA`，满足
  - `text[SA[0]:] < text[SA[1]:] < ... < text[SA[n-1]:]`（字典序）。

附加输出（演示中）：
- `LCP` 数组，`LCP[i]` 为后缀 `SA[i]` 与 `SA[i-1]` 的最长公共前缀长度，`LCP[0]=0`。

## R03

DC3/Skew 的核心思想：
- 将位置按 `i mod 3` 分成三组；
- 先对 `i mod 3 != 0` 的后缀（记作样本后缀）做排序；
- 用样本后缀的秩来帮助比较 `i mod 3 == 0` 的后缀；
- 最后线性归并两部分有序结果。

关键数学结构是 `mod 3` 的差覆盖（difference cover），使得比较任意两类后缀时，都能降解成常数项字符 + 已知秩比较。

## R04

`demo.py` 的总流程：
1. 把字符串映射为整数序列（字符码 + 1，预留 `0` 作为哨兵）。
2. 调用 `_suffix_array_dc3_int` 执行 DC3 递归排序。
3. 在样本后缀上三次基数排序（按三元组）。
4. 三元组重命名；若名字不唯一则递归。
5. 对 `mod 0` 后缀做一次基数排序。
6. 线性归并得到完整 `SA`。
7. 使用 `Kasai` 算法计算 `LCP`。
8. 运行固定+随机对照测试（对比朴素后缀数组）。

## R05

核心数据结构：
- `s`: 输入整数序列并在末尾补 3 个 `0` 哨兵。
- `s12`: 样本后缀索引或递归秩数组（`mod 1/2` 组）。
- `sa12`: 样本后缀的排序结果（在递归层中也可表示秩空间 SA）。
- `s0`: `mod 0` 后缀的索引列表。
- `sa0`: `mod 0` 后缀按首字符排序后的结果。
- `sa`: 最终后缀数组。

## R06

正确性要点：
- 三次稳定基数排序确保样本后缀按三元组正确排序。
- 三元组重命名后：
  - 若名字唯一，样本后缀序直接可得；
  - 若不唯一，递归求解保证样本后缀全序正确。
- `mod 0` 后缀通过首字符排序得到局部有序。
- 归并比较公式：
  - `mod 1` 样本与 `mod 0` 比较时用二元比较 `(s[i], rank(i+1))`；
  - `mod 2` 样本与 `mod 0` 比较时用三元比较 `(s[i], s[i+1], rank(i+2))`。
- 以上比较均可在 `O(1)` 完成，归并线性，故整体有序性成立。

## R07

复杂度：
- 时间复杂度：`O(n)`（DC3 理论线性时间）。
- 空间复杂度：`O(n)`（`s12/sa12/s0/sa0/sa` 等辅助数组）。

在本 MVP 规模下（教学与小样本验证），运行时间通常为秒级以内。

## R10

`demo.py` 主要函数职责：
- `_radix_pass`：对索引按 `s[i+offset]` 做稳定计数排序。
- `_suffix_array_dc3_int`：DC3 递归主过程（样本排序、命名、递归、归并）。
- `suffix_array_dc3`：字符串入口封装（字符转整数 + 哨兵）。
- `lcp_kasai`：由 `SA` 线性构造 `LCP`。
- `_naive_sa`：朴素对照实现（用于验真）。
- `_run_self_checks`：固定样例 + 随机样例断言。
- `main`：展示输出表格并打印通过标记。

## R15

与常见后缀数组构造法对比：
- 对比朴素排序：
  - 朴素法 `O(n^2 log n)` 或更差；
  - DC3 理论 `O(n)`。
- 对比前缀倍增（doubling）：
  - 倍增法常见 `O(n log n)`，实现直观；
  - DC3 更接近线性，但实现更复杂。
- 对比 SA-IS：
  - SA-IS 同样可线性；
  - DC3 在“按模分组 + 递归命名”思路上更直接展示差覆盖思想。

## R18

`demo.py` 的源码级算法流（8 步，非黑盒）如下：

1. `suffix_array_dc3` 将输入字符串映射到整数数组 `seq`，并追加三个 `0` 哨兵，调用 `_suffix_array_dc3_int`。  
2. `_suffix_array_dc3_int` 计算 `n0/n1/n2/n02`，收集所有 `i mod 3 != 0` 的样本后缀到 `s12`。  
3. 对 `s12` 依次按偏移 `2/1/0` 执行 `_radix_pass`，得到样本后缀按三元组 `(s[i], s[i+1], s[i+2])` 的稳定排序。  
4. 线性扫描排序结果做“重命名”：相同三元组同名，不同三元组名字递增，并把名字写回 `s12` 的递归坐标空间。  
5. 若名字数 `< n02`，递归调用 `_suffix_array_dc3_int(s12, n02, name)` 继续细分排序；否则直接由名字反推样本 SA。  
6. 从样本 SA 中抽取 `mod 1` 对应的 `mod 0` 位置，按首字符一次 `_radix_pass` 得到 `sa0`。  
7. 归并 `sa0` 与 `sa12`：对 `mod 1` 用二元比较 `(s[i], rank(i+1))`，对 `mod 2` 用三元比较 `(s[i], s[i+1], rank(i+2))`，逐项写入最终 `sa`。  
8. `main` 用 `lcp_kasai` 计算 LCP，并通过固定样例+随机样例对照 `_naive_sa` 的断言，验证实现正确性。
