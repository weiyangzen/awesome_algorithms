## R01

Sutherland-Hodgman 是经典二维多边形裁剪算法，用于计算“主体多边形 ∩ 裁剪窗口多边形”的结果。它在图形学管线中常用于视口裁剪、UI 形状裁切、地图几何预处理等任务。

算法特点是逐边裁剪：把主体多边形依次拿去和裁剪多边形的每一条边做半平面裁切，最终得到位于窗口内部的输出多边形。

## R02

本条目问题定义：

- 输入 1：主体多边形 `subject`（可凸可凹，顶点按顺序给出）；
- 输入 2：裁剪多边形 `clipper`（要求凸多边形，顶点按顺序给出，顺/逆时针都可）；
- 输出：裁剪后的多边形顶点序列（若无交集则返回空列表）。

`demo.py` 约定输出为 `list[tuple[float, float]]`。

## R03

核心思想（逐边裁剪）：

1. 令 `output = subject`；
2. 对 `clipper` 的每条边 `E`：
   - 读取上一轮 `output` 作为 `input`；
   - 遍历 `input` 的每条边（前一点 `S` 到当前点 `P`）；
   - 按 `S`、`P` 相对 `E` 的内外关系，决定输出 `P` 和/或交点；
3. 一轮结束得到新 `output`，继续下一条裁剪边；
4. 全部边处理完后的 `output` 即结果。

这就是 Sutherland-Hodgman 的标准流程。

## R04

单条裁剪边下的四种情况：

- `S` inside, `P` inside：输出 `P`
- `S` inside, `P` outside：输出交点
- `S` outside, `P` inside：输出交点，再输出 `P`
- `S` outside, `P` outside：不输出

其中 inside 判定来自“点是否位于裁剪边对应的内侧半平面”。

## R05

关键几何量：

- 叉积 `cross((b-a), (p-a))` 用于点在有向边 `a->b` 的左右侧判断；
- 直线交点公式：
  - 设主体边 `s + t*(e-s)`，裁剪边 `a + u*(b-a)`；
  - 由二维叉积可得 `t = cross((a-s), (b-a)) / cross((e-s), (b-a))`；
  - 交点 `I = s + t*(e-s)`。

`demo.py` 用这个显式公式实现交点，不依赖第三方黑盒几何裁剪函数。

## R06

复杂度分析：

- 设主体多边形顶点数为 `n`，裁剪多边形顶点数为 `m`；
- 每条裁剪边都会线性扫描一次当前输出顶点，量级约 `O(n)` 到 `O(n+m)`；
- 总体时间复杂度常写为 `O(n*m)`；
- 空间复杂度为 `O(n + k)`（`k` 为新生成交点数量，通常并入输出规模）。

## R07

本 MVP 的实现设计：

- `sutherland_hodgman`：核心裁剪循环；
- `_inside`：半平面内外判定（兼容顺/逆时针裁剪窗口）；
- `_line_intersection`：直线交点计算；
- `_clean_polygon`：清理连续重复点与尾首重复点；
- `polygons_equivalent`：固定样例中比较多边形（允许循环位移和反向顺序）；
- `run_fixed_cases` + `run_random_property_checks`：回归测试与性质测试。

## R08

前置知识：

- 平面向量叉积；
- 多边形顶点顺序与有向边；
- Shoelace 面积公式（用于方向和面积检查）。

运行环境：

- Python 3.10+
- `numpy`（用于面积计算、随机测试与数值检查）

## R09

适用场景：

- 二维图形渲染中的多边形窗口裁剪；
- GIS/几何处理中与凸区域求交；
- UI 裁切、矢量图可视区域截取。

限制：

- 标准 Sutherland-Hodgman 假设裁剪窗口是凸多边形；
- 若裁剪窗口是凹多边形，需改用更一般算法（如 Weiler-Atherton）。

## R10

正确性直觉（不写完整形式化证明）：

- 对每条裁剪边，算法输出的是“当前多边形与该边内侧半平面”的精确交；
- 多轮迭代后，相当于与所有半平面的交集；
- 对凸裁剪多边形，所有边内侧半平面的交集恰好是该凸多边形本体；
- 因此最终输出等于 `subject ∩ clipper`。

这也是该算法能稳定工作的核心不变式。

## R11

边界与鲁棒性处理：

- 主体或裁剪多边形顶点不足 3：抛 `ValueError`；
- 裁剪多边形非凸：抛 `ValueError`；
- 相邻重复点、闭合重复点：`_clean_polygon` 自动清理；
- 平行或近乎平行导致分母接近 0：交点函数使用稳定回退，避免数值爆炸；
- 裁剪后顶点不足 3：按空多边形返回。

## R12

实现细节取舍：

- inside 判定通过裁剪多边形有向面积自动识别顺/逆时针，避免强制输入方向；
- 输出比较不依赖“起点恰好一致”，而是做循环位移与反向等价比较；
- 浮点比较采用容差（`EPS`/`tol`），减少边界点误判。

## R13

数值稳定性说明：

- 叉积与交点使用 `float64`；
- 平行判断阈值 `EPS = 1e-9`；
- 多边形等价判定使用 `np.allclose(..., atol=1e-6)`；
- 随机性质测试检查所有坐标有限、且裁剪后面积不超过裁剪窗口面积（容差内）。

这些措施可覆盖绝大多数 MVP 级几何误差问题。

## R14

常见错误：

1. 把裁剪窗口当成可凹多边形直接用该算法，导致错误结果；
2. 忽略顶点方向，inside 判定号号方向写反；
3. 交点公式实现错误（分子分母叉积顺序颠倒）；
4. 不清理重复点，导致后续面积或比较逻辑异常；
5. 只做视觉检查，不做断言回归测试。

## R15

可扩展方向：

- 支持“保留边属性/来源边索引”的工程版裁剪；
- 对接扫描线填充与光栅化流程；
- 升级为多边形布尔运算框架（并/交/差）；
- 对复杂凹窗口改用 Weiler-Atherton 或 Vatti/Greiner-Hormann 系列算法。

## R16

相关算法对比：

- Cohen-Sutherland：主要用于线段裁剪；
- Liang-Barsky：参数方程线段裁剪，效率高；
- Weiler-Atherton：更通用的多边形裁剪，适合凹窗口与复杂拓扑；
- Sutherland-Hodgman：实现简洁、稳定，最适合“主体任意 + 凸窗口”场景。

## R18

`demo.py` 的源码级算法流程（8 步）：

1. `main` 先执行 `run_fixed_cases`，构建固定主体多边形与裁剪窗口，调用核心函数并断言期望结果。  
2. `sutherland_hodgman` 对输入做合法性检查：顶点数至少 3、裁剪窗口必须凸。  
3. `_clean_polygon` 清理输入中的连续重复点和尾首重复点，得到规范顶点序列。  
4. 通过 `polygon_signed_area` 判断裁剪窗口方向（顺/逆时针），据此在 `_inside` 中统一半平面判定符号。  
5. 对裁剪窗口每条边，遍历当前多边形边段 `(S -> P)`，按 inside/outside 四种组合决定追加 `P` 和/或 `_line_intersection` 交点。  
6. 每轮边裁剪结束后再做一次 `_clean_polygon`，抑制数值噪声造成的重复顶点。  
7. `run_random_property_checks` 生成随机主体多边形，验证输出点均在窗口内、坐标有限、且裁剪后面积不超过裁剪窗口面积。  
8. `main` 输出一组 showcase 的裁剪前后顶点与面积，最后打印全部检查通过信息。  

本实现没有调用第三方库的一步式黑盒“polygon clip”接口；`numpy` 仅用于数值计算与测试辅助，核心 Sutherland-Hodgman 流程在源码中逐步展开并可逐行追踪。
