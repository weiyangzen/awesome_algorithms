# Karatsuba乘法

## R01

Karatsuba 乘法是用于大整数乘法的经典分治算法，核心目标是在保证精确性的前提下，减少递归中“乘法”的次数。

- 输入: 两个十进制整数字符串（可含 `+/-`）。
- 输出: 乘积的十进制字符串。
- 核心思路:
  1. 把两个数按中点拆成高位和低位。
  2. 用 3 次子乘法替代朴素拆分后的 4 次子乘法。
  3. 通过十进制移位与加减重组结果。

## R02

- 提出者: Anatoly Karatsuba 与 Yuri Ofman。
- 时间: 1962 年。
- 意义: 首次将整数乘法从传统 `O(n^2)` 降到 `O(n^log2(3))`，是快速乘法体系（Toom-Cook、FFT/NTT）的重要起点。

## R03

对于位数较大的整数，朴素竖式乘法会执行大量位乘，整体复杂度为 `O(n^2)`。在密码学、任意精度计算、计算代数等场景中，`n` 往往很大，二次复杂度会成为瓶颈。

Karatsuba 通过代数恒等式减少子问题乘法数，得到更优渐进复杂度 `O(n^1.585)`，在中大规模位数下通常优于朴素法。

## R04

令 `n` 为输入位数，Karatsuba 的递推关系为:

- `T(n) = 3T(n/2) + O(n)`

由主定理可得:

- `T(n) = O(n^log2(3)) ≈ O(n^1.585)`

对比:

- 朴素乘法: `O(n^2)`
- Karatsuba: `O(n^1.585)`

## R05

空间开销主要来自递归与中间字符串:

- 递归栈深度: `O(log n)`
- 每层中间对象总量与输入长度同阶

在本目录的 Python MVP 实现中，可将额外空间近似视为 `O(n)` 到 `O(n log n)` 区间（取决于字符串分配与实现细节）。

## R06

示例: `1234 × 5678`

1. 设 `m=2`，拆分:
   - `1234 = 12*10^2 + 34`
   - `5678 = 56*10^2 + 78`
2. 计算三项:
   - `z2 = 12*56 = 672`
   - `z0 = 34*78 = 2652`
   - `z1 = (12+34)*(56+78) = 46*134 = 6164`
3. 中间项:
   - `mid = z1 - z2 - z0 = 6164 - 672 - 2652 = 2840`
4. 重组:
   - `result = z2*10^(2m) + mid*10^m + z0`
   - `= 672*10^4 + 2840*10^2 + 2652 = 7006652`

## R07

Karatsuba 的价值体现在三个层面:

- 理论层: 证明了可通过代数重排突破二次复杂度。
- 工程层: 常作为大整数库的中等规模乘法策略。
- 教学层: 是分治算法与复杂度分析的经典案例。

## R08

前置知识:

- 进位制整数表示与按位加减。
- 分治递归与递推式分析。
- 关键恒等式:
  - `(a*B^m+b)(c*B^m+d) = ac*B^(2m) + ((a+b)(c+d)-ac-bd)*B^m + bd`

## R09

适用条件与边界:

- 输入需是合法整数字符串。
- 小规模输入时，朴素法常数较小，可能更快。
- 工程实现通常设置阈值 `threshold`:
  - `n <= threshold` 走朴素乘法。
  - `n > threshold` 走 Karatsuba。

## R10

正确性可用数学归纳证明:

1. 基础情形: 小位数直接用朴素乘法，正确。
2. 归纳假设: 递归调用能正确求解更小规模子问题。
3. 使用恒等式分解:
   - `ac`、`bd`、`(a+b)(c+d)`
4. 推出交叉项:
   - `(a+b)(c+d)-ac-bd = ad+bc`
5. 重组得到:
   - `ac*B^(2m) + (ad+bc)*B^m + bd`
6. 与原式完全等价，故整体正确。

## R11

数值稳定性说明:

- 本算法是整数精确运算，不涉及浮点舍入误差。
- 风险主要来自实现错误:
  - 前导零处理不一致。
  - 借位/进位逻辑错误。
  - 符号解析和恢复错误。

## R12

工程成本（超越大 O）:

- 递归调用和字符串切片有额外常数开销。
- 中间字符串分配频繁，可能引入内存分配成本。
- 阈值选择影响较大:
  - 阈值过小: 递归开销偏大。
  - 阈值过大: 退化为较多朴素乘法。

## R13

- 近似比/概率保证: N/A。
- 原因: Karatsuba 是确定性精确算法，不是随机或近似算法。

## R14

鲁棒性与常见失效模式:

- 非法输入（空串、非数字字符）应立即报错。
- `sub_abs(a,b)` 的前提 `a>=b` 若被破坏，会导致错误结果。
- 超大输入可能受 Python 递归深度限制。
- 未规范化 `-0` 会造成符号不一致。

## R15

实现建议:

- 先统一输入为“符号 + 绝对值数字串”。
- 将加法、减法、比较、移位封装为独立函数。
- 采用“朴素 + Karatsuba”混合策略以平衡常数与渐进复杂度。
- 用 Python 内置 `int` 仅作为测试 oracle，不作为主算法实现。

## R16

前驱与后继关系:

- 前驱: 竖式乘法（`O(n^2)`）。
- 后继:
  - Toom-Cook 乘法
  - FFT/NTT 快速乘法
  - Schönhage-Strassen 及其后续改进

在实际大整数库中，通常按输入规模切换不同乘法内核。

## R17

本目录 `demo.py` 的 MVP 设计如下:

- 依赖: 仅 Python 标准库（`random`、`time`、`typing`）。
- 核心函数:
  - `schoolbook_mul_abs`: 朴素绝对值乘法。
  - `karatsuba_mul_abs`: Karatsuba 分治主体。
  - `multiply_karatsuba`: 处理符号后的统一入口。
- 运行内容:
  - 固定样例验证（含符号、前导零、超长数字）。
  - 随机回归测试（对照 Python `int`）。
  - 小规模性能对照（朴素 vs Karatsuba）。
- 运行命令:
  - `uv run python demo.py`

## R18

`demo.py` 中 Karatsuba 的源码级流程（8 步）:

1. `main()` 调用三组实验: 固定样例、随机回归、性能对照。
2. `multiply_karatsuba()` 先解析输入符号与绝对值，并处理零值特例。
3. `karatsuba_mul_abs()` 在位数不超过阈值时直接调用 `schoolbook_mul_abs()` 作为递归基。
4. 当位数超过阈值时，先将两个字符串左补零到等长偶数位，再各自拆成 `high` 与 `low` 两半。
5. 递归计算 `z2 = high_a*high_b` 与 `z0 = low_a*low_b`。
6. 递归计算 `z1 = (high_a+low_a)*(high_b+low_b)`，并用 `z1-z2-z0` 得到交叉项 `ad+bc`。
7. 使用十进制移位重组结果: `z2*10^(2m) + (ad+bc)*10^m + z0`，并通过字符串加法合并。
8. 返回到 `multiply_karatsuba()` 恢复符号，最后由测试逻辑与 Python `int` 结果逐例对照断言。
