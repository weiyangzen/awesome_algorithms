## R01

算法目标：在有限群（这里用素数域乘法群）中，给定 `g, h, p`，求解离散对数

`g^x ≡ h (mod p)`。

这就是经典 DLP（Discrete Logarithm Problem）。Pollard's Rho 是一种期望次指数前、在通用群模型下常用的随机化算法，空间占用很低。

## R02

输入与输出定义：

- 输入：
  - `p`：素数模数；
  - `g`：群生成元或至少是某个循环子群的生成元；
  - `h`：目标元素；
  - `order`：`g` 的阶（若不传，MVP 默认 `p-1`）。
- 输出：
  - 一个整数 `x`，满足 `g^x ≡ h (mod p)`。

## R03

核心思想：把“找对数”转成“找碰撞”。

构造伪随机游走状态 `(x_i, a_i, b_i)`，保持不变量：

`x_i = g^{a_i} * h^{b_i} (mod p)`。

当找到两个状态 `i != j` 且 `x_i = x_j`，可推导出关于 `x` 的线性同余方程，然后求解即可。

## R04

状态与分区函数：

Pollard's Rho 通常把当前 `x` 分到 3 个集合之一，并执行不同更新：

- `x mod 3 == 0`：`x <- x*h`, `b <- b+1`
- `x mod 3 == 1`：`x <- x*x`, `a <- 2a`, `b <- 2b`
- `x mod 3 == 2`：`x <- x*g`, `a <- a+1`

所有指数运算都按群阶 `n` 取模，所有群元素按 `p` 取模。

## R05

碰撞检测使用 Floyd 判圈（龟兔赛跑）：

- 慢指针每轮走一步；
- 快指针每轮走两步；
- 当两者到达同一个 `x`，得到一次碰撞。

这比保存完整哈希表更省内存，是 Pollard's Rho 的关键工程优势。

## R06

碰撞推导公式：

若 `x_t = x_h`，则

`g^{a_t} h^{b_t} = g^{a_h} h^{b_h}`

整理得

`g^{a_t-a_h} = h^{b_h-b_t} = g^{x*(b_h-b_t)}`。

令 `s = a_t-a_h (mod n)`，`r = b_h-b_t (mod n)`，得到线性同余：

`r * x ≡ s (mod n)`。

## R07

线性同余求解策略：

- 若 `d = gcd(r, n)` 且 `d` 不整除 `s`，无解（本轮失败，重启）；
- 否则有 `d` 个解：
  - 化简为 `(r/d) * x ≡ (s/d) (mod n/d)`；
  - 先求一个主解，再枚举 `d` 个同余类；
  - 逐个验证 `g^x mod p == h`，命中即返回。

因此本实现不仅覆盖 `gcd(r, n)=1`，也覆盖非互素情形。

## R08

算法流程（高层）：

1. 随机选初始 `(a0, b0)`；
2. 计算 `x0 = g^a0 * h^b0 mod p`；
3. 启动 Floyd 双指针游走；
4. 发生碰撞后构造 `r*x ≡ s (mod n)`；
5. 解线性同余并验证候选；
6. 成功则返回，失败则换随机种子重启。

## R09

复杂度（期望）：

- 时间：约 `O(sqrt(n))` 次群运算；
- 空间：`O(1)`（不计常数级状态和临时变量）。

对比 Baby-step Giant-step（BSGS）：

- BSGS 时间也约 `O(sqrt(n))`，但空间 `O(sqrt(n))`；
- Pollard's Rho 更省内存，适合内存受限场景。

## R10

正确性要点：

- 状态不变量 `x = g^a h^b` 在每次分区更新后都保持成立；
- 碰撞时得到的同余方程由群等式直接推导；
- 线性同余求解后再做“代回验证”，可排除伪解；
- 随机重启提升成功概率并规避退化轨迹。

## R11

失败模式与处理：

- `r = 0`：碰撞给不出有效方程，重启；
- `gcd(r,n)` 不整除 `s`：本轮无解，重启；
- 迭代步数超上限：重启；
- 多次重启仍失败：抛异常，建议提高 `restarts` 或检查参数合法性（例如 `h` 不在 `g` 生成子群中）。

## R12

`demo.py` 的实现结构：

- `State`：保存 `(x,a,b)`；
- `mod_inverse`：扩展欧几里得求逆；
- `solve_linear_congruence`：解 `a*x ≡ b (mod n)`；
- `rho_step`：三分区随机游走；
- `pollards_rho_discrete_log`：主流程（含重启）；
- `main`：固定参数演示并打印校验结果。

## R15

参数建议：

- `max_steps`：默认 `3*n`，一般足够；
- `restarts`：默认 32，若碰撞退化可增大；
- `seed`：固定种子保证 demo 可复现。

在大规模密码参数下，这个 MVP 仅用于教学与验证，不是高性能破解工具。

## R16

与常见离散对数算法对比：

- Pollard's Rho：
  - 优点：低内存、实现简洁；
  - 缺点：随机化，存在重启波动。
- BSGS：
  - 优点：确定性更强；
  - 缺点：需要大哈希表，内存高。
- Index Calculus：
  - 在特定群更快，但实现与预处理复杂得多，不适合本条目 MVP。

## R17

工程注意事项：

- 必须保证 `h` 属于 `g` 生成的子群；
- 若只给了 `p` 且使用 `order=p-1`，默认在 `Z_p^*` 全群求解；
- 对非素数模、椭圆曲线群等，需要替换群运算与阶信息来源；
- 生产级密码分析还会加入并行 rho、distinguished points 等优化，本 MVP 未覆盖。

## R18

源码级算法流（对应 `demo.py`，8 步）：

1. `main` 设定 `(p, g, secret_x)`，计算 `h = g^secret_x mod p`。
2. 调用 `pollards_rho_discrete_log(g, h, p, order)`，设置随机种子和重启上限。
3. 每次重启随机采样 `(a0, b0)`，构造初始状态 `x0 = g^a0 * h^b0 mod p`。
4. 通过 `rho_step` 让慢指针走 1 步、快指针走 2 步，持续保持 `x = g^a h^b` 不变量。
5. 一旦 `tortoise.x == hare.x`，从 `(a,b)` 差分构造 `r*x ≡ s (mod n)`。
6. 调用 `solve_linear_congruence(r, s, n)`；内部再通过 `mod_inverse` 求主解并枚举所有同余候选。
7. 对每个候选执行 `pow(g, candidate, p) == h` 验证，命中后立即返回该离散对数。
8. `main` 打印恢复结果与二次校验值，完成一次无交互、可复现的 MVP 演示。
