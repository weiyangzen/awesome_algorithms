# 大数乘法 - Toom-Cook算法

- UID: `MATH-0041`
- 学科: `数学`
- 分类: `算法`

- 目标目录: `Algorithms/数学-算法-0041-大数乘法_-_Toom-Cook算法`

## R01

Toom-Cook 是一类分治大数乘法算法。本条目实现的是最常用的 `Toom-3`（三路分割）。

- 输入: 两个任意精度整数 `x, y`。
- 输出: 乘积 `x * y`。
- 核心思想:
  1. 把大整数按基数 `B` 分成 3 段，视为二次多项式 `X(t), Y(t)` 在 `t=B^m` 处的取值。
  2. 在若干点（`0, 1, -1, 2, ∞`）上对多项式求值。
  3. 把“整数乘法”转成 5 次更小规模乘法（点值相乘）。
  4. 通过插值恢复乘积多项式系数，再在 `t=B^m` 回代得到最终整数。

## R02

历史脉络（简述）：

- 1963 年: Andrei Toom 提出把整数乘法转为多项式求值/插值的思路。
- 1966 年: Stephen Cook 进一步推广与系统化。
- 后续: 形成 `Toom-k` 系列，成为 Karatsuba 与 FFT/NTT 之间的重要中间层算法。

## R03

背景动机：

- 朴素学校乘法复杂度约为 `O(n^2)`，当位数增大后成本快速上升。
- Karatsuba 通过两路分治降到 `O(n^log_2 3)`，但在更大规模上仍可继续优化。
- Toom-Cook 通过 `k` 路分割进一步减少“递归乘法次数”，以更多加减和常数倍运算换取更低渐近复杂度。

## R04

复杂度（以 `Toom-3` 为例）：

- 递归关系可写作 `T(n) = 5T(n/3) + O(n)`。
- 由主定理得 `T(n) = O(n^log_3 5)`，指数约 `1.465`。
- 对比:
  - 学校乘法: `O(n^2)`
  - Karatsuba: `O(n^1.585)`
  - Toom-3: `O(n^1.465)`

说明: 这里的 `n` 可理解为“机器字/块数”，真实耗时还受常数项与底层大整数实现影响。

## R05

空间复杂度：

- 额外工作空间整体为 `O(n)`（中间点值、插值系数、重组缓存）。
- 递归深度约 `O(log_3 n)`。
- 在工程实现中，内存峰值通常由“同层临时对象分配策略”决定。

## R06

`Toom-3` 的结构化例子（符号级）：

设 `z = B^m`，把整数拆成：

- `x = x0 + x1*z + x2*z^2`
- `y = y0 + y1*z + y2*z^2`

构造多项式 `X(t), Y(t)`，并在点集 `{0, 1, -1, 2, ∞}` 计算：

- `v0 = X(0)Y(0)`
- `v1 = X(1)Y(1)`
- `v-1 = X(-1)Y(-1)`
- `v2 = X(2)Y(2)`
- `v∞ = X(∞)Y(∞) = x2*y2`

插值得到 `R(t) = r0 + r1*t + r2*t^2 + r3*t^3 + r4*t^4` 后，输出 `R(z)` 即乘积。

## R07

算法意义：

- 理论上: 展示了“求值-插值 + 分治”可突破 `n^2` 乘法瓶颈。
- 工程上: 是高性能大整数库中的经典分段策略之一。
- 体系上: 与 Karatsuba、FFT/NTT 共同构成多精度乘法的分层路线。

## R08

前置知识与依赖：

- 多项式表示与插值。
- 分治递归复杂度分析（主定理）。
- 任意精度整数表示（按基数分块）。
- 精确整数除法（Toom-3 插值阶段会用到对 `2`、`3` 的整除）。

## R09

适用前提与边界：

- 适合“足够大”的整数；对小整数往往不如学校乘法常数小。
- 对高度不平衡乘法（一个很短、一个很长）收益可能下降。
- 工程上通常设置阈值: 小于阈值时退化到学校乘法或 Karatsuba。
- 本目录 MVP 使用阈值切换，保证正确性与可运行性优先。

## R10

正确性依据（简述）：

1. 分块表示是整数在基数幂 `z` 上的唯一展开。
2. 点值计算得到的是同一多项式乘积 `R(t)=X(t)Y(t)` 在 5 个点上的值。
3. 四次多项式由 5 个互异点值唯一确定，因此插值可唯一恢复 `r0..r4`。
4. 代回 `t=z` 后得到的整数与原始 `x*y` 完全一致。

## R11

数值稳定性与误差传播：

- 纯整数算法，不涉及浮点舍入误差。
- 关键风险不是“数值不稳定”，而是实现错误导致插值除法不整除。
- 本实现在插值阶段使用“精确整除检查”（对 2 和 3），若不满足立即抛错，便于定位 bug。

## R12

真实成本（超越渐近记号）：

- Toom-3 增加了大量加减、移位式重组、点值管理，常数项明显高于 Karatsuba/学校乘法。
- 临界点依赖语言与库实现；在 Python 中，内建 `int` 乘法已高度优化，纯 Python Toom-3 常常更慢。
- 因此 MVP 主要目标是“可追踪算法机制与正确性”，而非击败 CPython 内建乘法性能。

## R13

近似比/误差界/概率保证：

- N/A。
- 原因: Toom-Cook 是确定性精确算法，不是随机算法或近似算法。

## R14

鲁棒性与失效模式：

- 插值公式写错: 会出现无法整除 2/3 或重组后结果错误。
- 基数/分块不一致: 点值与回代坐标错位，导致系统性错误。
- 阈值过低: 递归层数增加，反而拖慢性能。
- 负数符号处理缺失: 容易在 `X(-1)` 分支暴露错误。

## R15

工程实现要点：

- 采用“非负核心 + 符号包装”结构，减少分支复杂度。
- 设置学校乘法回退阈值，避免小规模递归过度开销。
- 在插值阶段统一封装 `exact_div`，显式检查整除性。
- 提供随机自检并与 Python 内建 `*` 对比，确保 MVP 可信。

## R16

前驱/后继与应用：

- 前驱:
  - 学校乘法（`O(n^2)`）
  - Karatsuba（`O(n^1.585)`）
- 后继:
  - 更高阶 Toom（Toom-4/Toom-6.5 等）
  - Schönhage-Strassen / NTT / FFT 类快速乘法
- 应用:
  - 大整数库（密码学、计算机代数、任意精度计算）
  - 高精度数学软件中的底层算术内核

## R17

Python MVP 方案（见 `demo.py`）：

- 依赖: 仅 Python 3 标准库（`random`、`time`、`typing`）。
- 主要函数:

  - `_toom3_nonnegative`: Toom-3 递归核心。
  - `schoolbook_multiply`: 小规模回退算法。
  - `_interpolate_toom3`: 从 `v0,v1,v-1,v2,v∞` 恢复 `r0..r4`。
- 演示内容:
  - 固定样例正确性验证。
  - 不同位数随机样例批量校验（与内建 `*` 对照）。
  - 单层 Toom-3 追踪输出（拆分、点评估、插值、重组）。

## R18

源码级算法流（对应本目录 `demo.py`，非第三方黑箱）可分 9 步：

1. `toom3_multiply` 读取输入并统一符号，转为非负乘法核心。
2. `_toom3_nonnegative` 判断阈值；若规模较小，转 `schoolbook_multiply`。
3. 对较大输入按 `z=B^m` 进行三段分块，得到 `a0,a1,a2` 与 `b0,b1,b2`。
4. 计算 `X(t),Y(t)` 在 `t=0,1,-1,2,∞` 的值。
5. 在 5 个点上递归调用乘法，得到 `v0,v1,v-1,v2,v∞`。
6. `_interpolate_toom3` 用精确整除恢复系数 `r0..r4`：
   - 先由对称和差得到 `r2`、`(r1+r3)`。
   - 再借助 `t=2` 点方程解出 `r3`，进而得到 `r1`。
7. 按 Horner 形式把 `r0..r4` 在 `z` 上回代，重组成整数结果。
8. 把符号恢复为最终乘积并返回。
9. `main` 中对固定样例与随机样例做断言校验，确保结果与 Python 内建乘法逐项一致。
