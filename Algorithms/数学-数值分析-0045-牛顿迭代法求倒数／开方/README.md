# 牛顿迭代法求倒数/开方

- UID: `MATH-0045`
- 学科: `数学`
- 分类: `数值分析`
- 源序号: `45`
- 目标目录: `Algorithms/数学-数值分析-0045-牛顿迭代法求倒数／开方`

## R01

本条目实现牛顿迭代（Newton's Method）在两个经典标量任务上的最小可运行版本：
- 求倒数 `1/a`（`a != 0`）；
- 求平方根 `sqrt(s)`（`s >= 0`）。

MVP 目标不是替代底层数学库，而是把“从方程建模到迭代收敛”的全过程写清楚、跑通并可观察误差收敛。

## R02

问题定义（有限精度计算版）：
- 输入：
  - 倒数任务输入实数 `a`，要求 `a != 0`；
  - 开方任务输入实数 `s`，要求 `s >= 0`；
  - 迭代控制参数 `max_iter`、`tol`。
- 输出：
  - 倒数近似值 `x ~= 1/a`；
  - 开方近似值 `x ~= sqrt(s)`；
  - 每次迭代的轨迹（当前估计与残差）。

## R03

数学建模与迭代公式：

1) 倒数
- 设 `f(x) = 1/x - a`，根为 `x* = 1/a`。
- 牛顿法 `x_{k+1} = x_k - f(x_k)/f'(x_k)`，其中 `f'(x) = -1/x^2`。
- 化简得
  `x_{k+1} = x_k * (2 - a * x_k)`。

2) 开方
- 设 `g(x) = x^2 - s`，根为 `x* = sqrt(s)`（取非负根）。
- 牛顿法给出
  `x_{k+1} = 0.5 * (x_k + s/x_k)`。

在初值合理时，两者都具有局部二次收敛：误差会近似“平方级”下降。

## R04

算法总览：
1. 对输入做定义域检查（`a != 0`、`s >= 0`、有限数）。
2. 使用 `frexp/ldexp` 构造便宜且稳定的初值，避免把库函数结果直接当黑盒答案。
3. 对倒数使用 `x <- x * (2 - a*x)` 迭代。
4. 对开方使用 `x <- 0.5 * (x + s/x)` 迭代。
5. 每轮记录迭代号、当前估计值、残差。
6. 用步长准则与残差准则共同判断停止。
7. 打印与真实值（`1/a` 或 `math.sqrt(s)`）的误差对照。

## R05

核心数据结构：
- `list[tuple[int, float, float]] history`：
  - 第 1 列是迭代编号；
  - 第 2 列是当前估计；
  - 第 3 列是残差。
- 标量参数：`max_iter`, `tol`。

MVP 只依赖 Python 标准库，结构直观，便于后续替换成向量化版本。

## R06

正确性要点：
- 倒数迭代是牛顿法对 `f(x)=1/x-a` 的代数化简结果；
- 开方迭代是牛顿法对 `g(x)=x^2-s` 的经典形式；
- 当输入满足定义域且初值在收敛吸引域内，迭代会收敛到目标根；
- `demo.py` 中用真实值对比可直接观察误差下降。

## R07

复杂度分析：
- 设倒数迭代次数为 `T_r`，开方迭代次数为 `T_s`。
- 时间复杂度：`O(T_r + T_s)`（单个样本）。
- 空间复杂度：
  - 若保留完整轨迹：`O(T)`；
  - 若仅保留当前值：`O(1)`。

由于二次收敛，`T` 通常很小（十几轮以内可达双精度高精度）。

## R08

边界与异常处理：
- 倒数：`a == 0` 抛出 `ValueError`；
- 开方：`s < 0` 抛出 `ValueError`；
- 非有限输入（`inf`/`nan`）抛出 `ValueError`；
- 若出现非有限迭代值（数值发散），抛出 `RuntimeError`。

## R09

MVP 取舍：
- 选择标量实现而非大规模批处理，优先保证“公式透明 + 轨迹可读”；
- 不直接调用第三方优化器或自动微分框架，避免把核心步骤藏在库内部；
- 初值采用轻量近似，兼顾稳定性与教学可解释性。

## R10

`demo.py` 函数职责：
- `check_finite_number`：输入有限性检查；
- `reciprocal_initial_guess`：倒数任务初值；
- `sqrt_initial_guess`：开方任务初值；
- `newton_reciprocal`：执行倒数牛顿迭代并返回轨迹；
- `newton_sqrt`：执行开方牛顿迭代并返回轨迹；
- `relative_error`：统一相对误差计算；
- `print_trace`：打印迭代明细；
- `run_reciprocal_demo` / `run_sqrt_demo`：分别演示两类任务；
- `main`：脚本入口，无交互运行。

## R11

运行方式：

```bash
cd Algorithms/数学-数值分析-0045-牛顿迭代法求倒数／开方
python3 demo.py
```

脚本不需要任何输入，会自动输出多组样例的迭代结果与误差统计。

## R12

输出解读：
- `estimate`：牛顿法最终近似值；
- `reference`：基准值（倒数任务用 `1/a`，开方任务用 `math.sqrt(s)`）；
- `relative_error`：相对误差；
- `iter`：迭代步；
- `residual`：
  - 倒数任务为 `|1 - a*x|`；
  - 开方任务为 `|x^2 - s|`。

如果实现正确，通常会看到残差快速下降到接近机器精度。

## R13

建议最小测试集：
- 倒数：`a in {0.2, 0.75, 3.0, -12.5, 1024.0}`；
- 开方：`s in {0.0, 2.0, 10.0, 1e-6, 1e6}`；
- 异常：`a=0`、`s<0`、`nan`、`inf`。

这些用例覆盖正数、负数、大小量级差异与边界输入。

## R14

可调参数：
- `max_iter`：最大迭代次数，默认 30；
- `tol`：停止阈值，默认 `1e-15`；
- `max_lines`：打印轨迹的最大行数，控制输出长度。

工程建议：先用严格 `tol` 验证正确性，再按性能需求适度放宽。

## R15

方法对比：
- 与“直接 `1/a` 或 `math.sqrt`”相比：
  - 直接调用更短，但看不到迭代行为；
  - 牛顿法可解释，可用于硬件实现与自定义精度控制。
- 与二分法开方相比：
  - 二分法线性收敛、稳定但慢；
  - 牛顿法通常二次收敛、步数更少。

应用场景：

## R18

源码级算法流（对应 `demo.py`，8 步）：
1. `main` 调用 `run_reciprocal_demo` 与 `run_sqrt_demo`，准备样例与统一迭代参数。  
2. 每个样例先进入 `check_finite_number` 做有限性校验，并在任务函数内做定义域检查。  
3. 倒数任务通过 `reciprocal_initial_guess`（`frexp/ldexp + 线性近似`）生成 `x0`。  
4. 开方任务通过 `sqrt_initial_guess` 生成正初值 `x0`，保证后续 `s/x` 稳定。  
5. `newton_reciprocal` 循环执行 `x <- x*(2-a*x)`，每轮记录 `|1-a*x|`。  
6. `newton_sqrt` 循环执行 `x <- 0.5*(x+s/x)`，每轮记录 `|x*x-s|`。  
7. 两个迭代器都用“步长阈值 + 残差阈值”判停，必要时检测非有限值并报错。  
8. 演示层计算 `reference` 与 `relative_error`，再用 `print_trace` 输出前若干步轨迹。
