## R01

Kruskal 算法用于在带权无向图中求解最小生成树（Minimum Spanning Tree, MST）。核心思想是：

1. 按边权从小到大排序。
2. 依次尝试加入当前最小边。
3. 若加入该边会形成环，则丢弃；否则保留。
4. 当已选边数达到 `|V|-1` 时结束，得到一棵最小生成树。

它是典型的“贪心 + 并查集（Disjoint Set Union, DSU）”组合算法，适合边集表示清晰、稀疏到中等规模的图。

## R02

历史脉络：

- 1926 年：Otakar Boruvka 提出最早的 MST 算法之一（Boruvka）。
- 1956 年：Joseph Kruskal 在论文中正式给出 Kruskal 算法。
- 1957 年：Prim 算法被广泛传播（Jarník-Prim-Dijkstra 路线）。

Kruskal 与 Prim 共同构成 MST 的两大基础范式。

## R03

问题背景与动机：

- 现实网络设计（道路、光纤、电网）常需要“全连通且总成本最低”的骨干结构。
- 若把节点看作站点、边权看作造价，MST 即为最低总造价连通方案（不要求唯一）。
- Kruskal 的优势在于实现简洁、正确性清晰、并查集优化后性能稳定，工程落地成本低。

## R04

时间复杂度：

- 排序阶段：`O(E log E)`（主耗时）。
- 并查集操作：每条边最多两次 `find` + 一次 `union`，均摊近似 `O(alpha(V))`。
- 总体：`O(E log E)`，其中 `alpha` 是反 Ackermann 函数，增长极慢，可近似视为常数。

## R05

空间复杂度：

- 边数组（排序前后）：`O(E)`。
- 并查集父节点、秩/大小数组：`O(V)`。
- 总体额外空间：`O(E + V)`（通常由边集存储主导）。

## R06

手算示例（6 个点）：

边集（`u-v:w`）：
`0-1:4, 0-2:4, 1-2:2, 1-3:5, 2-3:5, 2-4:11, 3-4:2, 3-5:6, 4-5:1`

按权排序后依次选边：

1. 选 `4-5:1`（无环）
2. 选 `1-2:2`（无环）
3. 选 `3-4:2`（无环）
4. 选 `0-1:4`（无环）
5. `0-2:4` 会成环，跳过
6. 选 `1-3:5`（连通两大连通块）

此时已选 `5 = |V|-1` 条边，结束。MST 总权重为 `1+2+2+4+5=14`。

## R07

算法意义：

- 在图论中，MST 是连接性与最优性结合的基础问题。
- Kruskal 体现了贪心策略的可证最优性（切分性质 / 交换论证）。
- 在工程上常作为网络骨架构建、聚类预处理、图压缩与近似算法子过程。

## R08

关键前置知识：

- 无向图、连通图、森林、生成树基本定义。
- 环检测思想（并查集是高效实现方式）。
- 贪心策略与“局部最优导向全局最优”的证明框架。
- 排序算法复杂度与数据结构开销认知。

## R09

适用前提与边界：

- 图应为无向图；有向图需转化或改用其他模型。
- 若图不连通，Kruskal 返回的是最小生成森林（MSF），而非单棵生成树。
- 边权可为负值、零或正值，算法仍成立。
- 平行边允许存在；自环应忽略（自环必成环，对 MST 无贡献）。

## R10

正确性依据（简述）：

- 切分性质：对任意割，跨越该割的最小边一定存在于某棵 MST 中。
- Kruskal 每一步都选择当前能安全加入（不成环）的最小边，等价于反复应用安全边选择。
- 通过交换论证可证明：若最优解未包含当前安全边，可替换其上对应更重边而不劣，最终得到包含该边的最优解。
- 因此迭代完成后得到全局最优的 MST（或 MSF）。

## R11

数值稳定性与精度问题：

- 算法本质是比较与加法，不涉及迭代数值逼近，无“误差累积发散”问题。
- 若使用浮点权重，可能出现近似相等比较的不确定顺序；建议工程上统一精度或改用整数权重（如放大后的毫单位）。
- 边权并列时会出现多棵合法 MST，总权重相同但边集合可能不同，这是数学上允许的非唯一性，不是错误。

## R12

真实计算成本（超越渐近符号）：

- 大规模图上，排序常常占据绝大部分 CPU 时间。
- 并查集采用路径压缩 + 按秩合并后，单次开销极低，但仍受内存访问局部性影响。
- Python 实现中，对象封装（如 dataclass Edge）带来可读性收益，但会有额外分配与属性访问成本。
- 若追求极限性能，常改为连续数组存储边并使用编译型语言实现。

## R13

结果性质与可复现性：

- Kruskal 是确定性算法：给定固定边排序规则，输出可复现。
- 当存在同权边时，不同稳定排序或 tie-break 规则可能得到不同 MST，但总权重一致。
- 因此评估时应优先验证“总权重最小 + 无环 + 覆盖全部节点（连通图）”，而非强制比对具体边集顺序。

## R14

鲁棒性与常见失效模式：

- 忘记并查集环检测，导致输出含环，不再是树。
- 误把有向边当无向边处理，语义错误。
- 节点编号越界或不连续映射处理不当，触发运行时异常或错误合并。
- 输入图不连通却仍假设“必有 MST”，会在业务层造成错误解释；应显式返回 `connected=False`。

## R15

工程实现要点：

- 边排序键建议统一为 `(weight, u, v)`，便于结果稳定复现。
- 并查集必须启用路径压缩与按秩/按大小合并。
- 选边计数达到 `V-1` 后可提前终止，避免无效扫描。
- 对数据导入层建议提前过滤自环与非法端点，减少核心循环分支压力。

## R16

相关算法与典型应用：

- 同类算法：Prim、Boruvka、Reverse-Delete。
- 扩展问题：最大生成树（改为降序）、受限 MST、动态 MST。
- 典型应用：
  - 网络布线与基础设施建设
  - 图聚类中的单链接层次聚类（与 MST 关系紧密）
  - 近似 TSP 的树结构预处理
  - 图像分割中的图结构简化

## R17

Python MVP 设计（见 `demo.py`）：

- 依赖：仅 Python 标准库（`dataclasses`、`typing`、`itertools`、`random`），无交互输入。
- 核心模块：
  - `UnionFind`：路径压缩 + 按秩合并。
  - `kruskal_mst`：Kruskal 主流程，返回总权重、选边列表、是否连通。
  - `bruteforce_mst_weight`：用于小图穷举校验，验证 MVP 正确性。
- 演示内容：
  - 固定样例图的 MST 计算与期望值验证。
  - 小随机图上与穷举最优值做交叉验证。
  - 非连通图返回最小生成森林并标记 `connected=False`。

运行：

```bash
python3 demo.py
```

## R18

`demo.py` 的源码级流程可拆成 8 步：

1. **输入检查**：`kruskal_mst` 验证 `num_nodes` 非负、边端点在 `[0, num_nodes-1]` 内，并忽略自环。
2. **并查集初始化**：创建 `UnionFind(num_nodes)`，每个节点先独立成集合。
3. **排序**：对边按 `(weight, u, v)` 升序排序，确保贪心顺序与结果可复现。
4. **贪心扫描**：依次取当前最小边，调用 `union(u, v)` 判断是否跨集合。
5. **安全选边**：若 `union` 成功则加入 MST 边集并累加权重；失败则说明成环，跳过。
6. **提前终止**：当已选边数达到 `num_nodes - 1` 时提前结束扫描。
7. **连通性判定**：若最终边数为 `num_nodes - 1`，则 `connected=True`；否则返回最小生成森林结果。
8. **结果验证与展示**：`main` 中先跑固定图，再用 `bruteforce_mst_weight` 在小图上对拍，最后展示非连通图边界行为。
