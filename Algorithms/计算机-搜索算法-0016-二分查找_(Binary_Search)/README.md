# 二分查找 (Binary Search)

- UID: `CS-0016`
- 学科: `计算机`
- 分类: `搜索算法`
- 源序号: `16`
- 目标目录: `Algorithms/计算机-搜索算法-0016-二分查找_(Binary_Search)`

## R01

二分查找用于在**有序序列**中查找目标值。它每一步都把搜索区间缩小为原来的一半，因此比线性扫描更高效。

## R02

问题形式化：
- 输入: 升序数组 `arr`，长度 `n`；目标值 `target`
- 输出: 若存在目标值，返回某个合法下标 `i` 使 `arr[i] == target`；否则返回 `-1`

## R03

适用前提：
- 序列必须按单调顺序排列（常见为升序）
- 元素支持可比较操作（`<`, `>`, `==`）
- 若存在重复值，需要预先约定返回哪一个位置（任意、最左、最右）

## R04

核心思想：
- 维护闭区间 `[left, right]`
- 每轮取中点 `mid = left + (right - left) // 2`
- 比较 `arr[mid]` 与 `target`：
  - 相等: 命中
  - `arr[mid] < target`: 目标只可能在右半边
  - `arr[mid] > target`: 目标只可能在左半边
- 区间为空时结束（未找到）

## R05

简化伪代码：

```text
left = 0, right = n - 1
while left <= right:
    mid = left + (right - left) // 2
    if arr[mid] == target:
        return mid
    if arr[mid] < target:
        left = mid + 1
    else:
        right = mid - 1
return -1
```

## R06

循环不变式（正确性要点）：
- 任意时刻，若目标存在，则必在当前区间 `[left, right]` 内
- 每次迭代后区间严格缩小，且不丢失可能位置
- 当区间变为空（`left > right`）时，说明不存在目标

## R07

复杂度：
- 时间复杂度: `O(log n)`
- 空间复杂度: `O(1)`（迭代版本）

## R08

边界与特殊情况：
- 空数组 `[]`，直接返回 `-1`
- 单元素数组
- 目标小于最小值或大于最大值
- 包含重复值时，基础版本可能返回任意一个命中位置

## R09

示例：
- `arr = [1, 3, 5, 7, 9]`, `target = 7`，返回 `3`
- `arr = [1, 3, 5, 7, 9]`, `target = 2`，返回 `-1`

## R12

典型应用：
- 字典/索引中的有序键定位
- 有序日志时间戳定位
- 参数阈值搜索（与“答案二分”结合）
- 作为更复杂算法（如分治、范围查询）的基础子过程

## R13

常见错误：
- 忘记先排序或错误假设“输入已排序”
- `while left < right` 与 `while left <= right` 混用导致漏检
- 中点更新和边界更新不匹配造成死循环
- 在重复值场景下未定义“最左/最右”语义

## R14

常见变体：
- 查找最左等于目标的位置（lower bound）
- 查找最右等于目标的位置（upper bound - 1）
- 在“可行性单调”条件下查找最小可行解（答案二分）

## R15

与线性查找对比：
- 线性查找: `O(n)`，无需有序
- 二分查找: `O(log n)`，要求有序
- 当数据静态且查询频繁时，先排序再二分通常更划算

## R16

工程实践建议：
- 大整数索引场景使用 `left + (right - left) // 2` 计算中点
- 对外暴露 API 时明确“输入必须有序”和“重复值返回策略”
- 用统一测试覆盖边界条件，防止回归

## R17

最小测试清单：
- 空数组、单元素数组
- 命中首元素、末元素、中间元素
- 未命中（小于最小值/介于中间/大于最大值）
- 重复值数组（验证返回值合法性）
- 非升序输入（应触发异常或明确处理）

## R18

源码级算法流程拆解（以本目录 `demo.py` 的 `binary_search` 为准）：
1. 接收 `arr` 与 `target`，并先用一次线性扫描校验 `arr` 是否非降序；若违例则抛出 `ValueError`。
2. 初始化双指针 `left = 0` 与 `right = len(arr) - 1`，定义当前闭区间。
3. 进入循环条件 `left <= right`，保证只要区间非空就继续搜索。
4. 计算中点 `mid = left + (right - left) // 2`，读取 `mid_value = arr[mid]`。
5. 若 `mid_value == target`，立即返回 `mid`（搜索成功）。
6. 若 `mid_value < target`，把左边界移动到 `mid + 1`，丢弃左半区间。
7. 否则把右边界移动到 `mid - 1`，丢弃右半区间。
8. 循环结束后区间为空，返回 `-1` 表示未找到。

说明：`demo.py` 同时用标准库 `bisect_left` 做交叉检查；`bisect_left` 的核心也是二分收缩区间，只是封装成库函数接口。
