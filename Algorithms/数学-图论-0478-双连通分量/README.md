# 双连通分量

- UID: `MATH-0478`
- 学科: `数学`
- 分类: `图论`
- 源序号: `478`
- 目标目录: `Algorithms/数学-图论-0478-双连通分量`

## R01

双连通分量（Biconnected Components，常简称 BCC）在本条目中指无向图的“点双连通分量”（也称 block）：

- 每个分量是一个极大连通子图；
- 该子图内部不存在割点（或等价地，任意两点间至少存在两条点不相交路径，针对平凡分量按标准 block 约定处理）。

`demo.py` 采用 Tarjan 的 DFS 时间戳 + low-link + 边栈方法，在 `O(V+E)` 时间内一次性求出全部点双分量与割点。

## R02

问题定义（面向本 MVP）：

- 输入：无向图 `G=(V,E)`，顶点编号 `0..n-1`，边列表 `[(u,v), ...]`。
- 输出：
1. `bcc_vertices`：每个点双分量对应的顶点集合；
2. `bcc_edges`：每个点双分量包含的边 `edge_id` 集合（用于验证边划分）；
3. `articulation_points`：所有割点。

实现支持：
- 非连通图；
- 重边（parallel edges）；
- 孤立点（作为平凡 block 输出）。

## R03

Tarjan 点双分解维护两个关键数组：

- `disc[u]`：顶点 `u` 在 DFS 中首次访问时间戳；
- `low[u]`：从 `u` 出发，经“若干树边 + 至多一条返祖边”可到达的最小 `disc`。

对 DFS 树边 `u -> v`：
- 若 `low[v] >= disc[u]`，则以该树边为界可以从边栈中弹出一个完整点双分量；
- 若 `u` 非根且存在子节点满足该条件，`u` 是割点；
- 若 `u` 是根且 DFS 子树数大于 1，`u` 是割点。

## R04

为什么需要“边栈”而不是“点栈”：

- 点双分量允许在不同分量间共享割点；
- 若只栈顶按点弹出，容易在割点处分组不准确；
- 按“边”入栈并在 `low[v] >= disc[u]` 时弹到当前树边，能精确得到一个 block 的边集，再转成顶点集。

这也是无向图点双分解的经典工程写法。

## R05

`demo.py` 的核心数据结构：

- `Graph`：
  - `edges`: 原始边列表；
  - `adjacency[u] = [(v, edge_id), ...]`；
  - 每条边有唯一 `edge_id`，便于重边和父边判定。
- `BiconnectedResult`：
  - `bcc_vertices`, `bcc_edges`, `articulation_points`；
  - `disc`, `low` 便于调试和教学展示。

## R06

主算法流程（Tarjan 点双）：

1. 构图并校验输入。
2. 初始化 `disc/low/is_articulation/edge_stack`。
3. 对每个未访问顶点启动 DFS（覆盖非连通图）。
4. 遇到树边时先入边栈，递归后回溯更新 `low`。
5. 遇到返祖边（祖先方向）时入栈并收紧 `low`。
6. 当树边满足 `low[child] >= disc[parent]`，从边栈弹出一个点双分量。
7. DFS 根节点子树数大于 1 时标记为割点，非根按 `low` 条件标记割点。
8. 若 DFS 根是孤立点，单独输出 `[root]` 平凡分量。

## R07

正确性直觉：

- `low[v]` 反映子树 `v` 是否还能绕回 `u` 以上祖先；
- 若不能绕回（`low[v] >= disc[u]`），则 `v` 子树与上方图的连接必须经过 `u`，对应一个完整 block 边界；
- 边栈保存了“尚未归属任何 block 的边”，按该边界弹栈即可得到极大无割点子图；
- 每条边仅在某一次弹栈中归属，保证分解互斥且完整。

## R08

复杂度分析：

- 时间复杂度：`O(V+E)`  
  每个顶点访问一次，每条无向边常数次处理（邻接表双向出现）。
- 空间复杂度：`O(V+E)`  
  邻接表 `O(V+E)`，辅助数组与边栈 `O(V+E)`（最坏情况下边栈可接近 `E`）。

## R09

边界与鲁棒性处理（已在实现中覆盖）：

- `n < 0`、端点越界、端点非整数：抛 `ValueError`；
- `n=0` 时不允许有边；
- 自环边在该 MVP 中直接拒绝（抛错）；
- 支持重边：通过 `edge_id == parent_edge_id` 跳过“进入当前节点的那条边”，不会误伤平行边；
- 支持孤立点：输出单点 block。

## R10

与相邻图论算法关系：

- 桥边检测（MATH-0479）与割点检测（MATH-0480）都依赖 `disc/low`；
- 点双分解可视为在割点检测基础上进一步输出“结构分块”；
- 与强连通分量（MATH-0476）不同：SCC 针对有向图，点双针对无向图。

## R18

`demo.py` 源码级流程（9 步）：

1. `main` 组织 5 个固定图用例并逐个调用 `run_case`。  
2. `run_case` 先用 `build_undirected_graph` 构建带 `edge_id` 的邻接表。  
3. `biconnected_components_tarjan` 初始化 `disc/low/is_articulation/timer/edge_stack`。  
4. 对每个未访问顶点发起 `dfs(root)`；若根是孤立点，直接登记为单点 block。  
5. DFS 遍历树边 `u->v` 时把该边入栈，递归返回后更新 `low[u] = min(low[u], low[v])`。  
6. DFS 遍历返祖边时（`disc[v] < disc[u]`）将该边入栈，并用 `disc[v]` 收紧 `low[u]`。  
7. 当某树边满足 `low[v] >= disc[u]`，调用 `pop_component_until` 从边栈弹到该边，形成一个点双分量。  
8. DFS 同步按根/非根规则标记割点；遍历结束后排序得到稳定输出顺序。  
9. `run_case` 再用 `articulation_points_bruteforce` 对拍割点、校验边划分完整性并与期望断言，最后打印 `disc/low` 与分量结果。
