## R01

向量空间模型（Vector Space Model, VSM）把文档和查询都表示为同一个高维向量空间中的向量。

核心思想是：
- 词项（term）作为维度；
- 文档在每个维度上的值表示该词项的重要性（常用 TF-IDF）；
- 查询也编码成同维向量；
- 通过相似度（通常余弦相似度）完成检索排序。

## R02

它解决的信息检索基本问题是“如何把自然语言匹配转成可计算排序”：
- 对一组候选文档给出与查询的相关性分数；
- 支持 Top-K 检索；
- 支持离线建索引 + 在线打分。

与布尔检索相比，VSM 不再是“命中/不命中”二值逻辑，而是可排序的连续相关性评分。

## R03

本条目采用标准 TF-IDF 版本的 VSM：
- `TF(t,d) = count(t,d) / sum_j count(j,d)`（归一化词频）；
- `IDF(t) = log((N + 1) / (DF(t) + 1)) + 1`（平滑逆文档频率）；
- `w(t,d) = TF(t,d) * IDF(t)`；
- 将文档和查询向量都做 L2 归一化；
- 相似度 `sim(q,d) = q · d`（归一化后等价于余弦相似度）。

## R04

算法流程（训练/检索分离）：
1. 分词并建立词表；
2. 构建文档-词项计数矩阵 `X`（稀疏）；
3. 计算每个词项的 `DF` 与 `IDF`；
4. 将 `X` 映射到 TF，再乘 `IDF` 得到文档 TF-IDF；
5. 对每个文档向量做 L2 归一化并缓存；
6. 查询时用同一词表编码查询；
7. 对查询做 TF-IDF + L2 归一化；
8. 计算 `score = D @ q` 并按分数降序返回 Top-K。

本 MVP 在源码里显式实现了步骤 3-8，没有把 `TfidfVectorizer` 当黑盒直接调用。

## R05

复杂度（设文档数 `N`，词表大小 `V`，非零项数 `nnz`）：
- 建索引：
  - 计数矩阵构建约 `O(nnz)`；
  - `DF/IDF` 计算约 `O(nnz + V)`；
  - 归一化约 `O(nnz)`。
- 单查询：
  - 查询编码与加权约 `O(nnz_q + V_q)`；
  - 稀疏点积打分约 `O(nnz_D_overlap)`，上界可写成 `O(nnz + N)`。

空间复杂度主要由稀疏矩阵占据，约 `O(nnz + V + N)`。

`demo.py` 的实验设计：

## R07

优点：
- 数学形式简单、可解释；
- 易于做离线索引与在线检索拆分；
- 适合作为倒排检索和语义检索之间的“经典基线”。

局限：
- 忽略词序与长程语义；
- 稀疏高维表示对同义改写不鲁棒；
- 在复杂语义任务上通常弱于现代稠密向量检索。

## R08

前置知识与运行环境：
- 向量与矩阵运算基础；
- 稀疏矩阵（CSR）基础；
- Python 3.10+；
- 依赖：`numpy`、`scipy`、`pandas`、`scikit-learn`、`torch`。

这些依赖在本 MVP 中分别用于数值计算、稀疏表示、结果表格化、词表构建与结果一致性校验。

## R09

适用场景：
- 小中型文本库关键词检索；
- 搜索功能原型搭建与可解释 baseline；
- 教学中演示 TF-IDF 与余弦相似度机制。

不适用场景：
- 强语义匹配（问答、跨句推理）；
- 对同义词/上下文依赖很强的检索任务；
- 多模态语义对齐场景。

## R10

正确性要点：
- 查询与文档使用同一词表与同一 IDF 参数；
- 归一化后内积与余弦相似度等价；
- 稀疏向量中未出现词项权重保持 0，不会引入伪相似度；
- 若查询词未出现在词表中，其向量为零向量，得分应全为 0。

`demo.py` 在代码里对这些条件做了显式处理。

## R11

鲁棒性与边界处理：
- 拒绝空文档集合；
- 拒绝重复 `doc_id`；
- 拒绝全空白文本；
- 查询为空白时抛出异常；
- `top_k` 必须为正整数；
- 查询全 OOV（词表外）时返回全 0 分数，不崩溃；
- 对 SciPy 与 PyTorch 两条打分路径做 `allclose` 一致性断言。

## R12

实现细节选择：
- 用 `CountVectorizer` 仅做“词表 + 计数矩阵”，TF-IDF 自行展开；
- 文档矩阵和查询向量都保留为 `scipy.sparse.csr_matrix`；
- 打分主路径使用稀疏乘法 `D @ q^T`；
- 用 PyTorch dense 路径做交叉验证，确保数值一致；
- 用 `pandas.DataFrame` 输出 Top-K 结果，提升可读性。

这与 R04 的流程定义严格一一对应。

## R14

常见错误：
1. 对查询和文档使用了不同的词表；
2. 忘记做 L2 归一化，导致长文档天然高分；
3. 把 IDF 用在查询但没用在文档（或反之）；
4. 直接黑盒调用 `TfidfVectorizer` 却无法解释内部步骤；
5. OOV 查询未处理，出现除零或空矩阵异常。

本条目实现显式规避了这些问题。

## R16

相关算法与机制（和 R08 的知识栈一致）：
- `TF-IDF`：VSM 最常见权重函数；
- `BM25`：在信息检索中常优于纯 TF-IDF；
- `LSI/LSA`：对 VSM 做低秩语义压缩；
- 稠密检索（DPR/双塔）：以神经向量替代稀疏词项向量；
- 倒排索引：工程上与 VSM 常配合使用。

## R18

`demo.py` 源码级算法流程（9 步）：
1. `build_demo_corpus` 生成 8 条已分词文档，每条含 `doc_id/title/text`。  
2. `fit` 使用 `CountVectorizer` 生成词表与稀疏计数矩阵 `X`。  
3. `_compute_idf` 从 `X` 的非零分布计算每个词项的 `DF` 与平滑 `IDF`。  
4. `_tfidf_from_counts` 对每一行先做 TF 归一化，再逐列乘 `IDF`，最后行级 L2 归一化，得到文档矩阵 `D`。  
5. 查询到来时，`encode_query` 用同一词表把查询映射到计数向量 `x_q`，并复用步骤 4 生成 `q`。  
6. `search` 通过稀疏矩阵乘法 `scores = D @ q^T` 计算所有文档相似度。  
7. `_score_with_torch` 将 `D` 与 `q` 转成 `torch.Tensor` 做一次独立打分，并与 SciPy 结果做 `allclose` 断言。  
8. `search` 按分数降序取 Top-K，封装为 `pandas.DataFrame` 输出检索结果。  
9. `run_demo` 在 4 条固定查询上计算并打印 `Precision@3`，形成可复现实验结论。  

该流程把第三方库调用拆解为可追踪的权重计算与打分步骤，不依赖不可解释的“一步黑盒检索”。
