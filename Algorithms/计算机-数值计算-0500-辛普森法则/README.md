## R01

辛普森法则（Simpson's Rule）是经典数值积分方法，用分段二次插值抛物线来逼近被积函数，再对逼近函数做解析积分。

本条目目标是实现一个最小可运行 MVP：
- 手写复合辛普森法主算法，不把积分主体交给黑盒库；
- 给出误差对照（真值误差 + Richardson 误差估计）；
- 在多个确定性函数上自动验证精度与稳定性；
- 保证 `uv run python demo.py` 可直接运行。

## R02

问题定义：给定可积函数 `f(x)` 与区间 `[a,b]`，求定积分

`I = ∫[a,b] f(x) dx`

输入：
- 函数 `f`；
- 积分区间端点 `a,b`（要求 `a < b`）；
- 子区间数 `n`（复合辛普森要求 `n` 为正偶数）。

输出：
- 辛普森近似值 `I_n`；
- 加密网格近似 `I_2n`；
- 绝对误差 `|I_2n - I_true|`（测试场景可得真值）；
- Richardson 误差估计 `|I_2n - I_n| / 15`。

## R03

核心数学公式：

1. 单区间辛普森法（把 `[x0,x2]` 分成两段，步长 `h=(x2-x0)/2`）：

`∫[x0,x2] f(x) dx ≈ (h/3) * (f(x0) + 4f(x1) + f(x2))`

2. 复合辛普森法（`n` 为偶数，`h=(b-a)/n`）：

`I_n = (h/3) * [f(x0) + f(xn) + 4 * Σ f(x_(2k-1)) + 2 * Σ f(x_(2k))]`

其中奇数节点求和不含端点，偶数节点求和不含端点。

3. 阶数性质：
- 全局截断误差阶为 `O(h^4)`；
- 对不超过三次的多项式，辛普森法可达到机器精度意义下“几乎精确”。

## R04

算法流程：
1. 校验参数：`a<b`、`n` 为正偶数、函数值有限。
2. 用 `linspace` 构建网格 `x0...xn`。
3. 在网格上计算 `y_i=f(x_i)`。
4. 取端点项 `y0 + yn`。
5. 取奇数索引项求和并乘 4。
6. 取偶数索引项求和并乘 2。
7. 按 `(h/3)` 缩放，得到 `I_n`。
8. 再计算 `I_2n`，形成误差估计与精度报告。

## R05

`demo.py` 中的数据结构与职责：
- `SimpsonCase`：描述测试用例（函数、区间、基准 `n`、真值、容差）；
- `SimpsonResult`：记录一次案例运行结果（`I_n`、`I_2n`、误差、估计）；
- `evaluate_on_grid`：把函数稳健地映射到网格，并检查有限值；
- `composite_simpson`：复合辛普森核心实现；
- `run_case`：执行单案例并做断言；
- `main`：批量运行案例并打印汇总。

## R06

正确性说明（实现级）：
- 公式正确性：代码严格对应复合辛普森权重结构 `1,4,2,4,...,2,4,1`；
- 收敛正确性：对光滑函数，`n` 翻倍后误差应明显下降，符合 `O(h^4)` 趋势；
- 估计正确性：`|I_2n-I_n|/15` 是基于四阶误差模型的常用 Richardson 估计；
- 可验证性：每个案例都有解析真值，程序用断言检查实际误差是否在容差内。

## R07

时间复杂度：
- 单次 `composite_simpson` 主要是函数采样与求和，复杂度 `O(n)`；
- 每个案例计算 `I_n` 与 `I_2n` 两次，总体 `O(n)`（常数约 3 倍节点量）；
- 若有 `m` 个案例，总复杂度 `O(mn)`。

## R08

空间复杂度：
- 需存储网格与函数值数组，空间 `O(n)`；
- 额外标量状态（步长、和项、误差）为 `O(1)`；
- 整体空间复杂度 `O(n)`。

## R09

边界与异常处理：
- `a >= b`：抛出 `ValueError`；
- `n <= 0` 或 `n` 为奇数：抛出 `ValueError`；
- 函数返回 `nan/inf`：抛出 `ValueError`；
- 函数未向量化：通过 `np.vectorize` 后备路径继续计算；
- 测试误差超出阈值：抛出 `AssertionError`，用于自动化验收。

## R10

MVP 取舍：
- 只实现 1D 定积分，不扩展到自适应区间细分与奇异点处理；
- 以 `numpy` 为核心，保持依赖最小；
- `scipy.integrate.quad` 仅作为可选参考，不参与主算法；
- 不引入绘图库，避免非必要依赖与运行复杂度。

## R13

输出字段说明：
- `integral_n`：基准网格 `n` 的辛普森积分值；
- `integral_2n`：加密网格 `2n` 的辛普森积分值；
- `abs_error`：`|integral_2n - true_value|`；
- `richardson_est`：`|integral_2n - integral_n| / 15`；
- `tol`：案例允许误差阈值；
- `scipy_abs_error`：若安装 SciPy，则给出与 `quad` 对照误差。

## R14

最小测试集（`demo.py` 已覆盖）：
- `∫[0,π] sin(x) dx = 2`；
- `∫[0,1] exp(x) dx = e-1`；
- `∫[0,1] 1/(1+x^2) dx = π/4`；
- `∫[-1,2] (x^3-2x+1) dx`（三次多项式，验证高精度精确性）。

此外还包含一条护栏检查：
- 故意传入奇数 `n`，确认函数会正确抛错。

## R15

参数建议：
- `n` 取偶数，且应随函数振荡程度增大而增大；
- 对平滑函数，可先用中等 `n`，再看 `I_n` 与 `I_2n` 差值决定是否加密；
- 若 `richardson_est` 远大于容差，应提升 `n`；
- 若函数含尖峰或端点奇异，本 MVP 可能不足，应改用自适应积分或变量代换。

## R16

方法对比：
- 对比梯形法：
  - 梯形法全局误差 `O(h^2)`；
  - 辛普森法在同等网格下通常更高精度（`O(h^4)`）。
- 对比 Gauss 求积：
  - Gauss 求积在高精度场景常更高效；
  - 辛普森法实现简单、权重固定、工程可解释性强。
- 对比黑盒积分库：
  - 黑盒更强健；
  - 本实现更透明，便于教学和审计。

## R18

`demo.py` 源码级算法流程（8 步）：
1. `main` 调用 `verify_guardrail_for_odd_n`，先验证“`n` 必须为偶数”的输入保护逻辑。  
2. `main` 通过 `build_cases` 构造多个带解析真值的测试函数与区间。  
3. 每个案例进入 `run_case`，先调用 `composite_simpson(..., n)` 计算 `I_n`。  
4. `composite_simpson` 在 `validate_interval_and_n` 中检查区间和 `n`，随后构造等距网格。  
5. `evaluate_on_grid` 负责安全计算 `f(x_i)`：优先走向量化调用，失败时回退到 `np.vectorize`，并检查有限值。  
6. `composite_simpson` 按权重模式 `1-4-2-...-4-1` 组装和式并乘以 `h/3`，返回积分近似值。  
7. `run_case` 再计算 `I_2n`，用 `richardson_error_estimate = |I_2n-I_n|/15` 估计离散误差，并与解析真值比较得到 `abs_error`。  
8. `run_case` 执行容差断言并汇总结果，`main` 最后打印表格；若装有 SciPy，再追加 `quad` 对照误差用于交叉验证。  

说明：核心积分流程完全在本地源码中展开实现，第三方库只用于数组运算与可选参考对照，不是算法黑盒。
