# 表面码 (Surface Code)

- UID: `PHYS-0240`
- 学科: `物理`
- 分类: `量子信息`
- 源序号: `243`
- 目标目录: `Algorithms/物理-量子信息-0243-表面码_(Surface_Code)`

## R01

表面码（Surface Code）是当前容错量子计算中最重要的拓扑量子纠错码之一。它把逻辑信息编码到二维局域耦合结构中，通过重复测量稳定子综合（syndrome）并执行解码，将物理误差率压低到可容错区间。

本条目提供一个可运行、可审计的最小实现：

- 建立平面码 patch（码距 `d`，数据比特在边上）；
- 只考虑 `Z` 误差子问题（由 `X` 稳定子检测）；
- 假设综合测量无误（perfect syndrome）；
- 用源码实现的最小权重匹配思路做解码；
- 估计逻辑失败率（logical failure rate）。

## R02

为什么做这个 MVP：

- 真实表面码工程很大，包含循环测量、时空解码、门噪声、测量噪声和硬件约束；
- 直接上全栈容易“能跑但不可解释”；
- 因此先做一个边界清晰的子问题，用最少工具建立完整闭环。

这个 MVP 的目标不是替代工业解码器，而是把核心算法链路讲清楚并跑通。

## R03

本条目求解的问题：

- 输入：距离为 `d` 的平面表面码 patch，边上数据比特独立发生 `Z` 误差，概率为 `p`；
- 可观测：内点 `X` 稳定子综合（缺陷点集合）；
- 输出：匹配/连接路径对应的纠错链，以及该次试验是否发生逻辑失败。

逻辑失败定义：纠错后残余误差链与边界构成非平凡同调类（本实现用一条参考切线上的奇偶校验判定）。

## R04

离散模型（与 `demo.py` 一一对应）：

- 顶点集合：`(x, y), x,y in {0,...,d-1}`；
- 数据比特：格点之间的边（水平边 `h` 与竖直边 `v`）；总数 `n=2d(d-1)`；
- 误差链：`E ⊆ 边集`，在 GF(2) 意义下按奇偶叠加；
- 综合：对每个被测量顶点，统计其关联误差边数的奇偶；奇数即缺陷；
- 解码：在“缺陷点 + 边界”上找最小总长度连接方案；
- 纠错链 `C` 与误差链 `E` 异或得到残余链 `R=E⊕C`。

## R05

核心思想是“把量子误差恢复问题转成图上的最短连接问题”：

- `Z` 误差的端点对应 `X` 综合缺陷；
- 缺陷必须成对连接，或连接到可吸收端点的边界；
- 连接总代价（曼哈顿长度）越小，越符合独立同分布小噪声假设下的最大似然近似。

这正是表面码 MWPM 解码器背后的基本结构。

## R06

本实现的数据结构：

- `Edge = ('h'|'v', x, y)`：一条数据比特边；
- `Vertex = (x, y)`：一个综合顶点；
- `set[Edge]`：误差链/纠错链/残余链；
- `list[Vertex]`：当前 shot 的缺陷点集合。

关键操作：

- `toggle_edge`：GF(2) 下边奇偶翻转；
- `syndrome_from_errors`：从误差链提取缺陷点；
- `min_weight_actions`：对缺陷子集做动态规划求最小权动作；
- `correction_from_actions`：把动作还原成实际纠错路径。

## R07

最小流程伪代码：

```text
for each trial:
    sample error chain E with iid Bernoulli(p) on each edge
    defects = syndrome(E)
    actions = minimum_weight_pair_or_boundary(defects)
    C = path_union(actions)
    R = E xor C
    fail = logical_parity(R)
```

其中 `minimum_weight_pair_or_boundary` 在本项目中不是黑箱库，而是子集动态规划精确求解当前缺陷集。

## R08

正确性要点：

- 综合定义与边端点奇偶一致，保证“链的边界 = 缺陷集”；
- 解码后残余链在被测量内点不再有缺陷（程序中有断言）；
- 逻辑失败判定基于参考切线奇偶，可检测是否跨越粗边界形成非平凡链。

在 `d=3` 上还做了全空间穷举（`2^12=4096` 个误差模式）验证。

## R09

复杂度：

- 单次综合提取：`O(d^2)`；
- 若缺陷数为 `m`，解码 DP 复杂度约 `O(2^m * m^2)`；
- 本 MVP 适合 `d=3,5` 等小码距教学与验证，不面向大规模生产。

备注：工业实现通常改用 Blossom/MWPM 专用高性能实现。

## R10

数值与工程处理：

- 全流程是离散组合运算，不依赖浮点线性代数稳定性；
- Monte Carlo 使用固定随机种子，保证复现实验；
- 关键不变量（如解码后综合应清零）用 `assert` 强约束。

## R12

`demo.py` 默认参数：

- Monte Carlo：`distance_mc=5`，`mc_shots=2000`；
- 噪声网格：`p in {0.01, 0.03, 0.05, 0.07, 0.10, 0.12, 0.15}`；

可调方向：

- 扩大 `p_grid` 观察高噪声区行为；
- 调大 `distance_mc` 验证码距收益与计算代价。

## R13

输出表包含：

- `logical_fail_d3_exact`：`d=3` 全枚举得到的精确逻辑失败率；
- `logical_fail_d5_mc`：`d=5` 蒙特卡洛估计逻辑失败率；
- `distance_gain(d3-d5)`：两者差值（正值表示 `d=5` 更好）；
- `mean_syndrome_weight` 与 `mean_error_weight`：综合与物理误差的统计量。

通常在低噪声区会观察到更大码距带来更低逻辑失败率。

## R14

该 MVP 体现的理论/工程信息：

- 误差链端点驱动综合，综合驱动恢复，这是稳定子码的核心闭环；
- 解码等价于组合优化问题，可在图结构上实现；
- 码距提升在低噪声区通常带来逻辑层收益。

## R15

常见失败与误用：

- 把“无测量噪声”结果误当成真实硬件性能；
- 忽略路径表示的坐标约定，导致综合与逻辑判据错位；
- 把小码距上的结论外推到大规模系统。

本条目已在代码中用断言约束最关键的一致性，但仍需结合更完整噪声模型做外推。

## R16

当前边界（有意简化）：

- 仅覆盖 `Z` 误差子问题，不含 `X/Y` 全噪声联动；
- 不含时域重复综合（3D 时空解码）；
- 不含测量错误、门错误、泄露错误和串扰模型；
- 不含硬件耦合图与调度约束。

另外，本实现采用了教学友好的边界离散化与确定性 tie-break 规则，低噪声区曲线不应直接等同于文献中的标准阈值曲线。
因此它是“算法骨架演示”，不是硬件级容错基准。

## R18

`demo.py` 源码级算法流程（8 步）：

1. `list_data_edges` 与 `measured_vertices` 构建距离 `d` 的平面码离散结构。  
2. `sample_z_errors` 在每条数据边按 Bernoulli(`p`) 采样 `Z` 误差，得到误差链 `E`。  
3. `syndrome_from_errors` 计算每个被测量顶点的奇偶，提取缺陷点集合。  
4. `min_weight_actions` 用“缺陷子集 DP”精确求当前 shot 的最小代价连接动作（缺陷-缺陷或缺陷-边界）。  
5. `correction_from_actions` 把连接动作映射为具体边路径并合并为纠错链 `C`。  
6. `residual_chain` 计算残余链 `R = E xor C`，并通过断言检查内点综合已清零。  
7. `logical_failure_from_residual` 在参考切线上做奇偶判定，得到本次是否逻辑失败。  
8. `run_mvp/main` 汇总 `d=3` 精确枚举与 `d=5` Monte Carlo 结果，输出失败率表与诊断信息。  

说明：这里没有调用第三方解码黑箱；匹配与路径还原都在源码中逐步实现，便于审计与教学。
