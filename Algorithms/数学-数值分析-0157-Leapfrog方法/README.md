# Leapfrog方法

- UID: `MATH-0157`
- 学科: `数学`
- 分类: `数值分析`
- 源序号: `157`
- 目标目录: `Algorithms/数学-数值分析-0157-Leapfrog方法`

## R01

本条目实现 Leapfrog（蛙跳）方法的最小可运行版本，聚焦哈密顿系统中最常见的二阶振子模型：

- 方程：`q' = p, p' = -omega^2 q`（等价于 `q'' + omega^2 q = 0`）
- 初值：`q(0)=q0, p(0)=p0`

MVP 目标：

- 透明展示 Leapfrog 的 kick-drift-kick 更新过程；
- 验证其二阶收敛行为；
- 展示其长期能量保持优于显式 Euler 的结构性优势。

## R02

问题定义（固定步长版）：

- 输入：`q0, p0, h, n_steps, omega`
- 输出：离散轨迹 `t_n, q_n, p_n` 与能量 `H_n = 0.5*(p_n^2 + omega^2*q_n^2)`
- 目标：在给定步数内稳定近似真实轨道，并量化相位误差与能量漂移

本实现只处理标量振子（1 自由度），优先保证算法链路清晰可审计。

## R03

Leapfrog（kick-drift-kick）单步公式：

1. `a_n = -omega^2 * q_n`
2. `p_{n+1/2} = p_n + (h/2) * a_n`
3. `q_{n+1} = q_n + h * p_{n+1/2}`
4. `a_{n+1} = -omega^2 * q_{n+1}`
5. `p_{n+1} = p_{n+1/2} + (h/2) * a_{n+1}`

该方法时间可逆、辛结构友好、全局误差阶为 `O(h^2)`（平滑问题下）。

## R04

算法流程（`demo.py` 实际执行）：

1. 校验输入合法性（`h>0`、`n_steps>0`、初值与 `omega` 有限）。
2. 分配 `t/q/p` 数组并写入初值。
3. 每步先做半步动量更新（kick）。
4. 用半步动量推进位置（drift）。
5. 再做半步动量更新得到整步动量（kick）。
6. 计算并记录整段轨迹能量。
7. 输出轨迹用于误差与稳定性评估。

## R05

核心数据结构：

- `HarmonicOscillator(dataclass)`：封装 `a(q)` 并计数函数评估次数 `nfev`
- `Trajectory(dataclass)`：
  - `t: np.ndarray`
  - `q: np.ndarray`
  - `p: np.ndarray`
  - `energy: np.ndarray`
  - `nfev: int`

这样可以同时支持“数值结果检查”和“开销可观测”。

## R06

正确性要点：

- 代码严格按 Leapfrog 的半步-整步更新顺序实现，不混用 Euler 或 RK 更新。
- 选用有解析解的测试方程 `q''+q=0`，可直接比较 `(q(T), p(T))` 相位误差。
- 步长减半实验中，终点相位误差比接近 `4`，符合二阶方法预期。
- 长时积分中，Leapfrog 能量误差有界振荡，而显式 Euler 能量会系统漂移。

## R07

复杂度分析（总步数 `N`）：

- 时间复杂度：`O(N)`（每步常数次算术与 2 次加速度评估）
- 空间复杂度：
  - 保存完整轨迹：`O(N)`（本 MVP 采用）
  - 仅保留当前状态：`O(1)`（可作为工程压缩版）

Leapfrog 的优势不在“更少步”，而在“结构保持带来的长期数值质量”。

## R08

边界与异常处理：

- `h <= 0` 或 `n_steps <= 0`：抛 `ValueError`
- `q0/p0/omega` 非有限，或 `omega <= 0`：抛 `ValueError`
- 积分中若出现 `nan/inf`：抛 `RuntimeError`

脚本设计为“显式失败”，避免静默传播错误状态。

## R09

## R12

输出字段解读：

- `h`：步长
- `steps`：积分步数
- `phase_error(T)`：终点相空间误差 `sqrt((dq)^2 + (dp)^2)`
- `prev/cur`：相邻步长误差比（理想接近 `4`）
- `max |H-H0|`：全程最大能量漂移
- `drift_ratio (Euler / Leapfrog)`：Euler 漂移与 Leapfrog 漂移比值（越大越说明 Leapfrog 稳定）

## R13

建议最小测试集：

1. 收敛测试：`h in {0.2, 0.1, 0.05, 0.025}`，验证二阶误差比。
2. 长时测试：`T=200, h=0.1`，比较 Leapfrog 与 Euler 能量漂移。
3. 输入异常：`h<=0`、`n_steps<=0`、`omega<=0`、`q0/p0` 非有限。
4. 数值异常监测：人为注入极端初值，确认非有限值保护生效。

## R14

关键可调参数：

- `h`：步长，决定精度与计算量；
- `n_steps` / `t_end`：积分时长，决定长期稳定性观察窗口；
- `omega`：系统频率，影响相位速度和稳定步长需求。

实践建议：

- 先用中等 `h` 验证流程，再逐步减小 `h` 看误差阶；
- 长期结构测试时保持 `h` 固定，拉长 `t_end` 观察能量行为。

## R15

方法对比（同类 ODE 时间推进）：

- 显式 Euler：一阶、实现最简单、长期能量漂移显著；
- RK4：短时高精度优秀，但非辛形式在超长时能量可能缓慢漂移；
- Leapfrog：二阶但辛结构友好，长期哈密顿量保持通常更稳健。

因此 Leapfrog 在分子动力学、天体力学等哈密顿问题里非常常见。

## R16

应用场景：

- 经典力学轨道积分（行星、摆、弹簧系统）
- 分子动力学中的 Verlet / velocity-Verlet 核心时间推进
- 需要长期保持几何结构（相空间体积、能量有界误差）的仿真
- 数值分析课程中“辛积分器”入门示例

## R18

源码级算法流程拆解（`leapfrog_solve`，8 步）：

1. 调用 `validate_inputs` 检查 `q0/p0/h/n_steps/omega`，提前拒绝非法参数。  
2. 初始化 `HarmonicOscillator`（负责加速度计算与 `nfev` 统计）。  
3. 分配 `t, q, p` 数组并写入初始状态 `(q0, p0)`。  
4. 在每个时间步 `n`，先计算当前位置加速度 `a_n = -omega^2 q_n`。  
5. 执行半步动量更新：`p_half = p_n + 0.5*h*a_n`。  
6. 执行整步位置更新：`q_{n+1} = q_n + h*p_half`，再计算 `a_{n+1}`。  
7. 完成第二次半步动量更新：`p_{n+1} = p_half + 0.5*h*a_{n+1}`，并检查非有限值。  
8. 循环结束后计算 `energy` 数组并返回 `Trajectory`，供误差和稳定性评估。  

本实现未调用任何黑盒常微分方程求解器，Leapfrog 的每个更新环节都在源码中可直接追踪。
