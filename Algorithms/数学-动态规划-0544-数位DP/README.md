# 数位DP

- UID: `MATH-0544`
- 学科: `数学`
- 分类: `动态规划`
- 源序号: `544`
- 目标目录: `Algorithms/数学-动态规划-0544-数位DP`

## R01

本条目实现一个可运行的数位 DP（Digit DP）最小版本，用于统计区间内满足数位约束的整数数量。  
MVP 目标是把“按位受限 + 区间计数”的通用套路落地为可验证代码，而不是只给公式。

本实现统计条件为：
1. 数位和可被 `3` 整除；
2. 任意相邻两位数字不能相同。

## R02

问题定义（MVP）：
- 输入：两个非负整数 `L, R`，满足 `0 <= L <= R`。
- 输出：区间 `[L, R]` 内满足上述两个条件的整数个数。

采用经典转换：
`count(L, R) = count_up_to(R) - count_up_to(L - 1)`，  
其中 `count_up_to(n)` 表示区间 `[0, n]` 的合法数量。

## R03

状态设计：

定义 `dfs(pos, mod3, prev_digit, started, tight)`：
- `pos`：当前处理到第几位（从高位到低位）；
- `mod3`：当前已选数位和对 `3` 取模；
- `prev_digit`：上一位数字（`10` 作为“尚未开始”的哨兵）；
- `started`：是否已经放置过非前导零数字；
- `tight`：当前前缀是否与上界前缀完全相同。

该状态足以表达“前缀合法性”和“后续可选空间”。

## R04

转移规则：
1. 枚举当前位数字 `d`，范围为 `0..upper`，其中 `upper = digits[pos]`（若 `tight=True`）否则 `9`；
2. 若仍处于前导零区间（`not started and d == 0`），则：
   - 不更新 `mod3`；
   - `prev_digit` 维持哨兵；
   - 继续下一位；
3. 否则进入真实数字状态：
   - 若 `started=True` 且 `d == prev_digit`，则剪枝（不合法）；
   - 更新 `next_mod3 = (mod3 + d) % 3`；
4. 边界 `pos == len(digits)` 时，若 `mod3 == 0` 返回 `1`，否则返回 `0`。

注：本实现把数字 `0` 视为合法数（数位和为 0，且不存在相邻冲突）。

## R05

核心数据结构：
- `digits: List[int]`：上界 `n` 的十进制位数组；
- `lru_cache`：缓存 `dfs` 子问题结果；
- `RunReport(dataclass)`：汇总每个测试用例的输出；
- `sentinel_prev = 10`：表示“尚无上一位”的状态哨兵。

## R06

正确性要点：
1. 完备性：每一位都枚举了可选数字，覆盖所有 `0..n` 数。
2. 无重计：每个具体数字对应唯一一条从高位到低位的决策路径。
3. 合法性：相邻相等在转移中被显式剪枝；`mod3` 在每次放置真实数字时准确累积。
4. 区间化：通过前缀计数差分 `count_up_to(R)-count_up_to(L-1)` 得到 `[L,R]` 精确结果。
5. 边界一致：`L=0`、`R=0`、大整数上界都由统一逻辑覆盖。

## R07

复杂度分析（十进制，`D` 为位数）：
- 状态数量上界约为 `D * 3 * 11 * 2 * 2 = O(D)`（常数较大但固定）；
- 每个状态枚举最多 `10` 个数字；
- 时间复杂度近似 `O(D * 3 * 11 * 2 * 2 * 10)`；
- 空间复杂度为缓存状态数，即 `O(D)`（在进制固定时）。

对于 `64` 位整数范围（最多 19 位十进制）依然非常快。

## R08

边界与异常处理：
- `L < 0`：抛出 `ValueError`（本 MVP 仅处理非负整数）；
- `L > R`：抛出 `ValueError`；
- `n < 0` 的 `count_up_to(n)` 直接返回 `0`（便于差分公式）；
- 大范围计数（例如到 `10^18-1`）不做暴力校验，仅运行 DP；
- 小范围自动启用暴力校验，若与 DP 不一致则抛 `AssertionError`。

## R09

MVP 设计取舍：
- 仅使用 Python 标准库（`functools/dataclasses/time`），避免外部黑盒；
- 约束选择为“模约束 + 相邻约束”，展示数位 DP 常见状态组合；
- 提供可验证小样例与不可暴力大样例，兼顾正确性与规模感；
- 不引入复杂模板框架，保持源码教学可读性。

## R10

`demo.py` 模块职责：
- `is_valid_number`：单数合法性判定（供暴力校验）；
- `brute_force_count`：小区间真值计算；
- `count_up_to`：核心数位 DP 前缀计数；
- `count_in_range`：区间差分封装；
- `run_case`：单样例执行、计时、可选断言；
- `main`：固定测试集批量运行并打印报告（无交互输入）。

## R11

运行方式：

```bash
cd Algorithms/数学-动态规划-0544-数位DP
uv run python demo.py
```

脚本将自动运行 4 组固定区间测试，不需要任何输入。

## R12

输出字段说明：
- `Range`：当前测试区间 `[L, R]`；
- `Digit-DP Count`：DP 计算出的合法数量；
- `Brute-force Count`：小区间暴力真值（大区间会显示 `skipped`）；
- `Visited DP States`：两次前缀 DP（`R` 与 `L-1`）累计访问状态数；
- `Elapsed Time (ms)`：本次区间统计耗时（毫秒）。

## R13

最小测试覆盖点：
1. `small_exact_1: [0, 200]`：覆盖 `0` 与低位边界；
2. `small_exact_2: [123, 4567]`：覆盖跨位数与一般区间；
3. `medium_exact_3: [10000, 99999]`：中等规模下继续做暴力一致性验证；
4. `large_dp_only: [0, 10^18-1]`：验证大范围可计算性与性能。

## R14

可调参数：
- 模数：当前固定为 `3`（可扩展为任意 `K`）；
- 暴力阈值：`brute_limit=200_000`；
- 测试集：`main()` 中 `cases` 可自由增减；
- 约束规则：目前是“相邻不等”，可扩展为“禁止某些数字/模式”。

## R15

与其他方法对比：
- 直接遍历：实现简单，但在大区间（如 `10^18`）不可行；
- 数学闭式：仅适用于极少数规则，不通用；
- 数位 DP：通用、可扩展，适合“按位条件 + 区间计数”问题。

本实现重点是“可解释的状态转移”，而不是追求最短代码。

## R16

典型应用场景：
- 统计满足某些数字特征的账号/编号数量；
- 区间内风控规则命中数估计（按位特征）；
- 算法课程中“记忆化搜索 + 状态压缩”的示例；
- 作为更复杂数位约束题（如含 `mask/自动机`）的基础模板。

## R17

可扩展方向：
- 将 `mod3` 泛化为 `modK`；
- 增加“禁用数字集合”或“数字出现次数限制”；
- 接入自动机状态（Aho-Corasick）处理“禁止子串”类约束；
- 增加统计维度（例如同时返回合法数量与数值总和）。

## R18

源码级算法流程（`demo.py`，8 步）：
1. `main` 定义固定测试区间，逐个调用 `run_case`。  
2. `run_case` 调用 `count_in_range(L, R)`，并记录耗时。  
3. `count_in_range` 分别计算 `count_up_to(R)` 与 `count_up_to(L-1)` 后做差。  
4. `count_up_to` 将上界转为 `digits` 数组，定义记忆化 `dfs`。  
5. `dfs` 在每一位根据 `tight` 决定枚举上界，并处理前导零分支。  
6. 对真实数字分支，先执行“相邻不等”剪枝，再更新 `mod3` 递归到下一位。  
7. 到达末位时按 `mod3==0` 决定是否计入 1，缓存子问题结果避免重复计算。  
8. 小区间下 `run_case` 还会调用 `brute_force_count` 对拍，确保 DP 与真值一致。
