## R01

ARIMA（AutoRegressive Integrated Moving Average）是经典单变量时间序列模型，用于处理具有自相关与趋势的序列。

- `AR(p)`：用过去观测值解释当前值；
- `I(d)`：通过差分让序列近似平稳；
- `MA(q)`：用过去随机扰动（残差）修正当前预测。

本目录提供一个可运行 MVP，演示 ARIMA 的最小实现链路：差分、参数估计、滚动预测、反差分恢复。

## R02

ARIMA 的典型应用场景：

- 销量、流量、价格等按时间采样的一维序列预测；
- 序列包含趋势或缓慢漂移，不满足直接平稳假设；
- 需要可解释且计算成本低的统计基线模型。

在工程中，ARIMA 常被当作“可解释的第一版时序模型”，用于与更复杂方法（如深度学习）做基线对比。

## R03

本实现建模任务定义如下：

- 输入：单变量序列 `y_t` 与模型阶数 `(p, d, q)`；
- 处理：先做 `d` 阶差分，得到近似平稳序列 `x_t`；
- 拟合：在 `x_t` 上拟合 ARMA(`p`,`q`)；
- 输出：
  - 拟合参数（截距、AR 系数、MA 系数）；
  - 训练残差、AIC/BIC；
  - 多步预测值（先预测差分，再反差分回原序列尺度）。

## R04

核心数学形式：

1. `d` 阶差分：`x_t = (1 - B)^d y_t`，`B` 为滞后算子。  
2. 差分后的 ARMA 模型：
   `x_t = c + sum_{i=1..p}(phi_i x_{t-i}) + eps_t + sum_{j=1..q}(theta_j eps_{t-j})`。  
3. 条件最小二乘（CSS）：通过迭代估计残差，最小化 `sum eps_t^2`。  
4. 高斯似然近似：`logL = -n/2 * [log(2*pi*sigma^2) + 1]`。  
5. 信息准则：`AIC = 2k - 2logL`，`BIC = k*log(n) - 2logL`。

## R05

`demo.py` 的高层流程：

1. 生成一段可复现的 ARIMA(1,1,1) 合成数据；
2. 划分训练集与测试集；
3. 对候选阶数组合做拟合并计算 AIC/BIC；
4. 选取 AIC 最优模型；
5. 用最优模型做多步预测；
6. 反差分得到原序列预测；
7. 与简单基线（Last Value）比较 RMSE/MAE；
8. 执行断言门禁，确保结果有效。

## R06

MVP 默认实验配置：

- 真模型：ARIMA(1,1,1)；
- 真实参数：
  - 差分序列漂移 `c=0.35`
  - AR 系数 `phi=0.55`
  - MA 系数 `theta=-0.30`
- 样本总长：`320`；
- 切分：训练 `260`，测试 `60`；
- 候选模型：`(0,1,1)`, `(1,1,0)`, `(1,1,1)`, `(2,1,1)`。

该设置保证模型可辨识，且能稳定展示 ARIMA 相对朴素基线的优势。

## R07

复杂度分析（样本长度 `n`，迭代轮数 `T`）：

- 单次 CSS 迭代需要构建设计矩阵并做最小二乘，约 `O(n * k^2 + k^3)`，`k=1+p+q`；
- 总拟合成本约 `O(T * (n * k^2 + k^3))`；
- 多步预测 `h` 的成本约 `O(h * (p+q+d))`；
- 空间复杂度约 `O(n)`（存储序列、残差、拟合值）。

对本目录的几十到几百样本规模，计算开销很小。

## R08

数值稳定与工程实现要点：

- 只用 `numpy/pandas`，避免隐藏黑箱依赖；
- OLS 用 `numpy.linalg.lstsq`，不显式求逆；
- 残差方差设置下界 `EPS=1e-10`，避免 `log(0)`；
- 迭代设置 `max_iter` 与 `tol`，并返回 `converged` 状态；
- 反差分采用状态递推，支持任意 `d>=0`。

## R09

参数含义解释：

- `intercept`：差分序列上的漂移项；
- `ar_params`：自回归系数，反映“惯性”；
- `ma_params`：移动平均系数，反映“噪声冲击的短期传播”；
- `sigma2`：残差方差估计；
- `aic/bic`：用于候选模型比较，值越小越好。

注意 AIC/BIC 只在同一数据与同一似然框架下可比较。

## R10

`demo.py` 主要函数分工：

- `difference_series`：执行 `d` 阶差分；
- `invert_differences`：把差分预测恢复到原序列尺度；
- `fit_arma_css`：在差分序列上用 CSS 拟合 ARMA(`p`,`q`)；
- `fit_arima_css`：封装 ARIMA(`p`,`d`,`q`) 拟合入口；
- `forecast_arima`：递推预测并反差分；
- `simulate_arima_111`：构造可控实验数据；
- `evaluate_candidate_orders`：评估候选阶数并汇总 DataFrame；
- `main`：组织实验、打印结果、执行断言。

## R13

正确性门禁（代码中断言）：

1. 候选评估表不允许 NaN；
2. 至少一个候选模型收敛；
3. 最优模型参数与 AIC/BIC 必须是有限值；
4. 预测长度必须与测试集长度一致；
5. ARIMA 的 RMSE 必须优于 Last Value 基线。

这些检查能防止“只运行不验证”的伪完成。

## R14

常见失效模式与对策：

- 失效 1：序列未平稳且 `d` 过小。  
  对策：增加差分阶数，或先做单位根检验后再定 `d`。

- 失效 2：候选 `(p,q)` 过大导致参数不稳。  
  对策：先从小阶数组合搜索，再按 AIC/BIC 筛选。

- 失效 3：样本太短，MA 项估计高方差。  
  对策：降低 `q`，或改为更稳健的简化模型。

- 失效 4：结构突变（节假日、策略切换）破坏线性假设。  
  对策：分段建模或引入外生变量（升级到 ARIMAX/SARIMAX）。

## R15

实践建议：

- 把 ARIMA 当作时序建模基线，先获得可解释基准；
- 先做数据清洗（缺失、异常值、时间戳对齐）再建模；
- 不要只看训练集 AIC/BIC，务必加测试集误差复核；
- 业务上可把 ARIMA 与规则基线（季节性、节假日）组合评估。

## R16

与其他方法对比：

- 相比指数平滑：ARIMA 能显式建模自相关与噪声项；
- 相比 Prophet/LSTM：ARIMA 参数更少、可解释性更强、调试更直观；
- 相比纯机器学习回归：ARIMA 原生适配时序依赖结构，不必手工构造大量滞后特征。

因此 ARIMA 常用于“先验统计建模 + 后续复杂模型”的第一阶段。

## R18

`demo.py` 的源码级算法流程（9 步）：

1. `simulate_arima_111` 先在差分空间生成 ARMA(1,1) 序列，再累加成原始序列，得到可复现实验数据。  
2. `main` 划分训练/测试集，并定义候选阶数 `(p,d,q)`。  
3. `fit_arima_css` 对每个候选先调用 `difference_series`，把原序列转换到 `d` 阶差分空间。  
4. `fit_arma_css` 在差分序列上迭代执行 CSS：用“历史值 + 历史残差”构造线性方程，最小二乘更新参数。  
5. 同一函数中按新参数递推重算整段残差，直到参数改变量低于 `tol` 或达到 `max_iter`。  
6. 用收敛后的残差估计 `sigma2`，再计算高斯 `log_likelihood` 与 `AIC/BIC`。  
7. `evaluate_candidate_orders` 汇总候选评分表，并按 AIC 选最优模型。  
8. `forecast_arima` 在差分空间递推未来 `h` 步，再通过 `invert_differences` 将预测逐步还原到原序列尺度。  
9. `main` 计算 ARIMA 与 Last Value 的 RMSE/MAE，执行断言门禁并输出结果。

这 9 步完整展开了 ARIMA 的最小实现链路，没有把核心估计逻辑留在第三方黑盒内部。
