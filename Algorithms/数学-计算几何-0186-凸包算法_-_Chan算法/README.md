## R01

Chan算法是二维凸包的输出敏感（output-sensitive）算法：

- 输入 `n` 个平面点；
- 输出其凸包顶点序列，记凸包点数为 `h`；
- 经典复杂度可达到 `O(n log h)`。

核心思想是“分治 + 包裹（gift wrapping）”融合：
先把点集切成多个小组求子凸包，再在子凸包之间执行Jarvis式逐边推进。

## R02

历史脉络（简要）：

- Graham Scan / Andrew Monotonic Chain：`O(n log n)`，实现简单；
- Jarvis March（Gift Wrapping）：`O(nh)`，输出敏感但在 `h` 大时不经济；
- Chan（1996）：把两类方法结合，达到 `O(n log h)`，同时保持较强可实现性。

Chan的价值在于：当 `h << n` 时，它通常比纯 `O(n log n)` 方法更有优势。

## R03

本条目问题定义：

- 输入：二维点集 `P = {p_i}`（允许重复点、共线点）；
- 输出：按逆时针（CCW）排列的凸包顶点，不重复首尾点。

几何约定：

- 使用叉积 `cross(o,a,b)` 判断转向；
- `cross > 0`：左转，`cross < 0`：右转，`cross = 0`：共线。

## R04

Chan算法标准流程可概括为：

1. 猜测一个上界 `m`（希望 `m >= h`）。
2. 把 `n` 个点按组大小 `m` 划分为若干组。
3. 每组用 `O(m log m)` 算法（如 Andrew）求子凸包。
4. 从全局最左下点出发，做最多 `m` 步包裹：
   - 每步在每个子凸包上找“对当前点最优的切点/支撑点”；
   - 在这些候选中选全局下一个凸包顶点。
5. 若在 `<= m` 步内回到起点，说明猜测成功；否则增大 `m` 重来。

## R05

复杂度分两层看：

- 经典Chan（子凸包切线可 `O(log m)` 查询）：
  - 单轮约 `O(n log m)`；
  - 通过特定增大策略，总体 `O(n log h)`。

- 本目录 MVP（教学版取舍）：
  - 子凸包支撑点使用线性扫描，而非二分切线；
  - 单步候选评估约 `O(n)`，单轮上界约 `O(n log m + n*m)`；
  - 外层使用 `m` 翻倍策略，整体更接近 `O(nh + n log^2 h)` 的实用实现。

即：保留Chan框架，不把“切线查找”写成黑箱，优先代码透明与可验证性。

## R06

微型示例（直觉）：

给定点集包含外层四边形点与若干内点。

- 若当前点是左下角 `p`，算法会在每个子凸包上给出一个候选点；
- 全局比较后，选出使“其它点都在边 `p->q` 左侧（或共线且更近）”的 `q`；
- 重复该过程，边界点会依次被包裹出来，直到回到起点。

这与Jarvis March一致，但候选来自“子凸包摘要”而非“所有原始点”。

## R07

算法意义：

- 在 `h` 远小于 `n` 的几何数据里，输出敏感策略很有价值；
- Chan将“易实现的小规模凸包算法”作为模块复用；
- 工程上适合作为“纯 `O(n log n)` 算法”与“纯 `O(nh)` 算法”之间的折中。

## R08

理论要点：

1. 凸包边界可由局部“最外转向”逐步确定（包裹思想）。
2. 点集分组后，每组内部非极点不会成为全局下一步候选。
3. 子凸包上的支撑点可代表该组对“下一条边”的最强竞争者。
4. 若 `m >= h`，包裹阶段最多 `h` 步就会闭合回起点。
5. 若未在 `m` 步内闭合，可判定本轮 `m` 过小，需要增大。

## R09

适用条件与边界：

- 适用：二维静态点集凸包；
- 可处理：重复点、含共线点（本实现保留端点，剔除中间共线点）；
- 不覆盖：
  - 三维及更高维凸包；
  - 动态点集（在线增删）；
  - 需要精确有理算术的高鲁棒几何内核。

## R10

正确性（工程可验证版）核心：

1. 子凸包正确：每组用Andrew算法得到CCW凸多边形。  
2. 下一点选择正确：`is_better_candidate` 用叉积规则逐候选比较，保持“最外侧”点。  
3. 共线处理正确：共线时取更远点，避免边上中间点干扰。  
4. 终止条件正确：当候选回到起点，得到闭合凸包。  
5. 验证闭环：`demo.py` 将Chan结果与全量Andrew参考凸包做顶点集比对。

## R11

数值与退化处理：

- 使用 `EPS=1e-12` 处理浮点比较；
- 去重后再求凸包，降低重复点导致的分支噪声；
- 全部共线时返回两个端点；
- 额外提供 `points_inside_or_on_hull` 检查所有输入点是否在结果凸包内/边上。

## R12

本MVP参数策略：

- 外层初始 `m=4`，失败则 `m*=2`；
- 子凸包构造：Andrew Monotonic Chain；
- 数据生成：固定随机种子，混合随机云点 + 外环点 + 共线点 + 重复点；
- 校验：
  - 与参考凸包顶点集一致；
  - 所有点均在结果凸包内部或边界上。

## R14

常见失效模式与防护：

- 失效：转向符号写反，导致顺/逆时针混乱。  
  - 防护：统一 `cross` 定义并在注释中固定几何语义。
- 失效：共线时不处理“更远点”，造成凸包缺失极点。  
  - 防护：比较函数加入距离平局规则。
- 失效：重复点未去重，导致无意义循环或边界判断波动。  
  - 防护：入口先 `sorted(set(points))`。
- 失效：只看顶点数量不看几何包含性。  
  - 防护：增加“所有点在凸包内/边上”的几何校验。

## R15

工程实践建议：

- 若目标是“可维护教学实现”，本MVP结构足够透明；
- 若目标是大规模性能，应把 `support_point_linear` 升级为子凸包二分切线查询；
- 在真实数据中，建议增加整数坐标快速路径或精确谓词（robust predicates）；
- 对大输入可加入计时与采样诊断，定位瓶颈在“子凸包构造”还是“包裹候选查询”。

## R16

算法关系与变体：

- 与 Graham/Andrew：后者固定 `O(n log n)`，Chan在 `h` 小时可更优。
- 与 Jarvis March：Jarvis是 `O(nh)`；Chan通过分组与子凸包候选压缩改进。
- 可替换模块：
  - 组内凸包可换任意正确 `O(m log m)` 方法；
  - 子凸包支撑点可从线性扫描升级为二分切线。

## R18

源码级流程拆解（对应 `demo.py`，8步）：

1. `make_demo_points` 生成固定种子的测试点集，并混入共线点与重复点。  
2. `unique_points` 对输入去重排序，得到稳定点序列。  
3. `chan_convex_hull` 以 `m=4` 开始，进入“尝试-失败则放大 m”的外层循环。  
4. `chan_once` 把点集按组大小 `m` 划分，每组调用 `monotonic_chain` 得到子凸包。  
5. 从全局最左下点起步，执行至多 `m` 次包裹迭代。  
6. 每步对每个子凸包调用 `support_point_linear`，再由 `is_better_candidate` 做全局候选竞争。  
7. 若候选回到起点则闭合成功返回；若 `m` 步内未闭合，`chan_once` 返回失败，由外层增大 `m`。  
8. `main` 将Chan结果与全量 `monotonic_chain` 参考结果比对，并做“全部点在凸包内/边上”断言。

说明：本实现没有把第三方库凸包当黑箱；每个几何判断与状态转移均在源码可追踪。
