# 哈密顿路径算法

- 目标目录: `Algorithms/计算机-图算法-0045-哈密顿路径算法`

## R01

哈密顿路径（Hamiltonian Path）是指在图中经过每个顶点恰好一次的一条路径。  
该问题是组合优化与图搜索中的经典问题，判定版本属于 NP-Complete。  
本条目聚焦无向简单图上的“是否存在 + 构造一条路径”的最小可运行实现。

## R02

本条目解决的问题：
- 输入：无向图邻接矩阵 `A`（`n x n`，`A[i,j] in {0,1}`，对角线为 `0`）。
- 输出：
  - 若存在哈密顿路径，返回顶点序列 `path`（长度为 `n`，每个顶点出现一次）；
  - 若不存在，返回 `None`；
  - 同时给出位压 DP 的存在性校验结果。

`demo.py` 使用固定样例图，无需用户输入。

## R03

核心定义与判定条件：
- 路径合法：相邻顶点间必须有边。
- 覆盖性：路径包含图中全部 `n` 个顶点。
- 不重复性：每个顶点仅出现一次。

形式化地，若路径为 `v0, v1, ..., v(n-1)`，则需满足：
- `A[vk, v(k+1)] = 1`（`k=0..n-2`）
- `{v0, ..., v(n-1)} = {0,1,...,n-1}`

## R04

MVP 采用“回溯搜索（构造）+ 位压 DP（校验）”双轨流程：
1. 校验邻接矩阵是否合法（方阵、0/1、对称、对角为 0）。
2. 回溯：从候选起点出发，逐步扩展路径，遇到死路回退。
3. 若路径长度达到 `n`，立即返回该哈密顿路径。
4. 若全部分支失败，则回溯结果为 `None`。
5. 额外使用位压 DP 计算“是否存在”并与回溯结果做一致性检查。

## R05

核心数据结构：
- `adj: np.ndarray (n, n)`：邻接矩阵。
- `path: list[int]`：当前构造中的路径。
- `used: np.ndarray (n,)`：顶点是否已被使用。
- `degrees: np.ndarray (n,)`：顶点度数，用于候选顺序启发。
- `dp: np.ndarray (2^n, n)`：位压 DP 状态，`dp[mask, v]` 表示“存在一条覆盖 `mask` 且终点为 `v` 的简单路径”。

## R06

正确性要点：
- 回溯过程中始终维持不变量：`path` 中顶点不重复，且相邻顶点有边。
- 当 `len(path)=n` 时，由不变量可知该路径覆盖全部顶点且合法，因此是哈密顿路径。
- 回溯穷举所有起点与可行扩展分支；若未找到路径，则不存在可行哈密顿路径。
- 位压 DP 的状态转移覆盖所有简单路径扩展，`any(dp[(1<<n)-1, :])` 与“存在哈密顿路径”逻辑等价。

## R07

复杂度分析：
- 回溯最坏时间复杂度：`O(n!)`，空间复杂度：`O(n)`（递归栈 + 路径状态）。
- 位压 DP 时间复杂度：`O(n^2 * 2^n)`，空间复杂度：`O(n * 2^n)`。

MVP 演示规模选在小图（`n<=12` 量级）以保证可运行性。

## R08

边界与异常处理：
- 非方阵、空矩阵、含非 0/1 元素、非对称矩阵、对角非 0：抛出 `ValueError`。
- 构图时若边端点越界或出现自环请求：抛出 `ValueError`。
- 位压 DP 对大规模图设置保护（`n > 20` 时拒绝执行）避免内存爆炸。

## R09

MVP 取舍：
- 不依赖 `networkx` 等现成求解器，避免把核心逻辑黑箱化。
- 回溯实现保持可读，加入“按度数排序”的轻量启发式提升平均搜索效率。
- 位压 DP 仅用于验证存在性，不做完整路径恢复，控制代码体量。

## R10

`demo.py` 函数职责：
- `validate_adjacency_matrix`：邻接矩阵合法性校验。
- `build_undirected_adjacency`：由边列表构建无向图邻接矩阵。
- `find_hamiltonian_path_backtracking`：回溯构造哈密顿路径。
- `hamiltonian_path_exists_bitmask_dp`：位压 DP 判定是否存在哈密顿路径。
- `is_valid_hamiltonian_path`：校验候选路径正确性。
- `run_case`：执行单个测试图并打印结果。
- `main`：组织样例并运行演示。

## R12

输出字段说明：
- `adjacency matrix`：测试图邻接矩阵。
- `backtracking path`：回溯找到的路径（或 `None`）。
- `backtracking found`：回溯是否找到解。
- `path validity`：返回路径是否通过合法性检查。
- `bitmask-dp exists`：位压 DP 的存在性结果。
- `consistency check`：两种方法是否一致。

## R13

建议最小测试集：
- 存在哈密顿路径的链式/稠密图（应返回有效路径）。
- 不存在哈密顿路径的星形图（叶子过多导致失败）。
- 单顶点图（边界条件）。
- 非法输入矩阵（非对称、非 0/1、对角非 0）应抛异常。

## R14

可调参数：
- 启发式顺序：当前为“优先扩展低度顶点”，可改成高连通优先。
- 位压 DP 的规模上限：默认 `n <= 20`。
- 演示图规模与边密度：可用于观察回溯耗时变化。

工程建议：实际大规模场景可引入剪枝、分支限界、SAT/ILP 编码或并行搜索。

## R15

与相关方法对比：
- 回溯：可直接给出一条路径，最坏复杂度高，但实现直观。
- 位压 DP：判定复杂度优于裸回溯（理论上），但内存开销大。
- SAT/ILP：可借助成熟求解器处理更大实例，但工程依赖更重。

本 MVP 选择“回溯构造 + DP 校验”，兼顾可解释性与结果可信度。

## R16

典型应用：
- 路径规划与任务编排中的“全点覆盖一次”子问题。
- 生物信息中的序列拼接/重构抽象模型。
- 测试覆盖顺序设计、巡检流程设计中的组合可达性分析。
- 作为旅行商问题（TSP）相关建模的基础组件。

## R17

可扩展方向：
- 在位压 DP 中恢复完整路径而非仅判定存在性。
- 增加割点/度约束等预判剪枝降低搜索空间。
- 支持有向图版本与哈密顿回路版本。
- 引入并行 DFS 或 meet-in-the-middle 策略处理更大 `n`。

## R18

`demo.py` 源码级算法流程（8 步）：
1. `main` 构造两个固定测试图：一个存在哈密顿路径，一个为无解星形图。  
2. `run_case` 调用 `find_hamiltonian_path_backtracking`，先做输入矩阵合法性检查。  
3. 回溯函数按顶点度数决定起点顺序，初始化 `used` 与 `path`。  
4. 深搜扩展时，只尝试“与当前顶点相邻且未访问”的候选点，并按度数排序。  
5. 每次扩展把候选点加入 `path`，若后续失败则撤销该步（回溯）。  
6. 当递归深度达到 `n`，立即返回当前路径作为哈密顿路径；所有分支失败则返回 `None`。  
7. `run_case` 继续执行 `hamiltonian_path_exists_bitmask_dp`：用 `dp[mask, v]` 逐步扩展状态直到全覆盖掩码。  
8. 最后输出回溯结果、路径合法性和 DP 存在性，并打印二者一致性检查结果。
