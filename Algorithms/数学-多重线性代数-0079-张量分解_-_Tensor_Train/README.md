## R01

Tensor Train（TT，张量列车）把高阶张量分解为一串三阶核心张量（首尾为边界核）：

`X(i1,...,id) = G1(1,i1,a1) G2(a1,i2,a2) ... Gd(a_{d-1},id,1)`

其中 `Gk ∈ R^(r_{k-1} x n_k x r_k)`，`r_k` 是 TT-rank，且边界满足 `r_0=r_d=1`。直观上，TT 用“链式低秩结构”替代全量 `n_1...n_d` 参数，避免维度灾难。

## R02

本条目 MVP 解决的问题：
- 输入：一个实值高阶张量 `X`、最大秩 `max_rank`、相对截断阈值 `rel_tol`；
- 输出：TT 核心序列 `G1...Gd`、估计秩 `r_0...r_d`、重建误差与压缩比；
- 目标：在可控误差下，用远少于稠密表示的参数完成近似表示。

`demo.py` 使用合成 4 阶张量，自动执行 TT-SVD 并打印验收指标。

## R03

核心数学对象：
- 原张量：`X ∈ R^(n1 x ... x nd)`；
- 第 `k` 步展开矩阵：`X_[k] ∈ R^((r_{k-1}n_k) x (n_{k+1}...n_d))`；
- SVD：`X_[k] = U S V^T`；
- 截断后秩：`r_k`，并令
  - `G_k = reshape(U[:,1:r_k], (r_{k-1}, n_k, r_k))`
  - 下一步工作矩阵 `W = S_{1:r_k,1:r_k} V_{1:r_k,:}^T`。

循环到 `k=d-1` 后，最后一个核心由 `W` reshape 得到。

## R04

本实现采用 TT-SVD（Sequential SVD）而不是黑盒库：
1. 从左到右逐步展开当前工作张量；
2. 对展开矩阵做截断 SVD；
3. 把左奇异向量重排为当前 TT 核；
4. 将奇异值与右奇异向量乘积传递给下一步。

每一步都可追踪到标准线性代数操作，便于审计与教学。

## R05

复杂度（`d` 阶张量）主要由每步 SVD 决定。第 `k` 步对矩阵
`(r_{k-1}n_k) x (n_{k+1}...n_d)` 做 SVD，近似代价：

`O(min(m_k n_k^2, m_k^2 n_k))`（记 `m_k = r_{k-1}n_k`, `n_k' = n_{k+1}...n_d`）。

总体复杂度是各步之和。与直接在全展开矩阵上一次分解相比，TT-SVD 通过链式秩把计算分散到多个较小子问题。

## R06

空间复杂度：
- 稠密张量占 `O(prod n_k)`；
- TT 核总参数为 `Σ_k r_{k-1} n_k r_k`；
- 额外临时空间来自每步 SVD 的 `U,S,V^T`。

当 TT-rank 远小于模式维度时，`Σ_k r_{k-1} n_k r_k << prod n_k`，压缩效果明显。

## R07

TT-SVD 的正确性直觉：
- 每一步对当前展开矩阵做截断 SVD，得到该步最优（Frobenius 意义）低秩近似；
- 已确定的左因子被“冻结”为 TT 核，不再回退；
- 误差由各步截断奇异值尾部能量累积。

因此它是“逐步局部最优”的构造法，工程上稳定且可解释。

## R08

为什么 TT 重要：
- 对高阶数据（视频、PDE 网格、多体波函数）可避免参数爆炸；
- 保留多模态结构，不需要粗暴展平；
- 推断与存储都能在低秩结构中进行。

TT 是高阶线性代数中最常用的可扩展分解之一。

## R09

适用条件与局限：
- 适合存在链式低秩结构的数据；
- TT-rank 需要手动上限或容差策略控制；
- 核之间存在规范自由度（gauge freedom），分解不唯一；
- 维度顺序会影响中间秩与压缩率，模式重排常有收益。

MVP 不做自动模式重排，只演示核心算法。

## R10

工程可验证正确性（本条目口径）：
1. 输出核数量等于张量阶数；
2. 边界秩满足 `r_0=r_d=1`；
3. 重建张量与输入张量相对误差在阈值内；
4. 压缩参数量小于稠密参数量；
5. 所有中间结果为有限值。

`demo.py` 中 `run_checks` 对这些条件做了自动断言。

## R11

数值稳定策略：
- 合成真值核按 `1/sqrt(r_{k-1} n_k)` 缩放，减少值域爆炸；
- `rel_tol` 按 `d-1` 个截断步骤均摊为局部阈值；
- 每步只保留必要秩，避免无意义大秩传播；
- 对输入范数、非有限值做显式检查。

这保证了 MVP 在普通 CPU 环境下稳定可复现。

## R12

`demo.py` 的核心函数分工：
- `reconstruct_tt`：将 TT 核链式收缩回稠密张量；
- `build_random_tt_cores`：生成可控尺度的真值 TT 核；
- `choose_rank_from_singular_values`：依据尾能量和 `max_rank` 选秩；
- `tt_svd`：主算法（展开→SVD→截断→重排）；
- `run_checks`：执行可验收断言。

没有调用第三方张量库 API，流程完全源码可见。

## R13

验收标准（当前目录）：
- `uv run python demo.py` 可直接执行；
- 输出应包含 `estimated_ranks`、误差、压缩比、局部截断能量；

## R14

常见失效模式：
- `max_rank` 过小：欠拟合，误差偏高；
- `rel_tol` 设得过松：过度截断导致信息丢失；
- 数据无明显低秩结构：TT 压缩效果有限；
- 模式顺序不佳：中间秩飙升，参数和计算都变大。

调参时通常先固定 `max_rank`，再扫描 `rel_tol` 观察误差-压缩折中。

## R15

与相邻方法对比：
- CP：参数更少但优化更敏感，且高阶时可能收敛慢；
- Tucker：表达灵活但核心张量可能仍偏大；
- TT：通过链式结构把高阶问题拆成连续低秩矩阵问题，扩展性更好。

因此 TT 常用于“阶数高、每维中等、需压缩”的任务。

## R16

本 MVP 的实验设计：
- 形状 `shape=(10,9,8,7)` 的 4 阶张量；
- 真值秩 `true_ranks=(1,3,4,3,1)`；
- 加入 `noise_std=0.002` 高斯噪声形成观测；
- 用 `max_rank=4`、`rel_tol=1e-8` 分解。

这样可以同时观察重建质量与压缩率。

## R18

`demo.py` 源码级算法流程拆解（8 步）：
1. `TTConfig` 校验形状、边界秩、`max_rank` 与噪声参数合法性。  
2. `make_synthetic_tensor` 调用 `build_random_tt_cores` 生成真值 TT 核，重建出干净张量并加噪声。  
3. `tt_svd` 读取观测张量范数，把全局 `rel_tol` 按 `d-1` 步分配为局部截断能量预算。  
4. 对每个模式执行“展开矩阵 -> `np.linalg.svd` -> `choose_rank_from_singular_values` 选秩”。  
5. 将截断后的 `U` reshape 为当前核心 `G_k`，并把 `S*V^T` 传递给下一步。  
6. 最后一步把剩余工作矩阵 reshape 为尾核心，得到完整 `cores` 与 `estimated_ranks`。  
7. `reconstruct_tt` 用 `np.tensordot` 逐核收缩得到 `x_hat`，计算相对误差与压缩比。  
8. `run_checks` 验证边界秩、误差阈值、参数压缩与数值有效性，全部通过后打印成功信息。  

说明：实现未调用任何现成 TT 黑盒分解函数，TT-SVD 的每个步骤都在源码中显式展开。
