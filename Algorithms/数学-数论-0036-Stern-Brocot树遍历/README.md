# Stern-Brocot树遍历

- UID: `MATH-0036`
- 学科: `数学`
- 分类: `数论`
- 源序号: `36`
- 目标目录: `Algorithms/数学-数论-0036-Stern-Brocot树遍历`

## R01

Stern-Brocot 树是一棵二叉搜索树，节点是所有正有理数（且每个既约分数只出现一次）。  
根节点为 `1/1`，任意节点的左右子树通过“中项（mediant）”规则构造：  
若左右边界分数为 `a/b` 与 `c/d`，则中项为 `(a+c)/(b+d)`。

## R02

本条目“遍历”关注两类核心操作：
1. 有限深度中序遍历：按从小到大的顺序枚举 Stern-Brocot 树中的分数。
2. 路径遍历（编码/解码）：在树上用 `L/R` 序列表示某个目标分数的位置，并可反向还原。

## R03

MVP 输入输出约定（脚本内固定样例，无交互）：
- 输入：
  - 固定最大深度 `max_depth=4` 用于中序遍历演示；
  - 固定若干目标分数（如 `5/2`、`13/8`、`7/5`）用于路径往返验证。
- 输出：
  - 遍历得到的分数列表（含路径与深度）；
  - 序列是否严格递增的检查结果；
  - 每个目标分数的路径编码与解码回写结果。

## R04

关键数学性质：
- 邻分数不变量：若 `a/b < c/d` 且 `bc-ad=1`，则它们的中项 `(a+c)/(b+d)` 位于两者之间。
- 有序性：中序遍历 Stern-Brocot 树可得到严格递增的有理数序列。
- 唯一性：任意正既约分数在树中有且仅有一个节点，因此路径编码唯一。

## R05

前置条件与约束：
- 目标分数必须满足 `numerator > 0`、`denominator > 0`；
- 路径编码函数要求目标是既约分数（内部会用 `gcd` 检查）；
- `max_depth >= 0`；
- MVP 使用整数交叉乘法比较大小，避免浮点误差。

## R06

算法高层流程：
1. 以哨兵边界 `0/1` 与 `1/0` 初始化 Stern-Brocot 搜索区间。
2. 路径编码时反复取中项并与目标分数比较：
   - 目标更小则走 `L`，更新右边界；
   - 目标更大则走 `R`，更新左边界；
   - 相等则结束。
3. 路径解码时按 `L/R` 重放边界更新，最终中项即目标分数。
4. 中序遍历在每层递归中执行“左子树 -> 当前中项 -> 右子树”。

## R07

正确性直觉：
- 路径编码本质是对有理数进行二分定位；中项始终位于当前边界内，区间单调收缩，最终命中目标既约分数。
- 路径解码与编码使用同一套边界更新规则，故具有可逆性（round-trip）。
- 中序遍历在二叉搜索树上天然保持有序，因此输出应严格递增。

## R08

时间复杂度：
- 路径编码 `path_to_fraction(n, d)`：`O(k)`，`k` 为路径长度（通常与连分数展开长度同阶）。
- 路径解码 `fraction_from_path(path)`：`O(k)`。
- 有限深度中序遍历：访问节点数为 `2^(h+1)-1`（`h=max_depth`），时间 `O(2^h)`。

## R09

空间复杂度：
- 路径编码/解码：`O(1)` 额外空间（不计输出路径字符串）。
- 中序遍历：
  - 结果列表 `O(2^h)`；
  - 递归栈 `O(h)`。

## R12

输出字段解释：
- `max_depth`：演示遍历深度；
- `node_count`：遍历得到的节点数；
- `strictly_increasing`：中序遍历分数是否严格递增；
- `path`：目标分数在 Stern-Brocot 树中的 `L/R` 编码；
- `decoded`：由 `path` 反解得到的分数，应与原目标一致。

## R13

边界与鲁棒性处理：
- `path_to_fraction` 对非正分数、非既约分数直接抛出 `ValueError`；
- `fraction_from_path` 若出现非法字符（非 `L`/`R`）抛出 `ValueError`；
- 为防实现错误导致死循环，路径搜索设有 `max_steps` 上限保护；
- `max_depth < 0` 时遍历函数抛出异常。

## R14

局限性：
- 本 MVP 演示的是“有限深度遍历”，并不尝试无限树的完整枚举；
- 输出以终端文本为主，没有可视化树结构；
- 未实现“按分母上界枚举所有有理数”的专门优化版本（例如 Farey 邻接迭代）。

## R15

可扩展方向：
- 增加 BFS 层序遍历，对比 DFS 中序结果；
- 增加路径压缩表示（如 `L^a R^b ...`）并与连分数系数互转；
- 增加按区间 `[x, y]` 或按分母上界 `Q` 的裁剪遍历；
- 输出 Graphviz `.dot` 文件用于树可视化。

## R16

最小测试建议：
- 基础往返：`1/1 -> "" -> 1/1`；
- 典型样例：`5/2`、`13/8`、`7/5` 的路径编码与解码一致；
- 有序性测试：中序遍历输出满足相邻项严格递增；
- 规模测试：`max_depth=4` 时节点数应为 `31`。

## R17

与替代方案对比：
- 与“连分数直接展开”相比：Stern-Brocot 更直观地展示二叉搜索与有理数定位过程；
- 与 Calkin-Wilf 树相比：两者都覆盖全部正有理数，但 Stern-Brocot 的中序遍历天然对应大小顺序；
- 对教学场景，Stern-Brocot 在“路径语义 + 有序遍历”上更易解释。

## R18

`demo.py` 源码级算法流程（8 步）：
1. `main()` 固定 `max_depth=4`，调用 `inorder_traversal(max_depth)` 生成节点列表。
2. `inorder_traversal` 以哨兵边界 `(0/1, 1/0)` 启动递归；每次先算中项，再按“左-根-右”顺序访问。
3. 每个访问到的中项封装成 `SBNode(numerator, denominator, path, depth)` 并加入结果。
4. `is_strictly_increasing` 通过交叉乘法检查遍历结果是否严格单调上升。
5. `run_roundtrip_cases` 遍历固定分数集合，逐个调用 `path_to_fraction` 求 `L/R` 路径。
6. `path_to_fraction` 循环计算当前中项，比较目标分数与中项大小，决定走 `L` 或 `R` 并更新边界，直到命中。
7. 对每条路径调用 `fraction_from_path`：重放同样的边界更新，最后中项即反解分数。
8. `main()` 打印节点摘要、单调性检查和 round-trip 结果；若任何回写不一致则抛出异常。
