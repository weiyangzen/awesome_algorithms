## R01

Jenkins-Traub 算法用于求解单变量多项式

`P(x) = a_0 x^n + a_1 x^(n-1) + ... + a_n`（`a_0 != 0`）

的全部（一般为复数）根。它是典型的“逐根提取 + 降阶”策略，核心是三阶段迭代：

1. 无移位阶段（no-shift）: 构造辅助多项式 `K(x)` 的初始形态。
2. 固定移位阶段（fixed-shift）: 在固定复移位 `s` 下迭代更新 `K(x)`，把迭代带入目标根盆地。
3. 可变移位阶段（variable-shift）: 联合 `P(s)` 与 `K(s)` 做 Newton-like 修正，收敛到一个根。

找到一个根后，用合成除法 deflation，继续求解剩余低一阶多项式。

## R02

- 提出者: M. A. Jenkins 与 J. F. Traub。
- 主要形成期: 1960 年代末到 1970 年代初。
- 历史定位: 在“高阶实系数多项式数值求根”领域长期作为工业级稳健方案之一，经典实现常见于 `rpoly` 系列代码。

## R03

历史动机是“高阶多项式根求解”的工程刚需：

- 闭式公式只适用于低阶（2/3/4 次）且数值上不稳健。
- 直接 Newton 依赖初值，容易发散或陷入坏盆地。
- 伴随矩阵特征值法虽然通用，但在某些系数组合下会受缩放和条件数影响。

Jenkins-Traub 通过“阶段化移位 + 辅助多项式”提升了从任意初始方向进入收敛区的概率与稳健性。

## R04

设多项式次数为 `n`，每次迭代评估与更新成本约 `O(n)`。

- 若每个根平均需要 `m` 次 variable-shift，逐根 deflation 后总成本通常可写为 `O(m * n^2)`。
- 在难例（重根、近重根、强聚簇）中，重启和补救迭代会增加常数项，实践中可能接近更高代价。
- 与伴随矩阵 `O(n^3)` 本征求解相比，Jenkins-Traub 在中等规模常有较好常数表现。

## R05

空间复杂度主要由多项式系数与中间向量构成。

- 工作内存规模约 `O(n)`。
- 逐根 deflation 后问题规模变小，内存随次数下降。

## R06

例: `P(x) = x^3 - 6x^2 + 11x - 6`。

- 根为 `1, 2, 3`（实根）。
- Jenkins-Traub 的一次运行流程是：
  1. 用移位阶段找到一个根（例如 `x=3`）。
  2. 把 `P(x)` 降阶为二次多项式。
  3. 再求剩余两个根。

## R07

算法意义：

- 理论层: 把“局部收敛迭代”组织成“可工程化的全局流程”。
- 工程层: 对复根、共轭根与多尺度系数具备较高实用鲁棒性。
- 生态层: 许多数值库历史实现都受到 Jenkins-Traub / RPOLY 思想影响。

## R08

前置知识与依赖组件：

- Horner 多项式求值。
- 合成除法（按 `(x-r)` deflation）。
- Cauchy 根界（用于初始移位半径）。
- 复数迭代与收敛判据（残差与步长双阈值）。
- 基本数值防护（缩放、退化分支、重启）。

## R09

适用条件与边界：

- 适用: 实系数或复系数的一元多项式全部根求解。
- 不适用: 非多项式方程（如 `exp(x)-x=0`）不能直接套用。
- 边界风险:
  - 系数跨度极大时需做缩放。
  - 重根或近重根会显著降低收敛速度并放大 deflation 误差。

## R10

正确性可分层理解：

1. 若 variable-shift 阶段收敛到某个简单根 `r`，则 `P(r)=0`。
2. 通过合成除法把 `P(x)` 分解为 `(x-r)Q(x) + rem`，`rem` 足够小即视为数值可接受 deflation。
3. 对 `Q(x)` 递归执行同样流程，最终得到全部根的近似。

因此它是“局部收敛机制 + 全局重启策略”的算法，而不是“单次迭代全局必收敛”的方法。

## R11

数值稳定性要点：

- 优点: 三阶段机制通常比裸 Newton 更稳健，尤其在初值质量一般时。
- 风险: deflation 会把先前根误差传递到后续子问题。
- 难点: 近重根会导致 `K(s)` 或导数项接近零，修正步可能异常放大。
- 工程实践: 常配合重启、根 polishing、残差复核。

## R12

真实运行成本主要受以下因素控制：

- 多项式求值次数（每轮至少需要 `P(s)`、`K(s)`）。
- 合成除法频率（每提取一个根一次）。
- 重启次数（不同移位角度尝试）。
- 系数条件数与根分布（聚簇根会显著拖慢）。

在 Python/Numpy 环境中，常数项由复数算术与数组分配开销主导。

## R13

保证类型说明：

- 不是随机化算法，一般不提供概率成功界。
- 对任意输入也不存在“统一有限步全局收敛定理”可直接工程套用。
- 常见保证是经验层面的稳健性: 对大量实务多项式表现良好。

## R14

鲁棒性与常见失效模式：

- `K(s)` 过小: 步长 `P(s)/K(s)` 失真，可能振荡。
- 初始移位不佳: 固定移位阶段无法把迭代带入收敛区。
- 重根/近重根: 收敛阶下降，迭代次数显著增加。
- deflation 残差偏大: 后续根整体精度被污染。

## R15

工程实现建议：

- 系数先归一化（首项归一）。
- 使用 Cauchy 界给移位半径，角度多起点重试。
- 采用“残差 + 步长”联合停止条件。
- 每次 deflation 后立即检查余项与当前根残差。
- 用独立方法交叉验证（本目录 demo 用 `numpy.roots` 仅做验算，不参与主求解流程）。

## R16

前驱/后继关系：

- 前驱方法: Newton、Graeffe、Bairstow、Durand-Kerner 等。
- 同代常见路线: Laguerre 法、伴随矩阵特征值法。
- 后继工程实践:
  - 高性能库常结合多种策略（缩放、分块、迭代切换）。
  - 验证层面常加入根 polishing 与 backward error 检查。

## R18

源码级流程拆解（对应 `demo.py`，8 步）：

1. `jenkins_traub_roots` 清洗输入系数并转为复数数组，进入逐根循环。
2. 调用 `cauchy_root_bound` 估算根界半径 `R`，并由不同角度生成复移位初值。
3. 在 `find_one_root_jenkins_traub_style` 中构造初始辅助多项式 `K = P'`（无移位阶段）。
4. 对固定移位 `s` 执行若干次 `jt_k_step`（固定移位阶段），把迭代推进到可收敛区域。
5. 进入可变移位阶段: 反复计算 `delta = P(s)/K(s)`，更新 `s <- s - delta`。
6. 每轮后再次调用 `jt_k_step` 更新 `K`，并用 `|P(s)|` 与 `|delta|` 双条件判断收敛。
7. 收敛后用 `synthetic_division` 按 `(x-s)` 对原多项式 deflation，得到下一阶子问题。
8. 所有根求完后，`evaluate_residuals` 与 `match_roots_error` 给出残差和与 `numpy.roots` 的对照误差，用于结果验算。

说明: `numpy.roots` 在本实现中只用于验证，不参与主求根迭代逻辑。
