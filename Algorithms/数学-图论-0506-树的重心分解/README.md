## R01

树的重心分解（Centroid Decomposition）是一种把树递归拆成“重心树”的方法。

- 树的重心定义：删除该点后，所有连通块大小都不超过原树节点数的一半。
- 每一层选一个重心作为分治中心，把剩余子树递归分解。
- 由此得到一棵新的“重心树”（每个原树节点在重心树中仍唯一出现一次）。

本条目实现的 MVP 目标：

- 构建重心树（给出每个点在重心树中的父节点与层级）；
- 支持经典应用：动态点染色（标记节点）后查询任意点到最近已标记点的距离；
- 保证 `python3 demo.py` 可直接运行并输出结果。

## R02

背景与脉络（工程视角）：

- 树分治思想长期存在于图论与算法设计中；
- 重心分解在竞赛与工程中常用于把“树上路径/距离查询”转为“沿重心祖先链聚合”；
- 常与 LCA、树链剖分、DSU on Tree 形成互补：
  - 重心分解擅长“全局动态点集 + 最近距离/统计”；
  - 实现复杂度适中，常数较小，便于落地。

## R03

动机：很多树上动态问题若直接暴力更新/查询会退化为 `O(n)` 每次。

典型例子：

- 频繁把某些节点激活（或染红）；
- 查询任意节点到最近激活节点的距离。

若每次查询都 BFS/DFS，总代价高。重心分解可将单次更新与查询都压到 `O(log n)`，在大规模操作序列中收益显著。

## R04

复杂度（`n` 为节点数，`q` 为操作数）：

- 构建重心分解：
  - 每层递归做子树大小统计与找重心；
  - 总时间复杂度 `O(n log n)`（均摊视角）。
- 最近激活点应用：
  - `update(x)`：沿 `x` 的重心祖先链更新，`O(log n)`；
  - `query(x)`：沿同一祖先链取最小值，`O(log n)`。
- 总运行：`O(n log n + q log n)`。

## R05

空间复杂度：

- 原树邻接表：`O(n)`（树边数 `n-1`）；
- 重心分解辅助数组：`size/removed/parent/level` 等 `O(n)`；
- 每个点保存到重心祖先的距离链，长度至多 `O(log n)`，总计 `O(n log n)`；
- 维护每个重心的当前最优激活距离 `best`：`O(n)`。

总体空间复杂度 `O(n log n)`。

## R06

最小示例（无权树）：

- 边：`0-1, 1-2, 1-3, 3-4, 3-5`
- 操作序列：
  1. `update(2)`（激活节点 2）
  2. `query(5)` 返回 `3`（路径 `5-3-1-2`）
  3. `update(4)`
  4. `query(5)` 返回 `2`（最近变为节点 4，路径 `5-3-4`）

该示例对应 `demo.py` 的固定测试。

## R07

算法意义：

- 理论层：将“树上全局问题”转为“分治层级上的局部聚合”；
- 工程层：适合高频更新/查询的在线服务；
- 实战层：常用于最近点、路径计数、距离统计、k 距离对数等树题。

它是树算法中“结构重写”思想的代表方法之一。

## R08

核心思想（本实现）：

1. 递归找重心，构造重心树父子关系。
2. 对每个原树节点 `u`，记录它到所有重心祖先 `c` 的距离 `dist(u,c)`。
3. 维护数组 `best[c]`：当前所有激活点到重心 `c` 的最小距离。
4. 更新 `u`：
   - 枚举 `u` 的重心祖先 `c`，执行 `best[c] = min(best[c], dist(u,c))`。
5. 查询 `u`：
   - 枚举同一祖先链，取 `min(best[c] + dist(u,c))`。

这样就把“全树搜索”替换为“重心祖先链聚合”。

## R09

适用范围与边界：

- 适用：
  - 静态树结构（边不变）；
  - 大量点更新 + 距离类查询；
  - 无权树（可扩展到正权树）。
- 不适用：
  - 图不是树（有环或不连通）；
  - 边频繁动态增删（需更复杂动态图结构）；
  - 需要维护非常复杂的路径函数且无法在线性可加聚合。

本 MVP 输入要求节点编号为 `[0, n-1]` 且边数恰为 `n-1`。

## R10

正确性直觉：

1. 分解正确性  
每次选到的重心把当前子树拆成若干不超过一半大小的部分，递归终止后每个节点恰好在某层被选为重心，因此重心树构建完整且无重复。

2. 更新正确性  
任一激活点 `r` 会把它到每个重心祖先 `c` 的距离写入 `best[c]` 的最小值，因此 `best[c]` 始终表示“已激活点到 `c` 的最短距离”。

3. 查询正确性  
对查询点 `u`，任取激活点 `r`，路径可经由它们在重心树链上的公共结构表示。枚举 `u` 的重心祖先 `c` 时，`best[c] + dist(u,c)` 覆盖了候选值，取最小即得到 `dist(u,r)` 的全局最小值。

因此 `query(u)` 返回最近激活点距离。

## R11

稳定性与实现注意：

- 本算法是整数距离运算，不涉及浮点数值误差；
- 主要风险在实现细节：
  - 重心查找时遗漏 `removed` 过滤；
  - 递归分治时对子树入口处理错误；
  - `dist_to_centroids` 未完整记录导致查询漏候选。
- `demo.py` 中加入固定样例与随机对拍，覆盖上述常见错误。

## R12

实际性能关键因素：

- 树形结构：链状/星状会影响常数，但层数仍受 `O(log n)` 约束；
- 操作分布：更新与查询占比不同会影响缓存命中与常数；
- Python 实现细节：列表/递归开销、邻接遍历局部性。

通常在 `q >> n` 的场景，重心分解相对暴力法会明显提速。

## R13

保证类型：

- 确定性算法，无随机误差；
- 在输入合法（确为树）前提下，查询结果与朴素最短路定义一致；
- 复杂度保证为上界型：构建 `O(n log n)`，操作 `O(log n)`。

本目录 demo 额外通过随机用例与朴素解交叉验证正确性。

## R14

常见失效模式：

- 输入不是树（边数错误或不连通），会破坏分解前提；
- 更新或查询节点越界；
- 构建时未剔除已选重心，导致无限递归或父子关系错误；
- 只记录重心父链但未记录对应距离，查询无法还原答案。

本实现对输入做显式校验，并在异常时抛出 `ValueError`。

## R17

`demo.py` 的 MVP 结构：

- `CentroidDecomposition`：
  - 构建重心树（`parent`, `level`）；
  - 预处理每个点到重心祖先的距离链；
  - 提供 `activate(node)` 与 `query_nearest(node)`。
- `TreeDistanceOracle`：
  - 用 BFS 预计算全点对距离；
  - 作为朴素正确性基线。
- `run_fixed_demo()`：
  - 打印一组可读示例结果。
- `run_random_crosscheck()`：
  - 生成随机树与随机操作；
  - 对比重心分解与朴素解，自动断言一致。

运行命令：

```bash
python3 demo.py
```

## R18

源码级流程拆解（对应 `demo.py`，9 步）：

1. `CentroidDecomposition.__init__` 校验输入为树，构建邻接表并初始化 `parent/level/removed/sub_size` 等数组。
2. 调用 `_decompose(entry, parent, depth)` 递归构建重心树：先 `_compute_subtree_sizes`，再 `_find_centroid` 得到当前子树重心。
3. 在当前重心 `c` 处执行 `_collect_distances(c, -1, 0, c)`，把该连通块内每个节点到 `c` 的距离追加到 `dist_to_centroids[node]`。
4. 标记 `removed[c] = True`，对每个未移除子树递归 `_decompose`，形成完整重心树层级。
5. 运行 `activate(x)` 时，遍历 `dist_to_centroids[x]` 中所有 `(c, dist_xc)`，执行 `best[c] = min(best[c], dist_xc)`。
6. 运行 `query_nearest(x)` 时，同样遍历祖先链，计算 `best[c] + dist_xc` 的最小值，若都不可达则返回 `None`。
7. `run_fixed_demo` 构建固定树与操作序列，直观展示更新后查询答案变化。
8. `run_random_crosscheck` 生成随机树与随机激活/查询，用 `TreeDistanceOracle` 的朴素答案对照每次查询结果。
9. `main` 依次执行固定示例与随机对拍；若断言全部通过，打印成功信息并退出。
