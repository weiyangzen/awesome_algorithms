## R01

双重差分（Difference-in-Differences, DiD）用于估计政策或干预的因果效应，核心思想是比较“处理组与对照组在政策前后的变化差异”。
在满足平行趋势等识别假设时，DiD 可把共同时间冲击和组间固定差异相互抵消，得到处理效应（ATT）的可识别估计。

## R02

典型适用场景：
- 一部分个体/地区/企业在某时间点后受到政策影响，另一部分未受影响。
- 存在政策前后观测数据（至少一个 pre 与一个 post）。
- 不能做随机对照试验，但具备可比对照组。

本目录 MVP 使用平衡面板模拟数据，演示最经典 2x2 DiD 结构。

## R03

问题形式化：
- 输入：面板数据 `D={(i,t,y_it,T_i,Post_t)}`
  - `T_i∈{0,1}` 表示个体是否属于处理组。
  - `Post_t∈{0,1}` 表示时间是否在政策后。
  - `y_it` 为结果变量。
- 输出：
  - `ATT` 的点估计（`treated:post` 系数）。
  - 标准误、t 统计量、p 值（本实现使用 cluster-robust SE）。

## R04

2x2 DiD 均值公式：

`ATT = (E[Y|T=1,Post=1]-E[Y|T=1,Post=0]) - (E[Y|T=0,Post=1]-E[Y|T=0,Post=0])`

等价回归形式：

`y_it = β0 + β1*T_i + β2*Post_t + β3*(T_i*Post_t) + ε_it`

其中 `β3` 即 DiD 估计量（ATT）。

## R05

关键识别假设：
- 平行趋势：若无处理，处理组与对照组结果趋势应平行。
- 无预期/无提前反应：处理前不应提前响应政策。
- SUTVA/无外溢：一组处理不应系统性影响另一组结果。
- 组别构成稳定：样本选择变化不应与政策共同驱动估计偏差。

## R06

`demo.py` 数据生成机制：
- `n_units=400`，`n_periods=4`，`t>=2` 为 post。
- 一半单位随机进入处理组。
- 真实效应设为 `true_tau=2.0`，仅在 `treated=1` 且 `post=1` 时生效。
- 添加个体固定异质性与随机噪声（含轻微异方差）。

该构造保证“期望意义下平行趋势成立”，便于验证 DiD 回归可回收真值。

## R07

MVP 估计流程：
1. 生成平衡面板数据。
2. 按 `(treated, post)` 计算 2x2 组均值，得到闭式 DiD。
3. 构造设计矩阵 `X=[1, treated, post, treated*post]`。
4. 用 OLS 估计 `β=(X'X)^(-1)X'Y`。
5. 以 `unit` 为聚类计算 cluster-robust 协方差与显著性。
6. 输出 `treated:post` 作为 ATT 并与真值对比。

## R08

统计推断采用 CR1 聚类稳健标准误（按 `unit` 聚类）：

`V = c*(X'X)^(-1) [Σ_g (X_g' u_g)(X_g' u_g)'] (X'X)^(-1)`

其中 `c=(G/(G-1))*((N-1)/(N-k))` 为有限样本修正，`G` 为簇数量。
该方法可缓解面板同一单位内误差相关导致的标准误低估问题。

## R09

复杂度（`N` 行样本，`k` 个回归变量，`G` 个簇）：
- OLS 主计算：`O(N*k^2 + k^3)`。
- 聚类稳健协方差：`O(N*k + G*k^2)`。
- 本 MVP 中 `k=4`，因此总体近似线性随样本规模增长。

## R10

结果解释口径：
- `treated:post > 0`：政策使处理组结果相对对照组上升。
- `treated:post < 0`：政策使处理组结果相对下降。
- 系数量级即单位处理效应（在模型设定下可解释为 ATT）。
- 显著性需要结合 `p_value` 与识别假设共同判断，不能只看 p 值。

## R11

常见误区与风险：
- 仅有一个 pre 时很难可信检验平行趋势。
- 政策分期实施（staggered adoption）下，简单 TWFE 可能产生加权偏差。
- 忽略聚类相关性会低估标准误，夸大显著性。
- 存在处理外溢、样本构成变化或并行政策时，ATT 解释会失真。

## R12

可扩展方向：
- 双向固定效应（单位 FE + 时间 FE）。
- 事件研究（Event Study）检验动态效应与政策前趋势。
- 倾向得分加权/匹配 + DiD（改善组间可比性）。
- 合成控制或矩阵补全等方法做稳健性对照。

## R13

与相邻因果方法比较：
- RCT：识别最强，但很多政策场景不可行。
- RD：依赖阈值与局部连续性，外推有限。
- IV：解决内生性但需强工具变量假设。
- DiD：实现成本低、可解释性高，但高度依赖平行趋势与设计质量。

## R14

本 MVP 的工程取舍：
- 不依赖高阶计量库，核心估计与协方差计算全部显式实现，避免黑箱。
- 使用可控模拟数据，便于验证估计量与真值一致性。
- 输出包括闭式 DiD、回归 DiD、placebo 预趋势检验，形成最小可用验证闭环。

## R15

运行方式：

```bash
uv run python Algorithms/数学-计量经济学-0340-因果推断_-_双重差分/demo.py
```

预期行为：
- 打印样本规模、四格均值、ATT 点估计、回归表（含 cluster-robust SE）、以及 pre-period placebo 结果。

## R16

一次典型运行中应观察到：
- `treated:post` 接近 `true_tau=2.0`。
- 闭式 DiD 与回归 DiD 数值接近。
- placebo 项 `treated:fake_post` 接近 0（支持平行趋势构造）。

若偏离显著，常见原因是样本太小、噪声过大或并行趋势被破坏。

## R17

最小验证清单：
- 功能正确性：`demo.py` 可直接运行，无交互输入。
- 数值一致性：闭式 DiD 与回归 DiD 差异较小。
- 因果方向：估计系数方向与设定真值一致。
- 预趋势 sanity check：placebo 交互项应不显著或接近 0。

## R18

`demo.py` 的源代码级算法流程（8 步）：
1. `simulate_panel` 随机生成单位、处理分组、时间维度，并按 `y_it = alpha_i + gamma_t + tau*(treated*post) + eps` 合成面板。
2. `did_from_group_means` 对 `(treated, post)` 四格分组求均值，按 DiD 公式直接得到 ATT。
3. `build_design_matrix` 构造回归矩阵 `[1, treated, post, treated*post]`。
4. `ols_cluster_robust` 用正规方程计算 OLS 系数 `beta = (X'X)^(-1)X'Y`。
5. 在同一函数中计算残差 `u = y - X beta`，按簇累积 `score_g = X_g' u_g`。
6. 基于 `Σ_g score_g score_g'` 构造 CR1 聚类稳健协方差矩阵，并给出 `std_err`。
7. 用 t 分布（自由度 `G-1`）计算每个系数的 t 值和 p 值，提取 `treated:post` 作为回归 ATT。
8. `run_placebo_pretrend_test` 在 pre 样本上构造 `fake_post` 重复 DiD 回归，输出 placebo 交互项作为并行趋势的最小诊断。
