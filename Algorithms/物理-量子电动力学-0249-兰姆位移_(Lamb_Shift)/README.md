# 兰姆位移 (Lamb Shift)

- UID: `PHYS-0246`

## R01

兰姆位移（Lamb Shift）是指氢原子（及类氢原子）中，本应在狄拉克理论下几乎简并的能级（典型是 `2S_{1/2}` 与 `2P_{1/2}`）由于量子电动力学辐射修正而出现可观测分裂。  
它是 QED 真空涨落、电子自能和真空极化可实验验证的经典证据之一。

## R02

本条目要解决的问题是：

- 在最小可运行脚本里，用可审计的一圈近似给出兰姆位移量级；
- 计算 `1S, 2S, 2P_{1/2}, 2P_{3/2}` 的位移（MHz）；
- 给出 `2S_{1/2}-2P_{1/2}` 纯一圈预测，并拟合一个简单高阶余项使其对齐实验中心值。

MVP 的目标不是高精度现象学拟合，而是建立一个可复现、可解释、可扩展的算法闭环。

## R03

`demo.py` 输入输出约定（无交互）：

- 输入（代码内固定）：
1. 氢原子态集合：`1S1/2, 2S1/2, 2P1/2, 2P3/2`；
2. 对应 Bethe logarithm 常数；
3. 目标分裂：`2S1/2-2P1/2 = 1057.844 MHz`；
4. 有效高阶余项形式：仅对 S 态加入 `C_s / n^3`。
- 输出：
1. 纯一圈贡献表；
2. 拟合后贡献表；
3. 关键摘要（分裂、拟合常数、`1S/2S` 比值）；
4. 所有断言通过后打印 `All checks passed.`。

## R04

采用的近似模型（自然单位换算到频率）：

1. 一圈公共因子：
`prefactor = (alpha/pi) * (Z*alpha)^4 * (m_e c^2 / h) / n^3`。

2. 一圈自能：
`Delta nu_SE = prefactor * [A40 + (4/3) delta_{l0} ln((Z alpha)^-2)]`。

3. A40 系数（本 MVP 采用）：
- `S1/2`: `A40 = 10/9 - (4/3) ln k0(nS)`；
- `P1/2`: `A40 = -1/6 - (4/3) ln k0(nP)`；
- `P3/2`: `A40 = 1/12 - (4/3) ln k0(nP)`。

4. 一圈真空极化（简化）：
`Delta nu_VP(ns) ~ -(4/15) * prefactor`，`l != 0` 取 0。

5. 有效高阶余项（参数化）：
`Delta nu_rem = C_s / n^3`（仅 S 态），通过匹配 `2S1/2-2P1/2` 的实验值拟合 `C_s`。

## R05

复杂度分析：

- 单态位移计算：`O(1)`；
- 本条目总态数固定（4 个），整体计算量：`O(1)`；
- 一维根求解（Brent）迭代次数有限，成本近似 `O(K)`，`K` 通常很小；
- 空间复杂度：`O(1)`（输出表格规模固定）。

## R06

MVP 算法闭环：

1. 定义氢能级及 Bethe log 常数；
2. 按态计算一圈自能和真空极化；
3. 先给出无余项（`C_s=0`）的 `2S-2P1/2` 预测；
4. 构造目标函数 `f(C_s)=splitting(C_s)-target`；
5. 用有界根求解得到 `C_s`；
6. 生成纯一圈与拟合后两张贡献表；
7. 做量级与趋势断言；
8. 打印结果。

## R10

正确性直觉：

1. S 态在原点波函数非零，受短程辐射修正更强，位移显著大于 P 态；
2. 低主量子数态位移更大，近似遵循 `1/n^3` 缩放；
3. 纯一圈 `2S-2P1/2` 应在 GHz 量级；
4. 加入小幅正的有效余项后可把分裂从一圈值推到实验中心值附近。

## R11

数值稳定策略：

- 全部计算在 SI 常数基础上统一到 Hz/MHz，避免混合单位；
- 根求解使用有界 bracket + Brent 法，避免牛顿法初值敏感；
- 在求根前先检查 bracket 端点异号；
- 用多个断言约束结果范围，防止静默跑偏。

## R12

关键参数与影响：

- Bethe logarithm：直接影响 `A40`，改变一圈自能大小；
- `C_s`：只调 S 态高阶余项，主要影响 `2S-2P1/2`；
- 目标分裂值：改变拟合得到的 `C_s`；
- `Z`（当前默认 1）：通过 `(Z alpha)^4` 快速放大/缩小位移尺度。

调参建议：

- 教学演示可先固定 `Z=1`，只比较一圈与拟合后；
- 若拓展到类氢离子，先检查 Bethe log 与适用近似是否仍成立；
- 若追求精度，应把 `C_s` 替换为显式高阶项而非单参数吸收。

## R14

常见失效模式：

1. 误写单位（Hz、MHz、eV 混用）导致数量级错误；
2. `A40` 系数或 Bethe log 常数抄写错误；
3. 根求解 bracket 不覆盖根，导致拟合失败；
4. 把余项错误施加到 P 态，破坏物理结构；
5. 忽略模型边界，把该 MVP 当成高精度最终理论。

## R18

`demo.py` 的源码级算法流程（8 步）：

1. `LEVELS` 定义 `1S1/2, 2S1/2, 2P1/2, 2P3/2` 与 Bethe logarithm，固定状态输入。  
2. `one_loop_prefactor_hz` 计算 `(alpha/pi)(Z alpha)^4(m_ec^2/h)/n^3`，给出统一频率尺度。  
3. `a40_term` 按 `(l,j)` 分支写出 `A40`，随后 `one_loop_self_energy_hz` 组装自能主项与 S 态对数项。  
4. `one_loop_vacuum_polarization_hz` 只对 S 态加入 `-(4/15)*prefactor`，形成最小 VP 修正。  
5. `total_shift_hz` 把 `SE + VP + C_s/n^3`（S 态）求和，`splitting_mhz` 负责任意两态差值。  
6. `fit_effective_remainder_c_s_mhz` 显式构造目标函数 `f(C_s)=splitting-target`，再调用 `scipy.optimize.root_scalar`（Brent 有界求根）得到 `C_s`。  
7. `build_contribution_table` 逐态输出 `SE/VP/remainder/total`，分别生成纯一圈表和拟合后表。  
8. `main` 汇总关键分裂、打印结果，并用断言检查量级、`n^{-3}` 缩放与拟合精度，最后输出 `All checks passed.`。

说明：`scipy` 仅用于标准一维求根；物理模型、目标函数、贡献拆解和验证条件均在源码中显式实现，非黑盒调用。
