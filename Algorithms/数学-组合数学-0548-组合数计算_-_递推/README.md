# 组合数计算 - 递推

- UID: `MATH-0548`

## R01

本条目实现“组合数计算 - 递推”的最小可运行版本（MVP）：
- 核心目标是计算 `C(n, k)`；
- 核心方法是杨辉三角递推 `C(n,k)=C(n-1,k-1)+C(n-1,k)`；
- 实现强调可读、可验证、无黑盒依赖。

`demo.py` 会自动执行固定样例和恒等式校验，不需要任何交互输入。

## R02

问题定义：
- 输入：非负整数 `n` 与整数 `k`。
- 输出：组合数 `C(n, k)` 的精确整数值。

约定：
- 当 `0 <= k <= n` 时，`C(n, k)` 按标准定义计算；
- 当 `k < 0` 或 `k > n` 时，返回 `0`（与杨辉三角边界扩展一致）。

## R03

数学基础：

1. 边界条件：
   - `C(n, 0) = 1`
   - `C(n, n) = 1`
2. 递推关系（Pascal 公式）：
   - `C(n, k) = C(n-1, k-1) + C(n-1, k)`
3. 对称性：
   - `C(n, k) = C(n, n-k)`

`demo.py` 使用上述关系逐行构造到第 `n` 行，并取第 `k` 列得到答案。

## R04

算法流程（高层）：
1. 校验输入类型与取值范围（`n` 非负，`n/k` 为整数）。
2. 处理边界：`k<0 or k>n -> 0`，`k==0 or k==n -> 1`。
3. 使用对称性将 `k` 压缩到 `min(k, n-k)`，减少计算量。
4. 初始化一维数组 `dp[0..k]`，其中 `dp[0]=1`。
5. 外层循环 `i=1..n`，表示逐行推进杨辉三角。
6. 内层循环 `j=min(i,k)..1` 逆序更新：`dp[j]=dp[j]+dp[j-1]`。
7. 循环结束后 `dp[k]` 即 `C(n,k)`。

## R05

核心数据结构：
- `dp: list[int]`
  - 含义：当前行到当前位置的组合数；
  - 大小：`k+1`；
  - 更新顺序：必须从右向左，避免覆盖本行尚未使用的旧值。
- `cases: list[tuple[int, int]]`
  - `main` 中固定演示样例。

## R06

正确性要点：
- 初始时 `dp[0]=1` 对应 `C(i,0)=1` 的不变量。
- 第 `i` 行逆序更新后，`dp[j]` 等于 `C(i,j)`：
  - 旧 `dp[j]` 是上一行 `C(i-1,j)`；
  - 旧 `dp[j-1]` 是上一行 `C(i-1,j-1)`；
  - 相加即得到当前行 `C(i,j)`。
- 逆序保证“上一行数据”在被读取前不被污染。
- 最终得到 `dp[k]=C(n,k)`。

## R07

复杂度分析：
- 设 `k' = min(k, n-k)`。
- 时间复杂度：`O(n * k')`。
- 空间复杂度：`O(k')`。

相比构造完整杨辉三角的 `O(n^2)` 空间，该实现只保留一行状态，空间更小。

## R08

边界与异常处理：
- `n` 或 `k` 不是整数：抛出 `TypeError`。
- `n < 0`：抛出 `ValueError`。
- `k < 0` 或 `k > n`：返回 `0`。
- `k == 0` 或 `k == n`：直接返回 `1`。

## R09

MVP 取舍：
- 使用 Python 标准库实现，不引入额外依赖。
- 不走阶乘公式 `n!/(k!(n-k)!)`，避免中间超大阶乘对象的额外构造。
- 不调用第三方黑盒组合数 API 作为主计算逻辑。
- 仅做固定样例输出与恒等式校验，保持实现最小但可靠。

## R10

`demo.py` 函数职责：
- `_validate_n_k`：输入类型和范围校验。
- `comb_recurrence`：递推主算法，返回精确 `C(n,k)`。
- `_check_core_identities`：验证对称性、行和恒等式、与 `math.comb` 对照。
- `_print_case_table`：打印固定样例结果表。
- `main`：组织校验与样例演示。

## R13

内置测试覆盖：
- 小规模：`(5,2)`、`(10,3)`，便于人工核对。
- 中规模：`(20,10)`、`(52,5)`。

## R15

方法对比：
- 递推法（本实现）：
  - 优点：基于定义、易解释、易验证；
  - 缺点：单次查询复杂度高于某些专用公式。
- 阶乘法：
  - 写法短；
  - 但会构造超大阶乘中间值，教学上对递推结构体现较弱。
- 乘法连乘法（逐项约分）：
  - 在单点查询时通常更高效；
  - 但不如 Pascal 递推直观展示组合结构。

## R16

## R18

`demo.py` 源码级算法流程（8 步）：
1. `main` 先调用 `_check_core_identities(max_n=30)` 做小范围严谨校验。
2. `_check_core_identities` 对每个 `n`、`k` 调用 `comb_recurrence(n,k)`。
3. `comb_recurrence` 中 `_validate_n_k` 先检查类型与 `n>=0`。
4. 命中边界条件时直接返回 `0` 或 `1`，避免无效循环。
5. 使用对称性 `k=min(k,n-k)` 压缩状态维度。
6. 初始化 `dp[0]=1`，并按 `i=1..n` 逐行推进。
7. 每行执行 `j` 逆序更新 `dp[j]=dp[j]+dp[j-1]`，对应 Pascal 递推。
8. 返回 `dp[k]`；随后在 `_print_case_table` 中与 `math.comb` 对照并输出 `PASS/FAIL`。
