## R01

引力波是广义相对论中的时空曲率扰动，以光速传播。对算法实现而言，核心任务通常不是“求解完整爱因斯坦方程组”，而是基于可观测器件输出，完成以下流程：

1. 构造可计算的波形近似模型（模板）。
2. 在噪声背景中进行统计检测。
3. 从最优模板反推出关键参数（如 chirp mass、合并时刻）。

本目录 MVP 专注于这一可执行链路：`波形生成 -> 噪声建模 -> 匹配滤波 -> 模板库搜索 -> 参数回收`。

## R02

在 LIGO/Virgo/KAGRA 这类地基干涉仪数据处理中，匹配滤波（matched filtering）是双致密天体并合信号检测的基础工具。

直观原因：

- 仪器噪声是强随机背景，时域直接“看波形”很困难。
- 广义相对论给出的并合前波形具有确定结构（chirp：频率和振幅随时间上升）。
- 当已知信号形状近似时，匹配滤波在高斯噪声下是最优线性检测器。

因此，本算法属于“广义相对论预测 + 统计信号处理”交叉路径。

## R03

本 MVP 使用并合前（inspiral）阶段的牛顿阶近似（leading-order PN）：

- 频率演化：`f(t) = A * (tc - t)^(-3/8)`。
- 其中 `A = (1/pi) * (5/256)^(3/8) * (G*Mc/c^3)^(-5/8)`。
- `Mc` 为 chirp mass，`tc` 为合并时刻。

振幅采用受限近似：

`h(t) ~ amplitude * (f(t)/100Hz)^(2/3) * cos(phi(t))`

相位通过解析积分得到：

`phi(t) = phi_c - (16/5) * (pi*A) * (tc - t)^(5/8)`。

并在 `[f_low, f_high]` 频段内截取有效波形。

## R04

检测统计量采用频域 PSD 加权匹配滤波：

`z(tau) ~ IFFT[ d(f) * h*(f) / S_n(f) ]`

其中：

- `d(f)` 是观测数据频谱。
- `h(f)` 是模板频谱。
- `S_n(f)` 是噪声功率谱密度（PSD）。

该加权意味着：

- 噪声大的频段权重自动降低。
- 噪声小且模板含信息的频段贡献更大。

本实现将 `z(tau)` 再标准化为经验 `z-score`（零均值、单位方差），用于跨模板比较峰值显著性。

## R05

算法主流程（高层）：

1. 用 chirp mass 与 `tc` 生成目标引力波注入信号。
2. 依据解析 PSD 模型合成彩色高斯噪声。
3. 将信号叠加到噪声得到观测数据。
4. 构建 chirp mass 模板库（模板 `tc` 先固定为参考值）。
5. 对每个模板做匹配滤波并提取峰值 `peak_z` 与时间滞后 `lag`。
6. 取 `peak_z` 最大模板为候选事件。
7. 用 `estimated_tc = template_tc + lag` 回收合并时刻。
8. 对比注入真值并执行阈值化校验。

## R06

复杂度估计（`N` 为采样点，`K` 为模板数）：

- 单模板匹配滤波：两次 FFT + 一次 IFFT，量级 `O(N log N)`。
- 模板库扫描：`O(K * N log N)`。
- 波形生成：`O(K * N)`（通常次于 FFT 成本）。

故主导复杂度是 `K` 次 FFT 卷积。对于中小模板库，CPU 上即可快速完成。

## R07

物理与统计前提：

- 仅覆盖 inspiral 阶段，不含 merger/ringdown 精细结构。
- 波形使用牛顿阶近似，未包含高阶自旋、偏心率、潮汐项。
- 噪声假设为平稳高斯，真实探测器通常存在非高斯瞬态。
- 使用单探测器单通道，不含多台干涉仪联合一致性检验。

因此它是“教学级可验证 MVP”，不是生产级搜寻管线。

## R08

本目录示例配置：

- 采样率 `fs = 1024 Hz`
- 时长 `16 s`
- 频带 `[28, 350] Hz`
- 注入真值：`Mc = 1.22 Msun`, `tc = 12.7 s`
- 模板库：`Mc in {1.12, 1.16, 1.20, 1.22, 1.24, 1.28, 1.32}`
- 模板参考合并时刻：`template_tc = 11.0 s`

匹配滤波输出会自动给出 lag，用于把参考 `tc` 修正回真实事件时刻。

## R09

前置知识建议：

- 广义相对论中双星 inspiral 的最基本 chirp 关系。
- 傅里叶变换、功率谱密度、白化/加权内积。
- 高斯噪声下的检测统计与阈值解释。
- 数值计算中的窗函数和边界效应。

## R10

适用边界：

- 适用：已知信号族较明确、需要在噪声中检测弱信号的场景。
- 谨慎：参数空间很大但模板库很稀疏时，可能失配。
- 不适用：强非高斯/非平稳噪声且缺少额外抗伪迹机制时。

若要用于真实观测数据，需要加入数据质量标记、伪迹剔除与多探测器联合统计。

## R11

本实现的正确性检查点：

1. 最优模板的 `peak_z` 应明显高于噪声基线峰值。
2. 回收 `Mc` 与注入真值误差应在容忍阈值内。
3. 回收 `tc`（由 lag 修正）应接近注入 `tc`。
4. 最优模板与次优模板峰值应有足够对比度。
5. 有效频段内瞬时频率应保持有限值（避免数值发散）。

`demo.py` 中这些检查全部显式打印，并在失败时以非零退出码结束。

## R12

数值稳定性策略：

- 在匹配滤波前对数据与模板去均值并施加 Hann 窗，减少频谱泄漏。
- `inv_psd` 只在 `[f_low, f_high]` 有效频段启用，抑制无意义低/高频放大。
- 用 `EPS` 防止标准化时除零。
- 对 `tau = tc - t` 设下界 `1e-9`，避免 `(tc-t)^(-3/8)` 奇异点。

这些设计使 MVP 在固定随机种子下稳定可复现。

## R13

与相关方法对比：

- 对比时域互相关：PSD 加权匹配滤波能显式利用噪声频谱信息，统计效率更高。
- 对比黑箱分类器：模板方法可解释性更强，易审计物理含义。
- 对比完整贝叶斯参数估计：本法更轻量快速，但不提供完整后验分布。

所以它适合作为“第一层触发器”和教学原型。

## R14

常见失效模式与防护：

- 模板库过稀：真值位于模板间隙，导致峰值被压低。
  - 防护：加密参数网格或采用插值/随机模板库。
- 噪声突发（glitch）造成伪高峰。
  - 防护：加入卡方一致性检验与多探测器相干约束。
- PSD 模型与真实噪声偏离大。
  - 防护：改为分段估计实测 PSD 并动态更新。

## R15

调参建议：

- `f_low_hz`：过低会被低频噪声淹没，过高会丢失信号累积相位信息。
- `f_high_hz`：应兼顾采样率 Nyquist 与模板物理适用区间。
- 模板库密度：越密越准但计算量越高。
- 注入振幅 `amplitude`：决定演示信号显著度；本例设置为中等可检测强度。

## R16

工程化扩展方向：

- 用真实探测器数据替换合成噪声，并按时间段估计 PSD。
- 将单探测器统计扩展为多探测器联合 ranking statistic。
- 把牛顿阶模板升级为更高阶 PN 或 EOB/IMR 波形族。
- 在候选事件后接入 MCMC/VI 做精细参数后验估计。

## R18

`demo.py` 源码级算法流程拆解（9 步）：

1. `ChirpConfig` 固定采样率、时长和有效频带，统一全局数值网格。
2. `generate_newtonian_inspiral` 根据 `Mc, tc, phi_c` 计算 `f(t)`、`phi(t)` 并生成受限 chirp 波形。
3. `detector_psd_model` 给出解析 PSD；`generate_colored_gaussian_noise` 在频域按 PSD 采样复高斯，再 `irfft` 回时域噪声。
4. `main` 将注入信号叠加到噪声形成观测数据 `data`，并记录信号/噪声 RMS。
5. `run_template_bank` 对每个候选 chirp mass 生成模板波形，调用 `matched_filter_zscore` 计算匹配滤波时间序列。
6. `matched_filter_zscore` 执行去均值、Hann 窗、频域 PSD 加权与 IFFT，得到 `z(tau)`，再标准化为 `z-score`。
7. 从 `|z(tau)|` 最大点提取 `peak_z` 与 `lag`，并用 `estimated_tc = template_tc + lag` 回收事件时刻。
8. `to_dataframe` 汇总所有模板结果并按 `peak_z` 排序；同时对 noise-only 数据做同样扫描，形成基线。
9. `main` 计算 `Mc/tc` 误差、峰值对比度和基线超额，执行 5 个阈值检查并输出 `PASS/FAIL`。

该实现没有把核心流程交给黑箱库：波形公式、PSD 加权、模板扫描与判据都在源码中逐步展开。
