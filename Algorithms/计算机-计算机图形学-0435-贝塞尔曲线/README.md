## R01

贝塞尔曲线（Bezier Curve）是由一组控制点定义的参数曲线。对 `n` 次贝塞尔曲线：

`B(t) = sum_{i=0..n} C(n,i) (1-t)^(n-i) t^i P_i,  t in [0,1]`

其中 `P_i` 为控制点，`C(n,i)` 为二项式系数。曲线经过首末控制点 `P_0` 与 `P_n`，中间控制点决定形状而不一定落在曲线上。

## R02

在计算机图形学中，贝塞尔曲线是矢量绘图、字体轮廓、路径动画、CAD 曲线建模和 UI 曲线编辑的基础构件。它的优势是：

- 参数化形式统一，易于插值和采样；
- 具有凸包性质，几何行为可控；
- 可通过 de Casteljau 算法稳定求值并支持曲线细分。

## R03

本条目 MVP 聚焦三个最小但完整的目标：

- 实现 Bernstein 多项式形式的贝塞尔曲线求值；
- 实现 de Casteljau 递推求值，并与 Bernstein 结果逐点比对；
- 实现曲线分割（subdivision）并验证“分割后两段与原曲线一致”的几何正确性。

## R04

本实现涉及的核心公式：

1. Bernstein 基函数：`b_{i,n}(t)=C(n,i)t^i(1-t)^(n-i)`。
2. 曲线表示：`B(t)=sum_i b_{i,n}(t)P_i`。
3. de Casteljau 递推：`P_i^{(r)}(t)=(1-t)P_i^{(r-1)}(t)+tP_{i+1}^{(r-1)}(t)`，`P_i^{(0)}=P_i`。
4. 最终曲线点：`B(t)=P_0^{(n)}(t)`。
5. 导矢端点性质：`B'(0)=n(P_1-P_0)`，`B'(1)=n(P_n-P_{n-1})`。

## R05

设曲线次数为 `n`，采样点个数为 `m`：

- Bernstein 向量化求值时间复杂度约 `O(mn)`；
- de Casteljau 单点求值为 `O(n^2)`，`m` 点为 `O(mn^2)`；
- 曲线分割一次为 `O(n^2)`；
- 空间复杂度主要来自采样结果，为 `O(m + n)`。

在低次数曲线（常见 2~5 次）场景中，以上复杂度足够轻量。

## R10

正确性直觉：

1. 两种数学等价表示（Bernstein、de Casteljau）应给出同一曲线点；
2. `t=0/1` 时曲线应回到首末控制点；
3. de Casteljau 在任意 `t` 分割后，两个子曲线拼接应与原曲线一致；
4. 曲线折线近似长度不应短于首尾直线距离。

这些性质都能在 `demo.py` 中被直接量化验证。

## R11

数值稳定策略：

- Bernstein 与 de Casteljau 双实现交叉验证，避免单一路径隐藏错误；
- de Casteljau 使用逐层线性插值，数值上通常比直接高阶多项式更稳定；
- 断言阈值使用 `1e-12` 量级，兼顾浮点误差与严格性；
- 采用固定采样网格，确保每次运行结果可复现。

## R14

常见失效模式：

1. `t` 不在 `[0,1]` 时对“线段插值曲线”的语义会变化；
2. 控制点维度不一致或输入形状错误导致计算失败；
3. 高次数下直接 Bernstein 计算可能出现数值放大；
4. 采样过稀时，几何性质检查可能漏检局部异常。

## R18

`demo.py` 的源码级算法流程（8 步）：

1. `_as_control_points` 校验输入控制点矩阵形状并统一为浮点数组。  
2. `bernstein_basis` 按 `C(n,i)t^i(1-t)^(n-i)` 构造基函数矩阵。  
3. `bezier_bernstein` 通过矩阵乘法 `basis @ control_points` 得到采样曲线点。  
4. `de_casteljau_point` 逐层执行线性插值递推，得到单个参数 `t` 对应曲线点。  
5. `bezier_de_casteljau` 对一组 `t_values` 循环调用第 4 步，形成第二套曲线求值结果。  
6. `bezier_split` 复用 de Casteljau 递推中每层首末点，构造左右子曲线控制点。  
7. `main` 中对两套求值结果做误差比较，并验证端点、端点导矢、分割一致性、长度与包围盒性质。  
8. 若所有断言通过则输出 `All checks passed.`，形成非交互、可复现的最小验收闭环。  

这里 `numpy` 仅用于数组表示与线性代数运算；贝塞尔核心逻辑（基函数、递推、分割、验证）均在源码中显式实现，没有依赖黑盒图形 API。
