## R01

三线性插值（Trilinear Interpolation）用于在三维规则网格中，从离散体素值估计任意连续坐标点的标量值。

它可以看成“双线性插值在第三个维度上的扩展”，是体渲染、医学影像重采样、3D 纹理采样中的基础算子。

## R02

设体数据 `V` 的形状为 `(D, H, W)`，索引顺序为 `V[z, y, x]`。  
给定连续查询点 `p=(x, y, z)`（在网格索引坐标系中），先定位其所在网格单元：

- `x0=floor(x), x1=min(x0+1, W-1)`
- `y0=floor(y), y1=min(y0+1, H-1)`
- `z0=floor(z), z1=min(z0+1, D-1)`

这 8 个角点构成包围 `p` 的体素立方体。

## R03

定义局部小数偏移：

- `xd = x - x0`
- `yd = y - y0`
- `zd = z - z0`

对应 8 个角点值：

- `c000 = V[z0, y0, x0]`，`c100 = V[z0, y0, x1]`
- `c010 = V[z0, y1, x0]`，`c110 = V[z0, y1, x1]`
- `c001 = V[z1, y0, x0]`，`c101 = V[z1, y0, x1]`
- `c011 = V[z1, y1, x0]`，`c111 = V[z1, y1, x1]`

## R04

插值按 x -> y -> z 三层线性组合进行：

```text
c00 = c000*(1-xd) + c100*xd
c10 = c010*(1-xd) + c110*xd
c01 = c001*(1-xd) + c101*xd
c11 = c011*(1-xd) + c111*xd

c0  = c00*(1-yd) + c10*yd
c1  = c01*(1-yd) + c11*yd

c   = c0*(1-zd) + c1*zd
```

这与 8 角点加权求和完全等价，且权重和为 1。

## R05

本目录 MVP 目标：

1. 手写实现三线性插值（不依赖黑盒 API）；
2. 同时提供向量化版本和朴素循环版本；
3. 提供越界策略：`raise` 与 `clip`；
4. 用线性场验证数值正确性；
5. 打印可读的结果表与性能指标。

## R06

`demo.py` 核心函数：

- `_validate_volume` / `_validate_points`：输入维度和类型检查；
- `_handle_bounds`：统一处理越界点（报错或裁剪）；
- `trilinear_interpolate`：向量化批量插值；
- `trilinear_interpolate_naive`：朴素循环插值；
- `build_linear_volume` / `analytic_linear_field`：构造可精确验证的线性体场。

## R07

正确性不变量：

1. `volume.ndim == 3`；
2. `points.shape == (N, 3)` 且列顺序为 `(x, y, z)`；
3. 插值权重都在 `[0, 1]`，权重总和为 `1`；
4. 在线性场上，插值结果应与解析表达式一致（浮点误差量级）；
5. 向量化版本与朴素版本输出一致（同一输入、同一边界策略）。

## R08

复杂度分析（`N` 为查询点数量）：

- 向量化版本：时间 `O(N)`，空间 `O(N)`；
- 朴素循环版本：时间 `O(N)`，但 Python 循环常数大；
- 单点插值常数操作数固定（8 角点 + 7 次线性合并）。

因此在工程中通常优先使用向量化实现。

## R09

数值与工程注意点：

- 坐标靠近上边界时，`x1/y1/z1` 需钳到末索引；
- 浮点比较需用容差（`np.allclose`）；
- 连续坐标语义必须与数组轴顺序严格对应；
- 插值不是外推，越界处理应显式声明；
- 大规模点查询时，向量化可显著降低 Python 调度开销。

## R10

实验设置（`demo.py`）：

- 体数据形状：`(D,H,W)=(20,24,28)`；
- 线性场：`f(x,y,z)=ax+by+cz+d`（`a,b,c,d` 固定）；
- 查询点：`50,000` 个随机点（种子 `2026`，均在合法范围）；
- 性能对比：向量化 vs 朴素循环（前 `5,000` 个点）；
- 边界演示：构造越界点，分别测试 `clip` 与 `raise`。

## R12

输出指标含义：

- `vectorized_ms`：向量化插值耗时；
- `naive_ms_subset`：朴素循环在子集上的耗时；
- `naive_over_vectorized_speedup`：朴素/向量化耗时比；
- `max_abs_err_linear_field`：线性场最大绝对误差；
- `rmse_linear_field`：线性场均方根误差；
- `out_of_bound_points`：边界演示越界点数量。

## R13

脚本内置质量门禁：

1. 线性场插值误差必须低于阈值；
2. 向量化与朴素版本必须一致；
3. `mode='raise'` 下越界输入必须抛出异常；
4. 输入形状不合法时必须报出明确 `ValueError`。

## R14

边界与异常策略：

- `mode='raise'`：任一点越界即抛错，并返回部分坏点索引；
- `mode='clip'`：把越界坐标裁剪到合法区间再插值；
- 非法 `mode` 会抛 `ValueError`；
- 若体数据或点数组形状错误，会在入口处直接失败。

## R15

## R18

源码级算法链路（8 步）：

1. `main()` 调用 `build_linear_volume` 生成规则 3D 线性场 `V[z,y,x]`，并用随机数生成连续查询点 `queries`。  
2. `trilinear_interpolate` 入口先做 `_validate_volume` 和 `_validate_points`，确保输入分别为 3D 体与 `(N,3)` 点集。  
3. `_handle_bounds` 按 `mode` 处理越界：`raise` 直接报错，`clip` 将每个坐标钳制到 `[0, max_index]`。  
4. 对每个点分解整数索引与小数偏移：`x0/x1, y0/y1, z0/z1` 与 `xd/yd/zd`，其中 `x1/y1/z1` 通过 `min(..., size-1)` 保证边界安全。  
5. 用 NumPy 高级索引一次性提取 8 个角点数组 `c000..c111`，避免 Python 层逐点访存。  
6. 按 `x -> y -> z` 依次执行 7 次线性插值合并，得到每个查询点的最终估计值 `c`。  
7. `main()` 使用解析函数 `analytic_linear_field` 计算真值，对比 `max_abs_err` 与 `rmse`，并断言误差阈值通过。  
8. `main()` 再运行 `trilinear_interpolate_naive` 做一致性和耗时对比，最后打印样本表、指标表与越界行为说明，形成完整可验证闭环。
