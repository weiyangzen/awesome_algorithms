# N体模拟 (N-body Simulation)

- UID: `PHYS-0360`
- 学科: `物理`
- 分类: `计算宇宙学`
- 源序号: `378`
- 目标目录: `Algorithms/物理-计算宇宙学-0378-N体模拟_(N-body_Simulation)`

## R01

本条目实现一个可运行、可审计的最小 N 体模拟（N-body）MVP：
- 粒子间相互作用采用软化牛顿引力；
- 时间推进采用 leapfrog（kick-drift-kick）辛积分；
- 输出能量守恒、维里比和径向密度分布等关键诊断量；
- 不依赖大型黑箱框架，核心算法都在 `demo.py` 源码中显式展开。

## R02

问题定义：给定 `N` 个质量点的初始位置和速度，数值求解其自引力系统的时间演化。该问题在计算宇宙学中是大尺度结构形成研究的基础模块。

本 MVP 的目标不是复现大规模生产级宇宙学仿真，而是提供一条最小、完整、可复现的算法链路，便于验证数值积分行为与物理量趋势。

## R03

核心方程：
- 软化引力加速度
`a_i = G * Σ_{j!=i} m_j * (x_j - x_i) / (|x_j-x_i|^2 + eps^2)^(3/2)`
- 动能
`K = 1/2 * Σ_i m_i |v_i|^2`
- 势能
`U = -G * Σ_{i<j} m_i m_j / sqrt(|x_i-x_j|^2 + eps^2)`
- 哈密顿量（总能量）
`E = K + U`
- 维里诊断
`2K/|U|`

## R04

`demo.py` 输入/输出约定（无交互）：
- 输入：脚本内固定参数（`N`、步长 `dt`、步数、软化长度、随机种子等）；
- 输出：
1. 轨迹采样表（时间、动能、势能、总能量、能量漂移、维里比、`r_rms`）；
2. 终态诊断摘要（最大能量漂移、质心漂移等）；
3. 径向壳层密度表；
4. 用 `scikit-learn` 与 `PyTorch` 拟合的密度幂律斜率。

## R05

默认参数（`NBodyConfig`）：
- `n_particles = 64`
- `n_steps = 320`
- `dt = 0.01`
- `grav_const = 1.0`
- `softening = 0.05`
- `init_pos_sigma = 0.35`
- `init_vel_sigma = 1.0`
- `init_virial_ratio = 0.30`
- `random_seed = 7`

## R06

实现策略：
1. 随机生成初始位置/速度，并去除质心漂移；
2. 通过速度缩放把初始 `Q=K/|U|` 调整到目标值；
3. 用 `O(N^2)` 直接求和计算每步加速度；
4. 用 leapfrog（KDK）推进状态，记录能量与结构诊断；
5. 在终态上做径向分箱密度统计并拟合幂律斜率。

## R07

复杂度：
- 单步加速度/势能主导复杂度为 `O(N^2)`；
- 总时间复杂度约为 `O(n_steps * N^2)`；
- 额外统计（分箱与拟合）复杂度远小于积分主流程。

在默认配置下（`N=64, steps=320`）可在普通 CPU 上快速运行。

## R08

数值稳定性与鲁棒性设计：
- 势能与加速度统一使用软化长度 `eps`，避免近距离发散；
- 对角元距离设为 `inf`，显式消除自作用；
- 能量漂移按 `|E0|` 归一化并设置最小分母，避免除零；
- 壳层体积计算设置下限，避免极小 bin 导致数值异常；
- 径向分箱的最小半径设置下界，避免 `log(0)`。

## R09

边界与失败条件：
- 若粒子数太小（极端情况下有效壳层点不足），密度斜率拟合会返回 `NaN`；
- 若步长 `dt` 过大，能量漂移会明显增大；
- 该实现未加入周期边界和宇宙学膨胀项，不适用于直接做精确宇宙学观测拟合。

## R10

MVP 诚实范围：
- 是教学/算法基线，不是生产级 N 体引擎；
- 采用直接求和，未实现 Barnes-Hut / TreePM / FFT-PM 加速；
- 采用无量纲单位与简化初值，不等价于真实宇宙初始条件生成器。

## R12

输出读法：
- `rel_energy_drift`：越接近 0 说明积分能量守恒越好；
- `virial_2K_over_|U|`：长期可能向 1 附近演化，反映系统向维里化靠近；
- `r_rms`：衡量粒子群整体尺度（塌缩或膨胀趋势）；
- 径向密度表可用于判断结构集中程度。

## R14

结果解释建议：

## R15

当前局限：
- 无周期边界、无 comoving 坐标、无尺度因子 `a(t)`；
- 无软化长度自适应、无多时间步策略；
- 无并行加速（GPU/多进程）与大规模 I/O。

## R17

工具栈与职责：
- `numpy`：粒子状态张量、`O(N^2)` 向量化求和；
- `scipy`：`binned_statistic` 做径向壳层统计；
- `pandas`：轨迹和密度结果表格化输出；
- `scikit-learn`：线性回归拟合密度幂律斜率；
- `PyTorch`：自动微分拟合同一幂律模型并给出优化残差。

## R18

`demo.py` 源码级算法流程（8 步）：
1. `initialize_state` 生成随机初始位置/速度，去除质心运动，再通过 `Q=K/|U|` 速度缩放设定初始动力学状态。
2. `pairwise_geometry` 构造三维位移张量 `diff[i,j]=x_j-x_i` 与软化逆距离 `inv_r`, `inv_r3`，并把对角元设为 `inf` 去掉自作用。
3. `accelerations` 通过 `np.einsum` 显式执行 `Σ_j m_j diff / r^3`，得到每个粒子的总加速度。
4. `leapfrog_step` 执行 kick-drift-kick：先半步速度，再整步位置，再用新位置求加速度，最后再半步速度。
5. `run_simulation` 在每个时间步计算 `K/U/E`、能量漂移、维里比、`r_rms`、质心范数，并累计为 `pandas` 轨迹表。
6. `radial_density_profile` 用 `scipy.stats.binned_statistic` 对终态半径做壳层质量统计，再除以壳层体积得到 `rho_shell`。
7. `fit_density_slope_sklearn` 在 `log10(r)-log10(rho)` 空间调用 `LinearRegression.fit` 拟合幂律斜率；这是显式的一元线性模型，不是黑箱端到端预测。
8. `fit_density_slope_torch` 用 `alpha` 与 `logA` 两个可训练标量，最小化 `mean((logA+alpha*log(r)-log(rho))^2)`，通过 Adam 迭代得到斜率与 RMSE，并在 `main` 中统一打印诊断报告。
