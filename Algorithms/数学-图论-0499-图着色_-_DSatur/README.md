# 图着色 - DSatur

- UID: `MATH-0499`
- 学科: `数学`
- 分类: `图论`
- 源序号: `499`
- 目标目录: `Algorithms/数学-图论-0499-图着色_-_DSatur`

## R01

图着色（Vertex Coloring）问题：
给定无向图 `G=(V,E)`，为每个顶点分配一个颜色编号，使得每条边 `(u,v)∈E` 的两端颜色不同，即 `color[u] != color[v]`。

目标通常是最小化使用颜色数，最优值记作图的色数（Chromatic Number）`χ(G)`。

## R02

DSatur（Degree of Saturation）是经典着色启发式：
- 每轮先选择“饱和度”最高的未着色顶点；
- 饱和度指该顶点已着色邻居里出现的**不同颜色种类数**；
- 对选中的顶点，分配当前可用的最小颜色（smallest available color）。

它不是精确算法，但在很多图上质量很高。

## R03

问题定位：
- 最小图着色是 NP-hard；
- 大图上通常难以用多项式时间求最优；
- DSatur 在工程上常作为“高质量、低复杂度”的实用方案；
- 本目录 MVP 采用 DSatur 作为主算法，并加入小规模精确回溯基线作对照。

## R04

本实现中的顶点选择规则（确定性）：
1. 先比较饱和度，取最大；
2. 若并列，比较静态度数（原图度），取最大；
3. 若仍并列，取编号最小顶点。

这样做的目的是保证每次运行结果可复现，不依赖随机数。

## R05

颜色分配规则：
- 对当前顶点 `v`，收集其已着色邻居的颜色集合 `F(v)`；
- 从 `0` 开始找最小的不在 `F(v)` 中的颜色并赋给 `v`。

该规则保证局部合法性（不会与已着色邻居冲突），并倾向于压缩颜色总数。

## R06

复杂度（邻接矩阵 + 朴素候选扫描）：
- 顶点选择共进行 `n` 轮；
- 每轮扫描所有未着色顶点并处理邻接关系，最坏约 `O(n^2)`；
- 总体最坏时间复杂度可看作 `O(n^3)` 量级；
- 空间复杂度 `O(n^2)`（邻接矩阵占主导）。

这对 MVP 来说足够清晰、稳定。

## R07

正确性（可行解层面）要点：
- 每个顶点赋色时，显式避开已着色邻居颜色；
- 因此任意边两端颜色不同，得到合法着色。

`demo.py` 用 `is_proper_coloring` 对 DSatur 结果与精确基线结果都做断言检查。

## R08

为什么需要“饱和度优先”：
- 饱和度高的顶点说明其颜色选择空间更紧；
- 提前处理这类“受限顶点”通常能减少后续冲突，降低新增颜色概率；
- 这与回溯中的“最受约束变量优先”思想一致。

## R09

本 MVP 的精确对照：
- 实现 `exact_chromatic_number_backtracking`（DFS + 分支限界）；
- 仅用于小图（默认 `n <= 18`）；
- 目的不是替代 DSatur，而是用于质量评估：比较 `DSatur_色数` 与 `最优色数`。

## R10

输入模型与假设：
- 无向简单图；
- 不允许自环；
- 邻接矩阵必须是方阵、对称、对角全 `False`；
- 顶点编号使用 `0..n-1`。

非法输入会抛出 `ValueError` 或内部一致性异常。

## R11

脚本输出指标包括：
- 顶点数、边数、图密度；
- DSatur 颜色向量、颜色种数、颜色直方图；
- DSatur 顶点选择顺序与饱和度更新次数；
- 精确色数与一个精确着色方案；
- `optimality_gap = DSatur_色数 - 最优色数`。

## R12

适用场景：
- 需要快速得到合法着色；
- 允许“通常较好但不保证最优”的结果；
- 需要稳定可复现的启发式输出。

不适用场景：
- 必须拿到全局最优且图规模较大；
- 对最优性证明有严格要求。

## R13

局限性说明：
- DSatur 是启发式，不存在通用最优保证；
- 邻接矩阵实现在稀疏大图上不够省内存；
- 精确基线是指数级，仅适合小图验证。

## R14

常见工程坑与处理：
- 把“编号顺序差异”误判为算法不稳定：本实现用确定性 tie-break 规避；
- 忽略输入对称性导致隐式有向边：构图和运行前都做校验；
- 在大图上误用精确回溯：函数参数 `max_n_for_exact` 显式限制。

## R15

`demo.py` 模块划分：
- `build_undirected_adjacency`：构图与合法性检查；
- `dsatur_coloring`：DSatur 主流程；
- `is_proper_coloring`：着色合法性验证；
- `exact_chromatic_number_backtracking`：小图精确对照；
- `edge_count` / `color_histogram`：统计辅助；
- `main`：组织样例、断言、报告输出。

## R16

可扩展方向：
- 用邻接表 + 位集优化大图性能；
- 结合多次随机重启（只改并列策略）提高解质量；
- 在 DSatur 输出基础上做 Kempe-chain 或局部交换改进；
- 与 ILP/CP-SAT 精确解器结合做“启发式初解 + 精确收敛”。

## R18

`demo.py` 的源码级算法流程（8 步）：
1. `main` 定义固定无向图边集，调用 `build_undirected_adjacency` 构造并校验邻接矩阵。  
2. `dsatur_coloring` 初始化未着色集合、颜色数组、每个顶点的“邻居已用颜色集合”。  
3. 每轮在未着色顶点中按 `(饱和度, 度数, -编号)` 选择优先级最高顶点 `v`。  
4. 收集 `v` 的禁用颜色集合，分配最小可用颜色（从 `0` 开始递增查找）。  
5. 将 `v` 标记为已着色，并把该颜色写入其未着色邻居的“已见颜色集合”，完成饱和度更新。  
6. 重复步骤 3-5 直到所有顶点着色，得到 DSatur 解及统计信息（颜色数、顺序、更新次数）。  
7. 调用 `exact_chromatic_number_backtracking`：用 DFS 在“已有颜色/新颜色”分支上搜索，并用当前最优颜色数做剪枝，得到精确色数。  
8. `main` 对两种结果做合法性断言，输出 DSatur 与精确基线对比指标以及 `optimality_gap`。
