## R01

逻辑回归（Logistic Regression）是经典的二分类概率模型。它用线性函数 `z = w^T x + b` 建模样本得分，再通过 Sigmoid 函数

`σ(z) = 1 / (1 + e^{-z})`

把得分映射成类别 1 的概率 `P(y=1|x)`。

## R02

它要解决的问题是：给定特征 `x`，预测标签 `y in {0, 1}` 的条件概率并做分类决策。

形式化目标：

- 概率模型：`p_i = P(y_i=1|x_i) = σ(w^T x_i + b)`
- 决策规则：`ŷ_i = 1[p_i >= τ]`，默认阈值 `τ=0.5`

本质上是“线性可解释 + 概率输出”的判别式模型。

## R03

核心思想：

- 用线性组合刻画特征对类别的方向与强度。
- 用对数几率（log-odds）保持线性：`log(p/(1-p)) = w^T x + b`。
- 通过极大似然（等价于最小化交叉熵）学习参数。
- 可加 `L2` 正则降低过拟合，提升泛化稳定性。

## R04

输入：

- 训练特征矩阵 `X in R^{n x d}`
- 二值标签向量 `y in {0,1}^n`
- 超参数：学习率、迭代轮数、正则强度、阈值等

输出：

- 参数 `w, b`
- 每个样本的正类概率 `p`
- 分类预测 `ŷ`
- 评估指标（Accuracy / Precision / Recall / F1 / LogLoss / ROC-AUC）

## R05

带 `L2` 正则的训练目标（平均损失）可写为：

`J(w,b) = - (1/n) * Σ [ y_i log p_i + (1-y_i) log (1-p_i) ] + (λ/(2n)) ||w||^2`

其中：

- 第一项是二元交叉熵（负对数似然）
- 第二项是权重衰减（通常不惩罚偏置 `b`）
- `λ` 越大，模型越保守，方差更低但可能欠拟合

## R06

标准训练流程：

1. 划分训练集/测试集，避免测试泄漏。
2. 用训练集统计量做标准化（均值方差）。
3. 初始化参数（常见为 0 向量）。
4. 前向计算 `p = σ(Xw+b)`。
5. 计算交叉熵与梯度。
6. 梯度下降更新参数，直到收敛或达到迭代上限。
7. 在测试集上输出概率、标签与指标。

## R07

复杂度（批量梯度下降）：

- 单次迭代主要开销是矩阵向量乘 `Xw`，时间复杂度 `O(n*d)`。
- 若迭代 `T` 次，总复杂度约 `O(T*n*d)`。
- 参数存储为 `w` 与少量中间量，空间复杂度约 `O(d)`（不含数据本身）。

## R08

优点：

- 概率输出天然可解释，便于阈值调优。
- 训练快、实现简单，是强基线模型。
- 系数符号和大小可解释特征影响方向与强度。

局限：

- 决策边界是线性的，难直接拟合复杂非线性关系。
- 对异常值、特征尺度较敏感。
- 类别极度不平衡时，默认阈值和默认损失可能效果一般。

## R09

适用场景：

- 二分类概率预测（欺诈识别、转化预测、医学风险分层）。
- 需要解释性的业务决策场景。
- 作为更复杂模型前的可靠基线。

不太适用：

- 强非线性、高阶交互为主且样本量充足时（可考虑树模型/核方法/深度网络）。

## R11

与相关方法对比：

- 与线性回归：线性回归直接拟合实数输出；逻辑回归拟合概率，分类更合理。
- 与感知机：感知机只关心分类边界；逻辑回归提供概率与可微损失，优化更平滑。
- 与朴素贝叶斯：朴素贝叶斯依赖条件独立假设；逻辑回归是判别式，通常判别性能更稳。

## R12

工程注意点：

- 标准化必须只用训练集统计量，否则会泄漏测试信息。
- 类别不平衡时要关注 Precision/Recall/F1，而不只看 Accuracy。
- 业务上常需调阈值（不一定固定 0.5）。
- 若训练不收敛，可减小学习率、增大正则、检查特征共线性。

## R16

常见问题：

- 损失不下降：学习率过大或数据未标准化。
- 过拟合：特征过多或噪声大，可增大 `L2`。
- 指标波动：样本量太小或切分偶然性大，建议多次重采样验证。
- AUC 异常：检查是否存在单一类别、概率常数化或标签编码错误。

## R17

可扩展方向：

- 增加 `L1` 或 Elastic Net，做稀疏特征选择。
- 改为小批量随机梯度下降（mini-batch SGD）提升大规模训练效率。
- 加入类别权重或 focal-like 改造，处理极端不平衡。
- 用多分类 softmax 回归扩展到 `K>2` 类。
- 做概率校准（Platt/Isotonic）提升下游决策稳定性。

## R18

本 MVP 的“源码级算法流”可拆成 8 步（不是黑箱调库）：

1. 数据准备：生成 `X, y` 并切分训练/测试；仅用训练集统计量完成标准化。
2. 参数初始化：构造增广矩阵 `[1, X]`（截距项），初始化权重向量 `w=0`。
3. 前向传播：计算 `z = Xw`，再用 `σ(z)` 得到每个样本的正类概率 `p`。
4. 损失计算：按二元交叉熵计算数据项，再加 `L2` 正则项得到总目标 `J`。
5. 梯度计算：`grad = X^T(p-y)/n + λw/n`（截距不做正则）。
6. 参数更新：执行 `w <- w - lr * grad`，并记录损失历史用于收敛诊断。
7. 收敛判定：若相邻两轮损失改变量小于 `tol` 则提前停止，否则继续迭代。
8. 推理评估：在测试集输出概率与标签，计算 `Accuracy/F1/LogLoss/ROC-AUC`；若 `sklearn` 可用，再训练 `lbfgs` 版本进行结果对照。

这 8 步对应了逻辑回归从“概率建模”到“数值优化”再到“预测评估”的完整闭环。
