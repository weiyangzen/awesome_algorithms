# 牛顿万有引力定律 (Newton's Law of Universal Gravitation)

- UID: `PHYS-0002`
- 学科: `物理`
- 分类: `经典力学`
- 源序号: `2`
- 目标目录: `Algorithms/物理-经典力学-0002-牛顿万有引力定律_(Newton's_Law_of_Universal_Gravitation)`

## R01

问题定义：实现一个“可运行、可审计、非黑盒”的最小 MVP，基于牛顿万有引力定律

`F = G * m1 * m2 / r^2`

构建二维平面内的地月双体系统数值仿真，并输出：
- 引力大小量级；
- 轨道周期（理论 vs 数值）；
- 能量守恒与角动量守恒漂移；
- 反平方律回归斜率校验。

## R02

物理背景：
- 万有引力是经典力学中描述天体相互作用的基础定律；
- 将引力写成矢量形式后，可直接进入 N 体数值积分；
- 双体问题在质心系中可用于验证轨道周期、守恒量与反平方关系。

本条目选择地月系统，是因为参数真实、量级清晰、验证指标直观。

## R03

本 MVP 的输入和输出定义：
- 输入（脚本内固定，不需要交互）：
  - 两个天体质量（地球、月球）；
  - 初始相对距离；
  - 质心系初始位置与速度；
  - 积分步长 `dt` 与步数 `num_steps`。
- 输出：
  - 指标表（`pandas.DataFrame`）：力、周期误差、守恒漂移、回归斜率等；
  - 轨迹快照表：指定时刻的距离、引力、月球坐标。

## R04

建模假设（MVP 级）：
- 仅考虑两体相互引力，不含太阳和其他行星扰动；
- 点质量近似，不考虑潮汐形变；
- 真空环境，无阻力与推力；
- 引力常数 `G` 取 `scipy.constants.G`；
- 采用质心系初值，减少平移漂移。

该模型是“最小可信”而非“天文台级高精度星历”。

## R05

核心方程（`demo.py` 已直接实现）：
- 矢量引力（作用在体 1 上）：
  - `F_1<-2 = G * m1 * m2 * (r2-r1) / |r2-r1|^3`
- 加速度：
  - `a_i = sum_{j!=i} G * m_j * (r_j-r_i) / |r_j-r_i|^3`
- 机械能：
  - `E = sum(0.5*m_i*|v_i|^2) - sum_{i<j}(G*m_i*m_j/|r_i-r_j|)`
- 总角动量：
  - `L = sum(m_i * (r_i x v_i))`
- 理论周期（双体圆轨道近似）：
  - `T = 2*pi*sqrt(r^3 / (G*(m1+m2)))`

## R06

数值算法流程：
1. 设置地月质量、初始距离和质心系初始速度；
2. 用 `accelerations` 计算当前引力加速度；
3. 用 Velocity-Verlet 更新位置 `r_{k+1}`；
4. 在新位置计算新加速度；
5. 用新旧加速度平均更新速度 `v_{k+1}`；
6. 每步记录能量和角动量模长；
7. 由相对位置极角 `atan2(y,x)` 的展开斜率估计数值周期；
8. 输出汇总指标和轨迹快照。

## R07

复杂度分析（`N` 为天体数量，`S` 为积分步数）：
- 每步加速度计算复杂度：`O(N^2)`；
- 总时间复杂度：`O(S*N^2)`；
- 总空间复杂度：`O(S*N)`（保存完整轨迹）。

本案例 `N=2`，因此每步是常数成本，主要耗时由 `S` 决定。

## R08

数值稳定性与鲁棒性策略：
- 距离为 0 时抛出异常，防止 `1/r^3` 奇异；
- 对 `dt` 和 `num_steps` 做正数校验；
- 采用 Velocity-Verlet（相较显式欧拉更保守、更稳）；
- 使用守恒漂移（能量、角动量）作为数值健康度监控。

## R09

适用场景：
- 经典力学/天体力学课程演示；
- 引力轨道数值方法入门；
- 反平方律与守恒量的程序化验证。

不适用场景：

- 多体强扰动长期混沌预测；
- 需要非球形天体、潮汐或相对论修正的工程任务。

## R10

脚本中的正确性检查点：
1. `period_relative_error <= 0.05`。
2. `energy_rel_drift <= 1e-3`。
3. `angular_momentum_rel_drift <= 1e-8`。
4. 反平方律线性回归斜率满足 `|slope + 2| <= 1e-10`。
5. `torch` 与 `numpy` 计算同一引力向量的差值不超过 `1e-6 N`。

这些检查将“物理正确性 + 数值正确性 + 实现一致性”连在一起。

## R14

常见失败模式与修复建议：
- 失败：`dt` 过大导致能量漂移放大。
  - 修复：减小 `dt` 或增加 `num_steps`。
- 失败：初始速度与圆轨道不匹配，轨道变得高偏心。
  - 修复：根据 `sqrt(G*(m1+m2)/r)` 重新设定速度。
- 失败：多体扩展时出现近距离掠过导致刚性增强。
  - 修复：自适应步长或软化策略（教学场景可先不启用）。
- 失败：单位混用（km 与 m）。
  - 修复：内部统一 SI 单位，只在输出层换算。

## R18

`demo.py` 源码级算法流拆解（8 步）：
1. `main` 设定地月质量、初始距离和质心系初始位置/速度，形成双体初值。  
2. `velocity_verlet` 初始化时间轴、状态数组、能量与角动量日志，并先计算 `a0`。  
3. 在每个时间步，先按 `r_{k+1}=r_k+v_k*dt+0.5*a_k*dt^2` 更新位置。  
4. 调用 `accelerations` 对每对天体显式计算 `G*m_j*(r_j-r_i)/|r_j-r_i|^3`，得到 `a_{k+1}`。  
5. 再按 `v_{k+1}=v_k+0.5*(a_k+a_{k+1})*dt` 更新速度，完成 Velocity-Verlet 一步。  
6. 每步调用 `total_energy` 与 `total_angular_momentum` 记录守恒量轨迹并统计相对漂移。  
7. 仿真结束后，用 `estimate_orbital_period` 从 `atan2` 展开角速度估计数值周期，并与开普勒理论周期对比。  
8. 用 `inverse_square_regression`（`scikit-learn`）验证斜率 `-2`，再用 `torch_force_consistency` 交叉核对一次引力矢量，最后统一输出 `pandas` 报表并执行断言。  

说明：第三方库只承担基础数值/回归/表格能力，核心引力计算与积分流程都在源码中逐步展开，不是黑盒调用。
