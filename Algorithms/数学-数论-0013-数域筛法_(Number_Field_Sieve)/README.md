# 数域筛法 (Number Field Sieve)

- UID: `MATH-0013`
- 学科: `数学`

## R01

数域筛法（NFS）是当前已知最快的通用大整数分解算法（针对经典计算模型）。

核心思想（GNFS 语境）是把“找同余平方”转化为“在两个不同结构中同时收集可平滑分解的关系”：

1. 选多项式：构造整数多项式 `f(x)` 与整数 `m`，满足 `f(m) ≡ 0 (mod N)`。
2. 双侧范数：对候选对 `(a,b)` 计算
   - 有理侧范数 `R(a,b)=a-bm`
   - 代数侧范数 `A(a,b)=b^d f(a/b)`（整数化后）
3. 关系收集：要求两侧都对各自因子基“足够平滑”（可分解为小素数乘积）。
4. 线性代数：把指数向量按模 2 组装成稀疏矩阵，找零空间依赖。
5. 平方根阶段：从依赖恢复两边平方关系，构造 `X^2 ≡ Y^2 (mod N)`，最终用 `gcd(X±Y,N)` 得非平凡因子。

输入：待分解奇合数 `N`（通常是大半素数乘积）。
输出：`N` 的非平凡因子 `p,q`。

## R02

时间脉络（常见叙述）：

- 1980s：Special NFS（SNFS）逐步成形（适合特殊形式整数）。
- 1990-1993：General NFS（GNFS）框架成熟，Buhler–Lenstra–Pomerance 等工作系统化了亚指数复杂度分析。
- 1990s 以后：NFS 成为 RSA 挑战数分解主力，工程实现不断演进（大规模分布式筛 + 稀疏线代）。

## R03

提出背景：

- 早期方法（试除、Pollard 系列）对超大合数失效较快。
- 二次筛（QS）在中等规模整数上很强，但位数继续增大后，复杂度常数和主项使其落后于 NFS。
- 现代公钥密码（RSA）安全强依赖“分解难度”，需要更快通用分解算法来校准密钥长度。

NFS 的历史意义在于：把分解问题从单一整数域扩展到代数数域 + 有理域的耦合关系搜集。

## R04

GNFS 启发式复杂度常写为：

`L_N[1/3, (64/9)^(1/3)] = exp(((64/9)^(1/3)+o(1)) * (ln N)^(1/3) * (ln ln N)^(2/3))`

SNFS 常数更优（结构化输入）：

`L_N[1/3, (32/9)^(1/3)]`

阶段主耗时：

- 关系筛选（sieving）
- 稀疏矩阵线性代数（Block Lanczos/Wiedemann）

## R05

空间复杂度（工程上）主要由关系库和稀疏矩阵决定：

- 关系条目通常远大于因子基规模。
- 矩阵行列规模可达千万级，需压缩存储与外存/分布式支持。
- 渐近上常以“与关系数近线性”的稀疏存储成本描述，但常数很大。

## R07

算法意义：

- 理论上：把整数分解复杂度推进到 `L_N[1/3, c]` 量级。
- 工程上：决定 RSA 密钥长度安全边界评估的核心参照。
- 方法论上：把“平滑数概率 + 超大规模稀疏线代 + 分布式计算”融合成一条完整密码分析流水线。

## R08

NFS 直接依赖的关键方法/工具：

- 模运算与大整数算法（快速乘法、模幂、gcd）
- 平滑数概率模型（Dickman 函数近似）
- 多项式选择启发式（如 Murphy-E 打分思想）
- 格筛/Lattice sieving
- 稀疏 GF(2) 线性代数（Block Lanczos / Block Wiedemann）
- CRT 与平方根恢复步骤

## R09

适用前提与边界：

- 目标为大奇合数分解；若 `N` 有很小因子，预处理会先剥离。
- GNFS 是“通用”算法，但只有在足够大位数时优势明显；中小规模时 QS/ECM 常更划算。
- 本目录 MVP 边界：
  - 只演示“关系→矩阵→依赖→gcd”主干；
  - 不实现 GNFS 的理想分解、大素数变体、过滤流水线、分布式块 Wiedemann。

## R10

正确性依据（完整 NFS）：

- 若通过线性代数得到指数全偶依赖，则对应两侧对象可写成平方。
- 通过数域到模 `N` 的映射把代数侧平方投影回整数同余，得到 `X^2 ≡ Y^2 (mod N)`。
- 当 `X ≠ ±Y (mod N)` 时，`gcd(X-Y,N)` 或 `gcd(X+Y,N)` 产生非平凡因子。

本 MVP 沿用该证明骨架，但在关系收集层采用可运行近似模型。

## R11

数值稳定性与误差传播：

- NFS 主体是整数/有限域精确运算，不存在浮点舍入误差主导的问题。
- 真正的不确定性来自启发式概率：关系收集速度、矩阵秩、依赖是否产出非平凡因子。
- 工程实现中风险更多是“统计失败/参数不佳”，而非“数值不稳定”。

## R12

真实计算成本（超越渐近符号）：

- 多项式选择质量对总成本有数量级影响。
- 筛阶段 CPU 与内存带宽占比高，且可强分布式并行。
- 过滤与线代 I/O 代价显著，磁盘/网络拓扑会成为瓶颈。
- 对超大实例，线代常是长尾阶段，需要大内存节点和长时间稳定运行。

## R13

近似界/概率保证（启发式）：

- 平滑概率常用 Dickman `ρ(u)` 模型估计，`u = ln|value| / ln B`。
- 关系数通常需超过因子基维数若干冗余（含过滤损失）才能形成可用零空间。
- 单次依赖不一定给出非平凡因子，因此要尝试多个依赖向量。

结论：NFS 成功率依赖参数与样本量，是“高概率可行”而非一步确定性分解。

## R14

鲁棒性与失效模式：

- 多项式差：导致平滑关系密度低，筛阶段几乎停滞。
- 关系不足或过滤过度：矩阵秩不够，线代无有效依赖。
- 依赖退化：得到 `X ≡ ±Y (mod N)`，gcd 只返回平凡因子。
- 参数失配：因子基过小/过大都会恶化总时间。

## R15

工程实现要点：

- 先做 trial division + ECM 预处理，尽早剥离小因子。
- 多项式选择与筛参数（factor base bound、large-prime 策略）要联调。
- 使用桶筛、分块缓存、位图压缩降低内存与带宽压力。
- 过滤阶段需要去重、合并、删除孤立点，保证矩阵质量。
- 线代需用专门稀疏 GF(2) 算法，而非通用稠密线代库。

## R16

前驱/后继与应用：

- 前驱：Dixon、Continued Fraction、Quadratic Sieve。
- 并行分支：ECM 常用于 NFS 前置找小因子。
- 后继工程：CADO-NFS、Msieve + GGNFS 等实现持续优化。
- 典型应用：RSA 挑战数分解、密码参数审计、学术 benchmark（按位数比较耗时）。

## R18

源码追踪与 3-10 步算法拆解（以主流 NFS 工程实现的模块化流水线为参照，如 CADO-NFS 的 `polyselect -> sieve/las -> filter -> linalg -> sqrt`）：

1. `polyselect`：枚举/优化多项式，评估根性质与平滑潜力，输出最优候选。
2. `sieve (las)`：在二维参数空间扫描 `(a,b)`，快速定位可能平滑的有理侧/代数侧范数。
3. `relation build`：对候选做精确分解，记录关系；支持 large-prime 变体以提高产率。
4. `filter`：合并重复、消去低度节点、压缩关系图，形成可线代矩阵。
5. `matrix build`：把关系转成 GF(2) 稀疏矩阵（按理想/素因子指数奇偶）。
6. `linalg`：用 Block Lanczos/Wiedemann 求零空间向量。
7. `sqrt`：从零空间依赖恢复两边平方并映射到 `mod N`。
8. `split`：计算 `gcd(X±Y,N)`，得到非平凡因子；失败则换依赖重试。

本目录 `demo.py` 对应了其中的第 2/3/5/6/7/8 步缩小版流程，并在注释中明确了“toy 近似”与“完整 GNFS”之间的差异，避免黑箱化描述。
