## R01

“搜索二维矩阵 II”问题：给定一个 `m x n` 整数矩阵 `matrix`，满足
- 每行从左到右非递减；
- 每列从上到下非递减；

判断目标值 `target` 是否存在于矩阵中（可返回布尔值，或额外返回一个命中坐标）。

## R02

该题是二维有序结构查找的经典案例（常见于在线题库 `Search a 2D Matrix II`）。

代表性在于：
- 输入不是“全局一维有序”，而是行列双单调；
- 可做 `O(m+n)` 的“楼梯搜索”，也可做子矩阵裁剪型分治；
- 适合作为“从一维二分扩展到二维边界裁剪”的教学桥梁。

## R03

形式化定义：
- 输入：`matrix ∈ Z^(m×n)`，满足行列单调性；整数 `target`。
- 输出：`found ∈ {True, False}`；若实现需要，也可附带 `(row, col)` 命中位置。

示例：
- `matrix = [[1,4,7],[2,5,8],[3,6,9]]`, `target=5`，输出 `True`。
- 同一矩阵 `target=10`，输出 `False`。

## R04

本题分治核心（按中列切分）：

1. 对当前子矩阵 `[top..bottom] x [left..right]`，先做范围剪枝：
   - 若 `target < matrix[top][left]` 或 `target > matrix[bottom][right]`，直接失败。
2. 取中列 `mid_col`，从 `top` 往下扫描该列，直到遇到 `> target` 的行或越界。
3. 若扫描中命中 `== target`，直接成功。
4. 否则把搜索空间切成两个互补子矩阵递归：
   - 右上子矩阵：`[top .. scan_row-1] x [mid_col+1 .. right]`
   - 左下子矩阵：`[scan_row .. bottom] x [left .. mid_col-1]`

理由：中列把“可能小于目标”和“可能大于目标”的区域分开，另外两个象限可被单调性排除。

## R05

MVP 的关键状态：
- `top, bottom, left, right`：当前递归子矩阵边界；
- `mid_col`：当前切分列；
- `scan_row`：中列向下扫描停止位置；
- `SearchStats`：记录递归调用、比较次数、范围剪枝次数、最大深度；
- `SearchResult`：返回是否命中、命中坐标与统计信息。

## R06

示例（目标 `target=5`）：

```text
matrix =
[
 [1, 4, 7, 11, 15],
 [2, 5, 8, 12, 19],
 [3, 6, 9, 16, 22],
 [10,13,14,17,24],
 [18,21,23,26,30]
]
```

- 初始子矩阵是全矩阵，`mid_col=2`（列值依次 `7,8,9,14,23`）。
- 从上往下扫描到第一行即发现 `7 > 5`，停止于 `scan_row=0`。
- 右上子矩阵为空（`top=0, scan_row-1=-1`），直接跳过。
- 进入左下子矩阵（仍是完整行，但列缩到 `[0..1]`）。
- 再取中列并扫描，很快在 `(1,1)` 命中 `5`，返回成功。

## R07

算法价值：
- 把“二维有序”转成“可裁剪子问题”，体现分治建模能力；
- 提供与楼梯法不同的思考路径：先切分再递归，不是单路径游走；
- 在很多“矩阵边界单调”的问题中可复用“范围剪枝 + 局部扫描 + 子区间递归”的套路。

## R08

前置知识：
- 二分/分治边界维护；
- 递归终止条件设计；
- 二维索引与子矩阵边界合法性；
- 基础复杂度分析与基线算法（暴力/楼梯搜索）。

## R09

复杂度（本目录的中列分治实现）：
- 时间复杂度：常见分析上界可写为 `O(m log n)`（按列递归切分并结合中列扫描）；在某些数据形态下可更快。
- 空间复杂度：`O(log n)` 级递归栈（取决于列方向切分深度）。

对比：
- 楼梯搜索（从右上角或左下角出发）是 `O(m+n)`、`O(1)` 空间；
- 暴力枚举是 `O(mn)`。

## R10

正确性要点（简述）：

1. 不变量：递归函数始终只在“尚未被单调性排除”的子矩阵中搜索。
2. 范围剪枝正确：
   - 子矩阵最小值位于左上角，最大值位于右下角；
   - 目标超出该区间时必不可能存在。
3. 中列扫描后，基于单调性可排除两个象限：
   - 中列上方（含）与右侧组合、
   - 中列下方与左侧组合，
   之外的候选区域恰为“右上 + 左下”两个子矩阵。
4. 递归每次严格缩减列区间（或快速命中/剪枝），终将终止。
5. 若命中则返回真实坐标；若所有子问题失败则目标不存在。

## R11

数值稳定性：
- 本题主要做整数比较，不涉及迭代数值误差传播；
- 主要风险不是“浮点不稳定”，而是“数据语义异常”：
  - 非规则矩阵（ragged rows）导致索引失效；
  - 输入含 `NaN`（若使用浮点）会破坏比较逻辑；
  - 若误把“行列非递减”当成“全局一维有序”会得到错误剪枝。

本 MVP 使用 `int64` 矩阵并显式检查矩阵规则性。

## R12

工程成本（超越大 O）：
- 递归版本代码表达清晰，但有函数调用开销；
- 若做高频在线查询，楼梯法通常更短更稳、更易维护；
- 转换输入为 `numpy.ndarray` 有一次性拷贝成本，但便于统一数据校验与测试；
- 真正工程瓶颈常在 I/O、数据准备与批量调度，而非单次查找。

## R14

常见失效模式：
- 忘记处理空矩阵或空列矩阵；
- 子矩阵边界更新错误，导致漏解或无限递归；
- 误剪枝：没先检查 `[min,max]` 范围就盲目递归，性能退化；
- 随机测试只测“存在”场景，不测“不存在”场景；
- 只验证布尔值，不验证返回坐标确实命中目标。

## R15

实现建议：
- 先写可读版本，再加统计信息（调用次数、剪枝次数）帮助调参；
- 同时保留楼梯法作为基线，做随机对拍；
- 测试覆盖应含：
  - `1x1`、`1xn`、`mx1`；
  - 命中首元素、尾元素、中间元素；
  - 超出范围（比最小值小、比最大值大）；
  - 大量随机单调矩阵。

## R17

本目录 `demo.py` 的 MVP 设计：
- 依赖：Python 标准库 + `numpy` + `pandas`。
- 核心函数：
  - `search_matrix_divide_conquer`：分治主算法，返回命中状态与坐标；
  - `search_matrix_staircase`：`O(m+n)` 基线；
  - `search_matrix_bruteforce`：真值校验器。
- 运行流程：
  - `run_fixed_cases()`：固定样例断言；
  - `run_random_regression()`：随机单调矩阵对拍；
  - `run_benchmark()`：分治 vs 楼梯法平均耗时表。
- 运行命令：`uv run python demo.py`。

## R18

`demo.py` 源码级流程（8 步）：

1. `main()` 依次调用固定样例、随机回归、基准测试，脚本全程无交互输入。  
2. `search_matrix_divide_conquer()` 先用 `_to_2d_int_array()` 把输入转为规则二维 `int64` 矩阵，并初始化 `SearchStats`。  
3. 内部递归 `solve(top,bottom,left,right,depth)` 更新调用计数与最大深度。  
4. 先做终止与剪枝：若子矩阵空或 `target` 超出该子矩阵 `[左上,右下]` 数值范围，直接返回失败。  
5. 取 `mid_col`，从 `top` 开始沿中列向下扫描；扫描中若遇到 `== target` 立即返回命中坐标。  
6. 扫描结束后，用停止行 `scan_row` 将候选空间分为“右上子矩阵”和“左下子矩阵”。  
7. 先递归右上；若未命中再递归左下，直到命中或所有候选子矩阵被排除。  
8. `run_fixed_cases` 与 `run_random_regression` 使用楼梯法、暴力法交叉验证分治结果，`run_benchmark` 用 `pandas` 输出耗时对比表。
