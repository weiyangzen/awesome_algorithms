## R01

同态加密（Homomorphic Encryption, HE）允许在**密文**上直接执行特定运算，解密后结果与“先对明文计算再加密”一致。它的核心价值是“数据可用不可见”，适用于隐私计算、外包计算与安全聚合。

## R02

本目录实现的是 Paillier 体制的最小可运行 MVP，属于**加法同态加密（PHE）**：
- 支持密文加法：`Enc(m1) * Enc(m2) = Enc(m1 + m2)`
- 支持明文标量乘法：`Enc(m)^k = Enc(k*m)`
- 不支持密文乘密文（因此不是全同态 FHE）

## R03

Paillier 关键数学对象：
- 选素数 `p, q`，令 `n = p*q`，模空间为 `n^2`
- 取 `g = n + 1`
- 私钥参数 `lambda = lcm(p-1, q-1)`
- 定义 `L(u) = (u - 1) / n`
- `mu = (L(g^lambda mod n^2))^{-1} mod n`

## R04

密钥生成流程（`generate_keypair`）：
1. 生成两个不同素数 `p, q`（Miller-Rabin 概率素性测试）。
2. 计算 `n, n^2, g, lambda`。
3. 计算 `L(g^lambda mod n^2)`，并求其模逆得到 `mu`。
4. 输出公钥 `(n, g, n^2)` 与私钥 `(lambda, mu)`。

## R05

加解密公式：
- 加密：随机选 `r` 且 `gcd(r, n)=1`，
  `c = g^m * r^n mod n^2`
- 解密：
  `m = L(c^lambda mod n^2) * mu mod n`

在实现中，明文约束为 `0 <= m < n`。

## R06

同态性质在代码中对应：
- `e_add(pk, c1, c2)`：返回 `c1*c2 mod n^2`，解密后等于 `m1+m2 mod n`
- `e_mul_plain(pk, c, k)`：返回 `c^k mod n^2`，解密后等于 `k*m mod n`

## R07

正确性直觉：
- Paillier 构造保证 `g^m` 部分承载明文，`r^n` 部分提供随机掩蔽。
- 密文相乘会让指数相加，因此对应明文加法。
- 使用 `L(·)` 与 `mu` 可以把指数结构恢复到 `Z_n` 上，得到原消息（模 `n`）。

- 成功结束时输出 `MVP run completed successfully.`

## R10

复杂度（设模数位数为 `k`）：
- 密钥生成：主要成本在素数搜索与幂模运算，近似为若干次 `O(k^3)` 级大整数运算。
- 单次加密：两次幂模运算，主导为 `O(k^3)`。
- 单次解密：一次幂模运算加常数次乘法，主导为 `O(k^3)`。
- 同态加法：一次模乘，远低于加解密成本。

## R11

安全性说明：
- Paillier 安全性通常基于复合剩余类困难假设（DCRA）。
- 本 MVP 仅演示算法流程，默认位数较小（`bits=48`）以保证运行快速。
- 真实生产需使用远大于本示例的安全参数，并采用成熟实现与审计。

## R12

边界与约束：
- 仅支持非负明文，范围 `[0, n)`。
- `e_mul_plain` 在本 MVP 中限制 `k >= 0`。
- 运算结果都在模 `n` 意义下解释，若明文和超过 `n` 会回绕。
- 不包含序列化、网络传输、密钥托管等工程能力。

## R13

与其他同态路线的关系：
- PHE（如 Paillier）：运算能力有限，但结构清晰、实现成本低。
- SHE/FHE（如 BGV/BFV/CKKS）：支持更复杂电路，但实现复杂度和开销显著更高。
- 本项目用于“理解同态思想与最小闭环”，不是高性能隐私计算框架。

## R14

典型应用场景：
- 安全求和（如匿名投票计票、隐私统计）。
- 多方上报后由聚合方在密文域求和。
- 需要“服务器可计算、不可见原文”的轻量场景。

## R18

源码级算法流（对应 `demo.py`，8 步）：
1. `generate_prime` 调用 `is_probable_prime`，通过 Miller-Rabin 生成两个候选素数 `p, q`。
2. `generate_keypair` 计算 `n, n_sq, g=n+1, lambda=lcm(p-1,q-1)`，再用 `L(g^lambda mod n^2)` 求得 `mu`。
3. `main` 固定示例明文数组 `values=[7,11,23]`，并生成对应密文列表。
4. `encrypt` 对每个明文随机采样互素 `r`，按 `g^m * r^n mod n^2` 形成密文。
5. `main` 循环调用 `e_add` 聚合多个密文，实现“密文域求和”。
6. `decrypt` 对聚合密文执行 `c^lambda mod n^2 -> L(·) -> *mu mod n`，恢复同态和。
7. `main` 再用 `e_mul_plain` + `e_add` 构造 `3*v0 + 2*v1 + v2` 的密文线性组合。
8. 最后两次 `decrypt` 结果与明文侧期望值比较并 `assert`，验证同态正确性。
