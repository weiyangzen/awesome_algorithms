# Nose-Hoover恒温器 (Nose-Hoover Thermostat)

- UID: `PHYS-0327`

- 分类: `计算物理`

## R01

Nose-Hoover 恒温器是分子动力学中用于采样正则系综 (NVT) 的确定性温控方法。它通过引入一个额外动力学变量 `xi`（摩擦系数）来持续调节粒子动量，使系统时间平均温度收敛到目标温度 `T`。

## R02

应用场景：
- 需要保持固定粒子数、体积、温度的经典分子动力学模拟。
- 关心温度控制但又希望动力学轨迹相对平滑（相比随机热浴）。
- 需要在简单模型上快速演示正则分布采样行为。

本目录的 MVP 选择“1D 谐振子集合”作为教学示例，重点展示恒温器机制而不是复杂势函数。

## R03

连续时间方程（`k_B=1` 可化简）为：

- `dq_i/dt = p_i / m`
- `dp_i/dt = F_i(q) - xi * p_i`
- `dxi/dt = (1/Q) * (sum_i(p_i^2/m) - g*k_B*T)`

符号说明：
- `q_i, p_i`: 第 `i` 个自由度的位置和动量。
- `m`: 质量。
- `F_i(q)`: 力；本例为谐振子 `F=-k*q`。
- `xi`: 恒温器摩擦变量。
- `Q`: 恒温器“质量”，控制控温响应快慢。
- `g`: 自由度数，本例 `g = n_oscillators`。

## R04

`demo.py` 的输入是代码内 `SimulationConfig` 参数（无需交互输入），输出是标准输出文本统计：
- 目标温度与生产段平均温度。
- 温度标准差与相对误差。
- `E[x^2]` 的理论值与模拟值（检验正则分布宽度）。
- 生产段平均机械能。

## R05

关键参数：
- `n_oscillators`: 振子数（也是总自由度）。
- `n_steps`: 总积分步数。
- `dt`: 时间步长。
- `sample_every`: 采样间隔（每多少步记录一次）。
- `burn_in_steps`: 热化阶段步数，统计时丢弃。
- `target_temperature`: 目标温度 `T`。
- `thermostat_mass`: Nose-Hoover 参数 `Q`。
- `mass`: 粒子质量 `m`。
- `k_spring`: 谐振势系数 `k`。
- `kb`: 玻尔兹曼常数（本例默认 1）。
- `seed`: 随机种子。

## R06

初始化策略：
- 位置 `q` 按高斯分布初始化，方差取 `k_B*T/k` 的尺度。
- 动量 `p` 按高斯分布初始化，方差取 `m*k_B*T` 的尺度。
- 恒温器变量 `xi` 初始为 `0`。
- 初始力通过 `harmonic_force(q, k)` 计算。

这样起步通常更快进入目标温度附近。

## R07

离散积分采用对称分裂（类似 velocity-Verlet + thermostat）单步流程：
- 半步更新 `xi`。
- 半步摩擦缩放 `p *= exp(-xi*dt/2)`。
- 半步力更新动量。
- 整步更新位置。
- 用新位置重算力。
- 再做半步力更新。
- 再做半步摩擦缩放。
- 再半步更新 `xi`。

该流程在稳定性和实现复杂度之间取得了较好的平衡。

## R08

正确性直觉：
- 当系统动能偏高时，`dxi/dt > 0`，摩擦增强，动量被抑制，温度下降。
- 当系统动能偏低时，`dxi/dt < 0`，等效“反摩擦”补能，温度上升。
- 这种闭环反馈使时间平均温度围绕目标值波动，并趋向正则统计。

## R09

复杂度：
- 时间复杂度 `O(n_steps * n_oscillators)`。
- 空间复杂度 `O(n_oscillators + n_records)`。

由于用 `numpy` 向量化，常数开销较低，适合做教学和快速验证。

## R10

数值稳定性注意点：
- `dt` 过大可能导致积分发散或温度大幅振荡。
- `Q` 过小会导致控温过激（高频震荡），过大则控温过慢。
- 自由度太小时 Nose-Hoover 可能遍历性不足；本例用 256 个振子减弱此问题。
- 统计应丢弃热化段 (`burn_in_steps`) 才更接近稳态性质。

## R11

结果解读建议：

- `temperature rel. error` 越小越好；MVP 级别通常在几个百分点内可接受。

- `temperature std` 反映热涨落大小，不应误解为算法错误。

## R14

边界与异常处理：
- `n_oscillators <= 0`、`n_steps <= 0`、`dt <= 0`、`Q <= 0` 等会抛出 `ValueError`。
- `burn_in_steps` 必须在 `[0, n_steps)`，否则统计区间无意义。
- 若输出出现明显异常，可优先减小 `dt` 或调大 `Q`。

## R15

与常见温控方法对比：
- Andersen 恒温器：引入随机碰撞，统计正确但动力学相关性改变更明显。
- Langevin 恒温器：有阻尼和随机力，稳健但轨迹不再完全确定性。
- Berendsen 恒温器：控温平滑但不严格生成正则系综。
- Nose-Hoover：确定性、可逼近正则系综，但参数和遍历性更敏感。

## R18

`demo.py` 中算法的源码级流程（非黑盒）可拆为 8 步：

1. `SimulationConfig` 定义积分参数、热浴参数和模型参数，构成实验输入。
2. `simulate()` 做参数合法性校验，确保步长、步数、`Q` 与热化区间可用。
3. 用 `np.random.default_rng(seed)` 生成初态：`q` 和 `p` 按目标温度尺度高斯采样。
4. 通过 `harmonic_force()` 计算初始力，并进入主时间循环。
5. 每步先用当前动能 `K` 更新半步 `xi`，实现温度偏差反馈。
6. 按对称分裂顺序更新动量与位置：摩擦半步、力半步、位置整步、力半步、摩擦半步。
7. 用更新后的动能再推进半步 `xi`，完成一个时间步闭环；按 `sample_every` 记录温度和 `x^2`。
8. 循环结束后丢弃热化段，计算温度均值/误差和 `E[x^2]` 与理论值的偏差，并在 `main()` 中打印。

这 8 步完整对应 Nose-Hoover 的数值积分逻辑，没有调用外部黑盒恒温器实现。
