# 坐标下降法

## R01

坐标下降法（Coordinate Descent）是求解优化问题的经典方法：
每次只优化一个坐标（一个变量），其余变量固定，然后循环扫描全部坐标直到收敛。

本目录选择最常见、也最适合坐标更新闭式求解的场景：
- `Lasso` 回归（`L1` 正则线性回归）
- 目标函数为凸函数
- 每个坐标更新可写成软阈值形式

## R02

本 MVP 要解决的问题：

给定样本矩阵 `X in R^(n*p)`、标签 `y in R^n` 和正则系数 `alpha > 0`，求解

`min_w  (1/(2n)) * ||y - Xw||_2^2 + alpha * ||w||_1`

输出：
- 估计参数 `w`
- 每轮 epoch 的目标函数值、最大坐标更新量、非零系数数量
- 收敛状态（是否在最大轮次内达到阈值）

## R03

选择该问题的原因：
- 坐标下降法在 Lasso 上有明确、可解释的单坐标闭式更新；
- 可直接展示“稀疏化”机制（`L1` 使部分坐标变成 0）；
- 相比把任务直接交给 `sklearn.linear_model.Lasso`，手写实现更利于理解算法细节。

## R04

单坐标更新公式（循环坐标下降，cyclic CD）：

设当前残差 `r = y - Xw`，更新第 `j` 个坐标时定义：

- `z_j = (1/n) * ||x_j||_2^2`
- `rho_j = (1/n) * x_j^T (r + x_j * w_j)`

软阈值算子：

`S(t, lam) = sign(t) * max(|t| - lam, 0)`

则更新为：

`w_j <- S(rho_j, alpha) / z_j`

更新后再把残差增量维护回去，避免每次重新计算 `y-Xw`。

## R05

算法流程（高层）：

1. 校验 `X/y` 形状与数值合法性，检查 `alpha/tol/max_epochs`。
2. 初始化 `w=0`、`residual=y`。
3. 预计算每列范数项 `z_j`。
4. 每个 epoch 依次遍历 `j=0..p-1`：
   - 计算 `rho_j`
   - 软阈值更新 `w_j`
   - 用增量更新残差
5. 记录 epoch 指标：`objective`、`max|Δw|`、`nnz`。
6. 若 `max|Δw| < tol` 则判定收敛。
7. 达到 `max_epochs` 仍未满足阈值则返回未收敛状态。

## R06

正确性要点：
- 在凸目标下，固定其余坐标时，单坐标子问题是一维凸问题；
- 软阈值更新等价于该一维子问题的最优解；
- 循环坐标下降会使目标函数非增（数值误差下可能有极小扰动）；
- `demo.py` 额外检查目标函数的“近似单调性”，作为实现层面的审计。

## R07

复杂度分析：
- 记样本数 `n`、特征数 `p`、迭代轮次 `T`。
- 每次坐标更新涉及一次长度 `n` 的向量内积和一次残差更新，成本 `O(n)`。
- 单个 epoch 扫描 `p` 个坐标，成本 `O(np)`。
- 总时间复杂度 `O(Tnp)`。
- 额外空间：
  - 参数向量 `w`：`O(p)`
  - 残差 `r`：`O(n)`
  - 预计算列范数 `z`：`O(p)`

## R08

边界与异常处理：
- `X` 必须是二维、`y` 必须是一维，且 `X.shape[0] == y.shape[0]`；
- 输入必须是有限数值（不允许 `nan/inf`）；
- `alpha <= 0`、`tol <= 0`、`max_epochs <= 0` 直接报错；
- 若某列方差过小导致 `z_j` 近零，报错并提示先做特征筛选/标准化。

## R09

MVP 取舍：
- 仅依赖 `numpy` 与标准库，避免引入重框架；
- 不实现随机坐标、并行坐标、强规则筛选等高级优化；
- 不调用现成 Lasso 黑盒接口，核心迭代完全手写；
- 保留可审计轨迹（history），优先可解释性而不是极限性能。

## R10

`demo.py` 函数职责：
- `validate_inputs`：输入形状、有限值与超参数合法性检查。
- `soft_threshold`：实现软阈值算子。
- `lasso_objective`：计算当前目标函数值。
- `coordinate_descent_lasso`：核心循环坐标下降。
- `make_synthetic_regression`：构造可复现实验数据。
- `support_metrics`：计算稀疏支持集的 precision/recall。
- `print_history`：打印迭代轨迹。
- `run_case`：运行单组 `alpha` 并输出指标。
- `main`：固定种子运行两组 `alpha` 并汇总。

## R12

输出字段说明：
- `epoch`：迭代轮次。
- `objective`：Lasso 目标函数值。
- `max|delta_w|`：本轮最大坐标更新幅度。
- `nnz`：当前非零系数个数。
- `converged`：是否在 `max_epochs` 内达到收敛阈值。
- `mse`：训练集均方误差（不含正则项）。
- `precision/recall`：估计支持集与真实支持集的一致性指标。
- `objective monotone check`：目标函数是否近似单调下降。

## R13

最小测试集（本脚本已覆盖）：
- 固定随机种子生成稀疏线性模型数据；
- 两组正则强度：
  - 较小 `alpha`（保留更多特征）
  - 较大 `alpha`（更稀疏）
- 对每组输出：
  - 收敛状态
  - 估计系数与真值的 L2 误差
  - 支持集 precision/recall
  - 目标函数单调性检查

## R14

关键参数与建议：
- `alpha`：L1 正则强度，越大越稀疏。
- `tol`：坐标更新阈值，越小收敛判据越严格。
- `max_epochs`：最大扫描轮数。
- `coef_tol`：判定“系数为非零”的阈值（用于支持集统计）。

经验上：
- 先固定 `tol=1e-8` 和较大 `max_epochs`；
- 再通过调整 `alpha` 观察稀疏度和拟合误差的折中。

## R15

与相关方法对比：
- 对比批量梯度下降：
  - 坐标下降每次只更新一个分量，单步更便宜，且在 Lasso 上有闭式更新。
- 对比 ISTA/FISTA：
  - ISTA/FISTA 用全向量近端更新；
  - 坐标下降用逐坐标精确子问题解，常在中小规模稀疏回归上更直接。
- 对比黑盒库：
  - 工程中可直接调用库；
  - 教学和审计场景下，手写版本更透明。

## R16

典型应用场景：
- 稀疏线性建模与特征选择；
- 高维但希望模型可解释（少量特征激活）的回归问题；
- 作为更复杂优化器前的可解释 baseline。

## R17

可扩展方向：

- 安全筛选（screening）减少无效坐标更新；

## R18

`demo.py` 源码级算法流程（8 步）：

1. `main` 固定随机种子生成标准化特征的合成稀疏回归数据，并定义两组 `alpha`。
2. 每组 `alpha` 由 `run_case` 调用 `coordinate_descent_lasso` 执行训练。
3. `coordinate_descent_lasso` 先调用 `validate_inputs` 检查维度、有限值与参数合法性。
4. 初始化 `w=0`、`residual=y`，并一次性预计算每列 `z_j=(1/n)||x_j||^2`。
5. 在每个 epoch 内按坐标顺序循环：基于当前残差计算 `rho_j`，再用 `soft_threshold` 得到 `w_j` 新值。
6. 计算 `delta = w_j_new - w_j_old` 后，用 `residual -= x_j * delta` 做增量更新，避免重复构造 `Xw`。
7. 每轮结束计算 `lasso_objective`、`max|delta_w|` 和 `nnz` 写入 `history`；若 `max|delta_w| < tol` 则收敛返回。
8. `run_case` 进一步计算 `mse`、支持集 precision/recall 与目标函数单调性检查，`main` 汇总两组案例结果。
