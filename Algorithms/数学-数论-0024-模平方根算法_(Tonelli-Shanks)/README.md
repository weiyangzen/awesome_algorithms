## R01

Tonelli-Shanks 算法用于求解奇素数模下的二次同余：

`x^2 ≡ n (mod p)`，其中 `p` 是奇素数。

给定 `n, p`，算法要么返回一个平方根 `r`（进而得到另一个根 `p-r`），要么判定无解（`n` 不是模 `p` 的二次剩余）。

它是“模平方根”最经典的通用算法之一，尤其适用于 `p % 4 != 3` 的情形。对于 `p % 4 == 3` 时可用更快特例公式，Tonelli-Shanks 也会自然退化到很短流程。

## R02

输入输出定义（本条目 MVP 采用奇素数模）：

- 输入：
  - `n`：待开平方的整数（可先归约到 `0..p-1`）
  - `p`：奇素数模数
- 输出：
  - 若有解，返回一组根：`{r, p-r}`（当 `n=0` 时只有 `0`）
  - 若无解，返回空集

必要判定：

- 欧拉判别法：`n^((p-1)/2) ≡ 1 (mod p)` 当且仅当 `n` 是非零二次剩余。
- 若判定值为 `-1`（即 `p-1`），则同余无解。

## R03

提出背景：

- 在密码学、编码理论和计算数论里，经常需要把“平方”反解回“平方根”。
- 与 `gcd`、`pow mod` 这类基础操作不同，模平方根在一般模数下并不总是存在，也没有统一的闭式解。
- 对奇素数模，Tonelli（1891）和 Shanks（1973）给出了高效可实现流程，使该问题从“特殊技巧”变成了稳定算法组件。

## R04

时间复杂度（奇素数模）：

- 前置判定（Legendre 符号，快速幂）：`O(log p)` 次模乘。
- 主循环复杂度常写为 `O(log^2 p)` 到 `O(log^3 p)`（按位复杂度模型会包含大整数乘法成本）。
- 工程上通常可以理解为“若干次快速幂 + 有界迭代”，在 64 位或中等大整数场景表现稳定。

直观上，循环次数由 `p-1 = q * 2^s` 中的 `s` 控制；每轮都会让 `m` 下降，因此会终止。

## R05

空间复杂度：

- 额外状态仅包含 `q, s, z, c, t, r, m` 等常数个整数变量。
- 因此可记为 `O(1)` 额外空间（不计输入输出与大整数对象自身存储）。

## R06

示例：求 `x^2 ≡ 5 (mod 41)`。

1. 分解 `p-1`：`40 = 5 * 2^3`，故 `q=5, s=3`。  
2. 找二次非剩余 `z`，取 `z=3`（可验 `3^20 ≡ -1 mod 41`）。  
3. 初始化：
   - `c = z^q = 3^5 mod 41 = 38`
   - `t = n^q = 5^5 mod 41 = 9`
   - `r = n^((q+1)/2) = 5^3 mod 41 = 2`
   - `m = s = 3`
4. 因 `t != 1`，找最小 `i` 使 `t^(2^i)=1`：
   - `9^2 = 40 != 1`
   - `9^4 = 1`，故 `i=2`
5. 计算 `b = c^(2^(m-i-1)) = c^(2^0)=38`。
6. 第一次更新：
   - `r = r*b = 2*38 mod 41 = 35`
   - `t = t*b^2 = 9*(38^2) mod 41 = 40`
   - `c = b^2 = 38^2 mod 41 = 9`
   - `m = i = 2`
7. 第二轮中 `t=40 != 1`，此时最小 `i=1`，取
   - `b = c^(2^(m-i-1)) = c^(2^0)=9`
   - `r = 35*9 mod 41 = 28`
   - `t = 40*9^2 mod 41 = 1`
8. `t==1` 停止，得到根 `r=28`，另一根 `41-28=13`。  
9. 验证：`28^2 mod 41 = 5`，`13^2 mod 41 = 5`。

## R07

算法意义：

- 在数论中，它是“二次剩余理论 -> 可计算算法”的代表案例。
- 在工程中，它出现在 ECC、同余方程构造、哈希到曲线、零知识证明相关原语中。
- 在教学上，它体现了“群结构 + 幂阶分解 + 迭代校正”的算法设计方法。

## R08

前置知识与依赖：

- 快速模幂 `pow(a, e, p)`。
- 欧拉判别法 / Legendre 符号。
- 模乘群 `F_p^*` 的阶为 `p-1`。
- 将 `p-1` 分解为 `q * 2^s`（`q` 为奇数）。
- 基础同余运算与模逆概念（本算法主流程通常不显式求逆）。

## R09

适用边界：

- 标准 Tonelli-Shanks 假设 `p` 是奇素数。
- 若 `p=2`，可直接返回 `n mod 2`。
- 若模数是奇素数幂或一般合数，需要 Hensel 提升或 CRT 组合，不是本 MVP 的范围。
- 若输入 `n` 不是二次剩余，应明确返回“无解”，而不是给出伪根。

## R10

正确性依据（概要）：

- 在 `F_p^*` 中，元素阶整除 `p-1 = q*2^s`。
- 初始化后维护不变量：`r^2 ≡ n*t (mod p)`。
- 当 `t != 1` 时，定位其 `2`-幂阶位置 `i`，用 `b` 调整 `r, t, c`，使 `t` 的 `2`-幂阶严格下降。
- 因 `m` 递减且下界有限，迭代终止于 `t=1`。
- 终态下不变量给出 `r^2 ≡ n (mod p)`，故 `r` 是所求平方根。

## R11

数值稳定性与误差传播：

- 全流程是整数模运算，不涉及浮点近似，不存在传统数值分析中的舍入误差累计。
- 主要风险来自实现错误：
  - 没有先做二次剩余判定；
  - `i` 搜索边界写错导致死循环；
  - 更新顺序错误破坏不变量。
- Python 大整数可避免固定宽度整型溢出问题。

## R12

真实成本（工程视角）：

- 核心成本在快速幂；Python 的三参 `pow` 已高度优化。
- 搜索二次非剩余 `z` 通常很快，平均尝试次数小（约常数级）。
- 当 `p-1` 含较高 `2`-幂（`s` 较大）时，循环轮数和轮内检查会增加，但仍可控。
- 对非常大的素数（密码学级别）仍建议依赖成熟大整数库与常数时间实现策略。

- 不是近似算法、不是 Monte Carlo 算法。

## R14

鲁棒性与失效模式：

- `p` 非素数：Legendre 判定与群结构前提失效，结果不可信。
- `p` 为偶数且未特判：会触发不正确行为。
- `n` 非二次剩余：若不提前判定，主循环可能异常或给出伪值。
- 实现错误（尤其 `b, c, t, r` 更新顺序）会破坏不变量，导致错误结果。

## R15

工程实现要点：

- 先做输入归约：`n %= p`。
- 快速分支：
  - `n == 0` -> 根为 `0`
  - `p == 2` -> 根为 `n`
  - `p % 4 == 3` -> 直接 `r = n^((p+1)/4) mod p`
- 通用分支使用 Tonelli-Shanks；找 `i` 时必须保证 `0 < i < m`。
- 输出根建议排序并去重，便于测试与对比。

## R16

前驱/后继关系：

- 前驱知识：
  - 欧拉准则与 Legendre 符号
  - 快速模幂
- 后继算法与应用：
  - Cipolla 算法（另一种模平方根通用方案）
  - Hensel lifting（扩展到 `p^k`）
  - CRT（扩展到合数模）
  - ECC 点恢复、密码协议中的二次剩余子过程

## R18

源码级算法流（对应本条目 `demo.py`，无第三方黑箱，8 步）：

1. `sqrt_mod_prime` 对输入做边界处理：`p==2`、`n%p==0`、`p` 素性检查。  
2. 调用 `legendre_symbol(n, p)` 执行欧拉判别；若为 `-1` 直接返回无解。  
3. 若 `p % 4 == 3`，走特例公式 `r = n^((p+1)/4) mod p`，立即构造双根。  
4. 否则在 `tonelli_shanks_one_root` 中分解 `p-1 = q * 2^s`（`q` 奇数）。  
5. 线性扫描找到最小二次非剩余 `z`，并初始化 `c=z^q, t=n^q, r=n^((q+1)/2), m=s`。  
6. 循环直到 `t==1`：搜索最小 `i` 使 `t^(2^i)==1`，据此计算修正因子 `b=c^(2^(m-i-1))`。  
7. 用 `b` 同步更新 `r, t, c, m`，保持 `r^2 ≡ n*t (mod p)` 且让 `t` 的 `2`-幂阶下降。  
8. 循环结束返回 `r`，上层函数再组合为 `(r, p-r)` 并排序去重；`main` 用暴力枚举结果做一致性验证。
