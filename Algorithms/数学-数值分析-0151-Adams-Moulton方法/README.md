# Adams-Moulton方法

- 源序号: `151`

## R01

本条目实现 Adams-Moulton 方法的最小可运行版本，聚焦最经典的一步隐式形式（梯形公式）：

- 常微分方程初值问题 `y' = f(t, y), y(t0)=y0`；
- 使用显式预测 + 隐式校正（固定点迭代）完成每一步推进；
- 输出数值解、隐式方程残差、校正迭代次数，并与解析解和显式 Euler 做对照。

目标是把“隐式多步思想如何落地到可执行代码”讲清楚，而不是依赖黑盒求解器。

## R02

问题定义（本 MVP）：

- 输入：
  - 右端函数 `f(t, y)`；
  - 初值 `t0, y0`；
  - 步长 `h` 与步数 `steps`；
  - 隐式校正参数 `tol, max_iter`。
- 输出：
  - 时间网格 `t[0..N]`；
  - 数值近似 `y[0..N]`；
  - 每步诊断信息（预测值、校正值、隐式残差、校正迭代次数）。

演示方程选为：

`y' = y - t^2 + 1, y(0)=0.5, t in [0, 2]`

其解析解为：

`y(t) = (t+1)^2 - 0.5*exp(t)`。

## R03

核心数学公式：

Adams-Moulton（一步/梯形）写为

`y_{n+1} = y_n + h/2 * [f(t_n, y_n) + f(t_{n+1}, y_{n+1})]`

这里 `y_{n+1}` 同时出现在等式左右，故为隐式格式。

为求 `y_{n+1}`，本实现使用固定点迭代：

`z_{k+1} = y_n + h/2 * [f(t_n, y_n) + f(t_{n+1}, z_k)]`

并以显式 Euler 预测值作为初始 `z_0`：

`z_0 = y_n + h * f(t_n, y_n)`。

当 `h` 足够小且 `f` 对 `y` 满足局部 Lipschitz 条件时，校正迭代通常快速收敛。

## R04

算法流程（单步）：

1. 已知 `(t_n, y_n)`，计算 `f_n = f(t_n, y_n)`。
2. 用显式 Euler 得到预测值 `y_pred = y_n + h*f_n`。
3. 令 `z = y_pred`，进入固定点迭代。
4. 反复更新 `z <- y_n + h/2*(f_n + f(t_{n+1}, z))`。
5. 若相邻两次迭代差值满足阈值，视为该步收敛。
6. 计算隐式残差 `|z - y_n - h/2*(f_n + f(t_{n+1}, z))|`。
7. 令 `y_{n+1}=z`，推进到下一步。

## R05

数据结构设计：

- `numpy.ndarray t, y`：保存时间网格与数值解；
- `StepDiagnostic`（`dataclass`）：
  - `step, t_n, y_n`
  - `y_predict, y_corrected`
  - `correction_iters`
  - `implicit_residual`
- `list[StepDiagnostic]`：保存逐步诊断，便于检查隐式求解行为。

这样既能保持实现简洁，也能提供足够可解释性。

## R06

正确性要点：

- 公式层面：代码严格按 Adams-Moulton 梯形公式离散；
- 隐式求解层面：每步通过不动点迭代近似求解 `y_{n+1}`；
- 终值验证层面：与解析解比较误差；
- 方法特性层面：在脚本中对比 `h=0.2/0.1/0.05` 时误差缩减速率，可观察到接近二阶收敛（Euler 接近一阶）。

## R07

复杂度分析：

- 设总步数为 `N`，每步平均校正迭代次数为 `K`；
- 时间复杂度：`O(N*K)`；
- 空间复杂度：
  - 存全轨迹与诊断：`O(N)`；
  - 仅保留当前步可降到 `O(1)`（本 MVP 为可观测性保留 `O(N)`）。

## R08

边界与异常处理：

- `h <= 0`、`steps <= 0`、`tol <= 0`、`max_iter <= 0` 直接抛 `ValueError`；
- `t0` 或 `y0` 非有限数抛 `ValueError`；
- 若校正迭代出现 `nan/inf` 抛 `RuntimeError`；
- 若单步迭代在 `max_iter` 内不收敛抛 `RuntimeError`。

这保证脚本在失败时“显式失败”，而不是静默返回错误数值。

## R09

MVP 取舍说明：

- 采用一步 Adams-Moulton（梯形）而非高阶多步，以降低启动复杂度；
- 采用固定点迭代而非牛顿法，减少导数依赖并保持实现短小；
- 选择带解析解的测试方程，方便直接检查全局误差；
- 仅实现标量 ODE，先保证核心路径可靠、可读。

## R10

`demo.py` 函数职责：

- `check_inputs`：校验迭代控制参数；
- `adams_moulton_trapezoid`：主求解器（预测 + 隐式校正）；
- `explicit_euler`：基线方法用于误差对比；
- `rhs_demo`：示例方程右端；
- `exact_solution_demo`：解析解；
- `safe_order`：根据误差估算收敛阶；
- `print_diagnostics`：打印每步校正诊断；
- `run_demo`：组织完整实验；
- `main`：脚本入口，无需交互输入。

运行后会自动输出：

## R12

输出字段解读：

- `y_AM(T)`：Adams-Moulton 在终点 `T` 的数值解；
- `y_Euler(T)`：显式 Euler 基线解；
- `y_exact(T)`：解析解；
- `|AM-exact|` / `|Euler-exact|`：绝对误差；
- `mean corr iters`：每步固定点平均迭代次数；
- `max implicit residual`：隐式方程残差最大值；
- `AM p≈...`：步长减半后的观测收敛阶。

常见现象：AM 误差显著小于 Euler，且阶数更高。

## R13

建议最小测试集：

1. 标准测试：`y' = y - t^2 + 1, y(0)=0.5, T=2`，`h in {0.2, 0.1, 0.05}`。
2. 稳定性观察：适度增大 `h`（如 `0.4`）看校正迭代次数与残差变化。
3. 参数鲁棒性：
   - 缩小 `tol`（`1e-10 -> 1e-12`）；
   - 减小 `max_iter`（例如设成 `2`，应触发不收敛异常）。
4. 输入异常：
   - `h <= 0`、`steps <= 0`；
   - 非有限 `t0/y0`。

## R14

关键参数建议：

- `h`：时间步长，决定离散误差与计算量；
- `steps`：总步数，通常与 `h` 联动（`steps=(T-t0)/h`）；
- `tol`：隐式校正停止阈值，默认 `1e-12`；
- `max_iter`：每步最大校正轮数，默认 `20`。

实践上建议先固定 `tol/max_iter`，再扫描不同 `h` 观察误差-成本曲线。

## R15

与常见方法对比：

- 对比显式 Euler：
  - Euler 一阶、实现最简单；
  - Adams-Moulton 梯形二阶，精度更高，但每步需解隐式方程。
- 对比 Adams-Bashforth（显式多步）：
  - AB 不需隐式求解，单步更便宜；
  - AM 通常稳定性更好，在刚性问题上更有优势。
- 对比黑盒库求解器（如 `scipy.integrate.solve_ivp`）：
  - 黑盒上手快；
  - 本实现更透明，适合教学、调试和算法理解。

## R16

典型应用场景：

- 数值分析课程中讲解隐式多步法；
- 作为更高阶 PECE 预测-校正器的基础模块；
- 需要在精度与稳定性间折中的工程 ODE 子问题；
- 为后续刚性系统求解（BDF/隐式 Runge-Kutta）做概念铺垫。

## R17

可扩展方向：

- 扩展到高阶 Adams-Moulton（AM3/AM4）并加入启动策略（如 RK4）；
- 将固定点迭代替换为牛顿/拟牛顿，提高强非线性场景鲁棒性；
- 支持向量状态 `y in R^d` 与 Jacobian 近似；
- 加入自适应步长控制（局部误差估计 + 步长调整）；
- 与 `scipy` 求解器做系统误差/性能基准。

## R18

源码级算法流（对应 `demo.py`，8 步）：

1. `main` 调用 `run_demo`，设定 `t0, y0, T, h` 列表及 `tol/max_iter`。  
2. 对每个步长，进入 `adams_moulton_trapezoid`，先由 `check_inputs` 完成参数合法性检查。  
3. 在每个时间步 `n`，先计算 `f_n = f(t_n, y_n)`，并用显式 Euler 生成预测值 `y_pred`。  
4. 以 `y_pred` 为初值执行固定点校正：`z <- y_n + h/2*(f_n + f(t_{n+1}, z))`。  
5. 若 `|z_k - z_{k-1}|` 达到阈值则收敛；超出最大迭代轮数则抛出异常。  
6. 收敛后计算隐式方程残差，写入 `StepDiagnostic`，并接受 `y_{n+1}=z`。  
7. 全部步推进结束后，`run_demo` 同时调用 `explicit_euler` 和 `exact_solution_demo` 计算误差。  
8. 使用 `safe_order` 估计阶数并输出结果，展示 AM 相对 Euler 的精度优势与收敛特征。
