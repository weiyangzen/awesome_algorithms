## R01

ADMM（Alternating Direction Method of Multipliers，交替方向乘子法）是一类用于**可分解凸优化**的算法。它把原问题拆成多个子问题，交替优化各子变量，并通过拉格朗日乘子（对偶变量）把子问题耦合起来，在分解效率与收敛稳定性之间取得平衡。

## R02

典型形式是：

\[
\min_{x,z} f(x) + g(z), \quad \text{s.t. } Ax + Bz = c
\]

其中 `f` 与 `g` 分别可能含有不同结构（例如一个平滑、一个非平滑）。ADMM 的优势在于：`x` 子问题和 `z` 子问题常常都能高效求解（闭式解、阈值算子或低成本线性代数）。

## R03

本目录中的 MVP 选择稀疏回归 LASSO：

\[
\min_x \frac{1}{2}\|Ax-b\|_2^2 + \lambda\|x\|_1
\]

通过变量分裂写成：

\[
\min_{x,z} \frac{1}{2}\|Ax-b\|_2^2 + \lambda\|z\|_1, \quad \text{s.t. } x-z=0
\]

这样 `x` 更新是二次优化，`z` 更新是软阈值，恰好适合 ADMM。

## R04

对约束 `x-z=0` 构造 scaled 增广拉格朗日：

\[
\mathcal{L}_\rho(x,z,u)=\frac{1}{2}\|Ax-b\|_2^2 + \lambda\|z\|_1 + \frac{\rho}{2}\|x-z+u\|_2^2 - \frac{\rho}{2}\|u\|_2^2
\]

其中 `u` 是 scaled dual variable，`rho > 0` 是罚参数。`rho` 越大，原始可行性通常收敛更快，但子问题数值性质也会变化。

## R05

LASSO 场景下的 ADMM 迭代：

1. `x` 更新（线性方程）：
\[
x^{k+1}=(A^TA+\rho I)^{-1}(A^Tb + \rho(z^k-u^k))
\]
2. `z` 更新（软阈值）：
\[
z^{k+1}=S_{\lambda/\rho}(x^{k+1}+u^k)
\]
3. `u` 更新（对偶上升）：
\[
u^{k+1}=u^k + x^{k+1}-z^{k+1}
\]

其中 `S_kappa(v)=sign(v)·max(|v|-kappa,0)`。

## R06

停止准则采用 primal/dual residual：

- 原始残差：`r^k = x^k - z^k`
- 对偶残差：`s^k = rho (z^k - z^{k-1})`

满足以下条件时停止：

- `||r^k||_2 <= eps_pri`
- `||s^k||_2 <= eps_dual`

`eps_pri`、`eps_dual` 由绝对阈值 `abs_tol` 与相对阈值 `rel_tol` 组合定义。

## R07

收敛直觉：

- `x-z=0` 由增广项与对偶更新共同驱动，逐步逼近可行性。
- `z` 的软阈值把稀疏先验直接注入迭代。
- 在闭凸条件下，ADMM 对原问题具有良好收敛性质；本例中 `f` 为二次凸，`g` 为 `L1` 范数，满足常见使用前提。

## R08

复杂度（本实现）：

- 预计算：Cholesky 分解 `A^TA + rho I`，约 `O(n^3)`（一次）
- 每次迭代：两次三角求解 `O(n^2)` + 向量阈值 `O(n)`

当迭代次数较多时，一次分解、重复回代通常比每轮重新求逆更高效也更稳定。

## R09

超参数建议：

- `lambda`：越大越稀疏，偏差也会增大。
- `rho`：常用 `0.1 ~ 10` 之间试探；过小可能 primal 收敛慢，过大可能 dual 收敛慢。
- `abs_tol`、`rel_tol`：决定停止精度。MVP 默认 `1e-4` 与 `1e-3`，兼顾速度和演示稳定性。

## R10

`demo.py` 使用可复现的合成数据：

- `A`：高斯随机设计矩阵（按 `1/sqrt(m)` 缩放）
- `x_true`：仅少数非零（稀疏真值）
- `b = A x_true + noise`

目标是检验 ADMM 是否能恢复稀疏结构并降低目标函数。

## R11

脚本输入输出：

- 输入：无命令行交互输入（脚本内部生成数据）
- 输出：
  - 迭代次数与是否收敛
  - 目标函数起止值
  - primal/dual 残差
  - 与真值的 MSE
  - 支持集（非零索引）重叠情况
  - 估计系数绝对值 Top-10

## R12

```bash
uv run python demo.py
```

该命令应直接完成一次 ADMM 求解并打印统计信息。

## R13

结果解读要点：

- `objective` 通常整体下降或趋于稳定。
- `primal_res` 与 `dual_res` 同时降低并触发阈值，说明可行性与对偶一致性在改善。
- `support overlap` 越高，表示稀疏结构恢复越好。
- `MSE` 越低，表示参数估计更接近真实稀疏向量。

## R14

本 MVP 的正确性保护：

- 固定随机种子，确保可复现。
- 使用 Cholesky + 三角求解，避免显式矩阵求逆。
- 每轮记录目标值与双残差，便于诊断是否收敛异常。
- 使用 `z`（阈值后变量）作为最终稀疏解，符合变量分裂语义。

## R15

常见问题：

- 若 `lambda` 太小：结果会变稠密，噪声被引入。
- 若 `lambda` 太大：系数被过度压缩，欠拟合。
- 若 `rho` 不合适：可能表现为某类残差下降慢。
- 若矩阵病态：线性子问题数值稳定性下降，可能需要标准化或预处理。

## R16

可扩展方向：

- 自适应 `rho`（基于 primal/dual 残差比值调整）。
- 替换为 Group LASSO、Total Variation 等不同正则项。
- 迁移到约束更复杂的形式 `Ax + Bz = c`。
- 在大规模稀疏矩阵下使用更高效的线性求解器。

## R17

与其他方法对比：

- 相比 ISTA/FISTA：ADMM 通过分裂把不可微项单独处理，子问题结构清晰。
- 相比纯次梯度法：ADMM 的迭代通常更稳定，收敛质量更好。
- 相比内点法：ADMM 在大规模场景更易分布式/并行化，单步成本更可控。

## R18

`demo.py` 的源码级算法流程（非黑箱）可拆为 8 步：

1. `build_synthetic_problem` 生成 `(A, b, x_true)`，构造稀疏回归测试问题。  
2. `solve_lasso_admm` 初始化 `x, z, u`，并预计算 `AtA, Atb`。  
3. 计算并缓存 `L = chol(AtA + rho I)`，为后续每轮 `x` 更新做快速线性求解。  
4. 每轮先做 `x` 子问题：解 `L L^T x = Atb + rho (z-u)`。  
5. 做 `z` 子问题：对 `x+u` 应用 `soft_threshold`，得到稀疏化后的 `z`。  
6. 做对偶更新：`u <- u + x - z`，累积约束违背信息。  
7. 计算 `objective`、`primal_res`、`dual_res` 与阈值，写入 `history` 并判断停止。  
8. `main` 用 `result.z` 输出 MSE、支持集重叠、Top-10 系数，完成可运行 MVP 验证。  

这 8 步完整对应 ADMM 的“分裂-交替-对偶校正-残差收敛”机制。
