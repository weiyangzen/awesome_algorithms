# 自动微分 - 反向模式

- UID: `MATH-0144`
- 学科: `数学`
- 分类: `数值分析`
- 源序号: `144`
- 目标目录: `Algorithms/数学-数值分析-0144-自动微分_-_反向模式`

## R01

本条目实现“自动微分（Automatic Differentiation, AD）- 反向模式”的最小可运行版本（MVP）。

目标：
- 用显式计算图 + 链式法则实现标量输出函数的梯度计算；
- 展示“前向构图、反向传播梯度”的完整可解释流程；
- 对比解析梯度与 PyTorch 自动求导结果，验证实现正确性。

## R02

问题定义（数值计算视角）：
- 输入：实数变量 `x, y, z`，以及由基本运算组合得到的标量目标函数 `f(x,y,z)`；
- 输出：
  - 函数值 `f`；
  - 一阶偏导 `df/dx, df/dy, df/dz`；
- 约束：
  - 不做符号推导；
  - 不用有限差分近似梯度作为主算法；
  - 梯度通过反向模式自动微分一次反传得到。

## R03

数学基础：

1) 计算图
- 每个中间变量是图上的一个节点；
- 每条边表示局部依赖关系（父节点到子节点）。

2) 局部导数
- 对每个基本算子（`+`、`*`、`pow`、`exp`、`sin`）预先写出局部梯度公式。

3) 链式法则
- 若 `u -> v -> f`，则 `df/du = (df/dv) * (dv/du)`；
- 在反向模式中，从输出节点设置 `df/df = 1`，沿拓扑逆序把梯度累积到每个前驱节点。

4) 反向模式优势
- 对“多输入、单输出”场景，一次反传就能拿到所有输入梯度；
- 代价约为常数倍前向计算成本。

## R04

算法总览（MVP）：
1. 用 `Value` 节点对象封装数值、梯度、父节点、算子类型和反传闭包；
2. 前向阶段：执行表达式计算，同时自动构建计算图；
3. 调用 `backward()`：
   - 先做 DFS 得到拓扑序；
   - 将目标节点梯度置为 `1.0`；
   - 按逆拓扑顺序执行各节点的 `_backward`，把梯度累加回父节点；
4. 读取输入节点梯度作为最终结果；
5. 与解析解和（可用时）PyTorch 结果比较误差。

## R05

核心数据结构：
- `Value` 类：
  - `data: float` 当前节点数值；
  - `grad: float` 当前节点梯度缓存；
  - `_prev: set[Value]` 前驱节点集合；
  - `_op: str` 由哪个算子产生；
  - `_backward: Callable[[], None]` 节点局部反传逻辑；
- `GradientReport`（`dataclass`）：保存单次实验的函数值、梯度和误差；
- `dict[str, float]`：用于按变量名保存梯度结果。

## R06

正确性要点：
- 每个基础算子都实现了正确的局部导数；
- `backward()` 使用拓扑逆序，保证某节点反传前其“上游梯度”已就绪；
- 梯度采用“累加”而非覆盖，适配一个节点被多条路径复用的情况；
- 通过解析梯度和 PyTorch 梯度进行数值交叉验证，误差应接近浮点舍入级别。

## R07

复杂度分析：
- 设计算图节点数 `V`、边数 `E`；
- 前向计算：`O(V)`；
- 拓扑构建（DFS）：`O(V + E)`；
- 反向传播：每条边至多处理一次，`O(E)`；
- 总体：`O(V + E)` 时间，`O(V + E)` 存图空间。

在表达式图较大时，主要成本来自中间节点存储与反传遍历。

## R08

边界与异常处理：
- 幂运算仅在当前示例中使用整数幂（避免负数底数与非整数幂导致复数分支）；
- 反向传播默认针对标量输出，若扩展到向量输出需显式提供上游梯度；
- 演示中使用 `float64`（含 PyTorch 对比）减少数值误差；
- 若本地未安装 PyTorch，脚本不会失败，只跳过该项对比并给出提示。

## R09

MVP 取舍：
- 采用纯 Python 手写微型 AD 引擎，突出算法机制，不依赖复杂框架；
- 支持的算子故意保持小集合（`+,-,*,/,pow,exp,sin`）；
- 不实现张量广播、批量图优化、内存复用等工程特性；
- 重点是“反向模式如何工作”，而不是工业级训练框架复刻。

## R10

`demo.py` 函数职责：
- `Value`：核心节点类，封装运算重载与 `backward()`；
- `target_function_value` / `target_function_float`：定义同一目标函数在计算图输入与纯浮点输入下的实现；
- `analytic_gradients`：返回解析梯度真值；
- `evaluate_with_engine`：运行手写反向模式并返回值与梯度；
- `try_evaluate_with_torch`：可选 PyTorch 交叉验证；
- `run_gradient_checks`：批量随机点检查误差；
- `main`：组织并打印完整演示结果。

## R15

方法对比：
- 与数值微分（有限差分）相比：
  - AD 精度高（避免差分步长折中）；
  - 计算更稳定，且通常更快。
- 与符号微分相比：
  - AD 不会出现表达式爆炸；
  - 更适合程序化计算流程。
- 与前向模式 AD 相比：
  - 在“输入维度大、输出标量”时，反向模式更高效；
  - 前向模式更适合“输入少、输出多”的雅可比列构建。

## R18

源码级算法流（对应 `demo.py`，8 步）：
1. `evaluate_with_engine` 将输入 `x,y,z` 封装为 `Value` 叶子节点，初始 `grad=0`。  
2. 调用 `target_function_value` 进行前向计算；每次算子重载（如 `__mul__`、`sin`、`exp`）都会创建新 `Value` 节点，并记录父节点 `_prev` 与本地 `_backward` 规则。  
3. 前向结束后得到标量输出节点 `out`，此时计算图已完整建立。  
4. `out.backward()` 先用 DFS 生成拓扑序，确保每个节点只访问一次。  
5. 将 `out.grad` 设为 `1.0`，作为链式法则起点 `df/df=1`。  
6. 按逆拓扑顺序依次执行节点 `_backward`：例如乘法节点把 `out.grad` 按局部导数分别分配给两个输入并累加。  
7. 当反传到叶子节点后，`x.grad / y.grad / z.grad` 即为目标函数对各输入的偏导。  
8. `analytic_gradients` 与 `try_evaluate_with_torch` 分别提供解析与框架级对照，输出误差并完成可验证闭环。
