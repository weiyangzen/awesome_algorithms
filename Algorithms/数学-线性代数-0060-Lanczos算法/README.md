## R01

Lanczos 算法是 Arnoldi 迭代在“实对称/厄米矩阵”上的特化版本。

给定对称矩阵 `A in R^{n x n}` 与非零初始向量 `b`，它在 Krylov 子空间
`K_m(A,b)=span{b,Ab,...,A^{m-1}b}` 上构造正交基 `Q_m`，并得到一个三对角投影矩阵 `T_m`，满足：

- `Q_m^T Q_m = I`
- `T_m = Q_m^T A Q_m`（对称三对角）
- `A Q_m = Q_m T_m + beta_m q_{m+1} e_m^T`

`T_m` 的特征值称为 Ritz 值，可近似 `A` 的特征值。

## R02

历史定位（简要）：

- C. Lanczos 在 1950 年前后提出该思想，用于把大规模对称特征问题降到小规模三对角问题；
- 该算法是现代大规模特征值求解器（如 ARPACK 的对称分支、LOBPCG 对比链路）的重要基础；
- 共轭梯度法（CG）与 Lanczos 在对称正定场景下有深层等价关系（同属 Krylov 子空间方法）。

## R03

本条目 MVP 解决的问题：

- 输入：
  - 对称实矩阵 `A`；
  - 初始向量 `b`；
  - 迭代步数 `m`；
  - breakdown 阈值 `tol`；
- 输出：
  - Lanczos 基 `Q_m`；
  - 三对角矩阵 `T_m`；
  - 正交性误差、Lanczos 关系残差；
  - `Q_m^T A Q_m` 与 `T_m` 的偏差；
  - Ritz 值与参考特征值的匹配误差。

## R04

时间复杂度（稠密矩阵）：

- 每步一次 `A @ q_k`，代价 `O(n^2)`；
- 三项递推本身仅 `O(n)`；
- 若启用全重正交（本 MVP 默认启用），第 `k` 步约 `O(kn)`；
- 总体约 `O(m n^2 + m^2 n)`，通常仍由 matvec 主导。

若 `A` 稀疏，matvec 可降为 `O(nnz(A))`，总体近似 `O(m*nnz(A) + m^2 n)`。

## R05

空间复杂度：

- 存储 `Q_m`：`O(nm)`；
- 存储三对角系数 `alpha/beta` 与 `T_m`：`O(m)` 到 `O(m^2)`；
- 输入矩阵稠密存储：`O(n^2)`。

若只看算法附加内存（不含 `A`），主要是 `O(nm)`。

## R06

微型直觉示例（二维对称矩阵）：

令
`A = [[2,1],[1,2]]`，`b=[1,0]^T`。

- 第一步从 `q1 = b/||b||` 出发，计算 `Aq1`，抽取沿 `q1` 的分量得到 `alpha1`；
- 把剩余分量归一化得到 `q2`，其系数是 `beta1`；
- 第二步同理得到 `alpha2`，二维情况下很快闭合。

最终 `T_2 = [[alpha1,beta1],[beta1,alpha2]]`，其特征值就是对 `A` 特征值的近似（在该小例中可做到精确）。

## R07

算法意义：

- 把“大规模对称特征分解”转化为“小规模三对角特征分解”；
- 三对角结构便于快速 `eigh`，并可聚焦极值特征对；
- 是许多工业级大规模谱分析管线的核心内核之一。

## R08

核心递推（对称情形）可写为：

1. `q1 = b / ||b||`，`beta0 = 0`，`q0 = 0`
2. 对 `k = 1..m`：
   - `z = A qk - beta_{k-1} q_{k-1}`
   - `alpha_k = qk^T z`
   - `z = z - alpha_k qk`
   - （数值实现中可加入重正交）
   - `beta_k = ||z||`
   - 若 `beta_k` 很小则 breakdown
   - 否则 `q_{k+1} = z / beta_k`

由此构造 `T_m = tridiag(beta_1..beta_{m-1}, alpha_1..alpha_m, beta_1..beta_{m-1})`。

## R09

适用条件与局限：

适用：

- `A` 为实对称或复厄米；
- 目标是少量特征值/特征向量（特别是极值端）；
- 能高效执行 matvec。

局限：

- 对非对称矩阵不适用（应回到 Arnoldi）；
- 有限精度下会出现“幽灵特征值（ghost Ritz values）”，需重正交或重启策略；
- 若谱聚簇严重，收敛速度可能受影响。

## R10

正确性要点（工程可验证版）：

1. 每一步将 `Aq_k` 去除 `q_{k-1}, q_k` 方向分量并归一化，形成新基向量；
2. 在精确算术中，`q_i` 两两正交，得到正交基 `Q_m`；
3. `Q_m^T A Q_m` 必为对称三对角，正是 `T_m`；
4. 组合列关系得到 `A Q_m = Q_m T_m + beta_m q_{m+1} e_m^T`；
5. `demo.py` 中显式计算上述误差指标做数值验证。

## R11

数值稳定性：

- 理论三项递推在浮点中会逐步丢失全局正交性；
- 本 MVP 默认启用“两次全重正交”，提升稳健性；
- 使用 `beta <= tol` 检测 breakdown，避免除零；
- 对长期迭代，建议结合重启与选择性重正交控制成本。

## R12

性能视角：

- 热点仍是 `A @ q`；
- `T_m` 只有三对角信息，后续谱计算代价低；
- 当仅需前几个特征值时，`m << n` 即可获得可用近似；
- 本 MVP 为可读性保留显式步骤，便于调试和审计。

## R13

本目录 MVP 的可验证保证：

- 固定构造对称测试矩阵与初值，运行可复现；
- 自动输出并检查：
  - `||Q^TQ - I||`（正交性）
  - `||A Q - Q T - beta*q_next*e_m^T||`（Lanczos 关系残差）
  - `||Q^T A Q - T||`（投影一致性）
  - `Q^T A Q` 的非三对角元素最大值
  - Ritz 值与 `A` 参考特征值的最大最小距离
- 通过后打印 `All checks passed.`。

## R14

常见失效模式：

- `A` 非方阵或非对称；
- `b` 为零向量，无法归一化；
- `m <= 0` 或阈值不合法；
- 未做重正交时，长迭代后正交性退化导致谱污染；
- breakdown 提前出现（Krylov 子空间已封闭）。

## R15

实现设计（`demo.py`）：

- `LanczosResult`：统一承载基、三对角矩阵和诊断指标；

- `off_tridiagonal_violation`、`max_min_eigen_distance`：结构和谱误差评估；
- `run_checks`：集中断言；
- `main`：执行实验并打印报告。

## R16

相关算法链路：

- 非对称扩展：Arnoldi 迭代；
- 线性方程组：CG 与 Lanczos 在 Krylov 结构上高度相关；
- 特征求解工程实现：常与隐式重启、谱变换（shift-invert）结合；
- 大规模稀疏场景：可将 `A@x` 替换为算子形式，避免显式存储矩阵。

## R18

`demo.py` 的源码级算法流程（8 步，非黑盒）如下：

1. `main` 先构造确定性的对称矩阵 `A` 与起始向量 `b`，并设置 `m/tol`。  
2. `lanczos_iteration` 调用 `validate_inputs` 校验对称性、维度与数值范围，然后归一化得到 `q1`。  
3. 每轮执行核心 matvec：`z = A @ q_k - beta_{k-1} q_{k-1}`，这是 Krylov 扩展入口。  
4. 计算 `alpha_k = q_k^T z` 并做 `z <- z - alpha_k q_k`，得到与当前向量正交的候选残差。  
5. 对已生成基做两次全重正交（`basis.T @ z` 再回代），抑制浮点误差造成的正交性损失。  
6. 令 `beta_k = ||z||`；若 `beta_k <= tol` 则判定 breakdown，否则归一化得到 `q_{k+1}`。  
7. 用 `alpha/beta` 显式组装三对角 `T_m`，并计算 `||Q^T A Q - T||` 与 Lanczos 关系残差。  
8. 仅在验算阶段调用 `np.linalg.eigvalsh` 比较 Ritz 值与参考谱；Lanczos 主体迭代与递推完全由源码手写实现。
