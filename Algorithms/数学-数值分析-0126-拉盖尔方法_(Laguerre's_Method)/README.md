# 拉盖尔方法 (Laguerre's Method)

## R01

拉盖尔方法（Laguerre's Method）是用于求解多项式方程 `P(x)=0` 的迭代算法，
尤其适合在复数域中稳定地寻找单个根，再配合“降阶（deflation）”求出全部根。

给定 `n` 次多项式及一个初值 `x_k`，其核心更新写为：

- `G = P'(x_k) / P(x_k)`
- `H = G^2 - P''(x_k) / P(x_k)`
- `a = n / (G ± sqrt((n-1)*(n*H - G^2)))`
- `x_{k+1} = x_k - a`

其中分母取 `|G + ...|` 与 `|G - ...|` 中模更大的那个，以降低数值不稳定风险。

## R02

- 该方法与 Edmond Laguerre 相关，后续在 20 世纪数值分析文献中被系统用于多项式求根。
- 与 Newton 法相比，Laguerre 法在复平面上对高次多项式通常更稳健，常作为工程实现中的默认策略之一。
- 经典数值库（如历史上的 `Numerical Recipes` 路径）也常把它作为“多项式全根求解”的核心迭代器。

## R03

多项式求根是数值分析基础问题：

- 控制、信号处理、振动分析中，系统极点/特征值问题经常转化为多项式根。
- 仅用 Newton 法时，初值敏感且在复根附近容易出现收敛慢或振荡。
- Laguerre 法通过引入 `P'` 与 `P''` 的组合信息，对复根场景有更强鲁棒性，且可统一处理实根与复根。

## R04

设多项式次数为 `n`。

- 单次迭代需计算 `P, P', P''`，可用 Horner 扩展在 `O(n)` 完成。
- 若某个根在 `k` 次内收敛，则该根成本约 `O(k*n)`。
- 若通过逐次降阶求全部根，粗略总成本可写为：
  `O(k_avg * (n + (n-1) + ... + 1)) = O(k_avg * n^2)`。

其中 `k_avg` 为平均迭代步数，工程上常是较小常数到几十级别。

## R05

- 单根迭代仅保存常量个复数变量，额外空间 `O(1)`（不含输入系数）。
- 若要求全部根并保存结果，空间为 `O(n)`。
- 若实施降阶，需要一份当前多项式系数副本，空间总体仍为 `O(n)`。

## R06

示例多项式：

`P(x) = (x-1)(x+2)(x^2+1) = x^4 + x^3 - x^2 + x - 2`

其真根为：

- `x = 1`
- `x = -2`
- `x = i`
- `x = -i`

Laguerre 法可在复平面任意初值附近迭代到其中某个根；结合降阶可依次恢复全部 4 个根。

## R07

主要应用：

- 一般多项式全根求解（实根 + 复根混合）。
- 特征多项式分析（控制系统极点、离散系统稳定性）。
- 作为 Jenkins-Traub、伴随矩阵法之外的“轻量可实现”数值求根方案。
- 教学中用于展示“高阶导信息如何增强迭代法鲁棒性”。

## R08

前置知识依赖：

- 多项式与导数（`P'`, `P''`）。
- 复数运算与复平面距离（模长）。
- Horner 法（高效求值）。
- 基本迭代收敛判据（如 `|P(x)| < tol` 或步长足够小）。
- 降阶（synthetic division）用于从已知根构造低一阶多项式。

## R09

适用前提与边界：

- 输入应为一元多项式系数，最高次项系数不可为 0。
- 算法天然支持复数域，实系数多项式会产生共轭复根对。
- 对重根或近重根，收敛可能变慢，且降阶对舍入误差更敏感。
- 若初值极差或分母接近 0，需要重启/扰动策略避免停滞。

## R10

正确性直觉（工程版）：

1. 对单根附近，Laguerre 迭代可视作利用 `P, P', P''` 构造的局部高信息更新，通常具备强局部收敛性。
2. 每次若 `|P(x_k)|` 下降且步长收缩，则趋向某个根。
3. 当找到根 `r` 后，可把 `P(x)` 分解为 `(x-r)Q(x)` 并继续在 `Q` 上迭代。
4. 重复 `n-1` 次后，剩余一阶多项式可直接解析求最后一根。
5. 因此“单根收敛 + 逐次降阶”给出全根流程。

## R11

数值稳定性分析：

- 优点：分母选择 `|G±...|` 较大者，可减小“相消导致的大步长爆炸”。
- 风险：
  - 当 `P(x)` 非常接近 0 时，`G/H` 计算可能受舍入放大。
  - 重根附近 `P'(x)` 也接近 0，迭代速度可能下降。
  - 降阶会把先前根误差带入后续多项式。
- 工程缓解：
  - 使用复数双精度。
  - 增加最大迭代数与多初值重启。
  - 在原多项式上做一次 root polishing。

## R12

真实成本（超越渐近记号）：

- 热点几乎都在复数乘加（Horner 求值）与开方运算。
- 对中等次数（如 `n <= 50`）Python + NumPy 也可快速演示。
- 若需要极高精度或超高次数，常转向更专门的高精度/稳定库。
- 本条目定位“最小可运行 MVP”，优先可读与可验证，不追求工业最优常数。

## R13

收敛与误差性质（简述）：

- Laguerre 法在简单根附近通常表现为快速收敛（常优于纯 Newton 的稳健性）。
- 实务中可用以下双阈值停止：
  - `|P(x_k)| < tol_residual`
  - `|x_{k+1} - x_k| < tol_step`
- 全根求解后可用残差 `max_i |P(r_i)|` 作为统一质量指标。
- 对重根问题，残差可能先降后平台化，需要更宽容的容差与迭代预算。

## R14

鲁棒性与失效模式：

- 失效 1：分母接近 0，导致更新量异常。
  - 处理：注入微扰或更换初值重启。
- 失效 2：达到 `max_iter` 仍未收敛。
  - 处理：提高迭代上限，或采用多初值尝试。
- 失效 3：降阶余项过大，表明当前根精度不足。
  - 处理：先在原多项式上 polishing，再执行 deflation。
- 失效 4：系数尺度跨越过大。
  - 处理：先做系数归一化/缩放。

## R15

工程实现建议：

- 用扩展 Horner 一次得到 `P, P', P''`，避免重复遍历系数。
- 复数统一路径实现，避免“实数/复数两套分支”增加维护成本。
- 单根求解器与全根求解器解耦：
  - `laguerre_single_root` 负责局部迭代。
  - `find_all_roots_laguerre` 负责重启与降阶。
- 输出时增加：迭代次数、最大残差、与 `numpy.roots` 的对照误差。

## R18

源码级算法流程（对应 `demo.py`，非第三方黑箱）可拆为 9 步：

1. 读取多项式系数并转为复数数组，确认最高次项非零。
2. 在 `eval_poly_and_derivatives` 中用扩展 Horner 单次扫描得到 `P(x), P'(x), P''(x)`。
3. 在 `laguerre_single_root` 里计算 `G=P'/P` 与 `H=G^2-P''/P`。
4. 计算 `sqrt((n-1)*(n*H-G^2))`，并在 `G±sqrt(...)` 中选择模更大的分母。
5. 按 `x_{k+1}=x_k-n/denom` 更新；若步长或残差满足阈值则认为单根收敛。
6. 若单次尝试失败（如分母过小或迭代耗尽），更换初值并重试，直到找到可用根。
7. 对已收敛根执行 `deflate_polynomial`，把 `n` 次问题降为 `n-1` 次。
8. 重复步骤 2-7 直到剩下一阶多项式，解析得到最后一根。
9. 汇总全部根，计算 `max |P(r_i)|` 与 `numpy.roots` 对照误差，输出可复现实验结果。
