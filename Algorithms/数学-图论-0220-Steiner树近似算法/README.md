# Steiner树近似算法

## R01

Steiner 树问题：给定带权无向图 `G=(V,E,w)` 和终端点集合 `R ⊆ V`，寻找一棵连通所有终端点的最小权值子树。  
当允许使用 `V\R` 中的中间点（Steiner 点）时，该问题一般是 NP-hard。  
本条目实现的是经典近似框架（KMB 思路）：通过最短路度量闭包把问题转成终端点完全图上的 MST，再映射回原图并剪枝。

## R02

Steiner 树在组合优化、VLSI 布线、网络设计中非常经典。  
常见的图 Steiner 近似基线包括：
- Takahashi-Matsuyama 增长式启发（1980）；
- Kou-Markowsky-Berman（KMB）度量闭包 + MST（1981），近似比不超过 2。  

本目录 MVP 采用可解释性强、实现简单且有明确保证的 KMB 型流程。

## R03

要解决的核心矛盾：
1. 精确最优解计算复杂度高（一般图上 NP-hard）。
2. 工程上仍需要“可快速运行、质量有下界”的方案。  

近似算法通过牺牲最优性换取多项式时间，并给出可验证的上界（这里是 2-近似）。

## R04

KMB 近似主流程：
1. 对每个终端点运行最短路，得到终端两两距离与路径。
2. 构造终端点的度量闭包完全图 `K_R`，边权为原图最短路距离。
3. 在 `K_R` 上求 MST。
4. 将 MST 的每条“终端边”替换为原图中的对应最短路径，得到子图 `H`。
5. 在 `H` 上再求一次 MST（去掉路径叠加造成的环）。
6. 反复删除“非终端叶子”，得到 Steiner 近似树。

## R05

记 `n=|V|, m=|E|, k=|R|`。
- 最短路阶段（从 `k` 个终端点出发的 Dijkstra）：`O(k * m log n)`。
- 度量闭包边数 `O(k^2)`，其 MST 复杂度 `O(k^2 log k)`。
- 路径展开与二次 MST 主要受展开子图规模影响，通常低于最短路阶段。  

总体主导项通常是 `O(k * m log n)`，空间复杂度约 `O(m + k*n)`（存图和若干最短路结果）。

## R06

示例思路（简化）：
- 终端点：`{A, D, G, H}`。
- 在原图中计算终端点两两最短路后，得到 `K_R`。
- 在 `K_R` 上取 MST，例如边可能是 `(A,D),(D,G),(G,H)`。
- 把这些边映射回原图最短路径并合并，会得到一个含若干中间点的连通子图。
- 再做一次 MST + 非终端叶剪枝，得到最终近似 Steiner 树。  

本目录 `demo.py` 会输出完整边集和总权重，并与小图暴力最优值对照。

## R07

优点：
- 有明确近似保证（≤2）。
- 思路直观，可解释性强。
- 实现复杂度适中，便于工程落地。  

局限：
- 不是精确最优。
- 对终端数量很多且图很大时，最短路阶段会成为瓶颈。
- 若权重异常或图不连通，需要额外防御性检查。

## R08

前置知识：
- 图最短路（Dijkstra，非负权）。
- 最小生成树（Kruskal/Prim）。
- 并查集（Union-Find）。
- 基本近似算法思想（用可计算代理问题替代原 NP-hard 问题）。

## R09

适用场景：
- 网络/管线/布线中只要求“较优且可计算”的方案。
- 终端点数量中等，且图边权非负。  

不适用或需改造：
- 存在负权边（Dijkstra 不直接适用）。
- 强依赖全局最优且规模较小（可考虑精确算法）。
- 动态实时变化图（需要增量算法或在线策略）。

## R10

实现正确性检查点：
1. 终端点必须都在图中，且终端间可达。
2. 度量闭包权重必须与原图最短路一致。
3. Kruskal 结果应满足“边数 = 节点数 - 1”（连通情形）。
4. 路径展开时每条边权应回溯到原图真实边权。
5. 剪枝只删除“非终端叶子”，绝不删除终端点。  

`demo.py` 中包含结构性断言，确保输出确实是一棵连通终端点的树。

## R11

数值/实现稳定性要点：
- Dijkstra 仅在非负权下使用。
- 对浮点比较用容差（`1e-9`）避免误判。
- 非法输入（负权、终端不可达）直接抛错，避免静默错误。
- 所有权值均使用 `float`，保证流程一致。

## R12

工程成本与性能建议：
- 若 `k << n`，从终端点跑 `k` 次最短路通常可接受。
- 若图稠密，可考虑更高效的最短路实现或并行化。
- 可对多次查询场景缓存“终端最短路结果”。
- 先跑近似算法给基线，再对局部热点做精细改进。

## R13

理论保证（经典结论）：
- 该框架在非负边权图上基于最短路度量闭包，满足近似比 `<= 2`（更精确地常写为 `2 - 2/|R|` 上界形式之一，最终不超过 2）。
- 算法是确定性的，不涉及随机成功概率。  

因此它是“有界近似”而非纯启发式黑盒。

## R14

常见失效模式与应对：
- 终端不可达：算法无法构造闭包。
  - 应对：预检查连通性并报错。
- 输入含负权边：Dijkstra 失效。
  - 应对：改用 Bellman-Ford 或先规整权重。
- 图极大导致运行慢。
  - 应对：图压缩、候选边筛选、并行最短路。
- 结果质量不满意（虽满足近似比但偏保守）。
  - 应对：在近似树上加局部搜索/替换优化。

## R15

实践建议：
- 保留“中间产物”可视化：终端闭包 MST、路径展开子图、最终树。
- 把输入合法性检查前置，避免隐蔽逻辑分叉。
- 对小规模样例增加“暴力最优值”回归测试，持续校验近似实现质量。
- 输出边集时排序，便于 CI 文本比对。

## R16

相关算法与扩展：
- 精确算法：Dreyfus-Wagner 动态规划（终端数小可行）。
- 其他近似/启发：最短路径增量法、局部改良法、Prize-Collecting Steiner Tree 等。
- 特殊图（平面图、欧式空间）可用更强结构性质做更高质量近似。  

本条目定位为“通用图 + 可快速实现”的基础版本。

## R17

`demo.py` 内容：
- 从零实现（`heapq + 并查集`），不依赖图算法黑盒库。
- 包含函数：
  - `dijkstra`：单源最短路；
  - `kruskal_mst`：最小生成树；
  - `kmb_steiner_tree`：Steiner 2-近似主流程；
  - `brute_force_optimal_steiner`：仅用于小图对照最优值；
  - `validate_steiner_tree`：结果结构校验。  

运行方式：

```bash
cd Algorithms/数学-图论-0220-Steiner树近似算法
python3 demo.py
```

脚本无交互输入，会直接打印近似解、暴力最优解和近似比。

## R18

源码级流程拆解（对应 `demo.py`，8 步）：
1. `build_demo_graph` 构造一个带权无向图和终端点集合，作为可复现实验输入。  
2. `kmb_steiner_tree` 先调用 `metric_closure_shortest_paths`，对每个终端跑 `dijkstra`，得到终端两两最短路距离与具体路径。  
3. 用这些距离构造终端完全图边集，并用 `kruskal_mst` 得到度量闭包 MST。  
4. 将闭包 MST 的每条边映射回原图最短路径，收集为展开子图的边集合。  
5. 在展开子图上再次运行 `kruskal_mst`，删除路径叠加带来的冗余环。  
6. `prune_non_terminal_leaves` 反复删除非终端叶子，压缩成 Steiner 近似树。  
7. `validate_steiner_tree` 校验输出是否满足树结构、终端连通和边权非负等条件。  
8. `main` 中额外调用 `brute_force_optimal_steiner`（仅小图）做最优值对照，打印近似比用于质量检查。  

整个流程没有把第三方库当黑箱调用；关键步骤都可直接追踪到函数级实现。
