# 切伦科夫探测器 (Cherenkov Detectors)

- UID: `PHYS-0410`
- 学科: `物理`
- 分类: `粒子物理实验`
- 源序号: `430`

## R01

切伦科夫探测器（尤其是 RICH/阈值计数器）把“粒子速度信息”转成“可测光学几何量”。
在均匀介质中，带电粒子满足 `beta*n > 1` 时会产生切伦科夫光，其锥角满足：

`cos(theta_c) = 1 / (beta*n)`。

在成像平面上，该锥角对应环半径 `r = L*tan(theta_c)`（`L` 为等效光路）。本条目 MVP 直接演示：由像面散点重建环，再反推 `beta` 和粒子类型。

## R02

工程价值：

1. 把实验读出（像面光子命中）映射到可解释参数（`theta_c`、`beta`、质量假设）。
2. 明确区分“有环事件”（超阈值）与“无环事件”（低于阈值）。
3. 在背景命中存在时，用鲁棒圆拟合稳定提取环半径。
4. 提供一个可复现的 PID 基线（`pi/K/p`）。

## R03

`demo.py` 输入/输出：

- 输入（脚本内配置、无交互）：
1. 固定动量 `p=3 GeV/c`，粒子种类混合 `pion/kaon/proton`；
2. 折射率 `n=1.03` 与几何参数 `L=1200 mm`；
3. 光子统计、径向噪声、背景命中率；
4. 圆拟合质量切阈值。
- 输出：
1. 事件级 `pandas.DataFrame`（真值、拟合、PID 与误差）；
2. 汇总指标（发光比例、重建效率、假环率、PID 准确率、角分辨等）；
3. 自动断言与 `All checks passed.`。

## R04

核心方程：

1. `beta = p / sqrt(p^2 + m^2)`。
2. `theta_c = arccos(1/(beta*n))`，若 `beta*n <= 1` 则 `theta_c=0`。
3. `r = L * tan(theta_c)`。
4. 拟合后反演：`theta_rec = arctan(r_rec/L)`。
5. `beta_rec = 1/(n*cos(theta_rec))`。
6. 已知动量下质量重建：`m_rec = p*sqrt(1/beta_rec^2 - 1)`。

## R05

复杂度设事件数 `E`，每事件平均命中点数 `H`：

- 事件生成（信号+背景）约 `O(E*H)`；
- 鲁棒圆拟合每事件迭代 `I` 次，约 `O(E*I*H)`；
- PID 评分是常数级（3 个假设），约 `O(E)`。

总复杂度近似 `O(E*I*H)`，本 MVP 参数下（几百事件、每事件几十点）运行很快。

## R06

最小闭环分为四段：

1. `simulate_event`：按粒子质量决定是否超阈值并生成环形信号光子。
2. 加入均匀背景命中，形成像面点云。
3. `fit_circle_robust`：用 `scipy.optimize.least_squares(loss="soft_l1")` 做鲁棒圆拟合。
4. `classify_species`：由 `r_rec` 与理论半径做似然评分，输出 `pi/K/p`。

优点：

## R10

正确性直觉：

1. 低于阈值的粒子（本配置下典型是质子）应很少出现稳定环；
2. 有效信号点越多、径向噪声越小，环半径误差越小；
3. `r -> theta -> beta` 是单调反演链，几何关系明确；
4. PID 本质上比较 `r_rec` 与各质量假设 `r_hyp` 的偏差，因此在固定动量下可区分 `pi/K/p`。

## R11

数值稳定策略：

- 对 `arccos` 输入做 `clip`，避免浮点越界；
- 圆拟合使用 `soft_l1` 降低离群点影响；
- 用内点数、内点比例、残差 RMS 三重门限判定“真环”；
- 对 `beta_rec` 做上限裁剪，防止反演质量出现非物理负根。

## R12

关键参数与调参方向：

- `refractive_index`：决定阈值与环半径尺度；
- `n0_photons`：控制信号光子数，直接影响环可重建性；
- `sigma_radial_mm`：单光子径向分辨；
- `background_mean_hits`：背景污染强度；
- `inlier_residual_mm`、`min_inlier_fraction`：环判定松紧度；
- `pid_sigma_floor_mm`：PID 似然宽度下限，避免过拟合统计涨落。

## R14

常见失效模式：

1. 背景过高导致假环率上升；
2. 光子数过低使拟合不稳定，`K` 与 `p` 易混淆；
3. 折射率或几何标定偏差造成系统性质量偏差；
4. 仅用半径做 PID，在边界动量区间可能分离度不足。

## R18

`demo.py` 中“鲁棒圆拟合 + PID”的源码级流程可拆为 8 步：

1. 由粒子质量计算 `beta_true`，用阈值条件 `beta*n>1` 决定是否产生信号环。  
2. 对发光事件按 `r_true=L*tan(theta)` 生成环上光子，并叠加高斯径向扰动；再加均匀背景点。  
3. 用点云均值和中位距离构造圆参数初值 `(cx0, cy0, r0)`。  
4. 构造残差 `res_i = sqrt((x_i-cx)^2+(y_i-cy)^2) - r`，这一步把“圆拟合”写成可优化的标量残差问题。  
5. 调用 `least_squares(loss="soft_l1")` 迭代最小化残差，`soft_l1` 在大残差区自动降权，抑制背景离群点。  
6. 对拟合结果按 `|res_i| < inlier_residual_mm` 做内点筛选，并以内点数/内点比例/残差 RMS 联合判定是否接受为“有效环”。  
7. 对有效环做几何反演：`r_rec -> theta_rec -> beta_rec -> m_rec`，形成可物理解释的重建参数。  
8. 在 PID 阶段，分别计算 `pi/K/p` 的理论半径并比较 `chi2=((r_rec-r_hyp)/sigma_eff)^2`，选择最小分数的粒子类型。  

说明：第三方库只负责数值优化器本体；事件生成、残差构造、内点规则、物理反演和 PID 决策都在源码中显式实现，非黑盒一行调用。
