# 三分查找 (Ternary Search)

- UID: `CS-0021`
- 学科: `计算机`
- 分类: `搜索算法`
- 源序号: `21`
- 目标目录: `Algorithms/计算机-搜索算法-0021-三分查找_(Ternary_Search)`

## R01

三分查找（Ternary Search）是一种在**有序数组**中进行查找的分治算法。每一轮不只取一个中点，而是把当前区间划分为三段，用两个分点值与目标比较，从而排除约三分之二的不可能区间。

## R02

问题形式化（本目录 MVP）：
- 输入：非降序数组 `arr`（长度 `n`）与目标值 `target`
- 输出：若存在下标 `i` 使 `arr[i] == target`，返回某个合法下标；否则返回 `-1`
- 前提：元素可比较，且 `arr` 已按非降序排列

## R03

适用边界与前置条件：
- 必须是有序数组，否则三分决策不成立
- 适合随机访问结构（如数组）；链表等结构不适合
- 对重复值，算法可返回任一命中位置（本实现也采用该约定）
- 渐进复杂度为 `O(log n)`，但单轮比较次数通常多于二分查找

## R04

核心思路：
- 在区间 `[left, right]` 内计算两个分点 `mid1`、`mid2`
- 先比较 `arr[mid1]` 和 `arr[mid2]` 与 `target`
- 若目标更小，进入左三分之一；若更大，进入右三分之一；否则进入中间三分之一
- 重复迭代直到命中或区间为空

## R05

简化伪代码：

```text
ensure arr is non-decreasing
left = 0, right = n - 1

while left <= right:
    third = (right - left) // 3
    mid1 = left + third
    mid2 = right - third

    if arr[mid1] == target:
        return mid1
    if arr[mid2] == target:
        return mid2

    if target < arr[mid1]:
        right = mid1 - 1
    else if target > arr[mid2]:
        left = mid2 + 1
    else:
        left = mid1 + 1
        right = mid2 - 1

return -1
```

## R06

正确性要点（不变式直观）：
- 每轮循环都维持不变式：若目标存在，则必在当前闭区间 `[left, right]` 内
- `target < arr[mid1]` 时，中间和右侧都可排除；`target > arr[mid2]` 时，左侧和中间可排除
- `arr[mid1] < target < arr[mid2]` 时，仅中间段可能包含目标
- 每轮至少缩小区间，不会无限循环；命中返回下标，区间耗尽返回 `-1`

## R07

复杂度分析：
- 时间复杂度：`O(log_3 n)`，按大 O 记号即 `O(log n)`
- 空间复杂度：`O(1)`，仅使用常量个辅助变量
- 与二分查找比较：两者同为 `O(log n)`；三分查找每轮通常做更多比较，常数项可能更大

## R08

边界与异常情况：
- 空数组：直接返回 `-1`
- 单元素数组：一轮比较后返回命中或未命中
- 目标小于最小值或大于最大值：会在若干轮后返回 `-1`
- 重复值：返回任一合法命中下标
- 输入无序：本实现显式抛出 `ValueError`，避免静默错误

## R09

示例（`arr=[1,3,5,7,9,11,13]`, `target=9`）：
- 初始 `left=0,right=6`，`mid1=2(值5)`，`mid2=4(值9)`
- 第二个分点直接命中，返回下标 `4`

示例（`arr=[2,4,6,8,10,12,14,16,18]`, `target=15`）：
- 首轮分点值为 `6` 和 `14`，目标在右段，更新为更小右侧区间
- 后续迭代区间继续收缩，最终区间为空，返回 `-1`

## R12

典型应用：

- 在有序数组上实现可读性较高的分治查找
- 作为从二分查找扩展到更一般 `k` 分查找的入门案例
- 与二分、跳跃、插值等搜索方法进行实验对比

## R13

常见错误：
- 忽略有序性前提，直接对无序数组执行
- 分点计算或区间更新写错，导致漏解或死循环
- 中间段更新写成闭区间错误边界，导致重复检查
- 未处理重复值语义，误以为必须返回第一个位置
- 没有验证 `result` 下标是否真命中目标值

## R14

常见变体：
- 连续域单峰函数极值搜索：在 `f(mid1)` 与 `f(mid2)` 比较后保留一侧区间
- 整数域单峰极值：在离散范围上用三分或邻域比较收缩
- `k` 分查找：把区间拆成更多段，但比较开销上升
- “查找目标值”与“求单峰极值”是两个不同问题，判定逻辑不同

## R15

与其他搜索算法对比：
- 二分查找：同阶复杂度 `O(log n)`，二分常数更小，工程上更常用
- 跳跃查找：`O(sqrt(n))`，实现直观但理论复杂度劣于三分/二分
- 插值查找：在分布近似均匀时可能更快，但最坏可能退化
- 线性查找：无需有序性但为 `O(n)`

## R16

工程实践建议：
- 明确 API 契约：输入必须为非降序数组
- 保持返回约定统一（未命中返回 `-1`）
- 用参考实现（如 `bisect_left`）做存在性回归校验
- 对性能敏感场景优先基准测试二分与三分的常数差异，再决定选型

## R17

最小测试清单：
- 空数组、单元素命中/未命中
- 命中首端、中段、末端
- 目标不存在（区间内缺口、超出上下界）
- 重复值数组（验证返回下标合法且值正确）
- 无序输入（应抛 `ValueError`）

## R18

源码级算法流程拆解（对应本目录 `demo.py` 的 `ternary_search`）：
1. 入口先调用 `_ensure_non_decreasing(arr)`，若发现逆序对立即抛出 `ValueError`。
2. 初始化搜索边界 `left=0`、`right=len(arr)-1`，表示候选区间为闭区间。
3. 循环条件为 `left <= right`；区间非空时继续搜索。
4. 每轮计算 `third=(right-left)//3`，再得到两个分点 `mid1=left+third`、`mid2=right-third`。
5. 读取 `arr[mid1]` 与 `arr[mid2]`；若任一等于 `target`，立即返回对应下标。
6. 若 `target < arr[mid1]`，则把 `right` 收缩到 `mid1-1`，保留左段。
7. 若 `target > arr[mid2]`，则把 `left` 推进到 `mid2+1`，保留右段；否则保留中段并更新为 `left=mid1+1`、`right=mid2-1`。
8. 当循环结束仍未命中时返回 `-1`；`main()` 再用 `_expected_via_bisect` 校验“是否存在”的结果一致性。
