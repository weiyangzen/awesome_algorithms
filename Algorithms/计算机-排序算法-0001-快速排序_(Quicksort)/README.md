## R01

快速排序（Quicksort）是一种经典的分治排序算法，核心思想是“选一个基准值（pivot），把序列按基准分成左右两部分，再分别递归排序”。

它的工程价值在于：

- 平均性能很高，常见实现平均时间复杂度为 `O(n log n)`；
- 主要在原地交换元素，额外空间通常较小；
- 实现可控，便于做 pivot 策略和小区间优化。

## R02

问题定义（升序）：

- 输入：长度为 `n` 的可比较序列 `A = [a1, a2, ..., an]`；
- 输出：序列 `B`，满足：
  - `B` 是 `A` 的重排；
  - `B[0] <= B[1] <= ... <= B[n-1]`。

本条目 MVP 目标：

- 给出源码级可追踪的 Quicksort 实现，不依赖第三方排序黑箱；
- 自动验证结果与 `sorted()` 一致；
- 展示 Quicksort 非稳定排序这一典型特性。

## R03

快速排序在排序算法体系中的定位：

- 是分治思想最具代表性的实战算法之一；
- 在内存内排序中常作为高性能基线实现；
- 通过 pivot 选择策略可明显影响常数项和退化风险；
- 有助于理解“划分（partition）驱动递归”的算法范式。

## R04

本 MVP 采用“median-of-three + Lomuto partition”机制：

1. 在 `lo / mid / hi` 三个位置中选中位数作为 pivot，降低有序输入退化概率；
2. 将 pivot 暂时换到右端 `hi`；
3. 扫描区间 `[lo, hi-1]`：
   - 若当前元素 `<= pivot`，则交换到左侧边界并推进边界；
4. 扫描结束后，把 pivot 放到最终位置；
5. 对 pivot 左右子区间递归执行相同流程。

说明：此实现是原地排序版本，默认不保证稳定性。

## R05

高层执行流程：

1. 输入序列复制为工作数组 `arr`；
2. 递归函数 `sort(lo, hi)` 处理闭区间；
3. `partition(lo, hi)` 完成一次原地划分并返回 pivot 落点；
4. 递归处理 `[lo, pivot_index-1]` 与 `[pivot_index+1, hi]`；
5. 递归结束后得到升序结果；
6. 返回排序数组与统计信息（比较次数、交换次数、分区次数、最大递归深度）。

## R06

正确性要点：

- 分区不变式：在扫描过程中，始终保持
  - 左段元素 `<= pivot`；
  - 中段元素 `> pivot`（尚未完成最终分布）；
  - 右端为待处理或 pivot 位置。
- 分区结束时，pivot 被交换到其最终位置，满足“左边都不大于 pivot、右边都大于 pivot”；
- 递归基是区间长度 `0` 或 `1`，天然有序；
- 由数学归纳可得整个数组最终有序。

## R07

复杂度分析：

- 平均时间复杂度：`O(n log n)`；
- 最坏时间复杂度：`O(n^2)`（例如持续极端不平衡划分）；
- 额外空间复杂度：
  - 原地交换部分为 `O(1)`；
  - 递归栈平均 `O(log n)`，最坏 `O(n)`。

## R08

边界与异常策略：

- 空数组：直接返回空结果；
- 单元素：直接返回；
- 重复值、负数、浮点数：正常处理；
- 不可比较混合类型：Python 比较阶段会抛错，MVP 保持显式失败，不吞错。

## R09

MVP 范围与取舍：

- 选择清晰、可审计的递归实现，不引入复杂框架；
- 使用 `numpy` 仅生成可复现随机样例，不参与排序核心；
- 统计指标保持最小集合，便于理解算法行为；
- 不做并行化、尾递归消除、三路划分等高级优化。

## R10

## R14

关键参数与行为观察：

- `n`（规模）增大时，比较与交换总量总体接近 `n log n` 趋势；
- pivot 质量影响递归深度和交换次数；
- 重复值比例高时，二路划分可能出现更多无效交换；
- 固定随机种子可复现实验输出，便于调试与回归验证。

## R15

与常见排序算法对比：

- 对比归并排序：Quicksort 通常更省额外内存，但不稳定且最坏可能 `O(n^2)`；
- 对比堆排序：两者都可原地，Quicksort 常数项常更优，堆排序最坏仍为 `O(n log n)`；
- 对比插入/冒泡/选择：大规模下 Quicksort 平均性能显著更好；
- 对比 Python `sorted()`（Timsort）：Timsort 稳定且工程优化更复杂，本 MVP 侧重算法可解释性。

## R17

可扩展方向：

## R18

`demo.py` 源码级流程拆解（8 步）：

1. `main` 构造固定测试集和 `numpy` 随机测试集，逐个交给 `run_case`。  
2. `run_case` 调用 `quick_sort(data)`，并用 `sorted(data)` 做基准比对。  
3. `quick_sort` 初始化工作数组与 `QuickSortStats` 统计对象。  
4. `choose_pivot_index` 在 `lo/mid/hi` 三点中取中位数，减少极端划分概率。  
5. `partition` 把 pivot 交换到右端，线性扫描并将 `<= pivot` 的元素交换到左侧区域。  
6. 扫描结束后将 pivot 放回最终落点，得到“左 <= pivot < 右”的分区结构。  
7. `sort` 递归处理 pivot 左右子区间，直到区间长度为 `0` 或 `1`。  
8. 全部递归结束后返回排序结果与统计；`main` 汇总输出并额外打印稳定性检查结果。  

说明：排序核心完全由源码实现；第三方库仅用于生成随机样例，不参与排序计算，不构成黑箱。
