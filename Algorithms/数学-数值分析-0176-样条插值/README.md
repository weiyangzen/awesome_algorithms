# 样条插值

- UID: `MATH-0176`
- 学科: `数学`
- 分类: `数值分析`
- 源序号: `176`
- 目标目录: `Algorithms/数学-数值分析-0176-样条插值`

## R01

本条目实现自然三次样条插值（Natural Cubic Spline）的最小可运行版本。给定一组严格递增节点 `x0 < x1 < ... < xn` 与函数值 `yi`，构造分段三次多项式 `S(x)`，满足：
- 插值条件：`S(xi) = yi`；
- 一阶、二阶导在内部节点连续；
- 自然边界：`S''(x0)=0`、`S''(xn)=0`。

MVP 目标是把数学推导直接落成代码，而不是只调用黑盒库函数。

## R02

问题定义：
- 输入：节点数组 `x`、值数组 `y`，长度相同且 `x` 严格递增。
- 输出：样条模型（每段多项式系数）与可评估函数。
- 约束：查询点必须落在 `[x0, xn]` 内，本 MVP 不做外推。

本实现采用自然边界条件，避免额外端点导数信息。

## R03

设每段区间 `[xi, x{i+1}]` 宽度 `hi = x{i+1} - xi`，记节点二阶导为 `Mi = S''(xi)`。内部节点满足三对角方程：

`h{i-1} M{i-1} + 2(h{i-1}+hi) Mi + hi M{i+1} = 6[(y{i+1}-yi)/hi - (yi-y{i-1})/h{i-1}]`

边界条件为 `M0 = Mn = 0`。解出全部 `Mi` 后，每段三次多项式可写成：
- `S_i(x) = a_i + b_i*dx + c_i*dx^2 + d_i*dx^3`，`dx = x - xi`
- `a_i = yi`
- `b_i = (y{i+1}-yi)/hi - hi*(2Mi + M{i+1})/6`
- `c_i = Mi/2`
- `d_i = (M{i+1}-Mi)/(6hi)`

## R04

算法流程：
1. 校验输入数组合法性（维度、长度、有限数、严格递增）。
2. 计算步长数组 `h`。
3. 构建内部节点对应的三对角线性系统。
4. 用 Thomas 算法求解内部二阶导。
5. 拼出完整 `M`（两端置零）。
6. 计算每段 `a,b,c,d` 系数。
7. 查询时通过二分定位区间并代入分段多项式。

## R05

`demo.py` 的核心数据结构：
- `SplineModel`：保存 `x`、`y`、`second_derivatives(M)`、`coeffs(a,b,c,d)`。
- `coeffs` 形状为 `(n_segments, 4)`，每行对应一段样条。
- 查询时按区间索引取对应系数，避免重复构建。

## R06

正确性验证思路：
- 解析函数取 `sin(x)`，节点为 `[0, 2pi]` 上等距 9 点；
- 稠密采样 401 点，对比 `spline(x)` 与真值 `sin(x)`；
- 与线性插值基线对比误差；
- 检查内部节点的一阶导、二阶导跳变应接近 0；
- 检查自然边界 `|M0|`、`|Mn|` 接近 0。

## R07

复杂度分析：
- 构建阶段：
  - 组装系统 `O(n)`；
  - Thomas 求解三对角系统 `O(n)`；
  - 计算分段系数 `O(n)`。
- 查询阶段：
  - 区间定位（`searchsorted`）约 `O(log n)`；
  - 区间内多项式求值 `O(1)`。
- 空间复杂度：`O(n)`。

## R08

异常与边界处理：
- `x`、`y` 非一维或长度不等：抛 `ValueError`；
- 节点数小于 2：抛 `ValueError`；
- 存在 `NaN/Inf`：抛 `ValueError`；
- `x` 非严格递增：抛 `ValueError`；
- 查询点超出节点区间：抛 `ValueError`；
- 三对角系统主元过小：抛 `ValueError`，提示矩阵奇异或病态。

## R09

MVP 的取舍：
- 选择自然三次样条，避免“夹持边界”对端点导数的额外依赖；
- 用 `numpy` 实现数组运算与求值，核心求解流程手写（非黑盒）；
- 不引入绘图或大型框架，保持可读和可验证。

## R10

函数职责拆分：
- `validate_xy`：输入合法性校验；
- `thomas_solve`：三对角系统求解；
- `build_natural_cubic_spline`：构建样条模型；
- `evaluate_spline`：在任意查询点上评估样条值；
- `continuity_report`：输出导数连续性的数值检查；
- `main`：组织示例数据、评估误差、打印结果。

## R11

运行方式：

```bash
cd Algorithms/数学-数值分析-0176-样条插值
python3 demo.py
```

脚本无交互输入，运行后会直接输出误差指标、边界检查、系数预览与样例查询结果。

## R12

输出字段说明：
- `spline max abs error`：样条在稠密网格上的最大绝对误差；
- `spline mean abs error`：样条平均绝对误差；
- `linear ... error`：线性插值的对应误差（作为基线）；
- `MAE improvement`：线性 MAE / 样条 MAE；
- `|M0|, |Mn|`：自然边界满足程度；
- `max ... knot jump`：内部节点导数连续性检查。

## R13

建议最小测试集合：
- 基本插值：`sin(x)`、等距节点；
- 非等距节点：手工给定不均匀 `x`；
- 退化规模：仅 2 个节点（退化为单段三次/线性趋势）；
- 非法输入：重复节点、逆序节点、含 `NaN`；
- 越界查询：小于 `x0` 或大于 `xn`。

## R14

可调参数与影响：
- `x_knots` 数量：节点越多，插值通常更精细，但构建成本上升；
- 节点分布：高曲率区加密节点能显著降低局部误差；
- `x_dense` 分辨率：越高越能准确估计误差上界；
- 函数平滑性：函数越光滑，三次样条优势通常越明显。

## R15

与常见插值方法对比：
- 线性插值：简单、稳定，但导数不连续；
- 全局高次多项式插值：可能出现 Runge 振荡；
- 三次样条：局部性好、`C2` 连续，工程中常用。

本 MVP 也输出线性基线误差，便于直观看到样条收益。

## R16

典型应用场景：
- 传感器离散采样数据的平滑重建；
- 数值积分/微分前的函数近似；
- 图形学路径或轮廓平滑；
- 科学计算中表格函数的连续化近似。

## R17

后续可扩展方向：

- 增加 not-a-knot 等边界策略；

## R18

源码级算法流（对应 `demo.py`，8 步）：
1. `main` 构造 `sin(x)` 节点样本并调用 `build_natural_cubic_spline`。  
2. `build_natural_cubic_spline` 先用 `validate_xy` 检查输入维度、单调性与有限数。  
3. 计算 `h = diff(x)`，按自然样条公式组装内部节点三对角系统。  
4. `thomas_solve` 通过前消元与回代在线性时间内求出内部 `M1..M_{n-1}`。  
5. 回填 `M0=Mn=0`，得到完整二阶导向量 `M`。  
6. 将每段区间转换为 `a,b,c,d` 多项式系数并存入 `SplineModel.coeffs`。  
7. `evaluate_spline` 对查询点做区间定位，代入 `a+b*dx+c*dx^2+d*dx^3` 得到插值值。  
8. `main` 计算样条与线性插值误差、导数连续性与边界检查并打印结果。
