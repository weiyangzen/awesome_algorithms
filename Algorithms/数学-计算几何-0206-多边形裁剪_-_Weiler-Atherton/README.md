## R01

Weiler-Atherton（WA）是经典多边形布尔裁剪算法，常用于计算：

- `subject ∩ clipper`（交集）
- 也可扩展到并集、差集（本 MVP 聚焦交集）

它通过“沿边界行走 + 在交点切换多边形”的方式构造结果边界，适合处理非凸简单多边形（不含洞的基本版本）。

## R02

算法由 Kevin Weiler 与 Peter Atherton 在 1977 年提出，核心价值是：

- 不是像扫描线那样全局切割平面，而是直接在多边形边界图上行走。
- 通过入口（entry）/出口（exit）交点标注，把“几何相交”转成“拓扑遍历”。

在图形学、CAD、GIS 与机器人几何模块中都属于基础算法家族。

## R03

本题目标：给定两个二维简单多边形 `subject` 与 `clipper`，输出它们的交集多边形集合。

输入约束（本实现）：

- 顶点按边界顺序给出，允许凹多边形；
- 重点支持一般位置的边相交；
- 不覆盖“边共线重叠、仅端点相切”这类退化情形。

输出：`List[np.ndarray]`，每个元素是一条结果环（`Nx2`）。

## R04

WA 的交集流程（本实现版本）：

1. 预处理两个多边形（去重、去重复闭合点、统一 CCW）。
2. 计算所有严格内部线段交点（排除端点触碰与共线重叠）。
3. 在两条边界链表上按参数 `alpha` 顺序插入交点节点，并建立双向 `neighbor` 配对。
4. 在 `subject` 链上把交点标成 `entry/exit`（从外到内为 entry）。
5. 从每个未访问的 entry 出发沿边界行走：
   - 正常沿当前多边形 `next` 前进；
   - 到 exit 点时切换到另一多边形同名交点继续走。
6. 闭环后做去重与共线清理，得到一个输出多边形。

## R05

设 `subject` 边数为 `n`，`clipper` 边数为 `m`，交点数为 `k`。

- 交点检测：`O(n*m)`
- 交点插入排序：约 `O(k log k)`（按每条边局部排序）
- 遍历构环：`O(n + m + k)`
- 空间复杂度：`O(n + m + k)`

因此本 MVP 总体可写成 `O(n*m + k log k)`。

## R06

`demo.py` 内置了三个可复现实例：

- 凹 subject 与矩形 clipper 相交（产生 1 个裁剪多边形）。
- 两多边形相离（输出空集合）。
- clipper 完全位于 subject 内（直接返回 clipper）。

运行后会打印每个结果环的顶点与面积。

## R07

优点：

- 对非凸简单多边形友好。
- 结果边界可直接用于渲染或下游几何流程。
- 思路可解释性强，便于教学与调试。

局限：

- 退化情形处理复杂（共线重叠、顶点接触、浮点抖动）。
- 链表节点与访问状态管理比 Sutherland-Hodgman 更繁琐。
- 工业级实现通常需要更强的鲁棒谓词与拓扑修复。

## R08

前置知识建议：

- 2D 向量叉积与线段相交参数方程。
- 点在多边形内判定（射线法）。
- 双向循环链表操作。
- 几何鲁棒性基础（`eps`、去重、共线简化）。

## R09

适用场景：

- 游戏/图形学中的裁剪窗口与遮罩。
- CAD/GIS 中简单多边形交集预处理。
- 需要输出显式多边形边界而非仅面积的任务。

不适用或需增强：

- 大量退化输入（重边、重顶点、共线重叠广泛）。
- 含洞多边形的完整布尔运算（需更完整拓扑结构）。

## R10

正确性直觉：

- 交点把边界切成“在对方内部/外部”的弧段。
- 从 entry 出发沿边界前进，遇到 exit 切换到另一边界，本质是在追踪 `subject ∩ clipper` 的边界前沿。
- 对简单相交情形，闭环一定回到起始交点，形成交集边界环。

该逻辑在本 MVP 的“一般位置”前提下成立。

## R11

数值与鲁棒性策略：

- 使用 `EPS = 1e-9` 做浮点阈值比较。
- 输入先归一化：去相邻重复点、去重复闭合点、统一 CCW。
- 输出环后处理：去相邻重复点、删除连续共线冗余点。

注意：本实现故意把端点相切/共线重叠排除在主路径外，避免把退化处理伪装成已完成。

## R14

常见失效模式：

1. 边共线重叠（本 MVP 未处理）。
2. 仅在顶点相切（本 MVP 的严格交点判定会忽略）。
3. 输入几何质量差（重复点、多段极短边）导致交点分类不稳定。
4. `eps` 与坐标量级不匹配导致误判。

工程对策：

- 对输入做几何清洗；
- 增加退化判定分支；
- 用高精度谓词或有理数几何库替代浮点判定。

## R15

实现层建议：

- 把流程拆分为：`预处理 -> 交点构图 -> 交点标记 -> 边界遍历 -> 后处理`。
- 用小函数隔离风险点（如 `segment_intersection_strict`、`mark_entry_exit`）。
- 对每类几何关系准备最小回归样例（相交/相离/包含/退化）。

## R16

相关算法对比：

- Sutherland-Hodgman：适合凸裁剪窗口，结构更简单。
- Greiner-Hormann：同样基于边界交点与遍历，工程上常作为 WA 的现代替代方案之一。
- Vatti / Martinez-Rueda：更通用、可处理复杂布尔操作与洞，但实现复杂度更高。

## R17

本目录 `demo.py` 的 MVP 范围：

- 使用 `numpy` + 手写几何逻辑实现 WA 交集主流程。
- 支持一般位置下的简单多边形相交。
- 自动执行 3 个确定性用例并断言结果数量/面积特征。
- 清晰暴露限制：不假装支持共线重叠与端点相切。

## R18

`demo.py` 源码级流程拆解（8 步）：

1. `normalize_polygon` 统一输入顶点表达（去重、去闭合重复点、统一 CCW）。
2. `build_ring` 把两个多边形转成循环双向链表节点结构。
3. `collect_intersections_and_insert` 双层遍历边对，调用 `segment_intersection_strict` 求交并按 `alpha` 插入交点节点，同时建立 `neighbor` 配对。 
4. 若无交点，`weiler_atherton_clip` 用 `point_in_polygon` 直接处理“包含/相离”三种退化分支并返回。 
5. `mark_entry_exit` 在 `subject` 链上按 inside 状态交替标记 entry/exit。 
6. `traverse_result_polygons` 从每个未访问 entry 出发，沿 `next` 走边界；遇到 exit 交点切换到 `neighbor` 并继续，直到闭环。 
7. `clean_output_polygon` 对每个结果环做重复点与共线点清理，保持输出紧凑可读。 
8. `main` 调用 `run_case` 执行三组固定样例并打印顶点与面积，保证 `python3 demo.py` 一次运行即可验证。
