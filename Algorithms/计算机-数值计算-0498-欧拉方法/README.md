## R01

欧拉方法（Euler Method）是最基础的一阶常微分方程（ODE）数值积分方法。  
它把连续时间上的微分问题离散化为“按小步前进”的差分迭代，是很多更高阶方法（如改进欧拉、Runge-Kutta）的起点。

## R02

目标问题是初值问题：

\[
\frac{dy}{dt} = f(t, y), \quad y(t_0)=y_0
\]

给定步长 \(h\) 与步数 \(N\)，在离散时刻 \(t_n=t_0+n h\) 上近似 \(y(t_n)\)。

## R03

核心思想：在每个离散点 \((t_n,y_n)\) 处，用导数 \(f(t_n,y_n)\) 给出的切线斜率做一次线性外推，得到下一点 \(y_{n+1}\)。  
这相当于“用当前斜率预测未来一个小步”的显式推进。

## R04

显式欧拉更新公式：

\[
t_{n+1}=t_n+h,\qquad
y_{n+1}=y_n+h\,f(t_n,y_n)
\]

当 \(h\to 0\) 时，数值解会收敛到真实解（在常见光滑条件下）。

## R05

伪代码：

```text
输入: f, t0, y0, h, N
t <- t0
y <- y0
for n in 0..N-1:
    y <- y + h * f(t, y)
    t <- t + h
输出: {t_n}, {y_n}
```

## R06

本目录 `demo.py` 的 MVP 示例选用：

- 方程：\(y' = y\)
- 初值：\(y(0)=1\)
- 解析解：\(y(t)=e^t\)
- 区间：\([0,1]\)

示例使用两组步长（`h=0.1` 与 `h=0.05`）比较终点误差，并估计实验收敛阶。

## R07

正确性直觉：

1. 泰勒展开有 \(y(t+h)=y(t)+h y'(t)+O(h^2)\)；
2. 用 \(f(t_n,y_n)\) 近似 \(y'(t_n)\)，得到欧拉更新；
3. 单步截断误差是 \(O(h^2)\)，累积到固定区间得到全局误差 \(O(h)\)。

因此欧拉法是一阶精度方法。

## R08

时间复杂度：`O(N)`。  
每一步只计算一次 `f(t, y)` 并做常数次加乘操作。

## R09

空间复杂度：

- 若只保留当前状态：`O(1)`；
- 若像 `demo.py` 一样保存整条轨迹用于分析/可视化：`O(N)`。

## R10

误差结论（固定积分区间）：

- 局部截断误差：`O(h^2)`；
- 全局离散误差：`O(h)`。

这意味着步长减半时，误差理论上约减半（实验中会接近该趋势）。

## R11

稳定性（线性测试方程 \(y'=\lambda y\)）：

\[
y_{n+1}=(1+h\lambda)y_n
\]

稳定条件约为 \(|1+h\lambda|<1\)。  
对刚性问题（\(\lambda\) 负且绝对值大），欧拉法要求很小的 \(h\)，否则可能震荡或发散。

## R12

步长选择建议：

1. 先用较大 `h` 快速试跑，检查是否稳定；
2. 使用 `h` 与 `h/2` 做误差对比（步长加密检验）；
3. 若误差不满足需求，优先减小步长或换高阶方法。

工程上常把“精度目标 + 计算预算 + 稳定性”一起考虑。

## R13

常见风险与边界情况：

- `h <= 0`：无意义，应直接报错；
- `N` 非正整数：离散网格不成立；
- 右端函数 `f` 不连续或剧烈变化：误差可能非常大；
- 长时间积分时误差会逐步积累，需定期校验。

## R14

与常见方法比较：

- 欧拉法：实现最简单、每步代价最低、精度一阶；
- 改进欧拉/Heun：每步多一次斜率评估，精度提升到二阶；
- RK4：每步四次斜率评估，精度高但计算更重。

因此欧拉法适合教学、原型验证、以及对精度要求不高的快速估算。

## R15

工程实现要点：

1. 用数组预分配保存轨迹，避免循环中频繁扩容；
2. 在入口处验证参数合法性（`h`、`n_steps`）；
3. 输出关键指标（终点误差、收敛阶估计）便于自动化验收；
4. 保留解析解对照（若可得）用于回归测试。

## R17

结果解读：

- 随着步长减小，终点误差应下降；
- 估计收敛阶应接近 `1`（允许因步长不够小而偏离）；
- 若误差不降反升，通常是实现错误、步长不稳定或问题本身更刚性。

## R18

`demo.py` 的源码级算法流程（8 步）：

1. 定义微分方程右端 `ode_rhs(t, y)`，本例返回 `y`。  
2. 定义解析解 `analytic_solution(t)`，用于误差对照。  
3. 在 `euler_solve(...)` 中校验 `h > 0` 且 `n_steps > 0`。  
4. 预分配 `ts`、`ys` 数组并写入初值 `(t0, y0)`。  
5. 对每个步 `n` 计算当前斜率 `slope = f(ts[n], ys[n])`。  
6. 依据欧拉公式执行一次前向更新：`ys[n+1] = ys[n] + h * slope`，并同步更新时间 `ts[n+1]`。  
7. 在 `run_case(h, t_end)` 中调用求解器，计算终点解析解与绝对误差。  
8. 在 `main()` 里运行两组步长，打印样本点和误差，并用对数比值公式估计收敛阶。
