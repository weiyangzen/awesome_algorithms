# 分发糖果问题

## R01

分发糖果问题（Candy）要求：

- 每个孩子至少分到 1 颗糖；
- 相邻孩子中，评分更高者必须拿到更多糖；
- 在满足约束前提下，使总糖果数最小。

这是贪心算法里的经典题目，核心是如何同时满足“左邻约束”和“右邻约束”。

## R02

形式化定义：

- 输入：长度为 `n` 的评分数组 `ratings`；
- 输出：最小糖果总数 `sum(candies)`；
- 约束：
  - `candies[i] >= 1`；
  - 若 `ratings[i] > ratings[i-1]`，则 `candies[i] > candies[i-1]`；
  - 若 `ratings[i] > ratings[i+1]`，则 `candies[i] > candies[i+1]`。

仅比较相邻评分，不要求跨越多个位置直接比较糖果数。

## R03

贪心关键观察：

- 单次从左到右只能保证“比左边评分高就多拿”；
- 单次从右到左只能保证“比右边评分高就多拿”；
- 最小可行解可表示为：
  - `left[i]`：满足左约束所需最少糖果；
  - `right[i]`：满足右约束所需最少糖果；
  - 最终 `candies[i] = max(left[i], right[i])`。

常见实现会把 `left` 写入结果数组，再从右向左做 `max` 修正。

## R04

两遍贪心流程：

1. 初始化 `candies = [1] * n`；
2. 左到右遍历：若 `ratings[i] > ratings[i-1]`，令 `candies[i] = candies[i-1] + 1`；
3. 右到左遍历：若 `ratings[i] > ratings[i+1]`，令
   `candies[i] = max(candies[i], candies[i+1] + 1)`；
4. 返回 `sum(candies)`（或同时返回分配方案）。

`demo.py` 的主函数 `candy_distribution_greedy` 即按此实现。

## R05

复杂度：

- 时间复杂度：`O(n)`（两次线性扫描）；
- 空间复杂度：`O(n)`（保存糖果分配数组）。

这是该题最常见且可读性最好的最优解法。

本目录 MVP 的验证策略：

## R09

适用场景：

- 一维序列、邻接关系上的“严格大小约束 + 最小资源分配”；
- 教学场景下演示“两遍贪心合并局部约束”的范式。

不适用场景：

- 非邻接、跨节点复杂依赖；
- 存在多种资源类型联动；
- 需要在不确定评分分布下做在线决策。

## R10

正确性直觉：

1. 左扫后，所有“右边评分更高”的关系都被满足且使用最小增量。  
2. 右扫时，只在必要处把 `candies[i]` 提高到满足右邻约束；`max` 保证不会破坏左扫已满足的条件。  
3. 任一位置若低于 `max(left[i], right[i])`，至少违反一个方向约束；因此该下界不可再降。  
4. 整体恰取该下界，故总和最小。

## R11

边界与鲁棒性：

- 空数组返回 `[]` 与总数 `0`；
- 单元素返回 `[1]` 与总数 `1`；
- 全等评分返回全 1；
- 严格递增返回 `1..n`；
- 严格递减返回 `n..1`。

`demo.py` 对输入做归一化，避免布尔值或非法类型混入。

## R12

实现细节：

- `_normalize_ratings`：统一输入并做类型检查；
- `candy_distribution_greedy`：返回完整糖果数组，便于验证与解释；
- `min_candies_greedy`：对外提供总数接口；
- `is_distribution_valid`：独立校验约束；
- `min_candies_bruteforce`：小规模穷举基线（仅用于对拍，不用于大输入）。

## R13

理论性质：

- 在题目原始约束下，两遍贪心给出精确最优解，不是近似；
- 最优总数唯一，但糖果分配数组在某些评分形态下可能不唯一；
- 本实现返回其中一个可行最优分配。

## R14

常见错误：

1. 只做左到右一遍扫描，漏掉右邻约束；
2. 右扫时直接覆盖 `candies[i] = candies[i+1] + 1`，忘记取 `max`；
3. 误把“评分相等”也当作必须分更多；
4. 忽略空输入与单元素边界；
5. 缺少独立校验函数，导致回归测试难定位问题。

## R18

`demo.py` 的源码级算法流程（8 步）：

1. `main()` 先跑固定样例，再跑随机对拍与性能快照。  
2. `_normalize_ratings()` 将评分序列转成整数列表，拒绝布尔等异常输入。  
3. `candy_distribution_greedy()` 初始化全 1 糖果数组，建立“每人至少 1 颗”的基线。  
4. 第一遍左扫：当 `ratings[i] > ratings[i-1]` 时，把 `candies[i]` 提升到 `candies[i-1] + 1`。  
5. 第二遍右扫：当 `ratings[i] > ratings[i+1]` 时，用 `max(candies[i], candies[i+1] + 1)` 修正。  
6. `min_candies_greedy()` 汇总糖果总数，`is_distribution_valid()` 独立复核所有邻接约束。  
7. `min_candies_bruteforce()` 在小规模上通过 DFS 枚举可行分配，求精确最小总数。  
8. 随机测试把贪心结果与穷举结果逐例比对，全部通过后输出成功摘要。  

实现未调用任何第三方“糖果分发黑盒函数”；`numpy` 仅用于测试数据生成。
