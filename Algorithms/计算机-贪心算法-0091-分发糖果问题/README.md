## R01

分发糖果问题（Candy Distribution）是经典的一维约束最小化贪心题。

给定每个孩子的评分数组 `ratings`，要求分配糖果数 `candies`，满足：
- 每个孩子至少 1 颗糖；
- 若 `ratings[i] > ratings[i-1]`，则 `candies[i] > candies[i-1]`；
- 若 `ratings[i] > ratings[i+1]`，则 `candies[i] > candies[i+1]`。

目标是在满足约束的前提下，让糖果总数 `sum(candies)` 最小。

## R02

问题定位：
- 学术上可视为“链式邻接不等式约束下的最小和可行解”；
- 工程上对应“局部优先级规则 + 全局成本最小化”；
- 在算法训练中常用于理解“单向贪心不足，需要双向修正”的思想。

该题的关键不在“给出任一可行解”，而在“保证最小总量”。

## R03

形式化定义：
- 输入：长度为 `n` 的评分序列 `ratings`（本实现使用有限数值，按整数比较）。
- 输出：长度为 `n` 的正整数序列 `candies`。
- 约束：
  - `candies[i] >= 1`
  - 若 `ratings[i] > ratings[i-1]`，则 `candies[i] >= candies[i-1] + 1`
  - 若 `ratings[i] > ratings[i+1]`，则 `candies[i] >= candies[i+1] + 1`
- 优化目标：最小化 `sum(candies)`。

空输入约定返回总糖果数 `0`。

## R04

核心贪心策略（双向扫描）：
1. 初始化所有孩子糖果数为 1。
2. 从左到右扫描：只处理“右边评分更高”的约束。
3. 从右到左扫描：只处理“左边评分更高”的约束，并用 `max` 保留左扫结果。
4. 两遍后同时满足两侧约束，且总和最小。

原因是任意单向扫描只能满足一侧单调关系，另一侧必须由反向补偿。

## R05

状态变量设计：
- `ratings`：输入评分；
- `candies`：当前糖果分配数组，初始化为全 1；
- 左扫阶段更新规则：
  - 若 `ratings[i] > ratings[i-1]`，设 `candies[i] = candies[i-1] + 1`；
- 右扫阶段更新规则：
  - 若 `ratings[i] > ratings[i+1]`，设 `candies[i] = max(candies[i], candies[i+1] + 1)`。

最终 `candies` 是可解释的逐位置决策结果，不是黑盒输出。

## R06

正确性直觉：
- 左扫后：所有“上升边”约束都被满足，且这些位置取到局部最小值；
- 右扫后：所有“下降边”约束补齐，且仅在必要时增加糖果；
- 使用 `max` 保证不会破坏左扫已满足的约束。

因此两遍结束时：
- 可行性成立（所有邻接约束都满足）；
- 由于每次只做必要增量，总和达到最小。

## R07

复杂度分析：
- 时间复杂度：`O(n)`（两次线性扫描）；
- 空间复杂度：`O(n)`（存储 `candies` 数组）；
- 若只求总和且可覆盖输入，可做更复杂的 `O(1)` 额外空间变体，但可读性较差。

本 MVP 优先使用清晰可审计的 `O(n)` 空间版本。

## R08

与常见错误方案对比：
- 只做左到右：无法满足右侧更高评分约束；
- 只做右到左：无法满足左侧更高评分约束；
- 局部反复微调直到稳定：实现复杂且可能退化到更高复杂度。

双向贪心是该问题最直接、稳定、可证明最优的线性方案。

## R09

边界情况：
- 空数组：返回 `0`；
- 单元素：返回 `1`；
- 全相等评分：每人 1 颗，总数 `n`；
- 严格递增：糖果为 `1..n`；
- 严格递减：糖果为 `n..1`；
- 含负数评分：允许，比较关系不受影响；
- 非有限数值（`NaN/Inf`）：拒绝输入。

## R10

MVP 的实现目标：
- 明确实现双向贪心主算法 `candy_greedy`；
- 输出每个案例的评分、分配和总糖果；
- 使用小规模精确枚举 `candy_exact_small` 做正确性对照；
- 使用 `numpy` 生成固定随机样本；
- 使用 `pandas` 打印按位置的结果表，方便人工审计。

## R11

伪代码：

```text
candy(ratings):
  n <- len(ratings)
  if n == 0: return 0, []
  candies <- [1] * n

  for i in [1 .. n-1]:
    if ratings[i] > ratings[i-1]:
      candies[i] <- candies[i-1] + 1

  for i in [n-2 .. 0]:
    if ratings[i] > ratings[i+1]:
      candies[i] <- max(candies[i], candies[i+1] + 1)

  return sum(candies), candies
```

## R12

正确性验证策略：
- 固定样例验证：
  - 峰谷混合；
  - 全相等；
  - 严格递增；
  - 严格递减；
  - 含负数评分。
- 随机小样本验证：
  - `numpy` 固定随机种子生成评分；
  - 与精确枚举最优值比对。
- 断言内容：
  - 邻接约束全部满足；
  - 贪心总和等于精确最优值（小规模）。

## R15

与其他方法关系：
- 差分约束视角：可转化为最小可行整数解问题；
- 图论视角：邻接关系形成链图上的方向约束；
- 线性规划视角：可写成线性约束最小化，但实现成本高于贪心。

在一维链结构下，双向贪心已经足够高效且最优。

## R16

适用与不适用场景：
- 适用：
  - 一维相邻比较约束；
  - 只需最小总量，不要求多目标优化；
  - 需要线性时间处理大输入。
- 不适用：
  - 非链式邻接图（例如二维网格、一般图）；
  - 约束含复杂全局依赖；
  - 目标函数不是总和最小（如最小最大值）。

## R18

`demo.py` 源码级流程（8 步，非黑盒）：
1. `main` 调用固定案例测试和随机案例测试，形成端到端执行。
2. `validate_ratings` 对输入做数值与有限性校验，统一为整数列表。
3. `candy_greedy` 初始化 `candies=[1]*n`，保证“每人至少 1 颗”基础约束。
4. 第一遍左扫：遇到 `ratings[i] > ratings[i-1]` 就把 `candies[i]` 提升到 `candies[i-1]+1`。
5. 第二遍右扫：遇到 `ratings[i] > ratings[i+1]` 就用 `max(candies[i], candies[i+1]+1)` 做必要补偿。
6. `assert_distribution_valid` 逐边检查相邻约束和下界约束，保证分配可行。
7. `candy_exact_small` 在小规模下用分支定界枚举所有可行分配，求出最小总糖果作为真值。
8. 将贪心总量与精确真值比对并打印 `pandas` 表格；一致则确认实现正确。

补充：第三方库仅用于数据生成和可视化输出，核心最优分配算法完全在源码中逐步实现。
