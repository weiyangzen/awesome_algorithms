## R01

二维变换矩阵用于把点 `(x, y)` 映射到新位置 `(x', y')`，是计算机图形学中最基础的几何操作表达。

常见操作包括：
- 平移（Translation）
- 旋转（Rotation）
- 缩放（Scaling）
- 错切（Shear）

这些操作都可以统一写成矩阵乘法，从而支持批量处理与高效组合。

## R02

为了把“平移”也纳入矩阵乘法框架，通常采用齐次坐标，把二维点扩展为三维向量：

- 欧氏坐标：`p = [x, y]^T`
- 齐次坐标：`p_h = [x, y, 1]^T`

于是任意二维仿射变换都可写为 `3x3` 矩阵：

- `p'_h = M @ p_h`
- 结果再投影回二维：`(x', y') = (p'_h[0]/w, p'_h[1]/w)`，其中 `w = p'_h[2]`

## R03

本题涉及的基础矩阵如下（列向量约定）：

```text
平移 T(tx, ty) = [[1, 0, tx],
                   [0, 1, ty],
                   [0, 0, 1 ]]

旋转 R(θ)      = [[cosθ, -sinθ, 0],
                   [sinθ,  cosθ, 0],
                   [0,      0,   1]]

缩放 S(sx, sy) = [[sx, 0,  0],
                   [0,  sy, 0],
                   [0,  0,  1]]

错切 H(shx,shy)= [[1,   shx, 0],
                   [shy, 1,   0],
                   [0,   0,   1]]
```

## R04

矩阵组合时顺序非常关键。

若对点依次应用 `M1 -> M2 -> M3`，则总矩阵为：

- `M_total = M3 @ M2 @ M1`

即“后应用的变换”写在左边。这是很多实现中最容易写错的地方。

## R05

本目录 MVP 的目标：

1. 提供平移/旋转/缩放/错切矩阵构造函数；
2. 提供围绕任意中心点（pivot）的变换封装；
3. 验证“逐步变换”与“组合矩阵一次变换”数值一致；
4. 验证逆矩阵可恢复原始点；
5. 给出小规模几何结果与大规模点云性能对比。

## R06

`demo.py` 的核心函数：

- `to_homogeneous` / `from_homogeneous`：二维与齐次坐标互转；
- `translation_matrix` / `rotation_matrix` / `scaling_matrix` / `shear_matrix`：基础变换矩阵生成；
- `transform_about_pivot`：把任意变换改写为“绕指定中心”；
- `compose_transforms`：按应用顺序生成总矩阵；
- `apply_transform`：对 `N x 2` 点集批量施加变换。

另外实现了 `polygon_area` 用于验证面积缩放关系。

## R07

正确性依赖以下不变量：

1. 输入点数组必须是 `(N, 2)`；
2. 变换矩阵必须是 `3x3`；
3. 齐次投影时 `w != 0`；
4. `sequential_result == composite_result`（在浮点容差内）；
5. `inverse(composite)` 可将结果点恢复到原始点（误差在阈值内）。

## R08

复杂度（`N` 为点数，`k` 为变换个数）：

- 单次 `apply_transform`：`O(N)`；
- 逐步应用 `k` 个矩阵：`O(kN)`；
- 先组合再应用：组合开销 `O(k)`（`3x3` 常数矩阵），应用开销 `O(N)`；
- 空间复杂度：`O(N)`（点集存储与中间数组）。

这解释了为什么组合矩阵通常更快。

## R09

数值稳定性与工程注意点：

- 浮点误差不可避免，比较时应使用 `np.allclose`；
- 连续多次变换会积累误差，使用组合矩阵可减少中间舍入；
- 若线性部分行列式接近 0，矩阵可能病态，逆变换会放大误差；
- 角度统一转弧度，避免角度单位混淆。

## R12

输出指标含义：

- `sequential_ms`：逐步应用多个矩阵的耗时；
- `composite_ms`：先合成总矩阵再一次变换的耗时；
- `speedup_seq_over_comp`：两者耗时比值；
- `area_ratio`：变换后面积 / 变换前面积；
- `abs_det_linear`：总矩阵线性部分行列式绝对值；
- `max_restore_err`：逆变换恢复原点的最大误差。

理论上 `area_ratio` 应接近 `abs_det_linear`。

## R13

`demo.py` 内置断言：

1. 逐步变换结果与组合矩阵结果必须一致；
2. 点云逐步结果与点云组合结果必须一致；
3. 逆变换恢复误差不得超过阈值（`1e-9`）；
4. 输入形状和矩阵维度不合法时抛出明确异常。

这些门禁保证“可运行”同时“可验证”。

## R14

边界与异常处理：

- 非 `(N,2)` 点集会触发 `ValueError`；
- 非 `3x3` 变换矩阵会触发 `ValueError`；
- 齐次坐标 `w=0` 会触发 `ValueError`，防止非法投影；
- 若变换矩阵不可逆，`np.linalg.inv` 会抛线性代数异常（本例参数避免该情况）。

## R18

源码级算法链路（8 步）：

1. `main()` 定义原始点集 `shape`，并计算几何中心 `centroid`。  
2. 构造基础矩阵：`scaling_matrix / rotation_matrix / shear_matrix / translation_matrix`，其中缩放和旋转通过 `transform_about_pivot` 改写为“先移到原点、变换、再移回”。  
3. `compose_transforms` 逐个读取变换矩阵并执行 `composite = mat @ composite`，得到与顺序应用语义一致的总矩阵。  
4. `apply_transform` 先调用 `to_homogeneous` 把 `N x 2` 点扩展为 `N x 3`，再做批量矩阵乘法 `(M @ P^T)^T`。  
5. `from_homogeneous` 对结果执行 `x/w, y/w` 投影，得到二维坐标；`main()` 分别得到 sequential 与 one-shot 结果并用 `np.allclose` 校验。  
6. `main()` 计算 `inv_composite = np.linalg.inv(composite)`，再用同一 `apply_transform` 把变换后点映射回去，统计恢复误差并设阈值断言。  
7. 在随机点云上分别执行“多次 apply”与“一次 apply”，比较 `sequential_ms` 与 `composite_ms`，验证组合矩阵减少重复遍历与中间数组写回。  
8. 最后计算线性部分行列式 `det(composite[:2,:2])` 与面积比 `area_after/area_before`，用 `pandas` 汇总打印，闭环验证几何量与线性代数结论一致。
