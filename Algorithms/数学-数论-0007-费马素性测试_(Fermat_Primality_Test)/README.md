# 费马素性测试 (Fermat Primality Test)

- UID: `MATH-0007`
- 学科: `数学`
- 分类: `数论`
- 源序号: `7`
- 目标目录: `Algorithms/数学-数论-0007-费马素性测试_(Fermat_Primality_Test)`

## R01

费马素性测试是一种随机化素性判定方法，利用费马小定理：若 `p` 为素数且 `gcd(a,p)=1`，则 `a^(p-1) ≡ 1 (mod p)`。输入为待测整数 `n` 和轮数 `k`；输出为“合数”或“疑似素数(probable prime)”。
主要流程：
1. 处理小数与偶数等快速分支。
2. 重复 `k` 轮，随机选底数 `a ∈ [2, n-2]`。
3. 若 `gcd(a,n) > 1`，直接判合数。
4. 计算 `pow(a, n-1, n)`；若结果不为 `1`，判合数。
5. 若 `k` 轮都通过，返回“疑似素数”。

## R02

时间脉络：
- 1640 年：Pierre de Fermat 提出费马小定理（通过书信流传）。
- 18 世纪：Euler 给出更系统证明并推广相关同余理论。
- 20 世纪后期：该思想被用于现代随机化素性测试流程，常作为更强测试（如 Miller-Rabin）前的快速过滤步骤。

## R03

提出背景是“试除法在大整数上过慢”。对接近 `N` 的整数，试除到 `sqrt(N)` 的成本很高；密码学和计算数论需要更快的候选筛选。费马测试把“证明是素数”放宽为“高概率不是合数”，用更低成本快速排除大部分合数。

## R04

设测试轮数为 `k`。
- 每轮核心是一次模幂 `a^(n-1) mod n`，使用快速幂需要 `O(log n)` 次模乘。
- 若按位复杂度计，单次大整数模乘成本依赖乘法实现，整体可写为 `O(k * log n * M(log n))`，其中 `M(t)` 为 `t` 位整数乘法复杂度。
- 在常见实现与中等规模输入下，可近似理解为“对 `k` 线性增长、对 `log n` 多项式增长”。

## R05

空间复杂度为 `O(1)` 个大整数寄存槽（不计输入本身），位空间量级约 `O(log n)`；实现中仅需维护当前底数、指数迭代状态和模幂中间值。

## R06

示例 1（素数）：`n=13`。
- 取 `a=2`，`2^12 mod 13 = 1`。
- 多个底数重复后都通过，返回“疑似素数”（此例真实为素数）。

示例 2（伪素数）：`n=341=11*31`。
- 取 `a=2`，`2^340 mod 341 = 1`，会“误过”。
- 说明费马测试可能把合数误判为疑似素数。

## R07

算法意义：
- 在理论上，它是“随机化素性测试”思想的经典起点。
- 在工程上，它能快速剔除大量明显合数，降低后续强测试负担。
- 在教学上，它清晰展示了“必要条件检验不等于充分条件检验”的算法设计边界。

## R08

直接依赖的先导知识与算法：
- 费马小定理（数论基础定理）。
- 欧几里得算法（通过 `gcd(a,n)` 快速发现非平凡因子）。
- 二进制快速幂/模重复平方法（高效计算模幂）。
- 基本随机采样机制（多轮独立底数抽样）。

## R09

适用前提与边界：
- 输入通常取 `n >= 2` 的整数。
- 算法只给“合数”或“疑似素数”，不提供确定性素性证明。
- 对 Carmichael 数（如 `561, 1105, 1729`）存在系统性弱点：对所有与 `n` 互素的底数都可能通过费马同余。

## R10

正确性依据：
- 若算法返回“合数”，其依据是存在证据 `a` 使得 `gcd(a,n)>1` 或 `a^(n-1) mod n != 1`，与素数必要条件矛盾，因此判定可靠。
- 若算法返回“疑似素数”，只说明“未找到反例”，并不构成充分证明。

## R11

该算法属于整数模运算，不涉及浮点迭代，因此不存在传统“数值舍入不稳定”。主要误差来源是统计误判（把合数看成疑似素数），不是数值舍入误差传播。

## R12

真实成本（超越渐近符号）主要来自：
- 大整数模幂常数项较大，是绝对主耗时。
- 轮数 `k` 线性影响总时长，`k` 每增加 1，几乎增加一次完整模幂开销。
- `gcd` 检查代价低于模幂，常用于提前失败。
- 对单个大整数判定，向量化收益有限；批处理时可通过多进程并行到数字级别，而不是算子级别。

## R13

概率保证：
- 对“非 Carmichael 合数”，随机底数作为“见证者”的概率通常有常数下界，因此多轮后误判概率呈指数下降（常见保守写法为每轮最多 `1/2` 级别的错误保留概率）。
- 对 Carmichael 数，上述保证不成立；若采样底数恰与 `n` 互素，可能全部通过费马条件，因此费马测试不是工业级最终判定器。

## R14

鲁棒性与失效模式：
- 失效模式 1：Fermat 伪素数（如 `341` 在某些底数下通过）。
- 失效模式 2：Carmichael 数对互素底数“全通过”，导致系统性误判风险。
- 工程缓解：将费马测试仅作为预筛，后接 Miller-Rabin 或确定性算法。

## R15

工程实现要点：
- 必须先处理 `n<2`、`n in {2,3}`、偶数等分支，避免无效随机区间。
- 每轮先做 `gcd(a,n)`，能提前发现因子并减少模幂次数。
- 为复现实验，应固定随机种子。
- 轮数建议参数化（例如 `k=5~20`），并在文档中明确“概率含义”。

## R16

前驱/后继与应用：
- 前驱基础：费马小定理、欧几里得算法、快速幂。
- 后继改进：Solovay-Strassen、Miller-Rabin（更强随机测试）、AKS（确定性多项式时间理论里程碑）。
- 典型应用：RSA 密钥生成时的大量候选数预筛、竞赛与教学中的快速素性初判。

## R18

源码追踪与算法拆解（避免把 `pow(a, b, mod)` 当黑箱）：

在实现层面，`pow(a, e, n)` 对应“模重复平方法（binary exponentiation with modular reduction）”，其核心流程可拆成 6 步：
1. 初始化：`result = 1`，`base = a mod n`，`exp = e`。
2. 读取 `exp` 当前最低位；若该位为 `1`，执行 `result = (result * base) mod n`。
3. 执行平方推进：`base = (base * base) mod n`。
4. 右移指数：`exp >>= 1`。
5. 重复步骤 2-4，直到 `exp == 0`。
6. 返回 `result`，即 `a^e mod n`。

费马测试在每一轮只是在该模幂内核外再包两层逻辑：`gcd` 早停 + 同余检查。也就是说，即便代码里只写了一行 `pow`，算法本体仍可还原为“随机选底数 + gcd 过滤 + 二进制模幂 + 同余判定”的可解释流程。
