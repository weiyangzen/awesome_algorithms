## R01

多项式求逆（更准确地说是“形式幂级数求逆”）问题如下。

给定多项式（或幂级数）
`A(x) = a0 + a1 x + a2 x^2 + ...`，在某个环/域上求 `B(x)`，使得
`A(x) * B(x) ≡ 1 (mod x^n)`。

- 输入: 系数序列 `a[0..m-1]`、目标截断阶 `n`、系数运算模数（本目录 demo 使用素数模）。
- 输出: 长度为 `n` 的系数序列 `b[0..n-1]`，满足卷积后前 `n` 项是单位元。
- 可解条件: 常数项 `a0` 在系数环中可逆（模素数场中即 `a0 != 0 mod p`）。
- 本条目采用 Newton 迭代（倍增精度）作为主算法。

## R02

历史脉络可分两层。

- 牛顿迭代思想来自 17 世纪 Newton 对函数方程的迭代修正思想。
- 将其系统化用于形式幂级数快速运算（逆、对数、指数、开方等）是现代计算代数与快速傅里叶卷积发展后的成果，成为计算机代数系统和竞赛算法中的标准做法。

## R03

问题背景是“高阶截断下的快速代数运算”。

朴素做法可由递推式逐项解出逆系数，时间 `O(n^2)`；当 `n` 上千或上万时，常成为瓶颈。Newton 迭代把“低精度逆”快速提升到“高精度逆”，并把核心代价压缩到若干次多项式卷积，便于借助 FFT/NTT 提速。

## R04

时间复杂度由多项式乘法复杂度 `M(n)` 决定。

- Newton 求逆复杂度: `O(M(n) log n)`。
- 若乘法用朴素卷积 `M(n)=O(n^2)`，则总复杂度约 `O(n^2 log n)`。
- 若乘法用 FFT/NTT 近线性乘法（常见记号 `M(n)=O(n log n)`），则总复杂度可写为 `O(n log n log n)`（工程上通常表现为“接近线性”）。

## R05

空间复杂度（工作内存）通常是 `O(n)` 到 `O(n log n)` 之间，取决于卷积实现。

- 本目录 MVP（含截断缓存）按数组计量约 `O(n)`。
- 若底层 FFT/NTT 实现做多缓冲，常数和附加缓存会更高，但阶数仍可控。

## R06

示例：在整数域（不取模）下，求
`A(x)=1+2x+3x^2` 的逆前 4 项。

设
`B(x)=b0+b1 x+b2 x^2+b3 x^3`，满足 `A(x)B(x) ≡ 1 (mod x^4)`。
逐项比较可得：

1. 常数项: `b0=1`
2. `x^1` 项: `2b0+b1=0 => b1=-2`
3. `x^2` 项: `3b0+2b1+b2=0 => b2=1`
4. `x^3` 项: `3b1+2b2+b3=0 => b3=4`

因此 `B(x)=1-2x+x^2+4x^3`，可验证 `(1+2x+3x^2)B(x)=1+O(x^4)`。

## R07

算法意义。

- 代数基础能力: 多项式逆是形式幂级数工具链的基础模块。
- 组合与生成函数: 许多生成函数求解依赖幂级数求逆。
- 高性能计算: 在 NTT 体系中是常用原语，用于快速除法、微分方程级数解、在线卷积等。

## R08

核心理论依赖。

- 形式幂级数环中的可逆性判定: `a0` 可逆当且仅当级数可逆。
- 卷积与截断: 只保留 `< x^n` 的项，定义 `mod x^n` 语义。
- Newton 修正公式:
  `B_{k+1} = B_k * (2 - A * B_k)  (mod x^{2k})`。
- 误差平方机制: 令 `E_k = 1 - A B_k`，则 `E_{k+1} = E_k^2`（在截断意义下）。

## R09

适用前提与边界。

- 必须保证常数项可逆；若 `a0=0 (mod p)`，算法应立即报错。
- 需要明确计算域：
  - 模素数域：结果精确、稳定、适合程序实现；
  - 浮点域：会有舍入误差，不适合作“严格等式”验证。
- `n<=0` 时通常返回空多项式。

## R10

正确性（简述）。

设当前 `B` 满足 `A B ≡ 1 (mod x^m)`，记误差 `E = 1 - A B`，则 `E = O(x^m)`。

构造
`B' = B(2 - AB) = B(1 + E)`。

于是
`A B' = AB(2-AB) = (1-E)(1+E) = 1 - E^2`。

因为 `E=O(x^m)`，所以 `E^2=O(x^{2m})`，得到
`A B' ≡ 1 (mod x^{2m})`。

即每次迭代“正确位数翻倍”，从 `m=1` 出发，经过 `O(log n)` 轮可达目标 `n`。

## R11

数值稳定性与误差。

- 本目录实现采用模素数整数运算，不存在浮点舍入误差传播，属于离散精确算法。
- 主要风险来自实现层：
  - 中间乘加可能超出固定宽度整型；
  - 模运算遗漏会导致错误累积。
- Python `int` 是任意精度整数，规避了溢出风险（以时间常数为代价）。

## R12

真实成本（工程视角）。

- 迭代轮数很少（按二倍扩展），大头成本在卷积。
- 当 `n` 较小，朴素卷积常数小，可能比 FFT/NTT 更快。
- 当 `n` 较大，应切换到 NTT/FFT 实现；Newton 框架本身无需改变。
- 实际系统常做“阈值分段”：小规模朴素、大规模频域。

## R14

鲁棒性与失效模式。

- 常数项不可逆: 无解，应抛出明确异常。
- 输入长度不足: 逻辑上缺失系数视为 `0`，实现需自动补零。
- 模数非素数时: 虽可运行，但 `a0` 可逆性不再等价于 `a0!=0`，需改为 `gcd(a0, mod)=1` 判定。
- 过度依赖黑箱库: 若直接调用高层“求逆 API”而不理解过程，排障困难；本条目显式实现 Newton 流程。

## R15

工程实现要点。

- 先做输入规约：全部系数映射到 `[0, mod)`。
- 每次卷积后立刻截断到目标阶，避免无意义高次项膨胀。
- 迭代规模采用 `1 -> 2 -> 4 -> ...`，直到覆盖 `n`。
- 用朴素递推结果做交叉校验，能快速发现实现错误。
- 产线实现可替换卷积内核（朴素/FFT/NTT），上层 Newton 框架保持不变。

## R16

前驱、后继与应用。

- 前驱基础:
  - 多项式乘法/卷积。
  - 模逆（常数项逆元）。
- 后继模块:
  - 多项式除法 `Q = F * G^{-1}`。
  - 形式幂级数 `ln/exp/sqrt` 等快速算法。
  - 高阶线性递推、生成函数求系数。
- 典型应用:
  - 竞赛中的 NTT 模板链路。
  - 计算机代数系统中的幂级数运算。

## R17

Python MVP 说明（见 `demo.py`）。

- 目标: 提供最小可运行、非交互、可自检的多项式求逆实现。
- 运算域: 模素数 `998244353`。
- 核心函数:
  - `poly_mul_trunc_mod`: 截断卷积（优先 `numpy.convolve`，缺失时回退朴素卷积）。
  - `poly_inv_newton_mod`: Newton 倍增求逆。
  - `poly_inv_naive_mod`: `O(n^2)` 递推求逆（用于交叉验证）。
- 演示内容:
  - 3 组固定样例 + 1 组伪随机样例。
  - 检查 `A(x)*B(x) mod x^n` 是否等于 `1`。
  - 检查 Newton 结果是否与朴素递推一致。
- 运行方式:
  - `python3 demo.py`

## R18

源码级算法流追踪（`demo.py`，非黑箱）。

1. `main` 组织样例，把原始系数规约到模域 `[0, MOD)`，并设置目标截断阶 `n`。
2. 调用 `poly_inv_newton_mod(a, n, MOD)`：先检查 `a[0]` 可逆，初始化 `b=[a0^{-1}]`（只保证 `mod x` 正确）。
3. 进入倍增循环，当前精度 `m` 扩到 `m2=min(2m, n)`；这一步决定本轮要修正到的截断阶。
4. 用 `poly_mul_trunc_mod` 计算 `ab = A*B (mod x^{m2})`，构造修正多项式 `c = 2 - ab`（常数项是 `2`，其余项取负）。
5. 再做一次截断卷积 `b <- B*c (mod x^{m2})`，根据 `AB' = 1 - (1-AB)^2` 把正确阶数翻倍。
6. 循环结束后返回前 `n` 项逆系数；`main` 再调用 `poly_inv_naive_mod` 做逐项递推对照，并验证 `A*B ≡ 1 (mod x^n)`，全部断言通过即输出成功。

补充：这里使用 `numpy.convolve` 仅作为“乘法内核”，并未调用任何“多项式逆”黑箱 API，Newton 主逻辑完全在源码中显式展开。
