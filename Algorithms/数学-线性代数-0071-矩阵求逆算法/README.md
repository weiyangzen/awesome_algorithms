# 矩阵求逆算法

- UID: `MATH-0071`
- 学科: `数学`
- 分类: `线性代数`

- 目标目录: `Algorithms/数学-线性代数-0071-矩阵求逆算法`

## R01

本条目实现“矩阵求逆算法”的最小可运行 MVP，核心目标是把从数学定义到可执行代码的完整链路打通，并清晰展示数值稳定性处理。

实现定位：
- 使用 Gauss-Jordan 消元法计算逆矩阵；
- 使用部分主元选取（partial pivoting）提升稳定性；
- 以 `numpy` 完成矩阵运算与结果校验；
- 提供可直接运行的 `demo.py`，无交互输入。

## R02

问题定义（本 MVP）：
- 输入：一个 `n x n` 实矩阵 `A`（`n >= 1`）。
- 输出：若 `A` 可逆，输出 `A^{-1}`；若不可逆，抛出明确异常。
- 验证目标：检查 `A @ A^{-1}` 是否接近单位阵 `I`，并报告残差范数。

可逆性判定原则：
- 消元过程中若当前列最大主元绝对值小于阈值 `eps`，判定为奇异或病态到不可稳定求逆。

## R03

数学基础：

1. 逆矩阵定义  
- 对方阵 `A`，若存在矩阵 `B` 使得 `AB = BA = I`，则 `B = A^{-1}`。

2. 增广矩阵思想  
- 构造 `[A | I]`；
- 通过初等行变换将左半部分 `A` 变为 `I`；
- 同步变换后右半部分即为 `A^{-1}`。

3. 初等行变换类型  
- 交换两行；
- 某行乘以非零常数；
- 某行加上另一行的倍数。

4. 稳定性关键点  
- 每一步优先选择当前列绝对值最大的可用主元（部分主元）以减小舍入误差放大。

## R04

算法流程（Gauss-Jordan + 部分主元）：
1. 校验输入是否为二维方阵。
2. 构造双精度增广矩阵 `aug = [A | I]`。
3. 对每一列 `col=0..n-1`：
   - 在 `col..n-1` 行中找绝对值最大主元行；
   - 若主元过小则报奇异；
   - 必要时交换主元行到当前行；
   - 将当前行除以主元，使主元归一；
   - 对其余行做消元，使该列除当前行外全为 0。
4. 循环结束后，`aug` 左半为单位阵，右半即逆矩阵。
5. 输出逆矩阵并给出数值残差。

## R05

核心数据结构：
- `A: ndarray (n, n)`：输入矩阵（`float64`）。
- `I: ndarray (n, n)`：单位阵。
- `aug: ndarray (n, 2n)`：增广矩阵，原地执行消元。
- `A_inv: ndarray (n, n)`：从 `aug[:, n:]` 切片得到。

主要中间量：
- `pivot_row`：当前列选出的主元行；
- `pivot_value`：主元值；
- `factor`：行消元系数。

## R06

正确性要点：
- 每一步都保持“增广矩阵对应同一组行变换”的不变式，因此若左侧最终为 `I`，右侧必为 `A^{-1}`。
- 行交换不会改变方程组解集，仅改变行顺序。
- 主元行归一 + 其他行消元后，当前列被严格化为标准基列。
- 经过 `n` 列处理后，左块成为 `I`，满足逆矩阵构造条件。
- 通过 `A @ A_inv ≈ I` 与 `A_inv @ A ≈ I` 的残差双向检查支持实现正确性。

## R07

复杂度分析：
- 时间复杂度：`O(n^3)`  
  原因：每列消元需处理 `O(n)` 行，每行含 `O(n)` 长度操作，共 `n` 列。
- 空间复杂度：`O(n^2)`  
  主要由增广矩阵 `(n, 2n)` 与若干同阶临时量构成。

适用规模：
- 适合中小规模教学与原型验证；
- 大规模问题通常优先使用 LU 分解复用或稀疏专用方法。

## R08

数值稳定性与异常处理：
- 采用部分主元选取，减少主元过小导致的不稳定放大。
- 通过 `eps` 阈值判定“近零主元”，避免除以数值噪声。
- 对奇异矩阵显式抛出 `numpy.linalg.LinAlgError`，而非静默产生错误结果。
- 使用 `float64` 以减小舍入误差影响。

## R09

MVP 取舍说明：
- 本实现优先“可读性 + 可验证性”，不追求 BLAS/LAPACK 级极致性能。
- 不引入复杂块分解、并行或 GPU，仅使用 `numpy` 保持环境轻量。
- 仍调用 `numpy.linalg.inv`，但仅作为交叉验证，不替代主算法主体。
- 对不可逆场景提供独立演示，确保边界行为可见。

## R10

`demo.py` 主要函数职责：
- `gauss_jordan_inverse`：执行矩阵求逆主流程（含主元选择、归一化、消元）。
- `compute_quality_metrics`：计算残差、双向误差与与参考解差异。
- `run_case`：运行单个案例，打印矩阵、逆矩阵与诊断指标。
- `main`：构造可逆/近病态/奇异示例并串行执行。

## R12

输出字段说明：
- `cond(A)`：矩阵条件数（2-范数），越大通常越病态。
- `residual_left`：`||A @ A_inv - I||_inf`。
- `residual_right`：`||A_inv @ A - I||_inf`。
- `distance_to_numpy_inv`：与 `numpy.linalg.inv` 结果差异的无穷范数。

解读建议：
- 在非奇异且条件数适中的场景下，残差通常应接近机器精度量级。
- 条件数极高时，算法仍可运行但误差会显著变大，属于问题本身病态而非实现错误。

## R13

建议最小测试集：
- 随机可逆矩阵：验证常规正确性与残差量级。
- 固定已知矩阵：便于人工复核中间结果与输出格式。
- 高条件数矩阵（如 Hilbert）：观察病态带来的误差放大。
- 奇异矩阵（行线性相关）：确认异常分支稳定触发。

## R14

关键可调参数：
- `eps`：主元零阈值，默认 `1e-12`；
- 随机种子：用于生成可复现实验矩阵；
- 案例矩阵规模 `n`：用于观察 `O(n^3)` 开销增长；
- 案例矩阵类型（随机、Hilbert、奇异构造）：用于覆盖稳定性与失败路径。

## R15

方法对比：
- Gauss-Jordan（本实现）
  - 优点：思路直接，输出逆矩阵一步到位；
  - 缺点：常数因子较大，数值稳定性依赖主元策略。
- LU 分解法
  - 常用于工程实践，通过分解后求解 `AX=I` 的多列右端；
  - 在复用场景（多次求解）更高效。
- SVD / 伪逆
  - 对奇异或近奇异更稳健；
  - 代价更高，且目标通常是最小二乘意义下的广义逆。

## R18

源码级算法流（`demo.py`，8 步）：
1. `main` 构造三个案例矩阵：可逆随机矩阵、Hilbert 病态矩阵、奇异矩阵。  
2. `run_case` 接收单个矩阵并先打印输入矩阵与基本信息（形状、条件数）。  
3. `run_case` 调用 `gauss_jordan_inverse(A, eps)`，函数先校验 `A` 为方阵并构造增广矩阵 `[A | I]`。  
4. 在 `gauss_jordan_inverse` 的每一列中，从当前行以下选择绝对值最大主元行；若主元小于 `eps`，立即抛出奇异异常。  
5. 若主元行不是当前行，先交换两行；随后把当前行整体除以主元，使该列主元变成 1。  
6. 对其他所有行执行“行减去系数乘主元行”，把当前列除主元外全部消为 0，逐列重复直到左块变为单位阵。  
7. 从增广矩阵右半块切出 `A_inv` 返回给 `run_case`；随后 `run_case` 调用 `compute_quality_metrics` 计算左右残差与和 `numpy.linalg.inv` 的差异。  
8. `run_case` 打印逆矩阵与误差指标；对奇异矩阵则捕获异常并输出“预期失败”，完成无交互演示闭环。
