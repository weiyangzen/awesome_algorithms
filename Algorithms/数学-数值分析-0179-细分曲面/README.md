## R01

细分曲面（Subdivision Surface）是把离散多边形网格通过迭代细分与重定位，逐步逼近光滑极限曲面的算法族。它不是直接拟合一个全局高次曲面，而是重复执行“局部规则”，因此在复杂拓扑和工业建模中非常实用。

在工程里，最常见的是：
- Catmull-Clark（四边形主导，动画与建模常用）
- Loop（三角网格主导）

本条目选择 Catmull-Clark 作为最小可运行 MVP。

## R02

问题定义（以 Catmull-Clark 为例）：

给定一个控制网格 `M=(V,F)`，其中 `V` 是顶点坐标集合，`F` 是面集合（本 MVP 限定为四边形面），构造一系列细分网格 `M_0, M_1, ..., M_k`，满足：
- `M_0` 为输入网格
- `M_{t+1} = S(M_t)`，`S` 为细分算子
- 随着 `t` 增大，网格表面更平滑

输入：
- 顶点数组 `V x 3`
- 四边形面索引列表
- 迭代次数 `k`

输出：
- 新顶点数组与新四边形面列表
- 可验证统计（`V/E/F` 数量、Euler 特征、边长和半径分布）

## R03

Catmull-Clark 核心思想：先创建三类新点，再重连拓扑。

1. 面点（Face Point）：每个旧面的顶点均值。
2. 边点（Edge Point）：每条旧边由端点与邻接面点共同决定。
3. 顶点点（Vertex Point）：每个旧顶点按局部邻域做加权更新。
4. 重建面：每个旧四边形被拆成 4 个新四边形。

这个过程是纯局部规则，不依赖全局求解器，因此实现清晰、可并行化。

## R04

本 MVP 使用的 Catmull-Clark 更新公式（与 `demo.py` 完全一致）：

设旧顶点 `v` 的价数（邻接面数）为 `n`，`F_avg` 为其邻接面点均值，`R_avg` 为其邻接边中点均值。

1. 面点：
`f = (1/m) * Σ v_i`（`m=4`，因为本 MVP 限四边形）

2. 内部边点（边邻接两个面）：
`e = (v_a + v_b + f_0 + f_1) / 4`

3. 内部顶点更新：
`v' = (F_avg + 2*R_avg + (n-3)*v) / n`

4. 新面重连（对旧面每个角各造一个四边形）：
`(v'_i, e_{i,i+1}, f_face, e_{i-1,i})`

补充：`demo.py` 还实现了边界回退规则（边中点与边界顶点平滑），但主实验输入是封闭立方体，不触发边界分支。

## R05

伪代码：

```text
input: quad mesh (V, F), iterations k

for t in 1..k:
  build topology: edge->faces, vertex->faces, vertex->edges

  face_points[f] = mean(vertices of face f)

  for each edge (a,b):
    if interior (2 adjacent faces):
      edge_points = (Va + Vb + F0 + F1) / 4
    else:
      edge_points = boundary fallback

  for each old vertex v:
    if interior:
      F_avg = mean(adjacent face points)
      R_avg = mean(midpoints of adjacent edges)
      v_new = (F_avg + 2*R_avg + (n-3)*v) / n
    else:
      v_new = boundary fallback

  concatenate new vertex array:
    [updated_old_vertices | edge_points | face_points]

  rebuild quads by local corner rule

return refined mesh
```

## R06

正确性直觉：

- 局部一致性：每个新点只依赖 1-ring 邻域（顶点邻域），不会出现远程跳变。
- 几何平滑：顶点更新把“面均值 + 边均值 + 原顶点”混合，能抑制尖锐折角。
- 拓扑守恒：在封闭流形条件下，细分保持网格连通结构并稳定细化。
- 极限曲面：反复迭代后，控制网格趋向光滑极限（非常规点处连续阶略低，但仍平滑可用）。

## R07

单次细分复杂度（`V` 顶点、`E` 边、`F` 面）：

- 建拓扑：`O(E + F)`
- 面点计算：`O(F)`
- 边点计算：`O(E)`
- 顶点更新：`O(V + 邻接访问)`，总体可视作 `O(V + E)`
- 重建新面：每旧面生成 4 面，`O(F)`

总时间复杂度：`O(V + E + F)`（单轮）

空间复杂度：`O(V + E + F)`（新点与邻接结构）

## R08

数值与建模稳定性要点：

- 拓扑前提：Catmull-Clark 的标准行为假设网格接近流形；封闭四边形网格最稳定。
- 非流形/孔洞：若边关联面数异常，会改变边点规则，可能导致局部伪影。
- 迭代次数：迭代越多越平滑，但也会带来体积收缩（shrinkage）。
- 浮点误差：仅做线性加权平均，误差通常温和，但深迭代仍会累积。

本 MVP 的主实验用封闭立方体网格，恰好满足“每条边邻接两面”的稳定条件。

## R09

参数经验：

- 迭代次数 `k`：
  - `k=1`：明显软化棱角
  - `k=2~3`：通常足够展示细分趋势
  - 更高 `k`：几何继续平滑，但点数快速增长
- 输入网格质量：初始网格过度拉伸会把细分误差放大到后续层。
- 若需要保体积，实践中常配合体积补偿或特征边约束（本 MVP 不包含）。

## R10

常见失败模式：

- 面不是四边形：本 MVP 直接报错（简化实现范围）。
- 网格非流形：某些边可能关联超过 2 个面，几何含义不再标准。
- 开边界网格：会触发边界回退规则，视觉结果与封闭网格不同。
- 迭代层数过高：`V/E/F` 爆炸增长，内存和计算成本显著上升。

## R11

本目录 MVP 设计（`demo.py`）：

- 算法：从零实现 Catmull-Clark，不调用图形库黑盒细分函数。
- 数据：使用封闭四边形立方体网格作为输入。
- 过程：执行 2 次细分。
- 输出：
  - 每轮 `V/E/F` 与 Euler 特征 `V-E+F`
  - 边长统计（min/mean/max/std）
  - 顶点到重心半径统计（min/mean/max/std）
  - 计数公式校验 `V' = V+E+F`、`F' = 4F`

这套输出和 R13 的解释、R14 的对比是一致的。

## R13

结果解释（对应 R11 输出项）：

- `Euler(V-E+F)`：封闭立方体拓扑应保持 `2`，用于检查重连是否破坏拓扑。
- `EdgeLen std`：边长标准差下降通常意味着网格更均匀、更平滑。
- `Radius std`：顶点半径分布更集中，表示形状从尖角向平滑体演化。
- `Predicted counts match`：若为 `True`，说明实现和 Catmull-Clark 计数规律一致。

## R14

与相近方法对比（与 R11/R13 同一观察口径）：

- 对比一次性线性细分（只加点不重定位）：
  - 线性细分只增加分辨率，不会显著消除尖角。
  - Catmull-Clark 会重定位旧顶点，平滑效果更明显（可通过 `EdgeLen/Radius` 统计看出）。
- 对比 Loop：
  - Loop 面向三角网；Catmull-Clark 面向四边形网。
  - 在 DCC/CAD 工作流中，四边形拓扑常更利于编辑和贴图。
- 对比 B-spline/NURBS：
  - NURBS 是参数曲面显式表示；细分曲面是控制网格迭代极限表示。
  - 细分曲面更容易处理复杂拓扑（洞、把手、多连通结构）。

## R15

工程实现注意事项：

- 需要稳定的拓扑索引：`edge -> faces`、`vertex -> faces/edges` 是核心。
- 点合并与去重策略要明确，否则会出现裂缝或重复边。
- 若接入渲染管线，通常要同时输出法线更新与 UV 细分规则。
- 大规模模型建议分块、并行或 GPU 化处理。

## R16

复现实验与稳定运行建议（与 R08 对齐）：

1. 优先使用封闭四边形流形网格进行基线验证（本 MVP 就是立方体）。
2. 先跑 `k=1` 检查拓扑统计，再升到 `k=2` 观察平滑趋势。
3. 若换成开边界网格，预期会触发边界回退规则，结果不应直接和封闭网格对比。
4. 若输入存在非流形边，先修复拓扑再细分，避免异常平均导致伪影。

这和 R08 的稳定性前提完全一致：封闭流形最稳，边界与非流形需额外处理。

## R17

建议测试清单：

1. 基础可运行性：`python3 demo.py` 能直接完成并输出 3 轮统计。
2. 公式一致性：每轮 `Predicted counts match` 应为 `True`。
3. 拓扑一致性：每轮 Euler 特征应维持常数（立方体应为 `2`）。
4. 平滑趋势：迭代后 `EdgeLen std`、`Radius std` 相比原始网格有下降趋势。
5. 边界分支回归：替换成开网格输入时程序仍可运行且无崩溃。

## R18

`demo.py` 源码级算法流程（8 步，非黑盒）：

1. `build_cube_mesh()` 构造初始封闭四边形立方体网格（顶点+面索引）。
2. `build_topology()` 建立 `edge_to_index`、`edge_faces`、`vertex_faces`、`vertex_edges` 四类邻接结构。
3. `catmull_clark_subdivide()` 先遍历每个面，计算面点 `face_points`（顶点均值）。
4. 同函数内遍历每条边，按邻接面数计算边点 `edge_points`（主路径为内部边四点平均）。
5. 再遍历旧顶点，用 `v' = (F_avg + 2R_avg + (n-3)v)/n` 更新内部顶点；边界顶点走回退分支。
6. 把三类点拼接成新顶点数组，并按局部角规则把每旧四边形拆成 4 个新四边形。
7. `main()` 对网格执行两轮细分；每轮输出 `V/E/F`、Euler、边长统计、半径统计，并校验 `V' = V+E+F`、`F' = 4F`。
8. 最后打印第 2 轮前 8 个顶点坐标样本，形成“拓扑正确 + 几何平滑”的最小验证闭环。

R18 的每一步都能在代码中对应到具体函数或代码段，并可由 R17 的测试项逐条验证。
