## R01

**算法目标**：对一个可比较序列进行升序排序。  
**本题算法**：快速排序（Quick Sort），属于典型分治算法。  
**适用场景**：
- 需要通用排序，且平均性能优先；
- 希望原地排序，减少额外内存；
- 数据规模中到大，且可接受极端情况下退化风险（可通过随机化主元缓解）。

## R02

**输入**：长度为 `n` 的整数列表 `A`（可扩展到任意支持 `<` 比较的类型）。  
**输出**：按非降序排列的新列表（本实现对输入做拷贝，不修改调用方原始列表）。  
**边界约定**：
- `n = 0` 或 `n = 1` 时直接返回；
- 允许重复元素、负数、已排序/逆序数据。

## R03

快速排序核心是“分而治之”：
1. 选一个主元（pivot）。
2. 通过一次线性扫描，把数组分成两段：左侧都 `< pivot`，右侧都 `>= pivot`。
3. 主元放到最终位置后，左右子区间独立递归排序。

当子问题足够小（区间长度 `<= 1`）时递归终止。

## R04

递归不变式（对任意调用 `quicksort_inplace(a, l, r)`）：
- 只重排区间 `a[l:r+1]`；
- 函数返回时，`a[l:r+1]` 有序；
- 区间外元素保持原位置不变。

分区不变式（Lomuto 分区思想）：
- 扫描指针 `i` 左侧维护 `< pivot` 区域；
- `i..j-1` 为“尚未确认归属”的扫描过程；
- 扫描结束后把 pivot 交换到边界位置，得到其全局正确排名。

## R05

本实现采用“**随机主元 + Lomuto 分区**”：
- 随机主元降低在已排序/逆序输入下反复劣化的概率；
- Lomuto 分区实现简洁，便于教学与验证；
- 通过“先递归较小区间、较大区间用循环处理”减少最大递归深度风险。

## R06

伪代码（简化版）：

```text
quick_sort(A):
    B = copy(A)
    sort(B, 0, len(B)-1)
    return B

sort(B, l, r):
    while l < r:
        p = random(l, r)
        m = partition(B, l, r, p)
        if (m - l) < (r - m):
            sort(B, l, m-1)
            l = m + 1
        else:
            sort(B, m+1, r)
            r = m - 1

partition(B, l, r, p):
    swap(B[p], B[r])
    pivot = B[r]
    i = l
    for j in [l..r-1]:
        if B[j] < pivot:
            swap(B[i], B[j]); i++
    swap(B[i], B[r])
    return i
```

## R07

正确性要点（简述）：
- `partition` 返回位置 `m` 后，满足：
  - `B[l:m] < B[m]`
  - `B[m+1:r+1] >= B[m]`
  - `B[m]` 已处于最终位置，不再移动。
- 对左右区间递归排序后，两段分别有序且都满足与 `B[m]` 的大小关系。
- 由归纳法可得 `B[l:r+1]` 全局有序，最终整个数组有序。

## R08

时间复杂度：
- 最好：`O(n log n)`（每次划分较均匀）
- 平均：`O(n log n)`（随机主元下常见）
- 最坏：`O(n^2)`（每次都划分极不均匀）

空间复杂度：
- 原地分区额外空间 `O(1)`；
- 递归栈平均 `O(log n)`，最坏 `O(n)`。

## R09

退化场景：
- 输入接近有序且主元总落在端点；
- 大量重复元素时，简单二路分区效果下降。

缓解策略：
- 随机主元（本实现已采用）；
- 三数取中（median-of-three）；
- 三路快排（<, =, >）提升重复元素场景表现；
- 小区间切换插入排序。

## R10

性质总结：
- **稳定性**：不稳定（相等元素相对顺序可能改变）；
- **是否原地**：是（对工作数组原地交换）；
- **是否比较排序**：是；
- **是否通用**：支持任意可比较对象（本 demo 使用整数）。

## R11

与常见排序对比（工程视角）：
- 对比归并排序：快排平均更快、缓存友好；归并稳定且最坏 `O(n log n)` 更稳。
- 对比堆排序：快排平均常数更小；堆排序最坏有保证但局部性较差。
- 对比 Python `sorted(Timsort)`：Timsort 稳定且对“部分有序”数据很强，生产中通常优先标准库。

## R12

实现注意点：
- 分区条件用 `< pivot`（非 `<=`），避免某些重复值场景下不必要交换模式；
- 递归基条件必须是 `l >= r`；
- 为避免调用方副作用，API 返回拷贝后的结果；
- 随机数种子固定可复现实验。

## R18

`demo.py` 中快速排序的源码级流程可分为 8 步：

1. `main()` 先固定随机种子，构造演示样例并调用 `quicksort(sample)`。
2. `quicksort()` 复制输入列表到 `arr`，避免修改调用方数据；若长度小于 2 直接返回。
3. 进入 `_quicksort_inplace(arr, 0, len(arr)-1)`，开始处理全区间。
4. 在每轮循环中随机选择 `pivot_idx`，调用 `_partition(...)` 执行一次分区。
5. `_partition` 先把主元换到区间末尾，然后线性扫描：
   - 维护 `store_index` 指向“下一个 `< pivot` 的放置位”；
   - 遇到 `< pivot` 的元素就与 `store_index` 交换并前移。
6. 扫描结束后把主元换回 `store_index`，返回该下标 `mid`；此时主元排名已确定。
7. `_quicksort_inplace` 比较左右子区间长度，优先递归较小区间，较大区间通过更新 `left/right` 在 `while` 循环继续处理，降低最大栈深。
8. 当所有区间都缩小到长度 `<= 1` 时排序完成，`main()` 再用 `sorted` 与 `numpy.sort` 进行一致性检查并输出耗时对比。
