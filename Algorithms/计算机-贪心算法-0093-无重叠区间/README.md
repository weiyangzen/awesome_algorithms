# 无重叠区间

- UID: `CS-0073`
- 学科: `计算机`
- 分类: `贪心算法`
- 源序号: `93`
- 目标目录: `Algorithms/计算机-贪心算法-0093-无重叠区间`

## R01

无重叠区间（Non-overlapping Intervals）要求：

- 输入若干区间 `intervals = [[s1,e1], [s2,e2], ...]`；
- 每次可以删除一个区间；
- 目标是让剩余区间两两不重叠；
- 求最少需要删除多少个区间。

该题本质是经典区间调度问题的等价变形：
`最少删除数 = 总区间数 - 最多可保留的不重叠区间数`。

## R02

该问题在工程中对应“冲突任务清理”场景，例如：

- 会议时段冲突消解（删除最少会议，保留最多可执行会议）；
- 批处理窗口冲突裁剪；
- 广告/作业时间片冲突过滤；
- 数据清洗中的冲突记录最小删除。

它常与“活动选择问题”“会议室安排”一起出现，是贪心算法的高频基础题。

## R03

本题采用的重叠判定约定：

- 若两个区间为 `[a,b]` 和 `[c,d]`；
- 当 `c < b` 时视为重叠；
- 当 `c >= b` 时视为不重叠（端点相接允许共存）。

因此像 `[1,2]` 与 `[2,3]` 在本题语义下是兼容的，不需要删除。

## R04

贪心策略（按右端点排序）：

1. 先按区间右端点 `end` 升序排序（若相同再按 `start`）；
2. 维护当前已选择集合最后一个区间的结束位置 `last_end`；
3. 扫描每个区间 `[s,e]`：
   - 若 `s >= last_end`，可保留该区间，更新 `last_end = e`；
   - 否则与已选区间冲突，删除该区间计数 `+1`。

右端点优先能给后续留出最大“可用空间”，是该问题的关键局部最优选择。

## R05

复杂度分析：

- 排序开销：`O(n log n)`；
- 线性扫描：`O(n)`；
- 总时间复杂度：`O(n log n)`。

空间复杂度：

- 若使用 `sorted` 生成新列表，额外 `O(n)`；
- 若原地排序（语言支持），可降到排序器相关的辅助空间。

## R06

## R07

方案优点：

- 实现短小，鲁棒性强；
- 复杂度达到区间问题常用最优量级；
- 可扩展输出“保留区间集合”用于下游调度。

局限：

- 当前实现是一维区间；
- 不处理带权收益（若每个区间价值不同，应改为加权区间调度 DP）；
- 不支持在线动态更新（高频增删需更复杂数据结构）。

前置知识：

## R09

适用场景：

- 目标是“最少删除冲突区间”；
- 区间语义明确且可比较端点；
- 离线批处理（一次性求解）。

不适用场景：

- 区间收益不等且要最大总收益；
- 需要实时在线更新；
- 多维几何冲突或复杂拓扑约束。

## R10

正确性要点（交换论证直觉）：

- 在所有可作为“当前选择”的候选区间中，结束最早的区间不会比结束更晚的区间更差；
- 若某最优解当前选了结束更晚的区间，可用结束更早的区间替换而不减少后续可行空间；
- 反复应用该替换，得到与贪心首选一致的最优解结构；
- 因此按右端点升序逐步选择，能得到最多可保留区间数，从而最小删除数也最优。

## R11

边界处理：

- 空输入：删除数为 `0`；
- 单区间：删除数为 `0`；
- 非法区间（`start > end`）：抛出 `ValueError`；
- 输入项长度不为 2：抛出 `ValueError`；
- 输出统一为整型删除数，并可选返回保留区间列表用于调试。

## R12

实现细节选择：

- 排序键为 `(end, start)`，优先最早结束；
- 判定条件使用 `s >= last_end`，与题目“端点相接不重叠”语义一致；
- 同时提供：
  - `erase_overlap_intervals` 返回最少删除数；
  - `select_max_non_overlapping` 返回保留区间集合（便于解释和验证）。

## R13

理论关系：

- 设总数为 `n`，可保留最大不重叠数为 `k`，则答案为 `n-k`；
- 贪心算法直接求得最大 `k`；
- 因而最少删除数也被直接确定。

这将“删除最小化”转成“保留最大化”，是本题建模关键。

## R14

常见错误：

1. 按 `start` 排序后直接删除，容易做出非最优决策；
2. 把条件写成 `s > last_end`，会错误地把端点相接当作冲突；
3. 冲突时删除“当前已选区间”而不比较结束时间，导致后续空间被压缩；
4. 忽略输入校验，出现反向区间导致结果不可解释。

## R17

`demo.py` 功能清单：

- `_normalize_and_validate`：输入校验与标准化；
- `select_max_non_overlapping`：按右端点贪心选择最大不重叠子集；
- `erase_overlap_intervals`：返回最少删除数；
- `max_non_overlap_bruteforce`：小规模穷举最优，用于校验贪心正确性；
- `run_fixed_cases` / `run_numpy_case`：执行固定样例与随机样例。

## R18

`demo.py` 的源码级算法流程（7 步）：

1. `main` 先执行固定样例，再执行 `numpy` 随机样例，并统一做断言校验。  
2. `_normalize_and_validate` 将输入转为 `List[Tuple[int,int]]`，并检查区间格式合法。  
3. `select_max_non_overlapping` 把区间按 `(end, start)` 升序排序，初始化 `last_end = None`。  
4. 线性扫描排序后的区间：若 `start >= last_end` 则保留并更新 `last_end`；否则跳过该区间。  
5. `erase_overlap_intervals` 用 `n - len(kept)` 计算最少删除数。  
6. `max_non_overlap_bruteforce` 对小规模样例穷举所有子集，求理论最优保留数，作为对照基准。  
7. 断言贪心删除数等于穷举最优删除数，确认实现与最优解一致。

本 MVP 没有调用第三方“黑盒求解器”；`numpy` 仅用于样例构造，核心贪心过程完全在源码中可追踪。
