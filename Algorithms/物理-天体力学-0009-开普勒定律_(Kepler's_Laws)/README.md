## R01

开普勒三定律描述了在中心引力主导下行星轨道的核心规律：

1. 第一定律（椭圆轨道定律）：天体沿椭圆轨道运行，中心天体位于一个焦点；
2. 第二定律（面积定律）：行星与太阳的连线在相等时间扫过相等面积；
3. 第三定律（调和定律）：同一中心天体下，不同行星满足 `T^2/a^3 = 常数`。

本条目通过一个最小可运行脚本把三条定律转为可计算、可断言、可回归的数值验证流程。

## R02

MVP 的目标不是构建完整天体力学库，而是打通以下闭环：

- 用二维二体模型数值积分一条椭圆轨道；
- 对同一轨道同时验证第 1、2 定律；
- 对多条不同半长轴轨道验证第 3 定律；
- 输出表格和指标，并以断言形式给出“通过/失败”。

## R03

建模假设：

- 采用经典牛顿引力与惯性系；
- 只考虑二体相对运动，不含摄动项（J2、第三体、阻力等）；
- 轨道限制为椭圆 `0 <= e < 1`；
- 在无量纲参数下演示，`mu = G(m1+m2)`。

该设定足以覆盖开普勒三定律的核心数学结构，同时保持实现紧凑可审查。

## R04

核心公式：

1. 二体相对运动方程

`r'' = -mu * r / |r|^3`

2. 椭圆近日点初值

`r_p = a(1-e)`

`v_p = sqrt(mu(1+e)/(a(1-e)))`

3. 第一定律对应焦点极坐标方程

`r(theta) = p / (1 + e cos(theta - omega))`

其中 `p = h^2/mu`，`h = x*vy - y*vx`。

4. 第二定律面积速度常量

`dA/dt = h/2`

离散近似：

`A_i ~= 0.5 * |x_i y_{i+1} - y_i x_{i+1}|`，`(dA/dt)_i ~= A_i / dt_i`

5. 第三定律

`T^2 / a^3 = 4*pi^2 / mu`

## R05

算法流程（高层）：

1. 由 `(a,e,mu)` 构造近日点初值；
2. 用 `solve_ivp(DOP853)` 积分一周期轨道；
3. 计算 `r, h, e_vec, energy, theta` 等不变量与几何量；
4. 用 `r(theta)=p/(1+e cos(theta-omega))` 重建半径并计算 `R^2/MAE`（第一定律）；
5. 用相邻采样点三角形面积计算面积速度稳定性（第二定律）；
6. 对多个 `a` 重复积分，并通过事件检测估计周期 `T`；
7. 统计 `T^2/a^3` 的误差与离散度（第三定律）；
8. 输出表格并执行阈值断言。

## R06

复杂度分析（设单轨道采样点数为 `n`，第三定律测试轨道条数为 `k`）：

- 单轨道积分与诊断：`O(n)`；
- 面积速度和椭圆拟合指标：`O(n)`；
- 第三定律多轨道周期估计：约 `O(k * n_event)`；
- 总体空间复杂度：`O(n)`（保留一条轨道状态与诊断量）。

在 MVP 场景中 `k` 很小（默认 5 条轨道），整体计算代价低。

## R07

正确性直觉：

- 中心力系统角动量守恒，直接对应第二定律中的面积速度常量；
- 逆平方中心力下轨道为圆锥曲线，椭圆是有界解，对应第一定律；
- 同一 `mu` 下的轨道族周期满足比例律，对应第三定律；
- 因此只要数值积分精度足够，三条定律可由守恒量与几何关系联合验证。

## R08

数值稳定策略：

- 使用高阶 `DOP853` 与严格容差 `rtol=1e-10, atol=1e-12`；
- 轨道初值限定在椭圆区间，避免非法参数；
- 对 `r` 过小情形加防护，避免 `1/r^3` 奇异发散；
- 第三定律周期估计使用事件根定位，避免粗网格采样造成的周期偏差。

## R09

`demo.py` 的最小工具栈：

- `numpy`：向量化计算轨道状态与守恒量；
- `scipy`：`solve_ivp` 做 ODE 积分与事件检测；
- `pandas`：构建轨道快照与第三定律对照表；
- `scikit-learn`：`r2_score`、`mean_absolute_error` 评估拟合与比例律误差；
- `torch`：统计面积速度和第三定律常数的张量跨度（与 numpy 交叉诊断）。

## R11

关键输出指标：

- `conic_radius_r2`、`conic_radius_mae`：第一定律几何一致性；
- `rel_energy_drift`、`max_abs_ecc_drift`：轨道积分守恒质量；
- `max_rel_area_rate_dev`、`area_rate_cv`：第二定律面积速度稳定性；
- `third_law_max_rel_error`、`third_law_rel_spread`：第三定律比例误差；
- `third_law_r2`、`third_law_mae_t2`：`T^2` 与 `a^3` 线性关系质量。

## R14

常见失效模式：

1. 参数非法：`e>=1` 却按椭圆初值构造；
2. 数值容差过松，导致能量与面积速度漂移过大；
3. 事件检测窗口过短，无法捕获下一次同相位过零点；
4. 仅看轨迹图像不看守恒量，可能掩盖积分误差；
5. 把受摄动轨道误当作严格开普勒轨道。

## R18

`demo.py` 的源码级算法流程（8 步）：

1. `OrbitConfig` 定义质量、轨道参数和采样规模，并提供 `mu`、理论周期属性。  
2. `make_periapsis_initial_state` 按椭圆解析公式生成 `r_p/v_p`，得到积分初值。  
3. `integrate_sampled_orbit` 调用 `solve_ivp(DOP853)` 积分一周期，得到离散状态序列。  
4. `compute_invariants` 显式计算 `radius/energy/h/e_vec/true_anomaly`，为三定律验证提供基础量。  
5. `evaluate_first_law` 用 `h,e_vec` 反推 `p,omega`，构造 `conic_radius_model` 并计算 `R^2/MAE`。  
6. `evaluate_second_law` 用相邻状态叉积近似扫面积，计算面积速度的相对偏差与变异系数。  
7. `evaluate_third_law` 对多组半长轴调用 `estimate_period_by_event` 得到 `T`，再统计 `T^2/a^3` 常量误差与离散度。  
8. `main` 汇总三类指标与表格并断言阈值，保证脚本可一键执行并给出确定性通过/失败结论。  

第三方库未被当作黑盒算法：`solve_ivp` 只承担通用微分方程数值积分，开普勒三定律对应的物理量定义、几何重建、比例验证和验收逻辑都在源码中逐步展开。
