## R01

多边形并/交/差是平面集合运算：

- 并集：`A ∪ B`
- 交集：`A ∩ B`
- 差集：`A \ B`

输入通常是简单多边形（可凸可凹，顶点按边界顺序给出）。输出一般仍是一个或多个多边形环（可能含孔洞）。这类运算是 GIS、CAD、机器人路径规划、地图叠加分析中的基础能力。

## R02

多边形布尔运算的发展路径大致是：

- 早期基于裁剪思想（Sutherland-Hodgman，适合凸裁剪）。
- 通用简单多边形算法（Weiler-Atherton、Greiner-Hormann）。
- 工程级扫描线方案（Vatti、Martinez-Rueda），成为 GEOS/JTS/Clipper 一类库的核心基础。

本目录 MVP 不依赖外部几何引擎，采用“栅格化 + 掩码布尔运算 + 轮廓追踪”的小实现，重点是可运行、可解释、可验证。

## R03

问题定义（本 MVP 版本）：

- 输入：两个二维简单多边形 `A, B`（`Nx2`、`Mx2` 顶点数组）。
- 输出：三种结果 `A ∪ B`、`A ∩ B`、`A \ B` 的近似区域与轮廓。
- 约束：不调用黑盒多边形布尔库（如 shapely/GEOS）。

难点：

- 一般凹多边形布尔运算需要处理边交点插入、拓扑重连、孔洞与退化情况。
- 从零写精确拓扑算法实现复杂度高，调试成本大。

因此这里选择“离散近似 MVP”：在统一网格上完成布尔运算，保证脚本稳定可跑并给出一致性校验。

## R04

`demo.py` 的算法主线：

1. 规范化输入多边形（去重复闭合点、统一 CCW）。
2. 根据两多边形联合包围盒构造规则网格。
3. 对每个网格中心做点在多边形内测试（ray casting），得到 `mask_A`、`mask_B`。
4. 直接做位运算：
   - `union = mask_A | mask_B`
   - `intersection = mask_A & mask_B`
   - `difference = mask_A & (~mask_B)`
5. 将结果掩码转边界：提取占用像素外轮廓边段并连接成闭环。
6. 将网格索引坐标映射回世界坐标，并做共线点简化。
7. 统计每个结果的组件数量、掩码面积、最大组件面积。
8. 执行面积恒等式检查，确保布尔逻辑正确。

## R05

设网格尺寸为 `H x W`，多边形边数分别为 `n, m`。

- 点内判定复杂度：`O(HW(n+m))`
- 掩码布尔运算：`O(HW)`
- 边界提取与追踪：`O(HW)`（常数与边界长度相关）
- 总体：`O(HW(n+m))`
- 空间复杂度：`O(HW)`

说明：这是离散近似法，复杂度受分辨率主导，而非仅受顶点数主导。

## R06

示例（`demo.py` 内置）：

- Case 1：凹多边形 vs 四边形。
- Case 2：星形凹多边形 vs 三角形。

脚本会输出：

- `union/intersection/difference` 各自的组件数量；
- 掩码面积近似值；
- 最大连通组件面积；
- 两条离散恒等式断言：
  - `|A ∪ B| = |A| + |B| - |A ∩ B|`
  - `|A| = |A \ B| + |A ∩ B|`

## R07

优点：

- 实现短小、依赖轻（`numpy + pandas`）。
- 可处理凹多边形，不需要手写复杂拓扑重连。
- 布尔结果在统一网格上严格自洽，便于自动校验。

局限：

- 几何结果是近似（精度受网格分辨率影响）。
- 细小特征可能被采样丢失或产生锯齿边界。
- 与精确计算几何库相比，无法提供严格拓扑保证。

## R08

前置知识：

- 射线法点在多边形内判定（奇偶规则）。
- 二值掩码布尔逻辑（位运算）。
- 多边形面积公式（shoelace）。
- 基础数值稳定性处理（容差 `EPS`、边界判定）。

## R09

适用场景：

- 需要快速验证多边形布尔逻辑原型。
- 几何可视化预处理、教学演示、离散域分析。
- 对“拓扑趋势正确”要求高于“坐标精确到机器精度”的任务。

不适用场景：

- CAD/GIS 生产级精确叠加。
- 对孔洞、切触、共线重合边有严格法律/工程精度要求。
- 后续流程依赖精确顶点级布尔结果。

## R10

正确性（在离散网格域内）要点：

1. 每个网格单元中心都被确定归类为 `A`、`B` 内外。
2. 掩码布尔运算遵循集合代数定义，逐单元逻辑等价。
3. 因此在离散域内，`union/intersection/difference` 的成员关系是严格正确的。
4. 面积恒等式在“单元计数”层面应严格成立，代码通过断言检查。
5. 轮廓追踪只是把离散集合边界显式化，不改变集合本身。

结论：该实现对“离散化后的集合运算”是正确的；连续几何误差来自采样分辨率而不是布尔逻辑错误。

## R11

数值与鲁棒性处理：

- `normalize_polygon` 处理重复收尾点与顺逆时针不一致。
- 点在边上时做边界包容（`on_edge` 判定），减少缝隙噪声。
- 点内测试分母加 `EPS`，避免近水平边除零不稳定。
- 轮廓提取后去共线顶点，减少栅格造成的冗余点。

## R12

关键参数与工程取舍：

- `resolution`：网格主分辨率，越大越精细但越慢。
- `pad_ratio`：包围盒外扩比例，避免边界贴格导致裁剪误差。
- `EPS`：边界判断容差。

经验：

- 原型阶段常用 `256~512` 分辨率。
- 若多边形细节尺度很小，应增大分辨率并做多组对比。

## R13

理论保证（本 MVP）：

- 近似比：无统一常数近似比承诺（误差依赖形状边界复杂度与网格步长）。
- 随机化成功概率：N/A（算法是确定性的）。

误差直觉：当网格步长减小时，面积与边界误差通常随步长收敛；但本实现未给出严格误差上界证明。

## R14

常见失效模式：

1. 分辨率过低导致细长结构断裂或消失。  
2. 多边形自交（非简单多边形）导致点内判定语义不稳定。  
3. 极端贴边输入引发局部锯齿或小孤岛。  
4. 像素角点接触导致轮廓追踪链断开。  

缓解策略：

- 提高 `resolution`，并做多尺度结果比对。
- 输入前做简单多边形校验与清洗。
- 对组件面积设置阈值，过滤微小噪声环。
- 若要生产级鲁棒性，切换到精确几何库/扫描线算法。

## R15

工程实践建议：

- 将流程拆成独立函数：归一化、栅格化、布尔、轮廓、统计。
- 始终保留离散恒等式断言，作为回归测试基线。
- 输出组件数量与面积，而不只输出一张图，便于 CI 自动判断。
- 若后续要升级为精确算法，可复用当前测试用例与统计框架。

## R16

相关精确算法（进阶方向）：

- Sutherland-Hodgman：凸裁剪高效，但不直接覆盖一般并/差。
- Weiler-Atherton / Greiner-Hormann：面向一般简单多边形布尔。
- Vatti / Martinez-Rueda 扫描线：工程中常见的通用方案。
- GEOS/JTS/Clipper：工业实现，覆盖大量退化处理。

本目录 MVP 适合作为“无外部几何引擎”的入门实现；若项目需要精确拓扑，建议迁移到上述算法/库。

## R18

源码级流程拆解（对应 `demo.py`，8 步）：

1. `normalize_polygon`：清洗输入顶点，删除重复收尾点并统一到 CCW。  
2. `make_grid`：根据 `A,B` 联合包围盒构造规则网格，确定 `dx,dy,width,height`。  
3. `grid_centers + points_in_polygon`：对所有网格中心执行射线法，分别得到 `mask_A`、`mask_B`。  
4. `boolean_masks`：逐位计算 `union/intersection/difference` 三张结果掩码。  
5. `mask_to_loops`：从结果掩码提取边界单元边段，并按有向边追踪成闭合环。  
6. `loops_to_world + simplify_loop`：将网格索引环映射回世界坐标并去除共线冗余点。  
7. `run_demo_case`：统计组件数量、掩码面积、最大组件面积，并做集合恒等式断言。  
8. `main`：运行两组预置多边形案例并汇总打印，形成可复现的最小演示。

该实现没有把第三方布尔库当黑盒；核心步骤（点内判定、掩码运算、轮廓追踪、一致性验证）都在源码内可逐行追踪。
