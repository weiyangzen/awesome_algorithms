# 范围搜索算法

- UID: `MATH-0194`
- 学科: `数学`
- 分类: `计算几何`
- 源序号: `194`
- 目标目录: `Algorithms/数学-计算几何-0194-范围搜索算法`

## R01

算法目标: 在二维点集上，给定一个轴对齐矩形 `[xmin, xmax] x [ymin, ymax]`，快速返回所有落在矩形内的点。  
本条目采用 `k-d tree`（二维）实现范围搜索，兼顾实现简洁性与查询效率。

## R02

问题形式化:
- 输入: 点集 `P = {p_i | p_i in R^2}`，查询矩形 `Q = [l_x, r_x] x [l_y, r_y]`
- 输出: `R = {p_i in P | l_x <= p_i.x <= r_x 且 l_y <= p_i.y <= r_y}`
- 支持多次查询时，先构建空间索引（k-d tree），再执行递归剪枝查询。

## R03

核心思想:
- 构建阶段: 递归地按坐标轴交替切分（x, y, x, y...），每层取中位点作为节点。
- 查询阶段: 若当前节点点在矩形内则收集；再依据矩形与切分平面的关系决定是否访问左右子树。
- 通过“与切分超平面无交集就剪枝”减少不必要遍历。

## R04

数据结构设计:
- `KDNode(point, index, axis, left, right)`
- `point`: 节点对应二维坐标
- `index`: 原始点在数组中的索引，便于结果回溯
- `axis`: 当前层切分轴（`0` 表示 x，`1` 表示 y）
- `left/right`: 子树引用

## R05

构建流程（递归）:
1. 若当前子数组为空，返回空节点。
2. 取 `axis = depth % 2`。
3. 按该轴排序并取中位索引作为根。
4. 左半部分递归构建左子树，右半部分递归构建右子树。
5. 返回根节点。

## R06

查询流程（递归）:
1. 访问当前节点，判断点是否落在查询矩形中，若是则加入答案。
2. 若 `rect_min[axis] <= node.point[axis]`，左子树可能有解，递归左子树。
3. 若 `rect_max[axis] >= node.point[axis]`，右子树可能有解，递归右子树。
4. 递归终止于空节点。

## R07

复杂度分析:
- 构建: 平均 `O(n log n)`（实现里每层排序，常数略大）
- 单次查询: 平均 `O(log n + k)`，`k` 为输出点数
- 最坏查询: `O(n)`（例如树严重退化或查询覆盖全空间）
- 空间复杂度: `O(n)`

## R08

正确性要点:
- 完备性: 任何可能包含答案的子树都不会被错误剪枝（由轴向区间相交条件保证）。
- 无冗余性: 仅当点满足二维区间约束时加入结果。
- 等价校验: `demo.py` 使用暴力扫描结果进行 100 次随机查询逐次比对，确保实现正确。

## R09

MVP 范围:
- 支持二维静态点集的轴对齐矩形范围查询。
- 不包含动态插入/删除、非轴对齐多边形查询或高维泛化接口。
- 以“可读 + 可验证 + 可直接运行”为目标。

## R10

实现文件说明:
- `demo.py`
- `build_kdtree(...)`: 构建索引树
- `range_search(...)`: 在索引树上执行剪枝搜索
- `brute_force_range(...)`: 线性扫描基线
- `main()`: 数据生成、正确性验证、性能对比输出

预期行为:

- 输出“100 次随机查询正确性校验: 通过”

## R12

样例输出解读:
- `固定查询命中数`: 单个矩形查询结果规模
- `总命中`: 所有测试查询命中数累计（用于确认两种方法结果一致）
- 若两种方法累计命中不同，程序会在断言处失败并抛出异常

## R13

与暴力扫描对比:
- 暴力法: 每次查询都扫描全部点，复杂度 `O(n)`
- k-d tree: 通过空间划分和剪枝，在典型分布下显著减少访问节点数
- 当查询窗口很大或数据分布极端时，k-d tree 优势会减弱

## R14

工程注意事项:
- 切分轴排序应稳定（代码中使用 `mergesort`），便于结果可复现。
- 边界定义采用闭区间（`<=` 与 `>=`），避免漏掉边界点。
- 结果统一排序后再比较，避免递归访问顺序影响一致性断言。

## R15

测试策略:
- 固定随机种子，保证可复现实验。
- 固定矩形 + 随机矩形双路径验证。
- 使用断言确保 k-d tree 与暴力扫描结果严格相等（索引集合一致）。

## R16

已知局限:
- 当前实现仅二维；扩展到 `d` 维仅需把 `axis = depth % d`，但性能与维度相关。
- 构建函数每层排序，简单直观但不是最优工程实现（可改进为线性选择中位数）。
- Python 递归深度在极端退化输入下可能受限。

## R17

可扩展方向:
- 动态更新: 采用平衡重建或批量重建策略
- 查询增强: 圆形范围、k 近邻、区间计数
- 工程优化: 向量化批查询、并行查询、持久化索引
- 替代结构: R-tree、Range tree、Quadtree（不同数据分布下更合适）

## R18

源码级算法流（`demo.py`，非黑盒拆解）:
1. `main()` 生成二维随机点 `points`，并创建 `indices = [0..n-1]`。
2. 调用 `build_kdtree(points, indices)` 进入递归构建；每层根据 `depth % 2` 选择切分轴。
3. 在当前轴上对候选索引排序，选中位索引作为当前节点，形成近似平衡树。
4. 对中位点左侧索引递归构建左子树、右侧索引递归构建右子树，直到子集为空。
5. 查询时调用 `one_query()`，其内部创建结果列表并调用 `range_search(root, rect_min, rect_max, hits)`。
6. `range_search` 先判断当前节点点是否在矩形内；若在则记录其原始索引。
7. 根据当前节点切分轴执行剪枝:  
   - 若 `rect_min[axis] <= point[axis]`，递归左子树；  
   - 若 `rect_max[axis] >= point[axis]`，递归右子树。
8. `main()` 用 `brute_force_range` 对同一查询做线性扫描比对；100 次随机查询全部断言通过后，再输出性能统计。
