## R01

Frank-Wolfe 算法（又称 Conditional Gradient）用于求解带凸约束集合 `D` 的光滑凸优化：

`min_{x in D} f(x)`，其中 `f` 可微且凸，`D` 为紧致凸集。

它的核心思想不是每步做欧氏投影，而是每轮解一个线性子问题（LMO, Linear Minimization Oracle）：

`s_k = argmin_{s in D} <grad f(x_k), s>`

然后沿着 `x_k -> s_k` 方向做凸组合更新。该特性在“投影很贵、线性优化很便宜”的约束集合（如单纯形、核范数球、迹范数球）上非常实用。

## R02

历史脉络：
- 1956 年由 Marguerite Frank 与 Philip Wolfe 提出，最初面向二次规划。
- 后续在机器学习与大规模优化中被重新重视，典型应用包括结构化稀疏学习、矩阵分解、低秩近似、交通分配等。
- 现代变体包括 Away-step FW、Pairwise FW、Fully-corrective FW 等，用于改善标准 FW 的边界锯齿和收敛常数。

本条目实现标准版 Frank-Wolfe，突出“最小可运行 + 可解释”。

## R03

要解决的核心问题：
1. 目标函数可微凸，但可行域 `D` 的投影 `Proj_D` 可能代价高或实现复杂。
2. 直接用投影梯度法时，每步需要投影；在某些几何集合上这是瓶颈。
3. 我们希望每轮只做一次梯度和一次线性最优化。

Frank-Wolfe 将难点从“投影”替换为“线性最小化 oracle”，在许多结构化集合上可以闭式或极快求解。

## R04

标准 Frank-Wolfe 迭代：
1. 给定 `x_k in D`，计算梯度 `g_k = grad f(x_k)`。
2. 线性子问题：`s_k = argmin_{s in D} <g_k, s>`。
3. 方向：`d_k = s_k - x_k`。
4. 步长：
   - 经典规则：`gamma_k = 2/(k+2)`；
   - 或线搜索：`gamma_k = argmin_{gamma in [0,1]} f(x_k + gamma d_k)`。
5. 更新：`x_{k+1} = x_k + gamma_k d_k`。
6. 终止：常用 Frank-Wolfe gap `gap_k = <x_k - s_k, g_k>` 小于阈值。

该 gap 同时是最优性证书（对凸问题为上界量）。

## R05

设变量维度为 `n`，二次目标 `f(x)=0.5 x^T Q x + c^T x` 且 `Q` 稠密：
- 梯度计算 `Qx + c`：`O(n^2)`。
- 在单纯形上的 LMO：取梯度最小分量对应基向量，`O(n)`。
- 线搜索（本实现为二次闭式）：`O(n^2)`（主要是 `Qd`）。

单次迭代总体约 `O(n^2)`，空间 `O(n^2)`（存储 `Q`）。

## R06

本 MVP 的演示问题：

`min 0.5 x^T Q x + c^T x`

`s.t. x >= 0, sum_i x_i = 1`（概率单纯形约束）

演示配置：
- 随机生成对称正定 `Q` 与向量 `c`；
- 以均匀分布 `x0 = (1/n,...,1/n)` 启动；
- 使用 Frank-Wolfe + 精确线搜索；
- 用 `scipy.optimize.minimize(SLSQP)` 得到参考最优值进行对照。

## R07

优点：
- 无需投影，迭代结构简单。
- 每次更新是凸组合，天然保持可行性。
- 可利用可行域结构（单纯形、原子集合）得到稀疏迭代点。

局限：
- 标准 FW 的理论速率通常为 `O(1/k)`，相对一阶加速法偏慢。
- 在最优点位于边界且曲率不利时，可能出现锯齿式前进。
- 对条件数较差问题，实际收敛常数可能大。

前置知识：

- 线性代数：向量内积、二次型、正定矩阵。

## R09

适用场景：
- 约束集合上线性优化易解、投影难解的问题。
- 需要保持迭代点稀疏或可解释（原子组合）的问题。
- 中大规模结构优化、在线/增量近似优化场景。

不太适用：
- 投影非常便宜而且需要更快收敛时（投影梯度/FISTA 可能更优）。
- 非光滑主导或强非凸问题（需改造或换算法）。

## R10

实现正确性的关键检查：
1. `x_k` 始终在可行域内（单纯形：非负且和为 1）。
2. LMO 的输出应是当前梯度最小分量对应的顶点。
3. 更新必须是 `x + gamma (s-x)` 且 `gamma in [0,1]`。
4. gap `=<x-s, grad>` 应非负（数值误差下允许极小负值）。
5. 目标值与 gap 随迭代整体下降（允许轻微非单调波动）。

## R11

数值稳定性要点：
- 线搜索分母 `d^T Q d` 可能非常小，需设 `eps` 保护。
- 步长裁剪到 `[0,1]`，避免越界更新。
- 判定可行性时对 `sum(x)`、`min(x)` 使用容差。
- 随机问题生成时给 `Q` 加 `mu I`，避免病态过强。

## R12

调参经验：
- `max_iter`：先给足够大（如 500~3000），再看 gap 截断。
- `tol_gap`：`1e-6` 到 `1e-8` 常见；越小迭代越多。
- 步长策略：
  - 若可做闭式线搜索，优先线搜索；
  - 若目标复杂，退回 `2/(k+2)` 也可保证收敛。
- 初始化：单纯形内任意点都可，均匀分布通常稳定。

## R13

理论保证（凸且梯度 Lipschitz）：
- 标准 FW 在曲率常数 `C_f` 下满足 `f(x_k)-f(x*) = O(C_f/k)`。
- Frank-Wolfe gap 可作为次优界：`f(x_k)-f(x*) <= gap_k`（在标准假设下成立）。
- 对强凸 + 多面体并配合 away-step 变体，可获得线性收敛（但标准版一般无此保证）。

## R14

常见失败模式与处理：
- gap 长时间不降：
  - 处理：检查 LMO 是否写错（索引方向常见反号错误）。
- 目标震荡明显：
  - 处理：启用线搜索或减小固定步长。
- 可行性被破坏：
  - 处理：检查更新公式和步长裁剪；必要时加轻量投影做诊断。
- 对照解差距过大：
  - 处理：增大迭代次数、收紧 `tol_gap`，并检查 `Q` 是否正定。

## R15

工程实践建议：
- 明确拆分三个模块：`objective/gradient`、`LMO`、`step_size`。
- 把 gap、目标值、步长都记录到 history，便于排障。
- 保留一个“高精度参考解”路径（例如 SLSQP）用于单元测试。
- 先在可解析结构（单纯形）上验证，再迁移到更复杂约束。

## R16

相关算法关系：
- 与投影梯度法：都是一阶法；FW 用 LMO 代替投影。
- 与 Mirror Descent：都可处理约束，但几何机制不同。
- 与坐标下降：都可产生稀疏更新，但选择规则不同。
- 常见扩展：Away-step FW、Pairwise FW、Stochastic FW。

## R18

`demo.py` 源码级流程拆解（8 步）：
1. `build_quadratic_problem` 生成正定矩阵 `Q`、线性项 `c` 与初始点 `x0`（均匀单纯形点）。
2. `quadratic_objective` / `quadratic_gradient` 提供目标与梯度计算。
3. `lmo_on_simplex` 根据当前梯度返回单纯形顶点 `e_i`（`i=argmin grad_i`）。
4. `exact_line_search_for_quadratic` 在方向 `d=s-x` 上求闭式最优步长并裁剪到 `[0,1]`。
5. `frank_wolfe_simplex_qp` 主循环：计算梯度、LMO、gap、步长，并更新 `x`。
6. 主循环每轮记录 `objective / gap / step_size`，并在 `gap < tol_gap` 时提前停止。
7. `solve_reference_with_slsqp` 用 SLSQP 解同一问题，作为参考最优值。
8. `main` 对比 FW 与参考解，打印结果并做断言（可行性、目标下降、精度达标）。

实现中没有把 Frank-Wolfe 交给第三方黑盒接口，关键步骤都在本地源码逐行可追踪。
