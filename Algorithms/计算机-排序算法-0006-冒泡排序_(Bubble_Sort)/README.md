## R01

冒泡排序（Bubble Sort）是一种经典的比较排序算法，核心思想是“相邻比较并交换”：

- 从左到右扫描数组，比较相邻元素；
- 若顺序错误（前者大于后者）则交换；
- 一轮扫描后，当前未排序区间中的最大元素会“冒泡”到末尾。

它实现简单、便于教学和调试，但在大规模数据上性能较弱。

## R02

问题定义（升序）：

- 输入：可比较序列 `A = [a1, a2, ..., an]`；
- 输出：升序序列 `B`，满足：
  - `B` 是 `A` 的重排；
  - `B[0] <= B[1] <= ... <= B[n-1]`。

本条目 MVP 目标：

- 提供源码级可追踪的冒泡排序实现（不调用第三方排序黑箱）；
- 输出关键运行统计（比较次数、交换次数、轮数、是否提前结束）；
- 自动校验结果与 `sorted()` 一致，并验证稳定性。

## R03

冒泡排序的价值主要在于：

- 算法机制直观，适合讲解“比较排序”与循环不变式；
- 原地排序，额外空间开销低（`O(1)`）；
- 通过“仅在 `>` 时交换”可天然保持稳定性；
- 可作为更复杂排序优化策略（例如提前终止、边界收缩）的入门案例。

## R04

核心机制（升序、带提前终止）：

1. 外层第 `i` 轮只处理前 `n-1-i` 个相邻对；
2. 内层比较 `arr[j]` 与 `arr[j+1]`；
3. 若 `arr[j] > arr[j+1]` 则交换；
4. 一轮结束后，位置 `n-1-i` 就位（该轮最大值）；
5. 若某一轮没有发生交换，说明数组已整体有序，可提前停止。

稳定性关键：

- 相等元素不交换（条件是 `>` 而不是 `>=`），相对顺序不被破坏。

## R05

高层流程：

1. 复制输入为工作数组 `arr`，避免修改原始数据；
2. 初始化统计器（比较、交换、轮数、提前结束标记）；
3. 进入外层轮次循环，逐轮缩小未排序区间；
4. 在每轮内做相邻比较与必要交换；
5. 若本轮无交换则提前结束；
6. 返回排序结果与统计信息。

## R06

正确性要点：

- 每轮内层扫描后，未排序区间最大元素会移动到该区间末端；
- 因此外层每完成一轮，就有一个元素在最终位置固定；
- 当外层轮次完成到 `n-1` 或提前终止时，整个数组有序；
- 全程仅交换逆序相邻对，且相等值不交换，因此算法稳定。

## R07

复杂度分析：

- 时间复杂度：
  - 最坏/平均：`O(n^2)`；
  - 最好（已排序且启用提前终止）：`O(n)`。
- 空间复杂度：
  - 额外空间 `O(1)`（原地交换，不依赖辅助数组）。

## R08

边界与异常策略：

- 空数组：直接返回空数组；
- 单元素：直接返回，无比较与交换；
- 重复值：正常处理，且可保持稳定；
- 含负数、浮点数：按可比较规则正常处理；
- 不可比较混合类型：抛出运行期异常，MVP 保持显式失败。

## R09

MVP 范围与取舍：

- 使用最小可审计实现，不引入复杂并行或向量化技巧；
- 引入“提前终止”作为唯一性能改进点，保持代码可读；
- 第三方库仅用于随机样本生成（`numpy`），不参与排序逻辑；
- 保持脚本一键运行，不需要交互输入。

## R14

关键参数与行为观察：

- `n`（输入规模）：比较和交换总量近似随 `n^2` 增长；
- 初始有序度：越接近有序，提前终止越容易触发；
- 重复值比例：重复值越多，交换次数可能下降；
- 随机种子：固定后可复现随机测试结果。

## R15

与常见排序算法对比：

- 对比选择排序：两者均 `O(n^2)`，但冒泡通常交换更多、稳定性更好；
- 对比插入排序：插入排序在“近乎有序”场景往往更高效；
- 对比归并/堆/快速排序：这些算法在中大规模下通常更实用（`O(n log n)` 主导）；
- 冒泡排序更适合作为教学与小规模验证工具，而非通用生产排序首选。

## R16

典型应用：

- 教学场景：演示相邻交换、稳定性与复杂度概念；
- 小规模数据快速原型验证；
- 对排序过程可视化（每轮“冒泡”效果非常直观）；
- 单元测试中作为“可读性高的基线实现”。

## R17

## R18

`demo.py` 源码级流程拆解（8 步）：

1. `main` 构造固定样例与 `numpy` 随机样例，逐个交给 `run_case`。  
2. `run_case` 调用 `bubble_sort(data)`，同时用 `sorted(data)` 做结果基准。  
3. `bubble_sort` 复制输入到 `arr`，初始化 `BubbleSortStats`。  
4. 进入外层轮次循环：第 `i` 轮仅扫描到 `n-1-i`，未排序区间逐轮缩小。  
5. 内层循环逐对比较 `arr[j]` 与 `arr[j+1]`，比较次数累加。  
6. 若发现逆序则交换并记录 `swaps`，本轮标记 `swapped=True`。  
7. 一轮结束后若 `swapped=False`，设置 `early_stop=True` 并 `break` 提前结束。  
8. 返回排序结果与统计；`main` 再执行记录型数据稳定性校验并输出汇总结论。

说明：排序核心完全由本地源码实现；第三方库仅用于随机样本生成，不构成算法黑箱。
