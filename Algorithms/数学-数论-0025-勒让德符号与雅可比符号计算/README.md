## R01

本条目讨论两个紧密相关的数论符号：

- 勒让德符号 `(a/p)`：其中 `p` 是奇素数，用于判断 `a` 在模 `p` 下是否是二次剩余。
- 雅可比符号 `(a/n)`：其中 `n` 是正奇整数（可合数），是勒让德符号向合数模数的代数推广。

判定规则：

- 勒让德符号：
  - `(a/p) = 0` 当 `p | a`
  - `(a/p) = 1` 当 `a` 是模 `p` 的非零二次剩余
  - `(a/p) = -1` 当 `a` 是模 `p` 的二次非剩余
- 雅可比符号：若 `n = p1^e1 * ... * pk^ek`，定义
  - `(a/n) = Π (a/pi)^ei`

MVP 的核心目标：

1. 实现 `legendre_symbol(a, p)`（基于 Euler 判别法）。
2. 实现 `jacobi_symbol(a, n)`（基于“提取 2 + 交换 + 二次互反律”的迭代法）。
3. 用可运行脚本展示“Jacobi=1 但不一定有平方根”的关键事实。

## R02

历史脉络：

- 1798 年：A.-M. Legendre 在《Essai sur la theorie des nombres》中系统使用勒让德符号，用于组织二次剩余理论。
- 19 世纪前半叶：二次互反律被高斯等人发展为数论核心定理。
- 1837 年前后：C. G. J. Jacobi 引入雅可比符号，使很多“只依赖符号演算”的流程可推广到奇合数模数。

因此这两个符号在时间上属于“先有勒让德，后有雅可比”的自然演进：从素数模判定走向合数模可计算框架。

## R03

提出背景与问题动机：

- 原始需求：快速判断同余方程 `x^2 ≡ a (mod m)` 是否可解。
- 素数模下可由勒让德符号高效完成判定。
- 合数模下直接判定更复杂，雅可比符号提供了一个可快速计算的“必要但非充分”信号。

工程语境中，雅可比符号常用于：

- 大整数算法中的预筛与分支剪枝。
- 概率素性测试（如 Solovay-Strassen）中的核心构件。
- 二次型/同余问题中的符号化推理。

## R04

时间复杂度（按本 MVP 实现）：

- `legendre_symbol(a, p)`：
  - 主操作是 `pow(a, (p - 1) // 2, p)`，复杂度约 `O(log p)` 次模乘。
- `jacobi_symbol(a, n)`：
  - 迭代次数与 `a,n` 缩减速度相关，算术层面可视作 `O(log n)` 轮。
  - 若按位复杂度估计（普通整数乘除），常写作 `~O((log n)^2)` 级别。
- `jacobi_from_factorization(a, n)`（仅用于交叉验证）：
  - 含试除分解，最坏 `O(sqrt(n))`，不适合超大 `n` 生产场景。

## R05

空间复杂度：

- 勒让德与雅可比主算法都为常数额外空间：`O(1)`（不计 Python 大整数对象本身）。
- 若做批量表格输出，额外空间由结果列表决定，约 `O(k)`（`k` 为样本数量）。

## R06

手算示例：计算 `(1001/9907)`（勒让德符号，`9907` 为奇素数）。

1. 先模化：`1001 mod 9907 = 1001`。
2. 用 Euler 判别法：
   - 计算 `1001^((9907 - 1) / 2) mod 9907 = 1001^4953 mod 9907`。
3. 若结果是 `1`，则符号为 `1`；若结果是 `9906`（即 `-1 mod 9907`），则符号为 `-1`。

再看雅可比反例：`(2/15)`

- `15 = 3 * 5`
- `(2/15) = (2/3)(2/5) = (-1) * (-1) = 1`
- 但 `x^2 mod 15` 取值集合为 `{0, 1, 4, 6, 9, 10}`，不含 `2`

结论：`(a/n) = 1` 对合数 `n` 并不保证存在平方根。

## R07

算法意义：

- 理论意义：把“二次剩余性”转换为可乘法组合、可递归化简的符号系统。
- 工程意义：在不显式求平方根的情况下快速筛选候选，节省大量分支成本。
- 教学意义：这是二次互反律在代码层面最直接、最经典的落地场景。

## R09

适用前提与边界：

- 勒让德符号要求 `p` 是奇素数。
- 雅可比符号要求 `n` 是正奇整数。
- `a` 可以是任意整数（实现中先做 `a %= n` 或 `a %= p`）。
- 若输入不满足定义域（如 `n` 偶数或非正），应抛出异常而非返回伪结果。

## R10

正确性依据（摘要）：

- 勒让德符号：依据 Euler 判别法
  - 对奇素数 `p` 与 `a`，`a^((p - 1) / 2) ≡ (a/p) (mod p)`。
- 雅可比符号迭代法：
  - 连续提取 `a` 中的 2 因子，并按 `n mod 8` 决定是否翻转符号。
  - 然后交换 `(a, n)`，并在 `a ≡ n ≡ 3 (mod 4)` 时翻转符号。
  - 交换后做 `a %= n`，问题规模下降，直到 `a = 0`。
  - 终止时 `n == 1` 返回累计符号，否则返回 `0`。

该流程等价于雅可比符号定义与二次互反律的组合应用。

## R11

数值稳定性与误差传播：

- 本算法是纯整数离散计算，不涉及浮点迭代，不存在传统舍入误差扩散问题。
- 主要风险来自实现错误：
  - 忘记先 `a %= n` 导致性能劣化或逻辑错误。
  - 对输入定义域检查不严导致“数学上无定义却返回数值”。
- Python 大整数可避免固定宽度溢出问题，稳定性较好。

## R12

真实成本分析：

- 单次勒让德计算通常很快，瓶颈主要在大模数下的模幂乘法。
- 雅可比计算通常比“先分解 n 再乘勒让德”明显更省，因为它不需要完整分解 `n`。
- 当要处理大量 `(a, n)` 批次时：
  - I/O 和字符串格式化有时比符号计算本身更慢。
  - 若同一 `n` 重复出现，可缓存部分中间统计（例如分类输出）。

## R14

鲁棒性与失效模式：

- 失效模式 1：把雅可比符号误当作“可解性充要条件”。
  - 典型误判：`(2/15) = 1` 但 `x^2 ≡ 2 (mod 15)` 无解。
- 失效模式 2：勒让德输入的 `p` 不是奇素数。
  - 这时 Euler 判别法不再保证语义正确。
- 失效模式 3：超大规模下仍使用试除分解做验证。
  - 会造成非必要性能灾难（MVP 中仅用于小规模 correctness check）。

## R15

工程实现要点：

- API 层严格输入校验：
  - `legendre_symbol(a, p)` 校验奇素数。
  - `jacobi_symbol(a, n)` 校验正奇数。
- 统一预处理 `a %= modulus`，避免负数与超大数影响迭代路径。
- 雅可比迭代中，符号翻转有两个独立触发器：
  - 提取 2 因子时看 `n mod 8 in {3, 5}`。
  - 交换后若 `a mod 4 = n mod 4 = 3` 再翻转。
- 终止条件要清晰：返回 `result if n == 1 else 0`。

## R18

源码级算法流程拆解（本 MVP 不依赖黑箱第三方包）：

1. 入口校验：检查 `p` 是否奇素数、`n` 是否正奇数；非法输入直接报错。  
2. 归一化：把 `a` 映射到模空间，执行 `a %= p` 或 `a %= n`。  
3. 勒让德求值：调用快速模幂计算 `pow(a, (p - 1) // 2, p)`，把结果 `1 / p - 1 / 0` 映射到 `1 / -1 / 0`。  
4. 雅可比提取 2：在 `a` 为偶数时不断除以 2，并根据 `n mod 8` 是否属于 `{3, 5}` 决定符号翻转。  
5. 变量交换：执行 `(a, n) = (n, a)`，把问题转换到更小规模；若二者同为 `3 (mod 4)` 则再次翻转符号。  
6. 模缩减：交换后做 `a %= n`，确保 `a` 单调缩小，循环最终可终止。  
7. 终止判定：若循环结束时 `n == 1` 返回累计符号；否则返回 `0`（代表有公共因子）。  
8. 自检闭环：用“分解乘积公式”与“暴力平方根存在性”做小规模对照，验证实现与数学语义一致。
