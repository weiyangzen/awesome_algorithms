## R01

凯撒密码（Caesar Cipher）是最经典的单表替换密码之一：把明文中的每个字母在字母表中按固定偏移量循环移动。

本条目目标：
- 给出一个可直接运行的 Python MVP；
- 同时覆盖“加密、解密、无密钥穷举尝试”三条最小闭环；
- 工具栈保持最小：仅使用 Python 标准库；
- 保留足够透明的实现，便于教学和代码审查。

## R02

历史背景：
- 该方法常被归因于古罗马时期凯撒用于军事通信；
- 本质是“固定偏移的替换规则”，属于古典密码体系；
- 在现代密码学中它已不具备安全性，但非常适合教学：
  - 可以直观看到密钥空间、穷举攻击、语言统计攻击这些核心概念。

## R03

问题定义（本 MVP 版本）：
- 输入：
  - 明文或密文字符串 `text`；
  - 整数偏移量 `shift`（可正可负）。
- 输出：
  - `caesar_encrypt(text, shift)` 返回密文；
  - `caesar_decrypt(text, shift)` 返回解密结果；
  - `brute_force_caesar(cipher)` 返回若干候选解密文本。

约束：
- 仅对英文字母 `A-Z` 与 `a-z` 做位移；
- 数字、空格、标点保持原样。

## R04

核心数学表达：

对字母表编号 `0..25`，记明文字母索引为 `x`，偏移为 `k`：
- 加密：`E_k(x) = (x + k) mod 26`
- 解密：`D_k(y) = (y - k) mod 26`

这里的 `mod 26` 使得位移具有环结构（`Z_26`），例如 `Z` 再右移 1 会回到 `A`。

## R05

实现中的核心数据结构：
- 字符串：承载输入输出文本；
- 常量频率表 `ENGLISH_FREQ`：用于对候选明文做英文分布打分；
- 候选列表 `List[Tuple[int, float, str]]`：每个元素记录
  - 假设密钥 `shift`；
  - 评分 `score`（越小越像英文）；
  - 对应解密文本。

## R06

示例：
- 明文：`Attack at dawn!`
- 偏移：`k = 3`
- 密文：`Dwwdfn dw gdzq!`

解释：
- `A -> D`，`t -> w`；
- 空格与 `!` 不变化；
- 解密时用 `-3` 即可恢复原文。

## R07

复杂度分析：
- 加密/解密：
  - 时间复杂度 `O(n)`，`n` 为文本长度；
  - 空间复杂度 `O(n)`（构造新字符串）。
- 穷举攻击（26 个移位全部尝试）：
  - 时间复杂度 `O(26n)`，可视作 `O(n)`（常数 26）；
  - 空间复杂度 `O(26n)`（保留候选文本时）。

## R08

正确性要点：
- 对每个字母都在同一个 26 字母环里做可逆映射；
- 解密函数等价于加密函数使用反向偏移；
- 非字母字符不参与替换，保证文本格式信息完整保留。

形式化上：`D_k(E_k(x)) = x`，因此在相同 `k` 下加密与解密互逆。

## R09

安全性评估：
- 密钥空间仅 26 种，暴力破解成本极低；
- 字母频率特征几乎完整保留，容易被统计攻击；
- 不提供现代机密性保障。

结论：
- 适合作为“密码学入门示例”；
- 不可用于真实安全场景。

## R10

`demo.py` 函数职责：
- `normalize_shift`：将任意整数偏移规约到 `0..25`；
- `shift_char`：对单字符执行位移；
- `caesar_encrypt / caesar_decrypt`：字符串级加解密；
- `chi_square_english_score`：按英文频率给文本打分；
- `brute_force_caesar`：遍历 26 个偏移并排序候选；
- `main`：执行演示并打印结果。

脚本无交互输入，运行后自动输出：

## R12

输出字段解读：
- `Plaintext`：原始明文；
- `Shift key`：用于加密的偏移量；
- `Ciphertext`：加密结果；
- `Recovered`：用同密钥解密后的文本；
- `Top brute-force candidates`：穷举出的候选按评分升序展示；
- `score`：卡方统计量，通常越小越接近英文分布。

## R13

最小测试建议：
- 回环测试：`decrypt(encrypt(text, k), k) == text`；
- 边界偏移：`k = 0, 25, 26, -1, 52`；
- 字符集测试：含大小写、数字、标点、空格；
- 破解测试：对英文句子加密后，检查穷举候选前几名是否包含正确明文。

## R14

边界与异常处理策略：
- 偏移可为任意整数，统一通过 `% 26` 归一化；
- 空字符串直接返回空字符串；
- 非字母字符不报错、不变换；
- 频率评分时若文本中无字母，返回 `inf` 代表“不可判别”。

## R15

参数与可调项：
- `shift`：主密钥参数；
- `top_k`：穷举输出候选数；
- `ENGLISH_FREQ`：语言分布先验（可替换为其它语言统计分布）。

如果要做教学扩展，可以把评分从单一频率改成：
- n-gram 语言模型；
- 词典匹配率；
- 混合打分。

## R16

## R18

源码级算法流（对应 `demo.py`，8 步）：
1. `main` 设定演示明文和偏移量 `shift`，先走“已知密钥加解密”主流程。  
2. `caesar_encrypt` 遍历字符串中的每个字符，并调用 `shift_char`。  
3. `shift_char` 先判定字符是否在 `A-Z/a-z`，若是则执行 `(index + shift) mod 26`；否则原样返回。  
4. `caesar_decrypt` 复用同一逻辑，但传入 `-shift`，得到解密文本并在 `main` 中校验回环正确性。  
5. `main` 再构造一个“未知密钥密文”场景，调用 `brute_force_caesar`。  
6. `brute_force_caesar` 对 `shift=0..25` 全部尝试解密，并对每个候选调用 `chi_square_english_score`。  
7. `chi_square_english_score` 统计候选文本字母频数，与 `ENGLISH_FREQ` 做卡方距离，得到可比较的语言相似度分数。  
8. `brute_force_caesar` 按 `score` 升序排序后返回 `top_k` 条候选，`main` 打印候选列表与最佳猜测结果。
