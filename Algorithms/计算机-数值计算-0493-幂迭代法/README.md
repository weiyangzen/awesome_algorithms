# 幂迭代法

- UID: `CS-0324`
- 学科: `计算机`
- 分类: `数值计算`
- 源序号: `493`
- 目标目录: `Algorithms/计算机-数值计算-0493-幂迭代法`

## R01

幂迭代法（Power Iteration）用于求矩阵的主特征对（dominant eigenpair），即绝对值最大的特征值 `lambda_max` 及其对应特征向量 `v_max`。  
它是最经典、最轻量的迭代特征值算法之一，常被当作更复杂方法（Lanczos、Arnoldi、PageRank 迭代）的基础模块。

## R02

问题定义：给定 `A in R^{n x n}`，希望找到

`A v = lambda v`, 且 `|lambda|` 在全谱中最大。

在很多任务里我们只关心“最重要模态”，不需要全谱分解，此时幂迭代比完整 `eig/eigh` 更省算力与内存。

## R03

核心思想：反复做“矩阵乘法 + 归一化”

`x_{k+1} = A x_k / ||A x_k||_2`

若 `x_0` 在主特征向量方向上的投影不为 0，迭代后向量方向会逐渐对齐到 `v_max`，再通过 Rayleigh 商估计特征值。

## R04

收敛直观（对可对角化矩阵）：

`x_0 = sum_i c_i v_i`, `A^k x_0 = sum_i c_i lambda_i^k v_i`

若 `|lambda_1| > |lambda_2| >= ...` 且 `c_1 != 0`，则随着 `k` 增大，`lambda_1^k` 项相对占优，归一化后方向趋向 `v_1`。

## R05

典型适用条件：
- 存在唯一主特征值（按绝对值严格最大）；
- 可以高效执行矩阵-向量乘（稠密或稀疏均可）；
- 仅需主特征对，不追求全特征分解。

对称正定矩阵场景最稳健，且主特征值为正、解释更直接。

## R06

特征值估计常用 Rayleigh 商：

`lambda_hat_k = (x_k^T A x_k) / (x_k^T x_k)`。

当 `x_k` 已接近真特征向量时，`lambda_hat_k` 会快速逼近真值；本 MVP 采用该估计并配合残差判停。

## R07

停止准则通常至少包含两类：
- 增量准则：`|lambda_hat_k - lambda_hat_{k-1}| < tol`；
- 残差准则：`||A x_k - lambda_hat_k x_k||_2 < residual_tol`。

只看增量可能误判“慢变化但未收敛”，只看残差又可能在早期迭代波动较大，因此二者联合更稳。

## R08

复杂度（`k` 为迭代次数）：
- 稠密矩阵：每轮 `O(n^2)`，总 `O(k n^2)`；
- 稀疏矩阵：每轮 `O(nnz(A))`，总 `O(k nnz(A))`；
- 额外内存：`O(n)`（当前向量、临时向量、历史统计）。

这就是幂迭代在大规模稀疏问题中仍然常见的原因。

## R09

收敛速度受谱间隙主导，近似按比率

`|lambda_2 / lambda_1|`

几何收敛。该比率越小，收敛越快；若接近 1，则会明显变慢。  
因此工程上常通过预处理、位移、块方法改善有效谱间隙。

## R10

数值稳定性要点：
- 每轮归一化，避免向量范数上溢/下溢；
- 使用 `float64` 降低舍入误差；
- 主特征向量符号本身不唯一（`v` 与 `-v` 等价），比较时应使用绝对夹角或 `|dot|`。

本条目 demo 对齐验证采用 `abs(dot(v_est, v_ref))`。

## R11

MVP 技术栈：
- `numpy`：矩阵构造、幂迭代主流程、参考特征分解；
- `pandas`：迭代历史表格化输出。

核心算法没有黑盒调用：主循环、收敛判据、残差评估均在源码显式实现，`numpy.linalg.eigh` 仅作结果校验。

## R12

运行方式（仓库根目录）：

```bash
uv run python Algorithms/计算机-数值计算-0493-幂迭代法/demo.py
```

脚本无交互输入，固定随机种子，直接输出参数摘要、迭代历史样本和验证结论。

## R13

输出字段说明：
- `iter`：迭代轮次；
- `lambda_est`：当前 Rayleigh 商估计；
- `delta_lambda`：相邻轮次特征值估计差；
- `residual_2norm`：残差范数 `||A x - lambda x||_2`。

最终还会打印 `lambda_ref`、`|lambda_est-lambda_ref|` 和向量对齐度 `|cos(theta)|`。

## R14

`demo.py` 内置验收阈值：
1. 必须在 `max_iters` 内触发收敛；
2. `|lambda_est - lambda_ref| < 1e-10`；
3. 终止残差 `< 1e-10`；
4. `|cos(theta)| > 1 - 1e-9`。

这些阈值针对本示例矩阵（有明显谱间隙）是可稳定满足的。

## R15

当前 MVP 边界：
- 仅求单个主特征对；
- 未实现 deflation（因此不能直接求前 `k` 个特征值）；
- 假设矩阵可在内存中存放；
- 对“主特征值非唯一”或“初始向量与主特征子空间正交”的病态情况不做专门修复。

## R16

可扩展方向：
- 加入 deflation，迭代求多个主特征对；
- 加入 shift-invert / 反幂迭代，求接近给定位移的特征值；
- 升级为块幂法（Block Power Method）提升多特征值场景效率；
- 针对稀疏矩阵改为算子接口（只暴露 matvec）。

## R17

常见错误与排查：
1. **忘记归一化**：向量可能数值溢出或全零化。  
2. **只看特征值不看残差**：可能出现“数值看似稳定但向量未收敛”。  
3. **把向量符号翻转误判为错误**：应比较 `|dot|` 而不是逐元素同号。  
4. **谱间隙太小还设过紧迭代上限**：会误报不收敛。  
5. **随机初值恰好退化**：可改随机种子或改用多初值重试。

## R18

`demo.py` 源码级算法流程（8 步）：
1. `PowerIterationConfig` 定义维度、随机种子、最大迭代、容差，并在 `validate()` 执行参数边界检查。  
2. `build_symmetric_matrix_with_known_spectrum` 通过 `Q * diag * Q^T` 构造可控谱的对称矩阵，确保主特征值唯一且已知。  
3. `power_iteration` 初始化随机单位向量 `x`，进入幂迭代主循环。  
4. 每轮计算 `y = A @ x` 并归一化得到下一轮方向，保证数值稳定。  
5. 在同轮内用 `rayleigh_quotient` 估计 `lambda`，并用 `residual_norm` 计算 `||A x - lambda x||_2`。  
6. 记录 `delta_lambda` 与残差到历史表，联合判据（增量 + 残差）决定是否停止。  
7. `main` 用 `numpy.linalg.eigh` 仅生成参考主特征对，计算特征值误差和向量夹角余弦。  
8. 将历史记录转为 `pandas` 表输出，并执行断言阈值，最终打印 `Validation: PASS`。
