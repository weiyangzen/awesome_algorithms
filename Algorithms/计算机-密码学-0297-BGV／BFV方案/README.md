## R01

BGV/BFV 都属于基于 RLWE（Ring Learning With Errors）的全同态加密（FHE）路线，核心思想是：

- 在多项式环里构造“带噪密文”；
- 解密时噪声可被控制或在模意义下消去；
- 密文可直接执行加法与乘法，从而在不解密的情况下完成计算。

本条目给出一个可运行的教学级 MVP，聚焦“环上代数流”与“同态正确性”，不引入工程级大框架。

## R02

BGV 与 BFV 的共同骨架：

- 工作在环 `R_q = Z_q[x]/(x^n + 1)`；
- 明文在 `R_t`（小模数）；
- 密文含随机项和误差项，安全性来自 RLWE 难题。

二者主要差异在“明文嵌入方式、乘法后缩放策略、模链管理细节”。本 MVP 采用两者共享的环上密文结构，并使用“噪声按 `t` 倍注入”的简化形式，便于直接验证同态加乘。

## R03

问题定义（本实现版本）：

- 输入：两个明文多项式 `m1, m2 in R_t`，长度 `n=8`；
- 输出：
  - `Enc(m)` 后可正确 `Dec`；
  - `Dec(Add(Enc(m1), Enc(m2))) = (m1 + m2) mod t`；
  - `Dec(Mul(Enc(m1), Enc(m2))) = (m1 * m2) mod (x^n+1, t)`。

其中 `Mul` 输出 3 分量密文（对应 `s^0, s^1, s^2`），是“未重线性化”的标准中间态。

## R04

本 MVP 使用的核心公式：

- 密钥生成：
  - 取小秘密 `s`、均匀 `a`、小误差 `e`；
  - 令 `b = -a*s + t*e (mod q)`；
  - 公钥 `pk=(a,b)`，私钥 `sk=s`。
- 加密：
  - 取小随机 `r,e1,e2`；
  - `c0 = b*r + t*e1 + m`；
  - `c1 = a*r + t*e2`。
- 解密：
  - 计算 `v = c0 + c1*s (+ c2*s^2 + ...) (mod q)`；
  - 输出 `v mod t`。

因为误差项都乘了 `t`，在 `mod t` 时会消失，便于教学演示正确性。

## R05

同态运算规则：

- 同态加法：密文按分量逐项相加；
- 同态乘法（fresh × fresh）：
  - `d0 = c00*c10`
  - `d1 = c00*c11 + c01*c10`
  - `d2 = c01*c11`

乘法后密文维度从 2 分量增长到 3 分量；工业 BFV/BGV 会在此后执行重线性化与模数链管理，本 MVP 故意保留该中间态，以便把代数路径看清楚。

## R06

复杂度（`n` 为环维度）：

- 单次多项式乘法（朴素卷积 + negacyclic 回折）：`O(n^2)`；
- 加密/解密各自包含常数次多项式乘法：总体 `O(n^2)`；
- 同态乘法包含 3 次核心乘法：仍是 `O(n^2)` 量级。

本实现刻意不使用 NTT，目的是把流程压到最小可读版本。

## R07

正确性直觉：

- 解密展开：
  - `c0 + c1*s = (b*r + t*e1 + m) + (a*r + t*e2)*s`
  - 代入 `b = -a*s + t*e` 后，`a*r*s` 项相消；
  - 得到 `m + t*(...)`；
- 对其取 `mod t`，误差项全部消失，只剩 `m`。

同态乘法时，`(m1 + t*η1)*(m2 + t*η2) = m1*m2 + t*(...)`，因此 `mod t` 后得到 `m1*m2`。

## R08

参数与安全说明：

- 演示参数：`n=8, t=17, q=17*4096=69632`；
- `q % t == 0` 是本教学实现的刻意选择，便于做“误差是 `t` 倍”的整洁推导；
- 这不是生产安全参数，不能用于真实数据保护。

真实 BGV/BFV 需要：更高维度、更严格噪声预算、RNS/NTT 优化、模数链与重线性化/密钥切换等完整机制。

## R09

MVP 设计取舍：

- 保留：RLWE 风格公私钥结构、环上密文、同态加乘主链路；
- 省略：重线性化密钥、模数切换链、批处理打包、NTT/RNS；
- 结果：代码量小、可运行、公式与实现逐行对应，适合入门和审阅。

## R11

`demo.py` 结构：

- `Params`：参数定义与合法性检查；
- `poly_mul_negacyclic`：`R_q` 中多项式乘法；
- `ToyBGVBFV._keygen / encrypt / decrypt`：核心密码流程；
- `add / mul`：同态运算；
- `run_demo`：固定样例、断言校验、结果输出。

所有核心算法均在源码显式实现，没有调用黑盒 FHE 库。

## R12

演示中的验证项：

1. `Dec(Enc(m1)) == m1`；
2. `Dec(Enc(m2)) == m2`；
3. `Dec(Add(ct1, ct2)) == (m1 + m2) mod t`；
4. `Dec(Mul(ct1, ct2)) == poly_mul_negacyclic(m1, m2, t)`。

每项都通过 `assert` 强制检查，失败会立即抛错。

## R13

边界与限制：

- 仅支持固定维度 `n=8`（可改但未做自动参数搜索）；
- `mul` 当前只支持 fresh(2分量)×fresh(2分量)；
- 乘法输出 3 分量密文，不支持继续深层电路评估；
- 该实现用于教学，不具备工程级安全与性能。

## R14

与相关方案关系：

- BGV：强调模数链与噪声管理，常见“模切 + 密钥切换”路径；
- BFV：同属 RLWE 全同态整数计算路线，工程实现常结合 RNS；
- CKKS：主打近似实数计算，语义与误差模型不同。

本条目可视作“BGV/BFV 共同代数骨架”的入门切片。

## R15

可扩展路线（从本 MVP 出发）：

1. 增加重线性化密钥，把 3 分量乘法密文压回 2 分量；
2. 引入模数链，支持多层乘法与噪声预算管理；
3. 接入 NTT/RNS，把 `O(n^2)` 乘法替换为高效实现；
4. 增加 batching（SIMD packing）能力。

## R16

常见实现错误：

- 把 `x^n + 1` 的回折写成 `+` 而不是 `-`（negacyclic 符号错）；
- 忘记所有系数都要在每步后 `mod q`；
- 明文域期望使用普通卷积而非 `mod (x^n+1)` 的环乘；
- 误把该教学参数当成安全参数。

## R18

`demo.py` 的源码级算法流程（8 步）：

1. `main -> run_demo` 初始化 `Params`，固定 `n,t,q` 与随机种子，保证结果可复现。  
2. `ToyBGVBFV.__init__` 调用 `_keygen` 生成 `sk=s` 与 `pk=(a,b)`，其中 `b = -a*s + t*e (mod q)`。  
3. `encrypt` 对每个明文多项式采样 `r,e1,e2`，构造 `c0 = b*r + t*e1 + m`、`c1 = a*r + t*e2`。  
4. `decrypt` 按密文分量次数累加 `c_i * s^i`（通过 `poly_mul_negacyclic`），最后取 `mod t` 还原明文。  
5. `add` 对两份密文逐分量相加，得到同态加法结果密文。  
6. `mul` 计算 `d0,d1,d2` 三项，得到 degree-2 乘法密文（未重线性化）。  
7. `run_demo` 对加法和乘法分别解密，并与明文域中的 `mod t` 期望结果逐项 `assert` 对比。  
8. 所有断言通过后打印结果与限制说明，明确该 MVP 未引入 NTT/RNS、重线性化、模数链等工程优化。

第三方库使用说明：`numpy` 仅用于数组与卷积计算，核心密码流程（密钥生成、加密、解密、同态加乘）均由源码显式实现，可逐行追踪。
