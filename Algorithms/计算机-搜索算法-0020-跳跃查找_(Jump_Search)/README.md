# 跳跃查找 (Jump Search)

- UID: `CS-0020`
- 学科: `计算机`
- 分类: `搜索算法`
- 源序号: `20`
- 目标目录: `Algorithms/计算机-搜索算法-0020-跳跃查找_(Jump_Search)`

## R01

跳跃查找（Jump Search）用于在**有序数组**中查找目标值。它不逐个元素线性扫描，而是按固定步长“跳块”前进，先定位候选区间，再在区间内顺序查找。

## R02

问题形式化（本目录 MVP）：
- 输入：非降序数组 `arr`（长度 `n`）与目标值 `target`
- 输出：若存在下标 `i` 使 `arr[i] == target`，返回某个合法下标；否则返回 `-1`
- 前提：元素支持比较操作，且数组已排序

## R03

算法适用边界：
- Jump Search 依赖“有序性”来决定跳跃和提前停止
- 在随机访问结构（如数组）上更适合；链表上跳跃优势不明显
- 常见步长选取为 `sqrt(n)`，是平均意义下平衡“跳跃次数”和“块内扫描长度”的经验最优级别

## R04

核心思路：
- 设步长 `step ≈ sqrt(n)`，每次查看一个块的末尾元素
- 若块末尾值仍小于目标，整块可排除，继续跳到下一块
- 当找到第一个“块末尾 >= target”的块后，只在该块内做线性查找

## R05

简化伪代码：

```text
if n == 0:
    return -1

step = floor(sqrt(n))
prev = 0

while prev < n and arr[min(prev + step, n) - 1] < target:
    prev += step
if prev >= n:
    return -1

for i from prev to min(prev + step, n) - 1:
    if arr[i] == target:
        return i
    if arr[i] > target:
        return -1

return -1
```

## R06

正确性要点（直观不变式）：
- 跳跃阶段每次判断 `arr[block_end] < target` 时，可安全排除该块全部元素
- 跳跃停止时，目标若存在，只可能落在当前候选块内
- 候选块内线性扫描是完备的；且利用有序性，当遇到 `arr[i] > target` 可立即判定不存在
- 因此算法不会漏报存在目标，也不会误报不存在目标为存在

## R07

复杂度：
- 时间复杂度：`O(sqrt(n))`（约 `n/step + step`，当 `step=sqrt(n)` 时最小）
- 空间复杂度：`O(1)`（仅常数级辅助变量）
- 对比：
  - 线性查找：`O(n)`
  - 二分查找：`O(log n)`（渐进更优，但实现与访存模式不同）

## R08

边界与异常情况：
- 空数组：直接返回 `-1`
- 单元素数组：一步即可完成
- `target` 小于首元素或大于末元素：可快速判定
- 重复值存在：本实现返回“任意一个命中下标”
- 输入无序：本实现显式抛出 `ValueError`，避免静默错误

## R09

示例：
- `arr=[1,3,5,7,9,11,13,15,17]`，`target=11`
  - 步长 `step=3`，看块尾 `arr[2]=5 < 11`，跳到下一块
  - 再看 `arr[5]=11`，锁定区间 `[3,5]`，线性扫描命中下标 `5`
- `arr=[2,4,6,8,10,12]`，`target=7`
  - 跳到候选块后线性扫描遇到 `8 > 7`，可立即返回 `-1`

## R12

典型应用：
- 中等规模有序数组上的低实现成本搜索
- 内存分层明显、希望减少逐元素比较次数的场景
- 教学场景中展示“分块 + 局部精查”的思维
- 作为更复杂索引结构前的入门级搜索策略

## R13

常见错误：
- 忽略有序性前提，直接在无序数组上执行
- 步长计算为 `0`（如 `n=0` 未特判）导致死循环
- 块边界写错（`min(prev + step, n) - 1`）导致越界
- 线性扫描上界错误，遗漏块尾元素
- 未做提前停止，导致不必要比较

## R14

常见变体：
- 自适应步长：根据数据分布动态调整跳跃步长
- 分层跳跃：先大步跳、后小步细化
- 与索引结构结合：先用稀疏索引定位块，再块内查找
- 返回第一个/最后一个命中位置：在命中后继续局部扫描

## R15

与其他搜索算法对比：
- 线性查找：无需有序性，但最坏 `O(n)`；Jump Search 在有序数组上更快
- 二分查找：`O(log n)`，理论更优；Jump Search 更直观，适合作为分块思想示例
- 指数查找：先指数探测区间再二分，适合上界不明确场景
- 插值查找：依赖分布近似均匀，最坏情况可能退化明显

## R16

工程实践建议：
- API 文档明确“输入必须非降序”
- 用统一返回约定（如未命中返回 `-1`）简化上层调用
- 对大批量查询可预先构建更高效索引（如二分友好结构）而非重复 Jump Search
- 保留基准对照（如 `bisect`）用于回归测试，防止边界回归

## R17

最小测试清单：
- 空数组、单元素命中/未命中
- 目标命中首段、中段、末段
- 目标不存在且位于缺口、超出范围
- 含重复值数组（验证返回下标合法且值正确）
- 无序输入（应抛 `ValueError`）

## R18

源码级算法流程拆解（对应本目录 `demo.py` 的 `jump_search`）：
1. 函数入口先调用 `_ensure_non_decreasing(arr)`，若检测到逆序对则抛出 `ValueError`，阻止错误前提下运行。
2. 处理空数组：`n == 0` 时直接返回 `-1`。
3. 计算步长 `step = max(1, isqrt(n))`，并初始化块起点 `prev = 0`。
4. 进入跳跃循环：计算当前块尾 `block_end = min(prev + step, n) - 1`，若 `arr[block_end] < target`，说明整块都不可能命中，执行 `prev += step` 跳到下一块。
5. 若 `prev >= n`，表示所有块都被排除，直接返回 `-1`。
6. 锁定候选块后，设置线性扫描右边界 `upper = min(prev + step, n)`，在 `[prev, upper)` 内逐个检查。
7. 扫描中若 `arr[i] == target`，立即返回下标 `i`；若 `arr[i] > target`，利用有序性提前返回 `-1`。
8. 候选块扫描完仍未命中，则返回 `-1`；`main()` 再用 `_expected_via_bisect` 校验“存在性”一致，验证实现正确性。
