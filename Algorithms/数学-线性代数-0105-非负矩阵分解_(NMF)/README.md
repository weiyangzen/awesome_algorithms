# 非负矩阵分解 (NMF)

- UID: `MATH-0105`
- 学科: `数学`
- 分类: `线性代数`
- 源序号: `105`
- 目标目录: `Algorithms/数学-线性代数-0105-非负矩阵分解_(NMF)`

## R01

非负矩阵分解（Non-negative Matrix Factorization, NMF）研究如何把非负矩阵 `V in R_{>=0}^{m*n}` 近似分解为两个低秩非负矩阵：

`V ~= W H`，其中 `W in R_{>=0}^{m*r}`, `H in R_{>=0}^{r*n}`, 且 `r << min(m,n)`。

- 目标: 在保持非负约束的同时，实现降维与可解释表示学习。
- 典型损失:
  - Frobenius 范数: `min ||V-WH||_F^2`
  - KL 散度: `min D_{KL}(V || WH)`
- 输出语义:
  - `W` 的列可理解为“基/部件/主题”。
  - `H` 的行可理解为每个基在样本中的激活强度。

## R02

- 早期思想可追溯到矩阵低秩近似与因子分析。
- 1999 年 Lee 与 Seung 提出经典的乘法更新规则，使 NMF 在图像与文本分解中迅速普及。
- 后续发展形成多条支线:
  - 稀疏 NMF（提升可解释性）
  - 带正则项的 NMF（L1/L2、图正则）
  - 在线/分布式 NMF（大规模数据）
  - 非负张量分解（高阶数据）

## R03

NMF 解决的是“既要降维，又希望表示可解释”的问题。

- 与 SVD/PCA 相比，NMF 强制系数非负，避免“正负抵消”导致的语义难解释。
- 在词频、像素强度、计数型生物数据中，非负约束符合物理意义。
- 实务价值:
  - 文本主题提取（词-主题、主题-文档）
  - 图像部件分解（眼睛/鼻子等局部基）
  - 推荐系统中的隐因子建模（带非负偏好）

## R04

以乘法更新（Frobenius 目标）为例，单次迭代主要成本为若干矩阵乘法:

- `W^T V` 与 `V H^T`: 约 `O(mnr)`
- `W^T W H` 与 `W H H^T`: 约 `O((m+n)r^2)`
- 单次迭代总复杂度: `O(mnr + (m+n)r^2)`，当 `r` 远小于 `m,n` 时主导项近似 `O(mnr)`。
- 若执行 `T` 次迭代，总体约 `O(T * (mnr + (m+n)r^2))`。

## R05

空间复杂度（稠密实现）:

- 输入矩阵 `V`: `O(mn)`
- 因子矩阵 `W,H`: `O(mr + rn)`
- 临时中间量（若不复用缓冲）: 约 `O(mr + rn + r^2)`
- 总体通常写作: `O(mn + mr + rn)`。

在高维文本场景，`V` 常为稀疏矩阵，实际实现可通过稀疏乘法显著降内存与算时。

## R06

示例（文本主题风格）:

设 `V` 为“词 x 文档”的非负计数矩阵，取 `r=2`:

- 主题 1（技术）在 `W` 对应列上可能对“算法/模型/训练”权重较高。
- 主题 2（运动）在 `W` 对应列上可能对“比赛/球员/进球”权重较高。
- 某篇技术文档在 `H` 中对主题 1 激活高、主题 2 低。

因此 NMF 把高维词频拆为“可读主题 + 每篇文档的主题混合强度”。

## R07

NMF 的关键价值是“部件可加性”:

- 结果可解释: 特征由若干非负基按权重叠加得到。
- 对业务沟通友好: 可以把每个隐因子映射成可理解概念。
- 对下游任务有效: 因子矩阵可作为聚类、分类、检索的低维输入。

它在“精确重构能力”上不一定优于无约束低秩分解，但在“解释性 + 非负先验一致性”上经常更优。

## R08

Frobenius 目标下的标准优化问题:

`min_{W,H>=0} 0.5 * ||V-WH||_F^2`

Lee-Seung 乘法更新（常见形式）:

- `H <- H * (W^T V) / (W^T W H + eps)`
- `W <- W * (V H^T) / (W H H^T + eps)`

其中 `*` 与 `/` 表示逐元素乘除，`eps` 防止除零。

性质与直觉:

- 若初始化 `W,H` 非负，更新后仍保持非负。
- 每步不需要显式学习率，形式简单。
- 目标函数通常单调不增到某个局部极值（整体问题非凸）。

## R09

适用条件与边界:

- `V` 必须是非负矩阵（可含零）。
- 分解秩 `r` 需满足 `1 <= r <= min(m,n)`。
- NMF 是非凸优化:
  - 结果依赖初始化。
  - 只保证收敛到局部最优或鞍点附近。
- 若数据包含大量负值，需先做平移/变换，或改用其他分解方法。

## R10

正确性关注点（工程可验证版本）:

1. 非负不变性: 初始化 `W,H>=0`，乘法更新后仍满足 `W,H>=0`。
2. 维度一致性: 始终保持 `W` 为 `m*r`，`H` 为 `r*n`，可重构 `WH` 与 `V` 同形。
3. 下降性观测: 记录损失序列，通常应呈下降或平台化。
4. 收敛停止: 采用相对改进阈值 `tol` 与最大迭代 `max_iter` 双保险。

## R11

数值稳定性要点:

- 除零风险: 分母加 `eps`（如 `1e-10`）避免 NaN/Inf。
- 零锁定（zero-locking）: 某些元素变为 0 后可能难以恢复；初始化常用正小值避免过早锁死。
- 缩放不确定性: `W D` 与 `D^{-1} H` 表示同一解，可通过列归一化提升可读性。
- 终止判据要稳健: 用相对改进而非绝对改进，可减少量纲影响。

## R12

真实成本（超越大 O）:

- 计算瓶颈是高频矩阵乘法；高效 BLAS 对性能影响极大。
- Python 层循环会拖慢速度，尽量向量化（本目录 MVP 即此策略）。
- 稀疏文本数据若转稠密会严重放大内存，工程中应选稀疏实现。
- NMF 常需要多次不同初始化重启以获得更好局部最优，总成本近似乘上重启次数。

## R13

误差与评估:

- 训练误差: `||V-WH||_F / ||V||_F`（相对重构误差）
- 解释性评估:
  - 每个基向量的高权重特征是否语义集中
  - 每个样本的激活是否稀疏且稳定
- 泛化/稳健性:
  - 改变随机种子，观察解的稳定程度
  - 对输入加小扰动，检查误差与主题漂移

NMF 通常提供“可解释近似”，而非唯一真解。

## R14

常见失效模式:

- `rank` 过小: 欠拟合，重构误差高。
- `rank` 过大: 过拟合与解释碎片化。
- 初始化差: 收敛慢或落入差局部最优。
- 输入含负值: 违反模型假设，需预处理。
- 早停过早: 尚未达到可用误差水平。

对应策略包括多次重启、秩扫描（网格搜索）、合理归一化与更长迭代预算。

## R15

工程实践建议:

- 数据预处理:
  - 保证非负
  - 可按列/行归一化，减少尺度失衡
- 参数设置:
  - `rank` 从小到大扫描
  - `max_iter` 给足（例如 500~3000）
  - `tol` 与业务误差容忍度联动
- 质量控制:
  - 记录损失曲线是否单调下降
  - 检查 `W,H` 是否出现大量无效零列/零行
  - 用重构误差 + 人工可解释性双指标决策

## R18

源码级流程拆解（对应 `demo.py` 的 `nmf_multiplicative_update`，8 步）:

1. 读取输入 `V`，检查其二维性、非负性与 `rank` 合法范围。
2. 用正随机数初始化 `W(m*r)` 与 `H(r*n)`，并记录初始损失。
3. 计算 `H` 的分子 `W^T V` 与分母 `W^T W H + eps`。
4. 执行逐元素更新 `H *= numerator/denominator`，保持非负。
5. 计算 `W` 的分子 `V H^T` 与分母 `W H H^T + eps`。
6. 执行逐元素更新 `W *= numerator/denominator`，随后做列归一化并把尺度吸收到 `H`。
7. 重新计算损失，比较相对改进 `(|old-new|/max(old,eps))`；若低于 `tol` 则提前停止。
8. 迭代至收敛或达到 `max_iter`，返回 `W,H`、损失历史和迭代信息；主程序再计算相对重构误差并做断言。
