## R01

预处理（Preconditioning）是把线性系统
`A x = b`
改写成“更容易迭代收敛”的等价系统。

常见形式：

- 左预处理：`M^{-1} A x = M^{-1} b`
- 右预处理：`A M^{-1} y = b, x = M^{-1} y`
- 对称预处理（SPD 常用）：`M^{-1/2} A M^{-1/2} y = M^{-1/2} b`

核心目标不是直接“精确解出”，而是让新系统的谱分布更有利（通常是降低条件数、聚集特征值），从而减少 Krylov 迭代次数。

## R02

历史定位（简要）：

- 20 世纪中后期，随着 CG、GMRES、BiCGSTAB 等 Krylov 方法成熟，预处理成为大规模线性求解的关键模块；
- 实务中常见预处理器包括 Jacobi、SSOR、ILU/IC、多重网格（MG/AMG）等；
- 在工程计算中，常说“迭代法是否好用，主要看预处理是否合适”。

## R03

本条目 MVP 聚焦问题：

- 任务：比较“无预处理 CG”与“Jacobi 预处理 CG（PCG）”在同一 SPD 问题上的收敛差异；
- 输入：程序内部构造的病态三对角 SPD 系统 `A x = b`；
- 输出：
  - 两种方法的迭代次数；
  - 相对残差与相对解误差；
  - `cond2(A)` 与 `cond2(D^{-1/2} A D^{-1/2})`（`D=diag(A)`）的对比。

## R04

时间复杂度：

- 本实现矩阵是三对角，`A @ v` 采用手写三对角 matvec，单次成本 `O(n)`；
- CG/PCG 每轮主要是一次 matvec + 若干向量内积/axpy，总体每轮 `O(n)`；
- 总成本约 `O(k n)`，`k` 为迭代步数；
- 诊断阶段计算条件数（密集 `cond`）为 `O(n^3)`，仅用于展示，不属于迭代核心路径。

## R05

空间复杂度：

- 三对角存储：`diag (n)` + `off (n-1)`，即 `O(n)`；
- 迭代向量 `x, r, z, p, Ap` 等为 `O(n)`；
- 残差历史为 `O(k)`；
- 若计入诊断中临时构造的密集矩阵，则会额外使用 `O(n^2)`（仅报告用途）。

## R06

2x2 直觉示例：

令
`A = [[1000, 0], [0, 1]]`，条件数 `kappa_2(A)=1000`。

- 无预处理时，两个方向尺度差异巨大；
- 取 Jacobi 预处理 `M = diag(A)`，则
  `M^{-1/2} A M^{-1/2} = I`；
- 条件数直接变成 1，迭代法会显著更快。

真实问题不会总是这么理想，但该例清晰说明了“尺度均衡化”的核心作用。

## R07

算法意义：

- 同样的 Krylov 框架，预处理往往决定了实际可用性；
- 好的预处理能把“不可接受的迭代次数”变成“工程可承受”；
- 预处理本质上是“用一个便宜近似 `M ≈ A` 换取更快的外层迭代收敛”。

## R08

PCG（左预处理写法）核心公式：

1. 初值 `x0`，`r0 = b - A x0`，`z0 = M^{-1} r0`，`p0 = z0`。
2. 对 `k = 0,1,2,...`：
   - `alpha_k = (r_k^T z_k) / (p_k^T A p_k)`
   - `x_{k+1} = x_k + alpha_k p_k`
   - `r_{k+1} = r_k - alpha_k A p_k`
   - 若 `||r_{k+1}||` 足够小则停止
   - `z_{k+1} = M^{-1} r_{k+1}`
   - `beta_k = (r_{k+1}^T z_{k+1}) / (r_k^T z_k)`
   - `p_{k+1} = z_{k+1} + beta_k p_k`

本 MVP 选择 `M = diag(A)`（Jacobi 预处理），`M^{-1}` 只需逐元素除法。

## R09

适用条件与局限：

适用：

- 系统为 SPD（CG/PCG 的标准假设）；
- 能高效实现 matvec；
- 需要中等到高精度迭代解。

局限：

- 预处理器并非越复杂越好，构造成本过高会抵消收益；
- Jacobi 很便宜，但对某些耦合强问题提升有限；
- 非 SPD 系统应改用 GMRES/BiCGSTAB + 对应预处理策略。

## R10

正确性要点（工程化表述）：

1. 在 `A` 与 `M` 均 SPD 时，PCG 在 `M^{-1}A` 诱导内积下保持共轭方向结构。  
2. 每一步都在 Krylov 子空间中推进并最小化相应误差。  
3. 有限精度下不保证精确 `n` 步结束，但残差应整体下降。  
4. 本实现用两类指标核验：
   - 迭代收敛（相对残差达到阈值）；
   - 相对解误差与条件数改善趋势。

## R11

数值稳定性与误差：

- 迭代终止采用相对残差 `||r_k|| / ||r_0|| <= tol`；
- 若出现 `p^T A p` 或 `r^T z` 退化到接近 0，则判定数值 breakdown 并停止；
- 病态系统中，无预处理即使达到残差阈值，解误差也可能不如预处理版本；
- Jacobi 属于尺度校正，不会破坏稀疏结构，稳定性与实现复杂度较友好。

## R12

性能视角：

- 迭代核心是 `tridiag_matvec`，完全向量化，单轮常数小；
- 预处理步骤 `z = D^{-1} r` 是逐元素乘法，代价几乎可以忽略；
- 当条件数显著下降时，迭代轮数减少可达数量级差异；
- 在大规模稀疏场景，这类“廉价预处理 + 快速 matvec”的组合非常常见。

## R13

本目录 MVP 的可验证保证：

- 无随机输入、无交互输入、结果可复现；
- 自动比较：
  - `CG(no preconditioner)`
  - `PCG(Jacobi)`
- 自动断言：
  - 两者都收敛；
  - Jacobi 迭代步数更少；
  - 预处理后条件数更低；
  - Jacobi 版本解误差达到高精度。

## R14

常见失效模式：

- `A` 非 SPD：CG/PCG 理论基础被破坏，可能发散或停滞；
- 预处理器不满足正定兼容：会导致 `r^T z <= 0`；
- 容忍阈值过严 + 最大迭代过小：表现为“未收敛”；
- 仅关注残差不关注解误差：在极病态问题上可能误判质量。

## R15

`demo.py` 结构设计：

- `build_ill_conditioned_tridiagonal_spd`：构造可控病态三对角 SPD 系统；
- `tridiag_matvec`：手写三对角乘法，避免黑盒稀疏求解器；
- `make_reference_solution`：生成确定性真解 `x_true`；
- `pcg_solve`：显式实现 PCG 主循环；
- `estimate_condition_numbers`：仅用于输出 `cond` 对比；
- `run_checks`：集中执行可验证断言；
- `main`：运行对照实验并打印报告。

## R16

相关算法链路：

- 预处理器：Jacobi、SSOR、ILU/IC、块预处理、AMG；
- 迭代器：CG（SPD）、MINRES（对称不定）、GMRES/BiCGSTAB（非对称）；
- 高阶实践：域分解 + 多层预处理 + 并行 Krylov；
- 现实工程常见模式：先用轻预处理快速试跑，再按瓶颈升级到 IC/AMG。

## R18

`demo.py` 的源码级流程（8 步，非黑盒）如下：

1. 在 `main` 中构造病态三对角 SPD 系统参数 `diag/off`，并生成真解 `x_true` 与右端 `b = A x_true`。  
2. 定义两种预处理算子：恒等算子（无预处理）与 `D^{-1}`（Jacobi）。  
3. `pcg_solve` 初始化 `x=0`、`r=b-Ax`、`z=M^{-1}r`、`p=z`，记录初始残差。  
4. 每轮先计算 `Ap`，再用 `alpha=(r^Tz)/(p^TAp)` 更新 `x` 与 `r`。  
5. 检查 `||r||/||r0||` 是否小于阈值；满足即判定收敛并退出。  
6. 未收敛时计算 `z_new=M^{-1}r`、`beta=(r_new^T z_new)/(r_old^T z_old)`，更新搜索方向 `p=z_new+beta*p`。  
7. 两组实验结束后，用 `estimate_condition_numbers` 计算 `cond(A)` 与 `cond(D^{-1/2}AD^{-1/2})`，量化预处理作用。  
8. `run_checks` 对“收敛性、迭代步数改善、条件数改善、解误差”做断言，最后打印报告。  

说明：这里没有调用任何现成 CG 黑盒求解器；PCG 的迭代与预处理应用均在源码中逐步展开实现。
