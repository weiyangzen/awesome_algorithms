## R01

最大子数组和（Maximum Subarray Sum）问题：
- 给定一个一维数值数组；
- 目标是在所有连续子数组中，找到元素和最大的那一段。

Kadane 算法是该问题的经典线性时间动态规划解法。
本目录提供一个最小可运行 MVP：输出最大和、区间下标和对应子数组，并通过暴力基线做一致性验证。

## R02

MVP 输入输出定义：
- 输入：`nums`，非空一维数值序列（`list/tuple/numpy.ndarray` 均可）。
- 输出：
  - `max_sum`：最大子数组和（`float`）；
  - `start_index` / `end_index`：最优连续区间闭区间下标；
  - `subarray`：该区间对应的子数组值。

约束与约定：
- 子数组必须非空；
- 当全部元素都为负数时，答案是“最大单个元素”；
- 若出现多个同和最优区间，MVP 采用稳定规则：优先起点更小，其次长度更短。

## R03

Kadane 状态定义：
- `best_ending_here[i]`：以 `i` 结尾的连续子数组最大和；
- `best_so_far`：遍历到当前位置时，所有结尾中的全局最大和。

实现时可空间压缩为常数变量：
- `current_sum` 对应当前 `best_ending_here`；
- `best_sum` 对应全局最优。

## R04

状态转移（线性扫描）：

对当前位置 `i` 的值 `x = nums[i]`：
- 若 `current_sum < 0`，则新段从 `i` 开始：`current_sum = x`；
- 否则继续累加：`current_sum = current_sum + x`。

随后比较 `current_sum` 与 `best_sum`：
- 若更优则更新全局最优区间；
- 若相等则按预设 tie-break 规则决定是否替换。

## R05

为什么这是动态规划：
- “以 `i` 结尾的最优子数组”只依赖“以 `i-1` 结尾的最优子数组”和当前元素；
- 对每个位置的最优值一旦确定，不会被未来状态反向影响（无后效性）；
- 全局最优可由局部最优逐步聚合得到。

因此可以在一次扫描中完成最优解求解。

## R06

复杂度分析：
- Kadane 主算法：
  - 时间复杂度 `O(n)`；
  - 空间复杂度 `O(1)`（不计输出子数组拷贝）。
- 暴力基线（前缀和 + 双层枚举）：
  - 时间复杂度 `O(n^2)`；
  - 空间复杂度 `O(n)`。

MVP 使用暴力法仅做交叉校验，不作为主求解路径。

## R07

边界与异常处理：
- 输入为空：抛 `ValueError`；
- 输入不是一维序列或包含非有限值（`NaN/Inf`）：抛 `ValueError`；
- 输入元素可为整数或浮点数，统一转为 `float64` 计算。

## R08

`demo.py` 模块结构：
- `MaxSubarrayResult`：封装算法输出；
- `to_1d_float_array`：输入校验并转换为 `numpy` 一维数组；
- `kadane_max_subarray`：`O(n)` 主算法；
- `bruteforce_max_subarray`：`O(n^2)` 基线算法；
- `run_case`：执行固定样例并断言主算法与基线一致；
- `randomized_cross_check`：随机数组对拍；
- `main`：组织固定样例、随机验证与汇总输出。

## R09

MVP 取舍：

## R13

随机对拍策略：
- 使用固定种子 `2026`，保证复现；
- 生成长度 `1..30` 的随机整数数组，元素范围 `[-20, 20]`；
- 默认执行 `300` 轮；
- 每轮断言：
  - Kadane 输出和暴力法输出完全一致（含区间 tie-break 一致性）。

若不一致，立即抛出 `AssertionError` 并打印失败样本。

## R14

正确性审计点：
- 扫描到每个位置时，`current_sum` 必须等于“以该位置结尾”的最优和；
- `best_sum` 必须是“遍历过所有位置中的最大值”；
- 区间更新遵循固定 tie-break，确保结果稳定可复现；
- 对所有固定样例与随机样例，Kadane 与暴力基线结果一致。

## R15

局限与工程注意：
- 当前实现仅处理一维数组，不直接支持二维最大子矩阵；
- tie-break 策略是实现约定，不同业务可能需要不同规则；
- 对极长序列的高吞吐场景，仍可进一步做 JIT/并行优化；
- 使用 `float64` 计算时，对极端大数会受浮点精度影响。

## R18

`demo.py` 源码级算法流程（8 步）：

1. `main` 定义 5 组固定样例，并逐个调用 `run_case`。  
2. `run_case` 先调用 `to_1d_float_array` 做输入校验与 `float64` 转换。  
3. `kadane_max_subarray` 初始化 `current_sum/best_sum` 与区间指针，起点设为第 0 位。  
4. 从第 1 位开始线性扫描：若 `current_sum < 0` 则“丢弃历史、从当前重开”，否则把当前值累加到 `current_sum`。  
5. 每一步将 `current_sum` 与 `best_sum` 比较，若更优或同和但 tie-break 更优，则更新全局最优和与区间。  
6. 扫描结束后，`kadane_max_subarray` 输出 `max_sum/start_index/end_index/subarray`。  
7. `run_case` 再调用 `bruteforce_max_subarray`（前缀和 + 双层枚举）求基线结果，并断言与 Kadane 完全一致。  
8. 固定样例完成后，`main` 调用 `randomized_cross_check` 进行 300 轮随机对拍，最后打印汇总和全局通过状态。
