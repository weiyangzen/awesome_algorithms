# 多普勒效应（光波） (Doppler Effect for Light)

- UID: `PHYS-0099`
- 学科: `物理`
- 分类: `光学`
- 源序号: `99`
- 目标目录: `Algorithms/物理-光学-0099-多普勒效应（光波）_(Doppler_Effect_for_Light)`

## R01

光波多普勒效应描述了光源与观测者沿视线方向的相对运动如何改变观测到的波长/频率。对速度不可忽略于光速 `c` 的情况，必须采用相对论形式，而不是经典声波近似。

本条目采用约定：`beta = v/c`，`beta>0` 表示光源远离观测者（红移），`beta<0` 表示光源接近观测者（蓝移）。

## R02

该效应是天体物理与精密测量中的基础工具：

- 通过光谱线红移反推星系或恒星视向速度；
- 在高速喷流、双星系统、系外行星径向速度法中做参数估计；
- 在高速度场景中校正经典近似误差，避免系统性偏差。

## R03

MVP 目标是建立一个最小、可运行且可断言的闭环：

- 正向：`beta -> (lambda_obs, f_obs, z)`；
- 反向：`z -> beta`；
- 评估：比较经典近似 `z≈beta` 在低速和高速下的误差表现；
- 应用：对带噪声光谱线测量执行速度反演并统计误差。

## R04

核心方程：

1. 相对论光学多普勒因子

`D(beta) = sqrt((1 + beta)/(1 - beta))`

2. 波长与频率变换

`lambda_obs = lambda_0 * D(beta)`

`f_obs = f_0 / D(beta)`

3. 红移定义

`1 + z = lambda_obs/lambda_0 = D(beta)`

4. 红移反演速度

`beta = ((1+z)^2 - 1) / ((1+z)^2 + 1)`

5. 低速近似（仅 `|beta| << 1`）

`z ≈ beta`

## R05

设样本数为 `n`：

- 正向计算 `D, lambda_obs, z`：`O(n)`；
- 反演 `z -> beta`：`O(n)`；
- 误差统计（MAE、max）：`O(n)`；
- 内存复杂度：`O(n)`（保留数组和结果表）。

整个 MVP 是线性复杂度，适合参数扫描和批量回归。

## R10

正确性直觉：

1. 当 `beta=0` 时，应有 `D=1`、`z=0`、`lambda_obs=lambda_0`；
2. `beta>0`（远离）应导致 `lambda_obs` 增大（红移）；
3. `beta<0`（接近）应导致 `lambda_obs` 减小（蓝移）；
4. 正反公式互为代数逆映射，组合后应恢复原始 `beta`；
5. 低速下泰勒展开给出 `z≈beta`，高速则二者偏差扩大。

## R11

数值与稳定性策略：

- 对 `beta` 强制检查 `|beta|<1`，避免物理域外输入；
- 对反演输入检查 `1+z>0`，避免非法红移值；
- 使用 `float64` 保证宽速度范围下的稳定精度；
- 所有 demo 采用确定性随机种子，保证复现实验结果。

## R14

常见失效模式：

1. 把声波多普勒公式直接用于光波高速场景；
2. 符号约定混乱（`beta` 的正负方向未固定）；
3. 忘记检查 `|beta|<1` 导致数值域错误；
4. 只用 `z≈beta` 做中高速反演，得到系统性低估/高估；
5. 把仪器偏置误当作随机噪声，导致误差评估过于乐观。

## R18

`demo.py` 源码级算法流程（8 步）：

1. `_as_beta_array` 将输入转为 `float64` 数组并强制 `|beta|<1`。  
2. `doppler_factor_from_beta` 计算 `D=sqrt((1+beta)/(1-beta))`，作为所有正向量的核心因子。  
3. `observed_wavelength` 与 `observed_frequency` 分别输出 `lambda_obs`、`f_obs`，并共享同一 `D`。  
4. `redshift_from_beta` 给出 `z=D-1`，`beta_from_redshift` 用闭式反演恢复 `beta`。  
5. `run_inverse_consistency_demo` 在宽速度网格上执行“正向再反向”，统计恢复误差并断言阈值。  
6. `run_classical_vs_relativistic_demo` 对比 `z_rel` 与 `z≈beta`，同时验证低速可近似与高速偏离。  
7. `run_spectral_line_velocity_estimation_demo` 构造含噪 H-alpha 测量数据，按 `z -> beta` 做速度估计并用 `mean_absolute_error` 评估。  
8. `main` 汇总打印三段结果、表格和指标，所有断言通过后输出 `All checks passed.`。  

第三方库未被当作黑盒算法：`numpy/pandas/scikit-learn` 仅用于数组、表格和误差统计；多普勒正反公式、符号约定、阈值验收逻辑均在源码中显式实现。
