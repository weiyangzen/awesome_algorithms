## R01

回归树（Regression Tree）是把输入空间递归切分成若干区域，并在每个叶节点输出常数预测值（通常是该叶样本均值）的监督学习方法。

本目录实现的是 `CART` 风格回归树：
- 二叉划分（每次只选一个特征、一个阈值）；
- 划分准则为平方误差下降（SSE 减少量最大）；
- 叶节点输出该叶样本 `y` 的均值。

## R02

MVP 问题定义：

给定训练集 `D={(x_i, y_i)}_{i=1}^n`，学习一个分段常数函数 `f(x)`，使预测误差尽量小。节点划分选择目标为：

`Gain = SSE(parent) - SSE(left) - SSE(right)`

其中

`SSE(S) = sum_{i in S} (y_i - mean(y_S))^2`

预测时从根节点按阈值向下走到叶子，输出叶均值作为回归结果。

## R03

为什么这个题目用手写实现而不是直接黑盒调用：
- 回归树核心就是“枚举切分点 + 误差下降比较”，代码结构清晰，适合算法教学；
- 能直接看到停止条件、叶子生成、阈值选取等关键机制；
- 与库实现对齐后，更容易理解随机森林、GBDT 等树模型家族。

`demo.py` 的训练核心是纯 `numpy` 手写，`scikit-learn` 仅作为可选对照检查。

## R04

数学要点（单次最优切分）：

对某个节点样本 `(X, y)`，若按特征 `j` 与阈值 `t` 划分为：
- `L = {i | X[i, j] <= t}`
- `R = {i | X[i, j] >  t}`

则该切分代价为：

`Cost(j,t) = SSE(L) + SSE(R)`

我们选择使 `Cost` 最小（等价于 `Gain` 最大）的 `(j,t)`。

实现上通过排序 + 前缀和计算，快速得到每个候选分割点的左右子集平方误差。

## R05

算法高层流程：

1. 检查输入维度、有限值、超参数合法性。
2. 在当前节点计算节点预测值（`y` 均值）和父节点 `SSE`。
3. 若满足停止条件（深度上限、样本过少、纯节点），直接建叶子。
4. 否则枚举全部特征，并在特征排序后扫描候选阈值。
5. 计算每个候选阈值的 `Gain`，选最优分裂。
6. 若最优增益低于 `min_impurity_decrease`，建叶子。
7. 否则按阈值划分左右子集，递归构建子树。
8. 预测时沿树路由到叶节点，输出叶均值。

## R06

正确性直觉：
- 每次分裂都在当前节点可行候选中选取 `SSE` 最小的二叉划分，因此对“该节点的一步分裂”是贪心最优；
- 叶子预测值取均值是最小化平方损失的最优常数；
- 递归重复该过程，得到分段常数近似函数。

注意：这是局部贪心最优，不保证全局最优树结构，这是 CART 的典型特性。

## R07

复杂度分析（单节点样本数记为 `m`，特征数为 `p`）：
- 对单个特征：排序 `O(m log m)`，扫描候选阈值 `O(m)`；
- 单节点找最优切分约 `O(p * m log m)`；
- 整棵树总复杂度与各节点样本规模分布有关，常见训练开销可近似看作多层节点代价之和。

空间复杂度：
- 树结构 `O(number_of_nodes)`；
- 节点内部排序和前缀统计由局部数组组成，随节点样本规模变化。

## R08

边界与异常处理：
- `X` 必须是二维，`y` 必须是一维，且样本数一致；
- `X/y` 不能包含 `nan` 或 `inf`；
- `max_depth >= 0`，`min_samples_split >= 2`，`min_samples_leaf >= 1`；
- `test_ratio` 必须在 `(0,1)`；
- 预测前必须先 `fit`，且预测特征维度必须与训练一致。

## R09

MVP 范围与取舍：
- 只实现最核心的 CART 回归树（平方误差、二叉切分、叶均值）；
- 不实现剪枝、缺失值原生处理、类别特征专门策略、多输出回归；
- 不做并行加速与直方图近似切分，优先代码可读性；
- 增加一个可选 `sklearn` 对照输出，帮助验证手写实现行为是否合理。

## R10

`demo.py` 关键函数职责：
- `TreeNode`：树节点数据结构（叶/非叶、阈值、左右子树等）。
- `RegressionTreeMVP.fit`：训练入口，递归构树。
- `RegressionTreeMVP._best_split`：按特征排序并扫描候选阈值，选最大增益分裂。
- `RegressionTreeMVP._build_tree`：递归建树与停止条件控制。
- `RegressionTreeMVP.predict` / `_predict_one`：批量与单样本预测。
- `make_piecewise_regression_data`：生成分段非线性回归数据。
- `train_test_split`：可复现训练/测试划分。
- `mse` / `r2_score`：回归指标。
- `maybe_compare_with_sklearn`：可选地与 `DecisionTreeRegressor` 做结果对照。
- `main`：端到端运行与打印结果。

## R12

输出解读：
- `train/test size`：训练与测试样本数量；
- `hyperparameters`：树深度、最小分裂样本数、最小叶子样本数等；
- `tree stats`：节点数、叶子数、树深；
- `train MSE`：训练集均方误差；
- `test MSE`、`test R2`：测试集泛化性能；
- `baseline mean-predictor test MSE`：常数均值基线，用于判断树是否学到结构；
- `feature importances`：按累计增益归一化的特征重要性；
- `Sample predictions`：部分测试样本的真值与预测值对照。

## R14

关键超参数与调参建议：
- `max_depth`：控制模型容量。太小欠拟合，太大易过拟合。
- `min_samples_split`：节点继续分裂所需最小样本数。
- `min_samples_leaf`：叶节点最小样本数，增大可提升稳定性。
- `min_impurity_decrease`：最小增益阈值，抑制无效细分。

实践中建议先固定较保守的 `max_depth` 和 `min_samples_leaf`，再微调增益阈值。

## R15

与相关模型对比：
- 对比线性回归：回归树可建模非线性和特征交互，但不擅长外推；
- 对比随机森林：单树可解释性强，随机森林泛化更稳但可解释性下降；
- 对比 GBDT：GBDT 精度常更高，但训练与调参更复杂。

本任务聚焦“单棵回归树”的机制透明与可运行最小实现。

## R18

`demo.py` 源码级算法流程（8 步）：

1. `main` 调用 `make_piecewise_regression_data` 生成可复现实验数据，并做训练/测试拆分。
2. 初始化 `RegressionTreeMVP`，设置 `max_depth/min_samples_split/min_samples_leaf/min_impurity_decrease`。
3. `fit` 调用 `_build_tree` 从根节点开始递归建树。
4. `_build_tree` 先计算当前节点预测值（`mean(y)`）与父节点 `SSE`，检查是否触发停止条件。
5. 若可继续分裂，调用 `_best_split`：对每个特征排序后，用前缀和扫描候选阈值并计算 `Gain`。
6. 选出最优 `(feature, threshold)` 后按阈值划分左右子集，递归构造左右子树；若增益不足则退化为叶节点。
7. 预测阶段 `predict -> _predict_one` 对每个样本自顶向下比较阈值，直到叶子并返回叶均值。
8. `main` 计算 `MSE/R2`、基线对比、树结构统计与特征重要性；若可用再执行 `maybe_compare_with_sklearn` 做外部 sanity check（非训练主流程）。
