# 整数规划 - 割平面法

## R01

割平面法（Cutting Plane）用于求解整数规划：

- 先解线性规划松弛（忽略整数约束）；
- 若最优解含分数，则构造一个对所有整数可行点都成立、但能切掉当前分数解的线性不等式（割）；
- 把该割加入模型并继续求解，直到得到整数解或达到停止条件。

本条目聚焦 Gomory 割思想，并在实现中采用 **GMI（Gomory Mixed-Integer）系数形式**，以便在加入连续割松弛变量后仍可继续切割。

## R02

历史脉络（简述）：

- 1950s，Ralph Gomory 提出系统化割平面法；
- 早期“纯割平面”在数值稳定和效率上受限；
- 现代 MIP 求解器普遍采用“分支定界 + 割平面”的混合框架（Branch-and-Cut）；
- Gomory 割依然是理解整数规划求解器内部机制的核心入门模块。

## R03

本目录 MVP 目标：实现一个可运行、可读、可验证的小型 Gomory/GMI 教学求解器。

问题形式：

- `max c^T x`
- `s.t. A x <= b`
- `x >= 0, x in Z^n`

约束：

- 以教学型小规模稠密问题为主；
- `demo.py` 不依赖外部黑盒 MIP 求解器；
- 不交互输入，直接运行即可看到完整流程和校验结果。

## R04

MVP 算法主线：

1. 用原始单纯形（primal simplex）求 LP 松弛最优基；
2. 检查当前解 `x` 是否整数；
3. 若有分数解，从单纯形表中选取“基变量应为整数且 RHS 含分数”的行；
4. 由该行构造一条 Gomory 分数割；
5. 把割加入表并引入新的割松弛变量；
6. 用对偶单纯形（dual simplex）恢复可行并继续最优；
7. 迭代直到整数解、无可用割行或达到割数量上限。

## R05

复杂度（教学实现视角）：

- 单次 pivot 的核心行变换是 `O(mn)`；
- 单次 LP 求解复杂度依赖 pivot 次数，理论上可指数，工程上常较快；
- 每加入一条割，约束数 `m` 和变量数 `n` 都会增加 1；
- 因此该 MVP 适合小规模演示，不适合大规模工业实例。

## R06

`demo.py` 的示例模型：

- `max x1 + x2`
- `2x1 + x2 <= 4`
- `x1 + 2x2 <= 4`
- `x1, x2 >= 0, integer`

该问题的 LP 松弛最优解为 `(4/3, 4/3)`（分数），割平面会切掉该点并最终得到整数最优值 `2`（如 `(2,0)`、`(1,1)`、`(0,2)`）。

## R07

算法价值：

- 展示“线性松弛 -> 分数割 -> 再优化”的整数规划核心思想；
- 帮助理解商用求解器中的 cut generation 子系统；
- 作为学习 branch-and-cut、混合整数线性规划（MILP）前的基础模块。

## R08

关键理论点：

- Gomory 分数割来自最终单纯形表的一行：当 `x_B` 应为整数且 `b̄` 含分数时，可构造能切掉当前分数解的有效不等式；
- 在本实现中，割系数按 GMI 规则计算：
  对整数变量使用分数部分分段公式，对连续变量按符号分段缩放；
  这能在后续出现连续割松弛变量时维持割的有效性；
- 该不等式对整数可行解有效，但会切掉当前分数基解；
- 连续添加有效割可逐步逼近整数凸包（实践中常与分支结合）。

## R09

适用范围与局限：

适用：

- 小规模 pure-integer 教学问题；
- 需要透明可追踪的算法过程；
- 需要源码级展示割是如何从表中生成。

局限：

- 纯割平面可能收敛慢；
- 数值稳定性不及工业级求解器；
- 复杂实例通常需要预处理、启发式、分支策略、切割池管理等组件。

## R10

正确性（工程化说明）：

1. primal simplex 得到 LP 松弛最优基；
2. 若该基对应的决策变量已整数，则它就是原整数规划的可行候选（并被脚本校验）；
3. 若出现分数，Gomory 割对所有整数点有效，因此不会误删整数可行域；
4. 该割会切除当前分数基解；
5. dual simplex 在加入新割后恢复可行并保持最优性条件；
6. 迭代后若得到整数解，再用暴力枚举（小盒子）做独立验证。

## R11

数值与实现注意事项：

- 用 `tol=1e-9` 处理“近似整数”和“近零 pivot”判断；
- pivot 时做整行消元，不显式求逆；
- 对偶单纯形中仅从 `a_rj < 0` 的列挑选入基；
- 对非常接近 0 的数做裁剪，降低浮点噪声；
- 若无有效入基列或无可用割行，返回对应状态而非静默失败。

## R12

性能取向：

- 本实现优先“可读性 + 最小依赖 + 可复现”；
- 没有做稀疏优化、抗退化规则（如 Bland）和并行 cut 分离；
- 与黑盒求解器相比速度不是目标，透明度是目标。

## R13

可验证性保证：

- `demo.py` 固定示例、无随机输入；
- 输出每轮 `x`、`obj`、割次数；
- 使用独立暴力枚举（有限范围）验证最优目标值一致；
- 包含可行性断言（非负与 `Ax<=b`）与状态断言；
- 通过后打印 `All checks passed.`。

## R14

异常与边界处理：

- `A/b/c` 维度不匹配或出现 `nan/inf`：抛 `ValueError`；
- 若 `b` 有负分量（本 MVP 不支持两阶段初始化）：抛 `ValueError`；
- LP 无界、对偶单纯形无可选入基列、迭代超限：抛 `RuntimeError` 或返回受控状态。

## R15

`demo.py` 模块结构：

- `SolveResult`：封装状态、目标值、解向量、迭代历史；
- `GomoryCuttingPlaneSolver`：主求解器类；
- `_build_initial_tableau` / `_pivot`：单纯形表搭建与枢轴；
- `primal_simplex` / `dual_simplex`：LP 优化与加割后重优化；
- `select_gomory_row` / `add_gomory_fractional_cut`：割行选择与 GMI 系数割生成；
- `brute_force_reference`：小规模独立校验；
- `main`：构造样例并执行全流程。

## R16

## R18

源码级算法流程拆解（`demo.py` 主路径，8 步）：

1. `main` 定义 ILP 示例矩阵 `A,b,c`，实例化 `GomoryCuttingPlaneSolver`。  
2. 构造初始单纯形表：约束行 `Ax + s = b`，目标行放置 `-c`（最大化约定）。  
3. 执行 `primal_simplex`：反复选择最负 reduced cost 入基，并做最小比值检验出基，得到 LP 松弛最优基。  
4. 读取当前基解 `x`；若决策变量都近似整数，直接返回 `optimal_integer`。  
5. 否则 `select_gomory_row` 在“基变量应为整数且 RHS 分数”的行中选割行。  
6. `add_gomory_fractional_cut` 按 GMI 分段系数（整数列/连续列分别处理）生成割，扩展单纯形表（新增一行一列，新增列为连续割松弛变量）。  
7. 加割后调用 `dual_simplex`：从负 RHS 行出发恢复可行，维持目标行的对偶可行性，获得新 LP 最优。  
8. 重复步骤 4-7，直到整数解/无可用割/达到上限；最后用 `brute_force_reference` 做独立最优值核对并输出 `All checks passed.`。

说明：本 MVP 的核心计算完全在源码内实现，没有把第三方优化库当黑盒求解器。
