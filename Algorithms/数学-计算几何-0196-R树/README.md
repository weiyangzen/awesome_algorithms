## R01

R 树（R-tree）是用于多维空间对象索引的平衡层次结构，核心对象是最小外接矩形（MBR, Minimum Bounding Rectangle）。

- 每个叶子节点存空间对象（本 MVP 中是二维点，对应退化矩形）。
- 每个内部节点存若干子节点的 MBR。
- 查询时通过“MBR 是否相交/可达”做剪枝，避免全表扫描。

它典型支持：
- 范围查询（window/range query）
- 近邻查询（nearest neighbor）
- 空间连接等更复杂的空间操作

## R02

历史上，R 树由 Antonin Guttman 在 1984 年提出，是数据库与 GIS（地理信息系统）中经典的空间访问方法。

常见后续变体包括：
- R* 树（改进分裂与重插入策略）
- R+ 树（允许对象复制到多个叶）
- Hilbert R 树（借助空间填充曲线排序）

本条目实现的是“原始 R 树思想 + 二次分裂（quadratic split）”的教学级可运行 MVP。

## R03

本目录 MVP 要解决的问题：

- 输入：一批二维点 `p_i=(x_i,y_i)`。
- 构建：把点插入 R 树（节点容量受 `max_entries/min_entries` 约束）。
- 输出能力：
  - 给定轴对齐矩形 `Q`，返回全部命中点 ID；
  - 给定查询点 `q`，返回 `k` 个最近点。

基线对照：
- 范围查询对照暴力筛选；
- kNN 对照暴力距离排序；
- 结果必须一致，否则脚本抛错。

## R04

本实现的核心流程（插入 + 分裂）：

1. **ChooseLeaf**：从根向下，选择“扩张代价最小”的子节点（若并列，选面积更小 MBR）。
2. 在叶节点插入新条目（点对应退化矩形）。
3. 若节点溢出（`len(entries) > max_entries`），触发 **Quadratic Split**：
   - 先选两颗种子（浪费面积最大的一对）；
   - 再逐条把剩余条目分配到扩张代价更小的组。
4. **AdjustTree**：向上更新父节点中对应子节点的 MBR；若父节点因新增兄弟也溢出，继续向上分裂。
5. 若根节点分裂，创建新根，树高 +1。

查询流程：
- 范围查询：DFS，仅访问与 `Q` 相交的 MBR。
- kNN：按 `mindist(MBR,q)` 做分支限界；下界超过当前第 `k` 近距离时剪枝。

## R05

设：
- 总对象数 `N`
- 每节点最大分支因子 `M`
- 树高 `h ~ O(log_M N)`（平衡假设下）

复杂度（典型平均情况）：
- 单次插入：`O(h * M)`（选择路径 + 局部分裂开销）
- 范围查询：`O(h + t)` 到 `O(N)`，其中 `t` 为输出条目数，取决于 MBR 重叠度
- kNN：常显著少于全扫描，但最坏仍可退化到 `O(N)`

空间复杂度：
- 存储索引结构总体 `O(N)`；
- 额外常数级临时空间（递归栈/堆结构不计输出）。

## R06

二维小例子（概念级）：

已插入点：
- `A(1,1), B(2,2), C(8,8), D(9,9)`

若叶容量上限是 2：
- 插入 `A,B` 后还在同一叶；
- 插入 `C` 时叶溢出，分裂为两组：`{A,B}` 与 `{C}`；
- 再插入 `D`，通常落到 `{C}` 那侧；
- 根节点保存两个子叶的 MBR。

范围查询 `Q=[0,0]-[3,3]`：
- 只需访问与 `Q` 相交的那个子叶，命中 `A,B`，剪掉 `C,D` 所在子树。

## R07

R 树的工程价值：

- 将“全量扫描”变为“层次剪枝”。
- 面向空间对象（点、矩形、多边形包围盒）统一处理。
- 动态插入友好，适合增量构建索引。

局限也很明显：
- 当数据分布导致 MBR 严重重叠时，查询退化明显。
- 分裂策略直接影响查询性能与树形质量。

## R08

本实现使用的关键量：

1. 矩形面积
   - `area(R) = (xmax-xmin)*(ymax-ymin)`
2. 合并矩形（并包围盒）
   - `union(R1,R2)`
3. 扩张代价（用于选择路径/分配条目）
   - `enlarge(R,E) = area(union(R,E)) - area(R)`
4. 点到矩形下界距离（用于 kNN 剪枝）
   - `mindist^2(q,R) = dx^2 + dy^2`
   - 若点在矩形投影内，对应轴距离为 0

这些量都在 `demo.py` 的 `Rect` 方法中显式实现。

## R09

适用场景：

- 中低维空间检索（尤其 2D/3D）。
- GIS、地图 POI、碰撞检测、空间过滤。
- 需要同时支持动态插入和空间查询。

不适用或需谨慎：

- 高维向量近邻（维度灾难下 MBR 剪枝弱）。
- 极端偏斜分布导致大面积重叠。
- 需要严格最优 I/O 边界时，通常要更复杂工程优化（如 R* 树、bulk-load）。

## R10

正确性要点（针对本实现）：

1. **覆盖不变式**：内部节点条目 MBR 始终覆盖其子树所有对象。
2. **查询完备性**：
   - 范围查询只剪掉“与查询矩形不相交”的 MBR；
   - 由于覆盖不变式，被剪分支不可能有命中对象。
3. **kNN 剪枝正确性**：
   - `mindist(MBR,q)` 是该子树任一点距离下界；
   - 若该下界已大于当前第 `k` 近距离，不可能产生更优候选，可安全剪枝。
4. `demo.py` 通过与暴力法逐项比对做实现层验证。

## R11

数值与边界处理：

- 坐标统一为 `float`。
- 距离比较优先使用平方距离，减少不必要 `sqrt`（最终展示时再开方）。
- 矩形相交采用闭区间判定，边界点视作命中。
- 入参维度检查：点/查询点必须是 `(2,)`。
- 空树情况下查询返回空列表，不抛异常。

## R12

性能经验（MVP 视角）：

- `max_entries` 太小会增高树高；太大则单节点线性扫描开销上升。
- `min_entries` 常设为 `max_entries/2` 左右，减少过度稀疏节点。
- 构建阶段最耗时通常在分裂与路径选择；查询阶段主要受重叠率影响。
- 该实现偏“可读与可验证”，未做磁盘页、批量加载、SIMD 等工程优化。

## R14

常见失效模式：

1. 节点分裂时忘记维护 `child.parent`，导致上溯更新路径错误。
2. 父节点 MBR 不刷新，后续查询出现漏检。
3. `mindist` 实现错误，kNN 会误剪枝。
4. 容量参数非法（如 `max_entries<3` 或 `min_entries>max_entries//2`）。

本实现通过：
- 显式 `_reattach_child_parent`；
- `AdjustTree` 逐层回写 MBR；
- 暴力对照断言；
来降低上述风险。

## R15

`demo.py` 的结构：

- `Rect`：几何原语（面积、并集、扩张、相交、点到矩形下界距离）。
- `Entry`：节点条目（叶条目含点和 payload，内部条目含 child）。
- `RTreeNode`：节点与节点级 MBR 计算。
- `RTree`：插入、分裂、上溯调整、范围查询、kNN。
- `brute_force_range / brute_force_knn`：正确性对照基线。
- `main`：构建 500 点索引并执行断言。

## R16

相关算法链路：

- 同族：R 树、R* 树、R+ 树、Hilbert R 树。
- 对比结构：
  - KD-tree：对点数据高效，但动态更新和复杂对象支持受限；
  - Quad-tree：空间分解固定，数据偏斜时易不平衡；
  - Ball-tree/VP-tree：偏向度量空间近邻。

在“动态空间对象 + 矩形过滤”场景，R 树通常是更自然的基础结构。

## R18

`demo.py` 源码级算法流（8 步）：

1. `main` 固定随机种子生成 500 个二维点，逐点调用 `RTree.insert` 建索引。  
2. `insert` 先把点转成退化矩形条目，再通过 `_choose_leaf` 按最小扩张代价下探到目标叶节点。  
3. 若叶节点溢出，调用 `_split_node` 进行二次分裂：`_pick_seeds_quadratic` 选种子，`_pick_next_entry` + `_assign_entry` 分配剩余条目。  
4. `_adjust_tree_after_insert` 将分裂和 MBR 变化向上传播；若根分裂则创建新根，保持树平衡。  
5. `range_query` 从根 DFS，仅访问与查询矩形相交的条目 MBR，叶层收集 payload。  
6. `k_nearest` 用最小堆按 `mindist(MBR,q)` 扩展候选子树，并用当前第 `k` 近距离做剪枝（分支限界）。  
7. `brute_force_range` 与 `brute_force_knn` 分别提供暴力基线，`main` 对每个查询做严格一致性断言。  
8. 所有断言通过后输出树统计、查询命中与近邻列表，最后打印 `All checks passed.`。  

该实现没有把第三方空间索引库当黑盒调用，索引维护与查询剪枝逻辑都可在源码中逐行追踪。
