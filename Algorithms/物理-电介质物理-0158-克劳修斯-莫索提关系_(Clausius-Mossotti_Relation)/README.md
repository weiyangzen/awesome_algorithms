## R01

克劳修斯-莫索提关系把宏观相对介电常数与微观分子极化率连接起来：

`(εr - 1)/(εr + 2) = Nα/(3ε0)`

其中：

- `εr`：相对介电常数；
- `N`：分子数密度（`1/m^3`）；
- `α`：单分子极化率（SI 单位 `C·m^2/V`）；
- `ε0`：真空介电常数。

该关系建立在局域场近似（Lorentz 局域场）与各向同性介质假设上。

## R02

该关系的工程价值在于“微观参数 <-> 宏观可测量量”的双向映射：

- 前向建模：给定 `N, α` 预测 `εr`；
- 反向辨识：由 `N, εr` 反推 `α`；
- 参数拟合：在多组带噪观测上估计统一的 `α`。

本条目 MVP 目标：实现前向、点反演、全局拟合三件事，并提供自动数值自检。

## R03

`demo.py` 输入输出（脚本内置样例、无交互）：

- 输入：
1. 合成数据中的数密度数组 `N`；
2. 真实极化率 `alpha_true`（用于构造真值）；
3. 确定性扰动后的 `epsilon_r_measured`。
- 输出：
1. 每个样本的 `εr_true`、`εr_measured`、线性/非线性拟合预测值；
2. 点估计 `alpha_pointwise_est`；
3. 汇总表：`alpha_linear_fit`、`alpha_nonlinear_fit`、相对误差、残差范数；
4. 断言通过后输出 `All checks passed.`。

## R04

MVP 用到的核心公式：

1. 原式：`(εr - 1)/(εr + 2) = Nα/(3ε0)`。
2. 前向表达：
`εr = (1 + 2x)/(1 - x)`，其中 `x = Nα/(3ε0)`。
3. 点反演表达：
`α = 3ε0/N * (εr - 1)/(εr + 2)`。
4. 线性化拟合：令 `y_i = (ε_i-1)/(ε_i+2)`，`x_i = N_i/(3ε0)`，
则 `y_i ≈ x_i α`，最小二乘闭式为 `α_hat = (x^T y)/(x^T x)`。
5. 非线性拟合：在 `ε` 空间最小化
`min_α || ε_model(N, α) - ε_measured ||_2`。

## R05

复杂度（`m` 为样本数）：

- 前向计算 `εr(N, α)`：`O(m)`；

- 线性化闭式拟合：`O(m)`；
- 非线性单参数最小二乘：每次迭代 `O(m)`，总计约 `O(km)`；

## R06

`demo.py` 的最小算法闭环：

- 闭环 A（前向）：`epsilon_r_from_number_density_alpha` 验证 `N, α -> εr`；
- 闭环 B（反向）：`alpha_from_epsilon_r_number_density` 验证 `N, εr -> α`；

## R09

适用场景：

- 教学演示微观极化率与宏观介电常数关系；
- 给定密度范围下快速估算 `εr`；

## R10

正确性直觉：

1. 原公式把微观极化率信息压缩为无量纲比值 `x=Nα/(3ε0)`；
2. 前向与反向公式互为代数逆变换，因此在无噪声下应互相恢复；
3. 多样本拟合本质是在“单一 `α` 解释全部观测”的假设下最小化残差；
4. 若模型与数据一致，线性化与非线性估计都应接近真值。

## R11

数值稳定性策略：

- 对 `N>0`、`εr>0`、数组长度与维度进行严格检查；
- 避免奇点：检查 `εr + 2 != 0`、`cm_ratio < 1`；
- 使用 `float64` 保持高精度；
- 非线性拟合对 `α` 使用正值有界约束，避免无物理意义解。

## R12

关键参数：

- `number_density`：数密度，单位 `1/m^3`；
- `alpha_true`：构造合成真值用的极化率；
- `perturbation`：确定性测量扰动；
- `alpha_init`：非线性拟合初值（由线性化结果提供）。

调参建议：

- 若要更明显观察非线性误差，可增大扰动幅度；
- 若要更接近稀薄气体，可降低 `N` 到 `1e25~1e26` 量级；
- 若 `x=Nα/(3ε0)` 接近 1，模型会趋向奇异，应回退到更低密度或更小 `α`。

## R14

常见失效模式：

1. 把 `N` 单位误写为 `mol/m^3` 而非 `1/m^3`；
2. 输入 `εr` 异常值（<=0 或接近 `-2`）；
3. 在 `x -> 1` 区域仍强行使用模型，导致数值爆炸；
4. 用单一 `α` 拟合实际多组分/各向异性样品；
5. 忽视观测噪声结构，直接把偏差全归因于模型参数。

## R18

`demo.py` 源码级算法流程（8 步）：

1. `CMObservation.validate` 对输入数组做维度、长度、正值和奇点检查，保证后续变换定义域合法。  
2. `cm_ratio_from_epsilon_r` 计算 `y=(εr-1)/(εr+2)`，把非线性介电常数映射到 CM 线性化空间。  
3. `epsilon_r_from_cm_ratio` 执行代数逆映射 `εr=(1+2y)/(1-y)`，并显式限制 `y<1` 防止极点。  
4. `epsilon_r_from_number_density_alpha` 先算 `x=Nα/(3ε0)` 再调用逆映射，构成前向物理模型。  
5. `alpha_from_epsilon_r_number_density` 用 `α=3ε0/N * y` 做逐点反演，验证公式可逆性。  
6. `fit_alpha_linearized` 构造 `x_i=N_i/(3ε0)`、`y_i`，通过 `α_hat=(x^Ty)/(x^Tx)` 求一参数闭式最小二乘解。  
7. `fit_alpha_nonlinear` 定义残差 `r_i(α)=ε_model(N_i,α)-ε_measured_i`，调用 `least_squares` 在有界区间内迭代求解。  
8. `main` 组装合成数据、执行前向/反向/拟合、打印表格，并用三类断言（无噪声一致性、拟合误差阈值、物理可行性）形成自证闭环。  

补充：`scipy.optimize.least_squares` 只承担“求解器”角色，物理模型、残差构造、参数约束与结果判定均在源码中显式实现。
