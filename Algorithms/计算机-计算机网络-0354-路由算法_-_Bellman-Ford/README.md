## R01

Bellman-Ford 解决的是单源最短路径问题：在带权有向图中，给定源点 `s`，计算 `s` 到其他顶点的最短距离。它与 Dijkstra 的关键区别是：Bellman-Ford 允许边权为负，并且能够检测从源点可达的负权环。

## R02

在计算机网络里，Bellman-Ford 是距离向量路由思想的基础。典型关系如下：
- RIP 协议的核心思想可视为分布式 Bellman-Ford。
- 每个路由器维护到各目的网络的代价估计，并根据邻居通告持续松弛更新。
- 当存在异常负环（在网络语义里通常对应错误度量配置）时，最短路不再有定义。

## R03

形式化输入输出：
- 输入：
  - 顶点数 `n`
  - 边集合 `E = {(u, v, w)}`，表示从 `u` 到 `v` 的边权为 `w`
  - 源点 `source`
- 输出：
  - `dist[v]`：`source -> v` 的最短距离（不可达则为 `inf`）
  - `pred[v]`：最短路径树中的前驱，用于重建路径
  - `affected_by_neg_cycle`：受负权环影响的点集合（其最短路无定义）

## R04

核心思想是“反复松弛（relaxation）”：
- 对于一条边 `(u, v, w)`，若 `dist[u] + w < dist[v]`，则可以改进 `dist[v]`。
- 在没有负环的情况下，任意最短路最多包含 `n-1` 条边，因此完整扫描所有边 `n-1` 轮即可收敛。
- 若第 `n` 轮还能改进，说明存在可达负权环。

## R05

正确性直觉：
- 第 `k` 轮结束后，`dist[v]` 至少不大于“使用不超过 `k` 条边到达 `v`”的最优代价。
- 由于简单路径最多 `n-1` 条边，做满 `n-1` 轮后，已覆盖所有候选简单路径。
- 若还能继续下降，必然利用了环；若该环使总代价持续下降，则为负权环。

## R06

算法流程（集中式 Bellman-Ford）：
1. 初始化 `dist[source]=0`，其余为 `inf`，`pred` 全为 `None`。
2. 重复 `n-1` 轮：
   - 依次扫描每条边并尝试松弛。
   - 若本轮没有任何更新，可提前停止（early stop）。
3. 再扫描一轮边：
   - 仍可松弛的顶点说明被负环影响或可由负环到达。
4. 通过图遍历扩散影响范围，标记所有受负环影响顶点。

## R07

复杂度：
- 时间复杂度：`O(V * E)`（最坏情况需做 `V-1` 轮，每轮扫描 `E` 条边）
- 空间复杂度：`O(V + E)`（距离、前驱、邻接关系）

对于大规模稠密图，Bellman-Ford 通常慢于 Dijkstra（非负权场景）或 Johnson（多源场景）。

## R08

与 Dijkstra 对比：
- Dijkstra 要求边权非负，通常更快（堆优化下 `O((V+E)logV)`）。
- Bellman-Ford 可处理负权边并检测负环，适用范围更广。
- 网络路由里的“距离向量”更新更接近 Bellman-Ford 的迭代松弛思想。

## R09

负权环处理策略（本项目实现）：
- 检测：第 `n` 轮仍可改进即判定存在可达负环。
- 标记：从可改进点出发做可达性扩散，将所有受影响点加入集合。
- 解释：这些点的最短路可以被负环反复降低，因此“最短值不存在”。

## R11

`demo.py` 的主要函数：

- `reconstruct_path(...)`：基于 `pred` 回溯一条最短路径。
- `print_result_table(...)`：格式化打印每个顶点的距离和路径。
- `run_demo_without_negative_cycle()`：稳定样例。
- `run_demo_with_negative_cycle()`：负环样例。

## R12

边界与异常考虑：
- 不可达点：距离保持 `inf`，路径显示为 `UNREACHABLE`。
- 源点：路径为自身，距离为 `0`。
- 负环影响点：距离不再可信，路径显示为 `UNDEFINED (negative cycle)`。
- 前驱回溯保护：若出现异常环状前驱，回溯函数会安全退出。

## R13

工程实践注意事项（网络视角）：
- 路由度量需保持一致语义，避免出现逻辑负环。
- 在分布式版本中，更新是异步的，可能出现“计数到无穷”问题。
- 实际协议通常配合抑制策略（如 split horizon、poison reverse）降低振荡风险。

## R18

源码级算法流（对应 `demo.py`）：
1. `bellman_ford` 初始化 `dist/pred`，将源点距离置零。
2. 外层进行 `n-1` 轮迭代，每轮顺序扫描全部边。
3. 对每条边执行松弛判定：若 `dist[u] + w < dist[v]`，更新 `dist[v]` 和 `pred[v]`。
4. 若一整轮没有更新，触发 early stop，提前收敛。
5. 额外再扫一轮边，找出仍可继续下降的顶点，作为负环候选入口。
6. 从候选入口沿邻接关系做 BFS/DFS 扩散，得到 `affected_by_neg_cycle` 全集。
7. 输出阶段仅对“可达且不受负环影响”的顶点调用 `reconstruct_path` 回溯路径。
8. 对不可达点或受负环影响点给出语义化状态，避免伪造最短路结果。
