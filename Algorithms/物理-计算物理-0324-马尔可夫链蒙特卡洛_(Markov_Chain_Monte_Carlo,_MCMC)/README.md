## R01

马尔可夫链蒙特卡洛（MCMC）是一类“用马尔可夫链构造样本”的随机数值算法，用于从难以直接采样的目标分布中估计统计量。

在计算物理中，它常用于平衡态统计物理、路径积分、参数反演等场景。核心价值是把高维积分

`E[f(x)] = ∫ f(x) pi(x) dx`

转化为样本平均

`E[f(x)] ≈ (1/N) Σ f(x_i)`。

## R02

本条目采用一个最小但真实的物理示例：

- 体系：一维双阱势 `U(x) = x^4/4 - x^2/2`。
- 系综：固定逆温 `beta` 的玻尔兹曼分布 `pi(x) ∝ exp(-beta * U(x))`。
- 任务：用 Metropolis-Hastings 采样，估计 `E[x]`、`E[x^2]`、`E[x^4]`，并与高分辨率数值积分基准比较。

该问题具备双峰结构，能够体现 MCMC 的“相关采样、接受率权衡、混合质量”这些关键工程点。

## R03

目标分布定义为：

`pi(x) = (1/Z) * exp(-beta * U(x))`

其中配分函数

`Z = ∫ exp(-beta * U(x)) dx`。

我们需要估计的物理量（矩）写为：

`<x^k> = (1/Z) ∫ x^k exp(-beta * U(x)) dx`。

MCMC 不需要显式知道 `Z`，只要知道未归一化密度比例即可执行采样。

## R04

Metropolis-Hastings 随机游走版本：

1. 当前状态 `x_t`。
2. 提议 `x' = x_t + eps`，`eps ~ N(0, sigma^2)`。
3. 计算
`alpha = min(1, pi(x') / pi(x_t)) = min(1, exp(log pi(x') - log pi(x_t)))`。
4. 以概率 `alpha` 接受新状态，否则留在原处。

由于提议分布是对称高斯，`q(x'|x)=q(x|x')`，接受率无需额外 Hastings 修正项。

## R05

本 MVP 的主要假设：

- 目标分布可由未归一化对数密度 `log pi(x)` 计算。
- 马尔可夫链经过足够 burn-in 后近似达到平稳分布。
- 相关样本的统计误差可用自相关时间和 ESS（有效样本量）粗略评估。
- 双阱势场在当前 `beta` 和步长下能够发生跨阱跳转，避免严重困链。

## R06

`demo.py` 输入输出约定：

- 输入：无交互输入；脚本内置 `beta`、`n_steps`、`burn_in`、`proposal_sigma`、`seed`。
- 输出：
1. `E[x]`、`E[x^2]`、`E[x^4]` 的 MCMC 估计与参考值对比表。
2. 诊断指标：接受率、积分自相关时间（IACT）、ESS。
3. 一组阈值检查与最终 `Validation: PASS/FAIL`。

## R07

算法主流程：

1. 定义势能 `U(x)` 与目标对数密度 `log pi(x)`。
2. 运行 Metropolis-Hastings 生成长度为 `n_steps` 的链。
3. 丢弃前 `burn_in` 样本，得到后验样本序列。
4. 计算样本矩估计（`x`、`x^2`、`x^4`）。
5. 用高分辨率网格积分计算“近真值”参考矩。
6. 计算误差、接受率、IACT 和 ESS。
7. 根据阈值判定实现是否通过。

## R08

复杂度分析（`N` 为链长度，`G` 为积分网格点数）：

- MCMC 采样：`O(N)` 时间，`O(N)` 存储（保存整条链）。
- 参考积分：`O(G)` 时间，`O(G)` 存储。
- 自相关估计（截断到 `L`）：`O(N * L)`，本实现中 `L <= 2000`。

在教学和中小规模实验中，这一复杂度足以支持快速迭代。

## R09

数值稳定与实现细节：

- 接受率判断在对数域完成，避免 `exp` 上溢/下溢。
- 参考积分对 `log w` 先减去最大值，再指数化，提升稳定性。
- IACT 采用“正序列截断”：自相关首次非正即停止，抑制高 lag 噪声。
- 使用固定随机种子，确保演示可复现。

## R10

MVP 技术栈：

- Python 3
- `numpy`：随机采样、向量化计算、数值积分
- `pandas`：结果表格化输出

实现中没有依赖 MCMC 黑箱库，转移核、接受率、诊断计算均在源码显式展开。

## R12

输出字段说明：

- `metric`：被估计的统计量名称（`mean_x`、`mean_x2`、`mean_x4`）。
- `mcmc_estimate`：MCMC 样本平均估计。
- `reference`：高分辨率网格积分给出的参考值。
- `abs_error`：绝对误差。
- `rel_error`：相对误差。
- `acceptance_rate`：提议被接受的比例。
- `IACT`：积分自相关时间，越小越好。
- `ESS`：有效样本量，近似衡量独立样本数。

## R13

正确性与质量验证（demo 内置阈值）：

1. 接受率在 `[0.20, 0.80]` 区间。
2. 对称势场下 `|E[x]| < 0.08`（对称性 sanity check）。
3. `E[x^2]` 相对误差小于 `5%`。
4. `E[x^4]` 相对误差小于 `8%`。
5. `ESS > 1000`，避免“样本数大但有效样本很少”的伪收敛。

全部通过时打印 `Validation: PASS`。

## R14

当前实现局限：

- 仅覆盖 1D 连续状态空间。
- 仅使用随机游走 Metropolis，未做梯度利用（如 HMC、MALA）。
- 未进行多链并行与 Gelman-Rubin (`R-hat`) 诊断。
- 参考“真值”来自高分辨率积分而非解析解，仍是数值近似。

## R15

可扩展方向：

- 扩展到 2D/高维势场（如 Ising 连续近似或多粒子势能面）。
- 增加自适应步长策略，使接受率自动靠近目标区间。
- 加入多链并行、`R-hat` 与分块 ESS 诊断。
- 用 HMC/NUTS 与随机游走 MH 做效率对比。

## R16

典型应用场景：

- 统计物理中配分函数相关观测量估计。
- 计算化学中的势能面采样。
- 贝叶斯参数反演（实验数据拟合中的后验采样）。
- 高维积分难以直接求解时的随机数值估计。

## R17

与常见替代方案对比：

- 直接积分：在低维可行，但维度升高后成本急剧上升。
- 重要性采样：需要高质量提议分布，失配时方差极大。
- 本方法（MH-MCMC）：实现简单、适应性强，但样本相关且需收敛诊断。

因此该 MVP 适合演示“从建模到诊断”的完整 MCMC 工程闭环。

## R18

`demo.py` 源码级流程拆解（8 步）：

1. `potential_energy` 定义双阱势 `U(x)`，`log_target_density` 构造未归一化目标密度。
2. `metropolis_hastings_1d` 迭代生成马尔可夫链：高斯提议、对数接受率、接受/拒绝更新。
3. 在 `metropolis_hastings_1d` 末尾丢弃 burn-in，返回样本与整体接受率。
4. `estimate_moments` 从样本直接计算 `E[x]`、`E[x^2]`、`E[x^4]`。
5. `reference_moments` 在高分辨率网格上计算 `exp(-beta U(x))` 加权积分，得到参考矩。
6. `integrated_autocorrelation_time` 基于自协方差序列估计 IACT，`effective_sample_size` 由 `N / IACT` 得 ESS。
7. `build_report` 汇总误差表和阈值检查，形成 PASS/FAIL 逻辑。
8. `main` 打印配置、结果表、诊断指标与验证结论，保证一次运行可复现。

可以看到，虽然底层数值操作使用了 `numpy`，但 MCMC 的转移核、接受机制、统计诊断和验证流程都在源码中透明可追踪。
