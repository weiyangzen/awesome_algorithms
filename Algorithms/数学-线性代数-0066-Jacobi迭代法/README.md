# Jacobi迭代法

- UID: `MATH-0066`
- 学科: `数学`
- 分类: `线性代数`
- 源序号: `66`
- 目标目录: `Algorithms/数学-线性代数-0066-Jacobi迭代法`

## R01

Jacobi 迭代法用于求解线性方程组 `Ax=b`，属于经典定常迭代法（stationary iterative method）。

- 核心思想：把 `A` 拆分为 `A = D + (L+U)`，其中 `D` 是对角矩阵。
- 迭代公式：
  - `x^(k+1) = D^{-1}(b - (L+U)x^(k))`
  - 等价为 `x^(k+1) = B x^(k) + c`，其中 `B = -D^{-1}(L+U)`，`c = D^{-1}b`。
- 每个分量的更新只依赖上一次迭代值，因此并行性较好。

## R02

历史定位（简要）：

- 方法以 Carl Gustav Jacob Jacobi 命名，19 世纪在线性方程数值解法体系中确立；
- 它是 Gauss-Seidel、SOR、多重网格平滑器等方法的基础参照；
- 在现代高性能计算中，Jacobi 常作为“简单、可并行、易分析”的基线迭代器。

## R03

本条目 MVP 解决的问题：

- 输入：
  - 方阵 `A` 与向量 `b`；
  - 初值 `x0`（可选，默认零向量）；
  - 终止阈值 `tol` 与最大迭代步数 `max_iter`。
- 输出：
  - 近似解 `x`；
  - 实际迭代次数与是否收敛；
  - 末次步长误差与残差 `||Ax-b||_2`；
  - 迭代矩阵 `B` 的谱半径估计（用于收敛性解释）。

## R04

时间复杂度（稠密矩阵）：

- 单次迭代主要开销是一次矩阵-向量乘法 `R @ x`（`R=A-D`），代价 `O(n^2)`；
- 进行 `k` 次迭代，总复杂度 `O(k n^2)`；
- 若 `A` 稀疏，单次开销可近似降到 `O(nnz(A))`，总复杂度 `O(k * nnz(A))`。

## R05

空间复杂度：

- 存储 `A` 需要 `O(n^2)`（稠密）；
- 迭代向量与临时向量（`x`、`x_new`、对角线等）为 `O(n)`；
- 若记录完整历史（本 MVP 记录有限诊断，不存全历史），可额外增加 `O(k)` 到 `O(kn)`。

仅看算法附加空间（不计输入矩阵）通常为 `O(n)`。

## R06

微型示例（2x2）：

设

- `A = [[4,1],[2,3]]`
- `b = [1,2]`
- `x^(0) = [0,0]`

则

- `x1^(1) = (1 - 1*x2^(0))/4 = 0.25`
- `x2^(1) = (2 - 2*x1^(0))/3 = 0.6667`

第二步：

- `x1^(2) = (1 - x2^(1))/4 = 0.0833`
- `x2^(2) = (2 - 2*x1^(1))/3 = 0.5`

可见解在迭代中逐步逼近真实解。

## R07

算法意义：

- 把“直接求解”转化为“固定点迭代”，实现简单、可扩展；
- 每个分量更新相互独立，天然适合并行硬件；
- 在大型线性系统中，常作为预处理、平滑器或教学/验证基线。

## R08

核心数学关系：

1. 矩阵分裂：`A = D + L + U`。  
2. Jacobi 迭代：`x^(k+1) = -D^{-1}(L+U)x^(k) + D^{-1}b`。  
3. 写成固定点形式：`x^(k+1) = Bx^(k)+c`。  
4. 充分必要收敛判据（线性固定点）：`rho(B) < 1`，`rho` 为谱半径。  
5. 常见充分条件：`A` 严格对角占优时 Jacobi 收敛。

## R09

适用条件与局限：

适用：

- `A` 对角元素全非零；
- `A` 满足严格对角占优，或可验证 `rho(B) < 1`；
- 问题规模大、需要简单并行迭代器。

局限：

- 相比 Gauss-Seidel/SOR，Jacobi 常收敛较慢；
- 对病态或不满足收敛条件的系统可能发散；
- 仅用残差阈值不能保证所有问题上的最优停止策略。

## R10

正确性要点（工程可验证版）：

1. 当 `A` 可逆且 `D` 非奇异时，真解 `x*` 满足固定点方程 `x*=Bx*+c`。  
2. 误差递推为 `e^(k+1)=B e^(k)`，其中 `e^(k)=x^(k)-x*`。  
3. 若 `rho(B)<1`，则 `B^k -> 0`，从而 `e^(k)->0`，迭代收敛到唯一固定点 `x*`。  
4. `demo.py` 同时检查步长误差、残差、与 `numpy.linalg.solve` 的差异，作为实现层验证。

## R11

数值稳定性与收敛性：

- 对角线元素过小会放大舍入误差，应提前做零/近零检查；
- 终止条件通常组合使用：
  - `||x^(k+1)-x^(k)||_inf < tol`
  - `||Ax-b||_2` 足够小；
- 本实现给出 `rho(B)` 估计，帮助解释“为什么收敛/不收敛”；
- 对高条件数系统，建议配合预处理或改用更高效迭代法（如 CG、GMRES）。

## R12

性能视角：

- 每轮主要热点是 `R @ x` 与向量逐元素除法；
- Python 层应尽量依赖 NumPy 向量化，避免纯 Python 双重循环；
- 若迁移到稀疏存储与并行硬件，Jacobi 的结构优势更明显；
- 但在同精度目标下，迭代次数可能显著多于更先进方法。

## R13

本目录 MVP 的可验证保证：

- 构造固定随机种子下的严格对角占优系统，保证可复现；
- 自动输出并校验：
  - `converged` 标志；
  - 最终残差 `||Ax-b||_2`；
  - 与直接法解的无穷范数误差；
  - 迭代矩阵谱半径是否小于 1。
- 若核心指标超阈值，脚本抛出异常并失败退出。

## R14

常见失效模式：

- `A` 非方阵或与 `b` 维度不匹配；
- 对角线出现 0（无法做 `D^{-1}`）；
- 系统不满足收敛条件，达到 `max_iter` 仍未收敛；
- 阈值设置过严/过松导致迭代次数异常或精度不足。

## R15

实现设计（`demo.py`）：

- `JacobiResult`：封装解、迭代诊断与收敛指标；
- `validate_inputs`：维度、有限值、对角线合法性检查；
- `jacobi_solve`：按公式显式实现 Jacobi 迭代，并记录最后步长与残差；
- `build_strictly_diagonally_dominant_system`：构造可复现测试系统；
- `run_checks`：统一断言收敛性与精度；
- `main`：执行实验、打印报告、无交互退出。

## R16

相关算法链路：

- 同类定常迭代：Gauss-Seidel、SOR；
- 非定常 Krylov 方法：CG、BiCGSTAB、GMRES；
- 多重网格中：Jacobi 常被用作平滑器（smoother）；
- 预处理体系中：对角预处理（Jacobi preconditioner）直接源于 `D^{-1}` 思想。

## R18

`demo.py` 的源码级算法流程（9 步，非黑盒）如下：

1. `main` 固定随机种子，构造严格对角占优矩阵 `A`、真解 `x_true` 与右端 `b=A@x_true`。  
2. `jacobi_solve` 先调用 `validate_inputs`，检查方阵性、维度匹配、有限值和对角线非零。  
3. 拆分出 `diag(A)` 与 `R=A-diag(diag(A))`，并准备初值 `x^(0)`（默认零向量）。  
4. 每轮按 `x^(k+1)=(b-Rx^(k))/diag(A)` 向量化更新全部分量。  
5. 计算步长误差 `||x^(k+1)-x^(k)||_inf`，若小于 `tol` 则标记收敛并提前停止。  
6. 迭代结束后计算残差 `||Ax-b||_2`，并记录实际迭代步数与收敛标志。  
7. 额外构造迭代矩阵 `B=-D^{-1}R`，用 `np.linalg.eigvals(B)` 得到谱半径估计。  
8. `main` 计算直接法参考解 `np.linalg.solve(A,b)`，并统计 Jacobi 解与真解/参考解误差。  
9. `run_checks` 对收敛性、残差、解误差、谱半径进行断言，全部通过后打印 `All checks passed.`。

说明：第三方库只提供基础线性代数运算与验算接口，Jacobi 迭代主流程由源码逐步实现。
