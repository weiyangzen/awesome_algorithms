# 漂移室 (Drift Chambers)

- UID: `PHYS-0413`
- 学科: `物理`
- 分类: `粒子物理实验`
- 源序号: `433`
- 目标目录: `Algorithms/物理-粒子物理实验-0433-漂移室_(Drift_Chambers)`

## R01

漂移室（Drift Chamber）是高能物理实验中常见的带电粒子径迹探测器。其核心观测量不是“击中点坐标”，而是电子漂移时间 `t_drift`。在已知漂移速度 `v_drift` 的标定条件下，时间可换算为“粒子轨迹到感应丝的最短距离” `d = v_drift * t_drift`。该距离只给出绝对值，因此天然存在左右二义性（left-right ambiguity）。

## R02

该算法条目的工程价值在于把“探测器原始时间读出”转为“可用于重建的几何轨迹参数”：

- 展示从时间域到空间域的最小重建链路；
- 体现漂移室最典型难点：左右二义性消解；
- 给后续动量重建（磁场中曲率拟合）提供二维直线基线；
- 提供可复现实验误差（时间噪声、离群点）下的稳健重建样例。

## R03

本 MVP 的输入输出约定（脚本内固定参数、无交互）：

- 输入：
1. 漂移室几何配置（层数、层间距、丝间距、交错结构）；
2. 漂移速度、时间测量噪声；
3. 一条真实直线轨迹 `x(y)=a*y+b`；
4. 单个离群时间点（模拟电子学异常）。
- 输出：
1. 迭代 OLS 与迭代 Huber 两种重建结果表；
2. 坡度误差、截距误差、轨迹 RMSE、左右符号准确率等指标；
3. 自动断言与 `All checks passed.`。

## R04

MVP 对应的核心物理/数学关系：

1. 漂移时间到距离：`d_i = v_drift * t_i`。
2. 轨迹模型（2D 简化）：`x(y)=a*y+b`。
3. 第 `i` 层测量方程（含左右符号）：
`x_i(hit) = x_i(wire) + s_i * d_i`, 其中 `s_i in {-1,+1}`。
4. 左右符号由当前拟合轨迹决定：
` s_i = sign(x_pred_i - x_i(wire)) `。
5. 目标函数：最小化 `sum rho(a*y_i+b-x_i(hit))`，其中 `rho` 取二次损失（OLS）或 Huber 损失（稳健拟合）。

## R05

设层数为 `N`，迭代轮数为 `I`：

- 每轮符号更新与伪击中构造为 `O(N)`；
- 每轮线性参数拟合（2 参数）为 `O(N)`；
- 总体复杂度约 `O(I*N)`；
- 空间复杂度 `O(N)`。

在该 MVP 中 `N=10`，因此重建成本很低，适合教学与单事件调试。

## R06

`demo.py` 实现了三个最小闭环：

- **闭环 A：测量模拟**
  从真实轨迹生成每层最近丝、真实漂移距离、含噪时间读出。
- **闭环 B：二义性消解 + 拟合**
  通过“预测轨迹→符号分配→拟合更新”的固定点迭代求解轨迹。
- **闭环 C：质量评估**
  与真值比较并输出误差表，最后用阈值断言确保脚本可自动验收。

## R09

## R10

正确性直觉：

1. 若漂移速度标定正确，`t -> d` 可恢复与丝的几何距离；
2. 给定一组 `s_i`，轨迹拟合是标准线性（或稳健）回归问题；
3. 轨迹更新后符号会逐步稳定，形成自洽解；
4. 离群层对 OLS 影响更大，而 Huber 会降低其权重，因此误差通常更小。

## R11

数值稳定策略：

- 用固定随机种子生成事件，保证结果可复现；
- 对测量时间裁剪为非负，避免非物理漂移距离；
- 对漂移距离设置上界，避免极端异常主导拟合；
- 采用迭代停止阈值控制（参数变化量足够小时提前收敛）。

## R12

关键参数：

- `drift_velocity_cm_per_ns`：时间到距离转换比例，标定误差会直接传导到轨迹偏差；
- `time_sigma_ns`：时间分辨率，决定距离噪声水平；
- `outlier_time_bias_ns`：离群点强度，用于测试稳健性；
- `max_iter`：左右歧义迭代次数上限；
- `f_scale`（Huber）：从“二次惩罚”过渡到“线性惩罚”的阈值。

调参建议：

- 若符号收敛慢，先增大层数或优化交错几何；
- 若离群点较多，优先使用 Huber 并降低 `f_scale`；
- 若噪声较小且无离群点，OLS 通常更高效。

## R14

常见失效模式：

1. 漂移速度标定错误导致系统性尺度偏差；
2. 几何参数错误（丝位置/层间距错误）导致拟合失真；
3. 多条近邻轨迹共享击中引发组合歧义（本 MVP 未处理）；
4. 初值过差时符号迭代陷入错误固定点。

## R18

`demo.py` 源码级算法流程（9 步）：

1. `simulate_event` 依据真值轨迹 `x=a*y+b` 和交错丝几何，确定每层最近丝位置 `x_wire`。  
2. 计算真实漂移距离 `d_true=|x_true-x_wire|`，再按 `t=d_true/v_drift + 噪声` 生成时间测量。  
3. 将测量时间裁剪为非负，并转回测量距离 `d_meas=v_drift*t_meas`；再注入一个离群层偏置。  
4. `reconstruct_track` 用丝中心点先做一次线性拟合作为初始轨迹参数。  
5. 在每次迭代中，先用当前轨迹预测 `x_pred`，再按 `sign(x_pred-x_wire)` 分配左右符号 `s_i`。  
6. 用 `x_hit=x_wire+s_i*d_meas` 构造伪击中点，把“距离观测”转为“坐标观测”。  
7. 执行参数更新：OLS 分支调用 `np.linalg.lstsq`；稳健分支调用 `scipy.optimize.least_squares(loss="huber")`。  
8. 以参数增量作为收敛判据，达到阈值后输出 `a,b,signs,iterations`。  
9. `evaluate_result` 计算 `track_rmse_cm`、`lr_sign_accuracy`、`slope/intercept` 误差，`main` 汇总表格并断言通过。  

说明：`scipy` 仅用于稳健目标求解器；左右消歧策略、几何构造、伪击中生成与指标评估均在源码中显式实现，不是黑盒调用。
