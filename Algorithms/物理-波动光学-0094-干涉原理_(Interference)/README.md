## R01

干涉原理的核心是波的叠加：
当两束同频波在空间相遇时，总振幅为矢量和，总强度不是简单相加，而是包含与相位差相关的交叉项。

对双缝模型而言，观测屏位置 `y` 处的明暗条纹来自相位差 `delta(y)` 的周期变化。

## R02

本条目要解决的最小问题是：
给定双缝参数和观测屏几何，数值生成干涉强度分布，并验证两条经典关系：

1. 条纹间距近轴近似 `Delta y ≈ lambda * L / d`；
2. 条纹可见度 `V = (Imax-Imin)/(Imax+Imin)` 与理论式的一致性。

## R03

MVP 采用“可运行 + 可核验”的建模边界：

- 两束同频光，允许振幅不等（`amplitude_1`, `amplitude_2`）；
- 用 `coherence_gamma in [0,1]` 描述部分相干导致的对比度衰减；
- 几何关系使用精确 `sin(theta)=y/sqrt(y^2+L^2)`，不强制只用小角近似；
- 可选单缝衍射包络 `sinc^2`（用于展示“干涉在包络内”）。

## R04

`demo.py` 使用的物理方程：

- 相位差：
  `delta(y) = (2*pi/lambda) * d * sin(theta) + phi0`
- 干涉强度：
  `I = I1 + I2 + 2*gamma*sqrt(I1*I2)*cos(delta)`
- 单缝包络（可选）：
  `E = sinc((a/lambda)*sin(theta))^2`
- 最终强度：
  `I_total = E * I`
- 近轴条纹间距理论值：
  `Delta y_theory = lambda*L/d`
- 理论可见度：
  `V_theory = 2*gamma*sqrt(I1*I2)/(I1+I2)`

## R05

脚本输出三类结果：

- 条纹间距验证：理论值 vs 峰值检测得到的观测值；
- 可见度验证：理论值 vs 数值极值恢复值；
- 一份含包络效应的剖面表（`y, theta, delta, envelope, intensity`）用于审计。

末尾给出阈值检查与 `Validation: PASS/FAIL`。

## R06

实现策略：

- 用 `InterferenceConfig` 统一管理参数；

## R07

## R09

设采样点数为 `N`。

- 强度计算：逐点向量化计算，时间复杂度 `O(N)`；
- 峰值检测：`find_peaks` 线性扫描量级，`O(N)`；
- 表格构建：`O(N)`；
- 总体时间复杂度 `O(N)`，空间复杂度 `O(N)`。

## R10

数值稳定性处理：

- 使用精确几何 `sin(theta)`，避免大角度时小角近似偏差；
- 强度结果做 `clip`，抑制浮点误差造成的极小负值；
- 峰值检测设置最小间隔和 prominence，减少噪声峰误检；
- 可见度检查采用解析可控场景（无包络）以减少估计偏差。

## R11

默认参数（见 `demo.py`）：

- `lambda = 532 nm`
- `d = 0.25 mm`
- `L = 1.5 m`
- 空间采样：`y in [-15 mm, 15 mm]`，`N=6001`

验证分两组：

- 条纹间距组：`A1=A2=1`, `gamma=1`；
- 可见度组：`A1=1`, `A2=0.55`, `gamma=0.75`；
- 展示组额外开启包络：`slit_width = 45 um`。

## R12

内置正确性阈值：

1. 条纹间距相对误差 `< 2%`；
2. 可见度绝对误差 `< 0.02`。

两项都满足才输出 `Validation: PASS`，否则进程以非零状态退出。

## R13

保证类型说明：

- 这是物理模型一致性验证，不是最优化问题，因此无“近似比”概念；
- 在固定参数下是确定性计算，不含随机性；
- “保证”体现在与理论关系的误差阈值，而非统计成功率。

## R14

常见失效模式：

1. 采样点太少导致峰值定位粗糙，间距误差上升；
2. 峰值检测阈值过低会引入伪峰；
3. 若把包络加入可见度验证，边缘暗化会干扰 `Imax/Imin` 估计；
4. 参数单位混用（nm/mm/m）会导致数量级错误。

## R18

`demo.py` 的源码级算法流程（8 步）：

1. `InterferenceConfig` 定义波长、双缝间距、屏距、两束振幅、相干因子、相位偏置与缝宽。  
2. `_sin_theta` 用精确几何关系 `sin(theta)=y/sqrt(y^2+L^2)` 计算每个屏幕坐标的出射角。  
3. `phase_difference` 计算几何相位差并叠加外加相位 `phi0`。  
4. `envelope_factor` 根据缝宽决定是否引入 `sinc^2` 单缝包络；缝宽<=0时返回全1。  
5. `intensity_profile` 按 `I1+I2+2*gamma*sqrt(I1I2)*cos(delta)` 计算干涉强度，再乘包络形成最终分布。  
6. `estimate_spacing_from_peaks` 调用 `find_peaks` 提取主极大位置，计算相邻峰间距中位数并与 `lambda*L/d` 比较。  
7. `build_visibility_report` 从数值 `Imax/Imin` 得到可见度，并与 `2*gamma*sqrt(I1I2)/(I1+I2)` 做误差比较。  
8. `main` 运行三组配置（间距验证、可见度验证、包络展示），打印表格和阈值检查，最终输出 `PASS/FAIL`。

第三方库职责被显式拆解：

- `numpy` 负责向量化物理计算；
- `scipy.signal.find_peaks` 仅做峰位检测；
- `pandas` 仅做结果结构化展示。

算法本体（相位、强度、可见度、阈值判定）均在源码中直接实现，而非黑盒调用。
