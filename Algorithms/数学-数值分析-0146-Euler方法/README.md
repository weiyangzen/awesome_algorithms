# Euler方法

## R01

本条目实现常微分方程初值问题的最小可运行 Euler 方法（显式一阶方法），目标是把“离散推进公式、误差行为、源码执行流程”完整打通。

MVP 选用标准教学例子：
- 方程：`y' = y - t^2 + 1`
- 初值：`y(0) = 0.5`
- 区间：`t in [0, 2]`

并与解析解对比，验证方法阶数与数值误差趋势。

## R02

问题定义（固定步长版）：
- 输入：微分方程右端函数 `f(t, y)`、初值 `(t0, y0)`、步长 `h`、步数 `N`
- 输出：离散网格点 `t_n = t0 + n*h` 及近似值 `y_n`
- 目标：近似满足 `y'(t)=f(t,y)`，并评估与解析解的误差

本 MVP 不处理自适应步长，只做固定 `h` 的基础版，以保持算法透明。

## R03

Euler 方法推导：
- 对真解在 `t_n` 处做一阶展开：
  `y(t_{n+1}) = y(t_n) + h*y'(t_n) + O(h^2)`
- 用方程 `y'(t_n)=f(t_n, y(t_n))` 替换导数，忽略高阶项：
  `y_{n+1} = y_n + h*f(t_n, y_n)`

这就是显式 Euler 更新公式。它是局部截断误差 `O(h^2)`、全局误差 `O(h)` 的一阶方法。

## R04

算法流程：
1. 校验输入（`h>0`、`steps>0`、数值有限）。
2. 初始化 `t=t0`、`y=y0`，并将初值写入轨迹数组。
3. 循环 `steps` 次：
   - 计算斜率 `s=f(t,y)`；
   - 执行更新 `y <- y + h*s`；
   - 执行更新 `t <- t + h`；
   - 保存新状态。
4. 返回全部网格点与数值解轨迹。

## R05

核心数据结构：
- `t_values: list[float]`：时间网格
- `y_values: list[float]`：Euler 近似解
- `results: list[tuple[h, steps, final_error, max_error]]`：多组步长对比结果

结构保持轻量，便于直接阅读与打印，不依赖复杂框架。

## R06

正确性要点：
- 若迭代使用 `y_{n+1}=y_n+h*f(t_n,y_n)`，则每一步都满足显式 Euler 定义。
- `demo.py` 选取存在解析解的例子，逐点计算 `|y_n - y(t_n)|`，可直接验证数值结果。
- 当 `h` 逐步减半时，最大误差近似按线性比例下降，对应一阶方法行为。

## R07

复杂度分析：
- 单次求解（`N` 步）：
  - 时间复杂度：`O(N)`
  - 空间复杂度：
    - 保留完整轨迹时 `O(N)`；
    - 若只要终值可降为 `O(1)`
- 多组步长实验：复杂度为各组 `N` 之和。

## R08

边界与异常处理：
- `h <= 0`：抛出 `ValueError`
- `steps <= 0`：抛出 `ValueError`
- `t0/y0/h` 非有限（`nan`/`inf`）：抛出 `ValueError`
- 迭代中斜率非有限：抛出 `RuntimeError`
- `t_end / h` 不是近整数（本 demo 固定步数网格要求）：抛出 `ValueError`

## R09

MVP 取舍：
- 采用纯 Python 标准库，不引入数值库黑盒求解器；
- 用一个有解析解的非自治方程，兼顾“算法通用性”和“可验证性”；
- 不加入自适应步长、刚性系统、高阶方法，先保证基础实现清晰可信。

## R10

`demo.py` 函数职责：
- `validate_inputs`：输入合法性校验
- `euler_solve`：执行通用 Euler 固定步长积分
- `ode_rhs`：示例方程右端函数
- `ode_exact`：示例解析解
- `max_abs_error`：网格最大绝对误差
- `run_single_resolution`：单个步长实验并返回误差统计
- `estimate_orders`：由相邻步长误差估计经验阶数
- `print_trajectory_sample`：打印前若干轨迹点
- `main`：组织批量实验并输出结论

## R12

输出解读：
- `h`：步长
- `steps`：对应网格步数
- `final_abs_error`：终点 `t=2` 的绝对误差
- `max_abs_error`：整段网格上的最大绝对误差
- `estimated order p`：由相邻 `h` 的误差比计算的经验收敛阶

对 Euler 方法，`p` 应接近 `1`。

## R13

建议最小测试集：
- 正常收敛：`h in {0.2, 0.1, 0.05, 0.025}`
- 参数异常：`h=0`、`h<0`、`steps=0`
- 数值异常：`t0=nan`、`y0=inf`
- 网格异常：`t_end=2.0, h=0.3`（不能整齐分割区间）

## R14

关键可调参数：
- `h_values`：步长列表，控制精度与开销
- `t_end`：积分终点
- `rows`：轨迹输出行数

工程上可先用较大步长快速检查，再用更小步长确认误差趋势。

## R15

方法对比（同一问题）：
- Euler：一阶、实现最简单、误差较大
- 改进 Euler / Heun：二阶，额外一次斜率评估，精度显著提升
- RK4：四阶，单步成本更高，但常用区间上精度更优

Euler 常用于教学、原型验证、或作为更高阶方法的对照基线。

## R18

源码级算法流（对应 `demo.py`，8 步）：
1. `main` 定义步长集合 `h_values`，逐个调用 `run_single_resolution`。  
2. `run_single_resolution` 将 `t_end/h` 转成整数步数，并检查网格一致性。  
3. 调用 `euler_solve(f, t0, y0, h, steps)` 进入核心积分器。  
4. `euler_solve` 先执行 `validate_inputs`，拒绝非法参数。  
5. 每一轮迭代计算当前斜率 `s=f(t,y)`，按 `y <- y + h*s`、`t <- t + h` 前推一步。  
6. 每步将新 `(t, y)` 追加到轨迹数组，形成完整离散解。  
7. 回到 `run_single_resolution` 后，利用 `ode_exact` 计算终点误差和全局最大误差。  
8. `main` 汇总所有步长误差，用 `estimate_orders` 计算经验阶，并打印轨迹样例。
