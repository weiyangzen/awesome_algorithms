# 树链剖分

- UID: `MATH-0507`
- 学科: `数学`
- 分类: `图论`
- 源序号: `507`
- 目标目录: `Algorithms/数学-图论-0507-树链剖分`

## R01

树链剖分（Heavy-Light Decomposition, HLD）用于把树上的路径问题转化为若干段连续区间问题。它特别适合处理如下高频操作：

- 路径查询：`u -> v` 路径上的和、最大值、最小值等。
- 路径更新：给路径上的点或边做加法、赋值等。
- 子树查询：查询某节点整棵子树的聚合值。

核心价值是把“树结构难以直接做区间操作”的问题，降维成“数组区间操作 + 线段树/树状数组”。

## R02

本任务采用“点权版本”的树链剖分，支持三类操作：

- 点更新：`update_node(u, new_value)`
- 路径查询：`query_path_sum(u, v)`、`query_path_max(u, v)`
- 子树查询：`query_subtree_sum(u)`、`query_subtree_max(u)`

约束设定：

- 输入是一棵 `n` 个节点的无向连通无环图。
- 以 `0` 为根（可改）。
- 节点编号默认 `0..n-1`。

## R03

重链思想：对每个节点选出一个“重儿子”（子树规模最大的儿子），该边记为重边；其余儿子连接的边记为轻边。

性质：任意节点到根路径上，轻边数量是 `O(log n)`。因此任意两点路径可被拆成 `O(log n)` 段重链连续片段。

只要给每个节点分配一个 DFS 序位置 `pos[u]`，并保证同一条重链在 `pos` 上连续，就能把树路径拆成若干数组区间。

## R04

实现中维护以下关键数组：

- `parent[u]`：父节点
- `depth[u]`：深度
- `size[u]`：子树大小
- `heavy[u]`：重儿子（无则 `-1`）
- `head[u]`：`u` 所在重链链头
- `pos[u]`：`u` 在线性化数组中的位置
- `node_at_pos[i]`：位置 `i` 对应的节点

并配套一个线段树，在线性化数组上支持：

- 区间和查询
- 区间最大值查询
- 单点更新

## R05

预处理分两次 DFS：

1. `dfs_sizes(u, p)`
- 计算 `size[u]`
- 找出重儿子 `heavy[u]`

2. `dfs_decompose(u, h)`
- 给当前节点分配 `pos[u]`
- 记录链头 `head[u]=h`
- 先递归重儿子（保证重链连续）
- 再递归轻儿子（每个轻儿子开启新链）

完成后，把节点权值按 `pos` 重排，建立线段树。

## R06

路径查询（以路径和为例）流程：

- 当 `head[u] != head[v]` 时：
- 始终让更深链头的一侧上跳。
- 查询该侧链头到当前节点的连续区间。
- 把该侧节点移动到链头父亲。
- 当二者进入同一条链后，只需查询 `[pos[min_depth], pos[max_depth]]`。

路径最大值同理，只是把“加法聚合”换成“max 聚合”。

## R07

点更新很直接：

- 节点 `u` 在数组中的位置是 `pos[u]`
- 在线段树执行单点更新 `point_update(pos[u], new_value)`

如果扩展到“路径区间加/赋值”，可在线段树加入懒标记；HLD 拆链逻辑无需变化。

## R08

时间复杂度：

- 建树链剖分预处理：`O(n)`
- 建线段树：`O(n)`
- 单次路径查询：`O(log^2 n)`
- 单次点更新：`O(log n)`
- 单次子树查询：`O(log n)`（子树对应一个连续区间）

空间复杂度：`O(n)`。

## R09

正确性要点：

- 重链连续性：`dfs_decompose` 先走重儿子，保证同一重链 `pos` 连续。
- 路径完整覆盖：`while head[u] != head[v]` 每次处理一整段链尾区间，最终落到同链补齐剩余区间。
- 无重叠无遗漏：链头深度严格上跳，路径被不相交区间分段，最终覆盖恰为 `u-v` 全路径。
- 子树连续性：DFS 序下，节点 `u` 子树恰对应 `[pos[u], pos[u] + size[u] - 1]`。

## R10

边界与实现细节：

- 单节点树：所有查询都退化为单点。
- 链式树：几乎只有重边，路径查询接近一次区间查询。
- 星型树：轻边多，但每条根路径轻边数仍满足理论上界。
- 负权值：路径最大值初始值需设为负无穷。
- 递归深度：大规模树需 `sys.setrecursionlimit`。

## R11

`demo.py` 结构：

- `SegmentTree`：提供 `range_sum/range_max/point_update`
- `HeavyLightDecomposition`：
- 预处理：`_dfs_sizes`、`_dfs_decompose`
- 查询：`query_path_sum`、`query_path_max`
- 子树：`query_subtree_sum`、`query_subtree_max`

并提供朴素路径/子树校验函数，运行时自动 `assert` 对比，保证 MVP 结果可验证。

## R12

- `naive_sum/naive_max`：朴素路径枚举结果

## R15

可扩展方向：

- 路径加法、路径赋值（线段树懒标记）
- 边权版本 HLD（将边权映射到子节点位置）
- 支持更多幺半群聚合（gcd、xor、自定义结构）
- 与 LCA 预处理结合，做更多路径统计问题

## R16

常见错误：

- 把轻儿子也延续到同一个 `head`，导致拆链错误。
- 区间边界写错（闭区间与半开区间混淆）。
- `head` 深度比较方向写反，上跳失效。
- 忘记同步更新外部权值数组，导致校验函数不一致。
- 边权题直接套点权模板，出现偏移一位错误。

## R17

和其他方案对比：

- 朴素 DFS/BFS 每次路径查询 `O(n)`，大量查询时不可用。
- 倍增 LCA 只擅长最近公共祖先，不直接解决路径聚合。
- 欧拉序 + RMQ 常用于 LCA，不足以直接支持动态路径更新。
- Link-Cut Tree 更通用但实现复杂；HLD 在静态树场景通常更实用。

结论：在“静态树结构 + 动态路径/子树统计”中，HLD 是工程与性能平衡点。

## R18

`demo.py` 的源码级算法流程可拆为 8 步：

1. 读取内置树结构与初始点权，构造邻接表。
2. 执行第一次 DFS，写入 `parent/depth/size/heavy`。
3. 执行第二次 DFS，写入 `head/pos/node_at_pos`，完成线性化。
4. 按 `pos` 重排点权，建立线段树的和与最大值数组。
5. 处理路径查询时，循环比较 `head[u]` 与 `head[v]`：每次把更深链头一侧映射为一个连续区间并聚合。
6. 当 `u,v` 进入同一条链后，聚合最后一段区间，得到整条路径结果。
7. 处理点更新时，把节点映射到 `pos[u]`，在线段树做单点更新，后续查询自动生效。
8. 每个查询都用朴素枚举路径/子树的结果做断言对比，最终输出 `All checks passed`。
